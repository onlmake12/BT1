### Title
IPv4-Compatible IPv6 Addresses Conflated with IPv4 in Peer Store Group Classification — (`network/src/network_group.rs`)

### Summary

Rust's `Ipv6Addr::to_ipv4()` returns `Some` for both IPv4-mapped (`::ffff:x.x.x.x`) and the deprecated IPv4-compatible (`::x.x.x.x`) address family. The `From<&Multiaddr> for Group` implementation unconditionally calls `to_ipv4()` on any IPv6 address, causing `::1.2.3.4` to be classified as `Group::IP4([1, 2])` — identical to the real IPv4 address `1.2.3.4`. An attacker who connects to a victim node can advertise IPv4-compatible listen addresses via the identify protocol, inflating a target `/16` group in the peer store until `check_purge` randomly evicts legitimate IPv4 peers from that group.

---

### Finding Description

**Root cause — `network/src/network_group.rs` lines 31–35:**

```rust

    if let Some(ipv4) = ipv6.to_ipv4() {   // returns Some for ::x.x.x.x too
        let bits = ipv4.octets();
        return Group::IP4([bits[0], bits[1]]);
    }
``` [1](#0-0) 

`Ipv6Addr::to_ipv4()` in Rust's standard library returns `Some` for the entire `::0.0.0.0/96` IPv4-compatible range (RFC 4291 §2.5.5.1, deprecated), not only for the `::ffff:0.0.0.0/96` IPv4-mapped range. There is no guard distinguishing the two families before the `Group::IP4` return.

**Injection path — identify protocol, `network/src/protocols/identify/mod.rs` lines 488–494:**

```rust

    for addr in addrs {
        if let Err(err) = peer_store.add_addr(addr.clone(), flags) { … }
    }

``` [2](#0-1) 

`add_remote_listen_addrs` adds every address a peer claims to listen on directly to the peer store **without** the `is_reachable` guard that the discovery protocol applies. An attacker who establishes a single inbound or outbound connection can advertise up to `MAX_ADDRS = 10` IPv4-compatible IPv6 addresses per identify exchange.

**Eviction trigger — `check_purge` in `network/src/peer_store/peer_store_impl.rs` lines 358–393:** [3](#0-2) 

When the peer store reaches `ADDR_COUNT_LIMIT = 16384`, `check_purge` groups all stored addresses by `Group` (calling the same buggy `From<&Multiaddr> for Group`), sorts groups by size, and for every group with `> 4` peers randomly evicts 2. Because `::1.2.x.x` and `1.2.x.x` share `Group::IP4([1, 2])`, attacker-injected addresses directly inflate the group that contains legitimate `1.2.x.x` peers. [4](#0-3) 

---

### Impact Explanation

Targeted, repeatable eviction of legitimate IPv4 peers from specific `/16` groups in the peer store. By choosing a `/16` that contains known good peers (e.g., seed nodes or well-connected peers), an attacker can systematically remove those peers from the victim's address book, degrading peer diversity and enabling a partial eclipse of the peer store over time.

---

### Likelihood Explanation

- **Entry point is standard P2P**: any node that accepts inbound connections (the default) is reachable.
- **No PoW, no key, no privilege required**: a single TCP connection suffices to send an identify message.
- **Filling the peer store to 16384** requires advertising many addresses across multiple connections or sessions, but is achievable by a determined attacker operating a small number of nodes.
- **Eviction is probabilistic** but repeatable; the attacker can re-inject after each eviction cycle.

---

### Recommendation

Replace `ipv6.to_ipv4()` with `ipv6.to_ipv4_mapped()` (stabilized in Rust 1.63), which returns `Some` **only** for the `::ffff:0.0.0.0/96` IPv4-mapped range and `None` for the deprecated `::0.0.0.0/96` IPv4-compatible range:

```rust
if let Some(ipv4) = ipv6.to_ipv4_mapped() {
    // only genuine IPv4-mapped addresses reach here
```

Alternatively, explicitly reject IPv4-compatible addresses before calling `to_ipv4()` by checking that the first 96 bits match `::ffff:0:0` rather than `::0:0`.

---

### Proof of Concept

```rust
use std::net::Ipv6Addr;
// IPv4-compatible ::1.2.3.4
let compat: Ipv6Addr = Ipv6Addr::new(0, 0, 0, 0, 0, 0, 0x0102, 0x0304);
// IPv4-mapped ::ffff:1.2.3.4
let mapped: Ipv6Addr = Ipv6Addr::new(0, 0, 0, 0, 0, 0xffff, 0x0102, 0x0304);

assert_eq!(compat.to_ipv4(), Some("1.2.3.4".parse().unwrap())); // true today
assert_eq!(mapped.to_ipv4(), Some("1.2.3.4".parse().unwrap())); // also true

// Both produce Group::IP4([1, 2]) — same bucket as real 1.2.x.x peers.
// With to_ipv4_mapped():
assert_eq!(compat.to_ipv4_mapped(), None);   // correctly rejected
assert_eq!(mapped.to_ipv4_mapped(), Some("1.2.3.4".parse().unwrap()));
```

A differential unit test asserting `Group::from(::1.2.3.4 multiaddr) != Group::from(1.2.3.4 multiaddr)` fails against the current code, confirming the invariant is broken.
