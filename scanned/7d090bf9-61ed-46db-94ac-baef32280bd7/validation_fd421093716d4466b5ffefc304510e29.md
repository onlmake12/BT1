### Title
Out-of-Bounds Index in `remove_unconfirmed_tx` Causes Node Panic — (`util/fee-estimator/src/estimator/confirmation_fraction.rs`)

---

### Summary

`TxConfirmStat::remove_unconfirmed_tx` correctly branches on `tx_age >= MAX_CONFIRM_BLOCKS` to handle the `block_unconfirmed_txs` array, but the subsequent `count_failure` write at line 215 is placed **outside** that branch and uses `tx_age - 1` as an index into `confirm_blocks_to_failed_txs` without a bounds check. When a tracked transaction has been in the pool for more than 1000 blocks and is then rejected, `tx_age - 1 >= 1000` exceeds the array's valid range (0..999), causing a Rust index-out-of-bounds **panic** that crashes the node.

---

### Finding Description

`confirm_blocks_to_failed_txs` is allocated with `MAX_CONFIRM_BLOCKS = 1000` rows (indices 0–999):

```rust
let confirm_blocks_to_failed_txs = vec![vec![0f64; buckets.len()]; max_confirm_blocks];
``` [1](#0-0) 

`remove_unconfirmed_tx` guards the `block_unconfirmed_txs` decrement with a bounds check, but the `count_failure` write is unconditionally placed after the if/else:

```rust
if tx_age >= self.block_unconfirmed_txs.len() {   // tx_age >= 1000
    self.bucket_stats[bucket_index].old_unconfirmed_txs -= 1;
} else {
    ...
}
if count_failure {
    // tx_age - 1 can be >= 1000 here — OUT OF BOUNDS
    self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64;
}
``` [2](#0-1) 

When `tx_age = 1001`, `tx_age - 1 = 1000` is past the last valid index (999). Rust panics with an index-out-of-bounds, crashing the node process.

The call chain that reaches this with `count_failure = true` is:

- `reject_tx` → `drop_tx` → `drop_tx_inner(tx_hash, true)` → `remove_unconfirmed_tx(..., count_failure=true)` [3](#0-2) 

Transactions are tracked at the block height they enter the pool (`track_tx` records `height = current_tip`). The `best_height` used in `drop_tx_inner` is the current chain tip at rejection time. If the gap exceeds 1000 blocks, the panic fires. [4](#0-3) 

The fee estimator callback is wired into the tx-pool rejection path in `register_tx_pool_callback`:

```rust
fee_estimator.reject_tx(&tx_hash);
``` [5](#0-4) 

---

### Impact Explanation

A Rust index-out-of-bounds panic is an unrecoverable abort of the thread (and in practice the process). Any CKB node running the `ConfirmationFraction` fee estimator that evicts or rejects a transaction that has been tracked for more than 1000 blocks will crash. This is a **remote node crash** reachable by an unprivileged transaction sender.

---

### Likelihood Explanation

At CKB's ~8-second block time, 1000 blocks ≈ 2.2 hours. The default `expiry_hours` for pool transactions is 72 hours, meaning transactions routinely remain in the pool far longer than 1000 blocks. An attacker submits a low-fee transaction (below the mining threshold) that will never be confirmed. After 1000+ blocks the node's own eviction logic (or an RBF replacement) calls `reject_tx`, triggering the panic. No special privilege is required — only the ability to submit a transaction to the mempool via P2P relay or the `send_transaction` RPC.

---

### Recommendation

Move the `count_failure` write inside the bounds-safe branch, or add an explicit cap:

```rust
if count_failure {
    let index = tx_age.saturating_sub(1).min(self.confirm_blocks_to_failed_txs.len() - 1);
    self.confirm_blocks_to_failed_txs[index][bucket_index] += 1f64;
}
```

Alternatively, mirror the same guard used for `block_unconfirmed_txs`:

```rust
if count_failure && tx_age <= self.confirm_blocks_to_failed_txs.len() {
    self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64;
}
``` [6](#0-5) 

---

### Proof of Concept

1. Node is configured with the `ConfirmationFraction` fee estimator.
2. Attacker submits a transaction with a fee rate below the mining threshold at block height `H`.
3. The estimator records it: `track_tx(tx_hash, fee_rate, H)`.
4. The chain advances past block `H + 1000`.
5. The node evicts the transaction (expiry, RBF, or pool-size eviction).
6. `reject_tx` → `drop_tx_inner(tx_hash, true)` → `remove_unconfirmed_tx(H, H+1001, bucket_index, true)`.
7. `tx_age = 1001`, `confirm_blocks_to_failed_txs[1000]` → **index out of bounds → node panic**. [7](#0-6)

    fn remove_unconfirmed_tx(
        &mut self,
        entry_height: u64,
        tip_height: u64,
        bucket_index: usize,
        count_failure: bool,
    ) {
        let tx_age = tip_height.saturating_sub(entry_height) as usize;
        if tx_age < 1 {
            return;
        }
        if tx_age >= self.block_unconfirmed_txs.len() {
            self.bucket_stats[bucket_index].old_unconfirmed_txs -= 1;
        } else {
            let block_index = (entry_height % self.block_unconfirmed_txs.len() as u64) as usize;
            self.block_unconfirmed_txs[block_index][bucket_index] -= 1;
        }
        if count_failure {
            self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64;
        }
    }
```
