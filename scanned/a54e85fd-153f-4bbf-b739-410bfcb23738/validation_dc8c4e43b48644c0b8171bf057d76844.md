### Title
Unbounded `tx_hashes` Array in `get_transaction_proof` / `get_transaction_and_witness_proof` RPC Methods Enables DoS - (`rpc/src/module/chain.rs`)

### Summary

The `get_transaction_proof` and `get_transaction_and_witness_proof` JSON-RPC methods accept an unbounded `tx_hashes: Vec<H256>` parameter with no upper-bound validation. Any RPC caller can submit a request containing tens of thousands of transaction hashes, forcing the node to perform a database lookup for each one. The equivalent light-client P2P handler enforces a hard limit of 1,000 hashes, but the RPC path has no such guard, making it a direct DoS vector.

### Finding Description

Both `get_transaction_proof` and `get_transaction_and_witness_proof` delegate to the private helper `get_tx_indices`, which iterates over every supplied hash and calls `snapshot.get_transaction_info` (a RocksDB read) for each one, with no length check:

```rust
// rpc/src/module/chain.rs  get_tx_indices()
if tx_hashes.is_empty() {
    return Err(RPCError::invalid_params("Empty transaction hashes"));
}
// ❌ No upper-bound check here
for tx_hash in tx_hashes {
    match snapshot.get_transaction_info(&(&tx_hash).into()) { ... }
}
``` [1](#0-0) 

The two public RPC entry points that call this helper are:

```rust
fn get_transaction_proof(
    &self,
    tx_hashes: Vec<H256>,   // ❌ unbounded
    block_hash: Option<H256>,
) -> Result<TransactionProof>
``` [2](#0-1) 

```rust
fn get_transaction_and_witness_proof(
    &self,
    tx_hashes: Vec<H256>,   // ❌ unbounded
    block_hash: Option<H256>,
) -> Result<TransactionAndWitnessProof>
``` [3](#0-2) 

By contrast, the light-client P2P server handler for the equivalent operation enforces a hard cap:

```rust
if self.message.tx_hashes().len() > constant::GET_TRANSACTIONS_PROOF_LIMIT {
    return StatusCode::MalformedProtocolMessage.with_context("too many transactions");
}
``` [4](#0-3) 

where `GET_TRANSACTIONS_PROOF_LIMIT = 1000`. [5](#0-4) 

The only existing protection is the default 10 MiB request body limit:

```toml
max_request_body_size = 10485760
``` [6](#0-5) 

### Impact Explanation

Each H256 in JSON encoding occupies ~69 bytes (`"0x<64 hex chars>"` + comma). Within the 10 MiB limit an attacker can pack approximately:

```
10,485,760 / 69 ≈ 151,967 hashes
```

For each hash the node performs a synchronous RocksDB read (`get_transaction_info`). Even if the first hash that does not belong to the same block causes an early return, an attacker who supplies valid hashes all belonging to the same high-transaction block forces the full iteration. After the loop, `get_transaction_proof` additionally calls `CBMT::build_merkle_proof` over all block transactions. [7](#0-6) 

Repeated rapid requests can saturate the RPC thread pool and the RocksDB read path, degrading or halting service for all other RPC callers.

### Likelihood Explanation

The RPC server is reachable by any process that can connect to the configured `listen_address`. While the default is `127.0.0.1:8114`, many operators expose the RPC to internal networks or behind a reverse proxy. No authentication is required. The attack requires only knowledge of a few valid transaction hashes (publicly available from any block explorer or by querying `get_block`), and can be automated trivially. The cost per request is zero beyond network bandwidth.

### Recommendation

Add an upper-bound check at the start of `get_tx_indices` (or at each call site), consistent with the limit already enforced in the light-client protocol handler:

```rust
const MAX_TX_HASHES_PER_PROOF_REQUEST: usize = 1000;

fn get_tx_indices(&self, tx_hashes: Vec<H256>, ...) -> Result<...> {
    if tx_hashes.is_empty() {
        return Err(RPCError::invalid_params("Empty transaction hashes"));
    }
    if tx_hashes.len() > MAX_TX_HASHES_PER_PROOF_REQUEST {
        return Err(RPCError::invalid_params(format!(
            "Too many transaction hashes, limit is {MAX_TX_HASHES_PER_PROOF_REQUEST}"
        )));
    }
    ...
}
```

The same limit should be applied to the `proof.indices` and `proof.lemmas` arrays accepted by `verify_transaction_proof` and `verify_transaction_and_witness_proof`, which are also unbounded and fed directly into `CBMT::retrieve_leaves`. [8](#0-7) 

### Proof of Concept

1. Obtain a valid block hash and collect all transaction hashes from a block with many transactions (e.g., via `get_block`).
2. Send the following request, repeating it in a tight loop:

```json
{
  "id": 1,
  "jsonrpc": "2.0",
  "method": "get_transaction_proof",
  "params": [
    [
      "0x<tx_hash_1>",
      "0x<tx_hash_2>",
      ...
      "0x<tx_hash_N>"   // up to ~150,000 entries within 10 MiB
    ],
    null
  ]
}
```

Each request forces `N` synchronous RocksDB reads in `get_tx_indices` followed by a Merkle proof construction over all block transactions. Flooding the endpoint with such requests will exhaust the RPC server's capacity to serve legitimate callers.

        snapshot
            .get_block(&(&tx_proof.block_hash).into())
            .ok_or_else(|| {
                RPCError::invalid_params(format!("Cannot find block {:#x}", tx_proof.block_hash))
            })
            .and_then(|block| {

                let merkle_proof = MerkleProof::new(
                    tx_proof
                        .proof
                        .indices
                        .into_iter()
                        .map(|index| index.value())
                        .collect(),
                    tx_proof
                        .proof
                        .lemmas
                        .into_iter()
                        .map(|lemma| lemma.into())
                        .collect(),

                                    None

    fn get_tx_indices(
        &self,
        tx_hashes: Vec<H256>,
        block_hash: Option<H256>,
    ) -> Result<(core::BlockView, Vec<u32>)> {
        if tx_hashes.is_empty() {
            return Err(RPCError::invalid_params("Empty transaction hashes"));
        }
        let snapshot = self.shared.snapshot();

        let mut retrieved_block_hash = None;
        let mut tx_indices = HashSet::new();
        for tx_hash in tx_hashes {
            match snapshot.get_transaction_info(&(&tx_hash).into()) {
                Some(tx_info) => {
                    if retrieved_block_hash.is_none() {
                        retrieved_block_hash = Some(tx_info.block_hash);
                    } else if Some(tx_info.block_hash) != retrieved_block_hash {
                        return Err(RPCError::invalid_params(
                            "Not all transactions found in retrieved block",
                        ));
                    }

                    if !tx_indices.insert(tx_info.index as u32) {
                        return Err(RPCError::invalid_params(format!(
                            "Duplicated tx_hash {tx_hash:#x}"
                        )));
                    }
                }
                None => {
                    return Err(RPCError::invalid_params(format!(
                        "Transaction {tx_hash:#x} not yet in block"
                    )));
                }
            }
        }
