Audit Report

## Title
Attacker-Controlled `observed_addr` with Mismatched PeerId Poisons Victim's Self-Advertised Addresses — (`network/src/protocols/identify/mod.rs`)

## Summary

`IdentifyCallback::add_observed_addr` only appends the local node's PeerId when the incoming `observed_addr` carries **no** `/p2p/` component. An attacker who pre-embeds `/p2p/<attacker_peer_id>` bypasses this guard entirely, causing the poisoned address to be stored verbatim and later broadcast to every subsequent peer as one of the victim's own listen addresses. Peers that cache or dial the poisoned address will fail the Noise/TLS handshake because the victim's actual keypair produces a different PeerId, degrading the victim's reachability.

## Finding Description

The guard at `network/src/protocols/identify/mod.rs` L497–502 is:

```rust
fn add_observed_addr(&mut self, mut addr: Multiaddr, session_id: SessionId) -> MisbehaveResult {
    if extract_peer_id(&addr).is_none() {   // only fires when NO /p2p/ present
        addr.push(Protocol::P2P(Cow::Borrowed(
            self.network_state.local_peer_id().as_bytes(),
        )))
    }
    self.network_state.add_observed_addr(session_id, addr);  // stored unconditionally
``` [1](#0-0) 

When the attacker sends `observed_addr = /ip4/<victim_ip>/tcp/<port>/p2p/<attacker_peer_id>`, `extract_peer_id` returns `Some(attacker_peer_id)`, the `if` branch is skipped, and the address is forwarded to `NetworkState::add_observed_addr` with no further validation.

`NetworkState::add_observed_addr` performs no PeerId consistency check — it simply inserts the address into the `observed_addrs` map keyed by `session_id`:

```rust
pub(crate) fn add_observed_addr(&self, session_id: SessionId, addr: Multiaddr) {
    let mut pending_observed_addrs = self.observed_addrs.write();
    pending_observed_addrs.insert(session_id, addr);
}
``` [2](#0-1) 

The poisoned entry is returned verbatim by `observed_addrs()`: [3](#0-2) 

It is then mixed into the victim's self-reported listen addresses in `local_listen_addrs()`: [4](#0-3) 

This list is sent verbatim to every new peer in the `connected` handshake: [5](#0-4) 

The same `observed_addrs` pool is also consumed by the hole-punching protocol's `listen_addrs` field, enabling further propagation: [6](#0-5) 

## Impact Explanation

Any peer that receives the poisoned entry (`/ip4/<victim_ip>/tcp/<port>/p2p/<attacker_peer_id>`) will attempt to dial the victim but authenticate against `<attacker_peer_id>`. The Noise/TLS handshake fails because the victim's actual keypair produces a different PeerId, causing the connection to be rejected. For nodes behind NAT that have no `public_addrs` and rely entirely on `observed_addrs`, an attacker who fills the entire `observed_addrs` map (one entry per concurrent inbound session) can make the victim effectively unreachable to all peers that rely on those addresses. This maps to **High — Vulnerabilities or bad designs which could cause CKB network congestion with few costs** (10001–15000 points), since a small number of attacker sessions can degrade a victim node's reachability across the network.

## Likelihood Explanation

The attack requires only a standard inbound P2P connection — no special privileges, no PoW, no key material. The crafted `IdentifyMessage` is a valid protobuf/molecule message with a well-formed Multiaddr. One attacker session poisons one `observed_addrs` map entry; multiple concurrent attacker sessions can saturate the entire map. The poisoned entries persist after disconnection because no cleanup of `observed_addrs` is performed on session teardown.

## Recommendation

In `add_observed_addr`, after the existing `extract_peer_id` check, validate that the extracted PeerId matches the local node's PeerId. If it does not match, strip the wrong `/p2p/` component and re-append the correct one (or treat it as misbehavior and return `MisbehaveResult::Disconnect`):

```rust
fn add_observed_addr(&mut self, mut addr: Multiaddr, session_id: SessionId) -> MisbehaveResult {
    match extract_peer_id(&addr) {
        Some(peer_id) if peer_id == *self.network_state.local_peer_id() => { /* ok */ }
        Some(_) => {
            addr = addr.into_iter()
                .filter(|p| !matches!(p, Protocol::P2P(_)))
                .collect();
            addr.push(Protocol::P2P(Cow::Borrowed(
                self.network_state.local_peer_id().as_bytes(),
            )));
        }
        None => {
            addr.push(Protocol::P2P(Cow::Borrowed(
                self.network_state.local_peer_id().as_bytes(),
            )));
        }
    }
    self.network_state.add_observed_addr(session_id, addr);
    MisbehaveResult::Continue
}
```

Additionally, consider evicting `observed_addrs` entries on session disconnect to prevent stale poisoned entries from persisting.

## Proof of Concept

1. Attacker node establishes a standard inbound P2P connection to the victim.
2. Attacker sends an `IdentifyMessage` where `observed_addr = /ip4/<victim_ip>/tcp/<victim_port>/p2p/<attacker_peer_id>`.
3. Victim's `received` → `process_observed` → `add_observed_addr`: `extract_peer_id` returns `Some(attacker_peer_id)`, the guard is skipped, the poisoned address is stored in `observed_addrs[attacker_session_id]`.
4. A third peer connects to the victim. `connected` calls `local_listen_addrs()`, which calls `observed_addrs(...)` and appends the poisoned entry to the list.
5. The victim sends `IdentifyMessage { listen_addrs: [..., /ip4/<victim_ip>/tcp/<port>/p2p/<attacker_peer_id>], ... }` to the third peer.
6. The third peer dials `/ip4/<victim_ip>/tcp/<port>/p2p/<attacker_peer_id>`; the Noise/TLS handshake fails because the victim's actual PeerId does not match `<attacker_peer_id>`.
7. Repeat with multiple concurrent attacker sessions to saturate the entire `observed_addrs` map, making the victim unreachable to all peers that rely on observed addresses.
