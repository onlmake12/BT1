### Title
RBF Timestamp Reset Allows Indefinite tx-pool Expiry Bypass and UTXO Griefing - (`tx-pool/src/pool.rs`, `tx-pool/src/component/entry.rs`)

---

### Summary

CKB's tx-pool has an `expiry` mechanism that evicts stale transactions after `expiry_hours` (default 12 hours). When Replace-By-Fee (RBF) is enabled, a transaction's pool entry always receives a **fresh wall-clock timestamp** upon replacement. An attacker can exploit this by repeatedly replacing their own transaction just before it expires, resetting the expiry timer indefinitely at near-zero cost. This defeats the eviction mechanism and allows the attacker to keep target UTXOs locked in the pool forever, preventing other parties from spending those UTXOs without outbidding the attacker.

---

### Finding Description

`TxEntry::new` unconditionally stamps the current wall-clock time as the entry's `timestamp`: [1](#0-0) 

The expiry check in `remove_expired` compares this timestamp against the configured `expiry` window: [2](#0-1) 

The `expiry` value is derived directly from `config.expiry_hours`: [3](#0-2) 

When RBF is enabled (`min_rbf_rate > min_fee_rate`), `check_rbf` validates the replacement but imposes no constraint on the original entry's timestamp — it does not carry the old timestamp forward: [4](#0-3) 

After `process_rbf` removes the old entry and `_submit_entry` inserts the new one, the new `TxEntry` is constructed with `TxEntry::new(...)`, which calls `unix_time_as_millis()` fresh: [5](#0-4) 

The minimum additional fee required per replacement is `min_rbf_rate.fee(size)`: [6](#0-5) 

With `min_rbf_rate = 1500 shannons/KB` (default) and a ~200-byte transaction, each replacement costs roughly **300 shannons** (~0.000003 CKB). An attacker can replace their transaction every 11.9 hours indefinitely, resetting the expiry timer each time.

---

### Impact Explanation

- **Expiry bypass**: The `remove_expired` mechanism, intended to clean up stale transactions and free pool resources, is permanently defeated for attacker-controlled entries. The eviction never fires.
- **UTXO griefing / censorship**: Any UTXO the attacker's transaction consumes is kept "locked" in the pool indefinitely. Any third party wishing to spend those same UTXOs must submit an RBF replacement with a higher fee than the attacker's current transaction. The attacker can always respond with another replacement, maintaining the lock at near-zero marginal cost.
- **Pool resource exhaustion**: Combined with the pool size limit, an attacker running many such chains can occupy pool slots indefinitely, crowding out legitimate transactions.

---

### Likelihood Explanation

- **Attacker entry point**: Any unprivileged RPC caller (`send_transaction`) or P2P peer relaying a transaction can trigger this.
- **Cost**: ~300 shannons per 12-hour cycle — effectively free on mainnet.
- **No special knowledge required**: The attacker only needs to know the `expiry_hours` configuration (publicly documented default: 12 hours) and submit a replacement before that window closes.
- **RBF is opt-in per node config**, but nodes that enable it (i.e., `min_rbf_rate > min_fee_rate`) are fully exposed.

---

### Recommendation

Preserve the **original** entry's timestamp when a transaction is replaced via RBF, rather than assigning a fresh one. Specifically:

1. In `check_rbf` or `process_rbf`, retrieve the oldest `timestamp` among all conflicting entries being replaced.
2. Pass that timestamp to `TxEntry::new_with_timestamp(...)` for the replacement entry, so the expiry clock is not reset.

This mirrors the fix suggested in the external report: the "timer" (here, the pool expiry timestamp) should not be reset by a state transition (RBF replacement) that the attacker fully controls. [7](#0-6) 

---

### Proof of Concept

1. Node is configured with `min_fee_rate = 1000`, `min_rbf_rate = 1500`, `expiry_hours = 12`.
2. Attacker submits **T1** spending UTXO `[A]` with fee 1000 shannons/KB. Pool records `T1.timestamp = now`.
3. After 11 hours 55 minutes, attacker submits **T2** spending the same UTXO `[A]` with fee 1500 shannons/KB (satisfying `check_rbf` Rule #3/#4). Pool records `T2.timestamp = now + 11h55m`.
4. `remove_expired` runs at the 12-hour mark: `expiry + T1.timestamp` would have evicted T1, but T1 is already gone; T2's expiry deadline is now `now + 11h55m + 12h` — safely in the future.
5. Attacker repeats step 3 every ~11.9 hours. UTXO `[A]` remains locked in the pool indefinitely. Any third party attempting to spend `[A]` must pay more than the attacker's current fee, and the attacker can always respond with another replacement. [8](#0-7) [9](#0-8)

    // Expire all transaction (and their dependencies) in the pool.
    pub(crate) fn remove_expired(&mut self, callbacks: &Callbacks) {
        let now_ms = ckb_systemtime::unix_time_as_millis();

        let removed: Vec<_> = self
            .pool_map
            .iter()
            .filter(|&entry| self.expiry + entry.inner.timestamp < now_ms)
            .map(|entry| entry.inner.clone())
            .collect();

        for entry in removed {
            let tx_hash = entry.transaction().hash();
            debug!("remove_expired {} timestamp({})", tx_hash, entry.timestamp);
            self.pool_map.remove_entry(&entry.proposal_short_id());
            let reject = Reject::Expiry(entry.timestamp);
            callbacks.call_reject(self, &entry, reject);
        }
    }

    pub(crate) fn check_rbf(
        &self,
        snapshot: &Snapshot,
        entry: &TxEntry,
    ) -> Result<HashSet<ProposalShortId>, Reject> {
        assert!(self.enable_rbf());
        let tx_inputs: Vec<OutPoint> = entry.transaction().input_pts_iter().collect();
        let conflict_ids = self.pool_map.find_conflict_tx(entry.transaction());

        if conflict_ids.is_empty() {
            return Ok(HashSet::new());
        }

        let short_id = entry.proposal_short_id();

        // Rule #1, the node has enabled RBF, which is checked by caller
        let conflicts = conflict_ids
            .iter()
            .filter_map(|id| self.get_pool_entry(id))
            .collect::<Vec<_>>();
        assert!(conflicts.len() == conflict_ids.len());

        // Rule #2, new tx don't contain any new unconfirmed inputs
        let mut inputs = HashSet::new();
        for c in conflicts.iter() {
            inputs.extend(c.inner.transaction().input_pts_iter());
        }

        if tx_inputs
            .iter()
            .any(|pt| !inputs.contains(pt) && !snapshot.transaction_exists(&pt.tx_hash()))
        {
            return Err(Reject::RBFRejected(
                "new Tx contains unconfirmed inputs".to_string(),
            ));
        }

        // Rule #5, the replaced tx's descendants can not more than 100
        // and the ancestor of the new tx don't have common set with the replaced tx's descendants
        let mut replace_count: usize = 0;
        let mut all_conflicted = conflicts.clone();
        let ancestors = self.pool_map.calc_ancestors(&short_id);
        for conflict in conflicts.iter() {
            let descendants = self.pool_map.calc_descendants(&conflict.id);
            replace_count += descendants.len() + 1;
            if replace_count > MAX_REPLACEMENT_CANDIDATES {
                return Err(Reject::RBFRejected(format!(
                    "Tx conflict with too many txs, conflict txs count: {}, expect <= {}",
                    replace_count, MAX_REPLACEMENT_CANDIDATES,
                )));
            }

            if !descendants.is_disjoint(&ancestors) {
                return Err(Reject::RBFRejected(
                    "Tx ancestors have common with conflict Tx descendants".to_string(),
                ));
            }

            let entries = descendants
                .iter()
                .filter_map(|id| self.get_pool_entry(id))
                .collect::<Vec<_>>();

            for entry in entries.iter() {
                let hash = entry.inner.transaction().hash();
                if tx_inputs.iter().any(|pt| pt.tx_hash() == hash) {
                    return Err(Reject::RBFRejected(
                        "new Tx contains inputs in descendants of to be replaced Tx".to_string(),
                    ));
                }
            }
            all_conflicted.extend(entries);
        }

        let tx_cells_deps: Vec<OutPoint> = entry
            .transaction()
            .cell_deps_iter()
            .map(|c| c.out_point())
            .collect();
        for entry in all_conflicted.iter() {
            let hash = entry.inner.transaction().hash();
            if tx_cells_deps.iter().any(|pt| pt.tx_hash() == hash) {
                return Err(Reject::RBFRejected(
                    "new Tx contains cell deps from conflicts".to_string(),
                ));
            }
        }

        // Rule #4, new tx's fee need to higher than min_rbf_fee computed from the tx_pool configuration
        // Rule #3, new tx's fee need to higher than conflicts, here we only check the all conflicted txs fee
        let fee = entry.fee;
        if let Some(min_replace_fee) = self.calculate_min_replace_fee(&all_conflicted, entry.size) {
            if fee < min_replace_fee {
                return Err(Reject::RBFRejected(format!(
                    "Tx's current fee is {}, expect it to >= {} to replace old txs",
                    fee, min_replace_fee,
                )));
            }
        } else {
            return Err(Reject::RBFRejected(
                "calculate_min_replace_fee failed".to_string(),
            ));
        }

        Ok(conflict_ids)
    }
