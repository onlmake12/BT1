### Title
Cellbase Reward Recipient Receives Only First-Output Lock Check, Not Full Capacity — (`File: verification/contextual/src/contextual_block_verifier.rs`)

---

### Summary

`RewardVerifier::verify` enforces that the cellbase's first output carries the correct target lock and that the *sum* of all outputs equals the block reward. It never enforces that the first output's *capacity* equals the block reward. A malicious miner can therefore split the cellbase into multiple outputs — the first satisfying the lock check with a dust amount, and the rest redirected to the miner's own lock — stealing the bulk of the reward from the intended recipient.

---

### Finding Description

`RewardVerifier::verify` performs exactly two checks when a finalization target exists and the reward is large enough to create a cell:

```rust
if cellbase.transaction.outputs_capacity()? != block_reward.total {
    return Err((CellbaseError::InvalidRewardAmount).into());
}
if cellbase
    .transaction
    .outputs()
    .get(0)
    .expect("cellbase should have output")
    .lock()
    != target_lock
{
    return Err((CellbaseError::InvalidRewardTarget).into());
}
``` [1](#0-0) 

Check 1 verifies the **total** outputs capacity equals `block_reward.total`. Check 2 verifies only the **first** output's lock equals `target_lock`. There is no check that:

- the cellbase has exactly one output, or
- the first output's individual capacity equals `block_reward.total`.

The `CapacityVerifier` explicitly skips the `OutputsSumOverflow` check for cellbase transactions, delegating that responsibility entirely to `RewardVerifier`:

```rust
if !(self.resolved_transaction.is_cellbase() || self.valid_dao_withdraw_transaction()) {
    // capacity sum check
}
``` [2](#0-1) 

The `CapacityVerifier` does still enforce a per-output minimum occupied capacity, so each additional output must be at least large enough to hold its lock script. This is the only lower bound on the dust output sent to the target lock.

---

### Impact Explanation

In CKB's delayed reward model, the miner of block `N + PROPOSAL_WINDOW.farthest + 1` is responsible for constructing the cellbase that pays block `N`'s miner. The target lock is read from block `N`'s cellbase witness:

```rust
let target_lock = CellbaseWitness::from_slice(
    &self.store.get_cellbase(&target.hash())
        .expect("target cellbase exist")
        .witnesses().get(0)
        ...
)
.expect("cellbase loaded from store should has non-empty witness")
.lock();
``` [3](#0-2) 

A malicious miner constructing the current block can craft a cellbase with two outputs:

| Output | Lock | Capacity |
|--------|------|----------|
| 0 | `target_lock` (block N's miner) | minimum occupied capacity (~61 CKB) |
| 1 | attacker's own lock | `block_reward.total − 61 CKB` |

Both checks in `RewardVerifier` pass:
- `outputs_capacity() == block_reward.total` ✓
- `outputs[0].lock() == target_lock` ✓

Block `N`'s miner receives only the dust amount. The attacker retains the remainder. For a mainnet block reward of ~1917 CKB, the attacker can steal ~1856 CKB per block they mine.

---

### Likelihood Explanation

Any miner who successfully mines a single block can execute this attack. No majority hashpower, no privileged keys, and no special access are required. The attack is a deliberate construction of the cellbase transaction before calling `submit_block`. It is silent — the block is valid and accepted by all nodes — and the victim (block `N`'s miner) has no recourse once the block is committed.

---

### Recommendation

Add a third check in `RewardVerifier::verify` that enforces the first (and only) cellbase output carries the full block reward:

```rust
// Enforce exactly one output and that it holds the full reward
if cellbase.transaction.outputs().len() != 1 {
    return Err((CellbaseError::InvalidRewardTarget).into());
}
if cellbase.transaction.outputs().get(0)
    .expect("cellbase should have output")
    .capacity()
    != block_reward.total.pack()
{
    return Err((CellbaseError::InvalidRewardAmount).into());
}
```

Alternatively, enforce that `outputs[0].capacity() == block_reward.total` without restricting output count, which would still prevent capacity theft while allowing future extensibility.

---

### Proof of Concept

1. Attacker mines block `H` (the block that finalizes block `H − PROPOSAL_WINDOW.farthest − 1`'s reward).
2. Attacker reads `target_lock` from block `H − PROPOSAL_WINDOW.farthest − 1`'s cellbase witness.
3. Attacker constructs a cellbase for block `H` with two outputs:
   - `output[0]`: lock = `target_lock`, capacity = 6100000000 shannons (61 CKB, minimum for a secp256k1 lock cell)
   - `output[1]`: lock = attacker's own lock, capacity = `block_reward.total − 6100000000`
4. Attacker submits block `H` via `submit_block` RPC.
5. `RewardVerifier::verify` passes: `outputs_capacity() == block_reward.total` and `outputs[0].lock() == target_lock`.
6. Block `H` is accepted by all nodes. Block `H − PROPOSAL_WINDOW.farthest − 1`'s miner receives 61 CKB instead of the full reward. Attacker receives the remainder. [4](#0-3)

    pub fn verify(&self) -> Result<(), Error> {
        let cellbase = &self.resolved[0];
        let no_finalization_target =
            (self.parent.number() + 1) <= self.context.consensus.finalization_delay_length();

        let (target_lock, block_reward) = self.context.finalize_block_reward(self.parent)?;
        let output = CellOutput::new_builder()
            .capacity(block_reward.total)
            .lock(target_lock.clone())
            .build();
        let insufficient_reward_to_create_cell = output.is_lack_of_capacity(Capacity::zero())?;

        if no_finalization_target || insufficient_reward_to_create_cell {
            let ret = if cellbase.transaction.outputs().is_empty() {
                Ok(())
            } else {
                Err((CellbaseError::InvalidRewardTarget).into())
            };
            return ret;
        }

        if !insufficient_reward_to_create_cell {
            if cellbase.transaction.outputs_capacity()? != block_reward.total {
                return Err((CellbaseError::InvalidRewardAmount).into());
            }
            if cellbase
                .transaction
                .outputs()
                .get(0)
                .expect("cellbase should have output")
                .lock()
                != target_lock
            {
                return Err((CellbaseError::InvalidRewardTarget).into());
            }
        }

        Ok(())
    }
