### Title
Score Reset via Discovery Re-Advertisement Erases Peer Penalties — (`network/src/peer_store/addr_manager.rs`, `network/src/peer_store/peer_store_impl.rs`)

### Summary

`AddrManager::add` unconditionally overwrites an existing `AddrInfo` entry — including its `score` field — whenever the incoming `last_connected_at_ms` is `>=` the stored value. Because `PeerStore::add_addr` always constructs `AddrInfo::new(addr, 0, default_score, flags)`, any peer whose stored `last_connected_at_ms` is also `0` (i.e., never successfully connected) will have its entire record replaced with a fresh one carrying `score = 100`. A remote peer can trigger this path by advertising the target address through the discovery protocol.

### Finding Description

**`PeerStore::add_addr`** always passes `last_connected_at_ms = 0` and `score = default_score (100)` to `AddrManager::add`: [1](#0-0) 

**`AddrManager::add`** then checks whether the incoming timestamp is `>=` the stored one. When both are `0`, the condition is satisfied and the full `AddrInfo` struct — including `score` — is replaced: [2](#0-1) 

**`PeerStore::report`** modifies `score` in-place on the existing `AddrInfo`. It does not touch `last_connected_at_ms`, so after a penalty the entry still has `last_connected_at_ms = 0` for any peer that was only ever discovered (never connected): [3](#0-2) 

**Attack sequence:**
1. Peer P is discovered → `add_addr` → `AddrInfo { score: 100, last_connected_at_ms: 0 }`.
2. P misbehaves → `report(P, bad_behaviour)` → `AddrInfo { score: 60, last_connected_at_ms: 0 }`.
3. Any remote peer (including P itself via the discovery relay) sends an `AddAddrs` message containing P's `Multiaddr`.
4. `add_addr` is called → constructs `AddrInfo::new(P_addr, 0, 100, flags)`.
5. Inside `AddrManager::add`: `0 >= 0` → `true` → `id_to_info.insert(id, addr_info)` replaces the entry.
6. P's score is now `100` again; the penalty is gone.

The discovery protocol entry point is confirmed at: [4](#0-3) 

### Impact Explanation

The scoring system's purpose is to accumulate evidence of misbehavior and eventually ban a peer when `score < ban_score (40)`. Resetting the score to `100` after each penalty cycle means a peer can misbehave indefinitely without ever reaching the ban threshold, as long as it (or a colluding peer) re-advertises its address after each penalty. This undermines the only soft-ban mechanism available below the hard IP ban.

### Likelihood Explanation

The precondition — `last_connected_at_ms == 0` — is the normal state for any peer that has been discovered but not yet successfully dialed. Penalties via `report` are applied during active sessions (e.g., for protocol violations), so the window where a peer is penalized but still has `last_connected_at_ms = 0` is real and common. The discovery protocol accepts `AddAddrs` from any connected peer without authentication, making the trigger trivially reachable.

### Recommendation

Preserve the existing `score` when updating an `AddrInfo` entry in `AddrManager::add`. Only update fields that reflect freshness (timestamps, flags, attempts). The score should only be modified through `report`. A minimal fix in `addr_manager.rs`:

```rust
if addr_info.last_connected_at_ms >= exist_last_connected_at_ms {
    let preserved_score = self.id_to_info.get(&id).expect("must exists").score;
    addr_info.random_id_pos = random_id_pos;
    addr_info.score = preserved_score;   // <-- preserve penalty
    self.id_to_info.insert(id, addr_info);
}
```

### Proof of Concept

```rust
// Pseudocode unit test
let mut peer_store = PeerStore::default();
let addr: Multiaddr = "/ip4/127.0.0.1/tcp/8000/p2p/Qm...".parse().unwrap();

// Step 1: discover peer
peer_store.add_addr(addr.clone(), Flags::COMPATIBILITY).unwrap();
assert_eq!(peer_store.addr_manager().get(&addr).unwrap().score, 100);

peer_store.report(&addr, Behaviour::UnexpectedDisconnect); // score -= 20 → 80
assert_eq!(peer_store.addr_manager().get(&addr).unwrap().score, 80);

// Step 3: re-advertise same addr (simulates remote discovery message)
peer_store.add_addr(addr.clone(), Flags::COMPATIBILITY).unwrap();

// Step 4: score is reset — penalty erased

    pub fn report(&mut self, addr: &Multiaddr, behaviour: Behaviour) -> ReportResult {
        if let Some(peer_addr) = self.addr_manager.get_mut(addr) {
            let score = peer_addr.score.saturating_add(behaviour.score());
            peer_addr.score = score;
            if score < self.score_config.ban_score {
                self.ban_addr(
                    addr,
                    self.score_config.ban_timeout_ms,
                    format!("report behaviour {behaviour:?}"),
                );
                return ReportResult::Banned;
            }
        }

    }
