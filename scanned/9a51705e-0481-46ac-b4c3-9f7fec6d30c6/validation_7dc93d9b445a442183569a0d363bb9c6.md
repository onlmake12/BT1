### Title
Unauthenticated Block Template Injection via Miner Notify Endpoint — (`File: miner/src/client.rs`)

---

### Summary

The CKB miner's HTTP notify-mode listener (`handle` function in `miner/src/client.rs`) accepts block template updates from **any** HTTP client without authentication or source verification. Any attacker who can reach the miner's listening port can inject a crafted `BlockTemplate` — including one with a substituted coinbase lock script — causing the miner to work on the attacker's template and surrender block rewards to the attacker.

---

### Finding Description

When the miner is configured in notify mode (`config.listen` is set), it binds a TCP listener and serves an HTTP endpoint via `listen_block_template_notify`. Every incoming HTTP POST is dispatched to the `handle` function:

```rust
// miner/src/client.rs:358-369
async fn handle(
    client: Client,
    req: Request<hyper::body::Incoming>,
) -> Result<Response<Empty<Bytes>>, Error> {
    let body = BodyExt::collect(req).await?.aggregate();

    if let Ok(template) = serde_json::from_reader(body.reader()) {
        client.update_block_template(template);
    }

    Ok(Response::new(Empty::new()))
}
```

There is **no authentication check, no IP allowlist, no shared secret, and no source validation** of any kind. The function deserializes the body as a `BlockTemplate` and immediately calls `client.update_block_template(template)`, replacing the current work for all mining workers.

The miner's RPC client does implement `parse_authorization` for outbound requests to the CKB node:

```rust
// miner/src/client.rs:380-394
fn parse_authorization(url: &Uri) -> Option<HeaderValue> { ... }
```

But this is only used for **outbound** calls to the node — the inbound notify endpoint has no corresponding check.

The `spawn_background` function shows that notify mode is explicitly opt-in and the listen address is operator-configured:

```rust
// miner/src/client.rs:204-232
pub fn spawn_background(self) {
    let client = self.clone();
    if let Some(addr) = self.config.listen {
        ...
        self.handle.spawn(async move {
            client.listen_block_template_notify(addr).await;
        });
```

When an operator binds the miner to a non-loopback address (e.g., `0.0.0.0:PORT` for a remote mining farm setup), the endpoint is reachable by any network peer.

---

### Impact Explanation

An attacker who can reach the miner's notify port can:

1. **Steal mining rewards**: Inject a `BlockTemplate` with the coinbase `lock` script replaced by the attacker's own address. The miner solves PoW for the attacker's block, and the resulting block — if valid — pays the block reward to the attacker.
2. **Waste hashpower / cause revenue loss**: Continuously inject templates with invalid parent hashes or malformed transactions, causing every solved block to be rejected by the node (`submit_block` will fail `HeaderVerifier` or `blocking_process_block`), wasting all hashpower.
3. **Suppress legitimate blocks**: By injecting templates faster than the node pushes them, the attacker can prevent the miner from ever working on a legitimate template.

The `submit_block` RPC does perform header and PoW verification:

```rust
// rpc/src/module/miner.rs:275-298
HeaderVerifier::new(snapshot, consensus)
    .verify(&header)
    .map_err(|err| handle_submit_error(&work_id, &err))?;
...
let is_new = self.chain.blocking_process_block(Arc::clone(&block))
    .map_err(|err| handle_submit_error(&work_id, &err))?;
```

However, a crafted template with a valid parent hash and a substituted coinbase address will pass all node-side verification — the node does not check that the coinbase lock matches any operator-configured address.

---

### Likelihood Explanation

- **Precondition**: The miner must be running in notify mode (`listen` address configured) and the port must be reachable by the attacker. This is a realistic deployment scenario for mining farms where the miner process runs on a different machine than the CKB node.
- **Attacker capability**: No privileged access required. A standard TCP connection and a single HTTP POST with a crafted JSON body is sufficient.
- **No cryptographic barrier**: There is no signature, token, or HMAC protecting the endpoint.

---

### Recommendation

Add source authentication to the miner's notify endpoint. Options in increasing strength:

1. **IP allowlist**: Only accept connections from the configured CKB node's IP address.
2. **Shared secret / Bearer token**: Require a configurable `Authorization` header on inbound notify requests, mirroring the existing `parse_authorization` logic used for outbound calls.
3. **Bind to loopback only by default**: Change the default (and document clearly) that `listen` should only be bound to `127.0.0.1` unless the operator explicitly understands the risk.

---

### Proof of Concept

1. Operator starts `ckb miner` with `listen = "0.0.0.0:18114"` in `ckb-miner.toml`.
2. Attacker obtains the current tip block hash (public via `get_tip_header` RPC on the node).
3. Attacker crafts a `BlockTemplate` JSON with:
   - `parent_hash` set to the current tip
   - `cellbase.data.outputs[0].lock` set to the attacker's own lock script
   - All other fields copied from a legitimate template
4. Attacker sends:
   ```
   POST http://<miner-ip>:18114/ HTTP/1.1
   Content-Type: application/json

   { <crafted BlockTemplate JSON> }
   ```
5. The `handle` function at `miner/src/client.rs:364-366` deserializes the template and calls `client.update_block_template(template)` with no checks.
6. All mining workers receive `WorkerMessage::NewWork` with the attacker's template.
7. When a nonce is found, `submit_block` is called with the attacker's coinbase — the block reward is paid to the attacker. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

fn parse_authorization(url: &Uri) -> Option<HeaderValue> {
    let a: Vec<&str> = url.authority()?.as_str().split('@').collect();
    if a.len() >= 2 {
        if a[0].is_empty() {
            return None;
        }
        let mut encoded = "Basic ".to_string();
        base64::prelude::BASE64_STANDARD.encode_string(a[0], &mut encoded);
        let mut header = HeaderValue::from_str(&encoded).unwrap();
        header.set_sensitive(true);
        Some(header)
    } else {
        None
    }
}
