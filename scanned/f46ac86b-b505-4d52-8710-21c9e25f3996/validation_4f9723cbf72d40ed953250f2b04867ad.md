### Title
Silent u128→u64 Truncation in DAO Maximum Withdrawal Calculation Causes Depositor Fund Loss — (`File: util/dao/src/lib.rs`)

### Summary

`DaoCalculator::calculate_maximum_withdraw` in `util/dao/src/lib.rs` uses an unchecked `as u64` cast to narrow a u128 intermediate result. When the intermediate value exceeds `u64::MAX`, the cast silently truncates instead of returning a `DaoError::Overflow`. This is inconsistent with every other analogous u128→u64 narrowing in the same codebase, all of which use `u64::try_from(...).map_err(|_| DaoError::Overflow)?`. The truncated value is orders of magnitude smaller than the correct withdrawal amount, causing the fee computation to either fail (rejecting a valid withdrawal) or silently accept a transaction where the depositor receives far less than their entitlement. The existing test `check_withdraw_calculation_overflows` is designed to assert `result.is_err()` for this exact overflow scenario, but the `as u64` cast means the function returns `Ok(truncated_small_value)` instead, so the test itself is broken by the bug.

### Finding Description

In `calculate_maximum_withdraw`, the withdrawal amount for the "counted" (non-occupied) portion of a DAO cell is computed as:

```rust
let withdraw_counted_capacity = u128::from(counted_capacity.as_u64())
    * u128::from(withdrawing_ar)
    / u128::from(deposit_ar);
let withdraw_capacity =
    Capacity::shannons(withdraw_counted_capacity as u64)   // ← unchecked truncation
        .safe_add(occupied_capacity)?;
``` [1](#0-0) 

The `ar` (accumulate rate) field stored in every block header starts at `DEFAULT_GENESIS_ACCUMULATE_RATE = 10_000_000_000_000_000` (10^16) and grows monotonically with each block's secondary issuance. [2](#0-1) 

The growth formula is `ar_increase = parent_ar * g2 / c`, applied every block. [3](#0-2) 

Over time, `withdrawing_ar / deposit_ar` grows. When `counted_capacity * withdrawing_ar / deposit_ar` exceeds `u64::MAX ≈ 1.844 × 10^19`, the `as u64` cast wraps the value modulo 2^64, producing a tiny result (e.g., ~1,842,384 shannons) instead of the correct large value.

Every other u128→u64 narrowing in the same file uses the safe pattern:

```rust
// secondary_block_reward (line 204):
let reward = u64::try_from(reward128).map_err(|_| DaoError::Overflow)?;

// dao_field_with_current_epoch (line 258):
let ar_increase = u64::try_from(ar_increase128).map_err(|_| DaoError::Overflow)?;
``` [4](#0-3) [5](#0-4) 

The `calculate_maximum_withdraw` function is called from two paths:

1. **Transaction fee verification** (`FeeCalculator::transaction_fee` → `DaoCalculator::transaction_fee` → `transaction_maximum_withdraw` → `calculate_maximum_withdraw`), which is invoked for every DAO withdrawal transaction submitted to the tx-pool and during block verification. [6](#0-5) [7](#0-6) 

2. **The `calculate_dao_maximum_withdraw` RPC**, which wallets call to determine how much to put in the withdrawal output. [8](#0-7) 

The existing overflow test confirms the intent but is broken by the bug:

```rust
assert!(result.is_err());  // fails: result is Ok(truncated_small_value)
``` [9](#0-8) 

### Impact Explanation

Two failure modes arise when `withdraw_counted_capacity` overflows u64:

**Mode A — Transaction permanently rejected (deadlock):** A wallet queries the RPC `calculate_dao_maximum_withdraw`, which calls the same buggy function and returns the truncated (tiny) value. The wallet builds a withdrawal transaction with `outputs_capacity` equal to the correct large amount. During verification, `transaction_fee = maximum_withdraw(truncated) - outputs_capacity(correct)` underflows in `safe_sub`, returning `DaoError::Overflow`. The transaction is rejected. The depositor's funds are permanently locked in the DAO cell with no valid withdrawal path. [6](#0-5) 

**Mode B — Silent fund loss:** If the wallet trusts the RPC's truncated value and sets `outputs_capacity` to that truncated amount, the transaction is accepted (fee = 0 or small), but the depositor receives only the truncated amount. The remainder of the principal + interest is consumed as miner fee.

The `CapacityVerifier` explicitly skips the inputs ≥ outputs check for DAO withdrawals, so no secondary guard catches this: [10](#0-9) 

### Likelihood Explanation

The overflow condition is `counted_capacity × (withdrawing_ar / deposit_ar) > u64::MAX`. The `ar` ratio grows at roughly 3–4% per year (secondary issuance ≈ 1.344 billion CKB/year over total capacity ≈ 33–50 billion CKB). For a deposit of ~3.36 × 10^18 shannons (≈ 33.6 billion CKB, the approximate genesis supply), the ratio needs to reach ~5.5×, which occurs after roughly 48 years. For a 1-billion-CKB deposit the threshold is ~148 years. These are long time horizons, making the likelihood low in the near term. However, the bug is demonstrably present today (the overflow test is broken), the protocol is designed to operate indefinitely, and large institutional DAO deposits are an explicit design goal of Nervos DAO.

### Recommendation

Replace the unchecked cast with the same checked pattern used everywhere else in the file:

```rust
// Before (buggy):
Capacity::shannons(withdraw_counted_capacity as u64).safe_add(occupied_capacity)?

// After (correct):
let withdraw_counted_capacity_u64 =
    u64::try_from(withdraw_counted_capacity).map_err(|_| DaoError::Overflow)?;
Capacity::shannons(withdraw_counted_capacity_u64).safe_add(occupied_capacity)?
``` [1](#0-0) 

### Proof of Concept

The existing test `check_withdraw_calculation_overflows` already constructs the overflow scenario with `capacity ≈ u64::MAX` and `withdrawing_ar > deposit_ar`. With the current `as u64` cast, `withdraw_counted_capacity` wraps to ~1,842,384 shannons and `safe_add(zero)` succeeds, so `result` is `Ok(...)`. The test's `assert!(result.is_err())` therefore panics, demonstrating the broken overflow guard. A depositor holding a large DAO cell over a long period would encounter Mode A or Mode B above when attempting withdrawal. [9](#0-8)

    fn calculate_dao_maximum_withdraw(
        &self,
        out_point: OutPoint,
        kind: DaoWithdrawingCalculationKind,
    ) -> Result<Capacity> {
        let snapshot: &Snapshot = &self.shared.snapshot();
        let consensus = snapshot.consensus();
        let out_point: packed::OutPoint = out_point.into();
        let data_loader = snapshot.borrow_as_data_loader();
        let calculator = DaoCalculator::new(consensus, &data_loader);
        match kind {
            DaoWithdrawingCalculationKind::WithdrawingHeaderHash(withdrawing_header_hash) => {
                let (tx, deposit_header_hash) = snapshot
                    .get_transaction(&out_point.tx_hash())
                    .ok_or_else(|| RPCError::invalid_params("invalid out_point"))?;
                let output = tx
                    .outputs()
                    .get(out_point.index().into())
                    .ok_or_else(|| RPCError::invalid_params("invalid out_point"))?;
                let output_data = tx
                    .outputs_data()
                    .get(out_point.index().into())
                    .ok_or_else(|| RPCError::invalid_params("invalid out_point"))?;

                match calculator.calculate_maximum_withdraw(
                    &output,
                    core::Capacity::bytes(output_data.len()).expect("should not overflow"),
                    &deposit_header_hash,
                    &withdrawing_header_hash.into(),
                ) {
                    Ok(capacity) => Ok(capacity.into()),
                    Err(err) => Err(RPCError::custom_with_error(RPCError::DaoError, err)),
                }
            }
            DaoWithdrawingCalculationKind::WithdrawingOutPoint(withdrawing_out_point) => {
                let (_tx, deposit_header_hash) = snapshot
                    .get_transaction(&out_point.tx_hash())
                    .ok_or_else(|| RPCError::invalid_params("invalid out_point"))?;

                let withdrawing_out_point: packed::OutPoint = withdrawing_out_point.into();
                let (withdrawing_tx, withdrawing_header_hash) = snapshot
                    .get_transaction(&withdrawing_out_point.tx_hash())
                    .ok_or_else(|| RPCError::invalid_params("invalid withdrawing_out_point"))?;

                let output = withdrawing_tx
                    .outputs()
                    .get(withdrawing_out_point.index().into())
                    .ok_or_else(|| RPCError::invalid_params("invalid withdrawing_out_point"))?;
                let output_data = withdrawing_tx
                    .outputs_data()
                    .get(withdrawing_out_point.index().into())
                    .ok_or_else(|| RPCError::invalid_params("invalid withdrawing_out_point"))?;

                match calculator.calculate_maximum_withdraw(
                    &output,

                    &deposit_header_hash,
                    &withdrawing_header_hash,
                ) {
                    Ok(capacity) => Ok(capacity.into()),
                    Err(err) => Err(RPCError::custom_with_error(RPCError::DaoError, err)),
                }
            }

#[test]
fn check_withdraw_calculation_overflows() {
    let output = CellOutput::new_builder()
        .capacity(Capacity::shannons(18_446_744_073_709_550_000))
        .build();
    let tx = TransactionBuilder::default().output(output.clone()).build();
    let epoch = EpochNumberWithFraction::new(1, 100, 1000);
    let deposit_header = HeaderBuilder::default()
        .number(100)
        .epoch(epoch)
        .dao(pack_dao_data(
            10_000_000_000_123_456,
            Default::default(),
            Default::default(),
            Default::default(),
        ))
        .build();
    let deposit_block = BlockBuilder::default()
        .header(deposit_header)
        .transaction(tx)
        .build();

    let epoch = EpochNumberWithFraction::new(1, 200, 1000);
    let withdrawing_header = HeaderBuilder::default()
        .number(200)
        .epoch(epoch)
        .dao(pack_dao_data(
            10_000_000_001_123_456,
            Default::default(),
            Default::default(),
            Default::default(),
        ))
        .build();
    let withdrawing_block = BlockBuilder::default().header(withdrawing_header).build();

    let tmp_dir = TempDir::new().unwrap();
    let db = RocksDB::open_in(&tmp_dir, COLUMNS);
    let store = ChainDB::new(db, Default::default());
    let txn = store.begin_transaction();
    txn.insert_block(&deposit_block).unwrap();
    txn.attach_block(&deposit_block).unwrap();
    txn.insert_block(&withdrawing_block).unwrap();
    txn.attach_block(&withdrawing_block).unwrap();
    txn.commit().unwrap();

    let consensus = Consensus::default();
    let data_loader = store.borrow_as_data_loader();
    let calculator = DaoCalculator::new(&consensus, &data_loader);
    let result = calculator.calculate_maximum_withdraw(
        &output,
        Capacity::bytes(0).expect("should not overflow"),
        &deposit_block.hash(),
        &withdrawing_block.hash(),
    );
    assert!(result.is_err());
}
