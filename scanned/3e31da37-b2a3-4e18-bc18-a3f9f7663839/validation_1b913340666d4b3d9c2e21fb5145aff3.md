Audit Report

## Title
Unbounded heap allocation via attacker-controlled fee rate in `do_estimate` — (`util/fee-estimator/src/estimator/weight_units_flow.rs`)

## Summary

`max_bucket_index_by_fee_rate` has no upper bound on its return value. When a transaction with an abnormally high fee rate is in the mempool, `do_estimate` allocates two `Vec<u64>` of size `max_bucket_index + 1`, growing linearly and without limit with the fee rate. An unprivileged attacker who owns CKB can force arbitrarily large heap allocations on every `estimate_fee_rate` RPC call, causing OOM crashes or severe memory pressure on any node running the `WeightUnitsFlow` fee estimator.

## Finding Description

**Root cause — `max_bucket_index_by_fee_rate` is unbounded:**

The final match arm in `max_bucket_index_by_fee_rate` applies no ceiling:

```rust
x => (x + t * 11_500) / (100 * t),  // no cap
``` [1](#0-0) 

For `fee_rate = 10^12` (achievable with e.g. 10,000 CKB fee on a minimal-weight tx), the index evaluates to approximately `10,000,000`.

**Allocation site — two unbounded `Vec` allocations in `do_estimate`:**

`max_fee_rate` is taken directly from the highest-fee-rate tx in the current mempool snapshot with no clamping:

```rust
let max_fee_rate = if let Some(fee_rate) = sorted_current_txs.first().map(|tx| tx.fee_rate) {
    fee_rate
} ...
let max_bucket_index = Self::max_bucket_index_by_fee_rate(max_fee_rate);
let mut buckets = vec![0u64; max_bucket_index + 1];  // line 219
``` [2](#0-1) 

A second identical allocation follows at line 248: [3](#0-2) 

**No cap on fee rate in tx pool admission:**

`check_tx_fee` only enforces a minimum fee rate floor; there is no maximum fee rate check: [4](#0-3) 

**Call chain:**

1. Attacker submits `send_transaction` RPC with a high-fee, minimal-weight tx.
2. `check_tx_fee` passes (only min fee rate enforced).
3. `register_pending` fires → `fee_estimator_clone.accept_tx(tx_hash, entry_info)` stores the `TxStatus` with the inflated fee rate. [5](#0-4) 

4. Any caller of `estimate_fee_rate` RPC → `do_estimate` → reads the high-fee tx as `max_fee_rate` → `vec![0u64; max_bucket_index + 1]` × 2.

**Existing tests do not cover the dangerous range:**

The continuity test only iterates `fee_rate` up to `3_000_000` (index ~30), far below the dangerous range triggered by realistic high-fee inputs: [6](#0-5) 

## Impact Explanation

Every `estimate_fee_rate` RPC call while the high-fee tx remains in the mempool triggers two heap allocations proportional to the attacker's fee rate. At `fee_rate = 10^12` (achievable with ~10,000 CKB burned on a minimal-weight tx), each call allocates ~160 MB. Repeated concurrent calls cause OOM crashes or severe memory pressure, crashing the node. This matches the **High** impact class: *Vulnerabilities which could easily crash a CKB node* (10001–15000 points).

## Likelihood Explanation

The `WeightUnitsFlow` estimator must be explicitly configured — the default is `FeeEstimator::Dummy`: [7](#0-6) 

Nodes that have opted into `WeightUnitsFlow` are fully exposed. The attacker must own and burn CKB (real economic cost), but the cost is not prohibitive for a motivated attacker targeting a specific node. The attack is repeatable as long as the high-fee tx remains unconfirmed in the mempool, and any user calling `estimate_fee_rate` (not just the attacker) triggers the allocation.

## Recommendation

1. **Cap `max_bucket_index_by_fee_rate`**: add a `const MAX_BUCKET_INDEX: usize` (e.g., 1000) and clamp the return value before use.
2. **Clamp `max_fee_rate` before computing `max_bucket_index`**: apply a safe maximum (e.g., `FeeRate::from_u64(2_000_000)`) to `max_fee_rate` in `do_estimate` before passing it to `max_bucket_index_by_fee_rate`.
3. **Add a maximum fee rate check in tx pool admission** in `check_tx_fee` to reject transactions with absurdly high fee rates.

## Proof of Concept

```rust
// Verify the index is unbounded:
let fee_rate = FeeRate::from_u64(1_000_000_000_000u64); // 10^12
let idx = Algorithm::max_bucket_index_by_fee_rate(fee_rate);
// idx ≈ 10_000_000
// vec![0u64; idx + 1] = 80 MB per allocation
// Two allocations in do_estimate = 160 MB per RPC call
// 20 concurrent calls → 3.2 GB → OOM
```

Minimal steps:
1. Configure node with `algorithm = "WeightUnitsFlow"` in `[fee_estimator]`.
2. Submit a transaction with a very small weight and a very large fee (e.g., 10,000 CKB fee, ~255 weight units from a secp256k1 input).
3. While the tx is pending, call `estimate_fee_rate` RPC repeatedly (or concurrently).
4. Observe OOM crash or severe memory pressure on the node.

    fn max_bucket_index_by_fee_rate(fee_rate: FeeRate) -> usize {
        let t = FEE_RATE_UNIT;
        let index = match fee_rate.as_u64() {
            x if x <= 10_000 => x / t,
            x if x <= 50_000 => (x + t * 10) / (2 * t),
            x if x <= 200_000 => (x + t * 100) / (5 * t),
            x if x <= 500_000 => (x + t * 400) / (10 * t),
            x if x <= 1_000_000 => (x + t * 1_300) / (20 * t),
            x if x <= 2_000_000 => (x + t * 4_750) / (50 * t),
            x => (x + t * 11_500) / (100 * t),
        };
        index as usize
    }

pub(crate) fn check_tx_fee(
    tx_pool: &TxPool,
    snapshot: &Snapshot,
    rtx: &ResolvedTransaction,
    tx_size: usize,
) -> Result<Capacity, Reject> {
    let fee = DaoCalculator::new(snapshot.consensus(), &snapshot.borrow_as_data_loader())
        .transaction_fee(rtx)
        .map_err(|err| {
            Reject::Malformed(
                format!("{err}"),
                "expect (outputs capacity) <= (inputs capacity)".to_owned(),
            )
        })?;
    // Theoretically we cannot use size as weight directly to calculate fee_rate,
    // here min fee rate is used as a cheap check,
    // so we will use size to calculate fee_rate directly
    let min_fee = tx_pool.config.min_fee_rate.fee(tx_size as u64);
    // reject txs which fee lower than min fee rate
    if fee < min_fee {
        let reject =
            Reject::LowFeeRate(tx_pool.config.min_fee_rate, min_fee.as_u64(), fee.as_u64());
        ckb_logger::debug!("Reject tx {}", reject);
        return Err(reject);
    }
    Ok(fee)
