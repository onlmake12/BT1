### Title
Expired Alert Replay via P2P: Missing `notice_until` Enforcement in `AlertRelayer::received` — (`util/network-alert/src/alert_relayer.rs`)

### Summary
The P2P `AlertRelayer::received` handler accepts and relays alert messages from peers without checking whether the alert's `notice_until` deadline has already passed. Any unprivileged P2P peer can replay a previously valid, now-expired alert (with authentic Nervos Foundation signatures) to cause it to propagate across the entire network, triggering stale/false warning displays on all connected nodes.

### Finding Description

The CKB network alert system allows the Nervos Foundation to broadcast critical messages to all nodes. Alerts are signed with a 2-of-4 multisig scheme and contain a `notice_until` timestamp field that is included in the signed hash — so the signature is cryptographically bound to the expiry time.

The RPC entry point `send_alert` correctly enforces the deadline:

```rust
// rpc/src/module/alert.rs:104-110
let now_ms = ckb_systemtime::unix_time_as_millis();
let notice_until: u64 = alert.raw().notice_until().into();
if notice_until < now_ms {
    return Err(RPCError::invalid_params(...));
}
```

However, the P2P relay handler `AlertRelayer::received` performs no such check:

```rust
// util/network-alert/src/alert_relayer.rs:98-178
async fn received(&mut self, nc: Arc<dyn CKBProtocolContext + Sync>, peer_index: PeerIndex, data: Bytes) {
    // ... parse alert ...
    if self.notifier.lock().has_received(alert_id) { return; }
    // verify cryptographic signatures only — no notice_until check
    if let Err(err) = self.verifier.verify_signatures(&alert) { ... return; }
    // immediately broadcast to all peers
    nc.quick_filter_broadcast(...);
    // add to notifier (displays to user)
    self.notifier.lock().add(&alert);
}
```

`clear_expired_alerts` is called only in `connected()`, not in `received()`. A peer sending an expired alert with a previously unseen alert ID bypasses the `has_received` deduplication check, passes signature verification (signatures are cryptographically valid), and causes the alert to be stored and relayed.

The signed data structure (`RawAlert`) includes `notice_until` in the hash: [1](#0-0) 

The `notice_until` field is part of the alert type: [2](#0-1) 

The RPC handler enforces the deadline: [3](#0-2) 

The P2P handler does not: [4](#0-3) 

### Impact Explanation

Any P2P peer can replay a past alert (e.g., the publicly known alert `20230001` with its published signatures) after its `notice_until` has expired. The replayed alert:

1. Passes signature verification (signatures are cryptographically valid and public).
2. Is stored in `received_alerts` and `noticed_alerts` on the receiving node.
3. Is immediately broadcast to all connected peers via `quick_filter_broadcast`.
4. Triggers `notify_controller.notify_network_alert` — displaying the stale warning to users.

This causes network-wide propagation of expired, stale critical warnings, potentially causing user panic, confusion about the current network state, and unnecessary operational responses. It also wastes P2P bandwidth across the entire network. [5](#0-4) 

### Likelihood Explanation

Likelihood is **high**. Past alerts with valid Nervos Foundation signatures are publicly observable on-chain and in node logs. Any peer connected to the CKB P2P network can craft a replay by simply re-sending a previously observed alert message. No key material, privileged access, or majority hashpower is required — only a standard P2P connection. [6](#0-5) 

### Recommendation

Add an expiry check in `AlertRelayer::received` immediately after parsing the alert, mirroring the check already present in the RPC handler:

```rust
// In received(), after parsing the alert:
let now_ms = ckb_systemtime::unix_time_as_millis();
let notice_until: u64 = alert.as_reader().raw().notice_until().into();
if notice_until < now_ms {
    debug!("Received expired alert {} from peer {}", alert_id, peer_index);
    return; // silently drop; optionally ban peer
}
```

This mirrors the existing guard in `rpc/src/module/alert.rs` lines 104–110 and closes the asymmetry between the RPC and P2P ingress paths. [3](#0-2) 

### Proof of Concept

1. Observe a past CKB network alert (e.g., alert `20230001`, `notice_until = 1681574400000`) with its two published signatures from the Nervos Foundation.
2. Connect to any CKB mainnet node as a standard P2P peer using the Alert protocol (`SupportProtocols::Alert`).
3. Serialize the expired `packed::Alert` and send it via the P2P connection.
4. The receiving node's `AlertRelayer::received` will:
   - Parse the message successfully.
   - Find `has_received(20230001) == false` (first time seeing this alert on this node).
   - Pass `verify_signatures` (the 2-of-4 signatures are cryptographically valid).
   - Broadcast the expired alert to all its connected peers.
   - Add the alert to its notifier, displaying the stale "CKB v0.105.* have bugs" warning to the node operator.
5. Each receiving peer repeats the process, propagating the expired alert network-wide.

        &mut self,
        nc: Arc<dyn CKBProtocolContext + Sync>,
        peer_index: PeerIndex,
        data: Bytes,

        let alert: packed::Alert = match packed::AlertReader::from_slice(&data) {
            Ok(alert) => {
                if alert.raw().message().is_utf8()
                    && alert

                        .min_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                    && alert
                        .raw()
                        .max_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                {
                    alert.to_entity()
                } else {
                    info!(
                        "A malformed message fromP peer {} : not utf-8 string",
                        peer_index
                    );
                    nc.ban_peer(
                        peer_index,
                        BAD_MESSAGE_BAN_TIME,
                        String::from("send us a malformed message: not utf-8 string"),
                    );
                    return;
                }
            }
            Err(err) => {
                info!("A malformed message from peer {}: {:?}", peer_index, err);
                nc.ban_peer(
                    peer_index,
                    BAD_MESSAGE_BAN_TIME,
                    String::from("send us a malformed message"),
                );
                return;
            }
        };
        let alert_id = alert.as_reader().raw().id().into();
        trace!("ReceiveD alert {} from peer {}", alert_id, peer_index);
        // ignore alert
        if self.notifier.lock().has_received(alert_id) {
            return;
        }
        // verify
        if let Err(err) = self.verifier.verify_signatures(&alert) {
            debug!(
                "An alert from peer {} with invalid signatures, error {:?}",
                peer_index, err
            );
            nc.ban_peer(
                peer_index,
                BAD_MESSAGE_BAN_TIME,
                String::from("send us an alert with invalid signatures"),
            );
            return;
        }
        // mark sender as known
        self.mark_as_known(peer_index, alert_id);
        // broadcast message
        let selected_peers: Vec<PeerIndex> = nc
            .connected_peers()
            .into_iter()
            .filter(|peer| self.mark_as_known(*peer, alert_id))
            .collect();
        if let Err(err) = nc.quick_filter_broadcast(
            TargetSession::Multi(Box::new(selected_peers.into_iter())),
            data,
        ) {
            debug!("alert broadcast error: {:?}", err);
        }
        // add to received alerts
        self.notifier.lock().add(&alert);

    pub fn add(&mut self, alert: &Alert) {
        let alert_id = alert.raw().id().into();
        let alert_cancel = alert.raw().cancel().into();
        if self.has_received(alert_id) {
            return;
        }
        // checkout cancel_id
        if alert_cancel > 0 {
            self.cancel(alert_cancel);
        }
        // add to received alerts
        self.received_alerts.insert(alert_id, alert.clone());

        // check conditions, figure out do we need to notice this alert
        if !self.is_version_effective(alert) {
            debug!("Received a version ineffective alert {:?}", alert);
            return;
        }

        if self.noticed_alerts.contains(alert) {
            return;
        }
        self.notify_controller.notify_network_alert(alert.clone());
        self.noticed_alerts.push(alert.clone());
        // sort by priority
        self.noticed_alerts.sort_by_key(|a| {
            let priority: u32 = a.raw().priority().into();
            u32::MAX - priority
        });
    }
