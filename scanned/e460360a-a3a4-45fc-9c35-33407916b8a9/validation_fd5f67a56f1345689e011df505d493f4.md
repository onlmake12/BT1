### Title
Unchecked `.unwrap()` on `get_pool_entry` in RBF Fee Calculation Causes Node Panic - (File: tx-pool/src/pool.rs)

### Summary

The `min_replace_fee` function in `tx-pool/src/pool.rs` calls `.unwrap()` directly on the result of `get_pool_entry()`, which returns an `Option<&PoolEntry>`. If the pool entry is absent (returns `None`), the node panics unconditionally. This is the direct Rust analog of calling `approve` instead of `safeApprove`: both substitute an unsafe, unchecked operation for one that should handle failure gracefully.

### Finding Description

In `tx-pool/src/pool.rs`, the `min_replace_fee` function computes the minimum fee required for a Replace-By-Fee (RBF) transaction to evict an existing pool entry:

```rust
pub fn min_replace_fee(&self, tx: &TxEntry) -> Option<Capacity> {
    if !self.enable_rbf() {
        return None;
    }

    let mut conflicts = vec![self.get_pool_entry(&tx.proposal_short_id()).unwrap()];
    ...
}
``` [1](#0-0) 

`get_pool_entry` returns `Option<&PoolEntry>`. The code calls `.unwrap()` on it without any guard. If the entry identified by `tx.proposal_short_id()` is not present in the pool at the moment of the call — due to a race between pool eviction and RBF admission, or because the caller passes a `TxEntry` whose short ID does not correspond to an existing pool entry — the thread panics.

Note that on line 95, the same function uses the safe pattern `filter_map(|id| self.get_pool_entry(id))` for descendants, correctly handling the `None` case. The inconsistency between line 91 (unsafe `.unwrap()`) and line 95 (safe `filter_map`) within the same function body makes the root cause clear. [2](#0-1) 

### Impact Explanation

A panic in the tx-pool service thread causes the CKB node to crash. Because the tx-pool is a long-running service that processes all incoming transactions, a crash terminates the node process entirely, requiring a manual restart. This constitutes a **remote denial-of-service** against any node that has RBF enabled (`min_rbf_rate > min_fee_rate`).

### Likelihood Explanation

RBF is an opt-in feature controlled by node configuration, but it is a documented and supported feature. Any unprivileged tx-pool submitter (RPC caller using `send_transaction`) can submit transactions. If the pool entry is evicted (e.g., due to expiry, capacity limits, or a concurrent block commit) between the point where the RBF conflict is detected and the point where `min_replace_fee` is called, `get_pool_entry` returns `None` and the panic fires. The tx-pool service processes requests concurrently with block processing, making this a realistic race window.

### Recommendation

Replace the unsafe `.unwrap()` with a safe alternative that returns `None` (or an appropriate error) when the pool entry is absent, consistent with how the function already handles descendants on line 95:

```rust
// Before (unsafe):
let mut conflicts = vec![self.get_pool_entry(&tx.proposal_short_id()).unwrap()];

// After (safe):
let root_entry = self.get_pool_entry(&tx.proposal_short_id())?;
let mut conflicts = vec![root_entry];
```

Since `min_replace_fee` already returns `Option<Capacity>`, using `?` propagates `None` cleanly without any panic.

### Proof of Concept

1. Enable RBF on a CKB node by setting `min_rbf_rate > min_fee_rate` in the config.
2. Submit transaction `T1` to the pool.
3. Arrange for `T1` to be evicted from the pool (e.g., wait for expiry, or trigger a block commit that includes `T1`).
4. Simultaneously submit a replacement transaction `T2` that conflicts with `T1`'s inputs, triggering the RBF path.
5. The RBF admission logic calls `min_replace_fee` with the `TxEntry` for the conflict; `get_pool_entry` returns `None` because `T1` is no longer in the pool; `.unwrap()` panics; the node crashes. [3](#0-2)

    pub fn min_replace_fee(&self, tx: &TxEntry) -> Option<Capacity> {
        if !self.enable_rbf() {
            return None;
        }

        let mut conflicts = vec![self.get_pool_entry(&tx.proposal_short_id()).unwrap()];
        let descendants = self.pool_map.calc_descendants(&tx.proposal_short_id());
        let descendants = descendants
            .iter()
            .filter_map(|id| self.get_pool_entry(id))
            .collect::<Vec<_>>();
        conflicts.extend(descendants);
        self.calculate_min_replace_fee(&conflicts, tx.size)
    }
