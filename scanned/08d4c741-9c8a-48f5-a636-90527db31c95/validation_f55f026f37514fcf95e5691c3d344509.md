Audit Report

## Title
`check_purge` integer-division yields `take(0)` when all addresses share `Group::None`, permanently blocking peer store insertion — (`network/src/peer_store/peer_store_impl.rs`)

## Summary
When all addresses in the peer store map to `Group::None` (e.g., Onion3 multiaddrs), `check_purge` computes `len / 2 == 1 / 2 == 0` in Rust integer arithmetic, causing `take(0)` to select zero eviction candidates. The function then returns `Err(PeerStoreError::EvictionFailed)`, permanently blocking any further `add_addr` calls. An attacker can trigger this by flooding the discovery protocol with 16384 distinct Onion3 addresses, all of which are freshly connectable and all of which hash to the same `Group::None` bucket.

## Finding Description
`Group::from(&Multiaddr)` in `network/src/network_group.rs` calls `multiaddr_to_socketaddr`, which returns `None` for Onion3 addresses, causing the function to fall through to `Group::None` (lines 39–41). All 16384 Onion3 addresses therefore land in a single `HashMap` entry.

In `check_purge` (`network/src/peer_store/peer_store_impl.rs`, lines 327–403):

- **Step 1 (lines 341–355):** Evict non-connectable addresses. Freshly added addresses have `last_connected_at_ms = 0` and `attempts_count = 0` (`add_addr` line 78). `is_connectable` returns `true` for these (types.rs lines 89–105), so step 1 finds zero candidates.
- **Step 2 (lines 357–401):** Group by network segment. With all addresses in `Group::None`, `peers_by_network_group` has exactly 1 entry, so `len = 1`. Line 376: `peers.into_iter().take(len / 2)` → `take(1 / 2)` → `take(0)`. The iterator yields nothing.
- **Line 399–400:** `candidate_peers.is_empty()` is true → `return Err(PeerStoreError::EvictionFailed)`.

`add_addr` propagates this error (line 75: `self.check_purge()?`), so every subsequent peer address insertion fails permanently until the store is manually cleared or the node is restarted with a clean database.

There is a secondary guard inside the `flat_map` at line 378 (`if addrs.len() > 4`) that would also suppress eviction for small groups, but it is never reached because `take(0)` short-circuits first.

## Impact Explanation
Once the store is saturated, the node cannot learn any new peer addresses from the discovery protocol. After a restart the node loads 16384 unconnectable Onion3 addresses and is unable to bootstrap new connections, effectively isolating it from the network. Applied at scale against multiple nodes this degrades CKB peer-to-peer connectivity with minimal attacker cost, matching **High: Vulnerabilities or bad designs which could cause CKB network congestion with few costs**.

## Likelihood Explanation
The attacker needs only a single connection to the target node via the discovery protocol and the ability to send `AddressesMessage` packets containing distinct Onion3 multiaddrs. No proof-of-work, no privileged role, and no Sybil majority is required. The 35-byte host + 2-byte port space provides far more than 16384 distinct addresses. The attack is repeatable: after a node restart the store can be reflooded immediately.

## Recommendation
Replace `take(len / 2)` with `take((len + 1) / 2)` (ceiling division) to guarantee at least one group is always selected when `len >= 1`. Additionally, enforce a per-group sub-limit on `Group::None` addresses (e.g., cap at `ADDR_COUNT_LIMIT / 4`) so that ungroupable addresses cannot monopolize the store.

## Proof of Concept
```rust
#[test]
fn test_check_purge_onion3_dos() {
    let mut store = PeerStore::default();
    // Fill store with 16383 distinct Onion3 addresses
    for i in 0u64..16383 {
        let mut host = [0u8; 35];
        host[..8].copy_from_slice(&i.to_le_bytes());
        let onion3 = Protocol::Onion3((host, 1234).into());
        let addr: Multiaddr = std::iter::once(onion3).collect();
        store.add_addr(addr, Flags::COMPATIBILITY).unwrap();
    }
    // 16384th address fills the store and triggers check_purge on the next call
    let mut host = [0xfeu8; 35];
    let onion3 = Protocol::Onion3((host, 1234).into());
    let addr: Multiaddr = std::iter::once(onion3).collect();
    store.add_addr(addr, Flags::COMPATIBILITY).unwrap();

    // Any further add_addr must now fail with EvictionFailed
    let mut host = [0xffu8; 35];
    let onion3 = Protocol::Onion3((host, 9999).into());
    let addr: Multiaddr = std::iter::once(onion3).collect();
    let result = store.add_addr(addr, Flags::COMPATIBILITY);
    assert!(result.is_err());
    assert!(result.unwrap_err().to_string().contains("EvictionFailed"));
}
```
