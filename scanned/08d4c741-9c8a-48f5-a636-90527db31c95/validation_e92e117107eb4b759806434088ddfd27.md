Audit Report

## Title
Peer Store Eviction Bypass via Uniform /16-Group Distribution — (`network/src/peer_store/peer_store_impl.rs`)

## Summary
`PeerStore::check_purge` contains two sequential eviction strategies. When the address store reaches `ADDR_COUNT_LIMIT` (16384) and all entries are distributed as exactly 4 addresses per IPv4 /16 group across 4096 distinct groups, both eviction strategies produce zero candidates and `PeerStoreError::EvictionFailed` is returned unconditionally. Every subsequent `add_addr` call fails permanently, freezing peer discovery. The poisoned state persists across node restarts.

## Finding Description

**Step 1 — Non-connectable eviction** (`peer_store_impl.rs` lines 341–355):

`is_connectable` (`types.rs` lines 89–105) returns `false` only when `last_connected_at_ms == 0 && attempts_count >= ADDR_MAX_RETRIES (3)`, or when the address has timed out with `attempts_count >= ADDR_MAX_FAILURES (10)`. Freshly-advertised addresses are created with `attempts_count = 0` (`types.rs` line 72) and `last_tried_at_ms = 0`, so `tried_in_last_minute` is false for any real `now_ms`, and neither failure condition is met. All fresh addresses are connectable; step 1 finds zero candidates.

**Step 2 — Network-group eviction** (`peer_store_impl.rs` lines 357–402):

Addresses are grouped by `Group`, which for IPv4 uses only the first two octets (`network_group.rs` lines 26–28), forming /16 buckets. The eviction loop takes only the top `len / 2` groups by size (line 376), then applies a strict `addrs.len() > 4` guard (line 378). Groups of exactly 4 return `None` from `flat_map`. If all groups have `len == 4`, `candidate_peers` is empty and `EvictionFailed` is returned at lines 399–401.

**Attack construction:**

- `ADDR_COUNT_LIMIT = 16384` (`mod.rs` line 26)
- 4096 distinct /16 subnets × 4 addresses each = 16384 entries, all connectable
- `check_purge` is called before insertion (line 75); at limit, both steps fail, `EvictionFailed` propagates, and the new address is never added
- `add_new_addrs` in the discovery protocol (`discovery/mod.rs` lines 347–363) calls `add_addr` for every address in received `Nodes` messages; errors are silently discarded at `debug` level
- The peer store is serialized to disk on shutdown and reloaded on startup (`peer_store_db.rs` lines 112–132), so the poisoned state survives restarts

## Impact Explanation

Once poisoned, the node cannot ingest any new peer addresses via discovery. With 16384 existing entries that churn over time and no ability to replenish, the node's reachable peer set degrades monotonically. This matches **Medium (2001–10000 points): Suboptimal implementation of CKB state storage mechanism** — the peer store's eviction logic contains a structural flaw that an unprivileged attacker can exploit to permanently disable peer address ingestion.

## Likelihood Explanation

The attacker requires only a single P2P connection to the victim. Sending ~16384 addresses across multiple `Nodes` messages is trivially achievable with no PoW, no special privileges, and no Sybil capability beyond one connection. The poisoned state persists across restarts, making remediation require manual intervention. The attack is deterministic and fully repeatable.

## Recommendation

Replace the strict `> 4` threshold at `peer_store_impl.rs` line 378 with `>= 1` so that any non-empty group is eligible for eviction. Alternatively, guarantee at least one eviction candidate regardless of group distribution — for example, always evict the oldest-tried address from the largest group, falling back to a random address if all groups are equal-sized. Additionally, enforce a per-session cap on the number of addresses accepted from a single `Nodes` message source in `add_new_addrs`.

## Proof of Concept

```rust
// Fill store with 4096 /16 groups × 4 addresses = 16384 entries (all connectable, attempts_count=0)
let mut peer_store = PeerStore::default();
for a in 1u8..=16 {
    for b in 0u8..=255 {
        for host in 1u8..=4 {
            let addr: Multiaddr = format!(
                "/ip4/{}.{}.0.{}/tcp/8115/p2p/{}",
                a, b, host, PeerId::random().to_base58()
            ).parse().unwrap();
            peer_store.add_addr(addr, Flags::COMPATIBILITY).unwrap();
        }
    }
}
// Store is now at ADDR_COUNT_LIMIT with 4096 groups of exactly 4

// Any new address fails permanently
let new_addr: Multiaddr = "/ip4/200.0.0.1/tcp/8115/p2p/Qm...".parse().unwrap();
let result = peer_store.add_addr(new_addr, Flags::COMPATIBILITY);
assert!(matches!(result, Err(e) if e.to_string().contains("EvictionFailed")));
```
