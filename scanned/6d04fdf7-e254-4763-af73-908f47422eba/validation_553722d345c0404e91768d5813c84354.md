### Title
`expiry_hours = 0` Causes Immediate Expiry of All Tx-Pool Transactions, Rendering the Pool Non-Functional — (File: `tx-pool/src/pool.rs`, `util/app-config/src/configs/tx_pool.rs`)

---

### Summary

The `expiry_hours` field in `TxPoolConfig` has no minimum-value validation. Setting it to `0` in `ckb.toml` causes the computed expiry window to be `0` milliseconds, making every transaction in the pool satisfy the expiry condition on every block. The tx-pool is silently drained of all pending transactions on each new block, making the node unable to relay or mine any user transactions.

---

### Finding Description

`TxPoolConfig` exposes `expiry_hours: u8` as a plain, unchecked field: [1](#0-0) 

The default is `12` hours: [2](#0-1) 

No minimum-bound deserializer (analogous to the `at_least_100` guard used for notify timeouts) is applied: [3](#0-2) 

When `TxPool::new()` is called, the expiry window is computed as:

```rust
let expiry = config.expiry_hours as u64 * 60 * 60 * 1000;
``` [4](#0-3) 

If `expiry_hours = 0`, then `expiry = 0`. The expiry filter in `remove_expired()` becomes:

```rust
filter(|&entry| self.expiry + entry.inner.timestamp < now_ms)
// => filter(|&entry| 0 + entry.inner.timestamp < now_ms)
// => always true for any tx added even 1 ms ago
``` [5](#0-4) 

`remove_expired()` is called on every new block during `_update_tx_pool_for_reorg`: [6](#0-5) 

The result is that every transaction submitted to the pool is evicted with `Reject::Expiry` on the very next block, making the pool permanently empty.

---

### Impact Explanation

- **Tx-pool completely non-functional**: No pending transaction survives to be proposed or committed. The node mines only empty blocks (cellbase only).
- **Transaction relay broken**: Transactions received from peers are accepted into the pool momentarily, then immediately evicted, so they are never relayed onward or included in blocks.
- **RPC callers affected**: Any user submitting via `send_transaction` receives `TransactionExpired (-1109)` on the next block, with no way to get their transaction confirmed through this node.
- **Recovery**: Requires the operator to correct the config and restart the node. Unlike the original report's `exodusMode`, this is reversible, but the node is fully non-functional until corrected.

---

### Likelihood Explanation

Low. The default value (`12`) is safe and the field is not commonly changed. However, an operator experimenting with aggressive pool cleanup, or one who misreads the unit (thinking it is seconds rather than hours), could set it to `0` or `1`. There is no validation at parse time to prevent this, unlike the `notify_tx_timeout` fields which enforce `at_least_100`.

---

### Recommendation

Add a minimum-value deserializer for `expiry_hours`, analogous to the `at_least_100` guard already used for notify timeouts in `util/app-config/src/configs/notify.rs`. A reasonable minimum is `1` hour. The deserializer should reject values of `0` with a clear error message at startup, preventing silent pool failure.

---

### Proof of Concept

1. Set in `ckb.toml`:
   ```toml
   [tx_pool]
   expiry_hours = 0
   ```
2. Start the node. `TxPool::new()` computes `expiry = 0`.
3. Submit any transaction via `send_transaction` RPC.
4. On the next block, `remove_expired()` evaluates `0 + tx.timestamp < now_ms` → `true` for every entry.
5. All transactions are evicted with `Reject::Expiry`. The pool remains permanently empty.
6. Observe via `tx_pool_info` RPC: `pending` and `proposed` counts stay at `0` regardless of submissions. [7](#0-6) [8](#0-7) [1](#0-0)

    D: serde::de::Deserializer<'de>,

    // Expire all transaction (and their dependencies) in the pool.
    pub(crate) fn remove_expired(&mut self, callbacks: &Callbacks) {
        let now_ms = ckb_systemtime::unix_time_as_millis();

        let removed: Vec<_> = self
            .pool_map
            .iter()
            .filter(|&entry| self.expiry + entry.inner.timestamp < now_ms)
            .map(|entry| entry.inner.clone())
            .collect();

        for entry in removed {
            let tx_hash = entry.transaction().hash();
            debug!("remove_expired {} timestamp({})", tx_hash, entry.timestamp);
            self.pool_map.remove_entry(&entry.proposal_short_id());
            let reject = Reject::Expiry(entry.timestamp);
            callbacks.call_reject(self, &entry, reject);
        }
    }
