### Title
Permanent Spent-Cell State Corruption via `break` in Input Loop — (`util/rich-indexer/src/indexer/mod.rs`)

### Summary

`AsyncRichIndexer::insert_transaction` uses `break` instead of `continue` when `spend_cell` returns `false`. This permanently leaves subsequent inputs of the same transaction with `is_spent = 0` in the `output` table, causing the indexer to report those cells as live indefinitely.

### Finding Description

In `insert_transaction`, the input-processing loop is:

```rust
for (input_index, input) in tx_view.inputs().into_iter().enumerate() {
    let out_point = input.previous_output();
    if !spend_cell(&out_point, tx).await? {
        break;          // ← BUG: should be `continue`
    }
    // ... build input rows
}
``` [1](#0-0) 

`spend_cell` issues an `UPDATE output SET is_spent = 1 WHERE tx_id = ... AND output_index = ...` and returns `false` when `rows_affected == 0` — i.e., when the referenced cell is absent from the `output` table. [2](#0-1) 

A cell is absent from the `output` table in two legitimate, production-reachable scenarios:

1. **`cell_filter` is configured**: the output was filtered out when its producing transaction was indexed.
2. **`block_filter` is configured**: the block that produced the cell was stored only as a simple header entry (via `bulk_insert_blocks_simple`), so none of its outputs were inserted into the `output` table. [3](#0-2) 

When `spend_cell` returns `false` for `input[0]`, the `break` exits the entire loop. `input[1]`, `input[2]`, … are never visited. If any of those later inputs reference cells that **are** present in the `output` table, those cells are never updated to `is_spent = 1`. The corruption is **permanent** — `insert_transaction` is only called once per block during the forward sync pass; there is no subsequent reconciliation.

### Impact Explanation

After the corrupted `append`, the `output` table contains rows for the affected cells with `is_spent = 0`. Every subsequent `get_cells` / `get_cells_capacity` RPC call that queries live cells will return those cells as unspent, even though they were definitively spent in a committed block. Any client or miner that relies on the indexer to determine cell liveness will receive incorrect data, potentially constructing transactions that reference already-spent cells (which consensus will reject). The indexer's state cannot self-heal without a full rollback and re-index.

### Likelihood Explanation

This requires only that:
- The node operator has configured `block_filter` or `cell_filter` (a documented, supported production feature).
- A committed block contains a non-coinbase transaction whose `input[0]` references a cell absent from the indexer DB, while at least one later input references a cell that **is** present.

No attacker action is required; this can occur naturally during normal chain operation with filters enabled. A malicious actor with any mining power (not majority) could deliberately craft such a transaction ordering to guarantee the trigger.

### Recommendation

Replace `break` with `continue` at line 232:

```rust
if !spend_cell(&out_point, tx).await? {
    continue;   // skip unindexed input, but still process remaining inputs
}
``` [4](#0-3) 

### Proof of Concept

1. Configure `cell_filter` to exclude cells with a specific lock script `L_filtered`.
2. Index block A containing tx_A with output[0] locked by `L_filtered` (filtered out) and output[1] locked by `L_indexed` (indexed, `is_spent = 0`).
3. Index block B containing tx_B with:
   - `input[0]` = `OutPoint { tx_hash: tx_A.hash, index: 0 }` (filtered cell, not in `output` table)
   - `input[1]` = `OutPoint { tx_hash: tx_A.hash, index: 1 }` (indexed cell, in `output` table)
4. After `append(block_B)`: query `SELECT is_spent FROM output WHERE tx_id = (SELECT id FROM ckb_transaction WHERE tx_hash = tx_A_hash) AND output_index = 1`.
5. **Observed**: `is_spent = 0`. **Expected**: `is_spent = 1`.
6. Call `get_cells` for `L_indexed` — the spent cell is returned as live. [5](#0-4)

```rust
pub(crate) async fn spend_cell(
    out_point: &OutPoint,
    tx: &mut Transaction<'_, Any>,
) -> Result<bool, Error> {
    let output_tx_hash = out_point.tx_hash().raw_data().to_vec();
    let output_index: u32 = out_point.index().into();

    let updated_rows = sqlx::query(
        r#"
            UPDATE output
            SET is_spent = 1
            WHERE
                tx_id = (SELECT ckb_transaction.id FROM ckb_transaction WHERE tx_hash = $1)
                AND output_index = $2
        "#,
    )
    .bind(output_tx_hash)
    .bind(output_index as i32)
    .execute(tx.as_mut())
    .await
    .map_err(|err| Error::DB(err.to_string()))?
    .rows_affected();

    Ok(updated_rows > 0)
}
