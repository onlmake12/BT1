### Title
Unauthenticated Block Template Injection via Miner HTTP Push Handler — (File: miner/src/client.rs)

### Summary
The miner's HTTP server `handle` function accepts incoming block template updates from any HTTP client without performing any authentication or authorization check. An unprivileged attacker who can reach the miner's HTTP listen port can inject an arbitrary `BlockTemplate`, causing the miner to mine on attacker-controlled content — including a template that redirects coinbase rewards to the attacker's address.

### Finding Description
In `miner/src/client.rs`, the miner runs a local HTTP server to receive pushed block templates from the CKB node. The `handle` function processes every incoming HTTP request unconditionally:

```rust
async fn handle(
    client: Client,
    req: Request<hyper::body::Incoming>,
) -> Result<Response<Empty<Bytes>>, Error> {
    let body = BodyExt::collect(req).await?.aggregate();

    if let Ok(template) = serde_json::from_reader(body.reader()) {
        client.update_block_template(template);
    }

    Ok(Response::new(Empty::new()))
}
``` [1](#0-0) 

There is no check on the request source, no token validation, no IP allowlist enforcement, and no HTTP Basic auth verification on the **incoming** path. The function simply deserializes the body as a `BlockTemplate` and calls `client.update_block_template(template)` — a privileged state mutation — for any caller.

By contrast, the miner does implement Basic auth for its **outgoing** requests to the CKB node RPC via `parse_authorization`:

```rust
fn parse_authorization(url: &Uri) -> Option<HeaderValue> {
    let a: Vec<&str> = url.authority()?.as_str().split('@').collect();
    ...
    let mut encoded = "Basic ".to_string();
    base64::prelude::BASE64_STANDARD.encode_string(a[0], &mut encoded);
    ...
}
``` [2](#0-1) 

This auth logic is applied only to outgoing `POST` requests the miner sends to the node, not to incoming requests the miner's HTTP server receives. The asymmetry means the push-mode HTTP endpoint is entirely open.

### Impact Explanation
An attacker who can send an HTTP POST to the miner's push-mode listen address can:

1. **Redirect coinbase rewards**: Inject a `BlockTemplate` with a modified `cellbase` transaction paying the block reward to the attacker's lock script. The miner will assemble and submit this block, permanently transferring the reward.
2. **Manipulate transaction inclusion**: Force inclusion or exclusion of specific transactions in mined blocks, enabling censorship or front-running.
3. **Waste hashpower on invalid blocks**: Inject a template that will be rejected by the network, causing the miner to expend real PoW work on a block that will never be accepted.

Impact is high: successful exploitation directly and permanently diverts mining revenue with no recovery path once a block is submitted.

### Likelihood Explanation
The miner's HTTP push server is a documented operational mode. If the listen address is bound to anything other than `127.0.0.1` (e.g., `0.0.0.0` for a multi-machine mining setup), the endpoint is reachable by any network peer. Even on localhost, any co-resident process or local user can exploit it. No credentials, keys, or special privileges are required — a single crafted HTTP POST suffices.

### Recommendation
Add authentication to the incoming HTTP handler. The simplest fix is to reuse the same Basic auth credential already configured for outgoing RPC calls: extract the `Authorization` header from the incoming request in `handle` and reject requests that do not present the correct credential before calling `update_block_template`. Alternatively, bind the push server exclusively to `127.0.0.1` and document this as a hard requirement.

fn parse_authorization(url: &Uri) -> Option<HeaderValue> {
    let a: Vec<&str> = url.authority()?.as_str().split('@').collect();
    if a.len() >= 2 {
        if a[0].is_empty() {
            return None;
        }
        let mut encoded = "Basic ".to_string();
        base64::prelude::BASE64_STANDARD.encode_string(a[0], &mut encoded);
        let mut header = HeaderValue::from_str(&encoded).unwrap();
        header.set_sensitive(true);
        Some(header)
    } else {
        None
    }
}
