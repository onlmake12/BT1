### Title
Missing Network Domain Separator in `RawAlert` Signature Scheme Enables Cross-Network Alert Replay - (File: `util/network-alert/src/verifier.rs`)

---

### Summary

The `RawAlert` signature scheme in CKB computes its signing hash as a plain `blake2b_256` of the raw alert bytes with no chain identifier, network name, or any domain separator. Because the same Nervos Foundation signing keys are used across all CKB networks (mainnet, testnet, etc.), a valid alert signature produced for testnet is cryptographically identical to a valid alert signature for mainnet when the `RawAlert` content is the same. An unprivileged attacker who observes a signed testnet alert can replay it against any mainnet node with the Alert RPC enabled, causing the alert to be accepted, stored, and broadcast network-wide via P2P.

---

### Finding Description

**Root cause — `calc_alert_hash` has no domain separator:**

`RawAlertReader::calc_alert_hash` delegates to the generic `CalcHash::calc_hash` trait:

```rust
// util/gen-types/src/extension/calc_hash.rs
impl<'r, R> CalcHash for R where R: Reader<'r> {
    fn calc_hash(&self) -> packed::Byte32 {
        blake2b_256(self.as_slice()).into()   // plain hash of raw bytes, no prefix
    }
}

impl<'r> packed::RawAlertReader<'r> {
    pub fn calc_alert_hash(&self) -> packed::Byte32 {
        self.calc_hash()   // no chain ID, no network name injected
    }
}

**Exploit flow:**

1. The Nervos Foundation legitimately signs a testnet alert (e.g., "CKB v0.105.x has bugs, upgrade to v0.105.1") with 2-of-4 Foundation keys.
2. The attacker observes this alert on the public testnet P2P network — no privileged access needed.
3. The attacker submits the identical `Alert` struct (same `RawAlert` bytes + same signatures) to a mainnet node via `send_alert` RPC.
4. `verify_signatures` computes `blake2b_256(raw_alert_bytes)` — identical to the testnet hash because `RawAlert` contains no chain identifier. Signature verification passes.
5. The mainnet node stores the alert and broadcasts it to all connected peers.
6. Every peer's `AlertRelayer::received` handler re-verifies (passes for the same reason) and re-broadcasts, propagating the false alert across the entire mainnet.

---

### Impact Explanation

All mainnet nodes display a false, replayed alert message. The alert system is the sole mechanism for the Nervos Foundation to warn users about critical bugs. A false alert can cause:

- Node operators to downgrade or upgrade their software unnecessarily, potentially introducing real vulnerabilities.
- Widespread user panic and incorrect operational responses.
- Erosion of trust in the alert system, making future legitimate alerts less effective.

The alert propagates to every connected peer via the P2P `AlertRelayer`, so a single successful injection affects the entire mainnet node population.

---

### Likelihood Explanation

The Nervos Foundation regularly signs testnet alerts (evidenced by hardcoded test signatures in `util/network-alert/src/tests/generate_alert_signature.rs`). Any testnet alert whose `RawAlert` content could plausibly match a mainnet context is replayable. The attacker requires no privileged access: testnet P2P traffic is public, and the `send_alert` RPC is a standard, documented endpoint. The only prerequisite is that the Foundation has signed a testnet alert with content that is meaningful on mainnet — a realistic condition given shared version strings and common message templates. [7](#0-6) 

---

### Recommendation

Include a network-specific domain separator in the alert signing hash. The genesis block hash (already used in `identify_name()` for P2P network identification) is the natural choice:

```rust
// In calc_alert_hash, bind to the chain:
pub fn calc_alert_hash_with_domain(
    &self,
    genesis_hash: &packed::Byte32,
) -> packed::Byte32 {
    let mut hasher = new_blake2b();
    hasher.update(genesis_hash.as_slice());   // domain separator
    hasher.update(self.as_slice());
    let mut result = [0u8; 32];
    hasher.finalize(&mut result);
    result.into()
}
```

Alternatively, prepend a fixed protocol-version tag (e.g., `b"ckb-alert-v1:"`) combined with the genesis hash before hashing. The `Verifier` and `AlertRelayer` must both use the domain-separated hash. Existing in-flight alerts would need re-signing after the upgrade. [8](#0-7)

### Proof of Concept

```
# 1. Observe a signed testnet alert (e.g., from testnet P2P or public explorer):
TESTNET_ALERT = {
  "id": "0x1",
  "cancel": "0x0",
  "priority": "0x1",
  "message": "CKB v0.105.* have bugs. Please upgrade.",
  "notice_until": "0x<future_timestamp>",
  "signatures": ["0x<sig1>", "0x<sig2>"]
}

# 2. Submit unchanged to a mainnet node with Alert RPC enabled:
curl -X POST http://mainnet-node:8114 \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"send_alert","params":[TESTNET_ALERT],"id":1}'

# Result: {"jsonrpc":"2.0","result":null,"id":1}
# The alert is accepted, stored, and broadcast to all mainnet peers.
# All mainnet nodes now display the false alert.
```

The `notice_until` field must be in the future (checked at line 106 of `rpc/src/module/alert.rs`), so the attacker may need to wait for a testnet alert with a sufficiently far future expiry, or the Foundation would need to re-sign with an updated timestamp — but the structural absence of a domain separator is the root cause regardless.     pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
        trace!("Verifying alert {:?}", alert);
        let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
        let signatures: Vec<Signature> = alert
            .signatures()
            .into_iter()
            .filter_map(
                |sig_data| match Signature::from_slice(sig_data.as_reader().raw_data()) {
                    Ok(sig) => {
                        if sig.is_valid() {
                            Some(sig)
                        } else {
                            debug!("invalid signature: {:?}", sig);
                            None
                        }
                    }
                    Err(err) => {
                        debug!("signature error: {}", err);
                        None
                    }
                },
            )
            .collect();
        verify_m_of_n(
            &message,
            self.config.signatures_threshold,
            &signatures,
            &self.pubkeys,
        )
        .map_err(|err| err.kind())?;
        Ok(())
    }
