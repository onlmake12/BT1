### Title
Missing `from == to == local_peer_id` Guard in `ConnectionSyncProcess::execute` Enables Attacker-Triggered Loopback NAT Traversal — (`network/src/protocols/hole_punching/component/connection_sync.rs`)

### Summary

An unprivileged remote peer can send two crafted `HolePunching` messages to cause the victim node to spawn repeated NAT traversal tasks that TCP-connect to the node's own listen address, registering loopback `raw_session`s as inbound connections and consuming connection slots.

### Finding Description

**Step 1 — Poison `pending_delivered[local_id]`**

The attacker sends `ConnectionRequest{from=local_id, to=local_id, listen_addrs=[victim_listen_addr], route=[], max_hops=1}`.

`ConnectionRequestProcess::execute()` has no guard against `from == to` or `from == local_peer_id`. [1](#0-0) 

The only route-based guard (`content.route.contains(self_peer_id)`) passes because the route is empty. [2](#0-1) 

Since `self_peer_id == &content.to` (both equal `local_id`), `respond_delivered(local_id, &local_id, [victim_listen_addr])` is called. [3](#0-2) 

`respond_delivered` accepts any TCP/IP4 or TCP/IP6 address in `remote_listens` (including the node's own listen address) and inserts it:

```
pending_delivered[local_id] = ([victim_listen_addr], now)
``` [4](#0-3) 

**Step 2 — Trigger loopback NAT traversal**

The attacker sends `ConnectionSync{from=local_id, to=local_id, route=[]}`.

`ConnectionSyncProcess::execute()` has no guard against `from == to` or either field equalling `local_peer_id`. [5](#0-4) 

With an empty route, the `None` branch is taken. `self_peer_id != &content.to` evaluates to `false` (both are `local_id`), so the else branch executes and looks up `pending_delivered.get(&content.from)` = `pending_delivered.get(&local_id)`, finding the poisoned entry. [6](#0-5) 

A NAT traversal task is spawned that retries TCP connections to `victim_listen_addr` for up to **30 seconds**: [7](#0-6) 

On success, the stream is registered as an inbound `raw_session`: [8](#0-7) 

### Impact Explanation

- Each successful loopback connection consumes one inbound connection slot.
- The `forward_rate_limiter` key is `(local_id, local_id, ConnectionSync_item_id)` — a fixed tuple — allowing 1 trigger per second. [9](#0-8) 
- Each spawned task runs for up to 30 seconds, so after 30 seconds there can be ~30 concurrent loopback-connecting tasks.
- The `pending_delivered` entry expires after 5 minutes (`TIMEOUT`) but can be refreshed every 2 minutes (`HOLE_PUNCHING_INTERVAL`) by re-sending the `ConnectionRequest`. [10](#0-9) 
- A loopback `raw_session` registered as inbound can confuse the peer registry (the node appears connected to itself) and degrades connectivity to honest peers by occupying slots.

### Likelihood Explanation

The attack requires only a single established P2P session — no special privileges, no PoW, no key material. The two-message sequence is straightforward to craft. The `from` and `to` fields are unauthenticated bytes in the molecule-encoded message. [11](#0-10) 

### Recommendation

Add an early-rejection guard at the top of both `ConnectionSyncProcess::execute` and `ConnectionRequestProcess::execute`:

```rust
if content.from == content.to
    || &content.from == self_peer_id
    || &content.to == self_peer_id
{
    return StatusCode::Ignore.with_context("invalid from/to peer id");
}
```

This should be checked **before** any `pending_delivered` lookup or rate-limiter keying.

### Proof of Concept

```
1. Attacker connects to victim (establishes session S).
2. Attacker sends over S:
     ConnectionRequest {
       from = victim_local_peer_id,
       to   = victim_local_peer_id,
       listen_addrs = [/ip4/<victim_ip>/tcp/<victim_port>],
       route = [],
       max_hops = 1,
     }
   → victim inserts pending_delivered[local_id] = ([victim_listen_addr], now)

3. Attacker sends over S (once per second):
     ConnectionSync {
       from  = victim_local_peer_id,
       to    = victim_local_peer_id,
       route = [],
     }
   → victim spawns try_nat_traversal → TCP connects to itself
   → on success: raw_session(loopback_stream, victim_addr, RawSessionInfo::inbound(...))

4. Assert: victim's inbound connection count increments; peer registry contains
   a self-referential entry.
```

pub(crate) async fn try_nat_traversal(
    bind_addr: Option<SocketAddr>,
    addr: Multiaddr,
) -> Result<(TcpStream, Multiaddr), std::io::Error> {
    let net_addr = match multiaddr_to_socketaddr(&addr) {
        Some(addr) => addr,
        None => {
            debug!("Failed to convert multiaddr to socketaddr");
            return Err(std::io::ErrorKind::InvalidInput.into());
        }
    };

    // Use a fixed interval but add a small amount of randomness
    let base_retry_interval = Duration::from_millis(200);

    // total time
    let timeout_duration = Duration::from_secs(30);
    let start_time = Instant::now();
    let mut retry_count = 0u32;
    while start_time.elapsed() < timeout_duration {
        retry_count += 1;

        // Add a small amount of random jitter (±25ms) to avoid conflicts
        // caused by continuous precise synchronization
        let jitter = Duration::from_millis(rand::random::<u64>() % 50);
        let actual_interval = if rand::random::<bool>() {
            base_retry_interval + jitter
        } else {
            base_retry_interval.saturating_sub(jitter)
        };

        let socket = create_socket(bind_addr, net_addr)?;

        match runtime::timeout(
            std::time::Duration::from_millis(200),
            socket.connect(net_addr),
        )
        .await
        {
            Ok(Ok(stream)) => {
                // try get the stored error in the underlying socket
                // if the socket is not connected, it will return an error
                if let Err(err) = check_connection(&stream) {
                    debug!("Failed to connect to NAT(base check): {}", err);
                }
                return Ok((stream, addr));
            }
            Err(err) => {
                debug!("Failed to connect to NAT(timeout): {}", err);
            }
            Ok(Err(err)) => {
                if err.kind() == std::io::ErrorKind::AddrNotAvailable {
                    return Err(err);
                }
                debug!(
                    "Failed to connect to NAT(other error): {}, {}",
                    err.kind(),
                    err
                );
            }
        }
        runtime::delay_for(actual_interval).await;
    }

    debug!("Failed to connect to NAT after {} retries", retry_count);
    Err(std::io::ErrorKind::TimedOut.into())
}
