### Title
Uncle Block Timestamps Are Completely Unvalidated, Allowing Miners to Inject Arbitrary Timestamps into On-Chain Headers - (File: `verification/contextual/src/uncles_verifier.rs`)

### Summary

CKB enforces a strict median-time-based timestamp rule on main chain block headers, but applies **no timestamp validation whatsoever** to uncle block headers. A miner can submit a block containing an uncle with `timestamp = 0` or `timestamp = u64::MAX`. Because uncle headers are stored on-chain and are reachable via `header_deps` in transactions, scripts that read uncle block timestamps via the `ckb_load_header` syscall receive a fully miner-controlled value, bypassing all consensus timestamp guarantees.

### Finding Description

CKB's `TimestampVerifier` in `verification/src/header_verifier.rs` enforces that every main chain block header's timestamp must be strictly greater than the median of the previous 37 blocks, and must not exceed `now + ALLOWED_FUTURE_BLOCKTIME` (15 seconds): [1](#0-0) 

This verifier is invoked for main chain headers in `HeaderVerifier::verify()`: [2](#0-1) 

However, the contextual `UnclesVerifier::verify()` in `verification/contextual/src/uncles_verifier.rs` checks compact target, epoch number, block number, descendant relationship, duplicate inclusion, PoW, and proposals hash — but **never calls `TimestampVerifier` or any equivalent check on the uncle's timestamp**: [3](#0-2) 

The full list of uncle checks (lines 72–128) contains no timestamp validation. An uncle with `timestamp = 0` or `timestamp = u64::MAX` passes all checks as long as its PoW is valid.

Uncle blocks are a first-class part of the CKB block structure: [4](#0-3) 

They are stored on-chain and are reachable by transactions via `header_deps`. Scripts can read the full uncle header — including its timestamp — through the `ckb_load_header` syscall.

The `submit_block` RPC entry point in `rpc/src/module/miner.rs` runs `HeaderVerifier` only on the main block header, then forwards to the chain controller which runs `UnclesVerifier` without any timestamp check on the embedded uncles: [5](#0-4) 

### Impact Explanation

Any smart contract that references an uncle block via `header_deps` and reads its timestamp (e.g., for a time-lock, oracle, or deadline check) receives a fully miner-controlled value. A miner can:

- Set `uncle.timestamp = u64::MAX` to make any `current_time < uncle_timestamp` check always pass, bypassing future-time guards.
- Set `uncle.timestamp = 0` to make any `uncle_timestamp > deadline` check always fail, permanently blocking time-locked funds.
- Set an arbitrary past or future timestamp to manipulate DeFi contract logic that uses `header_deps`-sourced timestamps.

This breaks the invariant that on-chain block timestamps have been validated by consensus, which is the foundational assumption for any script using `header_deps` for time-based logic.

### Likelihood Explanation

Any miner — even one with a small fraction of hashrate — can occasionally mine a valid uncle block (with valid Eaglesong PoW) and a main chain block that includes it. No 51% attack is required. The attacker only needs to:

1. Mine an uncle block with an arbitrary timestamp and valid PoW.
2. Mine a main chain block that includes that uncle (satisfying the epoch, number, and descendant rules).
3. Submit via `submit_block` RPC.

This is achievable by any participant operating a mining node, making it a realistic threat for any deployed contract that uses uncle `header_deps` timestamps.

### Recommendation

Apply `TimestampVerifier` (or an equivalent bounds check) to each uncle header inside `UnclesVerifier::verify()`. At minimum, enforce that each uncle's timestamp:

1. Is greater than the median time of the uncle's own parent chain (same rule as main chain blocks).
2. Does not exceed `now + ALLOWED_FUTURE_BLOCKTIME`.

This mirrors the fix applied in the Etherlink report (MR 13807/13832): enforce that all timestamp sources — not just the primary block — satisfy the same monotonicity and bounds constraints.

### Proof of Concept

1. Construct an uncle block with `timestamp = u64::MAX` and valid Eaglesong PoW for the current epoch.
2. Construct a main chain block whose parent is on the canonical chain, embedding the uncle (satisfying `uncle.number < block.number`, same epoch, valid descendant relationship).
3. Call `submit_block` via RPC. The block passes all validation:
   - Main header passes `TimestampVerifier`.
   - Uncle passes `UnclesVerifier` (PoW valid, epoch matches, number valid, descendant valid, not duplicate).
4. The uncle is now stored on-chain with `timestamp = u64::MAX`.
5. Deploy a contract whose `header_deps` reference this uncle's hash and whose logic branches on `uncle_header.timestamp > some_threshold`. The miner-controlled `u64::MAX` timestamp causes the branch to always be taken, regardless of actual wall-clock time.

The root cause is the absence of any timestamp check in `UnclesVerifier::verify()` at `verification/contextual/src/uncles_verifier.rs` lines 43–134, while `TimestampVerifier` at `verification/src/header_verifier.rs` lines 70–96 is only applied to main chain headers.

```rust
    type Target = HeaderView;
    fn verify(&self, header: &Self::Target) -> Result<(), Error> {
        // POW check first
        PowVerifier::new(header, self.consensus.pow_engine().as_ref()).verify()?;
        let parent_fields = self
            .data_loader
            .get_header_fields(&header.parent_hash())
            .ok_or_else(|| UnknownParentError {
                parent_hash: header.parent_hash(),
            })?;
        NumberVerifier::new(parent_fields.number, header).verify()?;
        EpochVerifier::new(parent_fields.epoch, header).verify()?;
        TimestampVerifier::new(
            self.data_loader,
            header,
            self.consensus.median_time_block_count(),
        )
        .verify()?;
        Ok(())
    }
}
```

```rust
    pub fn verify(&self) -> Result<(), Error> {
        // skip genesis block
        if self.header.is_genesis() {
            return Ok(());
        }

        let min = self.data_loader.block_median_time(
            &self.header.data().raw().parent_hash(),
            self.median_block_count,
        );
        if self.header.timestamp() <= min {
            return Err(TimestampError::BlockTimeTooOld {
                min,
                actual: self.header.timestamp(),
            }
            .into());
        }
        let max = self.now + ALLOWED_FUTURE_BLOCKTIME;
        if self.header.timestamp() > max {
            return Err(TimestampError::BlockTimeTooNew {
                max,
                actual: self.header.timestamp(),
            }
            .into());
        }
        Ok(())
    }
```

```rust
    pub fn verify(&self) -> Result<(), Error> {
        let uncles_count = self.block.data().uncles().len() as u32;

        // if self.block.uncles is empty, return
        if uncles_count == 0 {
            return Ok(());
        }

        // if block is genesis, which is expected with zero uncles, return error
        if self.block.is_genesis() {
            return Err(UnclesError::OverCount {
                max: 0,
                actual: uncles_count,
            }
            .into());
        }

        // verify uncles length =< max_uncles_num
        let max_uncles_num = self.provider.consensus().max_uncles_num() as u32;
        if uncles_count > max_uncles_num {
            return Err(UnclesError::OverCount {
                max: max_uncles_num,
                actual: uncles_count,
            }
            .into());
        }

        let mut included: HashMap<Byte32, BlockNumber> = HashMap::default();
        for uncle in self.block.uncles().into_iter() {
            if uncle.compact_target() != self.provider.epoch().compact_target() {
                return Err(UnclesError::InvalidTarget.into());
            }

            if self.provider.epoch().number() != uncle.epoch().number() {
                return Err((UnclesError::InvalidDifficultyEpoch).into());
            }

            if uncle.number() >= self.block.number() {
                return Err((UnclesError::InvalidNumber).into());
            }

            let embedded_descendant = included
                .get(&uncle.data().header().raw().parent_hash())
                .map(|number| (number + 1) == uncle.number())
                .unwrap_or(false);

            if !(embedded_descendant || self.provider.descendant(&uncle.header())) {
                return Err((UnclesError::DescendantLimit).into());
            }

            if included.contains_key(&uncle.hash()) {
                return Err((UnclesError::Duplicate(uncle.hash())).into());
            }

            if self.provider.double_inclusion(&uncle.hash()) {
                return Err((UnclesError::DoubleInclusion(uncle.hash())).into());
            }

            if uncle.data().proposals().len()
                > self.provider.consensus().max_block_proposals_limit() as usize
            {
                return Err((UnclesError::ExceededMaximumProposalsLimit).into());
            }

            if uncle.proposals_hash() != uncle.data().as_reader().calc_proposals_hash() {
                return Err((UnclesError::ProposalsHash).into());
            }

            let mut seen = HashSet::with_capacity(uncle.data().proposals().len());
            if !uncle
                .data()
                .proposals()
                .into_iter()
                .all(|id| seen.insert(id))
            {
                return Err((UnclesError::ProposalDuplicate).into());
            }

            if !self
                .provider
                .consensus()
                .pow_engine()
                .verify(&uncle.data().header())
            {
                return Err((PowError::InvalidNonce).into());
            }

            included.insert(uncle.hash(), uncle.number());
        }

        Ok(())
    }
```
