### Title
Unbounded DB Read Amplification via Unauthenticated `GetBlockFilterHashes` — (`sync/src/filter/get_block_filter_hashes_process.rs`)

### Summary

`GetBlockFilterHashesProcess::execute` performs up to 4002 DB reads per message with no per-peer rate limiting, no ban for valid messages, and no backpressure. Any connected peer can send `GetBlockFilterHashes(start_number=1)` in a tight loop, causing unbounded DB read amplification at the victim node.

### Finding Description

In `execute`, when `latest >= start_number` (satisfied as soon as the node has built filter data for ≥1 block):

1. **Parent lookup**: `get_block_hash(start_number - 1)` + `get_block_filter_hash(...)` → 2 DB reads
2. **Batch loop** (`BATCH_SIZE = 2000`): up to 2000 × (`get_block_hash` + `get_block_filter_hash`) → up to 4000 DB reads

Total: **up to 4002 DB reads per message**. [1](#0-0) [2](#0-1) 

The `received` handler in `BlockFilter` dispatches directly to `process` → `try_process` → `GetBlockFilterHashesProcess::execute` with no rate-limiting gate: [3](#0-2) 

The only ban path is for **malformed** messages (unparseable bytes). A well-formed `GetBlockFilterHashes` with any `start_number` never triggers `status.should_ban()`: [4](#0-3) 

No rate-limiting, token bucket, quota, or per-peer counter exists anywhere in `sync/src/filter/`:



### Impact Explanation

- **Amplification ratio**: ~4002 DB reads per small P2P message (a few bytes).
- **Attacker cost**: negligible — send a tight loop of identical small messages.
- **Victim cost**: saturated RocksDB read I/O, starving block processing, header sync, and transaction relay.
- A node under this attack falls behind chain tip, degrading its participation in block propagation and effectively causing local network congestion.

### Likelihood Explanation

- Precondition (node has built ≥1 filter block) is trivially met on mainnet.
- Any peer that completes the P2P handshake can send filter protocol messages — no privilege required.
- The attack is repeatable indefinitely from a single peer with no ban or backpressure applied.

### Recommendation

1. Add a **per-peer rate limit** (e.g., token bucket) on `GetBlockFilterHashes` processing in `BlockFilter::received` or `try_process`.
2. Return a ban-worthy `Status` (or at minimum a `Status::ignored()` with a cooldown) when a peer exceeds the allowed request rate.
3. Consider capping the response or adding a minimum interval between serving filter hash batches to the same peer.

### Proof of Concept

```
# Pseudocode
peer.connect(victim_node)
loop:
    peer.send(GetBlockFilterHashes { start_number: 1 })
    # No sleep needed — no ban, no backpressure
    # Victim performs 4002 DB reads per iteration
    # Victim sends back BlockFilterHashes(2000 hashes) — attacker drops it
```

Assert: victim node's RocksDB read IOPS scales linearly with message send rate; no ban event is logged; no backpressure is applied. [5](#0-4)

    pub async fn execute(self) -> Status {
        let active_chain = self.filter.shared.active_chain();
        let start_number: BlockNumber = self.message.to_entity().start_number().into();
        let latest: BlockNumber = active_chain.get_latest_built_filter_block_number();

        let mut block_filter_hashes = Vec::new();

        if latest >= start_number {
            let parent_block_filter_hash = if start_number > 0 {
                match active_chain
                    .get_block_hash(start_number - 1)
                    .and_then(|block_hash| active_chain.get_block_filter_hash(&block_hash))
                {
                    Some(parent_block_filter_hash) => parent_block_filter_hash,
                    None => return Status::ignored(),
                }
            } else {
                packed::Byte32::zero()
            };

            let mut block_number = start_number;
            for _ in 0..BATCH_SIZE {
                if let Some(block_filter_hash) = active_chain
                    .get_block_hash(block_number)
                    .and_then(|block_hash| active_chain.get_block_filter_hash(&block_hash))
                {
                    block_filter_hashes.push(block_filter_hash);
                } else {
                    break;
                }
                let Some(next_block_number) = block_number.checked_add(1) else {
                    break;
                };
                block_number = next_block_number;
            }
            let content = packed::BlockFilterHashes::new_builder()
                .start_number(start_number)
                .parent_block_filter_hash(parent_block_filter_hash)
                .block_filter_hashes(block_filter_hashes)
                .build();

            let message = packed::BlockFilterMessage::new_builder()
                .set(content)
                .build();
            async_send_message_to(&self.nc, self.peer, &message).await
        } else {
            Status::ignored()
        }
    }
