### Title
Rich-Indexer `insert_transaction` Unconditional `break` Leaves Subsequent Inputs Unspent When Cell Filter Is Active — (`util/rich-indexer/src/indexer/mod.rs`)

---

### Summary

When a `cell_filter` is configured, the `insert_transaction` function uses an unconditional `break` when `spend_cell` returns `false` for any input. Because `spend_cell` returns `false` whenever the referenced cell is absent from the indexer's `output` table (which is the normal case for cells that don't match the filter), any multi-input transaction whose first input references a non-indexed cell will silently skip marking all subsequent inputs as spent — even if those subsequent inputs reference cells that ARE indexed. The result is persistent phantom live-cell state in the DB.

---

### Finding Description

In `insert_transaction`, the input-processing loop is:

```rust
if tx_index != 0 {
    for (input_index, input) in tx_view.inputs().into_iter().enumerate() {
        let out_point = input.previous_output();
        if !spend_cell(&out_point, tx).await? {
            break;          // ← line 232
        }
        // ... build input_rows, set is_tx_matched
    }
}
``` [1](#0-0) 

`spend_cell` issues an `UPDATE output SET is_spent = 1 WHERE ...` and returns `Ok(rows_affected > 0)`. [2](#0-1) 

It returns `false` whenever the cell is **not present** in the `output` table — the normal outcome for any cell that was filtered out by the `cell_filter`. The `break` then exits the entire loop, so every input after the first non-indexed one is never visited. Those later inputs may reference cells that **are** indexed; their `is_spent` column stays `0`.

The correct fix is `continue` (skip this input, process the rest), not `break`.

---

### Impact Explanation

`get_cells` queries the `output` table with `output.is_spent = 0` to return live cells: [3](#0-2) 

Any cell whose `is_spent` was never flipped to `1` due to the premature `break` will be returned as a live cell indefinitely — surviving node restarts because the corruption is written to the persistent DB. `get_cells_capacity` is similarly affected. Downstream wallets or dApps querying the indexer will see phantom unspent cells and may attempt to re-spend them (failing at consensus) or compute incorrect balances.

---

### Likelihood Explanation

The precondition is that `cell_filter` is active (a supported, documented operator configuration). Once it is, any valid block containing a multi-input transaction where the first input's cell is not indexed (e.g., it was created before the indexer started, or it belongs to a different script type than the filter) triggers the bug. No special attacker capability is required beyond submitting or relaying a valid block — a normal P2P block relay path.

---

### Recommendation

Replace `break` with `continue` at line 232:

```rust
if !spend_cell(&out_point, tx).await? {
    continue;   // cell not in indexer — skip, but process remaining inputs
}
``` [4](#0-3) 

This preserves the existing semantics (non-indexed cells are silently skipped) while ensuring all inputs are visited so that indexed cells are correctly marked spent.

---

### Proof of Concept

1. Start a CKB node with the rich indexer and a `cell_filter` that matches only cells with script `S`.
2. Index a block containing tx `T1` that creates two outputs: output[0] with script `S` (indexed), output[1] with script `S` (indexed).
3. Construct tx `T2` with two inputs: input[0] spending a cell with script `X` (not matching the filter, so not in the `output` table), input[1] spending `T1:output[0]` (indexed, `is_spent=0`).
4. Submit a valid block containing `T2`.
5. After `append`, query the DB: `SELECT is_spent FROM output WHERE ...` for `T1:output[0]`.
6. **Expected**: `is_spent = 1`. **Actual**: `is_spent = 0` — the cell appears live.
7. Call `get_cells` for script `S`; `T1:output[0]` is returned as a live cell despite being consumed.

pub(crate) async fn spend_cell(
    out_point: &OutPoint,
    tx: &mut Transaction<'_, Any>,
) -> Result<bool, Error> {
    let output_tx_hash = out_point.tx_hash().raw_data().to_vec();
    let output_index: u32 = out_point.index().into();

    let updated_rows = sqlx::query(
        r#"
            UPDATE output
            SET is_spent = 1
            WHERE
                tx_id = (SELECT ckb_transaction.id FROM ckb_transaction WHERE tx_hash = $1)
                AND output_index = $2
        "#,
    )
    .bind(output_tx_hash)
    .bind(output_index as i32)
    .execute(tx.as_mut())
    .await
    .map_err(|err| Error::DB(err.to_string()))?
    .rows_affected();

    Ok(updated_rows > 0)
