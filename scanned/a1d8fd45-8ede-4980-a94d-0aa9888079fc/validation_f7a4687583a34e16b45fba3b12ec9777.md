### Title
Unvalidated `observed_addr` from Remote Peer Injected into Node's Advertised Address List — (`network/src/protocols/identify/mod.rs`)

### Summary

An established remote peer can send an `IdentifyMessage` with an arbitrary `observed_addr` Multiaddr. The address is stored in `NetworkState::observed_addrs` without any reachability or format validation, and is subsequently included in the node's own advertised listen addresses sent to every new peer it connects to.

### Finding Description

**Asymmetric validation between `process_listens` and `process_observed`:**

`process_listens` applies both a count limit and a reachability filter before storing remote listen addresses: [1](#0-0) 

`process_observed` applies **no validation at all** — it passes the raw attacker-controlled `Multiaddr` directly to `add_observed_addr`: [2](#0-1) 

`IdentifyCallback::add_observed_addr` only appends a P2P peer ID if missing, then stores the address unconditionally: [3](#0-2) 

**Re-advertisement path:**

`local_listen_addrs` fills remaining slots (up to `MAX_RETURN_LISTEN_ADDRS = 10`) with observed addresses when the node has fewer than 10 public addresses — common for NAT'd nodes: [4](#0-3) 

When the node connects to a new peer, it sends these addresses (with a reachability filter applied at send time): [5](#0-4) 

The send-time filter uses `is_reachable`, which passes **globally-routable IPs**. An attacker who sends a globally-routable but attacker-controlled IP as `observed_addr` will have it:
1. Stored without validation
2. Included in `local_listen_addrs()`
3. Pass the `is_reachable` filter at send time
4. Be propagated to every new peer the victim node connects to

**Memory growth:** The `observed_addrs` HashMap is keyed by `session_id`, so growth is bounded by the number of concurrent sessions — not truly unbounded, but each connected peer can inject one address.

### Impact Explanation

For nodes with fewer than 10 public addresses (NAT'd nodes, which are common), an attacker with a single established session can inject a globally-routable attacker-controlled IP into the node's advertised address list. Other peers receiving this address may attempt to connect to the attacker's infrastructure instead of the real node, enabling:
- Partial eclipse attack (redirecting some inbound connections)
- Network topology manipulation

### Likelihood Explanation

- Requires only an established P2P session — no special privileges
- The identify protocol is open by default
- NAT'd nodes (the majority of home/VPS nodes) are affected since they typically have fewer than 10 public addresses
- One attacker-controlled address per session is injected; multiple sessions multiply the effect

### Recommendation

Apply the same `is_reachable` filter in `process_observed` before calling `add_observed_addr`, mirroring the logic in `process_listens`: [6](#0-5) 

Additionally, validate that the observed address matches the node's actual outbound IP (e.g., cross-check against the session's local socket address) before storing it.

### Proof of Concept

1. Establish a P2P session with a target CKB node
2. Send an `IdentifyMessage` with `observed_addr` set to a globally-routable attacker-controlled IP (e.g., `1.2.3.4`) encoded as a valid Multiaddr
3. The address passes `Multiaddr::try_from` parsing in `decode` and is stored via `add_observed_addr`
4. When the victim node connects to any new peer, `local_listen_addrs()` includes `1.2.3.4`, which passes `is_reachable` and is sent to the new peer
5. The new peer now believes `1.2.3.4` is a valid address for the victim node [7](#0-6)

    pub(crate) fn decode(data: &'a [u8]) -> Option<Self> {
        let reader = packed::IdentifyMessageReader::from_compatible_slice(data).ok()?;

        let identify = reader.identify().raw_data();
        let observed_addr =
            Multiaddr::try_from(reader.observed_addr().bytes().raw_data().to_vec()).ok()?;
        let mut listen_addrs = Vec::with_capacity(reader.listen_addrs().len());
        for addr in reader.listen_addrs().iter() {
            match Multiaddr::try_from(addr.bytes().raw_data().to_vec()) {
                Ok(multi_addr) => {
                    listen_addrs.push(multi_addr);
                }
                Err(err) => warn!("failed to decode listen_addr to Multiaddr: {}", err),
            }
        }

        Some(IdentifyMessage {
            identify,
            observed_addr,
            listen_addrs,
        })
    }
