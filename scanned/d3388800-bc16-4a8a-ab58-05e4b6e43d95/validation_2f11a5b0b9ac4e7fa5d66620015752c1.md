### Title
Expired Network Alerts Accepted and Re-Broadcast Without `notice_until` Expiration Check in P2P Relay Path - (File: `util/network-alert/src/alert_relayer.rs`)

### Summary

The CKB P2P alert relay handler (`AlertRelayer::received`) accepts and re-broadcasts signed network alerts from peers without verifying that the alert's `notice_until` timestamp has not already passed. Any unprivileged P2P peer can replay a previously valid, now-expired alert (with cryptographically intact Nervos Foundation signatures) to cause receiving nodes to display stale emergency messages to operators and re-flood the network with expired alert traffic.

### Finding Description

CKB's network alert system uses secp256k1 multi-signatures (2-of-4 hardcoded Nervos Foundation public keys) to authenticate `Alert` messages broadcast over P2P. Each alert carries a `notice_until` field (a millisecond UTC timestamp) that marks when the alert expires. This field is part of the signed `RawAlert` payload, so it is cryptographically bound to the signature.

**The RPC entry point correctly enforces expiration:**

In `rpc/src/module/alert.rs`, `send_alert` checks:

```rust
let now_ms = ckb_systemtime::unix_time_as_millis();
let notice_until: u64 = alert.raw().notice_until().into();
if notice_until < now_ms {
    return Err(RPCError::invalid_params(...));
}
``` [1](#0-0) 

**The P2P relay path does NOT enforce expiration:**

In `util/network-alert/src/alert_relayer.rs`, the `received()` handler processes inbound P2P alert messages. After parsing and deduplication, it calls only `verify_signatures` (cryptographic check) and then immediately broadcasts and stores the alert — with no check against the current time:

```rust
if let Err(err) = self.verifier.verify_signatures(&alert) { ... return; }
// mark sender as known
self.mark_as_known(peer_index, alert_id);
// broadcast message
nc.quick_filter_broadcast(..., data);
// add to received alerts
self.notifier.lock().add(&alert);
``` [2](#0-1) 

`Verifier::verify_signatures` only checks cryptographic validity against the hardcoded public keys; it performs no temporal check: [3](#0-2) 

`Notifier::add` also performs no expiration check before calling `notify_controller.notify_network_alert(alert.clone())`, which surfaces the alert to the node operator: [4](#0-3) 

`clear_expired_alerts` is only invoked in the `connected()` handler (when a new peer connects), not in `received()`: [5](#0-4) 

This means that between peer connection events, any expired alert received over P2P is fully accepted, user-notified, and re-broadcast.

### Impact Explanation

An unprivileged P2P peer can capture any previously broadcast CKB network alert (all past alerts are public, their bytes are observable on the network) and replay it after its `notice_until` has expired. The receiving node will:

1. Accept the alert as valid (signatures are still cryptographically correct).
2. Trigger `notify_controller.notify_network_alert(...)`, surfacing a stale emergency message to the node operator.
3. Re-broadcast the expired alert to all connected peers, causing network-wide propagation of stale alert traffic.

This allows an attacker to cause false alarms among node operators (e.g., replaying an old "critical bug, upgrade immediately" alert) and to generate unnecessary P2P traffic. The impact is integrity of the alert system and potential operator confusion leading to incorrect operational decisions.

### Likelihood Explanation

Likelihood is **medium**. The attacker needs only a previously broadcast alert (publicly observable on the network) and a P2P connection to a target node — both are trivially achievable by any unprivileged peer. No key material, privileged access, or majority hashpower is required. The only constraint is that at least one real alert must have been broadcast historically, which has already occurred (e.g., alert `20230001` is hardcoded in tests). [6](#0-5) 

### Recommendation

In `AlertRelayer::received()`, add an expiration check immediately after signature verification and before broadcasting or storing the alert:

```rust
let now_ms = ckb_systemtime::unix_time_as_millis();
let notice_until: u64 = alert.raw().notice_until().into();
if notice_until < now_ms {
    debug!("Received expired alert {} from peer {}", alert_id, peer_index);
    return;
}
```

Additionally, `Notifier::add()` should guard against expired alerts being stored or notified, mirroring the existing `clear_expired_alerts` logic as a pre-condition on insertion.

### Proof of Concept

1. Observe any previously broadcast CKB alert on the P2P network (or use the publicly known alert `20230001` with its known signatures from the test suite).
2. After the alert's `notice_until` timestamp has passed, establish a P2P connection to a target CKB node.
3. Send the raw alert bytes directly over the Alert protocol channel.
4. The target node's `AlertRelayer::received()` will:
   - Parse the alert successfully.
   - Pass the `has_received` dedup check (if the node restarted or never saw it).
   - Pass `verify_signatures` (signatures remain cryptographically valid).
   - Call `notifier.lock().add(&alert)` → `notify_controller.notify_network_alert(...)` → operator sees the expired alert.
   - Broadcast the expired alert to all connected peers.

The `notice_until` field is never compared to the current time anywhere in the P2P receive path. [7](#0-6)

    async fn received(
        &mut self,
        nc: Arc<dyn CKBProtocolContext + Sync>,
        peer_index: PeerIndex,
        data: Bytes,
    ) {
        let alert: packed::Alert = match packed::AlertReader::from_slice(&data) {
            Ok(alert) => {
                if alert.raw().message().is_utf8()
                    && alert
                        .raw()
                        .min_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                    && alert
                        .raw()
                        .max_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                {
                    alert.to_entity()
                } else {
                    info!(
                        "A malformed message fromP peer {} : not utf-8 string",
                        peer_index
                    );
                    nc.ban_peer(
                        peer_index,
                        BAD_MESSAGE_BAN_TIME,
                        String::from("send us a malformed message: not utf-8 string"),
                    );
                    return;
                }
            }
            Err(err) => {
                info!("A malformed message from peer {}: {:?}", peer_index, err);
                nc.ban_peer(
                    peer_index,
                    BAD_MESSAGE_BAN_TIME,
                    String::from("send us a malformed message"),
                );
                return;
            }
        };
        let alert_id = alert.as_reader().raw().id().into();
        trace!("ReceiveD alert {} from peer {}", alert_id, peer_index);
        // ignore alert
        if self.notifier.lock().has_received(alert_id) {
            return;
        }
        // verify
        if let Err(err) = self.verifier.verify_signatures(&alert) {
            debug!(
                "An alert from peer {} with invalid signatures, error {:?}",
                peer_index, err
            );
            nc.ban_peer(
                peer_index,
                BAD_MESSAGE_BAN_TIME,
                String::from("send us an alert with invalid signatures"),
            );
            return;
        }
        // mark sender as known
        self.mark_as_known(peer_index, alert_id);
        // broadcast message
        let selected_peers: Vec<PeerIndex> = nc
            .connected_peers()
            .into_iter()
            .filter(|peer| self.mark_as_known(*peer, alert_id))
            .collect();
        if let Err(err) = nc.quick_filter_broadcast(
            TargetSession::Multi(Box::new(selected_peers.into_iter())),
            data,
        ) {
            debug!("alert broadcast error: {:?}", err);
        }
        // add to received alerts
        self.notifier.lock().add(&alert);
    }

    pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
        trace!("Verifying alert {:?}", alert);
        let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
        let signatures: Vec<Signature> = alert
            .signatures()
            .into_iter()
            .filter_map(
                |sig_data| match Signature::from_slice(sig_data.as_reader().raw_data()) {
                    Ok(sig) => {
                        if sig.is_valid() {
                            Some(sig)
                        } else {
                            debug!("invalid signature: {:?}", sig);
                            None
                        }
                    }
                    Err(err) => {
                        debug!("signature error: {}", err);
                        None
                    }
                },
            )
            .collect();
        verify_m_of_n(
            &message,
            self.config.signatures_threshold,
            &signatures,
            &self.pubkeys,
        )
        .map_err(|err| err.kind())?;
        Ok(())
    }

    pub fn add(&mut self, alert: &Alert) {
        let alert_id = alert.raw().id().into();
        let alert_cancel = alert.raw().cancel().into();
        if self.has_received(alert_id) {
            return;
        }
        // checkout cancel_id
        if alert_cancel > 0 {
            self.cancel(alert_cancel);
        }
        // add to received alerts
        self.received_alerts.insert(alert_id, alert.clone());

        // check conditions, figure out do we need to notice this alert
        if !self.is_version_effective(alert) {
            debug!("Received a version ineffective alert {:?}", alert);
            return;
        }

        if self.noticed_alerts.contains(alert) {
            return;
        }
        self.notify_controller.notify_network_alert(alert.clone());
        self.noticed_alerts.push(alert.clone());
        // sort by priority
        self.noticed_alerts.sort_by_key(|a| {
            let priority: u32 = a.raw().priority().into();
            u32::MAX - priority
        });
    }
