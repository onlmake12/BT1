### Title
Unauthenticated `from=to=self` `ConnectionRequest` Poisons `pending_delivered`, Forcing Outbound TCP Connections to Attacker-Controlled Addresses — (`network/src/protocols/hole_punching/component/connection_request.rs`)

### Summary
An unprivileged remote peer can send a crafted `ConnectionRequest` with `from = to = victim_node_peer_id` and attacker-controlled `listen_addrs`. Because no check enforces `from ≠ to` or that `from` matches the actual sender's peer ID, the victim node stores the attacker's TCP addresses in `pending_delivered` keyed by its own peer ID. A subsequent `ConnectionSync` with `from = to = victim_peer_id` and empty route causes the node to initiate outbound TCP connections to those attacker-controlled endpoints, and on success, establishes a full P2P session with the attacker's server.

### Finding Description
The hole-punching protocol's `ConnectionRequestProcess::execute` checks `self_peer_id == &content.to` to decide whether to act as the "destination" of a hole-punch request. When an attacker sets both `from` and `to` to the victim's own `PeerId`, this check passes and `respond_delivered` is called. There is no guard requiring `from ≠ to`, and no guard requiring `from` to equal the actual sender's authenticated peer ID.

Inside `respond_delivered`, the attacker-supplied `listen_addrs` are filtered only for transport type (TCP + IPv4/IPv6) and then stored verbatim in `pending_delivered[local_peer_id]`. The address validation only checks that any embedded P2P component matches `from` — since `from = local_peer_id`, the attacker can supply bare TCP addresses and the code appends `local_peer_id` automatically, passing all checks.

When the attacker then sends a `ConnectionSync` with `from = to = local_peer_id` and an empty `route`, `ConnectionSyncProcess::execute` reaches the branch where `self_peer_id == &content.to`, looks up `pending_delivered[local_peer_id]`, finds the attacker's addresses, and spawns `try_nat_traversal` tasks that retry TCP `connect()` to those addresses for up to 30 seconds. On success, `control.raw_session(...)` is called, establishing a live P2P session with the attacker's server.

### Impact Explanation
- The victim node initiates outbound TCP connections to arbitrary attacker-controlled IP:port endpoints.
- On a successful TCP handshake, a full P2P session is established with the attacker's server, revealing the victim's real IP and allowing the attacker to inject P2P protocol messages.
- This bypasses any inbound firewall rules since the connection is outbound from the victim.
- The attack requires only a single connected P2P peer slot (unprivileged) and two crafted messages.

### Likelihood Explanation
Any peer connected to the victim node can execute this with two messages. The victim's `PeerId` is publicly advertised on the P2P network. The `forward_rate_limiter` is keyed by `(from, to, item_id)` — since `from = to = local_peer_id` and `item_id` is fixed per message type, the rate limit is 1 attempt per second, which is sufficient for a persistent attack. No special privileges, keys, or majority hashpower are required.

### Recommendation
1. **Reject `from == to`**: Add an explicit check in `ConnectionRequestProcess::execute` that returns an error if `content.from == content.to`.
2. **Authenticate `from`**: Verify that `content.from` matches the actual sender's authenticated peer ID (available from the session context), so a peer cannot impersonate another peer ID in the `from` field.
3. **Validate `from ≠ self`**: The node should also reject any `ConnectionRequest` where `content.from == self_peer_id`, since a legitimate request would never originate from the node itself.

### Proof of Concept
```
1. Attacker connects to victim node as a normal P2P peer.
2. Attacker learns victim's PeerId (publicly advertised).
3. Attacker sends ConnectionRequest:
     from       = victim_peer_id
     to         = victim_peer_id
     listen_addrs = [/ip4/<attacker_ip>/tcp/<attacker_port>]
     route      = []
     max_hops   = 6
4. Victim: self_peer_id == content.to → calls respond_delivered(victim_peer_id, victim_peer_id, [attacker_addr])
   → pending_delivered[victim_peer_id] = ([/ip4/<attacker_ip>/tcp/<attacker_port>], now)
5. Attacker sends ConnectionSync:
     from  = victim_peer_id
     to    = victim_peer_id
     route = []
6. Victim: self_peer_id == content.to, pending_delivered[victim_peer_id] found
   → spawns try_nat_traversal([/ip4/<attacker_ip>/tcp/<attacker_port>])
   → victim initiates TCP connect() to attacker_ip:attacker_port for 30 seconds
   → on success: control.raw_session(stream, attacker_addr, RawSessionInfo::inbound(...))
     → full P2P session established with attacker's server
```

pub(crate) async fn try_nat_traversal(
    bind_addr: Option<SocketAddr>,
    addr: Multiaddr,
) -> Result<(TcpStream, Multiaddr), std::io::Error> {
    let net_addr = match multiaddr_to_socketaddr(&addr) {
        Some(addr) => addr,
        None => {
            debug!("Failed to convert multiaddr to socketaddr");
            return Err(std::io::ErrorKind::InvalidInput.into());
        }
    };

    // Use a fixed interval but add a small amount of randomness
    let base_retry_interval = Duration::from_millis(200);

    // total time
    let timeout_duration = Duration::from_secs(30);
    let start_time = Instant::now();
    let mut retry_count = 0u32;
    while start_time.elapsed() < timeout_duration {
        retry_count += 1;

        // Add a small amount of random jitter (±25ms) to avoid conflicts
        // caused by continuous precise synchronization
        let jitter = Duration::from_millis(rand::random::<u64>() % 50);
        let actual_interval = if rand::random::<bool>() {
            base_retry_interval + jitter
        } else {
            base_retry_interval.saturating_sub(jitter)
        };

        let socket = create_socket(bind_addr, net_addr)?;

        match runtime::timeout(
            std::time::Duration::from_millis(200),
            socket.connect(net_addr),
        )
        .await
        {
            Ok(Ok(stream)) => {
                // try get the stored error in the underlying socket
                // if the socket is not connected, it will return an error
                if let Err(err) = check_connection(&stream) {
                    debug!("Failed to connect to NAT(base check): {}", err);
                }
                return Ok((stream, addr));
            }
            Err(err) => {
                debug!("Failed to connect to NAT(timeout): {}", err);
            }
            Ok(Err(err)) => {
                if err.kind() == std::io::ErrorKind::AddrNotAvailable {
                    return Err(err);
                }
                debug!(
                    "Failed to connect to NAT(other error): {}, {}",
                    err.kind(),
                    err
                );
            }
        }
        runtime::delay_for(actual_interval).await;
    }

    debug!("Failed to connect to NAT after {} retries", retry_count);
    Err(std::io::ErrorKind::TimedOut.into())
}
