### Title
`Indexer::prune` Permanently Skips `Header`/`TxHash` Pruning When No `ConsumedOutPoint` Entries Exist — (`util/indexer/src/indexer.rs`)

### Summary

`Indexer::prune` uses `min_block_number` — which is only set inside the `ConsumedOutPoint` deletion loop — as the seek key for the subsequent `Header`/`TxHash` deletion pass. When the `ConsumedOutPoint` keyspace is empty (e.g., a chain of cellbase-only blocks), `min_block_number` stays at `BlockNumber::MAX`, the seek key becomes `[0xE0, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF]`, the iterator finds nothing, and every `Header` and `TxHash` entry ever written is silently retained forever.

### Finding Description

**Write path (`append`):**

For every block, `append` unconditionally writes a `Key::Header` entry and, for every matched transaction, a `Key::TxHash` entry. [1](#0-0) 

`Key::ConsumedOutPoint` is only written when a **non-cellbase** transaction (`tx_index > 0`) consumes a live cell that passes the cell filter. [2](#0-1) 

Therefore, a chain of cellbase-only blocks produces `Header` and `TxHash` entries but zero `ConsumedOutPoint` entries.

**Prune path (`prune`):**

`min_block_number` is initialised to `BlockNumber::MAX` and is only updated inside the `ConsumedOutPoint` loop body: [3](#0-2) 

When the loop body never executes (empty `ConsumedOutPoint` keyspace), `min_block_number` remains `u64::MAX`. The seek key for the Header pass is then built as:

```
[KeyPrefix::Header(224u8)] ++ [0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF]
``` [4](#0-3) 

No stored block has block number `u64::MAX`, so the RocksDB iterator returns nothing, and the entire `Header`/`TxHash` deletion loop is a no-op. Every `Header` and `TxHash` entry written since genesis is retained.

### Impact Explanation

- `Header` entries (prefix `0xE0`) and `TxHash` entries (prefix `0xC0`) accumulate at a rate of one `Header` + one `TxHash` per matched transaction per block, with no upper bound.
- RocksDB storage grows without limit, degrading all indexer read paths (range scans, point lookups) and eventually exhausting disk on any node running the indexer.
- The indexer is the primary query interface for wallets and dApps; its degradation or crash directly affects the usability of the CKB node.

### Likelihood Explanation

Cellbase-only blocks are the **normal** state of the CKB chain during low-activity periods. No special attacker capability is required — any miner producing blocks without user transactions triggers this path. The condition is also reachable when custom cell filters exclude all non-cellbase outputs, causing `ConsumedOutPoint` to remain empty even on a busy chain.

### Recommendation

Replace the `min_block_number`-based seek with an unconditional start from block 0 for the Header/TxHash pruning pass. The simplest fix:

```rust
// Replace:
let mut key_prefix_header = vec![KeyPrefix::Header as u8];
key_prefix_header.extend_from_slice(&min_block_number.to_be_bytes());

// With:
let start_number = if min_block_number == BlockNumber::MAX { 0u64 } else { min_block_number };
let mut key_prefix_header = vec![KeyPrefix::Header as u8];
key_prefix_header.extend_from_slice(&start_number.to_be_bytes());
```

This ensures Header/TxHash pruning always runs up to `prune_to_block`, regardless of whether any `ConsumedOutPoint` entries exist.

### Proof of Concept

1. Instantiate an `Indexer` with `keep_num = 10`, `prune_interval = 1`.
2. Append `N = 10 * keep_num` blocks, each containing only a cellbase transaction with one output matching the default cell filter.
3. After each `append`, call `prune` (it is called automatically at every `prune_interval`).
4. Count `Key::Header` and `Key::TxHash` entries in RocksDB.
5. **Expected (correct):** entries for blocks `0..N-keep_num-1` are deleted; count stabilises at `keep_num + 1`.
6. **Actual (buggy):** count equals `N` and never decreases, because `min_block_number == u64::MAX` causes the Header/TxHash deletion loop to be skipped on every `prune` call.

The existing test infrastructure in `util/indexer/src/indexer.rs` (lines 941+) already provides the `new_indexer` helper and block-building utilities needed to write this assertion directly. [5](#0-4)

    pub(crate) fn prune(&self) -> Result<(), Error> {
        let (tip_number, _tip_hash) = self.tip()?.expect("stored tip");
        let prune_number = self.keep_num + 1;
        if tip_number > prune_number {
            let prune_to_block = tip_number - prune_number;
            let mut batch = self.store.batch()?;
            // prune ConsumedOutPoint => Cell
            let key_prefix_consumed_out_point = vec![KeyPrefix::ConsumedOutPoint as u8];
            let iter = self
                .store
                .iter(&key_prefix_consumed_out_point, IteratorDirection::Forward)?
                .take_while(|(key, _value)| key.starts_with(&key_prefix_consumed_out_point));
            let mut min_block_number = BlockNumber::MAX;
            for (block_number, key) in iter
                .map(|(key, _value)| {
                    (
                        BlockNumber::from_be_bytes(
                            key[1..9].try_into().expect("stored block_number"),
                        ),
                        key,
                    )
                })
                .take_while(|(block_number, _key)| prune_to_block.gt(block_number))
            {
                if min_block_number == BlockNumber::MAX {
                    min_block_number = block_number;
                }
                batch.delete(key)?;
            }

            // prune Header => Transactions and TxHash => TransactionInputs
            let mut key_prefix_header = vec![KeyPrefix::Header as u8];
            key_prefix_header.extend_from_slice(&min_block_number.to_be_bytes());
            let iter = self
                .store
                .iter(&key_prefix_header, IteratorDirection::Forward)?
                .take_while(|(key, _value)| {
                    key.starts_with(&[KeyPrefix::Header as u8])
                        && BlockNumber::from_be_bytes(
                            key[1..9].try_into().expect("stored block_number"),
                        ) <= prune_to_block
                });
            for (txs, header_key) in iter.map(|(header_key, value)| {
                (
                    Value::parse_transactions_value(&value, header_key.len() == 42),
                    header_key,
                )
            }) {
                for (tx_hash, _outputs_len, _tx_index) in txs {
                    batch.delete(Key::TxHash(&tx_hash).into_vec())?;
                }
                batch.delete(header_key)?;
            }

            batch.commit()?;
        }
        Ok(())
    }
