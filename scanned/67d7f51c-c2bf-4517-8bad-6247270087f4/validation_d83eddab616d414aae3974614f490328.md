### Title
Out-of-Bounds Index Panic in `remove_unconfirmed_tx` When `tx_age > MAX_CONFIRM_BLOCKS` — (`util/fee-estimator/src/estimator/confirmation_fraction.rs`)

### Summary

`TxConfirmStat::remove_unconfirmed_tx` correctly guards the `block_unconfirmed_txs` array access when `tx_age >= 1000`, but unconditionally indexes `confirm_blocks_to_failed_txs[tx_age - 1]` without a corresponding bounds check. When a tracked tx is evicted from the pool after more than 1000 blocks, `tx_age - 1 >= 1000` exceeds the Vec's length of 1000, causing a Rust index-out-of-bounds panic that crashes the tx-pool service thread.

### Finding Description

`confirm_blocks_to_failed_txs` is initialized with length `MAX_CONFIRM_BLOCKS = 1000` (valid indices 0..=999): [1](#0-0) 

In `remove_unconfirmed_tx`, the guard at line 208 correctly routes `block_unconfirmed_txs` access for aged-out txs: [2](#0-1) 

But the `count_failure` branch immediately below has **no bounds check** on `tx_age`: [3](#0-2) 

When `tx_age = 1001`, `tx_age - 1 = 1000` is out of bounds for a Vec of length 1000 → **panic**.

`drop_tx` always passes `count_failure = true`: [4](#0-3) 

`drop_tx_inner` passes `self.best_height` as `tip_height`: [5](#0-4) 

`best_height` is updated on every committed block: [6](#0-5) 

### Impact Explanation

A Rust Vec index-out-of-bounds panics unconditionally. The tx-pool service thread panics, crashing the node's transaction processing. This matches the claimed impact: **node crash / network congestion**.

### Likelihood Explanation

The default pool expiry is 12 hours (`DEFAULT_EXPIRY_HOURS = 12`): [7](#0-6) 

At ~8 seconds/block, 12 hours ≈ 5400 blocks, far exceeding the 1001-block threshold. Any unprivileged user submitting a tx via RPC (`SubmitLocalTx`) or P2P relay that stays unconfirmed for >1000 blocks (e.g., a low-fee-rate tx) and is then evicted by pool size pressure or expiry will trigger the panic. The pool eviction path (`limit_size`, `remove_expired`) calls `callbacks.call_reject`, which propagates to `fee_estimator.reject_tx` → `drop_tx`: [8](#0-7) [9](#0-8) 

### Recommendation

Add a bounds check before accessing `confirm_blocks_to_failed_txs`:

```rust
if count_failure && tx_age <= self.confirm_blocks_to_failed_txs.len() {
    self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64;
}
```

Or use `get_mut` to silently skip out-of-range ages, consistent with how `block_unconfirmed_txs` already handles the overflow case via `old_unconfirmed_txs`.

    fn remove_unconfirmed_tx(
        &mut self,
        entry_height: u64,
        tip_height: u64,
        bucket_index: usize,
        count_failure: bool,
    ) {
        let tx_age = tip_height.saturating_sub(entry_height) as usize;
        if tx_age < 1 {
            return;
        }
        if tx_age >= self.block_unconfirmed_txs.len() {
            self.bucket_stats[bucket_index].old_unconfirmed_txs -= 1;
        } else {
            let block_index = (entry_height % self.block_unconfirmed_txs.len() as u64) as usize;
            self.block_unconfirmed_txs[block_index][bucket_index] -= 1;
        }
        if count_failure {
            self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64;
        }
    }

    pub(crate) fn remove_expired(&mut self, callbacks: &Callbacks) {
        let now_ms = ckb_systemtime::unix_time_as_millis();

        let removed: Vec<_> = self
            .pool_map
            .iter()
            .filter(|&entry| self.expiry + entry.inner.timestamp < now_ms)
            .map(|entry| entry.inner.clone())
            .collect();

        for entry in removed {
            let tx_hash = entry.transaction().hash();
            debug!("remove_expired {} timestamp({})", tx_hash, entry.timestamp);
            self.pool_map.remove_entry(&entry.proposal_short_id());
            let reject = Reject::Expiry(entry.timestamp);
            callbacks.call_reject(self, &entry, reject);
        }
    }
