### Title
Off-by-one in `MAX_VMS_COUNT` spawn guard allows 17th VM creation — (`script/src/scheduler.rs`)

### Summary

The spawn guard at line 532 uses `>` instead of `>=`, allowing exactly one VM beyond `MAX_VMS_COUNT` (16) to be created.

### Finding Description

`MAX_VMS_COUNT` is defined as `16`: [1](#0-0) 

The guard in the `Message::Spawn` arm is: [2](#0-1) 

When `suspended.len() + instantiated.len() == 16` (i.e., exactly 16 VMs are alive), the expression `16 > 16` evaluates to `false`, so the guard does **not** fire. `boot_vm` is called and a 17th VM is created. The correct operator is `>=`.

### Impact Explanation

- Any unprivileged script author can craft a V2 script that spawns 15 children (root + 15 = 16 total), then issues one additional `spawn` syscall.
- The 17th VM is created, violating the documented invariant.
- **Memory**: each VM allocates its own address space; one extra VM means one extra allocation beyond the intended cap. The cycle limit still bounds total execution, so this is not unbounded memory exhaustion.
- **Consensus divergence**: all nodes running the same binary have the same bug, so there is no divergence between same-version nodes. A divergence risk would arise between a patched node (rejects at 16) and an unpatched node (allows 17), making this a concern during a rolling upgrade.

### Likelihood Explanation

Trivially reachable: any script author can write a loop that spawns 16 VMs and then issues one more `spawn`. No privileged access, no PoW, no key material required.

### Recommendation

Change `>` to `>=` on line 532:

```rust
// Before
if self.suspended.len() + self.instantiated.len() > MAX_VMS_COUNT as usize {
// After
if self.suspended.len() + self.instantiated.len() >= MAX_VMS_COUNT as usize {
```

### Proof of Concept

Write a unit test that:
1. Spawns exactly 15 child VMs from the root (total = 16).
2. Issues one more `spawn` syscall.
3. Asserts the return code is `MAX_VMS_SPAWNED` (8).

With the current `>` check the return code will be `SUCCESS` (0), confirming the 17th VM was created. Changing to `>=` makes the test pass correctly. [2](#0-1)
