### Title
Network Alert Signature Replayable Across CKB Networks (Mainnet/Testnet) — (`util/network-alert/src/verifier.rs`)

### Summary
The `RawAlert` message that Nervos Foundation signers sign contains no network/chain domain separator (no genesis hash, no chain name, no network ID). The same four hardcoded public keys are the default for every CKB network. A valid alert signature produced for testnet is cryptographically identical to a valid alert signature for mainnet. Any unprivileged peer who observes a legitimately-signed alert on one network can replay it verbatim on any other CKB network.

### Finding Description

The `RawAlert` molecule schema contains only application-level fields:

```
table RawAlert {
    notice_until:   Uint64,
    id:             Uint32,
    cancel:         Uint32,
    priority:       Uint32,
    message:        Bytes,
    min_version:    BytesOpt,
    max_version:    BytesOpt,
}
```

No chain identifier, genesis hash, or network name is included. [1](#0-0) 

`calc_alert_hash()` is a plain blake2b hash of the raw bytes of `RawAlert` — nothing else:

```rust
pub fn calc_alert_hash(&self) -> packed::Byte32 {
    self.calc_hash()
}
``` [2](#0-1) 

`Verifier::verify_signatures()` derives the message to verify from `alert.calc_alert_hash()` and checks it against the hardcoded public-key set. There is no assertion that the alert was signed for the current network:

```rust
let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
// ...
verify_m_of_n(&message, self.config.signatures_threshold, &signatures, &self.pubkeys)
``` [3](#0-2) 

The default `NetworkAlertConfig` is loaded from a single shared TOML file that is identical for every network (mainnet, testnet, devnet):

```toml
signatures_threshold = 2
public_keys = [
  "0x03933a9b116c5017561742c37ae69acb0dca3329a52c479c85df5bb387c8ac8715",
  ...
]
``` [4](#0-3) [5](#0-4) 

CKB does have a per-network identifier (`identify_name()` returns `/<chain_id>/<genesis_hash_prefix>`), but it is used only for P2P peer handshake, never for alert signature binding: [6](#0-5) 

### Impact Explanation

An attacker who observes a legitimately-signed alert on the CKB testnet (or any other CKB-compatible network) can submit the exact same `Alert` struct — raw bytes and signatures unchanged — to a CKB mainnet node via the `send_alert` RPC or directly over the P2P Alert protocol. The `Verifier` will accept it (the signatures are cryptographically valid against the same keys), the `AlertNotifier` will store it, and the node will broadcast it to all connected peers. Every mainnet node will display the alert to operators and users.

Concrete consequences:
- A testnet alert saying "CKB v0.X.* has a critical bug, stop using it immediately" is replayed on mainnet, causing unnecessary panic, emergency upgrades, or service disruptions.
- A cancel alert from testnet can suppress a legitimate active mainnet alert (if IDs collide).
- The attack is silent — the replayed alert is indistinguishable from a genuine one.

### Likelihood Explanation

- All valid alerts are publicly observable: any node connected to the testnet P2P network receives them, and `get_blockchain_info` RPC exposes them with full signatures.
- The attacker needs no privileged access, no keys, and no special capability — only the ability to call `send_alert` on a mainnet RPC endpoint (which is a standard, documented, unauthenticated RPC) or connect as a P2P peer.
- The Nervos Foundation has historically issued real alerts (e.g., alert `20230001`), so signed material exists on-chain and in public archives.

### Recommendation

Include a network domain separator in the signed message. The simplest fix is to hash the genesis block hash (or `identify_name()` string) together with the `RawAlert` bytes before signing and verifying:

```rust
// In calc_alert_hash / verify_signatures:
let mut hasher = new_blake2b();
hasher.update(genesis_hash.as_slice()); // domain separator
hasher.update(alert.raw().as_slice());
let mut hash = [0u8; 32];
hasher.finalize(&mut hash);
let message = Message::from_slice(&hash)?;
```

Alternatively, add a `chain_id` or `genesis_hash` field to the `RawAlert` molecule schema and assert it matches the local node's genesis hash inside `verify_signatures`.

### Proof of Concept

1. On testnet, call `get_blockchain_info` on any node that has received a real alert. Extract the `signatures` array and the full alert fields.
2. On mainnet, call `send_alert` with the identical payload (same `id`, `message`, `notice_until`, `signatures`).
3. Observe that `verify_signatures` returns `Ok(())` and the alert is stored and broadcast to all mainnet peers.

The `notice_until` field must be in the future (checked by the RPC handler at `rpc/src/module/alert.rs:106`), so the attacker may need to craft a fresh alert on testnet with a far-future expiry before replaying it — still requiring no private keys, only the ability to submit a transaction to testnet RPC.     pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
        trace!("Verifying alert {:?}", alert);
        let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
        let signatures: Vec<Signature> = alert
            .signatures()
            .into_iter()
            .filter_map(
                |sig_data| match Signature::from_slice(sig_data.as_reader().raw_data()) {
                    Ok(sig) => {
                        if sig.is_valid() {
                            Some(sig)
                        } else {
                            debug!("invalid signature: {:?}", sig);
                            None
                        }
                    }
                    Err(err) => {
                        debug!("signature error: {}", err);
                        None
                    }
                },
            )
            .collect();
        verify_m_of_n(
            &message,
            self.config.signatures_threshold,
            &signatures,
            &self.pubkeys,
        )
        .map_err(|err| err.kind())?;
        Ok(())
    }

    fn send_alert(&self, alert: Alert) -> Result<()> {
        let alert: packed::Alert = alert.into();
        let now_ms = ckb_systemtime::unix_time_as_millis();
        let notice_until: u64 = alert.raw().notice_until().into();
        if notice_until < now_ms {
            return Err(RPCError::invalid_params(format!(
                "Expected `params[0].notice_until` in the future (> {now_ms}), got {notice_until}",
            )));
        }

        let result = self.verifier.verify_signatures(&alert);

        match result {
            Ok(()) => {
                // set self node notifier
                self.notifier.lock().add(&alert);

                self.network_controller.broadcast_with_handle(
                    SupportProtocols::Alert.protocol_id(),
                    alert.as_bytes(),
                    &self.handle,
                );
                Ok(())
            }
            Err(e) => Err(RPCError::custom_with_error(
                RPCError::AlertFailedToVerifySignatures,
                e,
            )),
        }
    }
