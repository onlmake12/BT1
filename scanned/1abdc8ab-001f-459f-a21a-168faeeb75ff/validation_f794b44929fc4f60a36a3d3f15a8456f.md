### Title
Off-by-One Eviction Threshold in `check_purge` Enables Peer Store Blockage via Crafted Discovery Messages — (`network/src/peer_store/peer_store_impl.rs`)

### Summary

The group-based eviction path in `check_purge` uses a strict `> 4` threshold instead of `>= 4`. An attacker with a single P2P connection can send crafted `Nodes` discovery messages advertising exactly 4 public IP addresses per `/16` subnet across 4096 distinct subnets, filling all 16384 `ADDR_COUNT_LIMIT` slots. When the store is full and all addresses are connectable, `check_purge` produces zero eviction candidates and returns `PeerStoreError::EvictionFailed`, permanently blocking `add_addr` for any new peer.

### Finding Description

**Root cause — `network/src/peer_store/peer_store_impl.rs`, line 378:** [1](#0-0) 

The comment at line 338 documents the intended invariant as "more than 4 peer," but the correct semantic for the eviction to be useful is `>= 4` (i.e., evict from any group that has reached the per-subnet cap). With `> 4`, a group of exactly 4 returns `None` from `flat_map`, contributing nothing to `candidate_peers`.

**Trigger condition:** [2](#0-1) 

If step 1 (non-connectable eviction) finds nothing, and every group in step 2 has `len == 4`, `candidate_peers` is empty and `EvictionFailed` is returned.

**Entry point — P2P Discovery protocol:** [3](#0-2) 

`add_new_addrs` is called for every received `DiscoveryMessage::Nodes` message. It calls `peer_store.add_addr` for each advertised address, which calls `check_purge` before insertion.

**Address validity check does not prevent the attack:** [4](#0-3) 

Only private/loopback IPs are filtered. Any public IP across 4096 distinct `/16` subnets passes `is_valid_addr`. The attacker does not need to own these IPs.

**Fresh addresses are permanently connectable:** [5](#0-4) 

`add_addr` inserts entries with `last_connected_at_ms = 0` and `attempts_count = 0`. Such entries return `true` from `is_connectable` until `ADDR_MAX_RETRIES` failed dial attempts accumulate. The attacker can re-send the same addresses periodically; `AddrManager::add` updates an existing entry when `last_connected_at_ms >= existing`, resetting `attempts_count` to 0 and preventing natural expiry. [6](#0-5) 

**Store limit:** [7](#0-6) 

`ADDR_COUNT_LIMIT = 16384`. With `MAX_ADDRS = 3` per `Node` item and `MAX_ADDR_TO_SEND = 1000` items per `Nodes` message, the attacker needs approximately 6 crafted messages to fill the store. [8](#0-7) 

### Impact Explanation

Once the store is full with 4096 groups of exactly 4 connectable addresses, every subsequent `add_addr` call returns `Err(PeerStoreError::EvictionFailed)`. The error is silently swallowed as a debug log in `add_new_addrs`. The node's peer store is frozen: legitimate peer addresses discovered via the network can no longer be recorded. Peer discovery degrades to zero new entries until the node is restarted or the peer store file is manually deleted.

### Likelihood Explanation

The attacker needs only one accepted inbound or outbound P2P connection to send `Nodes` messages. No PoW, no key material, no privileged access is required. The crafted addresses are arbitrary public IPs; the attacker does not need to control them. The attack is repeatable and can be maintained indefinitely by periodically re-advertising the same addresses to reset `attempts_count`.

### Recommendation

Change the strict inequality to `>=` on line 378:

```rust
// Before
if addrs.len() > 4 {
// After
if addrs.len() >= 4 {
```

This ensures that any group at or above the per-subnet cap of 4 is eligible for eviction, closing the off-by-one gap.

### Proof of Concept

```rust
// Fill store with exactly 4 addresses per /16 subnet across 4096 subnets
let mut peer_store = PeerStore::default();
for subnet in 0u16..4096 {
    for host in 1u8..=4 {
        let a = (subnet >> 8) as u8;
        let b = (subnet & 0xff) as u8;
        let addr: Multiaddr = format!(
            "/ip4/{}.{}.0.{}/tcp/8115/p2p/{}",
            a + 1, b, host, PeerId::random().to_base58()
        ).parse().unwrap();
        peer_store.add_addr(addr, Flags::COMPATIBILITY).unwrap();
    }
}
// Store is now at ADDR_COUNT_LIMIT with all groups having len == 4
// Next add_addr must succeed (evict one slot) but instead returns EvictionFailed
let new_addr: Multiaddr = format!(
    "/ip4/200.200.200.1/tcp/8115/p2p/{}", PeerId::random().to_base58()
).parse().unwrap();
assert!(peer_store.add_addr(new_addr, Flags::COMPATIBILITY).is_ok()); // FAILS
```
