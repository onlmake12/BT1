Audit Report

## Title
TOCTOU Race in `_process_tx` Admits Transactions With Dead Pool Inputs — (`tx-pool/src/process.rs`)

## Summary
`_process_tx` resolves transaction inputs under a read lock in `pre_check`, releases that lock, runs `verify_rtx` without any lock, then finalises admission in `submit_entry` under a write lock. When the chain tip has not advanced, `submit_entry` skips `check_rtx` entirely. A concurrent RBF replacement that evicts the parent transaction during the unlocked `verify_rtx` window causes the child to be admitted with a permanently unresolvable input, corrupting `edges.inputs` and enabling repeated pool-exhaustion attacks at the cost of two transaction fees per injected invalid entry.

## Finding Description

**Phase 1 — `pre_check` (read lock, then released)**

`pre_check` calls `resolve_tx` which uses `PoolCell::cell` to look up each input in `pool_map`. If TX_A is in the pool and produces cell C2, TX_B's input C2 resolves as `CellStatus::Live`. The read lock is then released and `tip_hash` is captured. [1](#0-0) [2](#0-1) 

**Phase 2 — `verify_rtx` (no lock held)**

Script execution is async and unbounded in duration. During this window a second worker can process TX_A' (an RBF replacement for TX_A), which calls `process_rbf` → `pool_map.remove_entry_and_descendants(&id)`, evicting TX_A and its output C2 from the pool entirely. [3](#0-2) [4](#0-3) 

**Phase 3 — `submit_entry` (write lock held)**

The guard `if pre_resolve_tip != tip_hash` is `false` because no new block arrived. `check_rtx` (which re-validates input liveness against the current pool state) is therefore never called. The only other check is `find_conflict_outpoint`, which only scans `edges.inputs` for direct double-spends; since TX_A's removal already cleared C2 from `edges.inputs`, no conflict is detected. TX_B is passed to `_submit_entry` and C2 is inserted into `edges.inputs` even though C2 no longer exists anywhere. [5](#0-4) 

The same flaw applies when RBF is enabled: `check_rbf` finds no conflict (TX_A is already gone) and returns an empty set, after which the tip-hash guard is still the only gate before `_submit_entry`. [6](#0-5) 

## Impact Explanation

The corruption has two concrete effects:

1. **Pool exhaustion / node DoS.** TX_B occupies a pool slot indefinitely (until the expiry timer or `limit_size` evicts it). The attacker can repeat the pattern at the cost of two transaction fees per injected invalid entry, filling the pool and evicting legitimate transactions via `limit_size`. This matches the allowed impact: **High — vulnerabilities or bad designs which could cause CKB network congestion with few costs**.

2. **Edge-tracking corruption.** C2 is recorded in `edges.inputs` as consumed by TX_B. Any subsequent transaction legitimately spending C2 (e.g., after a reorg that re-introduces TX_A) is rejected as a double-spend, even though TX_B is invalid and will never be mined. [7](#0-6) 

## Likelihood Explanation

- Requires only standard RPC access (`send_transaction`). No privileged role, no majority hashpower.
- The race window equals the full `verify_rtx` duration for TX_B. The attacker maximises this by crafting TX_B with a script consuming close to `max_tx_verify_cycles`, while TX_A' uses a trivial always-success script processed almost instantly by a second worker.
- `VerifyMgr` spawns multiple concurrent workers (`max_tx_verify_workers`), making the interleaving routine rather than exceptional. [8](#0-7) 

- Cost per injected invalid entry: fees for TX_A and TX_A' only; TX_B is never committed.

## Recommendation

In `submit_entry`, unconditionally re-run `check_rtx` against the current pool state before calling `_submit_entry`, regardless of whether the tip hash changed. The existing guard should only gate the *time-relative* re-verification:

```rust
// Always re-check input liveness against current pool state
status = check_rtx(tx_pool, &snapshot, &entry.rtx)?;

// Only redo time-relative verify if the chain tip advanced
if pre_resolve_tip != tip_hash {
    let tip_header = snapshot.tip_header();
    let tx_env = status.with_env(tip_header);
    time_relative_verify(snapshot, Arc::clone(&entry.rtx), tx_env)?;
}
``` [9](#0-8) 

## Proof of Concept

```
1. Attacker holds live cell C1 on-chain.

2. Submit TX_A (input: C1 → output: C2, trivial always-success script).
   → TX_A enters pool; C2 is live in pool_map outputs.

3. Submit TX_B (input: C2, script near max_tx_verify_cycles).
   → pre_check resolves C2 as Live (TX_A in pool); read lock released.
   → verify_rtx begins; worker is busy for ~seconds.

4. Submit TX_A' (input: C1, higher fee — valid RBF of TX_A).
   → second worker processes TX_A' quickly (trivial script).
   → submit_entry for TX_A': check_rbf finds TX_A as conflict,
     calls process_rbf → remove_entry_and_descendants(TX_A).
   → TX_A and C2 are gone from pool_map. TX_B is still in verify_queue.

5. TX_B's verify_rtx completes.
   → submit_entry for TX_B:
       pre_resolve_tip == tip_hash (no new block) → check_rtx skipped.
       find_conflict_outpoint(TX_B): C2 not in edges.inputs → no conflict.
       _submit_entry adds TX_B to pool; C2 inserted into edges.inputs.

6. TX_B is now in the pending pool with input C2 that does not exist.
   Block assembler skips TX_B silently.
   Any tx spending C2 is rejected as double-spend.
   Repeat steps 2–5 to exhaust pool capacity.
```

    async fn process_inner(&mut self) {
        loop {
            if self.exit_signal.is_cancelled() {
                info!("Verify worker::process_inner exit_signal is cancelled");
                return;
            }
            self.refresh_status();
            if self.status != ChunkCommand::Resume {
                return;
            }
            // cheap query to check queue is not empty
            if self.tasks.read().await.is_empty() {
                return;
            }

            self.refresh_status();
            if self.status != ChunkCommand::Resume {
                return;
            }

            // pick a entry to run verify
            let entry = {
                let mut tasks = self.tasks.write().await;
                match tasks.pop_front(self.role == WorkerRole::OnlySmallCycleTx) {
                    Some(entry) => entry,
                    None => {
                        if !tasks.is_empty() {
                            tasks.re_notify();
                            debug!(
                                "Worker (role: {:?}) didn't got tx after pop_front, but tasks is not empty, notify other Workers now",
                                self.role
                            );
                        }
                        return;
                    }
                }
            };

            if let Some((res, snapshot)) = self
                .service
                ._process_tx(
                    entry.tx.clone(),
                    entry.remote.map(|e| e.0),
                    Some(&mut self.command_rx),
                )
                .await
            {
                self.service
                    .after_process(entry.tx, entry.remote, &snapshot, &res)
                    .await;
            } else {
                info!("_process_tx for tx: {} returned none", entry.tx.hash());
            }
        }
    }
