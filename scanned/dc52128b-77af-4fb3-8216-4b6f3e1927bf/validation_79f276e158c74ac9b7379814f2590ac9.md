Audit Report

## Title
`GetBlocksProcess` Missing IBD Guard Causes Resource Drain on Syncing Nodes — (`File: sync/src/synchronizer/get_blocks_process.rs`)

## Summary
The module-level documentation in `sync/src/synchronizer/mod.rs` explicitly contracts that an IBD-mode node will respond with `packed::InIBD` to both `GetHeaders` and `GetBlocks` requests. `GetHeadersProcess::execute()` enforces this at line 53, but `GetBlocksProcess::execute()` contains no such guard. Any unprivileged P2P peer can send `GetBlocks` to an IBD node and force it to perform RocksDB reads, block serialization, and async network transmission — consuming resources the node needs for its own block download.

## Finding Description
The module documentation at `sync/src/synchronizer/mod.rs` L4 states the IBD contract for both message types: [1](#0-0) 

`GetHeadersProcess::execute()` correctly implements this at line 53: [2](#0-1) 

`GetBlocksProcess::execute()` has no equivalent check. After validating only message size, genesis hash, and deduplication, it unconditionally reads blocks from `active_chain` and spawns async send tasks: [3](#0-2) 

The relayer's IBD guard at `sync/src/relayer/mod.rs` L815–818 covers only relay-protocol messages and does not protect the sync protocol's `GetBlocks` path: [4](#0-3) 

The exploit path is: connect as a P2P peer → send `SyncMessage::GetBlocks` with valid early-block hashes → node performs disk reads, serializes blocks, and spawns async send tasks for each hash, up to `INIT_BLOCKS_IN_TRANSIT_PER_PEER` per message, with no IBD rejection. [5](#0-4) 

## Impact Explanation
The concrete impact is performance degradation of a node in IBD: disk I/O, CPU serialization, and outbound bandwidth are consumed serving attacker-requested blocks instead of being reserved for the node's own block download. This does not crash the node, cause consensus deviation, or damage the economy. It maps to **Low (501–2000 points): Any other important performance improvements for CKB** — the missing guard is a documented contract violation that measurably degrades IBD throughput under adversarial conditions.

## Likelihood Explanation
IBD is the normal startup state for any new or restarting node. No privilege, key, or special knowledge is required — any peer that completes the P2P handshake can immediately send `GetBlocks`. The attack is trivially automatable and repeatable from multiple connections simultaneously.

## Recommendation
Add an IBD guard at the top of `GetBlocksProcess::execute()`, mirroring `GetHeadersProcess`:

```rust
// sync/src/synchronizer/get_blocks_process.rs
let active_chain = self.synchronizer.shared.active_chain();
if active_chain.is_initial_block_download() {
    self.send_in_ibd();
    return Status::ignored();
}
```

`send_in_ibd()` would need to be added to `GetBlocksProcess` following the same pattern as in `GetHeadersProcess` at lines 101–115. [6](#0-5) 

Alternatively, add a single IBD gate in `Synchronizer::process()` covering both `GetHeaders` and `GetBlocks` dispatch arms.

## Proof of Concept
1. Start a CKB node from genesis; confirm `is_initial_block_download() == true`.
2. Connect a custom P2P client as an inbound peer.
3. Send `SyncMessage::GetBlocks` containing hashes of known-valid blocks (e.g., blocks 1–16).
4. Observe the node responds with `SendBlock` messages — no `InIBD` is sent.
5. Repeat in a tight loop from multiple connections; measure increasing disk I/O and outbound bandwidth on the IBD node while its own sync rate decreases.

The root cause is the absence of `active_chain.is_initial_block_download()` in `GetBlocksProcess::execute()`: [7](#0-6)

```rust
    pub fn execute(self) -> Status {
        let block_hashes = self.message.block_hashes();
        // use MAX_HEADERS_LEN as limit, we may increase the value of INIT_BLOCKS_IN_TRANSIT_PER_PEER in the future
        if block_hashes.len() > MAX_HEADERS_LEN {
            return StatusCode::ProtocolMessageIsMalformed.with_context(format!(
                "BlockHashes count({}) > MAX_HEADERS_LEN({})",
                block_hashes.len(),
                MAX_HEADERS_LEN,
            ));
        }
        let active_chain = self.synchronizer.shared.active_chain();

        let iter = block_hashes.iter().take(INIT_BLOCKS_IN_TRANSIT_PER_PEER);

        let mut dedup = HashSet::new();
        for block_hash in iter {
            debug!("get_blocks {} from peer {:?}", block_hash, self.peer);
            let block_hash = block_hash.to_entity();

            if block_hash == self.synchronizer.shared().consensus().genesis_hash() {
                return StatusCode::RequestGenesis.with_context("Request genesis block");
            }

            if !dedup.insert(block_hash.clone()) {
                return StatusCode::RequestDuplicate.with_context("Request duplicate block");
            }

            if !active_chain.contains_block_status(&block_hash, BlockStatus::BLOCK_VALID) {
                debug!(
                    "Ignoring get_block {} request from peer={} as it is not verified.",
                    block_hash, self.peer
                );
                continue;
            }

            if let Some(block) = active_chain.get_block(&block_hash) {
                debug!(
                    "respond_block {} {} to peer {:?}",
                    block.number(),
                    block.hash(),
                    self.peer,
                );
                let content = packed::SendBlock::new_builder().block(block.data()).build();
                let message = packed::SyncMessage::new_builder().set(content).build();

                let nc = Arc::clone(self.nc);
                self.synchronizer
                    .shared()
                    .shared()
                    .async_handle()
                    .spawn(async move { async_send_message_to(&nc, self.peer, &message).await });
            } else {
                // TODO response not found
                // TODO add timeout check in synchronizer

                // We expect that `block_hashes` is sorted descending by height.
                // So if we cannot find the current one from local, we cannot find
                // the next either.
                debug!("Stopping getblocks, since {} is not found", block_hash);
                break;
            }
        }

        Status::ok()
    }
```
