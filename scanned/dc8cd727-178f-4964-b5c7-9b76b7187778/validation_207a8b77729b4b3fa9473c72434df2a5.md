Audit Report

## Title
MMR `get_root()` I/O Amplification via Crafted `GetLastStateProof` ‚Äî (`util/light-client-protocol-server/src/components/get_last_state_proof.rs`)

## Summary

The `GetLastStateProofProcess::execute()` handler allows any unprivileged peer to trigger O(N √ó log H) RocksDB point reads per request by crafting a `GetLastStateProof` message with `last_n_blocks=499` and `difficulties=[]`. The per-request size limit (`GET_LAST_STATE_PROOF_LIMIT = 1000`) is satisfied while the `complete_headers` loop calls `chain_root_mmr(number-1).get_root()` once per block, each traversing O(log H) MMR nodes from disk. No rate limiting exists on the light client protocol handler, and valid requests return `Status::ok()` with no peer penalty.

## Finding Description

**Step 1 ‚Äî Limit check passes.**
The guard at `execute()` checks:
```
difficulties.len() + (last_n_blocks as usize) * 2 > GET_LAST_STATE_PROOF_LIMIT
```
With `difficulties=[]` and `last_n_blocks=499`: `0 + 998 = 998 ‚â§ 1000` ‚Äî passes. [1](#0-0) [2](#0-1) 

**Step 2 ‚Äî `last_n_numbers` vector is maximally populated.**
When `difficulty_boundary` equals the tip's total difficulty, `get_first_block_total_difficulty_is_not_less_than` resolves to the tip block itself. The guard at L313 fires (`last_block_number - tip < last_n_blocks`), setting `difficulty_boundary_block_number = last_block_number - 499`, and `last_n_numbers` becomes a 499-entry vector `[tip-499, tip-498, ‚Ä¶, tip-1]`. [3](#0-2) 

**Step 3 ‚Äî `complete_headers` calls `get_root()` once per block.**
For every entry in `block_numbers`, the loop unconditionally constructs a new `ChainRootMMR` and calls `get_root()`: [4](#0-3) 

**Step 4 ‚Äî Each `get_root()` issues O(log H) RocksDB reads.**
`chain_root_mmr(n)` creates an MMR of size `leaf_index_to_mmr_size(n)`. `get_root()` finds the MMR peaks and merges them; each peak lookup calls `get_elem(pos)` ‚Üí `store.get_header_digest(pos)`, a direct RocksDB read. For H ‚âà 10‚Å∂, this is ~20 reads per call. [5](#0-4) [6](#0-5) 

**Step 5 ‚Äî No rate limiting on the light client protocol.**
`LightClientProtocol` has no `rate_limiter` field (contrast with `Relayer`, which has a `governor`-based per-peer rate limiter). `try_process` dispatches directly to `GetLastStateProofProcess::execute()` with no throttle check. A valid request returns `Status::ok()`, triggering no ban. [7](#0-6) [8](#0-7) 

**Total I/O per request:** 499 √ó ~20 = ~9,980 RocksDB point reads for MMR roots alone, plus 499 `get_block` calls and uncle hash computations.

## Impact Explanation

This is a **bad design which could cause CKB network congestion with few costs** (High, 10001‚Äì15000 points). An attacker maintaining a persistent connection and sending back-to-back crafted requests sustains continuous RocksDB I/O amplification. Because the light client server shares the same RocksDB instance as the main chain processing pipeline, saturating I/O degrades block validation, peer sync, and all other storage-dependent operations on the targeted node. Multiple attackers targeting multiple light client servers simultaneously can degrade the CKB light client network broadly. The cost to the attacker is zero: no PoW, no stake, no privileged role.

## Likelihood Explanation

Any peer that can open a light-client protocol connection can send this message. The crafted parameters (`last_n_blocks=499`, `difficulties=[]`, `difficulty_boundary=tip_total_difficulty`, `last_hash=tip_hash`) are trivially derived from the tip state freely advertised via `GetLastState`. The attack is repeatable indefinitely with no ban risk, as valid requests always return `Status::ok()`. [9](#0-8) 

## Recommendation

1. **Cache the MMR root per request**: compute `chain_root_mmr(tip-1).get_root()` once before the `complete_headers` loop and reuse it for all sampled headers, since all headers in a single response share the same tip MMR.
2. **Pre-store `parent_chain_root`** alongside block headers at write time (in `reconcile_main_chain`), eliminating per-block MMR traversal at read time entirely.
3. **Add per-peer rate limiting** to `LightClientProtocol::try_process` using the same `governor`-based pattern already used in `Relayer`. [10](#0-9) 

## Proof of Concept

Send the following message to a node running the light client protocol server:

```
GetLastStateProof {
    last_n_blocks:        499,
    difficulties:         [],
    last_hash:            <current tip hash>,       // passes is_main_chain check
    start_hash:           <genesis hash>,
    start_number:         0,                        // reorg_last_n_numbers = []
    difficulty_boundary:  <tip total difficulty>    // resolves to tip, triggers guard
}
```

Limit check: `0 + 499√ó2 = 998 ‚â§ 1000` ‚Üí passes.
Server builds `last_n_numbers = [tip-499, ‚Ä¶, tip-1]` (499 entries).
`complete_headers` calls `chain_root_mmr(n-1).get_root()` 499 times.

**Instrumentation test**: add a counter to `Snapshot::get_header_digest`; send one such request and observe ~499 √ó ‚åälog‚ÇÇ(tip_height)‚åã calls. Send at maximum rate; observe RocksDB read IOPS scale linearly with request rate √ó log(H), with no ban applied and `Status::ok()` returned for every request. [11](#0-10) [6](#0-5)

    fn complete_headers(
        &self,
        positions: &mut Vec<u64>,
        last_hash: &packed::Byte32,
        numbers: &[BlockNumber],
    ) -> Result<Vec<packed::VerifiableHeader>, String> {
        let mut headers = Vec::new();

        for number in numbers {
            if let Some(ancestor_header) = self.snapshot.get_ancestor(last_hash, *number) {
                let position = leaf_index_to_pos(*number);
                positions.push(position);

                let ancestor_block = self
                    .snapshot
                    .get_block(&ancestor_header.hash())
                    .ok_or_else(|| {
                        format!(
                            "failed to find block for header#{} (hash: {:#x})",
                            number,
                            ancestor_header.hash()
                        )
                    })?;
                let uncles_hash = ancestor_block.calc_uncles_hash();
                let extension = ancestor_block.extension();

                let parent_chain_root = if *number == 0 {
                    Default::default()
                } else {
                    let mmr = self.snapshot.chain_root_mmr(*number - 1);
                    match mmr.get_root() {
                        Ok(root) => root,
                        Err(err) => {
                            let errmsg = format!(
                                "failed to generate a root for block#{number} since {err:?}"
                            );
                            return Err(errmsg);
                        }
                    }
                };

                let header = packed::VerifiableHeader::new_builder()
                    .header(ancestor_header.data())
                    .uncles_hash(uncles_hash)
                    .extension(Pack::pack(&extension))
                    .parent_chain_root(parent_chain_root)
                    .build();

                headers.push(header);
            } else {
                let errmsg = format!("failed to find ancestor header ({number})");
                return Err(errmsg);
            }
        }

        Ok(headers)
    }
