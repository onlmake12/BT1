Audit Report

## Title
Peer Store Slot Exhaustion via Peer-ID-Less Addresses Injected Through Discovery Protocol — (`network/src/peer_store/peer_store_impl.rs`, `network/src/protocols/discovery/mod.rs`)

## Summary

The discovery protocol accepts and stores multiaddrs that lack a `/p2p/<peer_id>` component. These addresses are permanently excluded from both feeler and attempt selection due to `extract_peer_id(...).unwrap_or_default()` evaluating to `false`. Because they are never dialed, `attempts_count` never increments, `is_connectable()` always returns `true`, and `check_purge()` cannot evict them. An attacker with a single connected peer can fill all 16,384 peer store slots with ~6 discovery messages, preventing the node from discovering or dialing any new peers.

## Finding Description

**Step 1 — Attacker-controlled entry point.**
A connected peer sends a `DiscoveryMessage::Nodes` message. `DiscoveryProtocol::received()` calls `self.addr_mgr.add_new_addrs(session.id, addrs)` with the raw multiaddrs from the message without any peer-id requirement. [1](#0-0) 

**Step 2 — `is_valid_addr` does not check for `/p2p/` component.**
`DiscoveryAddressManager::add_new_addrs()` filters only through `is_valid_addr()`, which checks only whether the IP is publicly reachable via `is_reachable(socket_addr.ip())`. An address like `/ip4/1.2.3.4/tcp/8114` passes this check with no peer-id present. [2](#0-1) [3](#0-2) 

**Step 3 — `add_addr()` stores the address without peer-id validation.**
`PeerStore::add_addr()` only checks the ban list, then calls `check_purge()` and stores the address unconditionally via `addr_manager.add(AddrInfo::new(addr, 0, score, flags.bits()))`. [4](#0-3) 

**Step 4 — `unwrap_or_default()` permanently excludes peer-id-less addresses from selection.**
Both `fetch_addrs_to_feeler()` and `fetch_addrs_to_attempt()` use:
```rust
extract_peer_id(&peer_addr.addr)
    .map(|peer_id| !peers.contains_key(&peer_id))
    .unwrap_or_default()
```
When `extract_peer_id` returns `None`, `.unwrap_or_default()` returns `false` (the default for `bool`), unconditionally excluding the address from both selection paths. [5](#0-4) [6](#0-5) 

**Step 5 — Addresses are never evicted.**
Since these addresses are never dialed, `attempts_count` stays at `0`. `is_connectable()` returns `true` when `last_connected_at_ms == 0 && attempts_count < ADDR_MAX_RETRIES (3)`, so they are never marked non-connectable. [7](#0-6) 

`check_purge()` step 1 only evicts addresses where `is_connectable()` is `false` — these addresses are immune. Step 2 only evicts from network segments with >4 peers. An attacker using one address per /16 segment (trivially achievable with any public IP range) bypasses both eviction paths, causing `check_purge()` to return `Err(PeerStoreError::EvictionFailed)` once the store is full. [8](#0-7) 

## Impact Explanation

The peer store has a hard cap of `ADDR_COUNT_LIMIT = 16384`. [9](#0-8) 

Once filled with peer-id-less addresses:
- `add_addr()` returns `Err(PeerStoreError::EvictionFailed)` for all new legitimate addresses.
- `fetch_addrs_to_feeler()` and `fetch_addrs_to_attempt()` return empty results.
- The node cannot discover or dial new peers. As existing connections drop, the node cannot replace them, leading to progressive isolation and eventual inability to participate in the network.

This matches **High (10001–15000 points): Vulnerabilities which could easily crash a CKB node** — the node's networking layer is rendered non-functional for peer discovery and outbound dialing.

## Likelihood Explanation

The attack requires only a single connected peer (no special privilege). `MAX_ADDR_TO_SEND = 1000` items × `MAX_ADDRS = 3` addresses per item = 3,000 addresses per non-announce `Nodes` message. [10](#0-9) 

Filling 16,384 slots requires approximately 6 messages. The attacker needs only to spread addresses across distinct /16 IP segments (e.g., `1.0.0.0/16`, `2.0.0.0/16`, …) to defeat the network-group eviction heuristic. This is trivially achievable with any public IP range and requires no special resources beyond a single peer connection.

## Recommendation

1. **Reject peer-id-less addresses at ingestion.** In `add_addr()` or `add_new_addrs()`, require `extract_peer_id(&addr).is_some()` before storing. This is the minimal fix.
2. **Fix the filter semantics.** The `unwrap_or_default()` pattern silently treats "no peer_id" as "already connected." The correct behavior is to reject such addresses at storage time, not silently exclude them at selection time.
3. **Add per-session rate limiting** in the discovery protocol to bound how many addresses a single peer can inject per time window.

## Proof of Concept

```rust
let mut peer_store = PeerStore::default();

// Fill store with peer-id-less addresses across distinct /16 segments
for i in 0u8..=255 {
    for j in 0u8..=63 {
        let addr: Multiaddr = format!("/ip4/{}.{}.1.1/tcp/8114", i, j)
            .parse().unwrap();
        // No /p2p/ component — accepted by add_addr
        let _ = peer_store.add_addr(addr, Flags::COMPATIBILITY);
    }
}

// Store is full; legitimate address is rejected
let legit: Multiaddr = "/ip4/8.8.8.8/tcp/8114/p2p/QmValid...".parse().unwrap();
assert!(peer_store.add_addr(legit, Flags::COMPATIBILITY).is_err());

// Feeler and attempt selection return empty
assert!(peer_store.fetch_addrs_to_feeler(10, |_| true).is_empty());
assert!(peer_store.fetch_addrs_to_attempt(10, Flags::COMPATIBILITY, |_| true).is_empty());

// Stored addresses are permanently connectable (never evicted)
let now = ckb_systemtime::unix_time_as_millis();
let sample: Multiaddr = "/ip4/0.0.1.1/tcp/8114".parse().unwrap();
let info = peer_store.addr_manager().get(&sample).unwrap();
assert!(info.is_connectable(now)); // true, attempts_count == 0
```

```rust
    fn check_purge(&mut self) -> Result<()> {
        if self.addr_manager.count() < ADDR_COUNT_LIMIT {
            return Ok(());
        }

        // Evicting invalid data in the peer store is a relatively rare operation
        // There are certain cleanup strategies here:
        // 1. First evict the nodes that have reached the eviction condition
        // 2. If the first step is unsuccessful, enter the network segment grouping mode
        //  2.1. Group current data according to network segment
        //  2.2. Sort according to the amount of data in the same network segment
        //  2.3. In the network segment with more than 4 peer, randomly evict 2 peer

        let now_ms = ckb_systemtime::unix_time_as_millis();
        let candidate_peers: Vec<_> = self
            .addr_manager
            .addrs_iter()
            .filter_map(|addr| {
                if !addr.is_connectable(now_ms) {
                    Some(addr.addr.clone())
                } else {
                    None
                }
            })
            .collect();

        for key in candidate_peers.iter() {
            self.addr_manager.remove(key);
        }

        if candidate_peers.is_empty() {
            let candidate_peers: Vec<_> = {
                let mut peers_by_network_group: HashMap<Group, Vec<_>> = HashMap::default();
                for addr in self.addr_manager.addrs_iter() {
                    peers_by_network_group
                        .entry((&addr.addr).into())
                        .or_default()
                        .push(addr);
                }
                let len = peers_by_network_group.len();
                let mut peers = peers_by_network_group
                    .drain()
                    .map(|(_, v)| v)
                    .collect::<Vec<Vec<_>>>();

                peers.sort_unstable_by_key(|k| std::cmp::Reverse(k.len()));

                peers
                    .into_iter()
                    .take(len / 2)
                    .flat_map(move |addrs| {
                        if addrs.len() > 4 {
                            Some(
                                addrs
                                    .iter()
                                    .choose_multiple(&mut rand::thread_rng(), 2)
                                    .into_iter()
                                    .map(|addr| addr.addr.clone())
                                    .collect::<Vec<Multiaddr>>(),
                            )
                        } else {
                            None
                        }
                    })
                    .flatten()
                    .collect()
            };

            for key in candidate_peers.iter() {
                self.addr_manager.remove(key);
            }

            if candidate_peers.is_empty() {
                return Err(PeerStoreError::EvictionFailed.into());
            }
        }
        Ok(())
    }
```

```rust
    pub fn is_connectable(&self, now_ms: u64) -> bool {
        // do not remove addr tried in last minute
        if self.tried_in_last_minute(now_ms) {
            return true;
        }
        // we give up if never connect to this addr
        if self.last_connected_at_ms == 0 && self.attempts_count >= ADDR_MAX_RETRIES {
            return false;
        }
        // consider addr is not connectable if failed too many times
        if now_ms.saturating_sub(self.last_connected_at_ms) > ADDR_TIMEOUT_MS
            && (self.attempts_count >= ADDR_MAX_FAILURES)
        {
            return false;
        }
        true
    }
```
