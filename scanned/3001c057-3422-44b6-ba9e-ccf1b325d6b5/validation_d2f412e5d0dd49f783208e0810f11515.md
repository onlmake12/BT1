### Title
Unbounded Growth of `pending_get_block_proposals` with Unconstrained Iteration — (`File: sync/src/types/mod.rs`, `sync/src/relayer/get_block_proposal_process.rs`)

---

### Summary

The `pending_get_block_proposals` map in `SyncState` has no size cap. An unprivileged P2P peer can repeatedly send `GetBlockProposal` relay messages containing unique, fabricated proposal IDs that are absent from the local tx pool. Each message causes `insert_get_block_proposals` to push new entries into the unbounded `DashMap`. When the periodic timer fires `prune_tx_proposal_request`, it clones and iterates the entire map, creating a CPU and memory spike proportional to the number of entries accumulated since the last drain.

### Impact Explanation

Between two consecutive timer ticks, a single connected peer can send a stream of `GetBlockProposal` messages, each carrying up to ~3 000 unique, fabricated `ProposalShortId` values (10 bytes each) that do not exist in the local tx pool. Every such ID is inserted into `pending_get_block_proposals`. When the timer fires:

1. **Memory**: the map is cloned in full — O(N) allocation.
2. **CPU**: the entire map is iterated — O(N) work on the async relay timer task, potentially stalling other relay operations.

Sustained flooding can cause the node to exhaust heap memory (OOM) or cause the relay timer task to lag, degrading block propagation for all peers.

---

### Likelihood Explanation

Any peer that has completed the P2P handshake can send `GetBlockProposal` relay messages. No authentication, stake, or special role is required. The per-message proposal count is bounded (~3 000), but there is no per-peer rate limit and no cap on the accumulated map size. A single malicious peer sending messages at network speed can fill the map faster than the timer drains it.

The CHANGELOG already records analogous fixes for related structures (`#3093: Resolve inflight proposals memory bloat issue`, `#3094: Fix inflight block potential memory bloat issues`, `#3110: Fix pending compact block memory bloat on abnormal flow`), confirming that this class of unbounded-collection bug is realistic and has been exploited in practice in the same codebase. [7](#0-6) 

---

### Recommendation

1. **Cap the map size**: reject or drop entries in `insert_get_block_proposals` when `pending_get_block_proposals.len()` exceeds a configurable limit (e.g., `max_block_proposals_limit * max_uncles_num`).
2. **Per-peer quota**: track how many pending proposal entries each peer has contributed and disconnect or ignore peers that exceed the quota.
3. **Avoid full clone on drain**: use `std::mem::take` or swap with an empty map instead of `clone()` to avoid the O(N) allocation on every timer tick.

---

### Proof of Concept

1. Connect a peer to a CKB node (standard P2P handshake via `RelayV3` protocol).
2. In a tight loop, send `GetBlockProposal` relay messages where `proposals` contains ~3 000 random 10-byte `ProposalShortId` values that do not correspond to any real transaction.
3. Observe `pending_get_block_proposals` growing without bound between timer ticks.
4. When the relay timer fires, observe the `prune_tx_proposal_request` task consuming O(N) CPU and memory proportional to the number of accumulated entries.

    pub async fn execute(self) -> Status {
        let shared = self.relayer.shared();
        let message_len = self.message.proposals().len();
        {
            // The block proposal request is separate from uncles,
            // so here the limit is only used to calculate the maximum value of uncles
            let limit = shared.consensus().max_block_proposals_limit()
                * (shared.consensus().max_uncles_num() as u64);
            if message_len as u64 > limit {
                return StatusCode::ProtocolMessageIsMalformed.with_context(format!(
                    "GetBlockProposal proposals count({message_len}) > consensus max_block_proposals_limit({limit})"
                ));
            }
        }

        let proposals: HashSet<packed::ProposalShortId> =
            self.message.proposals().to_entity().into_iter().collect();

        if proposals.len() != message_len {
            return StatusCode::RequestDuplicate.with_context("Request duplicate proposal");
        }

        let fetched_transactions = {
            let tx_pool = self.relayer.shared.shared().tx_pool_controller();
            let fetch_txs = tx_pool.fetch_txs(proposals.clone()).await;
            if let Err(e) = fetch_txs {
                debug_target!(
                    crate::LOG_TARGET_RELAY,
                    "relayer tx_pool_controller send fetch_txs error: {:?}",
                    e
                );
                return Status::ok();
            }
            fetch_txs.unwrap()
        };
        // Transactions that do not exist on this node
        let not_exist_proposals: Vec<packed::ProposalShortId> = proposals
            .into_iter()
            .filter(|short_id| !fetched_transactions.contains_key(short_id))
            .collect();

        // Cache request, try process on timer
        self.relayer
            .shared()
            .state()
            .insert_get_block_proposals(self.peer, not_exist_proposals);

    async fn prune_tx_proposal_request(&self, nc: &Arc<dyn CKBProtocolContext + Sync>) {
        let get_block_proposals = self.shared().state().drain_get_block_proposals();
        let tx_pool = self.shared.shared().tx_pool_controller();

        let fetch_txs = tx_pool
            .fetch_txs(
                get_block_proposals
                    .iter()
                    .map(|kv_pair| kv_pair.key().clone())
                    .collect(),
            )
            .await;
        if let Err(err) = fetch_txs {
            debug_target!(
                crate::LOG_TARGET_RELAY,
                "relayer prune_tx_proposal_request internal error: {:?}",
                err,
            );
            return;
        }

        let txs = fetch_txs.unwrap();

        let mut peer_txs = HashMap::new();
        for (id, peer_indices) in get_block_proposals.into_iter() {
            if let Some(tx) = txs.get(&id) {
                for peer_index in peer_indices {
                    let tx_set = peer_txs.entry(peer_index).or_insert_with(Vec::new);
                    tx_set.push(tx.clone());
                }
            }
        }

        let mut relay_bytes = 0;
        let mut relay_proposals = Vec::new();
        for (peer_index, txs) in peer_txs {
            for tx in txs {
                let data = tx.data();
                let tx_size = data.total_size();
                if relay_bytes + tx_size > MAX_RELAY_TXS_BYTES_PER_BATCH {
                    send_block_proposals(nc, peer_index, std::mem::take(&mut relay_proposals))
                        .await;
                    relay_bytes = tx_size;
                } else {
                    relay_bytes += tx_size;
                }
                relay_proposals.push(data);
            }
            if !relay_proposals.is_empty() {
                send_block_proposals(nc, peer_index, std::mem::take(&mut relay_proposals)).await;
                relay_bytes = 0;
            }
        }
