### Title
Unauthenticated Miner Notify HTTP Server Allows Any Caller to Redirect Mining Work and Steal Block Rewards - (File: `miner/src/client.rs`)

### Summary
The CKB miner's "notify mode" HTTP server (`listen_block_template_notify`) accepts block template updates from any network-reachable caller with no authentication, IP filtering, or token verification. The `handle` function unconditionally calls `update_block_template` on any valid JSON body it receives. An attacker who can reach the miner's listen port can inject a crafted `BlockTemplate` containing their own coinbase lock script, causing the miner to expend its hashpower on the attacker's block and submit a reward-bearing block to the CKB node under the attacker's address.

### Finding Description

The miner supports two operating modes: poll mode and notify mode. In notify mode, the CKB node's block assembler pushes new `BlockTemplate` objects to the miner via HTTP POST. The miner binds a TCP listener on the configured `listen` address and serves incoming connections through the `handle` function.

**Root cause — `handle` in `miner/src/client.rs` (lines 358–369):**

```rust
async fn handle(
    client: Client,
    req: Request<hyper::body::Incoming>,
) -> Result<Response<Empty<Bytes>>, Error> {
    let body = BodyExt::collect(req).await?.aggregate();

    if let Ok(template) = serde_json::from_reader(body.reader()) {
        client.update_block_template(template);
    }

    Ok(Response::new(Empty::new()))
}
```

There is no check on the source IP, no shared secret, no HTTP Basic Auth, and no HMAC or token validation. Any HTTP POST that deserializes as a valid `BlockTemplate` is accepted and immediately applied. [1](#0-0) 

**`listen_block_template_notify` binds and serves without any allowlist:** [2](#0-1) 

**`update_block_template` immediately replaces the active work:** [3](#0-2) 

The contrast is stark: the *outgoing* RPC client already implements HTTP Basic Auth via `parse_authorization`, but the *incoming* notify server has no equivalent protection: [4](#0-3) 

**`submit_block` on the node side verifies PoW and parent hash but does NOT verify that the coinbase lock script matches the configured `block_assembler`:** [5](#0-4) 

The coinbase lock script is set inside the `BlockTemplate` by the block assembler, sourced from `BlockAssemblerConfig.args` / `code_hash`: [6](#0-5) 

Because `submit_block` does not re-derive or re-check the coinbase against the node's own `block_assembler` config, a block whose coinbase was crafted by the attacker will be accepted and committed to the chain.

### Impact Explanation

An attacker who can reach the miner's notify port (e.g., the miner is bound to `0.0.0.0:8888`, or the attacker is on the same host/LAN) can:

1. **Steal block rewards**: Inject a `BlockTemplate` with their own coinbase lock script. The miner solves the PoW and submits the block; the node accepts it; the block reward is paid to the attacker's address.
2. **Waste/deny hashpower**: Continuously inject invalid or stale templates, causing the miner to work on blocks that will never be accepted, effectively zeroing the operator's mining output.

Both outcomes are direct financial losses to the miner operator with no recovery path once a block is committed.

### Likelihood Explanation

- Notify mode is an explicitly documented, production-supported feature (`resource/ckb-miner.toml` shows the `listen` option). [7](#0-6) 
- Operators who bind to `0.0.0.0` or expose the port through NAT/firewall rules are immediately vulnerable to any internet attacker.
- Even a `127.0.0.1` binding is exploitable by any co-located process (shared hosting, container escape, compromised dependency).
- The attacker needs only the ability to send a single HTTP POST with a valid JSON `BlockTemplate` body — no credentials, no keys, no privileged access.
- The attack is silent: the miner logs no warning when a template arrives from an unexpected source.

### Recommendation

1. **Require a shared secret**: Add a configurable bearer token or HMAC-signed header to the notify endpoint. The miner should reject any request that does not present the correct credential.
2. **Allowlist source IPs**: Restrict accepted connections to the configured CKB node's IP address.
3. **Re-derive and verify the coinbase**: In `submit_block`, the node should verify that the submitted block's coinbase lock script matches the locally configured `block_assembler` before accepting the block, closing the reward-theft path even if the miner is tricked.

### Proof of Concept

**Precondition**: Miner is running in notify mode with `listen = "0.0.0.0:8888"` (or any reachable address).

**Steps**:

1. Obtain a valid current `BlockTemplate` from the legitimate CKB node:
   ```bash
   curl -X POST http://<node>:8114/ \
     -H 'Content-Type: application/json' \
     -d '{"jsonrpc":"2.0","method":"get_block_template","params":[],"id":1}'
   ```

2. Modify the `cellbase_template.data` field to replace the coinbase lock script `args` with the attacker's public key hash.

3. POST the crafted template directly to the miner's notify port:
   ```bash
   curl -X POST http://<miner>:8888/ \
     -H 'Content-Type: application/json' \
     -d '<crafted_block_template_json>'
   ```

4. The miner's `handle` function deserializes the payload and calls `update_block_template` with no authentication check. The miner begins working on the attacker's template.

5. When the miner finds a valid nonce, it calls `submit_block` on the CKB node. The node verifies the PoW and parent hash (both valid), accepts the block, and the block reward is credited to the attacker's lock script.

**Expected outcome**: The miner's hashpower is redirected; the next block mined pays the reward to the attacker's address. The legitimate operator receives nothing.

    async fn listen_block_template_notify(&self, addr: SocketAddr) {
        let listener = TcpListener::bind(addr).await.unwrap();
        let server = auto::Builder::new(TokioExecutor::new());
        let graceful = GracefulShutdown::new();
        let stop_rx: CancellationToken = new_tokio_exit_rx();

        loop {
            let client = self.clone();
            let handle = service_fn(move |req| handle(client.clone(), req));
            tokio::select! {
                conn = listener.accept() => {
                    let (stream, _) = match conn {
                        Ok(conn) => conn,
                        Err(e) => {
                            info!("accept error: {}", e);
                            tokio::time::sleep(std::time::Duration::from_secs(1)).await;
                            continue;
                        }
                    };
                    let stream = hyper_util::rt::TokioIo::new(Box::pin(stream));
                    let conn = server.serve_connection_with_upgrades(stream, handle);

                    let conn = graceful.watch(conn.into_owned());
                    tokio::spawn(async move {
                        if let Err(err) = conn.await {
                            info!("connection error: {}", err);
                        }
                    });
                },
                _ = stop_rx.cancelled() => {
                    info!("Miner client received exit signal. Exit now");
                    break;
                }
            }
        }
        drop(listener);
        graceful.shutdown().await;
    }

        let work_id = block_template.work_id.into();
        let updated = |id| {
            if id != work_id || id == 0 {
                Some(work_id)
            } else {
                None
            }

                error!("notify_new_block error: {:?}", e);
            }
        }

fn parse_authorization(url: &Uri) -> Option<HeaderValue> {
    let a: Vec<&str> = url.authority()?.as_str().split('@').collect();
    if a.len() >= 2 {
        if a[0].is_empty() {
            return None;
        }
        let mut encoded = "Basic ".to_string();
        base64::prelude::BASE64_STANDARD.encode_string(a[0], &mut encoded);
        let mut header = HeaderValue::from_str(&encoded).unwrap();
        header.set_sensitive(true);
        Some(header)
    } else {
        None
    }
}

    pub(crate) fn build_cellbase_witness(
        config: &BlockAssemblerConfig,
        snapshot: &Snapshot,
    ) -> CellbaseWitness {
        let hash_type: ScriptHashType = config.hash_type.into();
        let cellbase_lock = Script::new_builder()
            .args(config.args.as_bytes())
            .code_hash(&config.code_hash)
            .hash_type(hash_type)
            .build();
        let tip = snapshot.tip_header();

        let mut message = vec![];
        if let Some(version) = snapshot.compute_versionbits(tip) {
            message.extend_from_slice(&version.to_le_bytes());
            message.extend_from_slice(b" ");
        }
        if config.use_binary_version_as_message_prefix {
            message.extend_from_slice(config.binary_version.as_bytes());
        }
        if !config.message.is_empty() {
            message.extend_from_slice(b" ");
            message.extend_from_slice(config.message.as_bytes());
        }

        CellbaseWitness::new_builder()
            .lock(cellbase_lock)
            .message(message)
            .build()
    }
