### Title
Unsanitized Peer Address and Network Interface Name Strings Returned by Terminal RPC to TUI Consumers — (`rpc/src/module/terminal.rs`)

---

### Summary

The CKB Terminal RPC module (`get_overview`) is explicitly designed for TUI (Terminal User Interface) applications. It returns peer connection addresses and OS network interface names as raw, unsanitized `String` values. No control-character stripping or format validation is applied anywhere in the pipeline before these strings reach the JSON response consumed by terminal-based monitoring tools. An inbound peer connecting to the node directly influences the `address` field returned to every TUI consumer.

---

### Finding Description

`rpc/src/module/terminal.rs` implements `get_network_info()`, which iterates over all connected peers and builds a `PeerInfo` struct for each one:

```rust
peer_infos.push(PeerInfo {
    peer_id,
    is_outbound,
    latency_ms: latency_ms.into(),
    address: peer.connected_addr.to_string(),   // ← raw, unsanitized
});
``` [1](#0-0) 

The `address` field in `PeerInfo` is declared as a plain `String` with no format constraint:

```rust
pub struct PeerInfo {
    pub peer_id: usize,
    pub is_outbound: bool,
    pub latency_ms: Uint64,
    pub address: String,   // ← no sanitization, no validation
}
``` [2](#0-1) 

Similarly, `get_sys_info()` collects OS network interface names via `sysinfo` and assigns them directly:

```rust
let networks = sys_networks
    .iter()
    .map(|(name, data)| Network {
        interface_name: name.clone(),   // ← raw OS string, unsanitized
        ...
    })
    .collect();
``` [3](#0-2) 

The `interface_name` field is also a plain `String`:

```rust
pub struct Network {
    pub interface_name: String,
    ...
}
``` [4](#0-3) 

Both of these unsanitized strings are aggregated into the `Overview` struct and returned by `get_overview`:

```rust
Ok(Overview {
    sys,
    cells,
    mining,
    pool,
    network,
    version: self.network_controller.version().to_owned(),
})
``` [5](#0-4) 

A grep for `sanitize`, `strip_control`, `non_printable`, or `escape` in `rpc/src/` returns zero results — there is no sanitization layer anywhere in the Terminal RPC pipeline.

---

### Impact Explanation

The module's own documentation states its primary consumers are TUI dashboards and terminal-based monitoring tools:

> "TUI Monitoring Dashboards: Real-time node status displays in terminal environments" [6](#0-5) 

A TUI application that renders the `address` or `interface_name` fields directly to a terminal (e.g., using `println!`, `write!`, or a TUI widget that passes strings to the terminal) will forward any embedded ANSI escape sequences or control characters (CR, LF, `\x1b[...`) to the terminal emulator. This enables:

- **Terminal control character injection**: Overwriting displayed lines, moving the cursor, clearing the screen, or spoofing status fields (e.g., faking "Status: Trusted" or "Balance: 1000 CKB" next to legitimate fields).
- **Log injection**: If the TUI or its backend logs the raw address string, log lines can be forged.
- **Context-sensitive injection in downstream consumers**: If the `get_overview` JSON is consumed by a web dashboard or stored in a database, the unsanitized string becomes an XSS or SQL injection vector depending on how it is handled.

---

### Likelihood Explanation

- **Peer address path**: Any inbound peer connecting to the CKB node controls the remote socket address. While standard TCP/IP addresses are constrained, the `connected_addr` is a `Multiaddr` type. In the P2P discovery protocol, peers exchange and store multiaddrs as raw byte sequences (`BytesVec`). If a crafted multiaddr containing control characters is accepted and stored as `connected_addr`, it flows unsanitized into the Terminal RPC response. The attack surface is every inbound connection — no authentication required.
- **Interface name path**: OS network interface names are locally controlled, but on Linux systems they can be set to arbitrary strings (up to 15 bytes including control characters) by a local user or container environment, making this a local privilege escalation vector for TUI consumers running in shared environments.
- The Terminal RPC module is new and purpose-built for TUI consumers, making it the highest-risk surface for this class of vulnerability.

---

### Recommendation

1. Add a sanitization step in `get_network_info()` before assigning `address`:
   ```rust
   address: peer.connected_addr.to_string()
       .chars()
       .filter(|c| !c.is_control())
       .collect(),
   ```
2. Apply the same filter to `interface_name` in `get_sys_info()`.
3. Validate that peer addresses conform to the expected multiaddr format before storing them.
4. Document in the Terminal RPC module that all string fields returned to TUI consumers must be sanitized of control characters before display.

---

### Proof of Concept

1. Start a CKB node with the Terminal RPC module enabled.
2. Connect an inbound peer whose multiaddr (as stored by the discovery layer) contains an ANSI escape sequence, e.g., `\x1b[2J` (clear screen).
3. Call `get_overview` via RPC (e.g., `curl -X POST ... -d '{"jsonrpc":"2.0","method":"get_overview","params":[null],"id":1}'`).
4. Observe that the `network.peers[N].address` field in the JSON response contains the raw escape sequence.
5. A TUI application that renders this field with `println!("{}", peer.address)` or equivalent will execute the terminal escape sequence, clearing the screen or spoofing displayed content. [7](#0-6) [8](#0-7)

    fn get_network_info(&self, refresh: RefreshKind) -> Result<NetworkInfo> {
        // Check cache first unless force refresh
        if !refresh.contains(RefreshKind::NETWORK_INFO)
            && let Some(cached) = self.cache.get_network_info()
        {
            return Ok(cached);
        }

        // Fetch fresh network data
        let peers = self.network_controller.connected_peers();
        let total_peers = peers.len();
        let mut outbound_peers = 0;
        let mut inbound_peers = 0;
        let mut peer_infos = Vec::new();

        for (peer_index, peer) in peers {
            // Count inbound vs outbound connections
            if peer.is_outbound() {
                outbound_peers += 1;
            } else {
                inbound_peers += 1;
            }

            // Extract peer ID and RTT information
            let peer_id = peer_index.value();
            let is_outbound = peer.is_outbound();
            let latency_ms = if let Some(rtt) = peer.ping_rtt {
                rtt.as_millis() as u64
            } else {
                0
            };

            peer_infos.push(PeerInfo {
                peer_id,
                is_outbound,
                latency_ms: latency_ms.into(),
                address: peer.connected_addr.to_string(),
            });
        }

        let network_info = NetworkInfo {
            connected_peers: (total_peers as u64).into(),
            outbound_peers: (outbound_peers as u64).into(),
            inbound_peers: (inbound_peers as u64).into(),
            peers: peer_infos,
        };

        // Cache the result
        self.cache.set_network_info(network_info.clone());
        Ok(network_info)
    }
