Audit Report

## Title
Off-by-one in `check_purge()` eviction condition allows a single remote peer to permanently freeze peer discovery — (`network/src/peer_store/peer_store_impl.rs`)

## Summary
`check_purge()` in `peer_store_impl.rs` uses `addrs.len() > 4` to decide whether to evict from a network group. A single malicious peer can advertise exactly 4 fake addresses per `/16` subnet across 4096 distinct subnets, filling `ADDR_COUNT_LIMIT=16384` such that Pass 1 finds no non-connectable addresses and Pass 2 finds no group exceeding 4 entries. Every subsequent `add_addr()` call returns `Err(EvictionFailed)`, which is silently swallowed at `debug!` level, permanently freezing peer discovery for the node.

## Finding Description
`check_purge()` runs when `addr_manager.count() >= ADDR_COUNT_LIMIT (16384)`.

**Pass 1** collects addresses where `!is_connectable(now_ms)`. Addresses inserted via `add_addr()` are created with `last_connected_at_ms=0` and `attempts_count=0`. In `is_connectable()`, the only non-connectable conditions require `attempts_count >= ADDR_MAX_RETRIES (3)` or `attempts_count >= ADDR_MAX_FAILURES (10)` — neither applies to freshly inserted addresses. Pass 1 yields zero candidates. [1](#0-0) [2](#0-1) 

**Pass 2** groups addresses by `/16` IPv4 prefix, sorts groups by descending size, takes the top `len/2` groups, and for each applies:
```rust
if addrs.len() > 4 { Some(...evict 2...) } else { None }
``` [3](#0-2) 

With exactly 4 addresses per group across 4096 groups: `4 > 4` is `false` for every group → zero candidates → `return Err(PeerStoreError::EvictionFailed.into())`. [4](#0-3) 

The network group key is `Group::IP4([bits[0], bits[1]])` — the first two octets of the IPv4 address — giving 65536 possible distinct groups, of which only 4096 are needed. [5](#0-4) 

The error propagates via `check_purge()?` in `add_addr()` and is caught in `add_new_addrs()` with only a `debug!` log — no ban, no disconnect, no alert. [6](#0-5) [7](#0-6) 

## Impact Explanation
Peer discovery is permanently frozen for the targeted node. The node cannot add any new peer addresses to its store. After existing connections drop, the node loses the ability to find replacement peers, eventually becoming isolated from the network. This constitutes a remotely-triggerable degradation of network connectivity that, if applied at scale, could cause CKB network congestion or partition with minimal cost — matching the **High** impact class: *"Vulnerabilities or bad designs which could cause CKB network congestion with few costs."*

## Likelihood Explanation
Requires only one accepted inbound or outbound P2P connection. The discovery protocol allows up to `MAX_ADDR_TO_SEND=1000` items × `MAX_ADDRS=3` addresses per `Nodes` message, yielding 3000 addresses per message. [8](#0-7) 
Approximately 6 messages suffice to deliver 4096 × 4 = 16384 addresses. Addresses only need to pass `is_reachable()` (a local IP-range check), not actual connectivity. No PoW, no privilege, no Sybil attack required. The attack is repeatable after node restart.

## Recommendation
Change the eviction condition at line 378 of `network/src/peer_store/peer_store_impl.rs` from:
```rust
if addrs.len() > 4 {
```
to:
```rust
if addrs.len() >= 4 {
``` [9](#0-8) 

Additionally, consider per-session and per-source-subnet rate limiting in `add_new_addrs()` to bound how many addresses a single peer can contribute to the store.

## Proof of Concept
```rust
// Fill store with 4096 groups × 4 addresses each (all connectable)
for subnet in 0u16..4096 {
    for host in 1u8..=4 {
        let ip = format!("{}.{}.1.{}", subnet >> 8, subnet & 0xff, host);
        let addr = format!("/ip4/{}/tcp/8115/p2p/Qm...", ip).parse().unwrap();
        peer_store.add_addr(addr, Flags::COMPATIBILITY).unwrap();
    }
}
assert_eq!(peer_store.addr_manager().count(), 16384);

// 16385th add_addr must fail with EvictionFailed
let extra = "/ip4/200.0.1.1/tcp/8115/p2p/Qm...".parse().unwrap();
let result = peer_store.add_addr(extra, Flags::COMPATIBILITY);
assert!(matches!(result, Err(Error::PeerStore(PeerStoreError::EvictionFailed))));
assert_eq!(peer_store.addr_manager().count(), 16384); // store frozen
```
