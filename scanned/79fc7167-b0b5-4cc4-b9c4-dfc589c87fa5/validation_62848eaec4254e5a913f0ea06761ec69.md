### Title
Unbounded Orphan Pool Growth and Unconditional RocksDB Write Without PoW Verification Enables Remote DoS — (`chain/src/chain_service.rs`, `chain/src/utils/orphan_block_pool.rs`)

### Summary
An unprivileged remote peer can flood a CKB node with syntactically valid blocks referencing fabricated parent hashes. Because `non_contextual_verify` does not check PoW, each block is cheap to craft. Every such block is unconditionally written to RocksDB and inserted into the orphan pool, which has no hard size cap and no eviction for current-epoch blocks, causing unbounded heap and disk growth leading to OOM or disk exhaustion.

### Finding Description
`ChainService::asynchronous_process_block` runs `non_contextual_verify` (structural checks only — no PoW), then calls `insert_block` to persist the block to RocksDB, then calls `orphan_broker.process_lonely_block`. When the parent hash is unknown (not stored, not pending), `orphan_blocks_broker.insert` is called. `InnerPool::insert` performs no size check before inserting into the `HashMap`. `with_capacity(ORPHAN_BLOCK_SIZE)` is only a pre-allocation hint. The only cleanup path (`clean_expired_blocks`) fires every 60 seconds and only removes blocks from epochs more than 6 behind the tip — current-epoch attack blocks are never cleaned.

### Impact Explanation
- **Heap exhaustion**: each orphan entry holds a `LonelyBlockHash` in the in-memory `HashMap`; N entries → O(N) heap.
- **Disk exhaustion**: `insert_block` writes the full block to RocksDB before any orphan check; N blocks → O(N × block_size) disk.
- Both resources grow without bound until the process OOMs or the disk fills, crashing the node. Targeting all reachable nodes simultaneously causes a network-wide outage.

### Likelihood Explanation
The attacker needs only one peer connection and the ability to construct structurally valid blocks (valid merkle root, valid cellbase, no duplicate txs). No PoW solution is required. Block construction is computationally trivial. The attack is fully automatable and requires no privileged access.

### Recommendation
1. **Enforce a hard cap in `InnerPool::insert`**: reject (or evict the oldest leader's subtree) when `self.parents.len() >= MAX_ORPHAN_POOL_SIZE`.
2. **Require PoW verification before `insert_block`**: move `PowVerifier` into `non_contextual_verify` in `chain_service.rs` — PoW verification is context-independent (it only needs the block's own `compact_target` and `nonce`) and should not require the parent.
3. **Rate-limit orphan insertions per peer** at the sync/relay layer.
4. **Defer `insert_block` until after orphan-pool admission**: do not write to RocksDB before confirming the block will be accepted into the pool.

### Proof of Concept
```
for i in 1..N:
    block = craft_valid_block(
        parent_hash = random_bytes(32),   # unknown parent
        epoch = current_epoch,            # avoids expiry
        # no PoW needed — BlockVerifier does not check it
    )
    peer.send_block(block)

assert orphan_pool.len() == N   # no eviction
assert rocksdb_size() grows by N * avg_block_size
```
After N >> `ORPHAN_BLOCK_SIZE` iterations, the node OOMs or exhausts disk.

    fn insert(&mut self, lonely_block: LonelyBlockHash) {
        let hash = lonely_block.hash();
        let parent_hash = lonely_block.parent_hash();
        self.blocks
            .entry(parent_hash.clone())
            .or_default()
            .insert(hash.clone(), lonely_block);
        // Out-of-order insertion needs to be deduplicated
        self.leaders.remove(&hash);
        // It is a possible optimization to make the judgment in advance,
        // because the parent of the block must not be equal to its own hash,
        // so we can judge first, which may reduce one arc clone
        if !self.parents.contains_key(&parent_hash) {
            // Block referenced by `parent_hash` is not in the pool,
            // and it has at least one child, the new inserted block, so add it to leaders.
            self.leaders.insert(parent_hash.clone());
        }
        self.parents.insert(hash, parent_hash);
