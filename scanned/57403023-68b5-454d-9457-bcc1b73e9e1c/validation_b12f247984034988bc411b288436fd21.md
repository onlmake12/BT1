### Title
`total_tx_size`/`total_tx_cycles` Permanently Inflated When Cell-Dep-Consuming Transactions Evict Pool Entries — (File: `tx-pool/src/component/pool_map.rs`)

---

### Summary

In `PoolMap::add_entry`, the aggregate pool counters `total_tx_size` and `total_tx_cycles` are computed **before** `check_and_record_ancestors` runs. When that function evicts existing pool entries (because the incoming transaction's inputs consume cells that those entries reference as cell deps), the evictions decrement the counters via `update_stat_for_remove_tx`. However, `add_entry` then **unconditionally overwrites** both counters with the stale pre-computed values, erasing the decrements. The counters are permanently inflated by the aggregate size and cycles of every evicted entry. An unprivileged tx-pool submitter can exploit this to push `total_tx_size` above `max_tx_pool_size`, causing the pool to prematurely evict or reject legitimate transactions.

---

### Finding Description

**Root cause — `add_entry` in `tx-pool/src/component/pool_map.rs` lines 200–221:**

```
Step 1 (line 210-211):
  let (total_tx_size, total_tx_cycles) =
      self.updated_stat_for_add_tx(entry.size, entry.cycles)?;
  // Snapshot: total_tx_size = self.total_tx_size + entry.size  (PRE-eviction)

Step 2 (line 213):
  evicts = self.check_and_record_ancestors(&mut entry)?;
  // For each evicted entry E_i:
  //   remove_entry(E_i) → update_stat_for_remove_tx(E_i.size, E_i.cycles)
  //   → self.total_tx_size -= E_i.size   (DECREMENTED)

Step 3 (lines 218-219):
  self.total_tx_size = total_tx_size;   // OVERWRITES the decremented value
  self.total_tx_cycles = total_tx_cycles;
```

The pre-computed snapshot at Step 1 is `original_total + entry.size`. After Step 2, the live field is `original_total − Σ(evicted_sizes)`. Step 3 overwrites it back to `original_total + entry.size`, so the final value is inflated by `Σ(evicted_sizes)`.

The same overwrite applies to `total_tx_cycles`. [1](#0-0)

---

### Impact Explanation

1. **Premature pool eviction (DoS):** `limit_size` loops while `total_tx_size > max_tx_pool_size`. An inflated counter causes the pool to evict legitimate, fee-paying transactions that would otherwise fit. Evicted transactions are rejected with `Reject::Full` and reported to peers, disrupting normal mempool operation.

2. **Premature pool admission rejection:** The `VerifyQueue::is_full` check and the pool-size guard in `submit_entry` both rely on `total_tx_size`. Inflation causes new transactions to be rejected earlier than the actual pool capacity warrants.

3. **Incorrect RPC reporting:** `tx_pool_info` exposes `total_tx_size` and `total_tx_cycles` directly. Inflated values mislead operators and downstream tooling.

4. **Cumulative, unbounded inflation:** Each eviction event adds `Σ(evicted_sizes)` to the counter permanently. An attacker can repeat the pattern to push the counter arbitrarily high, eventually making the pool reject all incoming transactions.

---

### Likelihood Explanation

The trigger condition — a new transaction whose input spends a cell that an existing pool transaction uses as a cell dep — is a normal, protocol-valid scenario. An unprivileged tx-pool submitter (RPC caller or P2P relayer) can craft this deliberately:

1. Obtain or create a live cell `C`.
2. Submit `T1` referencing `C` as a cell dep (valid, `T1` enters the pool).
3. Submit `T2` spending `C` as an input (valid, `T1` is evicted, counter inflated by `size(T1)`).
4. Repeat with fresh cells.

Each round costs only the transaction fees for `T1` and `T2`. The attacker does not need any privileged access, majority hashpower, or Sybil capability.

---

### Recommendation

Move the counter update to **after** `check_and_record_ancestors` completes, so it reflects the post-eviction state:

```rust
pub(crate) fn add_entry(
    &mut self,
    mut entry: TxEntry,
    status: Status,
) -> Result<(bool, HashSet<TxEntry>), Reject> {
    // ... duplicate check ...
    evicts = self.check_and_record_ancestors(&mut entry)?;  // evictions happen first
    self.record_entry_edges(&entry)?;
    self.insert_entry(&entry, status);
    self.record_entry_descendants(&entry);
    self.track_entry_statics(None, Some(status));
    // Now compute totals on the post-eviction state:
    let (total_tx_size, total_tx_cycles) =
        self.updated_stat_for_add_tx(entry.size, entry.cycles)?;
    self.total_tx_size = total_tx_size;
    self.total_tx_cycles = total_tx_cycles;
    Ok((true, evicts))
}
```

Alternatively, subtract `Σ(evicted_sizes)` from the pre-computed snapshot before applying it, or recompute via `recompute_total_stat` after evictions.

### Proof of Concept

**Initial state:** pool is empty, `total_tx_size = 0`.

1. Submit `T1` (serialized size = `S1`) with cell `C` as a cell dep. Pool accepts it. `total_tx_size = S1`. ✓

2. Submit `T2` (serialized size = `S2`) with cell `C` as an input. Inside `add_entry`:
   - Line 210: pre-computed `total_tx_size_snapshot = S1 + S2`.
   - Line 213: `check_and_record_ancestors` evicts `T1` → `update_stat_for_remove_tx(S1, ...)` → `self.total_tx_size = 0`.
   - Line 218: `self.total_tx_size = S1 + S2` (overwrites `0`).

3. **Observed:** `total_tx_size = S1 + S2`. **Expected:** `total_tx_size = S2` (only `T2` is in the pool). **Inflation = S1.**

4. Repeat steps 1–3 with fresh cells `C'`, `C''`, … Each round inflates `total_tx_size` by the size of the evicted entry. After `N` rounds, `total_tx_size` exceeds `max_tx_pool_size` (default ~180 MB), causing `limit_size` to evict all legitimate pending transactions and the pool to reject all new submissions with `Reject::Full`.     /// Update size and cycles statistics for remove tx
    /// cycles overflow is possible, currently obtaining cycles is not accurate
    fn update_stat_for_remove_tx(&mut self, tx_size: usize, cycles: Cycle) {
        match (
            self.total_tx_size.checked_sub(tx_size),
            self.total_tx_cycles.checked_sub(cycles),
        ) {
            (Some(total_tx_size), Some(total_tx_cycles)) => {
                self.total_tx_size = total_tx_size;
                self.total_tx_cycles = total_tx_cycles;
            }
            _ => {
                if let Some((total_tx_size, total_tx_cycles)) = self.recompute_total_stat() {
                    error!(
                        "tx-pool total stats underflowed when removing size {} cycles {}, recomputed size {} cycles {}",
                        tx_size, cycles, total_tx_size, total_tx_cycles
                    );
                    self.total_tx_size = total_tx_size;
                    self.total_tx_cycles = total_tx_cycles;
                } else {
                    error!(
                        "tx-pool total stats underflowed when removing size {} cycles {}, and recomputing overflowed",
                        tx_size, cycles
                    );
                }
            }
        }
    }

    pub(crate) fn limit_size(
        &mut self,
        callbacks: &Callbacks,
        current_entry_id: Option<&ProposalShortId>,
    ) -> Option<Reject> {
        let mut ret = None;
        while self.pool_map.total_tx_size > self.config.max_tx_pool_size {
            let next_evict_entry = || {
                self.pool_map
                    .next_evict_entry(Status::Pending)
                    .or_else(|| self.pool_map.next_evict_entry(Status::Gap))
                    .or_else(|| self.pool_map.next_evict_entry(Status::Proposed))
            };

            if let Some(id) = next_evict_entry() {
                let removed = self.pool_map.remove_entry_and_descendants(&id);
                for entry in removed {
                    let tx_hash = entry.transaction().hash();
                    debug!(
                        "Removed by size limit {} timestamp({})",
                        tx_hash, entry.timestamp
                    );
                    let reject = Reject::Full(format!(
                        "the fee_rate for this transaction is: {}",
                        entry.fee_rate()
                    ));
                    if let Some(short_id) = current_entry_id
                        && entry.proposal_short_id() == *short_id
                    {
                        ret = Some(reject.clone());
                    }
                    callbacks.call_reject(self, &entry, reject);
                }
            }
        }
        self.pool_map.entries.shrink_to_fit();
        ret
    }
