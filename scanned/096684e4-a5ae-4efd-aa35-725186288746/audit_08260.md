# LEAKED REPO, WRITE:PACKAGES GITHUB TOKEN

## Summary
The Anatha Desktop Wallet has a bug where it leaks a GitHub key with permissions to access and modify private repositories. This means that anyone who installs the wallet can use this key to push malicious code or packages to the Anatha GitHub organization. The key is located in two files and must be removed before the wallet is deployed. The score for this bug is 5 for both impact and likelihood.

## Details
##### Description

During the assessment, we have identified that the Anatha Desktop Wallet leaks a GitHub key with `repo` and `write:packages` permissions. The Token is leaked in the `[Installation Folder]/resources/app.asar` file. Any attacker who installs the Anatha Desktop Wallet can retrieve this Token and use it to push (backdoor) code to private Anatha GitHub repositories. They can also use to push packages to the GitHub package repo under the Anatha organization.

Code Location
-------------

These are the locations in the codebase where the GitHub Token exists, and which must be removed before deployment:

#### platform/wallet-desktop/electron-builder.env

```
APPLE_ID = "some@mail.com"
APPLE_ID_PASSWORD = "cbgs-jnze-vgpn-cgda"

GH_TOKEN = "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"

```

#### platform/wallet-desktop/src/renderer/electron-shared/Environment.ts

```
static readonly UPDATER_PROVIDER: any = {
    provider: 'github',
    repo: 'wallet-release-test',
    owner: 'anathatech',
    token: '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'
};

```
