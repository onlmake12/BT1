### Title
Expired Network Alert Can Be Replayed via P2P After Deduplication State Is Cleared — (`util/network-alert/src/notifier.rs`, `util/network-alert/src/alert_relayer.rs`)

---

### Summary

The CKB network alert system deduplicates received alerts by `alert_id` (u32) stored in an in-memory `HashMap`. When `clear_expired_alerts()` is called, expired alerts are evicted from this map, resetting the deduplication guard. Because the P2P `received()` handler never checks `notice_until`, any unprivileged peer can replay a previously broadcast, now-expired alert after the deduplication state has been cleared, causing the alert to be re-accepted, re-displayed, and re-broadcast network-wide.

---

### Finding Description

**Root cause — deduplication state is ephemeral and not replay-hardened:**

`Notifier::has_received()` checks only the in-memory `received_alerts` HashMap and the `cancel_filter` LRU cache:

```rust
pub fn has_received(&self, id: u32) -> bool {
    self.received_alerts.contains_key(&id) || self.cancel_filter.contains(&id)
}
``` [1](#0-0) 

`clear_expired_alerts()` removes expired entries from `received_alerts` (but not from `cancel_filter`, which is only populated for explicitly cancelled alerts):

```rust
pub fn clear_expired_alerts(&mut self, now: u64) {
    self.received_alerts.retain(|_id, alert| {
        let notice_until: u64 = alert.raw().notice_until().into();
        notice_until > now
    });
    ...
}
``` [2](#0-1) 

`clear_expired_alerts()` is triggered on every new peer connection and on every `get_blockchain_info()` RPC call: [3](#0-2) [4](#0-3) 

After eviction, `has_received(alert_id)` returns `false` for the expired alert's ID.

**The P2P `received()` handler never checks `notice_until`:**

```rust
async fn received(&mut self, nc: ..., peer_index: PeerIndex, data: Bytes) {
    let alert: packed::Alert = ...;  // parse only
    let alert_id = alert.as_reader().raw().id().into();
    // ignore alert
    if self.notifier.lock().has_received(alert_id) {  // ← only ID check, no expiry check
        return;
    }
    // verify
    if let Err(err) = self.verifier.verify_signatures(&alert) { ... }
    // broadcast + add
    ...
    self.notifier.lock().add(&alert);
}
``` [5](#0-4) 

The `notice_until` check exists **only** in the RPC `send_alert` handler: [6](#0-5) 

The signature covers the full `RawAlert` (all fields including `notice_until`, `id`, `message`, etc.) via `calc_alert_hash()`: [7](#0-6) 

So the signatures remain cryptographically valid forever. The only replay guard — the `received_alerts` HashMap — is periodically wiped.

**Exploit path (step by step):**

1. Attacker observes a legitimate alert (e.g., `id=42`, `notice_until=T`) broadcast on the P2P network and saves the raw bytes (including valid Nervos Foundation signatures).
2. After time `T`, any node that calls `clear_expired_alerts()` removes `id=42` from `received_alerts`. `has_received(42)` now returns `false`.
3. Attacker connects to any CKB node via the public P2P network and sends the saved alert bytes.
4. The node's `received()` handler: parses OK → `has_received(42)` = false → `verify_signatures()` passes (signatures are still valid) → broadcasts to all connected peers → calls `notifier.add()`.
5. `notifier.add()` re-inserts the alert into `received_alerts` and `noticed_alerts`, calls `notify_controller.notify_network_alert()` (re-triggering any configured notification scripts), and the alert propagates network-wide again.

---

### Impact Explanation

- **Network-wide re-propagation** of any previously broadcast alert, triggered by a single unprivileged peer with no keys.
- **Re-execution of operator notification scripts** (`network_alert_notify_script`) on every node that re-accepts the alert, potentially triggering automated responses (e.g., node restarts, paging on-call engineers).
- **Persistent harassment**: the attacker can repeat the replay indefinitely — each time `clear_expired_alerts()` runs, the deduplication guard resets, and the cycle can restart.
- **Confusion / false urgency**: node operators and monitoring systems see a previously resolved alert reappear as active, undermining trust in the alert system at the exact moment it is most needed (during real incidents).

---

### Likelihood Explanation

- Requires no privileged access, no keys, and no special tooling — only the ability to connect to a CKB P2P port (publicly reachable) and replay bytes observed on the network.
- All past alerts and their bytes are observable by any network participant.
- `clear_expired_alerts()` is called on every new peer connection, so the deduplication state resets frequently in a live network.
- The attack is deterministic and reproducible.

---

### Recommendation

1. **Check `notice_until` in the P2P `received()` handler** before accepting an alert, mirroring the check already present in the RPC handler:
   ```rust
   let notice_until: u64 = alert.as_reader().raw().notice_until().into();
   let now_ms = ckb_systemtime::unix_time_as_millis();
   if notice_until < now_ms {
       // ban or silently drop
       return;
   }
   ```
2. **Do not evict expired alert IDs from the deduplication set.** Move expired alert IDs into the `cancel_filter` (or a separate permanent seen-set) instead of simply dropping them, so that `has_received()` continues to return `true` for IDs that were ever accepted.

---

### Proof of Concept

```
// Preconditions: alert with id=42, notice_until=T was broadcast at time T-Δ.
// Attacker saved the raw alert bytes (public, observed on P2P network).

// Step 1: Wait until T passes and nodes call clear_expired_alerts().
//         This happens on every new peer connection or get_blockchain_info() call.

// Step 2: Connect to any CKB node via P2P (no authentication required).

// Step 3: Send the saved alert bytes over the Alert protocol channel.

// Step 4: Node executes received():
//   - has_received(42) → false  (cleared by clear_expired_alerts)
//   - verify_signatures() → Ok  (signatures are still cryptographically valid)
//   - quick_filter_broadcast() → alert re-sent to all connected peers
//   - notifier.add() → alert re-displayed, notification scripts re-triggered

// Step 5: Repeat from Step 3 after each clear_expired_alerts() cycle.
```

The `notice_until` field is part of the signed `RawAlert` and cannot be altered without invalidating the signature. The attacker replays the original bytes verbatim — no forgery is required. [8](#0-7) [9](#0-8)

    pub fn add(&mut self, alert: &Alert) {
        let alert_id = alert.raw().id().into();
        let alert_cancel = alert.raw().cancel().into();
        if self.has_received(alert_id) {
            return;
        }
        // checkout cancel_id
        if alert_cancel > 0 {

        self.received_alerts.insert(alert_id, alert.clone());

            return;

        self.noticed_alerts.push(alert.clone());
        // sort by priority
        self.noticed_alerts.sort_by_key(|a| {
            let priority: u32 = a.raw().priority().into();

    async fn received(
        &mut self,
        nc: Arc<dyn CKBProtocolContext + Sync>,
        peer_index: PeerIndex,
        data: Bytes,
    ) {
        let alert: packed::Alert = match packed::AlertReader::from_slice(&data) {
            Ok(alert) => {
                if alert.raw().message().is_utf8()
                    && alert
                        .raw()
                        .min_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                    && alert
                        .raw()
                        .max_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                {
                    alert.to_entity()
                } else {
                    info!(
                        "A malformed message fromP peer {} : not utf-8 string",
                        peer_index
                    );
                    nc.ban_peer(
                        peer_index,
                        BAD_MESSAGE_BAN_TIME,
                        String::from("send us a malformed message: not utf-8 string"),
                    );
                    return;
                }
            }
            Err(err) => {
                info!("A malformed message from peer {}: {:?}", peer_index, err);
                nc.ban_peer(
                    peer_index,
                    BAD_MESSAGE_BAN_TIME,
                    String::from("send us a malformed message"),
                );
                return;
            }
        };
        let alert_id = alert.as_reader().raw().id().into();
        trace!("ReceiveD alert {} from peer {}", alert_id, peer_index);
        // ignore alert
        if self.notifier.lock().has_received(alert_id) {
            return;
        }
        // verify
        if let Err(err) = self.verifier.verify_signatures(&alert) {
            debug!(
                "An alert from peer {} with invalid signatures, error {:?}",
                peer_index, err
            );
            nc.ban_peer(
                peer_index,
                BAD_MESSAGE_BAN_TIME,
                String::from("send us an alert with invalid signatures"),
            );
            return;
        }
        // mark sender as known
        self.mark_as_known(peer_index, alert_id);
        // broadcast message
        let selected_peers: Vec<PeerIndex> = nc
            .connected_peers()
            .into_iter()
            .filter(|peer| self.mark_as_known(*peer, alert_id))
            .collect();
        if let Err(err) = nc.quick_filter_broadcast(
            TargetSession::Multi(Box::new(selected_peers.into_iter())),
            data,
        ) {
            debug!("alert broadcast error: {:?}", err);
        }
        // add to received alerts
        self.notifier.lock().add(&alert);
    }
