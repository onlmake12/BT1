### Title
Missing Duplicate-Input Guard in `ResolvedTransaction::check()` Allows Per-Transaction Capacity Inflation — (File: `util/types/src/core/cell.rs`)

---

### Summary

`ResolvedTransaction::check()` uses a single `checked_cells` set for both cell-dep deduplication and input liveness checks. Because `seen_inputs` is only extended *after* all inputs are iterated, a transaction that lists the same `OutPoint` twice in its inputs passes the check silently. No separate duplicate-input verifier exists anywhere in the non-contextual or contextual transaction verification pipeline. As a result, `inputs_capacity()` sums the same cell's capacity N times, and the `CapacityVerifier` accepts outputs whose total capacity exceeds the actual on-chain cell capacity — creating CKB out of thin air.

---

### Finding Description

**Root cause — `util/types/src/core/cell.rs`, `ResolvedTransaction::check()`**

```
pub fn check<CC, HC, S>(
    &self,
    seen_inputs: &mut HashSet<OutPoint, S>,   // cross-tx double-spend set
    cell_checker: &CC,
    header_checker: &HC,
) -> Result<(), OutPointError> {
    let mut checked_cells: HashSet<OutPoint> = HashSet::new();   // ← local, per-call
    let mut check_cell = |out_point: &OutPoint| {
        if seen_inputs.contains(out_point) {          // cross-tx guard
            return Err(OutPointError::Dead(...));
        }
        if checked_cells.contains(out_point) {        // ← silently passes duplicates
            return Ok(());
        }
        // liveness check …
        checked_cells.insert(out_point.clone());
        Ok(())
    };

    for cell_meta in &self.resolved_inputs {
        check_cell(&cell_meta.out_point)?;            // ← duplicate passes on 2nd iter
    }
    // … deps …
    seen_inputs.extend(                               // ← extended AFTER the loop
        self.resolved_inputs.iter().map(|i| &i.out_point).cloned()
    );
    Ok(())
}
``` [1](#0-0) 

Walk-through for a transaction whose `inputs` list contains `OutPoint X` twice:

| Iteration | `seen_inputs` contains X? | `checked_cells` contains X? | Result |
|---|---|---|---|
| 1st | No | No | liveness check → live → insert into `checked_cells` → **Ok** |
| 2nd | No (not extended yet) | **Yes** | early-return **Ok** — no error |

After the loop, `seen_inputs` is extended with X once (from the iterator over `resolved_inputs`), so the cross-transaction guard is also not triggered for subsequent transactions.

**No duplicate-input verifier exists**

`NonContextualTransactionVerifier` contains `DuplicateDepsVerifier` for cell-deps and header-deps, but there is no analogous `DuplicateInputsVerifier`. [2](#0-1) 

`DuplicateDepsVerifier::verify()` explicitly iterates only `cell_deps_iter()` and `header_deps_iter()` — inputs are never checked. [3](#0-2) 

**Capacity inflation**

`inputs_capacity()` is a plain sum over `resolved_inputs`:

```rust
pub fn inputs_capacity(&self) -> CapacityResult<Capacity> {
    self.resolved_inputs
        .iter()
        .map(CellMeta::capacity)
        .try_fold(Capacity::zero(), Capacity::safe_add)
}
``` [4](#0-3) 

If `resolved_inputs` contains the same `CellMeta` N times, `inputs_capacity()` returns `N × cell.capacity`. The `CapacityVerifier` (part of `ContextualTransactionVerifier`) then accepts outputs whose total capacity equals up to `N × cell.capacity`, while only one cell of capacity `cell.capacity` is actually consumed. [5](#0-4) 

**Tx-pool does not protect block validation**

The tx-pool's `Edges::insert_input` maps `OutPoint → ProposalShortId` and would reject a duplicate-input transaction at submission time. However, this check is entirely absent from the block-validation path (`chain/src/verify.rs` → `ResolvedTransaction::check()`). A miner who directly assembles a block bypasses the tx-pool entirely. [6](#0-5) 

---

### Impact Explanation

A miner who crafts a block containing a transaction with N copies of the same input `OutPoint` can produce outputs whose total capacity is up to `N × C` (where `C` is the capacity of the referenced cell), while only destroying one cell of capacity `C`. This inflates the total CKB supply by `(N−1) × C` shannons per such transaction. The block passes all consensus validation on every full node, so the inflation is permanently committed to the canonical chain.

---

### Likelihood Explanation

Any miner — no privileged key or majority hashpower required — can craft such a block. The PoW puzzle must be solved, but that is the normal cost of block production. On mainnet the cost is non-trivial; on testnet/devnet it is negligible. The attack is silent: no existing error path in block validation rejects the transaction.

---

### Recommendation

Add a `DuplicateInputsVerifier` to `NonContextualTransactionVerifier`, analogous to the existing `DuplicateDepsVerifier`:

```rust
// In NonContextualTransactionVerifier::verify():
self.duplicate_inputs.verify()?;
```

```rust
pub struct DuplicateInputsVerifier<'a> {
    transaction: &'a TransactionView,
}

impl<'a> DuplicateInputsVerifier<'a> {
    pub fn verify(&self) -> Result<(), Error> {
        let mut seen = HashSet::with_capacity(self.transaction.inputs().len());
        if let Some(input) = self.transaction
            .inputs_iter()
            .find_map(|i| seen.replace(i.previous_output()))
        {
            return Err(TransactionError::DuplicateInputs { out_point: input }.into());
        }
        Ok(())
    }
}
```

Alternatively, inside `ResolvedTransaction::check()`, replace the `checked_cells` early-return for inputs with an explicit duplicate error:

```rust
for cell_meta in &self.resolved_inputs {

### Proof of Concept

1. Obtain a live cell `C` with capacity 100 CKB.
2. Construct a transaction:
   - `inputs`: `[OutPoint(C), OutPoint(C)]` — the same out-point twice.
   - `outputs`: one cell with capacity 199 CKB (99 CKB net gain; 1 CKB fee).
3. Mine a block that includes this transaction (bypassing the tx-pool).
4. Broadcast the block. Every full node calls `ResolvedTransaction::check()`:
   - First input `OutPoint(C)`: not in `seen_inputs`, not in `checked_cells` → liveness check passes → inserted into `checked_cells`.
   - Second input `OutPoint(C)`: not in `seen_inputs`, **is** in `checked_cells` → `Ok(())` returned immediately.
5. `inputs_capacity()` returns `100 + 100 = 200 CKB`.
6. `CapacityVerifier` accepts `199 CKB` outputs ≤ `200 CKB` inputs.
7. Block is committed; 99 CKB has been created from nothing. [7](#0-6) [2](#0-1)

    pub fn check<CC: CellChecker, HC: HeaderChecker, S: BuildHasher>(
        &self,
        seen_inputs: &mut HashSet<OutPoint, S>,
        cell_checker: &CC,
        header_checker: &HC,
    ) -> Result<(), OutPointError> {
        let mut checked_cells: HashSet<OutPoint> = HashSet::new();
        let mut check_cell = |out_point: &OutPoint| -> Result<(), OutPointError> {
            if seen_inputs.contains(out_point) {
                return Err(OutPointError::Dead(out_point.clone()));
            }

            if checked_cells.contains(out_point) {
                return Ok(());
            }

            match cell_checker.is_live(out_point) {
                Some(true) => {
                    checked_cells.insert(out_point.clone());
                    Ok(())
                }
                Some(false) => Err(OutPointError::Dead(out_point.clone())),
                None => Err(OutPointError::Unknown(out_point.clone())),
            }
        };

        // // check input
        for cell_meta in &self.resolved_inputs {
            check_cell(&cell_meta.out_point)?;
        }

        let mut resolved_system_deps: HashSet<&OutPoint> = HashSet::new();
        if let Some(system_cell) = SYSTEM_CELL.get() {
            for cell_meta in &self.resolved_dep_groups {
                let cell_dep = CellDep::new_builder()
                    .out_point(cell_meta.out_point.clone())
                    .dep_type(DepType::DepGroup)
                    .build();

                let dep_group = system_cell.get(&cell_dep);
                if let Some(ResolvedDep::Group(_, cell_deps)) = dep_group {
                    resolved_system_deps.extend(cell_deps.iter().map(|dep| &dep.out_point));
                } else {
                    check_cell(&cell_meta.out_point)?;
                }
            }

            for cell_meta in &self.resolved_cell_deps {
                let cell_dep = CellDep::new_builder()
                    .out_point(cell_meta.out_point.clone())
                    .dep_type(DepType::Code)
                    .build();

                if system_cell.get(&cell_dep).is_none()
                    && !resolved_system_deps.contains(&cell_meta.out_point)
                {
                    check_cell(&cell_meta.out_point)?;
                }
            }
        } else {
            for cell_meta in self
                .resolved_cell_deps
                .iter()
                .chain(self.resolved_dep_groups.iter())
            {
                check_cell(&cell_meta.out_point)?;
            }
        }

        for block_hash in self.transaction.header_deps_iter() {
            header_checker.check_valid(&block_hash)?
        }

        seen_inputs.extend(self.resolved_inputs.iter().map(|i| &i.out_point).cloned());

        Ok(())
    }

    pub fn verify(&self) -> Result<(), Error> {
        let transaction = self.transaction;
        let mut seen_cells = HashSet::with_capacity(self.transaction.cell_deps().len());
        let mut seen_headers = HashSet::with_capacity(self.transaction.header_deps().len());

        if let Some(dep) = transaction
            .cell_deps_iter()
            .find_map(|dep| seen_cells.replace(dep))
        {
            return Err(TransactionError::DuplicateCellDeps {
                out_point: dep.out_point(),
            }
            .into());
        }
        if let Some(hash) = transaction
            .header_deps_iter()
            .find_map(|hash| seen_headers.replace(hash))
        {
            return Err(TransactionError::DuplicateHeaderDeps { hash }.into());
        }
        Ok(())
    }
