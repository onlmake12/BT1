Audit Report

## Title
Out-of-Bounds Panic in `TxConfirmStat::remove_unconfirmed_tx` When `tx_age > MAX_CONFIRM_BLOCKS` — (`util/fee-estimator/src/estimator/confirmation_fraction.rs`)

## Summary

`confirm_blocks_to_failed_txs` is allocated with length `MAX_CONFIRM_BLOCKS = 1000` (valid indices `0..=999`). In `remove_unconfirmed_tx`, the existing bounds guard at line 208 protects only the `block_unconfirmed_txs` array; the subsequent indexing of `confirm_blocks_to_failed_txs[tx_age - 1]` at line 215 has no equivalent guard. When a tracked transaction is evicted after more than 1000 blocks, `tx_age - 1 >= 1000` exceeds the array length and causes a Rust index-out-of-bounds panic, crashing the node process.

## Finding Description

`confirm_blocks_to_failed_txs` is initialized with length `max_confirm_blocks` (= `MAX_CONFIRM_BLOCKS = 1000`): [1](#0-0) 

In `remove_unconfirmed_tx`, the guard at line 208 only protects `block_unconfirmed_txs`: [2](#0-1) 

When `tx_age >= 1001`, the branch at line 208 takes the `old_unconfirmed_txs` path (correctly), but execution falls through to line 215 where `confirm_blocks_to_failed_txs[tx_age - 1]` is indexed unconditionally. With `tx_age = 1001`, the index is `1000`, which is out of bounds for a slice of length `1000`, producing a Rust panic.

The reachable call chain is: `reject_tx` (line 475) → `drop_tx` (line 428) → `drop_tx_inner(tx_hash, true)` (line 416) → `remove_unconfirmed_tx(..., count_failure=true)` (line 418). The `count_failure=true` argument is hardcoded in `drop_tx`, so every eviction via `reject_tx` reaches the vulnerable line. [3](#0-2) 

The `reject_tx` callback is registered for all tx-pool eviction events (time expiry, pool size limit, conflict): [4](#0-3) 

## Impact Explanation

The panic unwinds through the fee estimator write-lock path and terminates the CKB node process. This matches the allowed bounty impact: **High (10001–15000 points) — Vulnerabilities which could easily crash a CKB node.**

## Likelihood Explanation

The `ConfirmationFraction` estimator is opt-in; the default is `FeeEstimator::Dummy`. Nodes that have set `fee_estimator.algorithm = "ConfirmationFraction"` in their config are affected. For such nodes, no special attacker capability is required: submitting a valid low-fee-rate transaction and waiting for 1001 blocks (~2.8 hours at 10 s/block, well within the 12-hour default expiry) is sufficient. The next pool eviction event (size pressure or expiry) will call `reject_tx` and trigger the panic. The condition is passively reachable and repeatable.

## Recommendation

Add a bounds check before the `confirm_blocks_to_failed_txs` indexing, mirroring the existing guard for `block_unconfirmed_txs`:

```rust
if count_failure && tx_age <= self.confirm_blocks_to_failed_txs.len() {
    self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64;
}
```

## Proof of Concept

```rust
// In a TxConfirmStat unit test:
let mut stat = TxConfirmStat::default(); // max_confirm_blocks = 1000
// Simulate tx entered at height 0, tip is now 1001 → tx_age = 1001
let bucket_index = stat.add_unconfirmed_tx(0, some_fee_rate).unwrap();
// Panics: index out of bounds: the len is 1000 but the index is 1000
stat.remove_unconfirmed_tx(0, 1001, bucket_index, true);
```

`tx_age - 1 = 1000` exceeds `confirm_blocks_to_failed_txs.len() = 1000` (valid range `0..=999`), producing `thread panicked at 'index out of bounds: the len is 1000 but the index is 1000'`. [5](#0-4)

```rust
    fn remove_unconfirmed_tx(
        &mut self,
        entry_height: u64,
        tip_height: u64,
        bucket_index: usize,
        count_failure: bool,
    ) {
        let tx_age = tip_height.saturating_sub(entry_height) as usize;
        if tx_age < 1 {
            return;
        }
        if tx_age >= self.block_unconfirmed_txs.len() {
            self.bucket_stats[bucket_index].old_unconfirmed_txs -= 1;
        } else {
            let block_index = (entry_height % self.block_unconfirmed_txs.len() as u64) as usize;
            self.block_unconfirmed_txs[block_index][bucket_index] -= 1;
        }
        if count_failure {
            self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64;
        }
    }
```
