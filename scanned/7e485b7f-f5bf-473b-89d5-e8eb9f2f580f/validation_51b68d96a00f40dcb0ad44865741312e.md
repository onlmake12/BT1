### Title
Attacker-Controlled Peer ID Stored as Local Observed Address via Identify Protocol — (`network/src/protocols/identify/mod.rs`)

### Summary

The `add_observed_addr` function in `IdentifyCallback` only appends the local node's peer ID to an observed address when no `/p2p/` component is present. An attacker can send an `observed_addr` that already contains a `/p2p/<attacker_peer_id>` component, bypassing the guard entirely. The poisoned address is then stored as the victim's own observed address and subsequently broadcast to all peers via `local_listen_addrs`.

### Finding Description

The vulnerable guard in `IdentifyCallback::add_observed_addr`:

```rust
fn add_observed_addr(&mut self, mut addr: Multiaddr, session_id: SessionId) -> MisbehaveResult {
    if extract_peer_id(&addr).is_none() {          // ← only appends local ID if /p2p/ is absent
        addr.push(Protocol::P2P(Cow::Borrowed(
            self.network_state.local_peer_id().as_bytes(),
        )))
    }
    self.network_state.add_observed_addr(session_id, addr);  // ← stores attacker-supplied addr
    MisbehaveResult::Continue
}
``` [1](#0-0) 

If the attacker sends `observed_addr = /ip4/<victim_ip>/tcp/8114/p2p/<attacker_peer_id>`, then `extract_peer_id` returns `Some(attacker_peer_id)`, the `if` branch is skipped, and the address with the wrong peer ID is stored verbatim.

The stored observed addresses are then included in `local_listen_addrs`, which is called on every new `connected` event to build the identify message sent to new peers:

```rust
fn local_listen_addrs(&mut self) -> Vec<Multiaddr> {
    let mut listens = self.listen_addrs();
    if listens.len() < MAX_RETURN_LISTEN_ADDRS {
        let observe_addrs = self.network_state.observed_addrs(...);
        listens.extend(observe_addrs);   // ← poisoned addr included here
    }
    listens
}
``` [2](#0-1) 

This is sent to every new peer on `connected`:

```rust

let data = IdentifyMessage::new(listen_addrs, session.address.clone(), identify).encode();

``` [3](#0-2)

### Impact Explanation

Any peer receiving the victim's identify message will store `/ip4/<victim_ip>/tcp/8114/p2p/<attacker_peer_id>` as a reachable address for the victim. When those peers attempt to connect to the victim using this address, the secio/TLS handshake will fail because the victim's actual peer ID does not match `<attacker_peer_id>`. This causes:

- **Identity spoofing**: The victim node advertises a peer ID it does not own.
- **Network routing disruption**: Peers that learned the victim's address from the poisoned identify message cannot connect to the victim node.
- **Persistence**: The poisoned address propagates transitively — peers that received it will include it in their own `listen_addrs` broadcasts to further peers.

### Likelihood Explanation

Any unprivileged peer that can establish a valid identify session (i.e., any peer on the same network with a matching network name) can trigger this. There is no authentication, rate-limiting, or peer-ID validation on the `observed_addr` field. The attacker only needs to craft a single `IdentifyMessage` with a `/p2p/` suffix containing an arbitrary peer ID.

### Recommendation

In `add_observed_addr`, unconditionally strip any existing `/p2p/` component from the incoming `observed_addr` before appending the local peer ID. The correct invariant is: the local node's peer ID is always authoritative for its own observed addresses. The fix should replace the current `if extract_peer_id(&addr).is_none()` guard with logic that removes any existing `Protocol::P2P(...)` component and then always appends `Protocol::P2P(local_peer_id)`.

### Proof of Concept

```rust
// Unit test sketch for add_observed_addr
let attacker_peer_id = PeerId::random();
let victim_ip = "/ip4/1.2.3.4/tcp/8114".parse::<Multiaddr>().unwrap();
let poisoned_addr = victim_ip.with(Protocol::P2P(attacker_peer_id.as_bytes().into()));

// Call add_observed_addr with the poisoned addr
callback.add_observed_addr(poisoned_addr, session_id);

// Retrieve stored observed addr
let stored = network_state.observed_addrs(10);
let stored_peer_id = extract_peer_id(&stored[0]).unwrap();

// This assertion FAILS — stored peer ID is attacker's, not local node's
assert_eq!(stored_peer_id, *network_state.local_peer_id());
```
