### Title
Missing Duplicate-Index Deduplication and Response Byte-Size Limit in `GetBlockTransactionsProcess::execute` Allows Multi-Peer Memory Amplification — (`sync/src/relayer/get_block_transactions_process.rs`)

---

### Summary

`GetBlockTransactionsProcess::execute` accepts up to `MAX_RELAY_TXS_NUM_PER_BATCH` (32767) transaction indexes per request but performs **no deduplication** and **no response byte-size check**. An attacker with many simultaneous peer connections can send requests with 32767 identical valid indexes pointing to a large stored block transaction, causing the node to allocate a serialized response buffer proportional to `32767 × tx_size` per request. With 128 peers each sending 30 req/sec (the per-peer rate limit), thousands of concurrent `execute()` calls can hold enormous in-memory allocations simultaneously, exhausting node memory.

---

### Finding Description

**Entrypoint**: Any unprivileged remote peer on the `RelayV3` P2P protocol (`/ckb/relay3`) can send a `GetBlockTransactions` message. The incoming message is dispatched through `Relayer::try_process` → `GetBlockTransactionsProcess::execute`.

**Rate limiter**: The rate limiter is keyed by `(PeerIndex, message.item_id())` at 30 req/sec per peer independently. [1](#0-0) [2](#0-1) 

With `MAX_RELAY_PEERS = 128`, the aggregate throughput ceiling is 128 × 30 = **3840 req/sec** of `GetBlockTransactions` messages reaching `execute()`. [3](#0-2) 

**The only guard in `execute()`** is a count check: [4](#0-3) 

This allows up to 32767 indexes. There is **no deduplication** and **no byte-size limit** on the response. The `filter_map` then collects one cloned `TransactionView` per index, including duplicates: [5](#0-4) 

The builder then serializes all collected transactions into a single flat byte buffer: [6](#0-5) 

This flat serialization (molecule format) allocates `32767 × tx_size` bytes in memory **before** `async_send_message_to` is called. Even if the send subsequently fails due to the 4MB `RelayV3` frame limit, the allocation has already occurred. [7](#0-6) 

**Contrast with sibling handlers** that correctly guard against this:

- `GetTransactionsProcess::execute` explicitly rejects duplicate hashes with `StatusCode::RequestDuplicate` and enforces `MAX_RELAY_TXS_BYTES_PER_BATCH` on the response. [8](#0-7) [9](#0-8) 

- `GetBlockProposalProcess::execute` deduplicates via `HashSet` and enforces `MAX_RELAY_TXS_BYTES_PER_BATCH`. [10](#0-9) [11](#0-10) 

`GetBlockTransactionsProcess` has neither protection. [12](#0-11) 

---

### Impact Explanation

- **Incoming message size**: 32767 × 4 bytes (uint32 indexes) ≈ 128KB — well within the 4MB `RelayV3` frame limit, so the message is accepted.
- **Outgoing allocation**: `32767 × tx_size`. For a moderately large transaction (e.g., 100KB), this is ~3.2GB per request. Even for a 10KB transaction, it is ~320MB per request.
- **Concurrent pressure**: 3840 req/sec across 128 peers, each holding a large allocation until `async_send_message_to` completes or fails, can exhaust available RAM and crash the node (OOM kill or Rust allocator panic).
- **Block data is public**: The attacker only needs a valid stored block hash and a valid transaction index within it — both are trivially discoverable from the public chain.

---

### Likelihood Explanation

The attack requires only 128 simultaneous TCP connections to the target node (the default `max_peers` for a CKB node is typically 125–500) and knowledge of any stored block containing a large transaction. No privileged access, no PoW, no key material is needed. The attack is fully scriptable and locally testable.

---

### Recommendation

Apply both fixes that sibling handlers already use:

1. **Deduplicate indexes** before processing: collect indexes into a `HashSet<u32>`, reject with `StatusCode::RequestDuplicate` if `set.len() != indexes.len()`.
2. **Enforce `MAX_RELAY_TXS_BYTES_PER_BATCH`** on the response: accumulate `tx.data().total_size()` and either truncate or split the response when the byte limit is reached, mirroring the pattern in `GetBlockProposalProcess` and `GetTransactionsProcess`.

---

### Proof of Concept

```
1. Identify any stored block hash H containing a large transaction at index 0.
2. Open 128 TCP connections to the target node on the RelayV3 protocol.
3. From each connection, send 30 GetBlockTransactions/sec with:
     block_hash = H
     indexes    = [0, 0, 0, ..., 0]  (32767 copies of index 0)
4. Each execute() call allocates ~32767 × tx_size bytes before attempting to send.
5. Monitor target node RSS; expect OOM crash within seconds.
```

    pub async fn execute(self) -> Status {
        let shared = self.relayer.shared();
        {
            let get_block_transactions = self.message;
            if get_block_transactions.indexes().len() > MAX_RELAY_TXS_NUM_PER_BATCH {
                return StatusCode::ProtocolMessageIsMalformed.with_context(format!(
                    "Indexes count({}) > MAX_RELAY_TXS_NUM_PER_BATCH({})",
                    get_block_transactions.indexes().len(),
                    MAX_RELAY_TXS_NUM_PER_BATCH,
                ));
            }
            if get_block_transactions.uncle_indexes().len() > shared.consensus().max_uncles_num() {
                return StatusCode::ProtocolMessageIsMalformed.with_context(format!(
                    "UncleIndexes count({}) > consensus max_uncles_num({})",
                    get_block_transactions.uncle_indexes().len(),
                    shared.consensus().max_uncles_num(),
                ));
            }
        }

        let block_hash = self.message.block_hash().to_entity();
        debug_target!(
            crate::LOG_TARGET_RELAY,
            "get_block_transactions {}",
            block_hash
        );

        if let Some(block) = shared.store().get_block(&block_hash) {
            let transactions = self
                .message
                .indexes()
                .iter()
                .filter_map(|i| {
                    block
                        .transactions()
                        .get(Into::<u32>::into(i) as usize)
                        .cloned()
                })
                .collect::<Vec<_>>();

            let uncles = self
                .message
                .uncle_indexes()
                .iter()
                .filter_map(|i| block.uncles().get(Into::<u32>::into(i) as usize))
                .collect::<Vec<_>>();

            let content = packed::BlockTransactions::new_builder()
                .block_hash(block_hash)
                .transactions(
                    transactions
                        .into_iter()
                        .map(|tx| tx.data())
                        .collect::<Vec<_>>(),
                )
                .uncles(
                    uncles
                        .into_iter()
                        .map(|uncle| uncle.data())
                        .collect::<Vec<_>>(),
                )
                .build();
            let message = packed::RelayMessage::new_builder().set(content).build();

            return async_send_message_to(&self.nc, self.peer, &message).await;
        }

        Status::ok()
    }
