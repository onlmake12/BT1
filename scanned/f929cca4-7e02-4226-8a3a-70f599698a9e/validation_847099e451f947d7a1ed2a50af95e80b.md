### Title
Ignored `open_protocols` Return Value in `IdentifyCallback::received_identify` Silently Fails to Establish Peer Protocols — (File: `network/src/protocols/identify/mod.rs`)

---

### Summary

In `IdentifyCallback::received_identify`, after an inbound peer passes all identify checks (network name, flags, required protocol support), the node calls `context.open_protocols(...)` to negotiate and open the actual working sub-protocols (Sync, Relay, Discovery, etc.) with that peer. The `Result` returned by both `open_protocols` call-sites is unconditionally discarded with `let _ = ...`. If either call fails, the TCP session remains alive, the peer is counted as a connected peer consuming a connection slot, but **no working sub-protocols are ever opened**. The function then returns `MisbehaveResult::Continue`, signalling success to the caller.

---

### Finding Description

`IdentifyCallback::received_identify` is the handler that runs when a remote peer sends its identify handshake. After verifying the peer belongs to the same network and advertises the required capability flags, the code branches into two `open_protocols` calls:

```rust
// branch 1 – feeler upgrade
let _ = context
    .open_protocols(
        context.session.id,
        TargetProtocol::Single(SupportProtocols::Feeler.protocol_id()),
    )
    .await;

// branch 2 – full protocol upgrade
let _ = context
    .open_protocols(
        context.session.id,
        TargetProtocol::Filter(Box::new(move |id| {
            id != &SupportProtocols::Feeler.protocol_id()
        })),
    )
    .await;
```

Both sites use `let _ = …` to throw away the `Result`. If `open_protocols` returns `Err` (e.g., the underlying Tentacle service queue is full, the session was concurrently closed, or an internal channel error occurs), the error is silently swallowed. The function then falls through and returns `MisbehaveResult::Continue`, which tells the identify state machine that the session is healthy and should be kept open.

The root cause is structurally identical to the ERC-20 `approve` pattern in the reference report: a call whose return value encodes success/failure is made, the return value is discarded, and the surrounding logic proceeds as if the call succeeded.

---

### Impact Explanation

A peer that has passed the identify handshake but for which `open_protocols` silently failed will:

1. **Occupy a connection slot** in the peer registry without contributing any protocol traffic (no block sync, no transaction relay, no discovery).
2. **Never be detected as broken** — because the identify protocol already marked it as `has_received = true` and returned `Continue`, no timeout or retry logic will fire for it.
3. **Reduce effective peer connectivity** — if several slots are consumed by such "zombie" sessions, the node's ability to download blocks and propagate transactions degrades proportionally.

Because the node's maximum peer count is bounded (configured via `max_peers`), filling slots with non-functional sessions is a reachable availability impact.

---

### Likelihood Explanation

The `open_protocols` call dispatches a command through the Tentacle service's internal channel. That channel can return an error when:

- The session is concurrently closed between the identify check and the `open_protocols` dispatch (a natural race on any busy node).
- The Tentacle service's command queue is temporarily full under high connection churn.

An unprivileged remote peer triggers this code path simply by completing the TCP handshake and sending a well-formed identify message. No special privilege is required. The race window is narrow but reproducible under connection-churn conditions (e.g., a peer that connects, sends identify, and immediately resets the TCP connection).

---

### Recommendation

Replace the `let _ = …` discards with explicit error handling. At minimum, log the error and disconnect the session so the slot is freed:

```rust
if let Err(e) = context
    .open_protocols(context.session.id, TargetProtocol::Single(...))
    .await
{
    error!("open_protocols failed for session {:?}: {:?}", context.session.id, e);
    let _ = context.disconnect(context.session.id).await;
    return MisbehaveResult::Disconnect;
}
```

The same pattern should be applied to the second `open_protocols` call-site and, for consistency, to the `disconnect` call-sites that also use `let _ = …` (lines 265, 277, 287, 310, 321).

---

### Proof of Concept

1. Attacker peer A connects to the CKB node.
2. A sends a syntactically valid `Identify` message with the correct network name and a flags value that satisfies `required_flags_filter`.
3. Immediately after sending the identify message, A resets the TCP connection (sends RST).
4. The CKB node's `received_identify` handler fires; all checks pass; it calls `context.open_protocols(...)`.
5. Because the session is already gone, Tentacle returns `Err`; the error is discarded by `let _ = …`.
6. The handler returns `MisbehaveResult::Continue`; the peer registry entry remains with `has_received = true`.
7. The slot is never reclaimed until the node's own keep-alive / ping timeout fires (up to minutes later), during which the slot is wasted.
8. Repeating this at scale (many short-lived connections) exhausts the node's `max_peers` budget, degrading its ability to maintain real sync peers.

    async fn received_identify(
        &mut self,
        context: &mut ProtocolContextMutRef<'_>,
        identify: &[u8],
    ) -> MisbehaveResult {
        match self.identify.verify(identify) {
            None => {
                self.network_state.ban_session(
                    &context.control().clone().into(),
                    context.session.id,
                    BAN_ON_NOT_SAME_NET,
                    "The nodes are not on the same network".to_string(),
                );
                MisbehaveResult::Disconnect
            }
            Some((flags, client_version)) => {
                let registry_client_version = |version: String| {
                    self.network_state.with_peer_registry_mut(|registry| {
                        if let Some(peer) = registry.get_peer_mut(context.session.id) {
                            peer.identify_info = Some(PeerIdentifyInfo {
                                client_version: version,
                                flags,
                            })
                        }
                    });
                };

                registry_client_version(client_version);

                let required_flags = self.network_state.required_flags;

                if context.session.ty.is_outbound() {
                    // why don't set inbound here?
                    // because inbound address can't feeler during staying connected
                    // and if set it to peer store, it will be broadcast to the entire network,
                    // but this is an unverified address

                    self.network_state.with_peer_store_mut(|peer_store| {
                        peer_store.add_outbound_addr(context.session.address.clone(), flags);
                    });

                    if self.network_state.with_peer_registry_mut(|reg| {
                        reg.change_feeler_flags(&context.session.address, flags)
                    }) {
                        let _ = context
                            .open_protocols(
                                context.session.id,
                                TargetProtocol::Single(SupportProtocols::Feeler.protocol_id()),
                            )
                            .await;
                    } else if required_flags_filter(required_flags, flags) {
                        // The remote end can support all local protocols.
                        let _ = context
                            .open_protocols(
                                context.session.id,
                                TargetProtocol::Filter(Box::new(move |id| {
                                    id != &SupportProtocols::Feeler.protocol_id()
                                })),
                            )
                            .await;
                    } else {
                        // The remote end cannot support all local protocols.
                        warn!(
                            "Session closed from IdentifyProtocol due to peer's flag not meeting the requirements"
                        );
                        return MisbehaveResult::Disconnect;
                    }
                }
                MisbehaveResult::Continue
            }
        }
    }
