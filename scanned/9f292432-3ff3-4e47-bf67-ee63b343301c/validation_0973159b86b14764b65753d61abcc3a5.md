### Title
Cross-Network Alert Signature Replay: `RawAlert` Signed Message Lacks Network Domain Separator — (`File: util/network-alert/src/verifier.rs`, `util/gen-types/schemas/extensions.mol`)

---

### Summary

The CKB network alert system signs a `RawAlert` message that contains no network/chain domain separator (no genesis hash, no chain ID, no network name). Because the same four hard-coded signing keys are used across all CKB networks (mainnet, testnet, and any custom network running the default config), a valid alert signature produced for one network is cryptographically valid on every other CKB network. Any unprivileged P2P peer can capture a legitimately-signed alert from testnet and replay it verbatim to mainnet nodes, causing all mainnet nodes to display a false or misleading critical-bug warning.

---

### Finding Description

**Signed message construction — no domain separator**

The `RawAlert` molecule schema defines the fields that are hashed and signed: [1](#0-0) 

The fields are: `notice_until`, `id`, `cancel`, `priority`, `message`, `min_version`, `max_version`. There is no `genesis_hash`, no `chain_id`, and no `network_name` field. The hash is computed by `calc_alert_hash`, which simply calls `blake2b_256(self.as_slice())` — a plain hash of the raw bytes with no domain prefix: [2](#0-1) 

**Signature verification — no network check**

`Verifier::verify_signatures` derives the message to verify from `alert.calc_alert_hash()` and checks it against the configured public keys. There is no step that binds the alert to a specific network: [3](#0-2) 

**Same keys used across all networks**

The default `NetworkAlertConfig` loads a single hard-coded `alert_signature.toml` containing four fixed public keys. This same default is used for both mainnet and testnet nodes: [4](#0-3) [5](#0-4) 

**P2P relay handler — unprivileged entry point**

The `AlertRelayer::received` handler accepts alert messages from any connected P2P peer. After parsing and deduplication, it calls `verify_signatures` and, on success, broadcasts the alert to all connected peers and adds it to the local notifier — with no check that the alert was intended for this network: [6](#0-5) 

---

### Impact Explanation

An attacker who relays a legitimately-signed testnet alert to mainnet nodes causes every receiving mainnet node to:
1. Accept the alert as cryptographically valid (signatures check out against the same keys).
2. Store and display the alert message to operators.
3. Re-broadcast the alert to all of their connected peers, achieving full network propagation.

The alert system is the CKB team's emergency channel for warning about critical bugs. A false alert injected by a replay attacker could cause operators to panic, halt nodes, or take other disruptive actions based on a message that was never intended for mainnet. The `notice_until` field provides a time bound, but any testnet alert that has not yet expired is immediately replayable.

---

### Likelihood Explanation

The attacker preconditions are minimal:
- Connect to the CKB testnet as a normal P2P peer (no privilege required).
- Observe a valid alert being relayed on testnet (alerts are broadcast to all peers).
- Connect to one or more mainnet nodes as a normal P2P peer.
- Forward the captured alert bytes verbatim.

The P2P network is public and permissionless. Testnet alerts are broadcast to all connected peers. The `send_alert` RPC path also exists but is typically localhost-restricted; the P2P path is fully reachable by any external peer. The only timing constraint is that the alert's `notice_until` timestamp must still be in the future at the time of replay via RPC — but the P2P `received` handler imposes no such check at all. [7](#0-6) 

---

### Recommendation

Include a network domain separator in the signed message. The natural choice is the genesis block hash, which already uniquely identifies each CKB network: [8](#0-7) 

Concretely, add a `genesis_hash: Byte32` field to `RawAlert` in `util/gen-types/schemas/extensions.mol`, populate it at signing time, and verify it matches the local node's `consensus.genesis_hash()` inside `Verifier::verify_signatures` before accepting the alert. Alternatively, prepend the genesis hash as a domain tag to the data passed to `blake2b_256` in `calc_alert_hash`, without changing the wire format.

---

### Proof of Concept

1. Run a CKB testnet node and a CKB mainnet node (both use the same default `alert_signature.toml` keys).
2. On testnet, have the Nervos Foundation (or a test setup with known keys) issue a valid alert via `send_alert`. All testnet peers receive the raw `packed::Alert` bytes over the P2P Alert protocol.
3. As an unprivileged P2P peer connected to both networks, capture the raw alert bytes from the testnet P2P stream.
4. Connect to a mainnet node as a P2P peer and send the identical raw bytes on the Alert protocol channel.
5. The mainnet `AlertRelayer::received` handler parses the alert, calls `verify_signatures` (which succeeds — same keys, same hash, no network field), and broadcasts the alert to all connected mainnet peers.
6. All mainnet nodes display the testnet alert message as a legitimate critical warning.

```rust
    pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
        trace!("Verifying alert {:?}", alert);
        let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
        let signatures: Vec<Signature> = alert
            .signatures()
            .into_iter()
            .filter_map(
                |sig_data| match Signature::from_slice(sig_data.as_reader().raw_data()) {
                    Ok(sig) => {
                        if sig.is_valid() {
                            Some(sig)
                        } else {
                            debug!("invalid signature: {:?}", sig);
                            None
                        }
                    }
                    Err(err) => {
                        debug!("signature error: {}", err);
                        None
                    }
                },
            )
            .collect();
        verify_m_of_n(
            &message,
            self.config.signatures_threshold,
            &signatures,
            &self.pubkeys,
        )
        .map_err(|err| err.kind())?;
        Ok(())
    }
```

```rust
    async fn received(
        &mut self,
        nc: Arc<dyn CKBProtocolContext + Sync>,
        peer_index: PeerIndex,
        data: Bytes,
    ) {
        let alert: packed::Alert = match packed::AlertReader::from_slice(&data) {
            Ok(alert) => {
                if alert.raw().message().is_utf8()
                    && alert
                        .raw()
                        .min_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                    && alert
                        .raw()
                        .max_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                {
                    alert.to_entity()
                } else {
                    info!(
                        "A malformed message fromP peer {} : not utf-8 string",
                        peer_index
                    );
                    nc.ban_peer(
                        peer_index,
                        BAD_MESSAGE_BAN_TIME,
                        String::from("send us a malformed message: not utf-8 string"),
                    );
                    return;
                }
            }
            Err(err) => {
                info!("A malformed message from peer {}: {:?}", peer_index, err);
                nc.ban_peer(
                    peer_index,
                    BAD_MESSAGE_BAN_TIME,
                    String::from("send us a malformed message"),
                );
                return;
            }
        };
        let alert_id = alert.as_reader().raw().id().into();
        trace!("ReceiveD alert {} from peer {}", alert_id, peer_index);
        // ignore alert
        if self.notifier.lock().has_received(alert_id) {
            return;
        }
        // verify
        if let Err(err) = self.verifier.verify_signatures(&alert) {
            debug!(
                "An alert from peer {} with invalid signatures, error {:?}",
                peer_index, err
            );
            nc.ban_peer(
                peer_index,
                BAD_MESSAGE_BAN_TIME,
                String::from("send us an alert with invalid signatures"),
            );
            return;
        }
        // mark sender as known
        self.mark_as_known(peer_index, alert_id);
        // broadcast message
        let selected_peers: Vec<PeerIndex> = nc
            .connected_peers()
            .into_iter()
            .filter(|peer| self.mark_as_known(*peer, alert_id))
            .collect();
        if let Err(err) = nc.quick_filter_broadcast(
            TargetSession::Multi(Box::new(selected_peers.into_iter())),
            data,
        ) {
            debug!("alert broadcast error: {:?}", err);
        }
        // add to received alerts
        self.notifier.lock().add(&alert);
    }
```
