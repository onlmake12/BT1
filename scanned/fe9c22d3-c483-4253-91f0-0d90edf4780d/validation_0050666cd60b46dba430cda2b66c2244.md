### Title
`min_replace_fee` RPC Reports Incorrect Minimum Fee for RBF When Replacement Transaction Is Larger Than Original — (File: `tx-pool/src/pool.rs`)

---

### Summary

The `min_replace_fee` function in `tx-pool/src/pool.rs` computes the minimum fee required to replace a pooled transaction using the **old** transaction's serialized size. However, the actual enforcement in `check_rbf` computes the required minimum using the **new** (replacement) transaction's size. When the replacement transaction is larger than the original, the value reported to the user via the `get_transaction_with_verbosity` RPC is lower than what `check_rbf` actually requires, causing the replacement to be rejected even though the user followed the node's own guidance.

---

### Finding Description

`calculate_min_replace_fee` is the shared helper that computes:

```
min_replace_fee = sum(replaced_txs.fee) + min_rbf_rate.fee(size)
```

The `size` argument is the critical parameter. It is supplied differently in two call sites:

**Call site 1 — `min_replace_fee` (user-facing RPC helper):**

```rust
// tx-pool/src/pool.rs  line 86-99
pub fn min_replace_fee(&self, tx: &TxEntry) -> Option<Capacity> {
    ...
    self.calculate_min_replace_fee(&conflicts, tx.size)   // ← OLD tx's size
}
``` [1](#0-0) 

**Call site 2 — `check_rbf` (actual enforcement):**

```rust

            "Tx's current fee is {}, expect it to >= {} to replace old txs",

The `calculate_min_replace_fee` helper itself:

The `min_replace_fee` result is surfaced to callers via the `get_transaction_with_verbosity` RPC (confirmed by the integration test assertions):

```rust

// replace fee is 100000363

``` [4](#0-3) 

A user who queries `min_replace_fee` for `tx1` and then submits a replacement `tx2` with `fee == min_replace_fee` will be rejected by `check_rbf` whenever `tx2.size > tx1.size`, because `check_rbf` recomputes the threshold using `tx2.size`, yielding a strictly higher required fee.

---

### Impact Explanation

Any RPC caller who:
1. Submits `tx1` to the pool,
2. Queries `get_transaction_with_verbosity` to read `min_replace_fee` (computed from `tx1.size`),
3. Constructs a replacement `tx2` with `fee == min_replace_fee` but `tx2.size > tx1.size` (e.g., more outputs, more witnesses, more cell deps),
4. Submits `tx2` via `send_transaction`,

will receive an `RBFRejected` error even though they followed the node's own published guidance. The replacement fails silently from the user's perspective. In time-sensitive scenarios (e.g., fee bumping before a deadline), this can result in the original transaction remaining stuck in the pool.

---

### Likelihood Explanation

Medium. RBF is an opt-in feature (`min_rbf_rate > min_fee_rate`). However, any user who legitimately needs to replace a transaction with a structurally different (larger) one — a common real-world pattern — and who relies on the `min_replace_fee` RPC field will hit this. The discrepancy is not documented, and the error message does not explain the size mismatch, making diagnosis difficult.

---

### Recommendation

Pass the **new** transaction's size to `calculate_min_replace_fee` inside `min_replace_fee`, or expose a variant of the RPC that accepts the prospective replacement transaction so the correct size can be used. At minimum, document that the returned `min_replace_fee` is a lower bound valid only when the replacement transaction is no larger than the original.

---

### Proof of Concept

1. Enable RBF: set `min_rbf_rate > min_fee_rate` in node config.
2. Submit `tx1` (size = S₁, fee = F₁) to the pool.
3. Call `get_transaction_with_verbosity(tx1.hash, 2)` → `min_replace_fee` = F₁ + `min_rbf_rate.fee(S₁)`.
4. Construct `tx2` with the same inputs as `tx1`, one extra output (so `tx2.size = S₂ > S₁`), and `tx2.fee = min_replace_fee`.
5. Call `send_transaction(tx2)`.
6. `check_rbf` computes required fee = F₁ + `min_rbf_rate.fee(S₂)` > `min_replace_fee`.
7. Node returns `RBFRejected: Tx's current fee is X, expect it to >= Y to replace old txs` — even though the user paid exactly what the RPC told them to pay.

    /// min_replace_fee = sum(replaced_txs.fee) + extra_rbf_fee
    fn calculate_min_replace_fee(&self, conflicts: &[&PoolEntry], size: usize) -> Option<Capacity> {
        let extra_rbf_fee = self.config.min_rbf_rate.fee(size as u64);
        // don't account for duplicate txs
        let replaced_fees: HashMap<_, _> = conflicts
            .iter()
            .map(|c| (c.id.clone(), c.inner.fee))
            .collect();
        let replaced_sum_fee = replaced_fees
            .values()
            .try_fold(Capacity::zero(), |acc, x| acc.safe_add(*x));
        let res = replaced_sum_fee.map_or(Err(CapacityError::Overflow), |sum| {
            sum.safe_add(extra_rbf_fee)
        });
        if let Ok(res) = res {
            Some(res)
        } else {
            let fees = conflicts.iter().map(|c| c.inner.fee).collect::<Vec<_>>();
            error!(
                "conflicts: {:?} replaced_sum_fee {:?} overflow by add {}",
                conflicts.iter().map(|e| e.id.clone()).collect::<Vec<_>>(),
                fees,
                extra_rbf_fee
            );
            None
        }
    }
