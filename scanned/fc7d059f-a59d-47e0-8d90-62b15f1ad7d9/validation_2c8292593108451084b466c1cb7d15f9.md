### Title
Missing per-peer rate limit for `GetHeaders` in Sync protocol allows DB read amplification — (`sync/src/synchronizer/get_headers_process.rs`)

### Summary
The `Synchronizer::try_process` handler for `GetHeaders` messages has no rate limiter, while the analogous `Relayer::try_process` does. The `getheaders_received` accounting hook is a confirmed no-op stub. An unprivileged peer out of IBD can flood the node with valid `GetHeaders` messages, each triggering up to 2000 DB header reads via `get_locator_response`, causing I/O saturation.

### Finding Description
`Synchronizer` has no `rate_limiter` field and `try_process` applies no per-peer throttle before dispatching `GetHeaders` to `GetHeadersProcess::execute`. The `Peers::getheaders_received` method — the intended accounting point — is an empty stub (`// TODO:`). Each valid `GetHeaders` (locator ending with genesis hash, ≤101 entries) causes:
- `locate_latest_common_block`: up to 101 snapshot hash lookups + a parent-chain DB walk
- `get_locator_response`: up to 2000 sequential `get_block_hash` + `get_block_header` DB reads

The `Relayer` protocol, by contrast, uses a `governor`-based rate limiter keyed by `(PeerIndex, message_item_id)` at 30 req/s. No equivalent exists for the Sync protocol.

### Impact Explanation
A single malicious peer sending `GetHeaders` at maximum TCP throughput forces proportional DB read IOPS on the victim node. With 2000 header reads per message and no throttle, sustained flooding causes I/O saturation, degrading block processing, peer sync, and overall node responsiveness.

### Likelihood Explanation
Any unprivileged peer can connect and send valid `GetHeaders` messages. The only structural guard (IBD check) is irrelevant for a synced node. The attack requires no special knowledge beyond the genesis hash.

### Recommendation
Add a `governor`-based rate limiter to `Synchronizer` (mirroring `Relayer`) and enforce it in `try_process` before dispatching `GetHeaders`. Implement `getheaders_received` to track per-peer request counts and enforce a hard cap.

### Proof of Concept
1. Connect to a fully-synced (non-IBD) CKB node as a peer on the Sync protocol.
2. In a tight loop, send `GetHeaders` messages with a locator of `[tip_hash, ..., genesis_hash]` (≤101 entries, genesis hash last).
3. Each message passes the `locator_size > MAX_LOCATOR_SIZE` check and the IBD check, then calls `locate_latest_common_block` + `get_locator_response`.
4. Benchmark DB read IOPS on the victim node; observe they scale linearly with send rate and are unbounded per peer.

    pub fn locate_latest_common_block(
        &self,
        _hash_stop: &Byte32,
        locator: &[Byte32],
    ) -> Option<BlockNumber> {
        if locator.is_empty() {
            return None;
        }

        let locator_hash = locator.last().expect("empty checked");
        if locator_hash != &self.sync_shared.consensus().genesis_hash() {
            return None;
        }

        // iterator are lazy
        let (index, latest_common) = locator
            .iter()
            .enumerate()
            .map(|(index, hash)| (index, self.snapshot.get_block_number(hash)))
            .find(|(_index, number)| number.is_some())
            .expect("locator last checked");

        if index == 0 || latest_common == Some(0) {
            return latest_common;
        }

        if let Some(header) = locator
            .get(index - 1)
            .and_then(|hash| self.sync_shared.store().get_block_header(hash))
        {
            let mut block_hash = header.data().raw().parent_hash();
            loop {
                let block_header = match self.sync_shared.store().get_block_header(&block_hash) {
                    None => break latest_common,
                    Some(block_header) => block_header,
                };

                if let Some(block_number) = self.snapshot.get_block_number(&block_hash) {
                    return Some(block_number);
                }

                block_hash = block_header.data().raw().parent_hash();
            }
        } else {
            latest_common
        }
    }

    pub fn get_locator_response(
        &self,
        block_number: BlockNumber,
        hash_stop: &Byte32,
    ) -> Vec<core::HeaderView> {
        let tip_number = self.tip_header().number();
        let Some(start_number) = block_number.checked_add(1) else {
            return Vec::new();
        };
        std::iter::successors(Some(start_number), |number| number.checked_add(1))
            .take_while(|number| *number <= tip_number)
            .take(MAX_HEADERS_LEN)
            .filter_map(|block_number| self.snapshot.get_block_hash(block_number))
            .take_while(|block_hash| block_hash != hash_stop)
            .filter_map(|block_hash| self.sync_shared.store().get_block_header(&block_hash))
            .collect()
    }
