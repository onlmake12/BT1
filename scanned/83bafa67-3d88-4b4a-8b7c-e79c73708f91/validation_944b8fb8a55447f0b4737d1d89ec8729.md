### Title
Unauthenticated `ConnectionRequestDelivered` Forwarding Allows Relay-Assisted Message Injection and NAT Traversal State Poisoning — (`network/src/protocols/hole_punching/component/connection_request_delivered.rs`)

---

### Summary

`ConnectionRequestDeliveredProcess::execute` contains no check that the peer who sent the `ConnectionRequestDelivered` message is actually the legitimate `to` peer (i.e., the one who received the original `ConnectionRequest`). When `route` is empty and `self_peer_id != content.from`, the relay unconditionally forwards the message to whatever peer matches `content.from` in its registry. An unprivileged attacker can exploit this to cause any relay node to deliver attacker-crafted `ConnectionRequestDelivered` messages — including attacker-controlled `listen_addrs` — to any peer the relay is connected to.

---

### Finding Description

The normal hole-punching flow is:

```
from ──ConnectionRequest──► relay ──► to
from ◄──ConnectionRequestDelivered──  relay ◄── to
```

The `to` peer is the only legitimate originator of `ConnectionRequestDelivered`. However, the relay never verifies this. In `execute()`:

```rust
match content.route.last() {
    Some(next_peer_id) => self.forward_delivered(next_peer_id).await,
    None => {
        let self_peer_id = self.protocol.network_state.local_peer_id();
        if self_peer_id != &content.from {
            // forward the message to the `from` peer
            self.forward_delivered(&content.from).await   // ← no origin check
        } else { ... }
    }
}
``` [1](#0-0) 

`forward_delivered` then looks up `content.from` in the peer registry and sends the message to that session:

```rust
async fn forward_delivered(&self, peer_id: &PeerId) -> Status {
    let target_sid = self.protocol.network_state.peer_registry
        .read().get_key_by_peer_id(peer_id);
    match target_sid {
        Some(next_peer) => {
            ...
            self.p2p_control.send_message_to(next_peer, proto_id, new_message).await
        }
        ...
    }
}
``` [2](#0-1) 

There is no check that `self.peer` (the session that sent the message) equals the session for `content.to`. The relay never records which `(from, to)` pairs it previously forwarded a `ConnectionRequest` for, so it cannot validate that the incoming `ConnectionRequestDelivered` corresponds to a legitimate prior exchange.

---

### Impact Explanation

**Tier 1 — Unconditional relay abuse (no precondition):**  
Any attacker connected to relay R can cause R to deliver a `ConnectionRequestDelivered` message to any peer V that is currently connected to R, simply by setting `content.from = V_peer_id` and `route = []`. The victim V receives the message and processes it. If V has no matching `inflight_requests` entry, it returns `Ignore` — but the relay has still been abused as a proxy and V has spent CPU on deserialization and registry lookups.

**Tier 2 — NAT traversal state poisoning (requires inflight request knowledge):**  
When V receives the forwarded message with `self_peer_id == content.from` and `route` empty, it executes:

```rust
let request_start = self.protocol.inflight_requests.remove(&content.to);
match request_start {
    Some(start) => {
        let res = self.respond_sync(content.from).await;
        ...
        self.try_nat_traversal(ttl, content.listen_addrs);  // ← attacker-controlled addrs
        Status::ok()
    }
    None => StatusCode::Ignore.with_context("the request is not in flight"),
}
``` [3](#0-2) 

If the attacker knows (or guesses) that V has an inflight request to peer X, they craft `content.to = X_peer_id` and `content.listen_addrs = [attacker_ip:port]`. The result:
1. V **removes** the legitimate inflight request entry for X — the real `ConnectionRequestDelivered` from X will subsequently be dropped as `Ignore`.
2. V spawns `try_nat_traversal` targeting attacker-controlled addresses, retrying TCP connections for up to **30 seconds**. [4](#0-3) 

The `inflight_requests` map is populated by the `notify` loop which broadcasts `ConnectionRequest` messages to peers fetched from the peer store. The `to` peer IDs used are derived from `fetch_nat_addrs`, which returns addresses stored in the peer store — these are not secret. [5](#0-4) 

---

### Likelihood Explanation

- **Attacker cost**: one P2P connection to any relay node. PeerIds are public (exchanged during identify protocol).
- **Tier 1** requires no prior knowledge beyond the victim's PeerId and that the victim is connected to the relay.
- **Tier 2** requires knowing a `to` PeerId the victim is currently trying to reach — obtainable by observing `ConnectionRequest` gossip traffic, since those are broadcast to `sqrt(total)` peers with no confidentiality.
- The `forward_rate_limiter` (1/sec per `(from, to, item_id)` triple) limits repeated attempts per pair but does not prevent the attack; the attacker can use many distinct `(from, to)` pairs within the outer 30/sec session limit. [6](#0-5) 

---

### Recommendation

1. **Verify sender identity**: In the `None` route branch, check that `self.peer` (the session that sent the message) corresponds to the session registered for `content.to` in the peer registry. If `get_key_by_peer_id(&content.to) != Some(self.peer)`, reject the message.
2. **Track forwarded pairs**: The relay should maintain a record of `(from, to)` pairs for which it forwarded a `ConnectionRequest`, and only forward `ConnectionRequestDelivered` for pairs it has seen.

---

### Proof of Concept

```
Attacker A connects to relay R (R also has victim V connected).
V has an inflight ConnectionRequest to peer X (observable from gossip).

A sends to R:
  ConnectionRequestDelivered {
    from: V_peer_id,
    to:   X_peer_id,
    route: [],          // empty → triggers the None branch
    sync_route: [],
    listen_addrs: [attacker_ip:attacker_port],
  }

R.execute():
  route.last() == None
  self_peer_id (R) != content.from (V)  → forward_delivered(&V_peer_id)
  peer_registry.get_key_by_peer_id(V_peer_id) → V's session
  send_message_to(V_session, message)   // no origin check performed

V.execute():
  route.last() == None
  self_peer_id (V) == content.from (V)  → else branch
  inflight_requests.remove(X_peer_id)  → Some(start)  // consumes V's legitimate state
  try_nat_traversal(ttl, [attacker_ip:attacker_port])  // 30s of outbound TCP to attacker
```

Assert: relay R forwards the message to V without verifying that the sender is X. V's inflight request to X is consumed and V initiates NAT traversal to attacker-controlled addresses.

pub(crate) async fn try_nat_traversal(
    bind_addr: Option<SocketAddr>,
    addr: Multiaddr,
) -> Result<(TcpStream, Multiaddr), std::io::Error> {
    let net_addr = match multiaddr_to_socketaddr(&addr) {
        Some(addr) => addr,
        None => {
            debug!("Failed to convert multiaddr to socketaddr");
            return Err(std::io::ErrorKind::InvalidInput.into());
        }
    };

    // Use a fixed interval but add a small amount of randomness
    let base_retry_interval = Duration::from_millis(200);

    // total time
    let timeout_duration = Duration::from_secs(30);
    let start_time = Instant::now();
    let mut retry_count = 0u32;
    while start_time.elapsed() < timeout_duration {
        retry_count += 1;

        // Add a small amount of random jitter (±25ms) to avoid conflicts
        // caused by continuous precise synchronization
        let jitter = Duration::from_millis(rand::random::<u64>() % 50);
        let actual_interval = if rand::random::<bool>() {
            base_retry_interval + jitter
        } else {
            base_retry_interval.saturating_sub(jitter)
        };

        let socket = create_socket(bind_addr, net_addr)?;

        match runtime::timeout(
            std::time::Duration::from_millis(200),
            socket.connect(net_addr),
        )
        .await
        {
            Ok(Ok(stream)) => {
                // try get the stored error in the underlying socket
                // if the socket is not connected, it will return an error
                if let Err(err) = check_connection(&stream) {
                    debug!("Failed to connect to NAT(base check): {}", err);
                }
                return Ok((stream, addr));
            }
            Err(err) => {
                debug!("Failed to connect to NAT(timeout): {}", err);
            }
            Ok(Err(err)) => {
                if err.kind() == std::io::ErrorKind::AddrNotAvailable {
                    return Err(err);
                }
                debug!(
                    "Failed to connect to NAT(other error): {}, {}",
                    err.kind(),
                    err
                );
            }
        }
        runtime::delay_for(actual_interval).await;
    }

    debug!("Failed to connect to NAT after {} retries", retry_count);
    Err(std::io::ErrorKind::TimedOut.into())
}
