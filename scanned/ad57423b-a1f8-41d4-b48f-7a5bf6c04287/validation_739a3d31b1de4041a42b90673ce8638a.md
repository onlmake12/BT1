Audit Report

## Title
Peer Store Permanent DoS via Crafted Discovery Addresses Exploiting Off-by-One in `check_purge` Eviction Guard — (`network/src/peer_store/peer_store_impl.rs`)

## Summary

`check_purge` uses a strict `> 4` threshold when selecting candidates for network-group eviction. An attacker who fills the peer store with exactly 4 addresses per `/16` group across 4096 distinct groups (totaling 16384 = `ADDR_COUNT_LIMIT`) causes every subsequent `add_addr` call to return `PeerStoreError::EvictionFailed`, permanently blocking honest peer address admission and enabling eclipse attacks.

## Finding Description

`check_purge` is invoked at the start of every `add_addr` call:

```rust
// peer_store_impl.rs L75
self.check_purge()?;
```

When `addr_manager.count() >= ADDR_COUNT_LIMIT` (16384), it runs two eviction stages:

**Stage 1** (L341–355): Collects addresses where `!is_connectable(now_ms)`. A freshly added address has `last_connected_at_ms = 0`, `attempts_count = 0`, and `last_tried_at_ms = 0`. In `is_connectable` (types.rs L89–105):
- `tried_in_last_minute`: `0 >= now_ms - 60000` → false
- `last_connected_at_ms == 0 && attempts_count >= 3` → `0 >= 3` → false
- Result: all fresh addresses are connectable → Stage 1 evicts nothing → `candidate_peers.is_empty()` is true

**Stage 2** (L357–403): Groups addresses by network group, sorts by group size descending, takes the top `len/2` groups, and for each group:

```rust
// peer_store_impl.rs L378
if addrs.len() > 4 {
    // evict 2 random addresses
} else {
    None  // groups of exactly 4 are NEVER evicted
}
```

With 4096 groups of exactly 4 addresses each, every group returns `None`, `candidate_peers` is empty, and the function returns:

```rust
// peer_store_impl.rs L399–401
if candidate_peers.is_empty() {
    return Err(PeerStoreError::EvictionFailed.into());
}
```

`AddrManager::add` (addr_manager.rs L22–34) deduplicates by exact address, so the attacker uses 4 distinct IPs per `/16` group — deduplication does not prevent the attack.

## Impact Explanation

Every `add_addr` call after the store is saturated returns `Err(EvictionFailed)`. The node permanently loses the ability to learn new peer addresses from discovery, DNS seeding, or identify handshakes. The node is left with only the 16384 attacker-supplied addresses, enabling a full eclipse attack. An eclipsed mining node can be fed a fake chain, causing consensus deviation. This matches **High (10001–15000 points): Vulnerabilities which could easily crash a CKB node** (via isolation/eclipse) and potentially **Critical: Vulnerabilities which could easily cause consensus deviation** if the victim is a block producer.

## Likelihood Explanation

The attacker requires 16384 routable IPv4 addresses spread across 4096 distinct `/16` subnets (4 per subnet). This is achievable with a mid-sized botnet or a cloud provider with diverse IP allocations. The discovery protocol accepts `Nodes` messages from any connected peer with no visible per-session address-count rate limit. A single TCP connection is sufficient to inject all 16384 crafted addresses. The attack is permanent until the node is restarted with a cleared peer store.

## Recommendation

Change the eviction threshold from strict `> 4` to `>= 4` at `peer_store_impl.rs` line 378:

```rust
// Before (vulnerable):
if addrs.len() > 4 {

// After (fixed):
if addrs.len() >= 4 {
```

Additionally:
1. Cap the number of addresses accepted per discovery session to limit injection speed.
2. Prefer eviction of addresses with `last_connected_at_ms == 0` (never successfully connected) before applying network-group eviction.

## Proof of Concept

```rust
#[test]
fn test_eviction_failed_with_exactly_4_per_group() {
    let mut peer_store = PeerStore::default();
    // Fill 4096 /16 groups × 4 addresses each = 16384 total
    for group in 0u16..4096 {
        let hi = (group >> 8) as u8;
        let lo = (group & 0xff) as u8;
        for host in 1u8..=4 {
            let addr: Multiaddr = format!("/ip4/{}.{}.1.{}/tcp/8115", hi, lo, host)
                .parse().unwrap();
            let _ = peer_store.add_addr(addr, Flags::COMPATIBILITY);
        }
    }
    // Any subsequent add_addr must fail with EvictionFailed
    let new_addr: Multiaddr = "/ip4/200.200.1.5/tcp/8115".parse().unwrap();
    assert!(matches!(
        peer_store.add_addr(new_addr, Flags::COMPATIBILITY),
        Err(e) if e.to_string().contains("EvictionFailed")
    ));
}
