Audit Report

## Title
`BlockReader::extension()` Unconditional `.unwrap()` on Malformed Extra Field Causes Node Crash — (`File: util/gen-types/src/extension/shortcut.rs`)

## Summary
`BlockReader::extension()` and `CompactBlockReader::extension()` call `.unwrap()` when parsing the first extra field as a molecule `Bytes` structure. `from_compatible_slice` validates outer molecule structure but does not validate the semantic content of extra fields. A peer sending a block or compact block with exactly one extra field whose bytes are not a valid molecule `Bytes` encoding triggers a panic in the relayer's async protocol handler. When the handler panics, tentacle reports `ProtocolHandleError::AbnormallyClosed`, which causes `broadcast_exit_signals()` to be called, shutting down the entire node.

## Finding Description

**Root cause — `extension()` panics on malformed extra field:**

`util/gen-types/src/extension/shortcut.rs` lines 251–254:
```rust
pub fn extension(&self) -> Option<packed::BytesReader<'_>> {
    self.extra_field(0)
        .map(|data| packed::BytesReader::from_slice(data).unwrap())
}
```
`from_slice` returns `Err` if the bytes do not satisfy the molecule `Bytes` layout (4-byte LE length prefix + exactly that many payload bytes). `.unwrap()` panics on `Err`.

**Why `from_compatible_slice` does not protect against this:**

`CompactBlockReader::verify()` (`util/gen-types/src/generated/extensions.rs` lines 7163–7203) with `compatible=true` validates the outer table structure (total size, offsets, field count ≥ 5) and verifies the five known fields. It does **not** call any verifier on extra fields. A `CompactBlock` with one extra field whose bytes are `[0x01, 0x00, 0x00, 0x00]` (claims length 1 but provides 0 payload bytes) passes `from_compatible_slice` successfully.

**Why the existing guard is insufficient:**

`sync/src/relayer/mod.rs` lines 823–841 only bans a peer if `count_extra_fields() > 1`. A single malformed extra field (`count_extra_fields() == 1`) passes through without any content validation.

**Panic propagation to node shutdown:**

The relayer's `received()` is an async handler registered with tentacle. When it panics, tentacle reports `ServiceError::ProtocolHandleError { error: AbnormallyClosed }`. The handler in `network/src/network.rs` lines 688–718 processes this by calling `broadcast_exit_signals()` at line 717, which sends a shutdown signal to all node services — crashing the entire node, not just the relay thread.

This is distinct from the chain verifier path (`chain/src/verify.rs` lines 86–96), which wraps `consume_unverified_blocks` in `catch_unwind` and recovers gracefully. The relayer has no equivalent protection.

## Impact Explanation

A single crafted P2P message causes the target CKB node to call `broadcast_exit_signals()`, terminating all node services. This matches the allowed impact: **High (10001–15000 points) — Vulnerabilities which could easily crash a CKB node.** The crash is deterministic and repeatable; the attacker can reconnect and crash the node again after each restart.

## Likelihood Explanation

Any unprivileged P2P peer can send a `CompactBlock` or `BlockTransactions` relay message. No mining power, stake, or authentication is required. The crafted message requires only a valid outer molecule `Block`/`CompactBlock` structure with one extra field whose content is not a valid molecule `Bytes`. This is trivially constructable. The attack is repeatable: after the node restarts, the attacker reconnects and sends the same message again.

## Recommendation

Replace `.unwrap()` with `.ok()` and use `and_then` to return `None` on malformed extra fields instead of panicking:

```rust
pub fn extension(&self) -> Option<packed::BytesReader<'_>> {
    self.extra_field(0)
        .and_then(|data| packed::BytesReader::from_slice(data).ok())
}
```

Additionally, add content validation of extra fields in `CompactBlockReader::verify()` when `compatible=true` and exactly one extra field is present, so malformed extensions are rejected at the deserialization boundary before reaching any caller of `extension()`.

## Proof of Concept

1. Construct a molecule-encoded `CompactBlock` with the standard 5 fields (valid header, empty short_ids, empty prefilled_transactions, empty uncles, empty proposals) plus one extra field whose bytes are `[0x01, 0x00, 0x00, 0x00]` — a molecule `Bytes` header claiming 1 byte of payload but providing none.
2. Wrap it in a `RelayMessage::CompactBlock` and send it to a CKB node via the RelayV3 P2P protocol.
3. `from_compatible_slice` accepts the message (outer structure is valid; `count_extra_fields() == 1`, not `> 1`, so the ban guard does not fire).
4. `try_process` dispatches to `CompactBlockProcess::execute()`.
5. During processing, `compact_block.extension()` is called; `packed::BytesReader::from_slice(&[0x01, 0x00, 0x00, 0x00])` returns `Err` (declared length 1, but slice is only 4 bytes total — the molecule `Bytes` layout requires `4 + 1 = 5` bytes); `.unwrap()` panics.
6. Tentacle catches the async task panic, emits `ProtocolHandleError::AbnormallyClosed`, and the network service handler calls `broadcast_exit_signals()`, shutting down the node.
