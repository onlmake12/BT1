### Title
Unauthenticated Miner Notify Port Allows Cellbase Witness Hijacking to Redirect Block Reward — (`miner/src/client.rs`)

---

### Summary

The miner's notify-mode HTTP listener accepts `BlockTemplate` POSTs from any source with no authentication. A `work_id=0` value in the crafted template unconditionally bypasses the deduplication guard in `update_block_template`. The miner then mines and submits the injected block. Because the cellbase **witness** lock (which designates where the *current* block's future reward is paid) is never validated against the operator's configured `block_assembler` lock, an attacker can redirect the current block's reward to an attacker-controlled address. The cellbase **output** lock (the reward for a *past* target block) is enforced by `RewardVerifier` and cannot be changed — but the witness lock is entirely unguarded.

---

### Finding Description

**1. No authentication on the notify port**

`listen_block_template_notify` binds a plain TCP listener and dispatches every incoming POST body directly to `update_block_template` with zero authentication, IP filtering, or token validation. [1](#0-0) 

The `handle` function deserializes any valid JSON body as a `BlockTemplate` and calls `update_block_template` unconditionally: [2](#0-1) 

**2. `work_id=0` unconditionally bypasses the deduplication guard**

```rust
let updated = |id| {
    if id != work_id || id == 0 {   // tautology when work_id == 0
        Some(work_id)
    } else {
        None
    }
};
```

When the attacker sends `work_id=0`, the predicate `id != 0 || id == 0` is a tautology — always `true` — so `fetch_update` always succeeds and the injected template is always dispatched to the mining workers. [3](#0-2) 

**3. Cellbase witness lock is not validated against `block_assembler` config**

`BlockAssembler::build_cellbase` embeds the operator's lock in the cellbase **witness** (the `CellbaseWitness.lock` field). This witness lock is what the network uses to determine where the *current* block's reward is paid when it is finalized `PROPOSAL_WINDOW.farthest + 1` blocks later. [4](#0-3) 

The non-contextual `CellbaseVerifier` only checks that the witness is a parseable `CellbaseWitness` with an enabled `hash_type`. It does **not** check that `CellbaseWitness.lock` matches the operator's configured `block_assembler` lock: [5](#0-4) 

**4. The cellbase OUTPUT lock IS enforced (important nuance)**

The contextual `RewardVerifier` enforces that the cellbase output lock equals `target_lock` — the lock from the *target block's* cellbase witness (a past block). An attacker who changes the output lock will be rejected with `InvalidRewardTarget`: [6](#0-5) 

Therefore the attacker must keep the output lock correct (easily obtained from public chain data or `get_block_template`) and only modify the **witness** lock to their own address.

---

### Impact Explanation

The attacker injects a template where the cellbase witness contains their own lock script. The miner mines a valid block (output lock is correct, passes all consensus checks). When that block's reward is finalized `PROPOSAL_WINDOW.farthest + 1` blocks later, the reward is paid to the attacker's address, not the operator's. The operator loses 100% of the block reward for every block mined on an injected template.

---

### Likelihood Explanation

- **Precondition**: The notify port must be reachable. The default config has `listen` commented out; when enabled, the suggested default is `127.0.0.1:8888`. However, operators running the miner on a separate machine from the node commonly bind to `0.0.0.0` or a LAN/public address, making the port reachable.
- **No credentials required**: There is zero authentication on the port.
- **Trivial to exploit**: The attacker fetches the legitimate template from the public `get_block_template` RPC, replaces `cellbase.witnesses[0]` with a `CellbaseWitness` encoding their own lock, sets `work_id=0`, and POSTs it.
- **Persistent**: The attacker can continuously re-inject every time a new template is issued. [7](#0-6) 

---

### Recommendation

1. **Bind the notify port to loopback only** and document this as a security requirement.
2. **Add a shared secret / HMAC token** to the notify endpoint; the CKB node's block assembler should include it in the `Authorization` header when POSTing.
3. **Validate the cellbase witness lock** in `update_block_template` against the locally configured `block_assembler` lock before dispatching the template to workers.
4. **Remove the `id == 0` special case** or treat `work_id=0` as invalid/rejected rather than always-accepted.

---

### Proof of Concept

```
1. Operator runs ckb-miner with listen = "0.0.0.0:8888"
2. Attacker fetches the current template:
     curl http://<node>:8114/ -d '{"id":1,"jsonrpc":"2.0","method":"get_block_template","params":[]}'
3. Attacker replaces cellbase.witnesses[0] with a CellbaseWitness
   encoding attacker_lock (any valid secp256k1 address they control).
   Keeps cellbase.outputs[0].lock unchanged (passes RewardVerifier).
   Sets work_id = "0x0".
4. Attacker POSTs the modified template to http://<miner>:8888/
5. update_block_template accepts it (work_id=0 tautology).
6. Miner mines the block; submit_block succeeds (all consensus checks pass).
7. PROPOSAL_WINDOW.farthest + 1 blocks later, the block reward is
   paid to attacker_lock, not the operator's configured lock.
``` [3](#0-2) [8](#0-7)

    async fn listen_block_template_notify(&self, addr: SocketAddr) {
        let listener = TcpListener::bind(addr).await.unwrap();
        let server = auto::Builder::new(TokioExecutor::new());
        let graceful = GracefulShutdown::new();
        let stop_rx: CancellationToken = new_tokio_exit_rx();

        loop {
            let client = self.clone();
            let handle = service_fn(move |req| handle(client.clone(), req));
            tokio::select! {
                conn = listener.accept() => {
                    let (stream, _) = match conn {
                        Ok(conn) => conn,
                        Err(e) => {
                            info!("accept error: {}", e);
                            tokio::time::sleep(std::time::Duration::from_secs(1)).await;
                            continue;
                        }
                    };
                    let stream = hyper_util::rt::TokioIo::new(Box::pin(stream));
                    let conn = server.serve_connection_with_upgrades(stream, handle);

                    let conn = graceful.watch(conn.into_owned());
                    tokio::spawn(async move {
                        if let Err(err) = conn.await {
                            info!("connection error: {}", err);
                        }
                    });
                },
                _ = stop_rx.cancelled() => {
                    info!("Miner client received exit signal. Exit now");
                    break;
                }
            }
        }
        drop(listener);
        graceful.shutdown().await;
    }

    pub(crate) fn build_cellbase_witness(
        config: &BlockAssemblerConfig,
        snapshot: &Snapshot,
    ) -> CellbaseWitness {
        let hash_type: ScriptHashType = config.hash_type.into();
        let cellbase_lock = Script::new_builder()
            .args(config.args.as_bytes())
            .code_hash(&config.code_hash)
            .hash_type(hash_type)
            .build();
        let tip = snapshot.tip_header();

        let mut message = vec![];
        if let Some(version) = snapshot.compute_versionbits(tip) {
            message.extend_from_slice(&version.to_le_bytes());
            message.extend_from_slice(b" ");
        }
        if config.use_binary_version_as_message_prefix {
            message.extend_from_slice(config.binary_version.as_bytes());
        }
        if !config.message.is_empty() {
            message.extend_from_slice(b" ");
            message.extend_from_slice(config.message.as_bytes());
        }

        CellbaseWitness::new_builder()
            .lock(cellbase_lock)
            .message(message)
            .build()
    }

        config: &BlockAssemblerConfig,
        snapshot: &Snapshot,
    ) -> Result<TransactionView, AnyError> {
        let tip = snapshot.tip_header();
        let candidate_number = tip
            .number()
            .checked_add(1)
            .ok_or(BlockAssemblerError::Overflow)?;
        let cellbase_witness = Self::build_cellbase_witness(config, snapshot);

        let tx = {
            let (target_lock, block_reward) = block_in_place(|| {
                RewardCalculator::new(snapshot.consensus(), snapshot).block_reward_to_finalize(tip)
            })?;
            let input = CellInput::new_cellbase_input(candidate_number);
            let output = CellOutput::new_builder()
                .capacity(block_reward.total)
                .lock(target_lock)
                .build();

            let witness = cellbase_witness.as_bytes();
            let no_finalization_target =
                candidate_number <= snapshot.consensus().finalization_delay_length();
            let tx_builder = TransactionBuilder::default().input(input).witness(witness);
            let insufficient_reward_to_create_cell = output.is_lack_of_capacity(Capacity::zero())?;
            if no_finalization_target || insufficient_reward_to_create_cell {
                tx_builder.build()
            } else {
                tx_builder
                    .output(output)
                    .output_data(Bytes::default())
                    .build()
            }
