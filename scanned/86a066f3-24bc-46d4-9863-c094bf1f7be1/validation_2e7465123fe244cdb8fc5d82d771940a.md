### Title
Unbounded `pending_compact_blocks` HashMap Enables Memory Exhaustion via Incomplete Compact Block Relay - (File: `sync/src/types/mod.rs`, `sync/src/relayer/compact_block_process.rs`)

---

### Summary

The `pending_compact_blocks` state in CKB's relay subsystem is an unbounded `HashMap` that accumulates entries whenever a compact block cannot be fully reconstructed (missing transactions). Entries are only removed upon successful block reconstruction. There is no size cap, no per-peer limit, and no timeout-based eviction. An attacker who can produce valid proof-of-work blocks can flood this map with incomplete compact blocks that never complete, causing unbounded memory growth.

---

### Finding Description

`PendingCompactBlockMap` is defined as a plain `HashMap` with no capacity bound:

```rust

        packed::CompactBlock,

        u64,

``` [1](#0-0) 

It is initialized without any bound in `SyncState`:

```rust

``` [2](#0-1) 

When a compact block arrives and reconstruction fails (missing transactions or short-id collision), `missing_or_collided_post_process` inserts an entry into this map:

```rust
shared
    .state()
    .pending_compact_blocks()
    .await
    .entry(block_hash.clone())
    .or_insert_with(|| (compact_block, HashMap::default(), unix_time_as_millis()))
    .1
    .insert(peer, (missing_transactions.clone(), missing_uncles.clone()));
``` [3](#0-2) 

Cleanup only occurs in the `ReconstructionResult::Block` branch — i.e., only when a block is successfully reconstructed:

```rust
// compact_block_process.rs lines 106-117
pending_compact_blocks.remove(&block_hash);
pending_compact_blocks.retain(|_, (v, _, _)| { ... });
``` [4](#0-3) 

The same pattern holds in `block_transactions_process.rs` — `pending.remove()` is called only on `ReconstructionResult::Block`: [5](#0-4) 

The `u64` timestamp stored in each entry is only used by `compare_with_pending_compact` to gate sync task creation (2-second window); it is **never** used for eviction: [6](#0-5) 

There is no background timer, no periodic maintenance task, and no size limit for `pending_compact_blocks`. By contrast, the orphan block pool has a 60-second cleanup timer (`clean_expired_orphans`), but no equivalent exists for `pending_compact_blocks`: [7](#0-6) 

---

### Impact Explanation

Each entry in `pending_compact_blocks` stores a full `packed::CompactBlock` (header + short IDs for all transactions + prefilled transactions + proposals + uncles). A compact block can be large — up to the consensus block size limit. An attacker who mines valid blocks and sends them as compact blocks with deliberately withheld transactions causes entries to accumulate indefinitely. With enough entries, the node's memory is exhausted, leading to OOM conditions, process crash, or severe performance degradation. The node stops participating in consensus relay.

---

### Likelihood Explanation

**Barrier**: The attacker must produce compact blocks with valid proof-of-work headers that pass `contextual_check` (valid PoW, valid parent, not stale). This requires real mining capability — it is not trivially cheap.

**Feasibility**: A well-resourced attacker (e.g., a mining pool with even a small fraction of network hashrate) can continuously mine valid blocks and relay them as compact blocks while never responding to `GetBlockTransactions` requests. Each mined block creates one persistent entry. The rate of accumulation scales with the attacker's hashrate. The `CompactBlockIsAlreadyPending` guard only prevents the same `(peer, block_hash)` pair from being re-inserted; different block hashes from the same peer are all stored: [8](#0-7) 

The CHANGELOG entry `#3110: Fix pending compact block memory bloat on abnormal flow` confirms this class of issue has been recognized before, but the current code still lacks a size bound or timeout eviction on `pending_compact_blocks`. [9](#0-8) 

---

### Recommendation

1. **Bound the map**: Cap `pending_compact_blocks` at a maximum number of entries (e.g., 512 or configurable), evicting the oldest entry (by timestamp) when the cap is reached.
2. **Timeout-based eviction**: Add a periodic background task (analogous to `clean_expired_orphans`) that removes entries whose timestamp is older than a threshold (e.g., 30 seconds).
3. **Per-peer limit**: Track how many pending entries each peer has contributed and reject new entries from peers that exceed the limit.
4. **Disconnect non-responding peers**: If a peer never responds to `GetBlockTransactions` for a pending entry, disconnect and remove their entries.

---

### Proof of Concept

**Attacker-controlled entry path**: Any peer connected via the `RelayV3` P2P protocol can send `CompactBlock` messages. No special privilege is required beyond valid PoW.

```
1. Attacker mines valid CKB blocks (requires hashpower, not validator status).
2. For each mined block, attacker sends a CompactBlock message to the target node
   with all short_ids listed but zero prefilled transactions.
3. The node calls reconstruct_block(), gets ReconstructionResult::Missing,
   and calls missing_or_collided_post_process() which inserts into pending_compact_blocks.
4. Attacker ignores all GetBlockTransactions responses — never sends the missing txs.
5. The entry remains in pending_compact_blocks forever (no timeout eviction).
6. Repeat for each newly mined block. Memory grows without bound.
```

The `non_contextual_check` passes because the block header is valid. The `contextual_check` passes because the PoW is valid and the parent is known. The entry is inserted and never cleaned up. [10](#0-9)

    pub async fn execute(self) -> Status {
        let instant = Instant::now();
        let shared = self.relayer.shared();
        let active_chain = shared.active_chain();
        let compact_block = self.message.to_entity();
        let header = compact_block.header().into_view();
        let block_hash = header.hash();

        let status =
            non_contextual_check(&compact_block, &header, shared.consensus(), &active_chain);
        if !status.is_ok() {
            return status;
        }

        let status = contextual_check(&header, shared, &active_chain, &self.nc, self.peer).await;
        if !status.is_ok() {
            return status;
        }

        // The new arrived has greater difficulty than local best known chain
        attempt!(CompactBlockVerifier::verify(&compact_block));
        // Header has been verified ok, update state
        shared.insert_valid_header(self.peer, &header);

        // Request proposal
        let proposals: Vec<_> = compact_block.proposals().into_iter().collect();
        self.relayer.request_proposal_txs(
            &self.nc,
            self.peer,
            (header.number(), block_hash.clone()).into(),
            proposals,
        );

        // Reconstruct block
        let ret = self
            .relayer
            .reconstruct_block(&active_chain, &compact_block, vec![], &[], &[])
            .await;

        // Accept block
        // `relayer.accept_block` will make sure the validity of block before persisting
        // into database
        match ret {
            ReconstructionResult::Block(block) => {
                if let Some(metrics) = ckb_metrics::handle() {
                    metrics
                        .ckb_relay_cb_transaction_count
                        .inc_by(block.transactions().len() as u64);
                    metrics.ckb_relay_cb_reconstruct_ok.inc();
                }
                let mut pending_compact_blocks = shared.state().pending_compact_blocks().await;
                pending_compact_blocks.remove(&block_hash);
                // remove all pending request below this block epoch
                //
                // use epoch as the judgment condition because we accept
                // all block in current epoch as uncle block
                pending_compact_blocks.retain(|_, (v, _, _)| {
                    Into::<EpochNumberWithFraction>::into(v.header().as_reader().raw().epoch())
                        .number()
                        >= block.epoch().number()
                });
                shrink_to_fit!(pending_compact_blocks, 20);
                self.relayer
                    .accept_block(Arc::clone(&self.nc), self.peer, block, "CompactBlock");

                if let Some(metrics) = ckb_metrics::handle() {
                    metrics
                        .ckb_relay_cb_verify_duration
                        .observe(instant.elapsed().as_secs_f64());
                }
                Status::ok()
            }
            ReconstructionResult::Missing(transactions, uncles) => {
                let missing_transactions: Vec<u32> =
                    transactions.into_iter().map(|i| i as u32).collect();

                if let Some(metrics) = ckb_metrics::handle() {
                    metrics
                        .ckb_relay_cb_fresh_tx_cnt
                        .inc_by(missing_transactions.len() as u64);
                    metrics.ckb_relay_cb_reconstruct_fail.inc();
                }

                let missing_uncles: Vec<u32> = uncles.into_iter().map(|i| i as u32).collect();
                missing_or_collided_post_process(
                    compact_block,
                    block_hash.clone(),
                    shared,
                    self.nc,
                    missing_transactions,
                    missing_uncles,
                    self.peer,
                )
                .await;

                StatusCode::CompactBlockRequiresFreshTransactions.with_context(&block_hash)
            }
            ReconstructionResult::Collided => {
                let missing_transactions: Vec<u32> = compact_block
                    .short_id_indexes()
                    .into_iter()
                    .map(|i| i as u32)
                    .collect();
                let missing_uncles: Vec<u32> = vec![];
                missing_or_collided_post_process(
                    compact_block,
                    block_hash.clone(),
                    shared,
                    self.nc,
                    missing_transactions,
                    missing_uncles,
                    self.peer,
                )
                .await;
                StatusCode::CompactBlockMeetsShortIdsCollision.with_context(&block_hash)
            }
            ReconstructionResult::Error(status) => status,
        }
    }
