### Title
Missing Canonical-Chain Check in `verify_transaction_proof` Allows Fork-Block Proof Acceptance — (`rpc/src/module/chain.rs`)

---

### Summary

`verify_transaction_proof` (and its sibling `verify_transaction_and_witness_proof`) fetches a block from the snapshot store using only the caller-supplied `block_hash`, with no check that the block is on the canonical (main) chain. Because the CKB store persists fork/orphan blocks alongside main-chain blocks, a caller can supply a `TransactionProof` whose `block_hash` points to a fork block, receive a cryptographically valid `Ok(tx_hashes)` response, and deceive any application that treats this response as proof of canonical-chain finality.

---

### Finding Description

In `rpc/src/module/chain.rs`, `verify_transaction_proof` is implemented as:

```rust
fn verify_transaction_proof(&self, tx_proof: TransactionProof) -> Result<Vec<H256>> {
    let snapshot = self.shared.snapshot();

    snapshot
        .get_block(&(&tx_proof.block_hash).into())   // ← no is_main_chain guard
        .ok_or_else(|| { ... })
        .and_then(|block| {
            // only verifies CBMT proof against block.transactions_root()
            ...
            Some(tx_hashes.iter().map(|hash| hash.into()).collect())
        })
}
``` [1](#0-0) 

`snapshot.get_block` is a pure hash-keyed lookup against `COLUMN_BLOCK_HEADER` / `COLUMN_BLOCK_BODY`: [2](#0-1) 

The canonical-chain membership test is a separate predicate that queries `COLUMN_INDEX`: [3](#0-2) 

`verify_transaction_proof` never calls `is_main_chain`. Compare this with `get_block_by_hash`, which explicitly gates on it: [4](#0-3) 

And `get_fork_block`, which explicitly checks `is_main_chain` to distinguish fork blocks from main-chain blocks: [5](#0-4) 

The same missing guard exists in `verify_transaction_and_witness_proof`: [6](#0-5) 

---

### Impact Explanation

An application (exchange, payment processor, light-client wallet) that calls `verify_transaction_proof` and treats `Ok(tx_hashes)` as "transaction confirmed on canonical chain" can be deceived. The attacker presents a proof for a transaction that exists only on a fork branch — one that was reorganized away or never became canonical. The node returns a successful verification, yet the transaction's outputs are not spendable on the main chain and may have been double-spent.

---

### Likelihood Explanation

- Fork blocks are routinely stored by every CKB full node during normal operation (natural forks, uncle candidates, reorgs).
- The `get_fork_block` RPC is a public, unauthenticated endpoint that returns fork block hashes, giving the attacker the required `block_hash` without any special access.
- `get_transaction_proof` itself accepts an optional `block_hash` parameter; an attacker can call it with a fork block hash to obtain a ready-made valid proof, then immediately pass it to `verify_transaction_proof`.
- No PoW, no privileged role, no key material is required — only two sequential unauthenticated RPC calls.

---

### Recommendation

Add an `is_main_chain` guard at the top of both `verify_transaction_proof` and `verify_transaction_and_witness_proof`, mirroring the pattern already used in `get_block_by_hash`:

```rust
fn verify_transaction_proof(&self, tx_proof: TransactionProof) -> Result<Vec<H256>> {
    let snapshot = self.shared.snapshot();
    let block_hash: packed::Byte32 = (&tx_proof.block_hash).into();

    if !snapshot.is_main_chain(&block_hash) {
        return Err(RPCError::invalid_params(format!(
            "Block {:#x} is not on the main chain",
            tx_proof.block_hash
        )));
    }
    // ... rest of existing logic
}
```

Apply the same fix to `verify_transaction_and_witness_proof`.

---

### Proof of Concept

1. Run a CKB dev-chain node.
2. Mine a fork: produce block A at height N on the main chain, then produce a competing block B at height N (different nonce/transactions, lower total difficulty so it stays a fork). Submit B via `process_block_without_verify` or natural sync so it is stored but not canonical.
3. Place transaction T only in block B.
4. Call `get_transaction_proof([T.hash], B.hash)` → receives a valid `TransactionProof` with `block_hash = B.hash`.
5. Call `verify_transaction_proof(<proof from step 4>)`.
6. **Observed**: returns `Ok([T.hash])`.
7. **Expected**: returns an error because B is not on the canonical chain.
8. Confirm T is absent from the canonical chain by calling `get_transaction(T.hash)` and observing `min_replace_fee` / `tx_status` shows it is not committed on the main chain.

        let snapshot = self.shared.snapshot();

            })

                let merkle_proof = MerkleProof::new(
                    tx_proof
                        .proof
                        .indices
                        .into_iter()
                        .map(|index| index.value())
                        .collect(),
                    tx_proof
                        .proof
                        .lemmas
                        .into_iter()
                        .map(|lemma| lemma.into())
                        .collect(),
                );

                CBMT::retrieve_leaves(block.tx_hashes(), &merkle_proof)
                    .and_then(|tx_hashes| {

                                    Some(tx_hashes.iter().map(|hash| hash.into()).collect())

                                    None

        let header = self.get_block_header(h)?;
        if let Some(freezer) = self.freezer()
            && header.number() > 0
            && header.number() < freezer.number()
        {
            let raw_block = freezer.retrieve(header.number()).expect("block frozen")?;
            let raw_block_reader =
                packed::BlockReader::from_compatible_slice(&raw_block).expect("checked data");
            if raw_block_reader.calc_header_hash().as_slice() == h.as_slice() {
                return Some(raw_block_reader.to_entity().into_view());
            }
        }
        let body = self.get_block_body(h);
        let uncles = self
            .get_block_uncles(h)
            .expect("block uncles must be stored");
        let proposals = self
            .get_block_proposal_txs_ids(h)
            .expect("block proposal_ids must be stored");
        let extension_opt = self.get_block_extension(h);

            BlockView::new_unchecked_with_extension(header, uncles, body, proposals, extension)

            BlockView::new_unchecked(header, uncles, body, proposals)
        };

    }
