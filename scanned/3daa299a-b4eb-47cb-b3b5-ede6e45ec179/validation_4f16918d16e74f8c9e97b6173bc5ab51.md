### Title
Unbounded `by_out_point` Index Growth in OrphanPool Allows Memory Amplification via High-Input Orphan Transactions — (`tx-pool/src/component/orphan.rs`)

### Summary
`OrphanPool` enforces a 100-transaction count limit via `DEFAULT_MAX_ORPHAN_TRANSACTIONS`, but the secondary index `by_out_point` is bounded only by the total number of input `OutPoint`s across all orphan transactions. An unprivileged remote peer can submit 100 transactions each with the maximum number of inputs (~11,636 given the 512KB tx size limit), causing `by_out_point` to grow to ~1.16M entries while `entries.len() == 100`.

### Finding Description
In `add_orphan_tx`, after inserting the transaction into `entries`, every input `OutPoint` is inserted into `by_out_point` with no cap on the total number of `OutPoint` entries:

```rust
for out_point in tx.input_pts_iter() {
    self.by_out_point
        .entry(out_point)
        .or_default()
        .insert(tx.proposal_short_id());
}
```

`limit_size()` then enforces `self.len() > DEFAULT_MAX_ORPHAN_TRANSACTIONS`, but `self.len()` is `self.entries.len()` — the transaction count — not the `by_out_point` entry count. The `by_out_point` map is never directly bounded. [6](#0-5) [7](#0-6) 

### Impact Explanation
Each `OutPoint` key is 36 bytes; each `HashSet<ProposalShortId>` value adds ~50–100 bytes of overhead. At ~1.16M entries, `by_out_point` alone consumes ~100–150MB of heap memory, far exceeding what the 100-tx count limit implies. This is a memory amplification DoS: the attacker uses 100 relay messages to force ~1.16M map entries.

### Likelihood Explanation
The attack path is fully reachable by an unprivileged remote peer via P2P relay (`relay_tx` → `add_orphan` → `add_orphan_tx`). The only upstream guard is `TRANSACTION_SIZE_LIMIT = 512KB`, which limits inputs per tx to ~11,636 but does not prevent the amplification. No PoW, no privileged access, and no special conditions are required. [8](#0-7) 

### Recommendation
Replace the transaction-count-based limit with a limit that accounts for total `by_out_point` entries (or total serialized size of all orphan transactions). For example:
- Cap `by_out_point.len()` directly (e.g., at `DEFAULT_MAX_ORPHAN_TRANSACTIONS * MAX_INPUTS_PER_TX`), or
- Track total input count across all orphan transactions and reject/evict when it exceeds a threshold, or
- Apply a per-peer orphan input quota to prevent a single peer from monopolizing the index.

    fn limit_size(&mut self) -> Vec<Byte32> {
        let now = ckb_systemtime::unix_time().as_secs();
        let expires: Vec<_> = self
            .entries
            .iter()
            .filter_map(|(id, entry)| {
                if entry.expires_at <= now {
                    Some(id)
                } else {
                    None
                }
            })
            .cloned()
            .collect();

        let mut evicted_txs = vec![];

        for id in expires {
            if let Some(entry) = self.remove_orphan_tx(&id) {
                evicted_txs.push(entry.tx.hash());
            }
        }

        while self.len() > DEFAULT_MAX_ORPHAN_TRANSACTIONS {
            // Evict a random orphan:
            let id = self.entries.keys().next().cloned().expect("bound checked");
            if let Some(entry) = self.remove_orphan_tx(&id) {
                evicted_txs.push(entry.tx.hash());
            }
        }

        if !evicted_txs.is_empty() {
            trace!("OrphanTxPool full, evicted {} tx", evicted_txs.len());
            self.shrink_to_fit();
        }
        evicted_txs
    }

    pub fn add_orphan_tx(
        &mut self,
        tx: TransactionView,
        peer: PeerIndex,
        declared_cycle: Cycle,
    ) -> Vec<Byte32> {
        if self.entries.contains_key(&tx.proposal_short_id()) {
            return vec![];
        }

        debug!("add_orphan_tx {}", tx.hash());
        self.entries.insert(
            tx.proposal_short_id(),
            Entry::new(tx.clone(), peer, declared_cycle),
        );

        for out_point in tx.input_pts_iter() {
            self.by_out_point
                .entry(out_point)
                .or_default()
                .insert(tx.proposal_short_id());
        }

        // DoS prevention: do not allow OrphanPool to grow unbounded
        self.limit_size()

pub(crate) fn non_contextual_verify(
    consensus: &Consensus,
    tx: &TransactionView,
) -> Result<(), Reject> {
    NonContextualTransactionVerifier::new(tx, consensus)
        .verify()
        .map_err(Reject::Verification)?;

    // The ckb consensus does not limit the size of a single transaction,
    // but if the size of the transaction is close to the limit of the block,
    // it may cause the transaction to fail to be packed
    let tx_size = tx.data().serialized_size_in_block() as u64;
    if tx_size > TRANSACTION_SIZE_LIMIT {
        return Err(Reject::ExceededTransactionSizeLimit(
            tx_size,
            TRANSACTION_SIZE_LIMIT,
        ));
    }
    // cellbase is only valid in a block, not as a loose transaction
    if tx.is_cellbase() {
        return Err(Reject::Malformed(
            "cellbase like".to_owned(),
            Default::default(),
        ));
    }

    Ok(())
}
