### Title
Arbitrary `providerToCredit` in `Echo.executeCallback` Allows Fee Theft After Exclusivity Period - (File: target_chains/ethereum/contracts/contracts/echo/Echo.sol)

### Summary
The `executeCallback` function in `Echo.sol` accepts a caller-controlled `providerToCredit` parameter that is only validated during the exclusivity window. Once the exclusivity period expires, any unprivileged caller can pass an arbitrary address as `providerToCredit`, redirecting the entire request fee away from the assigned provider to any address they control — including their own registered provider account.

### Finding Description
In `Echo.sol`, `executeCallback` enforces that `providerToCredit == req.provider` only while the exclusivity window is active:

```solidity
if (
    block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds
) {
    require(
        providerToCredit == req.provider,
        "Only assigned provider during exclusivity period"
    );
}
```

After the window closes, no such constraint exists. The fee is then unconditionally credited to whatever address was passed:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
```

There is no check that `providerToCredit` equals `msg.sender`, equals `req.provider`, or is even a registered provider. Provider registration is permissionless via `registerProvider`, and any registered provider can set themselves as their own fee manager via `setFeeManager`. The only withdrawal path for provider-accrued fees is `withdrawAsFeeManager`, which pays `msg.sender` when `msg.sender == _state.providers[provider].feeManager`.

This creates a complete, permissionless theft path:

1. Attacker calls `echo.registerProvider(fee, fee, fee)` — permissionless.
2. Attacker calls `echo.setFeeManager(attacker_address)` — sets self as fee manager.
3. Attacker monitors the mempool or on-chain state for pending requests whose exclusivity period has elapsed (default: 15 seconds after `publishTime`).
4. Attacker calls `echo.executeCallback(attacker_address, sequenceNumber, updateData, priceIds)` — `updateData` is public from Pyth price feeds.
5. `req.fee` (the full provider fee paid by the requester) is credited to `_state.providers[attacker_address].accruedFeesInWei`.
6. Attacker calls `echo.withdrawAsFeeManager(attacker_address, stolenFee)` — receives the stolen fee.

The assigned provider's transaction, if submitted concurrently, reverts with `NoSuchRequest` because the request was already cleared at step 4.

### Impact Explanation
The assigned provider (`req.provider`) loses the entire fee they were entitled to for fulfilling the request. The attacker gains that fee without having been the designated provider. Because `req.fee` is set at request time to `msg.value - _state.pythFeeInWei`, the stolen amount equals the full provider portion of the fee paid by the requester. This is a direct, repeatable financial loss for every assigned provider whose requests are targeted after the exclusivity window.

### Likelihood Explanation
The attack requires no privileged access. Provider registration is permissionless. The `updateData` needed to call `executeCallback` is publicly available from Pyth price feeds. The exclusivity period defaults to 15 seconds, meaning the attack window opens almost immediately after each request. An attacker can automate this by monitoring `PriceUpdateRequested` events and submitting `executeCallback` with `providerToCredit = attacker_address` as soon as `block.timestamp >= req.publishTime + exclusivityPeriodSeconds`. The attack is profitable whenever the stolen fee exceeds the attacker's gas cost.

### Recommendation
After the exclusivity period, restrict `providerToCredit` to either the actual caller (`msg.sender`) or the assigned provider (`req.provider`). The simplest fix is to remove the `providerToCredit` parameter entirely and always credit `msg.sender`, ensuring the entity doing the work receives the fee. Alternatively, add a post-exclusivity check:

```solidity
if (block.timestamp >= req.publishTime + _state.exclusivityPeriodSeconds) {
    require(
        providerToCredit == msg.sender,
        "providerToCredit must be caller after exclusivity"
    );
}
```

echo.withdrawAsFeeManager(attacker, stolen);
