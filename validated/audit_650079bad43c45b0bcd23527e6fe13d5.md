### Title
Permanent Freezing of All Protocol Update Fees Inside `PriceInfoObject` Dynamic Fields — No Withdrawal Path Exists — (`target_chains/sui/contracts/sources/price_info.move`)

---

### Summary

Every call to `update_single_price_feed` deposits SUI fee coins into a per-`PriceInfoObject` dynamic field. No function anywhere in the production codebase can extract those coins. The `fee_recipient_address` stored in `State` is explicitly documented as unused. All fees paid since deployment are permanently locked.

---

### Finding Description

**Step 1 — Fee deposit on every update.**

`update_single_price_feed` in `pyth.move` unconditionally calls `price_info::deposit_fee_coins` after the fee-sufficiency check: [1](#0-0) 

**Step 2 — Coins accumulate in a dynamic field with no drain.**

`deposit_fee_coins` in `price_info.move` either creates a new `Coin<SUI>` dynamic object field or joins into the existing one: [2](#0-1) 

The only other function touching this field is `get_balance`, which is read-only: [3](#0-2) 

**Step 3 — No withdrawal function exists anywhere.**

A full search of all `.move` files under `target_chains/sui/contracts/sources/` for `withdraw_fee`, `collect_fee`, or any function that borrows `FEE_STORAGE_KEY` mutably for extraction returns zero results. `deposit_fee_coins` and `get_balance` are the only two functions that reference `FEE_STORAGE_KEY`.

**Step 4 — `fee_recipient_address` is explicitly dead.**

`State` stores a `fee_recipient_address` field: [4](#0-3) 

The governance module that manages this field carries an explicit comment confirming it is no longer used: [5](#0-4) 

`set_fee_recipient` only updates the address field in `State`; it never touches any `PriceInfoObject` balance. There is no code path that routes coins from a `PriceInfoObject` to `fee_recipient_address`.

---

### Impact Explanation

Every SUI coin paid as an update fee is permanently locked inside the `PriceInfoObject` that was updated. With `N` price feeds each updated `K` times at fee `F`, the total locked amount is `N × K × F` MIST. These coins cannot be recovered by the protocol, the fee recipient, or any governance action. This is a permanent, irreversible freezing of protocol revenue funds.

---

### Likelihood Explanation

This is not a conditional or edge-case path. It triggers on every single successful call to `update_single_price_feed`, which is the primary production entry point for price updates. The accumulation is deterministic and begins from the first update after deployment.

---

### Recommendation

Implement a privileged `withdraw_fees` function in `price_info.move` (gated on `LatestOnly` or a governance capability) that removes the `Coin<SUI>` dynamic field from a `PriceInfoObject` and transfers it to `state::get_fee_recipient`. Wire this into a governance action or expose it as a callable entry function restricted to the fee recipient address. Alternatively, redirect fees at deposit time directly to the fee recipient via `sui::transfer::public_transfer` instead of storing them in the object.

---

### Proof of Concept

1. Deploy the contract with `base_update_fee = 50`.
2. Call `update_single_price_feed` on a `PriceInfoObject` 100 times, each time passing a `Coin<SUI>` of value 50.
3. Call `price_info::get_balance(&price_info_object)` — it returns `5000`.
4. Attempt to find any public or friend function in `pyth.move`, `price_info.move`, `state.move`, or any governance module that accepts a `&mut PriceInfoObject` and returns or transfers a `Coin<SUI>` — none exists.
5. Confirm `state::get_fee_recipient` returns a valid address but that address has received zero coins from any `PriceInfoObject`.

The invariant `sum(PriceInfoObject balances) == 0` (all fees forwarded to recipient) is permanently broken from the first update onward.

    public fun deposit_fee_coins(price_info_object: &mut PriceInfoObject, fee_coins: Coin<SUI>) {
        if (!dynamic_object_field::exists_with_type<vector<u8>, Coin<SUI>>(&price_info_object.id, FEE_STORAGE_KEY)) {
            dynamic_object_field::add(&mut price_info_object.id, FEE_STORAGE_KEY, fee_coins);
        }
        else {
            let current_fee = dynamic_object_field::borrow_mut<vector<u8>, Coin<SUI>>(
                &mut price_info_object.id,
                FEE_STORAGE_KEY
            );
            coin::join(current_fee, fee_coins);
        };
    }
