### Title
`PoolMap::add_entry` Overwrites Post-Eviction `total_tx_size`/`total_tx_cycles` With Stale Pre-Computed Values - (`tx-pool/src/component/pool_map.rs`)

### Summary

In `PoolMap::add_entry`, the new entry's contribution to `total_tx_size` and `total_tx_cycles` is pre-computed before any evictions occur. When `check_and_record_ancestors` evicts conflicting cell-dep-parent transactions (each of which correctly decrements the totals via `update_stat_for_remove_tx`), those decrements are immediately overwritten by the stale pre-computed values. The result is that `total_tx_size` and `total_tx_cycles` become permanently inflated by the sum of all evicted entries' sizes and cycles. An unprivileged tx-pool submitter can trigger this path repeatedly, causing the pool to falsely believe it is full and reject or evict legitimate transactions.

### Finding Description

In `PoolMap::add_entry` the sequence is:

1. **Pre-compute** new totals (before any evictions):
   ```rust
   let (total_tx_size, total_tx_cycles) =
       self.updated_stat_for_add_tx(entry.size, entry.cycles)?;
   ```
   At this point `total_tx_size = self.total_tx_size + entry.size`.

2. **Evict** conflicting cell-dep-parent entries inside `check_and_record_ancestors`:
   ```rust
   let removed = self.remove_entry_and_descendants(next_id);
   ```
   Each call to `remove_entry_and_descendants` → `remove_entry` → `update_stat_for_remove_tx` correctly **decrements** `self.total_tx_size` and `self.total_tx_cycles` for every evicted entry.

3. **Overwrite** with the stale pre-computed values:
   ```rust
   self.total_tx_size = total_tx_size;
   self.total_tx_cycles = total_tx_cycles;
   ```
   This discards all the decrements applied in step 2.

**Concrete arithmetic:**
- Before: `self.total_tx_size = S`, `self.total_tx_cycles = C`
- New entry: size `s`, cycles `c` → pre-computed: `S+s`, `C+c`
- Eviction of entry with size `e_s`, cycles `e_c` → `self.total_tx_size = S−e_s`, `self.total_tx_cycles = C−e_c`
- After overwrite: `self.total_tx_size = S+s` (should be `S−e_s+s`), `self.total_tx_cycles = C+c` (should be `C−e_c+c`)
- **Inflation per trigger: `e_s` bytes, `e_c` cycles**

The eviction path is reached when `ancestors_count > max_ancestors_count` but `ancestors_count − cell_ref_parents.len() ≤ max_ancestors_count`, i.e., the excess ancestors are all cell-dep-parent transactions that can be evicted. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

### Impact Explanation

`total_tx_size` is the sole guard for pool admission (`limit_size`) and the overflow check in `updated_stat_for_add_tx`. An inflated value causes:

1. **Spurious `Reject::Full`**: Legitimate transactions are rejected even when the pool has real capacity, because `total_tx_size > max_tx_pool_size` is evaluated against the inflated counter.
2. **Unnecessary eviction of honest transactions**: `limit_size` loops while `total_tx_size > max_tx_pool_size` and evicts lowest-fee-rate entries, permanently removing valid transactions from the pool.
3. **Inaccurate fee-rate estimation**: `estimate_fee_rate` and block-assembly decisions depend on the pool's apparent fullness.

The inflation is **permanent and cumulative**: each successful trigger adds `e_s` bytes and `e_c` cycles to the drift. The pool never self-corrects unless `recompute_total_stat` is called (only on underflow, not on this path). [5](#0-4) [6](#0-5) 

### Likelihood Explanation

The trigger requires:
1. A chain of in-pool transactions T1→T2→…→Tn where n approaches `max_ancestors_count` (default 25).
2. At least one other in-pool transaction that uses an output of a transaction in that chain as a **cell dep** (making it a `cell_ref_parent`).
3. A new transaction that spends that same output as an **input**, triggering the eviction branch.

An unprivileged attacker with a modest CKB balance can construct this setup entirely through the public `send_transaction` RPC or P2P relay. No privileged access, leaked keys, or majority hashpower is required. The attacker can repeat the trigger to accumulate unbounded inflation. [7](#0-6) [8](#0-7) 

### Recommendation

Move the stat pre-computation to **after** all evictions have completed, so it captures the post-eviction baseline:

```rust
pub(crate) fn add_entry(
    &mut self,
    mut entry: TxEntry,
    status: Status,
) -> Result<(bool, HashSet<TxEntry>), Reject> {
    let tx_short_id = entry.proposal_short_id();
    if self.entries.get_by_id(&tx_short_id).is_some() {
        return Ok((false, Default::default()));
    }
    // Validate capacity before evictions (overflow guard only)
    self.updated_stat_for_add_tx(entry.size, entry.cycles)?;

    let evicts = self.check_and_record_ancestors(&mut entry)?;
    self.record_entry_edges(&entry)?;
    self.insert_entry(&entry, status);
    self.record_entry_descendants(&entry);
    self.track_entry_statics(None, Some(status));

    // Re-compute AFTER evictions so evicted sizes are already subtracted
    let (total_tx_size, total_tx_cycles) =
        self.updated_stat_for_add_tx(entry.size, entry.cycles)?;
    self.total_tx_size = total_tx_size;
    self.total_tx_cycles = total_tx_cycles;
    Ok((true, evicts))
}
```

Alternatively, apply the new entry's increment directly (`self.total_tx_size += entry.size`) instead of using a pre-computed snapshot, so intermediate eviction decrements are preserved.

### Proof of Concept

1. Submit transactions A, B, C, … up to `max_ancestors_count − 1` in a chain (A→B→C→…).
2. Submit transaction X that uses output of A as a **cell dep** (X is now a `cell_ref_parent` of A).
3. Submit transaction Z that spends the same output of A as an **input** and has A as an ancestor.
   - `ancestors_count` = chain length + 1 > `max_ancestors_count`
   - `cell_ref_parents` = {X}; `ancestors_count − 1 ≤ max_ancestors_count` → eviction branch taken
   - X is evicted: `self.total_tx_size -= X.size` (correct decrement)
   - Z is inserted: `self.total_tx_size = pre_computed` (overwrites, re-inflates by `X.size`)
4. Query `tx_pool_info`: `total_tx_size` is now `X.size` bytes larger than the actual sum of entries.
5. Repeat to accumulate inflation until `total_tx_size > max_tx_pool_size`, causing all subsequent submissions to be rejected with `Reject::Full`. [1](#0-0) [9](#0-8) [10](#0-9)

    fn get_tx_ancenstors(
        &self,
        entry: &TransactionView,
    ) -> (
        HashSet<ProposalShortId>,
        HashSet<ProposalShortId>,
        HashSet<ProposalShortId>,
    ) {
        let mut parents: HashSet<ProposalShortId> =
            HashSet::with_capacity(entry.inputs().len() + entry.cell_deps().len());
        let mut cell_ref_parents: HashSet<ProposalShortId> = Default::default();

        for input in entry.inputs() {
            let input_pt = input.previous_output();
            if let Some(deps) = self.edges.deps.get(&input_pt) {
                cell_ref_parents.extend(deps.iter().cloned());
                parents.extend(deps.iter().cloned());
            }

            let id = ProposalShortId::from_tx_hash(&input_pt.tx_hash());
            if self.links.inner.contains_key(&id) {
                parents.insert(id);
            }
        }
        for cell_dep in entry.cell_deps() {
            let dep_pt = cell_dep.out_point();
            let id = ProposalShortId::from_tx_hash(&dep_pt.tx_hash());
            if self.links.inner.contains_key(&id) {
                parents.insert(id);
            }
        }

        let ancestors = self
            .links
            .calc_relation_ids(parents.clone(), Relation::Parents);

        (ancestors, parents, cell_ref_parents)
    }

    fn check_and_record_ancestors(
        &mut self,
        entry: &mut TxEntry,
    ) -> Result<HashSet<TxEntry>, Reject> {
        let tx = entry.transaction();
        let (ancestors, mut parents, cell_ref_parents) = self.get_tx_ancenstors(tx);

        let mut ancestors_count = ancestors.len() + 1;
        let mut evicted = Default::default();

        if ancestors_count <= self.max_ancestors_count {
            self._record_ancestors(entry, ancestors, parents);
            return Ok(evicted);
        }

        if ancestors_count.saturating_sub(cell_ref_parents.len()) <= self.max_ancestors_count {
            // if ancestors count exceed limitation,
            // try to evict some conflicted transactions due to ref cells

            // sort them to find out the transactions with lowest fees
            let evict_candidates: Vec<ProposalShortId> = self
                .entries
                .iter_by_evict_key()
                .filter(move |entry| cell_ref_parents.contains(&entry.id))
                .map(|x| x.id.clone())
                .collect();

            let mut iter = evict_candidates.iter();
            while ancestors_count > self.max_ancestors_count {
                if let Some(next_id) = iter.next() {
                    let removed = self.remove_entry_and_descendants(next_id);
                    ancestors_count = ancestors_count.saturating_sub(1);
                    parents.remove(next_id);
                    evicted.extend(removed);
                } else {
                    break;
                }
            }
        } else {
            return Err(Reject::ExceededMaximumAncestorsCount);
        }

        // some txs in `parents` are removed, now `ancestors` need to re-caculate,
        let ancestors = self
            .links
            .calc_relation_ids(parents.clone(), Relation::Parents);

        // we can assume the number now is less than `max_ancestors_count`
        assert!(ancestors.len() < self.max_ancestors_count);

        self._record_ancestors(entry, ancestors, parents);
        Ok(evicted)
    }

    fn track_entry_statics(&mut self, remove: Option<Status>, add: Option<Status>) {
        match remove {
            Some(Status::Pending) => self.pending_count -= 1,
            Some(Status::Gap) => self.gap_count -= 1,
            Some(Status::Proposed) => self.proposed_count -= 1,
            _ => {}

            Some(Status::Pending) => self.pending_count += 1,
            Some(Status::Gap) => self.gap_count += 1,
            Some(Status::Proposed) => self.proposed_count += 1,
            _ => {}

        assert_eq!(
            self.pending_count + self.gap_count + self.proposed_count,
            self.entries.len()

            metrics

            metrics
                .ckb_tx_pool_entry

    /// Calculate size and cycles statistics for adding a tx.
    fn updated_stat_for_add_tx(
        &self,
        tx_size: usize,
        cycles: Cycle,
    ) -> Result<(usize, Cycle), Reject> {
        let total_tx_size = self.total_tx_size.checked_add(tx_size).ok_or_else(|| {
            Reject::Full(format!(
                "tx-pool total_tx_size {} overflows by add {}",
                self.total_tx_size, tx_size
            ))
        })?;
        let total_tx_cycles = self.total_tx_cycles.checked_add(cycles).ok_or_else(|| {
            Reject::Full(format!(
                "tx-pool total_tx_cycles {} overflows by add {}",
                self.total_tx_cycles, cycles
            ))
        })?;
        Ok((total_tx_size, total_tx_cycles))
    }

    /// Update size and cycles statistics for remove tx
    /// cycles overflow is possible, currently obtaining cycles is not accurate
    fn update_stat_for_remove_tx(&mut self, tx_size: usize, cycles: Cycle) {
        match (
            self.total_tx_size.checked_sub(tx_size),
            self.total_tx_cycles.checked_sub(cycles),
        ) {
            (Some(total_tx_size), Some(total_tx_cycles)) => {
                self.total_tx_size = total_tx_size;
                self.total_tx_cycles = total_tx_cycles;
            }
            _ => {
                if let Some((total_tx_size, total_tx_cycles)) = self.recompute_total_stat() {
                    error!(
                        "tx-pool total stats underflowed when removing size {} cycles {}, recomputed size {} cycles {}",
                        tx_size, cycles, total_tx_size, total_tx_cycles
                    );
                    self.total_tx_size = total_tx_size;
                    self.total_tx_cycles = total_tx_cycles;
                } else {
                    error!(
                        "tx-pool total stats underflowed when removing size {} cycles {}, and recomputing overflowed",
                        tx_size, cycles
                    );
                }
            }
        }
    }

    pub(crate) fn limit_size(
        &mut self,
        callbacks: &Callbacks,
        current_entry_id: Option<&ProposalShortId>,
    ) -> Option<Reject> {
        let mut ret = None;
        while self.pool_map.total_tx_size > self.config.max_tx_pool_size {
            let next_evict_entry = || {
                self.pool_map
                    .next_evict_entry(Status::Pending)
                    .or_else(|| self.pool_map.next_evict_entry(Status::Gap))
                    .or_else(|| self.pool_map.next_evict_entry(Status::Proposed))
            };

            if let Some(id) = next_evict_entry() {
                let removed = self.pool_map.remove_entry_and_descendants(&id);
                for entry in removed {
                    let tx_hash = entry.transaction().hash();
                    debug!(
                        "Removed by size limit {} timestamp({})",
                        tx_hash, entry.timestamp
                    );
                    let reject = Reject::Full(format!(
                        "the fee_rate for this transaction is: {}",
                        entry.fee_rate()
                    ));
                    if let Some(short_id) = current_entry_id
                        && entry.proposal_short_id() == *short_id
                    {
                        ret = Some(reject.clone());
                    }
                    callbacks.call_reject(self, &entry, reject);
                }
            }
        }
        self.pool_map.entries.shrink_to_fit();
        ret
    }
