### Title
`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` Always Revert for Whitelisted Subscriptions Due to External `this.` Call Changing `msg.sender` — (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

In `Scheduler.sol`, the functions `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` delegate to `getPricesUnsafe` and `getEmaPricesUnsafe` respectively via `this.<fn>(...)` — an external call. This changes `msg.sender` inside the callee to the Scheduler contract's own address. Both callees are guarded by `onlyWhitelistedReader`, which rejects any caller that is not the subscription manager or an explicitly whitelisted address. The Scheduler contract itself is neither, so for any subscription with `whitelistEnabled = true`, the internal delegation always reverts with `Unauthorized`, making `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` permanently non-functional for those subscriptions.

---

### Finding Description

`getPricesNoOlderThan` (line 535) and `getEmaPricesNoOlderThan` (line 578) both carry the `onlyWhitelistedReader(subscriptionId)` modifier, so the original external caller is correctly gated. However, after passing that gate, each function delegates to a sibling function via `this.`:

```solidity
// Scheduler.sol line 554
prices = this.getPricesUnsafe(subscriptionId, priceIds);

// Scheduler.sol line 597
prices = this.getEmaPricesUnsafe(subscriptionId, priceIds);
```

Because `this.<fn>()` is an external call, Solidity sets `msg.sender` to `address(this)` (the Scheduler proxy) inside the callee. The `onlyWhitelistedReader` modifier in `getPricesUnsafe` (line 521) and `getEmaPricesUnsafe` (line 564) then evaluates:

1. Is `msg.sender` the subscription manager? → No (manager is the user who created the subscription).
2. Is `whitelistEnabled` false? → If `true`, proceed to whitelist check.
3. Is `msg.sender` in the `readerWhitelist`? → No (the Scheduler contract is never added to the whitelist).

Result: `revert SchedulerErrors.Unauthorized()` — every call to `getPricesNoOlderThan` or `getEmaPricesNoOlderThan` for a subscription with `whitelistEnabled = true` reverts unconditionally, regardless of whether the original caller is legitimately whitelisted.

The root cause is the use of `this.` for internal delegation, which is the same structural flaw as the reported pattern: a privileged check (`onlyWhitelistedReader`) is applied on the outer function but the inner function re-applies the same check against a different, unintended `msg.sender`.

---

### Impact Explanation

Any subscription manager who enables `whitelistEnabled = true` to restrict price-read access to a curated set of addresses renders `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` permanently broken for that subscription. Downstream DeFi protocols or keepers that depend on these functions for staleness-bounded price reads will receive a hard revert instead of a price, causing a denial-of-service on any logic that calls these entry points. The only fallback is `getPricesUnsafe` / `getEmaPricesUnsafe`, which carry no staleness guarantee — defeating the purpose of the time-bounded variants.

---

### Likelihood Explanation

`whitelistEnabled` is a first-class, user-configurable field in `SubscriptionParams`. Any subscription manager who sets it to `true` (a reasonable choice for protocols that want to gate price reads) will silently break `getPricesNoOlderThan` and `getEmaPricesNoOlderThan`. No privileged role, leaked key, or external oracle manipulation is required — the bug is triggered by normal, documented usage of the Scheduler API.

---

### Recommendation

Replace the external `this.` delegation with a direct call to the internal helper `_getPricesInternal`, which already exists and performs the same data retrieval without re-entering the access-control layer:

```solidity
// Before (broken for whitelisted subscriptions):
prices = this.getPricesUnsafe(subscriptionId, priceIds);

// After (correct):
PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
prices = new PythStructs.Price[](priceFeeds.length);
for (uint i = 0; i < priceFeeds.length; i++) {
    prices[i] = priceFeeds[i].price;
}
```

Apply the same fix to `getEmaPricesNoOlderThan` (replace `this.getEmaPricesUnsafe` with a direct call to `_getPricesInternal` extracting `emaPrice`).

---

### Proof of Concept

1. Deploy `SchedulerUpgradeable` and create a subscription with `whitelistEnabled = true`, adding address `alice` to `readerWhitelist`.
2. Fund the subscription and trigger a price update so `priceLastUpdatedAt` is recent.
3. As `alice` (a legitimately whitelisted reader), call `getPricesNoOlderThan(subscriptionId, priceIds, 60)`.
4. Execution flow:
   - `onlyWhitelistedReader` passes for `alice` ✓
   - Staleness check passes ✓
   - `this.getPricesUnsafe(subscriptionId, priceIds)` is dispatched as an external call; `msg.sender` inside `getPricesUnsafe` is now `address(scheduler)`.
   - `onlyWhitelistedReader` in `getPricesUnsafe`: `address(scheduler)` is not the manager, `whitelistEnabled` is `true`, `address(scheduler)` is not in `readerWhitelist` → `revert SchedulerErrors.Unauthorized()`.
5. The transaction reverts despite `alice` being a valid whitelisted reader.

```text
    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
```
