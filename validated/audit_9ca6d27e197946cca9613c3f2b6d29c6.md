### Title
ETH Sent to `Executor` Contract Is Permanently Locked — No Withdrawal Mechanism - (File: `target_chains/ethereum/contracts/contracts/executor/Executor.sol`)

---

### Summary

The `Executor` contract exposes a `receive() external payable {}` function and a `payable execute()` function, but contains no mechanism to withdraw ETH that accumulates in the contract. Any ETH sent directly to the contract (or any excess `msg.value` over `gi.value` forwarded during `execute()`) is permanently locked.

---

### Finding Description

`Executor.sol` has two ETH-receiving surfaces:

1. **`receive() external payable {}`** (line 113) — an unconditional, open fallback that accepts ETH from any sender with no calldata.
2. **`execute(bytes memory encodedVm) public payable`** (line 68) — a payable function that forwards exactly `gi.value` wei to `callAddress` via `address(callAddress).call{value: gi.value}(gi.callData)` (line 92). Any `msg.value` in excess of `gi.value` is silently retained by the contract.

There is no `withdraw`, `withdrawFee`, or any other function in `Executor.sol` that moves ETH out of the contract to an owner or admin. The contract holds no accounting of its own ETH balance, and governance cannot issue a VAA that calls a non-existent withdrawal function on itself. [1](#0-0) [2](#0-1) 

---

### Impact Explanation

Any ETH sent to the `Executor` contract — whether by a user calling `receive()` directly, or by a governance relayer who passes `msg.value > gi.value` to `execute()` — is permanently locked in the contract with no recovery path. There is no admin, owner, or governance action that can retrieve it. The contract has no `withdraw` function and no accounting of its own balance.

---

### Likelihood Explanation

The `receive()` function is completely permissionless — any unprivileged address can send ETH to the contract at any time. This is a realistic scenario: users or integrators may accidentally send ETH to the contract address (e.g., mistaking it for a fee-accepting contract), or a governance relayer may overpay `msg.value` relative to `gi.value` in an `execute()` call. The `receive()` function is explicitly documented as intentional (`@dev Called when msg.value is not zero and the call data is empty`), confirming the contract is designed to accept ETH, yet no withdrawal path exists. [1](#0-0) 

---

### Recommendation

Either:
1. **Remove `receive()`** if the contract is not intended to hold ETH independently of forwarding it via `execute()`.
2. **Add a governance-controlled withdrawal function** that allows the authorized governance emitter (via a VAA) to sweep the contract's ETH balance to a target address — analogous to `PythGovernance.withdrawFee()`.
3. **Add a refund in `execute()`**: after forwarding `gi.value`, refund `msg.value - gi.value` to `msg.sender`. [3](#0-2) 

---

### Proof of Concept

```solidity
// Anyone can permanently lock ETH in Executor:
(bool ok,) = address(executor).call{value: 1 ether}("");
// ok == true, 1 ether is now locked forever — no withdrawal function exists.

// Alternatively, a governance relayer overpays execute():
executor.execute{value: 2 ether}(encodedVaa);
// gi.value == 1 ether is forwarded to callAddress,
// but the remaining 1 ether stays in Executor with no recovery path.
``` [4](#0-3) [1](#0-0)

    function execute(
        bytes memory encodedVm
    ) public payable returns (bytes memory response) {
        IWormhole.VM memory vm = verifyGovernanceVM(encodedVm);

        GovernanceInstruction memory gi = parseGovernanceInstruction(
            vm.payload
        );

        if (gi.targetChainId != chainId && gi.targetChainId != 0)
            revert ExecutorErrors.InvalidGovernanceTarget();

        if (
            gi.action != ExecutorAction.Execute ||
            gi.executorAddress != address(this)
        ) revert ExecutorErrors.DeserializationError();

        // Check if the gi.callAddress is a contract account.
        uint len;
        address callAddress = address(gi.callAddress);
        assembly {
            len := extcodesize(callAddress)
        }
        if (len == 0) revert ExecutorErrors.InvalidContractTarget();

        bool success;
        (success, response) = address(callAddress).call{value: gi.value}(
            gi.callData
        );
