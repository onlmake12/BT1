### Title
Permanent Subscription ETH Permanently Locked in Scheduler — No Admin Recovery Mechanism (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

ETH deposited into permanent subscriptions in the `Scheduler` contract is permanently locked. The `withdrawFunds()` function unconditionally reverts for permanent subscriptions, and `SchedulerGovernance.sol` contains no ETH recovery function. There is no path — for the subscription manager, the admin, or governance — to retrieve the deposited native tokens.

---

### Finding Description

`Scheduler.createSubscription()` and `Scheduler.addFunds()` are both `payable` and credit `msg.value` to `status.balanceInWei`. [1](#0-0) 

`addFunds()` imposes no caller restriction — anyone can deposit ETH into any active permanent subscription: [2](#0-1) 

`withdrawFunds()` unconditionally reverts for permanent subscriptions before any ETH is returned:

```solidity
if (params.isPermanent) {
    revert SchedulerErrors.CannotUpdatePermanentSubscription();
}
``` [3](#0-2) 

`SchedulerGovernance.sol` exposes only admin-transfer and fee-parameter setters — there is no `withdrawFees`, `rescueETH`, or equivalent function: [4](#0-3) 

The only paths by which ETH leaves the contract are keeper payments (`_processFeesAndPayKeeper`) and Pyth oracle fees (`parsePriceFeedUpdatesWithConfig{value: pythFee}`). Both require a keeper to actively call `updatePriceFeeds`. If no keeper calls the function (e.g., the subscription's price feeds are deprecated, the keeper infrastructure is down, or the keeper fee exceeds the remaining balance), the residual ETH balance is permanently stranded. [5](#0-4) 

---

### Impact Explanation

**High.** Any ETH remaining in a permanent subscription's `balanceInWei` that is not consumed by keeper fees or Pyth fees is irrecoverable. The subscription manager cannot withdraw it, the admin cannot withdraw it, and there is no governance action that can transfer it. This is a direct loss of user funds.

---

### Likelihood Explanation

**High.** The `isPermanent` flag is a first-class feature exposed to all callers of `createSubscription`. Any user who creates a permanent subscription and deposits ETH is affected. Additionally, `addFunds()` has no caller restriction, so third parties can inadvertently or deliberately increase the locked balance of any permanent subscription.

---

### Recommendation

Add an admin-level ETH recovery function to `SchedulerGovernance`, analogous to the `withdrawFee` governance action in `PythGovernance.sol`:

```solidity
function withdrawFunds(address payable recipient, uint256 amount) external {
    _authorizeAdminAction();
    require(recipient != address(0), "zero address");
    (bool sent, ) = recipient.call{value: amount}("");
    require(sent, "transfer failed");
}
```

Alternatively, allow the subscription manager to recover residual funds from a permanent subscription once its balance falls below the minimum required to execute a single update, so the subscription is effectively exhausted.

---

### Proof of Concept

1. Alice calls `createSubscription({isPermanent: true, ...}){value: 10 ether}`. Her ETH is credited to `status.balanceInWei`. [1](#0-0) 
2. The keeper infrastructure goes offline; no one calls `updatePriceFeeds` for this subscription.
3. Alice calls `withdrawFunds(subscriptionId, 10 ether)`. The call reverts with `CannotUpdatePermanentSubscription`. [6](#0-5) 
4. The admin calls no recovery function because none exists in `SchedulerGovernance`. [7](#0-6) 
5. Alice's 10 ETH is permanently locked in the contract.

    function withdrawFunds(
        uint256 subscriptionId,
        uint256 amount
    ) external override onlyManager(subscriptionId) {
        SchedulerStructs.SubscriptionStatus storage status = _state
            .subscriptionStatuses[subscriptionId];
        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        // Prevent withdrawals from permanent subscriptions
        if (params.isPermanent) {
            revert SchedulerErrors.CannotUpdatePermanentSubscription();
        }

        if (status.balanceInWei < amount) {
            revert SchedulerErrors.InsufficientBalance();
        }

        // If subscription is active, ensure minimum balance is maintained
        if (params.isActive) {
            uint256 minimumBalance = this.getMinimumBalance(
                uint8(params.priceIds.length)
            );
            if (status.balanceInWei - amount < minimumBalance) {
                revert SchedulerErrors.InsufficientBalance();
            }
        }

        status.balanceInWei -= amount;

        (bool sent, ) = msg.sender.call{value: amount}("");
        require(sent, "Failed to send funds");
    }

    function _processFeesAndPayKeeper(
        SchedulerStructs.SubscriptionStatus storage status,
        uint256 startGas,
        uint256 numPriceIds
    ) internal {
        // Calculate fee components
        uint256 gasCost = (startGas - gasleft() + GAS_OVERHEAD) * tx.gasprice;
        uint256 keeperSpecificFee = uint256(_state.singleUpdateKeeperFeeInWei) *
            numPriceIds;
        uint256 totalKeeperFee = gasCost + keeperSpecificFee;

        // Check balance
        if (status.balanceInWei < totalKeeperFee) {
            revert SchedulerErrors.InsufficientBalance();
        }

        status.balanceInWei -= totalKeeperFee;
        status.totalSpent += totalKeeperFee;

        // Pay keeper and update status
        (bool sent, ) = msg.sender.call{value: totalKeeperFee}("");
        if (!sent) {
            revert SchedulerErrors.KeeperPaymentFailed();
        }
    }

pragma solidity ^0.8.0;

import "./SchedulerState.sol";
import "@pythnetwork/pulse-sdk-solidity/SchedulerErrors.sol";

/**
 * @dev `SchedulerGovernance` defines governance capabilities for the Pulse contract.
 */
abstract contract SchedulerGovernance is SchedulerState {
    event NewAdminProposed(address oldAdmin, address newAdmin);
    event NewAdminAccepted(address oldAdmin, address newAdmin);
    event SingleUpdateKeeperFeeSet(uint oldFee, uint newFee);
    event MinimumBalancePerFeedSet(uint oldBalance, uint newBalance);

    /**
     * @dev Returns the address of the proposed admin.
     */
    function proposedAdmin() public view virtual returns (address) {
        return _state.proposedAdmin;
    }

    /**
     * @dev Returns the address of the current admin.
     */
    function getAdmin() external view returns (address) {
        return _state.admin;
    }

    /**
     * @dev Proposes a new admin for the contract. Replaces the proposed admin if there is one.
     * Can only be called by either admin or owner.
     */
    function proposeAdmin(address newAdmin) public virtual {
        require(newAdmin != address(0), "newAdmin is zero address");

        _authorizeAdminAction();

        _state.proposedAdmin = newAdmin;
        emit NewAdminProposed(_state.admin, newAdmin);
    }

    /**
     * @dev The proposed admin accepts the admin transfer.
     */
    function acceptAdmin() external {
        if (msg.sender != _state.proposedAdmin)
            revert SchedulerErrors.Unauthorized();

        address oldAdmin = _state.admin;
        _state.admin = msg.sender;

        _state.proposedAdmin = address(0);
        emit NewAdminAccepted(oldAdmin, msg.sender);
    }

    /**
     * @dev Authorization check for admin actions
     * Must be implemented by the inheriting contract.
     */
    function _authorizeAdminAction() internal virtual;

    /**
     * @dev Set the keeper fee for single updates in Wei.
     * Calls {_authorizeAdminAction}.
     * Emits a {SingleUpdateKeeperFeeSet} event.
     */
    function setSingleUpdateKeeperFeeInWei(uint128 newFee) external {
        _authorizeAdminAction();

        uint oldFee = _state.singleUpdateKeeperFeeInWei;
        _state.singleUpdateKeeperFeeInWei = newFee;

        emit SingleUpdateKeeperFeeSet(oldFee, newFee);
    }

    /**
     * @dev Set the minimum balance required per feed in a subscription.
     * Calls {_authorizeAdminAction}.
     * Emits a {MinimumBalancePerFeedSet} event.
     */
    function setMinimumBalancePerFeed(uint128 newMinimumBalance) external {
        _authorizeAdminAction();

        uint oldBalance = _state.minimumBalancePerFeed;
        _state.minimumBalancePerFeed = newMinimumBalance;

        emit MinimumBalancePerFeedSet(oldBalance, newMinimumBalance);
    }
}
