### Title
Pyth Lazer `verifyUpdate` Does Not Enforce Payload Timestamp Freshness, Enabling Stale Price Submission — (File: `lazer/contracts/evm/src/PythLazer.sol`)

---

### Summary

`PythLazer.sol`'s `verifyUpdate` function verifies only that the ECDSA signature is valid and that the signer key has not expired. It does **not** validate the timestamp embedded in the payload against the current block time. Because there is also no on-chain mechanism to revoke a specific signed payload, any user who captures a Lazer price update can hold it indefinitely and submit it at a strategically chosen moment — exactly the "free option" pattern described in the reference report.

---

### Finding Description

`verifyUpdate` in `PythLazer.sol` performs three checks:

1. EVM magic number
2. ECDSA signature recovery
3. `isValidSigner(signer)` — checks `block.timestamp < trustedSignerToExpiresAtMapping[signer]` [1](#0-0) 

The function returns the raw `payload` bytes to the caller. The payload contains a `timestamp_us` field (parsed by `PythLazerLib.parsePayloadHeader`), but `verifyUpdate` never reads or bounds-checks that field against `block.timestamp`. [2](#0-1) 

The only revocation primitive available is `updateTrustedSigner`, which can set `expiresAt` to 0 to remove a signer entirely. [3](#0-2) 

This is a coarse-grained, all-or-nothing revocation: it invalidates **every** payload ever signed by that key, not a specific stale payload. There is no nonce, sequence number, or per-payload invalidation registry.

The same gap exists in the Sui implementation: `parse_and_verify_le_ecdsa_update_v2` checks only signer expiry, never the payload's `timestamp` field. [4](#0-3) 

---

### Impact Explanation

A user who subscribes to the Lazer WebSocket stream receives a continuous series of signed price payloads. Each payload is cryptographically valid for the entire lifetime of the signer key (which may be months or years). The user can:

1. Capture a payload at time T when the price is favorable (e.g., ETH = $3 000).
2. Wait until the market price moves to $3 100.
3. Submit the stale payload to any consumer contract that calls `verifyUpdate` and does not independently enforce a maximum age.
4. `verifyUpdate` accepts the payload (signer still valid), and the consumer executes logic at the stale $3 000 price.

Consumer contracts that delegate all validation to `verifyUpdate` — a reasonable assumption given that the function is the sole verification entry point — are directly exploitable. The Pyth Lazer integration guide shows consumers calling `verifyUpdate` and then parsing the returned payload, with no mandatory freshness check imposed by the SDK. [5](#0-4) 

---

### Likelihood Explanation

Lazer is marketed as an ultra-low-latency feed for institutional DeFi. Integrators are likely to trust `verifyUpdate` as a complete validity gate and omit a redundant timestamp check in their own contracts. The attack requires only that the attacker hold a valid Lazer subscription (permissionless) and that the target consumer omit or use a loose timestamp threshold. No privileged access, leaked key, or external oracle manipulation is needed.

---

### Recommendation

1. **Enforce a maximum payload age inside `verifyUpdate`**: parse the `timestamp_us` field from the payload header and revert if `block.timestamp * 1e6 - timestamp_us > MAX_PAYLOAD_AGE_US` (e.g., 60 seconds).
2. **Add a per-payload revocation registry**: maintain an on-chain mapping of revoked payload hashes so the Lazer operator can invalidate specific signed messages without rotating the entire signer key — directly analogous to the per-order-hash revocation recommended in the reference report.
3. **Document the freshness requirement explicitly** in the SDK and require consumers to call a `verifyUpdateNoOlderThan(update, maxAgeSeconds)` variant, mirroring the pattern already established in Pyth Core's `getPriceNoOlderThan`.

---

### Proof of Concept

```
1. Attacker subscribes to Lazer WebSocket (permissionless).
2. At block N, attacker captures a signed payload P where ETH price = $3 000.
3. Attacker waits until block N+500 (ETH spot = $3 100).
4. Attacker calls targetConsumer.updatePrice(P):
   a. targetConsumer calls PythLazer.verifyUpdate{value: fee}(P).
   b. verifyUpdate recovers signer, checks isValidSigner → true (key not expired).
   c. verifyUpdate returns payload bytes; no timestamp check performed.
   d. targetConsumer parses price = $3 000 and executes trade at stale price.
5. Attacker profits from the $100 spread with zero market risk.
```

        if (expiresAt == 0) {

                if (trustedSigners[i].pubkey == trustedSigner) {

    function verifyUpdate(
        bytes calldata update
    ) external payable returns (bytes calldata payload, address signer) {
        // Require fee and refund excess
        require(msg.value >= verification_fee, "Insufficient fee provided");
        if (msg.value > verification_fee) {
            payable(msg.sender).transfer(msg.value - verification_fee);
        }

        if (update.length < 71) {
            revert("input too short");
        }
        uint32 EVM_FORMAT_MAGIC = 706910618;

        uint32 evm_magic = uint32(bytes4(update[0:4]));
        if (evm_magic != EVM_FORMAT_MAGIC) {
            revert("invalid evm magic");
        }
        uint16 payload_len = uint16(bytes2(update[69:71]));
        if (update.length < 71 + payload_len) {
            revert("input too short");
        }
        payload = update[71:71 + payload_len];
        bytes32 hash = keccak256(payload);
        (signer, , ) = ECDSA.tryRecover(
            hash,
            uint8(update[68]) + 27,
            bytes32(update[4:36]),
            bytes32(update[36:68])
        );
        if (signer == address(0)) {
            revert("invalid signature");
        }
        if (!isValidSigner(signer)) {
            revert("invalid signer");
        }
    }

public(package) fun verify_le_ecdsa_message(
    state: &State,
    clock: &Clock,
    signature: &vector<u8>,
    payload: &vector<u8>,
) {
    let current_cap = state.current_cap();

    // 0 stands for keccak256 hash
    let pubkey = secp256k1_ecrecover(signature, payload, 0);

    // Check if the recovered pubkey is in the trusted signers list
    let trusted_signers = state.trusted_signers(&current_cap);
    let mut maybe_idx = trusted_signers.find_index!(|signer|
        signer.public_key() == &pubkey
    );

    assert!(maybe_idx.is_some(), ESignerNotTrusted);
    let idx = maybe_idx.extract();
    let expires_at_ms = trusted_signers[idx].expires_at_ms();
    assert!(clock.timestamp_ms() < expires_at_ms, ESignerExpired);

The `verifyUpdate` function will verify the price update and return the payload and the verification fee. This call takes a fee which can be queried from [`verification_fee(){:solidity}`](https://github.com/pyth-network/pyth-crosschain/blob/main/lazer/contracts/evm/src/PythLazer.sol#L9) function and passed to the `verifyUpdate` call. This fee is used to cover the cost of verifying the price update.

This SDK provides [`parsePayloadHeader`](https://github.com/pyth-network/pyth-crosschain/blob/main/lazer/contracts/evm/src/PythLazerLib.sol#L21) method to retrieve the values from the payload header.

```solidity copy
(uint64 _timestamp, Channel channel, uint8 feedsLen, uint16 pos) = parsePayloadHeader(payload);
```

This method returns:

- `_timestamp`: The timestamp of the price update.
- `channel`: The channel of the price update.
- `feedsLen`: The number of feeds in the price update.
- `pos`: The cursor position of the payload.
