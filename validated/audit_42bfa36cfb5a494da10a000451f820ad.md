### Title
Unauthenticated `from` Field in `ConnectionRequestDelivered` Allows Attacker to Drain `inflight_requests` and Trigger NAT Traversal to Attacker-Controlled Addresses — (`network/src/protocols/hole_punching/component/connection_request_delivered.rs`)

---

### Summary

The `ConnectionRequestDeliveredProcess::execute` function determines whether the local node is the intended final recipient of a `ConnectionRequestDelivered` message by comparing `self_peer_id` against the message's `content.from` field. Because `content.from` is parsed directly from attacker-supplied bytes with no cryptographic binding to the actual sender session, any connected peer can forge `from` = local peer ID, causing the node to remove a legitimate entry from `inflight_requests` and invoke `try_nat_traversal` with fully attacker-controlled `listen_addrs`.

---

### Finding Description

**Entry point — `received` in `mod.rs`:**

Any connected peer can send a `HolePunchingMessage::ConnectionRequestDelivered` over the P2P protocol. The message is dispatched to `ConnectionRequestDeliveredProcess::execute` with no prior authentication of the `from` field. [1](#0-0) 

**Parsing — `DeliverdContent::try_from`:**

`content.from` is decoded purely from the raw bytes supplied by the remote peer. No check ties it to the actual session's peer ID. [2](#0-1) 

**Routing decision — the forged `from` bypasses the forwarding branch:**

When `content.route` is empty and `self_peer_id == &content.from`, the code enters the "I am the originator" branch. An attacker sets `from` = local peer ID and `route` = `[]` to reach this branch unconditionally. [3](#0-2) 

**`inflight_requests` drain and `try_nat_traversal` with attacker addresses:**

Once in the branch, `inflight_requests.remove(&content.to)` removes the legitimate entry (keyed by the `to` peer ID the attacker supplies), and `try_nat_traversal` is called with `content.listen_addrs` — entirely attacker-controlled. [4](#0-3) 

**`try_nat_traversal` opens real outbound TCP connections:**

The function spawns an async task that repeatedly attempts `TcpSocket::connect` to each address in `remote_addrs` for up to 30 seconds with 200 ms retry intervals. [5](#0-4) 

**`inflight_requests` is populated by `notify()`:**

The `notify` timer broadcasts `ConnectionRequest` messages via gossip to `sqrt(total)` peers, inserting the `to` peer IDs into `inflight_requests`. Because the gossip fan-out includes arbitrary connected peers, an attacker who is connected can observe the `to` field directly from the broadcast. [6](#0-5) 

**Rate limiter does not block the attack:**

The `forward_rate_limiter` is keyed on `(content.from, content.to, msg_item_id)`. For a single-shot drain the attacker only needs one message; the limiter does not block the first request per key. [7](#0-6) 

---

### Impact Explanation

1. **Outbound TCP connections to attacker-controlled IP:port pairs.** The victim node becomes a TCP port-scan amplifier: the attacker directs the victim to probe arbitrary hosts/ports for up to 30 seconds per address, with up to 24 addresses per message (`ADDRS_COUNT_LIMIT`).
2. **Connection exhaustion.** Repeated messages (varying `to` across known or guessable peer IDs) spawn many concurrent `try_nat_traversal` tasks, each holding a socket for up to 30 s.
3. **Legitimate NAT traversal disruption.** Each attack message removes one real `inflight_requests` entry, preventing the victim from completing its own hole-punching sessions.

---

### Likelihood Explanation

- The local peer ID is publicly advertised in the P2P network; the attacker learns it trivially upon connecting.
- `ConnectionRequest` gossip is broadcast to `sqrt(total)` peers; an attacker who is one of those peers observes the `to` peer IDs directly.
- No cryptographic proof-of-origin is required for the `from` field anywhere in the message processing pipeline.
- A single connected peer (no special privilege) can trigger the full attack path.

---

### Recommendation

Bind the `from` field to the actual sender session: when `route` is empty and the node is about to treat itself as the originator, verify that `content.from` equals the peer ID of the session that delivered the message (i.e., `self.peer`'s registered peer ID), not just that it equals the local node's own ID. Reject the message with a ban-worthy status code if the check fails.

---

### Proof of Concept

```
1. Victim node V has local_peer_id = L.
2. V's notify() fires, inserts peer_id T into inflight_requests.
3. Attacker A (connected to V) observes the broadcast ConnectionRequest with to=T.
4. A sends ConnectionRequestDelivered {
       from: L,          // forged = victim's own peer ID
       to: T,            // known from gossip observation
       route: [],        // empty → triggers "I am the originator" branch
       listen_addrs: [attacker_ip:attacker_port, ...],  // up to 24 entries
       sync_route: [],
   }
5. V processes the message:
   - route.last() == None  ✓
   - self_peer_id (L) == content.from (L)  ✓  → enters else branch
   - inflight_requests.remove(&T) returns Some(start)  ✓
   - try_nat_traversal(ttl, [attacker_ip:attacker_port, ...]) is spawned
6. V opens outbound TCP connections to attacker_ip:attacker_port for 30 s.
7. inflight_requests no longer contains T; legitimate hole-punch to T is aborted.
```

pub(crate) async fn try_nat_traversal(
    bind_addr: Option<SocketAddr>,
    addr: Multiaddr,
) -> Result<(TcpStream, Multiaddr), std::io::Error> {
    let net_addr = match multiaddr_to_socketaddr(&addr) {
        Some(addr) => addr,
        None => {
            debug!("Failed to convert multiaddr to socketaddr");
            return Err(std::io::ErrorKind::InvalidInput.into());
        }
    };

    // Use a fixed interval but add a small amount of randomness
    let base_retry_interval = Duration::from_millis(200);

    // total time
    let timeout_duration = Duration::from_secs(30);
    let start_time = Instant::now();
    let mut retry_count = 0u32;
    while start_time.elapsed() < timeout_duration {
        retry_count += 1;

        // Add a small amount of random jitter (±25ms) to avoid conflicts
        // caused by continuous precise synchronization
        let jitter = Duration::from_millis(rand::random::<u64>() % 50);
        let actual_interval = if rand::random::<bool>() {
            base_retry_interval + jitter
        } else {
            base_retry_interval.saturating_sub(jitter)
        };

        let socket = create_socket(bind_addr, net_addr)?;

        match runtime::timeout(
            std::time::Duration::from_millis(200),
            socket.connect(net_addr),
        )
        .await
        {
            Ok(Ok(stream)) => {
                // try get the stored error in the underlying socket
                // if the socket is not connected, it will return an error
                if let Err(err) = check_connection(&stream) {
                    debug!("Failed to connect to NAT(base check): {}", err);
                }
                return Ok((stream, addr));
            }
            Err(err) => {
                debug!("Failed to connect to NAT(timeout): {}", err);
            }
            Ok(Err(err)) => {
                if err.kind() == std::io::ErrorKind::AddrNotAvailable {
                    return Err(err);
                }
                debug!(
                    "Failed to connect to NAT(other error): {}, {}",
                    err.kind(),
                    err
                );
            }
        }
        runtime::delay_for(actual_interval).await;
    }

    debug!("Failed to connect to NAT after {} retries", retry_count);
    Err(std::io::ErrorKind::TimedOut.into())
}
