### Title
`parseTwapPriceFeedUpdates` Accepts Duplicate `priceIds` Without Validation, Enabling Fee Evasion — (`File: target_chains/ethereum/contracts/contracts/pyth/Pyth.sol`)

---

### Summary

`parseTwapPriceFeedUpdates` in `Pyth.sol` does not validate that the caller-supplied `priceIds` array contains unique entries. Because the fee is computed from `updateData[0]` content (N unique price feeds), a caller can pass M > N duplicate entries in `priceIds` and receive M TWAP results while paying only for N updates.

---

### Finding Description

`parseTwapPriceFeedUpdates` iterates over the caller-supplied `priceIds` array and, for each entry, searches `startPriceIds` (extracted from `updateData`) for a match. There is no "already filled" guard and no duplicate check on the input array. [1](#0-0) 

For each `i`, the inner loop finds the first matching index `j` in `startPriceIds` and unconditionally writes `twapPriceFeeds[i]`:

```solidity
for (uint i = 0; i < priceIds.length; i++) {
    bytes32 requestedPriceId = priceIds[i];
    int startIdx = -1;
    for (uint j = 0; j < startPriceIds.length; j++) {
        if (startPriceIds[j] == requestedPriceId) {
            startIdx = int(j);
            break;
        }
    }
    if (startIdx >= 0) {
        ...
        twapPriceFeeds[i] = calculateTwap(...);
    }
}
```

If `priceIds = [A, A]` and `updateData` contains exactly one entry for `A`:
- `i=0`: finds `A` at `j=0`, fills `twapPriceFeeds[0]`
- `i=1`: finds `A` at `j=0` again (no guard), fills `twapPriceFeeds[1]`

The final completeness check only verifies `twapPriceFeeds[k].id != 0`, which passes for both slots: [2](#0-1) 

The fee is computed by `getTwapUpdateFee`, which counts updates **only in `updateData[0]`**, not in `priceIds.length`: [3](#0-2) 

So a caller with `updateData[0]` containing 1 entry for `A` pays `1 × singleUpdateFeeInWei + transactionFeeInWei`, but can pass `priceIds = [A, A, ..., A]` (M entries) and receive M TWAP results.

This contrasts with `parsePriceFeedUpdatesWithConfig`, which uses an "already filled" guard (`context.priceFeeds[k].id == 0`) that prevents the same slot from being filled twice — but that guard always finds `k=0` for duplicate entries and never fills `k=1`, causing a revert. `parseTwapPriceFeedUpdates` has no equivalent guard. [4](#0-3) 

The `Scheduler` contract avoids this class of issue by enforcing uniqueness on `priceIds` at subscription creation/update time: [5](#0-4) 

But `parseTwapPriceFeedUpdates` is a public entry point with no such guard.

---

### Impact Explanation

1. **Fee evasion**: A caller requesting M duplicates of N unique price feeds pays for N updates but receives M TWAP results. Fee savings = `(M − N) × singleUpdateFeeInWei`. With no upper bound on `priceIds.length`, M can be arbitrarily large.
2. **Downstream contract confusion**: Any contract that calls `parseTwapPriceFeedUpdates` and assumes the returned array contains unique price feeds (e.g., to settle a derivative or compute a portfolio value) will receive duplicate TWAP entries, potentially causing incorrect financial calculations.

---

### Likelihood Explanation

`parseTwapPriceFeedUpdates` is `external payable` with no access control. Any unprivileged transaction sender can call it with a crafted `priceIds` array. No special role, leaked key, or governance majority is required. The attacker only needs valid `updateData` for one price feed (obtainable from Hermes) and a `priceIds` array with repeated entries.

---

### Recommendation

Add a duplicate check on the caller-supplied `priceIds` array at the start of `parseTwapPriceFeedUpdates`, analogous to the check already present in `Scheduler._validateSubscriptionParams`:

```solidity
for (uint i = 0; i < priceIds.length; i++) {
    for (uint j = i + 1; j < priceIds.length; j++) {
        if (priceIds[i] == priceIds[j]) revert PythErrors.InvalidArgument();
    }
}
```

Alternatively, base the fee on `priceIds.length` rather than `updateData[0]` content, or add a "slot already filled" guard analogous to the one in `_parseSingleMerkleUpdate`.

---

### Proof of Concept

```solidity
// updateData[0] contains exactly 1 TWAP entry for priceId A
// updateData[1] contains exactly 1 TWAP entry for priceId A

bytes32[] memory priceIds = new bytes32[](3);
priceIds[0] = A;
priceIds[1] = A;  // duplicate
priceIds[2] = A;  // duplicate

// Fee paid: getTwapUpdateFee counts only updateData[0] → fee for 1 update
uint fee = pyth.getTwapUpdateFee(updateData);

// Returns 3 TWAP feeds for A, all identical, while paying for 1
PythStructs.TwapPriceFeed[] memory result =
    pyth.parseTwapPriceFeedUpdates{value: fee}(updateData, priceIds);

assert(result.length == 3);
assert(result[0].id == A);
assert(result[1].id == A);  // duplicate returned without extra fee
assert(result[2].id == A);  // duplicate returned without extra fee
```
