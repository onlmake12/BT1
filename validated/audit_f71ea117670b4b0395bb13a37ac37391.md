### Title
Uncapped Provider Fee Parameters in `Echo.sol` Cause `getFee()` to Revert via `SafeCast` Overflow, DOSing `requestPriceUpdatesWithCallback` — (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

Any registered Echo provider can set `feePerGasInWei` (or `feePerFeedInWei`) to a value that causes `SafeCast.toUint96()` to revert inside `getFee()`, making `requestPriceUpdatesWithCallback` permanently revert for all users of that provider. There is no cap enforced in `registerProvider()` or `setProviderFee()`.

---

### Finding Description

In `Echo.sol`, `getFee()` computes the total fee as:

```solidity
uint96 providerFeedFee = SafeCast.toUint96(
    priceIds.length * _state.providers[provider].feePerFeedInWei
);
uint256 gasFee = callbackGasLimit * providerFeeInWei;
feeAmount = baseFee + providerBaseFee + providerFeedFee + SafeCast.toUint96(gasFee);
```

`callbackGasLimit` is `uint32` (max ~4.29 × 10⁹) and `providerFeeInWei` is `uint96` (max ~7.92 × 10²⁸). Their product is computed in `uint256` space, but then cast back to `uint96` via `SafeCast.toUint96`. If `callbackGasLimit * feePerGasInWei > type(uint96).max`, `SafeCast.toUint96` reverts unconditionally.

Neither `registerProvider()` nor `setProviderFee()` impose any upper bound on the fee parameters:

```solidity
function registerProvider(
    uint96 baseFeeInWei,
    uint96 feePerFeedInWei,
    uint96 feePerGasInWei
) external override {
    ...
    provider.feePerGasInWei = feePerGasInWei;  // no cap
    ...
}

function setProviderFee(
    address provider,
    uint96 newBaseFeeInWei,
    uint96 newFeePerFeedInWei,
    uint96 newFeePerGasInWei
) external override {
    ...
    _state.providers[provider].feePerGasInWei = newFeePerGasInWei;  // no cap
    ...
}
```

Since `requestPriceUpdatesWithCallback` calls `getFee()` internally:

```solidity
uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
if (msg.value < requiredFee) revert InsufficientFee();
```

a revert in `getFee()` propagates and permanently blocks all requests to that provider.

The same overflow path exists for `feePerFeedInWei`: `SafeCast.toUint96(priceIds.length * feePerFeedInWei)` reverts if `priceIds.length * feePerFeedInWei > type(uint96).max`. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

---

### Impact Explanation

Any registered provider (registration is permissionless — anyone can call `registerProvider`) can set `feePerGasInWei = type(uint96).max`. For any `callbackGasLimit >= 2`, `getFee()` reverts, making `requestPriceUpdatesWithCallback` permanently revert for all users of that provider. If the affected provider is the default provider, the entire Echo service is effectively DOSed for all users who rely on the default. Even for non-default providers, users who have integrated with a specific provider address are fully blocked with no recourse on-chain. [5](#0-4) 

---

### Likelihood Explanation

Registration is permissionless — any address can call `registerProvider` and immediately set overflow fee values. An existing provider who turns malicious can call `setProviderFee` at any time to trigger the overflow. No admin action is required to register or update fees. The only privileged step needed to maximize impact (making the default provider malicious) requires admin cooperation, but the attack is still impactful against any provider's user base without admin involvement. [6](#0-5) 

---

### Recommendation

Add explicit upper bounds in both `registerProvider()` and `setProviderFee()`:

```solidity
uint96 constant MAX_FEE_PER_GAS = 1e9; // e.g., 1 gwei per gas unit
uint96 constant MAX_FEE_PER_FEED = 1e15; // reasonable upper bound
uint96 constant MAX_BASE_FEE = 1e18;

require(feePerGasInWei <= MAX_FEE_PER_GAS, "feePerGasInWei exceeds cap");
require(feePerFeedInWei <= MAX_FEE_PER_FEED, "feePerFeedInWei exceeds cap");
require(baseFeeInWei <= MAX_BASE_FEE, "baseFeeInWei exceeds cap");
```

Alternatively, perform the multiplication in `getFee()` with explicit overflow checks before the `SafeCast`, and return a sentinel value or revert with a descriptive error rather than a raw `SafeCast` panic. [7](#0-6) 

---

### Proof of Concept

1. Attacker calls `echo.registerProvider(0, 0, type(uint96).max)` — succeeds with no validation.
2. A user (or the admin sets attacker as default provider) calls `echo.getFee(attacker, 2, priceIds)`:
   - `gasFee = 2 * type(uint96).max = 2 * (2^96 - 1)` which is `> type(uint96).max`
   - `SafeCast.toUint96(gasFee)` reverts
3. User calls `echo.requestPriceUpdatesWithCallback{value: X}(attacker, publishTime, priceIds, 2)` → reverts at line 75 because `getFee()` reverts.
4. No amount of ETH sent as `msg.value` can bypass this — the revert occurs before the fee comparison.

Alternatively, attacker calls `echo.setProviderFee(attacker, 0, type(uint96).max, 0)` after registration to set `feePerFeedInWei = type(uint96).max`. Then `getFee()` with `priceIds.length >= 2` reverts via `SafeCast.toUint96(priceIds.length * type(uint96).max)`. [8](#0-7) [9](#0-8)

    function getFee(
        address provider,
        uint32 callbackGasLimit,
        bytes32[] calldata priceIds
    ) public view override returns (uint96 feeAmount) {
        uint96 baseFee = _state.pythFeeInWei; // Fixed fee to Pyth
        // Note: The provider needs to set its fees to include the fee charged by the Pyth contract.
        // Ideally, we would be able to automatically compute the pyth fees from the priceIds, but the
        // fee computation on IPyth assumes it has the full updated data.
        uint96 providerBaseFee = _state.providers[provider].baseFeeInWei;
        uint96 providerFeedFee = SafeCast.toUint96(
            priceIds.length * _state.providers[provider].feePerFeedInWei
        );
        uint96 providerFeeInWei = _state.providers[provider].feePerGasInWei; // Provider's per-gas rate
        uint256 gasFee = callbackGasLimit * providerFeeInWei; // Total provider fee based on gas
        feeAmount =
            baseFee +
            providerBaseFee +
            providerFeedFee +
            SafeCast.toUint96(gasFee); // Total fee user needs to pay
    }
