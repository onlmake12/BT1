### Title
Network Alert Signature Lacks Chain/Network Domain Binding, Enabling Cross-Network Alert Replay — (`util/network-alert/src/verifier.rs`, `util/gen-types/schemas/extensions.mol`)

---

### Summary

The CKB Network Alert system signs `RawAlert` messages using Nervos Foundation keys. The signed hash (`calc_alert_hash`) covers only the alert payload fields and contains no network or chain identifier. An unprivileged peer connected to both mainnet and testnet can capture a valid alert from one network and relay it verbatim to nodes on the other network, where it passes signature verification and is accepted and re-broadcast as legitimate.

---

### Finding Description

The `RawAlert` molecule type is defined with no network or chain identifier field:

```
table RawAlert {
    notice_until:   Uint64,
    id:             Uint32,
    cancel:         Uint32,
    priority:       Uint32,
    message:        Bytes,
    min_version:    BytesOpt,
    max_version:    BytesOpt,
}
``` [1](#0-0) 

`calc_alert_hash()` is implemented as a plain `blake2b_256` over the raw molecule bytes of `RawAlert` — no domain tag, no chain ID, no genesis hash:

```rust
impl<'r> packed::RawAlertReader<'r> {
    pub fn calc_alert_hash(&self) -> packed::Byte32 {
        self.calc_hash()   // = blake2b_256(self.as_slice())
    }
}
``` [2](#0-1) 

`Verifier::verify_signatures()` derives the message to verify from this hash alone and checks it against the configured Foundation public keys, with no additional context check:

```rust
pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
    let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
    ...
    verify_m_of_n(&message, self.config.signatures_threshold, &signatures, &self.pubkeys)
``` [3](#0-2) 

`AlertRelayer::received()` is the P2P entry point. Any peer can submit an `Alert` message; the only gate is `verify_signatures`:

```rust
if let Err(err) = self.verifier.verify_signatures(&alert) {
    ...
    nc.ban_peer(...);
    return;
}
// mark sender as known
self.mark_as_known(peer_index, alert_id);
// broadcast message
...
self.notifier.lock().add(&alert);
``` [4](#0-3) 

Because the signed hash is identical for the same `RawAlert` bytes regardless of which network processes it, a signature valid on mainnet is equally valid on testnet and vice versa — an exact analog of the `targetTracer != market` omission in the external report.

---

### Impact Explanation

An unprivileged attacker who is peered with both mainnet and testnet (trivially achievable by running two CKB nodes) can:

1. Capture a legitimately signed alert broadcast on mainnet.
2. Connect to testnet peers and relay the raw alert bytes.
3. Testnet nodes call `verify_signatures`, which passes because the Foundation keys and the hash are identical.
4. The alert is accepted, stored, and re-broadcast to all testnet peers, and surfaced to testnet users via the notify subsystem.

The reverse direction (testnet alert → mainnet) is equally possible. Consequences include:
- Testnet users receiving mainnet-specific upgrade warnings, causing unnecessary node churn.
- Mainnet users receiving testnet-only alerts, causing confusion or unwarranted alarm.
- If the Foundation ever issues a network-specific emergency alert (e.g., "halt mining on mainnet"), an attacker can replay it on testnet, disrupting testnet operations.

---

### Likelihood Explanation

- **No privileged access required.** Any node operator can peer with both mainnet and testnet simultaneously.
- **No key material required.** The attacker only relays an already-signed alert; they never need the Foundation's private keys.
- **Alerts are broadcast to all peers.** Any connected peer receives the full signed alert bytes, making capture trivial.
- **Mainnet and testnet share the same Foundation alert-signing keys** (the `NetworkAlertConfig::default()` used in production embeds the same Foundation public keys for both networks, as confirmed by `test_alert_20230001` using `NetworkAlertConfig::default()` with hardcoded mainnet signatures). [5](#0-4) 

---

### Recommendation

Include a network domain separator in the signed message. The simplest fix is to mix the genesis block hash (already available as `Consensus::genesis_hash`) into the alert hash before signing and verifying:

```rust
// In Verifier::verify_signatures:
let mut hasher = new_blake2b();
hasher.update(alert.calc_alert_hash().as_slice());
hasher.update(self.genesis_hash.as_slice()); // add per-network binding
let mut message_bytes = [0u8; 32];
hasher.finalize(&mut message_bytes);
let message = Message::from(message_bytes);
```

Alternatively, add a `network_id: Bytes` field to `RawAlert` and include it in the signed payload. Either approach ensures a mainnet-signed alert hash is cryptographically distinct from a testnet-signed alert hash, eliminating cross-network replay.

---

### Proof of Concept

1. Run a CKB mainnet node and a CKB testnet node on the same machine.
2. Connect the mainnet node to the mainnet P2P network; wait for a Foundation-signed alert to be broadcast (or use the known alert `20230001` with its published signatures).
3. Capture the raw alert bytes (the `Alert` molecule encoding) from the mainnet peer message.
4. Connect to a testnet peer and send the identical raw bytes as an `Alert` P2P message.
5. Observe that the testnet node's `AlertRelayer::received()` calls `verify_signatures`, which returns `Ok(())` because the hash and signatures are identical.
6. The testnet node stores the alert and re-broadcasts it to all its testnet peers; testnet users see the mainnet alert.

The known mainnet alert `20230001` with its two published signatures (visible in `generate_alert_signature.rs`) can be used directly as the replay payload without any key material. [6](#0-5) [7](#0-6)

    pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
        trace!("Verifying alert {:?}", alert);
        let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
        let signatures: Vec<Signature> = alert
            .signatures()
            .into_iter()
            .filter_map(
                |sig_data| match Signature::from_slice(sig_data.as_reader().raw_data()) {
                    Ok(sig) => {
                        if sig.is_valid() {
                            Some(sig)
                        } else {
                            debug!("invalid signature: {:?}", sig);
                            None
                        }
                    }
                    Err(err) => {
                        debug!("signature error: {}", err);
                        None
                    }
                },
            )
            .collect();
        verify_m_of_n(
            &message,
            self.config.signatures_threshold,
            &signatures,
            &self.pubkeys,
        )
        .map_err(|err| err.kind())?;
        Ok(())
    }

    async fn received(
        &mut self,
        nc: Arc<dyn CKBProtocolContext + Sync>,
        peer_index: PeerIndex,
        data: Bytes,
    ) {
        let alert: packed::Alert = match packed::AlertReader::from_slice(&data) {
            Ok(alert) => {
                if alert.raw().message().is_utf8()
                    && alert
                        .raw()
                        .min_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                    && alert
                        .raw()
                        .max_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                {
                    alert.to_entity()
                } else {
                    info!(
                        "A malformed message fromP peer {} : not utf-8 string",
                        peer_index
                    );
                    nc.ban_peer(
                        peer_index,
                        BAD_MESSAGE_BAN_TIME,
                        String::from("send us a malformed message: not utf-8 string"),
                    );
                    return;
                }
            }
            Err(err) => {
                info!("A malformed message from peer {}: {:?}", peer_index, err);
                nc.ban_peer(
                    peer_index,
                    BAD_MESSAGE_BAN_TIME,
                    String::from("send us a malformed message"),
                );
                return;
            }
        };
        let alert_id = alert.as_reader().raw().id().into();
        trace!("ReceiveD alert {} from peer {}", alert_id, peer_index);
        // ignore alert
        if self.notifier.lock().has_received(alert_id) {
            return;
        }
        // verify
        if let Err(err) = self.verifier.verify_signatures(&alert) {
            debug!(
                "An alert from peer {} with invalid signatures, error {:?}",
                peer_index, err
            );
            nc.ban_peer(
                peer_index,
                BAD_MESSAGE_BAN_TIME,
                String::from("send us an alert with invalid signatures"),
            );
            return;
        }
        // mark sender as known
        self.mark_as_known(peer_index, alert_id);
        // broadcast message
        let selected_peers: Vec<PeerIndex> = nc
            .connected_peers()
            .into_iter()
            .filter(|peer| self.mark_as_known(*peer, alert_id))
            .collect();
        if let Err(err) = nc.quick_filter_broadcast(
            TargetSession::Multi(Box::new(selected_peers.into_iter())),
            data,
        ) {
            debug!("alert broadcast error: {:?}", err);
        }
        // add to received alerts
        self.notifier.lock().add(&alert);
    }
