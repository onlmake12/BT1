### Title
`HeaderDigest::verify()` Skips Total-Difficulty Validation for Multi-Epoch Ranges, Allowing Falsified Difficulty in Light-Client Proofs - (File: `util/types/src/utilities/merkle_mountain_range.rs`)

---

### Summary

`HeaderDigest::verify()` validates `total_difficulty` only when a digest covers blocks within a **single epoch**. When the digest spans **multiple epochs**, the function performs no difficulty validation at all. A malicious light-client server peer can craft a `HeaderDigest` that crosses epoch boundaries with an arbitrary `total_difficulty` value and have it pass `verify()` without rejection.

---

### Finding Description

`HeaderDigest::verify()` in `util/types/src/utilities/merkle_mountain_range.rs` contains three checks:

1. Block-number ordering (`start_number ≤ end_number`)
2. Epoch ordering (`start_epoch ≤ end_epoch`)
3. Total-difficulty consistency — **only when `start_epoch_number == end_epoch_number`**

```rust
// 3. Check difficulties when in the same epoch.
if start_epoch_number == end_epoch_number {
    if start_compact_target != end_compact_target {
        // error: targets must match within one epoch
    } else {
        let blocks_count = end_number - start_number + 1;
        let block_difficulty = compact_to_difficulty(start_compact_target);
        let total_difficulty_calculated = block_difficulty * blocks_count;
        if total_difficulty != total_difficulty_calculated {
            // error: wrong total difficulty
        }
    }
}
// ← no else branch: cross-epoch digests skip all difficulty validation
``` [1](#0-0) 

When `start_epoch_number != end_epoch_number` the function falls through to `Ok(())` unconditionally. There is no lower-bound check, no per-epoch segment summation, and no cross-epoch difficulty accounting. Any `total_difficulty` value — zero, `U256::MAX`, or anything in between — passes verification for a multi-epoch digest.

The analog to the reported bug is exact:

| Original (`getPriceForQuantity`) | CKB (`HeaderDigest::verify`) |
|---|---|
| Calculates price over a token-id range | Validates difficulty over a block range |
| Only handles single-tier and two-tier cases | Only handles single-epoch (single-tier) case |
| Ignores intermediate price tiers | Ignores all difficulty validation for multi-epoch ranges |
| Result: wrong price charged | Result: arbitrary `total_difficulty` accepted |

---

### Impact Explanation

`HeaderDigest` is the core data structure of CKB's light-client MMR protocol (RFC 0044). It is transmitted by full-node peers to light clients inside `SendLastStateProof`, `SendBlocksProof`, and `SendTransactionsProof` messages. [2](#0-1) 

A malicious light-client server peer can:

1. Construct a `HeaderDigest` whose `start_epoch` and `end_epoch` differ (spanning an epoch boundary).
2. Set `total_difficulty` to an arbitrarily inflated value.
3. The digest passes `verify()` because the cross-epoch branch is absent.

The light client uses `total_difficulty` from the MMR root to determine which chain has the most accumulated proof-of-work. A falsified `total_difficulty` in an intermediate MMR node that is accepted without error allows the attacker to misrepresent the chain's cumulative work. This can be used to:

- Trick a light client into accepting a weaker chain as the canonical chain (false chain-tip attack).
- Cause the light client to reject the real canonical chain as having insufficient difficulty.

**Impact: High** — chain-selection integrity for light clients is directly undermined.

---

### Likelihood Explanation

**Likelihood: Medium.** Any peer that a light client connects to and designates as a light-client server can send crafted `HeaderDigest` items. No privileged access, key material, or majority hash power is required. The attack is triggered whenever the light client processes a proof that contains a `HeaderDigest` spanning an epoch boundary — a routine occurrence on mainnet where epochs change approximately every 4 hours. [3](#0-2) 

---

### Recommendation

Add a cross-epoch difficulty validation branch. For a digest spanning multiple epochs, the minimum verifiable lower bound is:

```
total_difficulty ≥ compact_to_difficulty(max(start_compact_target, end_compact_target))
                   * (end_number - start_number + 1)
```

A tighter check would iterate over each epoch segment between `start_epoch` and `end_epoch`, sum the per-segment difficulties using the known compact target for each epoch, and compare against `total_difficulty`. At minimum, add an `else` branch that enforces a non-zero lower bound and rejects values that are provably impossible given the block count and the minimum difficulty of either endpoint epoch.

use ckb_types::{
    packed,
    prelude::*,
    utilities::merkle_mountain_range::HeaderDigest,
    core::EpochNumberWithFraction,

// Craft a HeaderDigest spanning epoch 0 → epoch 1 with total_difficulty = U256::MAX
let start_epoch = EpochNumberWithFraction::new(0, 999, 1000); // last block of epoch 0
let end_epoch   = EpochNumberWithFraction::new(1,   0, 1000); // first block of epoch 1

let digest = packed::HeaderDigest::new_builder()
    .start_number(999u64)
    .end_number(1000u64)
    .start_epoch(start_epoch.full_value())
    .end_epoch(end_epoch.full_value())
    .start_compact_target(0x1e08_3126u32)  // real mainnet target
    .end_compact_target(0x1e07_a4c8u32)    // different target for epoch 1
    .total_difficulty(U256::MAX)            // completely fabricated
    .build();

// This returns Ok(()) — the fabricated total_difficulty is never checked
assert!(digest.verify().is_ok());
``` [4](#0-3) 

The single-epoch path (lines 99–118) is guarded by `if start_epoch_number == end_epoch_number`. When `start_epoch_number` is 0 and `end_epoch_number` is 1, the condition is false, the entire difficulty check is skipped, and `Ok(())` is returned regardless of the `total_difficulty` value. [5](#0-4)

impl HeaderDigest for packed::HeaderDigest {
    /// Verify the MMR header digest
    fn verify(&self) -> Result<(), String> {
        // 1. Check block numbers.
        let start_number: BlockNumber = self.start_number().into();
        let end_number: BlockNumber = self.end_number().into();
        if start_number > end_number {
            let errmsg = format!(
                "failed since the start block number is bigger than the end ([{start_number},{end_number}])"
            );
            return Err(errmsg);
        }

        // 2. Check epochs.
        let start_epoch: EpochNumberWithFraction = self.start_epoch().into();
        let end_epoch: EpochNumberWithFraction = self.end_epoch().into();
        let start_epoch_number = start_epoch.number();
        let end_epoch_number = end_epoch.number();
        if start_epoch != end_epoch
            && ((start_epoch_number > end_epoch_number)
                || (start_epoch_number == end_epoch_number
                    && start_epoch.index() > end_epoch.index()))
        {
            let errmsg = format!(
                "failed since the start epoch is bigger than the end ([{start_epoch:#},{end_epoch:#}])"
            );
            return Err(errmsg);
        }

        // 3. Check difficulties when in the same epoch.
        let start_compact_target: u32 = self.start_compact_target().into();
        let end_compact_target: u32 = self.end_compact_target().into();
        let total_difficulty: U256 = self.total_difficulty().into();
        if start_epoch_number == end_epoch_number {
            if start_compact_target != end_compact_target {
                // In the same epoch, all compact targets should be same.
                let errmsg = format!(
                    "failed since the compact targets should be same during epochs ([{start_epoch:#},{end_epoch:#}])"
                );
                return Err(errmsg);
            } else {
                // Sum all blocks difficulties to check total difficulty.
                let blocks_count = end_number - start_number + 1;
                let block_difficulty = compact_to_difficulty(start_compact_target);
                let total_difficulty_calculated = block_difficulty * blocks_count;
                if total_difficulty != total_difficulty_calculated {
                    let errmsg = format!(
                        "failed since total difficulty is {total_difficulty} but the calculated is {total_difficulty_calculated} \
                        during epochs ([{start_epoch:#},{end_epoch:#}])"
                    );
                    return Err(errmsg);
                }
            }
        }

        Ok(())
    }
}
