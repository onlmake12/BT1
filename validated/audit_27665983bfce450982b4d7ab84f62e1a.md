### Title
Stale and Negative Price Returned Without Validation in `PythAggregatorV3::latestRoundData()` and Related Functions — (File: `target_chains/ethereum/sdk/solidity/PythAggregatorV3.sol`)

---

### Summary

`PythAggregatorV3` is Pyth's official Chainlink-compatible adapter contract, explicitly recommended for migrating Chainlink-based protocols to Pyth. Every price-reading function in this contract calls `pyth.getPriceUnsafe()`, which by design returns a price from arbitrarily far in the past with no staleness check. The contract also performs no negative-price validation before returning `int256(price.price)`. Any downstream protocol that consumes `latestRoundData()`, `latestAnswer()`, `getRoundData()`, or `getAnswer()` expecting Chainlink-style freshness guarantees will silently receive a stale or negative price.

---

### Finding Description

`PythAggregatorV3` is the official Pyth SDK contract for Chainlink-compatible integrations. It is published in the `@pythnetwork/pyth-sdk-solidity` npm package and is the primary migration path documented for Chainlink users.

Every price-reading function in the contract calls `pyth.getPriceUnsafe(priceId)`:

```solidity
// latestAnswer — no staleness check, no negativity check
function latestAnswer() public view virtual returns (int256) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    return int256(price.price);
}

// latestRoundData — no staleness check, no negativity check
function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    roundId = uint80(price.publishTime);
    return (roundId, int256(price.price), price.publishTime, price.publishTime, roundId);
}

// getRoundData — same issue
function getRoundData(uint80 _roundId) external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    return (_roundId, int256(price.price), price.publishTime, price.publishTime, _roundId);
}
```

`getPriceUnsafe` is explicitly documented as returning a price "from arbitrarily far in the past." The Pyth SDK provides `getPriceNoOlderThan(id, age)` specifically to guard against this, but `PythAggregatorV3` never calls it.

Additionally, `price.price` is of type `int64` in `PythStructs.Price`. It can be negative (Pyth's protocol allows negative prices for certain instruments). The contract casts it directly to `int256` without any `<= 0` guard, so a negative price is silently returned as a valid answer.

By contrast, Pyth's own `PythPriceOracleGetter` (the Aave adapter) correctly uses `getPriceNoOlderThan` and explicitly checks `price.price <= 0`.

---

### Impact Explanation

Any protocol that deploys `PythAggregatorV3` as a drop-in Chainlink replacement and calls `latestRoundData()` or `latestAnswer()` without independently checking `updatedAt` will:

1. **Receive a stale price** during any Pyth network outage, market-hours gap (e.g., equity feeds), or if the on-chain price simply hasn't been pushed recently. The `updatedAt` field returned equals `price.publishTime`, which can be hours or days old — but the contract never reverts.
2. **Receive a negative price** for instruments where Pyth publishes a negative `int64` price. The cast `int256(price.price)` silently propagates this.

Downstream protocols (lending, derivatives, vaults) that trust the Chainlink interface contract to enforce freshness — as they would with a real Chainlink feed — will compute incorrect collateral values, liquidation thresholds, or exchange rates, enabling profitable exploits by a bad actor who can time transactions around stale prices.

**Impact: High** — incorrect price used in financial calculations, potential for fund loss.

---

### Likelihood Explanation

**Likelihood: Medium.** The contract is the official, documented migration path for Chainlink users. The Pyth documentation page "From Chainlink" explicitly instructs users to deploy `PythAggregatorV3` and pass its address to their existing Chainlink-compatible app. Chainlink-native protocols universally assume that `latestRoundData()` returns a fresh price (or reverts), so they do not add a secondary staleness check. Pyth is a pull oracle, meaning prices are only updated when someone calls `updateFeeds()`. If no keeper is running, the on-chain price can become arbitrarily stale while `latestRoundData()` continues to return it without error. Market-hours gaps (equities, FX) make this a near-certain occurrence for those asset classes.

---

### Recommendation

Replace all `getPriceUnsafe` calls in `PythAggregatorV3` with `getPriceNoOlderThan` using a configurable `maxAge` parameter set at construction time, mirroring the pattern used in `PythPriceOracleGetter`. Additionally, add a `price.price <= 0` guard before returning the answer:

```diff
+    uint public maxAge;

-    constructor(address _pyth, bytes32 _priceId) {
+    constructor(address _pyth, bytes32 _priceId, uint _maxAge) {
         priceId = _priceId;
         pyth = IPyth(_pyth);
+        maxAge = _maxAge;
     }

     function latestAnswer() public view virtual returns (int256) {
-        PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
+        PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
+        require(price.price > 0, "negative price");
         return int256(price.price);
     }

     function latestRoundData() external view returns (...) {
-        PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
+        PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
+        require(price.price > 0, "negative price");
         ...
     }
```

### Proof of Concept

1. Deploy `PythAggregatorV3` for an equity price feed (e.g., AAPL/USD).
2. Wait until market close — Pyth stops publishing updates. The on-chain price becomes stale (hours old).
3. Call `latestRoundData()`. It returns the stale price with no revert. `updatedAt` is hours in the past, but the caller (a Chainlink-native lending protocol) does not check it.
4. An attacker uses the stale (inflated or deflated) price to over-borrow or avoid liquidation, extracting value from the protocol.

For the negative-price vector: submit a Pyth price update with `price = -1` for a feed. Call `latestAnswer()` — it returns `-1` cast to `int256`, which downstream protocols may interpret as a near-zero or wrapped-around price, enabling further exploits.
