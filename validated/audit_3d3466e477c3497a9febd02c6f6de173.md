### Title
`_validateSubscriptionParams` Does Not Check Cross-Parameter Constraint Between `whitelistEnabled` and `readerWhitelist` ŌĆö (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`_validateSubscriptionParams` in `Scheduler.sol` validates individual subscription parameters but omits a cross-parameter check: it never verifies that when `whitelistEnabled = true`, the `readerWhitelist` array is non-empty. A subscriber who sets `whitelistEnabled = true` with an empty `readerWhitelist` creates a subscription where no address except the manager can ever read prices. For permanent subscriptions this state is irreversible, and deposited funds (up to 100 ETH) cannot be withdrawn.

---

### Finding Description

`_validateSubscriptionParams` performs the following checks on `SubscriptionParams`:

- `priceIds.length > 0`
- `priceIds.length <= MAX_PRICE_IDS_PER_SUBSCRIPTION`
- No duplicate price IDs
- `readerWhitelist.length <= MAX_READER_WHITELIST_SIZE`
- No duplicate whitelist addresses
- At least one update criterion enabled
- If `updateOnHeartbeat`, then `heartbeatSeconds > 0`
- If `updateOnDeviation`, then `deviationThresholdBps > 0` [1](#0-0) 

It does **not** check: `if (params.whitelistEnabled && params.readerWhitelist.length == 0)`.

The `onlyWhitelistedReader` modifier enforces access as follows:

1. Manager is always allowed.
2. If `whitelistEnabled == false`, anyone is allowed.
3. If `whitelistEnabled == true`, the caller must appear in `readerWhitelist`. [2](#0-1) 

When `whitelistEnabled = true` and `readerWhitelist = []`, step 3 iterates over an empty array, `isWhitelisted` stays `false`, and every non-manager caller receives `Unauthorized`. The subscription is accepted and funded, but its price data is permanently inaccessible to any intended reader.

The `SubscriptionParams` struct confirms both fields are independently settable with no enforced relationship: [3](#0-2) 

For **permanent subscriptions** (`isPermanent = true`), `updateSubscription` reverts unconditionally: [4](#0-3) 

And `withdrawFunds` also reverts for permanent subscriptions: [5](#0-4) 

The maximum deposit for a permanent subscription is `MAX_DEPOSIT_LIMIT = 100 ether`: [6](#0-5) 

---

### Impact Explanation

**Non-permanent subscription:** The subscription is created with broken access control. No non-manager address can read prices. The creator must call `updateSubscription` to fix it, wasting gas and potentially missing price update windows.

**Permanent subscription (worst case):**
- No non-manager address can ever read prices from the subscription ŌĆö the intended readers (e.g., a DeFi protocol's smart contracts) are permanently locked out.
- The subscription cannot be updated to add readers.
- Deposited funds (up to 100 ETH) cannot be withdrawn.
- Keeper fees continue draining the balance for updates that no intended reader can consume.

This is a direct financial loss and functional denial-of-service for the subscription creator.

---

### Likelihood Explanation

Any unprivileged user can call `createSubscription`. A developer building a DeFi integration who wants to restrict price feed access to their own contracts would naturally set `whitelistEnabled = true` but could easily forget to populate `readerWhitelist` ŌĆö especially when creating a permanent subscription intended for long-term use. The validation function gives no indication that this combination is invalid, so the transaction succeeds silently. The misconfiguration is only discovered when readers attempt to access prices and receive `Unauthorized`.

---

### Recommendation

Add a cross-parameter check in `_validateSubscriptionParams`:

```solidity
if (params.whitelistEnabled && params.readerWhitelist.length == 0) {
    revert SchedulerErrors.EmptyWhitelistWithWhitelistEnabled();
}
``` [1](#0-0) 

---

### Proof of Concept

```solidity
// Attacker/user creates a permanent subscription with whitelistEnabled=true but empty whitelist
bytes32[] memory priceIds = new bytes32[](1);
priceIds[0] = someValidPriceId;

SchedulerStructs.SubscriptionParams memory params = SchedulerStructs.SubscriptionParams({
    priceIds: priceIds,
    readerWhitelist: new address[](0),  // empty!
    whitelistEnabled: true,             // but whitelist is enabled
    isActive: true,
    isPermanent: true,                  // cannot be updated later
    updateCriteria: SchedulerStructs.UpdateCriteria({
        updateOnHeartbeat: true,
        heartbeatSeconds: 60,
        updateOnDeviation: false,
        deviationThresholdBps: 0
    })
});

// createSubscription succeeds ŌĆö no validation error
uint256 subId = scheduler.createSubscription{value: 10 ether}(params);

// Later, any non-manager reader tries to access prices:
// Reverts with Unauthorized ŌĆö no one can ever be added to the whitelist
// Funds cannot be withdrawn (permanent subscription)
scheduler.getPricesUnsafe(subId, priceIds); // REVERTS: Unauthorized

        SchedulerStructs.SubscriptionParams memory subscriptionParams
    ) external payable override returns (uint256 subscriptionId) {
        _validateSubscriptionParams(subscriptionParams);

        // Calculate minimum balance required for this subscription
        uint256 minimumBalance = this.getMinimumBalance(
            uint8(subscriptionParams.priceIds.length)
        );

        // Ensure enough funds were provided
        if (msg.value < minimumBalance) {
            revert SchedulerErrors.InsufficientBalance();
        }

        // Check deposit limit for permanent subscriptions
        if (subscriptionParams.isPermanent && msg.value > MAX_DEPOSIT_LIMIT) {
            revert SchedulerErrors.MaxDepositLimitExceeded();
        }

        // Set subscription to active
        subscriptionParams.isActive = true;

        subscriptionId = _state.subscriptionNumber++;

        // Store the subscription parameters
        _state.subscriptionParams[subscriptionId] = subscriptionParams;

        // Initialize subscription status
        SchedulerStructs.SubscriptionStatus storage status = _state
            .subscriptionStatuses[subscriptionId];
        status.priceLastUpdatedAt = 0;
        status.balanceInWei = msg.value;
        status.totalUpdates = 0;
        status.totalSpent = 0;

        // Map subscription ID to manager
        _state.subscriptionManager[subscriptionId] = msg.sender;

        _addToActiveSubscriptions(subscriptionId);

        emit SubscriptionCreated(subscriptionId, msg.sender);
        return subscriptionId;
    }

    function _validateSubscriptionParams(
        SchedulerStructs.SubscriptionParams memory params
    ) internal pure {
        // No zeroŌĆÉfeed subscriptions
        if (params.priceIds.length == 0) {
            revert SchedulerErrors.EmptyPriceIds();
        }

        // Price ID limits and uniqueness
        if (
            params.priceIds.length >
            SchedulerConstants.MAX_PRICE_IDS_PER_SUBSCRIPTION
        ) {
            revert SchedulerErrors.TooManyPriceIds(
                params.priceIds.length,
                SchedulerConstants.MAX_PRICE_IDS_PER_SUBSCRIPTION
            );
        }
        for (uint i = 0; i < params.priceIds.length; i++) {
            for (uint j = i + 1; j < params.priceIds.length; j++) {
                if (params.priceIds[i] == params.priceIds[j]) {
                    revert SchedulerErrors.DuplicatePriceId(params.priceIds[i]);
                }
            }
        }

        // Whitelist size limit and uniqueness
        if (params.readerWhitelist.length > MAX_READER_WHITELIST_SIZE) {
            revert SchedulerErrors.TooManyWhitelistedReaders(
                params.readerWhitelist.length,
                MAX_READER_WHITELIST_SIZE
            );
        }
        for (uint i = 0; i < params.readerWhitelist.length; i++) {
            for (uint j = i + 1; j < params.readerWhitelist.length; j++) {
                if (params.readerWhitelist[i] == params.readerWhitelist[j]) {
                    revert SchedulerErrors.DuplicateWhitelistAddress(
                        params.readerWhitelist[i]
                    );
                }
            }
        }

        // Validate update criteria
        if (
            !params.updateCriteria.updateOnHeartbeat &&
            !params.updateCriteria.updateOnDeviation
        ) {
            revert SchedulerErrors.InvalidUpdateCriteria();
        }
        if (
            params.updateCriteria.updateOnHeartbeat &&
            params.updateCriteria.heartbeatSeconds == 0
        ) {
            revert SchedulerErrors.InvalidUpdateCriteria();
        }
        if (
            params.updateCriteria.updateOnDeviation &&
            params.updateCriteria.deviationThresholdBps == 0
        ) {
            revert SchedulerErrors.InvalidUpdateCriteria();
        }
    }

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
