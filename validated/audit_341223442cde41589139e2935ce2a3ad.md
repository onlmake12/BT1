### Title
Cross-Network Alert Signature Replay — (`File: util/network-alert/src/verifier.rs`)

### Summary
The `RawAlert` signature schema does not include a chain/network identifier. A valid alert signed by the Nervos Foundation key holders on one CKB network (e.g., mainnet) can be replayed verbatim on another CKB network (e.g., testnet/Aggron), because `calc_alert_hash` hashes only the `RawAlert` payload with no domain separator. Any unprivileged RPC caller or P2P peer can submit the replayed alert.

---

### Finding Description

The `RawAlert` molecule schema contains only application-level fields:

```
table RawAlert {
    notice_until:   Uint64,
    id:             Uint32,
    cancel:         Uint32,
    priority:       Uint32,
    message:        Bytes,
    min_version:    BytesOpt,
    max_version:    BytesOpt,
}
``` [1](#0-0) 

There is no `chain_id`, network name, or any domain-separating field. The alert hash is computed as a plain Blake2b hash of the raw bytes of this struct:

```rust
pub fn calc_alert_hash(&self) -> packed::Byte32 {
    self.calc_hash()   // Blake2b of self.as_slice() — no domain prefix
}
``` [2](#0-1) 

The verifier then uses this hash directly as the ECDSA message:

```rust
let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
// ...
verify_m_of_n(&message, self.config.signatures_threshold, &signatures, &self.pubkeys)
``` [3](#0-2) 

Because the signing keys are the same across all CKB networks (hard-coded in source, per the RPC documentation: "signed by 2-of-4 signatures … belong to early CKB developers"), a `(RawAlert, signatures)` pair that is valid on mainnet produces an identical hash on testnet and passes `verify_m_of_n` there too. [4](#0-3) 

---

### Impact Explanation

An attacker who observes a legitimately signed alert on CKB mainnet can replay it on testnet (or vice versa) with two concrete effects:

1. **False alarm injection**: A mainnet emergency alert (e.g., "upgrade immediately") is broadcast across testnet, causing confusion among developers and CI infrastructure.
2. **Legitimate alert suppression via cancel-replay**: If a `cancel` alert (fields: `id=N, cancel=M`) was signed and broadcast on mainnet to cancel alert `M`, an attacker can replay that cancel-alert on testnet to silently suppress a live, legitimate testnet alert with `id=M`. The `Notifier::cancel` method removes the target alert from both `received_alerts` and `noticed_alerts` with no further authentication. [5](#0-4) 

---

### Likelihood Explanation

The attack requires only:
1. Observing a valid alert on any public CKB network (alerts are broadcast P2P and visible to all peers).
2. Submitting it to a node on the target network via the `send_alert` RPC (open to any caller) or by relaying it over P2P. [6](#0-5) [7](#0-6) 

No privileged access, leaked keys, or majority hashpower is required. The attacker is a standard RPC caller or P2P peer.

---

### Recommendation

Include a network/chain domain separator in the signed payload. The simplest approach is to prepend the consensus chain ID (e.g., `"ckb"` for mainnet, `"ckb_testnet"` for Aggron) to the data hashed for the alert signature, analogous to EIP-712 domain separation:

```rust
// In calc_alert_hash, mix in the chain id:
let mut hasher = new_blake2b();
hasher.update(chain_id.as_bytes());   // e.g. b"ckb" or b"ckb_testnet"
hasher.update(self.as_slice());
// ...
```

Alternatively, add a `chain_id: Bytes` field to `RawAlert` in `extensions.mol` so the domain is part of the signed struct itself. [8](#0-7) 

---

### Proof of Concept

1. Connect to a CKB mainnet node and observe a valid alert via `get_blockchain_info` (alerts field) or P2P sniffing.
2. Extract the `Alert` struct: `{ raw: RawAlert { id, cancel, message, notice_until, … }, signatures: [sig1, sig2] }`.
3. Submit the identical struct to a CKB testnet node's `send_alert` RPC endpoint.
4. `Verifier::verify_signatures` computes `blake2b(raw_alert.as_slice())` — identical on both networks — and `verify_m_of_n` passes because the same Foundation public keys are configured on testnet.
5. The alert is accepted, stored in `Notifier::received_alerts`, and broadcast to all connected testnet peers.

If the replayed alert has `cancel > 0`, it additionally removes the targeted testnet alert from `noticed_alerts`, suppressing a legitimate warning. [9](#0-8) [10](#0-9)

    pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
        trace!("Verifying alert {:?}", alert);
        let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
        let signatures: Vec<Signature> = alert
            .signatures()
            .into_iter()
            .filter_map(
                |sig_data| match Signature::from_slice(sig_data.as_reader().raw_data()) {
                    Ok(sig) => {
                        if sig.is_valid() {
                            Some(sig)
                        } else {
                            debug!("invalid signature: {:?}", sig);
                            None
                        }
                    }
                    Err(err) => {
                        debug!("signature error: {}", err);
                        None
                    }
                },
            )
            .collect();
        verify_m_of_n(
            &message,
            self.config.signatures_threshold,
            &signatures,
            &self.pubkeys,
        )
        .map_err(|err| err.kind())?;
        Ok(())
    }

    fn send_alert(&self, alert: Alert) -> Result<()> {
        let alert: packed::Alert = alert.into();
        let now_ms = ckb_systemtime::unix_time_as_millis();
        let notice_until: u64 = alert.raw().notice_until().into();
        if notice_until < now_ms {
            return Err(RPCError::invalid_params(format!(
                "Expected `params[0].notice_until` in the future (> {now_ms}), got {notice_until}",
            )));
        }

        let result = self.verifier.verify_signatures(&alert);

        match result {
            Ok(()) => {
                // set self node notifier
                self.notifier.lock().add(&alert);

                self.network_controller.broadcast_with_handle(
                    SupportProtocols::Alert.protocol_id(),
                    alert.as_bytes(),
                    &self.handle,
                );
                Ok(())
            }
            Err(e) => Err(RPCError::custom_with_error(
                RPCError::AlertFailedToVerifySignatures,
                e,
            )),
        }
    }

    pub fn add(&mut self, alert: &Alert) {
        let alert_id = alert.raw().id().into();
        let alert_cancel = alert.raw().cancel().into();
        if self.has_received(alert_id) {
            return;
        }
        // checkout cancel_id
        if alert_cancel > 0 {
            self.cancel(alert_cancel);
        }
        // add to received alerts
        self.received_alerts.insert(alert_id, alert.clone());

        // check conditions, figure out do we need to notice this alert
        if !self.is_version_effective(alert) {
            debug!("Received a version ineffective alert {:?}", alert);
            return;
        }

        if self.noticed_alerts.contains(alert) {
            return;
        }
        self.notify_controller.notify_network_alert(alert.clone());
        self.noticed_alerts.push(alert.clone());
        // sort by priority
        self.noticed_alerts.sort_by_key(|a| {
            let priority: u32 = a.raw().priority().into();
            u32::MAX - priority
        });
    }
