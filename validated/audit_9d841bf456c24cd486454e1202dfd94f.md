### Title
Expired Trusted Signers Permanently Occupy Fixed-Size Array Slots, Blocking Signer Rotation and Bricking `verifyUpdate` — (`lazer/contracts/evm/src/PythLazer.sol`)

---

### Summary

`PythLazer.sol` stores trusted signers in a fixed-size array of 100 entries. Expired signers are never automatically reclaimed. If all 100 slots are occupied by non-zero pubkeys (even fully expired ones), no new signer can be added, and every call to `verifyUpdate` reverts permanently — bricking the Lazer contract for all downstream consumers.

---

### Finding Description

`PythLazer.sol` declares a fixed-size storage array:

```solidity
TrustedSignerInfo[100] internal trustedSigners;
``` [1](#0-0) 

A slot is considered **empty** only when `pubkey == address(0)`. Explicit removal via `updateTrustedSigner(addr, 0)` sets `pubkey = address(0)`, freeing the slot:

```solidity
trustedSigners[i].pubkey = address(0);
trustedSigners[i].expiresAt = 0;
delete trustedSignerToExpiresAtMapping[trustedSigner];
``` [2](#0-1) 

However, when a signer simply **expires by time** (i.e., `block.timestamp >= expiresAt`), the slot is **not freed** — the entry retains a non-zero `pubkey`. The "add new signer" path only reclaims slots where `pubkey == address(0)`:

```solidity
for (uint8 i = 0; i < trustedSigners.length; i++) {
    if (trustedSigners[i].pubkey == address(0)) {
        trustedSigners[i].pubkey = trustedSigner;
        ...
        return;
    }
}
revert("no space for new signer");
``` [3](#0-2) 

Once all 100 slots are occupied by non-zero pubkeys (all expired), the owner cannot add a new signer. Simultaneously, `isValidSigner` returns `false` for every address because all entries are past their `expiresAt`:

```solidity
function isValidSigner(address signer) public view returns (bool) {
    return block.timestamp < trustedSignerToExpiresAtMapping[signer];
}
``` [4](#0-3) 

This causes `verifyUpdate` to revert on every call:

```solidity
if (!isValidSigner(signer)) {
    revert("invalid signer");
}
``` [5](#0-4) 

The contract is now in a stuck state: no valid signers exist, and no new signer can be added to restore service. Recovery requires the owner to call `updateTrustedSigner(expiredAddr, 0)` for each of the 100 expired entries — but the `trustedSigners` array is `internal` with no public enumeration getter, making it difficult to discover all occupied pubkeys on-chain.

---

### Impact Explanation

All downstream contracts and users that call `verifyUpdate` to consume Lazer price data receive a permanent revert. The Lazer price feed becomes completely non-functional. Any protocol that has locked funds contingent on receiving a Lazer price update (e.g., liquidation triggers, settlement logic) is bricked until the owner manually clears all 100 expired slots — a recovery that requires off-chain knowledge of every historical signer pubkey.

---

### Likelihood Explanation

The array has 100 slots. Over a long operational lifetime with periodic key rotations (e.g., annual or semi-annual), and without a disciplined practice of explicitly removing old signers before they expire, the array gradually fills. The condition is not triggered by any attacker action — it arises from normal governance operations. The absence of any automatic expiry-based slot reclamation means the risk grows monotonically with the number of signer rotations performed.

---

### Recommendation

Reclaim expired slots automatically in the "add new signer" path. After the first loop (exact pubkey match) and before the second loop (zero-pubkey slot), add a third pass that treats any slot where `expiresAt < block.timestamp` as reclaimable:

```solidity
// Reclaim expired slot
for (uint8 i = 0; i < trustedSigners.length; i++) {
    if (trustedSigners[i].expiresAt < block.timestamp) {
        delete trustedSignerToExpiresAtMapping[trustedSigners[i].pubkey];
        trustedSigners[i].pubkey = trustedSigner;
        trustedSigners[i].expiresAt = expiresAt;
        trustedSignerToExpiresAtMapping[trustedSigner] = expiresAt;
        return;
    }
}
```

Alternatively, replace the fixed-size array with a dynamic array or mapping, and add an explicit `cleanExpiredSigners()` function callable by anyone.

---

### Proof of Concept

1. Deploy `PythLazer` and call `updateTrustedSigner` 100 times with 100 distinct addresses, each with `expiresAt = block.timestamp + 1`.
2. Advance time by 2 seconds (`vm.warp(block.timestamp + 2)`).
3. All 100 signers are now expired. `isValidSigner` returns `false` for all of them.
4. Call `updateTrustedSigner(newSigner, block.timestamp + 1000)` → reverts with `"no space for new signer"` because every slot has a non-zero `pubkey` and none equal `address(0)`.
5. Call `verifyUpdate(validUpdate)` → reverts with `"invalid signer"`.
6. The contract is permanently non-functional until the owner iterates through all 100 known pubkeys and calls `updateTrustedSigner(addr, 0)` for each. [6](#0-5)
