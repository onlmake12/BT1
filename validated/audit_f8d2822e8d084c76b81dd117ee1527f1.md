### Title
Stale Price Usage in Chainlink-Compatible Adapter — (`File: target_chains/ethereum/sdk/solidity/PythAggregatorV3.sol`)

---

### Summary

`PythAggregatorV3` is Pyth's official Chainlink AggregatorV3-compatible adapter. Every price-reading function in the contract — `latestRoundData()`, `getRoundData()`, `latestAnswer()`, `latestTimestamp()`, and `decimals()` — calls `pyth.getPriceUnsafe(priceId)`, which explicitly performs **no staleness check** and can return a price from arbitrarily far in the past. Any protocol that deploys this adapter as a drop-in Chainlink oracle will silently consume stale prices with no on-chain protection.

---

### Finding Description

`PythAggregatorV3.sol` implements the Chainlink `AggregatorV3Interface` to allow protocols to use Pyth prices via the familiar Chainlink API. However, every function that reads a price delegates to `getPriceUnsafe`:

```solidity
// latestRoundData — line 110
PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);

// latestAnswer — line 54
PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);

// getRoundData — line 89
PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);

// decimals — line 41
PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
``` [1](#0-0) [2](#0-1) [3](#0-2) 

`getPriceUnsafe` is documented and implemented to return the last stored price with **zero recency enforcement**:

> "WARNING: the returned price can be from arbitrarily far in the past." [4](#0-3) 

The safe alternative, `getPriceNoOlderThan(id, age)`, exists in `AbstractPyth.sol` and enforces a `publishTime` freshness bound by reverting with `StalePrice` if the age threshold is exceeded: [5](#0-4) 

`PythAggregatorV3` never calls this safe variant. The returned `updatedAt` field in `latestRoundData()` is set to `price.publishTime`, which may be hours or days old, giving downstream callers a false sense of freshness if they check `updatedAt` against `block.timestamp`.

---

### Impact Explanation

Any lending protocol, derivatives platform, or liquidation engine that deploys `PythAggregatorV3` as a Chainlink-compatible oracle and relies on `latestRoundData()` will receive stale prices during Pyth feed outages or network congestion. An attacker can:

1. Wait for the on-chain Pyth price to become stale (no `updatePriceFeeds` call has been made recently).
2. Interact with the downstream protocol (e.g., borrow against an asset whose true price has dropped, or avoid liquidation when the true price has fallen below the liquidation threshold).
3. The protocol reads `latestRoundData()` → gets the old `price.publishTime` as `updatedAt` → no revert occurs → the stale price is used in a critical financial operation.

The impact is **direct loss of funds** for the protocol or its users via mispriced collateral, undercollateralized loans, or missed liquidations.

---

### Likelihood Explanation

- `PythAggregatorV3` is the official Pyth-provided Chainlink adapter, actively promoted for integrations.
- Pyth uses a pull-based update model; if no keeper calls `updatePriceFeeds`, the on-chain price ages indefinitely.
- During any network congestion, keeper downtime, or targeted griefing (withholding updates), the price becomes stale.
- No privileged access is required — any unprivileged user can exploit a downstream protocol at the moment the price is stale.

---

### Recommendation

Replace all `getPriceUnsafe` calls in `PythAggregatorV3` with `getPriceNoOlderThan(priceId, maxAge)`, where `maxAge` is a configurable parameter set at construction time (e.g., defaulting to `pyth.getValidTimePeriod()`). This mirrors the fix described in the external report: validate the `updatedAt` equivalent (`publishTime`) against a configurable expiration buffer.

```solidity
uint256 public maxAge; // set in constructor

function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
    ...
}
```

---

### Proof of Concept

1. Deploy `PythAggregatorV3` pointing to a live Pyth contract and a price feed ID.
2. Do **not** call `updateFeeds` for a period exceeding the protocol's acceptable staleness window (e.g., 60 seconds).
3. Call `latestRoundData()` — it returns successfully with `updatedAt = price.publishTime` from minutes/hours ago.
4. A downstream lending protocol that calls `latestRoundData()` and does not independently validate `updatedAt` will use this stale price for collateral valuation or liquidation logic.
5. An attacker opens or closes a position at the stale price, profiting from the discrepancy between the stale on-chain price and the true market price. [3](#0-2) [5](#0-4)

    function getPriceUnsafe(
        bytes32 id
    ) public view override returns (PythStructs.Price memory price) {
        PythInternalStructs.PriceInfo storage info = _state.latestPriceInfo[id];
        price.publishTime = info.publishTime;
        price.expo = info.expo;
        price.price = info.price;
        price.conf = info.conf;

        if (price.publishTime == 0) revert PythErrors.PriceFeedNotFound();
    }

    function getPriceNoOlderThan(
        bytes32 id,
        uint age
    ) public view virtual override returns (PythStructs.Price memory price) {
        price = getPriceUnsafe(id);

        if (diff(block.timestamp, price.publishTime) > age)
            revert PythErrors.StalePrice();

        return price;
    }
