Audit Report

## Title
Unbounded `GetBlocksProof` Request Rate with O(k·log(N)) MMR RocksDB Reads Enables Low-Cost DoS on Light Client Servers — (`util/light-client-protocol-server/src/lib.rs`, `util/light-client-protocol-server/src/components/get_blocks_proof.rs`)

## Summary
`LightClientProtocol` has no per-peer rate limiter, unlike `Relayer` which gates all non-PoW messages at 30 req/s. Any peer can repeatedly send `GetBlocksProof` messages with up to 1000 valid main-chain block hashes, each triggering O(k·log(N)) RocksDB reads via `mmr.gen_proof`. At modest request rates this saturates server I/O and can crash or severely degrade the node.

## Finding Description

**No rate limiter in struct or dispatch:**
`LightClientProtocol` holds only `pub shared: Shared` with no `rate_limiter` field. [1](#0-0) 
`received()` deserializes the message and immediately calls `self.try_process(...)`. [2](#0-1) 
`try_process()` dispatches `GetBlocksProof` directly to `GetBlocksProofProcess::execute()` with no interposed rate check. [3](#0-2) 

**Per-request limit — 1000 hashes, no frequency constraint:**
`GET_BLOCKS_PROOF_LIMIT` is 1000. [4](#0-3) 
`execute()` rejects messages exceeding this limit but imposes no frequency constraint. [5](#0-4) 

**MMR proof generation — O(k·log(N)) RocksDB reads per request:**
`execute()` calls `reply_proof` with up to 1000 leaf positions. [6](#0-5) 
Inside `reply_proof`, `mmr.gen_proof(items_positions)` is invoked. [7](#0-6) 
The `MMRStore<HeaderDigest>` implementation for `&Snapshot` issues one RocksDB read per MMR node accessed via `get_header_digest`. [8](#0-7) 

**Contrast with `Relayer`:**
`Relayer` holds a `governor`-based `rate_limiter` field keyed by `(PeerIndex, u32)` at 30 req/s. [9](#0-8) 
`Relayer::try_process` checks this limiter before any message handling and returns `TooManyRequests` on violation. [10](#0-9) 
No equivalent guard exists anywhere in `LightClientProtocol` — confirmed by absence of any `rate_limiter`, `RateLimiter`, `governor`, or `TooManyRequests` references in the entire `util/light-client-protocol-server/` tree. 

## Impact Explanation
A single attacker peer sending `GetBlocksProof` at 10 req/s with 1000 spread-out valid hashes forces ~230,000 RocksDB reads/second (≈23,000 reads/request × 10 req/s on a ~10M-block chain). This saturates storage I/O, starving the chain processing pipeline and causing the node to become unresponsive or crash. This matches the allowed **High** impact: *"Vulnerabilities which could easily crash a CKB node"* and *"Vulnerabilities or bad designs which could cause CKB network congestion with few costs."* [11](#0-10) 

## Likelihood Explanation
The attack requires only a valid P2P connection — no PoW, no keys, no privilege. The 1000 required main-chain block hashes are trivially obtained from any public block explorer or by syncing headers. The attack is repeatable indefinitely from a single peer and scales linearly with the number of attacker-controlled peers. The cost to the attacker is negligible network bandwidth; the cost to the server grows with log(chain height). [12](#0-11) 

## Recommendation
Add a per-peer rate limiter to `LightClientProtocol` analogous to the one in `Relayer`:

```rust
pub struct LightClientProtocol {
    pub shared: Shared,
    rate_limiter: RateLimiter<(PeerIndex, u32)>,
}
```

In `try_process`, check the limiter before dispatching any message, returning `StatusCode::TooManyRequests` on violation. Additionally, consider reducing `GET_BLOCKS_PROOF_LIMIT`, `GET_TRANSACTIONS_PROOF_LIMIT`, and `GET_LAST_STATE_PROOF_LIMIT` from 1000, or adding a global async work queue with backpressure for MMR proof generation. [12](#0-11) 

## Proof of Concept
1. Connect to a mainnet light client server via P2P (protocol id 120, `/ckb/lightclient`).
2. Collect 1000 valid main-chain block hashes spread across the full chain height (e.g., blocks 0, 10000, 20000, …, 9,990,000) from a public block explorer.
3. In a tight loop, send `GetBlocksProof { last_hash: <current tip hash>, block_hashes: [1000 hashes] }`.
4. Instrument `mmr.gen_proof` call count and wall time on chains of height 100, 10,000, and 1,000,000 to confirm O(k·log(N)) growth.
5. Observe server RocksDB I/O metrics saturating and chain processing latency increasing monotonically with request rate. [13](#0-12)

    pub(crate) async fn execute(self) -> Status {
        if self.message.block_hashes().is_empty() {
            return StatusCode::MalformedProtocolMessage.with_context("no block");
        }

        if self.message.block_hashes().len() > constant::GET_BLOCKS_PROOF_LIMIT {
            return StatusCode::MalformedProtocolMessage.with_context("too many blocks");
        }

        let snapshot = self.protocol.shared.snapshot();

        let last_block_hash = self.message.last_hash().to_entity();
        if !snapshot.is_main_chain(&last_block_hash) {
            return self
                .protocol
                .reply_tip_state::<packed::SendBlocksProof>(self.peer, self.nc)
                .await;
        }
        let last_block = snapshot
            .get_block(&last_block_hash)
            .expect("block should be in store");

        let block_hashes: Vec<_> = self
            .message
            .block_hashes()
            .to_entity()
            .into_iter()
            .collect();

        let mut uniq = HashSet::new();
        if !block_hashes
            .iter()
            .chain([last_block_hash].iter())
            .all(|hash| uniq.insert(hash))
        {
            return StatusCode::MalformedProtocolMessage
                .with_context("duplicate block hash exists");
        }

        let (found, missing): (Vec<_>, Vec<_>) = block_hashes
            .into_iter()
            .partition(|block_hash| snapshot.is_main_chain(block_hash));

        let mut positions = Vec::with_capacity(found.len());
        let mut block_headers = Vec::with_capacity(found.len());
        let mut uncles_hash = Vec::with_capacity(found.len());
        let mut extensions = Vec::with_capacity(found.len());

        for block_hash in found {
            let header = snapshot
                .get_block_header(&block_hash)
                .expect("header should be in store");
            positions.push(leaf_index_to_pos(header.number()));
            block_headers.push(header.data());

            let uncles = snapshot
                .get_block_uncles(&block_hash)
                .expect("block uncles must be stored");
            let extension = snapshot.get_block_extension(&block_hash);

            uncles_hash.push(uncles.data().calc_uncles_hash());
            extensions.push(packed::BytesOpt::new_builder().set(extension).build());
        }

        let proved_items = (
            block_headers.into(),
            uncles_hash.into(),
            packed::BytesOptVec::new_builder().set(extensions).build(),
        );
        let missing_items = missing.into();

        self.protocol
            .reply_proof::<packed::SendBlocksProofV1>(
                self.peer,
                self.nc,
                &last_block,
                positions,
                proved_items,
                missing_items,
            )
            .await
    }
