### Title
`SetValidPeriod` Governance Action Permanently Unexecutable on Starknet — (`File: target_chains/starknet/contracts/src/pyth/governance.cairo`)

---

### Summary

The Starknet Pyth contract recognizes `SetValidPeriod` (action byte `4`) as a valid governance action in its `GovernanceAction` enum and byte-to-action mapping, but the `parse_instruction` dispatcher unconditionally panics with `'unimplemented'` when this action is encountered. The `GovernancePayload` enum also has no corresponding variant. As a result, no governance VAA carrying `SetValidPeriod` can ever be executed on Starknet, permanently preventing Pyth governance from adjusting the price staleness threshold on that chain.

---

### Finding Description

In `target_chains/starknet/contracts/src/pyth/governance.cairo`, the `GovernanceAction` enum includes `SetValidPeriod` as a recognized variant: [1](#0-0) 

The `U8TryIntoGovernanceAction` implementation maps byte value `4` to `GovernanceAction::SetValidPeriod`, making it a parseable, non-error action: [2](#0-1) 

However, in `parse_instruction`, the match arm for `SetValidPeriod` unconditionally panics: [3](#0-2) 

The `GovernancePayload` enum, which is the output type of `parse_instruction`, has no `SetValidPeriod` variant — it is explicitly commented out as unsupported: [4](#0-3) 

The downstream executor in `target_chains/starknet/contracts/src/pyth.cairo` matches on `GovernancePayload` variants and would have no case to handle it even if parsing succeeded: [5](#0-4) 

By contrast, every other Pyth target chain fully implements `SetValidPeriod`. The EVM contract handles it: [6](#0-5)

### Impact Explanation

`SetValidPeriod` controls the maximum age of a price update before it is considered stale. If Pyth governance needs to tighten or loosen the staleness window on Starknet — for example, in response to a change in Wormhole guardian set update frequency, a market volatility event, or a misconfigured initial deployment — it is permanently blocked from doing so without a full contract upgrade. Any governance VAA with action byte `4` targeting Starknet will always revert. This is a governance execution failure that degrades the security model: the staleness parameter is frozen at its initial value indefinitely, and the governance authority over this critical parameter is effectively nullified on Starknet.

---

### Likelihood Explanation

The Pyth governance system regularly issues cross-chain parameter updates. `SetValidPeriod` is a standard governance action used on all other chains. A governance submitter — an unprivileged relayer — can submit any valid governance VAA to `execute_governance_instruction`. When a `SetValidPeriod` VAA is submitted to the Starknet contract, the transaction will always revert. The entry path requires no special privilege: any address can call `execute_governance_instruction` with a properly signed VAA.

---

### Recommendation

Implement the `SetValidPeriod` handler in the Starknet contract:

1. Add a `SetValidPeriod` variant to `GovernancePayload` in `governance.cairo` with a `valid_seconds: u64` field.
2. Replace the `panic_with_felt252('unimplemented')` arm in `parse_instruction` with logic that reads the `u64` valid period from the payload and constructs `GovernancePayload::SetValidPeriod(...)`.
3. Add a match arm for `GovernancePayload::SetValidPeriod` in the executor in `pyth.cairo` that calls the appropriate state-update function (analogous to `setValidPeriod` on EVM).

---

### Proof of Concept

1. Construct a governance VAA with:
   - Module: `Target` (byte `1`)
   - Action: `SetValidPeriod` (byte `4`)
   - Target chain: Starknet chain ID
   - Payload: 8-byte big-endian `u64` new valid period
2. Sign it with the Pyth governance key.
3. Call `execute_governance_instruction(vaa)` on the Starknet Pyth contract.
4. The call reverts with `'unimplemented'` at `governance.cairo` line 184 — the action is recognized as valid by the parser but the handler panics before any state change occurs.
5. The valid period on Starknet remains unchanged regardless of how many times governance attempts to update it.
