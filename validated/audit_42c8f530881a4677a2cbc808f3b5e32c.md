### Title
Inbound Peer Can Promote Ephemeral Port to Listen State via `GetNodes{version=1, listen_port=None}`, Causing Peer Store Pollution — (`network/src/protocols/discovery/mod.rs`, `network/src/protocols/discovery/state.rs`)

---

### Summary

An unprivileged inbound peer can send a `GetNodes` message with `version=1` (`REUSE_PORT_VERSION`) and `listen_port=None`. The receiver unconditionally calls `change_to_listen()` when `version >= REUSE_PORT_VERSION`, promoting the session's `RemoteAddress::Init(ephemeral_addr)` to `RemoteAddress::Listen(ephemeral_addr)` without any port correction. On the next discovery `notify` tick (within 60 seconds), `check_timer` returns the ephemeral address, which passes the `is_valid_addr` filter for public IPs, and the address is broadcast to up to 3 peers as a valid listen address.

---

### Finding Description

**Inbound session initialization** sets `remote_addr` to `Init` with the ephemeral source port: [1](#0-0) 

**GetNodes handler** — the two branches are independent and execute sequentially: [2](#0-1) 

When `listen_port=None`, the `if let Some(port)` block at line 124 is skipped entirely — no `update_port()` call, no port correction. Then at line 136, `change_to_listen()` is called unconditionally for any `version >= 1`: [3](#0-2) 

This promotes `Init(ip:ephemeral_port)` → `Listen(ip:ephemeral_port)` with the raw ephemeral source port intact.

**`check_timer`** returns the address on the first call (because `last_announce` is `None`, `unwrap_or(true)` fires immediately): [4](#0-3) 

**`notify` handler** has two gates before broadcasting: [5](#0-4) 

- **Gate 1 — `is_valid_addr`**: only checks that the IP is globally reachable (not private/loopback). It does not validate whether the port is a listen port. A public-IP attacker passes this trivially.
- **Gate 2 — `node_flags`**: requires `identify_info` to be set. The identify protocol sends its message immediately on `connected` via `quick_send_message`, completing within seconds. The attacker simply completes the normal identify handshake (sends a valid network-name + non-zero flags message), which sets `identify_info`: [6](#0-5) 

Both gates are passable by any ordinary connecting peer. The ephemeral address then enters `announce_list` and is pushed to up to 3 randomly selected peers: [7](#0-6) 

**The design intent is broken**: `REUSE_PORT_VERSION` was introduced for Linux SO_REUSEPORT, where the TCP source port genuinely equals the listen port. But the receiver has no way to verify this claim. Any peer on any platform can assert `version=1` with `listen_port=None`, and the receiver will unconditionally trust it.

This is made more glaring by an explicit comment in the identify protocol that deliberately avoids adding inbound addresses to the peer store precisely because they are unverified: [8](#0-7) 

The discovery `change_to_listen()` path silently bypasses this protection.

---

### Impact Explanation

Each attacker connection (from a public IP) injects one unreachable ephemeral address into the peer stores of up to 3 peers. The address is broadcast once per session (the 24-hour `ANNOUNCE_INTERVAL` prevents repetition within a session). Peers receiving the address will attempt connections to the ephemeral port, fail, and eventually evict the entry — but until eviction, their peer store is polluted and connection slots may be wasted. An attacker opening many concurrent inbound connections amplifies the effect linearly.

---

### Likelihood Explanation

The path requires only: (1) a public IP, (2) completing the standard identify handshake, and (3) sending one crafted `GetNodes` message. No special privileges, no PoW, no key material. The attack is trivially scriptable.

---

### Recommendation

The `change_to_listen()` call at line 138 must not be reached when `listen_port=None`. The correct fix is to restructure the two branches so that `change_to_listen()` is only called as a fallback when `listen_port` was already `Some(port)` and `update_port()` has already run (i.e., the address is already in `Listen` state and `change_to_listen()` is a no-op), **or** to require that `listen_port` must be `Some` for any promotion to `Listen` state regardless of version. The version field should only gate whether the node *sends* `listen_port=None` (trusting SO_REUSEPORT locally), not whether the *receiver* blindly promotes the remote address.

### Proof of Concept

```
1. Attacker opens inbound TCP connection from public_ip:54321 to victim node.
   → victim: remote_addr = Init(public_ip:54321)

2. Attacker completes identify handshake (sends valid network name + flags).
   → victim: peer.identify_info = Some(PeerIdentifyInfo { flags: COMPATIBILITY })

3. Attacker sends GetNodes { version: 1, listen_port: None, count: 1000, required_flags: 0 }
   → mod.rs line 124: listen_port is None → skip update_port()
   → mod.rs line 136: version(1) >= REUSE_PORT_VERSION(1) → call change_to_listen()
   → state.rs line 148: Init(public_ip:54321) → Listen(public_ip:54321)

4. Within 60 seconds, notify() fires.
   → check_timer: last_announce=None → unwrap_or(true) → returns Listen(public_ip:54321)
   → is_valid_addr(public_ip:54321): public IP → true
   → node_flags: identify_info set → Some(COMPATIBILITY)
   → announce_list = [(public_ip:54321, COMPATIBILITY)]

5. announce_list is broadcast to up to 3 randomly chosen peers.
   → Those peers store public_ip:54321 as a reachable listen address.
   → public_ip:54321 is an ephemeral port, not listening → all connection attempts fail.
```

    pub(crate) fn check_timer(&mut self, now: Instant, interval: Duration) -> Option<&Multiaddr> {
        if self
            .last_announce
            .map(|time| now.saturating_duration_since(time) > interval)
            .unwrap_or(true)
        {
            self.last_announce = Some(now);
            if let RemoteAddress::Listen(addr) = &self.remote_addr {
                Some(addr)
            } else {
                None
            }
        } else {
            None
        }
    }
