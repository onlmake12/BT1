### Title
Unbounded `activeSubscriptionIds` Array Growth via Unrestricted `createSubscription` Allows Keeper DoS — (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`Scheduler.createSubscription` is callable by any address and unconditionally pushes the new subscription ID into the unbounded `_state.activeSubscriptionIds` storage array. There is no cap on the total number of active subscriptions. A sufficiently funded attacker can create arbitrarily many subscriptions, bloating this array until off-chain keeper bots that call `getActiveSubscriptions` time out or run out of memory, preventing legitimate subscriptions from ever being updated.

---

### Finding Description

`createSubscription` is `external payable` with no caller restriction. [1](#0-0) 

After parameter validation it unconditionally calls `_addToActiveSubscriptions(subscriptionId)`: [2](#0-1) 

`_state.activeSubscriptionIds` is a plain dynamic `uint256[]` in `SchedulerState` with no declared maximum length: [3](#0-2) 

The only guard is a minimum-balance check: [4](#0-3) 

`_validateSubscriptionParams` enforces per-subscription limits (max price IDs, whitelist size, valid criteria) but imposes **no cap on the total number of subscriptions** that may exist simultaneously: [5](#0-4) 

Off-chain keepers discover work by calling `getActiveSubscriptions`, which iterates over the entire `activeSubscriptionIds` array. With a sufficiently large array the RPC call exceeds node memory/timeout limits, and keepers stop processing any subscription.

---

### Impact Explanation

Legitimate subscribers' price feeds stop being updated. Any DeFi protocol that depends on the Scheduler for fresh Pyth prices will read stale data, potentially enabling liquidation manipulation or mispriced trades. The degradation is sustained for as long as the attacker keeps their subscriptions active (funds locked, not lost).

---

### Likelihood Explanation

The attacker must lock `minimumBalancePerFeed × numFeeds` ETH per subscription. With 1 feed per subscription the cost equals `minimumBalancePerFeed` per slot. A well-capitalised attacker (or one on a low-fee chain where `minimumBalancePerFeed` is small) can create tens of thousands of subscriptions. The funds are recoverable upon deactivation, so the net cost is only gas and the opportunity cost of locked capital. No privileged access is required.

---

### Recommendation

1. Add a global cap constant (e.g. `MAX_ACTIVE_SUBSCRIPTIONS`) and revert in `createSubscription` when `_state.activeSubscriptionIds.length >= MAX_ACTIVE_SUBSCRIPTIONS`.
2. Alternatively, require a non-refundable creation fee (separate from the operational balance) to raise the economic cost of spam subscriptions.
3. Ensure `getActiveSubscriptions` is never called in a single unbounded on-chain loop; the current pagination parameters help off-chain but do not prevent array bloat.

---

### Proof of Concept

```
// Attacker script (pseudo-Solidity)
IScheduler scheduler = IScheduler(<deployed>);
SchedulerStructs.SubscriptionParams memory p = /* 1 feed, valid criteria */;
uint256 minBal = scheduler.getMinimumBalance(1);

for (uint i = 0; i < 100_000; i++) {
    scheduler.createSubscription{value: minBal}(p);
}

// Now getActiveSubscriptions(0, type(uint256).max) returns 100 000 entries.
// Keeper bots iterating this array time out; no legitimate subscription is updated.
```

    function createSubscription(
        SchedulerStructs.SubscriptionParams memory subscriptionParams
    ) external payable override returns (uint256 subscriptionId) {
        _validateSubscriptionParams(subscriptionParams);

        // Calculate minimum balance required for this subscription
        uint256 minimumBalance = this.getMinimumBalance(
            uint8(subscriptionParams.priceIds.length)
        );

        // Ensure enough funds were provided
        if (msg.value < minimumBalance) {
            revert SchedulerErrors.InsufficientBalance();
        }

        // Check deposit limit for permanent subscriptions
        if (subscriptionParams.isPermanent && msg.value > MAX_DEPOSIT_LIMIT) {
            revert SchedulerErrors.MaxDepositLimitExceeded();
        }

        // Set subscription to active
        subscriptionParams.isActive = true;

        subscriptionId = _state.subscriptionNumber++;

        // Store the subscription parameters
        _state.subscriptionParams[subscriptionId] = subscriptionParams;

        // Initialize subscription status
        SchedulerStructs.SubscriptionStatus storage status = _state
            .subscriptionStatuses[subscriptionId];
        status.priceLastUpdatedAt = 0;
        status.balanceInWei = msg.value;
        status.totalUpdates = 0;
        status.totalSpent = 0;

        // Map subscription ID to manager
        _state.subscriptionManager[subscriptionId] = msg.sender;

        _addToActiveSubscriptions(subscriptionId);

        emit SubscriptionCreated(subscriptionId, msg.sender);
        return subscriptionId;
    }

    function _validateSubscriptionParams(
        SchedulerStructs.SubscriptionParams memory params
    ) internal pure {
        // No zero‐feed subscriptions
        if (params.priceIds.length == 0) {
            revert SchedulerErrors.EmptyPriceIds();
        }

        // Price ID limits and uniqueness
        if (
            params.priceIds.length >
            SchedulerConstants.MAX_PRICE_IDS_PER_SUBSCRIPTION
        ) {
            revert SchedulerErrors.TooManyPriceIds(
                params.priceIds.length,
                SchedulerConstants.MAX_PRICE_IDS_PER_SUBSCRIPTION
            );
        }
        for (uint i = 0; i < params.priceIds.length; i++) {
            for (uint j = i + 1; j < params.priceIds.length; j++) {
                if (params.priceIds[i] == params.priceIds[j]) {
                    revert SchedulerErrors.DuplicatePriceId(params.priceIds[i]);
                }
            }
        }

        // Whitelist size limit and uniqueness
        if (params.readerWhitelist.length > MAX_READER_WHITELIST_SIZE) {
            revert SchedulerErrors.TooManyWhitelistedReaders(
                params.readerWhitelist.length,
                MAX_READER_WHITELIST_SIZE
            );
        }
        for (uint i = 0; i < params.readerWhitelist.length; i++) {
            for (uint j = i + 1; j < params.readerWhitelist.length; j++) {
                if (params.readerWhitelist[i] == params.readerWhitelist[j]) {
                    revert SchedulerErrors.DuplicateWhitelistAddress(
                        params.readerWhitelist[i]
                    );
                }
            }
        }

        // Validate update criteria
        if (
            !params.updateCriteria.updateOnHeartbeat &&
            !params.updateCriteria.updateOnDeviation
        ) {
            revert SchedulerErrors.InvalidUpdateCriteria();
        }
        if (
            params.updateCriteria.updateOnHeartbeat &&
            params.updateCriteria.heartbeatSeconds == 0
        ) {
            revert SchedulerErrors.InvalidUpdateCriteria();
        }
        if (
            params.updateCriteria.updateOnDeviation &&
            params.updateCriteria.deviationThresholdBps == 0
        ) {
            revert SchedulerErrors.InvalidUpdateCriteria();
        }
    }
