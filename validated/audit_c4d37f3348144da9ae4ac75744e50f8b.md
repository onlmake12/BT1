### Title
Deprecated `parse_and_verify_le_ecdsa_update()` Aborts on Valid `FixedRate1000ms` Channel Updates Due to Incomplete `channel::from_v2` Conversion - (File: `lazer/contracts/sui/sources/channel.move`)

---

### Summary

The deprecated `parse_and_verify_le_ecdsa_update()` function in `lazer/contracts/sui/sources/pyth_lazer.move` will unconditionally abort when processing any valid Lazer update that uses the `FixedRate1000ms` channel (channel value `4`). This is because the `channel::from_v2()` conversion helper it relies on only handles three of the four valid channel values, aborting on the fourth. The `FixedRate1000ms` channel is actively used in production (it is the channel in the canonical test data). Any Sui consumer contract that has not yet migrated away from the deprecated function will be permanently broken for this channel.

---

### Finding Description

`parse_and_verify_le_ecdsa_update` is marked `#[deprecated]` and delegates to `parse_and_verify_le_ecdsa_update_v2`, then converts the result back to the legacy `update_v1::Update` type via `update_v1::from_v2(update)`. [1](#0-0) 

`update_v1::from_v2` calls `channel::from_v2(update.channel())` to downcast the v2 `Channel` type to the legacy v1 `Channel` type: [2](#0-1) 

The `channel::from_v2` function only handles three channel variants — `RealTime` (1), `FixedRate50ms` (2), `FixedRate200ms` (3) — and **aborts** on any other value: [3](#0-2) 

However, `channel_v2` supports a fourth valid channel value: `FixedRate1000ms` (value `4`), which was added after the v1 channel enum was frozen: [4](#0-3) [5](#0-4) 

The canonical test data in `pyth_lazer_tests.move` explicitly uses the `fixed_rate@1000ms` channel, confirming this is a live production channel: [6](#0-5) 

---

### Impact Explanation

Any Sui consumer contract that calls the deprecated `parse_and_verify_le_ecdsa_update()` with a valid, properly-signed Lazer update on the `FixedRate1000ms` channel will have its transaction abort with `EInvalidChannel`. This is a **denial of service** on the consumer's price-update path: the consumer cannot ingest current price data, which can cause stale prices to be used in downstream DeFi protocols (lending, derivatives, etc.) that depend on Lazer for real-time pricing.

---

### Likelihood Explanation

**Medium.** The `FixedRate1000ms` channel is the channel used in the project's own integration test data, indicating it is the primary production channel. The deprecated function remains publicly callable (`public fun`) on Sui, so any consumer that has not yet migrated will silently break the moment a `FixedRate1000ms` update is submitted. A Lazer relayer submitting a legitimate signed update is the unprivileged trigger — no key compromise or governance attack is required.

---

### Recommendation

1. **Remove or gate the deprecated function**: Since `channel::from_v2` cannot be safely extended (the v1 `Channel` enum is frozen at 3 variants), the deprecated `parse_and_verify_le_ecdsa_update` should either be removed entirely or made to abort with a clear migration error before attempting the conversion.

2. **Fix `channel::from_v2`**: If the deprecated path must be preserved for backward compatibility, add a fallback mapping for `FixedRate1000ms` (e.g., map it to `FixedRate200ms` with a warning), or extend the v1 `Channel` enum to include `FixedRate1000ms`.

3. **Emit a compile-time error**: Replace the `#[deprecated]` annotation with a hard compile error or a runtime abort with a descriptive message directing callers to `parse_and_verify_le_ecdsa_update_v2`.

---

### Proof of Concept

1. A Lazer relayer submits a valid, trusted-signer-signed update with channel byte `0x04` (`FixedRate1000ms`).
2. A consumer contract calls `parse_and_verify_le_ecdsa_update(state, clock, update)`.
3. Internally: `parse_and_verify_le_ecdsa_update_v2` succeeds (signature valid, payload valid, channel `4` accepted by `channel_v2::from_u8`).
4. `update_v1::from_v2(update)` is called → `channel::from_v2(Channel(4))` is called.
5. None of the three `if` branches match → `abort EInvalidChannel` is executed.
6. The consumer's transaction reverts; price data is not updated.

The test file itself imports `parse_and_verify_le_ecdsa_update` alongside `parse_and_verify_le_ecdsa_update_v2` and uses `TEST_LAZER_UPDATE` (which carries channel `4`) — running the deprecated variant against this data would reproduce the abort: [7](#0-6)
