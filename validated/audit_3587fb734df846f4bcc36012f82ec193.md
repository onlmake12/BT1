### Title
RBF-Removed Transactions Permanently Lost When `limit_size` Evicts the Replacing Entry - (`File: tx-pool/src/process.rs`)

### Summary

In `tx-pool/src/process.rs`, the `submit_entry` function first removes conflicting transactions via `process_rbf` (storing them in `may_recovered_txs` for potential recovery), then inserts the new entry, and finally calls `limit_size`. If `limit_size` evicts the newly inserted entry, an early return via the `?` operator causes the `may_recovered_txs` recovery code to be skipped entirely. The RBF-removed transactions are permanently lost from the pool even though the replacing transaction also did not survive.

---

### Finding Description

In `submit_entry`, the sequence inside the `with_tx_pool_write_lock` closure is:

```rust
let may_recovered_txs = self.process_rbf(tx_pool, &entry, &conflicts);  // removes conflicting txs
let evicted = _submit_entry(tx_pool, status, entry.clone(), &self.callbacks)?;
// ...
tx_pool
    .limit_size(&self.callbacks, Some(&entry.proposal_short_id()))
    .map_or(Ok(()), Err)?;   // ← early return if new entry is evicted

if !may_recovered_txs.is_empty() {
    // ← NEVER REACHED when limit_size evicts the new entry
    tokio::spawn(async move {
        let mut queue = self_clone.verify_queue.write().await;
        for tx in may_recovered_txs {
            let _ = queue.add_tx(tx, false, None);
        }
    });
}
Ok(())
```

`process_rbf` removes the conflicting (RBF-replaced) transactions from the pool and returns them in `may_recovered_txs` so they can be re-queued if the replacing transaction does not ultimately stay in the pool. However, when `limit_size` determines that the newly inserted entry should be evicted (because it has the lowest fee rate among all remaining entries), it returns `Some(Reject::Full(...))`. The `.map_or(Ok(()), Err)?` converts this into an `Err` and the `?` operator causes the closure to return early — before the `tokio::spawn` recovery block at lines 154–163 is ever reached.

The result: the RBF-replaced transactions are gone, the replacing transaction is also gone, and no recovery occurs. [1](#0-0) 

The `limit_size` function itself correctly calls `callbacks.call_reject` for the evicted entry, but it has no knowledge of the `may_recovered_txs` that `process_rbf` produced. [2](#0-1) 

---

### Impact Explanation

An unprivileged transaction submitter can permanently evict one or more legitimate, fee-paying transactions from the mempool without those transactions being re-queued or re-broadcast. The attacker's own transaction is also evicted, but the net effect is that the victim's transactions disappear silently — they are not moved to `recent_reject`, not re-broadcast to peers, and not returned to the verify queue. From the victim's perspective the transaction simply vanishes from the pool with no error.

This enables:
- **Targeted transaction censorship**: an attacker can selectively remove a specific transaction (e.g., a time-sensitive channel-close or DAO withdrawal) from the pool.
- **Silent griefing**: the victim receives no rejection notification and may not resubmit in time.

---

### Likelihood Explanation

The preconditions are:
1. RBF is enabled (`min_rbf_rate > min_fee_rate` in config — this is the default-enabled path).
2. The pool is at or near its size limit (`max_tx_pool_size`).
3. The attacker crafts a replacing transaction T_rbf that:
   - Spends the same input as the target T_target (conflict).
   - Has a fee rate strictly higher than T_target (passes `check_rbf`).
   - Is physically larger than T_target, so that after the swap the pool exceeds `max_tx_pool_size`, causing `limit_size` to evict T_rbf (the lowest-fee-rate entry after T_target is removed).

All three conditions are observable and controllable by an unprivileged RPC caller who can inspect the pool via `get_raw_tx_pool`. The attack is cheap: the attacker's transaction is also evicted, so no fee is paid on-chain. [3](#0-2) 

---

### Recommendation

Move the `may_recovered_txs` recovery spawn **before** the `limit_size` call, or restructure the logic so that recovery is triggered regardless of whether `limit_size` evicts the new entry:

```rust
// Option A: recover before limit_size
if !may_recovered_txs.is_empty() {
    let self_clone = self.clone();
    tokio::spawn(async move { /* re-queue may_recovered_txs */ });
}
tx_pool
    .limit_size(&self.callbacks, Some(&entry.proposal_short_id()))
    .map_or(Ok(()), Err)?;
```

Alternatively, capture the `limit_size` result, perform recovery unconditionally, and then propagate the error:

```rust
let size_reject = tx_pool.limit_size(&self.callbacks, Some(&entry.proposal_short_id()));
if !may_recovered_txs.is_empty() {
    tokio::spawn(async move { /* re-queue */ });
}
size_reject.map_or(Ok(()), Err)?;
```

---

### Proof of Concept

1. Fill the pool to `max_tx_pool_size` with transactions having fee rates `[100, 90, 80, 70, 60]` shannons/byte.
2. The lowest-fee-rate transaction T_target (60 shannons/byte, 100 bytes) is in the pool.
3. Attacker submits T_rbf (65 shannons/byte, 300 bytes) that spends the same input as T_target.
   - `check_rbf` passes: 65 > 60.
   - `process_rbf` removes T_target; `may_recovered_txs = [T_target]`.
   - `_submit_entry` inserts T_rbf (+300 bytes, −100 bytes net = +200 bytes over limit).
   - `limit_size` finds T_rbf is now the lowest fee-rate entry; evicts it; returns `Some(Reject::Full(...))`.
   - `.map_or(Ok(()), Err)?` triggers early return.
   - Recovery spawn is never reached.
4. Pool now contains `[100, 90, 80, 70]` only. T_target is gone. T_rbf is gone. No rejection record for T_target. Victim's transaction silently disappeared. [1](#0-0) [4](#0-3)

    pub(crate) async fn submit_entry(
        &self,
        pre_resolve_tip: Byte32,
        entry: TxEntry,
        mut status: TxStatus,
    ) -> (Result<(), Reject>, Arc<Snapshot>) {
        let (ret, snapshot) = self
            .with_tx_pool_write_lock(move |tx_pool, snapshot| {
                // check_rbf must be invoked in `write` lock to avoid concurrent issues.
                let conflicts = if tx_pool.enable_rbf() {
                    tx_pool.check_rbf(&snapshot, &entry)?
                } else {
                    // RBF is disabled but we found conflicts, return error here
                    // after_process will put this tx into conflicts_pool
                    let conflicted_outpoint =
                        tx_pool.pool_map.find_conflict_outpoint(entry.transaction());
                    if let Some(outpoint) = conflicted_outpoint {
                        return Err(Reject::Resolve(OutPointError::Dead(outpoint)));
                    }
                    HashSet::new()
                };

                // if snapshot changed by context switch we need redo time_relative verify
                let tip_hash = snapshot.tip_hash();
                if pre_resolve_tip != tip_hash {
                    debug!(
                        "submit_entry {} context changed. previous:{} now:{}",
                        entry.proposal_short_id(),
                        pre_resolve_tip,
                        tip_hash
                    );

                    // destructuring assignments are not currently supported
                    status = check_rtx(tx_pool, &snapshot, &entry.rtx)?;

                    let tip_header = snapshot.tip_header();
                    let tx_env = status.with_env(tip_header);
                    time_relative_verify(snapshot, Arc::clone(&entry.rtx), tx_env)?;
                }

                let may_recovered_txs = self.process_rbf(tx_pool, &entry, &conflicts);
                let evicted = _submit_entry(tx_pool, status, entry.clone(), &self.callbacks)?;

                // in a corner case, a tx with lower fee rate may be rejected immediately
                // after inserting into pool, return proper reject error here
                for evict in evicted {
                    let reject = Reject::Invalidated(format!(
                        "invalidated by tx {}",
                        evict.transaction().hash()
                    ));
                    self.callbacks.call_reject(tx_pool, &evict, reject);
                }

                tx_pool.remove_conflict(&entry.proposal_short_id());
                tx_pool
                    .limit_size(&self.callbacks, Some(&entry.proposal_short_id()))
                    .map_or(Ok(()), Err)?;

                if !may_recovered_txs.is_empty() {
                    let self_clone = self.clone();
                    tokio::spawn(async move {
                        // push the recovered txs back to verify queue, so that they can be verified and submitted again
                        let mut queue = self_clone.verify_queue.write().await;
                        for tx in may_recovered_txs {
                            debug!("recover back: {:?}", tx.proposal_short_id());
                            let _ = queue.add_tx(tx, false, None);
                        }
                    });
                }
                Ok(())
            })
            .await;

        (ret, snapshot)
    }

    // Remove transactions from the pool until total size <= size_limit.
    // Return a `Reject` for current inserting entry if it's removed
    pub(crate) fn limit_size(
        &mut self,
        callbacks: &Callbacks,
        current_entry_id: Option<&ProposalShortId>,
    ) -> Option<Reject> {
        let mut ret = None;
        while self.pool_map.total_tx_size > self.config.max_tx_pool_size {
            let next_evict_entry = || {
                self.pool_map
                    .next_evict_entry(Status::Pending)
                    .or_else(|| self.pool_map.next_evict_entry(Status::Gap))
                    .or_else(|| self.pool_map.next_evict_entry(Status::Proposed))
            };

            if let Some(id) = next_evict_entry() {
                let removed = self.pool_map.remove_entry_and_descendants(&id);
                for entry in removed {
                    let tx_hash = entry.transaction().hash();
                    debug!(
                        "Removed by size limit {} timestamp({})",
                        tx_hash, entry.timestamp
                    );
                    let reject = Reject::Full(format!(
                        "the fee_rate for this transaction is: {}",
                        entry.fee_rate()
                    ));
                    if let Some(short_id) = current_entry_id
                        && entry.proposal_short_id() == *short_id
                    {
                        ret = Some(reject.clone());
                    }
                    callbacks.call_reject(self, &entry, reject);
                }
            }
        }
        self.pool_map.entries.shrink_to_fit();
        ret
