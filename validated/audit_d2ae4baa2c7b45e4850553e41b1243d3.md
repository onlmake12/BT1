### Title
Caller-Supplied `providerToCredit` Not Validated Against `req.provider` After Exclusivity Period — (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary
In `Echo.sol`, the `executeCallback` function accepts a caller-supplied `providerToCredit` address and credits all request fees to it. During the exclusivity period the contract enforces `providerToCredit == req.provider`, but after that window expires the check is dropped entirely. Any unprivileged actor can therefore call `executeCallback` with an arbitrary address as `providerToCredit`, redirecting the fee that belongs to the legitimate provider to a different account.

### Finding Description
`executeCallback` stores the assigned provider in the request struct at request time (`req.provider = provider`). When the callback is later executed, fees are credited to the caller-supplied `providerToCredit`:

```solidity
// Echo.sol lines 114-121
if (
    block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds
) {
    require(
        providerToCredit == req.provider,
        "Only assigned provider during exclusivity period"
    );
}
```

```solidity
// Echo.sol lines 161-162
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
```

Once `block.timestamp >= req.publishTime + exclusivityPeriodSeconds`, the identity check is skipped. The fee accumulation at line 161 uses `providerToCredit` — the caller-supplied value — rather than `req.provider` — the value that was validated and stored when the request was created. This is the direct analog to the reported pattern: a stored, correct identifier (`req.provider`) is available but the function instead uses a different, unvalidated identifier (`providerToCredit`) supplied by the caller.

### Impact Explanation
The legitimate provider (`req.provider`) receives zero fees for fulfilling a request whenever a third party races in after the exclusivity window. The attacker captures the full `req.fee + msg.value - pythFee` amount. Because `_state.providers[providerToCredit].accruedFeesInWei` is a plain mapping with no registration guard at the write site, the attacker only needs to pre-register as a provider (and optionally set themselves as their own `feeManager`) to later withdraw the stolen balance via `withdrawAsFeeManager`. The callback itself still executes correctly, so the requester is unaffected; only the provider's revenue is stolen.

### Likelihood Explanation
The attack requires no privileged access. Any EOA can:
1. Call `registerProvider` to become a registered provider.
2. Call `setFeeManager(self)` to enable withdrawal.
3. Monitor the mempool for pending `executeCallback` calls (or simply wait for the exclusivity period to expire on any open request).
4. Submit `executeCallback(self, sequenceNumber, updateData, priceIds)` with valid Pyth update data.

The exclusivity period is a configurable `uint32` set by the admin; if it is short (or zero), the window for exploitation opens immediately after request creation.

### Recommendation
After the exclusivity period, validate that `providerToCredit` equals `req.provider`, or remove the `providerToCredit` parameter entirely and always credit `req.provider`. If the design intent is to allow any provider to step in after the exclusivity window, add an explicit check that `providerToCredit` is a registered provider **and** emit a distinct event so the original provider can detect the substitution.

```solidity
// Suggested fix
if (providerToCredit != req.provider) {
    require(
        block.timestamp >= req.publishTime + _state.exclusivityPeriodSeconds,
        "Only assigned provider during exclusivity period"
    );
    require(
        _state.providers[providerToCredit].isRegistered,
        "providerToCredit must be a registered provider"
    );
}
```

### Proof of Concept
1. Alice (legitimate provider) is registered; Bob (attacker) calls `registerProvider` and `setFeeManager(Bob)`.
2. User calls `requestPriceUpdatesWithCallback(Alice, ...)` paying `fee = 1 ETH`. Request stored with `req.provider = Alice`.
3. Exclusivity period elapses (`block.timestamp >= req.publishTime + exclusivityPeriodSeconds`).
4. Bob calls `executeCallback(Bob, sequenceNumber, validUpdateData, priceIds)`.
5. Line 114–121: exclusivity check is skipped because the period has expired.
6. Line 161: `_state.providers[Bob].accruedFeesInWei += ~1 ETH`.
7. Bob calls `withdrawAsFeeManager(Bob, ~1 ETH)` and receives the funds.
8. Alice's `accruedFeesInWei` is unchanged; she earned nothing for the request she was assigned. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

    function withdrawAsFeeManager(
        address provider,
        uint128 amount
    ) external override {
        require(
            msg.sender == _state.providers[provider].feeManager,
            "Only fee manager"
        );
        require(
            _state.providers[provider].accruedFeesInWei >= amount,
            "Insufficient balance"
        );

        _state.providers[provider].accruedFeesInWei -= amount;

        (bool sent, ) = msg.sender.call{value: amount}("");
        require(sent, "Failed to send fees");

        emit FeesWithdrawn(msg.sender, amount);
    }
