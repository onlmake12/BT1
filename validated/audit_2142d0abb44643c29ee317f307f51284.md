### Title
Whitelisted Readers Permanently Denied Access to Age-Checked Price Functions via `this.` External Call Pattern — (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` in `Scheduler.sol` apply `onlyWhitelistedReader` to the outer call, but then delegate to `this.getPricesUnsafe(...)` and `this.getEmaPricesUnsafe(...)` using Solidity's `this.` external call syntax. This changes `msg.sender` inside the inner call from the original caller to the Scheduler contract itself. When `whitelistEnabled = true`, the inner `onlyWhitelistedReader` check sees the Scheduler contract as the caller — which is neither the subscription manager nor a whitelisted reader — and reverts with `Unauthorized`. Whitelisted readers are completely locked out of the age-checked price functions for any subscription with `whitelistEnabled = true`.

---

### Finding Description

`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` are guarded by `onlyWhitelistedReader` at the outer level, then internally call:

```solidity
prices = this.getPricesUnsafe(subscriptionId, priceIds);
```

and

```solidity
prices = this.getEmaPricesUnsafe(subscriptionId, priceIds);
``` [1](#0-0) 

In Solidity, `this.f()` is an **external** call. Inside `getPricesUnsafe`, `msg.sender` is therefore the Scheduler contract address, not the original caller.

`getPricesUnsafe` and `getEmaPricesUnsafe` are both decorated with `onlyWhitelistedReader(subscriptionId)`: [2](#0-1) 

The modifier logic is:

```solidity
modifier onlyWhitelistedReader(uint256 subscriptionId) {
    if (_state.subscriptionManager[subscriptionId] == msg.sender) { _; return; }
    if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) { _; return; }
    // linear scan of whitelist for msg.sender
    if (!isWhitelisted) { revert SchedulerErrors.Unauthorized(); }
    _;
}
``` [3](#0-2) 

When `whitelistEnabled = true`:
1. The Scheduler contract is not the subscription manager.
2. The Scheduler contract is not in `readerWhitelist`.
3. The modifier reverts with `Unauthorized`.

The outer `onlyWhitelistedReader` check on `getPricesNoOlderThan` is therefore **one-sided**: it validates the original caller correctly, but the inner `this.` call introduces a second, unintended access control check against the wrong identity (the Scheduler contract), which always fails.

This is the direct analog to the reported vulnerability: the external report's `_update()` only checked `_blacklist[_recipient]` but not `_blacklist[_sender]`; here, the whitelist check is applied correctly to the outer function but the `this.` delegation pattern causes the inner function's whitelist check to be applied to the wrong entity.

---

### Impact Explanation

Any subscription with `whitelistEnabled = true` is affected. Whitelisted readers — the intended consumers of these functions — receive `Unauthorized` reverts when calling `getPricesNoOlderThan` or `getEmaPricesNoOlderThan`. Only the subscription manager (who bypasses the whitelist check entirely) can successfully call these functions.

Protocols that integrate with Pyth Pulse and rely on `getPricesNoOlderThan` for staleness safety (the primary use case for this function) will be completely non-functional when the subscription uses a whitelist. The `getPricesUnsafe` variants remain accessible, but they provide no age guarantee, defeating the purpose of the age-checked API. [1](#0-0) 

---

### Likelihood Explanation

The whitelist feature (`whitelistEnabled = true`) is a first-class, documented feature of Pyth Pulse subscriptions. The `readerWhitelist` field is part of `SubscriptionParams` and is validated in `_validateSubscriptionParams`. Any subscription manager who enables the whitelist to restrict price data access to specific protocol contracts will trigger this bug for all their whitelisted readers. The bug is deterministic and reproducible on every call. [4](#0-3) 

---

### Recommendation

Replace the `this.getPricesUnsafe(...)` and `this.getEmaPricesUnsafe(...)` external calls with direct calls to the internal helper `_getPricesInternal`, which already exists and performs the same logic without the access control modifier:

```solidity
function getPricesNoOlderThan(...) external view override onlyWhitelistedReader(subscriptionId) ... {
    ...
    PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
    prices = new PythStructs.Price[](priceFeeds.length);
    for (uint i = 0; i < priceFeeds.length; i++) {
        prices[i] = priceFeeds[i].price;
    }
}
``` [5](#0-4) 

The same fix applies to `getEmaPricesNoOlderThan` calling `this.getEmaPricesUnsafe`.

    function _getPricesInternal(
        uint256 subscriptionId,
        bytes32[] calldata priceIds
    ) internal view returns (PythStructs.PriceFeed[] memory priceFeeds) {
        if (!_state.subscriptionParams[subscriptionId].isActive) {
            revert SchedulerErrors.InactiveSubscription();
        }

        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        // If no price IDs provided, return all price feeds for the subscription
        if (priceIds.length == 0) {
            PythStructs.PriceFeed[]
                memory allFeeds = new PythStructs.PriceFeed[](
                    params.priceIds.length
                );
            for (uint8 i = 0; i < params.priceIds.length; i++) {
                PythStructs.PriceFeed memory priceFeed = _state.priceUpdates[
                    subscriptionId
                ][params.priceIds[i]];
                // Check if the price feed exists (price ID is valid and has been updated)
                if (priceFeed.id == bytes32(0)) {
                    revert SchedulerErrors.InvalidPriceId(
                        params.priceIds[i],
                        bytes32(0)
                    );
                }
                allFeeds[i] = priceFeed;
            }
            return allFeeds;
        }

        // Return only the requested price feeds
        PythStructs.PriceFeed[]
            memory requestedFeeds = new PythStructs.PriceFeed[](
                priceIds.length
            );
        for (uint8 i = 0; i < priceIds.length; i++) {
            PythStructs.PriceFeed memory priceFeed = _state.priceUpdates[
                subscriptionId
            ][priceIds[i]];

            // Check if the price feed exists (price ID is valid and has been updated)
            if (priceFeed.id == bytes32(0)) {
                revert SchedulerErrors.InvalidPriceId(priceIds[i], bytes32(0));
            }
            requestedFeeds[i] = priceFeed;
        }
        return requestedFeeds;
    }

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
