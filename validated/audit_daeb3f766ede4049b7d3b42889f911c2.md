### Title
Unauthenticated `from` Field in Hole Punching Protocol Allows `pending_delivered` State Poisoning - (`network/src/protocols/hole_punching/component/connection_request.rs`)

---

### Summary

The CKB hole punching protocol stores NAT traversal state (`pending_delivered`) keyed by the `from` peer ID extracted from the message content. Because the `from` field is never verified against the actual network sender, any connected peer can spoof it to be any arbitrary `PeerId`. This allows an attacker to poison the `pending_delivered` map on a target node with attacker-controlled listen addresses, disrupting legitimate hole punching between two peers and causing the victim node to attempt TCP connections to attacker-controlled endpoints.

---

### Finding Description

The `HolePunching` protocol maintains two critical state maps:

- `pending_delivered: HashMap<PeerId, PendingDeliveredInfo>` — keyed by the `from` peer ID, stores the listen addresses of the original requester for later NAT traversal.
- `inflight_requests: HashMap<PeerId, u64>` — keyed by the `to` peer ID, tracks outgoing hole punch requests. [1](#0-0) 

When a node receives a `ConnectionRequest` and determines it is the intended `to` target, it calls `respond_delivered`, which inserts into `pending_delivered` using `from_peer_id` taken directly from `content.from` — the attacker-controlled message field: [2](#0-1) 

There is no check that `content.from` matches the actual network sender (`self.peer`). Because `ConnectionRequest` messages are relayed through intermediate nodes, `self.peer` is legitimately a relay, not the originator — but this also means the `from` field is entirely unverified and can be set to any `PeerId` by any peer in the network.

Later, when a `ConnectionSync` arrives, the target node looks up `pending_delivered` by `content.from` to retrieve listen addresses for NAT traversal: [3](#0-2) 

If an attacker has already poisoned `pending_delivered[Peer A]` with attacker-controlled addresses, the victim node will attempt NAT traversal to those addresses instead of Peer A's real addresses.

Additionally, `respond_delivered` contains a rate-limit guard that silently ignores new `ConnectionRequest` messages for the same `from` peer within `HOLE_PUNCHING_INTERVAL` (2 minutes): [4](#0-3) 

This means a poisoned entry blocks the legitimate peer from updating the state for 2 minutes per attack cycle.

---

### Impact Explanation

**Attack scenario:**

1. Attacker (Peer X) sends a `ConnectionRequest` with `from = Peer A` (spoofed), `to = Peer B` (victim), and `listen_addrs = [attacker_ip:port]`.
2. Peer B receives the request, passes the `from` field check, and calls `respond_delivered(Peer A, Peer B, [attacker_ip:port])`.
3. Peer B inserts `pending_delivered[Peer A] = ([attacker_ip:port], now)` and sends `ConnectionRequestDelivered` back toward the spoofed Peer A.
4. When Peer A legitimately sends a `ConnectionRequest` to Peer B, Peer B ignores it for 2 minutes (rate-limit guard fires because the entry already exists).
5. If Peer A's legitimate request eventually succeeds and a `ConnectionSync` arrives, Peer B looks up `pending_delivered[Peer A]` and attempts TCP NAT traversal to the attacker's address — not Peer A's real address.

**Consequences:**
- **Hole punching DoS**: Any connected peer can prevent two specific peers from establishing a hole-punched connection, indefinitely renewable every 2 minutes.
- **SSRF / unsolicited outbound TCP**: The victim node initiates TCP connections to arbitrary attacker-controlled IP:port combinations.
- **State pollution**: `pending_delivered` accumulates fake entries, consuming memory.

---

### Likelihood Explanation

Any peer with an active P2P session to a CKB node can send a `ConnectionRequest` message. The `HolePunching` protocol is a standard network protocol enabled by default. No special privileges, keys, or majority hash power are required. The attacker only needs to know the `PeerId` of the two peers they wish to disrupt (which is public information in the peer store / discovery protocol). The attack is trivially repeatable every 2 minutes.

---

### Recommendation

1. **Authenticate the `from` field**: When a node is the final `to` target and receives a `ConnectionRequest` directly (not forwarded), verify that `content.from` matches the actual sender's `PeerId` by looking it up via `self.peer` in the peer registry. For forwarded messages, consider requiring a cryptographic signature from the originating peer over the `from` field.

2. **Key `pending_delivered` by actual sender session**: For the non-forwarded case (direct connection), key the state by the verified sender identity rather than the message-supplied `from` field.

3. **Limit `pending_delivered` map size**: Add a hard cap on the number of entries to prevent memory exhaustion from state pollution.

---

### Proof of Concept

```
Attacker (Peer X) → [any relay] → Peer B:
  ConnectionRequest {
    from: <Peer A's PeerId>,   // spoofed
    to:   <Peer B's PeerId>,
    listen_addrs: [attacker_ip:1234],
    max_hops: 6,
    route: [],
  }

Peer B executes respond_delivered(Peer A, Peer B, [attacker_ip:1234]):
  pending_delivered.insert(Peer A, ([attacker_ip:1234], now))
  // sends ConnectionRequestDelivered back toward "Peer A" (which Peer A ignores)

Peer A (legitimate) → [relay] → Peer B:
  ConnectionRequest { from: Peer A, to: Peer B, listen_addrs: [real_addr] }

Peer B: pending_delivered.get(Peer A) exists and is < 2 min old → Ignore

// 2 minutes later, Peer A retries. Even if it succeeds, Peer B's
// pending_delivered[Peer A] still holds attacker_ip:1234 until overwritten.

Peer A → Peer B: ConnectionSync { from: Peer A, to: Peer B }
Peer B: pending_delivered.get(Peer A) = [attacker_ip:1234]
        → try_nat_traversal(bind_addr, attacker_ip:1234)  // TCP to attacker
        // Hole punch with real Peer A fails

    pub(crate) async fn execute(mut self) -> Status {
        let content = match RequestContent::try_from(&self.message) {
            Ok(content) => content,
            Err(status) => return status,
        };
        if content.listen_addrs.len() > ADDRS_COUNT_LIMIT || content.listen_addrs.is_empty() {
            return StatusCode::InvalidListenAddrLen
                .with_context("the listen address count is too large or empty");
        }

        if content.max_hops > MAX_HOPS {
            return StatusCode::InvalidMaxTTL.into();
        }
        if content.route.len() > MAX_HOPS as usize {
            return StatusCode::InvalidRoute.with_context("the route length is too long");
        }

        let self_peer_id = self.protocol.network_state.local_peer_id();
        if content.route.contains(self_peer_id) {
            return StatusCode::Ignore.with_context("the message is passed, ignore it");
        }

        if self
            .protocol
            .forward_rate_limiter
            .check_key(&(content.from.clone(), content.to.clone(), self.msg_item_id))
            .is_err()
        {
            debug!(
                "from: {}, to {}, item_name: {}, rate limit is reached",
                content.from, content.to, "ConnectionRequest",
            );
            return StatusCode::TooManyRequests.with_context("ConnectionRequest");
        }

        if self_peer_id == &content.to {
            self.respond_delivered(content.from, &content.to, content.listen_addrs)
                .await
        } else if content.max_hops == 0u8 {
            StatusCode::ReachedMaxHops.into()
        } else {
            self.forward_message(self_peer_id, &content.to).await
        }
    }

    pub(crate) async fn execute(self) -> Status {
        let content = match SyncContent::try_from(&self.message) {
            Ok(content) => content,
            Err(status) => return status,
        };

        if content.route.len() > MAX_HOPS as usize {
            return StatusCode::InvalidRoute.with_context("the route length is too long");
        }
        if self
            .protocol
            .forward_rate_limiter
            .check_key(&(content.from.clone(), content.to.clone(), self.msg_item_id))
            .is_err()
        {
            debug!(
                "from: {}, to {}, item_name: {}, rate limit is reached",
                content.from, content.to, "ConnectionSync",
            );
            return StatusCode::TooManyRequests.with_context("ConnectionSync");
        }

        match content.route.last() {
            Some(next_peer_id) => self.forward_sync(next_peer_id).await,
            None => {
                let self_peer_id = self.protocol.network_state.local_peer_id();
                if self_peer_id != &content.to {
                    // forward the message to the `to` peer
                    self.forward_sync(&content.to).await
                } else {
                    // Current node should be the `to` target.
                    if let Some(metrics) = ckb_metrics::handle() {
                        metrics.ckb_hole_punching_passive_count.inc();
                    }

                    let listens_info = self
                        .protocol
                        .pending_delivered
                        .get(&content.from)
                        .map(|info| info.0.clone());

                    match listens_info {
                        Some(listens) => {
                            let tasks = listens
                                .into_iter()
                                .map(|listen_addr| {
                                    Box::pin(try_nat_traversal(self.bind_addr, listen_addr))
                                })
                                .collect::<Vec<_>>();

                            if tasks.is_empty() {
                                return StatusCode::Ignore.with_context("no valid listen address");
                            }

                            debug!(
                                "current peer is the target peer {}, start NAT traversal",
                                content.to
                            );

                            match self
                                .protocol
                                .network_state
                                .config
                                .listen_addresses
                                .first()
                                .cloned()
                            {
                                Some(listen_addr) => {
                                    let control: ServiceAsyncControl = self.p2p_control.clone();
                                    runtime::spawn(async move {
                                        if let Ok(((stream, addr), _)) = select_ok(tasks).await {
                                            debug!("NAT traversal success, addr: {:?}", addr);
                                            if let Some(metrics) = ckb_metrics::handle() {
                                                metrics
                                                    .ckb_hole_punching_passive_success_count
                                                    .inc();
                                            }

                                            let _ignore = control
                                                .raw_session(
                                                    stream,
                                                    addr,
                                                    RawSessionInfo::inbound(listen_addr),
                                                )
                                                .await;
                                        }
                                    });
                                    Status::ok()
                                }
                                None => {
                                    StatusCode::Ignore.with_context("no listen address configured")
                                }
                            }
                        }
                        None => StatusCode::Ignore
                            .with_context("the from peer id is not in the pending list"),
                    }
                }
            }
        }
    }
