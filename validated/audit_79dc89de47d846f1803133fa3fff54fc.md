### Title
Insufficient Price ID Validation in `executeCallback` Allows Provider to Fulfill with Wrong Price Feed - (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary

`Echo.sol`'s `executeCallback` validates price IDs by comparing only the first 8 bytes (`bytes8` prefix) of each price ID rather than the full 32-byte identifier. A registered provider can fulfill a callback with a different price feed that shares the same 8-byte prefix as the originally requested feed, causing the consumer contract to receive and act on incorrect price data.

### Finding Description

In `requestPriceUpdatesWithCallback`, only the first 8 bytes of each requested price ID are stored:

```solidity
req.priceIdPrefixes = new bytes8[](priceIds.length);
for (uint8 i = 0; i < priceIds.length; i++) {
    bytes32 priceId = priceIds[i];
    bytes8 prefix;
    assembly {
        prefix := priceId
    }
    req.priceIdPrefixes[i] = prefix;
}
```

In `executeCallback`, the validation only compares these 8-byte prefixes:

```solidity
for (uint8 i = 0; i < req.priceIdPrefixes.length; i++) {
    bytes32 priceId = priceIds[i];
    bytes8 prefix;
    assembly {
        prefix := priceId
    }
    if (prefix != req.priceIdPrefixes[i]) {
        revert InvalidPriceIds(priceIds[i], req.priceIdPrefixes[i]);
    }
}
```

Since Pyth price IDs are 32 bytes, any price feed whose first 8 bytes match the requested feed's prefix will pass this check — even if the remaining 24 bytes differ entirely, making it a completely different price feed. [1](#0-0) [2](#0-1) 

### Impact Explanation

A registered provider can call `executeCallback` with `priceIds` that share only the first 8 bytes with the originally requested price IDs. The `parsePriceFeedUpdates` call on the Pyth oracle will then return data for the wrong price feed. The consumer's `_echoCallback` receives and processes incorrect price data — for example, a request for ETH/USD could be fulfilled with data from a different asset whose feed ID happens to share the same 8-byte prefix. This breaks the core correctness guarantee of the Echo protocol: that callbacks deliver the price data the requester asked for.

### Likelihood Explanation

Any registered provider can call `executeCallback`. Provider registration appears to be permissionless (any address can call `registerProvider`). The attack requires finding or constructing a Pyth price feed ID that shares the first 8 bytes with the target feed — with 2^64 possible prefix values and a growing set of Pyth feeds, collisions are unlikely to be accidental but a malicious provider can deliberately select a feed with a matching prefix. The Immunefi scope explicitly includes "Entropy user/provider" as a valid attacker role, and the Echo provider role is analogous.

### Recommendation

Replace the 8-byte prefix comparison with a full 32-byte price ID comparison:

```solidity
// In requestPriceUpdatesWithCallback: store full price IDs
req.priceIds = priceIds; // bytes32[] instead of bytes8[]

// In executeCallback: compare full price IDs
for (uint8 i = 0; i < req.priceIds.length; i++) {
    if (priceIds[i] != req.priceIds[i]) {
        revert InvalidPriceIds(priceIds[i], req.priceIds[i]);
    }
}
```

This mirrors the correct AND-style validation pattern: both the stored ID and the provided ID must match exactly, not just partially.

### Proof of Concept

1. User calls `requestPriceUpdatesWithCallback` requesting price feed `0xABCD000000000000XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX` (ETH/USD).
2. The contract stores only the prefix `0xABCD000000000000` in `req.priceIdPrefixes[0]`.
3. A malicious provider calls `executeCallback` with `priceIds[0] = 0xABCD000000000000YYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYY` (a different asset).
4. The prefix check passes: `bytes8(0xABCD000000000000) == bytes8(0xABCD000000000000)`.
5. `parsePriceFeedUpdates` is called with the wrong price ID, returning data for the wrong asset.
6. The consumer's `_echoCallback` receives incorrect price data and acts on it. [3](#0-2)

    function executeCallback(
        address providerToCredit,
        uint64 sequenceNumber,
        bytes[] calldata updateData,
        bytes32[] calldata priceIds
    ) external payable override {
        Request storage req = findActiveRequest(sequenceNumber);

        // Check provider exclusivity using configurable period
        if (
            block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds
        ) {
            require(
                providerToCredit == req.provider,
                "Only assigned provider during exclusivity period"
            );
        }

        // Verify priceIds match
        require(
            priceIds.length == req.priceIdPrefixes.length,
            "Price IDs length mismatch"
        );
        for (uint8 i = 0; i < req.priceIdPrefixes.length; i++) {
            // Extract first 8 bytes of the provided price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }

        }
