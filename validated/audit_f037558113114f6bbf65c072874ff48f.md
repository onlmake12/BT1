### Title
Stale Price Returned Without Staleness Check in `PythAggregatorV3` Chainlink Adapter - (File: `target_chains/ethereum/sdk/solidity/PythAggregatorV3.sol`)

---

### Summary

`PythAggregatorV3` is Pyth's official Chainlink `AggregatorV3Interface` adapter, intended as a drop-in replacement for Chainlink price feeds. Every price-returning function in this contract — `latestAnswer()`, `latestRoundData()`, `getRoundData()`, and `decimals()` — calls `pyth.getPriceUnsafe()`, which explicitly performs **no staleness check** and may return a price from arbitrarily far in the past. Additionally, `latestRoundData()` sets `answeredInRound == roundId` unconditionally, causing the standard Chainlink staleness guard (`answeredInRound >= roundId`) to always pass trivially, silently masking stale data.

---

### Finding Description

`PythAggregatorV3.sol` implements the Chainlink `AggregatorV3Interface` on top of Pyth. All four price-reading functions delegate to `pyth.getPriceUnsafe(priceId)`:

```solidity
// latestAnswer — no staleness check
function latestAnswer() public view virtual returns (int256) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    return int256(price.price);
}

// latestRoundData — no staleness check
function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    roundId = uint80(price.publishTime);
    return (roundId, int256(price.price), price.publishTime, price.publishTime, roundId);
}
```

`IPyth.getPriceUnsafe` is documented as:

> "This function returns the most recent price update in this contract without any recency checks. This function is unsafe as the returned price update may be arbitrarily far in the past."

The safe alternative, `getPriceNoOlderThan(id, age)`, is available but never used in this adapter.

A second compounding issue: `latestRoundData()` sets both `roundId` and `answeredInRound` to `uint80(price.publishTime)`. Integrating protocols that follow the standard Chainlink staleness pattern:

```solidity
require(answeredInRound >= roundId, "Stale price");
require(updatedAt != 0, "Round not complete");
```

will always pass both checks because `answeredInRound == roundId` by construction, and `updatedAt = price.publishTime` is always non-zero for any stored feed. The staleness guard is rendered inert. [1](#0-0) [2](#0-1) [3](#0-2) 

---

### Impact Explanation

Any DeFi protocol (lending, derivatives, liquidation engine) that deploys `PythAggregatorV3` as a Chainlink adapter and calls `latestAnswer()` or `latestRoundData()` will silently consume arbitrarily stale prices. Concrete impacts:

- **Lending/liquidation protocols**: A stale inflated price prevents legitimate liquidations; a stale deflated price triggers wrongful liquidations.
- **Derivatives/perps**: Stale prices allow adversarial selection — an attacker waits for a favorable stale price and opens/closes positions before the feed is updated.
- **Flash-loan exploitation**: An attacker can open and close a position in the same block using a stale price that diverges from the true market price, extracting value with no inventory risk.

The Pyth documentation explicitly warns that `getPriceUnsafe` must not be used without a manual `publishTime` check, yet the adapter exposes it through the standard Chainlink interface where callers have no reason to expect unsafe behavior. [4](#0-3) [5](#0-4) 

---

### Likelihood Explanation

- `PythAggregatorV3` is the official Pyth-provided migration path for protocols moving from Chainlink, documented in the developer hub under "Migrate an App to Pyth / Chainlink."
- Pyth uses a pull-oracle model: prices are only updated when someone calls `updatePriceFeeds`. During periods of low activity, network congestion, or outside market hours for equity feeds, the on-chain price can be hours or days old.
- Protocols that integrate this adapter and apply the standard Chainlink staleness check (`answeredInRound >= roundId`) receive a false sense of security — the check always passes.
- The attack requires no privileged access: any unprivileged transaction sender can exploit a protocol consuming stale prices. [6](#0-5) [7](#0-6) 

---

### Recommendation

Replace all `getPriceUnsafe` calls in `PythAggregatorV3` with `getPriceNoOlderThan(priceId, maxAge)`, where `maxAge` is a configurable constructor parameter (e.g., defaulting to `pyth.getValidTimePeriod()`):

```solidity
uint256 public maxPriceAge;

constructor(address _pyth, bytes32 _priceId, uint256 _maxPriceAge) {
    priceId = _priceId;
    pyth = IPyth(_pyth);
    maxPriceAge = _maxPriceAge;
}

function latestAnswer() public view virtual returns (int256) {
    PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxPriceAge);
    return int256(price.price);
}

function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxPriceAge);
    roundId = uint80(price.publishTime);
    return (roundId, int256(price.price), price.publishTime, price.publishTime, roundId);
}
```

This mirrors the recommendation in the external report: use the checked API and revert with a clear error when the price is stale. [5](#0-4)

### Proof of Concept

1. Deploy `PythAggregatorV3` pointing to a Pyth contract and a BTC/USD price feed.
2. Call `updatePriceFeeds` once to store a price at time `T`.
3. Advance the chain clock by 24 hours (no further `updatePriceFeeds` calls).
4. Call `latestRoundData()` on the adapter — it returns the 24-hour-old price with no revert.
5. The returned tuple has `answeredInRound == roundId` (both equal `uint80(T)`), so the standard Chainlink staleness check `require(answeredInRound >= roundId)` passes.
6. A lending protocol consuming this adapter will price collateral at the 24-hour-old value, enabling under-collateralized borrowing or wrongful liquidation.

```solidity
// Attacker PoC (simplified)
(uint80 roundId,int256 answer,,uint256 updatedAt,uint80 answeredInRound)
    = PythAggregatorV3(adapter).latestRoundData();

// Standard Chainlink staleness check — always passes in PythAggregatorV3
require(answeredInRound >= roundId, "Stale");   // roundId == answeredInRound always
require(updatedAt != 0, "Round not complete");  // publishTime always != 0

// answer is now used — but it may be arbitrarily old
``` [8](#0-7)

## Deploy Adapter Contract

First, deploy the `PythAggregatorV3` contract from `@pythnetwork/pyth-sdk-solidity` as a replacement for your application's Chainlink price feeds.

The adapter contract resolves this discrepancy by wrapping a single Pyth price feed.
Users should deploy an instance of this adapter for every required price feed, then point their existing app to the addresses of the deployed adapter contracts.
