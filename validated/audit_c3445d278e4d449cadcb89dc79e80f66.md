### Title
Unbounded `ban_time` in `set_ban` RPC Enables Permanent Peer Banning or Silent Ban Bypass via Integer Overflow — (File: `rpc/src/module/net.rs`)

---

### Summary

The `set_ban` RPC method accepts a `ban_time` parameter of type `Uint64` (milliseconds) with no upper-bound validation. Depending on the `absolute` flag, this allows either a permanent peer ban (when `absolute=true`, `ban_time = u64::MAX`) or a silent ban bypass via integer overflow (when `absolute=false`, `ban_time` is large enough to overflow `u64` addition), making the ban immediately ineffective.

---

### Finding Description

In `rpc/src/module/net.rs`, the `set_ban` implementation computes `ban_until` as follows:

```rust
let ban_until = if absolute.unwrap_or(false) {
    ban_time.unwrap_or_default().into()          // raw u64 timestamp
} else {
    unix_time_as_millis()
        + ban_time
            .unwrap_or_else(|| DEFAULT_BAN_DURATION.into())
            .value()                              // no overflow guard
};
self.network_controller
    .ban(ip_network, ban_until, reason.unwrap_or_default());
``` [1](#0-0) 

There is no upper-bound check on `ban_time` anywhere in the function.

**Path A — Permanent ban (`absolute=true`, `ban_time = u64::MAX`):**
`ban_until` is set to `u64::MAX` milliseconds since epoch (~year 584,542,046). The peer/subnet is effectively permanently banned. No validation prevents this.

**Path B — Silent ban bypass (`absolute=false`, large `ban_time`):**
`unix_time_as_millis()` is currently ~`1.75 × 10¹²`. Adding a value close to `u64::MAX` wraps around (Rust release-mode wrapping arithmetic), producing a `ban_until` in the distant past. The ban is stored but immediately considered expired, silently making the ban operation a no-op.

The `DEFAULT_BAN_DURATION` constant is 24 hours and is only used as a fallback when `ban_time` is `null`. [2](#0-1) 

---

### Impact Explanation

- **Path A**: A local RPC caller sets `ban_time = 0xffffffffffffffff` with `absolute=true`. Any IP or subnet (including critical bootstrap peers or honest validators) is permanently banned from the node's P2P layer. The node loses connectivity to those peers indefinitely with no automatic recovery.
- **Path B**: A local RPC caller sets `ban_time` to a value that overflows `u64` when added to the current timestamp. The ban is silently stored with an already-expired `ban_until`, making the ban operation completely ineffective. An operator who believes they have banned a malicious peer has not actually done so.

Both paths affect the node's P2P connectivity state retroactively and without any protocol-level safeguard.

---

### Likelihood Explanation

The `set_ban` RPC is part of the Net module, enabled by default. CKB's RPC server binds to localhost without authentication in its default configuration. Any local process — including a malicious script, compromised local service, or a user with shell access — can call this endpoint. The `ban_time` field is a plain `Uint64` with no documented or enforced maximum. The overflow path (Path B) is reachable with any `ban_time > (u64::MAX - unix_time_as_millis())`, which is a specific but easily computed value.

---

### Recommendation

1. **Add an upper bound** on `ban_time`. For example, cap it at 10 years in milliseconds (`315_360_000_000`).
2. **Add overflow-safe arithmetic** for the relative case: use `unix_time_as_millis().saturating_add(ban_time.value())` and then clamp the result to a maximum allowed `ban_until`.
3. **Add a `require`-style check** that rejects `ban_time` values exceeding the cap with a clear `InvalidParams` error.

Example fix for the relative path:

```rust
const MAX_BAN_DURATION_MS: u64 = 10 * 365 * 24 * 3600 * 1000; // 10 years

let duration = ban_time
    .unwrap_or_else(|| DEFAULT_BAN_DURATION.into())
    .value()
    .min(MAX_BAN_DURATION_MS);

let ban_until = unix_time_as_millis().saturating_add(duration);
```

### Proof of Concept

**Path A — Permanent ban:**
```json
{
  "id": 1,
  "jsonrpc": "2.0",
  "method": "set_ban",
  "params": [
    "192.168.1.1",
    "insert",
    "0xffffffffffffffff",
    true,
    "permanent ban PoC"
  ]
}
```
Result: peer `192.168.1.1` is banned until timestamp `u64::MAX` ms — effectively forever. No error is returned.

**Path B — Silent ban bypass (overflow):**
Let `T = unix_time_as_millis()` ≈ `0x197_3B5A_C000` (current epoch ms). Set `ban_time = u64::MAX - T + 1 = 0xFFFFFE68_C4A54000` (approximately). With `absolute=false`:

```
ban_until = T + (u64::MAX - T + 1) = u64::MAX + 1 → wraps to 0
```

The stored `ban_until` is `0` (or a very small past timestamp). The ban is immediately expired. The peer is not actually banned.

```json
{
  "id": 1,
  "jsonrpc": "2.0",
  "method": "set_ban",
  "params": [
    "10.0.0.1",
    "insert",
    "0xFFFFFE68C4A54000",
    false,
    "overflow PoC"
  ]
}
```

The RPC returns `null` (success), but the peer `10.0.0.1` remains unbanned because `ban_until` wrapped to a past timestamp. [3](#0-2)

    fn set_ban(
        &self,
        address: String,
        command: String,
        ban_time: Option<Timestamp>,
        absolute: Option<bool>,
        reason: Option<String>,
    ) -> Result<()> {
        let ip_network = address.parse().map_err(|_| {
            RPCError::invalid_params(format!(
                "Expected `params[0]` to be a valid IP address, got {address}"
            ))
        })?;

        match command.as_ref() {
            "insert" => {
                let ban_until = if absolute.unwrap_or(false) {
                    ban_time.unwrap_or_default().into()
                } else {
                    unix_time_as_millis()
                        + ban_time
                            .unwrap_or_else(|| DEFAULT_BAN_DURATION.into())
                            .value()
                };
                self.network_controller
                    .ban(ip_network, ban_until, reason.unwrap_or_default());
                Ok(())
            }
            "delete" => {
                self.network_controller.unban(&ip_network);
                Ok(())
            }
            _ => Err(RPCError::invalid_params(format!(
                "Expected `params[1]` to be in the list [insert, delete], got {address}"
            ))),
        }
    }
```
