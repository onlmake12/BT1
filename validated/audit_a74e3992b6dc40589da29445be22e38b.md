### Title
Missing `is_main_chain` Guard in `verify_transaction_proof` Allows Fork-Block Proof Acceptance — (`rpc/src/module/chain.rs`)

---

### Summary

`verify_transaction_proof` calls `snapshot.get_block()` on the caller-supplied `block_hash` without first checking `snapshot.is_main_chain()`. Because `get_block()` retrieves any block stored in the database — including fork (non-canonical) blocks — a valid Merkle proof for a transaction in a fork block passes verification and returns `Ok(vec![tx_hash])`. Every other block-hash-accepting RPC method in the same file performs the `is_main_chain` guard; this one does not.

---

### Finding Description

`verify_transaction_proof` is implemented as follows: [1](#0-0) 

```rust
fn verify_transaction_proof(&self, tx_proof: TransactionProof) -> Result<Vec<H256>> {
    let snapshot = self.shared.snapshot();
    snapshot
        .get_block(&(&tx_proof.block_hash).into())   // ← no is_main_chain check
        .ok_or_else(|| { ... })
        .and_then(|block| { /* Merkle proof verification only */ })
}
```

`ChainStore::get_block()` retrieves a block purely by hash from `COLUMN_BLOCK_HEADER` / `COLUMN_BLOCK_BODY`: [2](#0-1) 

It does not consult `COLUMN_INDEX`, which is the sole source of truth for canonical-chain membership. `is_main_chain()` is defined as: [3](#0-2) 

Fork blocks are written to `COLUMN_BLOCK_HEADER` and `COLUMN_BLOCK_BODY` during reorg processing but are **not** inserted into `COLUMN_INDEX`, so `get_block()` returns them while `is_main_chain()` returns `false` for them.

Every other block-hash-accepting RPC method in the same file guards with `is_main_chain` before proceeding:

- `get_block_by_hash` [4](#0-3) 
- `get_header` [5](#0-4) 
- `get_block_median_time` [6](#0-5) 
- `get_fork_block` (inverted check — returns `None` for main-chain blocks) [7](#0-6) 

`verify_transaction_proof` is the only block-hash-accepting method that omits this guard entirely. The same defect exists in `verify_transaction_and_witness_proof`: [8](#0-7) 

---

### Impact Explanation

An unprivileged caller can:

1. Obtain a fork block hash via `get_fork_block` (or P2P relay).
2. Compute a valid `TransactionProof` for any transaction in that fork block — `witnesses_root` and the CBMT Merkle proof are deterministic from the block body, which is publicly available.
3. Call `verify_transaction_proof` with the crafted proof.
4. Receive `Ok([tx_hash])` — a successful verification — for a transaction that was **never committed to the canonical chain**.

Applications (exchanges, DeFi bridges) that use `verify_transaction_proof` as a deposit-confirmation oracle will accept the fraudulent proof as evidence of canonical inclusion, enabling double-spend or phantom-deposit attacks.

---

### Likelihood Explanation

Fork blocks are a routine occurrence on any live CKB node; no special mining power or Sybil capability is required to trigger the precondition. The attacker only needs RPC access (default port 8114, often exposed) and knowledge of a fork block hash. Constructing the proof requires no privileged information — all inputs are derivable from the publicly accessible fork block. The attack is fully local-testable in a two-node devnet.

---

### Recommendation

Add an `is_main_chain` guard at the top of both `verify_transaction_proof` and `verify_transaction_and_witness_proof`, mirroring the pattern used by `get_block_by_hash`:

```rust
fn verify_transaction_proof(&self, tx_proof: TransactionProof) -> Result<Vec<H256>> {
    let snapshot = self.shared.snapshot();
    let block_hash: packed::Byte32 = (&tx_proof.block_hash).into();

    // ADD THIS GUARD:
    if !snapshot.is_main_chain(&block_hash) {
        return Err(RPCError::invalid_params(format!(
            "Block {:#x} is not on the main chain",
            tx_proof.block_hash
        )));
    }

    snapshot
        .get_block(&block_hash)
        ...
}
```

Apply the identical fix to `verify_transaction_and_witness_proof`.

### Proof of Concept

```
# 1. Start a two-node devnet; mine a common ancestor.
# 2. Disconnect nodes; mine one block on each (creating a fork).
# 3. Reconnect — one block becomes canonical, the other a fork block.
# 4. On the node that holds the fork block:

FORK_HASH=$(ckb-cli rpc get_fork_block --hash <fork_block_hash> | jq -r '.block_hash')

# 5. Build a TransactionProof for a tx in the fork block
#    (witnesses_root and CBMT proof are computable from the fork block body).

# 6. Call verify_transaction_proof:
curl -X POST http://127.0.0.1:8114 -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"verify_transaction_proof","params":[{
        "block_hash": "<fork_block_hash>",
        "witnesses_root": "<fork_block_witnesses_root>",
        "proof": {"indices":["0x0"],"lemmas":[]}
      }],"id":1}'

# Expected (buggy) result: {"result":["<tx_hash>"]}
# Confirm fork status: get_block_by_number returns a DIFFERENT block at that height.
```

    fn get_block(&self, h: &packed::Byte32) -> Option<BlockView> {
        let header = self.get_block_header(h)?;
        if let Some(freezer) = self.freezer()
            && header.number() > 0
            && header.number() < freezer.number()
        {
            let raw_block = freezer.retrieve(header.number()).expect("block frozen")?;
            let raw_block_reader =
                packed::BlockReader::from_compatible_slice(&raw_block).expect("checked data");
            if raw_block_reader.calc_header_hash().as_slice() == h.as_slice() {
                return Some(raw_block_reader.to_entity().into_view());
            }
        }
        let body = self.get_block_body(h);
        let uncles = self
            .get_block_uncles(h)
            .expect("block uncles must be stored");
        let proposals = self
            .get_block_proposal_txs_ids(h)
            .expect("block proposal_ids must be stored");
        let extension_opt = self.get_block_extension(h);

        let block = if let Some(extension) = extension_opt {
            BlockView::new_unchecked_with_extension(header, uncles, body, proposals, extension)
        } else {
            BlockView::new_unchecked(header, uncles, body, proposals)
        };
        Some(block)
    }
