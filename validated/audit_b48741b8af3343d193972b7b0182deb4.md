### Title
`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` Always Revert for Whitelisted Subscriptions Due to Broken Internal `this.method()` Call Through `onlyWhitelistedReader` - (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` in `Scheduler.sol` delegate to `this.getPricesUnsafe(...)` and `this.getEmaPricesUnsafe(...)` respectively via external `this.method()` calls. Because `this.method()` in Solidity is an external call, `msg.sender` inside the callee becomes `address(this)` (the Scheduler contract itself). The `onlyWhitelistedReader` modifier applied to those callees then checks whether the Scheduler contract's own address is the subscription manager or is in the whitelist — it is neither — causing an unconditional `Unauthorized` revert for every subscription with `whitelistEnabled = true`. The two "safe" age-checked read functions are therefore completely non-functional for any whitelisted subscription.

---

### Finding Description

**Root cause — `onlyWhitelistedReader` modifier:**

```solidity
modifier onlyWhitelistedReader(uint256 subscriptionId) {
    // Manager is always allowed
    if (_state.subscriptionManager[subscriptionId] == msg.sender) { _; return; }

    // If whitelist is not used, allow any reader
    if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) { _; return; }

    // Check if caller is in whitelist
    ...
    if (!isWhitelisted) { revert SchedulerErrors.Unauthorized(); }
    _;
}
``` [1](#0-0) 

**Broken delegation in `getPricesNoOlderThan`:**

```solidity
function getPricesNoOlderThan(...) external view override
    onlyWhitelistedReader(subscriptionId)   // ← checks original caller (correct)
    returns (PythStructs.Price[] memory prices)
{
    ...
    prices = this.getPricesUnsafe(subscriptionId, priceIds);  // ← external call: msg.sender = address(this)
}
``` [2](#0-1) 

**Same pattern in `getEmaPricesNoOlderThan`:** [3](#0-2) 

**Execution trace when `whitelistEnabled = true`:**

1. Whitelisted reader calls `getPricesNoOlderThan(subscriptionId, priceIds, age)`.
2. `onlyWhitelistedReader` runs with `msg.sender = reader` → reader is in whitelist → passes.
3. Age check passes.
4. `this.getPricesUnsafe(subscriptionId, priceIds)` is dispatched as an **external call**; `msg.sender` inside `getPricesUnsafe` is now `address(this)` (the Scheduler proxy/implementation).
5. `onlyWhitelistedReader` runs again with `msg.sender = address(this)`:
   - `address(this) != subscriptionManager` → false.
   - `whitelistEnabled == true` → whitelist path taken.
   - `address(this)` is not in `readerWhitelist` → `revert SchedulerErrors.Unauthorized()`.
6. The entire transaction reverts.

The same path applies to `getEmaPricesNoOlderThan` → `this.getEmaPricesUnsafe`.

---

### Impact Explanation

Any subscription that sets `whitelistEnabled = true` — the feature explicitly designed to restrict price-data access to trusted readers — renders `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` permanently unusable. These are the only functions that enforce a freshness guarantee on returned prices. Whitelisted readers who need age-bounded prices are forced to fall back to `getPricesUnsafe`, which returns arbitrarily stale data, or to abandon the Scheduler contract entirely. This is a complete denial of service on the two safety-critical read functions for the whitelist use-case.

---

### Likelihood Explanation

The whitelist feature is a first-class, documented feature of the Scheduler contract. Any integrator who creates a subscription with `whitelistEnabled = true` and then calls `getPricesNoOlderThan` or `getEmaPricesNoOlderThan` will hit this revert 100% of the time. No special attacker capability is required — a normal whitelisted reader triggers it by calling the function as intended. The bug is deterministic and reproducible.

---

### Recommendation

Replace the `this.method()` external delegation with a direct call to the shared internal helper `_getPricesInternal`, replicating the same pattern already used by `getPricesUnsafe` and `getEmaPricesUnsafe`:

```solidity
function getPricesNoOlderThan(
    uint256 subscriptionId,
    bytes32[] calldata priceIds,
    uint256 age_seconds
)
    external
    view
    override
    onlyWhitelistedReader(subscriptionId)
    returns (PythStructs.Price[] memory prices)
{
    SchedulerStructs.SubscriptionStatus memory status =
        _state.subscriptionStatuses[subscriptionId];

    if (distance(block.timestamp, status.priceLastUpdatedAt) > age_seconds)
        revert PythErrors.StalePrice();

    // Call internal helper directly — do NOT use this.getPricesUnsafe()
    PythStructs.PriceFeed[] memory priceFeeds =
        _getPricesInternal(subscriptionId, priceIds);
    prices = new PythStructs.Price[](priceFeeds.length);
    for (uint i = 0; i < priceFeeds.length; i++) {
        prices[i] = priceFeeds[i].price;
    }
}
```

Apply the same fix to `getEmaPricesNoOlderThan`.

### Proof of Concept

```solidity
// 1. Deploy SchedulerUpgradeable (standard setup).
// 2. Create a subscription with whitelistEnabled = true, readerWhitelist = [reader].
// 3. Fund the subscription and call updatePriceFeeds to populate price data.
// 4. As `reader` (a whitelisted address), call:
//      scheduler.getPricesNoOlderThan(subscriptionId, priceIds, 3600);
// Expected (correct): returns prices.
// Actual: reverts with SchedulerErrors.Unauthorized
//         because this.getPricesUnsafe() re-enters with msg.sender = address(scheduler),
//         which is not in the whitelist.
```

The existing test `testEnabledWhitelistEnforcesOnlyAuthorizedReads` confirms the whitelist is enforced for direct `getPricesUnsafe` calls but does not exercise `getPricesNoOlderThan`, leaving this path untested. [4](#0-3)
