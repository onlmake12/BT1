### Title
Lazer Price Update Replay — `verifyUpdate` Accepts Replayed Signed Messages Without Freshness or Uniqueness Enforcement - (File: lazer/contracts/evm/src/PythLazer.sol)

---

### Summary

`PythLazer.verifyUpdate` performs ECDSA signature recovery and trusted-signer validation but maintains no per-message state. There is no nonce, no consumed-hash set, and no timestamp-freshness floor enforced at the contract level. Any actor who captures a valid signed Lazer price update can replay it an unlimited number of times for as long as the signing key's `expiresAt` has not passed.

---

### Finding Description

`verifyUpdate` in `lazer/contracts/evm/src/PythLazer.sol` is the sole on-chain verification primitive for Lazer price updates on EVM chains. Its full logic is:

1. Collect the verification fee.
2. Check the EVM format magic number.
3. Recover the ECDSA signer from the payload hash.
4. Assert `isValidSigner(signer)` — i.e., `block.timestamp < trustedSignerToExpiresAtMapping[signer]`.
5. Return `(payload, signer)`. [1](#0-0) 

No state is written after a successful call. The contract never records the payload hash, a nonce, or the payload's embedded timestamp. The same `update` bytes pass all checks identically on every invocation until the signer key expires.

The Sui equivalent `parse_and_verify_le_ecdsa_update_v2` in `lazer/contracts/sui/sources/pyth_lazer.move` has the same property: it verifies the signature and signer expiry but stores nothing about the message itself. [2](#0-1) 

The payload does contain a `timestamp` field (parsed by `PythLazerLib`), but `verifyUpdate` never reads or enforces it. The contract's documentation and integration guides delegate timestamp staleness checking entirely to consumer contracts. [3](#0-2) 

---

### Impact Explanation

An unprivileged relayer or attacker who observes the public Lazer WebSocket stream can capture any valid signed price update and replay it to any consumer contract that calls `verifyUpdate`. Two concrete harm paths:

1. **Consumer without timestamp check**: The consumer accepts the replayed stale price as current. Downstream DeFi logic (liquidations, collateral valuations, AMM pricing) operates on an arbitrarily old price.

2. **Consumer with timestamp check**: The attacker replays the most recently captured message within the consumer's acceptable staleness window (e.g., within the last N seconds). Because the contract itself imposes no freshness floor, the attacker can keep a captured message "fresh" relative to the consumer's window by replaying it just before the window closes, preventing the consumer from ever seeing a newer price.

The Lazer governance VAA path correctly prevents replay via `seen_sequence`: [4](#0-3) 

But price update messages have no equivalent protection.

---

### Likelihood Explanation

- The Lazer price stream is publicly accessible via WebSocket.
- No privileged access is required to capture a signed update.
- Replaying requires only submitting the captured bytes to `verifyUpdate` with the fee attached.
- The replay window is bounded only by the signer's `expiresAt`, which is set far in the future (e.g., years).
- Consumer contracts that omit or implement timestamp checks incorrectly are common in practice.

---

### Recommendation

Enforce a maximum payload age inside `verifyUpdate`. Parse the `timestamp` field from the payload and revert if `block.timestamp - payloadTimestamp > MAX_AGE`. Alternatively, maintain a `mapping(bytes32 => bool) usedPayloadHashes` and revert on reuse. The governance path already demonstrates the correct pattern (sequence tracking); the same discipline should apply to price update messages.

---

### Proof of Concept

```solidity
// Attacker captures `update` bytes from the Lazer WebSocket at time T.
// At time T + 3600 (one hour later):
(bytes memory payload, address signer) = pythLazer.verifyUpdate{value: fee}(update);
// Succeeds. `payload` contains timestamp T, but the contract accepted it.
// A consumer that does not check payload timestamp will use a 1-hour-old price.

// Replay again at T + 7200:
(payload, signer) = pythLazer.verifyUpdate{value: fee}(update);
// Still succeeds. No state was written on the first call.
```

The test suite itself demonstrates this: `test_verify` in `lazer/contracts/evm/test/PythLazer.t.sol` calls `verifyUpdate` twice with the identical `update` bytes and both calls succeed without error. [5](#0-4)

    function verifyUpdate(
        bytes calldata update
    ) external payable returns (bytes calldata payload, address signer) {
        // Require fee and refund excess
        require(msg.value >= verification_fee, "Insufficient fee provided");
        if (msg.value > verification_fee) {
            payable(msg.sender).transfer(msg.value - verification_fee);
        }

        if (update.length < 71) {
            revert("input too short");
        }
        uint32 EVM_FORMAT_MAGIC = 706910618;

        uint32 evm_magic = uint32(bytes4(update[0:4]));
        if (evm_magic != EVM_FORMAT_MAGIC) {
            revert("invalid evm magic");
        }
        uint16 payload_len = uint16(bytes2(update[69:71]));
        if (update.length < 71 + payload_len) {
            revert("input too short");
        }
        payload = update[71:71 + payload_len];
        bytes32 hash = keccak256(payload);
        (signer, , ) = ECDSA.tryRecover(
            hash,
            uint8(update[68]) + 27,
            bytes32(update[4:36]),
            bytes32(update[36:68])
        );
        if (signer == address(0)) {
            revert("invalid signature");
        }
        if (!isValidSigner(signer)) {
            revert("invalid signer");
        }
    }

public fun parse_and_verify_le_ecdsa_update_v2(s: &State, clock: &Clock, update: vector<u8>): Update {
    let mut cursor = bcs::new(update);

    // Parse and validate message magic
    let magic = cursor.peel_u32();
    assert!(magic == UPDATE_MESSAGE_MAGIC, EInvalidUpdateMagic);

    // Parse signature
    let mut signature = vector::empty<u8>();
    let mut sig_i = 0;
    while (sig_i < SECP256K1_SIG_LEN) {
        signature.push_back(cursor.peel_u8());
        sig_i = sig_i + 1;
    };

    // Parse expected payload length and get remaining bytes as payload
    let payload_len = cursor.peel_u16();
    let payload = cursor.into_remainder_bytes();

    // Validate expected payload length
    assert!(payload_len as u64 == payload.length(), EInvalidPayloadLength);

    // Parse payload
    let mut payload_cursor = bcs::new(payload);
    let payload_magic = payload_cursor.peel_u32();
    assert!(payload_magic == PAYLOAD_MAGIC, EInvalidPayloadMagic);

    // Verify the signature against trusted signers
    verify_le_ecdsa_message(s, clock, &signature, &payload);

    update_v2::parse_from_cursor(payload_cursor)
}
