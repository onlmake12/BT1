### Title
Missing Chain Identifier in Alert Signing Message Enables Cross-Network Alert Replay — (`util/network-alert/src/verifier.rs`)

---

### Summary

The CKB network alert system signs and verifies alert messages using a multi-signature scheme, but the signed message hash (`calc_alert_hash`) is computed solely from the `RawAlert` payload, which contains no chain/network identifier. Because the same hard-coded public keys are used across all CKB networks (mainnet, testnet), a validly signed alert from one network can be replayed verbatim on any other network and will pass signature verification, causing all nodes on the target network to accept and display the false alert.

---

### Finding Description

The `RawAlert` molecule schema defines the alert payload as:

```
table RawAlert {
    notice_until:   Uint64,
    id:             Uint32,
    cancel:         Uint32,
    priority:       Uint32,
    message:        Bytes,
    min_version:    BytesOpt,
    max_version:    BytesOpt,
}
``` [1](#0-0) 

There is no `chain_id`, genesis hash, or network name field. The alert hash used as the signing message is computed as:

```rust
pub fn calc_alert_hash(&self) -> packed::Byte32 {
    self.calc_hash()  // blake2b of raw RawAlert bytes only
}
``` [2](#0-1) 

The `Verifier::verify_signatures` function verifies that the alert carries valid signatures from the required threshold of authorized keys, but the message it verifies is this chain-agnostic hash:

```rust
let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
``` [3](#0-2) 

The authorized public keys are the same hard-coded set for all networks (mainnet, testnet, devnet), loaded from the default `NetworkAlertConfig`: [4](#0-3) 

When an alert is received over P2P, `AlertRelayer::received` parses it, checks signatures via `self.verifier.verify_signatures(&alert)`, and if valid, broadcasts it to all connected peers and adds it to the local notifier — with no check that the alert was intended for this network: [5](#0-4) 

By contrast, the P2P `Identify` protocol does bind to a network-specific identifier (genesis hash prefix), but this binding is entirely absent from the alert subsystem:

```rust
pub fn identify_name(&self) -> String {
    let genesis_hash = format!("{:x}", Into::<H256>::into(&self.genesis_hash));
    format!("/{}/{}", self.id, &genesis_hash[..8])
}
``` [6](#0-5) 

---

### Impact Explanation

An attacker who observes a legitimately signed alert on the testnet P2P network (alerts are broadcast to all connected peers) can relay that exact alert message to mainnet nodes. Because the signature covers only the chain-agnostic `RawAlert` bytes, and the same developer keys authorize alerts on all networks, the mainnet `Verifier` accepts the alert as valid. All mainnet nodes then:

1. Store and display the alert to users via `get_blockchain_info`.
2. Re-broadcast it to all their peers, propagating the false alert across the entire mainnet.

The alert system is specifically designed for "critical problems" — messages that prompt users to upgrade, stop using the network, or take other urgent action. A false alert replayed from testnet to mainnet (e.g., "CKB v0.X.* have critical bugs, stop using the network") could cause widespread user panic, unnecessary software downgrades, or coordinated network disruption based on fabricated information. **Integrity impact is high.**

---

### Likelihood Explanation

- Testnet alerts are broadcast over the public P2P network; any observer (including an attacker running a testnet node) can capture a fully signed alert message.
- No special privilege is required beyond running a standard CKB node on testnet to receive alerts.
- Replaying the captured bytes to a mainnet peer requires only a standard P2P connection — the `AlertRelayer` processes incoming alert bytes from any connected peer.
- The same default public keys are used on all networks, so no key material needs to be compromised.
- The attack is fully passive until the replay step: capture on testnet, relay to mainnet.

---

### Recommendation

Include a chain-specific domain separator in the alert hash computation. The genesis hash (already used in `identify_name`) is the natural choice:

```
hash = blake2b(genesis_hash || raw_alert_bytes)
```

Alternatively, add a `chain_id` or `genesis_hash` field to `RawAlert` itself so that the signed commitment is chain-bound. This mirrors the fix pattern in EIP-155 (Ethereum) and CKB's own `identify_name` approach.

---

### Proof of Concept

**Step 1.** Run a CKB testnet node and connect to the testnet P2P network. Observe an incoming alert message (e.g., the real `alert_20230001` whose signatures are publicly known).

**Step 2.** The captured `packed::Alert` bytes are self-contained: `RawAlert` + signatures. No modification is needed.

**Step 3.** Connect to a mainnet CKB node and send the captured alert bytes over the `Alert` protocol channel (or submit via `send_alert` RPC).

**Step 4.** The mainnet `AlertRelayer::received` handler parses the alert, calls `self.verifier.verify_signatures(&alert)`, which computes `blake2b(raw_alert_bytes)` — identical to the testnet hash — and confirms the signatures are valid against the same hard-coded public keys.

**Step 5.** The alert is accepted, stored, re-broadcast to all mainnet peers, and displayed to all mainnet users via `get_blockchain_info().alerts`. [7](#0-6) [8](#0-7) [9](#0-8)

    pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
        trace!("Verifying alert {:?}", alert);
        let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
        let signatures: Vec<Signature> = alert
            .signatures()
            .into_iter()
            .filter_map(
                |sig_data| match Signature::from_slice(sig_data.as_reader().raw_data()) {
                    Ok(sig) => {
                        if sig.is_valid() {
                            Some(sig)
                        } else {
                            debug!("invalid signature: {:?}", sig);
                            None
                        }
                    }
                    Err(err) => {
                        debug!("signature error: {}", err);
                        None
                    }
                },
            )
            .collect();
        verify_m_of_n(
            &message,
            self.config.signatures_threshold,
            &signatures,
            &self.pubkeys,
        )
        .map_err(|err| err.kind())?;
        Ok(())
    }

    #[allow(clippy::needless_collect)]
    async fn received(
        &mut self,
        nc: Arc<dyn CKBProtocolContext + Sync>,
        peer_index: PeerIndex,
        data: Bytes,
    ) {
        let alert: packed::Alert = match packed::AlertReader::from_slice(&data) {
            Ok(alert) => {
                if alert.raw().message().is_utf8()
                    && alert
                        .raw()
                        .min_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                    && alert
                        .raw()
                        .max_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                {
                    alert.to_entity()
                } else {
                    info!(
                        "A malformed message fromP peer {} : not utf-8 string",
                        peer_index
                    );
                    nc.ban_peer(
                        peer_index,
                        BAD_MESSAGE_BAN_TIME,
                        String::from("send us a malformed message: not utf-8 string"),
                    );
                    return;
                }
            }
            Err(err) => {
                info!("A malformed message from peer {}: {:?}", peer_index, err);
                nc.ban_peer(
                    peer_index,
                    BAD_MESSAGE_BAN_TIME,
                    String::from("send us a malformed message"),
                );
                return;
            }
        };
        let alert_id = alert.as_reader().raw().id().into();
        trace!("ReceiveD alert {} from peer {}", alert_id, peer_index);
        // ignore alert
        if self.notifier.lock().has_received(alert_id) {
            return;
        }
        // verify
        if let Err(err) = self.verifier.verify_signatures(&alert) {
            debug!(
                "An alert from peer {} with invalid signatures, error {:?}",
                peer_index, err
            );
            nc.ban_peer(
                peer_index,
                BAD_MESSAGE_BAN_TIME,
                String::from("send us an alert with invalid signatures"),
            );
            return;
        }
        // mark sender as known
        self.mark_as_known(peer_index, alert_id);
        // broadcast message
        let selected_peers: Vec<PeerIndex> = nc
            .connected_peers()
            .into_iter()
            .filter(|peer| self.mark_as_known(*peer, alert_id))
            .collect();
        if let Err(err) = nc.quick_filter_broadcast(
            TargetSession::Multi(Box::new(selected_peers.into_iter())),
            data,
        ) {
            debug!("alert broadcast error: {:?}", err);
        }
        // add to received alerts
        self.notifier.lock().add(&alert);
    }
