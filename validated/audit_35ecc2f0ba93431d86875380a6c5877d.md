### Title
Missing Per-Peer Rate Limiter on BlockFilter Protocol Allows Single-Peer DoS via Unbounded DB Read Amplification — (`sync/src/filter/mod.rs`)

---

### Summary

The `BlockFilter` protocol handler has no rate limiter at any layer, while the `Relayer` protocol enforces a hard cap of 30 req/s per `(PeerIndex, message_item_id)`. A single unprivileged peer can flood a node with `GetBlockFilterHashes`, `GetBlockFilters`, or `GetBlockFilterCheckPoints` requests at the maximum TCP/tentacle frame rate, each triggering up to 2000 synchronous DB reads, causing sustained I/O exhaustion with no enforcement mechanism to stop it.

---

### Finding Description

**`Relayer` has a rate limiter; `BlockFilter` does not.**

`Relayer::new` initializes a `governor::RateLimiter<(PeerIndex, u32)>` at 30 req/s: [1](#0-0) 

`Relayer::try_process` checks it before dispatching every message type (except PoW-protected `CompactBlock`): [2](#0-1) 

`BlockFilter` has no such field — its struct contains only `shared: Arc<SyncShared>`: [3](#0-2) 

`BlockFilter::received` parses the message and immediately calls `self.process(...)` with zero rate-limit checks: [4](#0-3) 

`BlockFilter::try_process` dispatches to handlers with no rate-limit guard: [5](#0-4) 

**Per-request DB amplification:**

Each `GetBlockFilterHashes` request loops up to `BATCH_SIZE = 2000` times, performing two DB reads per iteration (`get_block_hash` + `get_block_filter_hash`): [6](#0-5) 

`GetBlockFilterCheckPoints` has the same `BATCH_SIZE = 2000` loop: [7](#0-6) 

`GetBlockFilters` loops up to `BATCH_SIZE = 1000` times reading full filter data (up to 1.8 MB per response): [8](#0-7) 

**Filter is enabled by default** in both the default config and `default_support_all_protocols()`: [9](#0-8) [10](#0-9) 

The launcher registers the `BlockFilter` handler when `Filter` is in `support_protocols`: [11](#0-10) 

---

### Impact Explanation

A single peer can send `GetBlockFilterHashes` at the maximum tentacle frame rate (the Filter protocol allows 2 MB frames, but the *request* message is tiny — just an 8-byte `start_number`). Each request triggers up to 4000 RocksDB reads (2000 × `get_block_hash` + `get_block_filter_hash`). At even 100 req/s this is 400,000 DB reads/s from one peer. This saturates the node's I/O, starving the sync and relay pipelines of DB access, causing the node to fall behind on block propagation — effective network congestion at low attacker cost.

---

### Likelihood Explanation

The Filter protocol is on by default, reachable by any peer that negotiates it. The `BLOCK_FILTER` flag is advertised in the `Identify` handshake, so any peer can discover and target it. No PoW, stake, or privileged role is required.

---

### Recommendation

Add a `governor::RateLimiter<(PeerIndex, u32)>` field to `BlockFilter` (mirroring `Relayer`), initialize it at the same 30 req/s quota in `BlockFilter::new`, and check it at the top of `BlockFilter::try_process` keyed by `(peer, message.item_id())`, returning `StatusCode::TooManyRequests` on failure — identical to the guard in `Relayer::try_process`.

---

### Proof of Concept

```
1. Connect to a CKB full node (default config, Filter protocol enabled).
2. Open the /ckb/filter protocol (ID 121).
3. In a tight loop, send GetBlockFilterHashes{start_number: 0} frames.
   - Each frame is ~12 bytes; the node processes each with up to 2000 DB reads.
4. Observe: node I/O saturates; block sync and relay latency increases.
5. Contrast: repeat with /ckb/relay3 (ID 101) sending GetBlockTransactions —
   after 30 req/s the node returns TooManyRequests and stops processing.
   No such cutoff exists for /ckb/filter.
```

    pub async fn execute(self) -> Status {
        let active_chain = self.filter.shared.active_chain();
        let start_number: BlockNumber = self.message.to_entity().start_number().into();
        let latest: BlockNumber = active_chain.get_latest_built_filter_block_number();

        let mut block_filter_hashes = Vec::new();

        if latest >= start_number {
            let parent_block_filter_hash = if start_number > 0 {
                match active_chain
                    .get_block_hash(start_number - 1)
                    .and_then(|block_hash| active_chain.get_block_filter_hash(&block_hash))
                {
                    Some(parent_block_filter_hash) => parent_block_filter_hash,
                    None => return Status::ignored(),
                }
            } else {
                packed::Byte32::zero()
            };

            let mut block_number = start_number;

                if let Some(block_filter_hash) = active_chain
                    .get_block_hash(block_number)
                    .and_then(|block_hash| active_chain.get_block_filter_hash(&block_hash))
                {
                    block_filter_hashes.push(block_filter_hash);
                } else {
                    break;
                }
                let Some(next_block_number) = block_number.checked_add(1) else {
                    break;
                };

        message: packed::GetBlockFiltersReader<'a>,
        filter: &'a BlockFilter,
        nc: Arc<dyn CKBProtocolContext + Sync>,
        peer: PeerIndex,
    ) -> Self {
        Self {
            message,
            nc,
            filter,
            peer,
        }

    pub async fn execute(self) -> Status {
        let active_chain = self.filter.shared.active_chain();
        let start_number: BlockNumber = self.message.to_entity().start_number().into();
        let latest: BlockNumber = active_chain.get_latest_built_filter_block_number();

        if latest >= start_number {
            let mut block_hashes = Vec::new();
            let mut filters = Vec::new();
            let mut current_content_size = 0;
            current_content_size += 8; // Size of start_number
            current_content_size += 4 * 2; // Size of the header field `full-size` of `block_hash` and `block_filter`
            let mut block_number = start_number;
            for _ in 0..BATCH_SIZE {
                if let Some(block_hash) = active_chain.get_block_hash(block_number) {
                    if let Some(block_filter) = active_chain.get_block_filter(&block_hash) {
                        if current_content_size
                            + block_hash.as_slice().len()
                            + 4
                            + block_filter.as_slice().len()
                            + 4
                            >= (1.8 * 1024.0 * 1024.0) as usize
                        {
                            // Break if the encoded size of `block_hash` + `block_filter` + `start_number` + molecule header increase reaches 1.8MB, to avoid frame size too large
                            break;
                        }
                        current_content_size +=
                            block_hash.as_slice().len() + block_filter.as_slice().len() + 4;
                        block_hashes.push(block_hash);
                        filters.push(block_filter);
                    } else {
                        break;
                    }
                } else {
                    break;
                }
                let Some(next_block_number) = block_number.checked_add(1) else {
                    break;
                };
                block_number = next_block_number;
            }
