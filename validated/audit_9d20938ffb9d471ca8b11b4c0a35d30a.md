### Title
`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` Permanently DoS'd for Whitelisted Subscriptions via `this.` External Call Changing `msg.sender` — (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`Scheduler.sol` implements a reader whitelist via the `onlyWhitelistedReader` modifier. The "freshness-checked" price functions `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` each carry this modifier, but then internally call `this.getPricesUnsafe(...)` and `this.getEmaPricesUnsafe(...)` using the `this.` keyword, which issues an **external call**. This changes `msg.sender` inside the callee to the Scheduler contract address itself. The Scheduler contract is never the subscription manager and is never in any reader whitelist, so the second `onlyWhitelistedReader` check always reverts when `whitelistEnabled = true`. Both freshness-guaranteed read functions are permanently broken for every subscription that enables the whitelist.

---

### Finding Description

**Root cause — `this.` external call changes `msg.sender`:**

`getPricesNoOlderThan` (line 543) and `getEmaPricesNoOlderThan` (line 586) both carry `onlyWhitelistedReader(subscriptionId)`. After passing that check for the original caller, they delegate to the underlying data-fetch functions via:

```solidity
// getPricesNoOlderThan, line 554
prices = this.getPricesUnsafe(subscriptionId, priceIds);

// getEmaPricesNoOlderThan, line 597
prices = this.getEmaPricesUnsafe(subscriptionId, priceIds);
``` [1](#0-0) [2](#0-1) 

Because `this.` triggers an **external** Solidity call, `msg.sender` inside `getPricesUnsafe` / `getEmaPricesUnsafe` becomes the **Scheduler contract's own address**, not the original EOA/contract that initiated the transaction.

`getPricesUnsafe` and `getEmaPricesUnsafe` both carry the same `onlyWhitelistedReader` modifier: [3](#0-2) [4](#0-3) 

The modifier logic is:

```solidity
modifier onlyWhitelistedReader(uint256 subscriptionId) {
    // Manager is always allowed
    if (_state.subscriptionManager[subscriptionId] == msg.sender) { _; return; }
    // If whitelist is not used, allow any reader
    if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) { _; return; }
    // Check if caller is in whitelist
    ...
    if (!isWhitelisted) { revert SchedulerErrors.Unauthorized(); }
    _;
}
``` [5](#0-4) 

When `whitelistEnabled = true`:
1. `msg.sender` = Scheduler contract address — **not** the subscription manager (manager is set to the user who called `createSubscription`).
2. The Scheduler contract address is **not** in any reader whitelist.
3. The modifier reverts with `Unauthorized`.

**Exploit path:**
1. A subscription manager creates a subscription with `whitelistEnabled = true` and adds legitimate reader addresses to `readerWhitelist`.
2. A whitelisted reader calls `getPricesNoOlderThan(subscriptionId, priceIds, age)`.
3. The outer `onlyWhitelistedReader` check passes (caller is whitelisted).
4. The function calls `this.getPricesUnsafe(subscriptionId, priceIds)` — external call, `msg.sender` = Scheduler address.
5. The inner `onlyWhitelistedReader` check in `getPricesUnsafe` reverts because the Scheduler address is not the manager and not in the whitelist.
6. The entire transaction reverts. Same for `getEmaPricesNoOlderThan` → `this.getEmaPricesUnsafe(...)`. [6](#0-5) [7](#0-6) 

---

### Impact Explanation

Any subscription with `whitelistEnabled = true` loses access to both freshness-guaranteed read functions (`getPricesNoOlderThan`, `getEmaPricesNoOlderThan`) entirely. These are the **recommended safe** read paths per the `IScheduler` interface documentation:

> *"This function is a sanity-checked version of `getPricesUnsafe` which is useful in applications that require a sufficiently-recent price."* [8](#0-7) 

Whitelisted integrators who rely on freshness guarantees (e.g., DeFi protocols that must reject stale prices) are permanently blocked from using these functions. They are forced to fall back to `getPricesUnsafe` / `getEmaPricesUnsafe` without any on-chain freshness enforcement, defeating the purpose of the whitelist-protected subscription model. This constitutes a **permanent DoS on core price-read functionality** for all whitelisted subscriptions.

---

### Likelihood Explanation

The whitelist feature (`whitelistEnabled = true`) is a first-class, documented feature of the Scheduler. Any subscription manager who enables it — the expected use case for access-controlled price feeds — immediately triggers this DoS. No special attacker is needed; the bug is triggered by normal, intended usage. Likelihood is **high**.

---

### Recommendation

Replace the `this.` external calls with direct calls to the internal helper `_getPricesInternal`, mirroring the pattern already used in `getPricesUnsafe` and `getEmaPricesUnsafe`:

```solidity
// getPricesNoOlderThan — replace line 554:
// BEFORE: prices = this.getPricesUnsafe(subscriptionId, priceIds);
// AFTER:
PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
prices = new PythStructs.Price[](priceFeeds.length);
for (uint i = 0; i < priceFeeds.length; i++) {
    prices[i] = priceFeeds[i].price;
}

// getEmaPricesNoOlderThan — replace line 597 similarly, using priceFeeds[i].emaPrice
``` [9](#0-8) 

This keeps all access control in the single outer `onlyWhitelistedReader` check and avoids the `msg.sender` context switch caused by `this.`.

---

### Proof of Concept

```solidity
// Setup: subscription with whitelistEnabled = true, reader = address(0xBEEF)
// Manager creates subscription:
uint256 subId = scheduler.createSubscription{value: 1 ether}(params); // params.whitelistEnabled = true, readerWhitelist = [0xBEEF]

// Keeper updates prices (works fine, no whitelist on updatePriceFeeds)
scheduler.updatePriceFeeds(subId, updateData);

// Whitelisted reader tries freshness-checked read:
vm.prank(address(0xBEEF));
// REVERTS with Unauthorized — msg.sender inside getPricesUnsafe = Scheduler address
scheduler.getPricesNoOlderThan(subId, priceIds, 60);

// Whitelisted reader can still call getPricesUnsafe directly (no freshness guarantee):
vm.prank(address(0xBEEF));
scheduler.getPricesUnsafe(subId, priceIds); // succeeds — msg.sender = 0xBEEF, passes whitelist
```

The revert is deterministic and reproducible for any subscription with `whitelistEnabled = true`. No privileged access is required; any subscription manager enabling the whitelist triggers the condition.

    function _getPricesInternal(
        uint256 subscriptionId,
        bytes32[] calldata priceIds
    ) internal view returns (PythStructs.PriceFeed[] memory priceFeeds) {
        if (!_state.subscriptionParams[subscriptionId].isActive) {
            revert SchedulerErrors.InactiveSubscription();
        }

        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        // If no price IDs provided, return all price feeds for the subscription
        if (priceIds.length == 0) {
            PythStructs.PriceFeed[]
                memory allFeeds = new PythStructs.PriceFeed[](
                    params.priceIds.length
                );
            for (uint8 i = 0; i < params.priceIds.length; i++) {
                PythStructs.PriceFeed memory priceFeed = _state.priceUpdates[
                    subscriptionId
                ][params.priceIds[i]];
                // Check if the price feed exists (price ID is valid and has been updated)
                if (priceFeed.id == bytes32(0)) {
                    revert SchedulerErrors.InvalidPriceId(
                        params.priceIds[i],
                        bytes32(0)
                    );
                }
                allFeeds[i] = priceFeed;
            }
            return allFeeds;
        }

        // Return only the requested price feeds
        PythStructs.PriceFeed[]
            memory requestedFeeds = new PythStructs.PriceFeed[](
                priceIds.length

            }
            requestedFeeds[i] = priceFeed;

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
