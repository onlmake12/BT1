### Title
Network Alert Signature Lacks Chain Domain Separation, Enabling Cross-Network Replay — (`File: util/network-alert/src/verifier.rs`)

### Summary

The CKB network alert system signs and verifies alerts using a hash of `RawAlert` content that contains no chain/network identifier. Because the same hardcoded public keys are used for both mainnet and testnet by default, a validly-signed alert from one network can be replayed verbatim on the other network and will pass signature verification.

### Finding Description

The `RawAlert` molecule schema contains only application-level fields — `notice_until`, `id`, `cancel`, `priority`, `message`, `min_version`, `max_version` — with no chain identifier or network domain separator. [1](#0-0) 

The alert hash used for signing is computed as a plain `blake2b_256` over the raw serialized bytes of `RawAlert`: [2](#0-1) 

`Verifier::verify_signatures()` derives the message to verify directly from `alert.calc_alert_hash()`, with no chain-specific prefix or domain tag mixed in: [3](#0-2) 

The default `NetworkAlertConfig` embeds the same four Nervos Foundation public keys for every node regardless of whether it is running mainnet or testnet: [4](#0-3) [5](#0-4) 

### Impact Explanation

An attacker who observes a legitimately signed alert on the CKB testnet P2P network can extract the raw `Alert` bytes (which already carry 2-of-4 valid Nervos Foundation signatures) and relay them unchanged to mainnet nodes. Because the signed hash covers only the `RawAlert` content — which is identical on both networks — `verify_signatures` accepts the message on mainnet. The alert is then stored by `Notifier::add` and propagated to all connected mainnet peers via `AlertRelayer::received`, causing every mainnet node to display the alert message. A testnet alert saying "critical bug, stop transactions" or "upgrade immediately" replayed on mainnet would cause widespread operational disruption and user panic across the live network. [6](#0-5) 

### Likelihood Explanation

The attack requires only passive observation of the testnet P2P network (no privilege needed) and the ability to connect to one mainnet peer and send a P2P message or call the `send_alert` RPC. The Nervos Foundation regularly issues real alerts (e.g., alert `20230001` is hardcoded in tests). Any such testnet alert is immediately replayable on mainnet. The entry path is fully reachable by an unprivileged peer or RPC caller. [7](#0-6) [8](#0-7) 

### Recommendation

Include a chain/network domain separator in the signed message. The simplest fix is to mix the genesis block hash (already available as `Consensus::genesis_hash`) into the alert hash computation, either by adding a `chain_id` field to `RawAlert` or by prepending the genesis hash before hashing in `calc_alert_hash`. This ensures a signature produced for one network is cryptographically invalid on any other network, directly mirroring the EIP712 `block.chainid` fix.

### Proof of Concept

1. Run a CKB testnet node and a CKB mainnet node (both use the default `alert_signature.toml` with the same four public keys).
2. On testnet, submit a valid alert via `send_alert` RPC with 2-of-4 Nervos Foundation signatures.
3. Capture the raw `Alert` molecule bytes from the testnet P2P wire.
4. Connect to any mainnet peer and send the identical bytes over the `Alert` protocol, or call `send_alert` on a mainnet RPC endpoint with the same payload.
5. Observe that `Verifier::verify_signatures` returns `Ok(())` on the mainnet node, the alert is added to `Notifier`, and it propagates to all connected mainnet peers — despite never being signed for mainnet. [9](#0-8) [10](#0-9)

    pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
        trace!("Verifying alert {:?}", alert);
        let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
        let signatures: Vec<Signature> = alert
            .signatures()
            .into_iter()
            .filter_map(
                |sig_data| match Signature::from_slice(sig_data.as_reader().raw_data()) {
                    Ok(sig) => {
                        if sig.is_valid() {
                            Some(sig)
                        } else {
                            debug!("invalid signature: {:?}", sig);
                            None
                        }
                    }
                    Err(err) => {
                        debug!("signature error: {}", err);
                        None
                    }
                },
            )
            .collect();
        verify_m_of_n(
            &message,
            self.config.signatures_threshold,
            &signatures,
            &self.pubkeys,
        )
        .map_err(|err| err.kind())?;
        Ok(())
    }

#[async_trait]
impl AlertRpc for AlertRpcImpl {
    fn send_alert(&self, alert: Alert) -> Result<()> {
        let alert: packed::Alert = alert.into();
        let now_ms = ckb_systemtime::unix_time_as_millis();
        let notice_until: u64 = alert.raw().notice_until().into();
        if notice_until < now_ms {
            return Err(RPCError::invalid_params(format!(
                "Expected `params[0].notice_until` in the future (> {now_ms}), got {notice_until}",
            )));
        }

        let result = self.verifier.verify_signatures(&alert);

        match result {
            Ok(()) => {
                // set self node notifier
                self.notifier.lock().add(&alert);

                self.network_controller.broadcast_with_handle(
                    SupportProtocols::Alert.protocol_id(),
                    alert.as_bytes(),
                    &self.handle,
                );
                Ok(())
            }
            Err(e) => Err(RPCError::custom_with_error(
                RPCError::AlertFailedToVerifySignatures,
                e,
            )),
        }
    }
