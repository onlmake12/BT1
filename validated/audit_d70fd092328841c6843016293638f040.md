### Title
Malicious Echo Consumer Returns Unbounded Returndata to Gas-Bomb `executeCallback` — (`target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

`Echo.executeCallback` dispatches to a user-supplied requester address via Solidity's `try/catch`. Solidity's `try/catch` compiles to a low-level `CALL` followed by an unconditional `RETURNDATACOPY` of the full returndata buffer into the **outer** context's memory before any branch decision is made. A malicious consumer can revert with a large payload (within the forwarded `callbackGasLimit`), forcing the outer `executeCallback` frame to pay unbounded memory-expansion gas for the copy. Because `clearRequest` executes **before** the callback, an OOG revert in the outer frame undoes the clear, leaving the request permanently stuck and the provider's gas wasted.

---

### Finding Description

`Echo.executeCallback` at line 176–201 of `Echo.sol` calls the consumer callback as:

```solidity
try
    IEchoConsumer(req.requester)._echoCallback{
        gas: req.callbackGasLimit
    }(sequenceNumber, priceFeeds)
{
    emitPriceUpdate(...);
} catch Error(string memory reason) {
    emit PriceUpdateCallbackFailed(..., reason);
} catch {
    emit PriceUpdateCallbackFailed(..., "low-level error (possibly out of gas)");
}
``` [1](#0-0) 

Solidity's `try/catch` desugars to a low-level `CALL` followed by `RETURNDATASIZE` / `RETURNDATACOPY` into the **caller's** memory before the branch selector runs. The `{gas: req.callbackGasLimit}` cap limits gas forwarded to the callee, but the cost of copying the returned bytes into the outer frame's memory is paid entirely by the **outer** context from its own remaining gas. These are two separate memory-expansion budgets.

A malicious consumer can use assembly to return a maximally large buffer within `callbackGasLimit`:

```solidity
function _echoCallback(uint64, PythStructs.PriceFeed[] memory) external {
    assembly {
        // allocate and return N words of zeros; memory expansion paid by callee
        return(0, N_BYTES)
    }
}
```

When `executeCallback` resumes after the call, the EVM copies those `N_BYTES` into the outer frame's memory. The quadratic memory-expansion formula `(words² / 512) + 3·words` means the outer frame can be forced to spend gas proportional to `callbackGasLimit` just on the copy — gas the provider did not budget for.

Contrast this with `Entropy.sol`'s new path, which explicitly caps returndata at 256 bytes via `excessivelySafeCall`:

```solidity
(success, ret) = req.requester.excessivelySafeCall(
    uint256(req.gasLimit10k) * TEN_THOUSAND,
    256, // copy at most 256 bytes of the return value into ret.
    ...
);
``` [2](#0-1) 

`Echo.sol` has no equivalent protection.

The compounding factor is the ordering of effects. `clearRequest` is called at line 164, **before** the callback:

```solidity
clearRequest(sequenceNumber);
// ... firstUnfulfilledSeq update ...
try IEchoConsumer(req.requester)._echoCallback{gas: req.callbackGasLimit}(...) { ... }
``` [3](#0-2) 

If the outer frame runs out of gas during the returndata copy, the entire transaction reverts, undoing `clearRequest`. The request remains active but the provider's gas is lost. The existing code comment at line 155–160 acknowledges that `executeCallback` must never revert to avoid locking funds, yet the returndata gas-bomb is precisely a path that causes it to revert. [4](#0-3) 

---

### Impact Explanation

A malicious requester can permanently prevent a provider from fulfilling their specific Echo request without providing unbounded extra gas. Each retry attempt by the provider reverts and wastes gas. Because the request is never cleared (the revert undoes `clearRequest`), the provider's accrued fee credit is also never finalized. On chains with lower block gas limits the attack can make the request entirely unfulfillable within a single block.

---

### Likelihood Explanation

Any unprivileged user can call `requestPriceUpdatesWithCallback` with an arbitrary `req.requester` address (their own malicious contract) and a `callbackGasLimit` they control. No privileged access, leaked key, or governance majority is required. The attacker pays the fee for the gas limit they set, but the provider pays the OOG cost on every retry. The attack is cheap to sustain and requires only deploying a one-function contract.

---

### Recommendation

Replace the `try/catch` dispatch in `Echo.executeCallback` with `excessivelySafeCall` (already used in `Entropy.sol`), capping the copied returndata to a small fixed size (e.g., 256 bytes):

```solidity
(bool success, bytes memory ret) = req.requester.excessivelySafeCall(
    req.callbackGasLimit,
    256,
    abi.encodeCall(IEchoConsumer._echoCallback, (sequenceNumber, priceFeeds))
);
if (!success) {
    emit PriceUpdateCallbackFailed(sequenceNumber, providerToCredit, priceIds, req.requester, string(ret));
} else {
    emitPriceUpdate(sequenceNumber, priceIds, priceFeeds);
}
```

This mirrors the protection already present in `Entropy.sol` at line 582–596. [2](#0-1)

### Proof of Concept

1. Deploy a malicious consumer:
```solidity
contract MaliciousEchoConsumer is IEchoConsumer {
    address private _echo;
    constructor(address echo) { _echo = echo; }
    function getEcho() internal view override returns (address) { return _echo; }
    function echoCallback(uint64, PythStructs.PriceFeed[] memory) internal override {
        assembly {
            // Return 512 KB of zeros; callee pays ~500k gas for memory expansion.
            // Outer executeCallback frame pays another ~500k gas to copy it.
            return(0, 0x80000)
        }
    }
}
```

2. Call `requestPriceUpdatesWithCallback` with `callbackGasLimit = 600_000` and `requester = address(maliciousConsumer)`. Pay the required fee.

3. Provider calls `executeCallback` with `gas = callbackGasLimit + reasonable_overhead` (e.g., 1.2M gas). The outer frame runs out of gas copying the 512 KB returndata buffer. Transaction reverts. `clearRequest` is undone. Provider's gas is lost.

4. Provider retries with more gas. Attacker's callback always returns the same large buffer. The provider must provide `callbackGasLimit + returndata_copy_cost + overhead` gas, where `returndata_copy_cost ≈ callbackGasLimit`, effectively doubling the required gas — an amount the provider did not budget for and that may exceed the block gas limit on constrained chains.
