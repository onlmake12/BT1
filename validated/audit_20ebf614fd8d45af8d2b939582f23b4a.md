### Title
Missing Duplicate-Input Validation in `NonContextualTransactionVerifier` Allows Capacity Inflation — (File: `verification/src/transaction_verifier.rs`)

### Summary

`NonContextualTransactionVerifier` enforces a `DuplicateDepsVerifier` check for `cell_deps` and `header_deps`, but has no equivalent check for duplicate transaction inputs. The `ResolvedTransaction::check()` function silently passes duplicate inputs via its `checked_cells` short-circuit, and `inputs_capacity()` sums every entry in `resolved_inputs` including duplicates. An unprivileged transaction sender can craft a transaction whose two inputs reference the same live cell, causing the `CapacityVerifier` to see a doubled input sum and accept outputs whose total capacity exceeds the actual cell capacity — creating CKB tokens out of thin air.

### Finding Description

**Root cause — missing non-contextual check:**

`NonContextualTransactionVerifier` in `verification/src/transaction_verifier.rs` runs six sub-verifiers:

```
version, size, empty, duplicate_deps, outputs_data_verifier, script_hash_type
```

`DuplicateDepsVerifier` (lines 428–459) explicitly rejects duplicate `cell_deps` and `header_deps`. There is no `DuplicateInputsVerifier`. The struct comment even lists "Check for duplicate deps" but says nothing about inputs.

**Root cause — `check()` silently passes duplicate inputs:**

`ResolvedTransaction::check()` in `util/types/src/core/cell.rs` (lines 309–385) uses a local `checked_cells: HashSet<OutPoint>`. For the first occurrence of an out-point it calls `cell_checker.is_live()` and inserts into `checked_cells`. For any subsequent occurrence of the **same** out-point it hits the early-return at line 321–323:

```rust
if checked_cells.contains(out_point) {
    return Ok(());   // duplicate input silently accepted
}
```

`seen_inputs` (the cross-transaction dead-cell guard) is only extended at the very end (line 382), so it cannot catch intra-transaction duplicates either.

**Root cause — `inputs_capacity()` double-counts:**

`ResolvedTransaction::inputs_capacity()` (lines 288–293) iterates over every element of `resolved_inputs` and sums their capacities:

```rust
self.resolved_inputs
    .iter()
    .map(CellMeta::capacity)
    .try_fold(Capacity::zero(), Capacity::safe_add)
```

If `resolved_inputs` contains two `CellMeta` entries for the same cell (because the transaction listed the same `OutPoint` twice), the capacity is counted twice.

**Attack path:**

1. Attacker calls `send_transaction` RPC (or relays via P2P) with a transaction whose `inputs` list contains the same `OutPoint` twice.
2. `non_contextual_verify` in `tx-pool/src/util.rs` (line 56) runs `NonContextualTransactionVerifier::verify()` — passes, no duplicate-input check.
3. `resolve_tx` resolves both inputs to the same live `CellMeta`; `resolved_inputs` now has two identical entries.
4. `ResolvedTransaction::check()` is called; the second duplicate hits `checked_cells` and returns `Ok(())`.
5. `CapacityVerifier::verify()` (lines 478–515) calls `inputs_capacity()` → returns `2 × cell_capacity`.
6. The verifier checks `inputs_sum >= outputs_sum`; with doubled input sum the attacker can create outputs totalling up to `2 × cell_capacity`.
7. The transaction is accepted into the pool and, if mined, into the canonical chain.

### Impact Explanation

A single unprivileged transaction sender can inflate CKB capacity: a cell holding C shannons can be used to fund outputs totalling 2C shannons. Repeating with N duplicate inputs yields N×C. This is a direct violation of CKB's core invariant that `outputs_capacity ≤ inputs_capacity`, enabling unbounded token creation and breaking the economic security of the chain.

### Likelihood Explanation

The entry point is the public `send_transaction` JSON-RPC method and the P2P relay path — both reachable by any unprivileged user with no keys beyond ownership of a single live cell. Constructing a transaction with a repeated input requires only standard tooling. No special timing, hashpower, or privileged access is needed.

### Recommendation

Add a `DuplicateInputsVerifier` to `NonContextualTransactionVerifier`, mirroring the existing `DuplicateDepsVerifier`:

```rust
// In NonContextualTransactionVerifier::verify():
self.duplicate_inputs.verify()?;
```

```rust
pub struct DuplicateInputsVerifier<'a> {
    transaction: &'a TransactionView,
}

impl<'a> DuplicateInputsVerifier<'a> {
    pub fn verify(&self) -> Result<(), Error> {
        let mut seen = HashSet::with_capacity(self.transaction.inputs().len());
        for input in self.transaction.inputs_iter() {
            let out_point = input.previous_output();
            if !seen.insert(out_point.clone()) {
                return Err(TransactionError::DuplicateInputs { out_point }.into());
            }
        }
        Ok(())
    }
}
```

This is a non-contextual check (requires no chain state) and should be placed alongside `DuplicateDepsVerifier` in `verification/src/transaction_verifier.rs`.

### Proof of Concept

1. Mine a block to obtain a live cell `C` with capacity 100 CKB at `OutPoint { tx_hash: H, index: 0 }`.
2. Construct a transaction:
   - `inputs`: `[CellInput { previous_output: (H, 0), since: 0 }, CellInput { previous_output: (H, 0), since: 0 }]`
   - `outputs`: one cell with capacity 200 CKB (lock = attacker's key)
   - `witnesses`: two valid signatures for the same key (lock script runs twice, both succeed)
3. Submit via `send_transaction` RPC.
4. Expected (current): transaction accepted; `inputs_capacity()` = 200 CKB ≥ `outputs_capacity()` = 200 CKB → `CapacityVerifier` passes.
5. Expected (fixed): rejected with `DuplicateInputs` error before reaching `CapacityVerifier`.

pub struct DuplicateDepsVerifier<'a> {
    transaction: &'a TransactionView,
}

impl<'a> DuplicateDepsVerifier<'a> {
    pub fn new(transaction: &'a TransactionView) -> Self {
        DuplicateDepsVerifier { transaction }
    }

    pub fn verify(&self) -> Result<(), Error> {
        let transaction = self.transaction;
        let mut seen_cells = HashSet::with_capacity(self.transaction.cell_deps().len());
        let mut seen_headers = HashSet::with_capacity(self.transaction.header_deps().len());

        if let Some(dep) = transaction
            .cell_deps_iter()
            .find_map(|dep| seen_cells.replace(dep))
        {
            return Err(TransactionError::DuplicateCellDeps {
                out_point: dep.out_point(),
            }
            .into());
        }
        if let Some(hash) = transaction
            .header_deps_iter()
            .find_map(|hash| seen_headers.replace(hash))
        {
            return Err(TransactionError::DuplicateHeaderDeps { hash }.into());
        }
        Ok(())
    }
}

    pub fn verify(&self) -> Result<(), Error> {
        // skip OutputsSumOverflow verification for resolved cellbase and DAO
        // withdraw transactions.
        // cellbase's outputs are verified by RewardVerifier
        // DAO withdraw transaction is verified via the type script of DAO cells
        if !(self.resolved_transaction.is_cellbase() || self.valid_dao_withdraw_transaction()) {
            let inputs_sum = self.resolved_transaction.inputs_capacity()?;
            let outputs_sum = self.resolved_transaction.outputs_capacity()?;

            if inputs_sum < outputs_sum {
                return Err((TransactionError::OutputsSumOverflow {
                    inputs_sum,
                    outputs_sum,
                })
                .into());
            }
        }

        for (index, (output, data)) in self
            .resolved_transaction
            .transaction
            .outputs_with_data_iter()
            .enumerate()
        {
            let data_occupied_capacity = Capacity::bytes(data.len())?;
            if output.is_lack_of_capacity(data_occupied_capacity)? {
                return Err((TransactionError::InsufficientCellCapacity {
                    index,
                    inner: TransactionErrorSource::Outputs,
                    capacity: output.capacity().into(),
                    occupied_capacity: output.occupied_capacity(data_occupied_capacity)?,
                })
                .into());
            }
        }

        Ok(())
    }

pub(crate) fn non_contextual_verify(
    consensus: &Consensus,
    tx: &TransactionView,
) -> Result<(), Reject> {
    NonContextualTransactionVerifier::new(tx, consensus)
        .verify()
        .map_err(Reject::Verification)?;

    // The ckb consensus does not limit the size of a single transaction,
    // but if the size of the transaction is close to the limit of the block,
    // it may cause the transaction to fail to be packed
    let tx_size = tx.data().serialized_size_in_block() as u64;
    if tx_size > TRANSACTION_SIZE_LIMIT {
        return Err(Reject::ExceededTransactionSizeLimit(
            tx_size,
            TRANSACTION_SIZE_LIMIT,
        ));
    }
    // cellbase is only valid in a block, not as a loose transaction
    if tx.is_cellbase() {
        return Err(Reject::Malformed(
            "cellbase like".to_owned(),
            Default::default(),
        ));
    }

    Ok(())
}
