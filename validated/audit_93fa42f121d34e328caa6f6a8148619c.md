### Title
Missing Duplicate-Input Deduplication Allows Capacity Double-Counting in Block Verification — (`util/types/src/core/cell.rs`, `verification/src/transaction_verifier.rs`)

---

### Summary

`NonContextualTransactionVerifier` explicitly rejects duplicate `CellDep` and `HeaderDep` entries, but has no analogous check for duplicate input `OutPoint`s within the same transaction. The `ResolvedTransaction::check()` function uses a shared `checked_cells` set for both input and dep liveness checks; a duplicate input silently passes on its second iteration. Because `inputs_capacity()` sums every entry in `resolved_inputs` without deduplication, a transaction whose inputs list contains the same `OutPoint` twice has its capacity counted twice, allowing outputs to exceed the real input capacity — i.e., creating CKB from nothing.

---

### Finding Description

**Step 1 — No duplicate-input verifier exists.**

`NonContextualTransactionVerifier` runs five sub-verifiers:

```
version / size / empty / duplicate_deps / outputs_data / script_hash_type
```

`DuplicateDepsVerifier` rejects duplicate `CellDep` and `HeaderDep` entries. There is no `DuplicateInputsVerifier`. [1](#0-0) 

**Step 2 — `check()` silently accepts duplicate inputs.**

`ResolvedTransaction::check()` initialises one `checked_cells: HashSet<OutPoint>` and uses it for both input and dep liveness checks:

```rust
let mut checked_cells: HashSet<OutPoint> = HashSet::new();
let mut check_cell = |out_point: &OutPoint| -> Result<(), OutPointError> {
    if seen_inputs.contains(out_point) { return Err(Dead); }
    if checked_cells.contains(out_point) { return Ok(()); }   // ← silent pass
    match cell_checker.is_live(out_point) { … }
};

for cell_meta in &self.resolved_inputs {
    check_cell(&cell_meta.out_point)?;
}
```

For the **first** occurrence of `OutPoint X`: not in `seen_inputs`, not in `checked_cells` → `is_live` → insert into `checked_cells` → `Ok`.  
For the **second** occurrence of `OutPoint X`: not in `seen_inputs`, **is** in `checked_cells` → `Ok` — no error raised. [2](#0-1) 

**Step 3 — `inputs_capacity()` sums every entry without deduplication.**

```rust
pub fn inputs_capacity(&self) -> CapacityResult<Capacity> {
    self.resolved_inputs
        .iter()
        .map(CellMeta::capacity)
        .try_fold(Capacity::zero(), Capacity::safe_add)
}
```

If `resolved_inputs` contains the same `CellMeta` twice (because the raw transaction listed the same `OutPoint` twice), the capacity is added twice. [3](#0-2) 

**Step 4 — `CapacityVerifier` uses the inflated sum.**

```rust
let inputs_sum = self.resolved_transaction.inputs_capacity()?;
let outputs_sum = self.resolved_transaction.outputs_capacity()?;
if inputs_sum < outputs_sum { return Err(OutputsSumOverflow …); }
```

With a doubled `inputs_sum`, outputs totalling up to `2 × cell_capacity` pass the check, even though only `1 × cell_capacity` is actually consumed. [4](#0-3) 

**Step 5 — The tx-pool edge check does not protect consensus.**

`Edges::insert_input` would reject a duplicate-input transaction at the mempool admission stage. However, a miner assembling a block directly (bypassing the tx-pool) can include such a transaction. Block verification in `resolve_block_transactions` calls `resolve_transaction` and then `ContextualBlockVerifier`, neither of which contains a duplicate-input guard. [5](#0-4) 

---

### Impact Explanation

A transaction with `N` copies of the same input `OutPoint` passes all consensus verification checks and is committed to the chain. `inputs_capacity()` returns `N × cell_capacity`, while only `1 × cell_capacity` is actually destroyed. The attacker can set outputs totalling up to `N × cell_capacity`, minting `(N-1) × cell_capacity` worth of CKB out of thin air. This is a direct capacity-inflation / double-spend at the consensus layer.

---

### Likelihood Explanation

The tx-pool's `insert_input` check prevents ordinary RPC users from submitting such transactions. However, any miner (an unprivileged role requiring only hashpower) can craft a block template that includes the malicious transaction directly, without routing it through the tx-pool. Once the block is mined and relayed, every full node's `ContextualBlockVerifier` accepts it because no consensus-layer duplicate-input check exists. The attack requires only that the attacker mine one valid block.

---

### Recommendation

1. Add a `DuplicateInputsVerifier` to `NonContextualTransactionVerifier` that iterates `tx.inputs_iter()`, inserts each `OutPoint` into a `HashSet`, and returns `TransactionError::DuplicateInputs` on collision — mirroring the existing `DuplicateDepsVerifier`. [6](#0-5) 

2. Fix `ResolvedTransaction::check()` to use a **separate** `HashSet` for input deduplication, distinct from `checked_cells` (which is legitimately shared for dep deduplication). [7](#0-6) 

---

### Proof of Concept

```
Attacker constructs a TransactionView where inputs = [CellInput(OutPoint X, 0), CellInput(OutPoint X, 0)]
(same OutPoint listed twice; cell X has capacity 100 CKB)

resolve_transaction produces:
  resolved_inputs = [CellMeta{out_point: X, capacity: 100}, CellMeta{out_point: X, capacity: 100}]

ResolvedTransaction::check():
  Iteration 1 (X): seen_inputs∌X, checked_cells∌X → is_live(X)=true → checked_cells={X} → Ok
  Iteration 2 (X): seen_inputs∌X, checked_cells∋X → Ok  ← no error

CapacityVerifier::verify():
  inputs_sum  = 100 + 100 = 200 CKB   ← inflated
  outputs_sum = 199 CKB               ← attacker keeps 199, burns 1 as fee
  200 >= 199 → PASS

Result: 99 CKB created from nothing; block committed to chain.
```

impl<'a> DuplicateDepsVerifier<'a> {
    pub fn new(transaction: &'a TransactionView) -> Self {
        DuplicateDepsVerifier { transaction }
    }

    pub fn verify(&self) -> Result<(), Error> {
        let transaction = self.transaction;
        let mut seen_cells = HashSet::with_capacity(self.transaction.cell_deps().len());
        let mut seen_headers = HashSet::with_capacity(self.transaction.header_deps().len());

        if let Some(dep) = transaction
            .cell_deps_iter()
            .find_map(|dep| seen_cells.replace(dep))
        {
            return Err(TransactionError::DuplicateCellDeps {
                out_point: dep.out_point(),
            }
            .into());
        }
        if let Some(hash) = transaction
            .header_deps_iter()
            .find_map(|hash| seen_headers.replace(hash))
        {
            return Err(TransactionError::DuplicateHeaderDeps { hash }.into());
        }
        Ok(())
    }
