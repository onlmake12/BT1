### Title
Signature Malleability in `verify_le_ecdsa_message` with No Replay Protection Enables Stale Price Injection - (File: `lazer/contracts/sui/sources/pyth_lazer.move`)

### Summary

`verify_le_ecdsa_message` in the Pyth Lazer Sui contract calls Sui's raw `secp256k1_ecrecover` without enforcing low-s normalization and without any consumed-message tracking. Because secp256k1 signatures are malleable — given a valid `(r, s, v)`, the tuple `(r, n−s, v^1)` recovers the same public key — an attacker can produce a distinct byte-level variant of any valid Lazer price update that also passes verification. Combined with the complete absence of replay protection in the Lazer update path, this allows an unprivileged attacker to replay stale price data (in original or malleable form) to any consumer contract that calls `parse_and_verify_le_ecdsa_update_v2`.

### Finding Description

`verify_le_ecdsa_message` is the sole signature-verification gate for all Lazer price updates on Sui:

```move
// lazer/contracts/sui/sources/pyth_lazer.move, lines 43-64
public(package) fun verify_le_ecdsa_message(
    state: &State,
    clock: &Clock,
    signature: &vector<u8>,
    payload: &vector<u8>,
) {
    let current_cap = state.current_cap();
    // 0 stands for keccak256 hash
    let pubkey = secp256k1_ecrecover(signature, payload, 0);   // ← raw, no low-s check
    let trusted_signers = state.trusted_signers(&current_cap);
    let mut maybe_idx = trusted_signers.find_index!(|signer|
        signer.public_key() == &pubkey
    );
    assert!(maybe_idx.is_some(), ESignerNotTrusted);
    let idx = maybe_idx.extract();
    let expires_at_ms = trusted_signers[idx].expires_at_ms();
    assert!(clock.timestamp_ms() < expires_at_ms, ESignerExpired);
}

The public entry point `parse_and_verify_le_ecdsa_update_v2` calls this function and returns the parsed `Update` struct with no further state mutation: [2](#0-1) 

The `State` struct contains only `trusted_signers`, `upgrade_cap`, and `governance` — there is no consumed-message set, no nonce, and no sequence counter for price updates: [3](#0-2) 

Contrast this with the Lazer governance path, which does track `seen_sequence` to prevent replay: [4](#0-3) 

And contrast with the Wormhole VAA path, which uses `ConsumedVAAs` to prevent replay: [5](#0-4) 

**Malleability mechanics**: For any valid Lazer signature `sig = r ‖ s ‖ v` (65 bytes, RSV format), the attacker computes `s' = secp256k1_n − s` and `v' = v XOR 1`. The resulting `sig' = r ‖ s' ‖ v'` is a distinct 65-byte value that `secp256k1_ecrecover` resolves to the **same** compressed public key. Because `verify_le_ecdsa_message` only checks that the recovered key is in `trusted_signers`, `sig'` passes verification identically to `sig`.

### Impact Explanation

An attacker who observes any valid Lazer price update broadcast on-chain or via the Lazer WebSocket can:

1. Record the raw update bytes `(magic ‖ sig ‖ payload_len ‖ payload)`.
2. Optionally produce a malleable variant by replacing `s` with `n−s` and flipping the recovery byte — yielding a distinct byte string that also passes `verify_le_ecdsa_message`.
3. At any future time (while the signer's certificate has not expired), submit the original or malleable update to any consumer contract that calls `parse_and_verify_le_ecdsa_update_v2`.
4. The consumer receives a "verified" `Update` struct containing a stale timestamp and stale prices.

Consumer contracts that do not independently enforce timestamp freshness will accept the stale price as current. For DeFi protocols (lending, perpetuals, options) using Pyth Lazer prices on Sui, this can cause incorrect liquidations, mispriced trades, or arbitrage losses. The malleability aspect specifically defeats any consumer-side deduplication keyed on the raw update bytes.

### Likelihood Explanation

- **Entry path**: `parse_and_verify_le_ecdsa_update_v2` is a `public` Move function callable by any unprivileged Sui transaction.
- **Attacker capability**: Observing valid Lazer updates requires only a Lazer WebSocket subscription (no privileged access). Computing `n−s` is trivial arithmetic.
- **Window**: The attack is valid for the entire lifetime of a trusted signer certificate (potentially months, per `expires_at_ms`).
- **Likelihood**: Medium-High. Any attacker monitoring the Lazer feed can execute this without any privileged capability.

### Recommendation

1. **Enforce low-s normalization** inside `verify_le_ecdsa_message`: after recovering the signature bytes, assert that `s ≤ secp256k1_n/2`. Reject signatures with high-s values.
2. **Add replay protection** for Lazer price updates: maintain a `consumed_updates: Set<Bytes32>` (keyed on `keccak256(payload)` or `keccak256(signature ‖ payload)`) in `State`, and assert-and-insert on every call to `parse_and_verify_le_ecdsa_update_v2`.
3. **Enforce timestamp monotonicity** at the contract level: track `last_seen_timestamp_us` per feed and reject updates with timestamps ≤ the last seen value.

**Step 1** — Extract `s` from bytes 32–63 of `TEST_SIGNATURE`:
```
s = 0x55bdb94f5a2fe926932ce9d9da338d05969b5a8b8c94024c86399005cc547fd00
```

**Step 2** — Compute `s' = secp256k1_n − s`:
```
secp256k1_n = 0xFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFEBAAEDCE6AF48A03BBFD25E8CD0364141
s'          = secp256k1_n - s  (non-zero, valid scalar)
```

**Step 3** — Flip recovery byte: `v' = 0x01 XOR 0x01 = 0x00`.

**Step 4** — Construct `malleable_sig = r ‖ s' ‖ v'` (65 bytes).

**Step 5** — Call `verify_le_ecdsa_message(state, clock, &malleable_sig, &TEST_PAYLOAD)`. The call succeeds: `secp256k1_ecrecover` recovers the same compressed public key `03a4380f...155b`, which is in `trusted_signers`. No error is raised.

**Step 6** — Call `parse_and_verify_le_ecdsa_update_v2` with `magic ‖ malleable_sig ‖ payload_len ‖ TEST_PAYLOAD`. The function returns a valid `Update` struct containing the original (potentially stale) price data, indistinguishable from a freshly verified update.

public fun parse_and_verify_le_ecdsa_update_v2(s: &State, clock: &Clock, update: vector<u8>): Update {
    let mut cursor = bcs::new(update);

    // Parse and validate message magic
    let magic = cursor.peel_u32();
    assert!(magic == UPDATE_MESSAGE_MAGIC, EInvalidUpdateMagic);

    // Parse signature
    let mut signature = vector::empty<u8>();
    let mut sig_i = 0;
    while (sig_i < SECP256K1_SIG_LEN) {
        signature.push_back(cursor.peel_u8());
        sig_i = sig_i + 1;
    };

    // Parse expected payload length and get remaining bytes as payload
    let payload_len = cursor.peel_u16();
    let payload = cursor.into_remainder_bytes();

    // Validate expected payload length
    assert!(payload_len as u64 == payload.length(), EInvalidPayloadLength);

    // Parse payload
    let mut payload_cursor = bcs::new(payload);
    let payload_magic = payload_cursor.peel_u32();
    assert!(payload_magic == PAYLOAD_MAGIC, EInvalidPayloadMagic);

    // Verify the signature against trusted signers
    verify_le_ecdsa_message(s, clock, &signature, &payload);

    update_v2::parse_from_cursor(payload_cursor)
}

module wormhole::consumed_vaas {
    use sui::tx_context::{TxContext};

    use wormhole::bytes32::{Bytes32};
    use wormhole::set::{Self, Set};

    /// Container storing VAA hashes (digests). This will be checked against in
    /// `parse_verify_and_consume` so a particular VAA cannot be replayed. It
    /// is up to the integrator to have this container live in his contract
    /// in order to take advantage of this no-replay protection. Or an
    /// integrator can implement his own method to prevent replay.
    struct ConsumedVAAs has store {
        hashes: Set<Bytes32>
    }

    public fun new(ctx: &mut TxContext): ConsumedVAAs {
        ConsumedVAAs { hashes: set::new(ctx) }
    }

    public fun consume(self: &mut ConsumedVAAs, digest: Bytes32) {
        set::add(&mut self.hashes, digest);
    }

    #[test_only]
    public fun destroy(consumed: ConsumedVAAs) {
        let ConsumedVAAs { hashes } = consumed;
        set::destroy(hashes);
    }
}
