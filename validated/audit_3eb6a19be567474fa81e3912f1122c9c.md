### Title
`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` Always Revert for Whitelist-Enabled Subscriptions Due to `msg.sender` Role Conflict in Internal `this.` Call - (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`Scheduler.sol` contains two view functions — `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` — that each apply the `onlyWhitelistedReader` access-control modifier and then internally call `this.getPricesUnsafe` / `this.getEmaPricesUnsafe`, which also carry the same modifier. Because the call is made via `this.`, it is an **external call**, so `msg.sender` inside the callee becomes the **Scheduler contract address itself** rather than the original caller. For any subscription with `whitelistEnabled = true`, the Scheduler contract is neither the subscription manager nor a whitelisted reader, so the inner modifier always reverts with `Unauthorized`. The two "safe" price-reading functions are permanently broken for every whitelist-enabled subscription.

---

### Finding Description

`getPricesNoOlderThan` is declared `external view` with the `onlyWhitelistedReader(subscriptionId)` modifier. After its staleness check it delegates to `getPricesUnsafe` via `this.getPricesUnsafe(subscriptionId, priceIds)`:

```solidity
// Scheduler.sol line 535-555
function getPricesNoOlderThan(
    uint256 subscriptionId,
    bytes32[] calldata priceIds,
    uint256 age_seconds
)
    external view override
    onlyWhitelistedReader(subscriptionId)   // ← passes for the real caller
    returns (PythStructs.Price[] memory prices)
{
    ...
    prices = this.getPricesUnsafe(subscriptionId, priceIds);  // ← external call
}
```

`getPricesUnsafe` is also `external view` with the same modifier:

```solidity
// Scheduler.sol line 514-533
function getPricesUnsafe(...)
    external view override
    onlyWhitelistedReader(subscriptionId)   // ← now msg.sender == address(this)
    ...
```

The modifier logic:

```solidity
// Scheduler.sol line 743-779
modifier onlyWhitelistedReader(uint256 subscriptionId) {
    if (_state.subscriptionManager[subscriptionId] == msg.sender) { _; return; }
    if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) { _; return; }
    // whitelist check ...
    if (!isWhitelisted) revert SchedulerErrors.Unauthorized();
    _;
}
```

When `getPricesNoOlderThan` calls `this.getPricesUnsafe(...)`, the EVM issues a new external call. Inside `getPricesUnsafe`, `msg.sender` is `address(this)` — the Scheduler proxy contract. The Scheduler contract is never stored as a subscription manager and is never added to a reader whitelist. Therefore:

- If `whitelistEnabled == false`: the second branch short-circuits and the call succeeds (no bug).
- If `whitelistEnabled == true`: the Scheduler contract is not in the whitelist → `revert Unauthorized`.

The identical pattern exists in `getEmaPricesNoOlderThan` → `this.getEmaPricesUnsafe` at line 597.

---

### Impact Explanation

For every subscription created with `whitelistEnabled = true`, the functions `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` are permanently and completely broken. No caller — including the subscription manager — can successfully invoke them. The only working read path is the unchecked `getPricesUnsafe` / `getEmaPricesUnsafe`, which provide no staleness guarantee. Integrators who rely on the age-validated functions (the recommended safe API per the SDK README) receive no prices and cannot detect stale data, leading to silent service unavailability for all whitelist-gated subscriptions.

---

### Likelihood Explanation

The `whitelistEnabled` flag is a first-class feature of `SubscriptionParams` and is explicitly documented in the Pulse README and SDK. Any subscription manager who enables the whitelist (a common pattern for protocols that want to restrict price-feed access) will trigger this bug the first time a reader calls `getPricesNoOlderThan`. No special attacker action is required; normal usage of the documented API is sufficient to hit the revert.

---

### Recommendation

Replace the `this.` external calls with direct internal calls to the shared helper `_getPricesInternal`, bypassing the re-entrant modifier:

```solidity
function getPricesNoOlderThan(...) external view onlyWhitelistedReader(subscriptionId) ... {
    if (distance(block.timestamp, status.priceLastUpdatedAt) > age_seconds)
        revert PythErrors.StalePrice();
    // Call internal helper directly instead of this.getPricesUnsafe(...)
    PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
    prices = new PythStructs.Price[](priceFeeds.length);
    for (uint i = 0; i < priceFeeds.length; i++) prices[i] = priceFeeds[i].price;
}
```

Apply the same fix to `getEmaPricesNoOlderThan`.

---

### Proof of Concept

1. Deploy `SchedulerUpgradeable`.
2. Manager calls `createSubscription` with `whitelistEnabled = true` and `readerWhitelist = [readerAddress]`.
3. Keeper calls `updatePriceFeeds` — succeeds, prices stored.
4. `readerAddress` calls `getPricesUnsafe(subscriptionId, priceIds)` — succeeds (modifier sees `msg.sender == readerAddress` ∈ whitelist).
5. `readerAddress` calls `getPricesNoOlderThan(subscriptionId, priceIds, 60)` — **reverts with `Unauthorized`** because internally `this.getPricesUnsafe(...)` is called as an external call, making `msg.sender == address(scheduler)` inside `getPricesUnsafe`, which is not in the whitelist.
6. Even the subscription manager calling `getPricesNoOlderThan` reverts for the same reason.

        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
