### Title
Decompression Bomb via Off-by-One in `LengthDelimitedCodecWithCompress::decode` Allows 8 MB Heap Allocation Per Message Regardless of Per-Protocol `max_frame_length` — (`network/src/compress.rs`)

---

### Summary

`LengthDelimitedCodecWithCompress::decode` enforces `max_frame_length` only on the **compressed wire size**, then checks the decompressed length against the global `MAX_UNCOMPRESSED_LEN` (8 MB) using a **strict greater-than** (`>`). A crafted snappy frame whose `decompress_len()` returns exactly `8388608` passes both checks and causes an unconditional `BytesMut::zeroed(8388608)` allocation, completely bypassing the per-protocol frame limit (e.g., Ping = 1 KB, Identify = 2 KB).

---

### Finding Description

**Check 1 — wire size (compressed):** `self.length_delimited.decode(src)` enforces `max_frame_length` on the raw bytes received over the wire. [1](#0-0) 

**Check 2 — decompressed size:** The guard uses strict `>`, so `decompressed_bytes_len == MAX_UNCOMPRESSED_LEN` (exactly 8 MB) is **allowed through**. [2](#0-1) 

**Allocation:** Immediately after the check passes, an 8 MB buffer is unconditionally zeroed. [3](#0-2) 

The per-protocol limits are:

| Protocol | `max_frame_length` |
|---|---|
| Ping | 1 KB |
| Identify | 2 KB |
| Feeler / DisconnectMessage / Time | 1 KB |
| Sync | 2 MB |
| RelayV3 | 4 MB | [4](#0-3) 

All of these protocols use `CKBProtocol::build()`, which wires `LengthDelimitedCodecWithCompress` as the codec with the per-protocol `max_frame_length` passed only to the inner `length_delimited` codec (i.e., it guards the wire size, not the decompressed size). [5](#0-4) 

Snappy can compress 8 MB of repetitive data (e.g., all-zero bytes) into well under 1 KB. Therefore, a Ping-protocol attacker can send a ~100-byte compressed frame (within the 1 KB wire limit) that decompresses to exactly 8 MB, triggering the full allocation.

---

### Impact Explanation

Each inbound connection on each protocol can trigger one 8 MB allocation per message. With the default max inbound peer count multiplied across all open protocols per session, an attacker filling all inbound slots and sending one crafted frame per protocol per connection can exhaust the victim node's heap, causing an OOM crash. Because honest nodes relay compressed frames without re-compressing, the same crafted frame propagates to all peers that received it, taking down the entire network simultaneously.

---

### Likelihood Explanation

The attack requires only:
1. A TCP connection (no authentication, no PoW, no stake).
2. A valid TLS/secio handshake (standard peer connection).
3. A single crafted snappy stream — trivially constructable with the `snap` crate.

No privileged access, no leaked keys, no majority hashpower. The path is fully reachable by any unprivileged remote peer.

---

### Recommendation

Replace the strict `>` with `>=` in the decompressed-length guard:

```rust
// network/src/compress.rs, line 235
if decompressed_bytes_len >= MAX_UNCOMPRESSED_LEN {  // was: >
```

More importantly, add a **second guard** that bounds the decompressed size to the protocol's own `max_frame_length`. `LengthDelimitedCodecWithCompress` already holds a reference to `self.length_delimited` whose `max_frame_length()` is accessible:

```rust
if decompressed_bytes_len > self.length_delimited.max_frame_length() {
    return Err(io::ErrorKind::InvalidData.into());
}
```

This ensures the post-decompression payload never exceeds the protocol's intended message size limit, regardless of `MAX_UNCOMPRESSED_LEN`. [6](#0-5) 

---

### Proof of Concept

```rust
// Craft a snappy stream that decompresses to exactly 8_388_608 bytes of zeros.
// The resulting compressed bytes will be well under 1 KB.
let uncompressed = vec![0u8; 8_388_608]; // exactly MAX_UNCOMPRESSED_LEN
let compressed = snap::raw::Encoder::new().compress_vec(&uncompressed).unwrap();
assert!(compressed.len() < 1024); // fits within Ping's max_frame_length

// Build the wire frame: 4-byte length prefix + compress_flag (0x80) + compressed payload
let mut frame = BytesMut::new();
let payload_len = compressed.len() + 1;
frame.put_uint(payload_len as u64, 4);
frame.put_u8(0x80); // COMPRESS_FLAG
frame.extend_from_slice(&compressed);

// Feed into the Ping codec (max_frame_length = 1024)
let mut codec = LengthDelimitedCodecWithCompress::new(
    true,
    length_delimited::Builder::new().max_frame_length(1024).new_codec(),
    0.into(),
);
// decode() will allocate BytesMut::zeroed(8_388_608) — 8 MB — for a 1 KB Ping message
let result = codec.decode(&mut frame).unwrap();
assert_eq!(result.unwrap().len(), 8_388_608);
```
