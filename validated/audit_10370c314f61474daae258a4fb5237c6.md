### Title
External Self-Call in `getPricesNoOlderThan`/`getEmaPricesNoOlderThan` Bypasses `onlyWhitelistedReader`, Causing Permanent DoS for Whitelist-Enabled Subscriptions — (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`Scheduler.getPricesNoOlderThan()` and `getEmaPricesNoOlderThan()` each perform an external self-call (`this.getPricesUnsafe(...)` / `this.getEmaPricesUnsafe(...)`) to delegate the actual price-fetch logic. Because these are external calls, `msg.sender` inside the callee becomes the Scheduler contract itself rather than the original caller. Both callees are guarded by the `onlyWhitelistedReader` modifier, which checks `msg.sender` against the subscription manager and the reader whitelist. The Scheduler contract is never the manager and is never in the whitelist, so whenever `whitelistEnabled = true` the inner call always reverts with `Unauthorized`, making both "no-older-than" functions permanently unusable for every caller on every whitelist-enabled subscription.

---

### Finding Description

`getPricesNoOlderThan()` is declared `external … onlyWhitelistedReader(subscriptionId)` and, after its staleness check, delegates to `this.getPricesUnsafe(subscriptionId, priceIds)`: [1](#0-0) 

`getPricesUnsafe()` is itself declared `external … onlyWhitelistedReader(subscriptionId)`: [2](#0-1) 

The `onlyWhitelistedReader` modifier resolves `msg.sender` at the point of the inner call. Because `this.getPricesUnsafe(...)` is an external call, the EVM sets `msg.sender = address(this)` (the Scheduler contract) inside `getPricesUnsafe`. The modifier then:

1. Checks `_state.subscriptionManager[subscriptionId] == msg.sender` — the Scheduler contract is never the manager.
2. If `whitelistEnabled == false`, allows any reader — no issue in this branch.
3. If `whitelistEnabled == true`, iterates the whitelist looking for `msg.sender` (the Scheduler contract) — it is never there — and reverts `Unauthorized`. [3](#0-2) 

The identical pattern exists in `getEmaPricesNoOlderThan()` → `this.getEmaPricesUnsafe(...)`: [4](#0-3) 

**Call chain:**
```
User → getPricesNoOlderThan() [onlyWhitelistedReader: user passes]
         └─ this.getPricesUnsafe() [msg.sender = Scheduler; onlyWhitelistedReader: Scheduler fails → Unauthorized]
```

**Root cause:** the use of `this.getPricesUnsafe(...)` / `this.getEmaPricesUnsafe(...)` as external self-calls changes the call context so that `msg.sender` inside the callee is the Scheduler contract, not the original whitelisted reader. The `onlyWhitelistedReader` guard in the callee then compares the Scheduler contract's address against the whitelist, where it is never present.

---

### Impact Explanation

**Medium.** When a subscription is created with `whitelistEnabled = true`, `getPricesNoOlderThan()` and `getEmaPricesNoOlderThan()` revert with `Unauthorized` for **every** caller — including the subscription manager and every explicitly whitelisted address. The two functions are completely non-functional for all whitelist-enabled subscriptions. Consumers that depend on freshness guarantees (the primary purpose of these functions) cannot obtain price data through the intended API. The whitelist feature — the access-control mechanism that Scheduler subscriptions are built around — is rendered incompatible with the staleness-checked read path. No funds are at risk, but a core product feature is permanently broken for any subscription that uses access control.

---

### Likelihood Explanation

**High.** The whitelist is a first-class feature of the Scheduler product; any subscription operator who enables it to restrict who can read their price data will encounter this bug deterministically on the first call to `getPricesNoOlderThan()` or `getEmaPricesNoOlderThan()`. The failure is 100% reproducible and requires no special conditions beyond `whitelistEnabled = true`. The only workaround — disabling the whitelist — defeats the purpose of access control.

---

### Recommendation

Replace the external self-calls with direct internal calls. Extract the shared logic of `getPricesUnsafe` and `getEmaPricesUnsafe` into `internal` helper functions (e.g., `_getPricesUnsafeInternal`) and call those helpers from both the `external` entry-points and from `getPricesNoOlderThan` / `getEmaPricesNoOlderThan`. This preserves the access-control check on the outer function while avoiding the `msg.sender` context switch:

```solidity
// Before (broken):
prices = this.getPricesUnsafe(subscriptionId, priceIds);

// After (fixed):
prices = _getPricesUnsafeInternal(subscriptionId, priceIds);
```

The `_getPricesInternal` helper already exists and is used by both `getPricesUnsafe` and `getEmaPricesUnsafe`; `getPricesNoOlderThan` can call `_getPricesInternal` directly after its staleness check, bypassing the redundant outer `onlyWhitelistedReader` guard that has already been satisfied. [5](#0-4)

```solidity

pragma solidity ^0.8.0;

// Demonstrates that getPricesNoOlderThan always reverts Unauthorized
// for a whitelist-enabled subscription, even when called by a whitelisted reader.

function test_getPricesNoOlderThan_whitelist_DoS() public {
    // 1. Manager creates a subscription with whitelistEnabled = true
    //    and adds `reader` to the whitelist.
    address[] memory whitelist = new address[](1);
    whitelist[0] = reader;
    SchedulerStructs.SubscriptionParams memory params = _buildParams(
        priceIds,
        /*whitelistEnabled=*/true,
        whitelist
    );
    vm.prank(manager);
    uint256 subId = scheduler.createSubscription{value: MIN_BALANCE}(params);

    // 2. Keeper pushes a fresh price update.
    vm.prank(keeper);
    scheduler.updatePriceFeeds(subId, updateData);

    // 3. Whitelisted reader calls getPricesNoOlderThan — should succeed,
    //    but always reverts Unauthorized due to the external self-call bug.
    vm.prank(reader);
    vm.expectRevert(SchedulerErrors.Unauthorized.selector);
    scheduler.getPricesNoOlderThan(subId, priceIds, 60);

    // 4. Even the manager is affected.
    vm.prank(manager);
    vm.expectRevert(SchedulerErrors.Unauthorized.selector);
    scheduler.getPricesNoOlderThan(subId, priceIds, 60);

    // 5. getPricesUnsafe works fine for the same caller (no self-call involved).
    vm.prank(reader);
    scheduler.getPricesUnsafe(subId, priceIds); // succeeds
}
```

The test shows that `getPricesNoOlderThan` reverts for both the manager and a whitelisted reader, while `getPricesUnsafe` (which does not use an external self-call) succeeds for the same callers under the same subscription.

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
