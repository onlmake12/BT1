### Title
Unprivileged Caller Can Permanently Brick `InitializePublisher` by Setting an Uncontrollable Authority During `Initialize` — (`target_chains/solana/programs/pyth-price-store/src/processor/initialize.rs`)

### Summary

The `initialize` instruction accepts an arbitrary `args.authority: [u8; 32]` from instruction data and writes it verbatim into the config PDA with no validation that the caller controls that key. Any signer can race the legitimate operator and set authority to `[0u8; 32]` or any other uncontrollable pubkey. Because the config PDA can only be initialized once and no update-authority instruction exists, this permanently prevents `InitializePublisher` from ever succeeding.

---

### Finding Description

`validate_payer` only checks that the first account is a signer and writable — it does not require the payer to be any specific key, and it does not require the payer to match `args.authority`. [1](#0-0) 

`initialize` then writes `args.authority` directly into the config account without any validation: [2](#0-1) 

`accounts::config::create` only guards against re-initialization (non-zero `format`), not against an invalid authority value: [3](#0-2) 

Once the config PDA exists with a poisoned authority, `validate_authority` in every subsequent `InitializePublisher` call will always fail because it requires the signer's key bytes to equal `config.authority`: [4](#0-3) 

There is no instruction in the program to update or replace the stored authority. The full instruction set is `Initialize`, `SubmitPrices`, and `InitializePublisher` — no admin override exists. [5](#0-4) 

---

### Impact Explanation

- `InitializePublisher` is permanently blocked; no publisher can ever be registered.
- `SubmitPrices` depends on a publisher config that can only be created via `InitializePublisher`, so price submission is also frozen for all new publishers.
- Recovery requires a program upgrade (redeployment), which is an out-of-band privileged operation.

---

### Likelihood Explanation

The attacker needs only to submit a valid `Initialize` transaction before the legitimate operator. This requires no special privilege, no leaked key, and no governance majority. It is a simple front-run or a race on a fresh deployment. The attack is one transaction.

---

### Recommendation

1. **Require the authority to co-sign `Initialize`**: add a check that the account at position `key[0]` (payer) matches `args.authority`, or add a separate required signer account that must equal `args.authority`.
2. **Reject the zero pubkey**: explicitly reject `args.authority == [0u8; 32]` in `accounts::config::create`.
3. **Add an `UpdateAuthority` instruction** gated on the current authority's signature to allow recovery from misconfiguration.

---

    account: Option<&'b AccountInfo<'a>>,
    config: &AccountInfo<'a>,

    let authority = account.ok_or(ProgramError::NotEnoughAccountKeys)?;
    ensure!(ProgramError::MissingRequiredSignature, authority.is_signer);
    ensure!(ProgramError::InvalidArgument, authority.is_writable);
    let config_data = config.data.borrow();
    let config = accounts::config::read(*config_data)?;
    ensure!(

pub fn initialize(
    program_id: &Pubkey,
    accounts: &[AccountInfo],
    args: &InitializeArgs,
) -> ProgramResult {
    let mut accounts = accounts.iter();
    let payer = validate_payer(accounts.next())?;
    let config = validate_config(accounts.next(), args.config_bump, program_id, true)?;
    let system = validate_system(accounts.next())?;

    let lamports = (Rent::get()?).minimum_balance(accounts::config::SIZE);

    invoke_signed(
        &system_instruction::create_account(
            payer.key,
            config.key,
            lamports,
            accounts::config::SIZE
                .try_into()
                .expect("unexpected overflow"),
            program_id,
        ),
        &[payer.clone(), config.clone(), system.clone()],
        &[&[CONFIG_SEED.as_bytes(), &[args.config_bump]]],
    )?;

    accounts::config::create(*config.data.borrow_mut(), args.authority)?;

    Ok(())
}

pub fn create(data: &mut [u8], authority: [u8; 32]) -> Result<&mut Config, ReadAccountError> {
    if data.len() < size_of::<Config>() {
        return Err(ReadAccountError::DataTooShort);
    }
    let data: &mut Config = from_bytes_mut(&mut data[..size_of::<Config>()]);
    if data.format != 0 {
        return Err(ReadAccountError::AlreadyInitialized);
    }
    data.format = FORMAT;
    data.authority = authority;
    Ok(data)
