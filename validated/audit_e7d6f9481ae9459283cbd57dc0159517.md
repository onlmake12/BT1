### Title
Provider-Controlled `defaultGasLimit` Without Minimum Bound Enables Disproportionate Fee Inflation in Entropy — (File: `target_chains/ethereum/contracts/contracts/entropy/Entropy.sol`)

---

### Summary

In `Entropy.sol`, the `getProviderFee` function uses `provider.defaultGasLimit` as a **divisor** in a proportional fee scaling formula. Any registered provider can call `setDefaultGasLimit(1)` — a value far below the minimum rounded gas unit of 10,000 — causing the fee for any request with `gasLimit > 0` to be inflated by up to `~MAX_GAS_LIMIT × feeInWei`. This is a direct analog to the reported "User Controlled Options Price" pattern: a user-controlled parameter is used as a divisor without a minimum bound, producing disproportionately large output values.

---

### Finding Description

`setDefaultGasLimit` accepts any `uint32` value from 0 to `MAX_GAS_LIMIT` with no minimum bound enforced:

```solidity
function setDefaultGasLimit(uint32 gasLimit) external override {
    ...
    roundTo10kGas(gasLimit); // only checks MAX, not MIN
    provider.defaultGasLimit = gasLimit;
}
``` [1](#0-0) 

The fee scaling formula in `getProviderFee` then uses `provider.defaultGasLimit` as the denominator:

```solidity
uint128 additionalFee = ((roundedGasLimit - provider.defaultGasLimit) * provider.feeInWei)
                        / provider.defaultGasLimit;
return provider.feeInWei + additionalFee;
``` [2](#0-1) 

The `roundTo10kGas` helper rounds any non-zero `gasLimit` **up** to the nearest 10,000, meaning the minimum possible `roundedGasLimit` for any `gasLimit ≥ 1` is **10,000**: [3](#0-2) 

**Attack scenario:**

A provider calls `setDefaultGasLimit(1)` and `setProviderFee(1)`.

| User `gasLimit` | `roundedGasLimit` | `additionalFee` | Total provider fee |
|---|---|---|---|
| 0 | 0 | 0 (condition false) | 1 wei |
| 1 | 10,000 | `(10000 - 1) * 1 / 1 = 9,999` | **10,000 wei** |
| 655,350,000 (MAX) | 655,350,000 | `≈ 655,350,000` | **~655,350,001 wei** |

The fee jumps from 1 wei (for `gasLimit = 0`) to 10,000 wei (for `gasLimit = 1`) — a **10,000× inflation** — and scales linearly up to **~655,350,001× inflation** at the maximum gas limit.

---

### Impact Explanation

Any user calling `requestV2(provider, gasLimit)` with `gasLimit > 0` against a provider that has set `defaultGasLimit = 1` will be required to pay a fee that is disproportionately large relative to the provider's advertised `feeInWei`. The provider collects these inflated fees. If the affected provider is the **default provider** (set by the protocol admin), every user of the Entropy protocol on that chain is impacted. Even for non-default providers, users who query `getFeeV2(provider, 0)` to estimate cost and then call `requestV2(provider, nonZeroGasLimit)` will receive a revert due to `InsufficientFee`, causing denial of service. [4](#0-3) 

---

### Likelihood Explanation

Any address can register as an Entropy provider permissionlessly. Once registered, `setDefaultGasLimit` is callable by the provider with no additional access control beyond `provider.sequenceNumber != 0`. The attack requires no privileged key, no governance majority, and no external oracle manipulation. A single transaction sets `defaultGasLimit = 1`. [5](#0-4) 

---

### Recommendation

Enforce a minimum value for `defaultGasLimit` in `setDefaultGasLimit`. The minimum should be at least `TEN_THOUSAND` (10,000), which is the smallest value `roundTo10kGas` can produce for any non-zero input. This ensures the divisor in `getProviderFee` is never smaller than the minimum possible `roundedGasLimit`, preventing fee inflation:

```solidity
if (gasLimit != 0 && gasLimit < TEN_THOUSAND) {
    revert EntropyErrors.GasLimitTooLow();
}
```

---

### Proof of Concept

1. Attacker registers as a provider via `register(...)`.
2. Attacker calls `setDefaultGasLimit(1)` and `setProviderFee(1)`.
3. Victim calls `getFeeV2(attacker, 0)` → returns `pythFeeInWei + 1` (appears cheap).
4. Victim calls `requestV2{value: pythFeeInWei + 1}(attacker, bytes32(0), 100_000)` → **reverts** with `InsufficientFee` because the actual fee is `pythFeeInWei + 10 * feeInWei = pythFeeInWei + 10` (for 100,000 gas rounded to 100,000 / 10,000 = 10 units, `additionalFee = (100000 - 1) * 1 / 1 = 99999`).
5. Victim must pay `pythFeeInWei + 100,000` wei instead of `pythFeeInWei + 1` wei — a **100,000× overcharge** — to successfully request entropy. [6](#0-5) [1](#0-0)

    function getProviderFee(
        address providerAddr,
        uint32 gasLimit
    ) internal view returns (uint128 feeAmount) {
        EntropyStructsV2.ProviderInfo memory provider = _state.providers[
            providerAddr
        ];

        // Providers charge a minimum of their configured feeInWei for every request.
        // Requests using more than the defaultGasLimit get a proportionally scaled fee.
        // This approach may be somewhat simplistic, but it allows us to continue using the
        // existing feeInWei parameter for the callback failure flow instead of defining new
        // configuration values.
        uint32 roundedGasLimit = uint32(roundTo10kGas(gasLimit)) * TEN_THOUSAND;
        if (
            provider.defaultGasLimit > 0 &&
            roundedGasLimit > provider.defaultGasLimit
        ) {
            // This calculation rounds down the fee, which means that users can get some gas in the callback for free.
            // However, the value of the free gas is < 1 wei, which is insignificant.
            uint128 additionalFee = ((roundedGasLimit -
                provider.defaultGasLimit) * provider.feeInWei) /
                provider.defaultGasLimit;
            return provider.feeInWei + additionalFee;
        } else {
            return provider.feeInWei;
        }
    }

    // Set the default gas limit for a request.
    function setDefaultGasLimit(uint32 gasLimit) external override {
        EntropyStructsV2.ProviderInfo storage provider = _state.providers[
            msg.sender
        ];
        if (provider.sequenceNumber == 0) {
            revert EntropyErrors.NoSuchProvider();
        }

        // Check that we can round the gas limit into the 10k gas. This reverts
        // if the provided value exceeds the max.
        roundTo10kGas(gasLimit);

        uint32 oldGasLimit = provider.defaultGasLimit;
        provider.defaultGasLimit = gasLimit;
        emit ProviderDefaultGasLimitUpdated(msg.sender, oldGasLimit, gasLimit);
        emit EntropyEventsV2.ProviderDefaultGasLimitUpdated(
            msg.sender,
            oldGasLimit,
            gasLimit,
            bytes("")
        );
    }

    function roundTo10kGas(uint32 gas) internal pure returns (uint16) {
        if (gas > MAX_GAS_LIMIT) {
            revert EntropyErrors.MaxGasLimitExceeded();
        }

        uint32 gas10k = gas / TEN_THOUSAND;
        if (gas10k * TEN_THOUSAND < gas) {
            gas10k += 1;
        }
        // Note: safe cast here should never revert due to the if statement above.
        return SafeCast.toUint16(gas10k);
    }
