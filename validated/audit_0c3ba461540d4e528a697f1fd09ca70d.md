### Title
Governance `SetValidPeriod` Action Lacks Minimum and Maximum Bounds on `validTimePeriodSeconds` — (File: `target_chains/ethereum/contracts/contracts/pyth/PythGovernance.sol`)

---

### Summary

The Pyth governance action `SetValidPeriod` allows the governance authority to set `validTimePeriodSeconds` to any arbitrary `uint64` value — including zero or `type(uint64).max` — with no bounds validation. Setting it to zero causes a permanent DoS on all `getPrice()` and `getEmaPrice()` calls across every integrated protocol. Setting it to an astronomically large value silently disables staleness protection, allowing arbitrarily old prices to be consumed as fresh.

---

### Finding Description

`parseSetValidPeriodPayload` in `PythGovernanceInstructions.sol` decodes the new valid period directly from the governance VAA payload as a raw `uint64` with no minimum or maximum check:

```solidity
svp.newValidPeriod = uint256(encodedPayload.toUint64(index));
```

`setValidPeriod` in `PythGovernance.sol` then writes this value unconditionally:

```solidity
function setValidPeriod(SetValidPeriodPayload memory payload) internal {
    uint oldValidPeriod = validTimePeriodSeconds();
    setValidTimePeriodSeconds(payload.newValidPeriod);
    emit ValidPeriodSet(oldValidPeriod, validTimePeriodSeconds());
}
```

The same unbounded pattern exists on every other chain implementation:
- Aptos: `set_stale_price_threshold.move` — `state::set_stale_price_threshold_secs(threshold)` with no bounds
- Sui: `set_stale_price_threshold.move` — identical pattern
- Fuel: `parse_set_valid_period_payload` — raw `u64` with no bounds
- Stylus: `set_valid_period` — `U256::from(valid_time_period_seconds)` with no bounds

The existing test `testSetValidPeriod` in `PythGovernance.t.sol` explicitly demonstrates that setting the value to `0` succeeds without revert, confirming the absence of any lower-bound guard.

---

### Impact Explanation

**If `validTimePeriodSeconds` is set to `0`:**

`AbstractPyth.sol`'s `getPrice()` and `getEmaPrice()` delegate to `getPriceNoOlderThan(id, getValidTimePeriod())`:

```solidity
if (diff(block.timestamp, price.publishTime) > age)
    revert PythErrors.StalePrice();
```

With `age = 0`, any price whose `publishTime` differs from the current block timestamp by even one second reverts with `StalePrice`. Since price updates are never published at the exact current block timestamp, this permanently breaks `getPrice()` and `getEmaPrice()` for every consumer of the Pyth oracle across all integrated DeFi protocols — lending, derivatives, and any protocol relying on the default valid period.

**If `validTimePeriodSeconds` is set to `type(uint64).max` (~584 billion years):**

Staleness protection is silently disabled. Any price, regardless of age, is returned as fresh. Protocols using `getPrice()` or `getEmaPrice()` will consume arbitrarily stale prices, enabling price manipulation via stale data.

---

### Likelihood Explanation

Execution requires a valid governance VAA signed by the Wormhole guardian set from the authorized governance emitter — the same trust model as the `DEFAULT_ADMIN_ROLE` in the original report. The VAA can be relayed by any transaction sender once signed. The likelihood is low under normal operations but non-zero under a governance key compromise or a malicious governance proposal, which is the same threat model the original report addresses.

---

### Recommendation

Add explicit bounds to `parseSetValidPeriodPayload` in `PythGovernanceInstructions.sol`:

```solidity
uint256 constant MIN_VALID_PERIOD = 1;       // at least 1 second
uint256 constant MAX_VALID_PERIOD = 30 days; // at most 30 days

function parseSetValidPeriodPayload(...) public pure returns (...) {
    svp.newValidPeriod = uint256(encodedPayload.toUint64(index));
    if (svp.newValidPeriod < MIN_VALID_PERIOD || svp.newValidPeriod > MAX_VALID_PERIOD)
        revert PythErrors.InvalidGovernanceMessage();
    ...
}
```

Apply the same bounds in all chain implementations (`set_stale_price_threshold.move`, Fuel, Stylus, CosmWasm, Near, TON).

---

### Proof of Concept

1. Governance authority signs a VAA with action `SetValidPeriod`, payload encoding `newValidPeriod = 0`.
2. Any relayer calls `executeGovernanceInstruction(vaa)` on the Pyth EVM contract.
3. `parseSetValidPeriodPayload` decodes `0` with no revert.
4. `setValidTimePeriodSeconds(0)` is written to state.
5. Any subsequent call to `getPrice(id)` or `getEmaPrice(id)` reverts with `StalePrice` because `diff(block.timestamp, price.publishTime) > 0` is always true.
