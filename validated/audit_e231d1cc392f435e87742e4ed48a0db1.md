### Title
Unbounded Revert-Reason Copy in `executeCallback` Enables Gas Griefing — (`File: target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

`Echo.sol`'s `executeCallback` uses a `catch Error(string memory reason)` clause to handle reverts from the consumer's `_echoCallback`. Because Solidity's `catch Error(string memory reason)` copies the entire revert-reason string from the external call's return buffer into memory with no size cap, a malicious requester can force the executor to copy and emit an arbitrarily large blob, consuming a disproportionate amount of gas and potentially causing the entire `executeCallback` transaction to run out of gas.

---

### Finding Description

In `Echo.sol`, `executeCallback` invokes the requester's callback inside a `try/catch`:

```solidity
try
    IEchoConsumer(req.requester)._echoCallback{
        gas: req.callbackGasLimit
    }(sequenceNumber, priceFeeds)
{
    emitPriceUpdate(sequenceNumber, priceIds, priceFeeds);
} catch Error(string memory reason) {
    emit PriceUpdateCallbackFailed(
        sequenceNumber,
        providerToCredit,
        priceIds,
        req.requester,
        reason          // <-- unbounded copy
    );
} catch { ... }
``` [1](#0-0) 

The `catch Error(string memory reason)` ABI-decodes the full revert payload into a Solidity `string`, allocating however many bytes the callee returned. There is no cap on this size. The decoded `reason` is then forwarded verbatim into the `PriceUpdateCallbackFailed` event, paying the full memory-expansion and LOG data cost a second time.

The `req.requester` field is set directly from `msg.sender` at request time:

```solidity
req.requester = msg.sender;
``` [2](#0-1) 

Any unprivileged user can therefore register a malicious contract as the requester by calling `requestPriceUpdatesWithCallback`. That contract's `echoCallback` implementation can revert with a string of arbitrary length (e.g., `revert(string(new bytes(500_000)))`). When the provider later calls `executeCallback`, the EVM copies the entire 500 kB revert payload into the executor's memory, consuming gas proportional to the payload size — gas that is paid by the executor, not the attacker.

By contrast, `Entropy.sol` already applies the exact fix for this class of bug: it uses `excessivelySafeCall` with a hard 256-byte copy limit:

```solidity
(success, ret) = req.requester.excessivelySafeCall(
    uint256(req.gasLimit10k) * TEN_THOUSAND,
    256, // copy at most 256 bytes of the return value into ret.
    ...
);
``` [3](#0-2) 

`Echo.sol` has no equivalent protection.

---

### Impact Explanation

An executor calling `executeCallback` pays all gas for the transaction. A malicious requester can force the executor to copy megabytes of revert data into memory, making the callback execution economically unviable or causing an out-of-gas revert that wastes the executor's gas entirely. Because `executeCallback` clears the request before the callback is invoked (`clearRequest` at line 164), a failed `executeCallback` due to OOG would leave the request cleared but the callback never delivered, permanently locking the requester's fee in the provider's accrued balance with no recourse. [4](#0-3) 

---

### Likelihood Explanation

The entry path requires only:
1. Deploying a contract whose `echoCallback` reverts with a large string.
2. Calling `requestPriceUpdatesWithCallback` with that contract as `msg.sender` and paying the normal fee.

No privileged access, leaked key, or governance majority is needed. The fee paid by the attacker is fixed and small; the gas cost imposed on the executor scales with the revert payload size. The attack is therefore cheap to mount and repeatable.

---

### Recommendation

Replace the `try/catch` callback invocation in `executeCallback` with `excessivelySafeCall` (already a dependency of the repo via `Entropy.sol`), capping the number of bytes copied from the return buffer to a small constant (e.g., 256 bytes), mirroring the pattern already used in `Entropy.sol`:

```solidity
(bool success, bytes memory ret) = req.requester.excessivelySafeCall(
    req.callbackGasLimit,
    256,
    abi.encodeWithSelector(
        IEchoConsumer._echoCallback.selector,
        sequenceNumber,
        priceFeeds
    )
);
if (!success) {
    // truncated reason in ret, safe to emit
    emit PriceUpdateCallbackFailed(..., string(ret));
}
```

contract MaliciousConsumer is IEchoConsumer {
    address echo;
    constructor(address _echo) { echo = _echo; }
    function getEcho() internal view override returns (address) { return echo; }
    function echoCallback(uint64, PythStructs.PriceFeed[] memory) internal override {
        // Revert with 500 kB of data
        bytes memory huge = new bytes(500_000);
        assembly { revert(add(huge, 32), mload(huge)) }
    }
}

// Attack sequence
MaliciousConsumer mc = new MaliciousConsumer(address(echo));
bytes32[] memory ids = new bytes32[](1);
ids[0] = SOME_PRICE_ID;

    provider, block.timestamp, ids, callbackGasLimit
);

// Provider calls executeCallback — copies 500 kB into memory,
// consuming ~500k * 3 gas = ~1.5M extra gas from the executor.
// If total gas budget is exceeded, the tx reverts and the executor
