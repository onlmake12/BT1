### Title
Missing Payload Timestamp Staleness Check in `verifyUpdate` Allows Indefinitely Replayable Signed Price Updates — (File: `lazer/contracts/evm/src/PythLazer.sol`)

---

### Summary

`PythLazer.verifyUpdate()` validates the ECDSA signature and checks that the signer's certificate has not expired, but it never checks whether the price payload's embedded timestamp is recent relative to `block.timestamp`. Any validly-signed Lazer price update can be replayed indefinitely — as long as the trusted signer's certificate is still valid — allowing stale prices to pass on-chain verification.

---

### Finding Description

`PythLazer.verifyUpdate()` performs two checks:

1. The ECDSA signature is valid (recovers a non-zero signer address).
2. The recovered signer address has a non-expired certificate (`isValidSigner`). [1](#0-0) 

`isValidSigner` only checks the signer's certificate expiry, not the age of the price data: [2](#0-1) 

The payload itself contains an embedded `timestamp` field (microseconds since epoch), parsed by `parsePayloadHeader` in `PythLazerLib`: [3](#0-2) 

`verifyUpdate` never calls `parsePayloadHeader` or any equivalent, so the payload's timestamp is never compared to `block.timestamp`. A price update signed an hour, a day, or a week ago will pass `verifyUpdate` without any staleness rejection, as long as the signer's certificate has not yet expired.

The Lazer Sui and Aptos contracts check the *signer* expiry but similarly do not enforce a maximum age on the payload timestamp itself: [4](#0-3) 

---

### Impact Explanation

Any consumer contract that calls `verifyUpdate` and trusts the returned `payload` as "fresh" can be fed arbitrarily stale price data. Because `verifyUpdate` is the authoritative on-chain verification step for Lazer price updates, consumer contracts have no protocol-enforced guarantee that the verified payload is recent. This enables:

- **Price manipulation**: A user can capture a favorable signed price update (e.g., a price spike or dip) and replay it later when market conditions have moved, to execute trades at stale prices.
- **Stale collateral valuation**: Lending protocols using Lazer prices can be manipulated by replaying old prices that no longer reflect current collateral values.

The impact is equivalent to the reference report's scenario: a signed agreement (price update) remains valid indefinitely, allowing it to be used under market conditions that have changed since it was signed.

---

### Likelihood Explanation

- Any unprivileged Lazer updater or relayer can call `verifyUpdate` with any previously captured signed update.
- No special access is required; the function is `external payable` and only requires the verification fee.
- Signed Lazer updates are broadcast over a public WebSocket feed, so historical signed payloads are readily available to any observer.
- The signer certificate validity window can be months or years (e.g., `3000000000000000` in the test), giving a large replay window. [5](#0-4) 

---

### Recommendation

Add a maximum payload age check inside `verifyUpdate`. After recovering the payload, parse its timestamp and reject updates older than a configurable staleness threshold (e.g., 60 seconds):

```solidity
// After: payload = update[71:71 + payload_len];
uint64 payloadTimestamp = uint64(bytes8(payload[4:12])); // skip 4-byte magic
// Lazer timestamps are in microseconds
uint256 payloadTimestampSecs = payloadTimestamp / 1_000_000;
if (block.timestamp > payloadTimestampSecs + MAX_PAYLOAD_AGE_SECS) {
    revert("payload too old");
}
```

Alternatively, expose a `verifyUpdateNoOlderThan(bytes calldata update, uint256 maxAgeSecs)` variant that enforces freshness, analogous to `getPriceNoOlderThan` in the classic Pyth price feed SDK: [6](#0-5) 

---

### Proof of Concept

1. At time `T`, a Lazer price update is broadcast with `payload.timestamp = T` and signed by a trusted signer with `expiresAt = T + 365 days`.
2. At time `T + 3600` (one hour later), market price has moved significantly.
3. An attacker calls `verifyUpdate{value: fee}(oldUpdate)` with the captured update from step 1.
4. `verifyUpdate` recovers the signer, checks `block.timestamp < expiresAt` (passes), and returns the stale `payload` without any timestamp check.
5. The attacker's consumer contract receives a "verified" payload containing the hour-old price and uses it to execute a trade at the stale price, profiting from the price discrepancy.

The root cause is the absence of any `block.timestamp` vs. `payloadTimestamp` comparison in `verifyUpdate`: [7](#0-6)

        bytes calldata update
    ) external payable returns (bytes calldata payload, address signer) {
        // Require fee and refund excess
        require(msg.value >= verification_fee, "Insufficient fee provided");
        if (msg.value > verification_fee) {
            payable(msg.sender).transfer(msg.value - verification_fee);
        }

        if (update.length < 71) {
            revert("input too short");
        }
        uint32 EVM_FORMAT_MAGIC = 706910618;

        uint32 evm_magic = uint32(bytes4(update[0:4]));
        if (evm_magic != EVM_FORMAT_MAGIC) {
            revert("invalid evm magic");
        }
        uint16 payload_len = uint16(bytes2(update[69:71]));
        if (update.length < 71 + payload_len) {
            revert("input too short");
        }
        payload = update[71:71 + payload_len];
        bytes32 hash = keccak256(payload);
        (signer, , ) = ECDSA.tryRecover(
            hash,
            uint8(update[68]) + 27,
            bytes32(update[4:36]),
            bytes32(update[36:68])
        );
        if (signer == address(0)) {
            revert("invalid signature");
        }
        if (!isValidSigner(signer)) {
            revert("invalid signer");
        }
    }

public(package) fun verify_le_ecdsa_message(
    state: &State,
    clock: &Clock,
    signature: &vector<u8>,
    payload: &vector<u8>,
) {
    let current_cap = state.current_cap();

    // 0 stands for keccak256 hash
    let pubkey = secp256k1_ecrecover(signature, payload, 0);

    // Check if the recovered pubkey is in the trusted signers list
    let trusted_signers = state.trusted_signers(&current_cap);
    let mut maybe_idx = trusted_signers.find_index!(|signer|
        signer.public_key() == &pubkey
    );

    assert!(maybe_idx.is_some(), ESignerNotTrusted);
    let idx = maybe_idx.extract();
    let expires_at_ms = trusted_signers[idx].expires_at_ms();
    assert!(clock.timestamp_ms() < expires_at_ms, ESignerExpired);
}
