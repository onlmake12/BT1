### Title
Outbound Dial Gate `can_dial` Skips Ban-List Check, Allowing Repeated Connections to Banned Peers — (`network/src/network.rs`)

### Summary
`can_dial()` — the pre-dial gate for every outbound connection — performs no ban-list lookup. The ban check exists only inside `accept_peer()`, which runs after the TCP handshake completes. For subnet-banned addresses (banned via `ban_network`), the peer-store address manager is never pruned, so `fetch_addrs_to_attempt` and `fetch_addrs_to_feeler` keep returning those addresses, `can_dial` approves them, and the node repeatedly opens TCP connections to banned peers before finally rejecting them at session registration. This is the structural analog of the BeraBitcoin bug: one direction (inbound, via `accept_peer`) is guarded; the other direction (outbound pre-dial, via `can_dial`) is not.

---

### Finding Description

**Inbound path — ban check present:**

`accept_peer()` in `network/src/peer_registry.rs` is called when any session (inbound or outbound) is registered. For non-whitelisted peers it calls `peer_store.is_addr_banned(&remote_addr)` and returns `Err(PeerError::Banned)` if the address is banned. [1](#0-0) 

**Outbound path — ban check absent:**

`can_dial()` in `network/src/network.rs` is the gate that every outbound dial must pass before the TCP connection is initiated. It checks for self-dial, already-connected peers, and duplicate dials — but it never consults the ban list. [2](#0-1) 

**Why the late check does not fully compensate — subnet bans:**

`ban_addr()` removes the individual address from the addr manager, so individually banned IPs are pruned from future dial candidates. However, `ban_network()` — used for subnet bans — only inserts into the ban list; it does **not** remove existing addr-manager entries for addresses within that subnet. [3](#0-2) 

As a result, `fetch_addrs_to_attempt` and `fetch_addrs_to_feeler` (called every `try_connect_interval` by `OutboundPeerService`) continue to return addresses that fall inside a banned subnet. [4](#0-3) 

`can_dial` approves those addresses (no ban check), the TCP handshake completes, and only then does `accept_peer` reject the session. The cycle repeats on every service tick. [5](#0-4) 

---

### Impact Explanation

1. **Ban-enforcement delay:** A subnet-banned peer receives a completed TCP connection from the local node on every `try_connect_interval` tick. The ban is not enforced until after the handshake, giving the banned peer a window to send data before disconnect.
2. **Resource exhaustion of outbound slots:** During the dial-to-reject window, outbound connection slots and `dialing_addrs` entries are consumed by banned peers, potentially starving legitimate outbound connections.
3. **Feeler pollution:** `dial_feeler` is intended to probe unknown addresses for reachability. Banned-subnet addresses that remain in the addr manager are probed repeatedly, wasting feeler budget and updating their `last_tried` timestamps, which can displace legitimate candidates.

---

### Likelihood Explanation

Any unprivileged peer that causes the local node to issue a subnet ban (e.g., by triggering the `ProtocolError` ban path, or via the operator's own `set_ban` RPC call) and whose subnet contains multiple addr-manager entries will trigger this condition automatically and continuously. No special access is required beyond being a peer that gets banned. [6](#0-5) 

---

### Recommendation

Add a ban-list check inside `can_dial()` before returning `true`:

```rust
pub(crate) fn can_dial(&self, addr: &Multiaddr) -> bool {
    // ... existing checks ...

    // NEW: reject banned addresses before initiating TCP connection
    if self.with_peer_store(|ps| ps.is_addr_banned(addr)) {
        trace!("Do not dial banned addr: {}", addr);
        return false;
    }

    true
}
```

Additionally, `ban_network()` should iterate the addr manager and remove all entries whose IP falls within the newly banned subnet, mirroring the cleanup already done by `ban_addr()`. [7](#0-6) 

---

### Proof of Concept

1. Node A has peer B (subnet `10.0.0.0/24`) in its addr manager with multiple entries (`10.0.0.1`, `10.0.0.2`, …).
2. Operator calls `set_ban("10.0.0.0/24", "insert", …)` → `ban_network` is called; addr-manager entries are **not** removed.
3. On the next `OutboundPeerService` tick, `fetch_addrs_to_attempt` returns `10.0.0.1:8115`.
4. `can_dial("10.0.0.1:8115/p2p/…")` returns `true` — no ban check.
5. TCP connection to `10.0.0.1:8115` is established; banned peer can send protocol bytes.
6. `accept_peer` is called → `is_addr_banned` returns `true` → session disconnected.
7. Steps 3–6 repeat every `try_connect_interval` for every address in the banned subnet. [8](#0-7) [2](#0-1) [9](#0-8)

    pub(crate) fn can_dial(&self, addr: &Multiaddr) -> bool {
        let peer_id = extract_peer_id(addr);
        if peer_id.is_none() {
            error!("Do not dial addr without peer id, addr: {}", addr);
            return false;
        }
        let peer_id = peer_id.as_ref().unwrap();

        if self.local_peer_id() == peer_id {
            trace!("Do not dial self: {:?}, {}", peer_id, addr);
            return false;
        }
        if self.public_addrs.read().contains(addr) {
            trace!(
                "Do not dial listened address(self): {:?}, {}",
                peer_id, addr
            );
            return false;
        }

        let peer_in_registry = self.with_peer_registry(|reg| {
            reg.get_key_by_peer_id(peer_id).is_some() || reg.is_feeler(addr)
        });
        if peer_in_registry {
            trace!("Do not dial peer in registry: {:?}, {}", peer_id, addr);
            return false;
        }

        if let Some(dial_started) = self.dialing_addrs.read().get(peer_id) {
            trace!(
                "Do not send repeated dial commands to network service: {:?}, {}",
                peer_id, addr
            );
            if Instant::now().saturating_duration_since(*dial_started) > DIAL_HANG_TIMEOUT {
                #[cfg(feature = "with_sentry")]
                with_scope(
                    |scope| scope.set_fingerprint(Some(&["ckb-network", "dialing-timeout"])),
                    || {
                        capture_message(
                            &format!(
                                "Dialing {:?}, {:?} for more than {} seconds, \
                                 something is wrong in network service",
                                peer_id,
                                addr,
                                DIAL_HANG_TIMEOUT.as_secs(),
                            ),
                            Level::Warning,
                        )
                    },
                );
            }
            return false;
        }

        true
    }

    pub(crate) fn ban_addr(&mut self, addr: &Multiaddr, timeout_ms: u64, ban_reason: String) {
        if let Some(addr) = multiaddr_to_socketaddr(addr) {
            let network = ip_to_network(addr.ip());
            self.ban_network(network, timeout_ms, ban_reason)
        }
        self.addr_manager.remove(addr);

    pub(crate) fn ban_network(&mut self, network: IpNetwork, timeout_ms: u64, ban_reason: String) {
        let now_ms = ckb_systemtime::unix_time_as_millis();
        let ban_addr = BannedAddr {
            address: network,
            ban_until: now_ms + timeout_ms,
            created_at: now_ms,
            ban_reason,
        };
        self.mut_ban_list().ban(ban_addr);
    }

    fn dial_feeler(&mut self) {

            TransportType::Wss => peer_addr

                .iter()

        let attempt_peers = self.network_state.with_peer_store_mut(|peer_store| {
            let paddrs = peer_store.fetch_addrs_to_feeler(FEELER_CONNECTION_COUNT, filter);
            for paddr in paddrs.iter() {
                // mark addr as tried
                if let Some(paddr) = peer_store.mut_addr_manager().get_mut(&paddr.addr) {
                    paddr.mark_tried(now_ms);
                }

            paddrs

            attempt_peers.len(),
            attempt_peers,

            self.network_state.dial_feeler(&self.p2p_control, {
                match &self.transport_type {
                    TransportType::Tcp => (),
                    TransportType::Ws => addr.push(Protocol::Ws),
                    TransportType::Wss => addr.push(Protocol::Wss),
                }
                addr
            });
        }
    }
