### Title
Provider Accrued Fees Permanently Locked if Provider Is a Smart Contract Without Withdrawal Capability — (File: `target_chains/ethereum/contracts/contracts/entropy/Entropy.sol`)

---

### Summary

The `withdraw()` function in `Entropy.sol` uses `msg.sender` as both the provider lookup key and the sole recipient of accrued fees. If a registered provider is a smart contract that cannot invoke `withdraw()` (e.g., an immutable contract with no such dispatch function), and has not previously set a `feeManager`, the accrued ETH fees are permanently locked in the contract with no recovery path.

---

### Finding Description

`Entropy.sol`'s `withdraw()` function is the only self-service path for a provider to retrieve accrued fees:

```solidity
function withdraw(uint128 amount) public override {
    EntropyStructsV2.ProviderInfo storage providerInfo = _state.providers[msg.sender];
    require(providerInfo.accruedFeesInWei >= amount, "Insufficient balance");
    providerInfo.accruedFeesInWei -= amount;
    (bool sent, ) = msg.sender.call{value: amount}("");
    require(sent, "withdrawal to msg.sender failed");
    ...
}
```

The provider is keyed by `msg.sender` and funds are sent exclusively to `msg.sender`. There is no overload accepting a `recipient` address.

The only alternative path is `withdrawAsFeeManager()`:

```solidity
function withdrawAsFeeManager(address provider, uint128 amount) external override {
    ...
    if (providerInfo.feeManager != msg.sender) revert EntropyErrors.Unauthorized();
    ...
    (bool sent, ) = msg.sender.call{value: amount}("");
    require(sent, "withdrawal to msg.sender failed");
    ...
}
```

This path requires the provider to have previously called `setFeeManager()` to designate a fee manager. If the provider is an immutable smart contract that:

1. Registered as a provider via `register()` (keyed by its own address), and
2. Accrued fees over time from fulfilled requests, and
3. Has no function to call `Entropy.withdraw()`, and
4. Never called `setFeeManager()` (default `feeManager` is `address(0)`)

…then all accrued fees are permanently frozen. No admin, governance, or permissionless path exists to recover them. [1](#0-0) [2](#0-1) 

---

### Impact Explanation

Accrued provider fees (real ETH) are permanently frozen in the `Entropy` contract. The provider cannot recover them, and no privileged role (admin, owner, governance) has a path to force-withdraw on behalf of a provider. The `withdrawAsFeeManager` escape hatch only works if the provider proactively set a fee manager before becoming unable to act. [3](#0-2) 

---

### Likelihood Explanation

Entropy providers can be smart contracts — the protocol explicitly supports contract-based providers (the `requestWithCallback` flow is designed for contract callers). A smart contract provider that was deployed without a `withdraw()` dispatch or without calling `setFeeManager()` during setup would permanently lose all accrued fees. This is a realistic deployment scenario for protocol-integrated or automated providers. [4](#0-3) 

---

### Recommendation

Add a `withdrawTo(address recipient, uint128 amount)` variant that allows the provider (`msg.sender`) to specify an arbitrary recipient address. This mirrors the `withdrawAsFeeManager` pattern but is callable by the provider itself:

```solidity
function withdrawTo(address recipient, uint128 amount) external {
    EntropyStructsV2.ProviderInfo storage providerInfo = _state.providers[msg.sender];
    require(providerInfo.accruedFeesInWei >= amount, "Insufficient balance");
    providerInfo.accruedFeesInWei -= amount;
    (bool sent, ) = recipient.call{value: amount}("");
    require(sent, "withdrawal to recipient failed");
    emit Withdrawal(msg.sender, recipient, amount);
}
```

Alternatively, allow the fee manager to withdraw to an arbitrary address (not just `msg.sender`) so that a fee manager set on a smart contract provider can route funds to a recoverable address. [5](#0-4) 

---

### Proof of Concept

1. Deploy an immutable smart contract `ProviderContract` that calls `entropy.register(fee, commitment, metadata, chainLength, uri)` in its constructor but has no function to call `entropy.withdraw()` and no call to `entropy.setFeeManager()`.
2. Users call `entropy.request{value: fee}(address(ProviderContract), ...)` — fees accrue to `_state.providers[address(ProviderContract)].accruedFeesInWei`.
3. Attempt `entropy.withdraw(amount)` from any EOA → reverts because `_state.providers[msg.sender].accruedFeesInWei == 0` for any EOA.
4. Attempt `entropy.withdrawAsFeeManager(address(ProviderContract), amount)` → reverts with `Unauthorized` because `providerInfo.feeManager == address(0)`.
5. All accrued fees in `_state.providers[address(ProviderContract)].accruedFeesInWei` are permanently inaccessible. [6](#0-5)

    function register(
        uint128 feeInWei,
        bytes32 commitment,
        bytes calldata commitmentMetadata,
        uint64 chainLength,
        bytes calldata uri
    ) public override {
        if (chainLength == 0) revert EntropyErrors.AssertionFailure();

        EntropyStructsV2.ProviderInfo storage provider = _state.providers[
            msg.sender
        ];

        // NOTE: this method implementation depends on the fact that ProviderInfo will be initialized to all-zero.
        // Specifically, accruedFeesInWei is intentionally not set. On initial registration, it will be zero,
        // then on future registrations, it will be unchanged. Similarly, provider.sequenceNumber defaults to 0
        // on initial registration.

        provider.feeInWei = feeInWei;

        provider.originalCommitment = commitment;
        provider.originalCommitmentSequenceNumber = provider.sequenceNumber;
        provider.currentCommitment = commitment;
        provider.currentCommitmentSequenceNumber = provider.sequenceNumber;
        provider.commitmentMetadata = commitmentMetadata;
        provider.endSequenceNumber = provider.sequenceNumber + chainLength;
        provider.uri = uri;

        provider.sequenceNumber += 1;

        emit EntropyEvents.Registered(
            EntropyStructConverter.toV1ProviderInfo(provider)
        );
        emit EntropyEventsV2.Registered(msg.sender, bytes(""));
    }
