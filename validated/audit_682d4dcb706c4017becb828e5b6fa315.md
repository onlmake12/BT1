### Title
Peer Store Permanently Locked via Crafted Discovery Addresses — (`network/src/peer_store/peer_store_impl.rs`)

### Summary
The `check_purge` eviction logic in `PeerStore` contains an off-by-one boundary condition (`addrs.len() > 4` instead of `>= 4`). A single malicious discovery peer can exploit this by filling the peer store's 16,384-address limit with exactly 4 connectable addresses per `/16` network group across 4,096 subnets. Once this state is reached, `check_purge` finds no eviction candidates and permanently returns `Err(EvictionFailed)`, silently blocking all future `add_addr` calls and preventing the victim node from ever learning new peer addresses via discovery.

---

### Finding Description

**Root cause — the `> 4` boundary in `check_purge`:** [1](#0-0) 

When the store is at capacity, `check_purge` first tries to evict non-connectable peers: [2](#0-1) 

If all stored peers are connectable (freshly added via discovery, `last_connected_at_ms = 0`, `attempts_count = 0`), it falls through to the network-group eviction path: [3](#0-2) 

The critical flaw is at line 378:

```rust
if addrs.len() > 4 {   // groups of exactly 4 are NEVER evicted
```

If every group has exactly 4 peers, this condition is always `false`, `candidate_peers` remains empty, and the function returns: [4](#0-3) 

**Attack entry point — P2P discovery protocol:**

`add_addr` is the untrusted discovery ingestion path: [5](#0-4) 

It is called from `DiscoveryAddressManager::add_new_addrs`, which processes inbound `Nodes` messages from any connected peer: [6](#0-5) 

The error is only debug-logged — the node continues silently with a permanently locked peer store.

**Capacity constant:** [7](#0-6) 

---

### Impact Explanation

Once the peer store is locked:
- Every subsequent `add_addr` call returns `Err(EvictionFailed)`.
- The victim node cannot learn any new peer addresses through the discovery protocol.
- Existing connections are unaffected, but as peers disconnect over time, the node cannot replenish its peer pool.
- The failure is silent (debug-level log only), so operators will not notice until the node becomes isolated.

---

### Likelihood Explanation

- Requires only **one** P2P connection to the victim (any discovery-capable peer).
- The discovery protocol allows up to 1,000 addresses per `Nodes` message (`MAX_ADDR_TO_SEND = 1000`), so filling 16,384 slots requires ~17 messages.
- Addresses need only pass `is_valid_addr` (public/reachable IP check). Public IPv4 space contains thousands of `/16` subnets; the attacker does not need to own or control those IPs — they only need to advertise them.
- The attack is **persistent**: once the store is filled with this distribution, it stays locked until the node is restarted or peers age out (7-day timeout).

---

### Recommendation

Change the eviction threshold from strict greater-than to greater-than-or-equal:

```rust
// Before (vulnerable):
if addrs.len() > 4 {

// After (fixed):
if addrs.len() >= 4 {
```

Additionally, consider rate-limiting the number of addresses accepted per discovery session and per source IP prefix to prevent bulk-filling attacks.

---

### Proof of Concept

```rust
// Fill peer store with 4096 groups × 4 addresses = 16384 (ADDR_COUNT_LIMIT)
// Each group uses a distinct /16 subnet: 1.0.x.x, 1.1.x.x, ..., 16.15.x.x
// All addresses are freshly added (last_connected_at_ms=0, attempts_count=0) → connectable

let mut peer_store = PeerStore::default();
let mut count = 0u32;
for subnet_a in 1u8..=16 {
    for subnet_b in 0u8..=255 {
        if count >= 4096 { break; }
        for host in 1u8..=4 {
            let addr: Multiaddr = format!(
                "/ip4/{}.{}.1.{}/tcp/8115/p2p/{}",
                subnet_a, subnet_b, host,
                PeerId::random().to_base58()
            ).parse().unwrap();
            peer_store.add_addr(addr, Flags::COMPATIBILITY).unwrap();
        }
        count += 1;
    }
}

// Peer store is now at ADDR_COUNT_LIMIT with 4 peers per /16 group
// Any new legitimate address now fails permanently:
let new_addr: Multiaddr = "/ip4/200.0.0.1/tcp/8115/p2p/...".parse().unwrap();
assert!(matches!(
    peer_store.add_addr(new_addr, Flags::COMPATIBILITY),
    Err(_) // PeerStoreError::EvictionFailed
));
```
