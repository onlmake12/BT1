### Title
`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` Always Revert for Whitelist-Enabled Subscriptions Due to Wrong `msg.sender` Context in `this.` External Call - (File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol)

---

### Summary

In `Scheduler.sol`, `getPricesNoOlderThan` delegates to `this.getPricesUnsafe(...)` and `getEmaPricesNoOlderThan` delegates to `this.getEmaPricesUnsafe(...)`. Using `this.` triggers an external call, causing `msg.sender` inside the callee to become the Scheduler contract address rather than the original caller. Both `getPricesUnsafe` and `getEmaPricesUnsafe` are guarded by the `onlyWhitelistedReader` modifier, which checks `msg.sender` against the subscription manager and whitelist. The Scheduler contract is never a subscription manager and is never added to any whitelist, so these calls unconditionally revert with `Unauthorized` whenever `whitelistEnabled` is `true`.

---

### Finding Description

In `Scheduler.sol`, `getPricesNoOlderThan` passes the staleness check and then calls:

```solidity
prices = this.getPricesUnsafe(subscriptionId, priceIds);
``` [1](#0-0) 

Similarly, `getEmaPricesNoOlderThan` calls:

```solidity
prices = this.getEmaPricesUnsafe(subscriptionId, priceIds);
``` [2](#0-1) 

Both `getPricesUnsafe` and `getEmaPricesUnsafe` carry the `onlyWhitelistedReader(subscriptionId)` modifier: [3](#0-2) 

The modifier logic is:

```solidity
modifier onlyWhitelistedReader(uint256 subscriptionId) {
    if (_state.subscriptionManager[subscriptionId] == msg.sender) { _; return; }
    if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) { _; return; }
    // ... check whitelist ...
    if (!isWhitelisted) { revert SchedulerErrors.Unauthorized(); }
    _;
}
``` [4](#0-3) 

When `getPricesNoOlderThan` calls `this.getPricesUnsafe(...)`, the EVM dispatches an external call. Inside `getPricesUnsafe`, `msg.sender` is now the **Scheduler contract address**, not the original whitelisted reader. The modifier evaluates:

1. Is `msg.sender` (Scheduler contract) the subscription manager? → **No**
2. Is `whitelistEnabled` false? → If `true`, falls through
3. Is the Scheduler contract in the reader whitelist? → **No** → `revert Unauthorized`

The result: for any subscription with `whitelistEnabled = true`, both `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` are permanently broken and always revert, even for legitimately whitelisted callers.

The root cause is structurally identical to the external report: using `this.f(...)` changes `msg.sender` to the contract itself inside `f`, breaking access control that was designed to check the original caller.

---

### Impact Explanation

Any subscription that enables the whitelist feature loses access to the two staleness-checked price reading functions (`getPricesNoOlderThan`, `getEmaPricesNoOlderThan`). These are the primary safe-read APIs that enforce price freshness guarantees. Whitelisted integrators relying on these functions for on-chain logic (e.g., lending protocols, derivatives) will receive permanent reverts, making the Scheduler's whitelist-gated staleness-checked reads completely non-functional. The subscription manager and all whitelisted readers are affected equally — there is no workaround short of disabling the whitelist entirely, which removes the access-control guarantee.

---

### Likelihood Explanation

The whitelist feature (`whitelistEnabled`) is an explicitly supported configuration option in `SchedulerStructs.SubscriptionParams`. Any subscription manager who enables it — a normal, expected usage — will trigger this bug. The broken functions are the primary staleness-safe read path, so affected users will encounter the revert on their first attempt to use them. Likelihood is high for any deployment where whitelist-enabled subscriptions are created.

---

### Recommendation

Replace the `this.` external calls with direct calls to the internal helper `_getPricesInternal`, which already performs the same data retrieval without re-entering the access-control modifier:

```solidity
// getPricesNoOlderThan — replace:
prices = this.getPricesUnsafe(subscriptionId, priceIds);
// with:
PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
prices = new PythStructs.Price[](priceFeeds.length);
for (uint i = 0; i < priceFeeds.length; i++) { prices[i] = priceFeeds[i].price; }
```

Apply the same pattern in `getEmaPricesNoOlderThan` (using `priceFeeds[i].emaPrice`). The outer `onlyWhitelistedReader` modifier on `getPricesNoOlderThan` / `getEmaPricesNoOlderThan` already enforces access control for the original caller; there is no need to re-check it on the inner call. [5](#0-4) 

---

### Proof of Concept

1. Deploy `SchedulerUpgradeable`.
2. Create a subscription with `whitelistEnabled = true` and add address `0xAlice` to `readerWhitelist`.
3. Have a keeper call `updatePriceFeeds` to populate price data.
4. Call `getPricesNoOlderThan(subscriptionId, priceIds, 60)` from `0xAlice` (a legitimately whitelisted reader).
5. **Observed**: reverts with `Unauthorized` — because inside `getPricesUnsafe`, `msg.sender` is the Scheduler contract, which is not in the whitelist.
6. Call `getPricesUnsafe(subscriptionId, priceIds)` directly from `0xAlice`.
7. **Observed**: succeeds — because `msg.sender` is `0xAlice`, which is whitelisted.

The discrepancy confirms that `this.getPricesUnsafe(...)` in `getPricesNoOlderThan` substitutes the Scheduler contract for the original caller, breaking the whitelist check.

    /// @notice Internal helper function to retrieve price feeds for a subscription.
    /// @param subscriptionId The ID of the subscription.
    /// @param priceIds The specific price IDs requested, or empty array to get all.
    /// @return priceFeeds An array of PriceFeed structs corresponding to the requested IDs.
    function _getPricesInternal(
        uint256 subscriptionId,
        bytes32[] calldata priceIds
    ) internal view returns (PythStructs.PriceFeed[] memory priceFeeds) {
        if (!_state.subscriptionParams[subscriptionId].isActive) {
            revert SchedulerErrors.InactiveSubscription();
        }

        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        // If no price IDs provided, return all price feeds for the subscription
        if (priceIds.length == 0) {
            PythStructs.PriceFeed[]
                memory allFeeds = new PythStructs.PriceFeed[](
                    params.priceIds.length
                );
            for (uint8 i = 0; i < params.priceIds.length; i++) {
                PythStructs.PriceFeed memory priceFeed = _state.priceUpdates[
                    subscriptionId
                ][params.priceIds[i]];
                // Check if the price feed exists (price ID is valid and has been updated)
                if (priceFeed.id == bytes32(0)) {
                    revert SchedulerErrors.InvalidPriceId(
                        params.priceIds[i],
                        bytes32(0)
                    );
                }
                allFeeds[i] = priceFeed;
            }
            return allFeeds;
        }

        // Return only the requested price feeds
        PythStructs.PriceFeed[]
            memory requestedFeeds = new PythStructs.PriceFeed[](
                priceIds.length
            );
        for (uint8 i = 0; i < priceIds.length; i++) {
            PythStructs.PriceFeed memory priceFeed = _state.priceUpdates[
                subscriptionId
            ][priceIds[i]];

            // Check if the price feed exists (price ID is valid and has been updated)
            if (priceFeed.id == bytes32(0)) {
                revert SchedulerErrors.InvalidPriceId(priceIds[i], bytes32(0));
            }
            requestedFeeds[i] = priceFeed;
        }
        return requestedFeeds;
    }

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
