### Title
`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` Always Revert for Whitelist-Enabled Subscriptions Due to Broken `this.` External Call Pattern - (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`Scheduler.sol` exposes four price-reading functions, all guarded by `onlyWhitelistedReader`. The two "safe" variants — `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` — delegate to the "unsafe" variants via `this.getPricesUnsafe(...)` and `this.getEmaPricesUnsafe(...)`. Because `this.` triggers an external call, `msg.sender` inside the inner function becomes `address(this)` (the Scheduler contract itself). The `onlyWhitelistedReader` modifier then rejects the call because the Scheduler contract is neither the subscription manager nor a whitelisted reader. For any subscription with `whitelistEnabled = true`, both safe read functions are permanently broken for all callers, including legitimately whitelisted readers.

---

### Finding Description

**Root cause — `this.` external call re-applies access control with wrong `msg.sender`:**

`getPricesNoOlderThan` at line 554:
```solidity
prices = this.getPricesUnsafe(subscriptionId, priceIds);
``` [1](#0-0) 

`getEmaPricesNoOlderThan` at line 597:
```solidity
prices = this.getEmaPricesUnsafe(subscriptionId, priceIds);
``` [2](#0-1) 

Both outer functions carry `onlyWhitelistedReader(subscriptionId)`, so the original caller's identity is verified correctly. However, the `this.` prefix causes Solidity to issue an external `CALL` opcode. Inside `getPricesUnsafe` / `getEmaPricesUnsafe`, `msg.sender` is now `address(this)` — the Scheduler contract — not the original whitelisted reader.

The modifier then evaluates three conditions in order: [3](#0-2) 

1. Is `msg.sender` the subscription manager? → **No** (Scheduler ≠ manager)
2. Is `whitelistEnabled` false? → **No** (whitelist is enabled)
3. Is `msg.sender` in `readerWhitelist`? → **No** (Scheduler is not listed)

Result: `revert SchedulerErrors.Unauthorized()` — every call to `getPricesNoOlderThan` or `getEmaPricesNoOlderThan` reverts when `whitelistEnabled = true`, regardless of who the original caller is.

The analogous inconsistency to the reference report: the "unsafe" variants (`getPricesUnsafe`, `getEmaPricesUnsafe`) work correctly for whitelisted callers, while the "safe" variants (`getPricesNoOlderThan`, `getEmaPricesNoOlderThan`) — which are the recommended interface per the SDK docs — are silently broken for the same callers. [4](#0-3) 

---

### Impact Explanation

**Impact: Medium**

Subscription managers who enable the whitelist pay for a service where only approved addresses can read prices. The two staleness-checked read functions (`getPricesNoOlderThan`, `getEmaPricesNoOlderThan`) are the recommended safe interface per the IScheduler NatSpec: [5](#0-4) 

For any whitelist-enabled subscription:
- Whitelisted readers calling `getPricesNoOlderThan` or `getEmaPricesNoOlderThan` always receive `Unauthorized`, making these functions completely unusable.
- Readers are forced to fall back to `getPricesUnsafe` / `getEmaPricesUnsafe`, which return prices without any staleness guarantee, defeating the purpose of the "no older than" API.
- Subscription managers who designed their protocol around the safe API (e.g., requiring freshness guarantees before executing trades) cannot use the intended interface.

---

### Likelihood Explanation

**Likelihood: High**

Any subscription created with `whitelistEnabled = true` triggers this bug unconditionally. The whitelist feature is a first-class, documented feature of the Scheduler. The README explicitly describes the Reader role and the whitelist mechanism: [6](#0-5) 

No special attacker action is required — the bug manifests for every legitimate whitelisted reader on every whitelist-enabled subscription.

---

### Recommendation

Replace the `this.` external calls with direct calls to the internal helper `_getPricesInternal`, mirroring the pattern already used in `getPricesUnsafe` and `getEmaPricesUnsafe`:

```solidity
// getPricesNoOlderThan — replace line 554:
// BEFORE: prices = this.getPricesUnsafe(subscriptionId, priceIds);
// AFTER:
PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
prices = new PythStructs.Price[](priceFeeds.length);
for (uint i = 0; i < priceFeeds.length; i++) {
    prices[i] = priceFeeds[i].price;
}

// getEmaPricesNoOlderThan — replace line 597 analogously, using priceFeeds[i].emaPrice
```

This keeps `msg.sender` as the original caller throughout, so `onlyWhitelistedReader` evaluates correctly in both the outer and inner logic paths.

### Proof of Concept

```
1. Manager calls createSubscription({
       priceIds: [feedA],
       readerWhitelist: [alice],
       whitelistEnabled: true,
       ...
   }) with sufficient ETH.

2. Keeper calls updatePriceFeeds(subscriptionId, updateData).
   → Prices stored on-chain. priceLastUpdatedAt = T.

3. Alice (whitelisted) calls getPricesUnsafe(subscriptionId, []).
   → onlyWhitelistedReader: msg.sender = alice, alice ∈ whitelist → PASSES.
   → Returns prices correctly. ✓

4. Alice calls getPricesNoOlderThan(subscriptionId, [], 60).
   → Outer onlyWhitelistedReader: msg.sender = alice → PASSES.
   → Staleness check: distance(now, T) ≤ 60 → PASSES.
   → this.getPricesUnsafe(subscriptionId, []) — external call issued.
   → Inside getPricesUnsafe: msg.sender = address(Scheduler).
   → onlyWhitelistedReader: Scheduler ≠ manager, whitelist enabled,
     Scheduler ∉ readerWhitelist → revert Unauthorized. ✗

Result: Alice cannot use getPricesNoOlderThan or getEmaPricesNoOlderThan
despite being a legitimately whitelisted reader.
```

```text
    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }

- **Manager:** The owner/creator of a subscription (`subscriptionManager`). They define the subscription parameters (feeds, triggers, whitelist), deposit funds (`addFunds`), and can modify (`updateSubscription`) or deactivate their subscription. Typically an EOA or a protocol controlled by its governance.
- **Reader:** A consumer of the price data stored within a specific subscription. They call read functions (`getPricesUnsafe`, `getEmaPriceUnsafe`) on this contract to get the latest pushed prices. Access can be permissionless or restricted to a whitelist (`readerWhitelist`) defined by the Manager.
- **Provider:** An off-chain, permissionless agent that runs Keeper node(s). Responsible for monitoring subscriptions and pushing updates (`updatePriceFeeds`) when trigger conditions are met. Providers are incentivized economically for successful pushes.
