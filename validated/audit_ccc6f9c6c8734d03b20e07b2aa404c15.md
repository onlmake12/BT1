### Title
No User Refund Mechanism for Locked Request Fees When Provider Fails to Execute Callback - (`File: target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

In `Echo.sol`, when a user calls `requestPriceUpdatesWithCallback`, the provider's fee portion is locked inside the pending `Request` struct. It is only released when the provider calls `executeCallback`. If the provider permanently stops operating, becomes deregistered, or refuses to fulfill, the user's locked fee has no recovery path. The contract itself acknowledges this risk in a TODO comment but provides no fix.

---

### Finding Description

When `requestPriceUpdatesWithCallback` is called, the user pays `msg.value`. The Pyth protocol fee is immediately credited:

```solidity
_state.accruedFeesInWei += _state.pythFeeInWei;
```

But the provider's portion is stored inside the pending request struct:

```solidity
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);
``` [1](#0-0) 

This `req.fee` is only credited to the provider when `executeCallback` is successfully called:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
clearRequest(sequenceNumber);
``` [2](#0-1) 

The `Request` struct confirms `fee` is stored per-request with no expiry or cancellation field: [3](#0-2) 

The contract itself acknowledges the risk in a TODO comment:

```
// TODO: if this effect occurs here, we need to guarantee that executeCallback can never revert.
// If executeCallback can revert, then funds can be permanently locked in the contract.
// TODO: there also needs to be some penalty mechanism in case the expected provider doesn't execute the callback.
``` [4](#0-3) 

There is no `cancelRequest`, `refundRequest`, or timeout-based expiry function anywhere in `Echo.sol`. The only withdrawal functions are `withdrawFees` (admin-only, for `_state.accruedFeesInWei`) and `withdrawAsFeeManager` (for provider's `accruedFeesInWei`). Neither returns funds to the requesting user. [5](#0-4) 

---

### Impact Explanation

User ETH paid as the provider fee portion is permanently locked inside the `Request` struct if the assigned provider never calls `executeCallback`. There is no on-chain path for the user to recover these funds. The Pyth admin can only withdraw `_state.accruedFeesInWei` (the protocol fee portion), not the per-request `req.fee` locked in pending requests.

---

### Likelihood Explanation

Low-to-medium. The scenario requires the assigned provider to stop operating after a request is submitted. This can happen if:
- The provider's off-chain infrastructure goes offline permanently.
- The provider is deregistered (no deregistration function exists, but the provider can simply stop responding).
- The exclusivity period passes and no alternative provider steps in to fulfill the request.

The `exclusivityPeriodSeconds` mechanism allows other providers to fulfill after the exclusivity window, but only if another provider is willing to call `executeCallback`. If no provider fulfills, the funds remain locked indefinitely. [6](#0-5) 

---

### Recommendation

Implement a user-callable `cancelRequest(uint64 sequenceNumber)` function that:
1. Verifies `msg.sender == req.requester`.
2. Enforces a minimum waiting period (e.g., after the exclusivity period has elapsed with no fulfillment).
3. Refunds `req.fee` to the requester and deducts `_state.accruedFeesInWei` by `_state.pythFeeInWei` (or accepts the Pyth fee as non-refundable).
4. Calls `clearRequest`.

Alternatively, add a deadline field to `Request` and allow anyone to trigger a refund after expiry.

---

### Proof of Concept

1. User calls `requestPriceUpdatesWithCallback(provider, publishTime, priceIds, gasLimit)` with `msg.value = 1 ETH`.
2. `_state.accruedFeesInWei += pythFeeInWei` (e.g., 0.001 ETH credited to protocol).
3. `req.fee = 0.999 ETH` is stored in the pending `Request` struct.
4. The assigned provider goes offline and never calls `executeCallback`.
5. No other provider calls `executeCallback` (or the exclusivity period never expires in a meaningful way).
6. The user has no function to call to recover `0.999 ETH`. The admin's `withdrawFees` only covers `_state.accruedFeesInWei`. The `req.fee` is inaccessible forever. [7](#0-6)

    function requestPriceUpdatesWithCallback(
        address provider,
        uint64 publishTime,
        bytes32[] calldata priceIds,
        uint32 callbackGasLimit
    ) external payable override returns (uint64 requestSequenceNumber) {
        require(
            _state.providers[provider].isRegistered,
            "Provider not registered"
        );

        // FIXME: this comment is wrong. (we're not using tx.gasprice)
        // NOTE: The 60-second future limit on publishTime prevents a DoS vector where
        //      attackers could submit many low-fee requests for far-future updates when gas prices
        //      are low, forcing executors to fulfill them later when gas prices might be much higher.
        //      Since tx.gasprice is used to calculate fees, allowing far-future requests would make
        //      the fee estimation unreliable.
        require(publishTime <= block.timestamp + 60, "Too far in future");
        if (priceIds.length > MAX_PRICE_IDS) {
            revert TooManyPriceIds(priceIds.length, MAX_PRICE_IDS);
        }
        requestSequenceNumber = _state.currentSequenceNumber++;

        uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
        if (msg.value < requiredFee) revert InsufficientFee();

        Request storage req = allocRequest(requestSequenceNumber);
        req.sequenceNumber = requestSequenceNumber;
        req.publishTime = publishTime;
        req.callbackGasLimit = callbackGasLimit;
        req.requester = msg.sender;
        req.provider = provider;
        req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);

        // Create array with the right size
        req.priceIdPrefixes = new bytes8[](priceIds.length);

        // Copy only the first 8 bytes of each price ID to storage
        for (uint8 i = 0; i < priceIds.length; i++) {
            // Extract first 8 bytes of the price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }
            req.priceIdPrefixes[i] = prefix;
        }
        _state.accruedFeesInWei += _state.pythFeeInWei;

        emit PriceUpdateRequested(req, priceIds);
    }

        require(msg.sender == _state.admin, "Only admin can withdraw fees");
        require(_state.accruedFeesInWei >= amount, "Insufficient balance");
