### Title
Unsafe Truncating `u128 as u64` Cast in DAO Withdrawal Capacity Calculation - (File: `util/dao/src/lib.rs`)

---

### Summary

`DaoCalculator::calculate_maximum_withdraw` computes an intermediate `u128` result to avoid overflow during multiplication, but then silently truncates it back to `u64` via an unchecked `as u64` cast. This is inconsistent with the safe `u64::try_from(...).map_err(|_| DaoError::Overflow)?` pattern used for identical u128→u64 conversions elsewhere in the same file. If the intermediate result exceeds `u64::MAX`, the high bits are silently dropped, producing a wrong (smaller) withdrawal capacity.

---

### Finding Description

In `util/dao/src/lib.rs`, `calculate_maximum_withdraw` computes the entitled withdrawal amount for a NervosDAO depositor:

```rust
// Lines 152–156
let withdraw_counted_capacity = u128::from(counted_capacity.as_u64())
    * u128::from(withdrawing_ar)
    / u128::from(deposit_ar);
let withdraw_capacity =
    Capacity::shannons(withdraw_counted_capacity as u64).safe_add(occupied_capacity)?;
``` [1](#0-0) 

The `u128` intermediate is intentional — it prevents overflow during the `counted_capacity * withdrawing_ar` multiplication. However, the final `as u64` cast is a **silent truncating downcast**: if `withdraw_counted_capacity > u64::MAX`, the upper 64 bits are discarded without any error, returning a drastically smaller capacity value.

The same file applies the correct pattern for analogous u128→u64 conversions in `dao_field_with_current_epoch`:

```rust
// Line 245
let miner_issuance =
    Capacity::shannons(u64::try_from(miner_issuance128).map_err(|_| DaoError::Overflow)?);
// Line 258
let ar_increase = u64::try_from(ar_increase128).map_err(|_| DaoError::Overflow)?;
``` [2](#0-1) 

The inconsistency is the root cause: `calculate_maximum_withdraw` omits the overflow guard that the rest of the file consistently applies.

**Overflow condition:**

`withdraw_counted_capacity = counted_capacity × withdrawing_ar / deposit_ar`

Overflow occurs when `counted_capacity × (withdrawing_ar / deposit_ar) > u64::MAX`. Since `deposit_ar` starts at `10^16` and grows monotonically with secondary issuance (~4% annually), the ratio `withdrawing_ar / deposit_ar` grows over time. For a cell holding a large fraction of the total CKB supply, the result can exceed `u64::MAX` once the accumulation rate has grown sufficiently (roughly a factor of ~5.5× from the deposit epoch to the withdrawal epoch for a cell near the total supply cap).

---

### Impact Explanation

When `withdraw_counted_capacity` silently wraps, `Capacity::shannons(withdraw_counted_capacity as u64)` returns a value far below the depositor's actual entitlement. This propagates through two paths:

1. **Transaction rejection / user fund loss**: `calculate_maximum_withdraw` is called from `transaction_maximum_withdraw`, which feeds into `transaction_fee`. A truncated maximum-withdraw makes the node believe the transaction outputs exceed the entitled inputs, causing valid DAO withdrawal transactions to be rejected. The depositor cannot reclaim their full principal + interest.

2. **Corrupted DAO field**: `transaction_maximum_withdraw` is also called from `withdrawed_interests`, which feeds into `dao_field_with_current_epoch`. An underestimated withdrawn interest corrupts the `s` (secondary issuance accumulator) field packed into the block's DAO field, causing consensus-level DAO state divergence between nodes that encounter the overflow and those that do not. [3](#0-2) 

---

### Likelihood Explanation

The overflow requires `withdrawing_ar / deposit_ar` to be large enough that `counted_capacity × ratio > u64::MAX`. Given the ~4% annual secondary issuance growth rate and the total CKB supply cap, this threshold is not reachable in the near term for typical cell sizes. However:

- The vulnerability is **unconditionally present** in the code path today.
- Any transaction sender submitting a DAO withdrawal RPC call or P2P transaction triggers this code path — no privilege is required.
- As the DAO accumulation rate grows over decades, or if a large fraction of CKB is concentrated in a single DAO cell, the overflow becomes reachable.
- The inconsistency with the safe pattern used elsewhere in the same file confirms this is an unintentional omission, not a deliberate design choice.

---

### Recommendation

Replace the unsafe truncating cast with the same checked conversion used elsewhere in the file:

```rust
// Before (unsafe):
Capacity::shannons(withdraw_counted_capacity as u64).safe_add(occupied_capacity)?

// After (safe, consistent with lines 245 and 258):
Capacity::shannons(
    u64::try_from(withdraw_counted_capacity).map_err(|_| DaoError::Overflow)?
).safe_add(occupied_capacity)?
``` [4](#0-3)

### Proof of Concept

**Setup:** A DAO cell is deposited at a block where `deposit_ar = 10_000_000_000_000_000` (the genesis value). The cell holds `counted_capacity = 18_000_000_000_000_000_000` shannons (≈ 180 billion CKB, within u64 range). The withdrawal occurs at a block where `withdrawing_ar = 11_000_000_000_000_000` (a 10% growth, reachable after ~2.5 years of secondary issuance).

**Computation:**
```
withdraw_counted_capacity (u128)
  = 18_000_000_000_000_000_000 × 11_000_000_000_000_000
    / 10_000_000_000_000_000
  = 19_800_000_000_000_000_000   ← exceeds u64::MAX (18_446_744_073_709_551_615)
```

**With `as u64` cast:**
```
19_800_000_000_000_000_000 as u64
  = 19_800_000_000_000_000_000 % 2^64
  = 1_353_255_926_290_448_384   ← silently truncated to ~1.35 × 10^18
```

The returned `withdraw_capacity` is `1_353_255_926_290_448_384 + occupied_capacity` instead of the correct `19_800_000_000_000_000_000 + occupied_capacity`. The depositor loses approximately 18.4 billion CKB worth of shannons, and any DAO withdrawal transaction claiming the correct amount is rejected by the node. [5](#0-4)

    fn transaction_maximum_withdraw(
        &self,
        rtx: &ResolvedTransaction,
    ) -> Result<Capacity, DaoError> {
        let header_deps: HashSet<Byte32> = rtx.transaction.header_deps_iter().collect();
        rtx.resolved_inputs.iter().enumerate().try_fold(
            Capacity::zero(),
            |capacities, (i, cell_meta)| {
                let capacity: Result<Capacity, DaoError> = {
                    let output = &cell_meta.cell_output;
                    let is_dao_type_script = |type_script: Script| {
                        Into::<u8>::into(type_script.hash_type())
                            == Into::<u8>::into(ScriptHashType::Type)
                            && type_script.code_hash() == self.consensus.dao_type_hash()
                    };
                    let is_dao_output = output
                        .type_()
                        .to_opt()
                        .map(is_dao_type_script)
                        .unwrap_or(false);
                    if is_dao_output {
                        // A withdrawing DAO cell has 8 bytes of cell data storing the
                        // block number of the original deposit.
                        let deposited_block_number =
                            match self.data_loader.load_cell_data(cell_meta) {
                                Some(data) if data.len() == 8 => LittleEndian::read_u64(&data),
                                _ => 0,
                            };
                        if deposited_block_number > 0 {
                            let withdrawing_header_hash = cell_meta
                                .transaction_info
                                .as_ref()
                                .map(|info| &info.block_hash)
                                .filter(|hash| header_deps.contains(hash))
                                .ok_or(DaoError::InvalidOutPoint)?;
                            let deposit_header_hash = rtx
                                .transaction
                                .witnesses()
                                .get(i)
                                .ok_or(DaoError::InvalidOutPoint)
                                .and_then(|witness_data| {
                                    // dao contract stores header deps index as u64 in the input_type field of WitnessArgs
                                    let witness =
                                        WitnessArgs::from_slice(&Into::<Bytes>::into(witness_data))
                                            .map_err(|_| DaoError::InvalidDaoFormat)?;
                                    let header_deps_index_data: Option<Bytes> =
                                        witness.input_type().to_opt().map(|witness| witness.into());
                                    if header_deps_index_data.is_none()
                                        || header_deps_index_data.clone().map(|data| data.len())
                                            != Some(8)
                                    {
                                        return Err(DaoError::InvalidDaoFormat);
                                    }
                                    Ok(LittleEndian::read_u64(&header_deps_index_data.unwrap()))
                                })
                                .and_then(|header_dep_index| {
                                    rtx.transaction
                                        .header_deps()
                                        .get(header_dep_index as usize)
                                        .and_then(|hash| header_deps.get(&hash))
                                        .ok_or(DaoError::InvalidOutPoint)
                                })?;

                            let deposit_header = self
                                .data_loader
                                .get_header(deposit_header_hash)
                                .ok_or(DaoError::InvalidHeader)?;
                            if deposit_header.number() != deposited_block_number {
                                return Err(DaoError::InvalidOutPoint);
                            }
                            self.calculate_maximum_withdraw(
                                output,
                                Capacity::bytes(cell_meta.data_bytes as usize)?,
                                deposit_header_hash,
                                withdrawing_header_hash,
                            )
                        } else {
                            Ok(output.capacity().into())
                        }
                    } else {
                        Ok(output.capacity().into())
                    }
                };
                capacity.and_then(|c| c.safe_add(capacities).map_err(Into::into))
            },
        )
    }
