### Title
Unprivileged RPC Caller Can Crash Node via Unconditional `.expect()` on `txs_sizes` in `FeeRateCollector::statistics` — (`rpc/src/util/fee_rate.rs`)

### Summary

`FeeRateCollector::statistics()` calls `.expect()` unconditionally on `block_ext.txs_sizes`, but the `collect()` loop does not filter out blocks whose `txs_sizes` is `None`. Any node that has blocks stored in the pre-v0.106 `BlockExt` format (5-field, no `txs_sizes`) will panic and crash when `get_fee_rate_statistics` is called by any unprivileged RPC caller.

### Finding Description

**Step 1 — The unconditional `.expect()` at the panic site:**

In `statistics()`, every `BlockExt` yielded by `collect()` is destructured and `txs_sizes` is immediately unwrapped: [1](#0-0) 

There is no `if let Some(txs_sizes) = txs_sizes` guard — the `.expect()` on line 93 fires unconditionally.

**Step 2 — The `collect()` loop does not filter `None` txs_sizes:** [2](#0-1) 

The `filter_map` on line 46 only drops blocks for which `get_block_ext_by_number` returns `None` (i.e., the block hash is absent from the store). It does **not** inspect `txs_sizes` inside the returned `BlockExt`.

**Step 3 — Old-format blocks yield `txs_sizes = None`:**

`get_block_ext` explicitly handles the pre-v0.106 5-field `BlockExt` format: [3](#0-2) 

When `count_extra_fields() == 0` (old format), it calls `reader.into()`, which maps to: [4](#0-3) 

`txs_sizes` is hardcoded to `None` for every pre-v0.106 block. No migration in `util/migrate/src/migrations/` rewrites these records to `BlockExtV1`.

**Step 4 — The schema confirms the two formats:** [5](#0-4) 

The old `BlockExt` table has 5 fields; `BlockExtV1` adds `cycles` and `txs_sizes`. Nodes upgraded from pre-v0.106 retain old-format rows in `COLUMN_BLOCK_EXT`.

### Impact Explanation

Any unprivileged caller who can reach the RPC port (default: localhost, but commonly exposed) can call `get_fee_rate_statistics` with any `target` value up to `MAX_TARGET = 101`. If the node's tip is within 101 blocks of the upgrade boundary, the `collect()` loop will include at least one old-format block, hit the `.expect()`, and the Rust runtime will `panic!`, terminating the node process. Impact: **node crash / denial of service**.

### Likelihood Explanation

Every CKB mainnet node that was running before v0.106 and has since been upgraded retains old-format `BlockExt` rows. During the 101-block window after the upgrade, a single unauthenticated RPC call deterministically triggers the panic. The RPC is enabled by default and requires no credentials.

### Recommendation

Replace the unconditional `.expect()` with a guard that skips blocks lacking `txs_sizes`:

```rust
// rpc/src/util/fee_rate.rs, inside the collect closure
let Some(txs_sizes) = txs_sizes else {
    return fee_rates; // skip pre-BlockExtV1 blocks
};
```

This matches the existing pattern used for `cycles` (line 97), which is already wrapped in `if let Some(cycles) = cycles`.

### Proof of Concept

1. Start a CKB node on a database that contains at least one block stored in the old 5-field `BlockExt` format (any mainnet node upgraded from pre-v0.106).
2. Ensure the tip block number is within 101 of the first `BlockExtV1`-format block.
3. Call: `curl -X POST http://localhost:8114 -H 'Content-Type: application/json' -d '{"jsonrpc":"2.0","method":"get_fee_rate_statistics","params":[null],"id":1}'`
4. The node panics with `expect txs_size's length >= 1` and terminates.

The exact code path: `get_fee_rate_statistics` RPC handler → `FeeRateCollector::new(&snapshot).statistics(target)` → `provider.collect(target, closure)` → `block_ext_iter.fold(...)` iterates an old-format block → closure line 93 `.expect()` on `None` → **panic**. [6](#0-5) [7](#0-6)

        self.get(COLUMN_BLOCK_EXT, block_hash.as_slice())
            .map(|slice| {
                let reader =
                    packed::BlockExtReader::from_compatible_slice_should_be_ok(slice.as_ref());

                }
