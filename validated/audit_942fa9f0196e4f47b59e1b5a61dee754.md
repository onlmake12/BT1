### Title
Network Alert `RawAlert` Signed Message Lacks Chain Domain Separator, Enabling Cross-Network Replay — (File: `util/network-alert/src/verifier.rs`, `util/gen-types/schemas/extensions.mol`)

---

### Summary

CKB's network alert system signs a `RawAlert` message whose serialized bytes contain no network or chain identifier. The `calc_alert_hash()` function hashes only the raw alert payload. If the same Nervos Foundation signing keys are configured on both mainnet and testnet nodes (as documented), a valid alert broadcast on one network can be replayed verbatim on the other, causing nodes to accept and display a misleading alert, or to suppress a legitimate alert with the same `id`.

---

### Finding Description

**Schema — no network field:**

`RawAlert` is defined in `util/gen-types/schemas/extensions.mol` as:

```
table RawAlert {
    notice_until:   Uint64,
    id:             Uint32,
    cancel:         Uint32,
    priority:       Uint32,
    message:        Bytes,
    min_version:    BytesOpt,
    max_version:    BytesOpt,
}
```

There is no `network_id`, `genesis_hash`, or any chain-specific field. [1](#0-0) 

**Hash computation — no domain separation:**

`calc_alert_hash()` is a plain `blake2b` over the raw alert bytes with no prefix or context tag:

```rust
impl<'r> packed::RawAlertReader<'r> {
    pub fn calc_alert_hash(&self) -> packed::Byte32 {
        self.calc_hash()   // blake2b(self.as_slice())
    }
}
``` [2](#0-1) 

**Verifier — no network check:**

`Verifier::verify_signatures` derives the signing message solely from `alert.calc_alert_hash()` and checks it against the configured public keys. There is no check that the alert was intended for this network:

```rust
pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
    let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
    ...
    verify_m_of_n(&message, self.config.signatures_threshold, &signatures, &self.pubkeys)
        .map_err(|err| err.kind())?;
    Ok(())
}
``` [3](#0-2) 

**Relayer — no network check:**

`AlertRelayer::received` accepts any alert that passes signature verification and deduplicates only by `alert_id` (a `u32`). There is no check that the alert originated from the same chain:

```rust
if self.notifier.lock().has_received(alert_id) { return; }
if let Err(err) = self.verifier.verify_signatures(&alert) { ... ban peer ... }
self.notifier.lock().add(&alert);
``` [4](#0-3) 

**Same keys documented for all networks:**

The RPC documentation states: *"The alerts must be signed by 2-of-4 signatures, where the public keys are hard-coded in the source code and belong to early CKB developers."* The same developer team and key set governs both mainnet and testnet. [5](#0-4) 

---

### Impact Explanation

**Cross-network replay:** An attacker who observes a valid alert on mainnet (alerts are public P2P messages) can relay the identical bytes to testnet nodes. Because the hash and signatures are identical, testnet nodes pass `verify_signatures` and accept the alert. Testnet users see a mainnet-specific message (e.g., "upgrade mainnet node v1.x immediately") that does not apply to them.

**Alert suppression via id collision:** Alert deduplication is keyed on the 32-bit `id` field. If an attacker replays a mainnet alert with `id=N` to mainnet nodes before the legitimate mainnet alert with `id=N` propagates, those nodes mark `id=N` as already received and permanently ignore the legitimate alert for the lifetime of the node process. This is a targeted denial-of-service against the alert delivery mechanism.

---

### Likelihood Explanation

- Alerts are broadcast over the open P2P network; any connected peer can observe and re-relay them.
- Connecting to nodes on a different network is trivially possible (no authentication barrier).
- The same Nervos Foundation keys are documented as governing all CKB networks.
- No code change or privileged access is required; the attacker only needs to be a normal P2P participant on one network and relay raw bytes to another.

---

### Recommendation

Include a chain-specific domain separator in the signed message. The genesis block hash (`consensus.genesis_hash()`) is already available at alert-verification time and uniquely identifies each network. The signing message should be:

```
blake2b(genesis_hash || raw_alert_bytes)
```

This is the direct analog of the `networkId` field recommended in the external report. The `Verifier` struct already holds a `NetworkAlertConfig`; it can be extended to carry the genesis hash and mix it into the message before calling `verify_m_of_n`.

---

### Proof of Concept

1. Run a mainnet node and a testnet node, both configured with the standard Nervos Foundation alert keys.
2. Submit a valid alert (id=42, message="mainnet alert") to the mainnet node via `send_alert` RPC with 2-of-4 valid signatures.
3. Capture the raw `packed::Alert` bytes from the P2P wire (or from the RPC response).
4. Connect to a testnet node and send the identical bytes via the Alert P2P protocol (protocol id from `SupportProtocols::Alert`).
5. Observe: the testnet node calls `verify_signatures`, computes `blake2b(raw_alert_bytes)` — identical to the mainnet hash — verifies the signatures successfully, and adds the alert to its notifier.
6. `get_blockchain_info` on the testnet node now returns the mainnet alert message.

For the suppression variant: send the captured mainnet alert bytes to mainnet nodes before the legitimate mainnet alert with the same `id` is broadcast; those nodes will deduplicate and ignore the legitimate alert. [6](#0-5)

    pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
        trace!("Verifying alert {:?}", alert);
        let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
        let signatures: Vec<Signature> = alert
            .signatures()
            .into_iter()
            .filter_map(
                |sig_data| match Signature::from_slice(sig_data.as_reader().raw_data()) {
                    Ok(sig) => {
                        if sig.is_valid() {
                            Some(sig)
                        } else {
                            debug!("invalid signature: {:?}", sig);
                            None
                        }
                    }
                    Err(err) => {
                        debug!("signature error: {}", err);
                        None
                    }
                },
            )
            .collect();
        verify_m_of_n(
            &message,
            self.config.signatures_threshold,
            &signatures,
            &self.pubkeys,
        )
        .map_err(|err| err.kind())?;
        Ok(())
    }
