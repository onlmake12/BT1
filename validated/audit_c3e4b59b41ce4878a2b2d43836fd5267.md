Let me look at the `observed_addrs` function and how it's used in the `public_addrs` context to fully understand the broadcast path.

### Title
Attacker-Controlled `observed_addr` with Mismatched PeerId Poisons Victim's Self-Advertised Addresses — (`network/src/protocols/identify/mod.rs`)

### Summary

`add_observed_addr` only appends the local node's PeerId when the incoming `observed_addr` has **no** `/p2p/` component. If the attacker pre-embeds a `/p2p/<attacker_peer_id>` component, the guard is bypassed and the poisoned address is stored verbatim and later broadcast to every subsequent peer as one of the victim's own listen addresses.

### Finding Description

The guard in `IdentifyCallback::add_observed_addr` is:

```rust
fn add_observed_addr(&mut self, mut addr: Multiaddr, session_id: SessionId) -> MisbehaveResult {
    if extract_peer_id(&addr).is_none() {          // ← only fires when NO /p2p/ present
        addr.push(Protocol::P2P(Cow::Borrowed(
            self.network_state.local_peer_id().as_bytes(),
        )))
    }
    self.network_state.add_observed_addr(session_id, addr);   // stored unconditionally
    MisbehaveResult::Continue
}
``` [1](#0-0) 

When `extract_peer_id` returns `Some` (because the attacker embedded `/p2p/<attacker_peer_id>`), the `if` branch is skipped entirely and the addr is forwarded to `NetworkState::add_observed_addr` with no further validation. [2](#0-1) 

`NetworkState::add_observed_addr` performs no PeerId consistency check — it simply inserts the address into the `observed_addrs` map keyed by `session_id`.

The poisoned entry is then returned by `observed_addrs()`: [3](#0-2) 

And mixed into the victim's self-reported listen addresses in `local_listen_addrs`: [4](#0-3) 

This list is sent verbatim to every new peer in the `connected` handshake: [5](#0-4) 

### Impact Explanation

Any peer that receives the poisoned `listen_addrs` entry (`/ip4/<victim_ip>/tcp/<port>/p2p/<attacker_peer_id>`) will attempt to dial the victim but expect to authenticate against `<attacker_peer_id>`. The TLS/Noise handshake will fail because the victim's actual keypair produces a different PeerId, causing the connection to be rejected. The victim's reachability is degraded for every peer that caches or dials the poisoned address. Because `observed_addrs` are also forwarded in the hole-punching protocol's `listen_addrs` field, the poisoned address can propagate further. [6](#0-5) 

### Likelihood Explanation

The attack requires only a single standard P2P connection to the victim — no special privileges, no PoW, no key material. The crafted `IdentifyMessage` is a valid protobuf/molecule message with a well-formed Multiaddr. One attacker session is sufficient to poison the map entry for that `session_id`; multiple attacker sessions can fill the entire `observed_addrs` map (bounded only by the number of concurrent inbound connections).

### Recommendation

In `add_observed_addr`, after the existing `extract_peer_id` check, validate that the extracted PeerId matches the local node's PeerId. If it does not match, either strip the `/p2p/` component and re-append the correct one, or treat it as misbehavior and return `MisbehaveResult::Disconnect`. The corrected logic should be:

```rust
fn add_observed_addr(&mut self, mut addr: Multiaddr, session_id: SessionId) -> MisbehaveResult {
    match extract_peer_id(&addr) {
        Some(peer_id) if peer_id == *self.network_state.local_peer_id() => { /* ok */ }
        Some(_) => {
            // strip the wrong /p2p/ component and append the correct one
            addr = addr.into_iter()
                .filter(|p| !matches!(p, Protocol::P2P(_)))
                .collect();
            addr.push(Protocol::P2P(Cow::Borrowed(
                self.network_state.local_peer_id().as_bytes(),
            )));
        }
        None => {
            addr.push(Protocol::P2P(Cow::Borrowed(
                self.network_state.local_peer_id().as_bytes(),
            )));
        }
    }
    self.network_state.add_observed_addr(session_id, addr);
    MisbehaveResult::Continue
}
```

### Proof of Concept

1. Attacker node connects to victim over P2P (any inbound slot).
2. Attacker sends an `IdentifyMessage` where `observed_addr = /ip4/<victim_ip>/tcp/<victim_port>/p2p/<attacker_peer_id>`.
3. `process_observed` → `add_observed_addr`: `extract_peer_id` returns `Some(attacker_peer_id)`, the guard is skipped, the poisoned address is stored in `observed_addrs[attacker_session_id]`.
4. A third peer connects to the victim. `connected` calls `local_listen_addrs()`, which calls `observed_addrs(...)` and appends the poisoned entry to the list.
5. The victim sends `IdentifyMessage { listen_addrs: [..., /ip4/<victim_ip>/tcp/<port>/p2p/<attacker_peer_id>], ... }` to the third peer.
6. The third peer dials `/ip4/<victim_ip>/tcp/<port>/p2p/<attacker_peer_id>` and the Noise/TLS handshake fails because the victim's actual PeerId does not match `<attacker_peer_id>`, breaking the connection.

            .read()
            .values()
            .cloned()
            .collect::<HashSet<_>>();
        if observed_addrs.len() <= count {
            return observed_addrs.into_iter().collect();

            observed_addrs
                .into_iter()
                .choose_multiple(&mut rand::thread_rng(), count)
