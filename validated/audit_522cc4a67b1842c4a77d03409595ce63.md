### Title
Fee Charged Without Price State Transition Due to Pre-Deposit Before Freshness Check — (`target_chains/sui/contracts/sources/pyth.move`)

### Summary
`update_single_price_feed` unconditionally deposits the caller's fee into the `PriceInfoObject` **before** checking whether the supplied price update is fresh. A front-runner who submits a newer valid VAA first can cause a victim relayer's subsequent call to pay the fee while writing nothing to the oracle state.

### Finding Description

In `update_single_price_feed`, the execution order is:

1. **Fee sufficiency check** (line 274) — asserts `base_update_fee <= coin::value(&fee)`
2. **Fee deposit** (line 277) — `price_info::deposit_fee_coins(price_info_object, fee)` — coins are irrevocably transferred into the `PriceInfoObject`
3. **Freshness check inside `update_cache`** (line 316) — `if (is_fresh_update(update, price_info_object))` — if false, the price write is silently skipped with no refund [1](#0-0) 

`is_fresh_update` returns `update_timestamp > cached_timestamp` strictly: [2](#0-1) 

`deposit_fee_coins` joins the coins into a dynamic field on the `PriceInfoObject` with no return path: [3](#0-2) 

`PriceInfoObject` is a **shared Sui object** (accessible by any transaction), and valid VAAs with arbitrary recent timestamps are publicly available from Pyth's price service. An attacker can:

1. Observe a victim's pending `update_single_price_feed` call carrying timestamp `T+1`.
2. Fetch a valid VAA with timestamp `T+2` (trivially available from Pythnet).
3. Submit their own `update_single_price_feed` on the same `PriceInfoObject` with `T+2`, which executes first and sets the cached timestamp to `T+2`.
4. Victim's transaction executes: fee is deposited at line 277, then `is_fresh_update` returns `false` (`T+1 ≤ T+2`), so `update_price_info_object` is never called. [4](#0-3) 

The attacker's own call also pays a fee, so there is no net profit — this is pure griefing.

### Impact Explanation
The victim relayer pays `base_update_fee` SUI into the `PriceInfoObject` with zero oracle state change. The invariant that every fee payment corresponds to a successful price state transition is broken. Repeated griefing can drain a relayer's SUI balance and disrupt price feed liveness.

### Likelihood Explanation
Sui's mempool is observable. Valid newer VAAs are freely obtainable from Pyth's public price service at any time. No privileged access, leaked keys, or governance majority is required. The attack is cheap (attacker pays one fee per grief) and fully automatable.

### Recommendation
Move the fee deposit **after** the freshness check, or refund the fee if `is_fresh_update` returns false. Concretely, restructure `update_single_price_feed` so that `deposit_fee_coins` is only called when `update_cache` actually writes a new price (i.e., inside the `if (is_fresh_update(...))` branch), or abort with a specific error code when the update is stale so the entire transaction — including the fee transfer — is rolled back.

### Proof of Concept

```
// Pseudocode test
// 1. Set cached timestamp to T in PriceInfoObject
// 2. Attacker calls update_single_price_feed with valid VAA timestamp T+2
//    -> cached timestamp becomes T+2, attacker pays fee
// 3. Victim calls update_single_price_feed with valid VAA timestamp T+1
//    -> deposit_fee_coins executes (victim pays fee)
//    -> is_fresh_update returns false (T+1 <= T+2)
//    -> update_price_info_object is NOT called
// Assert: price_info_object still holds timestamp T+2 (unchanged from step 2)
// Assert: price_info_object balance increased by victim's fee
```

This maps directly to the existing test harness in `pyth_tests` — set the cached price to timestamp `T`, call `update_single_price_feed` with `T+2` first, then call it again with `T+1` and assert the balance grew but the price state did not change. [5](#0-4)

    public fun update_single_price_feed(
        pyth_state: &PythState,
        price_updates: HotPotatoVector<PriceInfo>,
        price_info_object: &mut PriceInfoObject,
        fee: Coin<SUI>,
        clock: &Clock
    ): HotPotatoVector<PriceInfo> {
        let latest_only = state::assert_latest_only(pyth_state);

        // On Sui, users get to choose which price feeds to update. They specify a single price feed to
        // update at a time. We therefore charge the base fee for each such individual update.
        // This is a departure from Eth, where users don't get to necessarily choose.
        assert!(state::get_base_update_fee(pyth_state) <= coin::value(&fee), E_INSUFFICIENT_FEE);

        // store fee coins within price info object
        price_info::deposit_fee_coins(price_info_object, fee);

        // Find price update corresponding to PriceInfoObject within the array of price_updates
        // and use it to update PriceInfoObject.
        let i = 0;
        let found = false;
        while (i < hot_potato_vector::length<PriceInfo>(&price_updates)){
            let cur_price_info = hot_potato_vector::borrow<PriceInfo>(&price_updates, i);
            if (has_same_price_identifier(cur_price_info, price_info_object)){
                found = true;
                update_cache(latest_only, cur_price_info, price_info_object, clock);
                break
            };
            i = i + 1;
        };
        if (found==false){
            abort E_PRICE_UPDATE_NOT_FOUND_FOR_PRICE_INFO_OBJECT
        };
        price_updates
    }

    fun is_fresh_update(update: &PriceInfo, price_info_object: &PriceInfoObject): bool {
        // Get the timestamp of the update's current price
        let price_feed = price_info::get_price_feed(update);
        let update_timestamp = price::get_timestamp(&price_feed::get_price(price_feed));

        // Get the timestamp of the cached data for the price identifier
        let cached_price_info = price_info::get_price_info_from_price_info_object(price_info_object);
        let cached_price_feed =  price_info::get_price_feed(&cached_price_info);
        let cached_timestamp = price::get_timestamp(&price_feed::get_price(cached_price_feed));

        update_timestamp > cached_timestamp
    }

    public fun deposit_fee_coins(price_info_object: &mut PriceInfoObject, fee_coins: Coin<SUI>) {
        if (!dynamic_object_field::exists_with_type<vector<u8>, Coin<SUI>>(&price_info_object.id, FEE_STORAGE_KEY)) {
            dynamic_object_field::add(&mut price_info_object.id, FEE_STORAGE_KEY, fee_coins);

        else {
            let current_fee = dynamic_object_field::borrow_mut<vector<u8>, Coin<SUI>>(
                &mut price_info_object.id,
                FEE_STORAGE_KEY
            );
            coin::join(current_fee, fee_coins);
        };
    }
