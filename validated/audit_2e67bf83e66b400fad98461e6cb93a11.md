### Title
`this.getPricesUnsafe()` / `this.getEmaPricesUnsafe()` External Call Spoofs `msg.sender` to Contract Address, Permanently Breaking Staleness-Checked Price Reads for Whitelist-Enabled Subscriptions - (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`Scheduler.getPricesNoOlderThan` and `Scheduler.getEmaPricesNoOlderThan` delegate to `this.getPricesUnsafe(...)` and `this.getEmaPricesUnsafe(...)` respectively using an external `this.X()` call. Because Solidity's `this.X()` syntax creates a new message call rather than an internal jump, `msg.sender` inside the callee becomes `address(this)` (the Scheduler contract itself) instead of the original caller. Both `getPricesUnsafe` and `getEmaPricesUnsafe` are guarded by the `onlyWhitelistedReader` modifier, which checks `msg.sender` against the subscription's manager and whitelist. Since `address(this)` is never the manager and never in any user-defined whitelist, every call to `getPricesNoOlderThan` or `getEmaPricesNoOlderThan` on a whitelist-enabled subscription unconditionally reverts with `Unauthorized`, making these two functions permanently non-functional for the whitelist feature.

---

### Finding Description

In `Scheduler.sol`, `getPricesNoOlderThan` performs a staleness check and then delegates to `getPricesUnsafe` via `this.getPricesUnsafe(subscriptionId, priceIds)`: [1](#0-0) 

Similarly, `getEmaPricesNoOlderThan` delegates to `getEmaPricesUnsafe` via `this.getEmaPricesUnsafe(subscriptionId, priceIds)`: [2](#0-1) 

Both `getPricesUnsafe` and `getEmaPricesUnsafe` are decorated with the `onlyWhitelistedReader(subscriptionId)` modifier: [3](#0-2) [4](#0-3) 

The `onlyWhitelistedReader` modifier evaluates `msg.sender` in three steps: [5](#0-4) 

When `getPricesNoOlderThan` is called by a legitimately whitelisted address:

1. The outer `onlyWhitelistedReader` on `getPricesNoOlderThan` passes â€” the original caller is whitelisted.
2. `this.getPricesUnsafe(subscriptionId, priceIds)` is issued as an **external call**. Solidity's `this.X()` syntax creates a new EVM `CALL`, so inside `getPricesUnsafe`, `msg.sender` is now `address(this)` (the Scheduler proxy/contract), not the original caller.
3. The inner `onlyWhitelistedReader` on `getPricesUnsafe` runs with `msg.sender = address(this)`. The Scheduler contract is not the subscription manager and is not in any user-defined whitelist, so the modifier reaches the final `revert SchedulerErrors.Unauthorized()` branch.

The same logic applies identically to `getEmaPricesNoOlderThan` â†’ `getEmaPricesUnsafe`.

The root cause is the same class as the RubiconMarket H-05 finding: using `this.X()` instead of a direct internal call changes `msg.sender` in the callee.

---

### Impact Explanation

`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` are the **safe, staleness-checked** price reading functions intended for production use. For any subscription where `whitelistEnabled = true`, these two functions are completely non-functional â€” every call reverts with `Unauthorized`, regardless of whether the caller is legitimately whitelisted. Affected subscribers are forced to fall back to `getPricesUnsafe` / `getEmaPricesUnsafe`, which provide no staleness guarantee, defeating the purpose of the safer API. This is a permanent, protocol-level DoS on a core feature for all whitelist-enabled subscriptions.

---

### Likelihood Explanation

Any subscriber who enables the whitelist feature (`whitelistEnabled = true`) and then calls `getPricesNoOlderThan` or `getEmaPricesNoOlderThan` will immediately encounter the revert. No special attacker setup is required â€” the bug is triggered by normal, expected usage of the API. The whitelist feature is explicitly supported and documented in the `IScheduler` interface: [6](#0-5) 

---

### Recommendation

Replace the `this.X()` external calls with direct internal calls. Since `getPricesUnsafe` and `getEmaPricesUnsafe` are currently `external`, they must be changed to `public` (or their logic extracted into `internal` helpers) so they can be called without creating a new message context.

**In `getPricesNoOlderThan`**, change:
```solidity
prices = this.getPricesUnsafe(subscriptionId, priceIds);
```
to a direct internal call (after making `getPricesUnsafe` `public` or extracting its body into an `internal` function).

**In `getEmaPricesNoOlderThan`**, change:
```solidity
prices = this.getEmaPricesUnsafe(subscriptionId, priceIds);
```
similarly.

Alternatively, extract the shared logic of `getPricesUnsafe` / `getEmaPricesUnsafe` into `internal` helper functions (e.g., `_getPricesUnsafeInternal`) that can be called directly without the `onlyWhitelistedReader` modifier re-running, and have both the `external` entry points and the `NoOlderThan` wrappers call those helpers.

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
