### Title
Silent Arithmetic Truncation in `calculate_maximum_withdraw` Causes Wrong DAO Withdrawal Capacity — (`File: util/dao/src/lib.rs`)

### Summary

`calculate_maximum_withdraw` in `util/dao/src/lib.rs` uses a bare `as u64` truncating cast to convert a u128 intermediate result back to u64. Every other analogous u128→u64 conversion in the same file uses `u64::try_from(...).map_err(|_| DaoError::Overflow)?`. When the intermediate value exceeds `u64::MAX`, the truncating cast silently produces a wrong (far-too-small) capacity instead of returning an error, causing downstream fee and capacity checks to operate on a corrupted value.

### Finding Description

In `util/dao/src/lib.rs`, `calculate_maximum_withdraw` computes the maximum withdrawable capacity for a DAO cell:

```rust
let counted_capacity = output_capacity.safe_sub(occupied_capacity)?;
let withdraw_counted_capacity = u128::from(counted_capacity.as_u64())
    * u128::from(withdrawing_ar)
    / u128::from(deposit_ar);
let withdraw_capacity =
    Capacity::shannons(withdraw_counted_capacity as u64)   // ← silent truncation
        .safe_add(occupied_capacity)?;
``` [1](#0-0) 

The product `counted_capacity * withdrawing_ar` is computed in u128 to avoid overflow, but the final division result is cast back to u64 with `as u64`, which silently wraps/truncates when the value exceeds `u64::MAX`. Compare this with every other u128→u64 conversion in the same file, which all use the checked form:

```rust
// secondary_block_reward (line 204)
let reward = u64::try_from(reward128).map_err(|_| DaoError::Overflow)?;

// dao_field_with_current_epoch (line 244-245)
let miner_issuance =
    Capacity::shannons(u64::try_from(miner_issuance128).map_err(|_| DaoError::Overflow)?);

// dao_field_with_current_epoch (line 258)
let ar_increase = u64::try_from(ar_increase128).map_err(|_| DaoError::Overflow)?;
``` [2](#0-1) [3](#0-2) [4](#0-3) 

The `calculate_maximum_withdraw` function is the only site that uses the unsafe `as u64` cast.

### Impact Explanation

`calculate_maximum_withdraw` is called from two production paths:

**Path 1 — Transaction fee verification:**
`transaction_maximum_withdraw` → `transaction_fee` → `check_tx_fee` (tx-pool) and `FeeCalculator::transaction_fee` (block verification). [5](#0-4) [6](#0-5) 

When `withdraw_counted_capacity` overflows u64, the truncated value is far smaller than the true value. `transaction_fee` then computes `maximum_withdraw - outputs_capacity`. Because `maximum_withdraw` is now artificially small, this subtraction underflows and returns `Err(Overflow)`. A **valid DAO withdrawal transaction is rejected** with a misleading overflow error, even though the transaction is economically correct.

**Path 2 — RPC `calculate_dao_maximum_withdraw`:**
The RPC returns the truncated (wrong) capacity to the caller. [7](#0-6) 

A user or front-end relying on this RPC to construct a withdrawal transaction receives an incorrect maximum capacity, leading to a transaction that either fails validation or claims less than the user is entitled to.

### Likelihood Explanation

The overflow condition requires `counted_capacity * withdrawing_ar / deposit_ar > u64::MAX`. Since `withdrawing_ar > deposit_ar` always holds (DAO interest is always positive), any cell whose `counted_capacity` is large enough relative to the interest ratio can trigger this. The condition becomes reachable for cells holding a significant fraction of the total CKB supply that have been deposited for a long time. While not trivially triggered today, the condition is structurally guaranteed to become more reachable as the accumulate rate (`ar`) grows over time, and it is reachable by any unprivileged transaction sender who controls a sufficiently large DAO cell.

### Recommendation

Replace the truncating cast with the same checked conversion used everywhere else in the file:

```rust
// Before (unsafe):
Capacity::shannons(withdraw_counted_capacity as u64).safe_add(occupied_capacity)?

// After (consistent with the rest of the file):
let withdraw_counted_capacity_u64 =
    u64::try_from(withdraw_counted_capacity).map_err(|_| DaoError::Overflow)?;
Capacity::shannons(withdraw_counted_capacity_u64).safe_add(occupied_capacity)?
``` [8](#0-7) 

### Proof of Concept

The existing test `check_withdraw_calculation_overflows` in `util/dao/src/tests.rs` already demonstrates the overflow path, but it only catches the case where `safe_add` overflows at the end. A complementary test demonstrating the silent-truncation case (where `withdraw_counted_capacity > u64::MAX` but `safe_add` does not overflow because the truncated value is small) would show that the function returns `Ok(wrong_small_value)` instead of `Err(Overflow)`: [9](#0-8) 

```rust
// Construct a cell where counted_capacity * withdrawing_ar / deposit_ar
// overflows u64 but the truncated result + occupied_capacity does NOT overflow u64.
// With the current `as u64` cast, the function returns Ok(wrong_small_value).
// With the fix (u64::try_from), the function returns Err(DaoError::Overflow).
```

    fn calculate_dao_maximum_withdraw(
        &self,
        out_point: OutPoint,
        kind: DaoWithdrawingCalculationKind,
    ) -> Result<Capacity> {
        let snapshot: &Snapshot = &self.shared.snapshot();
        let consensus = snapshot.consensus();
        let out_point: packed::OutPoint = out_point.into();
        let data_loader = snapshot.borrow_as_data_loader();
        let calculator = DaoCalculator::new(consensus, &data_loader);
        match kind {
            DaoWithdrawingCalculationKind::WithdrawingHeaderHash(withdrawing_header_hash) => {
                let (tx, deposit_header_hash) = snapshot
                    .get_transaction(&out_point.tx_hash())
                    .ok_or_else(|| RPCError::invalid_params("invalid out_point"))?;
                let output = tx
                    .outputs()
                    .get(out_point.index().into())
                    .ok_or_else(|| RPCError::invalid_params("invalid out_point"))?;
                let output_data = tx
                    .outputs_data()
                    .get(out_point.index().into())
                    .ok_or_else(|| RPCError::invalid_params("invalid out_point"))?;

                match calculator.calculate_maximum_withdraw(
                    &output,
                    core::Capacity::bytes(output_data.len()).expect("should not overflow"),
                    &deposit_header_hash,
                    &withdrawing_header_hash.into(),
                ) {
                    Ok(capacity) => Ok(capacity.into()),
                    Err(err) => Err(RPCError::custom_with_error(RPCError::DaoError, err)),
                }
            }
            DaoWithdrawingCalculationKind::WithdrawingOutPoint(withdrawing_out_point) => {
                let (_tx, deposit_header_hash) = snapshot
                    .get_transaction(&out_point.tx_hash())

#[test]
fn check_withdraw_calculation_overflows() {
    let output = CellOutput::new_builder()
        .capacity(Capacity::shannons(18_446_744_073_709_550_000))
        .build();
    let tx = TransactionBuilder::default().output(output.clone()).build();
    let epoch = EpochNumberWithFraction::new(1, 100, 1000);
    let deposit_header = HeaderBuilder::default()
        .number(100)
        .epoch(epoch)
        .dao(pack_dao_data(
            10_000_000_000_123_456,
            Default::default(),
            Default::default(),
            Default::default(),
        ))
        .build();
    let deposit_block = BlockBuilder::default()
        .header(deposit_header)
        .transaction(tx)
        .build();

    let epoch = EpochNumberWithFraction::new(1, 200, 1000);
    let withdrawing_header = HeaderBuilder::default()
        .number(200)
        .epoch(epoch)
        .dao(pack_dao_data(
            10_000_000_001_123_456,
            Default::default(),
            Default::default(),
            Default::default(),
        ))
        .build();
    let withdrawing_block = BlockBuilder::default().header(withdrawing_header).build();

    let tmp_dir = TempDir::new().unwrap();
    let db = RocksDB::open_in(&tmp_dir, COLUMNS);
    let store = ChainDB::new(db, Default::default());
    let txn = store.begin_transaction();
    txn.insert_block(&deposit_block).unwrap();
    txn.attach_block(&deposit_block).unwrap();
    txn.insert_block(&withdrawing_block).unwrap();
    txn.attach_block(&withdrawing_block).unwrap();
    txn.commit().unwrap();

    let consensus = Consensus::default();
    let data_loader = store.borrow_as_data_loader();
    let calculator = DaoCalculator::new(&consensus, &data_loader);
    let result = calculator.calculate_maximum_withdraw(
        &output,
        Capacity::bytes(0).expect("should not overflow"),
        &deposit_block.hash(),
        &withdrawing_block.hash(),
    );
    assert!(result.is_err());
