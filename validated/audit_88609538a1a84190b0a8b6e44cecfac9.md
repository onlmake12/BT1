### Title
Legacy `revealWithCallback` Path Executes Callback Without Gas Stipend, Permanently Locking User Fees — (File: `target_chains/ethereum/contracts/contracts/entropy/Entropy.sol`)

---

### Summary

In `Entropy.sol`'s `revealWithCallback`, the legacy execution branch (triggered when `req.gasLimit10k == 0`) makes a direct external call to the requester's `_entropyCallback` with **no gas stipend and no try/catch**. A malicious or buggy callback can consume all available gas, reverting the entire transaction — including the preceding `clearRequest` call — leaving the request permanently active and the user's fee irrecoverably locked in the contract.

---

### Finding Description

`revealWithCallback` has two branches:

**New path** (`req.gasLimit10k != 0`, lines 574–660): uses `excessivelySafeCall` with an explicit gas cap, catches failures, and transitions the request to `CALLBACK_FAILED` state for recovery.

**Legacy path** (`req.gasLimit10k == 0`, lines 661–702): makes a bare, uncapped, uncaught external call:

```solidity
if (len != 0) {
    IEntropyConsumer(callAddress)._entropyCallback(   // line 676
        sequenceNumber,
        provider,
        randomNumber
    );
}
``` [1](#0-0) 

No gas stipend is attached, so the callee receives the full remaining gas. If it consumes all of it, the EVM reverts every state change in the transaction — including the `clearRequest` that was executed just before the call:

```solidity
clearRequest(provider, sequenceNumber);   // line 666
// WARNING: DO NOT USE req BELOW HERE AS ITS CONTENTS HAS BEEN CLEARED
``` [2](#0-1) 

Because the revert undoes `clearRequest`, the request stays permanently active. There is no `CALLBACK_FAILED` state, no retry path, and no refund mechanism in the legacy branch.

The legacy path is reached whenever `providerInfo.defaultGasLimit == 0`. In `requestHelper`, this condition explicitly sets `req.gasLimit10k = 0`:

```solidity
if (providerInfo.defaultGasLimit == 0) {
    // Provider doesn't support the new callback failure state flow
    req.gasLimit10k = 0;
}
``` [3](#0-2) 

The public `requestWithCallback` function always passes `gasLimit = 0`, unconditionally routing through this path for any provider whose `defaultGasLimit` is zero:

```solidity
function requestWithCallback(address provider, bytes32 userContribution)
    public payable override returns (uint64) {
    return requestV2(provider, userContribution, 0);
}
``` [4](#0-3) 

---

### Impact Explanation

- The user's fee (paid upfront at request time) is **permanently locked** in the contract — no cancel, no refund, no recovery path exists in the legacy branch.
- The provider can never fulfill the request and earn their accrued fee.
- Every provider attempt to call `revealWithCallback` wastes gas and reverts, making the slot permanently unresolvable.
- Unlike the new path (which has `CALLBACK_FAILED` → retry), the legacy path offers zero recourse.

---

### Likelihood Explanation

- Any provider that registered before the `defaultGasLimit` feature was introduced has `defaultGasLimit == 0`, making every `requestWithCallback` call against them vulnerable.
- A malicious requester can deliberately deploy a gas-exhausting callback (e.g., `while(true){}`) and call `requestWithCallback`, permanently bricking the slot.
- A legitimate user whose callback contract has an accidental infinite loop or unbounded storage write will also have their fee locked with no recovery.
- `revealWithCallback` is callable by anyone (line 541: "Anyone can call this method"), so the provider cannot prevent the attempt. [5](#0-4) 

---

### Recommendation

1. **Add a gas stipend** to the legacy callback call, mirroring the new path's `excessivelySafeCall` pattern.
2. **Wrap the call in try/catch** so a reverting or gas-exhausting callback does not roll back `clearRequest`.
3. **Add a `CALLBACK_FAILED` recovery state** to the legacy branch so stuck requests can be retried or cancelled.
4. Alternatively, **deprecate the legacy path** and require all providers to set a non-zero `defaultGasLimit` before accepting new requests.

---

### Proof of Concept

```solidity
// 1. Deploy malicious callback
contract MaliciousCallback is IEntropyConsumer {
    address entropyAddr;
    constructor(address e) { entropyAddr = e; }
    function getEntropy() internal view override returns (address) { return entropyAddr; }
    function _entropyCallback(uint64, address, bytes32) external override {
        assembly { invalid() }  // consumes all gas
    }
}

// 2. Request against a legacy provider (defaultGasLimit == 0)
MaliciousCallback mc = new MaliciousCallback(address(entropy));
uint64 seq = IEntropy(entropy).requestWithCallback{value: fee}(legacyProvider, userRandom);
// req.gasLimit10k == 0  →  legacy branch selected

// 3. Provider (or anyone) calls revealWithCallback
entropy.revealWithCallback(legacyProvider, seq, userRandom, providerProof);
// _entropyCallback consumes all gas → full revert → clearRequest undone
// seq is still active; fee is permanently locked; no recovery possible
```
