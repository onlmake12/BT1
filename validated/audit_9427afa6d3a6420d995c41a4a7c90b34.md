### Title
Unprivileged First-Caller Can Seize Program Authority on Initialize — (`target_chains/solana/programs/pyth-price-store/src/processor/initialize.rs`)

### Summary

The `initialize` instruction in `pyth-price-store` stores an arbitrary 32-byte `authority` value into the config PDA without requiring that value to sign the transaction, and without verifying the caller is the program's upgrade authority. Any account that calls `Initialize` first wins permanent control over `InitializePublisher`.

---

### Finding Description

The `initialize` function validates only that the payer is a signer and writable: [1](#0-0) 

`validate_payer` enforces only `is_signer` and `is_writable` — there is no check that the payer equals the program's upgrade authority or any other privileged key: [2](#0-1) 

The `authority` field in `InitializeArgs` is a plain `[u8; 32]` byte array — it is instruction data, not a required signer: [3](#0-2) 

`config::create` writes whatever bytes are passed as `authority` and only guards against re-initialization via `format != 0`: [4](#0-3) 

Because `system_instruction::create_account` atomically fails if the config PDA already exists, only the **first** caller of `Initialize` succeeds. There is no subsequent recovery path — the config PDA cannot be re-initialized once `format` is set. [5](#0-4) 

---

### Impact Explanation

`validate_authority` (used by `InitializePublisher`) checks that the signing account's key equals `config.authority`: [6](#0-5) 

Whoever controls the key stored in `config.authority` can call `InitializePublisher` at will, registering arbitrary publishers and associating them with attacker-controlled buffer accounts. This directly enables unauthorized price feed writes via `SubmitPrices`. [7](#0-6) 

---

### Likelihood Explanation

Solana program deployment and initialization are separate transactions. There is always a window — even if brief — between `solana program deploy` completing and the deployer's `Initialize` transaction landing. An attacker monitoring the chain for new deployments of this program ID can submit their own `Initialize` in the same or next slot. No privileged access, leaked keys, or social engineering is required.

---

### Recommendation

Add one of the following guards to `initialize`:

1. **Require the `authority` to co-sign the transaction** — add `authority` as a required signer account (not just instruction data), so the attacker cannot impersonate an arbitrary pubkey.
2. **Verify the payer is the program's upgrade authority** — load the program's `ProgramData` account and assert `payer.key == upgrade_authority_address`.

Option 1 is the minimal fix; option 2 is the stronger invariant.

---

### Proof of Concept

```
1. Deploy pyth-price-store to a test validator.
2. Before the deployer calls Initialize, submit:
     Initialize {
       payer:        attacker_keypair  (signer, writable),
       config:       PDA("CONFIG"),
       system:       system_program,
       args.authority: attacker_pubkey.to_bytes()
     }
3. Assert the transaction succeeds and config.authority == attacker_pubkey.
4. Deployer's Initialize now fails (AlreadyInitialized).
5. Call InitializePublisher signed by attacker_keypair — succeeds,
   registering an arbitrary publisher with an attacker-controlled buffer.
6. SubmitPrices from that publisher writes attacker-chosen prices into
   the buffer, corrupting the price feed.
``` [8](#0-7)

pub fn validate_payer<'a, 'b>(
    account: Option<&'b AccountInfo<'a>>,

pub fn validate_authority<'a, 'b>(
    account: Option<&'b AccountInfo<'a>>,
    config: &AccountInfo<'a>,
) -> Result<&'b AccountInfo<'a>, ProgramError> {
    let authority = account.ok_or(ProgramError::NotEnoughAccountKeys)?;
    ensure!(ProgramError::MissingRequiredSignature, authority.is_signer);
    ensure!(ProgramError::InvalidArgument, authority.is_writable);
    let config_data = config.data.borrow();
    let config = accounts::config::read(*config_data)?;
    ensure!(
        ProgramError::MissingRequiredSignature,
        authority.key.to_bytes() == config.authority
    );
    Ok(authority)

pub fn create(data: &mut [u8], authority: [u8; 32]) -> Result<&mut Config, ReadAccountError> {
    if data.len() < size_of::<Config>() {
        return Err(ReadAccountError::DataTooShort);
    }
    let data: &mut Config = from_bytes_mut(&mut data[..size_of::<Config>()]);
    if data.format != 0 {
        return Err(ReadAccountError::AlreadyInitialized);
    }
    data.format = FORMAT;
    data.authority = authority;
    Ok(data)
