### Title
Missing Staleness Check in `PythAggregatorV3` — All Price-Reading Functions Use `getPriceUnsafe()` Without Age Validation - (File: `target_chains/ethereum/sdk/solidity/PythAggregatorV3.sol`)

---

### Summary

`PythAggregatorV3` is Pyth's official Chainlink-compatible adapter contract, shipped as part of `@pythnetwork/pyth-sdk-solidity` and explicitly recommended for Chainlink-to-Pyth migrations. Every price-reading function in the contract calls `pyth.getPriceUnsafe()`, which returns prices from arbitrarily far in the past with no staleness check. Any protocol that integrates `PythAggregatorV3` as a drop-in Chainlink replacement will silently consume stale prices.

---

### Finding Description

`PythAggregatorV3` implements the Chainlink `AggregatorV3Interface` on top of Pyth. All six price-reading functions — `latestAnswer()`, `latestTimestamp()`, `latestRound()`, `getAnswer()`, `getRoundData()`, and `latestRoundData()` — call `pyth.getPriceUnsafe(priceId)` unconditionally: [1](#0-0) [2](#0-1) 

`getPriceUnsafe()` is explicitly documented as returning a price "from arbitrarily far in the past" with no recency guarantee: [3](#0-2) 

The safe alternative, `getPriceNoOlderThan(id, age)`, enforces a staleness threshold and reverts with `StalePrice` if the price is too old: [4](#0-3) 

`PythAggregatorV3` never calls `getPriceNoOlderThan()` and has no configurable `maxAge` parameter. The `updatedAt` field returned by `latestRoundData()` is simply `price.publishTime`, which can be hours or days old with no revert triggered: [5](#0-4) 

Pyth's own documentation and best-practices guide explicitly warn that `getPriceUnsafe` must never be used in production DeFi without a manual timestamp check, and that `getPriceNoOlderThan` should always be preferred: [6](#0-5) 

---

### Impact Explanation

Any lending, derivatives, or margin protocol that deploys `PythAggregatorV3` as a Chainlink feed replacement and calls `latestAnswer()` or `latestRoundData()` will receive stale prices without any revert or error signal. This can lead to:

- Incorrect collateral valuations enabling under-collateralized borrows
- Incorrect liquidation triggers (missed or spurious)
- Exploitable price discrepancies in perpetuals or options protocols

The Pyth documentation itself identifies this as a primary attack vector: [7](#0-6) 

---

### Likelihood Explanation

`PythAggregatorV3` is the officially documented migration path for Chainlink users. The developer hub explicitly instructs integrators to deploy it as a drop-in replacement: [8](#0-7) 

Pyth is a pull oracle — prices are only updated when someone submits an update transaction. If no keeper submits an update (due to network issues, market hours, or deliberate withholding), the on-chain price can become arbitrarily stale. Any protocol relying on `PythAggregatorV3` without adding its own staleness check downstream is fully exposed.

---

### Recommendation

Replace all `getPriceUnsafe()` calls in `PythAggregatorV3` with `getPriceNoOlderThan(priceId, maxAge)`, where `maxAge` is a configurable constructor or storage parameter. For example:

```solidity
uint public maxAge; // e.g., 60 seconds

constructor(address _pyth, bytes32 _priceId, uint _maxAge) {
    priceId = _priceId;
    pyth = IPyth(_pyth);
    maxAge = _maxAge;
}

function latestAnswer() public view virtual returns (int256) {
    PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
    return int256(price.price);
}

function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
    roundId = uint80(price.publishTime);
    return (roundId, int256(price.price), price.publishTime, price.publishTime, roundId);
}
```

This mirrors the fix applied in the referenced external report (adding a configurable `maxStaleness` field).

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {PythAggregatorV3} from "@pythnetwork/pyth-sdk-solidity/PythAggregatorV3.sol";
import {IPyth} from "@pythnetwork/pyth-sdk-solidity/IPyth.sol";
import {PythStructs} from "@pythnetwork/pyth-sdk-solidity/PythStructs.sol";

contract PythAggregatorV3StalenessTest is Test {
    address internal constant MOCK_PYTH = address(0xBEEF);
    bytes32 internal constant PRICE_ID = bytes32(uint256(1));

    function test_latestRoundData_acceptsStalePrice() public {
        vm.warp(block.timestamp + 10 days);

        PythStructs.Price memory stalePrice = PythStructs.Price({
            price: int64(100e8),
            conf: uint64(1e6),
            expo: int32(-8),
            publishTime: uint(block.timestamp - 1 days) // 1 day stale
        });

        vm.mockCall(
            MOCK_PYTH,
            abi.encodeWithSelector(IPyth.getPriceUnsafe.selector, PRICE_ID),
            abi.encode(stalePrice)
        );

        PythAggregatorV3 adapter = new PythAggregatorV3(MOCK_PYTH, PRICE_ID);

        // latestRoundData returns stale price without reverting
        (, int256 answer,, uint256 updatedAt,) = adapter.latestRoundData();

        // updatedAt is 1 day old — no revert, no error
        assertEq(answer, int256(100e8));
        assertEq(updatedAt, block.timestamp - 1 days);
    }
}
```

The test demonstrates that `latestRoundData()` returns a price with `updatedAt` one day in the past without any revert, exactly mirroring the vulnerability class in the external report. [9](#0-8) [1](#0-0) [10](#0-9) [11](#0-10) [5](#0-4)

    function getPriceNoOlderThan(
        bytes32 id,
        uint age
    ) public view virtual override returns (PythStructs.Price memory price) {
        price = getPriceUnsafe(id);

        if (diff(block.timestamp, price.publishTime) > age)
            revert PythErrors.StalePrice();

        return price;
    }
