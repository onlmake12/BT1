Audit Report

## Title
Attacker-Controlled Peer ID Accepted in Observed Address Poisons Victim's Broadcast Listen Addresses — (`network/src/protocols/identify/mod.rs`)

## Summary
The `add_observed_addr` implementation in `IdentifyCallback` only appends the local peer ID when no `/p2p/` component is present, but performs no validation when one already exists. A remote peer can send a crafted `observed_addr` containing a foreign peer ID, which is stored verbatim and subsequently included in the victim's `local_listen_addrs` broadcast to every newly connected peer, causing network-wide routing failures for the victim node.

## Finding Description
The guard in `IdentifyCallback::add_observed_addr` (L497–507) is gated solely on presence of a peer ID, not correctness:

```rust
fn add_observed_addr(&mut self, mut addr: Multiaddr, session_id: SessionId) -> MisbehaveResult {
    if extract_peer_id(&addr).is_none() {
        addr.push(Protocol::P2P(Cow::Borrowed(
            self.network_state.local_peer_id().as_bytes(),
        )))
    }
    self.network_state.add_observed_addr(session_id, addr);
    MisbehaveResult::Continue
}
``` [1](#0-0) 

If an attacker sends `/ip4/<victim_ip>/tcp/8114/p2p/<attacker_peer_id>`, `extract_peer_id` returns `Some(attacker_peer_id)`, the push is skipped, and the address with the wrong peer ID is stored verbatim. `NetworkState::add_observed_addr` performs no further validation:

```rust
pub(crate) fn add_observed_addr(&self, session_id: SessionId, addr: Multiaddr) {
    let mut pending_observed_addrs = self.observed_addrs.write();
    pending_observed_addrs.insert(session_id, addr);
}
``` [2](#0-1) 

`observed_addrs` also performs no filtering — it returns stored values as-is: [3](#0-2) 

These poisoned addresses are included in `local_listen_addrs`, which is sent to every newly connected peer on the `connected` event: [4](#0-3) [5](#0-4) 

The full exploit path: attacker establishes one valid inbound connection → sends `IdentifyMessage` with `observed_addr = /ip4/<victim_ip>/tcp/<port>/p2p/<attacker_peer_id>` → victim stores it via `add_observed_addr` → victim includes it in `local_listen_addrs` → victim broadcasts the poisoned address to all subsequent peers → those peers attempt to connect to `victim_ip:port` expecting `attacker_peer_id`, Noise/TLS handshake fails because the actual node presents `victim_peer_id`.

## Impact Explanation
This matches **High: Vulnerabilities or bad designs which could cause CKB network congestion with few costs**. The victim node's reachable address set is poisoned with a foreign peer ID. Every peer that receives the victim's `listen_addrs` will store an unconnectable address for the victim. As the victim connects to more peers, the poisoned address propagates network-wide, causing cascading connection failures and effectively isolating the victim from nodes that received the poisoned entry. The attacker needs only one accepted inbound connection to trigger this, and the effect is persistent until the victim restarts or the observed address is evicted.

## Likelihood Explanation
Any unprivileged peer that can establish a standard inbound connection to the victim can trigger this. No special privileges, no proof-of-work, no key material required. The attacker only needs to be accepted as a peer and send one crafted `IdentifyMessage`. The condition is trivially reachable on any publicly reachable CKB node.

## Recommendation
In `add_observed_addr`, when `extract_peer_id` returns `Some(peer_id)`, verify that `peer_id == self.network_state.local_peer_id()`. If it does not match, strip the foreign `/p2p/` component and append the correct local peer ID, or discard the address entirely:

```rust
fn add_observed_addr(&mut self, mut addr: Multiaddr, session_id: SessionId) -> MisbehaveResult {
    match extract_peer_id(&addr) {
        None => {
            addr.push(Protocol::P2P(Cow::Borrowed(
                self.network_state.local_peer_id().as_bytes(),
            )));
        }
        Some(peer_id) if peer_id != *self.network_state.local_peer_id() => {
            return MisbehaveResult::Continue; // discard or disconnect
        }
        _ => {}
    }
    self.network_state.add_observed_addr(session_id, addr);
    MisbehaveResult::Continue
}
```

## Proof of Concept
```rust
let local_peer_id = /* victim's real peer ID */;
let attacker_peer_id = /* attacker's peer ID */;

let mut addr: Multiaddr = "/ip4/1.2.3.4/tcp/8114".parse().unwrap();
addr.push(Protocol::P2P(Cow::Borrowed(attacker_peer_id.as_bytes())));

callback.add_observed_addr(addr, session_id);

let stored = network_state.observed_addrs(10);
let stored_peer_id = extract_peer_id(&stored[0]).unwrap();

// Fails on unpatched code: stored peer ID is attacker's, not victim's
assert_eq!(stored_peer_id, local_peer_id);
```

This unit test directly exercises `add_observed_addr` with a crafted address and asserts the invariant that the stored peer ID must equal the local peer ID. It will fail on the current code, confirming the vulnerability.
