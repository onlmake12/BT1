### Title
Unvalidated `providerToCredit` in `Echo.executeCallback` Allows Permanent Fee Locking - (File: target_chains/ethereum/contracts/contracts/echo/Echo.sol)

### Summary
`Echo.executeCallback` accepts a caller-supplied `providerToCredit` address with no zero-address or registration check. After the exclusivity period elapses, any unprivileged caller can invoke `executeCallback` with `providerToCredit = address(0)`, crediting the provider's earned fees to `_state.providers[address(0)].accruedFeesInWei` — permanently locking them inside the contract with no recovery path.

### Finding Description

`Echo.executeCallback` accepts `providerToCredit` as an externally supplied parameter: [1](#0-0) 

The fee crediting line inside `executeCallback` writes directly to the mapping keyed by `providerToCredit` without any zero-address or registration guard: [2](#0-1) 

The interface documentation explicitly acknowledges that after the exclusivity period, `providerToCredit` need not be the original request provider: [3](#0-2) 

The only withdrawal paths for provider fees are `withdrawAsFeeManager` (requires `msg.sender == feeManager`) and `withdrawFees` (admin-only, for Pyth protocol fees only): [4](#0-3) [5](#0-4) 

Neither path can recover fees credited to `address(0)`, since `msg.sender` can never equal `address(0)`. The fees are permanently locked in `_state.providers[address(0)].accruedFeesInWei`.

By contrast, `EntropyGovernance.withdrawFee` and `Echo._initialize` both explicitly guard against `address(0)`: [6](#0-5) [7](#0-6) 

The missing guard in `executeCallback` is the inconsistency that creates the vulnerability.

### Impact Explanation

A malicious actor waits for the exclusivity period (default 15 seconds) to elapse on any pending request, then calls `executeCallback(address(0), sequenceNumber, updateData, priceIds)`. The callback is fulfilled correctly for the consumer, but all fees (`req.fee + msg.value - pythFee`) are credited to `_state.providers[address(0)].accruedFeesInWei` and are irrecoverable. The legitimate provider loses their earned fees with no recourse.

**Impact: Medium** — direct, permanent loss of provider fee revenue.

### Likelihood Explanation

**Likelihood: Medium** — The exclusivity period is only 15 seconds by default. Any time a provider is slow to fulfill (network congestion, downtime, misconfiguration), a griefing actor monitoring the mempool can front-run or race the provider's `executeCallback` call with `providerToCredit = address(0)`. No privileged access is required; any EOA can execute this after the exclusivity window.

### Recommendation

Add a zero-address and/or registration check for `providerToCredit` at the top of `executeCallback`:

```solidity
require(providerToCredit != address(0), "providerToCredit is zero address");
require(_state.providers[providerToCredit].isRegistered, "providerToCredit not registered");
```

This mirrors the existing guard pattern already used in `_initialize` and `EntropyGovernance.withdrawFee`.

### Proof of Concept

```solidity
function test_providerToCreditZeroAddress() public {
    // 1. Consumer makes a request
    bytes32[] memory priceIds = createPriceIds();
    vm.deal(address(consumer), 1 gwei);
    vm.prank(address(consumer));
    uint64 seq = echo.requestPriceUpdatesWithCallback{value: calculateTotalFee()}(
        defaultProvider,
        SafeCast.toUint64(block.timestamp),
        priceIds,
        CALLBACK_GAS_LIMIT
    );

    // 2. Wait for exclusivity period to elapse
    vm.warp(block.timestamp + 16);

    // 3. Attacker calls executeCallback with address(0) as providerToCredit
    PythStructs.PriceFeed[] memory priceFeeds = createMockPriceFeeds(block.timestamp - 16);
    mockParsePriceFeedUpdates(pyth, priceFeeds);
    bytes[] memory updateData = createMockUpdateData(priceFeeds);

    vm.prank(address(0xdead)); // unprivileged attacker
    echo.executeCallback(address(0), seq, updateData, priceIds);

    // 4. Fees are now locked in address(0)'s mapping entry
    // defaultProvider earned nothing; address(0) has the fees but they are unrecoverable
    assertEq(echo.getProviderInfo(defaultProvider).accruedFeesInWei, 0);
    // _state.providers[address(0)].accruedFeesInWei > 0, permanently locked
}
```
