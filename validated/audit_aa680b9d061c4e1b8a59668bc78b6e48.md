### Title
Signature Replay in `PythLazer.verifyUpdate()` Allows Stale Price Injection — (File: `lazer/contracts/evm/src/PythLazer.sol`)

---

### Summary

`PythLazer.verifyUpdate()` performs ECDSA signature verification against a trusted-signer list but does **not** mark updates as consumed, enforce payload timestamp freshness, or include any nonce. Any previously valid signed Lazer update can be replayed indefinitely (until the signer key expires). Consumer contracts that treat a successful `verifyUpdate()` call as proof of freshness — without independently checking the embedded timestamp — are vulnerable to stale-price injection.

---

### Finding Description

`verifyUpdate()` in `PythLazer.sol` (lines 70–106) does the following:

1. Checks the fee.
2. Validates the EVM format magic.
3. Recovers the ECDSA signer from the raw bytes.
4. Checks `isValidSigner(signer)` — i.e., `block.timestamp < trustedSignerToExpiresAtMapping[signer]`. [1](#0-0) 

There is no:
- Set or mapping of consumed update hashes.
- Minimum-age / maximum-age check on the timestamp embedded in the payload.
- Nonce or sequence number bound to the signer.

The function simply returns `(payload, signer)` to the caller. The same `update` bytes will pass every subsequent call as long as the signer has not expired.

The Aptos Lazer contract has the identical structural gap: `verify_message()` checks the signer's expiry but never records the message as used. [2](#0-1) 

The Sui Lazer contract `verify_le_ecdsa_message()` is the same pattern. [3](#0-2) 

---

### Impact Explanation

A consumer contract that calls `verifyUpdate()` and writes the returned price to storage without checking `payload.timestamp > lastAcceptedTimestamp` can be fed an arbitrarily old price. Concretely:

- At time T₁, price = P₁. A valid signed update is broadcast and accepted on-chain.
- At time T₂ > T₁, price moves to P₂ (e.g., a large drop).
- Attacker replays the T₁ update bytes. `verifyUpdate()` succeeds (signer still valid, fee paid).
- Consumer contract stores P₁ as the current price.
- Attacker exploits the inflated price: borrows against over-valued collateral, avoids liquidation, or triggers incorrect liquidations of other users.

This is a direct loss-of-funds path for any DeFi protocol integrated with Pyth Lazer that does not independently enforce timestamp monotonicity.

---

### Likelihood Explanation

**Medium.** The attack requires:
1. A consumer contract that omits the `_timestamp > lastTimestamp` guard (a common integration mistake, especially given that the Pyth Lazer documentation shows the guard only as an example, not enforced by the contract).
2. A meaningful price movement after the captured update.
3. The trusted signer key to still be valid — signer expiry windows are set far in the future (e.g., `3000000000000000` in the test fixture), so this condition is almost always satisfied.

The attacker needs no privileged access; `verifyUpdate()` is `external payable` and callable by anyone. [4](#0-3) 

---

### Recommendation

Add replay protection directly inside `verifyUpdate()`:

```solidity
mapping(bytes32 => bool) private _usedUpdates;

function verifyUpdate(bytes calldata update)
    external payable
    returns (bytes calldata payload, address signer)
{
    // ... existing checks ...

    bytes32 updateHash = keccak256(update);
    require(!_usedUpdates[updateHash], "update already used");
    _usedUpdates[updateHash] = true;

    // ... return payload, signer ...
}
```

Alternatively (or additionally), parse the `uint64 timestamp` from the payload header and require it to be within an acceptable freshness window (e.g., `block.timestamp - MAX_AGE_SECONDS`). This is cheaper than storing hashes and also prevents cross-chain replay of the same signed bytes on a different EVM deployment. [5](#0-4) 

---

### Proof of Concept

```
1. Deploy a minimal consumer contract:
   function updatePrice(bytes calldata update) external payable {
       (bytes calldata payload, ) = pythLazer.verifyUpdate{value: fee}(update);
       (uint64 ts, , , ) = PythLazerLib.parsePayloadHeader(payload);
       // BUG: no check that ts > lastTimestamp
       price = PythLazerLib.getPrice(...);
   }

2. At block N, a keeper submits update_N (price = 1000 USDC).
   Consumer stores price = 1000.

3. At block N+100, real price drops to 500 USDC.
   A new update_N+100 is submitted; consumer stores price = 500.

4. Attacker calls consumer.updatePrice(update_N) — the old bytes.
   pythLazer.verifyUpdate() succeeds (signer valid, fee paid, no replay check).
   Consumer stores price = 1000 again.

5. Attacker borrows against the inflated collateral value, extracting funds.
```

    function verifyUpdate(
        bytes calldata update
    ) external payable returns (bytes calldata payload, address signer) {
        // Require fee and refund excess
        require(msg.value >= verification_fee, "Insufficient fee provided");
        if (msg.value > verification_fee) {
            payable(msg.sender).transfer(msg.value - verification_fee);
        }

        if (update.length < 71) {
            revert("input too short");
        }
        uint32 EVM_FORMAT_MAGIC = 706910618;

        uint32 evm_magic = uint32(bytes4(update[0:4]));
        if (evm_magic != EVM_FORMAT_MAGIC) {
            revert("invalid evm magic");
        }
        uint16 payload_len = uint16(bytes2(update[69:71]));
        if (update.length < 71 + payload_len) {
            revert("input too short");
        }
        payload = update[71:71 + payload_len];
        bytes32 hash = keccak256(payload);
        (signer, , ) = ECDSA.tryRecover(
            hash,
            uint8(update[68]) + 27,
            bytes32(update[4:36]),
            bytes32(update[36:68])
        );
        if (signer == address(0)) {
            revert("invalid signature");
        }
        if (!isValidSigner(signer)) {
            revert("invalid signer");
        }
    }

    public fun verify_message(
        message: vector<u8>,
        signature: vector<u8>,
        trusted_signer: vector<u8>,
        fee: coin::Coin<AptosCoin>
    ) acquires Storage {
        let storage = borrow_global<Storage>(@pyth_lazer);

        // Verify fee amount
        assert!(coin::value(&fee) >= storage.single_update_fee, EINSUFFICIENT_FEE);

        // Transfer fee to treasury
        coin::deposit(storage.treasury, fee);

        // Verify signer is trusted and not expired
        let i = 0;
        let valid = false;
        while (i < storage.trusted_signers.length()) {
            let signer_info = vector::borrow(&storage.trusted_signers, (i as u64));
            if (&signer_info.pubkey == &trusted_signer
                && signer_info.expires_at > timestamp::now_seconds()) {
                valid = true;
                break
            };
            i = i + 1;
        };
        assert!(valid, EINVALID_SIGNER);

        // Verify signature
        let sig = ed25519::new_signature_from_bytes(signature);
        let pk = ed25519::new_unvalidated_public_key_from_bytes(trusted_signer);
        assert!(
            ed25519::signature_verify_strict(&sig, &pk, message),
            EINVALID_SIGNATURE
        );
    }

public(package) fun verify_le_ecdsa_message(
    state: &State,
    clock: &Clock,
    signature: &vector<u8>,
    payload: &vector<u8>,
) {
    let current_cap = state.current_cap();

    // 0 stands for keccak256 hash
    let pubkey = secp256k1_ecrecover(signature, payload, 0);

    // Check if the recovered pubkey is in the trusted signers list
    let trusted_signers = state.trusted_signers(&current_cap);
    let mut maybe_idx = trusted_signers.find_index!(|signer|
        signer.public_key() == &pubkey
    );

    assert!(maybe_idx.is_some(), ESignerNotTrusted);
    let idx = maybe_idx.extract();
    let expires_at_ms = trusted_signers[idx].expires_at_ms();
    assert!(clock.timestamp_ms() < expires_at_ms, ESignerExpired);
}
