### Title
Signed Lazer Update Replay — No On-Chain Consumption Tracking in `verifyUpdate()` - (File: `lazer/contracts/evm/src/PythLazer.sol`)

---

### Summary

`PythLazer.verifyUpdate()` performs ECDSA signature verification against a trusted signer but never records that a given signed update has been consumed. Any caller can replay an identical signed update bytes an unlimited number of times. The contract will accept each replay as valid, returning the same stale payload to the consumer on every call.

---

### Finding Description

`PythLazer.verifyUpdate()` checks three things and nothing more:

1. `msg.value >= verification_fee`
2. The EVM magic bytes match
3. The recovered ECDSA signer is in the trusted-signer list and has not expired [1](#0-0) 

There is no mapping, set, or counter that records which `(hash, signer)` pairs have already been verified. The contract stores only `trustedSigners` and `trustedSignerToExpiresAtMapping`. [2](#0-1) 

The existing test suite explicitly demonstrates the same `update` bytes being submitted twice in the same test and both calls succeeding: [3](#0-2) 

The Sui analog (`parse_and_verify_le_ecdsa_update_v2`) has the same structure — it verifies the signature but stores nothing about the consumed update: [4](#0-3) 

---

### Impact Explanation

An attacker who captures a legitimately signed Lazer update (e.g., from a public mempool or by observing a prior on-chain call) can replay it to any consumer contract that calls `verifyUpdate()`. The consumer receives the same payload — including the old timestamp and old price — on every replay. Consumer contracts that do not enforce a strict freshness window (or that rely on the Pyth contract to enforce it) will accept the stale price as current. This enables:

- Forced liquidations at an artificially stale price
- Arbitrage against a DeFi protocol using a price that no longer reflects the market
- Denial-of-service by flooding a consumer's price-update function with old data, preventing the real current price from being applied (if the consumer uses a "latest wins" pattern without a timestamp guard)

---

### Likelihood Explanation

The attack requires only:
1. Observing any prior valid `verifyUpdate()` call (public on-chain data)
2. Paying the `verification_fee` (currently 1 wei)
3. Calling `verifyUpdate()` with the captured bytes

No privileged access, no key compromise, and no governance majority is needed. The fee is negligible. The window of opportunity is as long as the trusted signer's `expiresAt` has not passed, which is typically set far in the future. [5](#0-4) 

---

### Recommendation

Maintain a `mapping(bytes32 => bool) internal consumedUpdates` keyed on `keccak256(update)` (or on `keccak256(payload)`). Before returning, assert the hash has not been seen and then mark it consumed. Alternatively, enforce a maximum age on the `timestamp` field parsed from the payload inside `verifyUpdate()` itself, so that any update older than N seconds is rejected at the protocol level rather than relying solely on consumer-side checks.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity ^0.8.13;

interface IPythLazer {
    function verifyUpdate(bytes calldata update)
        external payable returns (bytes calldata payload, address signer);
    function verification_fee() external view returns (uint256);
}

contract LazerReplayPoC {
    IPythLazer public pythLazer;

    constructor(address _pythLazer) {
        pythLazer = IPythLazer(_pythLazer);
    }

    /// @notice Replay a previously observed signed update N times.
    /// Each call succeeds and returns the same stale payload.
    function replay(bytes calldata capturedUpdate, uint256 times) external payable {
        uint256 fee = pythLazer.verification_fee();
        require(msg.value >= fee * times, "insufficient ETH");
        for (uint256 i = 0; i < times; i++) {
            // verifyUpdate succeeds every iteration — no replay guard exists
            (bytes memory payload, ) =
                pythLazer.verifyUpdate{value: fee}(capturedUpdate);
            // payload contains the original (stale) timestamp and price
            // A vulnerable consumer would accept this as the current price
        }
    }
}
```

Steps:
1. Deploy `LazerReplayPoC` pointing at the live `PythLazer` proxy.
2. Observe any prior `verifyUpdate` call on-chain and copy its `update` calldata.
3. Call `replay(capturedUpdate, 10)` with `10 * verification_fee` ETH.
4. All 10 calls succeed; each returns the same stale payload.
5. Any consumer contract that does not independently validate `_timestamp > lastAcceptedTimestamp` will accept the stale price on every call.

        // Require fee and refund excess
        require(msg.value >= verification_fee, "Insufficient fee provided");
        if (msg.value > verification_fee) {
            payable(msg.sender).transfer(msg.value - verification_fee);
        }

        if (update.length < 71) {
            revert("input too short");
        }
        uint32 EVM_FORMAT_MAGIC = 706910618;

            revert("input too short");

public fun parse_and_verify_le_ecdsa_update_v2(s: &State, clock: &Clock, update: vector<u8>): Update {
    let mut cursor = bcs::new(update);

    // Parse and validate message magic
    let magic = cursor.peel_u32();
    assert!(magic == UPDATE_MESSAGE_MAGIC, EInvalidUpdateMagic);

    // Parse signature
    let mut signature = vector::empty<u8>();
    let mut sig_i = 0;
    while (sig_i < SECP256K1_SIG_LEN) {
        signature.push_back(cursor.peel_u8());
        sig_i = sig_i + 1;
    };

    // Parse expected payload length and get remaining bytes as payload
    let payload_len = cursor.peel_u16();
    let payload = cursor.into_remainder_bytes();

    // Validate expected payload length
    assert!(payload_len as u64 == payload.length(), EInvalidPayloadLength);

    // Parse payload
    let mut payload_cursor = bcs::new(payload);
    let payload_magic = payload_cursor.peel_u32();
    assert!(payload_magic == PAYLOAD_MAGIC, EInvalidPayloadMagic);

    verify_le_ecdsa_message(s, clock, &signature, &payload);
