### Title
Unbounded IO Amplification in `GetTransactionsProofProcess::execute` via 1000 Distinct-Block tx_hashes — (`util/light-client-protocol-server/src/components/get_transactions_proof.rs`)

---

### Summary

Any unprivileged peer connected to the light client protocol server can send a single `GetTransactionsProof` message with 1000 tx_hashes each from a different block. The handler performs ~4000 uncached DB reads plus 1000 full block body deserializations (all transactions in each block), with no per-peer rate limiting. Two additional redundant DB reads per block (uncles + extension) are also performed because `get_block` already loads them internally but the handler loads them again explicitly.

---

### Finding Description

**Entrypoint**: `LightClientProtocol::received` → `try_process` → `GetTransactionsProofProcess::execute` [1](#0-0) 

Unlike `Relayer` and `HolePunching`, the light client protocol handler has **no rate limiter** — every message is processed unconditionally. [2](#0-1) 

The limit is 1000 tx_hashes per request.

**Phase 1 — Partition (lines 54–64):** For each of the 1000 tx_hashes, `get_transaction_info` (1 `COLUMN_TRANSACTION_INFO` read) and `is_main_chain` (1 `COLUMN_INDEX` read) are called = **2000 DB reads**. [3](#0-2) 

**Phase 2 — Grouping (lines 67–75):** For each found tx_hash, `get_transaction_with_info` is called, which internally calls `get_transaction_info` again (duplicate read) then reads the individual tx from `COLUMN_BLOCK_BODY` = **2000 more DB reads** (1000 of which are duplicate `COLUMN_TRANSACTION_INFO` reads). [4](#0-3) [5](#0-4) 

**Phase 3 — Block loading (lines 82–126):** For each of the 1000 distinct blocks, `get_block` is called. `get_block` performs a full iterator scan over `COLUMN_BLOCK_BODY` deserializing **every transaction in the block** — not just the one requested — to build the CBMT proof. [6](#0-5) [7](#0-6) 

`get_block_body` uses `get_iter` (an iterator scan), which is **not cached** by `StoreCache` (only headers are cached).

**Redundant reads:** `get_block` already loads uncles and extension internally (store.rs lines 56–62), but lines 119–122 of the handler load them again separately, doubling those reads per block. [8](#0-7) 

---

### Impact Explanation

A single well-crafted request triggers:
- ~4000 uncached `COLUMN_TRANSACTION_INFO` / `COLUMN_INDEX` DB reads
- 1000 full block body iterator scans (each block may contain hundreds of transactions, all deserialized into memory)
- 2000 redundant uncles + extension reads (loaded twice per block)

The returned proof is tiny (1 tx per block × 1000 blocks), but the server-side work is proportional to the total transaction count across all 1000 blocks. This is a concrete IO amplification: proof output size ≪ DB bytes read. With no rate limiting on the light client protocol handler, an attacker can sustain this continuously, causing significant IO pressure and memory allocation on the serving node.

---

### Likelihood Explanation

The attacker only needs to know 1000 valid main-chain tx_hashes from 1000 different blocks — trivially obtained from any public block explorer. No authentication, no PoW, no privileged access required. The attack is repeatable at will.

---

### Recommendation

1. Add a per-peer rate limiter to `LightClientProtocol::received` (matching the pattern used in `Relayer` and `HolePunching`).
2. Eliminate the duplicate `get_transaction_info` call: the partition step already fetches `tx_info`; pass it through to the grouping step instead of calling `get_transaction_with_info` which re-fetches it.
3. Avoid calling `get_block` (full body deserialization) just to compute CBMT indices. Since `tx_info.index` is already known, only the tx hashes of the block are needed — use a lighter query (e.g., iterate `COLUMN_BLOCK_BODY` keys only, or store a separate tx-hash-list index).
4. Remove the redundant `get_block_uncles` / `get_block_extension` calls at lines 119–122 since `get_block` already loads them.

---

### Proof of Concept

```
1. Connect to a CKB full node with light client protocol enabled.
2. Collect 1000 confirmed tx_hashes, each from a different block (from a block explorer).
3. Send a single GetTransactionsProof P2P message with all 1000 hashes and a valid last_hash.
4. Instrument the node's RocksDB read counters before and after.
5. Observe: ~4000+ COLUMN_TRANSACTION_INFO/COLUMN_INDEX reads + 1000 full COLUMN_BLOCK_BODY
   iterator scans, while the returned SendTransactionsProof contains only 1000 small filtered blocks.
6. Repeat in a tight loop from a single peer — no ban is triggered for well-formed messages.
```

        &mut self,
        nc: Arc<dyn CKBProtocolContext + Sync>,
        peer: PeerIndex,

                return;

        let item_name = msg.item_name();
        let status = self.try_process(&nc, peer, msg).await;
        if let Some(ban_time) = status.should_ban() {
            error!(

                item_name, peer, ban_time, status
            );
            nc.ban_peer(peer, ban_time, status.to_string());
        } else if status.should_warn() {
            warn!("process {} from {}; result is {}", item_name, peer, status);
        } else if !status.is_ok() {
            debug!("process {} from {}; result is {}", item_name, peer, status);
        }
    }

    fn get_block(&self, h: &packed::Byte32) -> Option<BlockView> {
        let header = self.get_block_header(h)?;
        if let Some(freezer) = self.freezer()
            && header.number() > 0
            && header.number() < freezer.number()
        {
            let raw_block = freezer.retrieve(header.number()).expect("block frozen")?;
            let raw_block_reader =
                packed::BlockReader::from_compatible_slice(&raw_block).expect("checked data");
            if raw_block_reader.calc_header_hash().as_slice() == h.as_slice() {
                return Some(raw_block_reader.to_entity().into_view());
            }
        }
        let body = self.get_block_body(h);
        let uncles = self
            .get_block_uncles(h)
            .expect("block uncles must be stored");
        let proposals = self
            .get_block_proposal_txs_ids(h)
            .expect("block proposal_ids must be stored");
        let extension_opt = self.get_block_extension(h);

        let block = if let Some(extension) = extension_opt {
            BlockView::new_unchecked_with_extension(header, uncles, body, proposals, extension)
        } else {
            BlockView::new_unchecked(header, uncles, body, proposals)
        };
        Some(block)
    }

    fn get_block_body(&self, hash: &packed::Byte32) -> Vec<TransactionView> {
        let prefix = hash.as_slice();
        self.get_iter(
            COLUMN_BLOCK_BODY,
            IteratorMode::From(prefix, Direction::Forward),
        )
        .take_while(|(key, _)| key.starts_with(prefix))
        .map(|(_key, value)| {
            let reader = packed::TransactionViewReader::from_slice_should_be_ok(value.as_ref());
            Into::<TransactionView>::into(reader)
        })
        .collect()
    }

        &self,
        hash: &packed::Byte32,
    ) -> Option<(TransactionView, TransactionInfo)> {
        let tx_info = self.get_transaction_info(hash)?;
        if let Some(freezer) = self.freezer()
            && tx_info.block_number > 0
            && tx_info.block_number < freezer.number()
        {
            let raw_block = freezer
                .retrieve(tx_info.block_number)
                .expect("block frozen")?;
            let raw_block_reader =
                packed::BlockReader::from_compatible_slice(&raw_block).expect("checked data");
            if raw_block_reader.calc_header_hash().as_slice() == tx_info.block_hash.as_slice()
                && let Some(tx_reader) = raw_block_reader.transactions().get(tx_info.index)
                && tx_reader.calc_tx_hash().as_slice() == hash.as_slice()
            {
                return Some((tx_reader.to_entity().into_view(), tx_info));

        self.get(COLUMN_BLOCK_BODY, tx_info.key().as_slice())
            .map(|slice| {
                let reader = packed::TransactionViewReader::from_slice_should_be_ok(slice.as_ref());
                (reader.into(), tx_info)
            })
    }
