### Title
`estimate_fee_rate` Fallback Does Not Subtract Basic Block Overhead, Producing Systematically Low Fee Rate Estimates - (`tx-pool/src/pool.rs`, `tx-pool/src/component/pool_map.rs`)

---

### Summary

The fallback fee-rate estimator in `TxPool::estimate_fee_rate` passes the full consensus `max_block_bytes` as the available space for transactions. The actual block assembler (`BlockAssembler::update_full`) subtracts the basic block overhead (cellbase, uncles, proposals, extension) before packing transactions. This inconsistency causes the fallback to simulate more transactions fitting per block than actually can, producing a fee rate that is systematically too low.

---

### Finding Description

**Preview path — `TxPool::estimate_fee_rate` (fallback):** [1](#0-0) 

```rust
let fee_rate = self.pool_map.estimate_fee_rate(
    (target_to_be_committed - self.snapshot.consensus().tx_proposal_window().closest())
        as usize,
    self.snapshot.consensus().max_block_bytes() as usize,   // ← full block limit
    self.snapshot.consensus().max_block_cycles(),
    self.config.min_fee_rate,
);
```

`pool_map::estimate_fee_rate` then simulates block packing by accumulating transaction sizes until `current_block_bytes >= max_block_bytes`: [2](#0-1) 

No deduction for cellbase, uncles, proposals, or extension is made.

**Actual execution path — `BlockAssembler::update_full`:** [3](#0-2) 

```rust
let basic_size = Self::basic_block_size(
    current_template.cellbase.data(),
    uncles,
    proposals.iter(),
    current_template.extension.clone(),
);

let txs_size_limit = max_block_bytes
    .checked_sub(basic_size)          // ← overhead subtracted here
    .ok_or(BlockAssemblerError::Overflow)?;

let (txs, _txs_size, _cycles) =
    tx_pool_reader.package_txs(max_block_cycles, txs_size_limit);
```

The same pattern appears in `update_transactions`: [4](#0-3) 

The fallback estimate never calls `basic_block_size` and never subtracts it. The actual assembler always does.

**The fallback is invoked via the public `estimate_fee_rate` RPC** when the primary estimator (ConfirmationFraction / WeightUnitsFlow) lacks sufficient historical data: [5](#0-4) [6](#0-5) 

---

### Impact Explanation

The fallback simulates `N` transactions fitting in a block using the full `max_block_bytes` budget. In reality, the assembler reserves `basic_block_size` bytes for the cellbase, uncles, proposals, and extension before packing transactions. The fallback therefore overestimates how many transactions fit per block, which causes it to return a fee rate lower than what is actually required to be included within `target_blocks` blocks.

A user who calls `estimate_fee_rate` (especially on a freshly started node where the primary estimator has no data and the fallback is always used) will receive a fee rate that is too low. Their transaction may not be confirmed within the expected window, requiring a fee bump or extended wait.

The magnitude of the underestimate scales with `basic_block_size / max_block_bytes`. With mainnet `max_block_bytes = 597,000` bytes and a block carrying 2 uncles plus a full proposal list, the basic overhead can reach tens of kilobytes, producing a non-trivial underestimate.

---

### Likelihood Explanation

- The fallback is the **default** path (`enable_fallback: true`) and is always active when the primary estimator returns `LackData` or `NoProperFeeRate` — which is guaranteed on a freshly started or recently restarted node.
- Any unprivileged RPC caller can trigger it by calling `estimate_fee_rate` with no special permissions.
- The inconsistency is structural and reproducible; it is not gated by any configuration flag.

---

### Recommendation

In `TxPool::estimate_fee_rate`, compute and subtract the current basic block overhead before calling `pool_map.estimate_fee_rate`, mirroring the logic in `BlockAssembler::update_full`:

```rust
let basic_block_size = /* cellbase + uncles + proposals + extension */;
let effective_max_bytes = consensus.max_block_bytes() as usize
    .saturating_sub(basic_block_size);

let fee_rate = self.pool_map.estimate_fee_rate(
    target_blocks,
    effective_max_bytes,   // ← consistent with assembler
    consensus.max_block_cycles(),
    self.config.min_fee_rate,
);
```

Alternatively, expose `basic_block_size` from `BlockAssembler` so the pool can query it directly.

---

### Proof of Concept

1. Start a fresh CKB node (primary estimator has no data → fallback is always used).
2. Fill the tx-pool with transactions of varying fee rates.
3. Call `estimate_fee_rate` via RPC → returns fee rate `F`.
4. Observe that `BlockAssembler::update_full` packs fewer transactions per block than the fallback assumed (because it subtracts `basic_block_size`).
5. Submit a transaction at fee rate `F`; it will not be included within the estimated number of blocks because the actual per-block transaction capacity is smaller than the fallback assumed.

The root cause is the missing `basic_block_size` subtraction in `pool.rs` line 567 compared to `block_assembler/mod.rs` lines 206–208. [7](#0-6) [8](#0-7)

    pub(crate) fn estimate_fee_rate(
        &self,
        mut target_blocks: usize,
        max_block_bytes: usize,
        max_block_cycles: Cycle,
        min_fee_rate: FeeRate,
    ) -> FeeRate {
        debug_assert!(target_blocks > 0);
        let iter = self.entries.iter_by_score().rev();
        let mut current_block_bytes = 0;
        let mut current_block_cycles = 0;
        for entry in iter {
            current_block_bytes += entry.inner.size;
            current_block_cycles += entry.inner.cycles;
            if current_block_bytes >= max_block_bytes || current_block_cycles >= max_block_cycles {
                target_blocks -= 1;
                if target_blocks == 0 {
                    return entry.inner.fee_rate();
                }
                current_block_bytes = entry.inner.size;
                current_block_cycles = entry.inner.cycles;
            }
        }

        min_fee_rate
    }

        let all_entry_info = self.tx_pool.read().await.get_all_entry_info();

            .estimate_fee_rate(estimate_mode, all_entry_info)
        {
            Ok(fee_rate) => Ok(fee_rate),

                if enable_fallback {
                    let target_blocks =
                        FeeEstimator::target_blocks_for_estimate_mode(estimate_mode);
                    self.tx_pool
                        .read()
                        .await
                        .estimate_fee_rate(target_blocks)
                        .map_err(Into::into)
                } else {
                    Err(err.into())
                }
            }
        }
    }
