### Title
Deviation-Only Subscription Permanently Locked When All Stored Prices Are Zero - (`target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

### Summary

In `Scheduler.sol`, when a subscription is configured with `updateOnDeviation = true` and `updateOnHeartbeat = false`, and all stored previous prices for the subscription's feeds are zero, the deviation check loop silently skips every feed via `continue` and falls through to an unconditional `revert UpdateConditionsNotMet()`. No update can ever be accepted, permanently locking the subscription.

### Finding Description

In `_validateShouldUpdatePrices`, the deviation check loop contains an explicit zero-price guard:

```solidity
// Skip if either price is zero to avoid division by zero
if (previousPrice == 0 || currentPrice == 0) {
    continue;
}
``` [1](#0-0) 

If every feed in the subscription satisfies this guard (i.e., every stored `previousPrice` is 0, or every incoming `currentPrice` is 0), the loop completes without ever executing `return updateTimestamp`. Control falls through to:

```solidity
revert SchedulerErrors.UpdateConditionsNotMet();
``` [2](#0-1) 

The heartbeat branch is only reached when `updateOnHeartbeat = true`:

```solidity
if (params.updateCriteria.updateOnHeartbeat) {
    ...
    return updateTimestamp;
}
``` [3](#0-2) 

So for a deviation-only subscription (`updateOnHeartbeat = false`, `updateOnDeviation = true`), there is no fallback path. The full `_validateShouldUpdatePrices` function is: [4](#0-3) 

**Root cause:** The zero-price skip is a necessary guard against division by zero in `Math.mulDiv(numerator, 10_000, denominator)`, but there is no handling for the case where all feeds are skipped. The `updateOnDeviation` flag remains `true` (active) while the stored price values are zero — a direct analog to the Fluid DEX pattern where a flag indicates an operation is active but the corresponding value is zero.

**How the locked state is reached:**

1. A subscription is created with `updateOnDeviation = true`, `updateOnHeartbeat = false`, and `deviationThresholdBps > 0` (all pass `_validateSubscriptionParams`). [5](#0-4) 

2. The first `updatePriceFeeds` call succeeds because `previousFeed.id == bytes32(0)` triggers an early return. [6](#0-5) 

3. The stored price for every feed is written as `price = 0` (a valid `int64` value that Pyth can legitimately report for halted markets or newly listed assets). [7](#0-6) 

4. Every subsequent `updatePriceFeeds` call hits `previousPrice == 0` → `continue` for all feeds → falls through to `revert UpdateConditionsNotMet()`. The subscription can never be updated again.

### Impact Explanation

All operations that depend on fresh prices from this subscription are broken. Readers calling `getPricesNoOlderThan` will receive `StalePrice` errors. The subscription balance continues to be held but no keeper can successfully push an update. The subscription manager can recover by calling `updateSubscription` to add `updateOnHeartbeat = true`, but this requires an active manager response and is not automatic.

This is analogous to the Fluid DEX issue: the `updateOnDeviation` flag is set (active), but the stored price value is zero, making the guarded operation permanently unreachable — locking the subscription.

### Likelihood Explanation

- Pyth price feeds can legitimately report `price = 0` for halted markets, newly listed assets, or assets with zero-value reporting.
- A subscription with a single price feed is sufficient — only one feed needs a stored price of 0.
- Any unprivileged keeper who submits the first valid Pyth update containing a zero price triggers the locked state.
- No privileged access is required; the entry path is the public `updatePriceFeeds` function callable by anyone.

### Recommendation

When all feeds are skipped in the deviation loop, the function should not unconditionally revert. Instead, treat the all-skipped case as a non-triggering condition (not an error), or add a fallback that allows an update when no deviation can be computed:

```solidity
if (params.updateCriteria.updateOnDeviation) {
    bool anyCheckable = false;
    for (uint8 i = 0; i < priceFeeds.length; i++) {
        PythStructs.PriceFeed storage previousFeed = _state.priceUpdates[subscriptionId][priceFeeds[i].id];
        if (previousFeed.id == bytes32(0)) {
            return updateTimestamp;
        }
        int64 currentPrice = priceFeeds[i].price.price;
        int64 previousPrice = previousFeed.price.price;
        if (previousPrice == 0 || currentPrice == 0) {
            continue;
        }
        anyCheckable = true;
        uint256 numerator = SignedMath.abs(currentPrice - previousPrice);
        uint256 denominator = SignedMath.abs(previousPrice);
        uint256 deviationBps = Math.mulDiv(numerator, 10_000, denominator);
        if (deviationBps >= params.updateCriteria.deviationThresholdBps) {
            return updateTimestamp;
        }
    }
    // If no feed was checkable (all prices zero), allow the update to proceed
    // to avoid permanently locking the subscription.
    if (!anyCheckable) {
        return updateTimestamp;
    }
}
```

```text
    function _validateShouldUpdatePrices(
        uint256 subscriptionId,
        SchedulerStructs.SubscriptionParams storage params,
        SchedulerStructs.SubscriptionStatus storage status,
        PythStructs.PriceFeed[] memory priceFeeds
    ) internal view returns (uint256) {
        // Use the most recent timestamp, as some asset markets may be closed.
        // Closed markets will have a publishTime from their last trading period.
        // Since we verify all updates share the same Pythnet slot, we still ensure
        // that all price feeds are synchronized from the same update cycle.
        uint256 updateTimestamp = 0;
        for (uint8 i = 0; i < priceFeeds.length; i++) {
            if (priceFeeds[i].price.publishTime > updateTimestamp) {
                updateTimestamp = priceFeeds[i].price.publishTime;
            }
        }

        // Calculate the minimum acceptable timestamp (clamped at 0)
        // The maximum acceptable timestamp is enforced by the parsePriceFeedUpdatesWithSlots call
        uint256 minAllowedTimestamp = (block.timestamp >
            PAST_TIMESTAMP_MAX_VALIDITY_PERIOD)
            ? (block.timestamp - PAST_TIMESTAMP_MAX_VALIDITY_PERIOD)
            : 0;

        // Validate that the update timestamp is not too old
        if (updateTimestamp < minAllowedTimestamp) {
            revert SchedulerErrors.TimestampTooOld(
                updateTimestamp,
                block.timestamp
            );
        }

        // Reject updates if they're older than the latest stored ones
        if (
            status.priceLastUpdatedAt > 0 &&
            updateTimestamp <= status.priceLastUpdatedAt
        ) {
            revert SchedulerErrors.TimestampOlderThanLastUpdate(
                updateTimestamp,
                status.priceLastUpdatedAt
            );
        }

        // If updateOnHeartbeat is enabled and the heartbeat interval has passed, trigger update
        if (params.updateCriteria.updateOnHeartbeat) {
            uint256 lastUpdateTime = status.priceLastUpdatedAt;

            if (
                lastUpdateTime == 0 ||
                updateTimestamp >=
                lastUpdateTime + params.updateCriteria.heartbeatSeconds
            ) {
                return updateTimestamp;
            }
        }

        // If updateOnDeviation is enabled, check if any price has deviated enough
        if (params.updateCriteria.updateOnDeviation) {
            for (uint8 i = 0; i < priceFeeds.length; i++) {
                // Get the previous price feed for this price ID using subscriptionId
                PythStructs.PriceFeed storage previousFeed = _state
                    .priceUpdates[subscriptionId][priceFeeds[i].id];

                // If there's no previous price, this is the first update
                if (previousFeed.id == bytes32(0)) {
                    return updateTimestamp;
                }

                // Calculate the deviation percentage
                int64 currentPrice = priceFeeds[i].price.price;
                int64 previousPrice = previousFeed.price.price;

                // Skip if either price is zero to avoid division by zero
                if (previousPrice == 0 || currentPrice == 0) {
                    continue;
                }

                // Calculate absolute deviation basis points (scaled by 1e4)
                uint256 numerator = SignedMath.abs(
                    currentPrice - previousPrice
                );
                uint256 denominator = SignedMath.abs(previousPrice);
                uint256 deviationBps = Math.mulDiv(
                    numerator,
                    10_000,
                    denominator
                );

                // If deviation exceeds threshold, trigger update
                if (
                    deviationBps >= params.updateCriteria.deviationThresholdBps
                ) {
                    return updateTimestamp;
                }
            }
        }

        revert SchedulerErrors.UpdateConditionsNotMet();
    }
