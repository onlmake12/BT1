### Title
Unbounded O(peer_store_size) Fisher-Yates Shuffle Per GetNodes Connection — (`network/src/peer_store/addr_manager.rs`)

### Summary

Every inbound `GetNodes` message triggers a full or near-full Fisher-Yates shuffle over all `random_ids` in `AddrManager` (up to `ADDR_COUNT_LIMIT = 16384` entries). Because the `received_get_nodes` guard is per-session and resets on reconnect, an unprivileged attacker can open many short-lived TCP sessions — each sending one `GetNodes` — to drive O(16384) work per connection while holding the peer-store mutex, causing CPU exhaustion and peer-store lock contention proportional to `peer_store_size × connection_rate`.

---

### Finding Description

**Entry point — `received` in `mod.rs`:**

On the first `GetNodes` per session, the handler calls `get_random(2500, required_flags)` unconditionally: [1](#0-0) 

The `received_get_nodes` flag only blocks a *second* `GetNodes` on the *same* session. A fresh TCP connection resets the flag, so every new connection gets one free shuffle invocation.

**`get_random` → `fetch_random_addrs` → `fetch_random`:**

`DiscoveryAddressManager::get_random` delegates to `peer_store.fetch_random_addrs(n, flags)`: [2](#0-1) 

`fetch_random_addrs` applies a filter requiring `peer_addr.connected(|t| t > addr_expired_ms)` (connected within 7 days): [3](#0-2) 

**The O(N) loop in `AddrManager::fetch_random`:**

The loop iterates from `i = 0` to `random_ids.len()`, performing a Fisher-Yates swap on every step. It only breaks early when `count` qualifying entries are found: [4](#0-3) 

Each iteration performs:
- `rng.gen_range(i..len)` — random number generation
- `swap_random_id(j, i)` — two `HashMap` lookups + two `HashMap` mutations
- `id_to_info[&random_ids[i]].to_owned()` — one `HashMap` lookup + `AddrInfo` clone
- Filter evaluation

If the filter rejects most entries (see below), the loop runs over **all** `random_ids.len()` entries — up to `ADDR_COUNT_LIMIT = 16384`: [5](#0-4) 

**Worst-case filter rejection — never-connected addresses:**

Addresses added via `add_addr` (from received `Nodes` messages) are stored with `last_connected_at_ms = 0`: [6](#0-5) 

These addresses fail the `connected(|t| t > addr_expired_ms)` predicate in `fetch_random_addrs`, so the early-exit `break` is never reached and the loop runs over all 16384 entries.

**Mutex amplification:**

`get_random` acquires the peer-store write mutex via `with_peer_store_mut` for the entire duration of the shuffle. Every concurrent operation needing the peer store (outbound dialing, feeler connections, ban checks) is blocked for the full O(N) duration of each shuffle.

---

### Impact Explanation

- **CPU**: Each `GetNodes` connection costs O(16384) iterations of HashMap-heavy work. At ~200 ns/iteration, one call ≈ 3.3 ms. At 300 connections/second, this saturates a full CPU core.
- **Lock contention**: The peer-store mutex is held for the entire shuffle, serializing all peer-store operations across the node.
- **Cascading**: Stalled peer-store access blocks outbound connection attempts, feeler logic, and ban enforcement, degrading overall node connectivity.

---

### Likelihood Explanation

The attack requires only:
1. Sending `Nodes` messages (valid discovery protocol) to fill the peer store with 16384 never-connected addresses — straightforward since `add_new_addrs` accepts any valid multiaddr.
2. Opening TCP connections at high rate, each sending one `GetNodes` before disconnecting — no authentication, no PoW, no privileged role required.

There is no per-IP connection rate limit visible in the discovery protocol layer.

---

### Recommendation

1. **Early-exit the shuffle when the filter is known to be sparse**: Pre-filter `random_ids` to only those passing the `is_connectable` check before entering the shuffle loop, or maintain a separate index of connectable addresses.
2. **Cap shuffle work independently of store size**: Limit the number of iterations to `min(count * k, random_ids.len())` for some small constant `k` (e.g., 4×), accepting a slightly less uniform sample.
3. **Per-IP / per-session GetNodes rate limiting**: Track the source IP and reject `GetNodes` from IPs that have sent it more than once within a time window, independent of session ID.
4. **Release the mutex before post-processing**: Collect only IDs under the lock, then resolve `AddrInfo` outside it to reduce lock hold time.

### Proof of Concept

```
1. Connect to target CKB node; send Nodes messages advertising 16384 distinct
   never-before-seen IP:port addresses (each with last_connected_at_ms = 0).
   Verify peer store reaches ADDR_COUNT_LIMIT via RPC get_peers / logs.

2. Open 300 TCP connections/second to the discovery port.
   On each connection, immediately send a valid GetNodes message, then close.

3. Observe: CPU usage on the CKB process climbs to 100% on one core;
   peer-store mutex contention causes outbound dial attempts to stall;
   node connectivity degrades.

4. Assert: CPU time per connection scales linearly with peer_store_size,
   confirming O(N) work per GetNodes with N = ADDR_COUNT_LIMIT = 16384.
```

    fn get_random(&mut self, n: usize, flags: Flags) -> Vec<(Multiaddr, Flags)> {
        let fetch_random_addrs = self
            .network_state
            .with_peer_store_mut(|peer_store| peer_store.fetch_random_addrs(n, flags));
        let addrs = fetch_random_addrs
            .into_iter()
            .filter_map(|paddr| {
                if !self.is_valid_addr(&paddr.addr) {
                    return None;
                }
                let f = Flags::from_bits_truncate(paddr.flags);
                Some((paddr.addr, f))
            })
            .collect();
        trace!("Discovered random addrs: {:?}", addrs);
        addrs
    }

    pub fn fetch_random_addrs(&mut self, count: usize, required_flags: Flags) -> Vec<AddrInfo> {
        // Get info:
        // 1. Connected within 7 days

        let now_ms = ckb_systemtime::unix_time_as_millis();
        let addr_expired_ms = now_ms.saturating_sub(ADDR_TIMEOUT_MS);

        let filter = |peer_addr: &AddrInfo| {
            required_flags_filter(required_flags, Flags::from_bits_truncate(peer_addr.flags))
                && peer_addr.connected(|t| t > addr_expired_ms)
        };

        // get success connected addrs.
        self.addr_manager.fetch_random(count, filter)
    }
