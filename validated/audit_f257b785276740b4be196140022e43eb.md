### Title
`this.getPricesUnsafe()` / `this.getEmaPricesUnsafe()` Internal Calls Break Whitelist Access Control, Permanently DoS-ing Price Read Functions — (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` in `Scheduler.sol` delegate to `this.getPricesUnsafe()` and `this.getEmaPricesUnsafe()` respectively using an external `this.` call. Because Solidity external calls via `this` change `msg.sender` to `address(this)`, the `onlyWhitelistedReader` access-control modifier on the inner functions sees the Scheduler contract itself as the caller — not the original user. For any subscription with `whitelistEnabled = true`, the inner modifier always reverts with `Unauthorized`, making both `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` permanently unusable.

---

### Finding Description

`getPricesNoOlderThan` performs a staleness check and then delegates to `getPricesUnsafe` to retrieve prices:

```solidity
// Scheduler.sol line 554
prices = this.getPricesUnsafe(subscriptionId, priceIds);
```

`getPricesUnsafe` is declared `external` and carries the `onlyWhitelistedReader(subscriptionId)` modifier:

```solidity
// Scheduler.sol lines 514–521
function getPricesUnsafe(
    uint256 subscriptionId,
    bytes32[] calldata priceIds
)
    external
    view
    override
    onlyWhitelistedReader(subscriptionId)
    ...
```

The `onlyWhitelistedReader` modifier resolves access by checking `msg.sender`:

```solidity
// Scheduler.sol lines 750–778
modifier onlyWhitelistedReader(uint256 subscriptionId) {
    if (_state.subscriptionManager[subscriptionId] == msg.sender) { _; return; }
    if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) { _; return; }
    // check whitelist array for msg.sender ...
    if (!isWhitelisted) revert SchedulerErrors.Unauthorized();
    _;
}
```

When `getPricesNoOlderThan` calls `this.getPricesUnsafe(...)`, Solidity issues an external call to the contract's own address. Inside that call, `msg.sender` is `address(this)` — the Scheduler proxy — not the original EOA or contract that called `getPricesNoOlderThan`. The Scheduler contract is never registered as a subscription manager and is never placed in a reader whitelist, so for any subscription where `whitelistEnabled = true`, the modifier unconditionally reverts.

The identical pattern exists for `getEmaPricesNoOlderThan` → `this.getEmaPricesUnsafe()`:

```solidity
// Scheduler.sol line 597
prices = this.getEmaPricesUnsafe(subscriptionId, priceIds);

### Impact Explanation

Any subscription created with `whitelistEnabled = true` renders `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` permanently broken. Even the subscription manager — who passes the outer `onlyWhitelistedReader` check on `getPricesNoOlderThan` itself — cannot retrieve prices through these functions because the inner `this.` call substitutes `address(this)` for `msg.sender`. The only working read path is `getPricesUnsafe` / `getEmaPricesUnsafe` called directly, which omits the staleness guarantee. Consumers relying on the staleness-bounded API receive no data at all.

---

### Likelihood Explanation

The whitelist feature is an explicitly supported configuration option (`whitelistEnabled` flag in `SubscriptionParams`). Any subscription manager who enables it — a normal, unprivileged action — immediately triggers the bug for all callers of the two affected functions. No special attacker capability is required; the breakage is automatic and deterministic upon subscription creation. [5](#0-4) 

---

### Recommendation

Replace the `this.getPricesUnsafe(...)` and `this.getEmaPricesUnsafe(...)` external calls with direct calls to the shared internal helper `_getPricesInternal`, which already contains the retrieval logic without any access-control modifier:

```solidity
// getPricesNoOlderThan — replace line 554:
// Before: prices = this.getPricesUnsafe(subscriptionId, priceIds);
PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
prices = new PythStructs.Price[](priceFeeds.length);
for (uint i = 0; i < priceFeeds.length; i++) {
    prices[i] = priceFeeds[i].price;
}

// getEmaPricesNoOlderThan — replace line 597:
// Before: prices = this.getEmaPricesUnsafe(subscriptionId, priceIds);
PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
prices = new PythStructs.Price[](priceFeeds.length);
for (uint i = 0; i < priceFeeds.length; i++) {
    prices[i] = priceFeeds[i].emaPrice;
}
```

Access control is already enforced by the `onlyWhitelistedReader` modifier on the outer functions (`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` themselves), so no additional guard is needed. [6](#0-5)

### Proof of Concept

1. Deploy `SchedulerUpgradeable` (or use a live instance).
2. Call `createSubscription` with `whitelistEnabled = true` and a non-empty `readerWhitelist` containing address `A`. Fund it above the minimum balance.
3. Wait for a keeper to call `updatePriceFeeds`, populating stored prices.
4. From address `A` (a whitelisted reader), call `getPricesNoOlderThan(subscriptionId, priceIds, 3600)`.
5. **Observed**: the call reverts with `SchedulerErrors.Unauthorized`.
6. **Expected**: the call succeeds and returns prices, because `A` is whitelisted.

**Trace**:
- `getPricesNoOlderThan` outer `onlyWhitelistedReader` → passes (`A` is whitelisted).
- Staleness check → passes.
- `this.getPricesUnsafe(subscriptionId, priceIds)` → external call; `msg.sender` inside = `address(Scheduler)`.
- Inner `onlyWhitelistedReader`: `address(Scheduler) != manager`, `whitelistEnabled == true`, `address(Scheduler)` not in whitelist → **revert `Unauthorized`**. [7](#0-6) [8](#0-7)

        uint256 subscriptionId,
        bytes32[] calldata priceIds
    ) internal view returns (PythStructs.PriceFeed[] memory priceFeeds) {

            revert SchedulerErrors.InactiveSubscription();
        }

        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        // If no price IDs provided, return all price feeds for the subscription
        if (priceIds.length == 0) {
            PythStructs.PriceFeed[]
                memory allFeeds = new PythStructs.PriceFeed[](
                    params.priceIds.length
                );
            for (uint8 i = 0; i < params.priceIds.length; i++) {
                PythStructs.PriceFeed memory priceFeed = _state.priceUpdates[
                    subscriptionId
                ][params.priceIds[i]];
                // Check if the price feed exists (price ID is valid and has been updated)
                if (priceFeed.id == bytes32(0)) {
                    revert SchedulerErrors.InvalidPriceId(
                        params.priceIds[i],
                        bytes32(0)
                    );
                }
                allFeeds[i] = priceFeed;
            }
            return allFeeds;
        }

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
