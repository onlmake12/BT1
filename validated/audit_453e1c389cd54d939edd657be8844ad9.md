### Title
`PythAggregatorV3.latestAnswer()` / `latestRoundData()` / `getRoundData()` Return Stale Prices Without Any Staleness Check - (File: `target_chains/ethereum/sdk/solidity/PythAggregatorV3.sol`)

---

### Summary

`PythAggregatorV3`, the official Pyth Solidity SDK adapter that implements the Chainlink `AggregatorV3Interface`, calls `getPriceUnsafe()` in every price-returning function without performing any staleness check. This is the direct Pyth analog of the USSD M-1 finding: a Chainlink-compatible oracle adapter that returns prices from arbitrarily far in the past with no revert guard.

---

### Finding Description

`PythAggregatorV3.sol` is the official Pyth SDK contract recommended for migrating Chainlink-dependent protocols to Pyth. It implements `latestAnswer()`, `latestRoundData()`, and `getRoundData()` — the three functions that Chainlink-compatible DeFi protocols call to consume prices.

All three functions call `pyth.getPriceUnsafe(priceId)` and return the result directly:

```solidity
// line 53-56

    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    return int256(price.price);
}

// line 99-119

    ...
    return (roundId, int256(price.price), price.publishTime, price.publishTime, roundId);
}

// line 76-97
function getRoundData(uint80 _roundId) external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    ...

```

`getPriceUnsafe()` is explicitly documented in `IPyth.sol` as returning "the most recent price update in this contract without any recency checks" and that "the returned price update may be arbitrarily far in the past." The safe alternative, `getPriceNoOlderThan(id, age)`, is available in `AbstractPyth.sol` and enforces a staleness revert — but `PythAggregatorV3` never calls it. ### Impact Explanation

Any DeFi protocol (lending, perpetuals, stablecoin) that deploys `PythAggregatorV3` as a drop-in Chainlink replacement and calls `latestAnswer()` or `latestRoundData()` will silently receive a price from an arbitrarily old update. During network congestion, market closure, or a Pyth outage, the on-chain price can lag by minutes or hours. An attacker who observes the divergence between the stale on-chain price and the true market price can:

- Borrow against overvalued collateral (stale high price)
- Liquidate positions that are not actually undercollateralized (stale low price)
- Execute arbitrage trades at favorable stale prices

The impact is identical to the USSD M-1 finding: incorrect calculations in key financial functions due to unchecked stale oracle data. [5](#0-4) 

---

### Likelihood Explanation

`PythAggregatorV3` is the officially recommended migration path for Chainlink users, documented in the Pyth developer hub. Protocols following the official migration guide will deploy this contract and call `latestRoundData()` without any additional staleness guard, because the Chainlink interface convention is that the oracle itself enforces recency. The likelihood of exploitation is high whenever Pyth price updates are delayed (network congestion, market hours for non-crypto assets, or Pythnet outage). [6](#0-5) 

---

### Recommendation

Replace all `getPriceUnsafe(priceId)` calls in `PythAggregatorV3` with `getPriceNoOlderThan(priceId, maxAge)`, where `maxAge` is a configurable constructor parameter (e.g., defaulting to `getValidTimePeriod()`). If the price is stale, the call should revert, consistent with how Chainlink's own `latestRoundData()` behaves when the aggregator is stale.

```solidity
uint public maxAge;

constructor(address _pyth, bytes32 _priceId, uint _maxAge) {
    priceId = _priceId;
    pyth = IPyth(_pyth);
    maxAge = _maxAge;
}

function latestAnswer() public view virtual returns (int256) {
    PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
    return int256(price.price);
}
``` [4](#0-3)

### Proof of Concept

1. Deploy `PythAggregatorV3` pointing to the live Pyth EVM contract and a price feed ID (e.g., BTC/USD).
2. Do not call `updateFeeds()` for 2+ hours (simulating a Pyth outage or network congestion).
3. Call `latestAnswer()` — it returns the 2-hour-old price with no revert.
4. Call `latestRoundData()` — the returned `updatedAt` field equals `price.publishTime`, which is 2 hours in the past, but the function does not revert.
5. A lending protocol consuming this adapter would accept the stale price as current, enabling the attacker to borrow against mispriced collateral. [7](#0-6) [8](#0-7)

        bytes32 id,
        uint age
    ) public view virtual override returns (PythStructs.Price memory price) {
        price = getPriceUnsafe(id);

        if (diff(block.timestamp, price.publishTime) > age)
            revert PythErrors.StalePrice();

        return price;
    }
