Audit Report

## Title
Ping-RTT Eviction Protection Bypassed via `as_secs()` Truncation — (`network/src/peer_registry.rs`)

## Summary
`try_evict_inbound_peer` uses `Duration::as_secs()` to rank peers by ping RTT and by recency of last ping message, discarding all sub-second precision and collapsing the entire `[0 ms, 999 ms]` range to `0`. An attacker who controls 8 inbound connections and replies to pings immediately (guaranteed sub-second RTT) can monopolize all `EVICTION_PROTECT_PEERS = 8` low-ping protection slots whenever honest peers have RTTs ≥ 1 s, leaving honest peers as the sole eviction candidates and enabling a gradual eclipse attack.

## Finding Description
**Root cause — Pass 1 (L151–165, `network/src/peer_registry.rs`):**
```rust
sort_then_drop(
    &mut candidate_peers,
    EVICTION_PROTECT_PEERS,
    |peer1, peer2| {
        let peer1_ping = peer1.ping_rtt.map(|p| p.as_secs()).unwrap_or_else(|| u64::MAX);
        let peer2_ping = peer2.ping_rtt.map(|p| p.as_secs()).unwrap_or_else(|| u64::MAX);
        peer2_ping.cmp(&peer1_ping)   // descending
    },
);
```
`sort_then_drop` sorts descending then calls `list.truncate(list.len() - n)`, which removes the last `n` entries (lowest-ping peers) from `candidate_peers`, protecting them from eviction. Because `as_secs()` truncates, any peer with RTT in `[0 ms, 999 ms]` produces `0`. A peer with RTT = 1 000 ms produces `1`.

**Attack flow:**
1. Attacker opens 8 inbound TCP connections to the victim node.
2. Attacker replies to every ping frame immediately; `pong_received` records `ping_rtt = now.saturating_duration_since(last_ping)` ≈ 1–5 ms → `as_secs()` = 0.
3. Honest cross-continental or congested peers have RTT ≥ 1 000 ms → `as_secs()` = 1.
4. After descending sort: honest peers (value 1) appear first; attacker peers (value 0) appear last.
5. `truncate(list.len() - 8)` removes the last 8 (attacker peers) from `candidate_peers` — they are **protected**.
6. All honest peers remain in `candidate_peers` as eviction candidates.

**Root cause — Pass 2 (L168–183):**
```rust
.map(|t| now.saturating_duration_since(t).as_secs())
```
An attacker who sends frequent pings also has `last_ping_protocol_message_received_at` within the last second → `as_secs()` = 0, again filling the 8 "most recent message" protection slots.

**Pass 3 (L185–188)** uses `connected_time` directly without `as_secs()` and is not affected. However, after passes 1 and 2 are saturated by attacker peers, only honest peers remain as candidates; pass 3 protects half of them, leaving the other half exposed.

**`ping_rtt` is fully attacker-controlled:** `pong_received` (`ping.rs` L71–79) sets `ping_rtt = Some(now.saturating_duration_since(last_ping))`. The attacker simply replies to the ping frame immediately over the open TCP session — no special capability required.

## Impact Explanation
By repeatedly triggering `try_evict_inbound_peer` (each time a new inbound connection arrives), the attacker can evict honest inbound peers one by one and replace them with attacker-controlled peers. Over enough cycles this achieves an eclipse of the victim node's inbound peer set. An eclipsed node receives only attacker-curated blocks and transactions, enabling network partition and consensus deviation. This matches the allowed CKB bounty impact: **"Vulnerabilities which could easily cause consensus deviation"** (Critical, 15 001–25 000 points). The multi-cycle nature of the attack and the RTT ≥ 1 s precondition reduce practical severity toward the lower end of that band.

## Likelihood Explanation
- **Attacker cost:** 8 TCP connections from a single host; no PoW, no keys, no privileged access.
- **RTT precondition:** Honest peers must have RTT ≥ 1 000 ms. Typical cross-continental RTTs are 100–300 ms, so this requires congested or degraded network paths — a realistic but not universal condition in a global P2P network.
- **Repeatability:** The attack is deterministic and repeatable; each new inbound connection triggers another eviction cycle.
- **Detectability:** The attacker's behavior (fast pong replies) is indistinguishable from a well-connected local peer.

## Recommendation
Replace `as_secs()` with `as_millis()` in both comparators to preserve sub-second precision:

```rust
// Pass 1 — ping RTT
let peer1_ping = peer1.ping_rtt.map(|p| p.as_millis()).unwrap_or(u128::MAX);
let peer2_ping = peer2.ping_rtt.map(|p| p.as_millis()).unwrap_or(u128::MAX);

// Pass 2 — last message recency
let peer1_last = peer1.last_ping_protocol_message_received_at
    .map(|t| now.saturating_duration_since(t).as_millis())
    .unwrap_or(u128::MAX);
```

This ensures that a peer with RTT = 1 ms (value 1) is correctly distinguished from a peer with RTT = 999 ms (value 999), preventing trivial saturation of the protection slots.

## Proof of Concept
```rust
// Setup: 16 inbound peers total
//   8 attacker peers: ping_rtt = Duration::from_millis(5)    → as_secs() = 0
//   8 honest peers:   ping_rtt = Duration::from_millis(1200) → as_secs() = 1
//
// Call try_evict_inbound_peer():
//   Pass 1 sort (descending by as_secs):
//     [honest×8 (1), attacker×8 (0)]
//   truncate(16 - 8 = 8): keeps first 8 (honest), drops last 8 (attacker)
//   → attacker peers PROTECTED, honest peers remain as candidates
//
//   Pass 2: same truncation defect; attacker peers already gone from list,
//           8 honest peers sorted by last-message as_secs(); all 8 protected
//           if they all messaged within the last second, else some remain.
//
//   Pass 3: half of remaining honest peers protected by connection time.
//
//   Result: only honest peers are ever selected for eviction.
//
// Repeat 8 times → all honest inbound peers evicted, replaced by attacker peers.
// Node is eclipsed.
```
