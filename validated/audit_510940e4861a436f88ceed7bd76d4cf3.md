### Title
Silent No-Op for `UpgradeContract` Governance Action Silently Consumes Governance Sequence Without Upgrading Contract - (File: `target_chains/stylus/contracts/pyth-receiver/src/governance.rs`)

---

### Summary

In the Stylus Pyth receiver contract, the `execute_governance_instruction_internal` function dispatches governance actions via a `match` statement. The `GovernancePayload::UpgradeContract` arm is present but contains an empty body `{}`, making it a silent no-op. A valid governance VAA carrying an `UpgradeContract` action will pass all validation, permanently advance the governance sequence number, and return `Ok(())` — without performing any contract upgrade. The governance authority receives no error or event indicating failure.

---

### Finding Description

In `target_chains/stylus/contracts/pyth-receiver/src/governance.rs`, the `execute_governance_instruction_internal` function handles all governance actions: [1](#0-0) 

The `UpgradeContract` arm is:

```rust
GovernancePayload::UpgradeContract(_payload) => {}
```

This is a complete silent no-op. Every other action either performs its operation or returns an explicit error (e.g., `RequestGovernanceDataSourceTransfer` returns `Err(PythReceiverError::InvalidGovernanceMessage)`). Only `UpgradeContract` silently succeeds without doing anything. [2](#0-1) 

An `upgrade_contract` helper function exists but is never called from the handler, and would panic if it were: [3](#0-2) 

The governance sequence is consumed unconditionally before the dispatch, inside `verify_governance_vm`: [4](#0-3) 

The `GovernanceAction` enum in the Stylus structs file maps action byte `0` to `UpgradeContract` and parses its 32-byte payload correctly, so the instruction is fully accepted before the silent no-op: [5](#0-4) [6](#0-5) 

Compare with the EVM implementation, which correctly executes the upgrade: [7](#0-6) 

---

### Impact Explanation

1. A governance `UpgradeContract` VAA submitted to the Stylus Pyth receiver is fully validated and accepted.
2. The governance sequence number is permanently advanced — the same VAA cannot be replayed.
3. The contract is **not upgraded**. No event is emitted. No error is returned.
4. The governance authority has no on-chain signal that the upgrade failed; the transaction succeeds with `Ok(())`.
5. The governance authority must craft a new VAA with a higher sequence number to retry, but if they believe the upgrade succeeded (e.g., based on transaction success), the contract may remain on a stale implementation indefinitely.
6. Critical security patches or feature upgrades to the Stylus contract are silently blocked.

---

### Likelihood Explanation

The Stylus contract is a production deployment. Governance upgrade VAAs are issued by the Pyth governance authority (a governance message submitter — a listed valid actor in scope). The action byte `0` for `UpgradeContract` is a standard, well-defined governance action used across all Pyth chain integrations. Any future upgrade attempt to the Stylus contract will trigger this silent failure. The likelihood increases as the Stylus deployment matures and requires upgrades.

---

### Recommendation

Replace the silent no-op with an explicit error, consistent with how unimplemented actions are handled elsewhere (e.g., the Fuel contract reverts with `PythError::InvalidGovernanceMessage` for unimplemented actions):

```rust
GovernancePayload::UpgradeContract(_payload) => {
    return Err(PythReceiverError::InvalidGovernanceMessage);
}
```

Alternatively, implement the upgrade logic when the Stylus upgrade mechanism is available. Until then, the action must revert so the governance sequence is not consumed and the authority is notified of the failure.

---

### Proof of Concept

1. Construct a governance VAA with:
   - Valid governance emitter chain/address matching the Stylus contract's configured governance data source
   - Sequence number greater than `last_executed_governance_sequence`
   - Payload: `PTGM` magic + module `0x01` (Target) + action `0x00` (UpgradeContract) + target chain ID + 32-byte new implementation hash
2. Call `execute_governance_instruction(vaa_bytes)` on the deployed Stylus Pyth receiver.
3. Observe: transaction succeeds (`Ok(())`), `last_executed_governance_sequence` is incremented, no `ContractUpgraded` event is emitted, and the contract class hash is unchanged.
4. Attempt to replay the same VAA: reverts with `GovernanceMessageAlreadyExecuted` — the sequence is permanently consumed. [8](#0-7)

    pub fn execute_governance_instruction_internal(
        &mut self,
        data: Vec<u8>,
    ) -> Result<(), PythReceiverError> {
        let wormhole: IWormholeContract = IWormholeContract::new(self.wormhole.get());
        let config = Call::new();

        wormhole
            .parse_and_verify_vm(config, Vec::from(data.clone()))
            .map_err(|_| PythReceiverError::InvalidWormholeMessage)?;

        let vm = Vaa::read(&mut Vec::from(data.clone()).as_slice())
            .map_err(|_| PythReceiverError::InvalidVaa)?;

        verify_governance_vm(self, vm.clone())?;

        let instruction = governance_structs::parse_instruction(vm.body.payload.to_vec())
            .map_err(|_| PythReceiverError::InvalidGovernanceMessage)?;

        let chain_id_config = Call::new();

        let wormhole_id = wormhole
            .chain_id(chain_id_config)
            .map_err(|_| PythReceiverError::WormholeUninitialized)?;

        if instruction.target_chain_id != 0 && instruction.target_chain_id != wormhole_id {
            return Err(PythReceiverError::InvalidGovernanceTarget);
        }

        match instruction.payload {
            GovernancePayload::SetFee(payload) => {
                self.set_fee(payload.value, payload.expo);
            }
            GovernancePayload::SetDataSources(payload) => {
                set_data_sources(self, payload.sources);
            }
            GovernancePayload::SetWormholeAddress(payload) => {
                self.set_wormhole_address(payload.address, data.clone())?;
            }
            GovernancePayload::RequestGovernanceDataSourceTransfer(_) => {
                return Err(PythReceiverError::InvalidGovernanceMessage);
            }
            GovernancePayload::AuthorizeGovernanceDataSourceTransfer(payload) => {
                self.authorize_governance_transfer(payload.claim_vaa)?;
            }
            GovernancePayload::UpgradeContract(_payload) => {}
            GovernancePayload::SetValidPeriod(payload) => {
                self.set_valid_period(payload.valid_time_period_seconds);
            }
            GovernancePayload::SetTransactionFee(payload) => {
                self.set_transaction_fee(payload.value, payload.expo);
            }
            GovernancePayload::WithdrawFee(payload) => {
                self.withdraw_fee(payload.value, payload.expo, payload.target_address)?;
            }
        }

        Ok(())
    }
