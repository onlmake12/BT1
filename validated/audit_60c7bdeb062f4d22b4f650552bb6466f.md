### Title
PDA Pre-Funding DOS on Publisher Initialization via `system_instruction::create_account` — (File: `target_chains/solana/programs/pyth-price-store/src/processor/initialize_publisher.rs`)

---

### Summary

The `initialize_publisher` instruction in the `pyth-price-store` program uses `system_instruction::create_account` to create the `publisher_config` PDA. Because this instruction fails if the target account already holds any lamports, an attacker can permanently block any publisher from being initialized by pre-funding their deterministic PDA address with a trivial amount of SOL.

---

### Finding Description

The `initialize_publisher` handler calls `system_instruction::create_account` to allocate the `publisher_config` PDA:

```rust
// initialize_publisher.rs
invoke_signed(
    &system_instruction::create_account(
        authority.key,
        publisher_config.key,   // <-- deterministic PDA
        lamports,
        publisher_config::SIZE.try_into()...,
        program_id,
    ),
    ...
)?;
```

The `publisher_config` PDA is derived deterministically from `PUBLISHER_CONFIG_SEED + publisher_pubkey`:

```rust
// validate.rs
let (publisher_config_pda, expected_bump) = Pubkey::find_program_address(
    &[PUBLISHER_CONFIG_SEED.as_bytes(), &publisher.to_bytes()],
    program_id,
);
```

Because the publisher pubkey is public, anyone can compute this PDA address before the authority calls `initialize_publisher`. Sending even 1 lamport to that address causes `system_instruction::create_account` to fail with `SystemError::AccountAlreadyInUse`, permanently blocking initialization for that publisher.

The same pattern exists in `initialize.rs` for the global config PDA (derived from the fixed seed `"CONFIG"`), which can be pre-funded to block the one-time program initialization.

Notably, the `message_buffer` program in the same repository explicitly handles this case with a custom `create_account` helper that checks `if new_account_info.lamports() < target_rent` and only transfers the difference — confirming the team is aware of the pattern but did not apply it to `pyth-price-store`. ---

### Impact Explanation

The `pyth-price-store` program is the on-chain component on Pythnet that receives price submissions from publishers. Blocking `initialize_publisher` for a given publisher prevents that publisher from ever submitting prices to the program. An attacker targeting all known publishers (whose pubkeys are public) can pre-fund every publisher's PDA, permanently preventing any publisher from being onboarded. This disrupts the price aggregation pipeline at the source chain level.

**Impact: High** — permanent, permissionless DOS of publisher onboarding; no recovery path without a program upgrade. [4](#0-3) 

---

### Likelihood Explanation

**Likelihood: Medium**

- Publisher pubkeys are publicly observable on-chain and in governance tooling.
- The attack costs only a few lamports per publisher PDA.
- The attacker does not need to front-run a specific transaction; they can pre-fund PDAs proactively at any time before the authority calls `initialize_publisher`.
- The same attack applies to the global config PDA in `initialize.rs`, but that is a one-time setup that is likely already complete on mainnet. [5](#0-4) 

---

### Recommendation

Replace `system_instruction::create_account` with the pattern already used in `message_buffer/create_buffer.rs`: check the existing lamport balance and only transfer the deficit, then call `allocate` and `assign` separately:

```rust
let current_lamports = publisher_config.lamports();
if current_lamports == 0 {
    invoke_signed(&system_instruction::create_account(...), ...)?;
} else {
    // Transfer only the shortfall
    if current_lamports < lamports {
        invoke_signed(&system_instruction::transfer(..., lamports - current_lamports), ...)?;
    }
    invoke_signed(&system_instruction::allocate(...), ...)?;
    invoke_signed(&system_instruction::assign(...), ...)?;
}
```

Apply the same fix to `initialize.rs` for the config PDA. [6](#0-5) 

---

### Proof of Concept

1. Observe the `pyth-price-store` program ID on Pythnet (public).
2. For a target publisher pubkey `P`, compute:
   ```
   publisher_config_pda = find_program_address(["PUBLISHER_CONFIG", P], program_id)
   ```
3. Send 1 lamport to `publisher_config_pda` from any funded wallet.
4. When the Pyth authority calls `initialize_publisher` for publisher `P`, the transaction fails with `AccountAlreadyInUse`.
5. Repeat for all known publisher pubkeys to block all publisher onboarding.

The same attack applies to the config PDA by sending lamports to `find_program_address(["CONFIG"], program_id)` before the program is initialized. [7](#0-6) [1](#0-0)

/// Creates a publisher config account and stores the buffer account pubkey in it.
/// Verifies and initializes the buffer account.
/// See `Instruction` for the list of required accounts.
/// The config account must be an initialized PDA account with an expected seed.
/// The authority account that signed the instruction
/// must match the authority key stored in the config account.
/// The publisher config account must be a non-existing PDA account with an expected seed.
/// The buffer config must be an existing, zero-filled account owned by the program.
/// Note: we aren't using a PDA for the buffer because the program can't create
/// a PDA larger than 10240 bytes in a single transaction.
/// Note: currently, the publisher config can only be set once and can only contain
/// a single buffer key. If we need to modify the buffer key or create multiple buffer keys
/// per publisher, we'll need to upgrade the program.
pub fn initialize_publisher(
    program_id: &Pubkey,
    accounts: &[AccountInfo],
    args: &InitializePublisherArgs,
) -> ProgramResult {
    let mut accounts = accounts.iter();
    let first_account = accounts.next();
    let config = validate_config(accounts.next(), args.config_bump, program_id, false)?;
    let authority = validate_authority(first_account, config)?;
    let publisher_config = validate_publisher_config_for_init(
        accounts.next(),
        args.publisher_config_bump,
        &args.publisher.into(),
        program_id,
    )?;
    let buffer = validate_buffer(accounts.next(), program_id)?;
    let system = validate_system(accounts.next())?;

    // Deposit enough tokens to allocate the account.
    let rent = Rent::get()?;
    let lamports = rent.minimum_balance(publisher_config::SIZE);

    invoke_signed(
        &system_instruction::create_account(
            authority.key,
            publisher_config.key,
            lamports,
            publisher_config::SIZE
                .try_into()
                .expect("unexpected overflow"),
            program_id,
        ),
        &[authority.clone(), publisher_config.clone(), system.clone()],
        &[&[
            PUBLISHER_CONFIG_SEED.as_bytes(),
            &args.publisher,
            &[args.publisher_config_bump],
        ]],
    )?;

    let mut publisher_config_data = publisher_config.data.borrow_mut();
    publisher_config::create(
        *publisher_config_data,
        args.publisher,
        buffer.key.to_bytes(),
    )?;

    // Write an initial Header into the buffer account to prepare it to receive prices.
    let mut buffer_data = buffer.data.borrow_mut();
    ensure!(
        ProgramError::AccountNotRentExempt,
        buffer.lamports() >= rent.minimum_balance(buffer_data.len())
    );
    buffer::create(*buffer_data, args.publisher)?;

    Ok(())

    /// Manually invoke transfer, allocate & assign ixs to create an account
    /// to handle situation where an account already has lamports
    /// since system_program::create_account will fail in this case
    fn create_account<'a>(
        new_account_info: &AccountInfo<'a>,
        space: usize,
        payer: &Signer<'a>,
        seeds: &[&[&[u8]]],
        system_program: &AccountInfo<'a>,
    ) -> Result<()> {
        let target_rent = Rent::get()?.minimum_balance(space);
        if new_account_info.lamports() < target_rent {
            system_program::transfer(
                CpiContext::new_with_signer(
                    system_program.to_account_info(),
                    Transfer {
                        from: payer.to_account_info(),
                        to: new_account_info.to_account_info(),
                    },
                    seeds,
                ),
                target_rent - new_account_info.lamports(),
            )?;
        };

        system_program::allocate(
            CpiContext::new_with_signer(
                system_program.to_account_info(),
                Allocate {
                    account_to_allocate: new_account_info.to_account_info(),
                },
                seeds,
            ),
            space.try_into().unwrap(),
        )?;

        system_program::assign(
            CpiContext::new_with_signer(
                system_program.to_account_info(),
                Assign {
                    account_to_assign: new_account_info.to_account_info(),
                },
                seeds,
            ),
            &crate::ID,
        )?;

        Ok(())
