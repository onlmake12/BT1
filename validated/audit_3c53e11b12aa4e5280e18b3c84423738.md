### Title
`getPricesNoOlderThan` Staleness Check Uses Maximum Publish Time, Allowing Stale Individual Feed Prices to Pass — (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

### Summary

The `Scheduler` contract's `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` functions check freshness using `status.priceLastUpdatedAt`, which is set to the **maximum** publish time across all feeds in a subscription batch. For subscriptions containing mixed active-market and closed-market assets, the staleness check passes based on the most recently updated feed, while individual closed-market feeds can be arbitrarily stale. A reader calling `getPricesNoOlderThan` with a tight age window receives a false freshness guarantee.

### Finding Description

In `updatePriceFeeds`, after parsing the batch, `_validateShouldUpdatePrices` computes `updateTimestamp` as the **maximum** `publishTime` across all feeds:

```solidity
uint256 updateTimestamp = 0;
for (uint8 i = 0; i < priceFeeds.length; i++) {
    if (priceFeeds[i].price.publishTime > updateTimestamp) {
        updateTimestamp = priceFeeds[i].price.publishTime;
    }
}
```

This value is then stored as the subscription's last-updated timestamp:

```solidity
status.priceLastUpdatedAt = latestPublishTime;
```

Later, `getPricesNoOlderThan` enforces freshness using only this single max value:

```solidity
if (distance(block.timestamp, status.priceLastUpdatedAt) > age_seconds)
    revert PythErrors.StalePrice();

prices = this.getPricesUnsafe(subscriptionId, priceIds);
```

`getPricesUnsafe` returns the raw stored price for each feed without any per-feed timestamp check. For a subscription containing both BTC/USD (active, `publishTime ≈ now`) and AAPL/USD (closed market, `publishTime` from hours or days ago), the staleness check passes because `distance(block.timestamp, now) ≈ 0 < age_seconds`, but the AAPL/USD price returned is arbitrarily old.

The same flaw exists in `getEmaPricesNoOlderThan`.

### Impact Explanation

A consumer protocol integrating the Scheduler and calling `getPricesNoOlderThan(subscriptionId, priceIds, 60)` to enforce a 60-second freshness window will silently receive stale prices for closed-market assets. The function's name and interface imply a per-feed freshness guarantee that is not upheld. Downstream protocols using these prices for collateral valuation, liquidation thresholds, or settlement could operate on stale data, leading to incorrect financial outcomes (e.g., accepting overvalued collateral, failing to liquidate, or incorrect settlement).

### Likelihood Explanation

Subscriptions mixing active-market assets (crypto) with traditional-market assets (equities, commodities) are a natural and documented use case for the Scheduler. The Scheduler README explicitly notes that "closed markets will have a publishTime from their last trading period." Any reader using `getPricesNoOlderThan` on such a subscription is affected. The entry path is fully permissionless — any whitelisted reader can call `getPricesNoOlderThan`.

### Recommendation

Replace the single-max-timestamp staleness check in `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` with a per-feed check against each feed's individual `publishTime`:

```solidity
function getPricesNoOlderThan(
    uint256 subscriptionId,
    bytes32[] calldata priceIds,
    uint256 age_seconds
) external view override onlyWhitelistedReader(subscriptionId) returns (PythStructs.Price[] memory prices) {
    PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
    prices = new PythStructs.Price[](priceFeeds.length);
    for (uint i = 0; i < priceFeeds.length; i++) {
        if (distance(block.timestamp, priceFeeds[i].price.publishTime) > age_seconds)
            revert PythErrors.StalePrice();
        prices[i] = priceFeeds[i].price;
    }
}
```

Alternatively, document clearly that the age guarantee applies only to the most recently updated feed in the batch, and rename the function to avoid implying a per-feed freshness guarantee.

### Proof of Concept

1. A subscription is created with two feeds: `BTC/USD` and `AAPL/USD`.
2. A keeper calls `updatePriceFeeds` with a batch where `BTC/USD.publishTime = T` (current) and `AAPL/USD.publishTime = T - 8 hours` (market closed).
3. `_validateShouldUpdatePrices` sets `updateTimestamp = T` (the max).
4. `status.priceLastUpdatedAt = T`.
5. A reader calls `getPricesNoOlderThan(subscriptionId, priceIds, 60)` at time `T + 30`.
6. The check `distance(T+30, T) = 30 < 60` passes.
7. `getPricesUnsafe` returns both prices — including `AAPL/USD` with `publishTime = T - 8 hours`.
8. The reader receives a price 8 hours old despite calling the "no older than 60 seconds" function. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
