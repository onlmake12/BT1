Audit Report

## Title
Non-Contextual `EpochVerifier` Accepts Arbitrary Epoch Lengths at Epoch Transitions, Enabling Sync Disruption via Crafted Headers — (File: `verification/src/header_verifier.rs`)

## Summary
The non-contextual `EpochVerifier` used during P2P header sync relies on `is_successor_of()` and `is_well_formed()` to validate epoch continuity. At epoch boundaries, `is_successor_of()` does not validate the epoch length of the new epoch, and `is_well_formed()` accepts any length ≥ 1 when index = 0. A malicious peer can craft headers where each block claims epoch length = 1, pass all non-contextual checks (including PoW), and have those headers inserted into the sync header map. If the attacker's chain accumulates sufficient total_difficulty, the node will set it as `shared_best_header`, request full blocks that all fail contextual verification, and waste bandwidth and CPU during IBD.

## Finding Description

**Root cause — `is_successor_of()` skips epoch length at epoch transitions:**

In `util/types/src/core/extras.rs`, the epoch boundary branch only checks epoch number increment and `index == 0`, leaving `self.length()` unconstrained: [1](#0-0) 

Within an epoch the `else` branch enforces `self.length() == predecessor.length()`, but at the boundary it does not.

**`is_well_formed()` accepts length = 1 when index = 0:** [2](#0-1) 

For the first block of a new epoch (`index = 0`), any `length ≥ 1` satisfies `length > 0 && length > index`.

**The non-contextual `EpochVerifier` in `header_verifier.rs` relies solely on these two checks:** [3](#0-2) 

No epoch length bounds (`MIN_EPOCH_LENGTH = 300`, `MAX_EPOCH_LENGTH = 1800`) are checked here.

**`HeaderVerifier` in the sync path invokes this verifier for every received header:** [4](#0-3) 

**`HeadersProcess::execute()` runs this for each header in a `SendHeaders` message:** [5](#0-4) 

**Accepted headers are inserted into the header map and update `shared_best_header`:** [6](#0-5) 

**The epoch length and compact_target are only validated contextually**, in `contextual_block_verifier.rs`, which computes the expected `EpochExt` from `consensus.next_epoch_ext()` and compares both fields: [7](#0-6) 

This contextual check runs only when a full block is processed by the chain service — not during header-first sync.

**Consensus bounds exist but are never applied non-contextually:** [8](#0-7) 

## Impact Explanation

A malicious peer sends a `SendHeaders` message with up to 2000 headers where each block claims `epoch.length = 1`, `epoch.index = 0`, and `epoch.number` increments by 1 per block. Each header passes `is_well_formed()`, `is_successor_of()`, PoW, number, and timestamp checks. If the attacker's chain accumulates total_difficulty exceeding the honest chain (requiring real PoW work at the claimed `compact_target`), the node sets the attacker's chain as `shared_best_header`, begins requesting full blocks, and every block fails contextual verification with `EpochError::NumberMismatch`, stalling IBD. Additionally, during IBD, unprotected outbound peers that appear to have lower total_difficulty may be disconnected: [9](#0-8) 

This matches the allowed impact: **High — Vulnerabilities or bad designs which could cause CKB network congestion with few costs** (on testnet/devnet where PoW difficulty is low, the attack is cheap; on mainnet, it requires PoW work proportional to the honest chain's difficulty, making it expensive but not impossible for a well-resourced attacker).

## Likelihood Explanation

Any unprivileged P2P peer can send crafted `SendHeaders` messages. The attacker must perform real PoW work at the claimed `compact_target` to make the fake chain competitive in total_difficulty. On mainnet this is expensive (proportional to the honest chain's accumulated difficulty). On testnet or devnet the attack is cheap. No privileged access, leaked keys, or majority hashpower is required — only sufficient hashpower to exceed the target chain's total_difficulty, which is trivially achievable on low-difficulty networks.

## Recommendation

In `EpochVerifier::verify()` in `verification/src/header_verifier.rs`, when the header is the first block of a new epoch (`self.header.epoch().index() == 0`), add a range check on `self.header.epoch().length()` against `consensus.min_epoch_length()` and `consensus.max_epoch_length()`. This requires passing the `Consensus` reference into `EpochVerifier` (it is already available in `HeaderVerifier`). This mirrors the bounds enforced by `bounding_epoch_length()` in `spec/src/consensus.rs` at the cheaper, earlier header-sync stage, preventing malformed epoch-length headers from ever entering the header map. [10](#0-9) 

## Proof of Concept

1. Connect to a CKB node as a P2P peer.
2. Obtain a known block hash (e.g., genesis or any stored block).
3. Construct a chain of headers starting from that block where each header has:
   - `epoch.number` = previous + 1
   - `epoch.index` = 0
   - `epoch.length` = 1
   - `compact_target` = any value (set to match or exceed honest chain difficulty for total_difficulty competition)
   - A valid PoW nonce satisfying the claimed `compact_target`
   - `number` = previous + 1, `timestamp` > parent median time
4. Send these headers via `SendHeaders` (up to `MAX_HEADERS_LEN` = 2000 per message).
5. Observe that `HeadersProcess::execute()` accepts all headers without error (all pass `EpochVerifier::verify()`, `PowVerifier::verify()`, `NumberVerifier::verify()`, and `TimestampVerifier::verify()`).
6. If total_difficulty exceeds the honest chain, observe that `shared_best_header` is updated to the attacker's chain tip.
7. Observe that the node begins requesting full blocks from the attacker, which all fail contextual verification with `EpochError::NumberMismatch` (since `next_epoch_ext()` computes a length of 300–1800, not 1), stalling IBD.

    pub fn verify(&self) -> Result<(), Error> {
        let header = self.block.header();
        let actual_epoch_with_fraction = header.epoch();
        let block_number = header.number();
        let epoch_with_fraction = self.epoch.number_with_fraction(block_number);
        if actual_epoch_with_fraction != epoch_with_fraction {
            return Err(EpochError::NumberMismatch {
                expected: epoch_with_fraction.full_value(),
                actual: actual_epoch_with_fraction.full_value(),
            }
            .into());
        }
        let actual_compact_target = header.compact_target();
        if self.epoch.compact_target() != actual_compact_target {
            return Err(EpochError::TargetMismatch {
                expected: self.epoch.compact_target(),
                actual: actual_compact_target,
            }
            .into());
        }
        Ok(())
    }

    fn bounding_epoch_length(
        &self,
        length: BlockNumber,
        last_epoch_length: BlockNumber,
    ) -> (BlockNumber, bool) {
        let max_length = cmp::min(self.max_epoch_length(), last_epoch_length * TAU);
        let min_length = cmp::max(self.min_epoch_length(), last_epoch_length / TAU);
        if length > max_length {
            (max_length, true)
        } else if length < min_length {
            (min_length, true)
        } else {
            (length, false)
        }
    }
