### Title
Unvalidated `providerToCredit` in `executeCallback` Permanently Locks User Fees — (`target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

`Echo.sol`'s `executeCallback` accepts an attacker-controlled `providerToCredit` address with no validation that it is a registered provider or has a reachable fee manager. After the exclusivity period expires, any unprivileged caller can pass `address(0)` or any unregistered address as `providerToCredit`, causing the user's fee stored in `req.fee` to be credited to an address from which it can never be withdrawn. This violates the zero-balance invariant: user funds paid into the contract are permanently locked.

---

### Finding Description

When a user calls `requestPriceUpdatesWithCallback`, their fee is split and stored: [1](#0-0) 

```solidity
uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
if (msg.value < requiredFee) revert InsufficientFee();
...
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);
...
_state.accruedFeesInWei += _state.pythFeeInWei;
```

The provider's portion (`req.fee`) is held in the contract pending fulfillment. When `executeCallback` is called, the only access-control check on `providerToCredit` is the exclusivity-period guard: [2](#0-1) 

```solidity
if (block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds) {
    require(
        providerToCredit == req.provider,
        "Only assigned provider during exclusivity period"
    );
}
```

Once the exclusivity period has elapsed, **any caller may pass any address** as `providerToCredit`. The fee is then unconditionally credited: [3](#0-2) 

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
```

There is no check that `providerToCredit` is a registered provider or has a non-zero `feeManager`. The only withdrawal path for provider fees is `withdrawAsFeeManager`: [4](#0-3) 

```solidity
function withdrawAsFeeManager(address provider, uint128 amount) external override {
    require(msg.sender == _state.providers[provider].feeManager, "Only fee manager");
    ...
    (bool sent, ) = msg.sender.call{value: amount}("");
```

For any address that has never called `setFeeManager`, `_state.providers[addr].feeManager == address(0)`. Since `msg.sender` can never equal `address(0)`, fees credited to such an address are **permanently unrecoverable**.

The `ProviderInfo` struct confirms `feeManager` defaults to `address(0)` for unregistered addresses: [5](#0-4) 

---

### Impact Explanation

User funds paid as provider fees into the Echo contract are permanently locked. The contract's ETH balance grows without any corresponding claimable accounting entry, breaking the zero-balance invariant. The legitimate provider (`req.provider`) loses their earned fee, and the locked ETH is irrecoverable by any party.

---

### Likelihood Explanation

The attack requires only:
1. Waiting for the exclusivity period to expire on any unfulfilled request (a normal operational scenario when the assigned provider is slow or offline).
2. Calling `executeCallback` with `address(0)` as `providerToCredit` and valid Pyth update data (publicly available from the Pyth price service).

No privileged access, leaked keys, or external oracle manipulation is required. Any unprivileged transaction sender can execute this.

---

### Recommendation

Add a validation check in `executeCallback` that `providerToCredit` is a registered provider (i.e., `_state.providers[providerToCredit].isRegistered == true`) before crediting fees. Alternatively, fall back to crediting `req.provider` when `providerToCredit` is unregistered, or revert if `providerToCredit` has no reachable fee manager.

---

### Proof of Concept

1. User calls `requestPriceUpdatesWithCallback(provider, publishTime, priceIds, gasLimit)` with `msg.value = requiredFee`. `req.fee = msg.value - pythFeeInWei` is stored.
2. The assigned `provider` does not call `executeCallback` before `req.publishTime + exclusivityPeriodSeconds`.
3. Attacker calls:
   ```solidity
   echo.executeCallback(
       address(0),       // providerToCredit — unregistered, feeManager == address(0)
       sequenceNumber,
       validUpdateData,  // fetched from Pyth price service
       priceIds
   );
   ```
4. `_state.providers[address(0)].accruedFeesInWei += req.fee - pythFee` is executed.
5. No caller can ever satisfy `msg.sender == _state.providers[address(0)].feeManager` (which is `address(0)`), so `withdrawAsFeeManager` always reverts for this provider.
6. The user's fee is permanently locked in the contract.
