### Title
Unvalidated `block_hash` in `GetBlockProposalProcess::execute()` Allows Unauthenticated Peers to Trigger Repeated Tx Pool Read-Lock Contention — (`sync/src/relayer/get_block_proposal_process.rs`)

---

### Summary

`GetBlockProposalProcess::execute()` reads the `block_hash` field from the incoming `GetBlockProposal` message but **never validates it against chain state**. Any connected relay peer can supply an arbitrary hash (genesis, all-zeros, stale) and still cause the node to call `tx_pool.fetch_txs()` with up to 3,000 proposal short IDs. The only guards are a proposal-count ceiling and a per-peer rate limiter of 30 req/s. With many peers at the rate limit, the single-threaded tx-pool service loop can be kept continuously busy processing `FetchTxs` messages, delaying legitimate operations.

---

### Finding Description

In `execute()`, the message's `block_hash` field is never read after the struct is constructed: [1](#0-0) 

The only pre-`fetch_txs` checks are:

1. **Proposal count** — must not exceed `max_block_proposals_limit * max_uncles_num` (1,500 × 2 = **3,000** on mainnet). [2](#0-1) 

2. **Duplicate proposals** — the set size must equal the slice length. [3](#0-2) 

Neither check touches `block_hash`. After passing both, `fetch_txs` is called unconditionally: [4](#0-3) 

`fetch_txs` enqueues a `Message::FetchTxs` on the tx-pool service channel and awaits a oneshot response: [5](#0-4) 

The handler dispatches to `get_tx_for_compact_block`, which acquires **three sequential async read locks** (`verify_queue`, `orphan`, `tx_pool`) and iterates over all supplied short IDs under each lock: [6](#0-5) 

The tx-pool service is a **single-threaded async event loop** — all messages are processed one at a time. Flooding it with `FetchTxs` messages starves other operations (`SubmitTx`, `BlockTemplate`, chain reorg notifications, etc.).

The per-peer rate limiter is 30 req/s across all relay message types: [7](#0-6) [8](#0-7) 

With `MAX_RELAY_PEERS = 128` peers each at 30 req/s, the service receives up to **3,840 `FetchTxs` messages/s**, each requiring 3 read-lock acquisitions and up to 3,000 hash-map lookups. [9](#0-8) 

---

### Impact Explanation

- The tx-pool service message queue is saturated, delaying or stalling `BlockTemplate` generation (miner starvation) and `SubmitTx` processing.
- Read-lock contention on `tx_pool`, `verify_queue`, and `orphan` increases latency for all concurrent readers.
- The attack requires no special privileges — any connected relay peer qualifies.
- The `block_hash` field being ignored means the attacker does not need to know any valid recent block hash; genesis hash, all-zeros, or random bytes all work identically.

---

### Likelihood Explanation

The attack is low-cost: an attacker opens relay connections (up to `MAX_RELAY_PEERS`) and sends `GetBlockProposal` messages at the rate-limiter ceiling with 3,000 arbitrary short IDs and any `block_hash`. No PoW, no keys, no chain knowledge required. The rate limiter is the sole mitigation and it is permissive enough that many peers together can sustain continuous pressure on the service loop.

---

### Recommendation

Before calling `fetch_txs`, validate that `block_hash` refers to a block within the current proposal window relative to the chain tip (i.e., between `tip_number - farthest` and `tip_number + closest`). Reject requests whose `block_hash` is unknown or outside this window with a `StatusCode::ProtocolMessageIsMalformed` or a short ban. This mirrors the window already used in `BlockProposalProcess` and `request_proposal_txs`. [10](#0-9) [11](#0-10)

    pub async fn execute(self) -> Status {
        let shared = self.relayer.shared();
        let message_len = self.message.proposals().len();
        {
            // The block proposal request is separate from uncles,
            // so here the limit is only used to calculate the maximum value of uncles
            let limit = shared.consensus().max_block_proposals_limit()
                * (shared.consensus().max_uncles_num() as u64);
            if message_len as u64 > limit {
                return StatusCode::ProtocolMessageIsMalformed.with_context(format!(
                    "GetBlockProposal proposals count({message_len}) > consensus max_block_proposals_limit({limit})"
                ));
            }
        }

        let proposals: HashSet<packed::ProposalShortId> =
            self.message.proposals().to_entity().into_iter().collect();

        if proposals.len() != message_len {
            return StatusCode::RequestDuplicate.with_context("Request duplicate proposal");
        }

        let fetched_transactions = {
            let tx_pool = self.relayer.shared.shared().tx_pool_controller();
            let fetch_txs = tx_pool.fetch_txs(proposals.clone()).await;
            if let Err(e) = fetch_txs {
                debug_target!(
                    crate::LOG_TARGET_RELAY,
                    "relayer tx_pool_controller send fetch_txs error: {:?}",
                    e
                );
                return Status::ok();
            }
            fetch_txs.unwrap()
        };

    pub async fn get_tx_for_compact_block(
        &self,
        short_ids: HashSet<ProposalShortId>,
    ) -> HashMap<ProposalShortId, TransactionView> {
        let mut txs = HashMap::with_capacity(short_ids.len());
        {
            let verify_queue = self.verify_queue.read().await;
            txs.extend(short_ids.iter().filter_map(|short_id| {
                verify_queue
                    .get_tx_by_id(short_id)
                    .map(|entry| (short_id.to_owned(), entry.tx.to_owned()))
            }));
        }
        {
            let orphan = self.orphan.read().await;
            txs.extend(short_ids.iter().filter_map(|short_id| {
                orphan
                    .get(short_id)
                    .map(|entry| (short_id.to_owned(), entry.tx.to_owned()))
            }));
        }
        {
            let tx_pool = self.tx_pool.read().await;
            txs.extend(short_ids.iter().filter_map(|short_id| {
                tx_pool
                    .get_tx_from_pool_or_store(short_id)
                    .map(|tx| (short_id.to_owned(), tx))
            }));
        }
        txs
    }
