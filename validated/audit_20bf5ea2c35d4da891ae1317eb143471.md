### Title
Linear Storage Scan in `onlyWhitelistedReader` Modifier Causes Unbounded Gas Cost on Every Price Read — (`target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

The `onlyWhitelistedReader` modifier in `Scheduler.sol` performs a full linear scan over the `readerWhitelist` storage array on every invocation of `getPricesUnsafe`, `getPricesNoOlderThan`, `getEmaPricesUnsafe`, and `getEmaPricesNoOlderThan`. Additionally, `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` each make an external `this.` call to the underlying `getPricesUnsafe`/`getEmaPricesUnsafe` function, causing the whitelist scan to execute **twice** per call. This is a direct analog to the reported pattern of expensive looping with repeated storage reads.

---

### Finding Description

The `onlyWhitelistedReader` modifier reads the `readerWhitelist` array directly from contract storage and iterates over every element to check membership:

```solidity
address[] storage whitelist = _state
    .subscriptionParams[subscriptionId]
    .readerWhitelist;
bool isWhitelisted = false;
for (uint i = 0; i < whitelist.length; i++) {
    if (whitelist[i] == msg.sender) {
        isWhitelisted = true;
        break;
    }
}
``` [1](#0-0) 

Each `whitelist[i]` access is a cold SLOAD (2,100 gas) on first access per slot. With `MAX_READER_WHITELIST_SIZE` entries, the worst-case cost of the modifier alone is `MAX_READER_WHITELIST_SIZE × 2,100` gas.

The compounding issue is that `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` both carry the `onlyWhitelistedReader` modifier **and** make an external call to `this.getPricesUnsafe` / `this.getEmaPricesUnsafe`, which also carries `onlyWhitelistedReader`:

```solidity
prices = this.getPricesUnsafe(subscriptionId, priceIds);
``` [2](#0-1) 

```solidity
prices = this.getEmaPricesUnsafe(subscriptionId, priceIds);
``` [3](#0-2) 

This means the whitelist scan runs **twice** for every call to `getPricesNoOlderThan` or `getEmaPricesNoOlderThan`, doubling the gas cost. The external `this.` call also adds the overhead of an external call (2,100 gas base + ABI encoding/decoding).

The four affected public functions are: [4](#0-3) [5](#0-4) [6](#0-5) [7](#0-6) 

The `MAX_READER_WHITELIST_SIZE` constant caps the whitelist, but even at a moderate cap (e.g., 100), the worst-case modifier cost is ~210,000 gas per call, and ~420,000 gas for the double-scan functions — before any actual price-reading logic executes. [8](#0-7) 

---

### Impact Explanation

Any on-chain contract (DeFi protocol, liquidation bot, etc.) that calls `getPricesNoOlderThan` or `getEmaPricesNoOlderThan` on a subscription with a large whitelist pays O(N) storage reads **twice** per invocation. This can:

1. Make on-chain price consumption prohibitively expensive, effectively denying service to legitimate whitelisted consumers.
2. Cause on-chain consumers to run out of gas if they allocate a fixed gas budget for the price read.
3. Allow a subscription manager to silently degrade service for their own whitelisted readers by maximizing the whitelist size.

Even non-whitelisted callers trigger the full scan before receiving `Unauthorized`, wasting gas on every rejected call.

---

### Likelihood Explanation

- Any subscription manager (an unprivileged role — any user can create a subscription) can set `readerWhitelist` to `MAX_READER_WHITELIST_SIZE` entries at subscription creation or update time.
- The four affected functions are the primary price-reading interface for on-chain consumers of the Scheduler/Pulse product.
- The double-scan pattern in `getPricesNoOlderThan` / `getEmaPricesNoOlderThan` is triggered by any whitelisted on-chain consumer calling the most safety-checked variants of the API.

---

### Recommendation

1. **Replace the linear whitelist scan with a `mapping(address => bool)`** stored alongside the `readerWhitelist` array. Membership checks become O(1) (single SLOAD) instead of O(N).

2. **Eliminate the double-scan** in `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` by inlining the internal logic instead of making an external `this.` call. Call `_getPricesInternal` directly rather than routing through the public function with its own modifier.

Instead of:
```solidity
prices = this.getPricesUnsafe(subscriptionId, priceIds); // triggers modifier again
```

Use:
```solidity
PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
``` [9](#0-8) 

---

### Proof of Concept

1. Deploy `Scheduler` and create a subscription with `MAX_READER_WHITELIST_SIZE` unique addresses in `readerWhitelist` and `whitelistEnabled = true`.
2. From a whitelisted on-chain consumer contract, call `getPricesNoOlderThan(subscriptionId, priceIds, age)`.
3. Observe that `onlyWhitelistedReader` runs twice (once on `getPricesNoOlderThan`, once on the internal `this.getPricesUnsafe` call), each scanning all `MAX_READER_WHITELIST_SIZE` storage slots.
4. Measure gas: the whitelist scan alone consumes `2 × MAX_READER_WHITELIST_SIZE × 2,100` gas (cold SLOADs), before any price data is read.
5. Repeat with a non-whitelisted caller: the full scan still executes before the `Unauthorized` revert, wasting the caller's gas. [10](#0-9)

    function _getPricesInternal(
        uint256 subscriptionId,
        bytes32[] calldata priceIds
    ) internal view returns (PythStructs.PriceFeed[] memory priceFeeds) {
        if (!_state.subscriptionParams[subscriptionId].isActive) {
            revert SchedulerErrors.InactiveSubscription();
        }

        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        // If no price IDs provided, return all price feeds for the subscription
        if (priceIds.length == 0) {
            PythStructs.PriceFeed[]
                memory allFeeds = new PythStructs.PriceFeed[](
                    params.priceIds.length
                );
            for (uint8 i = 0; i < params.priceIds.length; i++) {
                PythStructs.PriceFeed memory priceFeed = _state.priceUpdates[
                    subscriptionId
                ][params.priceIds[i]];
                // Check if the price feed exists (price ID is valid and has been updated)
                if (priceFeed.id == bytes32(0)) {
                    revert SchedulerErrors.InvalidPriceId(
                        params.priceIds[i],
                        bytes32(0)
                    );
                }
                allFeeds[i] = priceFeed;
            }
            return allFeeds;
        }

        // Return only the requested price feeds
        PythStructs.PriceFeed[]
            memory requestedFeeds = new PythStructs.PriceFeed[](
                priceIds.length
            );
        for (uint8 i = 0; i < priceIds.length; i++) {
            PythStructs.PriceFeed memory priceFeed = _state.priceUpdates[
                subscriptionId
            ][priceIds[i]];

            // Check if the price feed exists (price ID is valid and has been updated)
            if (priceFeed.id == bytes32(0)) {
                revert SchedulerErrors.InvalidPriceId(priceIds[i], bytes32(0));
            }
            requestedFeeds[i] = priceFeed;
        }
        return requestedFeeds;
    }

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
