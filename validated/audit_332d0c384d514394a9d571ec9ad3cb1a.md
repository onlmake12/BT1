### Title
Fees Credited to Unregistered `providerToCredit` in `Echo::executeCallback` Are Permanently Locked - (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary

`Echo::executeCallback` accepts a caller-supplied `providerToCredit` address and credits the full request fee to `_state.providers[providerToCredit].accruedFeesInWei` without validating that `providerToCredit` is a registered provider. After the exclusivity period expires, any unprivileged caller can supply an unregistered address. Because the only withdrawal path for provider fees is `withdrawAsFeeManager`, which requires `msg.sender == _state.providers[provider].feeManager`, and an unregistered address has `feeManager == address(0)` by default and cannot call `setFeeManager` (which requires `isRegistered == true`), all fees credited to that address are permanently locked in the contract.

### Finding Description

In `Echo.sol`, `executeCallback` credits the provider fee unconditionally:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
``` [1](#0-0) 

The only guard on `providerToCredit` is the exclusivity-period check, which enforces that `providerToCredit == req.provider` only while `block.timestamp < req.publishTime + exclusivityPeriodSeconds`. After that window, the check is skipped entirely:

```solidity
if (block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds) {
    require(
        providerToCredit == req.provider,
        "Only assigned provider during exclusivity period"
    );
}
``` [2](#0-1) 

There is no subsequent check that `providerToCredit` is a registered provider.

The only withdrawal path for provider-accrued fees is `withdrawAsFeeManager`:

```solidity
require(
    msg.sender == _state.providers[provider].feeManager,
    "Only fee manager"
);
``` [3](#0-2) 

For an unregistered address, `feeManager` is `address(0)` (Solidity default). No EOA or contract can satisfy `msg.sender == address(0)`.

The only way to set a fee manager is `setFeeManager`, which requires the caller to be a registered provider:

```solidity
require(
    _state.providers[msg.sender].isRegistered,
    "Provider not registered"
);
``` [4](#0-3) 

An unregistered address cannot call `setFeeManager`, so it can never unlock its `accruedFeesInWei`. There is no admin-level recovery function for provider fees, and `withdrawFees` only covers `_state.accruedFeesInWei` (Pyth protocol fees), not per-provider balances. [5](#0-4) 

### Impact Explanation

Any ETH paid by a requester as the provider fee portion (`req.fee`) is permanently locked in the `Echo` contract when `executeCallback` is called with an unregistered `providerToCredit`. The legitimate provider who fulfilled the request receives nothing, and the funds cannot be recovered by any party — not the provider, not the requester, and not the admin. This is a direct, irrecoverable loss of user-paid funds.

### Likelihood Explanation

After the exclusivity period (configurable, default 15 seconds), `executeCallback` is permissionless — any address can call it with any `providerToCredit`. A malicious actor can monitor the mempool for pending requests, wait for the exclusivity window to expire, and front-run the legitimate provider's `executeCallback` call by supplying an unregistered address. This requires no privileged access, no leaked keys, and no external oracle manipulation. The attacker's only cost is gas.

### Recommendation

Add a registration check for `providerToCredit` inside `executeCallback`, analogous to the check in `requestPriceUpdatesWithCallback`:

```solidity
require(
    _state.providers[providerToCredit].isRegistered,
    "providerToCredit not registered"
);
```

This should be placed before the fee credit line at `Echo.sol:161`.

### Proof of Concept

1. Alice (requester) calls `requestPriceUpdatesWithCallback` with `provider = registeredProvider`, paying `totalFee`. The request is stored with `req.fee = totalFee - pythFeeInWei`.
2. The exclusivity period (15 seconds) elapses.
3. Attacker calls `executeCallback(address(0xdead), sequenceNumber, updateData, priceIds)` where `address(0xdead)` is not a registered provider.
4. The contract executes successfully: `_state.providers[0xdead].accruedFeesInWei += req.fee + msg.value - pythFee`.
5. `address(0xdead)` has `isRegistered == false` and `feeManager == address(0)`.
6. `withdrawAsFeeManager(0xdead, amount)` reverts because `msg.sender != address(0)` is always true.
7. `setFeeManager` reverts for `0xdead` because `isRegistered == false`.
8. The provider fee is permanently locked. The legitimate `registeredProvider` receives nothing.
