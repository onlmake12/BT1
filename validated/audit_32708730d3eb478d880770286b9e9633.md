### Title
Fee Charged Unconditionally Before Freshness Check in `update_single_price_feed` — (`target_chains/sui/contracts/sources/pyth.move`)

### Summary
`update_single_price_feed` deposits the caller's fee into the `PriceInfoObject` **before** evaluating whether the supplied price update is fresh. When the update timestamp is not strictly greater than the cached timestamp, `update_cache` silently no-ops (no abort, no revert), and the function returns normally. The caller permanently loses `base_update_fee` SUI with no price change applied and no refund path.

### Finding Description

The ordering in `update_single_price_feed` is:

1. **Fee sufficiency check** (line 274) — asserts `base_update_fee <= coin::value(&fee)`.
2. **Unconditional fee deposit** (line 277) — `price_info::deposit_fee_coins(price_info_object, fee)` executes regardless of whether the update will be applied.
3. **`update_cache` call** (line 287) — internally calls `is_fresh_update`.
4. **Freshness gate** (line 316) — `if (is_fresh_update(...))` wraps the actual state mutation; if false, the body is skipped and `update_cache` returns normally.
5. **Normal return** (line 295) — `update_single_price_feed` returns the hot potato with no abort. [1](#0-0) [2](#0-1) [3](#0-2) 

`is_fresh_update` returns `update_timestamp > cached_timestamp` — a strict greater-than. Any update with timestamp equal to or older than the cached value returns `false`, causing `update_cache` to skip the write silently. [4](#0-3) 

### Impact Explanation

The caller's `Coin<SUI>` of value `>= base_update_fee` is consumed by `deposit_fee_coins` and stored inside the `PriceInfoObject`. There is no refund branch anywhere in `update_single_price_feed`. The price feed state is unchanged. The caller has paid for a service that was not rendered.

This matches the scoped impact: **direct loss of caller funds** (the fee) with no compensating state change.

### Likelihood Explanation

This is reachable by any unprivileged caller in normal operation:

- **Race condition**: two independent updaters submit the same price feed update concurrently. The second transaction's update arrives with an equal or older timestamp; it pays the fee and gets nothing.
- **Deliberate replay**: a caller re-submits a previously valid (but now stale) authenticated `HotPotatoVector<PriceInfo>` — the Wormhole VAA replay protection is consumed at VAA verification time (in `create_price_infos_hot_potato` / `create_authenticated_price_infos_using_accumulator`), but the resulting `HotPotatoVector` carries no replay guard of its own. The same hot potato can be passed to `update_single_price_feed` once; if the cached price has already been updated to a newer timestamp by then, the fee is lost.
- **Equal-timestamp update**: an update with `timestamp == cached_timestamp` is not fresh (`>` not `>=`), so the fee is taken and no update occurs.

### Recommendation

Move the fee deposit **after** confirming the update is fresh, or abort the transaction when `is_fresh_update` returns false. The simplest fix:

```move
// In update_cache: abort instead of silently skipping
if (!is_fresh_update(update, price_info_object)) {
    // caller should not have been charged; abort here
    // OR: restructure update_single_price_feed to check freshness before deposit
    return
};
```

Preferred fix in `update_single_price_feed`: check freshness before `deposit_fee_coins`, and only deposit if the update will actually be applied.

### Proof of Concept

Invariant violated: for any call to `update_single_price_feed` where `update_timestamp <= cached_timestamp`, either the transaction aborts OR no fee is deducted. The code satisfies neither branch — it deducts the fee and returns successfully.

```
State: base_update_fee = 50, PriceInfoObject cached at timestamp T=100
Attacker: obtains HotPotatoVector<PriceInfo> with timestamp T-1=99 (valid VAA, valid data source)
Call: update_single_price_feed(state, hot_potato, price_info_object, coin(50), clock)
  → assert 50 <= 50  ✓
  → deposit_fee_coins(price_info_object, coin(50))  ← fee gone
  → update_cache(...)
      → is_fresh_update: 99 > 100 = false → skip, no abort
  → return hot_potato  ← no revert
Result: caller lost 50 SUI, price unchanged, no refund
``` [5](#0-4)

    public fun update_single_price_feed(
        pyth_state: &PythState,
        price_updates: HotPotatoVector<PriceInfo>,
        price_info_object: &mut PriceInfoObject,
        fee: Coin<SUI>,
        clock: &Clock
    ): HotPotatoVector<PriceInfo> {
        let latest_only = state::assert_latest_only(pyth_state);

        // On Sui, users get to choose which price feeds to update. They specify a single price feed to
        // update at a time. We therefore charge the base fee for each such individual update.
        // This is a departure from Eth, where users don't get to necessarily choose.
        assert!(state::get_base_update_fee(pyth_state) <= coin::value(&fee), E_INSUFFICIENT_FEE);

        // store fee coins within price info object
        price_info::deposit_fee_coins(price_info_object, fee);

        // Find price update corresponding to PriceInfoObject within the array of price_updates
        // and use it to update PriceInfoObject.
        let i = 0;
        let found = false;
        while (i < hot_potato_vector::length<PriceInfo>(&price_updates)){
            let cur_price_info = hot_potato_vector::borrow<PriceInfo>(&price_updates, i);
            if (has_same_price_identifier(cur_price_info, price_info_object)){
                found = true;
                update_cache(latest_only, cur_price_info, price_info_object, clock);
                break
            };
            i = i + 1;
        };
        if (found==false){
            abort E_PRICE_UPDATE_NOT_FOUND_FOR_PRICE_INFO_OBJECT
        };
        price_updates
    }

    fun is_fresh_update(update: &PriceInfo, price_info_object: &PriceInfoObject): bool {
        // Get the timestamp of the update's current price
        let price_feed = price_info::get_price_feed(update);
        let update_timestamp = price::get_timestamp(&price_feed::get_price(price_feed));

        // Get the timestamp of the cached data for the price identifier
        let cached_price_info = price_info::get_price_info_from_price_info_object(price_info_object);
        let cached_price_feed =  price_info::get_price_feed(&cached_price_info);
        let cached_timestamp = price::get_timestamp(&price_feed::get_price(cached_price_feed));

        update_timestamp > cached_timestamp
    }
