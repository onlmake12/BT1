### Title
`undelegate` Does Not Settle Accumulated OIS Rewards Before Removing Position - (`governance/pyth_staking_sdk/src/idl/integrity-pool.json`, `governance/pyth_staking_sdk/src/pyth-staking-client.ts`)

---

### Summary

The Oracle Integrity Staking (OIS) `undelegate` instruction removes a user's delegation position without first settling (advancing) the delegation record. The SDK methods `unstakeFromPublisher` and `unstakeFromAllPublishers` only submit `undelegate` instructions with no prior `advance_delegation_record` call. After the position is removed, the SDK's `getAdvanceDelegationRecordInstructions` filters out publishers for which the user has no remaining positions, making accumulated rewards permanently unclaimable.

---

### Finding Description

The `advance_delegation_record` instruction is the mechanism by which a delegator's accumulated OIS rewards are computed and transferred to their custody account. It reads the user's `stakeAccountPositions` to determine their historical stake and updates `delegationRecord.lastEpoch` to the current epoch.

The `undelegate` instruction's account list (from the IDL) does **not** include `delegation_record`:

```
owner, pool_data, pool_config, publisher, config_account,
stake_account_positions, stake_account_metadata,
stake_account_custody, staking_program, system_program
``` [1](#0-0) 

There is no `delegation_record` account — the instruction cannot settle rewards as part of undelegation.

The SDK's `unstakeFromPublisher` and `unstakeFromAllPublishers` methods build and send only `undelegate` instructions, with no preceding `advance_delegation_record`: [2](#0-1) [3](#0-2) 

After the position is removed by `undelegate`, `getAdvanceDelegationRecordInstructions` filters publishers to only those where the user still has active positions (`lowestEpoch !== undefined`). A publisher whose position was just removed is excluded: [4](#0-3) 

This means the SDK will never generate an `advance_delegation_record` instruction for that publisher after undelegation. If the user subsequently calls `advanceDelegationRecord` (the claim function), the removed publisher is silently skipped. [5](#0-4) 

The `claim` API entry point calls only `advanceDelegationRecord`: [6](#0-5) 

---

### Impact Explanation

A delegator who calls `unstakeFromPublisher` or `unstakeFromAllPublishers` without first calling `claim` (i.e., `advanceDelegationRecord`) forfeits all accumulated OIS rewards for the undelegated publisher positions. The `delegationRecord.lastEpoch` is never advanced for those publishers, but the positions are gone, so no future call to `advance_delegation_record` via the SDK will cover them. Rewards earned across all epochs since the last advance are permanently lost.

**Note:** Per OP-PIP-103, the reward rate `y` is currently set to 0, so no rewards are actively accruing at this time. However, the vulnerability is structural and becomes fully exploitable the moment `y` is set to a non-zero value. [7](#0-6) 

---

### Likelihood Explanation

Any staking user who undelegates via the standard SDK path (`unstakeIntegrityStaking` → `unstakeFromPublisher`) without separately claiming first will trigger this. The DAPP UI separates "Unstake" and "Claim" into distinct actions with no enforced ordering, making it easy for users to unstake without claiming. Direct on-chain callers face the same issue since `undelegate` has no built-in reward settlement. [8](#0-7) 

---

### Recommendation

In `unstakeFromPublisher` and `unstakeFromAllPublishers`, prepend `advance_delegation_record` instructions for each affected publisher before the `undelegate` instructions, mirroring the pattern already used in `advanceDelegationRecord`. Alternatively, the on-chain `undelegate` instruction should be modified to require and invoke `advance_delegation_record` as a CPI before closing the position.

---

### Proof of Concept

1. User delegates 1000 PYTH to publisher P at epoch E.
2. Several epochs pass; rewards accumulate (once `y > 0`). `delegationRecord.lastEpoch` = E.
3. User calls `unstakeFromPublisher(stakeAccountPositions, P, LOCKED, 1000n)`.
   - SDK sends only `undelegate` — no `advance_delegation_record`.
   - Position for publisher P is removed from `stakeAccountPositions`.
4. User calls `claim(stakeAccount)` → `advanceDelegationRecord(stakeAccountPositions)`.
   - `getAdvanceDelegationRecordInstructions` iterates all publishers, filters to those with `lowestEpoch !== undefined`.
   - Publisher P has no remaining positions → `lowestEpoch = undefined` → filtered out.
   - No `advance_delegation_record` instruction is generated for P.
5. Rewards for epochs E through currentEpoch for publisher P are permanently lost. [9](#0-8) [10](#0-9)

  public async unstakeFromPublisher(
    stakeAccountPositions: PublicKey,
    publisher: PublicKey,
    positionState: PositionState.LOCKED | PositionState.LOCKING,
    amount: bigint,
  ) {
    const stakeAccountPositionsData = await this.getStakeAccountPositions(
      stakeAccountPositions,
    );
    const currentEpoch = await getCurrentEpoch(this.connection);

    let remainingAmount = amount;
    const instructionPromises: Promise<TransactionInstruction>[] = [];

    const eligiblePositions = stakeAccountPositionsData.data.positions
      .map((p, i) => ({ index: i, position: p }))
      .reverse()
      .filter(
        ({ position }) =>
          position.targetWithParameters.integrityPool?.publisher !==
            undefined &&
          position.targetWithParameters.integrityPool.publisher.equals(
            publisher,
          ) &&
          positionState === getPositionState(position, currentEpoch),
      );

    for (const { position, index } of eligiblePositions) {
      if (position.amount < remainingAmount) {
        instructionPromises.push(
          this.integrityPoolProgram.methods
            .undelegate(index, convertBigIntToBN(position.amount))
            .accounts({
              publisher,
              stakeAccountPositions,
            })
            .instruction(),
        );
        remainingAmount -= position.amount;
      } else {
        instructionPromises.push(
          this.integrityPoolProgram.methods
            .undelegate(index, convertBigIntToBN(remainingAmount))
            .accounts({
              publisher,
              stakeAccountPositions,
            })
            .instruction(),
        );
        break;
      }
    }

    const instructions = await Promise.all(instructionPromises);
    return sendTransaction(instructions, this.connection, this.wallet);
  }

  public async unstakeFromAllPublishers(
    stakeAccountPositions: PublicKey,
    positionStates: (PositionState.LOCKED | PositionState.LOCKING)[],
  ) {
    const [stakeAccountPositionsData, currentEpoch] = await Promise.all([
      this.getStakeAccountPositions(stakeAccountPositions),
      getCurrentEpoch(this.connection),
    ]);

    const instructions = await Promise.all(
      stakeAccountPositionsData.data.positions
        .map((position, index) => {
          const publisher =
            position.targetWithParameters.integrityPool?.publisher;
          return publisher === undefined
            ? undefined
            : { index, position, publisher };
        })
        // By separating this filter from the next, typescript can narrow the
        // type and automatically infer that there will be no `undefined` values
        // in the array after this line.  If we combine those filters,
        // typescript won't narrow properly.
        .filter((positionInfo) => positionInfo !== undefined)
        .filter(({ position }) =>
          (positionStates as PositionState[]).includes(
            getPositionState(position, currentEpoch),
          ),
        )
        .reverse()
        .map(({ position, index, publisher }) =>
          this.integrityPoolProgram.methods
            .undelegate(index, convertBigIntToBN(position.amount))
            .accounts({ publisher, stakeAccountPositions })
            .instruction(),
        ),
    );

    return sendTransaction(instructions, this.connection, this.wallet);
  }

  async getAdvanceDelegationRecordInstructions(
    stakeAccountPositions: PublicKey,

    const stakeAccountPositionsData = await this.getStakeAccountPositions(
      stakeAccountPositions,

    const allPublishers = extractPublisherData(poolData);
    const publishers = allPublishers
