Audit Report

## Title
Attacker-Controlled Peer ID in `observed_addr` Poisons Victim's Broadcast Listen Addresses — (`network/src/protocols/identify/mod.rs`)

## Summary
`IdentifyCallback::add_observed_addr` guards only against a *missing* peer ID component, not a *wrong* one. A remote peer can send a crafted `observed_addr` containing an arbitrary foreign peer ID, which is stored verbatim and subsequently included in the victim's `local_listen_addrs` broadcast to every newly connected peer, causing network-wide routing failures for the victim node.

## Finding Description
The guard in `add_observed_addr` (L497–507) is gated solely on presence of a peer ID, not correctness:

```rust
fn add_observed_addr(&mut self, mut addr: Multiaddr, session_id: SessionId) -> MisbehaveResult {
    if extract_peer_id(&addr).is_none() {
        addr.push(Protocol::P2P(Cow::Borrowed(
            self.network_state.local_peer_id().as_bytes(),
        )))
    }
    self.network_state.add_observed_addr(session_id, addr);
    MisbehaveResult::Continue
}
``` [1](#0-0) 

If an attacker sends `/ip4/<victim_ip>/tcp/8114/p2p/<attacker_peer_id>`, `extract_peer_id` returns `Some(attacker_peer_id)`, the push is skipped, and the address with the wrong peer ID is stored verbatim. `NetworkState::add_observed_addr` performs no further validation: [2](#0-1) 

`observed_addrs` returns stored values as-is with no filtering: [3](#0-2) 

`local_listen_addrs` unconditionally extends its result with `observed_addrs`: [4](#0-3) 

On every `connected` event, the victim calls `local_listen_addrs()` and sends the result — including the poisoned address — to the newly connected peer: [5](#0-4) 

Full exploit path: attacker establishes one valid inbound connection → sends `IdentifyMessage` with `observed_addr = /ip4/<victim_ip>/tcp/<port>/p2p/<attacker_peer_id>` → victim stores it via `add_observed_addr` → victim includes it in `local_listen_addrs` → victim broadcasts the poisoned address to all subsequent peers → those peers attempt to connect to `victim_ip:port` expecting `attacker_peer_id`, Noise/TLS handshake fails because the actual node presents `victim_peer_id`.

## Impact Explanation
**High (10001–15000 points): Vulnerabilities or bad designs which could cause CKB network congestion with few costs.** The victim node's reachable address set is poisoned with a foreign peer ID. Every peer that receives the victim's `listen_addrs` stores an unconnectable address for the victim. As the victim connects to more peers, the poisoned address propagates network-wide, causing cascading connection failures and effectively degrading the victim's reachability across the network. The attacker needs only one accepted inbound connection to trigger this, and the effect is persistent until the victim restarts or the observed address is evicted.

## Likelihood Explanation
Any unprivileged peer that can establish a standard inbound connection to the victim can trigger this. No special privileges, no proof-of-work, no key material required. The attacker only needs to be accepted as a peer and send one crafted `IdentifyMessage`. The condition is trivially reachable on any publicly reachable CKB node.

## Recommendation
In `add_observed_addr`, when `extract_peer_id` returns `Some(peer_id)`, verify that `peer_id == self.network_state.local_peer_id()`. If it does not match, discard the address:

```rust
fn add_observed_addr(&mut self, mut addr: Multiaddr, session_id: SessionId) -> MisbehaveResult {
    match extract_peer_id(&addr) {
        None => {
            addr.push(Protocol::P2P(Cow::Borrowed(
                self.network_state.local_peer_id().as_bytes(),
            )));
        }
        Some(peer_id) if peer_id != *self.network_state.local_peer_id() => {
            return MisbehaveResult::Continue; // discard foreign peer ID
        }
        _ => {}
    }
    self.network_state.add_observed_addr(session_id, addr);
    MisbehaveResult::Continue
}
```

This unit test directly exercises `add_observed_addr` with a crafted address and asserts the invariant that the stored peer ID must equal the local peer ID. It will fail on the current code, confirming the vulnerability.
