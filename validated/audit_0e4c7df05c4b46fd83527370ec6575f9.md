### Title
Premature `add_anchors` Before Identify Completes Allows Attacker-Controlled Address Persistence in `anchors.db` — (`network/src/peer_registry.rs`)

---

### Summary

`accept_peer` unconditionally calls `peer_store.add_anchors(remote_addr)` at TCP session-open time, before the identify protocol runs. There is no code path that removes an address from the in-memory `Anchors` set when the session is subsequently closed (for any reason). On node shutdown, the in-memory anchors are flushed to `anchors.db`. On the next startup, those addresses are drained and dialed unconditionally. An attacker whose address is in the victim's peer store can exploit this to achieve persistent forced reconnection across every node restart.

---

### Finding Description

**Step 1 — `add_anchors` called at session open, before identify**

In `accept_peer`, when `non_whitelist_outbound >= max_outbound` and the block-relay-only slot is available, the code at line 130 calls `peer_store.add_anchors(remote_addr.clone())` and then sets `session_type = SessionType::BlockRelayOnly`. This happens inside the `ServiceEvent::SessionOpen` handler — the identify protocol has not yet run. [1](#0-0) 

**Step 2 — No removal from anchors on session close**

The `SessionClose` handler calls `peer_store.remove_disconnected_peer`, which only removes the peer from `connected_peers` (a `HashMap<PeerId, PeerInfo>`). It does not touch the `Anchors` set. [2](#0-1) [3](#0-2) 

**Step 3 — `Anchors` has no `remove` method**

The `Anchors` struct exposes only `add`, `count`, `dump_iter`, `drain`, and `contains`. There is no `remove`. Once an address is added, it can only leave the set via `drain()` (called once at startup) or by never being written to disk. [4](#0-3) 

**Step 4 — `ban_addr` also does not remove from anchors**

When identify fails due to wrong network, `ban_session` → `ban_addr` is called. `ban_addr` removes the address from `addr_manager` but does not touch `anchors`. [5](#0-4) 

**Step 5 — Anchors flushed to disk on shutdown**

`dump_to_dir` serializes the in-memory `Anchors` (up to `MAX_OUTBOUND_BLOCK_RELAY = 2` entries) to `anchors.db`. [6](#0-5) [7](#0-6) 

**Step 6 — Anchors dialed unconditionally on restart**

`NetworkService::start` drains the loaded anchors and dials every address via `dial_identify`, before any handshake validation. [8](#0-7) 

---

### Impact Explanation

An attacker whose address is in the victim's peer store (reachable via the discovery protocol) and who is dialed when the victim's regular outbound slots are full gets their address written into `anchors.db`. Even if the connection is immediately dropped (identify timeout, flag mismatch, or deliberate disconnect), the anchor entry survives in memory and is flushed to disk on shutdown. On every subsequent restart, the victim unconditionally dials the attacker's address. With `MAX_OUTBOUND_BLOCK_RELAY = 2`, an attacker controlling two addresses can fill both anchor slots, contributing to a long-term eclipse of the victim node and potential consensus deviation. [9](#0-8) 

---

### Likelihood Explanation

Preconditions are realistic on mainnet: a victim node with all regular outbound slots occupied is common under normal load. The attacker only needs to be discoverable (standard P2P gossip) and reachable. No privileged access, no majority hashpower, and no social engineering is required. The attacker does not need to maintain the connection — a single TCP handshake that reaches `SessionOpen` is sufficient to plant the anchor.

---

### Recommendation

1. Add a `remove(&mut self, addr: &Multiaddr)` method to `Anchors`.
2. In the `SessionClose` handler (or in `remove_disconnected_peer`), call `anchors.remove(&addr)` for any peer whose session closes before identify completes (i.e., before `peer.identify_info` is set).
3. Alternatively, defer `add_anchors` until `received_identify` succeeds and the peer is confirmed to be on the correct network with the required flags.

---

    pub(crate) fn accept_peer(
        &mut self,
        remote_addr: Multiaddr,
        session_id: SessionId,
        raw_session_type: RawSessionType,
        peer_store: &mut PeerStore,
    ) -> Result<Option<Peer>, Error> {
        if self.peers.contains_key(&session_id) {
            return Err(PeerError::SessionExists(session_id).into());
        }
        let peer_id = extract_peer_id(&remote_addr).expect("opened session should have peer id");
        if self.get_key_by_peer_id(&peer_id).is_some() {
            return Err(PeerError::PeerIdExists(peer_id).into());
        }

        let is_whitelist = self.whitelist_peers.contains(&peer_id);
        let mut evicted_peer: Option<Peer> = None;

        let mut session_type: SessionType = raw_session_type.into();
        if !is_whitelist {
            if self.whitelist_only {
                return Err(PeerError::NonReserved.into());
            }
            if peer_store.is_addr_banned(&remote_addr) {
                return Err(PeerError::Banned.into());
            }

            let connection_status = self.connection_status();
            // check peers connection limitation
            if session_type.is_inbound() {
                if connection_status.non_whitelist_inbound >= self.max_inbound {
                    if let Some(evicted_session) = self.try_evict_inbound_peer(peer_store) {
                        evicted_peer = self.remove_peer(evicted_session);
                    } else {
                        return Err(PeerError::ReachMaxInboundLimit.into());
                    }
                }
            } else if connection_status.non_whitelist_outbound >= self.max_outbound {
                if self.disable_block_relay_only_connection
                    || connection_status.block_relay_only_outbound_count
                        >= self.max_outbound_block_relay
                {
                    return Err(PeerError::ReachMaxOutboundLimit.into());
                } else {
                    peer_store.add_anchors(remote_addr.clone());
                    session_type = SessionType::BlockRelayOnly;
                }
            }
        }
        peer_store.add_connected_peer(remote_addr.clone(), session_type);
        let peer = Peer::new(session_id, session_type, remote_addr, is_whitelist);
        self.peers.insert(session_id, peer);
        Ok(evicted_peer)
    }

    /// Dump address list to disk
    pub fn dump(&self, mut file: File) -> Result<(), Error> {
        let addrs: Vec<_> = self.dump_iter().collect();
        debug!("Anchors dump {} addrs", addrs.len());
        // empty file and dump the json string to it
        file.set_len(0)
            .and_then(|_| serde_json::to_string(&addrs).map_err(Into::into))
            .and_then(|json_string| file.write_all(json_string.as_bytes()))
            .and_then(|_| file.sync_all())
            .map_err(Into::into)
    }
