### Title
Stale Price Returned Without Timestamp Validation in Chainlink Compatibility Adapter — (File: `target_chains/ethereum/sdk/solidity/PythAggregatorV3.sol`)

---

### Summary

`PythAggregatorV3.sol` is Pyth's official Chainlink AggregatorV3 compatibility adapter. Every price-returning function in this contract — including `latestAnswer()`, `latestRoundData()`, and `getRoundData()` — calls `pyth.getPriceUnsafe()` without any staleness check on the returned `publishTime`. A protocol integrating this adapter as a drop-in Chainlink replacement will silently consume arbitrarily old prices, with no revert and no signal that the data is stale.

---

### Finding Description

`PythAggregatorV3.sol` wraps the Pyth oracle behind the Chainlink `AggregatorV3` interface. All five price-reading functions delegate to `getPriceUnsafe()`:

```solidity
// line 53-56
function latestAnswer() public view virtual returns (int256) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    return int256(price.price);
}

// line 99-119
function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    roundId = uint80(price.publishTime);
    return (roundId, int256(price.price), price.publishTime, price.publishTime, roundId);
}

// line 76-97
function getRoundData(uint80 _roundId) external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    return (_roundId, int256(price.price), price.publishTime, price.publishTime, _roundId);
}
```

`getPriceUnsafe()` is explicitly documented in `IPyth.sol` as returning a price "from arbitrarily far in the past" with no recency guarantee. The safe alternative, `getPriceNoOlderThan()`, exists in `AbstractPyth.sol` and enforces `diff(block.timestamp, price.publishTime) > age` before returning.

The `latestRoundData()` return value sets `updatedAt = price.publishTime`. Downstream protocols that check `updatedAt` against `block.timestamp` themselves will catch staleness, but protocols that rely on the adapter to enforce freshness — as they would with a native Chainlink feed — will not.

Additionally, `latestAnswer()` is the deprecated Chainlink function (the exact function flagged in the Blex report), and it is exposed here using `getPriceUnsafe()` with no timestamp guard.

---

### Impact Explanation

Any DeFi protocol (lending, perpetuals, options) that integrates `PythAggregatorV3` as a Chainlink drop-in and does not implement its own staleness check will use arbitrarily old prices. Concrete impacts:

- **Incorrect liquidations**: A stale low price can trigger liquidation of healthy positions.
- **Bad debt / undercollateralization**: A stale high price allows over-borrowing against collateral.
- **Arbitrage / sandwich exploitation**: An attacker observes that the on-chain price is stale relative to the true market price and opens/closes positions against the protocol at the stale price.

---

### Likelihood Explanation

- `PythAggregatorV3` is Pyth's official SDK adapter, explicitly designed for protocols migrating from Chainlink. Its purpose is to be a drop-in replacement.
- Protocols migrating from Chainlink commonly assume the adapter enforces the same freshness guarantees as a native Chainlink feed.
- Pyth's pull-oracle model means prices are only updated when a user submits an update transaction. If no one submits an update, the on-chain price can become arbitrarily stale — and `PythAggregatorV3` will return it without reverting.
- An unprivileged attacker needs only to call the victim protocol's function at a moment when the Pyth feed is stale. No privileged access, no key compromise, no external oracle manipulation is required.

---

### Recommendation

Replace `getPriceUnsafe()` with `getPriceNoOlderThan()` in all price-returning functions, and add a configurable `maxAge` parameter to the constructor:

```solidity
uint256 public maxAge;

constructor(address _pyth, bytes32 _priceId, uint256 _maxAge) {
    priceId = _priceId;
    pyth = IPyth(_pyth);
    maxAge = _maxAge;
}

function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
    // ...
}
```

This mirrors the pattern already used in `AbstractPyth.sol` (`getPriceNoOlderThan` → `diff(block.timestamp, price.publishTime) > age` → revert `StalePrice`).

---

### Proof of Concept

1. Deploy `PythAggregatorV3` pointing at a live Pyth EVM contract.
2. Allow the Pyth feed to go stale (no `updatePriceFeeds()` call for, say, 10 minutes).
3. Call `latestRoundData()` on the adapter. It returns the 10-minute-old price with no revert.
4. A lending protocol using this adapter computes collateral value from the stale price and allows over-borrowing or incorrect liquidation.
5. Compare: calling `pyth.getPriceNoOlderThan(priceId, 60)` directly on the Pyth contract would revert with `StalePrice` under the same conditions.
