### Title
Missing Staleness Check in `PythAggregatorV3` Chainlink-Compatible Adapter Returns Arbitrarily Old Prices - (`File: target_chains/ethereum/sdk/solidity/PythAggregatorV3.sol`)

---

### Summary

`PythAggregatorV3.sol` is Pyth's official Chainlink `AggregatorV3Interface`-compatible adapter, recommended for protocols migrating from Chainlink. Every price-reading function in this contract calls `pyth.getPriceUnsafe()`, which explicitly performs **no staleness check** and can return a price from arbitrarily far in the past. There is no `publishTime` vs `block.timestamp` comparison, no revert on stale data, and no configurable age threshold. Any protocol that deploys this adapter and calls `latestRoundData()`, `latestAnswer()`, or `getRoundData()` will silently receive stale prices with no on-chain protection.

---

### Finding Description

`PythAggregatorV3` is the official Pyth adapter for Chainlink-compatible applications. It is published as part of `@pythnetwork/pyth-sdk-solidity` and is actively recommended in Pyth's migration documentation.

Every price-reading function delegates to `pyth.getPriceUnsafe()`:

```solidity
// latestAnswer — no staleness check
function latestAnswer() public view virtual returns (int256) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    return int256(price.price);
}

// latestRoundData — no staleness check
function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    roundId = uint80(price.publishTime);
    return (roundId, int256(price.price), price.publishTime, price.publishTime, roundId);
}

// getRoundData — no staleness check
function getRoundData(uint80 _roundId) external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    ...
}
``` [1](#0-0) [2](#0-1) [3](#0-2) 

`getPriceUnsafe()` is explicitly documented as returning the most recent cached price with **no recency checks**:

> "This function returns the most recent price update in this contract without any recency checks. This function is unsafe as the returned price update may be arbitrarily far in the past." [4](#0-3) 

By contrast, the safe variant `getPriceNoOlderThan()` in `AbstractPyth.sol` reverts if `block.timestamp - price.publishTime > age`: [5](#0-4) 

`PythAggregatorV3` never calls this safe variant in any of its price-reading paths.

---

### Impact Explanation

Protocols that deploy `PythAggregatorV3` as a drop-in Chainlink replacement and call `latestRoundData()` or `latestAnswer()` will receive stale prices silently — the call never reverts. This is directly analogous to the BendDAO finding:

- On an L2 (Arbitrum, Optimism — both are listed as supported chains in the Pyth contract registry), if the sequencer goes down, no one can call `updateFeeds()` to push fresh price data.
- When the sequencer recovers, the on-chain Pyth cache still holds the pre-outage price.
- Any protocol using `PythAggregatorV3` will continue operating at the stale pre-outage price.
- Users can exploit this window: borrow against inflated collateral, avoid liquidation on underwater positions, or open positions at favorable stale prices.

The `updatedAt` field returned by `latestRoundData()` is set to `price.publishTime`, which could be hours old. Protocols that do check `updatedAt` for staleness will catch this, but protocols that do not (the exact vulnerability class of the original report) will be fully exposed. [6](#0-5) 

---

### Likelihood Explanation

- `PythAggregatorV3` is the **official** Pyth-provided Chainlink migration path, actively documented and recommended.
- Pyth is deployed on Arbitrum and Optimism (L2s with sequencer downtime risk).
- Protocols migrating from Chainlink are likely to call `latestRoundData()` without adding their own staleness guard, since they expect the adapter to handle it (as Chainlink's native feeds do via heartbeat).
- Price update keepers can also fail independently of the sequencer, creating the same stale-price window on any chain.
- An unprivileged user needs only to call the downstream protocol's borrow/trade function during the stale window — no special access required.

---

### Recommendation

Replace all `getPriceUnsafe()` calls in `PythAggregatorV3` with `getPriceNoOlderThan()` using a configurable `maxAge` parameter set at construction time:

```solidity
uint256 public maxAge; // e.g., 60 seconds

constructor(address _pyth, bytes32 _priceId, uint256 _maxAge) {
    priceId = _priceId;
    pyth = IPyth(_pyth);
    maxAge = _maxAge;
}

function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
    ...
}
```

This causes `latestRoundData()` to revert with `StalePrice` when the cached price is too old, matching the behavior that Chainlink-migrating protocols expect and preventing silent stale-price consumption.

---

### Proof of Concept

1. Deploy `PythAggregatorV3` on Arbitrum pointing to the ETH/USD feed.
2. Simulate the Arbitrum sequencer going down: no new `updateFeeds()` calls are possible.
3. The ETH price drops 30% off-chain during the outage.
4. Sequencer recovers. The on-chain Pyth cache still holds the pre-outage price.
5. Call `PythAggregatorV3.latestRoundData()` — it returns the stale pre-outage price with no revert.
6. A lending protocol using this adapter allows a user to borrow against the inflated stale collateral value, creating an undercollateralized position that cannot be liquidated until the price is manually updated. [1](#0-0) [2](#0-1)

        bytes32 id,
