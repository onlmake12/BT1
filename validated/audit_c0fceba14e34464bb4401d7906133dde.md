Audit Report

## Title
Unauthenticated RPC Panic via `txs_sizes.expect()` on Legacy `BlockExt` Records — (`rpc/src/util/fee_rate.rs`)

## Summary

`FeeRateCollector::statistics` unconditionally calls `.expect()` on `block_ext.txs_sizes` at line 93 of `rpc/src/util/fee_rate.rs`. Legacy on-disk `BlockExt` records (pre-v0.106, 5-field molecule schema) always deserialize with `txs_sizes: None`. Any node whose database contains such blocks will panic in the RPC handler when `get_fee_rate_statistics` or `get_fee_rate_statics` is called with a target window covering those blocks.

## Finding Description

**Panic site** — `rpc/src/util/fee_rate.rs`, line 93:

```rust
let txs_sizes = txs_sizes.expect("expect txs_size's length >= 1");
```

The `collect` closure receives every `BlockExt` returned by `get_block_ext_by_number`; the `filter_map` at lines 45–47 only skips blocks absent from the database, not blocks with `txs_sizes: None`.

**Source of `None`** — `util/types/src/conversion/storage.rs`, lines 139–150: the `Unpack<core::BlockExt>` impl for the legacy `packed::BlockExtReader` (5-field schema) unconditionally sets `txs_sizes: None`.

**Store dispatch** — `store/src/store.rs`, lines 252–253: when the stored blob has `count_extra_fields() == 0` (old format), the store returns the legacy reader, yielding `txs_sizes: None`.

**RPC entry points** — both `get_fee_rate_statics` and `get_fee_rate_statistics` call `FeeRateCollector::statistics` with no authentication requirement.

The existing guard at line 94 (`if txs_sizes.len() > 1`) is never reached because the `.expect()` at line 93 panics first when `txs_sizes` is `None`.

## Impact Explanation

The `.expect()` panic terminates the RPC handler task. With Rust's default `unwind` panic strategy the node process survives but the RPC call aborts with an error. This matches the allowed CKB bounty impact: **Note (0–500 points) — Any local RPC API crash**.

## Likelihood Explanation

Any mainnet or testnet node that synced blocks before v0.106 retains legacy `BlockExt` rows in `COLUMN_BLOCK_EXT`. The RPC is unauthenticated; calling `get_fee_rate_statistics` with a `target` window large enough to reach those blocks is sufficient to trigger the panic. No special privileges or victim interaction are required.

## Recommendation

Replace the `.expect()` with a graceful skip, consistent with the existing `cycles` `Option` handling at line 97:

```rust
let Some(txs_sizes) = txs_sizes else {
    return fee_rates; // skip legacy blocks that lack txs_sizes
};
```
