### Title
Unlimited Concurrent SSE Connections Enable DoS of Hermes Price Feed Service - (File: `apps/hermes/server/src/api/rest/v2/sse.rs`)

---

### Summary
The Hermes price feed server exposes a public SSE streaming endpoint (`/v2/updates/price/stream`) that accepts an unlimited number of concurrent long-lived connections. There is no per-IP connection count limit, no global concurrency cap, and no throttle middleware on the Axum router. An unprivileged attacker can open thousands of SSE connections simultaneously, exhausting server memory and triggering an OOM kill of the Hermes process, making price feed data unavailable to all downstream consumers.

---

### Finding Description

The Hermes Axum router in `apps/hermes/server/src/api.rs` registers the SSE handler with no concurrency-limiting middleware:

```rust
let app = app
    .route("/v2/updates/price/stream", get(rest::price_stream_sse_handler))
    // ...
    .layer(CorsLayer::permissive())
    .layer(Extension(QsQueryConfig::new(5, false)));
    // No ConcurrencyLimitLayer, no throttle, no connection count guard
``` [1](#0-0) 

The SSE handler `price_stream_sse_handler` in `apps/hermes/server/src/api/rest/v2/sse.rs` allocates per-connection state for every accepted request:

- A `BroadcastStream` receiver cloned from the global aggregation channel
- An `Arc<AtomicBool>` (`should_end`) for connection lifecycle control
- A full clone of `ApiState<S>` (containing `Arc<S>`, `Arc<WsState>`, `Arc<ApiMetrics>`)
- A `SseConnectionGuard` — which **only increments/decrements a Prometheus counter** and enforces no limit

```rust
let guard = SseConnectionGuard::new(metrics);   // metrics only, no limit
let guarded_stream = futures::stream::poll_fn(move |cx| { ... });
Ok(Sse::new(guarded_stream).keep_alive(KeepAlive::default()))
``` [2](#0-1) [3](#0-2) 

Each connection is kept alive for up to 24 hours:

```rust
const MAX_CONNECTION_DURATION: Duration = Duration::from_secs(24 * 60 * 60);
``` [4](#0-3) 

The WebSocket endpoint (`/ws`) has a bytes-per-second rate limiter per IP address (256 KiB/s), but this limits **data throughput**, not the **number of concurrent connections**. The SSE endpoint has no equivalent protection at all. [5](#0-4) [6](#0-5) 

---

### Impact Explanation

Hermes is the canonical Pyth price feed oracle service. Its `/v2/updates/price/stream` SSE endpoint is publicly documented and used by dApps, on-chain protocols, and the `HermesClient` SDK to receive real-time price updates. If the Hermes process is killed by the OS OOM killer due to memory exhaustion from thousands of open SSE connections, all price feed consumers lose access to price data. On-chain protocols that depend on Hermes for price update VAAs would be unable to submit updates, potentially freezing liquidations, trades, and other time-sensitive operations.

---

### Likelihood Explanation

High. The endpoint requires no authentication for the public Hermes instance (`https://hermes.pyth.network`). An attacker needs only to issue repeated HTTP GET requests to `/v2/updates/price/stream?ids[]=<any_valid_id>` from one or more IPs. Each connection is trivially cheap to open from the attacker's side (a single `curl -N` or `EventSource` call) but expensive to hold on the server side due to the per-connection state allocation and Tokio task overhead. The 24-hour maximum connection duration means connections accumulate rapidly.

---

### Recommendation

1. Add a global concurrent connection limit using `tower::limit::ConcurrencyLimitLayer` on the Axum router, or specifically on the SSE route.
2. Add a per-IP SSE connection count limit (analogous to the existing WS bytes rate limiter) using a `DashMap<IpAddr, AtomicUsize>` counter checked at connection entry in `price_stream_sse_handler`.
3. Enforce the limit at the handler level by checking the current active SSE connection count (already tracked in `stream_active_connections` metrics) against a configurable maximum before accepting a new connection.

---

### Proof of Concept

```bash
# Open 5000 concurrent SSE connections to exhaust server memory
for i in $(seq 1 5000); do
  curl -sN "https://hermes.pyth.network/v2/updates/price/stream?ids[]=e62df6c8b4a85fe1a67db44dc12de5db330f7ac66b72dc658afedf0f4a415b43" &
done
wait
```

Each connection is accepted without restriction, allocates a `BroadcastStream` receiver, `Arc<AtomicBool>`, cloned `ApiState`, and a Tokio task. With enough connections the Hermes process RSS grows until the OS OOM killer terminates it, making price feeds unavailable to all consumers. [3](#0-2) [1](#0-0)

impl SseConnectionGuard {
    fn new(metrics: Arc<crate::api::metrics_middleware::ApiMetrics>) -> Self {
        metrics
            .stream_active_connections
            .get_or_create(&stream_protocol_label("sse"))
            .inc();
        Self { metrics }
    }
}

impl Drop for SseConnectionGuard {
    fn drop(&mut self) {
        self.metrics
            .stream_active_connections
            .get_or_create(&stream_protocol_label("sse"))
            .dec();
    }
}

pub async fn price_stream_sse_handler<S>(
    State(state): State<ApiState<S>>,
    QsQuery(params): QsQuery<StreamPriceUpdatesQueryParams>,
) -> Result<Sse<impl Stream<Item = Result<Event, Infallible>>>, RestError>
where
    S: Aggregates,
    S: Send + Sync + 'static,
{
    let price_id_inputs: Vec<PriceIdentifier> = params.ids.into_iter().map(Into::into).collect();
    let price_ids: Vec<PriceIdentifier> =
        validate_price_ids(&state, &price_id_inputs, params.ignore_invalid_price_ids).await?;

    // Clone the update_tx receiver to listen for new price updates
    let update_rx = Aggregates::subscribe(&*state.state);

    // Convert the broadcast receiver into a Stream
    let stream = BroadcastStream::new(update_rx);

    // Set connection start time
    let start_time = Instant::now();
    let disconnect_slow_consumers = state.streaming.disconnect_slow_consumers;
    let should_end = Arc::new(AtomicBool::new(false));
    let should_end_for_chain = should_end.clone();
    let metrics = state.metrics.clone();

    let mut inner_stream = futures::stream::StreamExt::boxed(
        stream
            .take_while({
                let should_end = should_end.clone();
                move |_| {
                    !should_end.load(Ordering::Relaxed)
                        && start_time.elapsed() < MAX_CONNECTION_DURATION
                }
            })
            .then(move |message| {
                let state_clone = state.clone();
                let price_ids_clone = price_ids.clone();
                let should_end = should_end.clone();
                async move {
                    match message {
                        Ok(event) => {
                            match handle_aggregation_event(
                                event,
                                state_clone,
                                price_ids_clone,
                                params.encoding,
                                params.parsed,
                                params.benchmarks_only,
                                params.allow_unordered,
                            )
                            .await
                            {
                                Ok(Some(update)) => Some(Ok(Event::default()
                                    .json_data(update)
                                    .unwrap_or_else(error_event))),
                                Ok(None) => None,
                                Err(e) => Some(Ok(error_event(e))),
                            }
                        }
                        Err(e) => {
                            if matches!(e, BroadcastStreamRecvError::Lagged(_)) {
                                state_clone.metrics.sse_broadcast_lagged.inc();
                            }
                            if should_disconnect_slow_sse_consumer(&e, disconnect_slow_consumers) {
                                tracing::info!(
                                    "Slow consumer disconnected (SSE broadcast lagged)."
                                );
                                state_clone
                                    .metrics
                                    .stream_slow_consumer_disconnects
                                    .get_or_create(&stream_protocol_label("sse"))
                                    .inc();
                                should_end.store(true, Ordering::Relaxed);
                                Some(Ok(slow_consumer_disconnect_event()))
                            } else {
                                Some(Ok(error_event(e)))
                            }
                        }
                    }
                }
            })
            .filter_map(|x| x)
            .chain(
                futures::stream::once(async move { should_end_for_chain.load(Ordering::Relaxed) })
                    .filter_map(|ended_due_to_slow_consumer| {
                        timeout_event_if_needed(ended_due_to_slow_consumer)
                    }),
            ),
    );

    let guard = SseConnectionGuard::new(metrics);
    let guarded_stream = futures::stream::poll_fn(move |cx| {
        let _ = &guard;
        futures::stream::StreamExt::poll_next_unpin(&mut inner_stream, cx)
    });

    Ok(Sse::new(guarded_stream).keep_alive(KeepAlive::default()))
