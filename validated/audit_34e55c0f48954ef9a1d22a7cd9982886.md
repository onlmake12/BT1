### Title
`PythAggregatorV3` Returns Stale Price Without Staleness Check in All Price Query Functions - (File: `target_chains/ethereum/sdk/solidity/PythAggregatorV3.sol`)

### Summary

`PythAggregatorV3.sol` is Pyth's official Chainlink `AggregatorV3Interface`-compatible adapter. Every price-returning function in this contract calls `pyth.getPriceUnsafe(priceId)`, which explicitly makes no recency guarantees. No staleness check is performed before returning the price to callers. Any protocol that integrates `PythAggregatorV3` as a drop-in Chainlink replacement will silently consume arbitrarily stale prices.

### Finding Description

`PythAggregatorV3` exposes the standard Chainlink aggregator surface: `latestAnswer()`, `latestRoundData()`, `getRoundData()`, and `decimals()`. All four functions resolve the price by calling `pyth.getPriceUnsafe(priceId)`: [1](#0-0) [2](#0-1) [3](#0-2) 

`IPyth.getPriceUnsafe` is explicitly documented as returning a price from arbitrarily far in the past with no recency checks: [4](#0-3) 

By contrast, `AbstractPyth.getPriceNoOlderThan` enforces a staleness bound and reverts with `StalePrice` if the price is too old: [5](#0-4) 

`PythAggregatorV3` never calls `getPriceNoOlderThan` or checks `price.publishTime` against `block.timestamp` before returning. The `updatedAt` field in the `latestRoundData` return tuple is set to `price.publishTime`, which is the Pyth publish timestamp — but no on-chain assertion is made that this timestamp is recent. [6](#0-5) 

### Impact Explanation

Protocols that deploy `PythAggregatorV3` as a Chainlink-compatible oracle adapter and call `latestRoundData()` or `latestAnswer()` will receive a price that may be hours or days old with no on-chain revert or signal. Downstream effects include:

- **Incorrect liquidations**: A stale price below the liquidation threshold can trigger liquidation of positions that are actually healthy at the current market price (the Case 2 scenario from the external report).
- **Mispriced collateral**: Lending protocols that accept collateral valued at a stale price can be drained by borrowing against inflated or deflated collateral values.
- **Incorrect margin/position sizing**: Any derivative or margin protocol using this adapter for position opens or debt checks will operate on outdated data.

The impact is protocol-level fund loss for any integrator that uses `PythAggregatorV3` without independently checking `updatedAt` — which is the entire point of a Chainlink-compatible adapter: callers expect the adapter itself to enforce recency.

### Likelihood Explanation

`PythAggregatorV3` is shipped as part of Pyth's official EVM Solidity SDK and is explicitly documented as a drop-in Chainlink replacement. Protocols migrating from Chainlink to Pyth using this adapter will not add a separate staleness check because the Chainlink `AggregatorV3Interface` convention is that the adapter is responsible for that check. The Pyth network's pull-based model means the on-chain price is only as fresh as the last `updatePriceFeeds` call; if no keeper updates the feed, the price silently ages. This is a realistic scenario during network congestion or keeper downtime.

### Recommendation

Replace `getPriceUnsafe` calls in `PythAggregatorV3` with `getPriceNoOlderThan(priceId, validAge)`, where `validAge` is a configurable constructor parameter (e.g., defaulting to `pyth.getValidTimePeriod()`). If the price is stale, the functions should revert, consistent with how Chainlink consumers expect stale-price protection to work.

```solidity
uint public maxAge;

constructor(address _pyth, bytes32 _priceId, uint _maxAge) {
    priceId = _priceId;
    pyth = IPyth(_pyth);
    maxAge = _maxAge;
}

function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
    ...
}
```

### Proof of Concept

1. Deploy `PythAggregatorV3` pointing at a live Pyth EVM contract and a valid `priceId`.
2. Call `updatePriceFeeds` once to seed the price.
3. Wait longer than the Pyth valid time period (e.g., 60 seconds) without calling `updatePriceFeeds` again.
4. Call `latestRoundData()` on the adapter — it returns the stale price with no revert.
5. A lending protocol using this adapter now values collateral at the stale price, enabling an attacker to borrow against it at incorrect rates.

The root cause is entirely within `PythAggregatorV3.sol` lines 54, 89, and 110, all of which call `getPriceUnsafe` with no subsequent timestamp validation. [7](#0-6)

    function getPriceNoOlderThan(
        bytes32 id,
        uint age
    ) public view virtual override returns (PythStructs.Price memory price) {
        price = getPriceUnsafe(id);

        if (diff(block.timestamp, price.publishTime) > age)
            revert PythErrors.StalePrice();

        return price;
    }
