### Title
Delegator Loses All Accrued OIS Rewards When Calling `undelegate` Before `advance_delegation_record` - (`governance/pyth_staking_sdk/src/pyth-staking-client.ts`)

---

### Summary

In Pyth's Oracle Integrity Staking (OIS) system, a delegator who calls `undelegate` to remove all positions for a publisher before calling `advance_delegation_record` (claim) permanently loses all accrued but unclaimed rewards for that publisher. The SDK's `unstakeFromPublisher` and `unstakeFromAllPublishers` functions do not call `advance_delegation_record` first, and after all positions are removed, the SDK's reward-claiming logic permanently filters out that publisher, making the rewards unclaimable forever.

---

### Finding Description

Rewards in the OIS system are computed lazily via `advance_delegation_record`, which reads the staker's current positions in `stake_account_positions` to determine their historical delegation share and transfers accrued rewards to their custody account. The `DelegationRecord` PDA (keyed by `[publisher, stake_account_positions]`) tracks `last_epoch` — the last epoch for which rewards were settled.

The `undelegate` instruction's account list does **not** include `delegation_record`: [1](#0-0) 

This means `undelegate` cannot enforce that rewards are settled before a position is closed. Neither `unstakeFromPublisher` nor `unstakeFromAllPublishers` in the SDK calls `advance_delegation_record` before submitting `undelegate` instructions: [2](#0-1) [3](#0-2) 

After all positions for a publisher are removed, `getAdvanceDelegationRecordInstructions` filters out that publisher entirely because `lowestEpoch` becomes `undefined` (no positions found): [4](#0-3) 

The filtered publisher list is then used to build all `advance_delegation_record` instructions: [5](#0-4) 

Since `advance_delegation_record` requires the positions to reconstruct the staker's historical delegation share, and those positions are now gone, the rewards for all epochs between `last_epoch` and the current epoch are permanently unclaimable. The `DelegationRecord` PDA remains on-chain but can never be advanced to yield a non-zero reward.

The `claim` function in `apps/staking/src/api.ts` is simply a wrapper around `advanceDelegationRecord`: [6](#0-5) 

The `DelegationRecord` struct only stores `last_epoch` and `next_slash_event_index` — there is no intermediate "pending rewards" accumulator that survives position removal: [7](#0-6) 

---

### Impact Explanation

A delegator who calls `undelegate` (directly on-chain or via the SDK) to remove all their positions for a given publisher, without first calling `advance_delegation_record`, permanently loses all OIS rewards accrued since their last claim for that publisher. These tokens remain locked in the pool reward custody and are never transferred to the staker. This is a direct monetary loss proportional to the staking duration and pool yield.

---

### Likelihood Explanation

Medium. The front-end UI enforces the correct order (claim before unstake), but:
1. Users can interact directly with the on-chain program or the SDK without going through the UI.
2. The SDK's `unstakeFromPublisher` and `unstakeFromAllPublishers` are public methods that do not enforce the correct ordering.
3. Any integrator or power user calling the SDK directly will silently lose rewards.
4. The `OutdatedDelegatorAccounting` error (6008) exists in the IDL but is not enforced in `undelegate` (the account is absent), giving a false sense of safety. [8](#0-7) 

---

### Recommendation

1. **Enforce ordering on-chain**: Add `delegation_record` to the `undelegate` instruction's account list and require that `delegation_record.last_epoch == current_epoch` (i.e., rewards are settled) before allowing undelegation. Return `OutdatedDelegatorAccounting` if not.
2. **Enforce ordering in the SDK**: In `unstakeFromPublisher` and `unstakeFromAllPublishers`, prepend `advance_delegation_record` instructions for the affected publisher(s) before the `undelegate` instructions in the same transaction.

---

### Proof of Concept

1. Alice stakes 1000 PYTH to publisher P at epoch 10. `DelegationRecord` for `(P, Alice)` is initialized with `last_epoch = 10`.
2. Epochs 11–20 pass. Alice accrues OIS rewards but never calls `advance_delegation_record`. `last_epoch` remains 10.
3. Alice calls `unstakeFromPublisher(stakeAccountPositions, P, LOCKED, 1000n)` directly via the SDK. This sends only `undelegate` instructions — no `advance_delegation_record`.
4. All positions for publisher P are removed from `stake_account_positions`.
5. Alice calls `claim` (→ `advanceDelegationRecord`). `getAdvanceDelegationRecordInstructions` iterates all publishers, finds no positions for P (`lowestEpoch = undefined`), and filters P out. No `advance_delegation_record` instruction is generated for P.
6. Alice's rewards for epochs 11–20 from publisher P are permanently lost. The `DelegationRecord` PDA still exists with `last_epoch = 10` but will never be advanced. [4](#0-3) [9](#0-8)

  public async unstakeFromPublisher(
    stakeAccountPositions: PublicKey,
    publisher: PublicKey,
    positionState: PositionState.LOCKED | PositionState.LOCKING,
    amount: bigint,
  ) {
    const stakeAccountPositionsData = await this.getStakeAccountPositions(
      stakeAccountPositions,
    );
    const currentEpoch = await getCurrentEpoch(this.connection);

    let remainingAmount = amount;
    const instructionPromises: Promise<TransactionInstruction>[] = [];

    const eligiblePositions = stakeAccountPositionsData.data.positions
      .map((p, i) => ({ index: i, position: p }))
      .reverse()
      .filter(
        ({ position }) =>
          position.targetWithParameters.integrityPool?.publisher !==
            undefined &&
          position.targetWithParameters.integrityPool.publisher.equals(
            publisher,
          ) &&
          positionState === getPositionState(position, currentEpoch),
      );

    for (const { position, index } of eligiblePositions) {
      if (position.amount < remainingAmount) {
        instructionPromises.push(
          this.integrityPoolProgram.methods
            .undelegate(index, convertBigIntToBN(position.amount))
            .accounts({
              publisher,
              stakeAccountPositions,
            })
            .instruction(),
        );
        remainingAmount -= position.amount;
      } else {
        instructionPromises.push(
          this.integrityPoolProgram.methods
            .undelegate(index, convertBigIntToBN(remainingAmount))
            .accounts({
              publisher,
              stakeAccountPositions,
            })
            .instruction(),
        );
        break;
      }
    }

    const instructions = await Promise.all(instructionPromises);
    return sendTransaction(instructions, this.connection, this.wallet);
  }

  public async unstakeFromAllPublishers(
    stakeAccountPositions: PublicKey,
    positionStates: (PositionState.LOCKED | PositionState.LOCKING)[],
  ) {
    const [stakeAccountPositionsData, currentEpoch] = await Promise.all([
      this.getStakeAccountPositions(stakeAccountPositions),
      getCurrentEpoch(this.connection),
    ]);

    const instructions = await Promise.all(
      stakeAccountPositionsData.data.positions
        .map((position, index) => {
          const publisher =
            position.targetWithParameters.integrityPool?.publisher;
          return publisher === undefined
            ? undefined
            : { index, position, publisher };
        })
        // By separating this filter from the next, typescript can narrow the
        // type and automatically infer that there will be no `undefined` values
        // in the array after this line.  If we combine those filters,
        // typescript won't narrow properly.
        .filter((positionInfo) => positionInfo !== undefined)
        .filter(({ position }) =>
          (positionStates as PositionState[]).includes(
            getPositionState(position, currentEpoch),
          ),
        )
        .reverse()
        .map(({ position, index, publisher }) =>
          this.integrityPoolProgram.methods
            .undelegate(index, convertBigIntToBN(position.amount))
            .accounts({ publisher, stakeAccountPositions })
            .instruction(),
        ),
    );

    return sendTransaction(instructions, this.connection, this.wallet);
  }
