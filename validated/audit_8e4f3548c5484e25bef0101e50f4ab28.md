### Title
Single-Step Ownership Transfer in `PythLazer` Allows Irrecoverable Loss of Contract Control — (File: `lazer/contracts/evm/src/PythLazer.sol`)

---

### Summary

`PythLazer.sol` inherits from `OwnableUpgradeable` (single-step) instead of `Ownable2StepUpgradeable`. A single call to `transferOwnership()` immediately and irrevocably transfers full contract control to the supplied address. If the owner accidentally provides a wrong address, that address instantly gains the ability to add or remove trusted Lazer signers and upgrade the contract, with no recovery path. This is inconsistent with every other upgradeable contract in the same codebase, which all use the two-step pattern.

---

### Finding Description

`PythLazer` is declared as:

```solidity
contract PythLazer is OwnableUpgradeable, UUPSUpgradeable {
``` [1](#0-0) 

`OwnableUpgradeable` from OpenZeppelin implements a **single-step** `transferOwnership()`: the new owner is set immediately in the same transaction, with no acceptance step required from the recipient.

The owner of `PythLazer` controls two irreversible, protocol-critical functions:

1. **`updateTrustedSigner(address trustedSigner, uint256 expiresAt)`** — adds or removes the set of signers whose signatures are accepted as valid Lazer price attestations.
2. **`_authorizeUpgrade(address)`** — authorises arbitrary contract upgrades via UUPS. [2](#0-1) 

The operational script `PythLazerChangeOwnership.s.sol` confirms that ownership transfers are a planned, recurring procedure:

```solidity
lazer.transferOwnership(NEW_OWNER);
``` [3](#0-2) 

By contrast, every other upgradeable contract in the same repository uses `Ownable2StepUpgradeable`, which requires the new owner to call `acceptOwnership()` before the transfer completes. For example, `EntropyUpgradable`:

```solidity
import "@openzeppelin/contracts-upgradeable/access/Ownable2StepUpgradeable.sol";
...
contract EntropyUpgradable is Initializable, Ownable2StepUpgradeable, UUPSUpgradeable, ...
``` [4](#0-3) 

`PythLazer` is the sole production contract in the repository that omits this protection.

---

### Impact Explanation

If the owner accidentally calls `transferOwnership(wrongAddress)`:

- `wrongAddress` **immediately** becomes the sole owner with no pending state.
- `wrongAddress` can call `updateTrustedSigner(maliciousSigner, farFutureTimestamp)` to register an attacker-controlled signer as trusted.
- All downstream consumers of `PythLazer.verifyUpdate()` will then accept price payloads signed by the malicious signer as authentic Lazer data.
- `wrongAddress` can also call `upgradeTo` / `upgradeToAndCall` to replace the implementation with arbitrary logic, draining all ETH held in the contract and corrupting all state.
- The original owner has **no recovery mechanism**: there is no `pendingOwner` to cancel, no timelock, and no admin override. [5](#0-4) 

---

### Likelihood Explanation

The `PythLazerChangeOwnership.s.sol` script shows that ownership transfers are a documented, scripted operational step performed by a human operator supplying `NEW_OWNER` via an environment variable. A copy-paste error, clipboard hijack, or simple typo in the address is a realistic and well-documented class of operational mistake. The absence of a two-step guard means there is zero tolerance for such errors. [6](#0-5) 

---

### Recommendation

Replace `OwnableUpgradeable` with `Ownable2StepUpgradeable` in `PythLazer.sol`, consistent with the pattern already enforced in `EntropyUpgradable`, `SchedulerUpgradeable`, `ExecutorUpgradable`, and `PythUpgradable`. The new owner must call `acceptOwnership()` to complete the transfer, ensuring that a wrong address can never silently acquire control. [7](#0-6) 

---

### Proof of Concept

1. Current owner initiates an ownership transfer to a governance executor contract but supplies a wrong address (e.g., a typo or stale address from clipboard):
   ```solidity
   lazer.transferOwnership(0xDeAdBeEf...); // wrong address
   ```
2. `OwnableUpgradeable.transferOwnership` immediately writes `_owner = 0xDeAdBeEf...`. No pending state exists; the transfer is final.
3. The attacker controlling `0xDeAdBeEf...` calls:
   ```solidity
   lazer.updateTrustedSigner(attackerSigner, block.timestamp + 365 days);
   ```
4. `attackerSigner` is now a valid signer. Any Lazer price payload signed by `attackerSigner` passes `verifyUpdate()` and is accepted as authentic by all on-chain consumers.
5. Alternatively, the attacker calls `lazer.upgradeToAndCall(maliciousImpl, "")` to replace the entire contract logic.
6. The original owner has no recourse: `transferOwnership` has no cancellation, no timelock, and no two-step acceptance. [8](#0-7)

    function updateTrustedSigner(
        address trustedSigner,
        uint256 expiresAt
    ) external onlyOwner {
        if (expiresAt == 0) {
            for (uint8 i = 0; i < trustedSigners.length; i++) {
                if (trustedSigners[i].pubkey == trustedSigner) {
                    trustedSigners[i].pubkey = address(0);
                    trustedSigners[i].expiresAt = 0;
                    delete trustedSignerToExpiresAtMapping[trustedSigner];
                    return;
                }
            }
            revert("no such pubkey");
        } else {
            for (uint8 i = 0; i < trustedSigners.length; i++) {
                if (trustedSigners[i].pubkey == trustedSigner) {
                    trustedSigners[i].expiresAt = expiresAt;
                    trustedSignerToExpiresAtMapping[trustedSigner] = expiresAt;
                    return;

        if (msg.value > verification_fee) {

            revert("input too short");

        if (evm_magic != EVM_FORMAT_MAGIC) {
