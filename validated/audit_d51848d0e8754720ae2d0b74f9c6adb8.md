### Title
Provider Fees Permanently Stuck in Echo Contract Due to Missing Direct Provider Withdrawal Function — (`target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary
The `Echo` contract accumulates fees for providers in `_state.providers[providerToCredit].accruedFeesInWei` during `executeCallback()`, but the **only** withdrawal path is `withdrawAsFeeManager()`, which requires `msg.sender == feeManager`. Since `feeManager` defaults to `address(0)` upon registration and no direct `withdraw()` function exists for providers, any provider that does not explicitly set a fee manager has their accrued fees permanently locked in the contract.

### Finding Description

When a callback is executed, provider fees are credited: [1](#0-0) 

The only mechanism to withdraw those provider fees is `withdrawAsFeeManager()`: [2](#0-1) 

This function enforces:
```solidity
require(
    msg.sender == _state.providers[provider].feeManager,
    "Only fee manager"
);
```

When a provider registers via `registerProvider()`, the `feeManager` field in `ProviderInfo` is never initialized and defaults to `address(0)`: [3](#0-2) [4](#0-3) 

Since `msg.sender` can never equal `address(0)`, `withdrawAsFeeManager()` will always revert for any provider that has not called `setFeeManager()`. There is no alternative `withdraw()` function for providers to withdraw their own fees directly. The `withdrawFees()` function is exclusively for the admin to withdraw Pyth protocol fees (`_state.accruedFeesInWei`), not provider fees: [5](#0-4) 

The `IEcho` interface confirms there is no provider-facing `withdraw()` function: [6](#0-5) 

By contrast, the Entropy contract correctly provides a `withdraw()` function allowing providers to withdraw their own fees directly as `msg.sender`: [7](#0-6) 

### Impact Explanation
Any provider who registers and accumulates fees without explicitly calling `setFeeManager()` has their entire `accruedFeesInWei` balance permanently locked in the Echo contract with no recovery path. This is a direct loss of funds for the provider.

### Likelihood Explanation
Setting a fee manager is optional and not enforced at registration time. A provider operating without a fee manager (the default state) will silently accumulate fees they can never retrieve. This is a realistic scenario for any provider who is unaware of the requirement or who intends to manage fees directly.

### Recommendation
Add a `withdraw(uint128 amount)` function to `Echo.sol` that allows a registered provider (`msg.sender`) to withdraw their own accrued fees directly, mirroring the pattern in `Entropy.sol`:

```solidity
function withdraw(uint128 amount) external {
    ProviderInfo storage provider = _state.providers[msg.sender];
    require(provider.isRegistered, "Provider not registered");
    require(provider.accruedFeesInWei >= amount, "Insufficient balance");
    provider.accruedFeesInWei -= amount;
    (bool sent, ) = msg.sender.call{value: amount}("");
    require(sent, "Failed to send fees");
    emit FeesWithdrawn(msg.sender, amount);
}
```

### Proof of Concept

```solidity
function testProviderFeesStuck() public {
    // 1. Provider registers without setting a fee manager
    vm.prank(defaultProvider);
    echo.registerProvider(1000, 100, 10);
    // feeManager defaults to address(0)

    // 2. User makes a request and provider executes callback, accruing fees
    bytes32[] memory priceIds = createPriceIds();
    vm.deal(address(consumer), 1 gwei);
    vm.prank(address(consumer));
    uint64 seq = echo.requestPriceUpdatesWithCallback{value: calculateTotalFee()}(
        defaultProvider, uint64(block.timestamp), priceIds, CALLBACK_GAS_LIMIT
    );
    // ... provider executes callback, fees accumulate in accruedFeesInWei

    // 3. Provider has no way to withdraw — no withdraw() function exists
    // withdrawAsFeeManager reverts because feeManager == address(0)
    vm.prank(defaultProvider);
    vm.expectRevert("Only fee manager");
    echo.withdrawAsFeeManager(defaultProvider, 1);

    // Fees are permanently stuck
    EchoState.ProviderInfo memory info = echo.getProviderInfo(defaultProvider);
    assertGt(info.accruedFeesInWei, 0); // fees exist but cannot be retrieved
}
```
