### Title
Missing Zero-Value Price ID Validation in `_validateSubscriptionParams()` Allows Subscriptions With Invalid Price IDs, Locking Funds — (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`Scheduler._validateSubscriptionParams()` validates price ID array length, uniqueness, and update criteria, but never checks that individual price IDs are non-zero (`bytes32(0)`). A user can create a subscription (including a **permanent** one) containing `bytes32(0)` as a price ID. Every subsequent `updatePriceFeeds()` call will revert because the Pyth oracle contract rejects zero price IDs. For permanent subscriptions, `updateSubscription()` is also blocked, permanently locking the deposited ETH.

---

### Finding Description

`_validateSubscriptionParams()` performs the following checks on `params.priceIds`: [1](#0-0) 

It checks for empty arrays, exceeding `MAX_PRICE_IDS_PER_SUBSCRIPTION`, and duplicates. **There is no check that any individual price ID is non-zero.**

A user calling `createSubscription()` with `priceIds = [bytes32(0)]` passes all validation and deposits ETH: [2](#0-1) 

When `updatePriceFeeds()` is later called, it forwards `params.priceIds` (including `bytes32(0)`) to `pyth.parsePriceFeedUpdatesWithConfig()`: [3](#0-2) 

The Pyth contract enforces that every requested price ID must be found in the update data: [4](#0-3) 

`bytes32(0)` will never appear in real Pyth update data, so this call always reverts with `PriceFeedNotFoundWithinRange`. The subscription can never be updated.

For **permanent subscriptions**, `updateSubscription()` is explicitly blocked: [5](#0-4) 

The deposited ETH (up to `MAX_DEPOSIT_LIMIT`) is permanently locked with no recovery path.

---

### Impact Explanation

A user who creates a permanent subscription with a zero price ID loses all deposited ETH permanently. For non-permanent subscriptions, the subscription is broken (no price updates can ever be stored), and the balance is inaccessible unless the user calls `updateSubscription()` to replace the zero price ID — but no warning or revert prevents the initial mistake. This is directly analogous to the reference report: missing input validation allows an invalid parameter to pass through to downstream processing that cannot handle it, freezing funds.

---

### Likelihood Explanation

Any unprivileged user calling `createSubscription()` or `updateSubscription()` can trigger this. A developer integrating the Scheduler programmatically could accidentally pass an uninitialized `bytes32` variable (which defaults to zero in Solidity) as a price ID. The path is fully permissionless and requires no special role.

---

### Recommendation

Add a non-zero check for each price ID inside `_validateSubscriptionParams()`:

```solidity
for (uint i = 0; i < params.priceIds.length; i++) {
    if (params.priceIds[i] == bytes32(0)) {
        revert SchedulerErrors.InvalidPriceId(bytes32(0), bytes32(0));
    }
    for (uint j = i + 1; j < params.priceIds.length; j++) {
        if (params.priceIds[i] == params.priceIds[j]) {
            revert SchedulerErrors.DuplicatePriceId(params.priceIds[i]);
        }
    }
}
``` [6](#0-5) 

---

### Proof of Concept

1. Deploy `Scheduler` on a testnet with a live Pyth contract.
2. Call `createSubscription({ priceIds: [bytes32(0)], isPermanent: true, ... })` with `msg.value >= minimumBalance`. The call succeeds and ETH is deposited.
3. Call `updatePriceFeeds(subscriptionId, validUpdateData)`. The call reverts with `PriceFeedNotFoundWithinRange` from the Pyth contract because `bytes32(0)` is not a valid price ID.
4. Call `updateSubscription(subscriptionId, newParams)`. The call reverts with `PermanentSubscription`.
5. The deposited ETH is permanently locked with no recovery path. [7](#0-6) [8](#0-7)

    function _validateSubscriptionParams(
        SchedulerStructs.SubscriptionParams memory params
    ) internal pure {
        // No zero‐feed subscriptions
        if (params.priceIds.length == 0) {
            revert SchedulerErrors.EmptyPriceIds();
        }

        // Price ID limits and uniqueness
        if (
            params.priceIds.length >
            SchedulerConstants.MAX_PRICE_IDS_PER_SUBSCRIPTION
        ) {
            revert SchedulerErrors.TooManyPriceIds(
                params.priceIds.length,
                SchedulerConstants.MAX_PRICE_IDS_PER_SUBSCRIPTION
            );
        }
        for (uint i = 0; i < params.priceIds.length; i++) {
            for (uint j = i + 1; j < params.priceIds.length; j++) {
                if (params.priceIds[i] == params.priceIds[j]) {
                    revert SchedulerErrors.DuplicatePriceId(params.priceIds[i]);
                }
            }
        }

        // Whitelist size limit and uniqueness
        if (params.readerWhitelist.length > MAX_READER_WHITELIST_SIZE) {
            revert SchedulerErrors.TooManyWhitelistedReaders(
                params.readerWhitelist.length,
                MAX_READER_WHITELIST_SIZE
            );
        }
        for (uint i = 0; i < params.readerWhitelist.length; i++) {
            for (uint j = i + 1; j < params.readerWhitelist.length; j++) {
                if (params.readerWhitelist[i] == params.readerWhitelist[j]) {
                    revert SchedulerErrors.DuplicateWhitelistAddress(
                        params.readerWhitelist[i]
                    );
                }
            }
        }

        // Validate update criteria
        if (
            !params.updateCriteria.updateOnHeartbeat &&
            !params.updateCriteria.updateOnDeviation
        ) {
            revert SchedulerErrors.InvalidUpdateCriteria();
        }
        if (
            params.updateCriteria.updateOnHeartbeat &&
            params.updateCriteria.heartbeatSeconds == 0
        ) {
            revert SchedulerErrors.InvalidUpdateCriteria();
        }
        if (
            params.updateCriteria.updateOnDeviation &&
            params.updateCriteria.deviationThresholdBps == 0
        ) {
            revert SchedulerErrors.InvalidUpdateCriteria();
        }
    }

    function updatePriceFeeds(
        uint256 subscriptionId,
        bytes[] calldata updateData
    ) external override {
        uint256 startGas = gasleft();

        SchedulerStructs.SubscriptionStatus storage status = _state
            .subscriptionStatuses[subscriptionId];
        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        if (!params.isActive) {
            revert SchedulerErrors.InactiveSubscription();
        }

        // Get the Pyth contract and parse price updates
        IPyth pyth = IPyth(_state.pyth);
        uint256 pythFee = pyth.getUpdateFee(updateData);

        // If we don't have enough balance, revert
        if (status.balanceInWei < pythFee) {
            revert SchedulerErrors.InsufficientBalance();
        }

        // Parse the price feed updates with an acceptable timestamp range of [0, now+10s].
        // Note: We don't want to reject update data if it contains a price
        // from a market that closed a few days ago, since it will contain a timestamp
        // from the last trading period. Thus, we use a minimum timestamp of zero while parsing,
        // and we enforce the past max validity ourselves in _validateShouldUpdatePrices using
        // the highest timestamp in the update data.
        status.balanceInWei -= pythFee;
        status.totalSpent += pythFee;
        uint64 curTime = SafeCast.toUint64(block.timestamp);
        (
            PythStructs.PriceFeed[] memory priceFeeds,
            uint64[] memory slots
        ) = pyth.parsePriceFeedUpdatesWithConfig{value: pythFee}(
                updateData,
                params.priceIds,
                0, // We enforce the past max validity ourselves in _validateShouldUpdatePrices
                curTime + FUTURE_TIMESTAMP_MAX_VALIDITY_PERIOD,
                false,
                true,
                false
            );

        // Verify all price feeds have the same Pythnet slot.
        // All feeds in a subscription must be updated at the same time.
        uint64 slot = slots[0];
        for (uint8 i = 1; i < slots.length; i++) {
            if (slots[i] != slot) {
                revert SchedulerErrors.PriceSlotMismatch();
            }
        }

        // Verify that update conditions are met, and that the timestamp
        // is more recent than latest stored update's. Reverts if not.
        uint256 latestPublishTime = _validateShouldUpdatePrices(
            subscriptionId,
            params,
            status,
            priceFeeds
        );

        // Update status and store the updates
        status.priceLastUpdatedAt = latestPublishTime;
        status.totalUpdates += priceFeeds.length;

        _storePriceUpdates(subscriptionId, priceFeeds);

        _processFeesAndPayKeeper(status, startGas, params.priceIds.length);

        emit PricesUpdated(subscriptionId, latestPublishTime);
    }
