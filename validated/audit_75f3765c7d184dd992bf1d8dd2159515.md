### Title
Immutable Treasury Address in Pyth Lazer Aptos Contract Cannot Be Updated After Initialization — (File: `lazer/contracts/aptos/sources/pyth_lazer.move`)

---

### Summary

The `pyth_lazer` Aptos contract stores the fee `treasury` address in its `Storage` struct, set once during `initialize()`. No admin-gated function exists to update this address post-deployment. All fees collected via `verify_message_with_funder()` and `verify_message()` are permanently routed to the original treasury address. If that account is compromised, the protocol has no on-chain recovery path — all user-paid fees flow to the attacker until the contract is redeployed.

---

### Finding Description

The `Storage` struct in `pyth_lazer.move` holds three fields:

```move
struct Storage has key {
    treasury: address,
    single_update_fee: u64,
    trusted_signers: vector<TrustedSignerInfo>
}
``` [1](#0-0) 

The `treasury` field is assigned exactly once, inside `initialize()`, which is a one-time operation enforced by Move's `move_to` semantics:

```move
public entry fun initialize(
    account: &signer, admin: address, treasury: address
) {
    assert!(signer::address_of(account) == @pyth_lazer, ENO_PERMISSIONS);
    let storage = Storage {
        treasury,
        ...
    };
    move_to(account, storage);
}
``` [2](#0-1) 

Every fee payment in both public entry points deposits directly to `storage.treasury`:

```move
coin::deposit(storage.treasury, fee);
``` [3](#0-2) 

The only admin-gated mutation function in the entire contract is `update_trusted_signer()`, which modifies `trusted_signers` only: [4](#0-3) 

There is no `set_treasury()`, `update_treasury()`, or any equivalent function. The `AdminCapability` holder has no on-chain path to redirect fees.

---

### Impact Explanation

If the treasury account's private key is compromised, every subsequent call to `verify_message_with_funder()` or `verify_message()` by any user will transfer their fee directly to the attacker-controlled address. The protocol cannot stop this without a full contract redeployment. All accumulated and future fee revenue is at permanent risk.

**Impact: Medium** — Direct, irreversible financial loss of protocol fee revenue with no on-chain mitigation path.

---

### Likelihood Explanation

Exploitation requires the treasury private key to be compromised (phishing, infrastructure breach, insider threat). This is a realistic operational risk for any long-lived production contract, especially one where the treasury address is set at deploy time and never rotated.

**Likelihood: Low** — Requires an external key compromise event, but the contract's inability to respond is the root cause.

---

### Recommendation

Add an admin-gated treasury update function:

```move
public entry fun set_treasury(
    account: &signer, new_treasury: address
) acquires Storage {
    assert!(
        exists<AdminCapability>(signer::address_of(account)),
        ENO_PERMISSIONS
    );
    let storage = borrow_global_mut<Storage>(@pyth_lazer);
    storage.treasury = new_treasury;
}
```

This mirrors the existing `update_trusted_signer()` pattern and gives the admin an on-chain recovery path without requiring redeployment.

---

### Proof of Concept

1. Contract is deployed with `treasury = @0xALICE`.
2. Alice's private key is compromised by an attacker.
3. Admin holds `AdminCapability` but has no function to call that changes `storage.treasury`.
4. Any user calling `verify_message_with_funder(user, message, sig, pubkey)` triggers:
   ```move
   let fee = coin::withdraw<AptosCoin>(account, storage.single_update_fee);
   verify_message(message, signature, trusted_signer, fee);
   // inside verify_message:
   coin::deposit(storage.treasury, fee); // → attacker's address
   ```
5. All fees are permanently diverted to the attacker until the contract is redeployed and users migrate. [5](#0-4) [6](#0-5)

    /// Upsert a trusted signer's information or remove them
    public entry fun update_trusted_signer(
        account: &signer, trusted_signer: vector<u8>, expires_at: u64
    ) acquires Storage {
        // Verify admin capability
        assert!(
            exists<AdminCapability>(signer::address_of(account)),
            ENO_PERMISSIONS
        );

        assert!(
            vector::length(&trusted_signer) == ED25519_PUBLIC_KEY_LENGTH,
            EINVALID_SIGNER
        );

        let storage = borrow_global_mut<Storage>(@pyth_lazer);
        let num_signers = storage.trusted_signers.length();
        let i = 0;
        let found = false;

        while (i < num_signers) {
            let signer_info = vector::borrow(&storage.trusted_signers, (i as u64));
            if (&signer_info.pubkey == &trusted_signer) {
                found = true;
                break
            };
            i = i + 1;
        };

        if (expires_at == 0) {
            // Remove signer
            assert!(found, ENO_SUCH_PUBKEY);
            vector::remove(&mut storage.trusted_signers, (i as u64));
        } else if (found) {
            // Update existing signer
            let signer_info = vector::borrow_mut(&mut storage.trusted_signers, (i as u64));
            signer_info.expires_at = expires_at;
        } else {
            // Add new signer
            vector::push_back(
                &mut storage.trusted_signers,
                TrustedSignerInfo { pubkey: trusted_signer, expires_at }
            );
        };
    }
