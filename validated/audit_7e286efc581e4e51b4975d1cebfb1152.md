### Title
Unbounded Provider Fee Setting Causes Arithmetic Overflow DoS in Entropy Contract - (File: `target_chains/ethereum/contracts/contracts/entropy/Entropy.sol`)

### Summary
The `setProviderFee` function in `Entropy.sol` allows any registered Entropy provider to set their `feeInWei` to an arbitrary `uint128` value with no upper or lower bound. Setting the fee to `type(uint128).max` causes an arithmetic overflow in `getFeeV2`, which reverts all fee queries and all randomness requests directed at that provider.

### Finding Description
`setProviderFee` (line 810–827) accepts any `uint128` value and writes it directly to `provider.feeInWei` with no validation:

```solidity
function setProviderFee(uint128 newFeeInWei) external override {
    EntropyStructsV2.ProviderInfo storage provider = _state.providers[msg.sender];
    if (provider.sequenceNumber == 0) revert EntropyErrors.NoSuchProvider();
    uint128 oldFeeInWei = provider.feeInWei;
    provider.feeInWei = newFeeInWei;   // ← no bounds check
    ...
}
``` [1](#0-0) 

The same absence of bounds applies to `register` (line 129) and `setProviderFeeAsFeeManager` (line 846). [2](#0-1) 

`getFeeV2(address provider, uint32 gasLimit)` computes the total fee as:

```solidity
return getProviderFee(provider, gasLimit) + _state.pythFeeInWei;
``` [3](#0-2) 

Both operands are `uint128`. When `provider.feeInWei = type(uint128).max` and `_state.pythFeeInWei > 0` (as is the case in any production deployment with a non-zero Pyth protocol fee), this addition overflows and reverts under Solidity 0.8+ checked arithmetic.

`requestHelper` calls `getFeeV2` unconditionally at line 234:

```solidity
uint128 requiredFee = getFeeV2(provider, callbackGasLimit);
``` [4](#0-3) 

Because `requestHelper` is the shared entry point for `request`, `requestWithCallback`, and `requestV2`, every randomness request to the affected provider reverts.

### Impact Explanation
A provider who sets `feeInWei = type(uint128).max` causes:

1. `getFeeV2(provider, *)` to revert with an arithmetic overflow — users cannot query the fee.
2. Every call to `request`, `requestWithCallback`, and `requestV2` targeting that provider to revert — complete DoS of randomness service for that provider.
3. If the affected provider is the **default provider**, the zero-argument `getFeeV2()` and `getFeeV2(gasLimit)` overloads (lines 750–758) also revert, breaking any integration that calls the default fee getter. [5](#0-4) 

Even without overflow (i.e., `pythFeeInWei == 0`), a fee of `type(uint128).max` is economically unpayable, achieving the same DoS outcome.

### Likelihood Explanation
Provider registration is fully permissionless — any address may call `register()` with arbitrary parameters and immediately become a valid provider. [6](#0-5) 

After registration, `setProviderFee` is callable by `msg.sender` with no further access control beyond the `sequenceNumber != 0` check. The attack requires only two transactions and zero privileged access.

### Recommendation
Add an upper bound to `setProviderFee`, `setProviderFeeAsFeeManager`, and `register`:

```solidity
uint128 constant MAX_PROVIDER_FEE = 1 ether; // or a governance-configurable cap

function setProviderFee(uint128 newFeeInWei) external override {
    require(newFeeInWei <= MAX_PROVIDER_FEE, "Fee exceeds maximum");
    ...
}
```

Alternatively, perform the `getProviderFee + pythFeeInWei` addition with an overflow-safe check (e.g., OpenZeppelin `SafeCast` or an explicit `require`) so that an extreme provider fee cannot cause `getFeeV2` to revert for unrelated callers.

### Proof of Concept

```solidity
// 1. Attacker registers as a provider
entropy.register(
    type(uint128).max,   // feeInWei — no upper bound enforced
    commitment,
    hex"",
    100,
    hex""
);

// 2. (Optional) Attacker updates fee after registration
entropy.setProviderFee(type(uint128).max);

// 3. Any user querying the fee for this provider reverts
//    (uint128 overflow when pythFeeInWei > 0)
entropy.getFeeV2(attacker, 0);  // reverts: arithmetic overflow

// 4. Any randomness request to this provider reverts
entropy.requestWithCallback{value: ...}(attacker, userCommitment);
// reverts inside requestHelper at: getFeeV2(provider, callbackGasLimit)
``` [7](#0-6)

    function setProviderFeeAsFeeManager(
        address provider,
        uint128 newFeeInWei
    ) external override {
        EntropyStructsV2.ProviderInfo storage providerInfo = _state.providers[
            provider
        ];

        if (providerInfo.sequenceNumber == 0) {
            revert EntropyErrors.NoSuchProvider();
        }

        if (providerInfo.feeManager != msg.sender) {
            revert EntropyErrors.Unauthorized();
        }

        uint128 oldFeeInWei = providerInfo.feeInWei;
        providerInfo.feeInWei = newFeeInWei;

        emit ProviderFeeUpdated(provider, oldFeeInWei, newFeeInWei);
        emit EntropyEventsV2.ProviderFeeUpdated(
            provider,
            oldFeeInWei,
            newFeeInWei,
            bytes("")
        );
    }
