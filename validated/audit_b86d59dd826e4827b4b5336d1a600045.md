Audit Report

## Title
`MaturityVerifier::verify()` Skips Cellbase Maturity Check on `resolved_dep_groups`, Allowing Consensus Rule Bypass via DepGroup Path — (`File: verification/src/transaction_verifier.rs`)

## Summary
`MaturityVerifier::verify()` enforces the cellbase maturity rule on `resolved_inputs` and `resolved_cell_deps` but never iterates `resolved_dep_groups`. Because `resolve_transaction_dep` routes the container cell of any `DepType::DepGroup` dependency exclusively into `resolved_dep_groups`, a miner can reference an immature cellbase output as a dep group container and have the transaction accepted without triggering `CellbaseImmaturity`. This is a concrete consensus rule bypass reachable by any miner.

## Finding Description
`MaturityVerifier::verify()` defines a `cellbase_immature` closure and applies it only to `resolved_inputs` and `resolved_cell_deps`, then returns `Ok(())`: [1](#0-0) 

`ResolvedTransaction` holds three distinct `Vec<CellMeta>` fields — `resolved_cell_deps`, `resolved_inputs`, and `resolved_dep_groups` — but only the first two are checked: [2](#0-1) 

In `resolve_transaction_dep`, when `dep_type == DepGroup`, the container cell (the actual out-point declared in `cell_deps`) is pushed to `resolved_dep_groups` (line 832), while only the sub-cells go to `resolved_cell_deps` (line 830): [3](#0-2) 

This means the container cell — which is the actual out-point declared in the transaction's `cell_deps` field — is never seen by `MaturityVerifier`. The error type documentation explicitly states the rule must cover dependency out-points: [4](#0-3) 

**Exploit flow:**
1. Miner produces a block whose cellbase output at index ≥ 1 has `data` = a valid molecule-encoded `OutPointVec` pointing to any live cells.
2. Before `block_epoch + cellbase_maturity` is reached, the miner submits a transaction with `cell_deps: [{ out_point: <cellbase_txhash, 1>, dep_type: "dep_group" }]`.
3. `resolve_transaction_dep` resolves the cellbase as the dep group container → `resolved_dep_groups`; sub-cells (mature) → `resolved_cell_deps`.
4. `MaturityVerifier::verify()` checks `resolved_cell_deps` (sub-cells, mature → passes) and `resolved_inputs` (unrelated → passes). `resolved_dep_groups` is never checked.
5. The transaction is admitted to the pool and committed, violating the cellbase maturity consensus rule.

The asymmetry is confirmed: submitting the same cellbase output with `dep_type: "code"` routes it to `resolved_cell_deps` and correctly triggers `CellbaseImmaturity(CellDeps[0])`.

## Impact Explanation
The cellbase maturity rule is a consensus rule. All current unpatched nodes share the same missing check, so a bypass transaction is committed everywhere. If any node is patched to enforce the rule on `resolved_dep_groups`, it will reject transactions that unpatched nodes accept, causing a consensus split. Even without a version split, the committed transaction carries a dependency on a cellbase still within the reorg window; a reorg of the cellbase block invalidates the dependent transaction, producing unexpected rollbacks. This matches the allowed critical impact: **consensus deviation**.

## Likelihood Explanation
Mining a single block is a normal, permissionless activity. No majority hashpower is required. The miner crafts a cellbase with valid `OutPointVec` data (any molecule-encoded list of live out-points), then immediately submits the bypass transaction. The attack is deterministic, requires no cryptographic break, no privileged key, and no victim mistake. It is repeatable every time the attacker mines a block.

## Recommendation
Add a third maturity check in `MaturityVerifier::verify()` covering `resolved_dep_groups`, mirroring the existing pattern:

```rust
if let Some(index) = self
    .transaction
    .resolved_dep_groups
    .iter()
    .position(cellbase_immature)
{
    return Err(TransactionError::CellbaseImmaturity {
        inner: TransactionErrorSource::CellDeps,
        index,
    }
    .into());
}
```

Optionally, add a `DepGroups` variant to `TransactionErrorSource` for diagnostic precision, and update the error type comment at `util/types/src/core/error.rs` line 168 to reflect the corrected coverage.

## Proof of Concept
1. Configure a CKB devnet with `cellbase_maturity = 4` epochs.
2. Mine a block whose cellbase output at index 1 has `data` = molecule-encoded `OutPointVec` containing the out-point of any live system cell.
3. Within the same epoch, submit a transaction: `cell_deps: [{ out_point: <cellbase_txhash, 1>, dep_type: "dep_group" }]`, with any spendable input and valid output.
4. **Expected (buggy) result:** Node accepts the transaction — no `CellbaseImmaturity` error.
5. **Control:** Repeat with `dep_type: "code"` on the same cellbase output. Node correctly rejects with `CellbaseImmaturity(CellDeps[0])`, confirming the check fires for `resolved_cell_deps` but is absent for `resolved_dep_groups`.

    pub fn verify(&self) -> Result<(), Error> {
        let cellbase_immature = |meta: &CellMeta| -> bool {
            meta.transaction_info
                .as_ref()
                .map(|info| {
                    info.block_number > 0 && info.is_cellbase() && {
                        let threshold =
                            self.cellbase_maturity.to_rational() + info.block_epoch.to_rational();
                        let current = self.epoch.to_rational();
                        current < threshold
                    }
                })
                .unwrap_or(false)
        };

        if let Some(index) = self
            .transaction
            .resolved_inputs
            .iter()
            .position(cellbase_immature)
        {
            return Err(TransactionError::CellbaseImmaturity {
                inner: TransactionErrorSource::Inputs,
                index,
            }
            .into());
        }

        if let Some(index) = self
            .transaction
            .resolved_cell_deps
            .iter()
            .position(cellbase_immature)
        {
            return Err(TransactionError::CellbaseImmaturity {
                inner: TransactionErrorSource::CellDeps,
                index,
            }
            .into());
        }

        Ok(())
    }
