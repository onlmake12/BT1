### Title
Stale Price Returned Without Staleness Check in `latestRoundData()` and All Price Functions - (File: `target_chains/ethereum/sdk/solidity/PythAggregatorV3.sol`)

---

### Summary

`PythAggregatorV3.sol` is Pyth's official Chainlink-compatible adapter contract, explicitly marketed as a drop-in replacement for Chainlink's `AggregatorV3Interface`. Every price-returning function in the contract — including `latestRoundData()`, `latestAnswer()`, `getRoundData()`, `latestTimestamp()`, and `getAnswer()` — calls `getPriceUnsafe()` with no staleness validation whatsoever. A stale price from arbitrarily far in the past is silently returned to any integrating protocol, with no revert and no indication of staleness.

---

### Finding Description

`PythAggregatorV3.sol` implements the Chainlink `AggregatorV3Interface` using Pyth price feeds. The contract is the official Pyth SDK artifact for Chainlink migration and is deployed by integrators as a per-feed adapter.

Every function that returns a price value delegates to `pyth.getPriceUnsafe(priceId)`:

`getPriceUnsafe()` in `Pyth.sol` performs only one check — that `publishTime != 0` (i.e., the feed has ever been initialized). It performs **no staleness check**:

```solidity
function getPriceUnsafe(bytes32 id) public view override returns (PythStructs.Price memory price) {
    PythInternalStructs.PriceInfo storage info = _state.latestPriceInfo[id];
    price.publishTime = info.publishTime;
    ...
    if (price.publishTime == 0) revert PythErrors.PriceFeedNotFound();
}
``` [3](#0-2) 

By contrast, `AbstractPyth.sol` shows the correct pattern — `getPriceNoOlderThan()` checks `diff(block.timestamp, price.publishTime) > age` and reverts with `StalePrice` if exceeded. `PythAggregatorV3` never calls this function. [4](#0-3) 

The contract contains no `validTimePeriod`, no `maxAge`, no staleness threshold, and no revert path for stale data across all 120 lines. [5](#0-4) 

---

### Impact Explanation

Any protocol that integrates `PythAggregatorV3` as a Chainlink-compatible price oracle — following Pyth's own migration guide — will silently receive prices from arbitrarily far in the past whenever the on-chain Pyth price has not been recently updated. This can occur during:

- Network outages or sequencer downtime on L2s
- Market closure periods (e.g., equity feeds)
- Deliberate withholding of price updates by a keeper

Downstream consequences include:
- **Incorrect liquidations**: A borrower's collateral may be liquidated at a stale (artificially high or low) price
- **Arbitrage / undercollateralized borrowing**: An attacker can borrow against collateral valued at a stale inflated price
- **Mispriced derivatives / perpetuals**: Positions opened or closed at incorrect prices

The `updatedAt` field returned by `latestRoundData()` is set to `price.publishTime` — the Pyth publish timestamp — not `block.timestamp`. Protocols that check `updatedAt` against a heartbeat threshold will detect staleness, but protocols that trust the Chainlink interface contract to enforce freshness (the standard assumption) will not.

---

### Likelihood Explanation

- `PythAggregatorV3` is the official Pyth SDK contract for Chainlink migration, actively documented and promoted at `apps/developer-hub/content/docs/price-feeds/core/migrate-an-app-to-pyth/chainlink.mdx`.
- Pyth is a pull oracle; on-chain prices are only as fresh as the last `updatePriceFeeds()` call. If no keeper is running, prices go stale.
- An unprivileged attacker needs only to observe that the on-chain price is stale (readable from `latestTimestamp()`) and then interact with the integrating protocol — no privileged access required.
- The attack is fully permissionless and requires no special capability beyond a standard transaction. [6](#0-5) 

---

### Recommendation

Replace all `getPriceUnsafe()` calls in `PythAggregatorV3` with `getPriceNoOlderThan()` using a configurable `maxAge` parameter set at construction time:

```solidity
uint256 public maxAge; // e.g., 60 seconds

constructor(address _pyth, bytes32 _priceId, uint256 _maxAge) {
    priceId = _priceId;
    pyth = IPyth(_pyth);
    maxAge = _maxAge;
}

function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
    // reverts with StalePrice if price.publishTime is too old
    ...
}
```

This mirrors the behavior of `AbstractPyth.getPriceNoOlderThan()` which already implements the correct staleness guard. Integrators should also be warned that `updatedAt` in the returned tuple reflects the Pyth `publishTime`, not `block.timestamp`, so Chainlink-style heartbeat checks on `updatedAt` must account for this. [4](#0-3)

### Proof of Concept

1. Deploy `PythAggregatorV3` pointing to the Pyth contract and a price feed (e.g., ETH/USD).
2. Call `updatePriceFeeds()` once to initialize the on-chain price at time `T`.
3. Wait `N` seconds (e.g., 3600 seconds / 1 hour) without calling `updatePriceFeeds()` again.
4. Call `latestRoundData()` on `PythAggregatorV3`.
5. Observe: the call **succeeds** and returns the price from time `T`, with `updatedAt = T`. No revert, no staleness error.
6. A protocol consuming this adapter will use the 1-hour-old price as if it were current.

```solidity
// PoC (Foundry)
function testStalePrice() public {
    PythAggregatorV3 adapter = new PythAggregatorV3(address(pyth), ETH_USD_FEED_ID);
    // Price updated at T=0
    adapter.updateFeeds{value: fee}(updateData);

    // Warp 1 hour forward — no further updates
    vm.warp(block.timestamp + 3600);

    // latestRoundData returns stale price with no revert
    (, int256 answer, , uint256 updatedAt, ) = adapter.latestRoundData();
    // answer == price from 1 hour ago
    // updatedAt == block.timestamp - 3600
    // No StalePrice revert — protocol uses stale data
}
