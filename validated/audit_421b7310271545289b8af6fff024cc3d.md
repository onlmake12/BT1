### Title
No Manager Transfer Mechanism in `Scheduler.createSubscription` Permanently Locks Subscription Control and Deposited ETH - (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

### Summary

`Scheduler.createSubscription` permanently assigns `msg.sender` as the subscription manager with no mechanism to transfer that role. When a smart contract calls `createSubscription` on behalf of a user (a common integration pattern), the intermediate contract becomes the irrevocable manager. The actual user is permanently locked out of `updateSubscription` and `withdrawFunds`, and the deposited ETH is unrecoverable if the intermediate contract cannot forward those calls.

### Finding Description

In `Scheduler.createSubscription`, the manager is set unconditionally to `msg.sender`:

```solidity
// Map subscription ID to manager
_state.subscriptionManager[subscriptionId] = msg.sender;
``` [1](#0-0) 

The `onlyManager` modifier gates both `updateSubscription` and `withdrawFunds`:

```solidity
modifier onlyManager(uint256 subscriptionId) {
    if (_state.subscriptionManager[subscriptionId] != msg.sender) {
        revert SchedulerErrors.Unauthorized();
    }
    _;
}
``` [2](#0-1) 

`withdrawFunds` also sends ETH to `msg.sender` (the manager), not to a configurable recipient:

```solidity
(bool sent, ) = msg.sender.call{value: amount}("");
require(sent, "Failed to send funds");
``` [3](#0-2) 

A grep across all `.sol` files confirms there is **no `transferManager`, `setManager`, or any other setter** for `_state.subscriptionManager` beyond the initial assignment in `createSubscription`. [4](#0-3) 

The `SchedulerState` struct stores the mapping but exposes no mutator for it: [5](#0-4) 

### Impact Explanation

Any smart contract (DeFi protocol, factory, wrapper, or on-chain automation contract) that calls `createSubscription` on behalf of a user becomes the permanent, irrevocable manager. The consequences are:

1. **Subscription permanently unmanageable by the intended user** — `updateSubscription` (deactivation, parameter changes) and `withdrawFunds` both revert with `Unauthorized` for anyone other than the intermediate contract.
2. **Deposited ETH permanently locked** — if the intermediate contract has no function to call `withdrawFunds`, or has no `receive()` fallback to accept the ETH transfer, all deposited funds are irretrievably stuck in the Scheduler.
3. **No admin escape hatch** — the Scheduler admin/owner has no override to reassign the manager of an existing subscription.

This matches the Swivel M-09 class exactly: an intermediate caller (`msg.sender`) is baked in as the privileged role, while the intended principal (the user) is permanently excluded.

### Likelihood Explanation

The Scheduler is explicitly designed for integration by other contracts — the `IScheduler` interface is published as an SDK, and the Argus off-chain service interacts with it. Any on-chain wrapper (e.g., a DeFi protocol that auto-creates subscriptions for its users, or a factory that batches subscription creation) will silently trigger this condition. The user has no warning and no recourse. The likelihood is **medium**: it does not require a malicious actor — any well-intentioned integrator that calls `createSubscription` through a contract will produce this outcome.

### Recommendation

Add a `transferManager` function gated by `onlyManager`, analogous to a two-step ownership transfer:

```solidity
function transferManager(uint256 subscriptionId, address newManager) external onlyManager(subscriptionId) {
    require(newManager != address(0), "new manager is zero address");
    _state.subscriptionManager[subscriptionId] = newManager;
    emit ManagerTransferred(subscriptionId, msg.sender, newManager);
}
```

Alternatively, allow `createSubscription` to accept an explicit `manager` address parameter (defaulting to `msg.sender` for EOA callers), mirroring the pattern used in `EntropyUpgradable.initialize` and `SchedulerUpgradeable.initialize` which accept explicit `owner` and `admin` addresses rather than using `msg.sender`. [6](#0-5)

```solidity

contract SubscriptionFactory {
    IScheduler public scheduler;

    constructor(address _scheduler) { scheduler = IScheduler(_scheduler); }

    // Called by user to create a subscription on their behalf
    function createForUser(
        SchedulerStructs.SubscriptionParams calldata params
    ) external payable returns (uint256 subId) {
        // msg.sender here is the user, but inside createSubscription,
        // msg.sender is address(this) — the factory contract.
        subId = scheduler.createSubscription{value: msg.value}(params);
        // subscriptionManager[subId] == address(this), NOT the user
    }
}

// Attack / impact demonstration
contract PoC {
    function demonstrate(SubscriptionFactory factory, IScheduler scheduler) external payable {
        uint256 subId = factory.createForUser{value: msg.value}(params);

        // User tries to withdraw — REVERTS with Unauthorized
        scheduler.withdrawFunds(subId, msg.value); // ← reverts

        // User tries to deactivate — REVERTS with Unauthorized
        scheduler.updateSubscription(subId, deactivatedParams); // ← reverts

        // ETH is permanently locked in the Scheduler
    }
}
