### Title
Cross-Network Alert Signature Replay via Missing Chain Domain Separation — (`util/network-alert/src/verifier.rs`)

---

### Summary

The CKB Network Alert system signs and verifies alerts using a hash of `RawAlert` that contains no chain identifier (no genesis hash, no network name, no chain ID). The same four hardcoded public keys are used as the default signer set for every CKB network (mainnet, testnet, devnet). As a result, a valid alert signature produced for one network is cryptographically valid on every other network. An unprivileged attacker who observes a legitimately signed alert on testnet can replay it verbatim on mainnet — via the public `send_alert` RPC or the P2P Alert protocol — and every mainnet node will accept and broadcast it.

---

### Finding Description

**Root cause — hash contains no chain context**

`calc_alert_hash()` is defined as a plain `blake2b_256` over the molecule-serialized bytes of `RawAlert`:

```rust
// util/gen-types/src/extension/calc_hash.rs
impl<'r> packed::RawAlertReader<'r> {
    pub fn calc_alert_hash(&self) -> packed::Byte32 {
        self.calc_hash()   // → blake2b_256(self.as_slice())
    }
}
``` [1](#0-0) 

The `RawAlert` molecule schema contains only application-level fields:

```
table RawAlert {
    notice_until:   Uint64,
    id:             Uint32,
    cancel:         Uint32,
    priority:       Uint32,
    message:        Bytes,
    min_version:    BytesOpt,
    max_version:    BytesOpt,
}
``` [2](#0-1) 

There is no genesis hash, chain name, or any other network-scoping field. Two `RawAlert` structs with identical field values on mainnet and testnet produce an identical 32-byte hash.

**Root cause — same public keys across all networks**

**Verification path**

`Verifier::verify_signatures` computes the message from `alert.calc_alert_hash()` and checks it against the hardcoded public key set. No chain context is consulted:

```rust

---

### Impact Explanation

An attacker who observes any legitimately signed alert on testnet (or any other CKB network sharing the same key set) can:

1. **Inject false alerts on mainnet**: Submit the testnet alert to a mainnet node via `send_alert` RPC or the P2P Alert protocol. Every mainnet node will accept it, store it, and broadcast it to all peers. Users will see a misleading or irrelevant alert message.

2. **Cancel a legitimate mainnet alert**: If the replayed alert carries a `cancel` field matching the ID of an active mainnet alert, the `Notifier` will remove that alert from all nodes' noticed-alert lists, silencing a real emergency warning.

3. **Amplify via P2P relay**: Once accepted by one node, the alert is relayed to all connected peers automatically, achieving network-wide propagation without further attacker involvement.

The alert system is described as a mechanism for communicating "critical problems" to all nodes. Injecting false alerts or suppressing real ones undermines the integrity of this emergency communication channel.

---

### Likelihood Explanation

- Testnet alerts are publicly visible on the P2P network; no privileged access is needed to observe them.
- The attacker needs no private keys — only a captured, already-signed alert message.
- The `send_alert` RPC endpoint is reachable by any local or remote RPC caller (depending on node configuration); the P2P Alert protocol is reachable by any peer.
- The Nervos Foundation has signed real testnet alerts (evidenced by `test_alert_20230001` which verifies a real historical signature against the default key set). [9](#0-8) 

---

### Recommendation

Include the genesis hash (or a chain-specific domain tag) in the data that is hashed and signed for alerts. For example, modify `calc_alert_hash` to prepend the genesis hash before hashing:

```rust
pub fn calc_alert_hash(&self, genesis_hash: &[u8]) -> packed::Byte32 {
    let mut hasher = new_blake2b();
    hasher.update(genesis_hash);
    hasher.update(self.as_slice());
    let mut hash = [0u8; 32];
    hasher.finalize(&mut hash);
    hash.into()
}
```

Alternatively, maintain separate public key sets for mainnet and testnet so that a signature valid on one network is not accepted on the other.

---

### Proof of Concept

1. Run a testnet node and a mainnet node, both using the default `alert_signature.toml`.
2. Obtain a legitimately signed testnet alert (e.g., by monitoring the testnet P2P Alert protocol or from a public source). Call it `alert_bytes`.
3. Submit it to the mainnet node's `send_alert` RPC:
   ```json
   {"jsonrpc":"2.0","method":"send_alert","params":[<alert_bytes>],"id":1}
   ```
4. The mainnet node calls `verify_signatures`, computes `blake2b_256(raw_alert_bytes)` — identical to the testnet hash — and finds 2-of-4 valid signatures against the same hardcoded keys.
5. The alert is accepted, stored, and broadcast to all mainnet peers. Mainnet users see the testnet alert message.

To demonstrate the cancel variant: craft a testnet alert with `cancel = <id_of_active_mainnet_alert>`, have it signed on testnet, then replay it on mainnet to silently remove the active mainnet alert from all nodes.

    pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
        trace!("Verifying alert {:?}", alert);
        let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
        let signatures: Vec<Signature> = alert
            .signatures()
            .into_iter()
            .filter_map(
                |sig_data| match Signature::from_slice(sig_data.as_reader().raw_data()) {
                    Ok(sig) => {
                        if sig.is_valid() {
                            Some(sig)
                        } else {
                            debug!("invalid signature: {:?}", sig);
                            None
                        }
                    }
                    Err(err) => {
                        debug!("signature error: {}", err);
                        None
                    }
                },
            )
            .collect();
        verify_m_of_n(
            &message,
            self.config.signatures_threshold,
            &signatures,
            &self.pubkeys,
        )
        .map_err(|err| err.kind())?;
        Ok(())
    }

    fn send_alert(&self, alert: Alert) -> Result<()> {
        let alert: packed::Alert = alert.into();
        let now_ms = ckb_systemtime::unix_time_as_millis();
        let notice_until: u64 = alert.raw().notice_until().into();
        if notice_until < now_ms {
            return Err(RPCError::invalid_params(format!(
                "Expected `params[0].notice_until` in the future (> {now_ms}), got {notice_until}",
            )));
        }

        let result = self.verifier.verify_signatures(&alert);

        match result {
            Ok(()) => {
                // set self node notifier
                self.notifier.lock().add(&alert);

                self.network_controller.broadcast_with_handle(
                    SupportProtocols::Alert.protocol_id(),
                    alert.as_bytes(),
                    &self.handle,
                );
                Ok(())
            }
            Err(e) => Err(RPCError::custom_with_error(
                RPCError::AlertFailedToVerifySignatures,
                e,
            )),
        }
    }
