### Title
Unpermissioned `set_publisher_stake_account` Allows Any Signer to Hijack a Publisher's Self-Delegation Rewards — (`governance/pyth_staking_sdk/src/idl/integrity-pool.json`)

---

### Summary

The `set_publisher_stake_account` instruction in the Integrity Pool program accepts an arbitrary `signer` with no constraint tying that signer to the `publisher` account being modified. Any unprivileged transaction sender can call this instruction to reassign or remove any publisher's associated stake account, redirecting or destroying that publisher's self-delegation rewards without the publisher's consent.

---

### Finding Description

The `set_publisher_stake_account` instruction's account structure in the IDL is:

```json
{
  "name": "signer",
  "signer": true
},
{
  "docs": ["CHECK : The publisher will be checked against data in the pool_data"],
  "name": "publisher"
},
{
  "name": "new_stake_account_positions_option",
  "optional": true
},
{
  "name": "current_stake_account_positions_option",
  "optional": true
}
```

The `signer` account carries **no `relations` constraint** linking it to `pool_config`, `pool_data`, or the `publisher` account. Compare this to privileged instructions such as `create_slash_event`, where the authority is constrained via `"relations": ["pool_config"]`. [1](#0-0) 

The `publisher` account is explicitly **not** a signer — it is only checked for existence in `pool_data`. [2](#0-1) 

The SDK's `setPublisherStakeAccount` method confirms this: `this.wallet` (the caller's wallet) is used as the signer, and `publisher` is passed as a separate, unsigned account. There is no assertion that `this.wallet.publicKey === publisher`. [3](#0-2) 

An attacker can therefore:
1. Call `set_publisher_stake_account` with `new_stake_account_positions_option = None` to **remove** a publisher's stake account association, causing the publisher to lose all self-delegation rewards.
2. Call `set_publisher_stake_account` with `new_stake_account_positions_option = attacker_stake_account` to **redirect** the publisher's self-delegation rewards to the attacker's own custody account.

The `current_stake_account_positions_option` (the existing stake account) is publicly readable on-chain, so the attacker has all the information needed to construct the transaction.

---

### Impact Explanation

The `publisherStakeAccounts` array in `poolData` determines which stake account custody receives self-delegation rewards when `advance_delegation_record` is executed. [4](#0-3) 

When `advance_delegation_record` runs, it uses `publisherStakeAccountPositions` and `publisherStakeAccountCustody` to distribute rewards. If an attacker has replaced the publisher's stake account with their own, the PYTH token rewards flow into the attacker's custody account instead of the legitimate publisher's. [5](#0-4) 

This is a direct, quantifiable loss of funds for the legitimate publisher.

---

### Likelihood Explanation

- The instruction is permissionless: any funded Solana wallet can submit it.
- All required inputs (`publisher` pubkey, `current_stake_account_positions_option`) are publicly readable from on-chain state.
- No special role, leaked key, or governance majority is required.
- The attack is cheap (one transaction) and repeatable — even if the publisher re-sets their stake account, the attacker can immediately overwrite it again.

---

### Recommendation

Add an on-chain constraint requiring that `signer == publisher`. In Anchor, this can be enforced with a `constraint` on the `signer` account:

```rust
#[account(
    constraint = signer.key() == publisher.key() @ IntegrityPoolError::Unauthorized
)]
pub signer: Signer<'info>,
```

Alternatively, if governance-only access is intended (as suggested by the xc_admin_cli usage), add a `relations = ["pool_config"]` constraint on `signer` tied to a `reward_program_authority` or equivalent admin field, consistent with how other privileged instructions are gated. [6](#0-5) 

---

### Proof of Concept

1. Observe that publisher `P` has stake account `S` stored in `poolData.publisherStakeAccounts[i]`.
2. Attacker creates their own stake account `A`.
3. Attacker submits `set_publisher_stake_account` with:
   - `signer`: attacker's wallet (any valid signer)
   - `publisher`: `P`
   - `current_stake_account_positions_option`: `S` (read from on-chain state)
   - `new_stake_account_positions_option`: `A`
4. `poolData.publisherStakeAccounts[i]` is now set to `A`.
5. On the next call to `advance_delegation_record` for publisher `P`, self-delegation rewards are transferred to `A`'s custody account (controlled by the attacker), not `S`'s custody account (controlled by the legitimate publisher).
6. Publisher `P` receives zero self-delegation rewards for that epoch. The attacker receives them instead. [7](#0-6) [8](#0-7)

  async getAdvanceDelegationRecordInstructions(
    stakeAccountPositions: PublicKey,
    payer?: PublicKey,
  ) {
    const poolData = await this.getPoolDataAccount();
    const stakeAccountPositionsData = await this.getStakeAccountPositions(
      stakeAccountPositions,
    );
    const allPublishers = extractPublisherData(poolData);
    const publishers = allPublishers
      .map((publisher) => {
        const positionsWithPublisher =
          stakeAccountPositionsData.data.positions.filter(
            ({ targetWithParameters }) =>
              targetWithParameters.integrityPool?.publisher.equals(
                publisher.pubkey,
              ),
          );

        let lowestEpoch;
        for (const position of positionsWithPublisher) {
          lowestEpoch = bigintMin(position.activationEpoch, lowestEpoch);
        }

        return {
          ...publisher,
          lowestEpoch,
        };
      })
      .filter(({ lowestEpoch }) => lowestEpoch !== undefined);

    const delegationRecords = await Promise.all(
      publishers.map(({ pubkey }) =>
        this.getDelegationRecord(stakeAccountPositions, pubkey),
      ),
    );

    let lowestEpoch: bigint | undefined;
    for (const [index, publisher] of publishers.entries()) {
      const maximum = bigintMax(
        publisher.lowestEpoch,
        delegationRecords[index]?.lastEpoch,
      );
      lowestEpoch = bigintMin(lowestEpoch, maximum);
    }

    const currentEpoch = await getCurrentEpoch(this.connection);

    // Filter out delegationRecord that are up to date
    const filteredPublishers = publishers.filter((_, index) => {
      return !(delegationRecords[index]?.lastEpoch === currentEpoch);
    });

    // anchor does not calculate the correct pda for other programs
    // therefore we need to manually calculate the pdas
    const advanceDelegationRecordInstructions = await Promise.all(
      filteredPublishers.map(({ pubkey, stakeAccount }) =>
        this.integrityPoolProgram.methods
          .advanceDelegationRecord()
          .accountsPartial({
            payer: payer ?? this.wallet.publicKey,
            publisher: pubkey,
            publisherStakeAccountCustody: stakeAccount
              ? getStakeAccountCustodyAddress(stakeAccount)
              : null, // eslint-disable-line unicorn/no-null
            publisherStakeAccountPositions: stakeAccount,
            stakeAccountCustody: getStakeAccountCustodyAddress(
              stakeAccountPositions,
            ),
            stakeAccountPositions,
          })
          .instruction(),
      ),
    );

  async setPublisherStakeAccount(
    publisher: PublicKey,
    stakeAccountPositions: PublicKey,
    newStakeAccountPositions: PublicKey | undefined,
  ) {
    const instruction = await this.integrityPoolProgram.methods
      .setPublisherStakeAccount()
      .accounts({
        currentStakeAccountPositionsOption: stakeAccountPositions,
        // eslint-disable-next-line unicorn/no-null
        newStakeAccountPositionsOption: newStakeAccountPositions ?? null,
        publisher,
      })
      .instruction();

    await sendTransaction([instruction], this.connection, this.wallet);
    return;
  }
