### Title
Missing Zero-Length Epoch Validation in `Since` Metric Extraction Causes Division-by-Zero Panic — (`verification/src/transaction_verifier.rs`)

---

### Summary

`Since::extract_metric()` constructs an `EpochNumberWithFraction` from an attacker-controlled `since` field using `from_full_value_unchecked`, which skips the zero-length normalization step. When the resulting value is subsequently passed to `to_rational()`, a division-by-zero panic occurs if the epoch length field is zero. Any unprivileged transaction sender can craft a `since` value that triggers this panic during tx-pool admission or block verification, crashing the node.

---

### Finding Description

In `verification/src/transaction_verifier.rs`, `Since::extract_metric()` parses the epoch-metric branch of the `since` field using `EpochNumberWithFraction::from_full_value_unchecked`: [1](#0-0) 

```rust
pub fn extract_metric(self) -> Option<SinceMetric> {
    let value = self.0 & VALUE_MASK;
    match self.0 & METRIC_TYPE_FLAG_MASK {
        0x0000_0000_0000_0000 => Some(SinceMetric::BlockNumber(value)),
        0x2000_0000_0000_0000 => Some(SinceMetric::EpochNumberWithFraction(
            EpochNumberWithFraction::from_full_value_unchecked(value),  // ← no normalization
        )),
        0x4000_0000_0000_0000 => value.checked_mul(1000).map(SinceMetric::Timestamp),
        _ => None,
    }
}
```

The safe counterpart, `from_full_value`, calls `.normalize()` to rewrite a zero-length epoch to length=1: [2](#0-1) 

```rust
pub fn from_full_value(value: u64) -> Self {
    Self::from_full_value_unchecked(value).normalize()
}

pub fn normalize(self) -> Self {
    if self.length() == 0 {
        Self::new(self.number(), 0, 1)
    } else {
        self
    }
}
```

`extract_metric()` uses the **unchecked** variant, so no normalization occurs. The code's own documentation explicitly warns about this: [3](#0-2) 

```
/// # Notice
/// The `EpochNumberWithFraction` constructed by this method has a potential risk that when
/// call `self.to_rational()` may lead to a panic if the user specifies a zero epoch length.
```

`to_rational()` panics when `self.0 != 0` but `self.length() == 0`: [4](#0-3) 

```rust
pub fn to_rational(self) -> RationalU256 {
    if self.0 == 0 {
        RationalU256::zero()
    } else {
        // panics if self.length() == 0
        RationalU256::new(self.index().into(), self.length().into()) + U256::from(self.number())
    }
}
```

The `SinceVerifier` calls `to_rational()` seven times on the epoch extracted from `extract_metric()` to perform epoch comparisons. [5](#0-4) 

The `flags_is_valid()` check does **not** guard against zero epoch length — it only checks that reserved flag bits are zero and the metric type is not `0b11`: [6](#0-5) 

The `NonContextualTransactionVerifier` checks version, size, empty inputs/outputs, duplicate deps, outputs data, and script hash type — none of which validate the `since` field's epoch length.

---

### Impact Explanation

An attacker submits a transaction whose `since` field encodes the epoch metric flag (`0x2000_0000_0000_0000`) with a zero epoch length but a non-zero epoch number or index (e.g., `since = 0x2000_0000_0000_0001`). When the node's tx-pool admission or block verification pipeline calls `SinceVerifier::verify()` → `extract_metric()` → `to_rational()`, the node panics and crashes. This is a remotely triggerable node crash with no authentication required.

---

### Likelihood Explanation

The `since` field is a plain `u64` in every transaction input, fully controlled by the transaction sender. No privileged access is needed. The crafted value passes all non-contextual checks. The attack requires only submitting one transaction via the RPC `send_transaction` endpoint or broadcasting it over P2P.

---

### Recommendation

Replace `from_full_value_unchecked` with `from_full_value` (which calls `.normalize()`) in `Since::extract_metric()`:

```rust
0x2000_0000_0000_0000 => Some(SinceMetric::EpochNumberWithFraction(
    EpochNumberWithFraction::from_full_value(value),  // safe: normalizes zero length
)),
```

Alternatively, add an explicit guard in `SinceVerifier` that rejects any epoch-metric `since` value whose extracted epoch length is zero before calling `to_rational()`.

---

### Proof of Concept

Craft a transaction input with:

```
since = 0x2000_0000_0000_0001
```

Bit breakdown:
- Bits 63: `0` → absolute since
- Bits 62–61: `01` → epoch metric (`0x2000_0000_0000_0000`)
- Bits 60–56: `0` → remain flags (valid)
- Bits 55–40 of value: `0` → epoch **length = 0**
- Bits 39–0 of value: `1` → epoch number = 1, index = 0

`extract_metric()` returns `SinceMetric::EpochNumberWithFraction(EpochNumberWithFraction(1))`.

`EpochNumberWithFraction(1).length()` = `(1 >> LENGTH_OFFSET) & LENGTH_MASK` = `0`.

`EpochNumberWithFraction(1).to_rational()`: `self.0 = 1 ≠ 0`, so the else branch executes `RationalU256::new(0.into(), 0.into())` — zero denominator → **panic / division by zero**.

Submit this transaction via `send_transaction` RPC. The node crashes during `SinceVerifier::verify()`. [7](#0-6) [4](#0-3)

const LOCK_TYPE_FLAG: u64 = 1 << 63;
const METRIC_TYPE_FLAG_MASK: u64 = 0x6000_0000_0000_0000;
const VALUE_MASK: u64 = 0x00ff_ffff_ffff_ffff;
const REMAIN_FLAGS_BITS: u64 = 0x1f00_0000_0000_0000;

/// Metric represent value
pub enum SinceMetric {
    /// The metric_flag is 00, `value` can be explained as a block number or a relative block number.
    BlockNumber(u64),
    /// The metric_flag is 01, value can be explained as an absolute epoch or relative epoch.
    EpochNumberWithFraction(EpochNumberWithFraction),
    /// The metric_flag is 10, value can be explained as a block timestamp(unix time) or a relative
    Timestamp(u64),
}

/// The struct define wrapper for (unsigned 64-bit integer) tx field since
///
/// See [tx-since](https://github.com/nervosnetwork/rfcs/blob/master/rfcs/0017-tx-valid-since/0017-tx-valid-since.md)
#[derive(Copy, Clone, Debug)]
pub struct Since(pub u64);

impl Since {
    /// Whether since represented absolute form
    pub fn is_absolute(self) -> bool {
        self.0 & LOCK_TYPE_FLAG == 0
    }

    /// Whether since represented relative form
    #[inline]
    pub fn is_relative(self) -> bool {
        !self.is_absolute()
    }

    /// Whether since flag is valid
    pub fn flags_is_valid(self) -> bool {
        (self.0 & REMAIN_FLAGS_BITS == 0)
            && ((self.0 & METRIC_TYPE_FLAG_MASK) != METRIC_TYPE_FLAG_MASK)
    }

    /// Extracts a `SinceMetric` from an unsigned 64-bit integer since
    pub fn extract_metric(self) -> Option<SinceMetric> {
        let value = self.0 & VALUE_MASK;
        match self.0 & METRIC_TYPE_FLAG_MASK {
            //0b0000_0000
            0x0000_0000_0000_0000 => Some(SinceMetric::BlockNumber(value)),
            //0b0010_0000
            0x2000_0000_0000_0000 => Some(SinceMetric::EpochNumberWithFraction(
                EpochNumberWithFraction::from_full_value_unchecked(value),
            )),
            //0b0100_0000
            0x4000_0000_0000_0000 => value.checked_mul(1000).map(SinceMetric::Timestamp),
            _ => None,
        }
    }
}
