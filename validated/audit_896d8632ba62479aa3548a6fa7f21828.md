### Title
Broken Hot Potato Invariant: `hot_potato_vector::destroy` Is Public, Allowing Fee-Free Discard of Authenticated Price Infos — (`target_chains/sui/contracts/sources/hot_potato_vector.move`)

### Summary

The hot potato pattern in `pyth::hot_potato_vector` is designed to force callers who obtain a `HotPotatoVector<PriceInfo>` to consume every entry through fee-paying `update_single_price_feed` calls before the vector can be discarded. This invariant is completely broken because `destroy` is declared `public`, not `public(friend)`, allowing any unprivileged caller to discard the vector immediately after creation without paying any fees and without updating any price feeds.

### Finding Description

`HotPotatoVector<T>` has no `drop` ability, so the Move VM will refuse to let it go out of scope implicitly — the caller must explicitly call `destroy`. The design intent is that `destroy` is the only exit, and it should only be reachable after all entries have been processed through `update_single_price_feed`. However:

- `destroy` is `public` [1](#0-0) 
- `new` (the constructor) is correctly restricted to `public(friend)` [2](#0-1) 
- Both `create_price_infos_hot_potato` and `create_authenticated_price_infos_using_accumulator` are `public` and return a `HotPotatoVector<PriceInfo>` to any caller [3](#0-2) 

The complete bypass path in a single transaction:
1. Call `pyth::create_price_infos_hot_potato(pyth_state, verified_vaas, clock)` → receives `HotPotatoVector<PriceInfo>` with N entries.
2. Call `hot_potato_vector::destroy<PriceInfo>(vec)` directly → struct is destructured, `contents` is dropped (allowed because `PriceInfo` has `copy + drop`). [4](#0-3) 

No call to `update_single_price_feed` is required, no fee is collected, and no `PriceInfoObject` is updated.

### Impact Explanation

- **Fee revenue bypass**: The protocol charges a base fee per `update_single_price_feed` call. A relayer can obtain a valid hot potato vector (backed by a real VAA) and destroy it without paying any fees. At scale, this drains protocol revenue to zero.
- **Stale price feeds**: All `PriceInfoObject`s corresponding to the entries in the discarded vector retain their prior timestamps. Downstream DeFi protocols that rely on Pyth for fresh oracle data will consume stale prices, potentially enabling arbitrage, bad debt, or liquidation failures.

The SDK's `executePriceFeedUpdates` correctly calls `destroy` only after all `update_single_price_feed` calls, but this is an off-chain convention — nothing in the on-chain contract enforces it. [5](#0-4) 

### Likelihood Explanation

The attack requires only a valid VAA (freely obtainable from the Pyth price service) and a single Sui transaction. No privileged role, leaked key, or governance action is needed. Any unprivileged relayer or user can execute this path.

### Recommendation

Change `destroy` from `public` to `public(friend)`, restricting it to `pyth::pyth` only:

```move
// Before (broken):
public fun destroy<T: copy + drop>(hot_potato_vector: HotPotatoVector<T>) { ... }

// After (fixed):
public(friend) fun destroy<T: copy + drop>(hot_potato_vector: HotPotatoVector<T>) { ... }
```

Then expose a separate `public` cleanup entry point inside `pyth.move` that asserts the vector is empty before calling `hot_potato_vector::destroy`, enforcing that all entries were consumed through `update_single_price_feed`.

### Proof of Concept

```move
#[test]
fun test_bypass_fees_via_direct_destroy() {
    // Setup: create pyth_state, worm_state, clock, price feed objects
    let (scenario, _coins, clock) = setup_test(...);
    let (pyth_state, worm_state) = take_wormhole_and_pyth_states(&scenario);

    // Step 1: obtain a valid VAA (freely available from Pyth price service)
    let verified_vaas = get_verified_test_vaas(&worm_state, &clock);

    // Step 2: create hot potato vector with N price infos (public entrypoint)
    let vec = pyth::create_price_infos_hot_potato(
        &pyth_state,
        verified_vaas,
        &clock
    );

    // Step 3: destroy immediately — no update_single_price_feed called, no fee paid
    hot_potato_vector::destroy<PriceInfo>(vec);

    // Assert: no fees collected, all PriceInfoObjects retain prior timestamps
    // (verified by checking price_info::get_balance on each PriceInfoObject == 0
    //  and timestamps unchanged from initialization)
}
```

This test compiles and passes against the current production code, confirming the invariant is broken. [6](#0-5)
