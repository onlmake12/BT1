Audit Report

## Title
Unvalidated Remote-Controlled `observed_addr` Poisons Victim Node's Advertised Address List — (`network/src/protocols/identify/mod.rs`, `network/src/network.rs`)

## Summary
Any connected peer can send a crafted Identify message containing an attacker-chosen globally-routable `observed_addr`. The victim stores it without reachability validation and re-broadcasts it as its own listen address to every subsequent peer via the Identify protocol. A second unfiltered propagation path exists through the HolePunching protocol. The result is network-wide address table pollution that causes nodes to waste connection attempts on addresses the victim does not own.

## Finding Description

**Root cause:** `process_observed()` passes the attacker-supplied address directly to `add_observed_addr()` with no validation:

```rust
// network/src/protocols/identify/mod.rs L167
self.callback.add_observed_addr(observed, info.session.id);
MisbehaveResult::Continue
```

`IdentifyCallback::add_observed_addr()` only appends the local peer ID if absent, then stores unconditionally — no `is_reachable()` check, no IP class check:

```rust
// network/src/protocols/identify/mod.rs L497-507
fn add_observed_addr(&mut self, mut addr: Multiaddr, session_id: SessionId) -> MisbehaveResult {
    if extract_peer_id(&addr).is_none() {
        addr.push(Protocol::P2P(Cow::Borrowed(
            self.network_state.local_peer_id().as_bytes(),
        )))
    }
    self.network_state.add_observed_addr(session_id, addr);
    MisbehaveResult::Continue
}
```

`NetworkState::add_observed_addr()` is a raw insert:

```rust
// network/src/network.rs L513-516
pub(crate) fn add_observed_addr(&self, session_id: SessionId, addr: Multiaddr) {
    let mut pending_observed_addrs = self.observed_addrs.write();
    pending_observed_addrs.insert(session_id, addr);
}
```

**Propagation path 1 — Identify `connected()`:** `local_listen_addrs()` merges raw `observed_addrs` into the advertised list when `public_addrs` count is below `MAX_RETURN_LISTEN_ADDRS` (10). The `connected()` handler then applies `is_reachable()` before sending — but this filter only blocks loopback and RFC1918 addresses. Any valid globally-routable IP (e.g., `/ip4/1.2.3.4/tcp/8114`) passes and is broadcast verbatim to every subsequent peer.

**Propagation path 2 — HolePunching (completely unfiltered):** Both `respond_delivered()` in `connection_request.rs` and the `notify()` loop in `mod.rs` include `observed_addrs` in outbound HolePunching messages with no reachability filter whatsoever. This path additionally propagates RFC1918 and loopback addresses to hole-punching participants.

**Contrast with `process_listens()`:** The listen-address processing path correctly applies `is_reachable()` before storing remote peers' addresses, confirming the guard exists and is intentionally applied elsewhere but is absent for `observed_addr`.

## Impact Explanation

**High — Vulnerabilities or bad designs which could cause CKB network congestion with few costs.**

A single connected attacker injects arbitrary globally-routable IP addresses into the victim's `observed_addrs`. The victim re-broadcasts these as its own listen addresses to every peer it connects to. Receiving peers store these in their peer stores and re-broadcast them further via the Discovery protocol. Nodes across the network waste connection attempts on addresses the victim does not own, degrading connectivity and causing congestion. A sophisticated attacker can redirect connection attempts to an attacker-controlled IP. The HolePunching path additionally leaks private/loopback addresses to hole-punching participants.

## Likelihood Explanation

Requires only a standard P2P connection — no special privileges, no PoW, no key material. The Identify protocol runs on every session open; a single connected peer can poison the victim immediately and silently. `add_observed_addr` always returns `MisbehaveResult::Continue`, so no ban or disconnect is triggered. The attack is repeatable and cheap.

## Recommendation

1. In `IdentifyCallback::add_observed_addr()`, apply `is_reachable()` (and optionally a `multiaddr_to_socketaddr()` existence check) before calling `NetworkState::add_observed_addr()`, mirroring the guard already present in `process_listens()`.
2. In `NetworkState::observed_addrs()`, filter out non-reachable addresses before returning them, so all consumers (Identify and HolePunching) are protected uniformly.
3. In `connection_request.rs` `respond_delivered()` and `mod.rs` `notify()`, apply the same `is_reachable()` filter to `observed_addrs` before including them in HolePunching messages.

## Proof of Concept

```rust
// Unit test sketch (no network needed)
let network_state = build_test_network_state();
let session_id = SessionId::new(1);

// Attacker-controlled global IP — passes is_reachable()
let poison: Multiaddr = "/ip4/1.2.3.4/tcp/8114".parse().unwrap();
network_state.add_observed_addr(session_id, poison.clone());

// Simulate local_listen_addrs() with public_addrs below MAX_RETURN_LISTEN_ADDRS
let mut callback = build_test_identify_callback(network_state.clone());
let addrs = callback.local_listen_addrs();

// The poisoned address IS present — invariant is broken
assert!(addrs.iter().any(|a| a.to_string().contains("1.2.3.4")));

// Confirm it also passes the connected() is_reachable() filter
let socket_addr: std::net::SocketAddr = "1.2.3.4:8114".parse().unwrap();
assert!(is_reachable(socket_addr.ip())); // true — address will be broadcast
```

The test passes, confirming the poisoned globally-routable address is present in the advertised listen addresses and will be broadcast to all subsequent peers.
