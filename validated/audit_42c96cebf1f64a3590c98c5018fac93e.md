### Title
O(n²) Nested Loops with Storage Reads in `_validateSubscriptionParams` and `_clearRemovedPriceUpdates` Can Cause Gas Exhaustion DoS — (`target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

The `Scheduler.sol` contract contains two O(n²) nested-loop patterns that, at the protocol-defined maximum of `MAX_PRICE_IDS_PER_SUBSCRIPTION = 255` and `MAX_READER_WHITELIST_SIZE = 255`, can consume ~7–8 M gas in a single `updateSubscription` call. On EVM chains with block gas limits below ~30 M (or when combined with other transaction costs), this can make it impossible for a subscription manager to update their subscription, permanently locking them into stale parameters.

---

### Finding Description

**Root cause 1 — `_validateSubscriptionParams` O(n²) memory loops** [1](#0-0) 

Two independent nested loops check for duplicate `priceIds` and duplicate `readerWhitelist` addresses. With `n = 255` each, the inner loops execute `255 × 254 / 2 = 32 385` iterations per check, totalling ~64 770 iterations. These are memory reads (~6 gas each), contributing ~388 620 gas.

**Root cause 2 — `_clearRemovedPriceUpdates` O(n²) storage reads** [2](#0-1) 

The function is called from `updateSubscription` with `currentPriceIds` as a **storage** array. The first loop reads each of the 255 storage slots cold (2 100 gas × 255 = 537 000 gas). The second loop then reads all 255 storage slots again for every element of `newPriceIds` — 255 × 255 = 65 025 warm reads at 100 gas each = **6 502 500 gas**. Total for this function alone: ~7 039 500 gas.

**Constants that set the ceiling** [3](#0-2) 

`MAX_PRICE_IDS_PER_SUBSCRIPTION = 255` and `MAX_READER_WHITELIST_SIZE = 255` are the enforced upper bounds, yet the O(n²) loops make the worst-case cost proportional to 255² ≈ 65 025 iterations with storage reads.

**Call path in `updateSubscription`** [4](#0-3) 

Any subscription manager (unprivileged EOA or contract) calls `updateSubscription`, which calls `_validateSubscriptionParams` and then `_clearRemovedPriceUpdates` unconditionally when the subscription is active.

---

### Impact Explanation

A subscription manager who created a subscription with 255 price IDs and 255 whitelist addresses, and later tries to update it (e.g., to change heartbeat parameters, add/remove a feed, or change the whitelist), will trigger ~7.4 M gas of O(n²) work. On chains whose block gas limit is ≤ 8–10 M (several EVM-compatible L2s and sidechains), the transaction will always revert out-of-gas, permanently preventing the manager from modifying their subscription. Even on Ethereum mainnet (30 M limit), this consumes ~25% of the block gas budget, making the transaction expensive and fragile under congestion.

---

### Likelihood Explanation

- `createSubscription` and `updateSubscription` are permissionless; any user can be a subscription manager.
- `MAX_PRICE_IDS_PER_SUBSCRIPTION = 255` is explicitly documented and enforced as the allowed maximum, so users are expected to use values up to 255.
- The Scheduler README explicitly states it targets a "decentralized keeper network" across multiple EVM chains, many of which have lower gas limits than Ethereum mainnet.
- No off-chain simulation or warning prevents a manager from hitting this limit.

---

### Recommendation

1. **Replace nested duplicate-check loops with a transient mapping** (or a `seen` bitmap for `bytes32` IDs) in `_validateSubscunctionParams` to reduce duplicate detection from O(n²) to O(n).
2. **Replace the O(n²) storage scan in `_clearRemovedPriceUpdates`** with a mapping-based lookup: store `priceIds` in a `mapping(bytes32 => bool)` alongside the array so membership tests are O(1).
3. **Lower `MAX_PRICE_IDS_PER_SUBSCRIPTION`** to a value whose worst-case O(n²) gas cost is safe on all target chains, or add an explicit gas guard.

---

    function updateSubscription(
        uint256 subscriptionId,
        SchedulerStructs.SubscriptionParams memory newParams
    ) external payable override onlyManager(subscriptionId) {
        SchedulerStructs.SubscriptionStatus storage currentStatus = _state
            .subscriptionStatuses[subscriptionId];
        SchedulerStructs.SubscriptionParams storage currentParams = _state
            .subscriptionParams[subscriptionId];

        // Add incoming funds to balance
        currentStatus.balanceInWei += msg.value;

        // Updates to permanent subscriptions are not allowed
        if (currentParams.isPermanent) {
            revert SchedulerErrors.CannotUpdatePermanentSubscription();
        }

        // If subscription is inactive and will remain inactive, no need to validate parameters
        bool wasActive = currentParams.isActive;
        bool willBeActive = newParams.isActive;
        if (!wasActive && !willBeActive) {
            // Update subscription parameters
            _state.subscriptionParams[subscriptionId] = newParams;
            emit SubscriptionUpdated(subscriptionId);
            return;
        }
        _validateSubscriptionParams(newParams);

        // Check minimum balance if subscription remains active
        if (willBeActive) {
            uint256 minimumBalance = this.getMinimumBalance(
                uint8(newParams.priceIds.length)
            );
            if (currentStatus.balanceInWei < minimumBalance) {
                revert SchedulerErrors.InsufficientBalance();
            }
        }

        // Handle activation/deactivation
        if (!wasActive && willBeActive) {
            // Reactivating a subscription - ensure minimum balance
            uint256 minimumBalance = this.getMinimumBalance(
                uint8(newParams.priceIds.length)
            );

            // Check if balance meets minimum requirement
            if (currentStatus.balanceInWei < minimumBalance) {
                revert SchedulerErrors.InsufficientBalance();
            }

            currentParams.isActive = true;
            _addToActiveSubscriptions(subscriptionId);
            emit SubscriptionActivated(subscriptionId);
        } else if (wasActive && !willBeActive) {
            // Deactivating a subscription
            currentParams.isActive = false;
            _removeFromActiveSubscriptions(subscriptionId);
            emit SubscriptionDeactivated(subscriptionId);
        }

        // Clear price updates for removed price IDs before updating params
        bool newPriceIdsAdded = _clearRemovedPriceUpdates(
            subscriptionId,
            currentParams.priceIds,
            newParams.priceIds
        );

        // Reset priceLastUpdatedAt to 0 if new price IDs were added
        if (newPriceIdsAdded) {
            _state.subscriptionStatuses[subscriptionId].priceLastUpdatedAt = 0;
        }

        // Update subscription parameters
        _state.subscriptionParams[subscriptionId] = newParams;

        emit SubscriptionUpdated(subscriptionId);
    }

    function _clearRemovedPriceUpdates(
        uint256 subscriptionId,
        bytes32[] storage currentPriceIds,
        bytes32[] memory newPriceIds
    ) internal returns (bool newPriceIdsAdded) {
        // Iterate through old price IDs
        for (uint i = 0; i < currentPriceIds.length; i++) {
            bytes32 oldPriceId = currentPriceIds[i];
            bool found = false;

            // Check if the old price ID exists in the new list
            for (uint j = 0; j < newPriceIds.length; j++) {
                if (newPriceIds[j] == oldPriceId) {
                    found = true;
                    break; // Found it, no need to check further
                }
            }

            // If not found in the new list, delete its stored update data
            if (!found) {
                delete _state.priceUpdates[subscriptionId][oldPriceId];
            }
        }

        // Check if any new price IDs were added
        for (uint i = 0; i < newPriceIds.length; i++) {
            bytes32 newPriceId = newPriceIds[i];
            bool found = false;

            // Check if the new price ID exists in the current list
            for (uint j = 0; j < currentPriceIds.length; j++) {
                if (currentPriceIds[j] == newPriceId) {
                    found = true;
                    break;
                }
            }

            // If a new price ID was added, mark as changed
            if (!found) {
                newPriceIdsAdded = true;
                break;
            }
        }

        return newPriceIdsAdded;
    }
