### Title
`ScriptHashTypeVerifier` Omits Output Type Script Hash-Type Validation, Allowing Not-Yet-Activated `ScriptHashType` Values to Bypass the Non-Contextual Gate — (File: `verification/src/transaction_verifier.rs`)

---

### Summary

`ScriptHashTypeVerifier::verify()` iterates over transaction outputs and validates **only the lock script's `hash_type`** against the `ENABLED_SCRIPT_HASH_TYPE` allowlist. The optional type script field (`output.type_()`) is never inspected. Because `ScriptHashType` already defines `Data3` through `Data127` (values 6–254) but only `{0, 1, 2, 4}` are in `ENABLED_SCRIPT_HASH_TYPE`, any unprivileged transaction sender can craft an output whose type script carries a not-yet-activated hash type (e.g., `Data3`). The transaction passes the non-contextual gate and can be relayed across the P2P network before being rejected at script-execution time.

---

### Finding Description

`ENABLED_SCRIPT_HASH_TYPE` in `util/constant/src/consensus.rs` is a compile-time static set containing exactly four values:

```
{0 = Data, 1 = Type, 2 = Data1, 4 = Data2}
``` [1](#0-0) 

The `ScriptHashType` enum is generated via `seq_macro` and defines `Data3` (= 6) through `Data127` (= 254) as valid Rust variants — they are parseable from the wire but not yet activated: [2](#0-1) 

`ScriptHashTypeVerifier::verify()` iterates over every output and checks **only** `output.lock().hash_type()`. The `output.type_()` field is never touched: [3](#0-2) 

This verifier is wired into `NonContextualTransactionVerifier`, which is the first gate applied to every submitted or relayed transaction: [4](#0-3) 

By contrast, `select_version()` — the contextual, script-execution-time check — correctly rejects any hash type outside `{Data, Data1, Data2, Type}` with `InvalidScriptHashType`: [5](#0-4) 

Similarly, `extract_script_and_dep_index()` falls through to an error for any hash type beyond `Data2`: [6](#0-5) 

The result is a split-gate inconsistency: lock script hash types are validated at the **non-contextual** level (early, cheap), while type script hash types are validated only at **script execution** (late, expensive). A transaction with a `Data3` type script on any output clears `ScriptHashTypeVerifier` and enters the relay/pool pipeline before the expensive contextual check fires.

---

### Impact Explanation

An unprivileged transaction sender submits a transaction whose output carries a `Data3` (or any `DataN`, N ≥ 3) type script. The transaction:

1. Passes `ScriptHashTypeVerifier` (lock script hash type is valid; type script is never checked).
2. Is admitted to the relay pipeline and propagated to peers.
3. Fails only when `select_version` is called during script execution, returning `InvalidScriptHashType`.

Every receiving node must re-run non-contextual verification, deserialize the transaction, and begin contextual verification before discovering the invalidity. A sustained flood of such transactions constitutes a network-wide resource-exhaustion (DoS) attack with no per-transaction cost beyond the RPC call. Additionally, if a future hardfork activates `Data3` but `ENABLED_SCRIPT_HASH_TYPE` is not updated, the inverse problem arises: valid `Data3` type scripts would still pass the non-contextual gate (since it never checks them), but the asymmetry between lock and type script enforcement would persist, creating a permanently inconsistent validation surface.

---

### Likelihood Explanation

The attack requires no special privilege — any RPC caller or P2P peer can submit a transaction. Crafting an output with a `Data3` type script is trivial: set `hash_type = 6` in the type script field. The `ScriptHashType` enum already has `Data3` as a named variant, so no raw-byte manipulation is needed. The gap is reachable on mainnet today.

---

### Recommendation

Extend `ScriptHashTypeVerifier::verify()` to also validate the type script hash type for every output that carries one:

```rust
pub fn verify(&self) -> Result<(), Error> {
    for output in self.transaction.outputs() {
        // existing lock-script check
        if let Ok(hash_type) = TryInto::<ScriptHashType>::try_into(output.lock().hash_type()) {
            let val: u8 = hash_type.into();
            if !ENABLED_SCRIPT_HASH_TYPE.contains(&val) {
                return Err(TransactionError::ScriptHashTypeNotPermitted { hash_type: val }.into());
            }
        } else {
            return Err(TransactionError::InvalidScriptHashType {
                hash_type: output.lock().hash_type(),
            }.into());
        }

        // NEW: type-script check
        if let Some(type_script) = output.type_().to_opt() {
            if let Ok(hash_type) = TryInto::<ScriptHashType>::try_into(type_script.hash_type()) {
                let val: u8 = hash_type.into();
                if !ENABLED_SCRIPT_HASH_TYPE.contains(&val) {
                    return Err(TransactionError::ScriptHashTypeNotPermitted { hash_type: val }.into());
                }
            } else {
                return Err(TransactionError::InvalidScriptHashType {
                    hash_type: type_script.hash_type(),
                }.into());
            }
        }
    }
    Ok(())
}
```

When a future hardfork activates `Data3`, `ENABLED_SCRIPT_HASH_TYPE` and `select_version` must be updated together, keeping both gates in sync.

// Craft a transaction whose output has a Data3 type script (hash_type = 6).
let bad_type_script = Script::default()
    .as_builder()
    .hash_type(ScriptHashType::Data3)   // value = 6, not in ENABLED_SCRIPT_HASH_TYPE
    .build();

    .lock(Script::default())            // valid lock (Data = 0)
    .type_(Some(bad_type_script))
    .build();

    .output(output)
    .build();

// ScriptHashTypeVerifier only checks lock; this passes.
let verifier = ScriptHashTypeVerifier::new(&tx);

// Any node that relays this tx before reaching script execution

    /// when the low 1 bit is 0, it indicates the data,

    pub enum ScriptHashType {
        /// Type "type" matches script code via cell type script hash.
        Type = 1,

        #[default]

        Data1 = 2,

/// Basic checks that don't depend on any context

    pub(crate) size: SizeVerifier<'a>,
    pub(crate) empty: EmptyVerifier<'a>,
    pub(crate) duplicate_deps: DuplicateDepsVerifier<'a>,

    /// Creates a new NonContextualTransactionVerifier
    pub fn new(tx: &'a TransactionView, consensus: &'a Consensus) -> Self {

            version: VersionVerifier::new(tx, consensus.tx_version()),
            size: SizeVerifier::new(tx, consensus.max_block_bytes()),
            empty: EmptyVerifier::new(tx),

        self.script_hash_type.verify()?;
        Ok(())
    }

    /// Returns the version of the machine based on the script and the consensus rules.
    pub fn select_version(&self, script: &Script) -> Result<ScriptVersion, ScriptError> {
        let is_vm_version_2_and_syscalls_3_enabled = self.is_vm_version_2_and_syscalls_3_enabled();
        let is_vm_version_1_and_syscalls_2_enabled = self.is_vm_version_1_and_syscalls_2_enabled();
        let script_hash_type = ScriptHashType::try_from(script.hash_type())
            .map_err(|err| ScriptError::InvalidScriptHashType(err.to_string()))?;
        match script_hash_type {
            ScriptHashType::Data => Ok(ScriptVersion::V0),
            ScriptHashType::Data1 => {
                if is_vm_version_1_and_syscalls_2_enabled {
                    Ok(ScriptVersion::V1)
                } else {
                    Err(ScriptError::InvalidVmVersion(1))
                }
            }
            ScriptHashType::Data2 => {
                if is_vm_version_2_and_syscalls_3_enabled {
                    Ok(ScriptVersion::V2)
                } else {
                    Err(ScriptError::InvalidVmVersion(2))
                }
            }
            ScriptHashType::Type => {
                if is_vm_version_2_and_syscalls_3_enabled {
                    Ok(ScriptVersion::V2)
                } else if is_vm_version_1_and_syscalls_2_enabled {
                    Ok(ScriptVersion::V1)
                } else {
                    Ok(ScriptVersion::V0)
                }

                return Err(ScriptError::InvalidScriptHashType(format!(
                    "The ScriptHashType/{:?} has not been activated, and is not permitted for use.",

                )));

        }
