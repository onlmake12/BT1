### Title
`PythAggregatorV3.latestRoundData()` Returns Stale Price Without Staleness Enforcement — (File: `target_chains/ethereum/sdk/solidity/PythAggregatorV3.sol`)

---

### Summary

`PythAggregatorV3` is Pyth's official Chainlink-compatible adapter contract, shipped as part of the production Solidity SDK. Every price-reading function in the contract — `latestRoundData()`, `getRoundData()`, `latestAnswer()`, `latestTimestamp()`, and `decimals()` — calls `pyth.getPriceUnsafe(priceId)`, which by design makes **no recency guarantee** and can return a price from arbitrarily far in the past. No staleness check is applied before the data is returned to callers.

---

### Finding Description

`PythAggregatorV3.sol` implements Chainlink's `AggregatorV3Interface` on top of Pyth price feeds. Its purpose is to serve as a drop-in replacement for Chainlink feeds for protocols migrating to Pyth.

The entire contract is built on a single internal call:

```solidity
PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
```

This is used in every public function:

- `latestRoundData()` — line 110
- `getRoundData()` — line 89
- `latestAnswer()` — line 54
- `latestTimestamp()` — line 59
- `decimals()` — line 41 [1](#0-0) 

`getPriceUnsafe()` is explicitly documented in `IPyth.sol` as returning "the most recent price update in this contract **without any recency checks**" and that "the returned price update may be **arbitrarily far in the past**." [2](#0-1) 

The `latestRoundData()` function returns `updatedAt = price.publishTime`, which could be a timestamp from hours or days ago. A downstream protocol that trusts this value as "the latest price" — which is the entire premise of the Chainlink `AggregatorV3Interface` — will silently consume stale data.

The safe alternative, `getPriceNoOlderThan(priceId, age)`, is available in the same `IPyth` interface and enforces a staleness bound by reverting if the price is too old: [3](#0-2) 

`PythAggregatorV3` never uses it.

---

### Impact Explanation

Any protocol that deploys `PythAggregatorV3` as a Chainlink-compatible price feed and calls `latestRoundData()` will receive a stale price without any revert or signal. Concrete impacts include:

- **Lending/borrowing protocols**: Collateral valued at a stale (inflated) price allows over-borrowing or prevents correct liquidations.
- **Derivatives/perpetuals**: Positions opened or closed at stale prices allow risk-free profit extraction.
- **Liquidation bots**: Incorrect prices cause missed or erroneous liquidations.

The risk is amplified because the contract is explicitly marketed as a migration path for Chainlink users. Protocols migrating from Chainlink expect the adapter to handle staleness the same way Chainlink does (i.e., by returning a fresh price or reverting). They are unlikely to add a second staleness check on top of the adapter.

---

### Likelihood Explanation

- Pyth is a pull oracle; prices are only updated when someone submits an update transaction. During market hours gaps (e.g., US equity feeds overnight), network outages, or periods of low activity, the on-chain price can become significantly stale.
- The `PythAggregatorV3` contract is the official, documented migration path for Chainlink users (see `apps/developer-hub/content/docs/price-feeds/core/migrate-an-app-to-pyth/chainlink.mdx`). It is actively deployed by third-party protocols.
- An unprivileged attacker needs only to wait for a staleness window and then interact with a victim protocol that uses this adapter — no special access is required. [4](#0-3) 

---

### Recommendation

Replace `getPriceUnsafe` with `getPriceNoOlderThan` in `latestRoundData()` and `latestAnswer()`, using a configurable `maxAge` parameter set at construction time:

```solidity
uint256 public maxAge; // e.g., 60 seconds

constructor(address _pyth, bytes32 _priceId, uint256 _maxAge) {
    priceId = _priceId;
    pyth = IPyth(_pyth);
    maxAge = _maxAge;
}

function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
    // ...
}
```

This causes `latestRoundData()` to revert with `StalePrice` when the feed is stale, matching the behavior downstream protocols expect from a Chainlink-compatible interface.

---

### Proof of Concept

1. Deploy `PythAggregatorV3` pointing to any Pyth price feed.
2. Do **not** call `updateFeeds()` for a period exceeding the staleness threshold (e.g., 60 seconds).
3. Call `latestRoundData()`.
4. Observe that it returns successfully with `updatedAt` = a timestamp from the past, and `answer` = the stale price. No revert occurs.
5. A lending protocol using this adapter will accept the stale price as current, enabling over-collateralized borrowing or incorrect liquidations.
