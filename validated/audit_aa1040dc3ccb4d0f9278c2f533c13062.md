### Title
Cross-Chain Replay of Lazer Price Update Signatures Due to Missing Domain Separation - (`File: lazer/contracts/evm/src/PythLazer.sol`)

---

### Summary

The `PythLazer.verifyUpdate()` function on EVM chains computes the verification hash as `keccak256(payload)` with no chain ID, contract address, or any other domain separator included in the signed data. Because the same trusted signer key is registered on every deployed EVM chain, a valid signed Lazer price update accepted on one chain is cryptographically identical and accepted on every other chain. An unprivileged attacker can replay a signed update from Chain A to Chain B, injecting stale or cross-chain price data that passes on-chain verification.

---

### Finding Description

In `PythLazer.verifyUpdate()`, the hash used for ECDSA recovery is computed as:

```solidity
bytes32 hash = keccak256(payload);
(signer, , ) = ECDSA.tryRecover(hash, ...);
```

No chain ID, no contract address, and no network-specific context is mixed into the hash before signature recovery. [1](#0-0) 

The contract is deployed at the **same proxy address** (`0xACeA761c27A909d4D3895128EBe6370FDE2dF481`) on more than 20 EVM chains (Ethereum, Base, Arbitrum, Polygon, BSC, Monad, Berachain, HyperEVM, etc.). [2](#0-1) 

The same trusted signer address is registered on all of these chains via `updateTrustedSigner`. Because the signed payload contains no chain-binding, a payload signed by the trusted signer for Ethereum is byte-for-byte valid on Arbitrum, Base, and every other chain in the deployment set.

The Sui Lazer contract has the same structural issue: `verify_le_ecdsa_message` recovers the public key from `secp256k1_ecrecover(signature, payload, 0)` with no chain-specific domain separation in the signed data. [3](#0-2) 

---

### Impact Explanation

An attacker who observes a valid signed Lazer price update on any chain can submit the identical bytes to `verifyUpdate()` on any other chain. The contract will:
1. Accept the magic bytes check.
2. Recover the same signer address from the same signature over the same payload.
3. Confirm the signer is valid (same signer is trusted everywhere).
4. Return the payload as verified.

A DeFi protocol consuming the returned `payload` will receive price data that was originally produced for a different chain. More critically, the attacker can replay **old** signed payloads (any payload signed while the signer was still valid) to inject stale prices. The Lazer payload contains a `timestamp` field, but the on-chain contract does not enforce any freshness bound — that responsibility is delegated entirely to the consumer. Protocols that do not independently enforce a tight staleness window are exposed to stale-price manipulation (incorrect liquidations, mispriced trades). [4](#0-3) 

---

### Likelihood Explanation

- **Attacker preconditions**: None. Any address can call `verifyUpdate()` with a fee of 1 wei.
- **Payload availability**: Lazer updates are streamed publicly via WebSocket. Any subscriber can capture signed payloads from any chain's subscription.
- **Same signer across chains**: Confirmed by the deployment registry — the same trusted signer address is registered on all chains.
- **No replay protection in the contract**: The contract stores no record of previously verified payloads; the same bytes can be submitted repeatedly.

---

### Recommendation

Include a chain-specific domain separator in the data that is signed and verified. The standard approach is EIP-712 domain separation, which binds the signature to `chainId` and `verifyingContract`. At minimum, the hash should be:

```solidity
bytes32 hash = keccak256(abi.encodePacked(block.chainid, address(this), payload));
```

This must be coordinated with the Lazer signing infrastructure so that the off-chain signer includes the target chain ID in the signed message. The Sui and Aptos Lazer contracts require the same fix.

---

### Proof of Concept

1. Subscribe to the Lazer WebSocket on Ethereum (`formats: ["evm"]`) and capture a signed update blob `U` (hex bytes starting with magic `0x2a22999a...`).
2. On Arbitrum (or any other chain), call:
   ```solidity
   pythLazer.verifyUpdate{value: 1 wei}(U);
   ```
   where `pythLazer` is `0xACeA761c27A909d4D3895128EBe6370FDE2dF481`.
3. The call succeeds and returns the payload as verified, despite the payload having been signed for Ethereum.
4. To demonstrate stale-price injection: capture `U` at time `T`, wait until the price has moved significantly, then submit `U` to a consumer protocol on Arbitrum. The protocol receives the price from time `T` as if it were current. [4](#0-3) [5](#0-4)

    function verifyUpdate(
        bytes calldata update
    ) external payable returns (bytes calldata payload, address signer) {
        // Require fee and refund excess
        require(msg.value >= verification_fee, "Insufficient fee provided");
        if (msg.value > verification_fee) {
            payable(msg.sender).transfer(msg.value - verification_fee);
        }

        if (update.length < 71) {
            revert("input too short");
        }
        uint32 EVM_FORMAT_MAGIC = 706910618;

        uint32 evm_magic = uint32(bytes4(update[0:4]));
        if (evm_magic != EVM_FORMAT_MAGIC) {
            revert("invalid evm magic");
        }
        uint16 payload_len = uint16(bytes2(update[69:71]));
        if (update.length < 71 + payload_len) {
            revert("input too short");
        }
        payload = update[71:71 + payload_len];
        bytes32 hash = keccak256(payload);
        (signer, , ) = ECDSA.tryRecover(
            hash,
            uint8(update[68]) + 27,
            bytes32(update[4:36]),
            bytes32(update[36:68])
        );
        if (signer == address(0)) {
            revert("invalid signature");
        }
        if (!isValidSigner(signer)) {
            revert("invalid signer");
        }
    }

public(package) fun verify_le_ecdsa_message(
    state: &State,
    clock: &Clock,
    signature: &vector<u8>,
    payload: &vector<u8>,
) {
    let current_cap = state.current_cap();

    // 0 stands for keccak256 hash
    let pubkey = secp256k1_ecrecover(signature, payload, 0);

    // Check if the recovered pubkey is in the trusted signers list
    let trusted_signers = state.trusted_signers(&current_cap);
    let mut maybe_idx = trusted_signers.find_index!(|signer|
        signer.public_key() == &pubkey
    );

    assert!(maybe_idx.is_some(), ESignerNotTrusted);
    let idx = maybe_idx.extract();
    let expires_at_ms = trusted_signers[idx].expires_at_ms();
    assert!(clock.timestamp_ms() < expires_at_ms, ESignerExpired);
