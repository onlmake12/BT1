### Title
Unbounded DB Read Loop with No Rate Limiter in `GetBlockFilterCheckPointsProcess::execute` Enables Remote DoS — (`sync/src/filter/get_block_filter_check_points_process.rs`)

### Summary

The `BlockFilter` P2P protocol handler has no per-peer rate limiter. A single unprivileged remote peer can repeatedly send `GetBlockFilterCheckPoints{start_number:0}` messages, each of which triggers up to `BATCH_SIZE=2000` synchronous DB reads (two per iteration: `get_block_hash` + `get_block_filter_hash`), saturating the node's RocksDB I/O and stalling its async executor, effectively taking the node offline.

### Finding Description

`GetBlockFilterCheckPointsProcess::execute` in `sync/src/filter/get_block_filter_check_points_process.rs` loops up to `BATCH_SIZE` (2000) times, stepping by `CHECK_POINT_INTERVAL` (2000) blocks, performing two DB reads per iteration: [1](#0-0) [2](#0-1) 

The `BlockFilter` handler struct carries no rate-limiter field and performs no rate check before dispatching to `execute`: [3](#0-2) [4](#0-3) 

By contrast, the `Relayer` handler explicitly carries a `governor::RateLimiter` and checks it before processing every message: [5](#0-4) [6](#0-5) 

The `Filter` protocol is included in the default `support_protocols` list, so it is reachable on every standard CKB node: [7](#0-6) 

### Impact Explanation

Each `GetBlockFilterCheckPoints{start_number:0}` message causes up to 4,000 synchronous RocksDB reads on a fully-synced node. Because the `BlockFilter::received` handler `await`s the entire processing inline, a flood of these 8-byte messages from a single peer monopolizes the async executor task and saturates DB I/O. This stalls block relay, sync, and all other protocol handlers that share the same executor and DB, rendering the node unable to participate in the network — a practical remote denial of service.

### Likelihood Explanation

The attack requires only a standard P2P connection (no PoW, no stake, no key). The message is 8 bytes. The Filter protocol is on by default. The missing rate limiter is a clear oversight relative to the Relayer, which was explicitly protected. Any peer on the public network can trigger this.

### Recommendation

Add a `governor::RateLimiter<(PeerIndex, u32)>` field to `BlockFilter` (mirroring `Relayer`) and check it at the top of `BlockFilter::try_process` before dispatching to any sub-processor. A limit of 30 req/s per peer per message type (matching the Relayer's quota) would be a reasonable starting point.

### Proof of Concept

1. Connect to a CKB full node (with filter data built) as a standard P2P peer on the `Filter` protocol.
2. In a tight loop, send `BlockFilterMessage { GetBlockFilterCheckPoints { start_number: 0 } }` (8-byte payload).
3. Observe: RocksDB read latency climbs, the node's async executor stalls, and block relay / sync message processing stops responding.
4. The sending peer is never banned (the handler only bans on malformed messages or explicit `should_ban()` status returns; a well-formed request always returns `Status::ok()` or `Status::ignored()`). [8](#0-7)

    pub async fn execute(self) -> Status {
        let active_chain = self.filter.shared.active_chain();
        let start_number: BlockNumber = self.message.to_entity().start_number().into();
        let latest: BlockNumber = active_chain.get_latest_built_filter_block_number();

        let mut block_filter_hashes = Vec::new();

        if latest >= start_number {
            let mut block_number = start_number;
            for _ in 0..BATCH_SIZE {
                if let Some(block_filter_hash) = active_chain
                    .get_block_hash(block_number)
                    .and_then(|block_hash| active_chain.get_block_filter_hash(&block_hash))
                {
                    block_filter_hashes.push(block_filter_hash);
                } else {
                    break;
                }
                let Some(next_block_number) = block_number.checked_add(CHECK_POINT_INTERVAL) else {
                    break;
                };
                block_number = next_block_number;
            }
            let content = packed::BlockFilterCheckPoints::new_builder()
                .start_number(start_number)
                .block_filter_hashes(block_filter_hashes)
                .build();

            let message = packed::BlockFilterMessage::new_builder()
                .set(content)
                .build();
            async_send_message_to(&self.nc, self.peer, &message).await
        } else {
            Status::ignored()
        }
    }
