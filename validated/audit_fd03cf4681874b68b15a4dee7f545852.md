### Title
Emitter Authority Can Redirect Payer's Lamports to Arbitrary Destination via `close_message_v1` — (`target_chains/solana/programs/core-bridge/src/processor/post_message/close_message_v1.rs`)

### Summary

`close_message_v1` only validates that the `emitter_authority` signer matches the authority stored in the draft message account. It does not track or enforce who originally funded the account. The `close_account_destination` is a fully unchecked account, so the `emitter_authority` can drain all lamports to any wallet — including their own — even when a separate payer funded the account creation.

### Finding Description

The `draft_message` account is explicitly designed to be created **outside** the Core Bridge using the system program before `init_message_v1` is called: [1](#0-0) 

`init_message_v1` records the `emitter_authority` key into `PostedMessageV1Info.emitter_authority` but records **no payer/funder field**: [2](#0-1) 

`PostedMessageV1Info` has no `payer` field at all: [3](#0-2) 

`close_message_v1` only checks `require_draft_message` (emitter_authority match + `Writing` status). The `close_account_destination` is declared `UncheckedAccount` with no constraint: [4](#0-3) 

`close_account` then unconditionally moves all lamports to whatever destination was passed: [5](#0-4) 

`require_draft_message` checks only the emitter_authority key and `Writing` status — nothing about the original funder: [6](#0-5) 

### Impact Explanation

When `payer != emitter_authority`, the emitter_authority can call `close_message_v1(close_account_destination = emitter_authority_wallet)` and receive all lamports that the payer deposited for rent. The payer has no recourse; the lamports are permanently redirected. The loss per account is the rent-exempt minimum for the message account size (scales with payload size, up to the 30 KiB maximum).

### Likelihood Explanation

The scenario requires `payer != emitter_authority`. This is a realistic pattern in integrations where:
- A user/relayer pays for account creation while a program PDA (derived via `[b"emitter"]` seeds) acts as the emitter authority
- A protocol sponsors account rent on behalf of users while a separate authority manages message lifecycle

The attack is fully permissionless once the account is initialized: the emitter_authority needs only their own signature and can choose any `close_account_destination`.

### Recommendation

1. Add a `payer` field to `PostedMessageV1Info` and populate it in `init_message_v1`.
2. In `CloseMessageV1`, constrain `close_account_destination` to equal the stored `payer` (or at minimum the `emitter_authority`), preventing lamport redirection to arbitrary accounts.

### Proof of Concept

```
1. payer_A: system_program::create_account(
       from: payer_A,
       to: draft_message_keypair,
       lamports: rent_exempt_minimum,
       space: PostedMessageV1::compute_size(payload_len),
       owner: core_bridge::ID
   )

2. emitter_authority_B: core_bridge::init_message_v1(
       emitter_authority: emitter_authority_B,
       draft_message: draft_message_keypair.pubkey(),
       args: { nonce: 0, commitment: Confirmed, cpi_program_id: None }
   )
   // PostedMessageV1Info.emitter_authority = emitter_authority_B
   // PostedMessageV1Info.status = Writing
   // No payer recorded anywhere

3. emitter_authority_B: core_bridge::close_message_v1(
       emitter_authority: emitter_authority_B,
       draft_message: draft_message_keypair.pubkey(),
       close_account_destination: emitter_authority_B   // ← arbitrary, unchecked
   )
   // All lamports (originally from payer_A) transferred to emitter_authority_B

Assert: payer_A.lamports == initial - rent_exempt_minimum  (never restored)
Assert: emitter_authority_B.lamports == initial + rent_exempt_minimum  (stolen)
```

    pub(crate) fn require_draft_message(
        acc_info: &AccountInfo,
        emitter_authority: &Signer,
    ) -> Result<bool> {
        let data = acc_info.try_borrow_data()?;
        require!(
            data.len() > 4 && data[..4] == *Self::LEGACY_DISCRIMINATOR,
            ErrorCode::AccountDidNotDeserialize
        );

        require_keys_eq!(
            Self::emitter_authority_unsafe(&data),
            emitter_authority.key(),
            CoreBridgeError::EmitterAuthorityMismatch
        );

        require!(
            Self::status_unsafe(&data) == MessageStatus::Writing,
            CoreBridgeError::NotInWritingStatus
        );

        Ok(true)
    }

#[derive(Accounts)]
pub struct CloseMessageV1<'info> {
    emitter_authority: Signer<'info>,

    /// CHECK: Message account. The payload will be written to and then finalized. This message can
    /// only be published when the message is finalized.
    #[account(
        mut,
        owner = crate::ID,
        constraint = PostedMessageV1::require_draft_message(&draft_message, &emitter_authority)?
    )]
    draft_message: UncheckedAccount<'info>,

    /// CHECK: Destination for lamports if the draft message account is closed.
    #[account(mut)]
    close_account_destination: UncheckedAccount<'info>,
}

pub fn close_message_v1(ctx: Context<CloseMessageV1>) -> Result<()> {
    crate::utils::close_account(
        &ctx.accounts.draft_message,
        &ctx.accounts.close_account_destination,
    )
}

pub(crate) fn close_account(info: &AccountInfo, sol_destination: &AccountInfo) -> Result<()> {
    // Transfer tokens from the account to the sol_destination.
    let dest_starting_lamports = sol_destination.lamports();
    **sol_destination.lamports.borrow_mut() =
        dest_starting_lamports.checked_add(info.lamports()).unwrap();
    **info.lamports.borrow_mut() = 0;

    info.assign(&system_program::ID);
    info.resize(0).map_err(Into::into)
}
