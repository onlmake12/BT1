### Title
`ema_price_no_older_than()` Missing Underflow Guard Causes Revert When `publish_time > current_time` - (File: `target_chains/fuel/contracts/pyth-contract/src/main.sw`)

---

### Summary

The private helper `ema_price_no_older_than()` in the Fuel Pyth contract performs an unchecked unsigned subtraction `current_time - price.publish_time`. In Sway, unsigned integer subtraction panics on underflow. The sibling function `price_no_older_than()` already has an explicit guard for this exact case, but the guard was never applied to the EMA variant. Any caller of the public `ema_price()` or `ema_price_no_older_than()` ABI functions will receive an unexpected arithmetic panic revert whenever a stored EMA price feed carries a `publish_time` that is even one second ahead of the Fuel chain's block timestamp.

---

### Finding Description

In `target_chains/fuel/contracts/pyth-contract/src/main.sw`, the private function `ema_price_no_older_than` computes the age of a price feed with a bare subtraction:

```sway
fn ema_price_no_older_than(time_period: u64, price_feed_id: PriceFeedId) -> Price {
    let price = ema_price_unsafe(price_feed_id);
    let current_time = timestamp();
    require(
        current_time - price.publish_time <= time_period,  // ← underflows if publish_time > current_time
        PythError::OutdatedPrice,
    );
    price
}
``` [1](#0-0) 

The sibling function `price_no_older_than` already carries an explicit comment and guard for this exact scenario:

```sway
fn price_no_older_than(time_period: u64, price_feed_id: PriceFeedId) -> Price {
    let price = price_unsafe(price_feed_id);
    let current_time = timestamp();
    // Mimicking saturating subtraction to avoid underflow
    let time_difference = if current_time > price.publish_time {
        current_time - price.publish_time
    } else {
        0
    };
    require(time_difference <= time_period, PythError::OutdatedPrice);
    price
}
``` [2](#0-1) 

The guard was applied to `price_no_older_than` but was never ported to `ema_price_no_older_than`. Both public ABI entry points `ema_price` and `ema_price_no_older_than` delegate directly to the unguarded private function: [3](#0-2) 

The same pattern is correctly handled in every other Pyth chain implementation:
- **TON**: uses `max(0, current_time - publish_time)` [4](#0-3) 
- **Starknet**: uses a conditional `if current >= publish_time` guard [5](#0-4) 
- **Stylus**: uses `.saturating_sub()` [6](#0-5) 

The Fuel EMA path is the only one that lacks this protection.

---

### Impact Explanation

When a price feed's stored `ema_price.publish_time` is greater than the Fuel chain's `timestamp()` — a realistic condition due to clock skew between Pythnet validators and the Fuel chain, or a price update submitted with a slightly future timestamp — the subtraction `current_time - price.publish_time` wraps around to a very large `u64` value and causes an arithmetic panic revert in Sway.

Any downstream DeFi protocol on Fuel that calls `ema_price()` or `ema_price_no_older_than()` will have all such transactions revert unexpectedly. This is a denial-of-service on the EMA price read path for the duration of the clock skew window.

---

### Likelihood Explanation

Pythnet validators and Fuel chain block producers run independent clocks. Minor clock drift (even a few seconds) between Pythnet and the Fuel chain is a normal operational condition. Price updates are published with the Pythnet timestamp, which can be slightly ahead of the Fuel chain's `block.timestamp`. This is not a theoretical edge case — it is the exact scenario the `price_no_older_than` guard was written to handle, as evidenced by the comment "Mimicking saturating subtraction to avoid underflow" already present in the codebase.

---

### Recommendation

Apply the same saturating-subtraction guard used in `price_no_older_than` to `ema_price_no_older_than`:

```sway
fn ema_price_no_older_than(time_period: u64, price_feed_id: PriceFeedId) -> Price {
    let price = ema_price_unsafe(price_feed_id);
    let current_time = timestamp();
    // Mimicking saturating subtraction to avoid underflow
    let time_difference = if current_time > price.publish_time {
        current_time - price.publish_time
    } else {
        0
    };
    require(time_difference <= time_period, PythError::OutdatedPrice);
    price
}
```

---

### Proof of Concept

1. A Pythnet validator publishes an EMA price update for ETH/USD with `publish_time = T`.
2. The update is relayed and stored on the Fuel chain via `update_price_feeds()`.
3. Due to normal clock skew, the Fuel chain's `timestamp()` at the time of the next read call returns `T - 2` (two seconds behind Pythnet).
4. A user or DeFi protocol calls `ema_price(ETH_USD_FEED_ID)`.
5. Inside `ema_price_no_older_than`, the expression `current_time - price.publish_time` evaluates as `(T - 2) - T`, which is `u64` underflow.
6. Sway's checked arithmetic panics, reverting the transaction with an arithmetic error instead of the expected `PythError::OutdatedPrice`.
7. All callers of `ema_price()` and `ema_price_no_older_than()` are denied service for the duration of the skew window.
