### Title
Stale Price Returned Without Staleness Check in `PythAggregatorV3.latestRoundData()` / `latestAnswer()` - (File: `target_chains/ethereum/sdk/solidity/PythAggregatorV3.sol`)

---

### Summary

`PythAggregatorV3`, Pyth's official Chainlink-compatible adapter contract shipped in `@pythnetwork/pyth-sdk-solidity`, implements `latestRoundData()`, `latestAnswer()`, `getRoundData()`, and `decimals()` exclusively via `getPriceUnsafe()`. This function performs **no staleness check** and can return a price from arbitrarily far in the past. Any protocol that deploys `PythAggregatorV3` as a drop-in Chainlink replacement — which is its stated purpose — will silently consume stale prices with no revert or warning.

---

### Finding Description

`PythAggregatorV3` is Pyth's official adapter that implements Chainlink's `AggregatorV3Interface` on top of Pyth price feeds. It is actively promoted as the migration path for Chainlink integrators.

Every price-returning function in the contract calls `pyth.getPriceUnsafe(priceId)`:

```solidity
// latestAnswer — no staleness check
function latestAnswer() public view virtual returns (int256) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    return int256(price.price);
}

// latestRoundData — no staleness check
function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    roundId = uint80(price.publishTime);
    return (roundId, int256(price.price), price.publishTime, price.publishTime, roundId);
}
``` [1](#0-0) [2](#0-1) 

`getPriceUnsafe()` in the core Pyth contract performs **only** a zero-check on `publishTime` — it applies no age bound:

```solidity
function getPriceUnsafe(bytes32 id) public view override returns (PythStructs.Price memory price) {
    PythInternalStructs.PriceInfo storage info = _state.latestPriceInfo[id];
    price.publishTime = info.publishTime;
    ...
    if (price.publishTime == 0) revert PythErrors.PriceFeedNotFound();
}
``` [3](#0-2) 

The safe alternative, `getPriceNoOlderThan()`, enforces a maximum age via `diff(block.timestamp, price.publishTime)`: [4](#0-3) 

`PythAggregatorV3` never calls this safe variant anywhere.

Additionally, `latestRoundData()` casts `price.publishTime` (a `uint256`) to `uint80` for `roundId` and `answeredInRound`. While this does not overflow with current Unix timestamps, it produces a semantically incorrect `roundId` that downstream Chainlink-style staleness guards (`answeredInRound >= roundId`) will always pass, masking the stale data. [5](#0-4) 

---

### Impact Explanation

Any protocol that deploys `PythAggregatorV3` as a Chainlink feed replacement — the contract's explicit purpose — will:

1. Receive prices from arbitrarily far in the past with no revert.
2. Receive an `updatedAt` value equal to `price.publishTime`, which may be hours or days old, with no on-chain enforcement preventing its use.
3. Have its downstream staleness guard (`answeredInRound >= roundId`) trivially satisfied because both values are set to the same truncated timestamp.

During a rapid market drop, if the Pyth on-chain feed is not updated (e.g., due to network congestion or no keeper calling `updateFeeds`), the adapter continues to return the pre-crash price. An attacker can exploit this to borrow against overvalued collateral, liquidate positions at incorrect prices, or extract value from any price-sensitive operation in the integrating protocol.

---

### Likelihood Explanation

- `PythAggregatorV3` is the official, documented migration path for Chainlink users. Multiple production protocols use it.
- Pyth is a pull oracle; on-chain prices are only updated when someone calls `updateFeeds`. During market stress, updates may lag.
- The `latestRoundData()` interface is the standard entry point for Chainlink-style price consumption. Protocols calling it have no indication that the returned price may be stale.
- No privileged access is required. Any user interacting with a downstream protocol that uses this adapter is exposed.

---

### Recommendation

Replace all `getPriceUnsafe()` calls in `PythAggregatorV3` with `getPriceNoOlderThan(priceId, maxAge)`, where `maxAge` is a configurable constructor parameter (e.g., defaulting to 3600 seconds). Alternatively, add an explicit staleness revert inside each price-returning function:

```solidity
uint256 constant MAX_PRICE_AGE = 3600; // configurable

function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    require(
        block.timestamp - price.publishTime <= MAX_PRICE_AGE,
        "Stale price"
    );
    roundId = uint80(price.publishTime);
    return (roundId, int256(price.price), price.publishTime, price.publishTime, roundId);
}
```

The safe `getPriceNoOlderThan` already exists in `AbstractPyth.sol` and should be used directly. [4](#0-3) 

---

### Proof of Concept

1. Deploy `PythAggregatorV3` pointing to the Pyth contract for ETH/USD.
2. Call `updateFeeds` once to set a price of $3000 at `publishTime = T`.
3. Advance block time by 24 hours (no further `updateFeeds` calls).
4. Call `latestRoundData()`:
   - Returns `answer = 3000e8`, `updatedAt = T` (24 hours ago), `roundId = answeredInRound = uint80(T)`.
   - No revert occurs.
5. A lending protocol consuming this adapter accepts $3000 as the current ETH price.
6. If the real market price is $1500, an attacker borrows against ETH collateral at the stale $3000 valuation, extracting double the intended credit.

The root cause is entirely within Pyth's own production SDK file `PythAggregatorV3.sol` — no third-party oracle misbehavior is required. [1](#0-0) [6](#0-5) [5](#0-4)
