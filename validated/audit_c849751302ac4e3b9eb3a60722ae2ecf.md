### Title
`onlyWhitelistedReader` Checks Wrong `msg.sender` in `getPricesNoOlderThan` / `getEmaPricesNoOlderThan`, Permanently Breaking Staleness-Checked Price Reads for Whitelisted Users — (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`Scheduler.getPricesNoOlderThan` and `Scheduler.getEmaPricesNoOlderThan` each apply the `onlyWhitelistedReader` modifier to the outer call, then delegate to `this.getPricesUnsafe` / `this.getEmaPricesUnsafe` via an **external** `this.` call. Because `this.` produces an external call in Solidity, `msg.sender` inside the inner function is the **Scheduler contract itself**, not the original caller. The inner function also carries `onlyWhitelistedReader`, which checks `msg.sender` against the subscription's whitelist. The Scheduler contract is never the subscription manager and is never in any whitelist, so the inner call always reverts with `Unauthorized` whenever `whitelistEnabled == true`. The staleness-checked read path is therefore permanently broken for every whitelisted user on every whitelist-enabled subscription.

---

### Finding Description

`onlyWhitelistedReader` is defined as:

```solidity
modifier onlyWhitelistedReader(uint256 subscriptionId) {
    if (_state.subscriptionManager[subscriptionId] == msg.sender) { _; return; }
    if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) { _; return; }
    // ... iterate whitelist, revert if msg.sender not found
}
``` [1](#0-0) 

`getPricesNoOlderThan` applies this modifier and then calls:

```solidity
prices = this.getPricesUnsafe(subscriptionId, priceIds);
``` [2](#0-1) 

`getPricesUnsafe` is declared `external` and also carries `onlyWhitelistedReader`:

```solidity
function getPricesUnsafe(...) external view override onlyWhitelistedReader(subscriptionId) ...
``` [3](#0-2) 

In Solidity, `this.f()` is an external call; `msg.sender` inside `getPricesUnsafe` is therefore `address(this)` — the Scheduler contract — not the original EOA or contract that initiated the transaction. The Scheduler contract is never registered as a subscription manager and is never added to any `readerWhitelist`, so the modifier always reaches the final `revert SchedulerErrors.Unauthorized()` branch when `whitelistEnabled == true`.

The identical defect exists in `getEmaPricesNoOlderThan` → `this.getEmaPricesUnsafe`: [4](#0-3) 

---

### Impact Explanation

Any subscription created with `whitelistEnabled = true` renders `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` permanently unusable for all callers, including the subscription manager. Downstream integrators who rely on these functions for staleness-bounded price reads are silently forced to fall back to `getPricesUnsafe` / `getEmaPricesUnsafe`, which perform no recency check. This defeats the safety guarantee those functions are designed to provide and can cause integrators to consume arbitrarily stale prices without any on-chain revert, directly undermining the correctness of any protocol that depends on the Scheduler's staleness-checked API.

---

### Likelihood Explanation

The whitelist feature is an explicitly supported configuration option (`whitelistEnabled` flag in `SubscriptionParams`). Any subscription manager who enables it — a normal, unprivileged action — immediately triggers the broken code path. No special privileges, leaked keys, or external oracle manipulation are required. The trigger is a standard subscription configuration choice.

---

### Recommendation

Replace the external `this.` delegation with a direct call to the internal helper `_getPricesInternal`, which already contains the shared retrieval logic and does not carry `onlyWhitelistedReader`. The staleness check can remain in the outer function, which has already verified the caller:

```solidity
function getPricesNoOlderThan(...) external view onlyWhitelistedReader(subscriptionId) ... {
    if (distance(block.timestamp, status.priceLastUpdatedAt) > age_seconds)
        revert PythErrors.StalePrice();
    // Call internal helper directly instead of this.getPricesUnsafe(...)
    PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
    ...
}
```

Apply the same fix to `getEmaPricesNoOlderThan`.

---

### Proof of Concept

1. Deploy `Scheduler` (or use a live instance).
2. Create a subscription with `whitelistEnabled = true` and add address `alice` to `readerWhitelist`.
3. Call `updatePriceFeeds` to populate price data.
4. As `alice`, call `getPricesUnsafe(subscriptionId, priceIds)` → **succeeds** (alice is whitelisted, `msg.sender == alice`).
5. As `alice`, call `getPricesNoOlderThan(subscriptionId, priceIds, 3600)` → **reverts with `Unauthorized`**.
   - Outer call: `msg.sender == alice` → passes `onlyWhitelistedReader`.
   - Inner `this.getPricesUnsafe(...)`: `msg.sender == address(Scheduler)` → not manager, not in whitelist → `revert Unauthorized`.
6. The same revert occurs for `getEmaPricesNoOlderThan`.

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
