### Title
Division by Zero in TWAP Calculation When `startSlot == endSlot` - (File: `target_chains/ethereum/contracts/contracts/pyth/Pyth.sol`)

### Summary
The `calculateTwap` function in `Pyth.sol` divides by `slotDiff` (the difference between end and start `publishSlot`) without verifying it is non-zero. The upstream `validateTwapPriceInfo` guard only checks `startSlot <= endSlot` (using strict `>`), which permits equal slots. When a caller submits valid Wormhole Merkle-attested TWAP update data whose start and end snapshots share the same `publishSlot`, all three divisions in `calculateTwap` revert with a division-by-zero panic, making `parseTwapPriceFeedUpdates` permanently unusable for that slot pair.

### Finding Description
`parseTwapPriceFeedUpdates` is a public payable entry point that accepts two Wormhole Merkle-attested TWAP update blobs (start and end), validates them, and calls `calculateTwap`.

`validateTwapPriceInfo` enforces ordering with:

```solidity
if (twapPriceInfoStart.publishSlot > twapPriceInfoEnd.publishSlot) {
    revert PythErrors.InvalidTwapUpdateDataSet();
}
``` [1](#0-0) 

This check uses strict `>`, so equal slots (`startSlot == endSlot`) pass validation. `calculateTwap` then computes:

```solidity
uint64 slotDiff = twapPriceInfoEnd.publishSlot - twapPriceInfoStart.publishSlot;
// slotDiff == 0 when slots are equal
int128 twapPrice = priceDiff / int128(uint128(slotDiff));   // DIV/0
uint128 twapConf  = confDiff  / uint128(slotDiff);           // DIV/0
uint64 downSlotsRatio = (totalDownSlots * 1_000_000) / slotDiff; // DIV/0
``` [2](#0-1) 

All three divisions use `slotDiff` as the denominator with no zero-guard. This is structurally identical to the reported `scoreAmplifiers[1] - scoreAmplifiers[1]` bug: a denominator that can evaluate to zero due to a missing or incorrect boundary check.

### Impact Explanation
Any call to `parseTwapPriceFeedUpdates` where the two provided Wormhole Merkle proofs carry the same `publishSlot` will revert with a Solidity panic (division by zero). This completely denies TWAP price feed data to any on-chain consumer (e.g., DeFi protocols relying on Pyth TWAP) for that slot pair. The fee paid by the caller is consumed and no price data is returned.

### Likelihood Explanation
Pyth publishes TWAP accumulator snapshots at each slot. A caller using Hermes to fetch a very short TWAP window (or querying two snapshots that happen to fall on the same slot boundary) can legitimately obtain two valid, guardian-signed Wormhole Merkle proofs with identical `publishSlot` values. The data passes all existing validation in `validateTwapPriceInfo` and proceeds directly to the unguarded division. No key compromise or privileged access is required — only a valid pair of Pyth-attested TWAP blobs.

### Recommendation
Add an explicit non-zero check for `slotDiff` in `validateTwapPriceInfo` (or at the top of `calculateTwap`):

```solidity
if (twapPriceInfoStart.publishSlot >= twapPriceInfoEnd.publishSlot) {
    revert PythErrors.InvalidTwapUpdateDataSet();
}
```

Changing `>` to `>=` at line 604 ensures `slotDiff` is always at least 1 before any division is attempted. [3](#0-2) 

### Proof of Concept
1. Obtain two valid Wormhole Merkle-attested TWAP update blobs from Hermes where both blobs encode the same `publishSlot` (e.g., `publishSlot = 1000` for both start and end).
2. Call `parseTwapPriceFeedUpdates{value: fee}(updateData, priceIds)`.
3. `validateTwapPriceInfo` passes: `1000 > 1000` is `false`, so no revert.
4. `calculateTwap` executes: `slotDiff = 1000 - 1000 = 0`.
5. `priceDiff / int128(uint128(0))` → Solidity panic 0x12 (division by zero), transaction reverts. [4](#0-3)

    function validateTwapPriceInfo(
        PythStructs.TwapPriceInfo memory twapPriceInfoStart,
        PythStructs.TwapPriceInfo memory twapPriceInfoEnd
    ) private pure {
        // First validate each individual price's uniqueness
        if (
            twapPriceInfoStart.prevPublishTime >= twapPriceInfoStart.publishTime
        ) {
            revert PythErrors.InvalidTwapUpdateData();
        }
        if (twapPriceInfoEnd.prevPublishTime >= twapPriceInfoEnd.publishTime) {
            revert PythErrors.InvalidTwapUpdateData();
        }

        // Then validate the relationship between the two data points

            revert PythErrors.InvalidTwapUpdateDataSet();

```text
    function calculateTwap(
        bytes32 priceId,
        PythStructs.TwapPriceInfo memory twapPriceInfoStart,
        PythStructs.TwapPriceInfo memory twapPriceInfoEnd
    ) private pure returns (PythStructs.TwapPriceFeed memory twapPriceFeed) {
        twapPriceFeed.id = priceId;
        twapPriceFeed.startTime = twapPriceInfoStart.publishTime;
        twapPriceFeed.endTime = twapPriceInfoEnd.publishTime;

        // Calculate differences between start and end points for slots and cumulative values
        uint64 slotDiff = twapPriceInfoEnd.publishSlot -
            twapPriceInfoStart.publishSlot;
        int128 priceDiff = twapPriceInfoEnd.cumulativePrice -
            twapPriceInfoStart.cumulativePrice;
        uint128 confDiff = twapPriceInfoEnd.cumulativeConf -
            twapPriceInfoStart.cumulativeConf;

        // Calculate time-weighted average price (TWAP) and confidence by dividing
        // the difference in cumulative values by the number of slots between data points
        int128 twapPrice = priceDiff / int128(uint128(slotDiff));
        uint128 twapConf = confDiff / uint128(slotDiff);

        // The conversion from int128 to int64 is safe because:
        // 1. Individual prices fit within int64 by protocol design
        // 2. TWAP is essentially an average price over time (cumulativePrice₂-cumulativePrice₁)/slotDiff
        // 3. This average must be within the range of individual prices that went into the calculation
        // We use int128 only as an intermediate type to safely handle cumulative sums
        twapPriceFeed.twap.price = int64(twapPrice);
        twapPriceFeed.twap.conf = uint64(twapConf);
        twapPriceFeed.twap.expo = twapPriceInfoStart.expo;
        twapPriceFeed.twap.publishTime = twapPriceInfoEnd.publishTime;

        // Calculate downSlotsRatio as a value between 0 and 1,000,000
        // 0 means no slots were missed, 1,000,000 means all slots were missed
        uint64 totalDownSlots = twapPriceInfoEnd.numDownSlots -
            twapPriceInfoStart.numDownSlots;
        uint64 downSlotsRatio = (totalDownSlots * 1_000_000) / slotDiff;
```
