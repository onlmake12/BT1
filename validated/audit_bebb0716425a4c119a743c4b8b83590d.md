### Title
Peer Store Permanently Blocked via Crafted Discovery Addresses Exploiting `check_purge` Eviction Dead Zone — (`network/src/peer_store/peer_store_impl.rs`)

---

### Summary

The `check_purge` function's second-pass eviction only removes peers from network groups containing **more than 4** entries. An unprivileged remote peer can fill the peer store with exactly 4 addresses per /16 subnet across 4096 distinct subnets (totalling `ADDR_COUNT_LIMIT = 16384`), causing every subsequent `add_addr` call to return `Err(PeerStoreError::EvictionFailed)` permanently, blocking all honest peer discovery for the lifetime of the node.

---

### Finding Description

`check_purge` is called on every `add_addr` invocation once the store reaches capacity. It has two eviction passes:

**Pass 1** — evict non-connectable addresses: [1](#0-0) 

Freshly advertised addresses have `last_connected_at_ms=0` and `attempts_count=0`. With `ADDR_MAX_RETRIES=3`, `is_connectable` returns `true` for all of them: [2](#0-1) 

So Pass 1 yields zero candidates and falls through to Pass 2.

**Pass 2** — evict from over-represented network groups: [3](#0-2) 

The critical guard is `if addrs.len() > 4`. If every group has **exactly 4** entries, `flat_map` returns `None` for every group, `candidate_peers` is empty, and the function returns `Err(PeerStoreError::EvictionFailed)`.

The network group key for IPv4 is the **first two octets** (a /16 subnet), not /24: [4](#0-3) 

So the attacker needs 4096 distinct /16 subnets × 4 addresses each = 16384 = `ADDR_COUNT_LIMIT`: [5](#0-4) 

---

### Impact Explanation

Once the store is in this state, `add_new_addrs` in `DiscoveryAddressManager` silently swallows the error: [6](#0-5) 

No new honest peer addresses can ever be stored. `fetch_addrs_to_feeler` and `fetch_addrs_to_attempt` will only return attacker-controlled (unreachable) addresses, permanently degrading outbound peer discovery for the node's lifetime without any crash or visible error.

---

### Likelihood Explanation

The attack path is fully reachable from a single connected peer via the Discovery protocol. `verify_nodes_message` allows up to `MAX_ADDR_TO_SEND=1000` items per non-announce message with up to `MAX_ADDRS=3` addresses per item: [7](#0-6) 

That is up to 3000 addresses per message. Filling 16384 slots requires only ~6 messages. The addresses must pass `is_valid_addr` (public IPs only), but 4096 distinct /16 public subnets are trivially constructable from the public IPv4 space. No privileged access, no PoW, no Sybil attack is required.

---

### Recommendation

Replace the hard `> 4` guard with a proportional eviction strategy that always produces at least one candidate when the store is full. For example, when no group exceeds 4, fall back to evicting the single oldest (lowest `last_connected_at_ms`) or lowest-scored entry unconditionally. Additionally, enforce a per-session cap on how many new addresses `add_new_addrs` will accept in a single call to limit the blast radius of a single malicious peer.

---

### Proof of Concept

```rust
// Pre-fill addr_manager with 16384 entries: 4096 /16 groups × 4 addresses each
// All entries: last_connected_at_ms=0, attempts_count=0 → is_connectable=true
let mut peer_store = PeerStore::default();
for subnet_a in 1u8..=16u8 {          // 16 class-A blocks
    for subnet_b in 0u8..=255u8 {      // 256 class-B blocks each → 4096 /16 groups
        for host in 1u8..=4u8 {        // 4 hosts per group
            let addr: Multiaddr = format!(
                "/ip4/{}.{}.1.{}/tcp/8115/p2p/{}",
                subnet_a, subnet_b, host,
                PeerId::random().to_base58()
            ).parse().unwrap();
            peer_store.add_addr(addr, Flags::COMPATIBILITY).unwrap();
        }
    }
}
assert_eq!(peer_store.addr_manager().count(), 16384);

// Now any new add_addr must fail
let fresh: Multiaddr = format!(
    "/ip4/200.0.0.1/tcp/8115/p2p/{}", PeerId::random().to_base58()
).parse().unwrap();
let result = peer_store.add_addr(fresh, Flags::COMPATIBILITY);
assert!(matches!(result, Err(e) if e.to_string().contains("EvictionFailed")));
// No honest peer can be added hereafter.
```

    pub fn is_connectable(&self, now_ms: u64) -> bool {
        // do not remove addr tried in last minute
        if self.tried_in_last_minute(now_ms) {
            return true;
        }
        // we give up if never connect to this addr
        if self.last_connected_at_ms == 0 && self.attempts_count >= ADDR_MAX_RETRIES {
            return false;
        }
        // consider addr is not connectable if failed too many times
        if now_ms.saturating_sub(self.last_connected_at_ms) > ADDR_TIMEOUT_MS
            && (self.attempts_count >= ADDR_MAX_FAILURES)
        {
            return false;
        }
        true
    }
