### Title
Missing IBD Phase Guard in `GetBlocksProcess` Allows Inbound Peers to Probe Block Inventory During Initial Block Download — (File: sync/src/synchronizer/get_blocks_process.rs)

### Summary
The CKB synchronizer's `GetBlocksProcess` handler does not check whether the local node is in Initial Block Download (IBD) mode before processing incoming `GetBlocks` requests. The module documentation explicitly states that during IBD the node "will respond `packed::InIBD` to `GetHeaders` and `GetBlocks` requests," but the `GetBlocksProcess::execute` implementation contains no such guard. The analogous `GetHeadersProcess::execute` does perform this check. This is a direct instance of the "confusing phase transitions" class: the IBD phase boundary is enforced inconsistently across handlers, with the intended invariant documented but not implemented in one of the two named handlers.

### Finding Description
The `sync/src/synchronizer/mod.rs` module comment declares the IBD contract:

> "When CKB node is in IBD mode, it will respond `packed::InIBD` to `GetHeaders` and `GetBlocks` requests"

`GetHeadersProcess::execute` correctly enforces this:

```rust
if active_chain.is_initial_block_download() {
    self.send_in_ibd();
    ...
    return Status::ignored();
}
```

`GetBlocksProcess::execute` contains no equivalent check. Any peer that connects during IBD and sends a `GetBlocks` message will receive full block bodies for any hashes it supplies that happen to have `BLOCK_VALID` status, without the node first sending `InIBD` or refusing the request. The node's IBD phase transition is therefore not uniformly enforced: the same conceptual "am I in IBD?" gate is applied to `GetHeaders` but silently omitted from `GetBlocks`.

The `Relayer::received` handler does guard with `is_initial_block_download()` at the top level, so the relay protocol is protected. The sync protocol's `GetBlocks` path is the gap.

### Impact Explanation
During IBD the node is supposed to dedicate its resources to a single outbound sync peer and refuse to serve block data to arbitrary inbound peers. Without the IBD guard in `GetBlocksProcess`:

1. **Resource exhaustion / bandwidth drain**: Any inbound peer can send up to `MAX_HEADERS_LEN` (2048) block hashes per message. For each hash that resolves to a `BLOCK_VALID` block, the node serializes and sends the full block body. An attacker with many inbound connections can force the IBD node to spend significant I/O and CPU serving block data while it is trying to catch up.
2. **Block inventory probing**: An attacker can enumerate which block hashes the node already has verified (`BLOCK_VALID`) by observing which hashes elicit a `SendBlock` response vs. a silent skip. This leaks information about the node's sync progress during IBD.
3. **Violation of the stated IBD invariant**: The documented protocol contract ("respond `InIBD` to `GetBlocks`") is broken, which can confuse well-behaved peers that rely on the `InIBD` signal to understand the node's state.

### Likelihood Explanation
Any unprivileged inbound peer can trigger this path. During IBD the node accepts inbound connections. Sending a `GetBlocks` message over the Sync protocol is a standard, low-cost operation. No special privileges, keys, or majority hashpower are required. The attacker only needs to establish a TCP connection and send a valid `SyncMessage::GetBlocks` molecule-encoded message.

### Recommendation
Add an IBD guard at the top of `GetBlocksProcess::execute`, mirroring the pattern in `GetHeadersProcess::execute`:

```rust
pub fn execute(self) -> Status {
    let active_chain = self.synchronizer.shared.active_chain();

    if active_chain.is_initial_block_download() {
        // Send InIBD and ignore, consistent with GetHeadersProcess and the
        // documented module invariant.
        return Status::ignored();
    }
    // ... rest of existing logic
}
```

Optionally, also send the `InIBD` message to the peer (as `GetHeadersProcess` does) so well-behaved peers can react correctly.

```rust
    pub fn execute(self) -> Status {
        let block_hashes = self.message.block_hashes();
        // use MAX_HEADERS_LEN as limit, we may increase the value of INIT_BLOCKS_IN_TRANSIT_PER_PEER in the future
        if block_hashes.len() > MAX_HEADERS_LEN {
            return StatusCode::ProtocolMessageIsMalformed.with_context(format!(
                "BlockHashes count({}) > MAX_HEADERS_LEN({})",
                block_hashes.len(),
                MAX_HEADERS_LEN,
            ));
        }
        let active_chain = self.synchronizer.shared.active_chain();

        let iter = block_hashes.iter().take(INIT_BLOCKS_IN_TRANSIT_PER_PEER);

        let mut dedup = HashSet::new();
        for block_hash in iter {
            debug!("get_blocks {} from peer {:?}", block_hash, self.peer);
            let block_hash = block_hash.to_entity();

            if block_hash == self.synchronizer.shared().consensus().genesis_hash() {
                return StatusCode::RequestGenesis.with_context("Request genesis block");
            }

            if !dedup.insert(block_hash.clone()) {
                return StatusCode::RequestDuplicate.with_context("Request duplicate block");
            }

            if !active_chain.contains_block_status(&block_hash, BlockStatus::BLOCK_VALID) {
                debug!(
                    "Ignoring get_block {} request from peer={} as it is not verified.",
                    block_hash, self.peer
                );
                continue;
            }

            if let Some(block) = active_chain.get_block(&block_hash) {
                debug!(
                    "respond_block {} {} to peer {:?}",
                    block.number(),
                    block.hash(),
                    self.peer,
                );
                let content = packed::SendBlock::new_builder().block(block.data()).build();
                let message = packed::SyncMessage::new_builder().set(content).build();

                let nc = Arc::clone(self.nc);
                self.synchronizer
                    .shared()
                    .shared()
                    .async_handle()
                    .spawn(async move { async_send_message_to(&nc, self.peer, &message).await });
            } else {
                // TODO response not found
                // TODO add timeout check in synchronizer

                // We expect that `block_hashes` is sorted descending by height.
                // So if we cannot find the current one from local, we cannot find
                // the next either.
                debug!("Stopping getblocks, since {} is not found", block_hash);
                break;
            }
        }

        Status::ok()
    }
```

```rust
        // Once this function has returned false, it must remain false.
        if self.ibd_finished.load(Ordering::Acquire) {
            false
        } else if unix_time_as_millis().saturating_sub(self.snapshot().tip_header().timestamp())

            true
        } else {
            self.ibd_finished.store(true, Ordering::Release);
            false
```
