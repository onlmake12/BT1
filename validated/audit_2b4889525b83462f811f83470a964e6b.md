### Title
Consensus `[params]` accepted without boundary validation — deferred node panic on epoch transition triggered by external block processing — (`spec/src/consensus.rs`)

### Summary

Multiple critical consensus parameters read from the chain spec `[params]` TOML section are passed directly into `build_genesis_epoch_ext()` and `ConsensusBuilder` without any boundary validation. Two distinct failure modes exist:

1. `genesis_epoch_length = 0` → immediate integer division-by-zero panic at node startup inside `build_genesis_epoch_ext()`.
2. `orphan_rate_target = (x, 0)` → zero-denominator `RationalU256` silently stored via `new_raw()` (bypassing the checked constructor), causing a deferred panic during the first epoch transition that has uncles — triggered by processing a valid block from an external peer.

The `ConsensusBuilder::build()` uses `debug_assert!` for a subset of checks, but `debug_assert!` is compiled out in release builds, and neither `genesis_epoch_length` nor `orphan_rate_target` denominator is checked even in debug builds.

### Finding Description

**Root cause 1 — `genesis_epoch_length = 0`**

`build_genesis_epoch_ext()` performs integer division by `genesis_epoch_length` with no prior guard:

```rust
// spec/src/consensus.rs:222-223
let block_reward = Capacity::shannons(epoch_reward.as_u64() / genesis_epoch_length);
let remainder_reward = Capacity::shannons(epoch_reward.as_u64() % genesis_epoch_length);
```

If `genesis_epoch_length = 0` is set in `[params]`, Rust panics with integer division by zero at node startup when `build_consensus()` calls this function. [1](#0-0) 

**Root cause 2 — `orphan_rate_target = (x, 0)`**

`ConsensusBuilder::orphan_rate_target()` stores the value using `RationalU256::new_raw()`, which explicitly skips the zero-denominator check:

```rust
// spec/src/consensus.rs:387-393
pub fn orphan_rate_target(mut self, orphan_rate_target: (u32, u32)) -> Self {
    self.inner.orphan_rate_target = RationalU256::new_raw(   // ← bypasses zero check
        U256::from(orphan_rate_target.0),
        U256::from(orphan_rate_target.1),
    );
    self
}
``` [2](#0-1) 

`RationalU256::new()` (the checked constructor) explicitly panics on zero denominator: [3](#0-2) 

The zero-denominator value is then used in `next_epoch_ext()` during epoch-boundary arithmetic. When `epoch_uncles_count > 0`, the code computes:

```rust
// spec/src/consensus.rs:887-896
let numerator = orphan_rate_target * ... * &last_epoch_length_u256;
let denominator = &last_orphan_rate * (orphan_rate_target + U256::one()) * &last_epoch_duration;
let raw_next_epoch_length = u256_low_u64((numerator / denominator).into_u256());
```

`into_u256()` performs `self.numer / self.denom`. Because `orphan_rate_target.denom = 0` propagates through the `new_raw()`-based `Mul` impl, the final `denom` is zero, and `into_u256()` panics. [4](#0-3) 

**The `debug_assert!` guards in `ConsensusBuilder::build()` are release-build no-ops and do not cover either parameter:** [5](#0-4) 

**The chain spec `Params` struct accepts both fields as plain `Option<u64>` / `Option<(u32,u32)>` with no validation:** [6](#0-5) 

**`build_consensus()` passes them directly to `build_genesis_epoch_ext()` and `ConsensusBuilder` without any guard:** [7](#0-6) 

### Impact Explanation

For `genesis_epoch_length = 0`: the node panics immediately at startup and cannot be started until the spec is corrected. The chain is permanently halted for that operator.

For `orphan_rate_target = (x, 0)`: the node starts and syncs normally. The panic is deferred until the first epoch boundary where at least one uncle block exists. At that point, any external peer that relays a valid block completing an epoch with uncles triggers `next_epoch_ext()`, which panics and crashes the node. The node cannot recover without correcting the spec and restarting. On a private/dev chain this is a permanent chain halt; on a network with multiple nodes, the misconfigured node is permanently partitioned.

### Likelihood Explanation

Low. Mainnet and testnet use bundled, correct specs. The risk applies to dev/private chain operators who customize `[params]`. The misconfiguration can be accidental (e.g., setting `orphan_rate_target = (0, 0)` intending to disable orphan rate adjustment) or deliberate (malicious chain deployer). The `orphan_rate_target` case is particularly subtle because the node starts successfully and the panic only manifests later when triggered by normal block propagation from any peer.

### Recommendation

1. Add explicit validation in `build_consensus()` (or `Params` accessors) rejecting `genesis_epoch_length == 0`, `epoch_duration_target == 0`, and `orphan_rate_target.1 == 0` with a descriptive `Err` return rather than a panic.
2. Replace `RationalU256::new_raw()` with `RationalU256::new()` in `ConsensusBuilder::orphan_rate_target()` so the zero-denominator is caught eagerly at construction time.
3. Promote the `debug_assert!` guards in `ConsensusBuilder::build()` to hard `assert!` (or return `Result`) so they fire in release builds.

### Proof of Concept

**Case 1 — immediate panic:**
```toml
# specs/dev.toml [params]
genesis_epoch_length = 0
```
Run `ckb run`. `build_consensus()` → `build_genesis_epoch_ext()` → `epoch_reward.as_u64() / 0` → thread panics with "attempt to divide by zero".

**Case 2 — deferred panic triggered by external block:**
```toml
# specs/dev.toml [params]
orphan_rate_target = [1, 0]   # denominator = 0
```
Node starts. Mine blocks normally. When the first epoch ends and the epoch-tail block has `epoch_uncles_count > 0` (any peer relays a block with an uncle), `next_epoch_ext()` is called, `orphan_rate_target` arithmetic produces a zero-denominator `RationalU256`, `into_u256()` executes `numer / 0`, and the node panics.

    pub fn build_consensus(&self) -> Result<Consensus, Box<dyn Error>> {
        let hardfork_switch = self.build_hardfork_switch()?;
        let genesis_epoch_ext = build_genesis_epoch_ext(
            self.params.initial_primary_epoch_reward(),
            self.genesis.compact_target,
            self.params.genesis_epoch_length(),
            self.params.epoch_duration_target(),
            self.params.orphan_rate_target(),

        self.verify_genesis_hash(&genesis_block)?;

        let mut builder = ConsensusBuilder::new(genesis_block, genesis_epoch_ext)
            .id(self.name.clone())
            .cellbase_maturity(EpochNumberWithFraction::from_full_value(
                self.params.cellbase_maturity(),
            ))
            .secondary_epoch_reward(self.params.secondary_epoch_reward())
            .max_block_cycles(self.params.max_block_cycles())
            .max_block_bytes(self.params.max_block_bytes())
            .pow(self.pow.clone())
            .satoshi_pubkey_hash(self.genesis.satoshi_gift.satoshi_pubkey_hash.clone())
            .satoshi_cell_occupied_ratio(self.genesis.satoshi_gift.satoshi_cell_occupied_ratio)
            .primary_epoch_reward_halving_interval(
                self.params.primary_epoch_reward_halving_interval(),
            )
            .initial_primary_epoch_reward(self.params.initial_primary_epoch_reward())
            .epoch_duration_target(self.params.epoch_duration_target())
            .permanent_difficulty_in_dummy(self.params.permanent_difficulty_in_dummy())
            .max_block_proposals_limit(self.params.max_block_proposals_limit())
            .orphan_rate_target(self.params.orphan_rate_target())
            .starting_block_limiting_dao_withdrawing_lock(
                self.params.starting_block_limiting_dao_withdrawing_lock(),
            )
            .hardfork_switch(hardfork_switch);

        if let Some(deployments) = self.softfork_deployments() {
            builder = builder.softfork_deployments(deployments);
