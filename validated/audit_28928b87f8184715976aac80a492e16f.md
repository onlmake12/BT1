### Title
Missing Validation for `whitelistEnabled=true` with Empty `readerWhitelist` in `Scheduler._validateSubscriptionParams` Permanently Locks All Non-Manager Readers for Permanent Subscriptions — (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`Scheduler._validateSubscriptionParams()` validates several interdependent parameter combinations (e.g., `updateOnHeartbeat && heartbeatSeconds == 0`) but omits a check for the combination `whitelistEnabled = true` with `readerWhitelist.length == 0`. For permanent subscriptions (`isPermanent = true`), this creates an irreversible state where all non-manager callers are permanently locked out from reading price data, because permanent subscriptions cannot be updated.

---

### Finding Description

`_validateSubscriptionParams` in `Scheduler.sol` performs the following checks:

```solidity
// Validates heartbeat criteria
if (params.updateCriteria.updateOnHeartbeat && params.updateCriteria.heartbeatSeconds == 0) {
    revert SchedulerErrors.InvalidUpdateCriteria();
}
// Validates deviation criteria
if (params.updateCriteria.updateOnDeviation && params.updateCriteria.deviationThresholdBps == 0) {
    revert SchedulerErrors.InvalidUpdateCriteria();
}
// @audit missing: if (params.whitelistEnabled && params.readerWhitelist.length == 0) { revert ... }
```

The function checks that a flag (`updateOnHeartbeat`) is not enabled without its required companion value (`heartbeatSeconds > 0`), but applies no equivalent check for `whitelistEnabled` requiring at least one entry in `readerWhitelist`.

The `onlyWhitelistedReader` modifier enforces access as follows:

```solidity
modifier onlyWhitelistedReader(uint256 subscriptionId) {
    if (_state.subscriptionManager[subscriptionId] == msg.sender) { _; return; }
    if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) { _; return; }
    // iterates whitelist — if empty, isWhitelisted stays false
    if (!isWhitelisted) { revert SchedulerErrors.Unauthorized(); }
    _;
}
```

When `whitelistEnabled = true` and `readerWhitelist` is empty, the loop finds no match and every non-manager caller is rejected. This modifier gates all four price-reading functions: `getPricesUnsafe`, `getPricesNoOlderThan`, `getEmaPricesUnsafe`, and `getEmaPricesNoOlderThan`.

For permanent subscriptions, `updateSubscription` unconditionally reverts:

```solidity
if (currentParams.isPermanent) {
    revert SchedulerErrors.CannotUpdatePermanentSubscription();
}
```

There is no other code path to modify subscription parameters. The misconfiguration is therefore **irreversible** once a permanent subscription is created with `whitelistEnabled = true` and an empty `readerWhitelist`.

---

### Impact Explanation

Any protocol or contract that creates a permanent subscription with `whitelistEnabled = true` and an empty `readerWhitelist` will permanently lock out all non-manager callers from reading price data. The subscription continues to receive keeper updates (keepers call `updatePriceFeeds`, which has no whitelist check), but the stored prices become permanently inaccessible to all downstream consumers. The manager address is the only address that can read prices, and there is no mechanism to add readers or disable the whitelist after the fact. This is a permanent, irreversible denial of service for all users of the affected subscription.

---

### Likelihood Explanation

A protocol deploying a permanent subscription might intend to add readers after creation, or might assume that `whitelistEnabled = true` with no entries is equivalent to "open" access. The validation function already establishes the pattern of checking flag+companion-value pairs (heartbeat, deviation), so a developer reading the code would reasonably expect the same pattern to apply to the whitelist flag. The absence of a guard makes this a plausible misconfiguration, particularly for permanent subscriptions where the stakes of getting the initial parameters wrong are highest.

---

### Recommendation

Add the following check inside `_validateSubscriptionParams`:

```solidity
if (params.whitelistEnabled && params.readerWhitelist.length == 0) {
    revert SchedulerErrors.EmptyWhitelist();
}
```

Add the corresponding error to `SchedulerErrors`:

```solidity
error EmptyWhitelist();
```

    function _validateSubscriptionParams(
        SchedulerStructs.SubscriptionParams memory params
    ) internal pure {
        // No zero‐feed subscriptions
        if (params.priceIds.length == 0) {
            revert SchedulerErrors.EmptyPriceIds();
        }

        // Price ID limits and uniqueness
        if (
            params.priceIds.length >
            SchedulerConstants.MAX_PRICE_IDS_PER_SUBSCRIPTION
        ) {
            revert SchedulerErrors.TooManyPriceIds(
                params.priceIds.length,
                SchedulerConstants.MAX_PRICE_IDS_PER_SUBSCRIPTION
            );
        }
        for (uint i = 0; i < params.priceIds.length; i++) {
            for (uint j = i + 1; j < params.priceIds.length; j++) {
                if (params.priceIds[i] == params.priceIds[j]) {
                    revert SchedulerErrors.DuplicatePriceId(params.priceIds[i]);
                }
            }
        }

        // Whitelist size limit and uniqueness
        if (params.readerWhitelist.length > MAX_READER_WHITELIST_SIZE) {
            revert SchedulerErrors.TooManyWhitelistedReaders(
                params.readerWhitelist.length,
                MAX_READER_WHITELIST_SIZE
            );
        }
        for (uint i = 0; i < params.readerWhitelist.length; i++) {
            for (uint j = i + 1; j < params.readerWhitelist.length; j++) {
                if (params.readerWhitelist[i] == params.readerWhitelist[j]) {
                    revert SchedulerErrors.DuplicateWhitelistAddress(
                        params.readerWhitelist[i]
                    );
                }
            }
        }

        // Validate update criteria
        if (
            !params.updateCriteria.updateOnHeartbeat &&
            !params.updateCriteria.updateOnDeviation
        ) {
            revert SchedulerErrors.InvalidUpdateCriteria();
        }
        if (
            params.updateCriteria.updateOnHeartbeat &&
            params.updateCriteria.heartbeatSeconds == 0
        ) {
            revert SchedulerErrors.InvalidUpdateCriteria();
        }

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
