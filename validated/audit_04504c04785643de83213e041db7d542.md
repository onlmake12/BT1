Audit Report

## Title
Inconsistent Address Normalization in `AddrManager::add()` Bypasses Score-Based Peer Banning for WebSocket Peers — (`network/src/peer_store/addr_manager.rs`)

## Summary
`AddrManager::add()` stores entries keyed by the raw `Multiaddr` (including `/ws`, `/wss`, `/tls`, `/memory` suffixes), while every read/write operation — `get()`, `get_mut()`, and `remove()` — normalizes via `base_addr()` before lookup. Any WebSocket peer address inserted through `add_outbound_addr()` during the identify handshake is stored under a key that can never be matched by subsequent lookups, silently breaking the score-based banning path in `report()` for all WebSocket-connected peers and leaving ghost entries that survive `ban_addr()`.

## Finding Description

**Root cause — `add()` uses raw key, all readers use normalized key:**

`add()` checks and inserts using `addr_info.addr` directly without normalization: [1](#0-0) 

`remove()`, `get()`, and `get_mut()` all call `base_addr()` first, which strips `/ws`, `/wss`, `/memory(_)`, `/tls(_)`: [2](#0-1) [3](#0-2) 

`base_addr()` is defined as: [4](#0-3) 

**Trigger path — identify protocol stores raw session address:**

During the identify handshake for outbound sessions, `context.session.address` (which includes `/ws` for WebSocket connections) is passed directly to `add_outbound_addr()`: [5](#0-4) 

`add_outbound_addr()` passes it unchanged to `addr_manager.add()`: [6](#0-5) 

**Score-update bypass — `report()` calls `get_mut()` which normalizes:**

`report()` calls `addr_manager.get_mut(addr)`. For a peer stored under `/ip4/A.A.A.A/tcp/8114/ws`, `get_mut()` normalizes to `/ip4/A.A.A.A/tcp/8114`, finds no entry, returns `None`, and the entire scoring block is skipped. The peer is never banned via the score mechanism regardless of how many misbehaviors are reported: [7](#0-6) 

**Ghost entry — `ban_addr()` calls `remove()` which also normalizes:**

Even if `ban_addr()` is called directly, `addr_manager.remove()` normalizes the address and fails to find the `/ws`-keyed entry. The IP-level ban via `ban_network()` still fires, but the addr_manager ghost entry persists and continues to be returned by `fetch_random()`, `fetch_addrs_to_attempt()`, and `fetch_addrs_to_feeler()`: [8](#0-7) 

**Duplicate entry inflation:**

If the same peer is seen both with and without the `/ws` suffix, `add()` stores two separate entries (since it checks the raw key), counting twice toward `ADDR_COUNT_LIMIT = 16384` and causing premature eviction of legitimate peers: [9](#0-8) 

## Impact Explanation

The score-based banning mechanism is completely bypassed for any peer that connects via WebSocket. An attacker operating one or more WebSocket peers can send malformed messages, trigger `report()` calls indefinitely, and never accumulate score penalties. This removes a primary defense against misbehaving peers and allows sustained resource-consuming misbehavior (bandwidth, CPU from protocol parsing) without any automatic ban. Additionally, ghost entries persist in the addr_manager and are repeatedly returned to the connection scheduler, causing the node to waste connection attempts toward entries that cannot be properly managed. This matches the allowed High impact: **"Vulnerabilities or bad designs which could cause CKB network congestion with few costs."**

## Likelihood Explanation

WebSocket is a first-class, explicitly supported transport in CKB's network stack — it is one of the four protocols handled by `base_addr()`. Any unprivileged external peer that connects to a CKB node via WebSocket automatically triggers `add_outbound_addr()` during the identify handshake. No special configuration, privilege, or victim mistake is required. The attacker only needs to establish a WebSocket connection and send misbehaving protocol messages. The condition is trivially and repeatably satisfiable.

## Recommendation

Normalize the address in `add()` using `base_addr()` before any lookup or insertion, consistent with all other operations:

```rust
pub fn add(&mut self, mut addr_info: AddrInfo) {
    let normalized = base_addr(&addr_info.addr);
    addr_info.addr = normalized.clone();
    if let Some(&id) = self.addr_to_id.get(&normalized) {
        let (exist_last_connected_at_ms, random_id_pos) = {
            let info = self.id_to_info.get(&id).expect("must exists");
            (info.last_connected_at_ms, info.random_id_pos)
        };
        if addr_info.last_connected_at_ms >= exist_last_connected_at_ms {
            addr_info.random_id_pos = random_id_pos;
            self.id_to_info.insert(id, addr_info);
        }
        return;
    }
    let id = self.next_id;
    self.addr_to_id.insert(normalized, id);
    addr_info.random_id_pos = self.random_ids.len();
    self.id_to_info.insert(id, addr_info);
    self.random_ids.push(id);
    self.next_id += 1;
}
```

This ensures a single canonical key is used across all `AddrManager` operations.

## Proof of Concept

**Manual steps:**

1. Attacker node connects to victim CKB node via WebSocket. Session address: `/ip4/A.A.A.A/tcp/8114/ws`.
2. Identify protocol fires on victim → `add_outbound_addr(/ip4/A.A.A.A/tcp/8114/ws)` → `addr_manager.add()` stores key `/ip4/A.A.A.A/tcp/8114/ws` → ID 0.
3. Attacker sends malformed messages. Victim calls `report(&"/ip4/A.A.A.A/tcp/8114/ws", Behaviour::UnexpectedMessage)`.
4. `report()` → `addr_manager.get_mut("/ip4/A.A.A.A/tcp/8114/ws")` → `base_addr()` → looks up `/ip4/A.A.A.A/tcp/8114` → **NOT FOUND** → returns `None`. Score unchanged. No ban triggered.
5. Repeat step 3 indefinitely. Peer is never banned via the score mechanism.
6. If `ban_addr("/ip4/A.A.A.A/tcp/8114/ws")` is called directly: IP-level ban fires, but `addr_manager.remove()` normalizes → `/ip4/A.A.A.A/tcp/8114` → **NOT FOUND**. Ghost entry ID 0 persists in `addr_manager` and continues to be returned by `fetch_random()`.

**Unit test:**

```rust
#[test]
fn test_ws_addr_score_bypass() {
    let mut peer_store = PeerStore::default();
    let ws_addr: Multiaddr = "/ip4/1.2.3.4/tcp/8114/ws".parse().unwrap();
    peer_store.add_outbound_addr(ws_addr.clone(), Flags::COMPATIBILITY);
    // Confirm entry exists via get — FAILS: returns None (key mismatch)
    assert!(peer_store.addr_manager().get(&ws_addr).is_some());
    // Confirm report does not update score
    let result = peer_store.report(&ws_addr, Behaviour::UnexpectedMessage);
    assert_eq!(result, ReportResult::Ok); // score never changed, never banned
    // Confirm remove fails — ghost entry persists
    peer_store.ban_addr(&ws_addr, 86400000, "test".to_string());
    assert_eq!(peer_store.addr_manager().count(), 1);
}
```

pub(crate) fn base_addr(addr: &Multiaddr) -> Multiaddr {
    addr.iter()
        .filter_map(|p| {
            if matches!(
                p,
                Protocol::Ws | Protocol::Wss | Protocol::Memory(_) | Protocol::Tls(_)
            ) {
                None
            } else {
                Some(p)
            }
        })
        .collect()
}
