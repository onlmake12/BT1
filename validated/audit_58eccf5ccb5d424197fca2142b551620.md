### Title
Unauthenticated Block Template Injection via Miner Notify HTTP Endpoint - (File: `miner/src/client.rs`)

### Summary

The CKB miner's "notify mode" HTTP server (`listen_block_template_notify`) accepts block template pushes from any unauthenticated source. The `handle()` function processes any valid JSON body as a `BlockTemplate` and immediately calls `client.update_block_template(template)` with no token, signature, or origin check. An attacker who can reach the miner's configured listen port can inject a crafted `BlockTemplate` — including a cellbase with an attacker-controlled lock script — causing the miner to direct its hashpower toward blocks whose rewards go to the attacker.

### Finding Description

When the miner is configured with a `listen` address in `ckb-miner.toml`, `spawn_background` starts an HTTP server via `listen_block_template_notify`: [1](#0-0) 

The TCP listener accepts connections from any source: [2](#0-1) 

Every accepted connection is dispatched to the `handle` function, which performs zero authentication: [3](#0-2) 

The body is deserialized directly into a `BlockTemplate` and passed to `update_block_template`. There is no shared secret, HMAC, bearer token, IP allowlist enforcement, or any other identity check. The `parse_authorization` helper that exists in the same file is used only for *outgoing* requests from the miner to the CKB node RPC — it is never applied to *incoming* notify requests: [4](#0-3) 

The `listen` field is a plain `SocketAddr` with no associated credential: [5](#0-4) 

The intended flow is that the CKB node's `block_assembler` POSTs legitimate templates to the miner's listen URL: [6](#0-5) 

Because the miner's endpoint is indistinguishable from the node's POST, any third party can send the same HTTP request.

### Impact Explanation

**Mining reward theft (primary):** The cellbase transaction in a `BlockTemplate` carries the lock script that receives the block subsidy and transaction fees. An attacker who injects a template with their own `cellbase.data.outputs[0].lock` causes the miner to solve PoW for a block whose coinbase pays the attacker. The CKB node's `submit_block` verifies the header and parent but does not re-validate the cellbase against the operator's configured `block_assembler` lock script: [7](#0-6) 

**Hashpower waste / chain disruption (secondary):** An attacker can inject a template with a stale `parent_hash` or a `compact_target` that differs from the current tip, causing the miner to expend hashpower on blocks that will be rejected by the node, effectively taking the miner offline.

### Likelihood Explanation

The `listen` option is documented and shown in the default config template: [8](#0-7) 

Operators who run the miner on a separate machine from the CKB node — a common production deployment — must enable this option and expose the port over the network. Any attacker with network access to that port (LAN peer, co-located tenant, or internet if the port is publicly exposed) can exploit this with a single HTTP POST. No credentials, keys, or prior knowledge are required beyond knowing the port number.

### Recommendation

1. **Shared secret token**: Add a configurable `notify_token` to `ClientConfig`. The CKB node includes it as a request header (e.g., `Authorization: Bearer <token>`); the miner's `handle` function rejects requests that do not present the matching value.
2. **IP allowlist**: Bind the listener to `127.0.0.1` by default and add an explicit allowlist of trusted source IPs checked before processing the body.
3. **HMAC signing**: The node signs the serialized template with a shared HMAC key; the miner verifies the signature before calling `update_block_template`.

### Proof of Concept

Assume the miner is configured with `listen = "0.0.0.0:8888"` and the operator's cellbase lock arg is `0xAABBCCDD...`.

```bash
# Attacker crafts a BlockTemplate JSON with their own cellbase lock script
# (attacker_lock_arg = 0xDEADBEEF...)
curl -X POST http://miner-host:8888/ \
  -H "Content-Type: application/json" \
  -d '{
    "version": "0x0",
    "compact_target": "0x1e083126",
    "current_time": "0x...",
    "number": "0x...",
    "epoch": "0x...",
    "parent_hash": "0x<current_tip_hash>",
    "cycles_limit": "0x...",
    "bytes_limit": "0x...",
    "uncles_count_limit": "0x2",
    "uncles": [],
    "transactions": [],
    "proposals": [],
    "cellbase": {
      "cycles": null,
      "data": {
        "inputs": [{"previous_output":{"tx_hash":"0x000...","index":"0xffffffff"},"since":"0x0"}],
        "outputs": [{"capacity":"0x...","lock":{"code_hash":"0x9bd7e06f...","hash_type":"type","args":"0xDEADBEEF..."}}],
        "outputs_data": ["0x"],
        "witnesses": ["0x..."]
      },
      "hash": "0x..."
    },
    "work_id": "0x1",
    "dao": "0x..."
  }'
```

The miner's `handle` function accepts this with HTTP 200. `update_block_template` replaces the current work. The next PoW solution the miner finds is submitted as a block paying `0xDEADBEEF...` — the attacker's address — instead of the operator's address. The CKB node accepts the block because the cellbase is consensus-valid; only the reward destination has changed.

    async fn listen_block_template_notify(&self, addr: SocketAddr) {
        let listener = TcpListener::bind(addr).await.unwrap();
        let server = auto::Builder::new(TokioExecutor::new());
        let graceful = GracefulShutdown::new();
        let stop_rx: CancellationToken = new_tokio_exit_rx();

        loop {
            let client = self.clone();
            let handle = service_fn(move |req| handle(client.clone(), req));
            tokio::select! {
                conn = listener.accept() => {
                    let (stream, _) = match conn {
                        Ok(conn) => conn,
                        Err(e) => {
                            info!("accept error: {}", e);
                            tokio::time::sleep(std::time::Duration::from_secs(1)).await;
                            continue;
                        }
                    };
                    let stream = hyper_util::rt::TokioIo::new(Box::pin(stream));
                    let conn = server.serve_connection_with_upgrades(stream, handle);

                    let conn = graceful.watch(conn.into_owned());
                    tokio::spawn(async move {
                        if let Err(err) = conn.await {
                            info!("connection error: {}", err);
                        }
                    });
                },
                _ = stop_rx.cancelled() => {
                    info!("Miner client received exit signal. Exit now");
                    break;
                }
            }
        }
        drop(listener);
        graceful.shutdown().await;
    }

async fn handle(

fn parse_authorization(url: &Uri) -> Option<HeaderValue> {
    let a: Vec<&str> = url.authority()?.as_str().split('@').collect();
    if a.len() >= 2 {
        if a[0].is_empty() {
            return None;
        }
        let mut encoded = "Basic ".to_string();
        base64::prelude::BASE64_STANDARD.encode_string(a[0], &mut encoded);
        let mut header = HeaderValue::from_str(&encoded).unwrap();
        header.set_sensitive(true);
        Some(header)
    } else {
        None
    }
}

            return;
        }
        let template = self.get_current().await;
        if let Ok(template_json) = serde_json::to_string(&template) {

                if let Ok(req) = Request::builder()
                    .method(Method::POST)
                    .uri(url.as_ref())
                    .header("content-type", "application/json")
                    .body(Full::new(template_json.to_owned().into()))

                    let client = Arc::clone(&self.poster);
                    let url = url.to_owned();

                        let _resp =

                                    ckb_logger::warn!(
                                        "block assembler notifying {} timed out",
