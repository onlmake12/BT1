### Title
`Scheduler.getEmaPricesNoOlderThan()` Staleness Check Uses On-Chain Push Timestamp Instead of Price `publishTime`, Allowing Stale EMA Prices to Pass Freshness Validation - (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

The `getEmaPricesNoOlderThan()` (and `getPricesNoOlderThan()`) function in the Pyth Pulse `Scheduler` contract validates freshness using `status.priceLastUpdatedAt` — the on-chain timestamp set when a keeper last called `updatePriceFeeds` — rather than the individual EMA price's `publishTime` field, which reflects the actual Pythnet data age. Because `updatePriceFeeds` is permissionless, an unprivileged actor can push valid but arbitrarily old Pythnet data (provided it is newer than what is currently stored), resetting `priceLastUpdatedAt` to `block.timestamp` while the stored EMA prices carry a `publishTime` far in the past. Downstream readers calling `getEmaPricesNoOlderThan` with a strict `age_seconds` will pass the staleness gate and receive EMA prices that are actually much older than the requested bound.

---

### Finding Description

`getEmaPricesNoOlderThan` performs its staleness check as follows:

```solidity
// Scheduler.sol
function getEmaPricesNoOlderThan(
    uint256 subscriptionId,
    bytes32[] calldata priceIds,
    uint256 age_seconds
) external view override onlyWhitelistedReader(subscriptionId)
    returns (PythStructs.Price[] memory prices)
{
    SchedulerStructs.SubscriptionStatus memory status = _state
        .subscriptionStatuses[subscriptionId];

    // Use distance (absolute difference) since pythnet timestamps
    // may be slightly ahead of this chain.
    if (distance(block.timestamp, status.priceLastUpdatedAt) > age_seconds)
        revert PythErrors.StalePrice();

    prices = this.getEmaPricesUnsafe(subscriptionId, priceIds);
}
``` [1](#0-0) 

The check `distance(block.timestamp, status.priceLastUpdatedAt) > age_seconds` measures how long ago the keeper last pushed an update on-chain, **not** how old the underlying Pythnet EMA price data is. The EMA prices returned by `getEmaPricesUnsafe` carry their original Pythnet `publishTime`, which can be arbitrarily older than `priceLastUpdatedAt`.

The same structural flaw exists in `getPricesNoOlderThan`:

```solidity
if (distance(block.timestamp, status.priceLastUpdatedAt) > age_seconds)
    revert PythErrors.StalePrice();

prices = this.getPricesUnsafe(subscriptionId, priceIds);
``` [2](#0-1) 

The Pulse README confirms `updatePriceFeeds` is permissionless — "Anyone can run a Keeper node; no registration is required to call `updatePriceFeeds`": [3](#0-2) 

The Scheduler verifies that pushed data is newer than what is currently stored (monotonically increasing `publishTime`), but it does **not** require the pushed data to be recent relative to `block.timestamp`. This means any actor can push valid Pythnet data whose `publishTime` is, say, 10 minutes in the past (as long as it is newer than the currently stored value), causing `priceLastUpdatedAt` to be set to `block.timestamp` while the actual EMA price `publishTime` remains 10 minutes old.

This is structurally identical to the ConicFinance pattern: `status.priceLastUpdatedAt` plays the role of `_cachedTotalUnderlying` (a raw cached value that can be refreshed without validating data freshness), while the individual price's `publishTime` plays the role of `cachedTotalUnderlying()` (the value that actually reflects data age).

---

### Impact Explanation

A DeFi protocol reading EMA prices from the Scheduler via `getEmaPricesNoOlderThan(subscriptionId, priceIds, 60)` expects a guarantee that the returned prices are no older than 60 seconds. Because the staleness gate checks `priceLastUpdatedAt` (on-chain push time) rather than `publishTime` (Pythnet data age), this guarantee is broken. The protocol receives EMA prices that may be minutes old while believing them to be fresh, potentially leading to:

- Incorrect collateral valuations in lending protocols
- Incorrect liquidation thresholds
- Incorrect derivative settlement prices

The `getEmaPricesUnsafe` path (which the `NoOlderThan` variant wraps) explicitly carries no freshness guarantee, as documented:

> "Readers are recommended to use the SDK's functions `get(Ema)PricesNoOlderThan`, which wrap the contract's `get(Ema)PricesUnsafe` functions and validate that the price is recent." [4](#0-3) 

The `NoOlderThan` variant is the safety wrapper, but its check is insufficient.

---

### Likelihood Explanation

The attack path requires no privileged access. Any actor can call `updatePriceFeeds` on the Scheduler. The attacker only needs:

1. A valid Pythnet price update (obtainable from Hermes) whose `publishTime` is newer than the currently stored value but still old relative to `block.timestamp`.
2. To call `updatePriceFeeds` with this data, resetting `priceLastUpdatedAt` to `block.timestamp`.

This is realistic during periods of low keeper activity (e.g., when a subscription's balance is depleted and the legitimate keeper stops pushing). The attacker can then keep the subscription's `priceLastUpdatedAt` current while the actual price data ages, defeating the staleness guarantee for any reader.

---

### Recommendation

Replace the subscription-level `priceLastUpdatedAt` check with a per-price `publishTime` check. For each requested price feed, validate that `distance(block.timestamp, price.publishTime) <= age_seconds` before returning. This mirrors the pattern used in `AbstractPyth.sol`:

```solidity
// AbstractPyth.sol — correct pattern
function getPriceNoOlderThan(bytes32 id, uint age)
    public view virtual override returns (PythStructs.Price memory price)
{
    price = getPriceUnsafe(id);
    if (diff(block.timestamp, price.publishTime) > age)
        revert PythErrors.StalePrice();
    return price;
}
``` [5](#0-4) 

The `getEmaPricesNoOlderThan` and `getPricesNoOlderThan` functions should iterate over the returned prices and check each `publishTime` individually, rather than relying on the subscription-level `priceLastUpdatedAt`.

---

### Proof of Concept

1. Subscription S has `heartbeatSeconds = 3600`. Stored EMA price has `publishTime = T - 7200` (2 hours ago). `priceLastUpdatedAt = T - 3700`.
2. Attacker fetches a valid Pythnet update from Hermes with `publishTime = T - 3600` (1 hour ago). This is newer than the stored value, so the Scheduler accepts it.
3. Attacker calls `Scheduler.updatePriceFeeds(subscriptionId, updateData)`. The Scheduler stores the new EMA price (`publishTime = T - 3600`) and sets `priceLastUpdatedAt = block.timestamp = T`.
4. A DeFi protocol calls `getEmaPricesNoOlderThan(subscriptionId, priceIds, 60)`.
5. Check: `distance(T, T) = 0 ≤ 60` → **passes**.
6. Returned EMA price has `publishTime = T - 3600` — **1 hour old** — while the caller believes it is no older than 60 seconds.
