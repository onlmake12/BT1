### Title
Permissionless `set_parameters` Initialization via `init_if_needed` Allows Any Signer to Seize `current_authority` and Corrupt OIS Staking Caps — (File: `pythnet/stake_caps_parameters/programs/stake_caps_parameters/src/lib.rs`)

---

### Summary

The `set_parameters` instruction in the `stake_caps_parameters` program uses Anchor's `init_if_needed` on the global `Parameters` PDA with no authorized-initializer guard. When the account does not yet exist, it is zero-initialized, making `current_authority == Pubkey::default()`. The access-control check explicitly allows any signer to proceed in that state. Any unprivileged transaction sender can therefore race the legitimate deployer, initialize the account themselves, install their own key as `current_authority`, and write arbitrary `m` and `z` values that govern every publisher's staking pool cap in the Oracle Integrity Staking (OIS) system.

---

### Finding Description

In `pythnet/stake_caps_parameters/programs/stake_caps_parameters/src/lib.rs`, the sole instruction is:

```rust
pub fn set_parameters(ctx: Context<SetParameters>, parameters: Parameters) -> Result<()> {
    let stored_parameters = &mut ctx.accounts.parameters;
    require!(
        ctx.accounts.signer.key() == stored_parameters.current_authority
            || stored_parameters.current_authority == Pubkey::default(),  // ← bypass
        ErrorCode::WrongAuthority
    );
    **stored_parameters = parameters;   // ← full overwrite including current_authority
    Ok(())
}
``` [1](#0-0) 

The account constraint is:

```rust
#[account(
    init_if_needed,          // ← creates the account if absent
    seeds = ["parameters".as_bytes()],
    bump,
    payer = signer,
    space = Parameters::LEN
)]
pub parameters: Account<'info, Parameters>,
``` [2](#0-1) 

When the PDA does not yet exist, Anchor creates it and zero-fills it. `Parameters.current_authority` is therefore `Pubkey::default()` (32 zero bytes). The `|| stored_parameters.current_authority == Pubkey::default()` branch is immediately true, so the `require!` passes for **any** signer. The attacker's supplied `parameters` struct — including a new `current_authority` pointing to their own key and arbitrary `m`/`z` values — is written atomically.

There is no separate `initialize` instruction, no `GlobalConfig` PDA, no allowlist, and no deployer-key check. The program has a single instruction that doubles as both initializer and updater, with the only guard being the zero-address bypass. [3](#0-2) 

---

### Impact Explanation

`m` (target stake per symbol) and `z` (minimum publisher count floor) are the two free parameters in the OIS pool-cap formula:

```
C_p = M · Σ_{s ∈ Symbols_p}  1 / max(n_s, Z)
``` [4](#0-3) 

These values are read from Pythnet by the staking front-end and by the off-chain publisher-caps computation pipeline that feeds the on-chain `publisher_caps` account consumed by the `integrity_pool` `advance` instruction. [5](#0-4) 

An attacker who seizes `current_authority` can:

- Set `m = 0` → all pool caps collapse to zero, making every publisher's pool appear full; stakers cannot earn rewards and the UI shows 0-capacity pools.
- Set `z = u64::MAX` → each term becomes `1/u64::MAX ≈ 0`, same effect.
- Set `m = u64::MAX` → overflow/saturation in cap arithmetic, potentially causing panics or wildly inflated caps that bypass the soft-cap enforcement.
- Continuously rotate `current_authority` to themselves after each epoch, permanently locking out the legitimate Pyth DAO operator.

Because the attacker becomes the permanent `current_authority`, recovery requires a program upgrade — a governance-gated operation.

---

### Likelihood Explanation

The attack window is the gap between program deployment and the first legitimate `set_parameters` call. On Solana/Pythnet, transactions are public in the mempool; a bot watching for the deployer's initialization transaction can front-run it in the same slot or the next. The attacker needs only a funded keypair and a single transaction. No privileged access, no leaked keys, and no social engineering are required. The program is already deployed at `ujSFv8q8woXW5PUnby52PQyxYGUudxkrvgN6A631Qmm`. [6](#0-5) 

---

### Recommendation

Replace the `init_if_needed` + zero-address bypass pattern with a dedicated, one-shot `initialize` instruction that hard-codes or verifies the authorized initializer:

```rust
// Option A: hard-code the deployer / DAO key
pub fn initialize(ctx: Context<Initialize>, parameters: Parameters) -> Result<()> {
    require!(
        ctx.accounts.signer.key() == AUTHORIZED_INITIALIZER,
        ErrorCode::WrongAuthority
    );
    ctx.accounts.parameters.set_inner(parameters);
    Ok(())
}

// Option B: store an authorized_initializer in a GlobalConfig PDA
// checked during initialize, then removed or locked after first use.
```

The `set_parameters` instruction should use `#[account(mut, seeds = [...], bump)]` (no `init_if_needed`) and require `signer == current_authority` with no zero-address bypass.

#[test]
fn test_stake_caps_parameters() {
    let mut svm = litesvm::LiteSVM::new();
    let payer_1 = Keypair::new();
    let payer_2 = Keypair::new();

    svm.add_program_from_file(
        stake_caps_parameters::ID,
        "../../target/deploy/stake_caps_parameters.so",
    )
    .unwrap();

    svm.airdrop(&payer_1.pubkey(), LAMPORTS_PER_SOL).unwrap();
    svm.airdrop(&payer_2.pubkey(), LAMPORTS_PER_SOL).unwrap();

    let parameter_1 = Parameters {
        current_authority: payer_1.pubkey(),
        m: 1,
        z: 2,
    };

    assert!(set_parameters(&mut svm, &payer_1, parameter_1).is_ok());
    assert_eq!(fetch_current_parameters(&mut svm), parameter_1);

    let parameter_2 = Parameters {
        current_authority: payer_2.pubkey(),
        m: 3,
        z: 4,
    };

    require_wrong_authority_error(set_parameters(&mut svm, &payer_2, parameter_2));
    assert_eq!(fetch_current_parameters(&mut svm), parameter_1);

    assert!(set_parameters(&mut svm, &payer_1, parameter_2).is_ok());
    assert_eq!(fetch_current_parameters(&mut svm), parameter_2);

    let parameter_3 = Parameters {
        current_authority: payer_1.pubkey(),
        m: 5,
        z: 6,
    };

    require_wrong_authority_error(set_parameters(&mut svm, &payer_1, parameter_3));
    assert!(set_parameters(&mut svm, &payer_2, parameter_3).is_ok());
    assert_eq!(fetch_current_parameters(&mut svm), parameter_3);
}
