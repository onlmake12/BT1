### Title
`EpochNumberWithFraction::from_full_value_unchecked` Accepts Invalid Epoch Values with Non-Zero Upper 8 Bits Without Validation — (`util/types/src/conversion/blockchain.rs`)

### Summary

The Molecule unpack path for `EpochNumberWithFraction` uses `from_full_value_unchecked`, which wraps a raw `u64` without checking that the upper 8 bits are zero. The `EpochNumberWithFraction` type only uses 56 bits (24 for epoch number, 16 for index, 16 for length), leaving the top 8 bits of the `u64` unused and semantically invalid. The epoch verifier does not check for non-zero upper bits either. A block relayer can craft a block header with a malformed epoch field that passes all verification and is accepted into the chain.
