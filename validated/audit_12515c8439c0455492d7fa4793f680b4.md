### Title
Unbounded Per-Message CPU/IO Cost in `GetBlocksProofProcess::execute` with No Per-Peer Rate Limiting — (`util/light-client-protocol-server/src/components/get_blocks_proof.rs`)

### Summary

An unprivileged remote peer can send repeated `GetBlocksProof` messages each containing up to 1000 distinct valid on-chain block hashes. The server performs 3 database reads per block (header, uncles, extension), then calls `mmr.gen_proof` with up to 1000 positions (O(N log N) MMR node reads), then serializes and sends a large `SendBlocksProofV1` response — all with no per-peer rate limiting. This allows a single peer to sustain high CPU and I/O pressure on the server indefinitely.

### Finding Description

`constant::GET_BLOCKS_PROOF_LIMIT` is set to 1000. [1](#0-0) 

The `execute` method enforces this limit and rejects duplicates, but both checks are purely structural — they do not limit the computational cost of a valid request: [2](#0-1) 

For every hash in `found`, the server performs three synchronous store reads: [3](#0-2) 

Then `reply_proof` calls `mmr.gen_proof(items_positions)` with up to 1000 positions, which requires O(N log N) MMR node reads against the backing store: [4](#0-3) 

The `received` handler in `lib.rs` dispatches every incoming message directly to `try_process` with no token bucket, no per-peer message counter, no cooldown, and no concurrency cap: [5](#0-4) 

A peer is only banned for sending a *malformed* message (unparseable bytes) or a message that returns a `should_ban` status. A structurally valid `GetBlocksProof` with 1000 real on-chain hashes returns `Status::ok()` and is never penalized. [6](#0-5) 

### Impact Explanation

A single malicious peer can saturate the light-client server's CPU and RocksDB I/O by sending back-to-back `GetBlocksProof` messages at line rate. Because the handler `await`s each message to completion before returning, and there is no concurrency limit or rate gate, the attacker can keep the server busy processing 3000 DB reads + O(1000 log 1000) MMR reads + large response serialization per message, degrading or denying service to all legitimate light clients connected to the same node.

### Likelihood Explanation

The attack requires only a TCP connection to a node with the light client protocol enabled and knowledge of 1000 valid block hashes (trivially scraped from any public chain explorer or by syncing headers). No PoW, no key, no privileged role is needed.

### Recommendation

- Add a per-peer sliding-window rate limiter (e.g., token bucket) in `received` before dispatching to `try_process`, shared across all light-client message types.
- Reduce `GET_BLOCKS_PROOF_LIMIT` to a smaller value (e.g., 64–128) that bounds worst-case per-message cost.
- Consider processing `GetBlocksProof` in a bounded worker pool so a flood from one peer cannot monopolize the async executor.

### Proof of Concept

1. Run a CKB node with light client protocol enabled and mine (or sync) at least 1000 blocks.
2. Collect 1000 distinct on-chain block hashes `h_1 … h_1000` and the current tip hash `last_hash`.
3. In a loop, send `GetBlocksProof { last_hash, block_hashes: [h_1, …, h_1000] }` as fast as the TCP socket allows.
4. Observe via `perf`/`htop` that the server's CPU and RocksDB read IOPS scale linearly with message rate, with no back-pressure applied to the sending peer. [1](#0-0) [7](#0-6)

    pub(crate) async fn execute(self) -> Status {
        if self.message.block_hashes().is_empty() {
            return StatusCode::MalformedProtocolMessage.with_context("no block");
        }

        if self.message.block_hashes().len() > constant::GET_BLOCKS_PROOF_LIMIT {
            return StatusCode::MalformedProtocolMessage.with_context("too many blocks");
        }

        let snapshot = self.protocol.shared.snapshot();

        let last_block_hash = self.message.last_hash().to_entity();
        if !snapshot.is_main_chain(&last_block_hash) {
            return self
                .protocol
                .reply_tip_state::<packed::SendBlocksProof>(self.peer, self.nc)
                .await;
        }
        let last_block = snapshot
            .get_block(&last_block_hash)
            .expect("block should be in store");

        let block_hashes: Vec<_> = self
            .message
            .block_hashes()
            .to_entity()
            .into_iter()
            .collect();

        let mut uniq = HashSet::new();
        if !block_hashes
            .iter()
            .chain([last_block_hash].iter())
            .all(|hash| uniq.insert(hash))
        {
            return StatusCode::MalformedProtocolMessage
                .with_context("duplicate block hash exists");
        }

        let (found, missing): (Vec<_>, Vec<_>) = block_hashes
            .into_iter()
            .partition(|block_hash| snapshot.is_main_chain(block_hash));

        let mut positions = Vec::with_capacity(found.len());
        let mut block_headers = Vec::with_capacity(found.len());
        let mut uncles_hash = Vec::with_capacity(found.len());
        let mut extensions = Vec::with_capacity(found.len());

        for block_hash in found {
            let header = snapshot
                .get_block_header(&block_hash)
                .expect("header should be in store");
            positions.push(leaf_index_to_pos(header.number()));
            block_headers.push(header.data());

            let uncles = snapshot
                .get_block_uncles(&block_hash)
                .expect("block uncles must be stored");
            let extension = snapshot.get_block_extension(&block_hash);

            uncles_hash.push(uncles.data().calc_uncles_hash());
            extensions.push(packed::BytesOpt::new_builder().set(extension).build());
        }

        let proved_items = (
            block_headers.into(),
            uncles_hash.into(),
            packed::BytesOptVec::new_builder().set(extensions).build(),
        );
        let missing_items = missing.into();

        self.protocol
            .reply_proof::<packed::SendBlocksProofV1>(
                self.peer,
                self.nc,
                &last_block,
                positions,
                proved_items,
                missing_items,
            )
            .await
    }

    async fn received(
        &mut self,
        nc: Arc<dyn CKBProtocolContext + Sync>,
        peer: PeerIndex,
        data: Bytes,
    ) {
        trace!("LightClient.received peer={}", peer);

        let msg = match packed::LightClientMessageReader::from_slice(&data) {
            Ok(msg) => msg.to_enum(),
            _ => {
                warn!(
                    "LightClient.received a malformed message from Peer({})",
                    peer
                );
                nc.ban_peer(
                    peer,
                    constant::BAD_MESSAGE_BAN_TIME,
                    String::from("send us a malformed message"),
                );
                return;
            }
        };

        let item_name = msg.item_name();
        let status = self.try_process(&nc, peer, msg).await;
        if let Some(ban_time) = status.should_ban() {
            error!(
                "process {} from {}; ban {:?} since result is {}",
                item_name, peer, ban_time, status
            );
            nc.ban_peer(peer, ban_time, status.to_string());
        } else if status.should_warn() {
            warn!("process {} from {}; result is {}", item_name, peer, status);
        } else if !status.is_ok() {
            debug!("process {} from {}; result is {}", item_name, peer, status);
        }
    }
