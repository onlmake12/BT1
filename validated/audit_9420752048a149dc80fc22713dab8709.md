### Title
Permissionless `initialize` Allows Attacker to Seize Program Authority and Inject Arbitrary Oracle Prices — (`target_chains/solana/programs/pyth-price-store/src/processor/initialize.rs`)

---

### Summary

The `initialize` instruction is fully permissionless: any account may be the payer and any pubkey may be supplied as `args.authority`. Because the config PDA is a singleton (one deterministic address per program deployment), the first caller permanently owns the authority. An attacker who calls `initialize` before the legitimate operator — even with a small timing window between deployment and initialization — locks the operator out and gains full control over publisher registration and price submission.

---

### Finding Description

**Root cause — `initialize` has no deployer check.**

`initialize` accepts any signer as `payer` and stores whatever pubkey is in `args.authority` into the config PDA: [1](#0-0) 

There is no check that `payer` equals the program's upgrade authority or any other privileged key. The config PDA is derived solely from `CONFIG_SEED` and `program_id`: [2](#0-1) 

Because `system_instruction::create_account` fails if the account already exists, the first caller wins permanently. The legitimate operator's subsequent `initialize` call will revert with "account already in use."

**Step 2 — Attacker creates a buffer owned by the program.**

`validate_buffer` only checks `is_writable` and `owner == program_id`: [3](#0-2) 

Any account created with `create_account` or `create_account_with_seed` pointing to `program_id` as owner satisfies this check. No proof of legitimate creation is required.

**Step 3 — Attacker calls `InitializePublisher` with their authority.**

`validate_authority` reads the authority stored in the config PDA and checks the signer matches it: [4](#0-3) 

Since the attacker set their own key as authority in step 1, this check passes. The attacker registers any publisher pubkey (e.g., their own) with the attacker-controlled buffer.

**Step 4 — Attacker calls `SubmitPrices` to inject arbitrary prices.**

`submit_prices` verifies only that the publisher signed, the publisher_config PDA matches the publisher key, and the buffer key matches what is stored in publisher_config: [5](#0-4) 

All three checks pass for the attacker's legitimately-registered (but maliciously-controlled) publisher. Arbitrary price data is written into the buffer and consumed by the validator at end-of-slot.

---

### Impact Explanation

The buffer is read by the Solana validator plugin to produce price feeds consumed by downstream DeFi protocols. Arbitrary prices in the buffer directly translate to manipulated oracle outputs. Protocols using Pyth prices for liquidations, collateral valuation, or derivatives settlement would act on fabricated data, causing incorrect liquidations or fund loss.

---

### Likelihood Explanation

The attack does not require mempool front-running (Solana has none). It only requires calling `initialize` before the legitimate operator does — a window that exists between program deployment and the first `initialize` transaction. An attacker monitoring the chain for new deployments of this program ID can exploit this window reliably. The entire 4-step sequence is executable in a single `program-test` environment with no privileged access.

---

### Recommendation

Restrict `initialize` to the program's upgrade authority. Retrieve the upgrade authority from the program's `ProgramData` account and assert that `payer.key` matches it:

```rust
// In initialize(), after validate_payer():
let program_data = validate_program_data(accounts.next(), program_id)?;
let upgrade_authority = read_upgrade_authority(&program_data)?;
ensure!(ProgramError::MissingRequiredSignature,
    payer.key.to_bytes() == upgrade_authority);
```

This ensures only the entity that deployed the program can set the authority, eliminating the race condition entirely.

---

### Proof of Concept

```rust
// 1. Deploy program, do NOT call initialize yet.
// 2. Attacker calls initialize with attacker_keypair as authority.
let mut data = vec![Instruction::Initialize as u8, config_bump];
data.extend_from_slice(&attacker.pubkey().to_bytes());
// → config PDA now stores attacker pubkey as authority

// 3. Attacker creates buffer owned by program_id.
create_account_with_seed(&attacker, &buffer_key, &attacker, "evil", lamports, size, &program_id);

// 4. Attacker calls InitializePublisher (authority check passes).
// → publisher_config PDA created, buffer_account = attacker's buffer

// 5. Attacker calls SubmitPrices with fabricated data.
let prices = [BufferedPrice::new(fake_price, fake_conf, fake_slot, fake_expo).unwrap()];
// → assert prices are accepted and readable from buffer
```

All four steps succeed; the legitimate operator's `initialize` call fails with "account already in use."

```rust
pub fn initialize(
    program_id: &Pubkey,
    accounts: &[AccountInfo],
    args: &InitializeArgs,
) -> ProgramResult {
    let mut accounts = accounts.iter();
    let payer = validate_payer(accounts.next())?;
    let config = validate_config(accounts.next(), args.config_bump, program_id, true)?;
    let system = validate_system(accounts.next())?;

    let lamports = (Rent::get()?).minimum_balance(accounts::config::SIZE);

    invoke_signed(
        &system_instruction::create_account(
            payer.key,
            config.key,
            lamports,
            accounts::config::SIZE
                .try_into()
                .expect("unexpected overflow"),
            program_id,
        ),
        &[payer.clone(), config.clone(), system.clone()],
        &[&[CONFIG_SEED.as_bytes(), &[args.config_bump]]],
    )?;

    accounts::config::create(*config.data.borrow_mut(), args.authority)?;

    Ok(())
```

```rust
pub fn validate_authority<'a, 'b>(
    account: Option<&'b AccountInfo<'a>>,
    config: &AccountInfo<'a>,
) -> Result<&'b AccountInfo<'a>, ProgramError> {
    let authority = account.ok_or(ProgramError::NotEnoughAccountKeys)?;
    ensure!(ProgramError::MissingRequiredSignature, authority.is_signer);
    ensure!(ProgramError::InvalidArgument, authority.is_writable);
    let config_data = config.data.borrow();
    let config = accounts::config::read(*config_data)?;
    ensure!(
        ProgramError::MissingRequiredSignature,
        authority.key.to_bytes() == config.authority
    );
    Ok(authority)
```

```rust
pub fn validate_buffer<'a, 'b>(
    account: Option<&'b AccountInfo<'a>>,
    program_id: &Pubkey,
) -> Result<&'b AccountInfo<'a>, ProgramError> {
    let buffer = account.ok_or(ProgramError::NotEnoughAccountKeys)?;
    ensure!(ProgramError::InvalidArgument, buffer.is_writable);
    ensure!(ProgramError::IllegalOwner, buffer.owner == program_id);
    Ok(buffer)
```
