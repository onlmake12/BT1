### Title
`uint8` Loop Counter Overflow in `_getPricesInternal` Causes Permanent DoS on Price Getters for Large Subscriptions — (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`Scheduler._getPricesInternal` uses a `uint8` loop counter to iterate over a subscription's `priceIds` array. If a subscription is created with 256 or more price IDs, the counter wraps to zero and the loop never terminates, permanently bricking all price-read view functions for that subscription. A separate `uint8` truncation in `createSubscription` silently zeroes the minimum-balance requirement for the same 256-price-ID boundary, making such a subscription trivially creatable.

---

### Finding Description

**Loop counter overflow — `Scheduler.sol` lines 479–491**

```solidity
for (uint8 i = 0; i < params.priceIds.length; i++) {
    PythStructs.PriceFeed memory priceFeed = _state.priceUpdates[
        subscriptionId
    ][params.priceIds[i]];
    ...
    allFeeds[i] = priceFeed;
}
```

`i` is declared `uint8`. When `params.priceIds.length >= 256`, `i` increments from 255 to 0 (wraps), the condition `i < params.priceIds.length` remains true forever, and the call reverts out-of-gas. Every external view function that delegates to `_getPricesInternal` with an empty `priceIds` argument (`getPricesUnsafe`, `getPricesNoOlderThan`, `getEmaPricesUnsafe`, `getEmaPricesNoOlderThan`) becomes permanently unusable for that subscription.

**Minimum-balance bypass — `Scheduler.sol` lines 38–45**

```solidity
uint256 minimumBalance = this.getMinimumBalance(
    uint8(subscriptionParams.priceIds.length)   // truncates at 256
);
if (msg.value < minimumBalance) {
    revert SchedulerErrors.InsufficientBalance();
}
```

`uint8(256) == 0`, so `getMinimumBalance(0) == 0`. A subscription with exactly 256 price IDs passes the balance check with `msg.value == 0`, allowing the malformed subscription to be stored at no cost. [1](#0-0) [2](#0-1) 

---

### Impact Explanation

All four external price-read view functions (`getPricesUnsafe`, `getPricesNoOlderThan`, `getEmaPricesUnsafe`, `getEmaPricesNoOlderThan`) call `_getPricesInternal` with an empty `priceIds` slice to return all feeds for a subscription. For any subscription whose `params.priceIds.length >= 256`, every such call reverts out-of-gas. Whitelisted readers — the intended consumers of the Scheduler — permanently lose the ability to read price data for that subscription through the contract's own interface. [3](#0-2) 

---

### Likelihood Explanation

Any address can call `createSubscription`. The minimum-balance bypass at the 256-boundary means the attacker pays nothing beyond gas. `_validateSubscriptionParams` is called before the balance check; whether it independently caps `priceIds.length` was not confirmed from the available code, but no explicit cap is visible in the call path. The attack is a single transaction. [4](#0-3) 

---

### Recommendation

1. Change the loop counter in `_getPricesInternal` from `uint8 i` to `uint256 i`.
2. Change the cast in `createSubscription` from `uint8(subscriptionParams.priceIds.length)` to the raw `uint256` value, or add an explicit upper-bound check in `_validateSubscriptionParams` (e.g., `require(params.priceIds.length <= 255)`).
3. Consider adding a paginated variant of `getPricesUnsafe(subscriptionId, startIndex, maxResults)` analogous to the already-paginated `getActiveSubscriptions`, so that even large subscriptions remain readable. [5](#0-4) 

---

### Proof of Concept

```
1. Attacker calls createSubscription({ priceIds: [id_0, ..., id_255], ... }, value=0)
   - uint8(256) == 0 → getMinimumBalance(0) == 0 → balance check passes
   - Subscription stored with subscriptionId = N

2. Whitelisted reader calls getPricesUnsafe(N, [])
   - Delegates to _getPricesInternal(N, [])
   - priceIds.length == 0 → enters the "return all feeds" branch
   - Loop: uint8 i = 0; i < 256; i++
     → i reaches 255, i++ wraps to 0, condition still true → infinite loop
   - Transaction reverts: out of gas

3. Same result for getPricesNoOlderThan, getEmaPricesUnsafe, getEmaPricesNoOlderThan.
   Subscription price data is permanently inaccessible via the contract interface.
``` [6](#0-5) [7](#0-6)

    function _getPricesInternal(
        uint256 subscriptionId,
        bytes32[] calldata priceIds
    ) internal view returns (PythStructs.PriceFeed[] memory priceFeeds) {
        if (!_state.subscriptionParams[subscriptionId].isActive) {
            revert SchedulerErrors.InactiveSubscription();
        }

        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        // If no price IDs provided, return all price feeds for the subscription
        if (priceIds.length == 0) {
            PythStructs.PriceFeed[]
                memory allFeeds = new PythStructs.PriceFeed[](
                    params.priceIds.length
                );
            for (uint8 i = 0; i < params.priceIds.length; i++) {
                PythStructs.PriceFeed memory priceFeed = _state.priceUpdates[
                    subscriptionId
                ][params.priceIds[i]];
                // Check if the price feed exists (price ID is valid and has been updated)
                if (priceFeed.id == bytes32(0)) {
                    revert SchedulerErrors.InvalidPriceId(
                        params.priceIds[i],
                        bytes32(0)
                    );
                }
                allFeeds[i] = priceFeed;
            }
            return allFeeds;
        }

        // Return only the requested price feeds
        PythStructs.PriceFeed[]
            memory requestedFeeds = new PythStructs.PriceFeed[](
                priceIds.length
            );
        for (uint8 i = 0; i < priceIds.length; i++) {
            PythStructs.PriceFeed memory priceFeed = _state.priceUpdates[
                subscriptionId
            ][priceIds[i]];

            // Check if the price feed exists (price ID is valid and has been updated)
            if (priceFeed.id == bytes32(0)) {
                revert SchedulerErrors.InvalidPriceId(priceIds[i], bytes32(0));
            }
            requestedFeeds[i] = priceFeed;
        }
        return requestedFeeds;
    }
