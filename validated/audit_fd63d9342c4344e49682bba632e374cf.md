### Title
Arbitrary `providerToCredit` in `Echo.executeCallback` Allows Fee Theft After Exclusivity Period - (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary
`Echo.executeCallback` accepts a caller-supplied `address providerToCredit` parameter and credits the entire request fee to that address's accrued balance. The exclusivity-period guard only enforces `providerToCredit == req.provider` while the exclusivity window is active. Once the window expires, the check is skipped entirely, allowing any caller to redirect the legitimate provider's earned fee to an arbitrary address they control.

### Finding Description
In `Echo.executeCallback`, the fee credit logic is:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
```

The only guard on `providerToCredit` is:

```solidity
if (block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds) {
    require(
        providerToCredit == req.provider,
        "Only assigned provider during exclusivity period"
    );
}
```

After `req.publishTime + exclusivityPeriodSeconds` elapses, this `if`-block is never entered, and `providerToCredit` is completely unconstrained. The function never falls back to `req.provider` (the stored, designated provider). Any caller may pass their own address as `providerToCredit` and receive the full fee credit.

The stored provider address is available in `req.provider` (set at request creation time in `requestPriceUpdatesWithCallback`), but it is only used inside the exclusivity guard — never as the unconditional fee recipient.

### Impact Explanation
An attacker who:
1. Registers as a provider via `registerProvider` (permissionless),
2. Calls `setFeeManager(attacker_address)` to designate themselves as their own fee manager,
3. Waits for the exclusivity period to expire on any pending request,
4. Calls `executeCallback(attacker_address, sequenceNumber, updateData, priceIds)` with valid price update data,

…will have `req.fee` (the full provider portion of the user's payment) credited to their own `accruedFeesInWei` balance instead of the legitimate provider's. They then drain it via `withdrawAsFeeManager(attacker_address, amount)`.

The legitimate provider who was assigned the request receives nothing. Every pending request whose exclusivity window has elapsed is exploitable. The attacker only needs to supply valid `updateData` and cover the Pyth parsing fee (`pythFee`), which is typically small.

### Likelihood Explanation
`executeCallback` is a public, permissionless function with no caller restriction. The exclusivity period is a finite, configurable window (default set at deployment). Any request that has not been fulfilled within that window is permanently vulnerable. A monitoring bot can trivially detect unfulfilled requests and front-run or race the legitimate provider after the window expires. No privileged access, leaked keys, or governance majority is required.

### Recommendation
Remove the `address providerToCredit` parameter from `executeCallback`. Instead, always credit `req.provider` (the stored provider from the request), which is the only address that legitimately earned the fee:

```solidity
// Before (vulnerable):
_state.providers[providerToCredit].accruedFeesInWei += ...;

// After (fixed):
_state.providers[req.provider].accruedFeesInWei += ...;
```

The exclusivity-period logic can be retained as a separate check on `msg.sender` if the intent is to restrict *who may call* the function, rather than *who gets credited*.

### Proof of Concept

1. User calls `requestPriceUpdatesWithCallback(legitimateProvider, publishTime, priceIds, gasLimit)` paying `totalFee`. `req.provider = legitimateProvider`, `req.fee = totalFee - pythFeeInWei`.
2. Attacker calls `registerProvider(0, 0, 0)` — registers with zero fees.
3. Attacker calls `setFeeManager(attacker)` — sets self as own fee manager.
4. `block.timestamp >= req.publishTime + exclusivityPeriodSeconds` — exclusivity window has elapsed.
5. Attacker calls `executeCallback(attacker, sequenceNumber, updateData, priceIds)`.
   - The `if (block.timestamp < ...)` guard is **not entered**.
   - `_state.providers[attacker].accruedFeesInWei += (req.fee + 0) - pythFee` — fee credited to attacker.
   - `legitimateProvider`'s balance is unchanged.
6. Attacker calls `withdrawAsFeeManager(attacker, amount)` — drains the stolen fee. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)
