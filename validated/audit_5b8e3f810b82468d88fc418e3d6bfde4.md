### Title
`getPriceUnsafe()` Used in All Price-Returning Functions of `PythAggregatorV3`, Including `latestRoundData()` — No Staleness Enforcement — (File: `target_chains/ethereum/sdk/solidity/PythAggregatorV3.sol`)

---

### Summary

`PythAggregatorV3` is Pyth's production Chainlink-compatible adapter contract. Every price-returning function in the contract — including `latestRoundData()`, `getRoundData()`, `latestAnswer()`, and `getAnswer()` — calls `getPriceUnsafe()` internally. `getPriceUnsafe()` is explicitly documented in `IPyth.sol` as returning a price that "may be arbitrarily far in the past" with "no recency checks." No staleness guard is applied at any point inside the adapter. Any DeFi protocol that deploys this adapter and reads prices from it can silently consume arbitrarily stale data.

---

### Finding Description

`PythAggregatorV3.sol` implements the Chainlink `AggregatorV3Interface` as a drop-in replacement for Chainlink price feeds. All six price-reading functions delegate to `getPriceUnsafe()`:

- `latestAnswer()` — line 54 [1](#0-0) 
- `latestTimestamp()` — line 59 [2](#0-1) 
- `getAnswer()` — line 68–70, delegates to `latestAnswer()` [3](#0-2) 
- `getRoundData()` — line 89 [4](#0-3) 
- `latestRoundData()` — line 110 [5](#0-4) 
- `decimals()` — line 41 [6](#0-5) 

`IPyth.sol` explicitly documents `getPriceUnsafe()` as:

> "This function is unsafe as the returned price update may be arbitrarily far in the past … If you are considering using this function, it may be safer / easier to use `getPriceNoOlderThan`." [7](#0-6) 

The safe alternative, `getPriceNoOlderThan(id, age)`, reverts if the stored price is older than `age` seconds. [8](#0-7) 

`latestRoundData()` does return `updatedAt = price.publishTime`, which technically allows a careful caller to check staleness themselves. However, `latestAnswer()` and `getAnswer()` return only the raw `int256` price with zero staleness metadata — making it impossible for callers of those functions to detect stale data. Neither function reverts on stale input.

---

### Impact Explanation

Any DeFi protocol (lending, derivatives, liquidation engine) that deploys `PythAggregatorV3` as a Chainlink-compatible price feed and reads from `latestAnswer()` or `latestRoundData()` without manually validating `updatedAt` will silently consume an arbitrarily old price. This can cause:

- Incorrect collateral valuations and under-collateralized loans going unliquidated
- Incorrect or missed liquidations at wrong prices
- Arbitrage against the protocol using the stale on-chain price vs. the true market price

The severity is amplified because the contract is explicitly marketed as a Chainlink drop-in replacement. Protocols migrating from Chainlink expect `latestRoundData()` to behave safely (Chainlink's own aggregators revert on incomplete rounds); the Pyth adapter silently returns stale data instead.

---

### Likelihood Explanation

An unprivileged attacker can exploit this passively: simply refrain from calling `updateFeeds()` (or allow the price update to lapse), then interact with a victim protocol that reads from the adapter. No privileged access is required. The Pyth network is a pull oracle — prices are only updated when someone pays to push them on-chain. If no one updates the feed, the stored price ages indefinitely and `getPriceUnsafe()` returns it without complaint.

---

### Recommendation

Replace `getPriceUnsafe()` with `getPriceNoOlderThan(priceId, maxAge)` in all price-returning functions, where `maxAge` is a constructor-configurable parameter (e.g., defaulting to a reasonable value such as 60 seconds). For `latestAnswer()` and `getAnswer()`, which cannot surface a timestamp to the caller, this is the only way to enforce freshness. For `latestRoundData()` and `getRoundData()`, it provides defense-in-depth even for callers that do check `updatedAt`.

```solidity
uint public maxAge;

constructor(address _pyth, bytes32 _priceId, uint _maxAge) {
    pyth = IPyth(_pyth);
    priceId = _priceId;
    maxAge = _maxAge;
}

function latestAnswer() public view virtual returns (int256) {
    PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
    return int256(price.price);
}
```

---

### Proof of Concept

1. Deploy `PythAggregatorV3` pointing to a live Pyth contract and a price feed ID.
2. Do not call `updateFeeds()` for an extended period (hours/days).
3. Call `latestAnswer()` — it returns the stale price from days ago with no revert.
4. Call `latestRoundData()` — it returns the same stale price; `updatedAt` reflects the old `publishTime` but the function itself does not revert.
5. A lending protocol reading `latestAnswer()` has no way to detect the staleness and will use the wrong price for all collateral calculations. [1](#0-0) [9](#0-8)
