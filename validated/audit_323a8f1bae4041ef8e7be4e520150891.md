### Title
Missing Zero Address Check for Immutable `pyth` Parameter in Constructor - (File: target_chains/ethereum/contracts/contracts/aave/PythPriceOracleGetter.sol)

### Summary
`PythPriceOracleGetter.sol` accepts a `pyth` address in its constructor and stores it permanently via `PythAssetRegistry.setPyth()`. There is no zero address check for this parameter. Because `setPyth` is `internal` and called only from the constructor, the stored address cannot be updated after deployment. If `pyth` is mistakenly set to `address(0)`, the oracle is permanently broken.

### Finding Description
In `PythPriceOracleGetter.sol`, the constructor accepts a `pyth` address and passes it directly to `PythAssetRegistry.setPyth(pyth)` without any zero address validation:

```solidity
// PythPriceOracleGetter.sol lines 29-50
constructor(
    address pyth,          // <-- no zero address check
    address[] memory assets,
    bytes32[] memory priceIds,
    address baseCurrency,
    uint256 baseCurrencyUnit,
    uint validTimePeriodSeconds
) {
    if (baseCurrencyUnit == 0) {
        revert InvalidBaseCurrencyUnit();
    }
    PythAssetRegistry.setPyth(pyth);   // stored with no validation
    ...
}
``` [1](#0-0) 

In `PythAssetRegistry`, `setPyth` is declared `internal` and has no zero address guard:

```solidity
function setPyth(address pythAddress) internal {
    _registryState.pyth = payable(pythAddress);
}
``` [2](#0-1) 

There is no external or public setter for `_registryState.pyth` anywhere in `PythAssetRegistry`, making the stored address effectively permanent after construction. [3](#0-2) 

Additionally, `baseCurrency` is also passed without a zero address check and is assigned to the `immutable` variable `BASE_CURRENCY`:

```solidity
address public immutable override BASE_CURRENCY;
...
PythAssetRegistry.setBaseCurrency(baseCurrency, baseCurrencyUnit);
BASE_CURRENCY = _registryState.BASE_CURRENCY;
``` [4](#0-3) 

### Impact Explanation
If `pyth` is set to `address(0)` at deployment:
- Every call to `getAssetPrice()` for any non-base-currency asset invokes `pyth().getPriceNoOlderThan(...)`, which is a call into `address(0)`.
- Calls into `address(0)` return empty data in the EVM; the ABI decoder will either revert or return zero/garbage values.
- This permanently breaks the Aave price oracle integration — all asset prices become unavailable or incorrect.
- Since there is no recovery path (no setter, no upgrade mechanism in this non-upgradeable contract), redeployment is the only fix. [5](#0-4) 

### Likelihood Explanation
Deployment misconfiguration is a realistic and documented class of error, especially in multi-parameter constructors. The original report describes exactly this scenario in analogous contracts. The `PythPriceOracleGetter` constructor takes six parameters; a deployer scripting error or copy-paste mistake could easily pass `address(0)` for `pyth`. Unlike upgradeable contracts, this contract has no recovery path, amplifying the consequence of even a low-probability mistake.

### Recommendation
Add explicit zero address checks for `pyth` and `baseCurrency` at the top of the `PythPriceOracleGetter` constructor, mirroring the pattern already used in other Pyth contracts such as `EntropyUpgradable` and `SchedulerUpgradeable`:

```solidity
constructor(...) {
    require(pyth != address(0), "pyth is zero address");
    require(baseCurrency != address(0), "baseCurrency is zero address");
    if (baseCurrencyUnit == 0) { revert InvalidBaseCurrencyUnit(); }
    ...
}
```

Alternatively, add the zero address guard inside `PythAssetRegistry.setPyth()` so all callers benefit. [2](#0-1) 

### Proof of Concept
1. Deploy `PythPriceOracleGetter` with `pyth = address(0)`, valid `assets`/`priceIds`, a non-zero `baseCurrency`, and a valid `baseCurrencyUnit`.
2. The constructor completes without revert — no zero address check exists.
3. Call `getAssetPrice(someAsset)` where `someAsset != BASE_CURRENCY`.
4. Execution reaches `pyth().getPriceNoOlderThan(priceId, validTimePeriodSeconds)` — a call to `address(0)`.
5. The call returns empty bytes; the ABI decode of `PythStructs.Price` reverts or returns zeroed data, making the oracle permanently non-functional.
6. No admin function exists to update `_registryState.pyth`; the contract must be redeployed. [6](#0-5)

    /// @inheritdoc IPriceOracleGetter
    function getAssetPrice(
        address asset
    ) external view override returns (uint256) {
        bytes32 priceId = _registryState.assetsPriceIds[asset];
        if (asset == BASE_CURRENCY) {
            return BASE_CURRENCY_UNIT;
        }
        if (priceId == 0) {
            revert PythErrors.PriceFeedNotFound();
        }
        PythStructs.Price memory price = pyth().getPriceNoOlderThan(
            priceId,
            PythAssetRegistry.validTimePeriodSeconds()
        );

        // Aave is not using any price feeds < 0 for now.
        if (price.price <= 0) {
            revert InvalidNonPositivePrice();
        }
        uint256 normalizedPrice = uint64(price.price);
        int32 normalizerExpo = price.expo + int8(BASE_NUM_DECIMALS);
        bool isNormalizerExpoNeg = normalizerExpo < 0;
        uint256 normalizer = isNormalizerExpoNeg
            ? 10 ** uint32(-normalizerExpo)
            : 10 ** uint32(normalizerExpo);

        // this check prevents overflow in normalized price
        if (!isNormalizerExpoNeg && normalizer > type(uint192).max) {
            revert NormalizationOverflow();
        }

        normalizedPrice = isNormalizerExpoNeg
            ? normalizedPrice / normalizer
            : normalizedPrice * normalizer;

        if (normalizedPrice <= 0) {
            revert InvalidNonPositivePrice();
        }

        return normalizedPrice;
    }

contract PythAssetRegistry {
    PythAssetRegistryStorage.State _registryState;

    /**
     * @dev Emitted after the base currency is set
     * @param baseCurrency The base currency of used for price quotes
     * @param baseCurrencyUnit The unit of the base currency
     */
    event BaseCurrencySet(
        address indexed baseCurrency,
        uint256 baseCurrencyUnit
    );

    /**
     * @dev Emitted after the price source of an asset is updated
     * @param asset The address of the asset
     * @param source The priceId of the asset
     */
    event AssetSourceUpdated(address indexed asset, bytes32 indexed source);

    function pyth() public view returns (IPyth) {
        return IPyth(_registryState.pyth);
    }

    function setPyth(address pythAddress) internal {
        _registryState.pyth = payable(pythAddress);
    }

    function setAssetsSources(
        address[] memory assets,
        bytes32[] memory priceIds
    ) internal {
        if (assets.length != priceIds.length) {
            revert InconsistentParamsLength();
        }
        for (uint256 i = 0; i < assets.length; i++) {
            _registryState.assetsPriceIds[assets[i]] = priceIds[i];
            emit AssetSourceUpdated(assets[i], priceIds[i]);
        }
    }

    function setBaseCurrency(
        address baseCurrency,
        uint256 baseCurrencyUnit
    ) internal {
        _registryState.BASE_CURRENCY = baseCurrency;
        _registryState.BASE_CURRENCY_UNIT = baseCurrencyUnit;
        emit BaseCurrencySet(baseCurrency, baseCurrencyUnit);
    }

    function setValidTimePeriodSeconds(uint validTimePeriodInSeconds) internal {
        _registryState.validTimePeriodSeconds = validTimePeriodInSeconds;
    }

    function validTimePeriodSeconds() public view returns (uint) {
        return _registryState.validTimePeriodSeconds;
    }
}
