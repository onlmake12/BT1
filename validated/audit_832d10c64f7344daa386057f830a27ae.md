### Title
Unconditional `expect()` on `txs_sizes` in `FeeRateCollector::statistics` panics and crashes node when legacy pre-v0.106 `BlockExt` records are in the tip window — (`rpc/src/util/fee_rate.rs`)

---

### Summary

`FeeRateCollector::statistics` calls `.expect()` unconditionally on `BlockExt::txs_sizes`. When a node's DB contains pre-v0.106 blocks (5-field `BlockExt`, deserialized with `txs_sizes: None`) and those blocks fall within the RPC's sliding tip window (up to 101 blocks), any unprivileged caller invoking `get_fee_rate_statistics` or `get_fee_rate_statics` triggers a Rust panic that terminates the node process.

---

### Finding Description

**Panic site** — `rpc/src/util/fee_rate.rs` line 93:

```rust
let txs_sizes = txs_sizes.expect("expect txs_size's length >= 1");
```

This line is inside the closure passed to `collect()` and executes for **every** block in the tip window with no prior `None` guard. [1](#0-0) 

**How `txs_sizes: None` enters the live path** — `store/src/store.rs` `get_block_ext` reads the raw DB bytes and branches on `count_extra_fields()`. When the stored record has 5 fields (old format, `count_extra_fields() == 0`), it calls `reader.into()`: [2](#0-1) 

That conversion impl (`From<packed::BlockExtReader<'r>> for core::BlockExt`) hard-codes `txs_sizes: None`: [3](#0-2) 

**Schema confirmation** — The old molecule schema `BlockExt` has `FIELD_COUNT = 5` and has no `cycles` or `txs_sizes` fields; `BlockExtV1` (7 fields) added them in v0.106: [4](#0-3) 

**RPC entry point** — Both `get_fee_rate_statistics` and the deprecated `get_fee_rate_statics` call `FeeRateCollector::statistics` directly with no pre-flight guard: [5](#0-4) 

**Tip window** — `collect()` iterates `[max(1, tip − target), tip]`, where `target ≤ 101`. Any legacy block in that range triggers the panic: [6](#0-5) 

**No migration converts old records** — The migration list contains no pass that rewrites `COLUMN_BLOCK_EXT` rows from 5-field to 7-field format. `get_block_ext` is purely a read-time compatibility shim; it never upgrades the stored bytes.


---

### Impact Explanation

A Rust `panic!` in a non-`catch_unwind` context terminates the node process. The RPC handler does not wrap `statistics()` in any panic boundary. Result: **full node crash / denial of service** for any node whose DB contains legacy blocks within the current tip window.

---

### Likelihood Explanation

The precondition is met on any node that:

1. Was running CKB before v0.106 and upgraded in-place (DB not wiped), **and**
2. The current chain tip is within ~100 blocks of the block height at which the node first ran v0.106 code.

On mainnet this window is ~17 minutes of wall-clock time after upgrade. On testnets or private networks with low block heights the window can be permanent (e.g., a testnet upgraded at block 50 with tip still below 151). The RPC is unauthenticated by default, so any network-reachable peer can trigger it.

---

### Recommendation

Replace the unconditional `.expect()` with a graceful skip:

```rust
// In the collect() closure:
let Some(txs_sizes) = txs_sizes else {
    return fee_rates; // legacy block — no size data, skip
};
```

This matches the existing pattern used for `cycles` (checked with `if let Some(cycles) = cycles` on line 97) and makes the function safe for mixed-format databases. [7](#0-6) 

---

### Proof of Concept

Using the existing `DummyFeeRateProvider` test harness:

```rust
#[test]
#[should_panic(expected = "expect txs_size's length >= 1")]
fn test_legacy_block_ext_panics() {
    use ckb_types::core::{BlockExt, Capacity};
    let mut provider = DummyFeeRateProvider::new(30);
    provider.append(1, BlockExt {
        received_at: 0,
        total_difficulty: 0u64.into(),
        total_uncles_count: 0,
        verified: Some(true),
        txs_fees: vec![Capacity::shannons(1000)], // non-empty fees
        cycles: Some(vec![100]),
        txs_sizes: None,  // ← legacy 5-field record
    });
    // This panics at the .expect() on line 93
    let _ = FeeRateCollector::new(&provider).statistics(None);
}
```

On a live node: call `get_fee_rate_statistics` via JSON-RPC on any node whose DB was created before v0.106 and whose tip is within 101 blocks of the upgrade height. The node process will terminate with `thread 'main' panicked at 'expect txs_size's length >= 1'`.
