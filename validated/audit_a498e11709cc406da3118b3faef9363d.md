### Title
`Executor.execute()` Does Not Validate `msg.value` Against Encoded `gi.value` — (File: `target_chains/ethereum/contracts/contracts/executor/Executor.sol`)

---

### Summary

`Executor.execute()` is `public payable` but never checks that `msg.value` equals or covers the `gi.value` encoded in the governance VAA. Because the contract also exposes a `receive()` fallback that allows it to accumulate ETH, any unprivileged caller can front-run a legitimate governance execution with `msg.value = 0`, causing the Executor to drain its own accumulated ETH balance to fund the call instead of the intended caller's ETH.

---

### Finding Description

`Executor.execute()` decodes `gi.value` from the Wormhole VAA payload and forwards exactly that amount to the target:

```solidity
(success, response) = address(callAddress).call{value: gi.value}(
    gi.callData
);
``` [1](#0-0) 

The function is declared `public payable` with no guard on `msg.value`:

```solidity
function execute(
    bytes memory encodedVm
) public payable returns (bytes memory response) {
``` [2](#0-1) 

The contract also has an unrestricted `receive()`:

```solidity
/// @dev Called when `msg.value` is not zero and the call data is empty.
receive() external payable {}
``` [3](#0-2) 

There is no statement of the form `require(msg.value >= gi.value)` anywhere in `execute()`. [4](#0-3) 

Two distinct problems follow:

**A — ETH drain via front-running.** When the Executor holds accumulated ETH (received via `receive()` or from prior over-funded calls), any unprivileged actor can observe a pending governance VAA with `gi.value > 0` in the mempool and call `execute(vaa)` with `msg.value = 0`. The Executor's own balance funds the call; the sequence number is consumed; the legitimate executor's transaction reverts. The Executor's ETH is drained without the attacker spending any ETH.

**B — Excess ETH locked.** If a legitimate caller sends `msg.value > gi.value`, the surplus is silently retained by the Executor. There is no refund path and no withdrawal function for arbitrary ETH, so the excess is effectively locked until a future governance call happens to consume it.

---

### Impact Explanation

- **ETH drain**: The Executor's accumulated ETH balance can be depleted by an unprivileged actor front-running any governance VAA that carries `gi.value > 0`. This can prevent subsequent governance calls that legitimately require ETH from succeeding (governance DoS / loss of funds held by the Executor).
- **ETH lock**: Callers who over-fund `execute()` permanently lose the excess ETH to the contract with no recovery mechanism. [1](#0-0) 

---

### Likelihood Explanation

**Low-to-Medium.** The attacker needs no special privilege — `execute()` is `public`. The only precondition is that the Executor holds a non-zero ETH balance (plausible given the `receive()` function) and that a governance VAA with `gi.value > 0` is pending in the mempool. Front-running is straightforward on EVM chains with a public mempool.

---

### Recommendation

Add a check that `msg.value` exactly equals (or is at least) `gi.value`, and refund any surplus:

```solidity
// After parsing gi:
require(msg.value >= gi.value, "Insufficient msg.value");

(success, response) = address(callAddress).call{value: gi.value}(gi.callData);

// Refund surplus
if (msg.value > gi.value) {
    (bool refunded, ) = msg.sender.call{value: msg.value - gi.value}("");
    require(refunded, "Refund failed");
}
```

This mirrors the fix applied to `BridgedGovernor` and ensures the caller always provides exactly the ETH the governance instruction requires, eliminating both the drain and the lock vectors.

---

### Proof of Concept

1. Governance publishes a VAA encoding `gi.value = 1 ether`, `gi.callAddress = someContract`, `gi.callData = somePayload`.
2. The Executor contract holds 2 ETH accumulated via prior `receive()` calls.
3. Attacker observes the VAA in the mempool before the legitimate executor submits their transaction.
4. Attacker calls `Executor.execute(vaa)` with `msg.value = 0` and a higher gas price.
5. `verifyGovernanceVM` passes (valid VAA, correct emitter, sequence `N`). `lastExecutedSequence` is updated to `N`.
6. `address(callAddress).call{value: 1 ether}(callData)` executes using the Executor's own balance — 1 ETH is drained from the Executor.
7. Legitimate executor's transaction reverts with `MessageOutOfOrder` (sequence already consumed).
8. Net result: Executor lost 1 ETH; attacker spent 0 ETH; governance action executed (correctly, but funded from the wrong source). Repeat for every subsequent VAA with `gi.value > 0` until the Executor is empty. [5](#0-4) [1](#0-0)

    function execute(
        bytes memory encodedVm
    ) public payable returns (bytes memory response) {
        IWormhole.VM memory vm = verifyGovernanceVM(encodedVm);

        GovernanceInstruction memory gi = parseGovernanceInstruction(
            vm.payload
        );

        if (gi.targetChainId != chainId && gi.targetChainId != 0)
            revert ExecutorErrors.InvalidGovernanceTarget();

        if (
            gi.action != ExecutorAction.Execute ||
            gi.executorAddress != address(this)
        ) revert ExecutorErrors.DeserializationError();

        // Check if the gi.callAddress is a contract account.
        uint len;
        address callAddress = address(gi.callAddress);
        assembly {
            len := extcodesize(callAddress)
        }
        if (len == 0) revert ExecutorErrors.InvalidContractTarget();

        bool success;
        (success, response) = address(callAddress).call{value: gi.value}(
            gi.callData
        );

        // Check if the call was successful or not.
        if (!success) {
            // If there is return data, the delegate call reverted with a reason or a custom error, which we bubble up.
            if (response.length > 0) {
                // The first word of response is the length, so when we call revert we add 1 word (32 bytes)
                // to give the pointer to the beginning of the revert data and pass the size as the second argument.

                    let returndata_size := mload(response)
                    revert(add(32, response), returndata_size)
                }
            } else {
                revert ExecutorErrors.ExecutionReverted();
            }
        }
    }

    function verifyGovernanceVM(
        bytes memory encodedVM
    ) internal returns (IWormhole.VM memory parsedVM) {
        (IWormhole.VM memory vm, bool valid, ) = wormhole.parseAndVerifyVM(
            encodedVM
        );

        if (!valid) revert ExecutorErrors.InvalidWormholeVaa();

        if (
            vm.emitterChainId != ownerEmitterChainId ||
            vm.emitterAddress != ownerEmitterAddress
        ) revert ExecutorErrors.UnauthorizedEmitter();

        if (vm.sequence <= lastExecutedSequence)
            revert ExecutorErrors.MessageOutOfOrder();

        lastExecutedSequence = vm.sequence;

        return vm;
