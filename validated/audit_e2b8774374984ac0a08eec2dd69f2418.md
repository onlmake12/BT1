Audit Report

## Title
Integer Division by Zero in `check_purge` Network-Group Eviction Allows Single-Group Monopoly to Permanently Block Peer Discovery — (`network/src/peer_store/peer_store_impl.rs`)

## Summary
When `check_purge` is invoked and the first-pass eviction (non-connectable addresses) yields nothing, the second pass groups addresses by network segment and calls `take(len / 2)` where `len` is the number of distinct groups. If all stored addresses belong to a single `/16` network group, `len = 1`, integer division yields `0`, `take(0)` produces an empty iterator, `candidate_peers` is empty, and `Err(PeerStoreError::EvictionFailed)` is returned. Because `add_addr` propagates this error via `self.check_purge()?`, every subsequent discovery-sourced address insertion fails permanently for the lifetime of the process.

## Finding Description

**Root cause — `take(len / 2)` with `len = 1`:**

In `check_purge`, after the first pass removes nothing (all freshly-added addresses are connectable), the second pass executes:

```rust
let len = peers_by_network_group.len();   // = 1 when all addrs share one /16
...
peers.into_iter().take(len / 2)           // take(0) — empty iterator
``` [1](#0-0) 

`take(0)` means the `flat_map` closure (which requires `addrs.len() > 4`) is never reached. `candidate_peers` is empty, triggering:

```rust
if candidate_peers.is_empty() {
    return Err(PeerStoreError::EvictionFailed.into());
}
``` [2](#0-1) 

**Why freshly-added addresses survive the first pass:**

`AddrInfo::new` initializes `last_connected_at_ms=0`, `last_tried_at_ms=0`, `attempts_count=0`. [3](#0-2) 

`is_connectable` returns `true` for all three conditions: `tried_in_last_minute` is false (0 < now_ms − 60000), the `attempts_count >= ADDR_MAX_RETRIES(3)` branch is false (0 < 3), and the `attempts_count >= ADDR_MAX_FAILURES(10)` branch is false (0 < 10). [4](#0-3) 

**Why a single /16 is sufficient:**

The `Group` enum keys on the first two octets of IPv4, so all addresses in `10.0.0.0/16` (65536 addresses) map to `Group::IP4([10, 0])`. [5](#0-4) 

`ADDR_COUNT_LIMIT = 16384`, which fits entirely within a single /16. [6](#0-5) 

**Error propagation:**

`add_addr` calls `self.check_purge()?`, so the `EvictionFailed` error is returned to every caller attempting to insert a discovery-sourced address. [7](#0-6) 

## Impact Explanation

Once the peer store is saturated with 16384 addresses from a single /16, the node permanently rejects all new addresses from the discovery protocol via `add_addr`. The node cannot learn new outbound peers, degrading its ability to sync blocks and relay transactions. This matches **Low (501–2000 points) — any other important performance improvements for CKB**, as it impairs a node's peer discovery capability for the lifetime of the process without crashing it.

## Likelihood Explanation

The CKB discovery protocol (`GetNodes`/`Nodes`) is unauthenticated. An attacker controlling a single peer can advertise 16384 syntactically valid multiaddrs within the same /16 subnet (e.g., `10.0.0.0–10.0.63.255`). The addresses do not need to be reachable — `add_addr` performs no reachability check before inserting. No proof-of-work, key material, or privileged access is required. The attack is deterministic and repeatable.

## Recommendation

Replace `take(len / 2)` with a minimum-of-one guard to ensure at least the largest group is always a candidate for eviction:

```rust
peers.into_iter().take((len / 2).max(1))
```

Additionally, consider enforcing a per-network-group cap during `add_addr` to prevent monopoly pre-filling before `check_purge` is even reached.

## Proof of Concept

```rust
let mut peer_store = PeerStore::default();
// Fill to ADDR_COUNT_LIMIT with addresses from the same /16 (10.0.x.x)
for i in 0u32..16384 {
    let ip = std::net::Ipv4Addr::new(10, 0, (i / 256) as u8, (i % 256) as u8);
    let addr: Multiaddr = format!("/ip4/{}/tcp/8115", ip).parse().unwrap();
    let _ = peer_store.add_addr(addr, Flags::COMPATIBILITY);
}
// Attempt to add a legitimate address from a different /16
let new_addr: Multiaddr = "/ip4/1.2.3.4/tcp/8115".parse().unwrap();
let result = peer_store.add_addr(new_addr, Flags::COMPATIBILITY);
// Buggy: Err(EvictionFailed) — Fixed: Ok(())
assert!(result.is_ok(), "peer discovery permanently blocked");
```

        }
        true
    }
