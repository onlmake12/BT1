### Title
Unrestricted `clear_tx_pool` RPC Allows Any Caller to Wipe the Entire Transaction Pool — (`File: rpc/src/module/pool.rs`)

### Summary
The `clear_tx_pool` RPC method removes every pending transaction from the node's mempool without performing any caller authentication or authorization check. The RPC server itself has no authentication middleware. Any entity that can reach the RPC HTTP endpoint — including any local process or any remote caller when the port is exposed — can silently destroy all pending transactions at will, with no credential required.

### Finding Description

**Root cause — no authorization in `clear_tx_pool`:**

`clear_tx_pool` is implemented in `rpc/src/module/pool.rs`:

```rust
fn clear_tx_pool(&self) -> Result<()> {
    let snapshot = Arc::clone(&self.shared.snapshot());
    let tx_pool = self.shared.tx_pool_controller();
    tx_pool
        .clear_pool(snapshot)
        .map_err(|err| RPCError::custom(RPCError::Invalid, err.to_string()))?;
    Ok(())
}
```

There is no caller identity check, no token, no role guard, and no rate limit before `clear_pool` is invoked. [1](#0-0) 

**Root cause — no authentication middleware in the RPC server:**

The HTTP handler `handle_jsonrpc` in `rpc/src/server.rs` receives the raw request body, deserializes it as JSON-RPC, and dispatches it directly to the registered handler. There is no `Authorization` header check, no token validation, and no IP allowlist enforcement at the application layer. The server is built with `CorsLayer::permissive()`, which accepts requests from any origin.

```rust

The same absence of authorization applies to `remove_transaction` (removes a specific tx and all its dependents) and `clear_tx_verify_queue` (clears the verification queue). [4](#0-3) 

**Analogy to the reference vulnerability:**
Just as `mint()` is a privileged token-creation operation that should be restricted to the owner, `clear_tx_pool` is a privileged token-destruction operation (destroying pending economic activity) that should be restricted to the node operator. Neither has any access control in the code.

### Impact Explanation

Any caller who can reach the RPC port can:
1. Instantly wipe the entire mempool with a single HTTP POST.
2. Repeat the call in a tight loop to prevent any transaction from ever being confirmed.
3. Target specific high-value transactions using `remove_transaction` to selectively censor or grief users.

Concrete consequences:
- **User funds delayed or lost**: transactions are silently dropped and must be resubmitted, potentially at higher fees.
- **Miner revenue loss**: all pending fee-bearing transactions are destroyed before block assembly.
- **Sustained mempool DoS**: the attacker can continuously clear the pool faster than users can resubmit, effectively halting transaction processing on the targeted node.
- **Selective censorship**: `remove_transaction` allows targeted removal of specific transactions (e.g., a competitor's high-fee transaction).

### Likelihood Explanation

The default `rpc.listen_address` is `127.0.0.1:8114` (localhost). However:
- Many operators intentionally expose the RPC port publicly for wallet/dApp integration; the documentation warns against this but provides no enforcement mechanism.
- Any local process on the same machine (e.g., a malicious dependency, a compromised co-located service) can reach the localhost endpoint with no credential.
- The RPC server has `CorsLayer::permissive()`, meaning a malicious web page can trigger the call via a browser on the same machine (CSRF-style).
- The miner client supports sending HTTP Basic Auth credentials in the URL (`parse_authorization` in `miner/src/client.rs`), but the **server never validates them** — confirming the server-side auth gap is not an oversight of the client but a missing server feature. [5](#0-4) 

### Recommendation

1. **Add server-side authentication**: implement an optional bearer token or HTTP Basic Auth check in `handle_jsonrpc` (or as an Axum middleware layer) that gates all state-mutating RPC calls.
2. **Introduce a privilege tier**: separate read-only RPC methods from destructive/administrative ones (`clear_tx_pool`, `remove_transaction`, `clear_tx_verify_queue`, `set_network_active`, `set_ban`, `clear_banned_addresses`) and require explicit credentials for the latter tier.
3. **Remove or restrict `clear_tx_pool` from the default module list**: it is currently included in the default `modules` list alongside `Pool`, making it reachable by default.

### Proof of Concept

With the node running at default configuration (`127.0.0.1:8114`), any local process can execute:

```bash
# Step 1: Submit a transaction to the pool
curl -s -X POST http://127.0.0.1:8114 \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"send_transaction","params":[<tx_json>,"passthrough"],"id":1}'

# Step 2: Immediately wipe the entire pool — no credentials required
curl -s -X POST http://127.0.0.1:8114 \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"clear_tx_pool","params":[],"id":2}'
```

The second call succeeds unconditionally. Repeating it in a loop prevents any transaction from surviving long enough to be included in a block. The `remove_transaction` variant allows targeted removal of a known tx hash, enabling selective censorship without clearing the entire pool. [1](#0-0) [6](#0-5)

async fn handle_jsonrpc<T: Default + Metadata>(
    Extension(io): Extension<Arc<MetaIoHandler<T>>>,
    req_body: Bytes,
) -> Response {
    let make_error_response = |error| {
        Json(jsonrpc_core::Failure {
            jsonrpc: Some(jsonrpc_core::Version::V2),
            id: jsonrpc_core::Id::Null,
            error,
        })
        .into_response()
    };

    let req = match std::str::from_utf8(req_body.as_ref()) {
        Ok(req) => req,
        Err(_) => {
            return make_error_response(jsonrpc_core::Error::parse_error());
        }
    };

    let req = serde_json::from_str::<Request>(req);
    match req {
        Err(_error) => {
            let response = RpcResponse::from(
                Error::new(ErrorCode::ParseError),
                Some(jsonrpc_core::Version::V2),
            );

            serde_json::to_string(&response)
                .map(|json| {
                    (
                        [(axum::http::header::CONTENT_TYPE, "application/json")],
                        json,
                    )
                        .into_response()
                })
                .unwrap_or_else(|_| StatusCode::INTERNAL_SERVER_ERROR.into_response())
        }
        Ok(request) => match request {
            Request::Single(call) => {
                let result = io.handle_call(call, T::default()).await;

                if let Some(response) = result {
                    serde_json::to_string(&response)
                        .map(|json| {
                            (
                                [(axum::http::header::CONTENT_TYPE, "application/json")],
                                json,
                            )
                                .into_response()
                        })
                        .unwrap_or_else(|_| StatusCode::INTERNAL_SERVER_ERROR.into_response())
                } else {
                    StatusCode::NO_CONTENT.into_response()
                }
            }
            Request::Batch(calls) => {
                if let Some(batch_size) = JSONRPC_BATCH_LIMIT.get()
                    && calls.len() > *batch_size
                {
                    return make_error_response(jsonrpc_core::Error::invalid_params(format!(
                        "batch size is too large, expect it less than: {}",
                        batch_size
                    )));
                }

                let stream = stream::iter(calls)
                    .then(move |call| {
                        let io = Arc::clone(&io);
                        async move { io.handle_call(call, T::default()).await }
                    })
                    .filter_map(|response| async move { response });

                (
                    [(axum::http::header::CONTENT_TYPE, "application/json")],
                    StreamBodyAs::json_array(stream),
                )
                    .into_response()
            }
        },
    }
}

fn parse_authorization(url: &Uri) -> Option<HeaderValue> {
    let a: Vec<&str> = url.authority()?.as_str().split('@').collect();
    if a.len() >= 2 {
        if a[0].is_empty() {
            return None;
        }
        let mut encoded = "Basic ".to_string();
        base64::prelude::BASE64_STANDARD.encode_string(a[0], &mut encoded);
        let mut header = HeaderValue::from_str(&encoded).unwrap();
        header.set_sensitive(true);
        Some(header)
    } else {
        None
    }
}
