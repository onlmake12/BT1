### Title
Excess ETH Overpayment in `requestPriceUpdatesWithCallback` Silently Credited to Provider Without Refund - (File: target_chains/ethereum/contracts/contracts/echo/Echo.sol)

### Summary
`Echo.sol`'s `requestPriceUpdatesWithCallback` enforces only a minimum fee floor (`msg.value >= requiredFee`) but never refunds any excess. The entire surplus above `_state.pythFeeInWei` is silently credited to the provider's `accruedFeesInWei`. The user receives no additional service and has no mechanism to recover the overpaid ETH, while the provider (or their fee manager) can withdraw it at will via `withdrawAsFeeManager`.

### Finding Description
In `requestPriceUpdatesWithCallback`, the stored provider fee is computed as:

```solidity
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);
``` [1](#0-0) 

The check only enforces a minimum:

```solidity
uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
if (msg.value < requiredFee) revert InsufficientFee();
``` [2](#0-1) 

Any `msg.value` above `requiredFee` is silently absorbed: the full `msg.value - pythFeeInWei` is stored as `req.fee`, and when `executeCallback` is later called, that entire amount (including the excess) is credited to the provider:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
``` [3](#0-2) 

The provider (or their designated fee manager) can then withdraw the excess via `withdrawAsFeeManager`:

```solidity
function withdrawAsFeeManager(
    address provider,
    uint128 amount
) external override {
    require(
        msg.sender == _state.providers[provider].feeManager,
        "Only fee manager"
    );
``` [4](#0-3) 

There is no `cancelRequest` or refund path in the contract for the requester to recover overpaid ETH.

The interface documentation states `"The msg.value must be equal to getFee(callbackGasLimit)"`, implying exact payment is expected, yet the contract silently accepts and permanently retains any surplus. [5](#0-4) 

### Impact Explanation
A user who sends more ETH than `getFee(provider, callbackGasLimit, priceIds)` — whether by using a round number, a stale fee estimate, or a frontend rounding error — permanently loses the excess to the provider. The user receives exactly the same service as if they had paid the exact fee. The provider gains a windfall they did not earn. There is no on-chain refund path for the requester.

### Likelihood Explanation
Realistic. Users and integrating contracts commonly send a small buffer above the quoted fee to guard against fee increases between the `getFee` call and the transaction landing. Frontend SDKs that add a safety margin, or contracts that hardcode a slightly higher value, will silently donate the surplus to the provider on every request. The `executeCallback` function is also `payable` and any ETH sent with it is similarly absorbed into provider fees with no refund.

### Recommendation
1. Compute the exact required fee inside `requestPriceUpdatesWithCallback` and revert (or refund) if `msg.value` exceeds it, or explicitly refund the surplus to `msg.sender`:
   ```solidity
   uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
   if (msg.value < requiredFee) revert InsufficientFee();
   uint256 excess = msg.value - requiredFee;
   if (excess > 0) {
       (bool ok,) = msg.sender.call{value: excess}("");
       require(ok, "Refund failed");
   }
   req.fee = SafeCast.toUint96(requiredFee - _state.pythFeeInWei);
   ```
2. Remove the `payable` modifier from `executeCallback` if no ETH is legitimately needed from the caller, or add equivalent refund logic there.

### Proof of Concept
1. Provider registers with `baseFeeInWei = 1 wei`, `feePerFeedInWei = 10 wei`, `feePerGasInWei = 1 wei`.
2. `getFee(provider, 100_000, [priceId])` returns `1 + 1 + 10 + 100_000 = 100_012 wei`.
3. User calls `requestPriceUpdatesWithCallback{value: 1 ether}(...)` — a common pattern when a contract pre-funds itself with 1 ETH.
4. `req.fee = 1 ether - 1 wei` (pythFee) is stored.
5. Provider calls `executeCallback`; `accruedFeesInWei` for the provider increases by `~1 ether - pythFee_on_pyth_contract`.
6. Provider calls `withdrawAsFeeManager` and withdraws ~1 ETH.
7. User paid ~1 ETH for a service worth ~100_012 wei; the ~999_999_988 wei surplus is unrecoverable by the user. [6](#0-5)

    function requestPriceUpdatesWithCallback(
        address provider,
        uint64 publishTime,
        bytes32[] calldata priceIds,
        uint32 callbackGasLimit
    ) external payable override returns (uint64 requestSequenceNumber) {
        require(
            _state.providers[provider].isRegistered,
            "Provider not registered"
        );

        // FIXME: this comment is wrong. (we're not using tx.gasprice)
        // NOTE: The 60-second future limit on publishTime prevents a DoS vector where
        //      attackers could submit many low-fee requests for far-future updates when gas prices
        //      are low, forcing executors to fulfill them later when gas prices might be much higher.
        //      Since tx.gasprice is used to calculate fees, allowing far-future requests would make
        //      the fee estimation unreliable.
        require(publishTime <= block.timestamp + 60, "Too far in future");
        if (priceIds.length > MAX_PRICE_IDS) {
            revert TooManyPriceIds(priceIds.length, MAX_PRICE_IDS);
        }
        requestSequenceNumber = _state.currentSequenceNumber++;

        uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
        if (msg.value < requiredFee) revert InsufficientFee();

        Request storage req = allocRequest(requestSequenceNumber);
        req.sequenceNumber = requestSequenceNumber;
        req.publishTime = publishTime;
        req.callbackGasLimit = callbackGasLimit;
        req.requester = msg.sender;
        req.provider = provider;
        req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);

        // Create array with the right size
        req.priceIdPrefixes = new bytes8[](priceIds.length);

        // Copy only the first 8 bytes of each price ID to storage
        for (uint8 i = 0; i < priceIds.length; i++) {
            // Extract first 8 bytes of the price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }
            req.priceIdPrefixes[i] = prefix;
        }
        _state.accruedFeesInWei += _state.pythFeeInWei;

        emit PriceUpdateRequested(req, priceIds);
    }
