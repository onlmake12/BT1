### Title
`getPricesNoOlderThan` Uses Aggregate MAX Timestamp Instead of Per-Feed Timestamps for Freshness Validation - (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

The Pyth Pulse `Scheduler.sol` contract stores a single subscription-level `priceLastUpdatedAt` value equal to the **maximum** publish time across all feeds in a batch. `getPricesNoOlderThan` validates freshness against this aggregate MAX value rather than each feed's individual `publishTime`. As a result, a consumer calling `getPricesNoOlderThan(subscriptionId, priceIds, maxAge)` can receive individual price feeds whose per-feed `publishTime` is arbitrarily older than `maxAge`, while the age check silently passes.

---

### Finding Description

In `updatePriceFeeds`, after parsing all feeds, `_validateShouldUpdatePrices` computes a single `updateTimestamp` as the **maximum** publish time across all feeds in the batch:

```solidity
uint256 updateTimestamp = 0;
for (uint8 i = 0; i < priceFeeds.length; i++) {
    if (priceFeeds[i].price.publishTime > updateTimestamp) {
        updateTimestamp = priceFeeds[i].price.publishTime;
    }
}
``` [1](#0-0) 

This single aggregate value is then stored as the subscription-level `status.priceLastUpdatedAt`:

```solidity
status.priceLastUpdatedAt = latestPublishTime;
``` [2](#0-1) 

Meanwhile, `_storePriceUpdates` correctly stores each feed's full struct (including its own per-feed `publishTime`) keyed by feed ID:

```solidity
for (uint8 i = 0; i < priceFeeds.length; i++) {
    _state.priceUpdates[subscriptionId][priceFeeds[i].id] = priceFeeds[i];
}
``` [3](#0-2) 

`getPricesNoOlderThan` reads `status.priceLastUpdatedAt` (the aggregate MAX) for its age check, then delegates to `_getPricesInternal` which returns the raw per-feed structs with their individual (potentially much older) `publishTime` values: [4](#0-3) 

The mismatch: the age gate uses one aggregate value, but the returned data carries per-feed timestamps that can be far outside the requested age window.

---

### Impact Explanation

A DeFi protocol integrating Pulse calls `getPricesNoOlderThan(subscriptionId, priceIds, 60)` expecting **all** returned prices to be at most 60 seconds old. The check passes because `status.priceLastUpdatedAt` (the MAX across all feeds) is within 60 seconds. However, individual feeds for closed or illiquid markets may carry `publishTime` values from hours or days ago. The protocol consumes these stale prices believing they are fresh, potentially leading to incorrect liquidations, mispriced derivatives, or other financial harm.

This directly mirrors the external report: an aggregate value (`totalXPs[i]` = average across all questions) is used where a per-item value (per-question XP) is required, causing downstream consumers to receive incorrect data.

---

### Likelihood Explanation

Any Pulse subscription containing a mix of actively-traded and closed/illiquid markets (e.g., equities + crypto) will exhibit this behavior on every update. No special attacker action is required — the stale per-feed prices are already stored in the contract. Any whitelisted reader calling `getPricesNoOlderThan` is affected. The `SubscriptionStatus` documentation itself acknowledges the design:

> `priceLastUpdatedAt` — Timestamp of the last update. All feeds in the subscription are updated together. [5](#0-4) 

This confirms the aggregate MAX is intentionally stored, but the per-feed staleness gap is not surfaced to consumers of `getPricesNoOlderThan`.

---

### Recommendation

`getPricesNoOlderThan` should validate freshness against each individual feed's `publishTime` rather than the subscription-level `priceLastUpdatedAt`. Either:

1. Iterate over the returned feeds and revert (or skip) any feed whose `publishTime` violates the age constraint, or
2. Store per-feed timestamps separately and check each one in `getPricesNoOlderThan`.

---

### Proof of Concept

1. Create a Pulse subscription with two price IDs: one for an actively-traded asset (e.g., BTC/USD) and one for a closed equity market (e.g., AAPL/USD).
2. A keeper calls `updatePriceFeeds` with update data where BTC/USD has `publishTime = block.timestamp` and AAPL/USD has `publishTime = block.timestamp - 86400` (1 day old, last trading close).
3. `_validateShouldUpdatePrices` sets `updateTimestamp = block.timestamp` (the MAX). Staleness check passes. `status.priceLastUpdatedAt = block.timestamp`.
4. `_storePriceUpdates` stores both feeds with their respective `publishTime` values.
5. A consumer calls `getPricesNoOlderThan(subscriptionId, [AAPL_ID], 300)` (5-minute freshness requirement).
6. The age check compares `distance(block.timestamp, status.priceLastUpdatedAt)` = 0 ≤ 300. **Check passes.**
7. `_getPricesInternal` returns the AAPL feed with `publishTime = block.timestamp - 86400` — 1 day stale — while the consumer believes it is fresh. [6](#0-5) [7](#0-6)

    function updatePriceFeeds(
        uint256 subscriptionId,
        bytes[] calldata updateData
    ) external override {
        uint256 startGas = gasleft();

        SchedulerStructs.SubscriptionStatus storage status = _state
            .subscriptionStatuses[subscriptionId];
        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        if (!params.isActive) {
            revert SchedulerErrors.InactiveSubscription();
        }

        // Get the Pyth contract and parse price updates
        IPyth pyth = IPyth(_state.pyth);
        uint256 pythFee = pyth.getUpdateFee(updateData);

        // If we don't have enough balance, revert
        if (status.balanceInWei < pythFee) {
            revert SchedulerErrors.InsufficientBalance();
        }

        // Parse the price feed updates with an acceptable timestamp range of [0, now+10s].
        // Note: We don't want to reject update data if it contains a price
        // from a market that closed a few days ago, since it will contain a timestamp
        // from the last trading period. Thus, we use a minimum timestamp of zero while parsing,
        // and we enforce the past max validity ourselves in _validateShouldUpdatePrices using
        // the highest timestamp in the update data.
        status.balanceInWei -= pythFee;
        status.totalSpent += pythFee;
        uint64 curTime = SafeCast.toUint64(block.timestamp);
        (
            PythStructs.PriceFeed[] memory priceFeeds,
            uint64[] memory slots
        ) = pyth.parsePriceFeedUpdatesWithConfig{value: pythFee}(
                updateData,
                params.priceIds,
                0, // We enforce the past max validity ourselves in _validateShouldUpdatePrices
                curTime + FUTURE_TIMESTAMP_MAX_VALIDITY_PERIOD,
                false,
                true,
                false
            );

        // Verify all price feeds have the same Pythnet slot.
        // All feeds in a subscription must be updated at the same time.
        uint64 slot = slots[0];
        for (uint8 i = 1; i < slots.length; i++) {
            if (slots[i] != slot) {
                revert SchedulerErrors.PriceSlotMismatch();
            }
        }

        // Verify that update conditions are met, and that the timestamp
        // is more recent than latest stored update's. Reverts if not.
        uint256 latestPublishTime = _validateShouldUpdatePrices(
            subscriptionId,
            params,
            status,
            priceFeeds
        );

        // Update status and store the updates
        status.priceLastUpdatedAt = latestPublishTime;
        status.totalUpdates += priceFeeds.length;

        _storePriceUpdates(subscriptionId, priceFeeds);

        _processFeesAndPayKeeper(status, startGas, params.priceIds.length);

        emit PricesUpdated(subscriptionId, latestPublishTime);
    }

    function _validateShouldUpdatePrices(
        uint256 subscriptionId,
        SchedulerStructs.SubscriptionParams storage params,
        SchedulerStructs.SubscriptionStatus storage status,
        PythStructs.PriceFeed[] memory priceFeeds
    ) internal view returns (uint256) {
        // Use the most recent timestamp, as some asset markets may be closed.
        // Closed markets will have a publishTime from their last trading period.
        // Since we verify all updates share the same Pythnet slot, we still ensure
        // that all price feeds are synchronized from the same update cycle.
        uint256 updateTimestamp = 0;
        for (uint8 i = 0; i < priceFeeds.length; i++) {
            if (priceFeeds[i].price.publishTime > updateTimestamp) {
                updateTimestamp = priceFeeds[i].price.publishTime;
            }
        }

        // Calculate the minimum acceptable timestamp (clamped at 0)
        // The maximum acceptable timestamp is enforced by the parsePriceFeedUpdatesWithSlots call
        uint256 minAllowedTimestamp = (block.timestamp >
            PAST_TIMESTAMP_MAX_VALIDITY_PERIOD)
            ? (block.timestamp - PAST_TIMESTAMP_MAX_VALIDITY_PERIOD)
            : 0;

        // Validate that the update timestamp is not too old
        if (updateTimestamp < minAllowedTimestamp) {
            revert SchedulerErrors.TimestampTooOld(
                updateTimestamp,
                block.timestamp
            );
        }

        // Reject updates if they're older than the latest stored ones
        if (
            status.priceLastUpdatedAt > 0 &&
            updateTimestamp <= status.priceLastUpdatedAt
        ) {
            revert SchedulerErrors.TimestampOlderThanLastUpdate(
                updateTimestamp,
                status.priceLastUpdatedAt
            );
        }

        // If updateOnHeartbeat is enabled and the heartbeat interval has passed, trigger update
        if (params.updateCriteria.updateOnHeartbeat) {
            uint256 lastUpdateTime = status.priceLastUpdatedAt;

            if (
                lastUpdateTime == 0 ||
                updateTimestamp >=
                lastUpdateTime + params.updateCriteria.heartbeatSeconds
            ) {
                return updateTimestamp;
            }
        }

        // If updateOnDeviation is enabled, check if any price has deviated enough
        if (params.updateCriteria.updateOnDeviation) {
            for (uint8 i = 0; i < priceFeeds.length; i++) {
                // Get the previous price feed for this price ID using subscriptionId
                PythStructs.PriceFeed storage previousFeed = _state
                    .priceUpdates[subscriptionId][priceFeeds[i].id];

                // If there's no previous price, this is the first update
                if (previousFeed.id == bytes32(0)) {
                    return updateTimestamp;
                }

                // Calculate the deviation percentage
                int64 currentPrice = priceFeeds[i].price.price;
                int64 previousPrice = previousFeed.price.price;

                // Skip if either price is zero to avoid division by zero
                if (previousPrice == 0 || currentPrice == 0) {
                    continue;
                }

                // Calculate absolute deviation basis points (scaled by 1e4)
                uint256 numerator = SignedMath.abs(
                    currentPrice - previousPrice
                );
                uint256 denominator = SignedMath.abs(previousPrice);
                uint256 deviationBps = Math.mulDiv(
                    numerator,
                    10_000,
                    denominator
                );

                // If deviation exceeds threshold, trigger update
                if (
                    deviationBps >= params.updateCriteria.deviationThresholdBps
                ) {
                    return updateTimestamp;
                }
            }
        }

        revert SchedulerErrors.UpdateConditionsNotMet();
    }
