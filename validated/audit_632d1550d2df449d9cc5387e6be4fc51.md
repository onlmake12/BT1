### Title
`executeCallback()` and `requestPriceUpdatesWithCallback()` Accept Excess ETH That Is Permanently Transferred to the Provider Instead of Being Refunded — (`target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary
`Echo.sol` contains two `payable` functions — `requestPriceUpdatesWithCallback()` and `executeCallback()` — that silently absorb any ETH sent beyond what is strictly required. In `requestPriceUpdatesWithCallback()`, the fee check only enforces a minimum (`msg.value < requiredFee`), so any excess is permanently credited to the provider's accrued balance rather than refunded to the caller. In `executeCallback()`, the function is marked `payable` despite having no legitimate reason to accept ETH from the caller; any ETH sent is again credited to the provider, not returned to the sender.

### Finding Description

**Root cause 1 — `requestPriceUpdatesWithCallback`:**

The function enforces only a lower bound on `msg.value`:

```solidity
uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
if (msg.value < requiredFee) revert InsufficientFee();
``` [1](#0-0) 

Immediately after, the entire `msg.value - pythFeeInWei` is stored as the provider's fee:

```solidity
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);
``` [2](#0-1) 

When `executeCallback` later runs, `req.fee` is credited in full to the provider:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
``` [3](#0-2) 

Any ETH above `requiredFee` sent by the user flows into `req.fee` and is ultimately paid out to the provider. There is no refund path.

**Root cause 2 — `executeCallback`:**

The function is declared `payable` even though the Pyth oracle fee it must pay (`pythFee`) is already funded by `req.fee` (collected at request time). The developer's own TODO comment acknowledges this:

```solidity
// TODO: does this need to be payable? Any cost paid to Pyth could be taken out of the provider's accrued fees.
function executeCallback(...) external payable override {
``` [4](#0-3) 

Any ETH a caller sends to `executeCallback` is added to `providerToCredit`'s accrued fees — not returned to the sender.

### Impact Explanation
Any user who overpays `requestPriceUpdatesWithCallback` (e.g., by using a stale fee estimate, or by sending a round-number ETH amount) permanently loses the excess to the provider. Any caller who sends ETH to `executeCallback` (e.g., by mistake or via a wrapper contract that forwards `msg.value`) permanently loses that ETH to the provider. In neither case is the ETH locked in the contract — it is transferred to the provider's withdrawable balance — but the original sender has no recourse to recover it.

### Likelihood Explanation
Fee overpayment is a common user error, especially when fee rates change between the time a user queries `getFee()` and the time the transaction is mined. Wrapper contracts or integrations that forward `msg.value` to `executeCallback` (which is `external payable`) are a realistic trigger for the second issue. The developer's own TODO comment on `executeCallback` confirms awareness that the `payable` modifier is unnecessary.

### Recommendation
1. In `requestPriceUpdatesWithCallback`, add an exact-fee check or refund the excess:
   ```solidity
   require(msg.value == requiredFee, "Incorrect fee");
   // or: if (msg.value > requiredFee) { payable(msg.sender).transfer(msg.value - requiredFee); }
   ```
2. Remove the `payable` modifier from `executeCallback` entirely, since the Pyth oracle fee is already covered by `req.fee`. If `payable` must be kept for forward-compatibility, add `require(msg.value == 0, "No ETH expected")`.

### Proof of Concept

**Scenario A — excess ETH in `requestPriceUpdatesWithCallback`:**

```solidity
uint96 correctFee = echo.getFee(provider, gasLimit, priceIds);
// User sends 2x the required fee by mistake
echo.requestPriceUpdatesWithCallback{value: correctFee * 2}(
    provider, publishTime, priceIds, gasLimit
);
// req.fee = (correctFee * 2) - pythFeeInWei
// When executeCallback runs, provider receives correctFee + excess
// User's excess ETH is gone with no refund
```

**Scenario B — ETH sent to `executeCallback`:**

```solidity
// Anyone can call executeCallback with ETH
echo.executeCallback{value: 1 ether}(
    providerToCredit, sequenceNumber, updateData, priceIds
);
// The 1 ether is added to providerToCredit.accruedFeesInWei
// Caller loses 1 ether permanently
``` [5](#0-4) [6](#0-5)

    function requestPriceUpdatesWithCallback(
        address provider,
        uint64 publishTime,
        bytes32[] calldata priceIds,
        uint32 callbackGasLimit
    ) external payable override returns (uint64 requestSequenceNumber) {
        require(
            _state.providers[provider].isRegistered,
            "Provider not registered"
        );

        // FIXME: this comment is wrong. (we're not using tx.gasprice)
        // NOTE: The 60-second future limit on publishTime prevents a DoS vector where
        //      attackers could submit many low-fee requests for far-future updates when gas prices
        //      are low, forcing executors to fulfill them later when gas prices might be much higher.
        //      Since tx.gasprice is used to calculate fees, allowing far-future requests would make
        //      the fee estimation unreliable.
        require(publishTime <= block.timestamp + 60, "Too far in future");
        if (priceIds.length > MAX_PRICE_IDS) {
            revert TooManyPriceIds(priceIds.length, MAX_PRICE_IDS);
        }
        requestSequenceNumber = _state.currentSequenceNumber++;

        uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
        if (msg.value < requiredFee) revert InsufficientFee();

        Request storage req = allocRequest(requestSequenceNumber);
        req.sequenceNumber = requestSequenceNumber;
        req.publishTime = publishTime;
        req.callbackGasLimit = callbackGasLimit;
        req.requester = msg.sender;
        req.provider = provider;
        req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);

        // Create array with the right size
        req.priceIdPrefixes = new bytes8[](priceIds.length);

        // Copy only the first 8 bytes of each price ID to storage
        for (uint8 i = 0; i < priceIds.length; i++) {
            // Extract first 8 bytes of the price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }
            req.priceIdPrefixes[i] = prefix;
        }
        _state.accruedFeesInWei += _state.pythFeeInWei;

        emit PriceUpdateRequested(req, priceIds);
    }

        address providerToCredit,
        uint64 sequenceNumber,
        bytes[] calldata updateData,
        bytes32[] calldata priceIds
    ) external payable override {
        Request storage req = findActiveRequest(sequenceNumber);

        // Check provider exclusivity using configurable period
        if (
            block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds
        ) {
            require(
                providerToCredit == req.provider,
                "Only assigned provider during exclusivity period"
            );
        }

        // Verify priceIds match
        require(
            priceIds.length == req.priceIdPrefixes.length,
            "Price IDs length mismatch"
        );
        for (uint8 i = 0; i < req.priceIdPrefixes.length; i++) {
            // Extract first 8 bytes of the provided price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }

            // Compare with stored prefix
            if (prefix != req.priceIdPrefixes[i]) {
                // Now we can directly use the bytes8 prefix in the error
                revert InvalidPriceIds(priceIds[i], req.priceIdPrefixes[i]);
            }
        }

        // TODO: should this use parsePriceFeedUpdatesUnique? also, do we need to add 1 to maxPublishTime?
        IPyth pyth = IPyth(_state.pyth);
        uint256 pythFee = pyth.getUpdateFee(updateData);
        PythStructs.PriceFeed[] memory priceFeeds = pyth.parsePriceFeedUpdates{
            value: pythFee
        }(

            priceIds,
            SafeCast.toUint64(req.publishTime),
            SafeCast.toUint64(req.publishTime)
        );

        // If executeCallback can revert, then funds can be permanently locked in the contract.
