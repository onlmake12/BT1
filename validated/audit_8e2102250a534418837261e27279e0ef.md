### Title
Unbounded `complete_headers` Work via `difficulty_boundary=0` Bypasses `GET_LAST_STATE_PROOF_LIMIT` — (`util/light-client-protocol-server/src/components/get_last_state_proof.rs`)

### Summary

An unprivileged remote peer can send a `GetLastStateProof` message with `difficulty_boundary=U256::zero()` and `difficulties=[]` to force the server to build `last_n_numbers` spanning the **entire chain** (0..N blocks), causing `complete_headers` to perform N MMR root computations and N block lookups. The `GET_LAST_STATE_PROOF_LIMIT=1000` guard does not bound this work at all.

### Finding Description

**Guard check (lines 201–205):**

```rust
if self.message.difficulties().len() + (last_n_blocks as usize) * 2
    > constant::GET_LAST_STATE_PROOF_LIMIT
{
    return StatusCode::MalformedProtocolMessage.with_context("too many samples");
}
```

With `difficulties=[]` and `last_n_blocks=10`, this evaluates to `0 + 20 = 20 > 1000` → **false**. The guard passes. [1](#0-0) 

**Else-branch entered (line 291):**

`last_block_number - start_block_number <= last_n_blocks` → `N <= 10` → false for any chain longer than 10 blocks. The else branch is taken. [2](#0-1) 

**`get_first_block_total_difficulty_is_not_less_than` with `min_total_difficulty=0` (lines 30–32):**

Since every block's `total_difficulty >= U256::zero()`, the very first check `start_total_difficulty >= *min_total_difficulty` is always true, and the function immediately returns `Some((start_block_number, start_total_difficulty))`. With `start_block_number=0`, this sets `difficulty_boundary_block_number = 0`. [3](#0-2) 

**Adjustment at lines 313–316 is skipped:**

`last_block_number - difficulty_boundary_block_number < last_n_blocks` → `N - 0 < 10` → false for N > 10. The adjustment that would cap `difficulty_boundary_block_number` to `last_block_number - last_n_blocks` is **not applied**. [4](#0-3) 

**`last_n_numbers` spans the entire chain (lines 318–319):**

```rust
let last_n_numbers = (difficulty_boundary_block_number..last_block_number).collect::<Vec<_>>();
// = (0..N) → N entries
``` [5](#0-4) 

**`difficulty_boundary_block_number > 0` is false (line 321), so no sampled_numbers trimming occurs:** [6](#0-5) 

**`complete_headers` iterates all N entries (lines 132–177):**

For each of the N block numbers, it calls `self.snapshot.chain_root_mmr(*number - 1).get_root()` (an MMR computation) plus `get_ancestor` and `get_block` (disk I/O). This is O(N) work per request. [7](#0-6) 

### Impact Explanation

A single peer can exhaust server CPU and disk I/O proportional to the full chain length with a single small message. On mainnet (millions of blocks), this is a severe amplification: the attacker sends ~100 bytes; the server performs millions of MMR root computations and block lookups. Multiple concurrent requests from different peer connections multiply the effect. The `GET_LAST_STATE_PROOF_LIMIT=1000` invariant is completely bypassed. [8](#0-7) 

### Likelihood Explanation

The exploit requires no privileges, no PoW, no keys, and no special state. Any peer that can send a `GetLastStateProof` message (the standard light-client protocol) can trigger it. The crafted message is trivially constructable: valid `last_hash` (the current tip, publicly known), `start_number=0`, `start_hash=genesis_hash` (publicly known), `last_n_blocks=10`, `difficulty_boundary=U256::zero()`, `difficulties=[]`.

### Recommendation

The guard must bound the **actual output size** of `last_n_numbers`, not just `difficulties.len() + last_n_blocks * 2`. After computing `difficulty_boundary_block_number`, assert:

```rust
let last_n_count = last_block_number - difficulty_boundary_block_number;
if last_n_count > constant::GET_LAST_STATE_PROOF_LIMIT as u64 {
    return StatusCode::MalformedProtocolMessage.with_context("too many last_n blocks");
}
```

Alternatively, enforce that `difficulty_boundary` must be strictly greater than the total difficulty of `start_block_number`, preventing the degenerate `difficulty_boundary=0` case from collapsing `difficulty_boundary_block_number` to `start_block_number`.

### Proof of Concept

1. Construct a chain of N = 50,000 blocks.
2. Send `GetLastStateProof { last_hash: tip_hash, start_number: 0, start_hash: genesis_hash, last_n_blocks: 10, difficulty_boundary: U256::zero(), difficulties: [] }`.
3. Observe that `last_n_numbers = (0..50000)` (50,000 entries).
4. Observe that `complete_headers` is called with 50,000 entries, performing 50,000 MMR root computations.
5. Repeat with N = 100,000 and confirm linear scaling — demonstrating that `GET_LAST_STATE_PROOF_LIMIT=1000` provides zero protection.
