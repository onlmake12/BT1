### Title
Accrued OIS Staking Rewards Permanently Lost When Publisher Removed From Pool — (`governance/pyth_staking_sdk/src/utils/pool.ts`)

---

### Summary

The Oracle Integrity Staking (OIS) SDK's `extractPublisherData` function reads publisher reward data using **post-filter array indices** against data arrays that are indexed by **original (pre-filter) positions**. When a publisher is removed from `poolData.publishers` (set to `PublicKey.default`), two compounding issues arise: (1) all publishers positioned after the removed slot have their reward history, delegation fees, and delegation state read from the wrong index; and (2) the removed publisher is silently excluded from `advance_delegation_record` instruction generation, making any accrued-but-unclaimed rewards for that publisher permanently unclaimable.

---

### Finding Description

**Root cause — index mismatch in `extractPublisherData`:** [1](#0-0) 

```typescript
return poolData.publishers
  .filter((publisher) => !publisher.equals(PublicKey.default))  // removes slots
  .map((publisher, index) => ({                                  // index is POST-filter
    apyHistory: poolData.events.map((event) => ({
      apy: convertEpochYieldToApy(
        (event.y * (event.eventData[index]?.otherRewardRatio ?? 0n)) / ...
      ),                                                         // index is PRE-filter in storage
    })),
    delegationFee: poolData.delegationFees[index] ?? 0n,        // same mismatch
    selfDelegation: poolData.selfDelState[index]?.totalDelegation ?? 0n,
    totalDelegation: (poolData.delState[index]?.totalDelegation ?? 0n) + ...
  }));
```

`poolData.publishers`, `poolData.events[*].eventData`, `poolData.delegationFees`, `poolData.delState`, and `poolData.selfDelState` are all parallel arrays indexed by the publisher's **original slot position**. After `.filter()` removes `PublicKey.default` entries, the `.map()` callback receives a **compressed index** that no longer matches the original slot. Every publisher whose original slot comes after a removed publisher reads reward data from the wrong slot.

**Concrete example:**

| Original slot | 0 (Publisher A) | 1 (Publisher B — removed) | 2 (Publisher C) |
|---|---|---|---|
| `delegationFees` | `feeA` | `feeB` (zeroed) | `feeC` |
| Post-filter index | 0 | — | **1** (wrong, should be 2) |
| Fee read for C | `feeA` ✓ | — | `feeB` (zeroed) ✗ |

**Root cause — removed publisher excluded from reward claiming:** [2](#0-1) 

`getAdvanceDelegationRecordInstructions` calls `extractPublisherData(poolData)` to enumerate publishers, then generates one `advance_delegation_record` on-chain instruction per publisher. Because `extractPublisherData` filters out `PublicKey.default` entries, a removed publisher is never included in this list. No instruction is ever generated for them. [3](#0-2) 

The on-chain program enforces `PublisherNotFound` (error code `6000`) when the supplied publisher account is not present in `pool_data`: [4](#0-3) 

This means a delegator cannot manually construct and submit an `advance_delegation_record` instruction for a removed publisher either — the on-chain program rejects it. Any rewards that accrued in `pool_data.events` for that publisher's slot, but were not yet claimed by the delegator, are permanently locked in the pool reward custody.

---

### Impact Explanation

- **Incorrect reward accounting for all publishers after a removed slot**: Delegators see wrong APY history, wrong delegation fees, and wrong pool utilization for every publisher whose original slot index is greater than the removed publisher's slot. This corrupts the reward display and any off-chain reward simulation (`getClaimableRewards`).

- **Permanent loss of accrued rewards**: Any delegator who had an active position with a publisher at the time of removal, and had not yet called `advance_delegation_record` for that publisher, permanently loses all rewards that accrued since their last claim. The PYTH tokens remain locked in `poolRewardCustody` with no recovery path. [5](#0-4) 

---

### Likelihood Explanation

Publisher removal is a normal governance operation — publishers leave the Pyth network, are replaced, or are removed after slashing events. The OIS documentation explicitly describes publisher removal as part of the slashing process. The 7-day epoch cycle means there is always a window between when rewards accrue and when a delegator claims them. Any delegator who does not claim within the same epoch as a publisher removal loses their rewards for that publisher permanently.

---

### Recommendation

1. **Fix the index mismatch**: Replace the post-filter index with the original slot index:

```typescript
return poolData.publishers
  .map((publisher, originalIndex) => ({ publisher, originalIndex }))
  .filter(({ publisher }) => !publisher.equals(PublicKey.default))
  .map(({ publisher, originalIndex }) => ({
    apyHistory: poolData.events.map((event) => ({
      apy: convertEpochYieldToApy(
        (event.y * (event.eventData[originalIndex]?.otherRewardRatio ?? 0n)) / FRACTION_PRECISION_N,
      ) * ...,
    })),
    delegationFee: poolData.delegationFees[originalIndex] ?? 0n,
    selfDelegation: poolData.selfDelState[originalIndex]?.totalDelegation ?? 0n,
    ...
  }));
```

2. **Allow reward claiming for removed publishers**: The on-chain program should permit `advance_delegation_record` to be called for a publisher that has been removed from the active list, as long as a `DelegationRecord` PDA exists for that publisher and the delegator. Alternatively, store the publisher's public key inside the `DelegationRecord` so rewards can be settled independently of the publisher's presence in `pool_data`.

---

### Proof of Concept

1. Publisher A is at slot 0, Publisher B is at slot 1, Publisher C is at slot 2 in `poolData.publishers`.
2. A delegator stakes to Publisher C. Rewards accrue over epoch N.
3. Governance removes Publisher B (slot 1 set to `PublicKey.default`).
4. The delegator calls `getClaimableRewards` / `advanceDelegationRecord`:
   - `extractPublisherData` filters out slot 1, producing `[A (index 0), C (index 1)]`.
   - Publisher C's reward data is read from `event.eventData[1]` (B's old slot, now zeroed) instead of `event.eventData[2]`.
   - The displayed claimable rewards for C are `0` (or stale/wrong).
5. If the delegator had staked to Publisher B before removal:
   - `extractPublisherData` does not include B.
   - `getAdvanceDelegationRecordInstructions` generates no instruction for B.
   - Any manual call to `advance_delegation_record` with B's pubkey fails with `PublisherNotFound` (error 6000).
   - All of B's accrued rewards are permanently lost. [6](#0-5) [7](#0-6) [4](#0-3)

export const extractPublisherData = (
  poolData: PoolDataAccount,
): PublisherData => {
  return poolData.publishers
    .filter((publisher) => !publisher.equals(PublicKey.default))
    .map((publisher, index) => ({
      apyHistory: poolData.events
        .filter((event) => event.epoch > 0n)
        .map((event) => ({
          apy:
            convertEpochYieldToApy(
              (event.y * (event.eventData[index]?.otherRewardRatio ?? 0n)) /
                FRACTION_PRECISION_N,
            ) *
            computeDelegatorRewardPercentage(
              poolData.delegationFees[index] ?? 0n,
            ),
          epoch: event.epoch,
          selfApy: convertEpochYieldToApy(
            (event.y * (event.eventData[index]?.selfRewardRatio ?? 0n)) /
              FRACTION_PRECISION_N,
          ),
        }))
        .sort((a, b) => Number(a.epoch) - Number(b.epoch)),
      delegationFee: poolData.delegationFees[index] ?? 0n,
      pubkey: publisher,
      selfDelegation: poolData.selfDelState[index]?.totalDelegation ?? 0n,
      selfDelegationDelta: poolData.selfDelState[index]?.deltaDelegation ?? 0n,

  public async getClaimableRewards(
    stakeAccountPositions: PublicKey,
    simulationPayer?: PublicKey,
  ) {
    const instructions = await this.getAdvanceDelegationRecordInstructions(
      stakeAccountPositions,
      simulationPayer,
    );

    let totalRewards = 0n;

    for (const instruction of instructions.advanceDelegationRecordInstructions) {
      const tx = new Transaction().add(instruction);
      tx.feePayer = simulationPayer ?? this.wallet.publicKey;
      // eslint-disable-next-line @typescript-eslint/no-deprecated
      const res = await this.connection.simulateTransaction(tx);
      const val = res.value.returnData?.data[0];
      if (val === undefined) {
        continue;
      }
      const buffer = Buffer.from(val, "base64").reverse();
      totalRewards += BigInt("0x" + buffer.toString("hex"));
    }

    return {
      expiry:
        instructions.lowestEpoch === undefined
          ? undefined
          : epochToDate(instructions.lowestEpoch + 53n),
      totalRewards,
    };
  }
