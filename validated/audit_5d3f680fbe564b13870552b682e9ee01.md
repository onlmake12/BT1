### Title
Cross-Network Alert Replay: `RawAlert` Hash Excludes Network Identifier, Enabling Replay Across CKB Networks — (File: `util/network-alert/src/verifier.rs`)

### Summary

The `RawAlert` molecule schema and its `calc_alert_hash()` implementation hash only the alert payload fields (`notice_until`, `id`, `cancel`, `priority`, `message`, `min_version`, `max_version`). No network identifier (genesis hash, chain spec name) is included in the signed digest. Because the Nervos Foundation uses the same secp256k1 key set across all CKB networks (mainnet, testnet, devnet), a validly signed alert from one network is cryptographically valid on every other CKB network. Any unprivileged attacker who observes a signed alert on one network can replay it verbatim on another via the `send_alert` RPC or the P2P Alert protocol, and every receiving node will accept and re-broadcast it.

### Finding Description

**Schema — no network field:**

The `RawAlert` molecule table contains only:

```
table RawAlert {
    notice_until:   Uint64,
    id:             Uint32,
    cancel:         Uint32,
    priority:       Uint32,
    message:        Bytes,
    min_version:    BytesOpt,
    max_version:    BytesOpt,
}
``` [1](#0-0) 

There is no `genesis_hash`, `chain_spec_name`, or any other network-scoping field.

**Hash computation — only raw alert bytes:**

`calc_alert_hash()` delegates to `calc_hash()`, which is a plain `blake2b_256` over `self.as_slice()` — the serialized bytes of `RawAlert` with no domain separator or network tag. [2](#0-1) 

**Signature verification — no network check:**

`Verifier::verify_signatures` constructs the message directly from `alert.calc_alert_hash()` and checks it against the hardcoded Foundation public keys. There is no step that binds the alert to a specific network. [3](#0-2) 

**Same key set across all networks:**

The Foundation public keys are loaded from `NetworkAlertConfig`, which is hardcoded in the binary and identical for mainnet and testnet deployments. There is no per-network key rotation. [4](#0-3) 

**Contrast with the identify protocol**, which correctly includes the genesis hash in the network name:

```rust
pub fn identify_name(&self) -> String {
    let genesis_hash = format!("{:x}", Into::<H256>::into(&self.genesis_hash));
    format!("/{}/{}", self.id, &genesis_hash[..8])
}
``` [5](#0-4) 

The alert signing path has no equivalent binding.

**Attacker-controlled entry path:**

The `send_alert` RPC endpoint accepts any `Alert` value from any RPC caller and, after signature verification, broadcasts it to all connected peers via the P2P Alert protocol. [6](#0-5) 

The P2P `AlertRelayer::received` handler also accepts and re-broadcasts any alert that passes `verify_signatures`, with no network check. [7](#0-6) 

**Exploit flow:**

1. Attacker monitors the testnet P2P network or calls `get_blockchain_info` / observes a testnet `send_alert` RPC call to obtain a fully signed `Alert` (raw bytes + signatures).
2. Attacker submits the identical `Alert` to a mainnet node via `send_alert` RPC (no authentication required beyond having RPC access, which is the normal operator-facing interface).
3. The mainnet `Verifier` computes `blake2b(raw_alert_bytes)`, recovers the Foundation public keys from the signatures, and finds a valid 2-of-4 match — because the signed digest is identical on both networks.
4. The alert is accepted, stored in the notifier, and broadcast to all mainnet peers, which repeat the same verification and re-broadcast.
5. The false alert propagates across the entire mainnet.

### Impact Explanation

A successfully replayed alert is indistinguishable from a legitimately issued mainnet alert. Node operators who trust the alert system will act on it: they may downgrade or upgrade their node software based on `min_version`/`max_version` fields that were intended for a different network, halt operations, or take other disruptive actions. Because alerts are signed by the Nervos Foundation and carry institutional authority, operators have strong reason to comply. A testnet alert about a critical bug in version range X replayed on mainnet could cause mass unnecessary node upgrades or downgrades, degrading network stability. The impact is integrity failure of the alert trust model and potential operational disruption across the mainnet.

### Likelihood Explanation

The preconditions are minimal:
- A signed alert must exist on any CKB network (testnet, devnet). The Foundation has issued real alerts historically (e.g., alert ID 20230001 is hardcoded in tests).
- The attacker needs only RPC access to a mainnet node with the Alert RPC module enabled, or a P2P connection to inject the alert via the relay protocol.
- No private keys, no privileged access, no cryptographic break required.

The attack is fully passive until the replay step: observe, copy, submit.

### Recommendation

Include a network-scoping field in `RawAlert` — specifically the genesis block hash or the chain spec name — so that the signed digest is unique per network. This mirrors the approach already used in the identify protocol (`identify_name` embeds the genesis hash). Alternatively, prepend a domain separation tag containing the genesis hash to the data hashed by `calc_alert_hash()`, without changing the wire format, by modifying `RawAlertReader::calc_alert_hash` to hash `genesis_hash || raw_alert_bytes`.

### Proof of Concept

```

# (observe any broadcast alert via P2P sniffing or testnet RPC)

# Step 2: Replay the identical alert on mainnet
curl -X POST mainnet-node:8114 -H 'Content-Type: application/json' -d '{
  "jsonrpc":"2.0","method":"send_alert","id":1,
  "params":[{
    "id":"0x4E2F01",
    "cancel":"0x0",
    "priority":"0x14",
    "message":"CKB v0.105.* have bugs. Please upgrade.",
    "notice_until":"0x...",
    "signatures":["<testnet_sig_1>","<testnet_sig_2>"]
  }]
}'

# Expected result: mainnet node accepts the alert (signatures verify against
# the same Foundation pubkeys), stores it, and broadcasts it to all mainnet peers.
# Verifier::verify_signatures passes because calc_alert_hash produces the same
# digest on both networks — no network identifier is included.
```

The signed digest is `blake2b_256(raw_alert_bytes)` where `raw_alert_bytes` contains no network field, making it identical on mainnet and testnet for the same alert content. [8](#0-7) [9](#0-8)

    pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
        trace!("Verifying alert {:?}", alert);
        let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
        let signatures: Vec<Signature> = alert
            .signatures()
            .into_iter()
            .filter_map(
                |sig_data| match Signature::from_slice(sig_data.as_reader().raw_data()) {
                    Ok(sig) => {
                        if sig.is_valid() {
                            Some(sig)
                        } else {
                            debug!("invalid signature: {:?}", sig);
                            None
                        }
                    }
                    Err(err) => {
                        debug!("signature error: {}", err);
                        None
                    }
                },
            )
            .collect();
        verify_m_of_n(
            &message,
            self.config.signatures_threshold,
            &signatures,
            &self.pubkeys,
        )
        .map_err(|err| err.kind())?;
        Ok(())
    }

    fn send_alert(&self, alert: Alert) -> Result<()> {
        let alert: packed::Alert = alert.into();
        let now_ms = ckb_systemtime::unix_time_as_millis();
        let notice_until: u64 = alert.raw().notice_until().into();
        if notice_until < now_ms {
            return Err(RPCError::invalid_params(format!(
                "Expected `params[0].notice_until` in the future (> {now_ms}), got {notice_until}",
            )));
        }

        let result = self.verifier.verify_signatures(&alert);

        match result {
            Ok(()) => {
                // set self node notifier
                self.notifier.lock().add(&alert);

                self.network_controller.broadcast_with_handle(
                    SupportProtocols::Alert.protocol_id(),
                    alert.as_bytes(),
                    &self.handle,
                );
                Ok(())
            }
            Err(e) => Err(RPCError::custom_with_error(
                RPCError::AlertFailedToVerifySignatures,
                e,
            )),
        }
    }

    async fn received(
        &mut self,
        nc: Arc<dyn CKBProtocolContext + Sync>,
        peer_index: PeerIndex,
        data: Bytes,
    ) {
        let alert: packed::Alert = match packed::AlertReader::from_slice(&data) {
            Ok(alert) => {
                if alert.raw().message().is_utf8()
                    && alert
                        .raw()
                        .min_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                    && alert
                        .raw()
                        .max_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                {
                    alert.to_entity()
                } else {
                    info!(
                        "A malformed message fromP peer {} : not utf-8 string",
                        peer_index
                    );
                    nc.ban_peer(
                        peer_index,
                        BAD_MESSAGE_BAN_TIME,
                        String::from("send us a malformed message: not utf-8 string"),
                    );
                    return;
                }
            }
            Err(err) => {
                info!("A malformed message from peer {}: {:?}", peer_index, err);
                nc.ban_peer(
                    peer_index,
                    BAD_MESSAGE_BAN_TIME,
                    String::from("send us a malformed message"),
                );
                return;
            }
        };
        let alert_id = alert.as_reader().raw().id().into();
        trace!("ReceiveD alert {} from peer {}", alert_id, peer_index);
        // ignore alert
        if self.notifier.lock().has_received(alert_id) {
            return;
        }
        // verify
        if let Err(err) = self.verifier.verify_signatures(&alert) {
            debug!(
                "An alert from peer {} with invalid signatures, error {:?}",
                peer_index, err
            );
            nc.ban_peer(
                peer_index,
                BAD_MESSAGE_BAN_TIME,
                String::from("send us an alert with invalid signatures"),
            );
            return;
        }
        // mark sender as known
        self.mark_as_known(peer_index, alert_id);
        // broadcast message
        let selected_peers: Vec<PeerIndex> = nc
            .connected_peers()
            .into_iter()
            .filter(|peer| self.mark_as_known(*peer, alert_id))
            .collect();
        if let Err(err) = nc.quick_filter_broadcast(
            TargetSession::Multi(Box::new(selected_peers.into_iter())),
            data,
        ) {
            debug!("alert broadcast error: {:?}", err);
        }
        // add to received alerts
        self.notifier.lock().add(&alert);
    }
