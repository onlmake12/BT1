### Title
Unauthenticated Notify Endpoint Allows Arbitrary BlockTemplate Injection, Causing Miner Hashrate Diversion — (`miner/src/client.rs`)

### Summary

The miner's HTTP notify listener (`handle` function) accepts any inbound `BlockTemplate` POST without authentication or source validation. An attacker who can reach the listen socket can inject a crafted `BlockTemplate` with an arbitrary `compact_target`, causing all worker threads to switch to mining against the attacker-controlled template and wasting the operator's hashrate.

### Finding Description

When the miner is configured in notify mode (`config.listen = Some(addr)`), `spawn_background` binds a TCP listener and serves every incoming connection through the `handle` function: [1](#0-0) 

The `handle` function collects the full request body, deserializes it as a `BlockTemplate` via `serde_json::from_reader`, and immediately calls `update_block_template` — with **no check on the source IP, no shared secret, no token, and no HTTP method guard**: [2](#0-1) 

`update_block_template` accepts the template as long as its `work_id` differs from the currently stored one (trivially satisfied by the attacker by incrementing the field): [3](#0-2) 

The accepted template is converted to a `Work` and dispatched to all worker threads. In `notify_new_work`, `compact_to_target` is called directly on the attacker-supplied `compact_target` field with no bounds or consensus check: [4](#0-3) 

Workers then mine against the attacker-controlled target until the next legitimate template arrives or another crafted POST is sent.

### Impact Explanation

The miner's current work state — stored in `current_work_id` and propagated to all `WorkerController` channels — is fully overwritten by the injected template. All worker threads switch to mining the crafted block. Because the crafted `compact_target` can be set to minimum difficulty (e.g., `0x207fffff`), workers find "solutions" almost instantly, but the resulting blocks are rejected by the CKB node's consensus validation. The operator loses real hashrate and mining revenue for the duration of the attack. A persistent attacker can keep sending new crafted POSTs (with incrementing `work_id`) to prevent the miner from ever returning to legitimate work.

### Likelihood Explanation

The listen address is operator-configured. The default template comments it out as `127.0.0.1:8888`: [5](#0-4) 

However, the `ClientConfig.listen` field accepts any `SocketAddr`: [6](#0-5) 

Operators who bind to `0.0.0.0` (e.g., when the miner and node run on separate hosts, a common production setup) expose the endpoint to any network-reachable attacker. Even with `127.0.0.1`, any co-located process (container escape, shared hosting, compromised dependency) can exploit this. There is no mechanism in the code for an operator to add authentication even if they wanted to.

### Recommendation

Add source-IP allowlisting (accept connections only from the configured CKB node's address) and/or a shared-secret token checked in `handle` before calling `update_block_template`. At minimum, validate that the incoming `BlockTemplate`'s `compact_target` is within the expected consensus range before dispatching it to workers.

### Proof of Concept

```bash
# Craft a minimal BlockTemplate JSON with compact_target = 0x207fffff (minimum difficulty)
# and a work_id different from the current one (e.g., 9999)
curl -X POST http://<miner-listen-addr>:<port>/ \
  -H "Content-Type: application/json" \
  -d '{
    "version": "0x0",
    "compact_target": "0x207fffff",
    "current_time": "0x...",
    "number": "0x1",
    "epoch": "0x...",
    "parent_hash": "0x0000...0000",
    "cycles_limit": "0x...",
    "bytes_limit": "0x...",
    "uncles_count_limit": "0x2",
    "uncles": [],
    "transactions": [],
    "proposals": [],
    "cellbase": { ... },
    "work_id": "0x9999",
    "dao": "0x..."
  }'
```

The `handle` function deserializes this as a valid `BlockTemplate`, `update_block_template` accepts it (work_id `0x9999` ≠ current), and all workers immediately switch to mining against `compact_target = 0x207fffff`. The miner's legitimate work state is overwritten. [1](#0-0) [7](#0-6)

        let work_id = block_template.work_id.into();
        let updated = |id| {

                Some(work_id)
