### Title
Silent u128→u64 Truncation in NervosDAO Withdrawal Capacity Calculation — (`File: util/dao/src/lib.rs`)

---

### Summary

`DaoCalculator::calculate_maximum_withdraw` computes the maximum CKB a depositor can withdraw from NervosDAO using a u128 intermediate value, then casts it to u64 with a bare `as u64` (truncating cast) instead of a checked `u64::try_from(...)`. When the product overflows u64, the result is silently truncated to a wrong, much smaller value rather than returning an error. Every other analogous u128→u64 narrowing in the same file uses the checked form.

---

### Finding Description

In `util/dao/src/lib.rs`, `calculate_maximum_withdraw` computes:

```rust
let withdraw_counted_capacity = u128::from(counted_capacity.as_u64())
    * u128::from(withdrawing_ar)
    / u128::from(deposit_ar);
let withdraw_capacity =
    Capacity::shannons(withdraw_counted_capacity as u64)   // ← truncating cast
        .safe_add(occupied_capacity)?;
``` [1](#0-0) 

The `as u64` cast silently discards the upper 64 bits when `withdraw_counted_capacity > u64::MAX`. The correct pattern, used consistently everywhere else in the same file, is:

```rust
u64::try_from(reward128).map_err(|_| DaoError::Overflow)?
``` [2](#0-1) [3](#0-2) [4](#0-3) 

The existing test `check_withdraw_calculation_overflows` explicitly constructs an overflow scenario and asserts `result.is_err()`, but with the current `as u64` cast the function returns `Ok(truncated_wrong_value)` instead, meaning the test assertion fails. [5](#0-4) 

`calculate_maximum_withdraw` is called from two production paths:

1. `transaction_maximum_withdraw` → `transaction_fee`, used by the tx-pool to compute fees for DAO withdrawal transactions.
2. The `calculate_dao_maximum_withdraw` JSON-RPC endpoint, directly reachable by any RPC caller. [6](#0-5) 

---

### Impact Explanation

When `counted_capacity * withdrawing_ar / deposit_ar` exceeds `u64::MAX`, the truncated result is a value modulo 2^64 — potentially orders of magnitude smaller than the correct withdrawal amount. Two concrete effects:

- **RPC callers** invoking `calculate_dao_maximum_withdraw` receive a silently wrong (far too small) capacity figure. If they use this to construct a phase-2 withdrawal transaction, the on-chain DAO type script will reject it (output capacity does not match the correct maximum withdraw), causing the transaction to be permanently invalid.
- **Tx-pool fee accounting**: `transaction_fee` subtracts `outputs_capacity` from the (wrong) `maximum_withdraw`. If the truncated value is less than `outputs_capacity`, `safe_sub` returns an error and the withdrawal transaction is rejected from the pool even though it is valid on-chain.

---

### Likelihood Explanation

The overflow requires `counted_capacity * withdrawing_ar > deposit_ar * u64::MAX`. With the total CKB supply ≈ 3.36 × 10¹⁸ shannons and AR starting at 10¹⁶, the AR would need to grow by roughly 5.5× relative to the deposit AR for a full-supply deposit to overflow. That is a long-horizon condition. However, the test fixture already demonstrates the overflow is reachable with a capacity near `u64::MAX` and a modest AR delta, and the existing test is broken by the bug today. Any RPC caller can trigger the wrong return value by referencing a sufficiently large live DAO cell.

---

### Recommendation

Replace the truncating cast on line 156 with a checked conversion, consistent with every other u128→u64 narrowing in the file:

```rust
let withdraw_counted_capacity_u64 =
    u64::try_from(withdraw_counted_capacity).map_err(|_| DaoError::Overflow)?;
let withdraw_capacity =
    Capacity::shannons(withdraw_counted_capacity_u64).safe_add(occupied_capacity)?;
``` [7](#0-6) 

---

### Proof of Concept

The repository already contains a test that demonstrates the bug. With the current `as u64` cast, `check_withdraw_calculation_overflows` returns `Ok(truncated_value)` instead of `Err(DaoError::Overflow)`, causing the `assert!(result.is_err())` assertion to fail:

```
deposit_ar      = 10_000_000_000_123_456
withdrawing_ar  = 10_000_000_001_123_456
counted_capacity = 18_446_744_073_709_550_000  (≈ u64::MAX)

withdraw_counted_capacity (u128)
  = 18_446_744_073_709_550_000 × 10_000_000_001_123_456
    / 10_000_000_000_123_456
  ≈ 18_447_000_000_000_000_000   (> u64::MAX = 18_446_744_073_709_551_615)

as u64 truncates to ≈ 255_926_290_448_384  (wrong, ~72× too small)
``` [8](#0-7)

fn check_withdraw_calculation_overflows() {
    let output = CellOutput::new_builder()
        .capacity(Capacity::shannons(18_446_744_073_709_550_000))
        .build();
    let tx = TransactionBuilder::default().output(output.clone()).build();
    let epoch = EpochNumberWithFraction::new(1, 100, 1000);
    let deposit_header = HeaderBuilder::default()
        .number(100)
        .epoch(epoch)
        .dao(pack_dao_data(
            10_000_000_000_123_456,
            Default::default(),
            Default::default(),
            Default::default(),
        ))
        .build();
    let deposit_block = BlockBuilder::default()
        .header(deposit_header)
        .transaction(tx)
        .build();

    let epoch = EpochNumberWithFraction::new(1, 200, 1000);
    let withdrawing_header = HeaderBuilder::default()
        .number(200)
        .epoch(epoch)
        .dao(pack_dao_data(
            10_000_000_001_123_456,
            Default::default(),
            Default::default(),
            Default::default(),
        ))
        .build();
    let withdrawing_block = BlockBuilder::default().header(withdrawing_header).build();

    let tmp_dir = TempDir::new().unwrap();
    let db = RocksDB::open_in(&tmp_dir, COLUMNS);
    let store = ChainDB::new(db, Default::default());
    let txn = store.begin_transaction();
    txn.insert_block(&deposit_block).unwrap();
    txn.attach_block(&deposit_block).unwrap();
    txn.insert_block(&withdrawing_block).unwrap();
    txn.attach_block(&withdrawing_block).unwrap();
    txn.commit().unwrap();

    let consensus = Consensus::default();
    let data_loader = store.borrow_as_data_loader();
    let calculator = DaoCalculator::new(&consensus, &data_loader);
    let result = calculator.calculate_maximum_withdraw(
        &output,
        Capacity::bytes(0).expect("should not overflow"),
        &deposit_block.hash(),
        &withdrawing_block.hash(),
    );
    assert!(result.is_err());
}

    fn calculate_dao_maximum_withdraw(
        &self,
        out_point: OutPoint,
        kind: DaoWithdrawingCalculationKind,
    ) -> Result<Capacity> {
        let snapshot: &Snapshot = &self.shared.snapshot();
        let consensus = snapshot.consensus();
        let out_point: packed::OutPoint = out_point.into();
        let data_loader = snapshot.borrow_as_data_loader();
        let calculator = DaoCalculator::new(consensus, &data_loader);
        match kind {
            DaoWithdrawingCalculationKind::WithdrawingHeaderHash(withdrawing_header_hash) => {
                let (tx, deposit_header_hash) = snapshot
                    .get_transaction(&out_point.tx_hash())
                    .ok_or_else(|| RPCError::invalid_params("invalid out_point"))?;
                let output = tx
                    .outputs()
                    .get(out_point.index().into())
                    .ok_or_else(|| RPCError::invalid_params("invalid out_point"))?;
                let output_data = tx
                    .outputs_data()
                    .get(out_point.index().into())
                    .ok_or_else(|| RPCError::invalid_params("invalid out_point"))?;

                match calculator.calculate_maximum_withdraw(
                    &output,
                    core::Capacity::bytes(output_data.len()).expect("should not overflow"),
                    &deposit_header_hash,
                    &withdrawing_header_hash.into(),
                ) {
                    Ok(capacity) => Ok(capacity.into()),
                    Err(err) => Err(RPCError::custom_with_error(RPCError::DaoError, err)),
                }
            }
            DaoWithdrawingCalculationKind::WithdrawingOutPoint(withdrawing_out_point) => {
                let (_tx, deposit_header_hash) = snapshot
                    .get_transaction(&out_point.tx_hash())
                    .ok_or_else(|| RPCError::invalid_params("invalid out_point"))?;

                let withdrawing_out_point: packed::OutPoint = withdrawing_out_point.into();
                let (withdrawing_tx, withdrawing_header_hash) = snapshot
                    .get_transaction(&withdrawing_out_point.tx_hash())
                    .ok_or_else(|| RPCError::invalid_params("invalid withdrawing_out_point"))?;

                let output = withdrawing_tx
                    .outputs()
                    .get(withdrawing_out_point.index().into())
                    .ok_or_else(|| RPCError::invalid_params("invalid withdrawing_out_point"))?;
                let output_data = withdrawing_tx
                    .outputs_data()
                    .get(withdrawing_out_point.index().into())
                    .ok_or_else(|| RPCError::invalid_params("invalid withdrawing_out_point"))?;

                match calculator.calculate_maximum_withdraw(
                    &output,

                    &deposit_header_hash,
                    &withdrawing_header_hash,
                ) {
                    Ok(capacity) => Ok(capacity.into()),
                    Err(err) => Err(RPCError::custom_with_error(RPCError::DaoError, err)),
                }
            }
        }
