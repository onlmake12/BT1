### Title
Arithmetic Underflow in `ema_price_no_older_than` Timestamp Comparison Causes Unrecoverable Panic - (File: `target_chains/fuel/contracts/pyth-contract/src/main.sw`)

### Summary

The `ema_price_no_older_than` function in the Fuel Pyth contract performs an unguarded unsigned integer subtraction `current_time - price.publish_time`. If a stored EMA price carries a `publish_time` greater than the current chain timestamp (possible via a future-timestamped price update within the protocol's allowed window), the subtraction underflows in Sway, causing a runtime panic rather than the expected `PythError::OutdatedPrice` revert. The sibling function `price_no_older_than` was already patched with an explicit saturating-subtraction guard for exactly this reason, but `ema_price_no_older_than` was not updated.

### Finding Description

`ema_price_no_older_than` computes the staleness check as:

```sway
require(
    current_time - price.publish_time <= time_period,
    PythError::OutdatedPrice,
);
``` [1](#0-0) 

The adjacent `price_no_older_than` function carries an explicit comment and guard that was added to prevent exactly this underflow:

```sway
// Mimicking saturating subtraction to avoid underflow
let time_difference = if current_time > price.publish_time {
    current_time - price.publish_time
} else {
    0
};
require(time_difference <= time_period, PythError::OutdatedPrice);
``` [2](#0-1) 

The fix was applied to `price_no_older_than` but was never propagated to `ema_price_no_older_than`. In Sway, unsigned integer underflow causes a VM panic (not a graceful revert with a typed error), so callers that attempt to catch `PythError::OutdatedPrice` will instead receive an unhandled panic.

### Impact Explanation

Any on-chain consumer calling `ema_price_no_older_than` while the stored EMA price has `publish_time > current_time` receives a hard panic revert. This:

1. Breaks all downstream contracts that rely on `ema_price_no_older_than` for the affected feed until the chain clock catches up.
2. Produces an unexpected panic rather than the typed `PythError::OutdatedPrice`, breaking any caller that handles the typed error.
3. Is analogous to the original report: a stored accumulated value (`publish_time`) exceeds a freshly computed value (`current_time`), causing the subtraction to underflow and stall the function.

### Likelihood Explanation

Pyth price updates explicitly permit timestamps slightly in the future (the EVM Scheduler uses `FUTURE_TIMESTAMP_MAX_VALIDITY_PERIOD`). [3](#0-2)  An unprivileged updater who submits a valid VAA containing a price with a timestamp a few seconds ahead of the Fuel chain's clock will cause the stored EMA price to have `publish_time > current_time`. This is a realistic, low-effort trigger requiring no privileged access.

### Recommendation

Apply the same saturating-subtraction guard already present in `price_no_older_than` to `ema_price_no_older_than`:

```sway
fn ema_price_no_older_than(time_period: u64, price_feed_id: PriceFeedId) -> Price {
    let price = ema_price_unsafe(price_feed_id);
    let current_time = timestamp();
    // Mimicking saturating subtraction to avoid underflow
    let time_difference = if current_time > price.publish_time {
        current_time - price.publish_time
    } else {
        0
    };
    require(time_difference <= time_period, PythError::OutdatedPrice);
    price
}
```

### Proof of Concept

1. An unprivileged updater submits a Pyth price update VAA to the Fuel Pyth contract where the EMA price's `publish_time` is `current_fuel_time + 5` seconds (within the protocol's allowed future-timestamp window).
2. The update is accepted and stored; `storage.latest_price_feed` now holds an EMA price with `publish_time > current_time`.
3. Any contract calling `ema_price_no_older_than(60, feed_id)` immediately panics at `current_time - price.publish_time` (unsigned underflow in Sway), returning a hard revert instead of `PythError::OutdatedPrice`.
4. The panic persists for every caller until the Fuel chain's clock advances past the stored `publish_time`, during which all EMA price consumers for that feed are bricked. [1](#0-0) [2](#0-1)
