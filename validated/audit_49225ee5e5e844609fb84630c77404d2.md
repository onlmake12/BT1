### Title
`this.getPricesUnsafe()` / `this.getEmaPricesUnsafe()` Internal Calls Break `msg.sender` Context, Causing DoS for Whitelist-Enabled Subscriptions - (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`Scheduler.getPricesNoOlderThan()` and `Scheduler.getEmaPricesNoOlderThan()` delegate to `this.getPricesUnsafe()` and `this.getEmaPricesUnsafe()` via an external `this.` call. This changes `msg.sender` to the Scheduler contract's own address inside the callee. Both callee functions are guarded by the `onlyWhitelistedReader` modifier, which checks `msg.sender` against the subscription's whitelist. Because the Scheduler contract itself is never the subscription manager and is never in any reader whitelist, the modifier always reverts for whitelist-enabled subscriptions. The result is a complete, permanent DoS of the recency-checked price getters for every subscription that has `whitelistEnabled = true`.

---

### Finding Description

`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` both perform their staleness check and then forward to the unsafe variants via `this.`:

```solidity
// Scheduler.sol line 554
prices = this.getPricesUnsafe(subscriptionId, priceIds);

// Scheduler.sol line 597
prices = this.getEmaPricesUnsafe(subscriptionId, priceIds);
``` [1](#0-0) [2](#0-1) 

`this.f()` in Solidity compiles to an external `CALL` opcode. The callee therefore sees `msg.sender == address(this)` (the Scheduler contract), not the original EOA or contract that initiated the transaction.

`getPricesUnsafe` and `getEmaPricesUnsafe` are both decorated with `onlyWhitelistedReader(subscriptionId)`:

```solidity
modifier onlyWhitelistedReader(uint256 subscriptionId) {
    if (_state.subscriptionManager[subscriptionId] == msg.sender) { _; return; }
    if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) { _; return; }
    // whitelist loop checks msg.sender …
    if (!isWhitelisted) revert SchedulerErrors.Unauthorized();
    _;
}
``` [3](#0-2) 

When the Scheduler calls itself via `this.getPricesUnsafe(...)`:
1. `msg.sender` is the Scheduler contract address.
2. The Scheduler is not the subscription manager.
3. If `whitelistEnabled == true`, the whitelist loop runs against the Scheduler's own address — which is never present — and reverts with `SchedulerErrors.Unauthorized`. [4](#0-3) [5](#0-4) 

---

### Impact Explanation

For any subscription created with `whitelistEnabled = true`:

- `getPricesNoOlderThan` always reverts, even for legitimately whitelisted readers.
- `getEmaPricesNoOlderThan` always reverts, even for legitimately whitelisted readers.

These are the only functions that enforce a recency guarantee on returned prices. Whitelisted consumers that rely on them for safe, time-bounded price reads are completely blocked. They are forced to fall back to `getPricesUnsafe` / `getEmaPricesUnsafe`, which carry no staleness protection, silently degrading the security of any downstream protocol.

---

### Likelihood Explanation

**High.** The whitelist feature is an explicit, documented access-control mechanism. Any subscription manager who enables it (the expected use case for permissioned DeFi integrations) immediately triggers the bug for all whitelisted readers. No special attacker action is required; the DoS is self-inflicted by normal, correct usage of the API.

---

### Recommendation

Remove the redundant `this.` dispatch. Since `getPricesNoOlderThan` already passes the `onlyWhitelistedReader` check itself, it can call the internal helper `_getPricesInternal` directly:

```solidity
// Before (broken):
prices = this.getPricesUnsafe(subscriptionId, priceIds);

// After (fixed):
PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
prices = new PythStructs.Price[](priceFeeds.length);
for (uint i = 0; i < priceFeeds.length; i++) {
    prices[i] = priceFeeds[i].price;
}
```

Apply the same fix to `getEmaPricesNoOlderThan` (use `priceFeeds[i].emaPrice`). This preserves the staleness check while keeping `msg.sender` as the original caller throughout. [6](#0-5) 

---

### Proof of Concept

1. Deploy `SchedulerUpgradeable` and create a subscription with `whitelistEnabled = true`, adding address `Alice` to `readerWhitelist`.
2. Call `updatePriceFeeds` to populate price data.
3. As `Alice`, call `getPricesNoOlderThan(subscriptionId, priceIds, 3600)`.
4. Execution flow:
   - `onlyWhitelistedReader` in `getPricesNoOlderThan` passes (Alice is whitelisted).
   - Staleness check passes.
   - `this.getPricesUnsafe(subscriptionId, priceIds)` is dispatched as an external call; `msg.sender` inside `getPricesUnsafe` is now the Scheduler contract.
   - `onlyWhitelistedReader` in `getPricesUnsafe` runs: Scheduler ≠ manager, `whitelistEnabled == true`, Scheduler not in whitelist → `revert SchedulerErrors.Unauthorized()`.
5. Alice's transaction reverts despite being a legitimately whitelisted reader. [7](#0-6) [8](#0-7)

    function _getPricesInternal(
        uint256 subscriptionId,
        bytes32[] calldata priceIds
    ) internal view returns (PythStructs.PriceFeed[] memory priceFeeds) {
        if (!_state.subscriptionParams[subscriptionId].isActive) {
            revert SchedulerErrors.InactiveSubscription();
        }

        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        // If no price IDs provided, return all price feeds for the subscription
        if (priceIds.length == 0) {
            PythStructs.PriceFeed[]
                memory allFeeds = new PythStructs.PriceFeed[](
                    params.priceIds.length
                );
            for (uint8 i = 0; i < params.priceIds.length; i++) {
                PythStructs.PriceFeed memory priceFeed = _state.priceUpdates[
                    subscriptionId
                ][params.priceIds[i]];
                // Check if the price feed exists (price ID is valid and has been updated)
                if (priceFeed.id == bytes32(0)) {
                    revert SchedulerErrors.InvalidPriceId(
                        params.priceIds[i],
                        bytes32(0)
                    );
                }
                allFeeds[i] = priceFeed;
            }
            return allFeeds;
        }

        // Return only the requested price feeds
        PythStructs.PriceFeed[]
            memory requestedFeeds = new PythStructs.PriceFeed[](
                priceIds.length
            );
        for (uint8 i = 0; i < priceIds.length; i++) {
            PythStructs.PriceFeed memory priceFeed = _state.priceUpdates[
                subscriptionId
            ][priceIds[i]];

            // Check if the price feed exists (price ID is valid and has been updated)
            if (priceFeed.id == bytes32(0)) {
                revert SchedulerErrors.InvalidPriceId(priceIds[i], bytes32(0));
            }
            requestedFeeds[i] = priceFeed;
        }
        return requestedFeeds;
    }

        // may be slightly ahead of this chain.

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
