### Title
Silent Truncating Cast in NervosDAO Withdrawal Calculation Produces Incorrect Withdrawal Amount - (File: `util/dao/src/lib.rs`)

### Summary
`DaoCalculator::calculate_maximum_withdraw` uses a silent truncating `as u64` cast to narrow a `u128` intermediate result, whereas every analogous `u128`→`u64` narrowing in the same file uses the checked `u64::try_from(...).map_err(|_| DaoError::Overflow)?` pattern. If the intermediate value exceeds `u64::MAX`, the cast silently wraps, returning a drastically reduced (incorrect) withdrawal capacity instead of propagating an error. The existing overflow test passes only because the error is caught by a later `safe_add`, not by the cast itself, leaving the silent-truncation path untested and undetected.

### Finding Description

In `util/dao/src/lib.rs`, `calculate_maximum_withdraw` computes the maximum NervosDAO withdrawal as:

```rust
let withdraw_counted_capacity = u128::from(counted_capacity.as_u64())
    * u128::from(withdrawing_ar)
    / u128::from(deposit_ar);
let withdraw_capacity =
    Capacity::shannons(withdraw_counted_capacity as u64)   // ← silent truncation
        .safe_add(occupied_capacity)?;
``` [1](#0-0) 

The `as u64` cast is a **truncating** (wrapping) narrowing. If `withdraw_counted_capacity` exceeds `u64::MAX`, the high bits are silently discarded, producing a value that is `withdraw_counted_capacity mod 2^64` — potentially orders of magnitude smaller than the correct result.

Every other `u128`→`u64` narrowing in the same file uses the checked pattern:

```rust
// secondary_block_reward (line 204)
let reward = u64::try_from(reward128).map_err(|_| DaoError::Overflow)?;

// dao_field_with_current_epoch (line 245)
Capacity::shannons(u64::try_from(miner_issuance128).map_err(|_| DaoError::Overflow)?)

// dao_field_with_current_epoch (line 258)
let ar_increase = u64::try_from(ar_increase128).map_err(|_| DaoError::Overflow)?;
``` [2](#0-1) [3](#0-2) [4](#0-3) 

`calculate_maximum_withdraw` is the sole exception. The existing test `check_withdraw_calculation_overflows` asserts `result.is_err()` for a near-`u64::MAX` capacity, but the error is produced by the subsequent `safe_add(occupied_capacity)` overflowing — not by the `as u64` cast — so the silent-truncation path (where `safe_add` does not overflow after truncation) is never exercised. [5](#0-4) 

### Impact Explanation

If `withdraw_counted_capacity` (a `u128`) exceeds `u64::MAX` and the truncated value plus `occupied_capacity` does not itself overflow `u64`, `calculate_maximum_withdraw` returns `Ok(Capacity::shannons(truncated_value))` — a value far below the depositor's entitlement. This propagates into:

1. **`transaction_fee`** (called from tx-pool's `check_tx_fee`): the fee is computed as `maximum_withdraw − outputs_capacity`. A truncated `maximum_withdraw` can make a valid withdrawal appear to have a negative fee, causing the node to reject a legitimate transaction.
2. **Block verification**: `transaction_maximum_withdraw` (which calls `calculate_maximum_withdraw`) is used during contextual verification to bound the interest a depositor may claim. A truncated result could cause a node to accept a block that over-pays a depositor (if the on-chain script allows it) or to reject a valid block. [6](#0-5) [7](#0-6) 

### Likelihood Explanation

`withdraw_counted_capacity` overflowing `u64` requires:

```
counted_capacity × withdrawing_ar / deposit_ar  >  u64::MAX  ≈  1.84 × 10¹⁹
```

The total CKB supply is approximately 3.36 × 10¹⁸ shannons, so `counted_capacity` is bounded by that. For the product to exceed `u64::MAX`, the AR ratio `withdrawing_ar / deposit_ar` must exceed ≈ 5.5. AR starts at 10¹⁶ and grows proportionally to cumulative secondary issuance divided by total capacity. At the current secondary issuance rate (~1.344 billion CKB/year), reaching a 5.5× AR ratio would take on the order of a century. **Likelihood is very low in the near term** but the code is provably incorrect and inconsistent with every other narrowing in the same file, and the condition will eventually be reachable on a long-lived chain.

### Recommendation

Replace the silent cast with the same checked pattern used everywhere else in the file:

```rust
// Before (line 155-156):
let withdraw_capacity =
    Capacity::shannons(withdraw_counted_capacity as u64).safe_add(occupied_capacity)?;

// After:
let withdraw_counted_capacity_u64 =
    u64::try_from(withdraw_counted_capacity).map_err(|_| DaoError::Overflow)?;
let withdraw_capacity =
    Capacity::shannons(withdraw_counted_capacity_u64).safe_add(occupied_capacity)?;
```

Add a dedicated unit test that constructs a scenario where `withdraw_counted_capacity` itself (before `safe_add`) exceeds `u64::MAX` and verifies that `DaoError::Overflow` is returned, distinct from the existing test that only exercises the `safe_add` overflow path.

### Proof of Concept

**Triggering the silent truncation (conceptual):**

```
deposit_ar      = 10_000_000_000_000_000   (10^16, genesis value)
withdrawing_ar  = 60_000_000_000_000_000   (6× growth — reachable after ~125 years)
counted_capacity = 3_360_000_000_000_000_000  (3.36 × 10^18 shannons, entire supply in one cell)

withdraw_counted_capacity (u128)
  = 3_360_000_000_000_000_000 × 60_000_000_000_000_000 / 10_000_000_000_000_000
  = 20_160_000_000_000_000_000   (> u64::MAX = 18_446_744_073_709_551_615)

withdraw_counted_capacity as u64
  = 20_160_000_000_000_000_000 mod 2^64
  = 1_713_255_926_290_448_384   (≈ 1.71 × 10^18, ~85% loss)

safe_add(occupied_capacity) succeeds (no overflow)
→ returns Ok(Capacity::shannons(1_713_255_926_290_448_384))
  instead of Err(DaoError::Overflow)
```

A NervosDAO depositor who deposited the entire CKB supply and waited long enough would receive approximately 85% less than their entitled withdrawal, with the node accepting the transaction as valid. [8](#0-7)

        let withdrawing_header = self
            .data_loader
            .get_header(withdrawing_header_hash)
            .ok_or(DaoError::InvalidHeader)?;
        if deposit_header.number() >= withdrawing_header.number() {
            return Err(DaoError::InvalidOutPoint);

            / u128::from(deposit_ar);

            Capacity::shannons(withdraw_counted_capacity as u64).safe_add(occupied_capacity)?;

#[test]
fn check_withdraw_calculation_overflows() {
    let output = CellOutput::new_builder()
        .capacity(Capacity::shannons(18_446_744_073_709_550_000))
        .build();
    let tx = TransactionBuilder::default().output(output.clone()).build();
    let epoch = EpochNumberWithFraction::new(1, 100, 1000);
    let deposit_header = HeaderBuilder::default()
        .number(100)
        .epoch(epoch)
        .dao(pack_dao_data(
            10_000_000_000_123_456,
            Default::default(),
            Default::default(),
            Default::default(),
        ))
        .build();
    let deposit_block = BlockBuilder::default()
        .header(deposit_header)
        .transaction(tx)
        .build();

    let epoch = EpochNumberWithFraction::new(1, 200, 1000);
    let withdrawing_header = HeaderBuilder::default()
        .number(200)
        .epoch(epoch)
        .dao(pack_dao_data(
            10_000_000_001_123_456,
            Default::default(),
            Default::default(),
            Default::default(),
        ))
        .build();
    let withdrawing_block = BlockBuilder::default().header(withdrawing_header).build();

    let tmp_dir = TempDir::new().unwrap();
    let db = RocksDB::open_in(&tmp_dir, COLUMNS);
    let store = ChainDB::new(db, Default::default());
    let txn = store.begin_transaction();
    txn.insert_block(&deposit_block).unwrap();
    txn.attach_block(&deposit_block).unwrap();
    txn.insert_block(&withdrawing_block).unwrap();
    txn.attach_block(&withdrawing_block).unwrap();
    txn.commit().unwrap();

    let consensus = Consensus::default();
    let data_loader = store.borrow_as_data_loader();
    let calculator = DaoCalculator::new(&consensus, &data_loader);
    let result = calculator.calculate_maximum_withdraw(
        &output,
        Capacity::bytes(0).expect("should not overflow"),
        &deposit_block.hash(),
        &withdrawing_block.hash(),
    );
    assert!(result.is_err());
}
