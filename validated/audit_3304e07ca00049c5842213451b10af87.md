### Title
Unconditionally Permissive CORS Policy Allows Any Web Origin to Access Local CKB RPC — (`rpc/src/server.rs`)

### Summary
The CKB RPC HTTP/WebSocket server applies `CorsLayer::permissive()` unconditionally, which instructs browsers to allow cross-origin requests from **any** origin to the node's RPC endpoint. A malicious web page visited by a user running a local CKB node can silently invoke any enabled RPC method — including `send_transaction`, `clear_tx_pool`, `set_ban`, `clear_banned_addresses`, `add_node`, `remove_node`, and `set_network_active` — without any user interaction or authentication.

### Finding Description
In `rpc/src/server.rs`, the `start_server` function builds the Axum router and unconditionally attaches `CorsLayer::permissive()`:

```rust
let app = Router::new()
    .route("/", method_router.clone())
    .route("/{*path}", method_router)
    .route("/ping", get(ping_handler))
    .layer(Extension(Arc::clone(rpc)))
    .layer(CorsLayer::permissive())   // ← allows ALL origins
    ...
```

`tower_http::cors::CorsLayer::permissive()` sets:
- `Access-Control-Allow-Origin: *`
- `Access-Control-Allow-Methods: *`
- `Access-Control-Allow-Headers: *`

This applies to both the HTTP server and the WebSocket server (both call `start_server`). There is no distinction between production and development builds, no configurable origin allowlist, and no restriction to `null` or `localhost` origins.

The default `listen_address` is `127.0.0.1:8114`, which is only reachable from the local machine. However, browsers treat web pages as cross-origin relative to `127.0.0.1`, so the browser's Same-Origin Policy would normally block `https://evil.com` from reading responses from `http://127.0.0.1:8114`. The permissive CORS response headers explicitly override this protection, instructing the browser to allow the cross-origin request to complete. [1](#0-0) 

The `default_trusted_proxies` function in the network config hardcodes localhost as a trusted proxy, but this is a separate P2P-layer concern and does not restrict RPC access. [2](#0-1) 

### Impact Explanation
Any web page a CKB node operator visits can silently call the full set of enabled RPC methods. The default production module list includes `Net`, `Pool`, `Miner`, `Chain`, `Stats`, `Subscription`, `Experiment`, and `Terminal`. [3](#0-2) 

Sensitive operations reachable via cross-origin requests include:

- **`send_transaction`** — submit arbitrary transactions from the node's perspective
- **`clear_tx_pool`** / **`remove_transaction`** — evict pending transactions, disrupting the operator's mempool
- **`set_ban`** / **`clear_banned_addresses`** — manipulate the peer ban list, re-admitting banned peers or banning legitimate ones
- **`add_node`** / **`remove_node`** — force connections to or disconnections from specific peers
- **`set_network_active`** — disable the node's P2P networking entirely [4](#0-3) 

### Likelihood Explanation
The attack requires only that the node operator visit a web page while their CKB node is running — a routine occurrence. No special privileges, leaked keys, or social engineering beyond a single page visit are required. The RPC is bound to `127.0.0.1` by default, but this is irrelevant because the browser itself is on the same machine and the permissive CORS headers explicitly authorize the cross-origin read. This is a well-known browser-based attack class (localhost CORS abuse) with documented real-world exploitation.

### Recommendation
Replace `CorsLayer::permissive()` with a restrictive policy. For a node whose RPC is intended only for local tooling (CLI, miner), the correct policy is to allow only `null` origin (direct requests) or no cross-origin access at all:

```rust
.layer(CorsLayer::new()) // deny all cross-origin by default
```

If cross-origin access is needed (e.g., for a local web UI), expose a configurable `rpc.allowed_origins` list in `RpcConfig` and validate against it, analogous to how `whitelist_peers` restricts P2P access. Never use `permissive()` in a production build. [5](#0-4) 

### Proof of Concept
1. Start a CKB node with default config (`listen_address = "127.0.0.1:8114"`).
2. Open a browser and navigate to any web page (or use the browser console on any HTTPS page).
3. Execute:

```javascript
fetch("http://127.0.0.1:8114", {
  method: "POST",
  headers: {"Content-Type": "application/json"},
  body: JSON.stringify({
    id: 1, jsonrpc: "2.0",
    method: "clear_banned_addresses", params: []
  })
}).then(r => r.json()).then(console.log);
```

4. The browser sends a CORS preflight OPTIONS request; the CKB server responds with `Access-Control-Allow-Origin: *`.
5. The browser completes the POST. The response `{"id":1,"jsonrpc":"2.0","result":null}` confirms the RPC call succeeded from a foreign origin.

The same technique applies to `send_transaction`, `set_ban`, `set_network_active`, and all other enabled methods. [6](#0-5)

```rust
    fn start_server(
        rpc: &Arc<MetaIoHandler<Option<Session>>>,
        address: String,
        handler: Handle,
        enable_websocket: bool,
    ) -> Result<SocketAddr, AnyError> {
        let stream_config = StreamServerConfig::default()
            .with_keep_alive(true)
            .with_pipeline_size(4)
            .with_shutdown(async move {
                new_tokio_exit_rx().cancelled().await;
            });

        // HTTP and WS server.
        let post_router = post(handle_jsonrpc::<Option<Session>>);
        let get_router = if enable_websocket {
            get(handle_jsonrpc_ws::<Option<Session>>)
        } else {
            get(get_error_handler)
        };
        let method_router = post_router.merge(get_router);

        let app = Router::new()
            .route("/", method_router.clone())
            .route("/{*path}", method_router)
            .route("/ping", get(ping_handler))
            .layer(Extension(Arc::clone(rpc)))
            .layer(CorsLayer::permissive())
            .layer(TimeoutLayer::with_status_code(
                StatusCode::REQUEST_TIMEOUT,
                Duration::from_secs(30),
            ))
            .layer(Extension(stream_config));

```
