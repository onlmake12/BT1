### Title
Unauthenticated `from` Field in Hole-Punching `ConnectionRequest` Enables Peer-ID Spoofing and SSRF — (`network/src/protocols/hole_punching/component/connection_request.rs`)

### Summary

The CKB hole-punching protocol accepts a `ConnectionRequest` message whose `from` field is taken verbatim from the attacker-controlled message payload. The handler never verifies that `content.from` matches the cryptographically authenticated peer ID of the actual TCP session. Any connected peer can therefore impersonate any other peer ID, poison the `pending_delivered` state table with arbitrary listen addresses, and cause the victim node to initiate outbound TCP connections to attacker-chosen IP addresses (SSRF). The `forward_rate_limiter` is also keyed on the spoofed `from`, allowing rate-limit exhaustion against arbitrary peer-ID pairs.

### Finding Description

**Root cause — `from` is never bound to the session's authenticated peer ID.**

In `ConnectionRequestProcess::execute`, the `from` field is parsed directly from the wire message: [1](#0-0) 

The struct that carries it: [2](#0-1) 

The `ConnectionRequestProcess` struct holds the actual session index (`peer: PeerIndex`) alongside the parsed message, but the handler never cross-checks `content.from` against the peer ID that the Secio handshake authenticated for that session: [3](#0-2) 

The session's real peer ID is available via `peer_registry.get_peer(session_id)` → `extract_peer_id(&peer.connected_addr)`, but this lookup is never performed for the `from` field.

**Attack path 1 — `pending_delivered` poisoning → SSRF.**

When the receiving node is the `to` target, `respond_delivered` is called with the attacker-supplied `from_peer_id` and the attacker-supplied `listen_addrs`: [4](#0-3) 

The node stores `from_peer_id → (remote_listens, now)` in `pending_delivered`: [5](#0-4) 

Later, when a `ConnectionSync` arrives with `from = <spoofed_peer_id>`, the node looks up `pending_delivered.get(&content.from)` and initiates TCP connections to whatever addresses were stored there: [6](#0-5) 

The `try_nat_traversal` function then opens real outbound TCP sockets to those addresses: [7](#0-6) 

**Attack path 2 — `forward_rate_limiter` exhaustion / bypass.**

The forward rate limiter is keyed on `(content.from, content.to, msg_item_id)`: [8](#0-7) 

Because `content.from` is attacker-controlled, a single connected peer can exhaust rate-limit slots for arbitrary peer-ID pairs, blocking legitimate hole-punching requests from those pairs. Conversely, the attacker can rotate `from` values to bypass the rate limit that would otherwise apply to their real peer ID.

**Attack path 3 — `inflight_requests` state corruption.**

In `ConnectionRequestDeliveredProcess::execute`, `inflight_requests.remove(&content.to)` is called with the attacker-controlled `to` field: [9](#0-8) 

An attacker can remove in-flight hole-punching state for arbitrary peer IDs, silently cancelling legitimate NAT traversal attempts.

**Wire schema confirms all three fields are fully attacker-controlled:** [10](#0-9) 

### Impact Explanation

- **SSRF / connection hijacking**: Any connected peer can cause the victim CKB node to open outbound TCP connections to arbitrary IP:port pairs. This can be used to probe internal network services, trigger connection-based side effects, or exhaust file-descriptor / connection-slot resources.
- **Rate-limit bypass and exhaustion**: The `forward_rate_limiter` is rendered ineffective as a DoS mitigation because its key is fully attacker-controlled. A single peer can block hole-punching for any pair of legitimate peers.
- **State corruption**: Legitimate `inflight_requests` entries can be silently deleted, permanently breaking NAT traversal for targeted peers without any error visible to the victim.

### Likelihood Explanation

The hole-punching protocol is open to any connected peer — no special role or privilege is required. The `HolePunching` protocol handler is registered as a standard service protocol: [11](#0-10) 

An attacker needs only to establish a single inbound or outbound P2P session to the target node and send a crafted `HolePunchingMessage::ConnectionRequest`. The message format is public and the molecule schema is published. Exploitation requires no cryptographic material and no privileged access.

### Recommendation

1. **Bind `from` to the authenticated session.** At the start of `ConnectionRequestProcess::execute`, resolve the actual peer ID from the session registry and reject any message where `content.from != actual_session_peer_id`. The session peer ID is already available via `self.peer` (a `PeerIndex`) and `network_state.with_peer_registry(|r| r.get_peer(self.peer))`.
2. **Key the `forward_rate_limiter` on the session ID**, not on the message-supplied `from` field, so it cannot be bypassed by rotating spoofed identities.
3. **Validate `content.to` against `inflight_requests`** before removing entries, or remove entries only when the message arrives from the authenticated session that originally issued the request.

### Proof of Concept

```
Attacker (connected peer, session_id=S, real_peer_id=A) sends:

HolePunchingMessage::ConnectionRequest {
    from: <victim_peer_id V>,   // spoofed — not A
    to:   <target_node T>,      // T is the receiving node
    listen_addrs: [attacker_ip:1234],
    max_hops: 6,
    route: [],
}

T receives the message. Because content.to == T's own peer_id:
  → respond_delivered(V, T, [attacker_ip:1234]) is called
  → pending_delivered.insert(V, ([attacker_ip:1234], now))

Attacker then sends (or relays):

HolePunchingMessage::ConnectionSync {
    from: V,
    to:   T,
    route: [],
}

T receives it. content.to == T, so it looks up pending_delivered[V]
  → gets [attacker_ip:1234]
  → spawns try_nat_traversal(bind_addr, attacker_ip:1234)
  → opens outbound TCP connection to attacker_ip:1234
```

The attacker controls the destination of the TCP connection entirely through the unauthenticated `from` and `listen_addrs` fields.

    pub(crate) async fn execute(mut self) -> Status {
        let content = match RequestContent::try_from(&self.message) {
            Ok(content) => content,
            Err(status) => return status,
        };
        if content.listen_addrs.len() > ADDRS_COUNT_LIMIT || content.listen_addrs.is_empty() {
            return StatusCode::InvalidListenAddrLen
                .with_context("the listen address count is too large or empty");
        }

        if content.max_hops > MAX_HOPS {
            return StatusCode::InvalidMaxTTL.into();
        }
        if content.route.len() > MAX_HOPS as usize {
            return StatusCode::InvalidRoute.with_context("the route length is too long");
        }

        let self_peer_id = self.protocol.network_state.local_peer_id();
        if content.route.contains(self_peer_id) {
            return StatusCode::Ignore.with_context("the message is passed, ignore it");
        }

        if self
            .protocol
            .forward_rate_limiter
            .check_key(&(content.from.clone(), content.to.clone(), self.msg_item_id))
            .is_err()
        {
            debug!(
                "from: {}, to {}, item_name: {}, rate limit is reached",
                content.from, content.to, "ConnectionRequest",
            );
            return StatusCode::TooManyRequests.with_context("ConnectionRequest");
        }

        if self_peer_id == &content.to {
            self.respond_delivered(content.from, &content.to, content.listen_addrs)
                .await
        } else if content.max_hops == 0u8 {
            StatusCode::ReachedMaxHops.into()
        } else {
            self.forward_message(self_peer_id, &content.to).await
        }
    }

    async fn respond_delivered(
        &mut self,
        from_peer_id: PeerId,
        to_peer_id: &PeerId,
        remote_listens: Vec<Multiaddr>,
    ) -> Status {
        if let Some((_, t)) = self.protocol.pending_delivered.get(&from_peer_id) {
            let now = unix_time_as_millis();
            if now - t < HOLE_PUNCHING_INTERVAL {
                return StatusCode::Ignore
                    .with_context("a same message is already replied in a moment ago");
            }
        }
        let listen_addrs = {
            let public_addr = self.protocol.network_state.public_addrs(ADDRS_COUNT_LIMIT);
            if public_addr.len() < ADDRS_COUNT_LIMIT {
                let observed_addrs = self
                    .protocol
                    .network_state
                    .observed_addrs(ADDRS_COUNT_LIMIT - public_addr.len());
                let iter = public_addr
                    .iter()
                    .chain(observed_addrs.iter())
                    .map(Multiaddr::to_vec)
                    .map(|v| packed::Address::new_builder().bytes(v).build());
                packed::AddressVec::new_builder().extend(iter).build()
            } else {
                let iter = public_addr
                    .iter()
                    .map(Multiaddr::to_vec)
                    .map(|v| packed::Address::new_builder().bytes(v).build());
                packed::AddressVec::new_builder().extend(iter).build()
            }
        };
        let content = init_delivered(self.message, listen_addrs);
        let new_message = packed::HolePunchingMessage::new_builder()
            .set(content)
            .build()
            .as_bytes();
        let proto_id = SupportProtocols::HolePunching.protocol_id();

        let remote_listens: Vec<Multiaddr> = remote_listens
            .into_iter()
            .filter_map(|addr| match find_type(&addr) {
                TransportType::Memory
                | TransportType::Onion
                | TransportType::Ws
                | TransportType::Wss
                | TransportType::Tls => None,
                TransportType::Tcp => {
                    if addr
                        .iter()
                        .any(|p| matches!(p, Protocol::Ip4(_) | Protocol::Ip6(_)))
                    {
                        Some(addr)
                    } else {
                        None
                    }
                }
            })
            .collect();

        if remote_listens.is_empty() {
            return StatusCode::Ignore.with_context("remote listen address is empty");
        }

        debug!(
            "current peer is the target peer {}, send a response back",
            to_peer_id

        if let Err(error) = self

            .send_message_to(self.peer, proto_id, new_message)
            .await
        {
            return StatusCode::ForwardError.with_context(error);
        }

        let now = unix_time_as_millis();
        self.protocol
            .pending_delivered
            .insert(from_peer_id, (remote_listens, now));

pub(crate) async fn try_nat_traversal(
    bind_addr: Option<SocketAddr>,
    addr: Multiaddr,
) -> Result<(TcpStream, Multiaddr), std::io::Error> {
    let net_addr = match multiaddr_to_socketaddr(&addr) {
        Some(addr) => addr,
        None => {
            debug!("Failed to convert multiaddr to socketaddr");
            return Err(std::io::ErrorKind::InvalidInput.into());
        }
    };

    // Use a fixed interval but add a small amount of randomness
    let base_retry_interval = Duration::from_millis(200);

    // total time
    let timeout_duration = Duration::from_secs(30);
    let start_time = Instant::now();
    let mut retry_count = 0u32;
    while start_time.elapsed() < timeout_duration {
        retry_count += 1;

        // Add a small amount of random jitter (±25ms) to avoid conflicts
        // caused by continuous precise synchronization
        let jitter = Duration::from_millis(rand::random::<u64>() % 50);
        let actual_interval = if rand::random::<bool>() {
            base_retry_interval + jitter
        } else {
            base_retry_interval.saturating_sub(jitter)
        };

        let socket = create_socket(bind_addr, net_addr)?;

        match runtime::timeout(
            std::time::Duration::from_millis(200),
            socket.connect(net_addr),
        )
        .await
        {
            Ok(Ok(stream)) => {
                // try get the stored error in the underlying socket
                // if the socket is not connected, it will return an error

                    debug!("Failed to connect to NAT(base check): {}", err);
                }
                return Ok((stream, addr));
