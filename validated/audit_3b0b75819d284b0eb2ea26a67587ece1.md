### Title
OIS Slash Evasion via `undelegate` Frontrun of `create_slash_event` — (`governance/pyth_staking_sdk/src/idl/integrity-pool.json`)

---

### Summary

The Oracle Integrity Staking (OIS) slashing process is two-phase and public: first `create_slash_event` is called by the `rewardProgramAuthority`, then `slash` is called (permissionlessly) per-staker. Because the Pythian Council process is fully public (forum posts, governance votes) and spans up to two epochs before `create_slash_event` is submitted on-chain, a delegator who was active during the slash epoch can observe the pending slash and call `undelegate` before `create_slash_event` lands. If the per-staker `slash` instruction resolves slashable stake from the current position state rather than from a historical epoch snapshot, the staker escapes the penalty entirely.

---

### Finding Description

The OIS slash lifecycle has two distinct on-chain steps:

**Step 1 — `create_slash_event`** (privileged, `rewardProgramAuthority` signer):
Creates a `SlashEvent` PDA keyed by `(publisher, index)` storing `epoch`, `slash_ratio`, and `slash_custody`. [1](#0-0) 

**Step 2 — `slash`** (permissionless, any `signer`):
Applies the slash to a specific staker by reading the staker's `delegation_record` and calling into the staking program's `slash_account`. [2](#0-1) 

The `SlashEvent` struct records the target `epoch`: [3](#0-2) 

The `undelegate` instruction is permissionless for any staker and moves a position from `LOCKED` to `PREUNLOCKING` (sets `unlockingStart`): [4](#0-3) 

Position state transitions are epoch-gated: [5](#0-4) 

The Pythian Council process is explicitly public and takes up to two full epochs (14 days) from incident to slash execution: [6](#0-5) 

During this window, a staker can call `undelegate` (via `unstakeFromPublisher` / `integrityPoolProgram.methods.undelegate`) before `create_slash_event` is submitted: [7](#0-6) 

If the `slash` instruction resolves slashable stake from the staker's **current** position state (i.e., only `LOCKED` positions) rather than from a historical snapshot of who was `LOCKED` during `slash_event.epoch`, the staker's positions are already in `PREUNLOCKING` or `UNLOCKING` at the time `slash` is called and are not reduced.

---

### Impact Explanation

A delegator who was staked during the misprint epoch escapes the slash penalty entirely by undelegating before `create_slash_event` is submitted. This directly undermines the OIS security model: the economic deterrent against supporting inaccurate publishers is nullified for any staker who monitors the Pythian Council forum. Slashed amounts that should flow to the DAO treasury are not collected, and the pool's total slashable stake is artificially reduced, weakening the protocol's accountability guarantee.

---

### Likelihood Explanation

- The Pythian Council process is fully public (forum.pyth.network posts, governance votes) and takes up to two epochs — a large advance-notice window.
- `undelegate` is permissionless and costs only a transaction fee.
- The `slash` instruction is also permissionless, so the staker cannot be forced to be slashed before they undelegate.
- Solana's transaction submission model does not provide a traditional mempool, but the multi-epoch governance window makes mempool-level frontrunning unnecessary — the staker has days of advance notice.
- Sophisticated delegators (e.g., large institutional stakers) have strong financial incentive to monitor governance forums.

---

### Recommendation

1. **Snapshot-based slash eligibility**: Record the set of delegators and their staked amounts at the start of the slash epoch (or at `create_slash_event` time) and enforce that `slash` applies the ratio to the snapshotted amount, regardless of current position state.
2. **Slash-on-undelegate**: When `undelegate` is called, check whether any pending `SlashEvent` exists for the publisher whose epoch overlaps with the position's active range, and apply the slash before allowing the position to transition.
3. **Freeze undelegation during active slash window**: After `create_slash_event` is emitted for a publisher, block `undelegate` calls for positions that were `LOCKED` during the slash epoch until `slash` has been applied to that staker.

---

### Proof of Concept

1. Epoch `N`: Publisher P produces a misprint. Staker S has 1,000,000 PYTH delegated to P's pool (`LOCKED`).
2. Epoch `N+1`: Pythian Council posts on forum.pyth.network announcing a 500 bps slash against P's pool.
3. Staker S observes the forum post and calls `undelegate(position_index, 1_000_000)` — positions move to `PREUNLOCKING`.
4. Epoch `N+2`: Pythian Council calls `create_slash_event(index=0, slash_ratio=500_bps)` for publisher P.
5. Anyone calls `slash(index=0)` for staker S's `stake_account_positions`.
6. If `slash` only reduces `LOCKED` positions, S's positions are now `PREUNLOCKING`/`UNLOCKING` and are not reduced. S retains all 1,000,000 PYTH.
7. S re-delegates after cooldown ends, having suffered zero penalty for the misprint epoch. [1](#0-0) [2](#0-1) [6](#0-5)

export const getPositionState = (
  position: Position,
  currentEpoch: bigint,
): PositionState => {
  if (currentEpoch < position.activationEpoch) {
    return PositionState.LOCKING;
  }
  if (!position.unlockingStart) {
    return PositionState.LOCKED;
  }
  const hasActivated = position.activationEpoch <= currentEpoch;
  const unlockStarted = position.unlockingStart <= currentEpoch;
  const unlockEnded = position.unlockingStart + 1n <= currentEpoch;

  if (hasActivated && !unlockStarted) {
    return PositionState.PREUNLOCKING;
  } else if (unlockStarted && !unlockEnded) {
    return PositionState.UNLOCKING;
  } else {
    return PositionState.UNLOCKED;
  }
};

  public async unstakeFromPublisher(
    stakeAccountPositions: PublicKey,
    publisher: PublicKey,
    positionState: PositionState.LOCKED | PositionState.LOCKING,
    amount: bigint,
  ) {
    const stakeAccountPositionsData = await this.getStakeAccountPositions(
      stakeAccountPositions,
    );
    const currentEpoch = await getCurrentEpoch(this.connection);

    let remainingAmount = amount;
    const instructionPromises: Promise<TransactionInstruction>[] = [];

    const eligiblePositions = stakeAccountPositionsData.data.positions
      .map((p, i) => ({ index: i, position: p }))
      .reverse()
      .filter(
        ({ position }) =>
          position.targetWithParameters.integrityPool?.publisher !==
            undefined &&
          position.targetWithParameters.integrityPool.publisher.equals(
            publisher,
          ) &&
          positionState === getPositionState(position, currentEpoch),
      );

    for (const { position, index } of eligiblePositions) {
      if (position.amount < remainingAmount) {
        instructionPromises.push(
          this.integrityPoolProgram.methods
            .undelegate(index, convertBigIntToBN(position.amount))
            .accounts({
              publisher,
              stakeAccountPositions,
            })
            .instruction(),
        );
        remainingAmount -= position.amount;
      } else {
        instructionPromises.push(
          this.integrityPoolProgram.methods
            .undelegate(index, convertBigIntToBN(remainingAmount))
            .accounts({
              publisher,
              stakeAccountPositions,
            })
            .instruction(),
        );
        break;
      }
    }

    const instructions = await Promise.all(instructionPromises);
    return sendTransaction(instructions, this.connection, this.wallet);
  }
