Audit Report

## Title
`GetBlocksProcess` Missing IBD State Check Contradicts Documented Design Contract - (File: sync/src/synchronizer/get_blocks_process.rs)

## Summary
The module-level documentation in `sync/src/synchronizer/mod.rs` explicitly contracts that an IBD-mode node must respond with `InIBD` to both `GetHeaders` and `GetBlocks` requests. `GetHeadersProcess::execute()` enforces this correctly, but `GetBlocksProcess::execute()` contains no IBD check, allowing any peer to force the node to perform RocksDB lookups and serve full block data during IBD, directly violating the stated design contract and degrading IBD throughput.

## Finding Description
The design contract is stated at `sync/src/synchronizer/mod.rs` L4: [1](#0-0) 

`GetHeadersProcess::execute()` enforces this at `sync/src/synchronizer/get_headers_process.rs` L53–65 by calling `active_chain.is_initial_block_download()`, invoking `send_in_ibd()`, and returning `Status::ignored()`: [2](#0-1) 

`GetBlocksProcess::execute()` has no such check. After the `MAX_HEADERS_LEN` bounds check, it immediately acquires `active_chain` and begins iterating block hashes: [3](#0-2) 

For each hash it performs a `contains_block_status` RocksDB lookup and, if the block is found valid, a `get_block` retrieval followed by spawning an async task to send the full block to the peer: [4](#0-3) 

The dispatch in `sync/src/synchronizer/mod.rs` L407–411 applies no top-level IBD guard before invoking `GetBlocksProcess`: [5](#0-4) 

The IBD guard in `sync/src/relayer/mod.rs` L815–818 is irrelevant because `GetBlocks` is a Sync protocol message, not a Relay protocol message: [6](#0-5) 

The exploit path: any peer opens a Sync protocol connection and sends a `GetBlocks` message containing up to `MAX_HEADERS_LEN` (2000) block hashes. The node processes up to `INIT_BLOCKS_IN_TRANSIT_PER_PEER` (32) per call, performing `contains_block_status` RocksDB lookups and `get_block` retrievals for each, and spawning async send tasks — all while its own IBD pipeline competes for the same disk I/O and CPU resources. [7](#0-6) 

## Impact Explanation
This is a **Low** impact finding (501–2000 points): any other important performance improvement for CKB. The missing check violates the explicitly documented IBD design contract and forces an IBD node to perform unnecessary disk I/O (RocksDB status lookups and block retrievals) and spawn async send tasks in response to attacker-controlled messages. This degrades IBD throughput. It does not crash the node, cause consensus deviation, or damage the CKB economy, so it does not qualify for High or Critical.

## Likelihood Explanation
Any peer that opens a Sync protocol connection can trigger this. No special privilege, key, or hashpower is required. Block hashes are public on-chain data. The attack is repeatable at negligible cost and requires only a TCP connection. During IBD, the node accepts inbound connections and processes Sync messages normally.

## Recommendation
Add an IBD check at the start of `GetBlocksProcess::execute()`, mirroring `GetHeadersProcess`. When `active_chain.is_initial_block_download()` returns `true`, send an `InIBD` response and return `Status::ignored()`. A `send_in_ibd()` helper identical to the one in `GetHeadersProcess` (L101–115) should be added to `GetBlocksProcess`: [8](#0-7) 

## Proof of Concept
1. Start a CKB node from genesis; confirm `is_initial_block_download()` returns `true` (tip timestamp is far behind wall clock).
2. Connect a custom peer that opens the Sync protocol.
3. Send a `GetBlocks` message containing the hash of block #1 (publicly known).
4. Observe the node responds with a `SendBlock` message containing full block data instead of `InIBD`.
5. Send a `GetHeaders` message to the same node; observe it correctly returns `InIBD` and logs: *"Ignoring getheaders from peer=X because the node is in initial block download stage."*
6. The asymmetry between steps 4 and 5 confirms the missing guard.

**File:** sync/src/synchronizer/get_blocks_process.rs (L33-94)
```rust
    pub fn execute(self) -> Status {
        let block_hashes = self.message.block_hashes();
        // use MAX_HEADERS_LEN as limit, we may increase the value of INIT_BLOCKS_IN_TRANSIT_PER_PEER in the future
        if block_hashes.len() > MAX_HEADERS_LEN {
            return StatusCode::ProtocolMessageIsMalformed.with_context(format!(
                "BlockHashes count({}) > MAX_HEADERS_LEN({})",
                block_hashes.len(),
                MAX_HEADERS_LEN,
            ));
        }
        let active_chain = self.synchronizer.shared.active_chain();

        let iter = block_hashes.iter().take(INIT_BLOCKS_IN_TRANSIT_PER_PEER);

        let mut dedup = HashSet::new();
        for block_hash in iter {
            debug!("get_blocks {} from peer {:?}", block_hash, self.peer);
            let block_hash = block_hash.to_entity();

            if block_hash == self.synchronizer.shared().consensus().genesis_hash() {
                return StatusCode::RequestGenesis.with_context("Request genesis block");
            }

            if !dedup.insert(block_hash.clone()) {
                return StatusCode::RequestDuplicate.with_context("Request duplicate block");
            }

            if !active_chain.contains_block_status(&block_hash, BlockStatus::BLOCK_VALID) {
                debug!(
                    "Ignoring get_block {} request from peer={} as it is not verified.",
                    block_hash, self.peer
                );
                continue;
            }

            if let Some(block) = active_chain.get_block(&block_hash) {
                debug!(
                    "respond_block {} {} to peer {:?}",
                    block.number(),
                    block.hash(),
                    self.peer,
                );
                let content = packed::SendBlock::new_builder().block(block.data()).build();
                let message = packed::SyncMessage::new_builder().set(content).build();

                let nc = Arc::clone(self.nc);
                self.synchronizer
                    .shared()
                    .shared()
                    .async_handle()
                    .spawn(async move { async_send_message_to(&nc, self.peer, &message).await });
            } else {
                // TODO response not found
                // TODO add timeout check in synchronizer

                // We expect that `block_hashes` is sorted descending by height.
                // So if we cannot find the current one from local, we cannot find
                // the next either.
                debug!("Stopping getblocks, since {} is not found", block_hash);
                break;
            }
        }
```
