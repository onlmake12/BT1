### Title
Unbounded Orphan-Chain BFS Resolution in `process_orphan_tx` Enables CPU/Lock-Contention DoS — (File: `tx-pool/src/process.rs`)

---

### Summary

When a transaction is accepted into the CKB tx-pool, `TxPoolService::process_orphan_tx` performs a BFS traversal over every orphan transaction that depends on the newly accepted one, calling full CKB-VM script verification (`_process_tx`) for each orphan synchronously within the loop. An unprivileged P2P peer can pre-populate the orphan pool (hard-capped at 100 entries) with transactions all depending on a single parent output, then submit the parent to trigger processing of all 100 orphans in one BFS invocation. Each orphan can consume up to `max_tx_verify_cycles` (70 M cycles) of CKB-VM execution, producing a burst of up to 7 B cycles of CPU work — twice the `max_block_cycles` consensus limit — in a single async task, causing severe lock contention and tx-pool service stall.

---

### Finding Description

`process_orphan_tx` in `tx-pool/src/process.rs` implements a BFS loop with no per-invocation iteration cap:

For each orphan whose declared cycles are ≤ `max_tx_verify_cycles`, `_process_tx` is called directly inside the loop. `_process_tx` runs full CKB-VM script verification and acquires the tx-pool write lock: [2](#0-1) 

The orphan pool is capped at `DEFAULT_MAX_ORPHAN_TRANSACTIONS = 100`, but this cap only limits *storage*, not the number of entries processed per BFS invocation: [3](#0-2) [4](#0-3) 

The `by_out_point` index allows multiple orphans to share the same parent output. A transaction with multiple outputs can fan out to many orphans simultaneously: [5](#0-4) 

The `max_tx_verify_cycles` config is 70 M cycles per transaction: [6](#0-5)

With 100 orphans each at 70 M cycles, a single `process_orphan_tx` invocation can trigger up to **7 B cycles** of CKB-VM work — 2× the block cycle limit — in one async task, while holding the tx-pool write lock repeatedly.

`process_orphan_tx` is called from `after_process` every time a remote or local transaction is accepted: [8](#0-7) [9](#0-8) 

It is also called for every transaction in an attached block during reorg: [10](#0-9) 

---

### Impact Explanation

An attacker who fills the orphan pool with 100 crafted transactions (each with a script that legitimately consumes close to 70 M cycles and all depending on a single parent output) and then submits the parent causes:

1. **CPU spike**: Up to 7 B cycles of CKB-VM execution in a single BFS call, stalling the async task.
2. **Write-lock contention**: Each `_process_tx` call acquires the tx-pool write lock (`with_tx_pool_write_lock`), blocking all concurrent tx-pool reads and writes for the duration of the burst.
3. **Tx-pool service stall**: New transactions submitted during the burst experience significant latency or timeout, degrading node liveness.
4. **Repeatable**: The attacker can repeat the attack after the orphan pool refills.

---

### Likelihood Explanation

The attack is reachable by any unprivileged P2P peer via the `RelayV3` protocol. `TransactionsProcess::execute` accepts relayed transactions and submits them to the tx-pool: [11](#0-10) 

The only per-message guard is that declared cycles must not exceed `max_block_cycles` (3.5 B), which is far above `max_tx_verify_cycles` (70 M): [12](#0-11) 

An attacker operating multiple peers (or using multiple IP addresses) can fill the 100-entry orphan pool with their crafted transactions. Crafting a script that consumes a precise number of cycles is straightforward in CKB-VM (a simple loop). The parent transaction can be submitted by the attacker themselves. The attack requires no privileged access, no majority hashpower, and no social engineering.

---

### Recommendation

1. **Add a per-invocation iteration cap** to `process_orphan_tx`: limit the total number of orphans processed in a single BFS call (e.g., 16 or 32), deferring the remainder to a background task or the next invocation.
2. **Charge the BFS cost to the triggering peer**: track which peer submitted the parent and apply backpressure or rate-limiting if the resulting orphan resolution burst is large.
3. **Limit fan-out per output**: cap the number of orphans that can reference a single `OutPoint` in `by_out_point` (e.g., to 5–10), reducing the maximum BFS width.
4. **Move orphan resolution off the hot path**: instead of awaiting `process_orphan_tx` inline in `after_process`, enqueue resolved orphans into the existing `verify_queue` for bounded, rate-limited processing.

---

### Proof of Concept

1. Attacker connects to a CKB node as a P2P peer via `RelayV3`.
2. Attacker crafts a "parent" transaction `P` with one output `O`.
3. Attacker crafts 100 "orphan" transactions `C_1 … C_100`, each spending `O` (double-spend variants are allowed in the orphan pool per `test_orphan_allows_double_spends_of_unknown_input`), each with a script that consumes exactly 69,999,999 cycles (just below `max_tx_verify_cycles`).
4. Attacker relays `C_1 … C_100` to the node. Each is accepted into the orphan pool (evicting legitimate orphans if needed). The pool fills to 100 entries.
5. Attacker submits `P` via `send_transaction` RPC or relays it via P2P.
6. `after_process` is called for `P`, which calls `process_orphan_tx(&P)`.
7. `find_orphan_by_previous(&P)` returns all 100 orphans.
8. The BFS loop calls `_process_tx` for each of the 100 orphans, running CKB-VM for up to 70 M cycles each.
9. Total CKB-VM work: ~7 B cycles. The tx-pool write lock is acquired and released 100 times in rapid succession.
10. During this period, all other tx-pool operations (new transaction submissions, block assembly) are blocked or severely delayed.

    pub(crate) async fn process_orphan_tx(&self, tx: &TransactionView) {
        let mut orphan_queue: VecDeque<TransactionView> = VecDeque::new();
        orphan_queue.push_back(tx.clone());

        while let Some(previous) = orphan_queue.pop_front() {
            let orphans = self.find_orphan_by_previous(&previous).await;
            for orphan in orphans.into_iter() {
                if orphan.cycle > self.tx_pool_config.max_tx_verify_cycles {
                    debug!(
                        "process_orphan {} added to verify queue; find previous from {}",
                        orphan.tx.hash(),
                        tx.hash(),
                    );
                    let orphan_id = orphan.tx.proposal_short_id();
                    match self
                        .enqueue_verify_queue(
                            orphan.tx.clone(),
                            false,
                            Some((orphan.cycle, orphan.peer)),
                        )
                        .await
                    {
                        Ok(_) => {
                            self.remove_orphan_tx(&orphan_id).await;
                        }
                        Err(reject) => {
                            warn!(
                                "process_orphan {} failed to enqueue verify queue: {}; keep orphan from {}",
                                orphan.tx.hash(),
                                reject,
                                tx.hash(),
                            );
                        }
                    }
                } else if let Some((ret, _snapshot)) = self
                    ._process_tx(orphan.tx.clone(), Some(orphan.cycle), None)
                    .await
                {
                    match ret {
                        Ok(_) => {
                            self.send_result_to_relayer(TxVerificationResult::Ok {
                                original_peer: Some(orphan.peer),
                                tx_hash: orphan.tx.hash(),
                            });
                            debug!(
                                "process_orphan {} success, find previous from {}",
                                orphan.tx.hash(),
                                tx.hash()
                            );
                            self.remove_orphan_tx(&orphan.tx.proposal_short_id()).await;
                            orphan_queue.push_back(orphan.tx);
                        }
                        Err(reject) => {
                            debug!(
                                "process_orphan {} reject {}, find previous from {}",
                                orphan.tx.hash(),
                                reject,
                                tx.hash(),
                            );

                            if !is_missing_input(&reject) {
                                self.remove_orphan_tx(&orphan.tx.proposal_short_id()).await;
                                if reject.is_malformed_tx() {
                                    self.ban_malformed(orphan.peer, format!("reject {reject}"))
                                        .await;
                                }
                                if reject.is_allowed_relay() {
                                    self.send_result_to_relayer(TxVerificationResult::Reject {
                                        tx_hash: orphan.tx.hash(),
                                    });
                                }
                                if reject.should_recorded() {
                                    self.put_recent_reject(&orphan.tx.hash(), &reject).await;
                                }
                            }
                        }
                    }
                }
            }
        }
    }

    pub fn execute(self) -> Status {
        let shared_state = self.relayer.shared().state();
        let txs: Vec<(TransactionView, Cycle)> = {
            // ignore the tx if it's already known or it has never been requested before
            let mut tx_filter = shared_state.tx_filter();
            tx_filter.remove_expired();
            let unknown_tx_hashes = shared_state.unknown_tx_hashes();

            self.message
                .transactions()
                .iter()
                .map(|tx| (tx.transaction().to_entity().into_view(), tx.cycles().into()))
                .filter(|(tx, _)| {
                    !tx_filter.contains(&tx.hash())
                        && unknown_tx_hashes
                            .get_priority(&tx.hash())
                            .map(|priority| priority.requesting_peer() == Some(self.peer))
                            .unwrap_or_default()
                })
                .collect()
        };

        if txs.is_empty() {
            return Status::ok();
        }

        let max_block_cycles = self.relayer.shared().consensus().max_block_cycles();
        if txs
            .iter()
            .any(|(_, declared_cycles)| declared_cycles > &max_block_cycles)
        {
            self.nc.ban_peer(
                self.peer,
                DEFAULT_BAN_TIME,
                String::from("relay declared cycles greater than max_block_cycles"),
            );
            return Status::ok();
        }

        shared_state.mark_as_known_txs(txs.iter().map(|(tx, _)| tx.hash()));

        let tx_pool = self.relayer.shared.shared().tx_pool_controller().clone();
        let peer = self.peer;
        self.relayer
            .shared
            .shared()
            .async_handle()
            .spawn(async move {
                for (tx, declared_cycles) in txs {
                    if let Err(e) = tx_pool
                        .submit_remote_tx(tx.clone(), declared_cycles, peer)
                        .await
                    {
                        error!("submit_tx error {}", e);
                    }
                }
            });

        Status::ok()
    }
