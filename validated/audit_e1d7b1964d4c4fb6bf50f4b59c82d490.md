### Title
`PythAggregatorV3` Returns Stale Prices Without Staleness Enforcement in All Price-Reading Functions - (File: `target_chains/ethereum/sdk/solidity/PythAggregatorV3.sol`)

---

### Summary

`PythAggregatorV3.sol` is the official Pyth SDK Chainlink AggregatorV3 compatibility adapter. Every price-reading function in this contract — `latestRoundData()`, `latestAnswer()`, `getRoundData()`, `decimals()`, `latestTimestamp()`, and `getAnswer()` — calls `pyth.getPriceUnsafe(priceId)` with no staleness check. Any protocol that deploys this adapter as a drop-in Chainlink replacement will silently consume arbitrarily stale prices, enabling an unprivileged attacker to exploit the stale price window for financial gain.

---

### Finding Description

`PythAggregatorV3` wraps the Pyth oracle into the Chainlink `AggregatorV3` interface. The contract exposes a standalone `updateFeeds()` helper, but calling it is entirely optional and not enforced before any read. All six price-reading entry points delegate to `getPriceUnsafe`:

```solidity
// latestRoundData — the primary function called by Chainlink-compatible protocols
function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId); // no staleness check
    roundId = uint80(price.publishTime);
    return (roundId, int256(price.price), price.publishTime, price.publishTime, roundId);
}

// latestAnswer — returns raw price with no staleness check
function latestAnswer() public view virtual returns (int256) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    return int256(price.price);
}
```

`getPriceUnsafe` is explicitly documented as returning a price "from arbitrarily far in the past." The safe alternative, `getPriceNoOlderThan(id, age)`, is never called anywhere in this contract.

The `updatedAt` field returned by `latestRoundData()` is set to `price.publishTime` (the Pyth publish timestamp), not `block.timestamp`. Protocols that do check `updatedAt` will see a stale timestamp, but the contract itself never reverts — it silently serves the stale value. Protocols that omit the `updatedAt` check (common in Chainlink integrations) receive no signal at all.

---

### Impact Explanation

Any protocol that deploys `PythAggregatorV3` as a Chainlink-compatible oracle and calls `latestRoundData()` or `latestAnswer()` will receive a price that may be arbitrarily old. Concrete impacts include:

- **Lending/borrowing protocols**: Collateral valued at a stale (inflated) price allows over-borrowing; debt valued at a stale (deflated) price blocks legitimate liquidations.
- **Derivatives/perpetuals**: Settlement or margin calculations at stale prices allow profitable round-trips.
- **AMMs using oracle pricing**: Stale prices allow arbitrage extraction against the pool.

Because `latestAnswer()` returns only `int256` with no timestamp, callers of that function have no mechanism to detect staleness at all.

---

### Likelihood Explanation

- `PythAggregatorV3` is the official, published Pyth SDK adapter for Chainlink-compatible protocols. It is the primary integration path for any EVM protocol already using Chainlink that migrates to Pyth.
- Pyth operates on a pull model: prices are only updated when someone calls `updatePriceFeeds`. If no keeper or user submits an update, the on-chain price ages indefinitely.
- Many Chainlink-compatible protocols do not validate `updatedAt` (a well-known integration pitfall). For those that do, the contract still does not revert — it merely returns a stale timestamp.
- An unprivileged attacker needs only to observe that no price update has been submitted recently and then call the target protocol's function that reads from `PythAggregatorV3`.

---

### Recommendation

Replace `getPriceUnsafe` with `getPriceNoOlderThan` in all price-reading functions, using a configurable `maxAge` parameter set at construction time:

```solidity
uint256 public maxAge; // e.g., 60 seconds

constructor(address _pyth, bytes32 _priceId, uint256 _maxAge) {
    priceId = _priceId;
    pyth = IPyth(_pyth);
    maxAge = _maxAge;
}

function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
    // ...
}
```

This causes `latestRoundData()` to revert with `StalePrice` when the price is too old, matching the behavior that Chainlink-compatible protocols expect from a healthy oracle.

---

### Proof of Concept

1. Deploy `PythAggregatorV3` pointing at a Pyth price feed.
2. Do **not** call `updateFeeds()`. Allow the on-chain Pyth price to age past any reasonable freshness threshold (e.g., 60 seconds).
3. Call `latestRoundData()`. Observe that it returns successfully with `updatedAt` = a timestamp far in the past and `answer` = the stale price. No revert occurs.
4. Call `latestAnswer()`. Observe that it returns the stale price with no timestamp at all — the caller has no way to detect staleness.

        bytes32 id,
        uint age
    ) public view virtual override returns (PythStructs.Price memory price) {
        price = getPriceUnsafe(id);

        if (diff(block.timestamp, price.publishTime) > age)
            revert PythErrors.StalePrice();
