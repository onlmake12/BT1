### Title
Peer Store Permanently Fillable via Crafted 4-per-/16 Address Distribution, Causing Permanent `EvictionFailed` on Discovery — (`network/src/peer_store/peer_store_impl.rs`)

### Summary

`PeerStore::check_purge` contains a logical flaw in its fallback eviction path: it only evicts from network groups with **strictly more than 4** addresses. An attacker who pre-fills the store with exactly 4 addresses per `/16` subnet across 4096 distinct subnets (totaling `ADDR_COUNT_LIMIT = 16384` entries) causes both eviction passes to find zero candidates, returning `PeerStoreError::EvictionFailed` on every subsequent `add_addr` call. The victim node can no longer learn new peer addresses from the discovery protocol.

**`add_addr`** calls `check_purge` before inserting, propagating any error to the caller: [2](#0-1)

**Pass 1** — remove non-connectable addresses: [3](#0-2) 

**Pass 2** — group by network segment, evict 2 from groups with `addrs.len() > 4`: [4](#0-3) 

The critical flaw is the **strictly-greater-than-4** threshold on line 378. If every group has exactly 4 addresses, the inner `Some(...)` branch is never taken, `candidate_peers` stays empty, and the function falls through to: [5](#0-4) 

**Network grouping** uses the first two octets of IPv4 (i.e., `/16` subnet): [6](#0-5)

### Impact Explanation

Once the store is filled with the crafted distribution (4 addresses × 4096 `/16` subnets = 16384 entries):

- Pass 1 finds zero non-connectable entries (all have `attempts_count=0`).
- Pass 2 finds zero groups with `> 4` members (all have exactly 4).
- Every `add_addr` call returns `Err(PeerStoreError::EvictionFailed)`.
- The node stops accepting new peer addresses from discovery, starving outbound connection attempts and causing effective network isolation.

---

### Likelihood Explanation

- Requires only **one P2P connection** to the victim.
- The discovery protocol accepts peer-advertised addresses without IP ownership verification.
- A single malicious peer can advertise 16384 fake addresses from 4096 different `/16` subnets in repeated discovery messages — a one-time, low-cost operation.
- The store remains locked until the fake addresses naturally become non-connectable (after 3 failed dial attempts each), but the attacker can continuously re-fill it.

---

### Recommendation

Change the eviction threshold from strictly-greater-than to greater-than-or-equal-to-4, or use `>= 2` to ensure at least one group is always evictable when the store is full:

```rust

```

Additionally, consider capping the number of addresses accepted per `/16` subnet during `add_addr` (before insertion), rather than relying solely on post-hoc eviction.

    fn check_purge(&mut self) -> Result<()> {
        if self.addr_manager.count() < ADDR_COUNT_LIMIT {
            return Ok(());
        }

        // Evicting invalid data in the peer store is a relatively rare operation
        // There are certain cleanup strategies here:
        // 1. First evict the nodes that have reached the eviction condition
        // 2. If the first step is unsuccessful, enter the network segment grouping mode
        //  2.1. Group current data according to network segment
        //  2.2. Sort according to the amount of data in the same network segment
        //  2.3. In the network segment with more than 4 peer, randomly evict 2 peer

        let now_ms = ckb_systemtime::unix_time_as_millis();
        let candidate_peers: Vec<_> = self
            .addr_manager
            .addrs_iter()
            .filter_map(|addr| {
                if !addr.is_connectable(now_ms) {
                    Some(addr.addr.clone())
                } else {
                    None
                }
            })
            .collect();

        for key in candidate_peers.iter() {
            self.addr_manager.remove(key);
        }

        if candidate_peers.is_empty() {
            let candidate_peers: Vec<_> = {
                let mut peers_by_network_group: HashMap<Group, Vec<_>> = HashMap::default();
                for addr in self.addr_manager.addrs_iter() {
                    peers_by_network_group
                        .entry((&addr.addr).into())
                        .or_default()
                        .push(addr);
                }
                let len = peers_by_network_group.len();
                let mut peers = peers_by_network_group
                    .drain()
                    .map(|(_, v)| v)
                    .collect::<Vec<Vec<_>>>();

                peers.sort_unstable_by_key(|k| std::cmp::Reverse(k.len()));

                peers
                    .into_iter()
                    .take(len / 2)
                    .flat_map(move |addrs| {
                        if addrs.len() > 4 {
                            Some(
                                addrs
                                    .iter()
                                    .choose_multiple(&mut rand::thread_rng(), 2)
                                    .into_iter()
                                    .map(|addr| addr.addr.clone())
                                    .collect::<Vec<Multiaddr>>(),
                            )
                        } else {
                            None
                        }
                    })
                    .flatten()
                    .collect()
            };

            for key in candidate_peers.iter() {
                self.addr_manager.remove(key);
            }

            if candidate_peers.is_empty() {
                return Err(PeerStoreError::EvictionFailed.into());
            }
        }
        Ok(())
    }

        // do not remove addr tried in last minute
