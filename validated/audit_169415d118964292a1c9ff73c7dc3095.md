### Title
Raw Exception Messages Exposed in HTTP API Response — (File: `governance/xc_admin/packages/proposer_server/src/index.ts`)

### Summary
The `/api/propose` endpoint in the `proposer_server` catches exceptions and returns `error.message` verbatim in the HTTP 500 response body. Any caller who can reach the endpoint and supply input that triggers an exception receives the raw internal error string, which may include internal RPC URLs, Solana SDK internals, or library-level context.

### Finding Description
In `governance/xc_admin/packages/proposer_server/src/index.ts`, the single route handler wraps all logic in a `try/catch` and, on any `Error` subclass, serialises the raw `.message` property directly into the JSON response body:

```typescript
} catch (error) {
    if (error instanceof Error) {
      res.status(500).json(error.message);   // raw internal message returned
    } else {
      res.status(500).json("An unknown error occurred");
    }
  }
``` [1](#0-0) 

The handler processes attacker-supplied JSON fields before the error can occur. For example, `new PublicKey(ix.programId)` and `new PublicKey(key.pubkey)` throw `Error` objects whose `.message` strings are returned verbatim when the supplied value is not a valid base-58 key. [2](#0-1) 

Further into the same handler, `vault.proposeInstructions(...)` makes live RPC calls to Solana. Any network-level or SDK-level failure (wrong cluster, RPC node error, insufficient funds, etc.) also propagates its `.message` to the caller. [3](#0-2) 

The server registers no authentication middleware and enables permissive CORS (`app.use(cors())`), so any origin can POST to it. [4](#0-3) 

A secondary, lower-severity instance exists in the Hermes REST layer: `RestError::RpcConnectionError { message }` is constructed with `format!("RPC connection error: {e}")` — where `{e}` is the raw `anyhow` error from an internal metadata service call — and that string is returned directly as the HTTP body. [5](#0-4) [6](#0-5) 

### Impact Explanation
An attacker who can reach the `proposer_server` endpoint learns:
- Internal Solana RPC node URLs (embedded in connection-failure messages from `@solana/web3.js` or `@sqds/mesh`)
- Internal cluster topology (mainnet/devnet/testnet RPC strings stored in `RPC_URLS`)
- SDK-level error context that reveals library versions, call paths, or account addresses involved in failed transactions

This information lowers the cost of follow-on attacks (e.g., targeting the specific RPC node, crafting transactions that exploit known SDK behaviour, or enumerating valid vs. invalid governance accounts).

### Likelihood Explanation
The entry path requires only an HTTP POST with a JSON body — no credentials, no on-chain transaction, no privileged role. The CORS wildcard means a browser-based attacker can reach it cross-origin. Triggering an error is trivial: supply any string that is not a valid base-58 public key for `programId` or any key in `keys`, and the handler immediately throws and returns the SDK error message.

### Recommendation
- Replace `res.status(500).json(error.message)` with a generic message (e.g., `"Internal server error"`).
- Log the full error server-side only (already partially done via the existing logger pattern elsewhere in the repo).
- Add authentication/network-level access control to the `proposer_server` so only authorised governance operators can reach `/api/propose`.
- For the Hermes `RpcConnectionError` variant, replace `format!("RPC connection error: {e}")` with a static string and log `{e}` at `warn` level only (the `tracing::warn!` call already exists on the line above — the formatted string should not be forwarded to the client).

### Proof of Concept
```
POST /api/propose HTTP/1.1
Host: <proposer-server-host>:4000
Content-Type: application/json

{
  "cluster": "mainnet-beta",
  "instructions": [
    {
      "programId": "NOT_A_VALID_PUBKEY",
      "data": [],
      "keys": []
    }
  ]
}
```

Response (HTTP 500):
```json
"Invalid public key input"
```

To obtain RPC-level internals, supply a structurally valid instruction set that causes `vault.proposeInstructions` to fail (e.g., point the server at a non-existent cluster by manipulating the `cluster` field to a value that resolves to an unreachable RPC URL). The resulting `FetchError` or `SendTransactionError` message — including the RPC endpoint URL — is returned verbatim. [7](#0-6)

      wallet,
