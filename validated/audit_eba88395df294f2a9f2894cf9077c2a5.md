### Title
IPv4-mapped IPv6 Ban Evasion: `ip_to_network` Preserves Version, Allowing Banned IPv4 Peers to Reconnect via `::ffff:x.x.x.x` — (`network/src/peer_store/ban_list.rs`, `network/src/peer_store/types.rs`)

---

### Summary

`ip_to_network` converts `IpAddr` to `IpNetwork` without normalizing IPv4-mapped IPv6 addresses. As a result, banning `1.2.3.4` stores `IpNetwork::V4(1.2.3.4/32)`, but a reconnection from `::ffff:1.2.3.4` (IPv4-mapped IPv6, which is what a dual-stack OS socket presents for the same IPv4 peer) is checked against `IpNetwork::V6(::ffff:1.2.3.4/128)`. The `ipnetwork` crate's `contains` returns `false` for version mismatches, so the ban check passes and the connection is accepted.

---

### Finding Description

**`ip_to_network` — no normalization:** [1](#0-0) 

`IpAddr::V4(1.2.3.4)` → `IpNetwork::V4(1.2.3.4/32)`; `IpAddr::V6(::ffff:1.2.3.4)` → `IpNetwork::V6(::ffff:1.2.3.4/128)`. No cross-version mapping is performed.

**`is_ip_banned_until` — two-stage check, both fail for version mismatch:** [2](#0-1) 

- Stage 1 (line 49–53): `inner.get(&ip_to_network(::ffff:1.2.3.4))` → key is `IpNetwork::V6(::ffff:1.2.3.4/128)`, ban list holds `IpNetwork::V4(1.2.3.4/32)` → **no match**.
- Stage 2 (line 56–58): `IpNetwork::V4(1.2.3.4/32).contains(IpAddr::V6(::ffff:1.2.3.4))` → the `ipnetwork` crate returns `false` for any `(V4 network, V6 addr)` pair → **no match**.

Result: `is_ip_banned_until` returns `false`.

**`accept_peer` — ban check gates connection admission:** [3](#0-2) 

`is_addr_banned` returns `false` → `PeerError::Banned` is never raised → the connection is accepted.

**`ban_addr` — stores the exact IP version of the original connection:** [4](#0-3) 

If the original misbehaving peer connected via an IPv4 socket (or was banned via RPC `set_ban("1.2.3.4", "insert")`), the stored key is `IpNetwork::V4(1.2.3.4/32)`.

**`network_group.rs` — IPv4-mapped IPv6 collapses to the same eviction group:** [5](#0-4) 

`ipv6.to_ipv4()` maps `::ffff:1.2.3.4` back to `1.2.3.4`, so the reconnecting peer lands in `Group::IP4([1, 2])` — the same group as the banned peer — creating an eviction-grouping inconsistency on top of the ban bypass.

---

### Impact Explanation

A peer banned for misbehavior (e.g., sending invalid blocks/headers, triggering automatic ban via `ban_session`) can immediately reconnect from the same IPv4 address if the node's listening socket is dual-stack (the default on Linux with `IPV6_V6ONLY=0`). The OS presents the reconnection as `::ffff:1.2.3.4`, which passes the ban check. The ban mechanism — the primary tool for ejecting and excluding misbehaving P2P peers — is silently ineffective in this configuration.

---

### Likelihood Explanation

Dual-stack sockets are the default on Linux. Any node listening on `0.0.0.0` or `::` with `IPV6_V6ONLY=0` (the common case) will receive IPv4 connections as IPv4-mapped IPv6 addresses. The mismatch between how `ban_addr` stores the key and how `is_ip_banned_until` looks it up is deterministic and requires no special attacker capability beyond reconnecting from the same IP.

---

### Recommendation

Normalize IPv4-mapped IPv6 addresses to their IPv4 form at the point of entry. In `ip_to_network` (or at the call sites in `ban_addr` and `is_ip_banned_until`), apply `to_ipv4()` before constructing the `IpNetwork`:

```rust
pub fn ip_to_network(ip: IpAddr) -> IpNetwork {
    let ip = match ip {
        IpAddr::V6(v6) => v6.to_ipv4().map(IpAddr::V4).unwrap_or(ip),
        v4 => v4,
    };
    match ip {
        IpAddr::V4(ipv4) => IpNetwork::V4(ipv4.into()),
        IpAddr::V6(ipv6) => IpNetwork::V6(ipv6.into()),
    }
}
```

This ensures `::ffff:1.2.3.4` and `1.2.3.4` map to the same `IpNetwork::V4(1.2.3.4/32)` key in both `ban_addr` and `is_ip_banned_until`, closing the bypass in both directions.

### Proof of Concept

```rust
use std::net::{IpAddr, Ipv4Addr, Ipv6Addr};
use ipnetwork::IpNetwork;

fn ip_to_network(ip: IpAddr) -> IpNetwork {
    match ip {
        IpAddr::V4(ipv4) => IpNetwork::V4(ipv4.into()),
        IpAddr::V6(ipv6) => IpNetwork::V6(ipv6.into()),
    }
}

fn main() {
    let banned_ipv4: IpAddr = IpAddr::V4(Ipv4Addr::new(1, 2, 3, 4));
    let ban_network = ip_to_network(banned_ipv4); // IpNetwork::V4(1.2.3.4/32)

    // Attacker reconnects via dual-stack socket → OS presents ::ffff:1.2.3.4
    let reconnect_ip: IpAddr = IpAddr::V6(Ipv6Addr::new(0, 0, 0, 0, 0, 0xffff, 0x0102, 0x0304));
    let reconnect_network = ip_to_network(reconnect_ip); // IpNetwork::V6(::ffff:1.2.3.4/128)

    // Stage 1: direct key lookup — different keys, no match
    assert_ne!(ban_network, reconnect_network);

    // Stage 2: contains check — version mismatch, returns false
    assert!(!ban_network.contains(reconnect_ip));

    // Result: is_ip_banned_until returns false → ban bypassed
    println!("Ban bypassed: reconnect_ip not detected as banned");
}
```
