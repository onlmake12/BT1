### Title
Silent No-Op for `UpgradeContract` Governance Action Permanently Consumes Sequence Number Without Upgrading — (`File: target_chains/stylus/contracts/pyth-receiver/src/governance.rs`)

---

### Summary

The Stylus Pyth receiver's `execute_governance_instruction_internal` silently accepts a valid `UpgradeContract` governance VAA, advances the governance sequence number, and returns `Ok(())` — without performing any upgrade. This is a direct analog to the reported `BasePongHandler` bug: a governance instruction handler that silently drops a critical action instead of executing it or reverting.

---

### Finding Description

In `target_chains/stylus/contracts/pyth-receiver/src/governance.rs`, the match arm for `GovernancePayload::UpgradeContract` is an empty block:

```rust
GovernancePayload::UpgradeContract(_payload) => {}
``` [1](#0-0) 

The function `execute_governance_instruction_internal` first calls `verify_governance_vm`, which validates the VAA and **permanently advances the governance sequence number** before the match dispatch: [2](#0-1) 

The `upgrade_contract` helper exists in the same file but is never invoked from the match arm — it only panics with `unimplemented!`: [3](#0-2) 

The `GovernanceAction` enum in `governance_structs.rs` correctly maps byte `0` to `UpgradeContract`, so the action is parsed and dispatched — it just silently disappears: [4](#0-3) 

By contrast, every other Pyth chain implementation either executes the upgrade or explicitly reverts. The EVM contract calls `upgradeUpgradableContract`: [5](#0-4) 

The Fuel contract reverts with `revert(0u64)`: [6](#0-5) 

---

### Impact Explanation

When Pyth governance issues an `UpgradeContract` VAA targeting the Stylus receiver (or with `target_chain_id = 0` for all chains):

1. The VAA passes all signature and source verification checks.
2. The governance sequence number is permanently advanced.
3. The match arm executes nothing.
4. The function returns `Ok(())` — success.

The VAA is now consumed and can never be replayed (sequence guard prevents reuse). The Stylus receiver **cannot be upgraded via governance**. Any critical security patch delivered as an `UpgradeContract` VAA will be silently discarded, leaving the contract permanently stuck on the current (potentially vulnerable) implementation.

---

### Likelihood Explanation

The `executeGovernanceInstruction` entry point is publicly callable — any unprivileged relayer can submit a valid governance VAA. Once Pyth governance issues an `UpgradeContract` VAA (e.g., to patch a vulnerability), any actor who observes it on-chain can relay it to the Stylus receiver, consuming the sequence number before the legitimate upgrade process completes. Even without a malicious front-runner, the governance team itself would trigger the silent failure by submitting the VAA normally.

---

### Recommendation

Replace the empty match arm with either:

1. A call to `self.upgrade_contract(payload.new_implementation)` once the Stylus upgrade mechanism is implemented, or
2. An explicit error return to prevent silent consumption of the sequence number:

```rust
GovernancePayload::UpgradeContract(_payload) => {
    return Err(PythReceiverError::UpgradeNotImplemented);
}
```

This mirrors the Fuel contract's explicit revert and prevents the governance sequence from being permanently advanced without effect.

---

### Proof of Concept

1. Obtain or construct a valid Wormhole-signed governance VAA with:
   - Emitter matching the Pyth governance data source
   - Sequence number > `last_executed_governance_sequence`
   - Payload: `PTGM | module=1 | action=0 (UpgradeContract) | target_chain_id=<stylus_chain_id> | <32-byte implementation hash>`
2. Call `executeGovernanceInstruction(vaa_bytes)` on the deployed Stylus receiver.
3. Observe: transaction succeeds, `last_executed_governance_sequence` is incremented, but no upgrade occurs.
4. Attempt to replay the same VAA — it reverts with `OldGovernanceMessage`, confirming the sequence was consumed.
5. The contract remains on the old implementation with no path to upgrade via this VAA. [7](#0-6)

    pub fn execute_governance_instruction_internal(
        &mut self,
        data: Vec<u8>,
    ) -> Result<(), PythReceiverError> {
        let wormhole: IWormholeContract = IWormholeContract::new(self.wormhole.get());
        let config = Call::new();

        wormhole
            .parse_and_verify_vm(config, Vec::from(data.clone()))
            .map_err(|_| PythReceiverError::InvalidWormholeMessage)?;

        let vm = Vaa::read(&mut Vec::from(data.clone()).as_slice())
            .map_err(|_| PythReceiverError::InvalidVaa)?;

        verify_governance_vm(self, vm.clone())?;

        let instruction = governance_structs::parse_instruction(vm.body.payload.to_vec())
            .map_err(|_| PythReceiverError::InvalidGovernanceMessage)?;

        let chain_id_config = Call::new();

        let wormhole_id = wormhole
            .chain_id(chain_id_config)
            .map_err(|_| PythReceiverError::WormholeUninitialized)?;

        if instruction.target_chain_id != 0 && instruction.target_chain_id != wormhole_id {
            return Err(PythReceiverError::InvalidGovernanceTarget);
        }

        match instruction.payload {
            GovernancePayload::SetFee(payload) => {
                self.set_fee(payload.value, payload.expo);
            }
            GovernancePayload::SetDataSources(payload) => {
                set_data_sources(self, payload.sources);
            }
            GovernancePayload::SetWormholeAddress(payload) => {
                self.set_wormhole_address(payload.address, data.clone())?;
            }
            GovernancePayload::RequestGovernanceDataSourceTransfer(_) => {
                return Err(PythReceiverError::InvalidGovernanceMessage);
            }
            GovernancePayload::AuthorizeGovernanceDataSourceTransfer(payload) => {
                self.authorize_governance_transfer(payload.claim_vaa)?;
            }
            GovernancePayload::UpgradeContract(_payload) => {}
            GovernancePayload::SetValidPeriod(payload) => {
                self.set_valid_period(payload.valid_time_period_seconds);
            }
            GovernancePayload::SetTransactionFee(payload) => {
                self.set_transaction_fee(payload.value, payload.expo);
            }
            GovernancePayload::WithdrawFee(payload) => {
                self.withdraw_fee(payload.value, payload.expo, payload.target_address)?;
            }
        }

        Ok(())
    }
