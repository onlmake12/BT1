### Title
Multi-Peer Reconstruction Amplification via Unbounded `peers_map` in `pending_compact_blocks` — (`sync/src/relayer/block_transactions_process.rs`)

---

### Summary

`BlockTransactionsProcess::execute` processes each peer's `BlockTransactions` message independently, calling `reconstruct_block` (which includes `tx_pool.fetch_txs`) and emitting a `GetBlockTransactions` re-request **once per peer** with no cross-peer deduplication. Because `PendingCompactBlockMap`'s inner `peers_map` is an unbounded `HashMap<PeerIndex, …>`, M attacker-controlled peers registered for the same compact block hash each trigger a full, independent reconstruction attempt, multiplying relay processing cost by M per round-trip.

---

### Finding Description

**Data structure — no cap on `peers_map`:**

`PendingCompactBlockMap` is defined as:

```
HashMap<Byte32, (CompactBlock, HashMap<PeerIndex, (Vec<u32>, Vec<u32>)>, u64)>
``` [1](#0-0) 

The inner `HashMap<PeerIndex, …>` has no size limit. `missing_or_collided_post_process` inserts each new peer unconditionally:

```rust
.insert(peer, (missing_transactions.clone(), missing_uncles.clone()));
``` [2](#0-1) 

The only guard in `contextual_check` prevents the **same** peer from being inserted twice; it does not cap the total number of distinct peers per block:

```rust
if pending_compact_blocks
    .get(&block_hash)
    .map(|(_, peers_map, _)| peers_map.contains_key(&peer))
    .unwrap_or(false)
{ return StatusCode::CompactBlockIsAlreadyPending… }
``` [3](#0-2) 

**Per-peer reconstruction with no deduplication:**

In `execute`, the `pending_compact_blocks` `tokio::sync::Mutex` is acquired and its guard is held for the entire `if let` block (lines 65–187). Inside that block, `reconstruct_block` is awaited while the guard is live:

```rust
if let Entry::Occupied(mut pending) = shared
    .state()
    .pending_compact_blocks()
    .await                          // MutexGuard acquired, held until end of if-let
    .entry(block_hash.clone())
{
    …
    let ret = self.relayer.reconstruct_block(…).await;  // guard still held
    …
    async_send_message_to(&self.nc, self.peer, &message).await;  // guard still held
``` [4](#0-3) [5](#0-4) 

`reconstruct_block` calls `tx_pool.fetch_txs` for every remaining short ID:

```rust
if !short_ids_set.is_empty() {
    let tx_pool = self.shared.shared().tx_pool_controller();
    let fetch_txs = tx_pool.fetch_txs(short_ids_set).await;
``` [6](#0-5) 

There is **no check** such as "if another peer already triggered reconstruction for this block hash, skip." Every peer entry in `peers_map` independently reaches `reconstruct_block` and the subsequent `GetBlockTransactions` send.

---

### Impact Explanation

With M attacker peers registered in `peers_map` for one block hash:

| Per round-trip | Cost |
|---|---|
| `tx_pool.fetch_txs` calls | M (sequential, each holding the global `pending_compact_blocks` mutex) |
| `GetBlockTransactions` messages sent | M (one per peer) |
| `pending_compact_blocks` mutex hold time | O(M × per-reconstruction latency) |

Because the mutex is held across every `reconstruct_block` await, **all** compact block processing on the node is serialized behind M sequential reconstructions. The attacker sustains the cycle indefinitely by always replying with a partial `BlockTransactions` (still missing one tx), keeping the block in `pending_compact_blocks` and re-triggering the loop.

---

### Likelihood Explanation

The attack requires only M standard P2P relay peers connecting to the victim and sending valid (but incomplete) compact block relay messages — no PoW, no keys, no privileged access. The cost per round is O(M) small messages; the victim pays O(M) tx-pool queries and O(M) mutex-serialized reconstruction attempts. This is concretely reachable on mainnet.

---

### Recommendation

1. **Cap `peers_map` per block hash** — reject a new peer's compact block if `peers_map.len() >= MAX_COMPACT_BLOCK_PEERS` (e.g., 3–5).
2. **Deduplicate reconstruction work** — once one peer has triggered `reconstruct_block` for a given block hash and the result is `Missing`, record a "reconstruction in progress" flag; subsequent peers' `BlockTransactions` for the same hash should only update their own `peers_map` entry and re-send `GetBlockTransactions` without calling `reconstruct_block` again.
3. **Release the mutex before awaiting** — clone the needed data out of `pending_compact_blocks`, drop the guard, then call `reconstruct_block`. This prevents the global mutex from being held across async I/O.

---

### Proof of Concept

```
1. Attacker connects M peers (P1…PM) to victim node.
2. Each Pi sends CompactBlock(H) with short_ids=[tx_A, tx_B], prefilled=[],
   where tx_A and tx_B are not in victim's tx pool.
   → missing_or_collided_post_process inserts Pi into peers_map[H]
     with missing_transactions=[0,1].
   → GetBlockTransactions([0,1]) sent to Pi.

3. Each Pi replies with BlockTransactions(H, transactions=[tx_A])
   (deliberately omitting tx_B — still one tx missing).

4. Victim processes P1's message:
   - acquires pending_compact_blocks mutex
   - calls reconstruct_block → tx_pool.fetch_txs({tx_B}) → still missing
   - sends GetBlockTransactions([1]) to P1
   - releases mutex

5. Victim processes P2's message (same block hash, same partial response):
   - acquires mutex
   - calls reconstruct_block → tx_pool.fetch_txs({tx_B}) again
   - sends GetBlockTransactions([1]) to P2
   - releases mutex

   … repeated for P3…PM.

6. Each Pi receives GetBlockTransactions and replies with tx_A again → goto 4.

Assert: M tx_pool.fetch_txs calls and M GetBlockTransactions messages per round.
The pending_compact_blocks mutex is held for O(M × fetch_txs_latency) per round,
blocking all other compact block relay on the node.
``` [7](#0-6) [8](#0-7) [1](#0-0)

/// request missing txs and uncles from peer
async fn missing_or_collided_post_process(
    compact_block: CompactBlock,
    block_hash: Byte32,
    shared: &SyncShared,
    nc: Arc<dyn CKBProtocolContext + Sync>,
    missing_transactions: Vec<u32>,
    missing_uncles: Vec<u32>,
    peer: PeerIndex,
) {
    shared
        .state()
        .pending_compact_blocks()
        .await
        .entry(block_hash.clone())
        .or_insert_with(|| (compact_block, HashMap::default(), unix_time_as_millis()))
        .1
        .insert(peer, (missing_transactions.clone(), missing_uncles.clone()));

    let content = packed::GetBlockTransactions::new_builder()
        .block_hash(block_hash)
        .indexes(missing_transactions.as_slice())
        .uncle_indexes(missing_uncles.as_slice())
        .build();
    let message = packed::RelayMessage::new_builder().set(content).build();
    shared.shared().async_handle().spawn(async move {
        let sending = async_send_message_to(&nc, peer, &message).await;
        if !sending.is_ok() {
            ckb_logger::warn_target!(
                crate::LOG_TARGET_RELAY,
                "ignore the sending message error, error: {}",
                sending
            );
        }
    });
}

            if let Entry::Occupied(mut value) = peers_map.entry(self.peer) {
                let (expected_transaction_indexes, expected_uncle_indexes) = value.get_mut();
                ckb_logger::info!(
                    "relayer receive BLOCKTXN of {}, peer: {}",
                    block_hash,
                    self.peer
                );

                attempt!(BlockTransactionsVerifier::verify(
                    compact_block,
                    expected_transaction_indexes,
                    &received_transactions,
                ));
                attempt!(BlockUnclesVerifier::verify(
                    compact_block,
                    expected_uncle_indexes,
                    &received_uncles,
                ));

                let ret = self
                    .relayer
                    .reconstruct_block(
                        &active_chain,
                        compact_block,
                        received_transactions,
                        expected_uncle_indexes,
                        &received_uncles,
                    )
                    .await;

                // Request proposal
                {
                    let proposals: Vec<_> = received_uncles
                        .into_iter()
                        .flat_map(|u| u.data().proposals().into_iter())
                        .collect();
                    self.relayer.request_proposal_txs(
                        &self.nc,
                        self.peer,
                        (
                            compact_block.header().into_view().number(),
                            block_hash.clone(),
                        )
                            .into(),
                        proposals,
                    );
                }

                match ret {
                    ReconstructionResult::Block(block) => {
                        pending.remove();
                        self.relayer
                            .accept_block(self.nc, self.peer, block, "BlockTransactions");
                        return Status::ok();
                    }
                    ReconstructionResult::Missing(transactions, uncles) => {
                        // We need to get all transactions and uncles that do not exist locally
                        // at once to restore a block
                        //
                        // Under normal circumstances, one request is enough, when the chain occurs fork,
                        // the transaction pool may drop some transactions due to double spend check, at
                        // this time, the previously issued request to obtain transactions may not meet
                        // the needs of a one-time construction, we need to send another complete request
                        // to do so. That is, the current miss + the miss of the previous request are
                        // combined and requested once. This is a small probability event
                        missing_transactions = transactions
                            .into_iter()
                            .map(|i| i as u32)
                            .chain(expected_transaction_indexes.iter().copied())
                            .collect();
                        missing_uncles = uncles
                            .into_iter()
                            .map(|i| i as u32)
                            .chain(expected_uncle_indexes.iter().copied())
                            .collect();

                        missing_transactions.sort_unstable();
                        missing_uncles.sort_unstable();
                    }
                    ReconstructionResult::Collided => {
                        missing_transactions = compact_block
                            .short_id_indexes()
                            .into_iter()
                            .map(|i| i as u32)
                            .collect();
                        collision = true;
                        missing_uncles = vec![];
                    }
                    ReconstructionResult::Error(status) => {
                        return status;
                    }
                }

                assert!(!missing_transactions.is_empty() || !missing_uncles.is_empty());

                let content = packed::GetBlockTransactions::new_builder()
                    .block_hash(block_hash.clone())
                    .indexes(missing_transactions.as_slice())
                    .uncle_indexes(missing_uncles.as_slice())
                    .build();
                let message = packed::RelayMessage::new_builder().set(content).build();

                let _ignore = async_send_message_to(&self.nc, self.peer, &message).await;

                let _ignore_prev_value =
                    mem::replace(expected_transaction_indexes, missing_transactions);
                let _ignore_prev_value = mem::replace(expected_uncle_indexes, missing_uncles);

                if collision {
                    return StatusCode::CompactBlockMeetsShortIdsCollision.with_context(block_hash);
                } else {
                    return StatusCode::CompactBlockRequiresFreshTransactions
                        .with_context(block_hash);
                }
            }
        }
