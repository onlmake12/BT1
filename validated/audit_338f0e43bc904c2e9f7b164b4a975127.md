### Title
Stale Storage Pointer After `clearRequest` Delivers Callback to `address(0)` for Overflow Requests — (File: target_chains/ethereum/contracts/contracts/echo/Echo.sol)

---

### Summary

In `Echo.executeCallback`, a `Request storage req` pointer is obtained before `clearRequest` is called. For requests stored in the overflow mapping (`_state.requestsOverflow`), `clearRequest` uses `delete` which zeros **all** struct fields including `requester` and `callbackGasLimit`. The `req` pointer then reads `address(0)` and `0` respectively. The subsequent external call delivers the callback to `address(0)` instead of the actual requester, causing the requester to permanently lose their callback while their fee is credited to the provider.

---

### Finding Description

In `executeCallback` (Echo.sol lines 105–202), the execution order is:

1. `req = findActiveRequest(sequenceNumber)` — returns a `storage` pointer
2. For overflow requests, `req` points to `_state.requestsOverflow[key]`
3. `_state.providers[providerToCredit].accruedFeesInWei += ...` — fee credited from `req.fee`
4. `clearRequest(sequenceNumber)` — for overflow requests, calls `delete _state.requestsOverflow[key]`, zeroing **all** fields of the struct
5. `req.requester` is now `address(0)`, `req.callbackGasLimit` is now `0`
6. `IEchoConsumer(req.requester)._echoCallback{gas: req.callbackGasLimit}(...)` is called with `address(0)` and `0` gas [1](#0-0) [2](#0-1) [3](#0-2) 

The `clearRequest` function for the overflow path uses `delete`, which zeros all struct fields: [4](#0-3) 

`findRequest` returns a storage pointer to `_state.requestsOverflow[key]` for overflow requests: [5](#0-4) 

A request enters overflow when `allocRequest` finds the main slot already occupied and copies the existing request to `_state.requestsOverflow`: [6](#0-5) 

After `delete _state.requestsOverflow[key]`, the `req` storage pointer reads all zeros. The call `IEchoConsumer(address(0))._echoCallback{gas: 0}(...)` targets an address with no code. In Solidity, a `try` call to a codeless address succeeds with empty return data, so the `try` block succeeds and `emitPriceUpdate` is called — but the actual requester's contract is never invoked.

---

### Impact Explanation

- The requester paid a fee (stored in `req.fee`) for a price-update callback. That fee is credited to the provider at line 161–162 before `clearRequest`.
- After `clearRequest` zeros the overflow slot, the callback is silently delivered to `address(0)` instead of the requester's contract.
- The requester's application never receives the price update it paid for. Funds are non-refundable.
- The `PriceUpdateExecuted` event is emitted with the correct data, making the failure invisible on-chain to observers.

---

### Likelihood Explanation

A request enters the overflow mapping whenever a later request hashes to the same `shortKey` slot. `shortKey = uint8(keccak256(sequenceNumber)[0] & NUM_REQUESTS_MASK)`. With `NUM_REQUESTS = 32` slots, on average every 32nd sequential sequence number collides with a given slot.

An unprivileged attacker can:
1. Observe a victim's pending request at sequence number N (slot X).
2. Compute the next sequence number M that hashes to slot X (deterministic, ~32 requests away).
3. Make requests until sequence number M is assigned to them, pushing the victim's request to overflow.
4. Wait for the exclusivity period (default 15 s) to expire.
5. Call `executeCallback` for sequence number N with valid price data obtained from Pyth's public API.

The attack cost is ~32 request fees plus gas. No privileged access is required. `executeCallback` is permissionless after the exclusivity period. [7](#0-6) 

---

### Recommendation

Save `req.requester` and `req.callbackGasLimit` to **memory** variables before calling `clearRequest`, then use the memory copies for the external call:

```solidity
address requester = req.requester;
uint32 callbackGasLimit = req.callbackGasLimit;

clearRequest(sequenceNumber);

// ... firstUnfulfilledSeq update ...

try IEchoConsumer(requester)._echoCallback{gas: callbackGasLimit}(sequenceNumber, priceFeeds) {
    emitPriceUpdate(sequenceNumber, priceIds, priceFeeds);
} catch Error(string memory reason) {
    emit PriceUpdateCallbackFailed(sequenceNumber, providerToCredit, priceIds, requester, reason);
} catch {
    emit PriceUpdateCallbackFailed(sequenceNumber, providerToCredit, priceIds, requester, "low-level error (possibly out of gas)");
}
```

This mirrors the pattern already used in `Entropy.revealWithCallback` (the non-gasLimit path), which explicitly copies `req.requester` to a local variable before calling `clearRequest`: [8](#0-7)

### Proof of Concept

```
Setup:
  NUM_REQUESTS = 32, exclusivityPeriodSeconds = 15

Step 1: Victim calls requestPriceUpdatesWithCallback(...)
        → assigned sequenceNumber = N, hashes to slot X

Step 2: Attacker computes M = next sequence number hashing to slot X
        Attacker makes (M - currentSeq) requests to reach sequence M
        → allocRequest for M finds slot X occupied by N
        → copies N to _state.requestsOverflow[keccak256(N)]
        → M takes the main slot

Step 3: block.timestamp >= req.publishTime + 15 (exclusivity expired)

Step 4: Attacker calls executeCallback(anyProvider, N, validUpdateData, priceIds)
        → findActiveRequest(N) returns storage pointer to _state.requestsOverflow[keccak256(N)]
        → provider fee credited from req.fee (victim's funds)
        → clearRequest(N) → delete _state.requestsOverflow[keccak256(N)] → all fields zeroed
        → req.requester = address(0), req.callbackGasLimit = 0
        → IEchoConsumer(address(0))._echoCallback{gas: 0}(...) → succeeds (no code at address(0))
        → PriceUpdateExecuted emitted

Result: Victim's contract never receives _echoCallback.
        Victim's fee is permanently credited to the provider.
        No revert, no error event — failure is silent.
```

    function clearRequest(uint64 sequenceNumber) internal {
        (bytes32 key, uint8 shortKey) = requestKey(sequenceNumber);

        Request storage req = _state.requests[shortKey];
        if (req.sequenceNumber == sequenceNumber) {
            req.sequenceNumber = 0;
