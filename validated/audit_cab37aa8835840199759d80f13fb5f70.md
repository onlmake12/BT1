### Title
Missing IP address reachability check in `ConnectionRequestDelivered` listen_addrs allows attacker-directed TCP probing of internal network addresses — (`network/src/protocols/hole_punching/component/connection_request_delivered.rs`)

### Summary
The `DeliverdContent::try_from` function validates `listen_addrs` only for peer ID consistency, not for IP address reachability. A connected peer can craft a `ConnectionRequestDelivered` message whose `listen_addrs` contain loopback, link-local, or RFC-1918 addresses, causing the victim node to perform repeated TCP connection attempts to those addresses via `try_nat_traversal`.

### Finding Description
In `connection_request_delivered.rs`, `DeliverdContent::try_from` (lines 51–71) parses each `listen_addr` from the message and only rejects it if a `/p2p/` component is present with a peer ID that does not match `content.to`. There is no call to `is_reachable()` or any equivalent guard. The parsed addresses flow unchanged into `try_nat_traversal` (lines 237–257), which further filters only by transport type (`Tcp`) and IP protocol presence (`Ip4`/`Ip6`), then passes the address to the `try_nat_traversal` async function in `mod.rs`. That function converts the multiaddr to a `SocketAddr` and enters a 30-second retry loop making TCP `connect()` calls every ~200 ms with no IP address validation.

Every other address-handling path in the codebase (`identify`, `discovery`, `NetworkState::from_config`) calls `is_reachable()` before acting on a remote-supplied address. The hole-punching path is the sole exception.

### Impact Explanation
The victim node performs TCP `connect()` attempts to attacker-specified addresses — including `127.0.0.1`, `192.168.x.x`, `10.x.x.x`, `172.16.x.x`, `169.254.x.x` — for up to 30 seconds per crafted message, with retries every ~200 ms. This enables:
- **Internal network port scanning**: the attacker learns which ports are open on the victim's localhost or LAN by timing connection outcomes.
- **SSRF-adjacent behavior**: the victim's TCP stack initiates connections to internal services on behalf of the attacker.
- **Repeated exploitation**: the victim re-populates `inflight_requests` on every `notify` tick (every 5 minutes), so the attack can be repeated indefinitely.

### Likelihood Explanation
The attacker only needs to be a connected P2P peer (no privilege required). The `ConnectionRequest` gossip broadcast exposes the required `to` peer ID to all connected peers. Crafting the malformed `ConnectionRequestDelivered` message requires only knowledge of the victim's peer ID (public) and the observed `to` peer ID. No cryptographic material or special access is needed.

### Recommendation
Add an `is_reachable()` check in `DeliverdContent::try_from` when parsing each `listen_addr`, consistent with how `identify` and `discovery` handle remote-supplied addresses:

```rust
// After parsing addr, before Ok(addr):
if let Some(socket_addr) = multiaddr_to_socketaddr(&addr) {
    if !is_reachable(socket_addr.ip()) {
        return Err(StatusCode::InvalidListenAddrLen
            .with_context("non-routable address in listen_addrs"));
    }
}
```

Alternatively, apply the same filter inside the `try_nat_traversal` method before spawning the async task.

### Proof of Concept
1. Connect to the victim node as a normal P2P peer.
2. Observe a `ConnectionRequest` gossip message; extract `to_peer_id` and `from_peer_id` (victim).
3. Craft a `ConnectionRequestDelivered` molecule message:
   - `from` = victim's peer ID bytes
   - `to` = `to_peer_id` bytes
   - `route` = empty `BytesVec`
   - `sync_route` = empty `BytesVec`
   - `listen_addrs` = `[/ip4/127.0.0.1/tcp/8080/p2p/<to_peer_id>]`
4. Send the message to the victim on the `HolePunching` protocol channel.
5. Observe (e.g., via `tcpdump` on the victim) repeated TCP SYN packets to `127.0.0.1:8080` for ~30 seconds.

pub(crate) async fn try_nat_traversal(
    bind_addr: Option<SocketAddr>,
    addr: Multiaddr,
) -> Result<(TcpStream, Multiaddr), std::io::Error> {
    let net_addr = match multiaddr_to_socketaddr(&addr) {
        Some(addr) => addr,
        None => {
            debug!("Failed to convert multiaddr to socketaddr");
            return Err(std::io::ErrorKind::InvalidInput.into());
        }
    };

    // Use a fixed interval but add a small amount of randomness
    let base_retry_interval = Duration::from_millis(200);

    // total time
    let timeout_duration = Duration::from_secs(30);
    let start_time = Instant::now();
    let mut retry_count = 0u32;
    while start_time.elapsed() < timeout_duration {
        retry_count += 1;

        // Add a small amount of random jitter (±25ms) to avoid conflicts
        // caused by continuous precise synchronization
        let jitter = Duration::from_millis(rand::random::<u64>() % 50);
        let actual_interval = if rand::random::<bool>() {
            base_retry_interval + jitter
        } else {
            base_retry_interval.saturating_sub(jitter)
        };

        let socket = create_socket(bind_addr, net_addr)?;

        match runtime::timeout(
            std::time::Duration::from_millis(200),
            socket.connect(net_addr),
        )
        .await
        {
            Ok(Ok(stream)) => {
                // try get the stored error in the underlying socket
                // if the socket is not connected, it will return an error
                if let Err(err) = check_connection(&stream) {
                    debug!("Failed to connect to NAT(base check): {}", err);
                }
                return Ok((stream, addr));
            }
            Err(err) => {
                debug!("Failed to connect to NAT(timeout): {}", err);
            }
            Ok(Err(err)) => {
                if err.kind() == std::io::ErrorKind::AddrNotAvailable {
                    return Err(err);
                }
                debug!(
                    "Failed to connect to NAT(other error): {}, {}",
                    err.kind(),
                    err
                );
            }
        }
        runtime::delay_for(actual_interval).await;
    }

    debug!("Failed to connect to NAT after {} retries", retry_count);
    Err(std::io::ErrorKind::TimedOut.into())
}
