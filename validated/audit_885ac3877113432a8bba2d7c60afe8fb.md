### Title
`CapacityVerifier` Skips Global Capacity Overflow Check for Any Transaction Containing a DAO Input, Allowing Non-DAO Output Capacity Inflation - (File: `verification/src/transaction_verifier.rs`)

---

### Summary

`CapacityVerifier::verify()` unconditionally skips the `OutputsSumOverflow` check for the **entire transaction** whenever any single input cell uses the DAO type script. Because the DAO type script only enforces rules for DAO-tagged cells and not for co-mingled non-DAO outputs, an unprivileged transaction sender can craft a transaction that pairs a legitimate DAO prepare cell with extra non-DAO inputs and inflated non-DAO outputs, bypassing the node's capacity conservation check at the tx-pool admission layer.

---

### Finding Description

In `CapacityVerifier::verify()`, the guard condition is:

```rust
if !(self.resolved_transaction.is_cellbase() || self.valid_dao_withdraw_transaction()) {
    let inputs_sum = self.resolved_transaction.inputs_capacity()?;
    let outputs_sum = self.resolved_transaction.outputs_capacity()?;
    if inputs_sum < outputs_sum {
        return Err(TransactionError::OutputsSumOverflow { ... }.into());
    }
}
``` [1](#0-0) 

`valid_dao_withdraw_transaction()` uses `.any()` — it returns `true` the moment **any** resolved input carries the DAO type script:

```rust
fn valid_dao_withdraw_transaction(&self) -> bool {
    self.resolved_transaction
        .resolved_inputs
        .iter()
        .any(|cell_meta| cell_uses_dao_type_script(&cell_meta.cell_output, &self.dao_type_hash))
}
``` [2](#0-1) 

`cell_uses_dao_type_script` checks only `hash_type == Type` and `code_hash == dao_type_hash`: [3](#0-2) 

When the guard fires, the **entire** `OutputsSumOverflow` check is skipped — not just for the DAO cells, but for every output in the transaction. The inline comment states the assumption: *"DAO withdraw transaction is verified via the type script of DAO cells."* [4](#0-3) 

However, the on-chain DAO type script only verifies the correctness of the DAO withdrawal amount for cells that carry the DAO type script. It does not enforce that the sum of **all** outputs ≤ sum of **all** inputs + DAO interest. Non-DAO outputs in the same transaction are invisible to the DAO type script.

The `DaoCalculator::transaction_fee` function does compute the correct global bound:

```rust
pub fn transaction_fee(&self, rtx: &ResolvedTransaction) -> Result<Capacity, DaoError> {
    let maximum_withdraw = self.transaction_maximum_withdraw(rtx)?;
    rtx.transaction
        .outputs_capacity()
        .and_then(|y| maximum_withdraw.safe_sub(y))
        .map_err(Into::into)
}
``` [5](#0-4) 

`transaction_maximum_withdraw` sums DAO interest for DAO inputs and face-value capacity for non-DAO inputs: [6](#0-5) 

This check is wired into the contextual block verifier (`verification/contextual/src/contextual_block_verifier.rs`), **not** into the tx-pool admission path. The `CapacityVerifier` is the only capacity gate applied at tx-pool admission time, and it is bypassed for all DAO-touching transactions.

---

### Impact Explanation

An attacker who holds a valid DAO prepare cell can craft a transaction such as:

| Role | Cell | Capacity |
|---|---|---|
| Input 0 | DAO prepare cell | 100 CKB |
| Input 1 | Regular cell | 10 CKB |
| Output 0 | Regular cell (DAO withdrawal) | 105 CKB (legitimate interest) |
| Output 1 | Regular cell (attacker-controlled) | 100 CKB (**inflated by 90 CKB**) |

- Total inputs: 110 CKB; total outputs: 205 CKB.
- `CapacityVerifier` skips the check → transaction is admitted to the tx-pool.
- The DAO type script verifies only that Output 0 is a valid DAO withdrawal; it does not inspect Output 1.
- `DaoCalculator::transaction_fee` in the contextual block verifier catches the overflow and rejects any block that includes this transaction.

The primary confirmed impact is **tx-pool resource exhaustion / DoS**: the attacker can flood the tx-pool with capacity-inflating DAO transactions that pass admission but are permanently unmineable. Because the DAO prepare cell is not consumed until the transaction is committed to a block, the same UTXO can be reused to submit the same invalid transaction repeatedly after each eviction, sustaining the attack at low cost.

If `DaoCalculator::transaction_fee` were ever not enforced at block-verification time (e.g., due to a future refactor or a code path that skips contextual verification), the impact escalates to **CKB capacity minting out of thin air** — the direct analog of minting tokens with a worthless ERC-20.

---

### Likelihood Explanation

- Entry path: any unprivileged transaction sender who has previously deposited CKB into the NervosDAO (a public, permissionless operation) and advanced to the "prepare" phase.
- No privileged access, no key leakage, no majority hashpower required.
- The DAO prepare cell is not consumed by a tx-pool-only submission, so a single prepare cell can sustain a prolonged attack.
- Likelihood of tx-pool DoS: **Medium-High** (low barrier, repeatable).
- Likelihood of actual capacity minting (given current code): **Low** (blocked by contextual block verifier), but the structural gap makes it a latent high-severity risk.

---

### Recommendation

Replace the coarse transaction-level bypass with a per-cell accounting approach:

1. Compute `non_dao_inputs_sum` = sum of capacity of all inputs that do **not** use the DAO type script.
2. Compute `non_dao_outputs_sum` = sum of capacity of all outputs that do **not** use the DAO type script.
3. Enforce `non_dao_inputs_sum >= non_dao_outputs_sum` unconditionally.
4. Continue to delegate the DAO-cell capacity check to the DAO type script (or inline `DaoCalculator::transaction_fee` into the `CapacityVerifier` for DAO cells).

This mirrors the fix recommended in the external report: validate each "token type" independently rather than granting a blanket bypass when any one recognized type is present.

### Proof of Concept

**Attacker-controlled transaction structure** (pseudocode):

```
inputs:
  [0] DAO prepare cell  (capacity = 100 CKB, type = DAO, data = <deposit_block_number>)
  [1] Regular cell      (capacity = 10 CKB,  type = None)

outputs:
  [0] Regular cell      (capacity = 105 CKB, type = None)   ← legitimate DAO withdrawal
  [1] Regular cell      (capacity = 100 CKB, type = None)   ← 90 CKB inflated from thin air

witnesses:
  [0] WitnessArgs { input_type: <deposit_header_dep_index> }
  [1] <lock witness for regular cell>

header_deps: [deposit_block_hash, prepare_block_hash]
cell_deps:   [dao_script_dep, lock_script_dep]
```

**Verification trace at tx-pool admission:**

1. `CapacityVerifier::verify()` → `valid_dao_withdraw_transaction()` returns `true` (Input 0 has DAO type script) → `OutputsSumOverflow` check **skipped entirely**.
2. DAO type script executes for Input 0 → verifies Output 0 withdrawal amount is ≤ maximum withdraw → **passes** (Output 1 is invisible to the DAO script group).
3. Transaction enters tx-pool. ✓

**Verification trace at block inclusion:**

4. `DaoCalculator::transaction_fee` → `maximum_withdraw` = 105 + 10 = 115 CKB; `outputs_capacity` = 205 CKB → `safe_sub` overflows → **block rejected**. ✗

The transaction is permanently unmineable but occupies tx-pool slots, demonstrating the DoS vector. The structural root cause — the unconditional transaction-wide bypass in `CapacityVerifier` — is the direct analog of accepting any arbitrary ERC-20 token without a whitelist check.

    fn transaction_maximum_withdraw(
        &self,
        rtx: &ResolvedTransaction,
    ) -> Result<Capacity, DaoError> {
        let header_deps: HashSet<Byte32> = rtx.transaction.header_deps_iter().collect();
        rtx.resolved_inputs.iter().enumerate().try_fold(
            Capacity::zero(),
            |capacities, (i, cell_meta)| {
                let capacity: Result<Capacity, DaoError> = {
                    let output = &cell_meta.cell_output;
                    let is_dao_type_script = |type_script: Script| {
                        Into::<u8>::into(type_script.hash_type())
                            == Into::<u8>::into(ScriptHashType::Type)
                            && type_script.code_hash() == self.consensus.dao_type_hash()
                    };
                    let is_dao_output = output
                        .type_()
                        .to_opt()
                        .map(is_dao_type_script)
                        .unwrap_or(false);
                    if is_dao_output {
                        // A withdrawing DAO cell has 8 bytes of cell data storing the
                        // block number of the original deposit.
                        let deposited_block_number =
                            match self.data_loader.load_cell_data(cell_meta) {
                                Some(data) if data.len() == 8 => LittleEndian::read_u64(&data),
                                _ => 0,
                            };
                        if deposited_block_number > 0 {
                            let withdrawing_header_hash = cell_meta
                                .transaction_info
                                .as_ref()
                                .map(|info| &info.block_hash)
                                .filter(|hash| header_deps.contains(hash))
                                .ok_or(DaoError::InvalidOutPoint)?;
                            let deposit_header_hash = rtx
                                .transaction
                                .witnesses()
                                .get(i)
                                .ok_or(DaoError::InvalidOutPoint)
                                .and_then(|witness_data| {
                                    // dao contract stores header deps index as u64 in the input_type field of WitnessArgs
                                    let witness =
                                        WitnessArgs::from_slice(&Into::<Bytes>::into(witness_data))
                                            .map_err(|_| DaoError::InvalidDaoFormat)?;
                                    let header_deps_index_data: Option<Bytes> =
                                        witness.input_type().to_opt().map(|witness| witness.into());
                                    if header_deps_index_data.is_none()
                                        || header_deps_index_data.clone().map(|data| data.len())
                                            != Some(8)
                                    {
                                        return Err(DaoError::InvalidDaoFormat);
                                    }
                                    Ok(LittleEndian::read_u64(&header_deps_index_data.unwrap()))
                                })
                                .and_then(|header_dep_index| {
                                    rtx.transaction
                                        .header_deps()
                                        .get(header_dep_index as usize)
                                        .and_then(|hash| header_deps.get(&hash))
                                        .ok_or(DaoError::InvalidOutPoint)
                                })?;

                            let deposit_header = self
                                .data_loader
                                .get_header(deposit_header_hash)
                                .ok_or(DaoError::InvalidHeader)?;
                            if deposit_header.number() != deposited_block_number {
                                return Err(DaoError::InvalidOutPoint);
                            }
                            self.calculate_maximum_withdraw(
                                output,
                                Capacity::bytes(cell_meta.data_bytes as usize)?,
                                deposit_header_hash,
                                withdrawing_header_hash,
                            )
                        } else {
                            Ok(output.capacity().into())
                        }
                    } else {
                        Ok(output.capacity().into())
                    }
                };
                capacity.and_then(|c| c.safe_add(capacities).map_err(Into::into))
            },
        )
    }
