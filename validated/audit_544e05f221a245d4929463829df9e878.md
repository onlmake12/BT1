Audit Report

## Title
Unconditional `data.len() < 2` Guard in `LengthDelimitedCodecWithCompress::decode` Enables Low-Cost IP Ban via Single Malformed Frame — (`network/src/compress.rs`)

## Summary
The `data.len() < 2` check at `compress.rs:228` fires unconditionally regardless of `enable_compress`. Any peer that completes a secio handshake and sends a 5-byte TCP payload (`[0x00,0x00,0x00,0x01,0x00]`) causes `decode` to return `Err(io::ErrorKind::InvalidData)`, which the tentacle framework surfaces as `ServiceError::ProtocolError`, which `handle_error` in `network.rs` converts into an unconditional 300-second IP ban with no prior misbehavior scoring. An attacker with N distinct IPs can ban all N from the victim's peer store simultaneously, continuously degrading peer diversity and inbound connection capacity.

## Finding Description

**Step 1 — Unconditional codec guard**

`LengthDelimitedCodecWithCompress::decode` at `compress.rs:226-230` checks `data.len() < 2` without consulting `self.enable_compress`. A frame whose length-delimited payload is exactly 1 byte (flag byte only, no application payload) satisfies `max_frame_length` for every protocol but fails this guard, returning `Err(io::ErrorKind::InvalidData)`:

```rust
match self.length_delimited.decode(src)? {
    Some(mut data) => {
        if data.len() < 2 {
            return Err(io::ErrorKind::InvalidData.into());
        }
```

**Step 2 — Codec always installed for all CKB protocols**

`CKBProtocol::build()` at `protocols/mod.rs:280-288` always wraps the inner `LengthDelimitedCodec` with `LengthDelimitedCodecWithCompress`, regardless of the `compress` flag value. All protocols have `max_frame_length` values ranging from 1 KB to 4 MB (`support_protocols.rs:122-136`), so a 1-byte payload frame passes the length check and reaches the `data.len() < 2` guard.

**Step 3 — Codec error → unconditional 300-second ban**

`ServiceError::ProtocolError` is handled at `network.rs:643-656` with a direct `ban_session` call at a hardcoded 300-second duration, with no misbehavior scoring or threshold:

```rust
ServiceError::ProtocolError { id, proto_id, error } => {
    let message = format!("ProtocolError id={proto_id}");
    self.network_state.ban_session(
        &context.control().clone().into(),
        id,
        Duration::from_secs(300),
        message,
    );
```

By contrast, `ServiceError::MuxerError` and `ServiceError::SessionTimeout` at `network.rs:658-672` result only in a debug log with no ban.

**Step 4 — `ban_session` bans the peer's exact IP**

`ban_session` at `network.rs:248-274` filters only explicitly whitelisted peers, then calls `peer_store.lock().ban_addr(...)` for all others. The ban is stored as an exact `/32` (IPv4) or `/128` (IPv6) `IpNetwork` entry via `ip_to_network` at `types.rs:152-156`. The `BanList` at `ban_list.rs:48-58` checks both exact-IP and network-containment on every inbound connection attempt.

**Step 5 — Unit test confirms the exact byte sequence**

The existing test at `tests/compress.rs:154-155` explicitly confirms that `[0u8, 0, 0, 1, 0]` (4-byte length prefix = 1, followed by 1-byte payload) triggers the error:

```rust
let mut buf = BytesMut::from(&[0u8, 0, 0, 1, 0][..]); // invalid data, length 1
assert!(codec_with_compress.decode(&mut buf).is_err());
```

**Existing guards are insufficient**: The `is_whitelist` filter in `ban_session` only protects explicitly whitelisted peers. No rate-limiting or misbehavior scoring precedes the ban decision for codec-level errors.

## Impact Explanation

**High severity** — matches *"Vulnerabilities or bad designs which could cause CKB network congestion with few costs."*

An attacker controlling N distinct IP addresses can ban all N IPs from the victim's peer store simultaneously, each for 300 seconds. By cycling across IP pools every 300 seconds, the attacker continuously degrades the victim node's peer diversity and inbound connection capacity. At sufficient scale (rotating through large IP pools), this can effectively isolate a target node from legitimate peers, constituting a sustained low-cost DoS against individual CKB nodes and degrading overall network connectivity.

## Likelihood Explanation

- Requires only a TCP connection and a valid secio handshake — no privileged access, no PoW, no special keys.
- The attack payload is exactly 5 bytes: `[0x00, 0x00, 0x00, 0x01, 0x00]`.
- The existing unit test at `network/src/tests/compress.rs:154-155` explicitly confirms this byte sequence triggers the error.
- No rate-limiting or scoring precedes the ban; each connection attempt results in an immediate ban.
- Cloud providers and residential proxy networks make N-IP attacks cheap and accessible to unprivileged attackers.

## Recommendation

1. **Do not ban on codec errors without prior scoring.** Codec-level `InvalidData` should increment a misbehavior score; only ban after the score exceeds a threshold.
2. **Alternatively**, treat a codec error as a disconnect (not a ban), consistent with how `ServiceError::MuxerError` and `ServiceError::SessionTimeout` are handled — neither triggers a ban.
3. **Or**, add a minimum-frame-size check inside the tentacle codec layer before the application-level `data.len() < 2` guard, so that a 1-byte frame is silently dropped rather than treated as peer misbehavior.
