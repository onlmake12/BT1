### Title
Provider Fee in Entropy Lacks Upper Bound and Can Be Set Arbitrarily — (File: `target_chains/ethereum/contracts/contracts/entropy/Entropy.sol`)

---

### Summary

`setProviderFee` and `setProviderFeeAsFeeManager` in `Entropy.sol` accept any `uint128` value with no upper bound. A registered provider can set `feeInWei` to `type(uint128).max`, causing `getFeeV2` to revert on overflow when adding `pythFeeInWei`, permanently DoS-ing that provider's service and — if the provider is the default — the entire Entropy contract's fee-query and request path.

---

### Finding Description

`setProviderFee` at line 810 of `Entropy.sol` performs only a registration check before writing the caller-supplied value directly to storage:

```solidity
function setProviderFee(uint128 newFeeInWei) external override {
    EntropyStructsV2.ProviderInfo storage provider = _state.providers[msg.sender];
    if (provider.sequenceNumber == 0) revert EntropyErrors.NoSuchProvider();
    provider.feeInWei = newFeeInWei;          // ← no upper-bound check
    ...
}
``` [1](#0-0) 

The same pattern exists in `setProviderFeeAsFeeManager`: [2](#0-1) 

And in the initial `register` call, which also sets `feeInWei` without bounds: [3](#0-2) 

When `feeInWei == type(uint128).max`, `getProviderFee` returns `MAX_UINT128` (the `defaultGasLimit == 0` branch simply returns `provider.feeInWei`): [4](#0-3) 

`getFeeV2` then computes `MAX_UINT128 + _state.pythFeeInWei`, which overflows `uint128` and reverts: [5](#0-4) 

The existing test suite explicitly demonstrates this overflow:

```solidity
vm.prank(provider1);
random.register(MAX_UINT128, provider1Proofs[0], hex"0100", 100, provider1Uri);
vm.expectRevert();
random.getFee(provider1);
``` [6](#0-5) 

Because `requestHelper` calls `getFeeV2` before accepting any payment, the overflow also propagates to every `request`, `requestWithCallback`, and `requestV2` call for that provider: [7](#0-6) 

Note that `setDefaultGasLimit` — by contrast — does enforce a `MAX_GAS_LIMIT` bound, showing the pattern is intentionally applied elsewhere but was omitted for fees: [8](#0-7) 

---

### Impact Explanation

A registered provider who sets `feeInWei = type(uint128).max` causes:

1. `getFee(provider)` / `getFeeV2(provider, gasLimit)` to revert for all callers — no user can query the fee.
2. Every `request*` call targeting that provider to revert at the fee check — complete DoS of that provider's randomness service.
3. If the affected provider is the **default provider**, the zero-argument `getFee()` and `requestV2()` overloads also revert, DoS-ing the entire default Entropy path for all users.

The `IEntropy` interface exposes `setProviderFee` as a public, permissionless function callable by any registered provider: [9](#0-8) 

---

### Likelihood Explanation

- Provider registration is **permissionless** — any address can call `register`.
- After registration, `setProviderFee(type(uint128).max)` requires a single transaction.
- The most impactful scenario (DoS of the default provider path) requires the attacker to first be designated as the default provider by the admin, but the overflow DoS of a non-default provider is reachable with zero privilege beyond self-registration.
- Likelihood: **Low** (requires a malicious or compromised provider), but the attack is trivially executable once the provider role is held.

---

### Recommendation

- **Short term**: Add an upper-bound check in `setProviderFee`, `setProviderFeeAsFeeManager`, and `register` — e.g., `require(newFeeInWei <= MAX_FEE_IN_WEI)` — where `MAX_FEE_IN_WEI` is chosen so that `MAX_FEE_IN_WEI + pythFeeInWei` cannot overflow `uint128`.
- **Long term**: Audit all provider-configurable parameters for missing bounds, following the same pattern already applied to `setDefaultGasLimit` (which enforces `MAX_GAS_LIMIT`).

---

### Proof of Concept

```solidity
// 1. Register as a new provider with MAX fee
entropy.register{value: 0}(
    type(uint128).max,   // feeInWei — no upper bound check
    commitment,
    metadata,
    100,
    uri
);

// 2. Alternatively, existing provider calls:
entropy.setProviderFee(type(uint128).max);

// 3. Any user querying the fee now gets a revert:
entropy.getFee(attackerProvider);   // reverts: uint128 overflow

// 4. Any user requesting randomness from this provider also reverts:
entropy.requestWithCallback{value: 1 ether}(attackerProvider, userCommitment);
// reverts inside requestHelper → getFeeV2 → overflow
```

    function getProviderFee(
        address providerAddr,
        uint32 gasLimit
    ) internal view returns (uint128 feeAmount) {
        EntropyStructsV2.ProviderInfo memory provider = _state.providers[
            providerAddr
        ];

        // Providers charge a minimum of their configured feeInWei for every request.
        // Requests using more than the defaultGasLimit get a proportionally scaled fee.
        // This approach may be somewhat simplistic, but it allows us to continue using the
        // existing feeInWei parameter for the callback failure flow instead of defining new
        // configuration values.
        uint32 roundedGasLimit = uint32(roundTo10kGas(gasLimit)) * TEN_THOUSAND;
        if (
            provider.defaultGasLimit > 0 &&
            roundedGasLimit > provider.defaultGasLimit
        ) {
            // This calculation rounds down the fee, which means that users can get some gas in the callback for free.
            // However, the value of the free gas is < 1 wei, which is insignificant.
            uint128 additionalFee = ((roundedGasLimit -
                provider.defaultGasLimit) * provider.feeInWei) /
                provider.defaultGasLimit;
            return provider.feeInWei + additionalFee;
        } else {
            return provider.feeInWei;
        }
    }
```

    function setProviderFeeAsFeeManager(
        address provider,
        uint128 newFeeInWei
    ) external override {
        EntropyStructsV2.ProviderInfo storage providerInfo = _state.providers[
            provider
        ];

        if (providerInfo.sequenceNumber == 0) {
            revert EntropyErrors.NoSuchProvider();
        }

        if (providerInfo.feeManager != msg.sender) {
            revert EntropyErrors.Unauthorized();
        }

        uint128 oldFeeInWei = providerInfo.feeInWei;
        providerInfo.feeInWei = newFeeInWei;

        emit ProviderFeeUpdated(provider, oldFeeInWei, newFeeInWei);
        emit EntropyEventsV2.ProviderFeeUpdated(
            provider,
            oldFeeInWei,
            newFeeInWei,
            bytes("")
        );
    }
