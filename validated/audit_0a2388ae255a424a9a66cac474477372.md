### Title
Slashed PYTH Tokens Permanently Locked in `slash_custody` — No Withdrawal Instruction Exists - (File: `governance/pyth_staking_sdk/src/idl/integrity-pool.json`)

---

### Summary

The `integrity_pool` program accumulates slashed PYTH tokens into a `slash_custody` token account PDA during every `slash` instruction execution. However, the program exposes no instruction to withdraw, redistribute, or recover those tokens. The slashed funds are permanently locked with no on-chain recovery path.

---

### Finding Description

When a slash event is executed via the `slash` instruction, tokens are transferred from a staker's `stake_account_custody` into the `slash_custody` token account. The `slash_custody` is initialized during `initialize_pool` as `getStakeAccountCustodyAddress(poolData)` — a PDA token account whose authority is the `custody_authority` PDA derived from `poolData`. [1](#0-0) 

The full set of instructions exposed by the `integrity_pool` program is:

- `advance`
- `advance_delegation_record`
- `create_slash_event`
- `initialize_pool`
- `merge_delegation_positions`
- `set_publisher_stake_account`
- `slash`
- `update_delegation_fee`
- `update_reward_program_authority`
- `update_y` [2](#0-1) 

None of these instructions withdraw from or drain `slash_custody`. The `slash` instruction writes to `slash_custody` as a writable account: [3](#0-2) 

The `custody_authority` PDA that controls `slash_custody` is derived from `["authority", stake_account_positions]`: [4](#0-3) 

Because `slash_custody` is keyed to `poolData` (not a user-owned stake account), the staking program's `withdraw_stake` instruction — which requires the stake account owner to sign — cannot be used to drain it. There is no other path. [5](#0-4) 

---

### Impact Explanation

Every successful `slash` execution permanently locks the slashed PYTH tokens in `slash_custody`. These tokens can never be redistributed to the reward pool, burned, or recovered by any party — including the `reward_program_authority`. Over time, as slashing events accumulate, a growing quantity of PYTH tokens becomes irrecoverably locked on-chain.

**Impact: Medium** — funds are permanently locked, not stolen.

---

### Likelihood Explanation

**Likelihood: High** — the `slash` instruction is a core operational path of the OIS protocol. Every slash event (triggered by `reward_program_authority` via `create_slash_event` followed by permissionless `slash` calls) contributes tokens to `slash_custody` that can never leave. The locking is unconditional and automatic. [6](#0-5) 

---

### Recommendation

Add a `withdraw_slash_custody` (or equivalent) instruction to the `integrity_pool` program, callable by `reward_program_authority`, that transfers the accumulated balance of `slash_custody` to a designated destination (e.g., the `pool_reward_custody` for redistribution, or a burn address). This mirrors the pattern already used by `advance_delegation_record`, which transfers from `pool_reward_custody` to stakers. [7](#0-6) 

---

### Proof of Concept

1. `reward_program_authority` calls `create_slash_event` for a publisher with a non-zero `slash_ratio`.
2. Any signer calls `slash` for a delegator's `stake_account_positions`, transferring tokens from `stake_account_custody` → `slash_custody`.
3. Inspect `slash_custody` token balance on-chain — it increases.
4. Attempt to call any instruction in `integrity_pool` that drains `slash_custody` — none exists.
5. Attempt to call `withdraw_stake` on the staking program for `slash_custody` — fails because `poolData` is not a user-owned stake account and no owner can sign.
6. Tokens remain locked indefinitely. [8](#0-7)

  public async withdrawTokensFromStakeAccountCustody(
    stakeAccountPositions: PublicKey,
    amount: bigint,
  ) {
    const globalConfig = await this.getGlobalConfig();
    const mint = globalConfig.pythTokenMint;
    const instructions = [];

    const receiverTokenAccount = getAssociatedTokenAddressSync(
      mint,
      this.wallet.publicKey,
      true,
    );

    // Edge case: if the user doesn't have an ATA, create one
    try {
      await this.getOwnerPythAtaAccount();
    } catch {
      instructions.push(
        createAssociatedTokenAccountInstruction(
          this.wallet.publicKey,
          receiverTokenAccount,
          this.wallet.publicKey,
          mint,
        ),
      );
    }

    instructions.push(
      await this.stakingProgram.methods
        .withdrawStake(new BN(amount.toString()))
        .accounts({
          destination: receiverTokenAccount,
          stakeAccountPositions,
        })
        .instruction(),
    );

    return sendTransaction(instructions, this.connection, this.wallet);
