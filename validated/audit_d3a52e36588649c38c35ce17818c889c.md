Audit Report

## Title
Unconditional `.expect()` on `Option<Vec<u64>>` in `FeeRateCollector::statistics` Panics on Legacy `BlockExt` Records — (`rpc/src/util/fee_rate.rs`)

## Summary

`FeeRateCollector::statistics` calls `.expect()` unconditionally on `BlockExt.txs_sizes`, which is `Option<Vec<u64>>`. Legacy `BlockExt` records stored before `BlockExtV1` was introduced always deserialize with `txs_sizes: None`. Any node with such records in the window iterated by `collect` will panic when `get_fee_rate_statistics` is called, crashing the RPC handler for that request.

## Finding Description

The panic site is confirmed at `rpc/src/util/fee_rate.rs` line 93:

```rust
let txs_sizes = txs_sizes.expect("expect txs_size's length >= 1");
``` [1](#0-0) 

`txs_sizes` is destructured from `BlockExt`, where the field is declared as `Option<Vec<u64>>`: [2](#0-1) 

The legacy on-disk schema `BlockExt` (as opposed to `BlockExtV1`) has no `txs_sizes` field: [3](#0-2) 

`BlockExtV1` adds `txs_sizes` as an optional field: [4](#0-3) 

When the store deserializes a legacy `BlockExt` record, the conversion code explicitly sets `txs_sizes: None`: [5](#0-4) 

The `collect` helper uses `filter_map` to skip blocks where `get_block_ext_by_number` returns `None`, but does **not** skip blocks where the returned `BlockExt` has `txs_sizes = None`: [6](#0-5) 

The RPC handler passes the result directly with no panic guard: [7](#0-6) 

The existing test suite only exercises `txs_sizes: Some(...)` cases, leaving the `None` path untested: [8](#0-7) 

## Impact Explanation

A panic in the RPC handler thread causes a crash for that request. Depending on whether the jsonrpc-http-server framework wraps the handler in `catch_unwind`, the result is either a 500 error for that request or termination of the RPC serving thread, making `get_fee_rate_statistics` (and the deprecated `get_fee_rate_statics`) permanently unavailable. This matches the allowed CKB bounty impact: **Note (0–500 points): Any local RPC API crash**.

## Likelihood Explanation

Any node that has been running since before `BlockExtV1` was introduced will have legacy `BlockExt` records in `COLUMN_BLOCK_EXT`. The `collect` function iterates up to 101 recent blocks; if any of those blocks has a legacy record, the panic fires. The RPC endpoint requires no authentication. An attacker needs only to send a single HTTP POST with `method: "get_fee_rate_statistics"` to trigger the panic.

## Recommendation

Replace the unconditional `.expect()` with a graceful skip, consistent with how `cycles` (also `Option`) is handled:

```rust
let Some(txs_sizes) = txs_sizes else {
    return fee_rates; // skip blocks without txs_sizes data
};
```

This matches the existing pattern at line 97 where `cycles` is handled with `if let Some(cycles) = cycles`. [9](#0-8) 

## Proof of Concept

Using the existing `DummyFeeRateProvider` test harness in `rpc/src/tests/fee_rate.rs`, add a block with `txs_sizes: None`:

```rust
#[test]
fn test_fee_rate_panics_on_legacy_block_ext() {
    let mut provider = DummyFeeRateProvider::new(30);
    provider.append(1, BlockExt {
        received_at: 0,
        total_difficulty: 0u64.into(),
        total_uncles_count: 0,
        verified: Some(true),
        txs_fees: vec![Capacity::shannons(1000)],
        cycles: Some(vec![100]),
        txs_sizes: None,   // ← legacy record: panics unconditionally
    });
    // This call panics at fee_rate.rs:93
    let _ = FeeRateCollector::new(&provider).statistics(None);
}
```

The test infrastructure already exists; only the `txs_sizes: None` case is missing from the test suite, confirming the gap. [10](#0-9)

use crate::util::{FeeRateCollector, FeeRateProvider};
use ckb_jsonrpc_types::FeeRateStatistics;
use ckb_types::core::{BlockExt, BlockNumber, Capacity};
use std::collections::HashMap;

struct DummyFeeRateProvider {
    tip_number: BlockNumber,
    block_exts: HashMap<BlockNumber, BlockExt>,
    max_target: u64,
}

impl DummyFeeRateProvider {
    pub fn new(max_target: u64) -> DummyFeeRateProvider {
        DummyFeeRateProvider {
            tip_number: 0,
            block_exts: HashMap::new(),
            max_target,
        }
    }

    pub fn append(&mut self, number: BlockNumber, ext: BlockExt) {
        if number > self.tip_number {
            self.tip_number = number;
        }
        self.block_exts.insert(number, ext);
    }

    pub fn set_max_target(&mut self, max_target: u64) {
        self.max_target = max_target
    }
}

impl FeeRateProvider for DummyFeeRateProvider {
    fn get_tip_number(&self) -> BlockNumber {
        self.tip_number
    }

    fn get_block_ext_by_number(&self, number: BlockNumber) -> Option<BlockExt> {
        self.block_exts.get(&number).cloned()
    }

    fn max_target(&self) -> u64 {
        self.max_target
    }
}

#[test]
fn test_fee_rate_statics() {
    let mut provider = DummyFeeRateProvider::new(30);
    for i in 0..=21 {
        let ext = BlockExt {
            received_at: 0,
            total_difficulty: 0u64.into(),
            total_uncles_count: 0,
            verified: None,

            // txs_fees length is equal to block_ext.cycles length
            // and txs_fees does not include cellbase
            txs_fees: vec![Capacity::shannons(i * i * 100)],
            // cycles does not include cellbase
            cycles: Some(vec![i * 100]),
            // txs_sizes length is equal to block_ext.txs_fees length + 1
            // first element in txs_sizes is belong to cellbase
            txs_sizes: Some(vec![i * 5678, i * 100]),
        };
        provider.append(i, ext);
    }

    let statistics = FeeRateCollector::new(&provider).statistics(None);
    assert_eq!(
        statistics,
        Some(FeeRateStatistics {
            mean: 11_000.into(),
            median: 11_000.into(),
