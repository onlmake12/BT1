### Title
Unauthenticated `remove_transaction` and `clear_tx_pool` RPC Methods Allow Any Caller to Arbitrarily Purge Pending Transactions — (`File: rpc/src/module/pool.rs`)

### Summary

The CKB JSON-RPC server exposes `remove_transaction` and `clear_tx_pool` as public, unauthenticated endpoints in the Pool module. These are privileged, state-mutating operations — analogous to `mintNFT()` lacking `onlyOwner` — that should only be callable by the node operator. Because the RPC server has no authentication middleware and the `RpcConfig` struct carries no credential fields, any caller who can reach the RPC port can remove any specific pending transaction or wipe the entire pool without restriction.

### Finding Description

**Root cause — no auth check in the handler:**

`remove_transaction` at `rpc/src/module/pool.rs:662` directly calls `tx_pool.remove_local_tx(tx_hash.into())` with no caller verification:

```rust
fn remove_transaction(&self, tx_hash: H256) -> Result<bool> {
    let tx_pool = self.shared.tx_pool_controller();
    tx_pool.remove_local_tx(tx_hash.into()).map_err(|e| { ... })
}
``` [1](#0-0) 

`clear_tx_pool` at line 684 similarly calls `tx_pool.clear_pool(snapshot)` with no caller verification:

```rust
fn clear_tx_pool(&self) -> Result<()> {
    let snapshot = Arc::clone(&self.shared.snapshot());
    let tx_pool = self.shared.tx_pool_controller();
    tx_pool.clear_pool(snapshot)
        .map_err(|err| RPCError::custom(RPCError::Invalid, err.to_string()))?;
    Ok(())
}
``` [2](#0-1) 

**Root cause — no auth middleware in the server:**

The RPC server in `rpc/src/server.rs` builds its Axum router with `CorsLayer::permissive()` and a `TimeoutLayer`, but no authentication layer whatsoever:

```rust
let app = Router::new()
    .route("/", method_router.clone())
    .route("/{*path}", method_router)
    .route("/ping", get(ping_handler))
    .layer(Extension(Arc::clone(rpc)))
    .layer(CorsLayer::permissive())
    .layer(TimeoutLayer::with_status_code(...));
``` [3](#0-2) 

**Root cause — `RpcConfig` has no auth fields:**

The `RpcConfig` struct in `util/app-config/src/configs/rpc.rs` has no token, password, or credential field — there is no mechanism to configure server-side authentication at all: [4](#0-3) 

The miner client does support sending HTTP Basic Auth headers (`parse_authorization` in `miner/src/client.rs:380`), but this is client-side only — the server never validates any `Authorization` header. [5](#0-4) 

**Exposed surface:**

The default `listen_address = "127.0.0.1:8114"` is localhost-only, but the config explicitly supports binding to any address, and the documentation acknowledges this is a common deployment pattern for dApp integration. The `Pool` module is enabled by default in the standard module list: [6](#0-5) 

### Impact Explanation

An attacker who can reach the RPC port can:

1. **Targeted transaction censorship** — call `remove_transaction` with any known `tx_hash` to silently evict a specific pending transaction (e.g., a high-fee transaction, a time-sensitive DAO withdrawal, or a competitor miner's transaction) from the pool. The transaction disappears without error to the original submitter.

2. **Full pool wipe** — call `clear_tx_pool` to atomically remove every pending and proposed transaction from the pool, causing miners to lose all queued fee revenue and forcing every user to resubmit.

3. **Verification queue disruption** — call `clear_tx_verify_queue` to drop all transactions currently being verified, stalling throughput.

These operations directly affect the node's ability to include transactions in blocks, impacting miner revenue and user transaction finality.

### Likelihood Explanation

The RPC is localhost-only by default, but:
- Many operators expose it to `0.0.0.0` for dApp backends, block explorers, and wallet integrations — this is a documented and supported configuration.
- The `Pool` module is enabled by default; operators have no way to split "read-only" from "write" pool operations within the module.
- No credential is required — the attack is a single unauthenticated HTTP POST with a known transaction hash, trivially scriptable.
- Transaction hashes are public (visible on-chain explorers and via `get_raw_tx_pool`), so an attacker can enumerate targets.

### Recommendation

1. Add an optional `rpc_secret_token` field to `RpcConfig`. When set, the server middleware rejects any request whose `Authorization: Bearer <token>` header does not match.
2. Split the Pool module into read-only (`tx_pool_info`, `get_raw_tx_pool`) and write (`remove_transaction`, `clear_tx_pool`, `clear_tx_verify_queue`) sub-modules, allowing operators to expose read endpoints without exposing destructive ones.
3. At minimum, document `remove_transaction` and `clear_tx_pool` as operator-only and add a prominent warning in the RPC README that these methods must not be reachable by untrusted callers.

### Proof of Concept

**Precondition:** Node has `rpc.listen_address = "0.0.0.0:8114"` (common for dApp nodes) with the `Pool` module enabled (default).

**Step 1 — Identify a pending transaction:**
```bash
curl -s http://<node>:8114 -X POST -H 'Content-Type: application/json' \
  -d '{"id":1,"jsonrpc":"2.0","method":"get_raw_tx_pool","params":[false]}'
# Returns list of pending tx hashes
```

**Step 2 — Remove it without any credentials:**
```bash
curl -s http://<node>:8114 -X POST -H 'Content-Type: application/json' \
  -d '{"id":1,"jsonrpc":"2.0","method":"remove_transaction","params":["0x<tx_hash>"]}'
# Returns: {"result": true}
```

**Step 3 — Wipe the entire pool:**
```bash
curl -s http://<node>:8114 -X POST -H 'Content-Type: application/json' \
  -d '{"id":1,"jsonrpc":"2.0","method":"clear_tx_pool","params":[]}'
# Returns: {"result": null}
```

The transaction is gone from the pool. No authentication was required at any step. The `remove_transaction` handler at `rpc/src/module/pool.rs:662` and `clear_tx_pool` at line 684 execute unconditionally for any caller. [7](#0-6)

fn parse_authorization(url: &Uri) -> Option<HeaderValue> {
    let a: Vec<&str> = url.authority()?.as_str().split('@').collect();
    if a.len() >= 2 {
        if a[0].is_empty() {
            return None;
        }
        let mut encoded = "Basic ".to_string();
        base64::prelude::BASE64_STANDARD.encode_string(a[0], &mut encoded);
        let mut header = HeaderValue::from_str(&encoded).unwrap();
        header.set_sensitive(true);
        Some(header)
    } else {
        None
    }
}
