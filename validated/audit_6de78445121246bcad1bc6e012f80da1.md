### Title
Governance Global `freeze` Flag Blocks All User Withdrawals and Unstaking in Pyth Staking Program — (`governance/pyth_staking_sdk/src/idl/staking.json`)

---

### Summary

The Pyth staking program's `GlobalConfig` contains a `freeze: bool` field. When governance sets this flag to `true`, it triggers the `Frozen` error (code 6017) across all protocol instructions — including `withdraw_stake` and `close_position` (the unstaking path). There is no granular mechanism to pause only new deposits while allowing existing stakers to exit. This is the direct analog of the Homora M02 finding: a governance-controlled global pause that traps user funds with no exit path.

---

### Finding Description

The `GlobalConfig` struct in the Pyth staking program contains a `freeze` boolean field: [1](#0-0) 

The program defines a `Frozen` error (code 6017, "Protocol is frozen"): [2](#0-1) 

Both the `withdraw_stake` instruction and the `close_position` instruction (used for unstaking from governance and OIS) load the `config` account as a required account:

`withdraw_stake` loads `config`: [3](#0-2) 

`close_position` loads `config`: [4](#0-3) 

The `Frozen` error exists solely to be checked against `config.freeze` inside these instructions. Since both the token-withdrawal path (`withdraw_stake`) and the position-closing path (`close_position`) load `config`, setting `freeze = true` blocks:

1. Unstaking from Pyth Governance (`closePosition` with `voting` target via `unstakeFromGovernance`)
2. Unstaking from OIS (`undelegate` via `unstakeFromPublisher` / `unstakeFromAllPublishers`)
3. Withdrawing tokens from the stake account custody (`withdraw_stake`)

The `governance_authority` (the Pyth DAO governance key) can update `GlobalConfig` including the `freeze` field: [5](#0-4) 

There is no separate "pause deposits only" flag. The single `freeze` boolean is the only available pause mechanism, and it applies uniformly to all operations.

---

### Impact Explanation

If governance sets `freeze = true` — even for a legitimate security reason (e.g., a discovered vulnerability in the staking logic) — every staker in the protocol loses the ability to:

- Unstake from governance positions (`close_position`)
- Undelegate from OIS publisher pools (`undelegate`)
- Withdraw PYTH tokens from their stake account custody (`withdraw_stake`)

All PYTH tokens held in stake account custodies across the entire protocol are effectively frozen. The staking client confirms these are the only exit paths: [6](#0-5) [7](#0-6) 

Unlike the Homora partial fix (per-token pause), Pyth has no per-position or per-operation granularity at all.

---

### Likelihood Explanation

The scenario does not require malicious governance. It requires only that governance decides to pause the protocol for any reason (security incident, upgrade, regulatory pressure) and uses the only available mechanism — `freeze = true`. At that point, all users with staked positions are unable to exit until governance unfreezes. If governance is compromised, the freeze could be permanent. The likelihood is moderate: the freeze mechanism exists precisely to be used, and its use for any reason traps all user funds.

---

### Recommendation

Introduce separate, granular pause flags:

- `pause_deposits: bool` — blocks `create_position` and new staking
- `pause_withdrawals: bool` — blocks `withdraw_stake` (should default to always-off or require a separate, higher-threshold governance action)

Withdrawals and unstaking should never be blocked by a routine governance pause. At minimum, `withdraw_stake` and `close_position` should be exempt from the `freeze` check, mirroring the Homora recommendation to remove the whitelist requirement from repayments.

---

### Proof of Concept

1. User stakes 1,000,000 PYTH into governance via `create_position` + `close_position` cycle.
2. Governance calls `update_global_config` setting `freeze = true`.
3. User calls `close_position` (unstake from governance) → transaction reverts with `Frozen` (6017).
4. User calls `withdraw_stake` → transaction reverts with `Frozen` (6017).
5. User's 1,000,000 PYTH are locked in the stake account custody PDA with no exit path until governance sets `freeze = false`.

  public async unstakeFromGovernance(
    stakeAccountPositions: PublicKey,
    positionState: PositionState.LOCKED | PositionState.LOCKING,
    amount: bigint,
  ) {
    const stakeAccountPositionsData = await this.getStakeAccountPositions(
      stakeAccountPositions,
    );
    const currentEpoch = await getCurrentEpoch(this.connection);

    let remainingAmount = amount;
    const instructionPromises: Promise<TransactionInstruction>[] = [];

    const eligiblePositions = stakeAccountPositionsData.data.positions
      .map((p, i) => ({ index: i, position: p }))
      .reverse()
      .filter(
        ({ position }) =>
          position.targetWithParameters.voting !== undefined &&
          positionState === getPositionState(position, currentEpoch),
      );

    for (const { position, index } of eligiblePositions) {
      if (position.amount < remainingAmount) {
        instructionPromises.push(
          this.stakingProgram.methods
            .closePosition(index, convertBigIntToBN(position.amount), {
              voting: {},
            })
            .accounts({
              stakeAccountPositions,
            })
            .instruction(),
        );
        remainingAmount -= position.amount;
      } else {
        instructionPromises.push(
          this.stakingProgram.methods
            .closePosition(index, convertBigIntToBN(remainingAmount), {
              voting: {},
            })
            .accounts({
              stakeAccountPositions,
            })
            .instruction(),
        );
        break;
      }
    }

    const instructions = await Promise.all(instructionPromises);
    return sendTransaction(instructions, this.connection, this.wallet);

  public async unstakeFromAllPublishers(
    stakeAccountPositions: PublicKey,
    positionStates: (PositionState.LOCKED | PositionState.LOCKING)[],
  ) {
    const [stakeAccountPositionsData, currentEpoch] = await Promise.all([
      this.getStakeAccountPositions(stakeAccountPositions),
      getCurrentEpoch(this.connection),
    ]);

    const instructions = await Promise.all(
      stakeAccountPositionsData.data.positions
        .map((position, index) => {
          const publisher =
            position.targetWithParameters.integrityPool?.publisher;
          return publisher === undefined
            ? undefined
            : { index, position, publisher };
        })
        // By separating this filter from the next, typescript can narrow the
        // type and automatically infer that there will be no `undefined` values
        // in the array after this line.  If we combine those filters,
        // typescript won't narrow properly.
        .filter((positionInfo) => positionInfo !== undefined)
        .filter(({ position }) =>
          (positionStates as PositionState[]).includes(
            getPositionState(position, currentEpoch),
          ),
        )
        .reverse()
        .map(({ position, index, publisher }) =>
          this.integrityPoolProgram.methods
            .undelegate(index, convertBigIntToBN(position.amount))
            .accounts({ publisher, stakeAccountPositions })
            .instruction(),
        ),
    );

    return sendTransaction(instructions, this.connection, this.wallet);
  }
