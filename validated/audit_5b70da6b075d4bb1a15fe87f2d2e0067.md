### Title
Unfulfilled Echo Requests Lock User ETH Permanently With No Refund Path - (File: target_chains/ethereum/contracts/contracts/echo/Echo.sol)

### Summary
The `Echo` contract accepts ETH from users via `requestPriceUpdatesWithCallback` and stores it as `req.fee` in pending requests. If a provider never calls `executeCallback`, the deposited ETH is permanently locked in the contract. There is no cancellation function, no timeout-based refund, and the admin's `withdrawFees` only covers `_state.accruedFeesInWei` (Pyth protocol fees), not the ETH held in pending requests.

### Finding Description

In `Echo.sol`, `requestPriceUpdatesWithCallback` accepts `msg.value` from the caller and stores the provider portion as `req.fee`:

```solidity
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);
...
_state.accruedFeesInWei += _state.pythFeeInWei;
``` [1](#0-0) 

The Pyth protocol fee is immediately credited to `_state.accruedFeesInWei`, but the provider fee (`req.fee`) remains locked in the contract until `executeCallback` is called. Only at that point is it credited to the provider:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
``` [2](#0-1) 

The admin `withdrawFees` function is scoped exclusively to `_state.accruedFeesInWei` (Pyth protocol fees) and cannot touch `req.fee` held in pending requests:

```solidity
function withdrawFees(uint128 amount) external override {
    require(msg.sender == _state.admin, "Only admin can withdraw fees");
    require(_state.accruedFeesInWei >= amount, "Insufficient balance");
    _state.accruedFeesInWei -= amount;
    (bool sent, ) = msg.sender.call{value: amount}("");
``` [3](#0-2) 

There is no `cancelRequest`, no timeout-based refund, and no admin emergency sweep for pending request fees. The developers themselves acknowledge this risk in a TODO comment:

> "If executeCallback can revert, then funds can be permanently locked in the contract." [4](#0-3) 

The `IEcho` interface exposes no cancellation or user-refund function: [5](#0-4) 

### Impact Explanation

Any ETH deposited by a user in a pending request that is never fulfilled by the provider is permanently unrecoverable. Neither the user (no cancel/refund), nor the admin (scoped `withdrawFees` only covers Pyth fees), nor any other party can retrieve the locked ETH. The contract balance grows with each unfulfilled request and the funds are irrecoverable without a proxy upgrade.

### Likelihood Explanation

Providers are off-chain actors. A provider can go offline, be decommissioned, or simply fail to fulfill a request. Because the exclusivity period only gates *which* provider can fulfill (not whether fulfillment is mandatory), and there is no penalty or timeout that forces fulfillment, any request can remain pending indefinitely. Users who overpay (since `msg.value >= requiredFee` is accepted without refunding the excess) compound the locked amount.

### Recommendation

1. Add a `cancelRequest(uint64 sequenceNumber)` function callable by the original requester after a timeout (e.g., after the exclusivity period + a grace period), which refunds `req.fee` to `req.requester`.
2. Alternatively, add an admin emergency sweep that can recover ETH from requests older than a configurable staleness threshold.
3. Enforce exact payment (`msg.value == requiredFee`) or refund the excess to prevent silent overpayment accumulation.

### Proof of Concept

1. User calls `requestPriceUpdatesWithCallback{value: getFee(...)}(provider, publishTime, priceIds, gasLimit)`.
2. `req.fee = msg.value - pythFeeInWei` is stored; `_state.accruedFeesInWei += pythFeeInWei`.
3. Provider goes offline and never calls `executeCallback`.
4. User attempts to recover ETH — no `cancelRequest` exists.
5. Admin calls `withdrawFees(amount)` — reverts or only recovers `_state.accruedFeesInWei`, leaving `req.fee` locked.
6. ETH in `req.fee` is permanently unrecoverable without a proxy upgrade. [6](#0-5) [7](#0-6)

    function requestPriceUpdatesWithCallback(
        address provider,
        uint64 publishTime,
        bytes32[] calldata priceIds,
        uint32 callbackGasLimit
    ) external payable override returns (uint64 requestSequenceNumber) {
        require(
            _state.providers[provider].isRegistered,
            "Provider not registered"
        );

        // FIXME: this comment is wrong. (we're not using tx.gasprice)
        // NOTE: The 60-second future limit on publishTime prevents a DoS vector where
        //      attackers could submit many low-fee requests for far-future updates when gas prices
        //      are low, forcing executors to fulfill them later when gas prices might be much higher.
        //      Since tx.gasprice is used to calculate fees, allowing far-future requests would make
        //      the fee estimation unreliable.
        require(publishTime <= block.timestamp + 60, "Too far in future");
        if (priceIds.length > MAX_PRICE_IDS) {
            revert TooManyPriceIds(priceIds.length, MAX_PRICE_IDS);
        }
        requestSequenceNumber = _state.currentSequenceNumber++;

        uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
        if (msg.value < requiredFee) revert InsufficientFee();

        Request storage req = allocRequest(requestSequenceNumber);
        req.sequenceNumber = requestSequenceNumber;
        req.publishTime = publishTime;
        req.callbackGasLimit = callbackGasLimit;
        req.requester = msg.sender;
        req.provider = provider;
        req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);

        // Create array with the right size
        req.priceIdPrefixes = new bytes8[](priceIds.length);

        // Copy only the first 8 bytes of each price ID to storage
        for (uint8 i = 0; i < priceIds.length; i++) {
            // Extract first 8 bytes of the price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }
            req.priceIdPrefixes[i] = prefix;
        }
        _state.accruedFeesInWei += _state.pythFeeInWei;

        emit PriceUpdateRequested(req, priceIds);
    }
