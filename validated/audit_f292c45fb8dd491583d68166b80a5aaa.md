### Title
Onion3 Peers Are Unbannnable Due to IP-Only Ban Infrastructure — (`network/src/peer_store/peer_store_impl.rs`)

### Summary
`ban_addr` silently skips adding Onion3 addresses to the ban list because `multiaddr_to_socketaddr` returns `None` for them. Simultaneously, `is_addr_banned` always returns `false` for Onion3 addresses for the same reason. The result is that any Onion3 peer that triggers a ban is only transiently removed from `addr_manager` and can be immediately re-added via discovery, cycling indefinitely.

### Finding Description
The ban mechanism in CKB's peer store is entirely IP-network-based. `BanList::inner` maps `IpNetwork → BannedAddr`. Both the write path (`ban_addr` → `ban_network`) and the read path (`is_addr_banned` → `is_ip_banned`) go through `multiaddr_to_socketaddr`, which returns `None` for Onion3 multiaddrs. This creates a symmetric gap: banning never writes an entry for Onion3, and the ban check never reads one, so the guard in `add_addr` is permanently bypassed for Onion3 addresses.

### Impact Explanation
An Onion3 peer can repeatedly misbehave (e.g., send invalid headers, spam transactions, trigger score penalties via `report()`), get "banned" (only removed from `addr_manager`), re-announce itself via the discovery protocol, be re-added to `addr_manager` through `add_addr`, and repeat the cycle indefinitely. The invariant that a banned peer cannot re-enter the peer store is violated for all Onion3 peers.

### Likelihood Explanation
Any peer operating as a Tor hidden service (Onion3) can exploit this. No special privileges are required — only the ability to connect via P2P and trigger misbehavior scoring. The discovery re-add path is a standard production code path.

### Recommendation
Introduce a separate ban set for non-IP addresses (e.g., keyed on the raw Onion3 bytes or peer ID). In `ban_addr`, when `multiaddr_to_socketaddr` returns `None`, extract the Onion3 component and store it in this set. In `is_addr_banned`, check this set for Onion3 addresses before returning `false`.
