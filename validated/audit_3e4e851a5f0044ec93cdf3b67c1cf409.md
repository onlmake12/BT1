### Title
No User-Controlled Maximum Fee in Entropy `requestV2()` — Provider Can Instantly Raise Fee Without Notice - (File: target_chains/ethereum/contracts/contracts/entropy/Entropy.sol)

### Summary
The Entropy contract allows providers (and their fee managers) to instantly update `feeInWei` via `setProviderFee()` / `setProviderFeeAsFeeManager()` with no timelock, no upper bound, and no notice period. The `requestV2()` family of functions provides no `maxFee` parameter for callers to guard against fee increases between fee-query and execution. Critically, excess `msg.value` is explicitly **not refunded**, so a user who sends a buffer to avoid reverts can be drained of that buffer by a fee front-run.

### Finding Description
`Entropy.setProviderFee()` and `setProviderFeeAsFeeManager()` update `provider.feeInWei` atomically and immediately:

```solidity
// Entropy.sol L810-L827
function setProviderFee(uint128 newFeeInWei) external override {
    ...
    provider.feeInWei = newFeeInWei;   // instant, no timelock
    emit ProviderFeeUpdated(...);
}
``` [1](#0-0) 

The fee is consumed at execution time inside `requestHelper()`:

```solidity
// Entropy.sol L234-L235
uint128 requiredFee = getFeeV2(provider, callbackGasLimit);
if (msg.value < requiredFee) revert EntropyErrors.InsufficientFee();
``` [2](#0-1) 

The `IEntropyV2` interface itself warns that fees change over time and that **excess value is not refunded**:

> "Note that the fee can change over time… Further note that excess value is *not* refunded to the caller." [3](#0-2) 

None of the `requestV2()` overloads accept a `maxFee` parameter: [4](#0-3) 

The Fortuna keeper's `adjust_fee_wrapper` loop runs continuously and updates the on-chain fee automatically based on gas prices, making fee changes a routine, non-malicious occurrence: [5](#0-4) 

### Impact Explanation
A user who sends `requestV2{value: fee + buffer}()` to avoid reverts during gas-price volatility can have the entire `fee + buffer` consumed if the provider's fee is raised to match `msg.value` between the user's fee query and transaction execution. Because excess value is never refunded, the user loses the buffer. Alternatively, a user who sends exactly the queried fee will have their transaction revert, causing a DoS and wasted gas. In both cases the user has no on-chain mechanism to bound the fee they are willing to pay.

### Likelihood Explanation
The Fortuna keeper runs a continuous `adjust_fee_if_necessary` loop that updates `feeInWei` on-chain whenever gas prices move outside a target band. This is a documented, production behavior — not a hypothetical. Any user transaction that lands in the same block as a keeper fee update is subject to this race. On high-throughput chains with frequent gas-price swings, this is a regular occurrence. A malicious provider (any permissionlessly registered provider) can also deliberately front-run user transactions.

### Recommendation
1. Add a `maxFee` parameter to `requestV2()` variants and revert if `requiredFee > maxFee`.
2. Refund excess `msg.value` (`msg.value - requiredFee`) back to the caller inside `requestHelper()`.
3. Optionally, introduce a timelock or minimum notice period (e.g., one block or one epoch) before a fee increase takes effect, analogous to the recommendation in the reference report.

### Proof of Concept
1. Provider's current `feeInWei` = 1000 wei. User calls `getFeeV2()` off-chain → 1000 wei.
2. User submits `requestV2{value: 2000 wei}()` with a 1000 wei buffer to avoid reverts.
3. Provider (or Fortuna keeper) calls `setProviderFee(2000)` in the same block (or front-runs via mempool).
4. `requestHelper` executes: `requiredFee = getFeeV2(...) = 2000 wei`. Check passes (`msg.value == 2000`).
5. `providerInfo.accruedFeesInWei += providerFee` — provider collects 2000 wei.
6. No refund is issued. User paid 1000 wei more than the fee they observed. [6](#0-5) [7](#0-6) [8](#0-7)

    function setProviderFee(uint128 newFeeInWei) external override {
        EntropyStructsV2.ProviderInfo storage provider = _state.providers[
            msg.sender
        ];

        if (provider.sequenceNumber == 0) {
            revert EntropyErrors.NoSuchProvider();
        }
        uint128 oldFeeInWei = provider.feeInWei;
        provider.feeInWei = newFeeInWei;
        emit ProviderFeeUpdated(msg.sender, oldFeeInWei, newFeeInWei);
        emit EntropyEventsV2.ProviderFeeUpdated(
            msg.sender,
            oldFeeInWei,
            newFeeInWei,
            bytes("")
        );
    }

    function setProviderFeeAsFeeManager(
        address provider,
        uint128 newFeeInWei
    ) external override {
        EntropyStructsV2.ProviderInfo storage providerInfo = _state.providers[
            provider
        ];

        if (providerInfo.sequenceNumber == 0) {
            revert EntropyErrors.NoSuchProvider();
        }

        if (providerInfo.feeManager != msg.sender) {
            revert EntropyErrors.Unauthorized();
        }

        uint128 oldFeeInWei = providerInfo.feeInWei;
        providerInfo.feeInWei = newFeeInWei;

        emit ProviderFeeUpdated(provider, oldFeeInWei, newFeeInWei);
        emit EntropyEventsV2.ProviderFeeUpdated(
            provider,
            oldFeeInWei,
            newFeeInWei,
            bytes("")
        );
    }
