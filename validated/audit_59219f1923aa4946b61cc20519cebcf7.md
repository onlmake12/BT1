### Title
Missing Sufficiency Check on Actual Pyth Oracle Fee in `executeCallback` Causes Permanent Fund Lockup - (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary
`Echo.executeCallback()` computes the actual Pyth oracle fee dynamically at execution time via `pyth.getUpdateFee(updateData)`, but the stored `req.fee` was computed at request time using the static `_state.pythFeeInWei` estimate. There is no check that `req.fee + msg.value >= pythFee`. If the actual Pyth fee exceeds the stored provider fee, the arithmetic subtraction underflows (Solidity 0.8 reverts), permanently locking user funds with no cancellation path.

### Finding Description

**At request time** (`requestPriceUpdatesWithCallback`):

```solidity
uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
if (msg.value < requiredFee) revert InsufficientFee();
// ...
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);  // static estimate
_state.accruedFeesInWei += _state.pythFeeInWei;                // immediately credited
```

`req.fee` stores `msg.value - _state.pythFeeInWei`. The static `_state.pythFeeInWei` is immediately credited to Echo's accrued fees, so the contract only retains `req.fee` for the provider.

**At execution time** (`executeCallback`):

```solidity
uint256 pythFee = pyth.getUpdateFee(updateData);   // dynamic, actual Pyth fee
// ...
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);   // NO check: pythFee <= req.fee + msg.value
```

`pyth.getUpdateFee(updateData)` is dynamic and depends on the actual `updateData` submitted. If `pythFee > req.fee + msg.value_executor`, the subtraction underflows → revert. The contract's own TODO comment acknowledges this: *"if executeCallback can revert, then funds can be permanently locked in the contract."*

The mismatch arises because:
- `_state.pythFeeInWei` is a static admin-configured estimate of what Pyth will charge
- `pyth.getUpdateFee(updateData)` is the real fee at execution time, which can differ if: (a) Pyth governance raises fees between request and execution, (b) `_state.pythFeeInWei` is misconfigured below the actual Pyth fee, or (c) the `updateData` submitted contains more price feeds than the minimum

`getFee()` itself acknowledges the estimation problem:

```solidity
// Note: The provider needs to set its fees to include the fee charged by the Pyth contract.
// Ideally, we would be able to automatically compute the pyth fees from the priceIds, but the
// fee computation on IPyth assumes it has the full updated data.
```

There is no cancellation or refund function in the Echo contract. Once a request is stored, the only way to release funds is via `clearRequest`, which is only called inside `executeCallback` after the fee arithmetic succeeds.

### Impact Explanation

If `pyth.getUpdateFee(updateData) > req.fee + msg.value_executor`:
- `executeCallback` reverts unconditionally
- The request can never be fulfilled
- User funds (`req.fee` worth of ETH) are permanently locked in the Echo contract with no recovery path
- The provider also cannot earn their fee

**Impact: High** — permanent loss of user funds with no recovery mechanism.

### Likelihood Explanation

**Likelihood: Medium.** The trigger conditions are realistic:
1. Pyth governance raises `pythFeeInWei` on the Pyth oracle contract between request and execution (a normal governance action)
2. The Echo admin sets `_state.pythFeeInWei` lower than the actual Pyth oracle fee (misconfiguration)
3. A provider submits `updateData` with more feeds than the minimum, causing `getUpdateFee` to return more than `_state.pythFeeInWei`

Any unprivileged user who calls `requestPriceUpdatesWithCallback` during a window where `_state.pythFeeInWei < pyth.getUpdateFee(updateData)` will have their funds locked.

### Recommendation

Add an explicit sufficiency check before the subtraction in `executeCallback`:

```solidity
uint256 pythFee = pyth.getUpdateFee(updateData);
require(req.fee + msg.value >= pythFee, "Insufficient funds to cover Pyth fee");
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
```

Additionally, implement a request cancellation/refund mechanism so users can recover funds if a request becomes permanently unfulfillable. Consider also dynamically computing the Pyth fee at request time (e.g., by requiring the user to pass `updateData` at request time, or by using a conservative upper-bound estimate).

### Proof of Concept

1. Pyth admin raises `pythFeeInWei` on the Pyth oracle contract from 1 wei to 1000 wei (normal governance)
2. Echo admin has `_state.pythFeeInWei = 1` (not yet updated)
3. User calls `requestPriceUpdatesWithCallback{value: getFee(...)}(...)` — pays exactly `getFee()` which includes only 1 wei for Pyth
4. `req.fee = msg.value - 1` (provider portion)
5. Provider calls `executeCallback(...)` with valid `updateData`
6. `pythFee = pyth.getUpdateFee(updateData)` returns 1000
7. `(req.fee + 0) - 1000` underflows → revert
8. User funds are permanently locked; no cancellation path exists [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

```text
    function getFee(
        address provider,
        uint32 callbackGasLimit,
        bytes32[] calldata priceIds
    ) public view override returns (uint96 feeAmount) {
        uint96 baseFee = _state.pythFeeInWei; // Fixed fee to Pyth
        // Note: The provider needs to set its fees to include the fee charged by the Pyth contract.
        // Ideally, we would be able to automatically compute the pyth fees from the priceIds, but the
        // fee computation on IPyth assumes it has the full updated data.
        uint96 providerBaseFee = _state.providers[provider].baseFeeInWei;
        uint96 providerFeedFee = SafeCast.toUint96(
            priceIds.length * _state.providers[provider].feePerFeedInWei
        );
        uint96 providerFeeInWei = _state.providers[provider].feePerGasInWei; // Provider's per-gas rate
        uint256 gasFee = callbackGasLimit * providerFeeInWei; // Total provider fee based on gas
        feeAmount =
            baseFee +
            providerBaseFee +
            providerFeedFee +
            SafeCast.toUint96(gasFee); // Total fee user needs to pay
```
