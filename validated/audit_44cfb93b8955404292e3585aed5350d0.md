### Title
Excess ETH Not Refunded to Caller in `requestPriceUpdatesWithCallback` — (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

`Echo.sol`'s `requestPriceUpdatesWithCallback` function does not refund excess ETH to the caller. Any ETH sent beyond `_state.pythFeeInWei` is silently stored as `req.fee` and later paid out to the provider via `executeCallback`, causing users who overpay to permanently lose their excess ETH to the provider.

---

### Finding Description

In `requestPriceUpdatesWithCallback`, the required fee is computed at line 75:

```solidity
uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
if (msg.value < requiredFee) revert InsufficientFee();
```

`getFee` returns `pythFeeInWei + providerBaseFee + providerFeedFee + gasFee`. The function enforces a minimum but places no upper bound on `msg.value`. The entire surplus is then silently absorbed at line 84:

```solidity
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);
```

Only `pythFeeInWei` is credited to the protocol at line 99:

```solidity
_state.accruedFeesInWei += _state.pythFeeInWei;
```

The remainder — which includes both the legitimate provider fee **and any user overpayment** — is stored in `req.fee`. When `executeCallback` is later called, the provider collects the full `req.fee` at line 161–162:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
```

There is no code path that returns excess ETH to the original requester. [1](#0-0) [2](#0-1) [3](#0-2) 

This contrasts with `PythLazer.sol`'s `verifyUpdate`, which explicitly refunds excess:

```solidity
if (msg.value > verification_fee) {
    payable(msg.sender).transfer(msg.value - verification_fee);
}
``` [4](#0-3) 

The Entropy contract also explicitly documents its analogous behavior as intentional ("excess value is *not* refunded to the caller"), but Echo carries no such documentation and the excess silently enriches the provider rather than being retained by the protocol. [5](#0-4) 

---

### Impact Explanation

Any user who sends `msg.value > requiredFee` to `requestPriceUpdatesWithCallback` permanently loses the excess ETH to the provider. The provider's `accruedFeesInWei` is inflated by the overpayment, and the user has no mechanism to recover it. This is a direct, irreversible financial loss for the caller.

---

### Likelihood Explanation

Fee overpayment is a realistic scenario:

1. **Fee changes between estimation and submission**: A user calls `getFee()` off-chain, then the provider updates their fee before the transaction lands. The user's `msg.value` now exceeds the new required fee, and the excess is silently taken.
2. **Defensive overpayment by integrators**: Smart contract integrations commonly add a small buffer (e.g., `fee * 110 / 100`) to ensure the transaction does not revert due to fee increases. This buffer is permanently lost.
3. **No documentation warning**: Unlike Entropy's interface, Echo carries no NatSpec warning that excess is non-refundable, so integrators have no reason to guard against it.

---

### Recommendation

Add a refund of excess ETH to the caller at the end of `requestPriceUpdatesWithCallback`, mirroring the pattern already used in `PythLazer.verifyUpdate`:

```solidity
uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
if (msg.value < requiredFee) revert InsufficientFee();
uint96 excess = SafeCast.toUint96(msg.value) - requiredFee;
if (excess > 0) {
    (bool sent, ) = msg.sender.call{value: excess}("");
    require(sent, "Refund failed");
}
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei - excess);
```

Alternatively, document explicitly that excess is non-refundable and is credited to the provider, so integrators can guard against it.

---

### Proof of Concept

1. Provider registers with `baseFeeInWei = 500`, `feePerFeedInWei = 100`, `feePerGasInWei = 0`. `pythFeeInWei = 200`.
2. User calls `getFee(provider, 0, [priceId])` → returns `200 + 500 + 100 = 800 wei`.
3. Provider raises `baseFeeInWei` to `600` in a front-running or concurrent transaction.
4. User's transaction lands with `msg.value = 800`. New `requiredFee = 900`. Transaction does **not** revert because `msg.value >= old requiredFee` — wait, actually it would revert here. Let me adjust:
5. More precisely: user sends `msg.value = 1000` as a buffer. `requiredFee = 800`. `req.fee = 1000 - 200 = 800`. Legitimate provider fee = `500 + 100 = 600`. Excess = `200 wei` silently stored in `req.fee`.
6. Provider calls `executeCallback`. Provider receives `req.fee + msg.value_executor - pythFee_actual = 800 + 0 - 0 = 800`, which includes the user's 200 wei overpayment.
7. User loses 200 wei with no recourse. [6](#0-5)

        address provider,
        uint64 publishTime,
        bytes32[] calldata priceIds,
        uint32 callbackGasLimit
    ) external payable override returns (uint64 requestSequenceNumber) {
        require(
            _state.providers[provider].isRegistered,
            "Provider not registered"
        );

        // FIXME: this comment is wrong. (we're not using tx.gasprice)
        // NOTE: The 60-second future limit on publishTime prevents a DoS vector where
        //      attackers could submit many low-fee requests for far-future updates when gas prices
        //      are low, forcing executors to fulfill them later when gas prices might be much higher.
        //      Since tx.gasprice is used to calculate fees, allowing far-future requests would make
        //      the fee estimation unreliable.
        require(publishTime <= block.timestamp + 60, "Too far in future");
        if (priceIds.length > MAX_PRICE_IDS) {
            revert TooManyPriceIds(priceIds.length, MAX_PRICE_IDS);
        }
        requestSequenceNumber = _state.currentSequenceNumber++;

        uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
        if (msg.value < requiredFee) revert InsufficientFee();

        Request storage req = allocRequest(requestSequenceNumber);
        req.sequenceNumber = requestSequenceNumber;
        req.publishTime = publishTime;
        req.callbackGasLimit = callbackGasLimit;
        req.requester = msg.sender;
        req.provider = provider;
        req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);

        // Create array with the right size
        req.priceIdPrefixes = new bytes8[](priceIds.length);

        // Copy only the first 8 bytes of each price ID to storage
        for (uint8 i = 0; i < priceIds.length; i++) {
            // Extract first 8 bytes of the price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }
            req.priceIdPrefixes[i] = prefix;
        }
        _state.accruedFeesInWei += _state.pythFeeInWei;

        emit PriceUpdateRequested(req, priceIds);
    }
