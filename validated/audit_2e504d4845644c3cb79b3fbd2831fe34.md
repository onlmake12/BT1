### Title
Unauthenticated Inbound Peer Can Inject Arbitrary (IP, Port) Into Network-Wide Address Gossip — (`network/src/protocols/discovery/mod.rs`, `state.rs`)

### Summary
An inbound peer that sends `GetNodes{version=1, listen_port=Some(chosen_port)}` causes the victim node to replace the session's ephemeral TCP source port with the attacker-supplied `chosen_port` and promote the address to `RemoteAddress::Listen`. This address is then broadcast as an announcement to up to 3 random peers on every `notify()` tick, with no verification that `chosen_port` is the peer's actual listening port.

### Finding Description
In `DiscoveryProtocol::received()`, when a `GetNodes` message is received with `listen_port = Some(port)`, `state.remote_addr.update_port(port)` is called unconditionally regardless of session direction. `update_port()` replaces the TCP port in the stored `RemoteAddress::Init` with the peer-supplied value and transitions the state to `RemoteAddress::Listen`. Subsequently, if `version >= REUSE_PORT_VERSION` (1), `change_to_listen()` is also called (a no-op at this point since the state is already `Listen`).

Once in the `Listen` state, `check_timer()` returns this address during every `notify()` call. The only filter applied is `is_valid_addr()`, which only checks whether the IP is globally routable via `is_reachable()`. The port value is never independently verified. The resulting `(attacker_ip:chosen_port, flags)` tuple is added to `announce_list` and forwarded to up to 3 randomly selected peers as `Nodes{announce: true}`.

### Impact Explanation
An attacker controlling a single routable IP can inject an arbitrary `(IP, port)` pair into the network-wide peer discovery gossip. Concretely:
- **Eclipse attack**: Inject the addresses of victim nodes with wrong ports, causing other peers to fail connections to them and eventually drop them from their peer stores.
- **DoS of specific services**: Inject `(target_ip:target_port)` for any non-CKB service, causing the CKB network to flood it with connection attempts.
- **Address table poisoning**: Continuously inject addresses to fill peer stores with unreachable entries, degrading new-peer discovery for the entire network.

### Likelihood Explanation
Exploitation requires only a single inbound TCP connection from a routable IP and one crafted `GetNodes` message. No authentication, no PoW, no special privileges. The `REUSE_PORT_VERSION` (1) is the current version used on Linux (the dominant server OS), so the triggering condition is met by default in production deployments. [9](#0-8) 

### Recommendation
1. **Reject `listen_port` from inbound sessions**: In the `GetNodes` handler, only call `update_port()` / `change_to_listen()` when `context.session.ty.is_outbound()` is false — i.e., skip port promotion entirely for inbound peers, since the victim cannot verify the claimed port.
2. **Alternatively, require independent confirmation**: Only promote an inbound session's address to `Listen` after the identify protocol has confirmed the peer's listen address through a separate, authenticated channel.
3. **Add direction check before `update_port`**: Guard the block at lines 124–139 of `mod.rs` with `if !session.ty.is_inbound()` to prevent inbound peers from influencing their own stored listen address. [10](#0-9) 

### Proof of Concept
1. Start a CKB node (victim) with a routable IP.
2. Attacker opens an inbound TCP connection to the victim from a routable IP (e.g., `1.2.3.4`).
3. Attacker sends `GetNodes{version=1, count=100, listen_port=Some(9999), required_flags=0}`.
4. Victim calls `update_port(9999)` → `remote_addr` becomes `RemoteAddress::Listen(/ip4/1.2.3.4/tcp/9999)`.
5. On the next `notify()` tick (default 60s), `check_timer()` returns `/ip4/1.2.3.4/tcp/9999`, `is_valid_addr()` returns `true` (routable IP).
6. The address is pushed into `announce_list` and sent as `Nodes{announce: true}` to 3 random peers.
7. Assert: those 3 peers now have `/ip4/1.2.3.4/tcp/9999` in their peer stores, despite `9999` never being verified as the attacker's actual listen port.

        if let RemoteAddress::Init(addr) = self {
