### Title
Unbounded `pending_get_block_proposals` DashMap Growth via Unauthenticated `GetBlockProposal` P2P Messages — (`File: sync/src/types/mod.rs`)

---

### Summary

Any connected peer can send repeated `GetBlockProposal` relay messages containing fabricated, non-existent `ProposalShortId` values. The handler inserts every unresolved proposal ID into `pending_get_block_proposals`, an unbounded `DashMap` with no per-peer quota and no total size cap. The only eviction is a periodic full drain on a timer. A single malicious peer can flood this map faster than it is drained, causing sustained unbounded memory growth and eventual OOM crash of the victim node.

---

### Finding Description

`SyncState` in `sync/src/types/mod.rs` declares `pending_get_block_proposals` as a plain, unbounded `DashMap`: [1](#0-0) 

It is initialized with no capacity limit: [2](#0-1) 

The insertion function `insert_get_block_proposals` performs no size check, no per-peer quota, and no deduplication guard against attacker-controlled IDs: [3](#0-2) 

This function is called unconditionally from `GetBlockProposalProcess::execute()` for every proposal ID that is not found in the local tx-pool: [4](#0-3) 

The only guard in the handler is a per-message count ceiling (`max_block_proposals_limit * max_uncles_num`), which limits proposals per single message but places no bound on the total accumulation rate across repeated messages: [5](#0-4) 

The sole cleanup path is `drain_get_block_proposals`, which atomically clears the entire map and is called only from the periodic `prune_tx_proposal_request` timer task: [6](#0-5) [7](#0-6) 

Between two consecutive timer firings, an attacker sending a continuous stream of `GetBlockProposal` messages with distinct, fabricated `ProposalShortId` values causes the map to grow without bound. Because `ProposalShortId` is a 10-byte value with a 2^80 key space, the attacker trivially generates unique IDs that never collide with existing entries, guaranteeing a new allocation per ID.

---

### Impact Explanation

Each `DashMap` entry consumes heap memory for the key (`ProposalShortId`, 10 bytes), the `HashSet<PeerIndex>` value, and internal DashMap shard bookkeeping. At a sustained rate of thousands of messages per second, each carrying the per-message maximum of proposals, the map can grow to hundreds of megabytes within seconds. When the process exhausts available RAM, the OS OOM-killer terminates the node, causing a complete node shutdown. Because the attack requires only a single connected peer sending standard relay protocol messages, it is reachable from any unprivileged network participant.

---

### Likelihood Explanation

The `GetBlockProposal` message is part of the standard `RelayV3` protocol. Any peer that completes the P2P handshake can send it without staking, authentication, or any privileged role. The fabricated `ProposalShortId` values pass all structural validation (correct length, no duplicate check within the message). The attack requires no PoW, no valid transaction, and no chain knowledge. A single attacker node with a normal network connection is sufficient.

---

### Recommendation

1. **Enforce a total size cap** on `pending_get_block_proposals`. Replace the unbounded `DashMap` with a bounded structure (e.g., `LruCache` or a `DashMap` with an atomic counter guard) that rejects insertions once a configurable maximum (e.g., 50 000 entries) is reached.
2. **Add per-peer insertion quotas** inside `insert_get_block_proposals`: track how many pending entries each `PeerIndex` has contributed and reject or disconnect peers that exceed a per-peer limit.
3. **Disconnect or ban peers** that repeatedly send `GetBlockProposal` messages whose proposal IDs are never resolved, analogous to the `TooManyUnknownTransactions` status already implemented for `add_ask_for_txs`. [8](#0-7) 

---

### Proof of Concept

**Attacker-controlled entry path:**

1. Connect a peer to the victim CKB node using the standard `RelayV3` protocol.
2. In a tight loop, send `GetBlockProposal` messages where `proposals` is filled with randomly generated 10-byte `ProposalShortId` values (guaranteed not to exist in the victim's tx-pool).
3. Each message passes the per-message count check and the duplicate-within-message check.
4. `GetBlockProposalProcess::execute` calls `insert_get_block_proposals`, inserting all IDs into the unbounded `DashMap`.
5. Repeat faster than the `prune_tx_proposal_request` timer drains the map.
6. Observe the victim node's RSS grow monotonically until OOM termination.

    pub fn add_ask_for_txs(&self, peer_index: PeerIndex, tx_hashes: Vec<Byte32>) -> Status {
        let mut unknown_tx_hashes = self.unknown_tx_hashes.lock();

        for tx_hash in tx_hashes
            .into_iter()
            .take(MAX_UNKNOWN_TX_HASHES_SIZE_PER_PEER)
        {
            match unknown_tx_hashes.entry(tx_hash) {
                keyed_priority_queue::Entry::Occupied(entry) => {
                    let mut priority = entry.get_priority().clone();
                    priority.push_peer(peer_index);
                    entry.set_priority(priority);
                }
                keyed_priority_queue::Entry::Vacant(entry) => {
                    entry.set_priority(UnknownTxHashPriority {
                        request_time: Instant::now(),
                        peers: vec![peer_index],
                        requested: false,
                    })
                }
            }
        }

        // Check `unknown_tx_hashes`'s length after inserting the arrival `tx_hashes`
        if unknown_tx_hashes.len() >= MAX_UNKNOWN_TX_HASHES_SIZE
            || unknown_tx_hashes.len()
                >= self.peers.state.len() * MAX_UNKNOWN_TX_HASHES_SIZE_PER_PEER
        {
            warn!(
                "unknown_tx_hashes is too long, len: {}",
                unknown_tx_hashes.len()
            );

            let mut peer_unknown_counter = 0;
            for (_hash, priority) in unknown_tx_hashes.iter() {
                for peer in priority.peers.iter() {
                    if *peer == peer_index {
                        peer_unknown_counter += 1;
                    }
                }
            }
            if peer_unknown_counter >= MAX_UNKNOWN_TX_HASHES_SIZE_PER_PEER {
                return StatusCode::TooManyUnknownTransactions.into();

    pub async fn execute(self) -> Status {
        let shared = self.relayer.shared();
        let message_len = self.message.proposals().len();
        {
            // The block proposal request is separate from uncles,
            // so here the limit is only used to calculate the maximum value of uncles
            let limit = shared.consensus().max_block_proposals_limit()
                * (shared.consensus().max_uncles_num() as u64);
            if message_len as u64 > limit {
                return StatusCode::ProtocolMessageIsMalformed.with_context(format!(
                    "GetBlockProposal proposals count({message_len}) > consensus max_block_proposals_limit({limit})"
                ));
            }
        }

        let proposals: HashSet<packed::ProposalShortId> =
            self.message.proposals().to_entity().into_iter().collect();

        if proposals.len() != message_len {
            return StatusCode::RequestDuplicate.with_context("Request duplicate proposal");
        }

        let fetched_transactions = {
            let tx_pool = self.relayer.shared.shared().tx_pool_controller();
            let fetch_txs = tx_pool.fetch_txs(proposals.clone()).await;
            if let Err(e) = fetch_txs {
                debug_target!(
                    crate::LOG_TARGET_RELAY,
                    "relayer tx_pool_controller send fetch_txs error: {:?}",
                    e
                );
                return Status::ok();
            }
            fetch_txs.unwrap()
        };
        // Transactions that do not exist on this node
        let not_exist_proposals: Vec<packed::ProposalShortId> = proposals
            .into_iter()
            .filter(|short_id| !fetched_transactions.contains_key(short_id))
            .collect();

        // Cache request, try process on timer
        self.relayer
            .shared()
            .state()
            .insert_get_block_proposals(self.peer, not_exist_proposals);
