### Title
Unvalidated `providerToCredit` in `executeCallback` Allows Fee Theft from Legitimate Providers — (`File: target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary

In `Echo.sol`, the `executeCallback` function accepts an arbitrary `providerToCredit` address and credits the full request fee to it without verifying that the address is the provider originally assigned to the request. After the exclusivity period expires, any registered provider can call `executeCallback` with themselves as `providerToCredit`, stealing the fee that belongs to the originally assigned provider.

### Finding Description

`executeCallback` enforces `providerToCredit == req.provider` only during the exclusivity window: [1](#0-0) 

Once `block.timestamp >= req.publishTime + exclusivityPeriodSeconds`, the check is skipped entirely and the fee is credited unconditionally to the caller-supplied address: [2](#0-1) 

There is no subsequent validation that `providerToCredit` equals `req.provider` or is even a registered provider. The `ProviderInfo` struct stores `accruedFeesInWei` per address: [3](#0-2) 

Fees credited to a provider address are only withdrawable via `withdrawAsFeeManager`, which requires `msg.sender == _state.providers[provider].feeManager`: [4](#0-3) 

An attacker who has registered as a provider and set themselves as their own fee manager can therefore:

1. Register via `registerProvider(...)` — sets `isRegistered = true`.
2. Call `setFeeManager(attackerAddress)` — sets `feeManager = attackerAddress`.
3. Wait for the exclusivity period on any pending request to expire.
4. Call `executeCallback(attackerAddress, sequenceNumber, updateData, priceIds)`.
5. The full `req.fee + msg.value - pythFee` is credited to `_state.providers[attackerAddress].accruedFeesInWei`.
6. Call `withdrawAsFeeManager(attackerAddress, amount)` to drain the stolen fees.

The legitimate provider (`req.provider`) receives nothing for fulfilling the request they were originally assigned. [5](#0-4) 

### Impact Explanation

**High.** The legitimate provider permanently loses the fee they were owed for a fulfilled request. The attacker, who is any permissionlessly registered provider, can steal fees from every request whose exclusivity period has elapsed. The user's callback is still executed (no user fund loss), but the provider's fee accounting is corrupted and the stolen ETH is unrecoverable by the legitimate provider.

### Likelihood Explanation

**Medium.** The exclusivity period is configurable (default 15 seconds per the test setup). Any registered provider — a role that is permissionless — can monitor the mempool or chain for requests whose exclusivity period is about to expire and front-run the legitimate provider's `executeCallback` call. No privileged access is required. [6](#0-5) 

### Recommendation

After the exclusivity period, restrict `providerToCredit` to registered providers only, or always credit `req.provider` regardless of who calls `executeCallback`. At minimum, add:

```solidity
require(
    _state.providers[providerToCredit].isRegistered,
    "providerToCredit not registered"
);
```

A stronger fix is to always credit `req.provider` and only allow a different address to be credited if the original provider has explicitly forfeited the request (e.g., via a timeout-and-penalty mechanism).

### Proof of Concept

```solidity
// 1. Attacker registers as provider
vm.prank(attacker);
echo.registerProvider(1 wei, 1 wei, 1 wei);

// 2. Attacker sets themselves as fee manager
vm.prank(attacker);
echo.setFeeManager(attacker);

// 3. User submits a request assigned to legitimateProvider
vm.prank(user);
uint64 seq = echo.requestPriceUpdatesWithCallback{value: fee}(
    legitimateProvider, publishTime, priceIds, gasLimit
);

// 4. Wait for exclusivity period to expire
vm.warp(block.timestamp + exclusivityPeriod + 1);

// 5. Attacker steals the fee by crediting themselves
vm.prank(attacker);
echo.executeCallback(attacker, seq, updateData, priceIds);

// 6. Attacker withdraws stolen fees
uint128 stolen = echo.getProviderInfo(attacker).accruedFeesInWei;
vm.prank(attacker);
echo.withdrawAsFeeManager(attacker, stolen);
// legitimateProvider.accruedFeesInWei == 0 (stolen)
``` [7](#0-6)

    function withdrawAsFeeManager(
        address provider,
        uint128 amount
    ) external override {
        require(
            msg.sender == _state.providers[provider].feeManager,
            "Only fee manager"
        );
        require(
            _state.providers[provider].accruedFeesInWei >= amount,
            "Insufficient balance"
        );

        _state.providers[provider].accruedFeesInWei -= amount;

        (bool sent, ) = msg.sender.call{value: amount}("");
        require(sent, "Failed to send fees");

        emit FeesWithdrawn(msg.sender, amount);
    }
