### Title
`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` Permanently Broken for Whitelist-Enabled Subscriptions Due to `this.function()` External Call — (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`Scheduler.getPricesNoOlderThan` and `Scheduler.getEmaPricesNoOlderThan` internally call `this.getPricesUnsafe(...)` and `this.getEmaPricesUnsafe(...)` as external calls. This changes `msg.sender` to `address(this)` inside those functions. Both target functions are guarded by the `onlyWhitelistedReader` modifier, which checks whether `msg.sender` is the subscription manager or is in the whitelist. Since `address(this)` is never either, every call to `getPricesNoOlderThan` or `getEmaPricesNoOlderThan` on a subscription with `whitelistEnabled = true` unconditionally reverts with `Unauthorized`, permanently breaking freshness-checked price reads for that subscription class.

---

### Finding Description

`getPricesNoOlderThan` (line 554) and `getEmaPricesNoOlderThan` (line 597) delegate to the freshness-unchecked variants via an **external** `this.` call:

```solidity
// getPricesNoOlderThan (line 554)
prices = this.getPricesUnsafe(subscriptionId, priceIds);

// getEmaPricesNoOlderThan (line 597)
prices = this.getEmaPricesUnsafe(subscriptionId, priceIds);
``` [1](#0-0) [2](#0-1) 

Both `getPricesUnsafe` and `getEmaPricesUnsafe` carry the `onlyWhitelistedReader(subscriptionId)` modifier:

```solidity
function getPricesUnsafe(
    uint256 subscriptionId,
    bytes32[] calldata priceIds
)
    external
    view
    override
    onlyWhitelistedReader(subscriptionId)   // <-- checked with msg.sender = address(this)
    ...
``` [3](#0-2) 

The modifier evaluates three conditions in order:

```solidity
modifier onlyWhitelistedReader(uint256 subscriptionId) {
    if (_state.subscriptionManager[subscriptionId] == msg.sender) { _; return; }
    if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) { _; return; }
    // ... check whitelist array for msg.sender ...
    if (!isWhitelisted) revert SchedulerErrors.Unauthorized();
    _;
}
``` [4](#0-3) 

When the call arrives via `this.getPricesUnsafe(...)`, `msg.sender` is `address(this)` — the Scheduler contract itself. The Scheduler is never stored as a subscription manager, and it is never placed in any subscription's `readerWhitelist`. Therefore:

- **`whitelistEnabled = false`**: second branch passes → no issue.
- **`whitelistEnabled = true`**: all three checks fail for `address(this)` → `revert Unauthorized()`.

The result is that `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` are **completely non-functional** for every subscription that uses `whitelistEnabled = true`, regardless of who the actual caller is.

---

### Impact Explanation

Any subscription owner who enables the whitelist feature (`whitelistEnabled = true`) to restrict price-read access loses the ability to use the freshness-enforced read functions `getPricesNoOlderThan` and `getEmaPricesNoOlderThan`. These are the only functions that enforce a maximum data age (`age_seconds`), which is the primary safety guarantee for consumers that need fresh prices. Affected callers are silently forced to fall back to `getPricesUnsafe`, which provides no staleness protection. This is a complete, permanent denial of service for the safe read path on whitelist-enabled subscriptions.

---

### Likelihood Explanation

`whitelistEnabled = true` is an explicitly supported and documented configuration in `SubscriptionParams`. Any subscription manager who sets it — a realistic choice for any protocol that wants to gate price-read access — will immediately and permanently trigger this bug. No special attacker action is required; the breakage is automatic upon subscription creation with `whitelistEnabled = true`. The entry path is fully unprivileged: any user can create such a subscription and observe the failure.

---

### Recommendation

Replace the external `this.` calls with direct calls to the internal helper `_getPricesInternal`, bypassing the access control modifier (which has already been checked in the outer function):

```diff
function getPricesNoOlderThan(...) external view onlyWhitelistedReader(subscriptionId) ... {
    ...
-   prices = this.getPricesUnsafe(subscriptionId, priceIds);
+   PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
+   prices = new PythStructs.Price[](priceFeeds.length);
+   for (uint i = 0; i < priceFeeds.length; i++) { prices[i] = priceFeeds[i].price; }
}

function getEmaPricesNoOlderThan(...) external view onlyWhitelistedReader(subscriptionId) ... {
    ...
-   prices = this.getEmaPricesUnsafe(subscriptionId, priceIds);
+   PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
+   prices = new PythStructs.Price[](priceFeeds.length);
+   for (uint i = 0; i < priceFeeds.length; i++) { prices[i] = priceFeeds[i].emaPrice; }
}
```

The access control check is already enforced by the `onlyWhitelistedReader` modifier on the outer functions, so calling the internal helper directly is safe.

---

### Proof of Concept

1. Deploy `SchedulerUpgradeable`.
2. Call `createSubscription` with `whitelistEnabled = true` and `readerWhitelist = [alice]`, sending sufficient ETH.
3. Have a keeper call `updatePriceFeeds` to populate price data.
4. Have `alice` call `getPricesNoOlderThan(subscriptionId, priceIds, 60)`.
5. Observe: the call reverts with `Unauthorized`, even though `alice` is in the whitelist.
6. Root cause: inside `getPricesUnsafe`, `msg.sender == address(this)` (the Scheduler), which is not `alice` and not in the whitelist, so the modifier reverts.
7. Confirm: `alice` calling `getPricesUnsafe` directly succeeds, proving the whitelist check itself is correct — only the `this.` indirection is broken.

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
