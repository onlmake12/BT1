### Title
Verify Queue Lacks Time-Based Expiry, Enabling Persistent Resource Exhaustion DoS — (`tx-pool/src/component/verify_queue.rs`)

---

### Summary

The `VerifyQueue` in CKB's transaction pool has no time-based expiry mechanism. While `TxPool::remove_expired()` cleans stale entries from the main pool on every block, it does not cover the `verify_queue`. An unprivileged RPC caller or P2P relay peer can fill the verify queue with transactions that are never expired by time, blocking legitimate transaction processing indefinitely until the queue is manually drained.

---

### Finding Description

The CKB tx-pool uses a two-stage pipeline: transactions first enter `VerifyQueue` for script verification, then graduate to the main `pool_map`. The main pool has a well-defined expiry path:

`TxPool::remove_expired()` in `tx-pool/src/pool.rs` iterates only over `self.pool_map`: [1](#0-0) 

This is called from `_update_tx_pool_for_reorg` in `tx-pool/src/process.rs` on every block: [2](#0-1) 

However, `VerifyQueue` in `tx-pool/src/component/verify_queue.rs` has **no `remove_expired` method and no TTL field**. Its only cleanup paths are:
- `remove_tx` / `remove_txs` — explicit manual removal
- `remove_txs_by_peer` — triggered on peer disconnect
- `pop_front` — normal processing by a worker
- `clear` — full wipe [3](#0-2) 

The only guard against unbounded growth is a 256 MB size cap (`DEFAULT_MAX_VERIFY_QUEUE_TX_SIZE`): [4](#0-3) 

There is no periodic or per-block sweep that removes entries from `verify_queue` that have exceeded the pool's configured `expiry` threshold. By contrast, the orphan tx pool (`tx-pool/src/component/orphan.rs`) correctly implements time-based expiry via `ORPHAN_TX_EXPIRE_TIME` and calls `limit_size()` on every insertion: [5](#0-4) [6](#0-5) 

The `VerifyQueue` has no equivalent.

---

### Impact Explanation

An unprivileged RPC caller (`send_transaction`) or P2P relay peer can submit many large-serialized transactions (up to the per-tx size limit) with high declared cycle counts. These are enqueued in `VerifyQueue` and processed slowly by the fixed worker pool. Because there is no time-based eviction, entries that have been waiting past the pool's `expiry` window remain in the queue. Once the 256 MB cap is reached, every subsequent `add_tx` call returns `Reject::Full("verify_queue total_tx_size exceeded")`, blocking all legitimate transaction submissions for the duration of the attack. The node's tx-pool is effectively DoS'd without any on-chain cost to the attacker beyond the initial submission fees (which are never paid if the transactions are invalid or never confirmed).

---

### Likelihood Explanation

The attack is reachable by any unprivileged actor with RPC access or P2P connectivity. Filling 256 MB requires submitting roughly hundreds to low thousands of maximum-size transactions. With high-cycle transactions that stall the two verify workers, the queue drains slowly, sustaining the DoS. No special privilege, key material, or majority hashpower is required.

---

### Recommendation

Add a time-based expiry sweep to `VerifyQueue`, mirroring `TxPool::remove_expired()`. Specifically:

1. Add a `timestamp` field to `VerifyEntry` (already present as `added_time`).
2. Implement a `remove_expired(expiry_ms: u64)` method on `VerifyQueue` that evicts entries whose `added_time + expiry_ms < now`.
3. Call this method from `_update_tx_pool_for_reorg` (in `tx-pool/src/process.rs`) alongside the existing `tx_pool.remove_expired(callbacks)` call, using the same configured `expiry` duration.

---

### Proof of Concept

1. Connect to a CKB node's RPC endpoint.
2. Repeatedly call `send_transaction` with transactions that are syntactically valid but have maximum serialized size and high declared cycle counts (so workers process them slowly).
3. Observe that `VerifyQueue::total_tx_size` grows toward 256 MB.
4. Once the cap is hit, all subsequent `send_transaction` calls return `Reject::Full("verify_queue total_tx_size exceeded")`.
5. Mine or wait for blocks: `_update_tx_pool_for_reorg` calls `tx_pool.remove_expired()` but **not** any expiry sweep on `verify_queue`, so the queue remains full.
6. Legitimate users cannot submit transactions until the attacker's transactions are individually processed by the workers — which, for high-cycle transactions, can take an extended period.

    pub(crate) fn remove_expired(&mut self, callbacks: &Callbacks) {
        let now_ms = ckb_systemtime::unix_time_as_millis();

        let removed: Vec<_> = self
            .pool_map
            .iter()
            .filter(|&entry| self.expiry + entry.inner.timestamp < now_ms)
            .map(|entry| entry.inner.clone())
            .collect();

        for entry in removed {
            let tx_hash = entry.transaction().hash();
            debug!("remove_expired {} timestamp({})", tx_hash, entry.timestamp);
            self.pool_map.remove_entry(&entry.proposal_short_id());
            let reject = Reject::Expiry(entry.timestamp);
            callbacks.call_reject(self, &entry, reject);
        }
    }

    fn limit_size(&mut self) -> Vec<Byte32> {
        let now = ckb_systemtime::unix_time().as_secs();
        let expires: Vec<_> = self
            .entries
            .iter()
            .filter_map(|(id, entry)| {
                if entry.expires_at <= now {
                    Some(id)
                } else {
                    None
                }
            })
            .cloned()
            .collect();

        let mut evicted_txs = vec![];

        for id in expires {
            if let Some(entry) = self.remove_orphan_tx(&id) {
                evicted_txs.push(entry.tx.hash());
            }
        }

        while self.len() > DEFAULT_MAX_ORPHAN_TRANSACTIONS {
            // Evict a random orphan:
            let id = self.entries.keys().next().cloned().expect("bound checked");
            if let Some(entry) = self.remove_orphan_tx(&id) {
                evicted_txs.push(entry.tx.hash());
            }
        }

        if !evicted_txs.is_empty() {
            trace!("OrphanTxPool full, evicted {} tx", evicted_txs.len());
            self.shrink_to_fit();
        }
        evicted_txs
    }
