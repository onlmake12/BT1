### Title
Permissionless Provider Registration with No Deregistration or User Refund Path Permanently Locks User Fees — (`File: target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

`Echo.sol` allows any address to permissionlessly register as a price-update provider. Once registered, a provider cannot be deregistered or paused by the admin. When a user pays fees to request a price update from a malicious provider who never fulfills the request, those fees are permanently locked in the contract — there is no `cancelRequest`, no refund function, and no admin mechanism to recover user funds.

---

### Finding Description

`Echo.registerProvider()` is fully permissionless:

```solidity
function registerProvider(
    uint96 baseFeeInWei,
    uint96 feePerFeedInWei,
    uint96 feePerGasInWei
) external override {
    ProviderInfo storage provider = _state.providers[msg.sender];
    require(!provider.isRegistered, "Provider already registered");
    ...
    provider.isRegistered = true;
    emit ProviderRegistered(msg.sender, feePerGasInWei);
}
``` [1](#0-0) 

Any EOA or contract can register as a provider with zero fees (`baseFeeInWei=0, feePerFeedInWei=0, feePerGasInWei=0`) to attract users. There is no admin approval step and no `deregisterProvider()` function anywhere in the contract or its interface. [2](#0-1) 

When a user calls `requestPriceUpdatesWithCallback()`, the provider fee portion is stored in `req.fee` inside the request struct:

```solidity
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);
``` [3](#0-2) 

This fee is **only** credited to a provider when `executeCallback()` is successfully called:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
``` [4](#0-3) 

If `executeCallback()` is never called, `req.fee` remains locked in the contract indefinitely. The contract's own TODO comment at line 157 explicitly acknowledges this gap:

> "TODO: there also needs to be some penalty mechanism in case the expected provider doesn't execute the callback." [5](#0-4) 

While the exclusivity period allows other providers to fulfill after it expires, this requires off-chain coordination and valid Pyth update data — there is no on-chain incentive for a third party to rescue a victim's request. The admin has no mechanism to pause a specific provider or force-cancel a request. [6](#0-5) 

---

### Impact Explanation

User fees (the provider portion of `msg.value`) are permanently locked in the `Echo` contract with no recovery path. The `withdrawFees()` function is admin-only and only covers the Pyth protocol fee (`_state.accruedFeesInWei`), not `req.fee` stored in individual requests. [7](#0-6) 

There is no `cancelRequest()`, no `refundRequest()`, and no `deregisterProvider()` in the contract or its interface. [8](#0-7) 

---

### Likelihood Explanation

The attack requires only a single permissionless transaction (`registerProvider`) with zero fees to attract users. The attacker needs no privileged access, no leaked keys, and no governance majority. Any user who calls `requestPriceUpdatesWithCallback()` targeting the malicious provider loses their fee. The attack is cheap to execute and the damage is irreversible.

---

### Recommendation

1. **Add a `cancelRequest()` function** that allows the original requester to reclaim `req.fee` after a timeout (e.g., if the request is unfulfilled after `publishTime + exclusivityPeriod + buffer`).
2. **Add an admin-controlled `deregisterProvider()` or `pauseProvider()` function** to block a malicious provider from receiving new requests.
3. **Require admin approval** (or a stake/bond) before a provider can be used in `requestPriceUpdatesWithCallback()`, analogous to a whitelist.

---

### Proof of Concept

```solidity
// 1. Attacker registers as a provider with zero fees
vm.prank(attacker);
echo.registerProvider(0, 0, 0);

// 2. User requests a price update from the attacker's provider
bytes32[] memory priceIds = new bytes32[](1);
priceIds[0] = somePriceId;
uint96 fee = echo.getFee(attacker, callbackGasLimit, priceIds);
vm.deal(user, fee);
vm.prank(user);
uint64 seq = echo.requestPriceUpdatesWithCallback{value: fee}(
    attacker,
    block.timestamp,
    priceIds,
    callbackGasLimit
);

// 3. Attacker never calls executeCallback()
// req.fee is now permanently locked in the contract.
// No cancelRequest(), no refund, no admin recovery path.
// User's fee (fee - pythFeeInWei) is irrecoverable.
assertEq(echo.getRequest(seq).fee, fee - echo.getPythFeeInWei());
// No function exists to reclaim this.
```

interface IEcho is EchoEvents {
    // Core functions
    /**
     * @notice Requests price updates with a callback
     * @dev The msg.value must be equal to getFee(callbackGasLimit)
     * @param provider The provider to fulfill the request
     * @param publishTime The minimum publish time for price updates, it should be less than or equal to block.timestamp + 60
     * @param priceIds The price feed IDs to update. Maximum 10 price feeds per request.
     *        Requests requiring more feeds should be split into multiple calls.
     * @param callbackGasLimit The amount of gas allocated for the callback execution
     * @return sequenceNumber The sequence number assigned to this request
     * @dev Security note: The 60-second future limit on publishTime prevents a DoS vector where
     *      attackers could submit many low-fee requests for far-future updates when gas prices
     *      are low, forcing executors to fulfill them later when gas prices might be much higher.
     *      Since tx.gasprice is used to calculate fees, allowing far-future requests would make
     *      the fee estimation unreliable.
     */
    function requestPriceUpdatesWithCallback(
        address provider,
        uint64 publishTime,
        bytes32[] calldata priceIds,
        uint32 callbackGasLimit
    ) external payable returns (uint64 sequenceNumber);

    /**
     * @notice Executes the callback for a price update request
     * @dev Requires 1.5x the callback gas limit to account for cross-contract call overhead
     * For example, if callbackGasLimit is 1M, the transaction needs at least 1.5M gas + some gas for some other operations in the function before the callback
     * @param providerToCredit The provider to credit for fulfilling the request. This may not be the provider that submitted the request (if the exclusivity period has elapsed).
     * @param sequenceNumber The sequence number of the request
     * @param updateData The raw price update data from Pyth
     * @param priceIds The price feed IDs to update, must match the request
     */
    function executeCallback(
        address providerToCredit,
        uint64 sequenceNumber,
        bytes[] calldata updateData,
        bytes32[] calldata priceIds
    ) external payable;

    // Getters
    /**
     * @notice Gets the base fee charged by Pyth protocol
     * @dev This is a fixed fee per request that goes to the Pyth protocol, separate from gas costs
     * @return pythFeeInWei The base fee in wei that every request must pay
     */
    function getPythFeeInWei() external view returns (uint96 pythFeeInWei);

    /**
     * @notice Calculates the total fee required for a price update request
     * @dev Total fee = base Pyth protocol fee + base provider fee + provider fee per feed + gas costs for callback
     * @param provider The provider to fulfill the request
     * @param callbackGasLimit The amount of gas allocated for callback execution
     * @param priceIds The price feed IDs to update.
     * @return feeAmount The total fee in wei that must be provided as msg.value
     */
