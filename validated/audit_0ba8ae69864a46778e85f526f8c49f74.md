### Title
Unbounded Prometheus Metrics Family Growth via Arbitrary URI Paths — (`apps/hermes/server/src/api/metrics_middleware.rs`)

### Summary

The `track_metrics` middleware in Hermes falls back to the raw request URI path when no `MatchedPath` extension is present. Because `prometheus_client`'s `Family` type stores one counter per unique label set in an internal `HashMap` that is never pruned, any unprivileged caller can force unbounded heap growth by sending requests to an unlimited number of distinct paths.

### Finding Description

In `track_metrics`, the `path` variable is resolved as follows:

```rust
let path = if let Some(matched_path) = req.extensions().get::<MatchedPath>() {
    matched_path.as_str().to_owned()
} else {
    req.uri().path().to_owned()   // ← raw, attacker-controlled string
};
``` [1](#0-0) 

`MatchedPath` is only populated by Axum's router when a route actually matches. For every request that does not match a registered route (i.e., any 404), Axum still runs all `Router::layer` middleware, but `MatchedPath` is absent, so the raw URI path is used.

That path is then embedded in a `Labels` struct and passed to `get_or_create`:

```rust
api_state.metrics.requests.get_or_create(&labels).inc();
``` [2](#0-1) 

`Family::get_or_create` inserts a new `Counter` into an internal `HashMap<Labels, Counter>` for every previously-unseen label combination. There is no eviction, no size cap, and no TTL on this map. Each unique path permanently occupies heap memory for the lifetime of the process.

The same `path` string is also inserted into the `latencies` `Family<LatencyLabels, Histogram>`:

```rust
api_state.metrics.latencies.get_or_create(&latency_labels).observe(latency);
``` [3](#0-2) 

A `Histogram` entry is significantly larger than a `Counter` entry (it stores bucket arrays), so the per-path allocation is non-trivial.

Additionally, the `Labels` struct includes a `token_suffix` field (last 4 characters of the caller's `Authorization: Bearer` token): [4](#0-3) 

This multiplies the cardinality: for each unique `(path, token_suffix)` pair, a separate entry is created. An attacker rotating tokens while also varying paths amplifies the leak rate.

The analogous root cause in the SEDA report is a per-call allocation (`C.CString`) that is never freed. Here the per-request allocation is a `HashMap` entry (plus a `Histogram` with bucket arrays) that is never removed.

### Impact Explanation

A Hermes node operator suffers a continuous, unbounded heap growth. Each unique `(method, path, status, token_suffix)` tuple permanently allocates at minimum one `Counter` and one `Histogram` (with ~11 f64 buckets) on the heap. Sending 1 million distinct paths consumes hundreds of megabytes. Over extended operation the process will exhaust available memory, causing degraded performance and eventual OOM termination of the Hermes price-feed relay, making price updates unavailable to all downstream consumers.

### Likelihood Explanation

The Hermes HTTP API is publicly reachable. No authentication is required to trigger the fallback path (unauthenticated requests to non-existent routes still pass through the middleware). An attacker needs only to send HTTP GET requests with incrementing path segments (e.g., `/x/1`, `/x/2`, …) at any rate. No special privileges, keys, or on-chain interaction are required.

### Recommendation

1. **Never use the raw URI path as a metrics label.** Replace the fallback with a fixed sentinel string such as `"unknown"` or `"unmatched"`:

```rust
let path = req.extensions()
    .get::<MatchedPath>()
    .map(|p| p.as_str().to_owned())
    .unwrap_or_else(|| "unmatched".to_owned());
```

2. **Validate or truncate `token_suffix`** to a fixed allow-list of known suffixes, or omit it from the label set entirely, to prevent cardinality explosion on the `requests` Family even for matched routes.

### Proof of Concept

```bash
# Send 100 000 requests to distinct paths; each creates a permanent heap entry
for i in $(seq 1 100000); do
  curl -s "https://hermes.pyth.network/leak/$i" > /dev/null &
done
wait
# Hermes heap now contains ≥100 000 permanent Counter + Histogram entries
# that will never be freed for the lifetime of the process.
```

The `requests` Family at `apps/hermes/server/src/api/metrics_middleware.rs` line 179 and the `latencies` Family at line 183 will each hold 100 000 entries. Repeating this periodically will eventually exhaust the node's memory. [5](#0-4)

```rust
pub async fn track_metrics<B, S>(
    State(api_state): State<ApiState<S>>,
    req: Request<B>,
    next: Next<B>,
) -> impl IntoResponse {
    let start = Instant::now();
    let path = if let Some(matched_path) = req.extensions().get::<MatchedPath>() {
        matched_path.as_str().to_owned()
    } else {
        req.uri().path().to_owned()
    };

    let method = req.method().clone();

    // Extract the token from the request
    let token = token::extract_token_from_headers_and_uri(req.headers(), req.uri());
    let token_suffix = token::get_token_suffix(token.as_deref());

    let response = next.run(req).await;
    let latency = start.elapsed().as_secs_f64();
    let status = response.status().as_u16();
    let labels = Labels {
        method: method.to_string(),
        path: path.clone(),
        status,
        token_suffix,
    };
    let latency_labels = LatencyLabels {
        method: method.to_string(),
        path,
        status,
    };

    api_state.metrics.requests.get_or_create(&labels).inc();
    api_state
        .metrics
        .latencies
        .get_or_create(&latency_labels)
        .observe(latency);

    response
}
