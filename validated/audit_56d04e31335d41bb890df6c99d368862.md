### Title
Malicious Peer Can Delay Block Synchronization via Crafted Compact Block Poisoning of `pending_compact_blocks` Gate — (`sync/src/synchronizer/block_fetcher.rs`)

---

### Summary

A malicious peer can send a crafted `CompactBlock` message (valid header/PoW, but with a wrong nonce so short IDs do not match the victim's mempool) for any newly-propagated block. This causes the block hash to be inserted into `pending_compact_blocks` with the current timestamp. For the next 2 seconds, `compare_with_pending_compact` returns `false` for that hash, and the synchronizer's `BlockFetcher` silently skips requesting the block from any peer. The attacker can repeat this for every new block, creating a sustained per-block delay that causes miners to waste hashpower on stale tips.

---

### Finding Description

**Gate function — `sync/src/types/mod.rs`**

`compare_with_pending_compact` is called as a prerequisite gate inside `BlockFetcher::fetch` before a block is added to the inflight download queue: [1](#0-0) 

It returns `false` (blocking the fetch) when:
- `pending_compact_blocks` is **not** empty, AND
- the target `hash` **is** present in the map, AND
- fewer than 2 000 ms have elapsed since the entry was created.

**Gate call site — `sync/src/synchronizer/block_fetcher.rs`** [2](#0-1) 

When `compare_with_pending_compact` returns `false` (and the node is not in IBD), the block is **not** pushed into `fetch`, so no `GetBlocks` request is issued for it.

**How entries enter `pending_compact_blocks`**

In `compact_block_process.rs`, when a compact block cannot be reconstructed (missing transactions or short-ID collision), `missing_or_collided_post_process` inserts the block hash with `unix_time_as_millis()` as the timestamp: [3](#0-2) 

The `or_insert_with` call means the timestamp is set **only on first insertion** and is never refreshed by subsequent failed reconstructions from the same or different peers.

**Admission checks an attacker must pass**

Before reaching `missing_or_collided_post_process`, a compact block must pass:

1. `non_contextual_check` — uncles/proposals count limits, and block height within `epoch_length` of the current tip. [4](#0-3) 
2. `contextual_check` — block not already stored, parent known, header PoW valid. [5](#0-4) 

An attacker does **not** need to mine a new block. They observe a legitimately-propagated block, copy its header verbatim (preserving valid PoW), but substitute a different `nonce` field in the compact block encoding. The receiver uses this nonce to derive short IDs from its mempool; a wrong nonce produces short IDs that match nothing, triggering the "missing transactions" path and the insertion into `pending_compact_blocks`.

**Secondary issue — `blocking_lock()` on a `tokio::sync::Mutex`**

`compare_with_pending_compact` acquires the lock with `blocking_lock()`: [6](#0-5) 

The same mutex is held (via `.await`) during header verification and block reconstruction in the async relayer tasks. If an attacker floods the node with many crafted compact blocks concurrently, the async tasks hold the mutex for extended periods, and `blocking_lock()` stalls the Tokio worker thread that runs the synchronizer, amplifying the DoS.

---

### Impact Explanation

- **Miners**: A 2-second delay per block means the miner is always building on a tip that is one block behind the network. With CKB's ~10-second target block time, this represents ~20% wasted hashpower per attack cycle.
- **Normal nodes**: Sustained delay in receiving the canonical tip degrades transaction confirmation latency and can cause the node to appear out-of-sync.
- The attacker can sustain the attack indefinitely by sending one crafted compact block per new block propagation event — a very low bandwidth cost.

---

### Likelihood Explanation

- The attacker only needs a single peer connection to the victim.
- No PoW mining is required; the attacker reuses the legitimate block's header.
- The attacker must win a race against the legitimate compact block relay, which is feasible with a well-positioned network node (e.g., co-located with a mining pool).
- No privileged access, leaked keys, or majority hashpower is required.

---

### Recommendation

1. **Short term**: In `missing_or_collided_post_process`, verify that the peer sending the compact block is the same peer from whom the block was originally expected (e.g., cross-check against the inflight block map). Reject compact blocks whose short IDs produce a miss rate above a configurable threshold (analogous to the `buyAmount` minimum check in the original report).

2. **Short term**: Replace `blocking_lock()` in `compare_with_pending_compact` with a non-blocking `try_lock()` that returns `true` (allow fetch) on contention, so the synchronizer is never stalled by async relayer tasks holding the mutex.

3. **Long term**: Rate-limit `CompactBlock` messages per peer and per block hash. Penalise or disconnect peers that repeatedly send compact blocks that fail reconstruction for the same block hash.

---

### Proof of Concept

1. Victim node `V` is at tip `N`. A new block `N+1` is mined and begins propagating.
2. Attacker peer `A` (connected to `V`) receives block `N+1` from the network.
3. `A` constructs a `CompactBlock` message: copies the header of `N+1` exactly (valid PoW, valid parent), but sets the compact block `nonce` to an arbitrary value different from the legitimate one.
4. `A` sends this message to `V` before the legitimate compact block arrives.
5. `V` passes `non_contextual_check` and `contextual_check` (header is valid), calls `reconstruct_block`, which fails because the wrong nonce produces short IDs that match nothing in `V`'s mempool.
6. `missing_or_collided_post_process` inserts `hash(N+1)` into `pending_compact_blocks` with `timestamp = now`.
7. For the next 2 000 ms, every call to `BlockFetcher::fetch` skips `hash(N+1)` because `compare_with_pending_compact` returns `false`.
8. `V` does not issue a `GetBlocks` request for `N+1` during this window; it continues mining on tip `N`.
9. After 2 seconds, `V` finally fetches `N+1` — but `A` immediately repeats step 3 for block `N+2`, sustaining the delay indefinitely.

/// * check compact block's uncles and proposals length
/// * check compact block height
fn non_contextual_check(
    compact_block: &CompactBlock,
    header: &HeaderView,
    consensus: &Consensus,
    active_chain: &ActiveChain,
) -> Status {
    if compact_block.uncles().len() > consensus.max_uncles_num() {
        return StatusCode::ProtocolMessageIsMalformed.with_context(format!(
            "CompactBlock uncles count({}) > consensus max_uncles_num({})",
            compact_block.uncles().len(),
            consensus.max_uncles_num()
        ));
    }
    if (compact_block.proposals().len() as u64) > consensus.max_block_proposals_limit() {
        return StatusCode::ProtocolMessageIsMalformed.with_context(format!(
            "CompactBlock proposals count({}) > consensus max_block_proposals_limit({})",
            compact_block.proposals().len(),
            consensus.max_block_proposals_limit(),
        ));
    }

    // Only accept blocks with a height greater than tip - N
    // where N is the current epoch length
    let block_hash = header.hash();
    let tip = active_chain.tip_header();
    let epoch_length = active_chain.epoch_ext().length();
    let lowest_number = tip.number().saturating_sub(epoch_length);

    if lowest_number > header.number() {
        return StatusCode::CompactBlockIsStaled.with_context(block_hash);
    }

    Status::ok()
}

async fn contextual_check(
    compact_block_header: &HeaderView,
    shared: &Arc<SyncShared>,
    active_chain: &ActiveChain,
    nc: &Arc<dyn CKBProtocolContext + Sync>,
    peer: PeerIndex,
) -> Status {
    let block_hash = compact_block_header.hash();
    let tip = active_chain.tip_header();

    let status = active_chain.get_block_status(&block_hash);
    if status.contains(BlockStatus::BLOCK_STORED) {
        // update last common header and best known
        let parent = shared
            .get_header_index_view(&compact_block_header.data().raw().parent_hash(), true)
            .expect("parent block must exist");

        let header_index = HeaderIndex::new(
            compact_block_header.number(),
            block_hash.clone(),
            parent.total_difficulty() + compact_block_header.difficulty(),
        );
        let state = shared.state().peers();
        state.may_set_best_known_header(peer, header_index);

        return StatusCode::CompactBlockAlreadyStored.with_context(block_hash);
    } else if status.contains(BlockStatus::BLOCK_RECEIVED) {
        // block already in orphan pool
        return Status::ignored();
    } else if status.contains(BlockStatus::BLOCK_INVALID) {
        return StatusCode::BlockIsInvalid.with_context(block_hash);
    }

    let store_first = tip.number() + 1 >= compact_block_header.number();
    let parent = shared.get_header_index_view(
        &compact_block_header.data().raw().parent_hash(),
        store_first,
    );
    if parent.is_none() {
        debug_target!(
            crate::LOG_TARGET_RELAY,
            "UnknownParent: {}, send_getheaders_to_peer({})",
            block_hash,
            peer
        );
        active_chain.send_getheaders_to_peer(nc, peer, (&tip).into());
        return StatusCode::CompactBlockRequiresParent.with_context(format!(
            "{} parent: {}",
            block_hash,
            compact_block_header.data().raw().parent_hash(),
        ));
    }

    // compact block is in pending
    let pending_compact_blocks = shared.state().pending_compact_blocks().await;
    if pending_compact_blocks
        .get(&block_hash)
        .map(|(_, peers_map, _)| peers_map.contains_key(&peer))
        .unwrap_or(false)
    {
        return StatusCode::CompactBlockIsAlreadyPending.with_context(block_hash);
    }

    // compact header verification
    let fn_get_pending_header = {
        |block_hash| {
            pending_compact_blocks
                .get(&block_hash)
                .map(|(compact_block, _, _)| {
                    let header = compact_block.header().into_view();
                    HeaderFields {
                        hash: header.hash(),
                        number: header.number(),
                        epoch: header.epoch(),
                        timestamp: header.timestamp(),
                        parent_hash: header.parent_hash(),
                    }
                })
                .or_else(|| {
                    shared
                        .get_header_index_view(&block_hash, false)
                        .map(|header| HeaderFields {
                            hash: header.hash(),
                            number: header.number(),
                            epoch: header.epoch(),
                            timestamp: header.timestamp(),
                            parent_hash: header.parent_hash(),
                        })
                })
        }
    };
    let median_time_context = CompactBlockMedianTimeView {
        fn_get_pending_header: Box::new(fn_get_pending_header),
    };
    let header_verifier = HeaderVerifier::new(&median_time_context, shared.consensus());
    if let Err(err) = header_verifier.verify(compact_block_header) {
        if err
            .downcast_ref::<HeaderError>()
            .map(|e| e.is_too_new())
            .unwrap_or(false)
        {
            return Status::ignored();
        } else {
            shared
                .shared()
                .insert_block_status(block_hash.clone(), BlockStatus::BLOCK_INVALID);
            return StatusCode::CompactBlockHasInvalidHeader
                .with_context(format!("{block_hash} {err}"));
        }
    }

    Status::ok()
}
