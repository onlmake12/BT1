### Title
Unguarded Authority Initialization in `set_parameters` Allows Any Signer to Seize Control of Stake Cap Parameters — (File: `pythnet/stake_caps_parameters/programs/stake_caps_parameters/src/lib.rs`)

---

### Summary

The `stake_caps_parameters` program's sole instruction `set_parameters` contains a bypass condition that allows **any unprivileged signer** to call it when the on-chain `Parameters` account is uninitialized (or when `current_authority` has been reset to `Pubkey::default()`). An attacker can front-run legitimate initialization, set themselves as `current_authority`, and write arbitrary `m` and `z` stake-cap values.

---

### Finding Description

The `set_parameters` instruction uses `init_if_needed` on the `parameters` PDA, meaning the account is created with all-zero data on first invocation. The authority check is:

```rust
require!(
    ctx.accounts.signer.key() == stored_parameters.current_authority
        || stored_parameters.current_authority == Pubkey::default(),
    ErrorCode::WrongAuthority
);
**stored_parameters = parameters;
``` [1](#0-0) 

Because a freshly initialized account has `current_authority == Pubkey::default()` (all zeros), the second branch of the `require!` always passes for any signer on first call. The instruction then overwrites the **entire** `Parameters` struct — including `current_authority` — with attacker-supplied data:

```rust
**stored_parameters = parameters;
``` [2](#0-1) 

The account struct definition confirms all three fields (`current_authority`, `m`, `z`) are fully attacker-controlled:

```rust
pub struct Parameters {
    pub current_authority: Pubkey,
    pub m: u64,
    pub z: u64,
}
``` [3](#0-2) 

A secondary trigger exists even after initialization: if the legitimate authority ever calls `set_parameters` with `parameters.current_authority = Pubkey::default()` (e.g., by mistake or a misconfigured script), the bypass re-opens permanently.

The `signer` account carries no Anchor `address`, `has_one`, or `constraint` attribute — it is a bare `Signer<'info>` with only `#[account(mut)]`:

```rust
pub struct SetParameters<'info> {
    #[account(mut)]
    pub signer: Signer<'info>,
    ...
}
``` [4](#0-3) 

---

### Impact Explanation

`m` and `z` are the two parameters that govern stake caps for publishers in the integrity pool reward system. An attacker who seizes `current_authority` can:

1. Set `m = 0` or `z = 0` to collapse all publisher stake caps to zero, effectively zeroing out publisher rewards or breaking the cap computation.
2. Set `m` / `z` to extreme values to inflate or deflate caps, distorting the staking economics for all delegators and publishers.
3. Permanently lock out the legitimate Pyth governance authority by setting `current_authority` to an address they control.

This constitutes unauthorized manipulation of core staking/reward accounting parameters — a direct protocol-level impact on the integrity pool.

---

### Likelihood Explanation

The attack window is the period between program deployment and the first legitimate call to `set_parameters`. Because the PDA address is deterministic (`seeds = ["parameters"]`), an attacker can monitor the mempool for the legitimate initialization transaction and front-run it. On Solana, front-running is feasible by submitting a transaction with a higher priority fee. The secondary trigger (authority reset to default) is lower likelihood but non-zero given operational error.

---

### Recommendation

Replace the `init_if_needed` + default-pubkey bypass pattern with a two-step approach:

1. Add a dedicated `initialize` instruction that is gated by a **hard-coded** admin pubkey (analogous to Anchor's `address` constraint), similar to how `message_buffer` uses `has_one = admin` on its `UpdateWhitelist` context.
2. In `set_parameters`, remove the `|| stored_parameters.current_authority == Pubkey::default()` branch entirely and enforce `signer.key() == stored_parameters.current_authority` unconditionally.

```rust
// Recommended: hard-code the initial authority
pub const INITIAL_AUTHORITY: Pubkey = pubkey!("<governance_pubkey>");

#[account(mut, address = INITIAL_AUTHORITY)]
pub signer: Signer<'info>,
``` [5](#0-4) 

---

### Proof of Concept

```
1. Observe that the `parameters` PDA (seeds=["parameters"]) does not yet exist on-chain
   (or current_authority == Pubkey::default()).

2. Craft a transaction calling set_parameters with:
     signer        = attacker_keypair
     parameters    = Parameters {
                       current_authority: attacker_pubkey,
                       m: 0,   // collapses all stake caps
                       z: 0,
                     }

3. The require! check evaluates:
     attacker_pubkey == Pubkey::default()  → false
     Pubkey::default() == Pubkey::default() → TRUE  ← bypass

4. **stored_parameters = parameters executes, writing attacker_pubkey as
   current_authority and 0/0 as m/z.

5. Legitimate governance can no longer call set_parameters
   (their key ≠ attacker_pubkey), and stake caps are now broken.
```
