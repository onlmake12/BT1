### Title
Immediate Trusted-Signer Removal Without Timelock Allows Owner to Front-Run In-Flight `verifyUpdate` Calls, and Single-Step Ownership Transfer Risks Permanent Contract Lock - (`File: lazer/contracts/evm/src/PythLazer.sol`)

### Summary

`PythLazer` inherits `OwnableUpgradeable` (single-step ownership) instead of `Ownable2StepUpgradeable`, and `updateTrustedSigner` takes effect in the same block it is called. These two properties together mirror the exact vulnerability class in the zAuction report: (1) a privileged role can silently change critical contract state with no advance notice, causing in-flight user transactions to revert unexpectedly, and (2) a single mistaken `transferOwnership` call permanently bricks the contract with no recovery path.

### Finding Description

`PythLazer` is a UUPS-upgradeable contract that gates all price-update verification on a set of trusted signers managed by the owner.

**Root cause 1 – immediate signer removal:**
`updateTrustedSigner(address, 0)` deletes a signer from both the array and the mapping atomically in the same transaction. [1](#0-0) 

`verifyUpdate` then calls `isValidSigner`, which reads `trustedSignerToExpiresAtMapping[signer]`. If the signer was just removed, the mapping entry is `0`, and `block.timestamp < 0` is always false, so the call reverts with `"invalid signer"`. [2](#0-1) [3](#0-2) 

There is no timelock, no pending-removal queue, and no grace period. The change is irreversible from the user's perspective the moment the owner's transaction is mined.

**Root cause 2 – single-step ownership:**
`PythLazer` inherits `OwnableUpgradeable`, which makes `transferOwnership` take effect immediately with no confirmation step. [4](#0-3) 

Every other upgradeable contract in the same repo uses `Ownable2StepUpgradeable`: [5](#0-4) [6](#0-5) [7](#0-6) 

`PythLazer` is the sole exception. The deployment script confirms the single-step pattern is actively used in production: [8](#0-7) 

### Impact Explanation

**Signer front-running:** A Lazer consumer submits a `verifyUpdate` transaction to the mempool. The owner observes it and front-runs with `updateTrustedSigner(signer, 0)`. The consumer's transaction lands in the next block and reverts with `"invalid signer"`. The consumer paid gas and received no service. Because `verifyUpdate` is the sole on-chain verification entry point for Lazer price data, any downstream contract that depends on it (e.g., a DeFi protocol calling `verifyUpdate` before executing a trade) is silently denied service with no warning.

**Permanent contract lock:** If the owner calls `transferOwnership(wrongAddress)` — even accidentally — ownership transfers immediately. The wrong address cannot call `acceptOwnership` (no such function exists in `OwnableUpgradeable`). All `onlyOwner` functions become permanently inaccessible:
- `updateTrustedSigner` — no new signers can be added; existing signers expire and cannot be renewed
- `_authorizeUpgrade` — the contract can never be upgraded or fixed [9](#0-8) 

The result is a permanently bricked Lazer EVM contract: all `verifyUpdate` calls eventually revert once every signer's `expiresAt` timestamp passes, with no recovery path.

### Likelihood Explanation

- The owner is a governance executor contract (per the deployment script), so the signer-removal front-run requires a governance action to be timed maliciously or accidentally against a pending user transaction. On chains with a public mempool this is straightforward.
- The single-step ownership risk is realized any time a governance proposal encodes a wrong target address — a realistic operational error given the multi-chain deployment complexity visible in the contract manager scripts.
- All other Pyth upgradeable contracts already use `Ownable2StepUpgradeable`, confirming the team is aware of the pattern; `PythLazer` is an oversight.

### Recommendation

1. Replace `OwnableUpgradeable` with `Ownable2StepUpgradeable` in `PythLazer`, matching `EntropyUpgradable`, `ExecutorUpgradable`, and `SchedulerUpgradeable`.
2. Introduce a two-step signer removal: a `proposeSigner Removal(address)` that records a pending removal with a timestamp, and a `commitSignerRemoval(address)` callable only after a mandatory delay (e.g., 48 hours). This gives in-flight `verifyUpdate` callers time to observe the pending change and react.
3. Validate that the new owner address is non-zero before accepting any ownership transfer (already enforced by `Ownable2StepUpgradeable`'s `pendingOwner` pattern).

    function updateTrustedSigner(
        address trustedSigner,
        uint256 expiresAt
    ) external onlyOwner {
        if (expiresAt == 0) {
            for (uint8 i = 0; i < trustedSigners.length; i++) {
                if (trustedSigners[i].pubkey == trustedSigner) {
                    trustedSigners[i].pubkey = address(0);
                    trustedSigners[i].expiresAt = 0;
                    delete trustedSignerToExpiresAtMapping[trustedSigner];
                    return;
                }
            }
            revert("no such pubkey");
        } else {
            for (uint8 i = 0; i < trustedSigners.length; i++) {
                if (trustedSigners[i].pubkey == trustedSigner) {
                    trustedSigners[i].expiresAt = expiresAt;
                    trustedSignerToExpiresAtMapping[trustedSigner] = expiresAt;
                    return;
                }
            }
            // Signer not found - adding a new signer.
            for (uint8 i = 0; i < trustedSigners.length; i++) {
                if (trustedSigners[i].pubkey == address(0)) {
                    trustedSigners[i].pubkey = trustedSigner;
                    trustedSigners[i].expiresAt = expiresAt;
                    trustedSignerToExpiresAtMapping[trustedSigner] = expiresAt;
                    return;
                }
            }
            revert("no space for new signer");
        }
    }

        bytes calldata update
    ) external payable returns (bytes calldata payload, address signer) {
        // Require fee and refund excess
        require(msg.value >= verification_fee, "Insufficient fee provided");
        if (msg.value > verification_fee) {
            payable(msg.sender).transfer(msg.value - verification_fee);
        }

        if (update.length < 71) {
            revert("input too short");
        }
        uint32 EVM_FORMAT_MAGIC = 706910618;

        uint32 evm_magic = uint32(bytes4(update[0:4]));
        if (evm_magic != EVM_FORMAT_MAGIC) {
            revert("invalid evm magic");
        }
        uint16 payload_len = uint16(bytes2(update[69:71]));
        if (update.length < 71 + payload_len) {
            revert("input too short");
        }
        payload = update[71:71 + payload_len];
        bytes32 hash = keccak256(payload);
        (signer, , ) = ECDSA.tryRecover(
            hash,

            bytes32(update[4:36]),

        );

            revert("invalid signature");
        }
