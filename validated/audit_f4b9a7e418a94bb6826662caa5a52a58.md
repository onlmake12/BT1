### Title
Unregistered Treasury Address Causes Permanent DoS of Lazer Verification — (`File: lazer/contracts/aptos/sources/pyth_lazer.move`)

---

### Summary

In `pyth_lazer.move`, the `verify_message` function calls `coin::deposit(storage.treasury, fee)` without first checking whether the treasury address has a registered `CoinStore<AptosCoin>`. On Aptos, `coin::deposit` aborts with `ECOIN_STORE_NOT_PUBLISHED` if the recipient has no registered coin store. If the treasury address set during `initialize` is not registered for `AptosCoin`, every call to `verify_message` and `verify_message_with_funder` will abort, permanently DoS-ing the entire Lazer price verification system on Aptos.

---

### Finding Description

`initialize` accepts an arbitrary `treasury: address` with no registration check: [1](#0-0) 

The treasury address is stored immutably in `Storage`: [2](#0-1) 

There is no `set_treasury` or admin function to update it after deployment. Every fee-paying call routes through `verify_message`, which unconditionally calls: [3](#0-2) 

In Aptos Move, `coin::deposit` aborts if the recipient address does not have a `CoinStore<AptosCoin>` registered. There is no `coin::is_account_registered<AptosCoin>(storage.treasury)` guard anywhere in the function. The public entry point `verify_message_with_funder` also routes through this path: [4](#0-3) 

---

### Impact Explanation

If the treasury address is not registered for `AptosCoin` at the time of deployment (or becomes inaccessible), **all calls to `verify_message` and `verify_message_with_funder` abort**. Since there is no `set_treasury` function, the contract cannot be reconfigured without a full package upgrade. This is a complete, permanent DoS of the Lazer price verification system on Aptos — no user or integrator can verify any Lazer price update.

---

### Likelihood Explanation

The `initialize` function accepts any `address` as treasury with no on-chain enforcement that it is registered for `AptosCoin`. A fresh account, a resource account without a coin store, or a typo in the treasury address at deployment time would trigger this condition. Because there is no recovery path (no `set_treasury`), a single misconfiguration permanently bricks the contract. The risk is elevated because the treasury address is a deployment-time parameter with no validation gate.

---

### Recommendation

Add a `coin::is_account_registered<AptosCoin>(treasury)` check inside `initialize` before storing the treasury address:

```move
use aptos_framework::coin;

public entry fun initialize(
    account: &signer, admin: address, treasury: address
) {
    assert!(signer::address_of(account) == @pyth_lazer, ENO_PERMISSIONS);
    assert!(
        coin::is_account_registered<AptosCoin>(treasury),
        ETREASURY_NOT_REGISTERED
    );
    // ... rest of initialization
}
```

Additionally, consider adding a privileged `set_treasury` entry function so the treasury can be updated by the admin without requiring a full contract upgrade.

---

### Proof of Concept

1. Deploy `pyth_lazer` with `initialize(&contract, admin_addr, @0xDEAD)` where `@0xDEAD` has never called `coin::register<AptosCoin>()`.
2. Call `verify_message_with_funder(&user, message, signature, trusted_signer)` with a valid message, valid signature, and sufficient fee.
3. Execution reaches `coin::deposit(storage.treasury, fee)` at line 118.
4. Aptos VM aborts with `ECOIN_STORE_NOT_PUBLISHED` because `@0xDEAD` has no `CoinStore<AptosCoin>`.
5. The entire transaction is rolled back (user fee is not lost, but verification is impossible).
6. Every subsequent call to `verify_message` or `verify_message_with_funder` aborts identically — permanent DoS with no on-chain recovery path. [5](#0-4)

    public fun verify_message(
        message: vector<u8>,
        signature: vector<u8>,
        trusted_signer: vector<u8>,
        fee: coin::Coin<AptosCoin>
    ) acquires Storage {
        let storage = borrow_global<Storage>(@pyth_lazer);

        // Verify fee amount
        assert!(coin::value(&fee) >= storage.single_update_fee, EINSUFFICIENT_FEE);

        // Transfer fee to treasury
        coin::deposit(storage.treasury, fee);

        // Verify signer is trusted and not expired
        let i = 0;
        let valid = false;
        while (i < storage.trusted_signers.length()) {
            let signer_info = vector::borrow(&storage.trusted_signers, (i as u64));
            if (&signer_info.pubkey == &trusted_signer
                && signer_info.expires_at > timestamp::now_seconds()) {
                valid = true;
                break
            };
            i = i + 1;
        };
        assert!(valid, EINVALID_SIGNER);

        // Verify signature
        let sig = ed25519::new_signature_from_bytes(signature);
        let pk = ed25519::new_unvalidated_public_key_from_bytes(trusted_signer);
        assert!(
            ed25519::signature_verify_strict(&sig, &pk, message),
            EINVALID_SIGNATURE
        );
    }
