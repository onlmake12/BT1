### Title
`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` Always Revert for Whitelist-Enabled Subscriptions Due to `this.` External Call Bypassing `onlyWhitelistedReader` - (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

In `Scheduler.sol`, `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` each carry the `onlyWhitelistedReader` modifier and then delegate to `this.getPricesUnsafe` / `this.getEmaPricesUnsafe` via an **external call**. Because `this.` triggers a new EVM call frame, `msg.sender` inside the callee becomes the **Scheduler contract address itself**, which is neither the subscription manager nor a whitelisted reader. When `whitelistEnabled` is `true`, the inner call always reverts with `Unauthorized`, making both "safe" price-reading functions permanently broken for any whitelist-enabled subscription.

---

### Finding Description

`getPricesNoOlderThan` passes the `onlyWhitelistedReader` check for the original caller, then calls:

```solidity
prices = this.getPricesUnsafe(subscriptionId, priceIds);
```

`this.` forces an external call. Inside `getPricesUnsafe`, `msg.sender` is now the Scheduler contract itself. The `onlyWhitelistedReader` modifier evaluates:

1. Is `msg.sender` (= Scheduler) the subscription manager? → No.
2. Is `whitelistEnabled` false? → No (it is `true`).
3. Is `msg.sender` (= Scheduler) in `readerWhitelist`? → No.

Result: `revert SchedulerErrors.Unauthorized()`.

The identical pattern exists in `getEmaPricesNoOlderThan` → `this.getEmaPricesUnsafe`.

---

### Impact Explanation

Any subscription created with `whitelistEnabled = true` loses access to `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` — the two functions explicitly recommended as the safer alternatives to the `*Unsafe` variants. Whitelisted readers who call these functions always receive a revert. The only workaround is to call `getPricesUnsafe` / `getEmaPricesUnsafe` directly, which removes the staleness protection and defeats the purpose of the safer API. This is a functional breakage of a core feature for all whitelist-enabled subscriptions.

---

### Likelihood Explanation

The whitelist feature is a first-class, documented feature of the Scheduler contract. Any subscription manager who enables it and whose consumers call the recommended `getPricesNoOlderThan` or `getEmaPricesNoOlderThan` will hit this revert unconditionally. No special attacker action is required — normal usage triggers the bug.

---

### Recommendation

Replace the external `this.` calls with direct internal calls to the shared helper `_getPricesInternal`, which already exists and carries no access modifier:

```solidity
function getPricesNoOlderThan(...) external view onlyWhitelistedReader(subscriptionId) {
    ...
    // Instead of: prices = this.getPricesUnsafe(subscriptionId, priceIds);
    PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
    prices = new PythStructs.Price[](priceFeeds.length);
    for (uint i = 0; i < priceFeeds.length; i++) {
        prices[i] = priceFeeds[i].price;
    }
}
```

Apply the same fix to `getEmaPricesNoOlderThan`.

---

### Proof of Concept

1. Deploy `SchedulerUpgradeable`.
2. Create a subscription with `whitelistEnabled = true` and a non-empty `readerWhitelist` containing address `alice`.
3. Fund and update the subscription so prices are fresh.
4. Call `getPricesNoOlderThan(subscriptionId, priceIds, 3600)` as `alice` (a whitelisted reader).
5. The call reverts with `SchedulerErrors.Unauthorized` because the inner `this.getPricesUnsafe` sees `msg.sender == address(scheduler)`, which is not in the whitelist.

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
