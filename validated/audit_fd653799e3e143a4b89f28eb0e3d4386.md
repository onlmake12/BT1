Looking at the exact execution order in `update_single_price_feed` and the absence of a withdrawal path in `price_info.move`.

The code confirms the vulnerability. Here is the complete analysis:

---

### Title
Fee Collected Before Freshness Check Causes Permanent SUI Lock in PriceInfoObject — (`target_chains/sui/contracts/sources/pyth.move`)

### Summary
`update_single_price_feed` unconditionally calls `price_info::deposit_fee_coins` at line 277 before `update_cache` evaluates `is_fresh_update`. When the submitted price update is not fresh (`update_timestamp <= cached_timestamp`), the fee is deposited but the price state is unchanged. Because no `withdraw_fee_coins` function exists anywhere in the codebase, those SUI coins are permanently locked inside the `PriceInfoObject`.

### Finding Description

The execution order in `update_single_price_feed` is:

1. Fee sufficiency check (line 274)
2. **`deposit_fee_coins` — fee locked unconditionally (line 277)**
3. Loop to find matching price identifier (lines 281–291)
4. `update_cache` → `is_fresh_update` — freshness evaluated here (lines 287, 316) [1](#0-0) 

Inside `update_cache`, the write to state is gated by `is_fresh_update`: [2](#0-1) 

`is_fresh_update` returns `false` when `update_timestamp <= cached_timestamp`: [3](#0-2) 

`deposit_fee_coins` stores the coins as a dynamic object field on the `PriceInfoObject`: [4](#0-3) 

A grep across the entire repository for `withdraw_fee_coins` returns **zero matches**. The only fee-related public functions are `deposit_fee_coins` and `get_balance`: [5](#0-4) 

There is no protocol-owned withdrawal path, no admin reclaim, and no refund branch anywhere in `pyth.move` or `price_info.move`.

### Impact Explanation

Any caller who submits a valid but stale price update (one whose `publish_time` is equal to or older than the cached timestamp) pays the base update fee and receives nothing in return. The paid SUI is deposited into the `PriceInfoObject` dynamic field and can never be recovered. Because `PriceInfoObject` is a shared, non-deletable Sui object and no withdrawal entry point exists, the funds are frozen permanently. This satisfies the "permanent freezing of funds" criterion in the stated scope.

### Likelihood Explanation

The path is fully unprivileged and requires only publicly available inputs:

- Pyth publishes signed VAAs continuously. An attacker (or an honest user who is simply slow) can obtain a valid, guardian-signed VAA whose price timestamp equals or precedes the currently cached value.
- Wormhole's `parse_and_verify` on Sui does not maintain a global consumed-VAA registry at the application layer; replay protection is the responsibility of the consuming contract. `create_price_infos_hot_potato` performs no sequence-number or hash deduplication.
- The attacker calls `create_price_infos_hot_potato` (succeeds — valid data source, valid guardian signatures), then `update_single_price_feed` (fee deposited, no price written).

This is trivially reproducible with a unit test: initialize a `PriceInfoObject` at timestamp T, submit a VAA at timestamp T, assert `get_balance` increases while `get_price_info_from_price_info_object` is unchanged.

### Recommendation

Move `deposit_fee_coins` to **after** the freshness check, so fees are only collected when `is_fresh_update` returns `true`:

```move
if (is_fresh_update(update, price_info_object)) {
    price_info::deposit_fee_coins(price_info_object, fee);
    // emit event and update state
} else {
    // return fee to caller or abort
}
```

Alternatively, add a `withdraw_fee_coins` function (gated by a `LatestOnly` capability or a governance-controlled admin) so that mistakenly locked funds can be recovered.

        clock: &Clock

        let latest_only = state::assert_latest_only(pyth_state);

        // On Sui, users get to choose which price feeds to update. They specify a single price feed to
        // update at a time. We therefore charge the base fee for each such individual update.
        // This is a departure from Eth, where users don't get to necessarily choose.
        assert!(state::get_base_update_fee(pyth_state) <= coin::value(&fee), E_INSUFFICIENT_FEE);

        // store fee coins within price info object
        price_info::deposit_fee_coins(price_info_object, fee);

        // Find price update corresponding to PriceInfoObject within the array of price_updates
        // and use it to update PriceInfoObject.
        let i = 0;
        let found = false;
        while (i < hot_potato_vector::length<PriceInfo>(&price_updates)){
            let cur_price_info = hot_potato_vector::borrow<PriceInfo>(&price_updates, i);
            if (has_same_price_identifier(cur_price_info, price_info_object)){
                found = true;
                update_cache(latest_only, cur_price_info, price_info_object, clock);
                break
            };
            i = i + 1;
        };
        if (found==false){
            abort E_PRICE_UPDATE_NOT_FOUND_FOR_PRICE_INFO_OBJECT
        };
        price_updates
    }

    fun is_fresh_update(update: &PriceInfo, price_info_object: &PriceInfoObject): bool {
        // Get the timestamp of the update's current price
        let price_feed = price_info::get_price_feed(update);
        let update_timestamp = price::get_timestamp(&price_feed::get_price(price_feed));

        // Get the timestamp of the cached data for the price identifier
        let cached_price_info = price_info::get_price_info_from_price_info_object(price_info_object);
        let cached_price_feed =  price_info::get_price_feed(&cached_price_info);
        let cached_timestamp = price::get_timestamp(&price_feed::get_price(cached_price_feed));

        update_timestamp > cached_timestamp
    }

    public fun get_balance(price_info_object: &PriceInfoObject): u64 {
        if (!dynamic_object_field::exists_with_type<vector<u8>, Coin<SUI>>(&price_info_object.id, FEE_STORAGE_KEY)) {
            return 0
        };
        let fee = dynamic_object_field::borrow<vector<u8>, Coin<SUI>>(&price_info_object.id, FEE_STORAGE_KEY);
        coin::value(fee)
    }

    public fun deposit_fee_coins(price_info_object: &mut PriceInfoObject, fee_coins: Coin<SUI>) {
        if (!dynamic_object_field::exists_with_type<vector<u8>, Coin<SUI>>(&price_info_object.id, FEE_STORAGE_KEY)) {
            dynamic_object_field::add(&mut price_info_object.id, FEE_STORAGE_KEY, fee_coins);
        }
        else {
            let current_fee = dynamic_object_field::borrow_mut<vector<u8>, Coin<SUI>>(
                &mut price_info_object.id,
                FEE_STORAGE_KEY
            );
            coin::join(current_fee, fee_coins);
        };
    }
