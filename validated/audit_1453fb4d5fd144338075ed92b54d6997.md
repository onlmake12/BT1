### Title
Unauthenticated RPC Panic via `txs_sizes.expect()` on Legacy `BlockExt` Records — (`rpc/src/util/fee_rate.rs`)

### Summary

`FeeRateCollector::statistics` unconditionally calls `.expect()` on `block_ext.txs_sizes`. The legacy on-disk `BlockExt` format (5-field molecule schema, pre-v0.106) always deserializes with `txs_sizes: None`. Any node whose database contains such blocks will panic when `get_fee_rate_statistics` / `get_fee_rate_statics` is called with a target window covering those blocks.

### Finding Description

**Panic site** — `rpc/src/util/fee_rate.rs`, line 93:

```rust
let txs_sizes = txs_sizes.expect("expect txs_size's length >= 1");
``` [1](#0-0) 

**Source of `None`** — `util/types/src/conversion/storage.rs`, lines 139–150: the `Unpack<core::BlockExt>` impl for the legacy `packed::BlockExtReader` (5-field schema) unconditionally sets `txs_sizes: None`:

```rust
impl<'r> Unpack<core::BlockExt> for packed::BlockExtReader<'r> {
    fn unpack(&self) -> core::BlockExt {
        core::BlockExt {
            ...
            cycles: None,
            txs_sizes: None,   // always None for old format
        }
    }
}
``` [2](#0-1) 

**Store dispatch** — `store/src/store.rs`, lines 247–263: when the stored blob has `count_extra_fields() == 0` (old format), the store returns the legacy reader, yielding `txs_sizes: None`:

```rust
match reader.count_extra_fields() {
    0 => reader.into(),   // → txs_sizes: None
    2 => packed::BlockExtV1Reader::from_slice_should_be_ok(...).into(),
    _ => panic!(...),
}
``` [3](#0-2) 

**`collect` does not guard against `None`** — the `filter_map` in `FeeRateProvider::collect` only skips blocks absent from the database; it passes every found `BlockExt` (including those with `txs_sizes: None`) into the fold closure: [4](#0-3) 

**RPC entry points** — both `get_fee_rate_statics` and `get_fee_rate_statistics` call `FeeRateCollector::statistics` with no authentication requirement: [5](#0-4) 

### Impact Explanation

Any node that synced blocks before the `BlockExtV1` format was introduced (v0.106) retains old-format `BlockExt` rows in `COLUMN_BLOCK_EXT`. A single unauthenticated RPC call with a `target` window covering those blocks reaches the `.expect()` and panics the handler thread. With Rust's default `unwind` panic strategy this terminates the handler task and returns an error to the caller; if the release profile uses `panic = "abort"` (common for embedded/node binaries to avoid unsound unwinds), the entire process is killed. Either way the invariant "RPC handlers must never panic on attacker-controlled input" is broken.

### Likelihood Explanation

- Any mainnet/testnet node that has been running since before v0.106 has legacy blocks on disk.
- The RPC is typically exposed on localhost but is unauthenticated; many operators expose it to internal networks or via proxies.
- The CHANGELOG entry `#4654` (v0.119.0) addressed the `txs_sizes` length invariant (`txs_fees.len() + 1`) but did **not** replace the `.expect()` with a graceful `None` handler, leaving the `None` path unguarded. [6](#0-5) 

### Recommendation

Replace the `.expect()` with a graceful skip:

```rust
// In FeeRateCollector::statistics, inside the fold closure:
let Some(txs_sizes) = txs_sizes else {
    return fee_rates; // skip legacy blocks that lack txs_sizes
};
```

This matches the existing pattern used for `cycles` (which is also `Option`) at line 97 and eliminates the panic entirely.
