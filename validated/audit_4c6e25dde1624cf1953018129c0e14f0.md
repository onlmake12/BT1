### Title
Unauthenticated `from` Field in `ConnectionRequest` Allows Overwriting `pending_delivered` Entries — (`network/src/protocols/hole_punching/component/connection_request.rs`)

### Summary

`ConnectionRequestProcess::respond_delivered` stores attacker-supplied `listen_addrs` in `pending_delivered` keyed by the message's `from` field, which is never verified against the actual sender's session. Any connected peer can spoof `from` to match a legitimate peer's `PeerId`, wait for `HOLE_PUNCHING_INTERVAL` to expire, then overwrite the stored addresses with attacker-controlled endpoints. When the legitimate `ConnectionSync` subsequently arrives, the victim attempts NAT traversal to the attacker's endpoints instead of the real peer's.

### Finding Description

The `from` field in a `ConnectionRequest` is parsed directly from the message payload: [1](#0-0) 

The actual sender's session identity (`self.peer`, a `PeerIndex`) is only used to route the reply back — it is never compared against `content.from`: [2](#0-1) 

In `respond_delivered`, the sole guard is a timestamp check against `HOLE_PUNCHING_INTERVAL` (2 minutes): [3](#0-2) 

If no entry exists, or if the existing entry is older than 2 minutes, the function unconditionally overwrites `pending_delivered` with the attacker-supplied `remote_listens`: [4](#0-3) 

The `listen_addrs` validation only checks that any embedded `PeerId` in an address matches `content.from` — since the attacker controls `from`, they can supply bare IP addresses (no embedded peer ID), which then have the spoofed `from` peer ID appended automatically: [5](#0-4) 

When `ConnectionSync` arrives with `from=A`, the victim looks up `pending_delivered` and uses whatever addresses are stored there for NAT traversal: [6](#0-5) [7](#0-6) 

### Impact Explanation

The victim node establishes TCP connections to attacker-controlled endpoints (up to 30-second timeout per address, up to `ADDRS_COUNT_LIMIT` = 24 addresses). The p2p noise handshake will then fail because the attacker cannot prove ownership of peer A's private key. The net effect is:

1. The legitimate hole-punching attempt for peer A is silently sabotaged.
2. The victim wastes up to 30 seconds × 24 addresses of connection resources per attack cycle.
3. The attacker can repeat the attack every 2 minutes (`HOLE_PUNCHING_INTERVAL`) indefinitely, permanently preventing V from hole-punching to A. [8](#0-7) 

### Likelihood Explanation

Any peer with a valid P2P session to the victim can execute this. No special privileges, keys, or majority hashpower are required. The attacker only needs to know the target peer A's `PeerId` (which is public) and be connected to V. The `forward_rate_limiter` (1 req/sec per `(from, to, item_id)` tuple) is trivially bypassed by waiting 2 minutes between attacks, which is exactly the `HOLE_PUNCHING_INTERVAL` window. [9](#0-8) 

### Recommendation

Verify that `content.from` matches the actual sender's authenticated peer ID before storing into `pending_delivered`. In `respond_delivered`, retrieve the sender's `PeerId` from the peer registry using `self.peer` (the session ID) and reject the message if it does not equal `content.from`:

```rust
// In respond_delivered, before the pending_delivered insert:
let actual_sender_peer_id = self.protocol.network_state
    .with_peer_registry(|reg| reg.get_peer(self.peer).map(|p| p.peer_id.clone()));
if actual_sender_peer_id.as_ref() != Some(&from_peer_id) {
    return StatusCode::Ignore.with_context("from peer id does not match actual sender");
}
```

### Proof of Concept

1. Victim V is running with `local_peer_id = V`.
2. Legitimate peer A sends a `ConnectionRequest` (`from=A`, `to=V`, `listen_addrs=[A_real_ip:port]`). V stores `pending_delivered[A] = ([A_real_ip:port], t0)`.
3. Attacker (connected to V) waits `HOLE_PUNCHING_INTERVAL` (2 minutes) for the timestamp to become stale.
4. Attacker sends `ConnectionRequest` (`from=A` spoofed, `to=V`, `listen_addrs=[attacker_ip:port]`).
5. V's `respond_delivered` checks: `now - t0 >= HOLE_PUNCHING_INTERVAL` → proceeds. Overwrites: `pending_delivered[A] = ([attacker_ip:port], now)`.
6. Legitimate `ConnectionSync` arrives with `from=A`. V looks up `pending_delivered[A]`, gets `[attacker_ip:port]`, and spawns `try_nat_traversal(bind_addr, attacker_ip:port)`.
7. Assert: `pending_delivered[A].0 == [attacker_ip:port]` (original `A_real_ip:port` is gone). V never connects to A. [10](#0-9)

    async fn respond_delivered(
        &mut self,
        from_peer_id: PeerId,
        to_peer_id: &PeerId,
        remote_listens: Vec<Multiaddr>,
    ) -> Status {
        if let Some((_, t)) = self.protocol.pending_delivered.get(&from_peer_id) {
            let now = unix_time_as_millis();
            if now - t < HOLE_PUNCHING_INTERVAL {
                return StatusCode::Ignore
                    .with_context("a same message is already replied in a moment ago");
            }
        }
        let listen_addrs = {
            let public_addr = self.protocol.network_state.public_addrs(ADDRS_COUNT_LIMIT);
            if public_addr.len() < ADDRS_COUNT_LIMIT {
                let observed_addrs = self
                    .protocol
                    .network_state
                    .observed_addrs(ADDRS_COUNT_LIMIT - public_addr.len());
                let iter = public_addr
                    .iter()
                    .chain(observed_addrs.iter())
                    .map(Multiaddr::to_vec)
                    .map(|v| packed::Address::new_builder().bytes(v).build());
                packed::AddressVec::new_builder().extend(iter).build()
            } else {
                let iter = public_addr
                    .iter()
                    .map(Multiaddr::to_vec)
                    .map(|v| packed::Address::new_builder().bytes(v).build());
                packed::AddressVec::new_builder().extend(iter).build()
            }
        };
        let content = init_delivered(self.message, listen_addrs);
        let new_message = packed::HolePunchingMessage::new_builder()
            .set(content)
            .build()
            .as_bytes();
        let proto_id = SupportProtocols::HolePunching.protocol_id();

        let remote_listens: Vec<Multiaddr> = remote_listens
            .into_iter()
            .filter_map(|addr| match find_type(&addr) {
                TransportType::Memory
                | TransportType::Onion
                | TransportType::Ws
                | TransportType::Wss
                | TransportType::Tls => None,
                TransportType::Tcp => {
                    if addr
                        .iter()
                        .any(|p| matches!(p, Protocol::Ip4(_) | Protocol::Ip6(_)))
                    {
                        Some(addr)
                    } else {
                        None
                    }
                }
            })
            .collect();

        if remote_listens.is_empty() {
            return StatusCode::Ignore.with_context("remote listen address is empty");
        }

        debug!(
            "current peer is the target peer {}, send a response back",
            to_peer_id
        );

        if let Err(error) = self
            .p2p_control
            .send_message_to(self.peer, proto_id, new_message)
            .await
        {
            return StatusCode::ForwardError.with_context(error);
        }

        let now = unix_time_as_millis();
        self.protocol
            .pending_delivered
            .insert(from_peer_id, (remote_listens, now));

        Status::ok()
