### Title
Missing Domain Separator in Network Alert Signature Scheme Enables Cross-Network Replay and Post-Expiry Replay - (File: `util/gen-types/src/extension/calc_hash.rs`, `util/network-alert/src/verifier.rs`)

---

### Summary

The CKB Network Alert system signs and verifies alerts using a raw `blake2b_256` hash of the `RawAlert` molecule-encoded bytes, with no domain separator, no chain/network identifier, and no nonce. This is structurally identical to the EIP-712 vulnerability in Factory.sol: because the signed message contains no binding to a specific network, a valid alert signature produced for CKB testnet is cryptographically valid on CKB mainnet (and vice versa), and a captured alert can be replayed after it expires and is evicted from the notifier's deduplication map.

---

### Finding Description

**Root cause — `calc_alert_hash` has no domain separation:**

`RawAlertReader::calc_alert_hash()` in `util/gen-types/src/extension/calc_hash.rs` (lines 292–298) delegates to the generic `calc_hash()` trait, which is:

```rust
fn calc_hash(&self) -> packed::Byte32 {
    blake2b_256(self.as_slice()).into()
}
```

The `RawAlert` molecule schema (`util/gen-types/schemas/extensions.mol`, lines 445–453) contains only:

```
table RawAlert {
    notice_until:   Uint64,
    id:             Uint32,
    cancel:         Uint32,
    priority:       Uint32,
    message:        Bytes,
    min_version:    BytesOpt,
    max_version:    BytesOpt,
}
```

There is no `chain_id`, no `genesis_hash`, no `network_name`, and no nonce. The hash is a plain BLAKE2b of the raw payload bytes.

**Verification path:**

`Verifier::verify_signatures()` in `util/network-alert/src/verifier.rs` (lines 33–64) uses this hash directly as the secp256k1 message:

```rust
let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
// ...
verify_m_of_n(&message, self.config.signatures_threshold, &signatures, &self.pubkeys)
```

**Attack vector 1 — Cross-network replay:**

CKB mainnet and testnet use the same hard-coded Nervos Foundation public keys (configured via `NetworkAlertConfig`). Because `calc_alert_hash()` produces an identical 32-byte digest for the same `RawAlert` content regardless of which network the node is running on, a signed alert broadcast on testnet is byte-for-byte valid on mainnet. An unprivileged P2P observer or RPC caller can:

1. Observe a valid alert propagating on the testnet P2P network (or via `get_blockchain_info`).
2. Capture the raw `Alert` bytes (including the valid 2-of-4 signatures).
3. Submit the identical alert to a mainnet node via `send_alert` RPC (`rpc/src/module/alert.rs`, line 102) or inject it directly into the P2P alert protocol (`util/network-alert/src/alert_relayer.rs`, line 151).
4. `verify_signatures` accepts it — the signature is valid because the hash is identical.

**Attack vector 2 — Post-expiry replay:**

`Notifier::clear_expired_alerts()` (`util/network-alert/src/notifier.rs`, lines 135–144) removes expired alerts from `received_alerts` (a `HashMap<u32, Alert>`). After removal, `has_received(alert_id)` returns `false` (line 147–149), because the `cancel_filter` LRU only tracks *cancelled* alerts, not expired ones. The `AlertRelayer::received()` handler checks `has_received` before verifying signatures (line 147 of `alert_relayer.rs`). Once the alert is evicted, the same captured alert bytes pass through the deduplication gate and `verify_signatures` accepts them again, causing the alert to be re-displayed to all node operators.

---

### Impact Explanation

**Cross-network replay:** An unprivileged attacker who captures a legitimately signed testnet alert can force it to propagate across the entire CKB mainnet P2P network. Mainnet node operators will display the alert message (which may be testnet-specific, outdated, or misleading), potentially triggering unnecessary emergency upgrades, node shutdowns, or user panic. The inverse (mainnet alert replayed on testnet) also holds. This undermines the integrity of the alert system as a trusted emergency communication channel.

**Post-expiry replay:** After an alert's `notice_until` timestamp passes and `clear_expired_alerts` evicts it, any P2P peer that captured the original alert bytes can re-inject it. The alert re-propagates across the network and is re-displayed to all node operators, constituting a persistent misinformation/disruption capability for any unprivileged network participant.

---

### Likelihood Explanation

Both attacks require only passive observation of the CKB P2P network or a single `get_blockchain_info` RPC call to capture a live alert. The P2P alert protocol broadcasts alert bytes to all connected peers (`alert_relayer.rs` lines 166–176), making capture trivial. No privileged access, key material, or cryptographic break is required. The cross-network replay is immediately exploitable whenever a valid alert exists on one network; the post-expiry replay is exploitable after any alert's `notice_until` passes.

---

### Recommendation

Include a domain separator in the signed message. The alert hash should commit to at least the genesis hash of the target chain (available as `consensus.genesis_hash()`), analogous to EIP-712's domain separator. For example:

```rust
pub fn calc_alert_hash_with_domain(raw: &RawAlert, genesis_hash: &Byte32) -> Byte32 {
    let mut hasher = new_blake2b();
    hasher.update(genesis_hash.as_slice());   // domain separator
    hasher.update(raw.as_slice());
    let mut result = [0u8; 32];
    hasher.finalize(&mut result);
    result.into()
}
```

Additionally, to prevent post-expiry replay, the `cancel_filter` LRU should also track expired alert IDs (not only cancelled ones), or a separate `expired_filter` should be maintained.

---

### Proof of Concept

**Cross-network replay (step-by-step):**

1. Run a CKB testnet node and a CKB mainnet node.
2. On testnet, observe a valid alert via `get_blockchain_info` or P2P capture. Extract the raw `Alert` bytes including signatures.
3. On mainnet, call `send_alert` RPC with the identical alert JSON. The `verify_signatures` call at `rpc/src/module/alert.rs:112` invokes `Verifier::verify_signatures`, which computes `blake2b_256(raw_alert.as_slice())` — identical to the testnet hash — and the secp256k1 signature check passes.
4. The alert is added to the mainnet notifier and broadcast to all mainnet peers via `network_controller.broadcast_with_handle` (`alert.rs:119`).

**Post-expiry replay (step-by-step):**

1. Capture a valid alert from the P2P network while it is live.
2. Wait until `notice_until` passes and `clear_expired_alerts` is called (triggered on each new peer connection at `alert_relayer.rs:87`).
3. Re-submit the captured alert via `send_alert` RPC or direct P2P injection.
4. `has_received(alert_id)` returns `false` (alert evicted from `received_alerts`, not in `cancel_filter`); `verify_signatures` passes; the alert re-propagates across the network.

    pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
        trace!("Verifying alert {:?}", alert);
        let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
        let signatures: Vec<Signature> = alert
            .signatures()
            .into_iter()
            .filter_map(
                |sig_data| match Signature::from_slice(sig_data.as_reader().raw_data()) {
                    Ok(sig) => {
                        if sig.is_valid() {
                            Some(sig)
                        } else {
                            debug!("invalid signature: {:?}", sig);
                            None
                        }
                    }
                    Err(err) => {
                        debug!("signature error: {}", err);
                        None
                    }
                },
            )
            .collect();
        verify_m_of_n(
            &message,
            self.config.signatures_threshold,
            &signatures,
            &self.pubkeys,
        )
        .map_err(|err| err.kind())?;
        Ok(())
    }

    pub fn clear_expired_alerts(&mut self, now: u64) {
        self.received_alerts.retain(|_id, alert| {
            let notice_until: u64 = alert.raw().notice_until().into();
            notice_until > now
        });
        self.noticed_alerts.retain(|a| {
            let notice_until: u64 = a.raw().notice_until().into();
            notice_until > now
        });
    }

        let alert: packed::Alert = alert.into();

        let notice_until: u64 = alert.raw().notice_until().into();

                    &self.handle,
                );
                Ok(())
