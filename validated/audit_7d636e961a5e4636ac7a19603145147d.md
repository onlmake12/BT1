### Title
Overpayment in `requestPriceUpdatesWithCallback` is silently credited to provider instead of being refunded to user - (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

`Echo.sol::requestPriceUpdatesWithCallback` accepts any `msg.value >= requiredFee` but stores the entire surplus as `req.fee`, which is later credited to the provider in `executeCallback`. Any ETH a user sends beyond the required fee is permanently lost to them and transferred to the provider's accrued balance.

---

### Finding Description

In `requestPriceUpdatesWithCallback`, the fee check is a one-sided minimum:

```solidity
uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
if (msg.value < requiredFee) revert InsufficientFee();
```

Immediately after, the stored provider fee is computed as:

```solidity
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);
```

`requiredFee` decomposes as `_state.pythFeeInWei + providerBaseFee + providerFeedFee + gasFee`. If a user sends `msg.value = requiredFee + excess`, then:

```
req.fee = (requiredFee + excess) - _state.pythFeeInWei
        = providerFees + excess
```

Later, in `executeCallback`, the provider is credited:

```solidity
_state.providers[providerToCredit].accruedFeesInWei +=
    SafeCast.toUint128((req.fee + msg.value) - pythFee);
```

The `excess` embedded in `req.fee` flows directly into the provider's `accruedFeesInWei`. There is no refund path back to the original requester.

Two realistic scenarios cause overpayment:

1. **Stale fee estimate**: A user calls `getFee()` off-chain, then the provider calls `setProviderFee()` to lower their fee before the user's transaction lands. The user's `msg.value` is now larger than the new `requiredFee`, and the surplus is credited to the provider.
2. **Accidental overpayment**: A user or integrating contract sends a rounded-up or hardcoded ETH amount larger than the exact fee. [1](#0-0) [2](#0-1) 

---

### Impact Explanation

**Impact: Medium**

Any ETH sent above `requiredFee` is permanently transferred to the provider's `accruedFeesInWei` balance. The user has no mechanism to recover it. The loss is bounded by the overpayment amount per transaction, but it is irreversible once the request is created. The provider can then withdraw the windfall via `withdrawAsFeeManager` or direct withdrawal. [3](#0-2) 

---

### Likelihood Explanation

**Likelihood: Medium**

Provider fees are mutable via `setProviderFee()`, callable at any time by the provider or their fee manager. A race condition between a user's off-chain `getFee()` call and the on-chain transaction submission is realistic, especially during periods of fee adjustment. Additionally, integrating contracts that add a small buffer to fee estimates (a common defensive pattern) will routinely overpay. [4](#0-3) 

---

### Recommendation

Refund any excess ETH to the caller after computing the exact required fee, or enforce an exact-match check:

```diff
uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
if (msg.value < requiredFee) revert InsufficientFee();

+// Refund overpayment
+if (msg.value > requiredFee) {
+    (bool refunded, ) = payable(msg.sender).call{value: msg.value - requiredFee}("");
+    require(refunded, "Refund failed");
+}

req.fee = SafeCast.toUint96(requiredFee - _state.pythFeeInWei);
```

Alternatively, replace the `>=` check with strict equality:

```diff
-if (msg.value < requiredFee) revert InsufficientFee();
+if (msg.value != requiredFee) revert InsufficientFee();
``` [1](#0-0) 

---

### Proof of Concept

1. Provider registers with `baseFeeInWei = 1000 wei`, `feePerFeedInWei = 0`, `feePerGasInWei = 0`. Assume `_state.pythFeeInWei = 100 wei`.
2. User calls `getFee(provider, gasLimit, priceIds)` off-chain → returns `1100 wei`.
3. Provider calls `setProviderFee(provider, 500, 0, 0)` — fee drops to `600 wei` total.
4. User's transaction lands with `msg.value = 1100 wei`.
5. New `requiredFee = 600 wei`; check `1100 >= 600` passes.
6. `req.fee = 1100 - 100 = 1000 wei` (should be `500 wei`).
7. In `executeCallback`, provider is credited `1000 + 0 - pythFee_actual` — receiving `500 wei` of the user's ETH as an unearned windfall.
8. User has permanently lost `500 wei` with no recourse. [5](#0-4) [6](#0-5)

    function requestPriceUpdatesWithCallback(
        address provider,
        uint64 publishTime,
        bytes32[] calldata priceIds,
        uint32 callbackGasLimit
    ) external payable override returns (uint64 requestSequenceNumber) {
        require(
            _state.providers[provider].isRegistered,
            "Provider not registered"
        );

        // FIXME: this comment is wrong. (we're not using tx.gasprice)
        // NOTE: The 60-second future limit on publishTime prevents a DoS vector where
        //      attackers could submit many low-fee requests for far-future updates when gas prices
        //      are low, forcing executors to fulfill them later when gas prices might be much higher.
        //      Since tx.gasprice is used to calculate fees, allowing far-future requests would make
        //      the fee estimation unreliable.
        require(publishTime <= block.timestamp + 60, "Too far in future");
        if (priceIds.length > MAX_PRICE_IDS) {
            revert TooManyPriceIds(priceIds.length, MAX_PRICE_IDS);
        }
        requestSequenceNumber = _state.currentSequenceNumber++;

        uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
        if (msg.value < requiredFee) revert InsufficientFee();

        Request storage req = allocRequest(requestSequenceNumber);
        req.sequenceNumber = requestSequenceNumber;
        req.publishTime = publishTime;
        req.callbackGasLimit = callbackGasLimit;
        req.requester = msg.sender;
        req.provider = provider;
        req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);

        // Create array with the right size
        req.priceIdPrefixes = new bytes8[](priceIds.length);

        // Copy only the first 8 bytes of each price ID to storage
        for (uint8 i = 0; i < priceIds.length; i++) {
            // Extract first 8 bytes of the price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }
            req.priceIdPrefixes[i] = prefix;
        }
        _state.accruedFeesInWei += _state.pythFeeInWei;

        emit PriceUpdateRequested(req, priceIds);
    }

    function getFee(
        address provider,
        uint32 callbackGasLimit,
        bytes32[] calldata priceIds
    ) public view override returns (uint96 feeAmount) {
        uint96 baseFee = _state.pythFeeInWei; // Fixed fee to Pyth
        // Note: The provider needs to set its fees to include the fee charged by the Pyth contract.
        // Ideally, we would be able to automatically compute the pyth fees from the priceIds, but the
        // fee computation on IPyth assumes it has the full updated data.
        uint96 providerBaseFee = _state.providers[provider].baseFeeInWei;
        uint96 providerFeedFee = SafeCast.toUint96(
            priceIds.length * _state.providers[provider].feePerFeedInWei
        );
        uint96 providerFeeInWei = _state.providers[provider].feePerGasInWei; // Provider's per-gas rate
        uint256 gasFee = callbackGasLimit * providerFeeInWei; // Total provider fee based on gas
        feeAmount =
            baseFee +
            providerBaseFee +
            providerFeedFee +
            SafeCast.toUint96(gasFee); // Total fee user needs to pay
    }

            _state.providers[provider].isRegistered,

            oldBaseFee,
