Audit Report

## Title
Out-of-Bounds Panic in `TxConfirmStat::remove_unconfirmed_tx` When `tx_age` Exceeds Circular Buffer Size — (File: `util/fee-estimator/src/estimator/confirmation_fraction.rs`)

## Summary
`TxConfirmStat` initializes `confirm_blocks_to_failed_txs` and `block_unconfirmed_txs` to length `MAX_CONFIRM_BLOCKS = 1000`. In `remove_unconfirmed_tx`, a bounds guard protects `block_unconfirmed_txs` when `tx_age >= 1000`, but `confirm_blocks_to_failed_txs[tx_age - 1]` is indexed unconditionally. When a tracked transaction is rejected or evicted after more than 1000 blocks, `tx_age - 1 >= 1000` is out of bounds and Rust panics, crashing the node.

## Finding Description
Both arrays are initialized to length 1000: [1](#0-0) 

In `remove_unconfirmed_tx`, the guard at line 208 routes `block_unconfirmed_txs` safely when `tx_age >= self.block_unconfirmed_txs.len()`, but line 215 indexes `confirm_blocks_to_failed_txs[tx_age - 1]` with no corresponding guard: [2](#0-1) 

The call chain that sets `count_failure = true`:
- `reject_tx` (L475–477) calls `drop_tx` (L428–430), which calls `drop_tx_inner(tx_hash, true)` (L416–425), which calls `remove_unconfirmed_tx(..., count_failure: true)`. [3](#0-2) [4](#0-3) 

`remove_expired` in `pool.rs` calls `callbacks.call_reject`, which invokes `fee_estimator.reject_tx`, completing the reachable path from an external trigger: [5](#0-4) 

## Impact Explanation
When `ConfirmationFraction` is enabled and a tracked transaction remains unconfirmed for more than 1000 blocks before being rejected or evicted, the node panics unconditionally with an index-out-of-bounds error in both debug and release builds. This is a deterministic, reproducible node crash. Impact: **High (10001–15000 points): Vulnerabilities which could easily crash a CKB node**.

## Likelihood Explanation
`ConfirmationFraction` is opt-in via `fee_estimator.algorithm = "ConfirmationFraction"` in `ckb.toml`. Once enabled, the trigger requires no special privileges: submit a low-fee transaction that miners ignore, wait for 1001+ blocks (~2.8 hours at 10 s/block), then allow the default 12-hour expiry timer (`remove_expired`) or pool size eviction (`limit_size`) to fire. Any unprivileged user with RPC access to `send_transaction` can trigger this condition. The crash is repeatable.

## Recommendation
Add a bounds guard before indexing `confirm_blocks_to_failed_txs`, mirroring the existing guard for `block_unconfirmed_txs`:

```rust
if count_failure && tx_age <= self.confirm_blocks_to_failed_txs.len() {
    self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64;
}
```

This ensures failure counts are only recorded for transactions whose age falls within the tracked window, consistent with the design intent of `MAX_CONFIRM_BLOCKS`. [6](#0-5) 

## Proof of Concept
1. Configure the node: `fee_estimator.algorithm = "ConfirmationFraction"` in `ckb.toml`.
2. Submit a transaction with a fee rate just above `min_fee_rate` but low enough that miners do not include it. The tx is accepted and tracked at height `H` with `best_height = H`.
3. Allow 1001+ blocks to be produced. The tx remains unconfirmed; `tx_age` grows beyond 1000.
4. Wait for the 12-hour expiry timer (`remove_expired` → `callbacks.call_reject` → `fee_estimator.reject_tx`) or fill the pool to trigger `limit_size` eviction.
5. `remove_unconfirmed_tx` is called with `tx_age > 1000` and `count_failure = true`.
6. `self.confirm_blocks_to_failed_txs[tx_age - 1]` panics: **thread panicked at 'index out of bounds: the len is 1000 but the index is N'**, crashing the node.

Unit test to confirm: construct a `TxConfirmStat` with `max_confirm_blocks = 5`, call `add_unconfirmed_tx` at height 0, then call `remove_unconfirmed_tx(entry_height=0, tip_height=6, bucket_index, count_failure=true)` — this produces `tx_age = 6`, which panics at `confirm_blocks_to_failed_txs[5]` on a `Vec` of length 5. [7](#0-6)

```rust
    fn remove_unconfirmed_tx(
        &mut self,
        entry_height: u64,
        tip_height: u64,
        bucket_index: usize,
        count_failure: bool,
    ) {
        let tx_age = tip_height.saturating_sub(entry_height) as usize;
        if tx_age < 1 {
            return;
        }
        if tx_age >= self.block_unconfirmed_txs.len() {
            self.bucket_stats[bucket_index].old_unconfirmed_txs -= 1;
        } else {
            let block_index = (entry_height % self.block_unconfirmed_txs.len() as u64) as usize;
            self.block_unconfirmed_txs[block_index][bucket_index] -= 1;
        }
        if count_failure {
            self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64;
        }
    }
```
