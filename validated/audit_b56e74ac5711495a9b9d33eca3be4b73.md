### Title
Global `tx_filter` Poisoning via Relay Transaction with Suppressed Cleanup — (File: `sync/src/relayer/transactions_process.rs`)

### Summary
In `TransactionsProcess::execute`, the global `SyncState::tx_filter` is updated with `mark_as_known_txs` **before** the async pool submission completes. When the pool rejects a transaction with a reason where `is_allowed_relay()` returns `false`, `remove_from_known_txs` is never called. The transaction hash remains in the global `tx_filter` for the full 4-hour TTL (`FILTER_TTL = 4 * 60 * 60` seconds), silently suppressing all future relay announcements of that hash from every peer on the node.

### Finding Description

**Root cause — premature global state write before async result:**

In `sync/src/relayer/transactions_process.rs` at line 76, `mark_as_known_txs` inserts the transaction hash into the global `tx_filter` before the async `submit_remote_tx` call is even dispatched: [1](#0-0) 

The only cleanup path is in `sync/src/relayer/mod.rs` inside `send_bulk_of_tx_hashes`: [2](#0-1)
