### Title
Unvalidated `authority` in `initialize` + missing `UpdateAuthority` permanently bricks publisher registration — (`target_chains/solana/programs/pyth-price-store/src/processor/initialize.rs`)

### Summary

The `initialize` instruction accepts an arbitrary 32-byte `authority` pubkey with no validation that it is a signable (non-PDA) key. Because there is no `UpdateAuthority` instruction and the config account can only be initialized once, an attacker who front-runs the deployer's `initialize` call can permanently set an inaccessible PDA as authority, making `InitializePublisher` permanently unreachable.

---

### Finding Description

**Step 1 — No access control on `initialize`.**
`validate_payer` only checks that the payer is a signer and writable; it does not restrict who may call `initialize`. [1](#0-0) 

**Step 2 — `authority` is stored verbatim, no validation.**
`accounts::config::create` accepts any 32-byte value and writes it directly into the config account. There is no check that the pubkey is off-curve (i.e., a valid signer key) or that it is not a PDA. [2](#0-1) 

**Step 3 — Config is write-once.**
`create` returns `AlreadyInitialized` if `format != 0`, so the config can never be re-initialized after the first call. [3](#0-2) 

**Step 4 — `validate_authority` requires `is_signer`.**
Every `InitializePublisher` call goes through `validate_authority`, which enforces that the authority account is a transaction signer. PDAs can never be transaction signers on Solana. [4](#0-3) 

**Step 5 — No `UpdateAuthority` instruction exists.**
The entire instruction set is `Initialize`, `SubmitPrices`, and `InitializePublisher`. There is no mechanism to rotate the stored authority. [5](#0-4) 

---

### Impact Explanation

Once the config is initialized with a PDA as authority, every subsequent `InitializePublisher` call fails at `validate_authority` with `MissingRequiredSignature`. No publisher can ever be registered or de-registered. The price-store program's publisher set is permanently frozen with no on-chain recovery path. [6](#0-5) 

---

### Likelihood Explanation

The attack window is the gap between program deployment and the deployer's `initialize` transaction. An attacker monitoring the mempool (or simply racing the deployer) can submit `initialize` with `authority = config_PDA` (a publicly computable address) before the legitimate call lands. No privileged access is required; only a funded Solana account is needed. [7](#0-6) 

---

### Recommendation

1. **Validate `authority` at initialization time**: reject any pubkey that lies on the curve of a known PDA, or require the authority account to co-sign the `initialize` transaction (add it to the accounts list with `is_signer = true`).
2. **Add an `UpdateAuthority` instruction** gated on the current authority's signature, so the authority can be rotated if the key is ever lost or set incorrectly. [2](#0-1)

pub fn validate_payer<'a, 'b>(
    account: Option<&'b AccountInfo<'a>>,

    let payer = account.ok_or(ProgramError::NotEnoughAccountKeys)?;

    ensure!(ProgramError::InvalidArgument, payer.is_writable);

pub fn validate_authority<'a, 'b>(
    account: Option<&'b AccountInfo<'a>>,
    config: &AccountInfo<'a>,
) -> Result<&'b AccountInfo<'a>, ProgramError> {
    let authority = account.ok_or(ProgramError::NotEnoughAccountKeys)?;
    ensure!(ProgramError::MissingRequiredSignature, authority.is_signer);
    ensure!(ProgramError::InvalidArgument, authority.is_writable);
    let config_data = config.data.borrow();
    let config = accounts::config::read(*config_data)?;
    ensure!(
        ProgramError::MissingRequiredSignature,
        authority.key.to_bytes() == config.authority
    );
    Ok(authority)

pub fn initialize(
    program_id: &Pubkey,
    accounts: &[AccountInfo],
    args: &InitializeArgs,
) -> ProgramResult {
    let mut accounts = accounts.iter();
    let payer = validate_payer(accounts.next())?;
    let config = validate_config(accounts.next(), args.config_bump, program_id, true)?;
    let system = validate_system(accounts.next())?;

    let lamports = (Rent::get()?).minimum_balance(accounts::config::SIZE);

    invoke_signed(
        &system_instruction::create_account(
            payer.key,
            config.key,
            lamports,
            accounts::config::SIZE
                .try_into()
                .expect("unexpected overflow"),
            program_id,
        ),
        &[payer.clone(), config.clone(), system.clone()],
        &[&[CONFIG_SEED.as_bytes(), &[args.config_bump]]],
    )?;

    accounts::config::create(*config.data.borrow_mut(), args.authority)?;

    Ok(())
