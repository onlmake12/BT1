### Title
`MaturityVerifier` Skips Cellbase Immaturity Check on `resolved_dep_groups` Container Cells - (`File: verification/src/transaction_verifier.rs`)

---

### Summary

`MaturityVerifier::verify()` enforces the cellbase maturity rule on `resolved_inputs` and `resolved_cell_deps`, but silently omits the same check on `resolved_dep_groups`. A `DepType::DepGroup` cell dep whose container cell is itself an immature cellbase output passes maturity verification unchallenged, violating a core consensus invariant.

---

### Finding Description

In CKB's cell model, a `CellDep` can have `dep_type = DepGroup`. When resolved, the dep group container cell (the cell whose data is an `OutPointVec`) is stored in `resolved_dep_groups`, while the cells it points to are expanded into `resolved_cell_deps`.

`resolve_transaction_dep` in `util/types/src/core/cell.rs` makes this split explicit:

```rust
// dep group container → resolved_dep_groups
resolved_dep_groups.push(dep_group);
// expanded sub-cells  → resolved_cell_deps
for sub_out_point in sub_out_points.into_iter() {
    resolved_cell_deps.push(cell_resolver(&sub_out_point, eager_load)?);
}
``` [1](#0-0) 

`MaturityVerifier::verify()` then checks `resolved_inputs` and `resolved_cell_deps` for cellbase immaturity, but never iterates over `resolved_dep_groups`:

```rust
// ✓ checked
resolved_inputs.iter().position(cellbase_immature)
// ✓ checked
resolved_cell_deps.iter().position(cellbase_immature)
// ✗ NEVER CHECKED
// resolved_dep_groups — omitted entirely
``` [2](#0-1) 

The `cellbase_immature` predicate correctly identifies any cell whose `transaction_info` shows `block_number > 0`, `is_cellbase() == true`, and whose epoch has not yet crossed the maturity threshold. That predicate is simply never applied to `resolved_dep_groups`. [3](#0-2) 

`MaturityVerifier` is composed into both `TimeRelativeTransactionVerifier` (used during tx-pool admission re-verification) and `ContextualTransactionVerifier` (used during block verification), so the gap is present in both paths. [4](#0-3) 

---

### Impact Explanation

The cellbase maturity rule is a consensus invariant: no transaction may reference an immature cellbase output as an input or dependency. Bypassing it for dep group container cells means:

- A transaction can be submitted to the tx-pool and committed into a block while referencing an immature cellbase output as a `DepType::DepGroup` dep.
- Any node that enforces the rule correctly would reject such a block, causing a **consensus split** between nodes that apply the check and those that do not (all current nodes share the same missing check, so the split would appear after a fix is deployed).
- The immature cellbase output is consumed as a dependency before the protocol allows it, breaking the economic guarantee that miners cannot immediately re-spend coinbase rewards through the dep group path.

---

### Likelihood Explanation

The attacker is any transaction sender. The steps are:

1. Observe a recently mined block whose cellbase output has data that is (or can be crafted to be) a valid `OutPointVec`.
2. Construct a transaction with a `CellDep { out_point: <cellbase_outpoint>, dep_type: DepGroup }`.
3. Submit via RPC (`send_transaction`). The tx-pool resolves the dep group, places the container cell in `resolved_dep_groups`, and `MaturityVerifier` passes without checking it.
4. The transaction is proposed and committed before the maturity epoch is reached.

A miner can trivially craft a cellbase output whose data is a valid `OutPointVec` pointing to any live cells, making this reliably exploitable.

---

### Recommendation

Add a third immaturity check in `MaturityVerifier::verify()` covering `resolved_dep_groups`:

```rust
if let Some(index) = self
    .transaction
    .resolved_dep_groups
    .iter()
    .position(cellbase_immature)
{
    return Err(TransactionError::CellbaseImmaturity {
        inner: TransactionErrorSource::CellDeps,
        index,
    }
    .into());
}
``` [5](#0-4) 

---

### Proof of Concept

**Setup:**
- Chain with `cellbase_maturity = 5 epochs`.
- Miner mines block N, producing cellbase output `C` at index 0 whose `output_data` is a valid `OutPointVec` pointing to two live cells `[A, B]`.

**Attack:**
- Before 5 epochs elapse, attacker submits:
  ```
  Transaction {
    cell_deps: [CellDep { out_point: C, dep_type: DepGroup }],
    inputs: [...],
    outputs: [...],
  }
  ```
- `resolve_transaction_dep` places `C` into `resolved_dep_groups` and `[A, B]` into `resolved_cell_deps`.
- `MaturityVerifier::verify()` checks `resolved_inputs` (none immature) and `resolved_cell_deps` (`A`, `B` — not cellbase outputs, pass). `resolved_dep_groups` (`C` — immature cellbase) is **never checked**.
- Transaction passes all verification, enters the pool, and is committed into a block. [6](#0-5) [7](#0-6)

fn resolve_transaction_dep<F: FnMut(&OutPoint, bool) -> Result<CellMeta, OutPointError>>(
    cell_dep: &CellDep,
    cell_resolver: &mut F,
    resolved_cell_deps: &mut Vec<CellMeta>,
    resolved_dep_groups: &mut Vec<CellMeta>,
    eager_load: bool,
    remaining_dep_slots: &mut usize,
) -> Result<(), OutPointError> {
    if cell_dep.dep_type() == DepType::DepGroup.into() {
        let outpoint = cell_dep.out_point();
        let dep_group = cell_resolver(&outpoint, true)?;
        let data = dep_group
            .mem_cell_data
            .as_ref()
            .expect("Load cell meta must with data");
        let sub_out_points =
            parse_dep_group_data(data).map_err(|_| OutPointError::InvalidDepGroup(outpoint))?;

        *remaining_dep_slots = remaining_dep_slots
            .checked_sub(sub_out_points.len())
            .ok_or(OutPointError::OverMaxDepExpansionLimit)?;

        for sub_out_point in sub_out_points.into_iter() {
            resolved_cell_deps.push(cell_resolver(&sub_out_point, eager_load)?);
        }
        resolved_dep_groups.push(dep_group);
    } else {
        *remaining_dep_slots = remaining_dep_slots
            .checked_sub(1)
            .ok_or(OutPointError::OverMaxDepExpansionLimit)?;

        resolved_cell_deps.push(cell_resolver(&cell_dep.out_point(), eager_load)?);
    }
    Ok(())
}

```rust
    pub fn verify(&self) -> Result<(), Error> {
        let cellbase_immature = |meta: &CellMeta| -> bool {
            meta.transaction_info
                .as_ref()
                .map(|info| {
                    info.block_number > 0 && info.is_cellbase() && {
                        let threshold =
                            self.cellbase_maturity.to_rational() + info.block_epoch.to_rational();
                        let current = self.epoch.to_rational();
                        current < threshold
                    }
                })
                .unwrap_or(false)
        };

        if let Some(index) = self
            .transaction
            .resolved_inputs
            .iter()
            .position(cellbase_immature)
        {
            return Err(TransactionError::CellbaseImmaturity {
                inner: TransactionErrorSource::Inputs,
                index,
            }
            .into());
        }

        if let Some(index) = self
            .transaction
            .resolved_cell_deps
            .iter()
            .position(cellbase_immature)
        {
            return Err(TransactionError::CellbaseImmaturity {
                inner: TransactionErrorSource::CellDeps,
                index,
            }
            .into());
        }

        Ok(())
    }
