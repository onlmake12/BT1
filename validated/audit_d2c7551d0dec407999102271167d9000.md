### Title
Lazer `verifyUpdate()` Accepts Arbitrarily Old Signed Payloads Without Timestamp Freshness Enforcement — (File: `lazer/contracts/evm/src/PythLazer.sol`)

---

### Summary

`PythLazer.sol::verifyUpdate()` — the core on-chain verification primitive for Pyth Lazer (Pro) price updates — validates only that the ECDSA signature was produced by a currently-trusted signer. It performs no check that the timestamp embedded in the payload is recent relative to `block.timestamp`. Any unprivileged caller can replay an arbitrarily old, validly-signed Lazer update and receive a successful verification result. Consumer contracts that rely on `verifyUpdate()` as their sole security gate and do not independently enforce a staleness window will silently accept stale prices.

---

### Finding Description

`verifyUpdate()` in `PythLazer.sol` performs the following checks:

1. Fee ≥ `verification_fee`
2. Magic bytes match `EVM_FORMAT_MAGIC`
3. ECDSA signature recovers to a signer whose `expiresAt > block.timestamp`

It then returns `(payload, signer)` to the caller. [1](#0-0) 

The payload contains a `uint64 timestamp` field (microseconds since epoch) that is parsed by `PythLazerLib::parsePayloadHeader()` and exposed to consumers as `_timestamp`. [2](#0-1) 

`verifyUpdate()` never reads or validates this timestamp field. The signer's `expiresAt` is a certificate-rotation deadline (typically months in the future), not a per-message freshness bound. [3](#0-2) 

The same structural gap exists in the Aptos Lazer contract (`verify_message()`) and the Sui Lazer contract (`parse_and_verify_le_ecdsa_update_v2()`): [4](#0-3) [5](#0-4) 

The Cardano integration docs explicitly acknowledge this design gap: *"It does not enforce freshness of the update, nor does it disallow verifying the same update multiple times."* [6](#0-5) 

The EVM integration guide's example code shows only a monotonicity guard (`_timestamp > timestamp`) — not a wall-clock freshness check against `block.timestamp`: [7](#0-6) 

---

### Impact Explanation

A consumer contract that calls `verifyUpdate()` and then uses the returned price without comparing `_timestamp` to `block.timestamp` within a tight bound will accept prices from arbitrarily far in the past. Because Lazer is marketed as a sub-second, ultra-low-latency feed, integrators may reasonably assume the verification step enforces freshness. An attacker who replays a stale update with a favorable price can:

- Borrow against inflated collateral
- Liquidate positions using a stale low price
- Open/close derivative positions at a price that no longer reflects the market

The impact is direct financial loss to protocols and their users. The Lazer product targets high-frequency DeFi (perps, lending), where even seconds of staleness are exploitable.

---

### Likelihood Explanation

Signed Lazer payloads are observable on-chain (emitted in transaction calldata) and off-chain (WebSocket stream). Any past update signed by a still-trusted signer remains replayable. The EVM integration guide's example code does not include a `block.timestamp` freshness check, making it likely that consumer contracts will omit it. The attack requires no privileged access — only the ability to submit a transaction with a previously-observed signed payload.

---

### Recommendation

**Short term:** Add a `maxAgeSeconds` parameter to `verifyUpdate()` and assert `block.timestamp - (payload_timestamp_us / 1_000_000) <= maxAgeSeconds` before returning. Alternatively, add a `verifyUpdateNoOlderThan(bytes calldata update, uint256 maxAgeSeconds)` variant and deprecate the unchecked form.

**Long term:** Mirror the staleness-enforcement pattern already present in the core price-feed contracts (`getPriceNoOlderThan`, `check_price_is_fresh`) across all Lazer verification primitives on all chains. Update integration guides to show a mandatory `block.timestamp` freshness check as part of the canonical consumer pattern.

---

### Proof of Concept

1. At time `T`, observe a signed Lazer update `U` (e.g., from on-chain calldata) where `U.payload.timestamp = T` and the price is favorable to the attacker (e.g., BTC at $95,000 when current price is $100,000).
2. At time `T + Δ` (e.g., Δ = 30 minutes), call `pythLazer.verifyUpdate{value: fee}(U)`.
3. `verifyUpdate` recovers the signer, checks `block.timestamp < signer.expiresAt` (still valid), and returns `(payload, signer)` — **no revert**.
4. A consumer contract that does not check `_timestamp` against `block.timestamp` stores the stale price and uses it for a financial operation (e.g., collateral valuation), enabling the attacker to profit from the price discrepancy. [8](#0-7)

    function verifyUpdate(
        bytes calldata update
    ) external payable returns (bytes calldata payload, address signer) {
        // Require fee and refund excess
        require(msg.value >= verification_fee, "Insufficient fee provided");
        if (msg.value > verification_fee) {
            payable(msg.sender).transfer(msg.value - verification_fee);
        }

        if (update.length < 71) {
            revert("input too short");
        }
        uint32 EVM_FORMAT_MAGIC = 706910618;

        uint32 evm_magic = uint32(bytes4(update[0:4]));
        if (evm_magic != EVM_FORMAT_MAGIC) {
            revert("invalid evm magic");
        }
        uint16 payload_len = uint16(bytes2(update[69:71]));
        if (update.length < 71 + payload_len) {
            revert("input too short");
        }
        payload = update[71:71 + payload_len];
        bytes32 hash = keccak256(payload);
        (signer, , ) = ECDSA.tryRecover(
            hash,
            uint8(update[68]) + 27,
            bytes32(update[4:36]),
            bytes32(update[36:68])
        );
        if (signer == address(0)) {
            revert("invalid signature");
        }
        if (!isValidSigner(signer)) {
            revert("invalid signer");
        }
    }

    public fun verify_message(
        message: vector<u8>,
        signature: vector<u8>,
        trusted_signer: vector<u8>,
        fee: coin::Coin<AptosCoin>
    ) acquires Storage {
        let storage = borrow_global<Storage>(@pyth_lazer);

        // Verify fee amount
        assert!(coin::value(&fee) >= storage.single_update_fee, EINSUFFICIENT_FEE);

        // Transfer fee to treasury
        coin::deposit(storage.treasury, fee);

        // Verify signer is trusted and not expired
        let i = 0;
        let valid = false;
        while (i < storage.trusted_signers.length()) {
            let signer_info = vector::borrow(&storage.trusted_signers, (i as u64));
            if (&signer_info.pubkey == &trusted_signer
                && signer_info.expires_at > timestamp::now_seconds()) {
                valid = true;
                break
            };
            i = i + 1;
        };
        assert!(valid, EINVALID_SIGNER);

        // Verify signature
        let sig = ed25519::new_signature_from_bytes(signature);
        let pk = ed25519::new_unvalidated_public_key_from_bytes(trusted_signer);
        assert!(
            ed25519::signature_verify_strict(&sig, &pk, message),
            EINVALID_SIGNATURE
        );
    }

public fun parse_and_verify_le_ecdsa_update_v2(s: &State, clock: &Clock, update: vector<u8>): Update {
    let mut cursor = bcs::new(update);

    // Parse and validate message magic
    let magic = cursor.peel_u32();
    assert!(magic == UPDATE_MESSAGE_MAGIC, EInvalidUpdateMagic);

    // Parse signature
    let mut signature = vector::empty<u8>();
    let mut sig_i = 0;
    while (sig_i < SECP256K1_SIG_LEN) {
        signature.push_back(cursor.peel_u8());
        sig_i = sig_i + 1;
    };

    // Parse expected payload length and get remaining bytes as payload
    let payload_len = cursor.peel_u16();
    let payload = cursor.into_remainder_bytes();

    // Validate expected payload length
    assert!(payload_len as u64 == payload.length(), EInvalidPayloadLength);

    // Parse payload
    let mut payload_cursor = bcs::new(payload);
    let payload_magic = payload_cursor.peel_u32();
    assert!(payload_magic == PAYLOAD_MAGIC, EInvalidPayloadMagic);

    // Verify the signature against trusted signers
    verify_le_ecdsa_message(s, clock, &signature, &payload);

    update_v2::parse_from_cursor(payload_cursor)
}
