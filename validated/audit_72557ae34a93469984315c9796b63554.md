### Title
Hole-Punching `ConnectionRequest` with `from == to == self_peer_id` Poisons `pending_delivered` and Forces NAT Traversal to Attacker-Controlled Addresses — (`network/src/protocols/hole_punching/component/connection_request.rs`)

### Summary
An unprivileged remote peer with a direct connection to the victim can send a `ConnectionRequest` where both `from` and `to` are set to the victim's own peer ID. Because no guard rejects `from == to`, the victim inserts its own peer ID into `pending_delivered` with attacker-supplied listen addresses. A subsequent `ConnectionSync(from=victim_peer_id, to=victim_peer_id)` causes the victim to initiate outbound TCP connections to those attacker-controlled addresses and, on success, establish a full P2P session with the attacker's endpoint.

### Finding Description
`ConnectionRequestProcess::execute()` checks route loops, hop counts, address counts, and rate limits, but never validates that `content.from != content.to` or that `content.from != self_peer_id`. [6](#0-5) 

When `self_peer_id == &content.to` (line 145), `respond_delivered` is called with `from_peer_id = content.from`. If `content.from` was also set to `victim_peer_id`, `respond_delivered` inserts `victim_peer_id → ([attacker_addr], now)` into `pending_delivered` with no check that `from_peer_id != self_peer_id`. [7](#0-6) 

`ConnectionSyncProcess::execute()` then reads `pending_delivered.get(&content.from)` where `content.from` is again set to `victim_peer_id`, retrieves the poisoned entry, and calls `try_nat_traversal` for each attacker-controlled address. [8](#0-7) 

`try_nat_traversal` retries TCP `connect()` to the attacker's socket address for up to 30 seconds, and on success hands the stream to `control.raw_session(...)`, opening a full P2P session. [9](#0-8) 

### Impact Explanation
1. **Forced outbound TCP connections (SSRF)**: The victim node makes repeated TCP connection attempts to arbitrary attacker-controlled IP:port pairs for 30 seconds per `ConnectionSync` message.
2. **Unauthorized P2P session establishment**: If the TCP handshake succeeds, a full P2P session is opened to the attacker's endpoint via `raw_session`, bypassing normal peer-selection logic.
3. **State corruption**: `pending_delivered` is poisoned with `victim_peer_id → attacker_addrs` for up to 5 minutes (`TIMEOUT`), blocking legitimate hole-punching responses for the victim's own peer ID during that window. [10](#0-9) 

### Likelihood Explanation
- The attacker only needs to know the victim's peer ID (publicly discoverable on the P2P network) and have a single direct connection to the victim.
- No special privileges, no PoW, no key material required.
- The `forward_rate_limiter` key is `(from, to, msg_item_id)`. With `from == to`, the attacker can vary `msg_item_id` or wait 1 second between attempts to bypass it.
- The `HOLE_PUNCHING_INTERVAL` (2 minutes) only throttles re-insertion into `pending_delivered`, not the initial poisoning. [11](#0-10) [12](#0-11) 

### Recommendation
Add an explicit rejection at the start of `execute()` for the `from == to` case and for `from == self_peer_id`:

```rust
// In ConnectionRequestProcess::execute(), after parsing content:
if content.from == content.to {
    return StatusCode::InvalidFromPeerId
        .with_context("from and to must be different peers");
}
if &content.from == self_peer_id {
    return StatusCode::InvalidFromPeerId
        .with_context("from must not be the local peer");
}
``` [6](#0-5) 

Similarly, `respond_delivered` should assert `from_peer_id != self_peer_id` as a defense-in-depth guard. [13](#0-12) 

### Proof of Concept
```
1. Attacker (peer A, connected to victim V) learns V's peer_id via normal P2P discovery.
2. A sends: ConnectionRequest { from: V.peer_id, to: V.peer_id, listen_addrs: [attacker_ip:port], max_hops: 6, route: [] }
3. V.execute(): self_peer_id == content.to → calls respond_delivered(from=V.peer_id, to=V.peer_id, [attacker_ip:port])
4. V.respond_delivered(): pending_delivered.insert(V.peer_id, ([attacker_ip:port], now))
5. A sends: ConnectionSync { from: V.peer_id, to: V.peer_id, route: [] }
6. V.ConnectionSyncProcess::execute(): self_peer_id == content.to → pending_delivered.get(V.peer_id) → [attacker_ip:port]
7. V spawns try_nat_traversal(bind_addr, attacker_ip:port) → 30-second TCP connect loop to attacker-controlled address
8. On success: control.raw_session(stream, attacker_addr, ...) → full P2P session opened to attacker
```

            return StatusCode::InvalidMaxTTL.into();
        }

pub(crate) async fn try_nat_traversal(
    bind_addr: Option<SocketAddr>,
    addr: Multiaddr,
) -> Result<(TcpStream, Multiaddr), std::io::Error> {
    let net_addr = match multiaddr_to_socketaddr(&addr) {
        Some(addr) => addr,
        None => {
            debug!("Failed to convert multiaddr to socketaddr");
            return Err(std::io::ErrorKind::InvalidInput.into());
        }
    };

    // Use a fixed interval but add a small amount of randomness
    let base_retry_interval = Duration::from_millis(200);

    // total time
    let timeout_duration = Duration::from_secs(30);
    let start_time = Instant::now();
    let mut retry_count = 0u32;
    while start_time.elapsed() < timeout_duration {
        retry_count += 1;

        // Add a small amount of random jitter (±25ms) to avoid conflicts
        // caused by continuous precise synchronization
        let jitter = Duration::from_millis(rand::random::<u64>() % 50);
        let actual_interval = if rand::random::<bool>() {
            base_retry_interval + jitter
        } else {
            base_retry_interval.saturating_sub(jitter)
        };

        let socket = create_socket(bind_addr, net_addr)?;

        match runtime::timeout(
            std::time::Duration::from_millis(200),
            socket.connect(net_addr),
        )
        .await
        {
            Ok(Ok(stream)) => {
                // try get the stored error in the underlying socket
                // if the socket is not connected, it will return an error
                if let Err(err) = check_connection(&stream) {
                    debug!("Failed to connect to NAT(base check): {}", err);
                }
                return Ok((stream, addr));
            }
            Err(err) => {
                debug!("Failed to connect to NAT(timeout): {}", err);
            }
            Ok(Err(err)) => {
                if err.kind() == std::io::ErrorKind::AddrNotAvailable {
                    return Err(err);
                }
                debug!(
                    "Failed to connect to NAT(other error): {}, {}",
                    err.kind(),
                    err
                );
            }
        }
        runtime::delay_for(actual_interval).await;
    }

    debug!("Failed to connect to NAT after {} retries", retry_count);
    Err(std::io::ErrorKind::TimedOut.into())
