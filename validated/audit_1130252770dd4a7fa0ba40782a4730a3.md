### Title
Fallback Fee Rate Estimator Overestimates Block Capacity by Ignoring Block Overhead — (`tx-pool/src/pool.rs` and `tx-pool/src/component/pool_map.rs`)

---

### Summary

The fallback `estimate_fee_rate` function in `tx-pool/src/pool.rs` passes `max_block_bytes` as the per-block transaction capacity to `PoolMap::estimate_fee_rate`, but the actual space available for user transactions in a real block is `max_block_bytes − basic_block_size`, where `basic_block_size` accounts for the serialized cellbase, header, uncles, proposals, and extension. This causes the estimator to systematically overestimate how many transactions fit per block, producing a fee rate that is lower than what is actually required for timely confirmation.

---

### Finding Description

The fallback path of `TxPool::estimate_fee_rate` in `tx-pool/src/pool.rs` calls:

```rust
let fee_rate = self.pool_map.estimate_fee_rate(
    (target_to_be_committed - self.snapshot.consensus().tx_proposal_window().closest())
        as usize,
    self.snapshot.consensus().max_block_bytes() as usize,   // ← full block limit
    self.snapshot.consensus().max_block_cycles(),
    self.config.min_fee_rate,
);
``` [1](#0-0) 

`PoolMap::estimate_fee_rate` then simulates filling blocks by accumulating `entry.inner.size` bytes until `current_block_bytes >= max_block_bytes`:

```rust
for entry in iter {
    current_block_bytes += entry.inner.size;
    current_block_cycles += entry.inner.cycles;
    if current_block_bytes >= max_block_bytes || current_block_cycles >= max_block_cycles {
        target_blocks -= 1;
        ...
    }
}
``` [2](#0-1) 

However, in actual block assembly (`BlockAssembler::update_full` and `update_transactions`), the space available for committed transactions is not `max_block_bytes` but `max_block_bytes − basic_block_size`:

```rust
let txs_size_limit = max_block_bytes
    .checked_sub(basic_size)
    .ok_or(BlockAssemblerError::Overflow)?;
``` [3](#0-2) 

`basic_block_size` is computed by `BlockAssembler::basic_block_size`, which serializes a block containing the cellbase transaction, all uncle blocks, all proposal short IDs, and the extension (MMR hash after RFC0044), then measures its size: [4](#0-3) 

On mainnet, `max_block_bytes = 597,000` bytes. The overhead includes:
- Block header: 224 bytes
- Cellbase transaction: several hundred bytes
- Extension (RFC0044 active): 32 bytes
- Up to 2 uncle blocks: 228 bytes each
- Up to 1,500 proposal short IDs: 10 bytes each (up to 15,000 bytes) [5](#0-4) 

The estimator treats the full `max_block_bytes` as available for user transactions, ignoring this overhead entirely.

---

### Impact Explanation

The fallback fee estimator simulates more transactions fitting per block than actually can. As a result, it concludes that fewer blocks are needed to clear the mempool queue, and returns a fee rate that is lower than the true minimum required for confirmation within the requested number of blocks. Users who rely on the `estimate_fee_rate` RPC (with `enable_fallback = true`) to set their transaction fees will systematically underpay, causing their transactions to be confirmed later than expected. In a congested mempool, this can result in transactions being stuck for significantly longer than the user intended.

---

### Likelihood Explanation

The fallback algorithm is the default path when the primary fee estimator (`ConfirmationFraction` or `WeightUnitsFlow`) lacks sufficient historical data — which is common after node restarts, IBD completion, or on nodes that have not been running long. Any RPC caller invoking `estimate_fee_rate` (with or without `enable_fallback`) can trigger this path. The underestimation is systematic and reproducible, not probabilistic. [6](#0-5) 

---

### Recommendation

In `TxPool::estimate_fee_rate`, compute the actual per-block transaction byte budget by subtracting the minimum expected `basic_block_size` from `max_block_bytes` before passing it to `PoolMap::estimate_fee_rate`. A conservative lower bound for `basic_block_size` (e.g., header + cellbase + extension, without proposals or uncles) should be used so the estimate is never optimistic. This mirrors how `BlockAssembler::update_full` and `update_transactions` compute `txs_size_limit`.

---

### Proof of Concept

1. Node is freshly started (primary fee estimator not ready → fallback is used).
2. Mempool is filled with 1,000 transactions each of size 597 bytes (one "standard" tx).
3. Call `estimate_fee_rate` with `estimate_mode: "no_priority"` and `enable_fallback: true`.
4. The fallback simulates `597 * 1000 = 597,000` bytes per block, concluding all 1,000 transactions fit in exactly 1 block.
5. In reality, `basic_block_size` ≈ 500–1,000+ bytes (header + cellbase + extension), so only ~999 transactions fit per block.
6. The returned fee rate is lower than the true minimum, causing a user who sets fees at this rate to experience delayed confirmation. [7](#0-6) [8](#0-7)

    pub(crate) fn estimate_fee_rate(
        &self,
        mut target_blocks: usize,
        max_block_bytes: usize,
        max_block_cycles: Cycle,
        min_fee_rate: FeeRate,
    ) -> FeeRate {
        debug_assert!(target_blocks > 0);
        let iter = self.entries.iter_by_score().rev();
        let mut current_block_bytes = 0;
        let mut current_block_cycles = 0;
        for entry in iter {
            current_block_bytes += entry.inner.size;
            current_block_cycles += entry.inner.cycles;
            if current_block_bytes >= max_block_bytes || current_block_cycles >= max_block_cycles {
                target_blocks -= 1;
                if target_blocks == 0 {
                    return entry.inner.fee_rate();
                }
                current_block_bytes = entry.inner.size;
                current_block_cycles = entry.inner.cycles;
            }
        }

        min_fee_rate
    }

    pub(crate) async fn estimate_fee_rate(
        &self,
        estimate_mode: EstimateMode,
        enable_fallback: bool,
    ) -> Result<FeeRate, AnyError> {
        let all_entry_info = self.tx_pool.read().await.get_all_entry_info();
        match self
            .fee_estimator
            .estimate_fee_rate(estimate_mode, all_entry_info)
        {
            Ok(fee_rate) => Ok(fee_rate),
            Err(err) => {
                if enable_fallback {
                    let target_blocks =
                        FeeEstimator::target_blocks_for_estimate_mode(estimate_mode);
                    self.tx_pool
                        .read()
                        .await
                        .estimate_fee_rate(target_blocks)
                        .map_err(Into::into)
                } else {
                    Err(err.into())
                }
            }
        }
    }
