### Title
Unauthenticated Miner Notify HTTP Endpoint Accepts Arbitrary Block Template Injections - (File: miner/src/client.rs)

### Summary
When `ckb-miner` is configured in notify mode (`[miner.client] listen = "<addr>"`), it binds an HTTP server that accepts block template pushes from any source without authentication, rate limiting, or source IP validation. Any attacker who can reach the bound address can POST crafted `BlockTemplate` JSON to the endpoint, causing the miner to abandon its current valid work and switch to attacker-controlled templates, wasting all hashpower until the next legitimate push arrives.

### Impact Explanation

An attacker who can reach the miner's notify port can:

1. **Inject fake block templates** with arbitrary `work_id` values, causing all mining workers to immediately stop their current PoW search and begin hashing on an attacker-controlled (invalid) template. Every injected template with a new `work_id` triggers a full worker reset via `Works::New`.
2. **Sustain the disruption** by continuously sending templates with incrementing `work_id` values, keeping the miner permanently on invalid work. The miner will never produce a valid block submission during the attack.
3. **Cause financial loss** to the miner operator: all hashpower is wasted, block rewards are missed, and the miner's subscription to the CKB node's block assembler is effectively hijacked.

There is no mechanism for the miner to distinguish a legitimate push from the CKB node from an attacker's push.

### Likelihood Explanation

- Notify mode is a documented, supported production feature (`ckb-miner.toml` ships with the option).
- Operators running distributed miner farms (workers on separate machines from the CKB node) must bind to a non-loopback address, directly exposing the endpoint.
- No privilege is required: any TCP client that can reach the port can exploit this.
- The attack is trivially scriptable: a single `curl -X POST -d '<crafted_json>' http://<miner>:8888/` suffices.

### Recommendation

1. Add a shared-secret token to the notify URL (e.g., `http://127.0.0.1:8888/?token=<secret>`) and validate it in `handle` before processing the body.
2. Alternatively, enforce an IP allowlist in `listen_block_template_notify` by comparing the accepted connection's remote address against a configured set of trusted CKB node IPs.
3. Add per-IP rate limiting analogous to the existing `forward_rate_limiter` used in the hole-punching protocol.

### Proof of Concept

1. Configure `ckb-miner.toml` with `listen = "0.0.0.0:8888"` and start the miner in notify mode.
2. From any reachable host, send a crafted template with a new `work_id`:
   ```bash
   curl -s -X POST http://<miner-ip>:8888/ \
     -H 'Content-Type: application/json' \
     -d '{"version":"0x0","compact_target":"0x1e083126","current_time":"0x174c45e17a3",
          "number":"0x1","epoch":"0x7080018000001","parent_hash":"0x000...000",
          "cycles_limit":"0xd09dc300","bytes_limit":"0x91c08","uncles_count_limit":"0x2",
          "uncles":[],"transactions":[],"proposals":[],"cellbase":{"cycles":null,"data":{...},"hash":"0x..."},
          "work_id":"0x9999","dao":"0x000...000"}'
   ```
3. Observe that all mining workers immediately switch to the injected template (logged as `WorkerMessage::NewWork`).
4. Repeat with incrementing `work_id` values in a tight loop to permanently prevent the miner from working on any legitimate template.

    async fn listen_block_template_notify(&self, addr: SocketAddr) {
        let listener = TcpListener::bind(addr).await.unwrap();
        let server = auto::Builder::new(TokioExecutor::new());
        let graceful = GracefulShutdown::new();
        let stop_rx: CancellationToken = new_tokio_exit_rx();

        loop {
            let client = self.clone();
            let handle = service_fn(move |req| handle(client.clone(), req));
            tokio::select! {
                conn = listener.accept() => {
                    let (stream, _) = match conn {
                        Ok(conn) => conn,
                        Err(e) => {
                            info!("accept error: {}", e);
                            tokio::time::sleep(std::time::Duration::from_secs(1)).await;
                            continue;
                        }
                    };
                    let stream = hyper_util::rt::TokioIo::new(Box::pin(stream));
                    let conn = server.serve_connection_with_upgrades(stream, handle);

                    let conn = graceful.watch(conn.into_owned());
                    tokio::spawn(async move {
                        if let Err(err) = conn.await {
                            info!("connection error: {}", err);
                        }
                    });
                },
                _ = stop_rx.cancelled() => {
                    info!("Miner client received exit signal. Exit now");
                    break;
                }
            }
        }
        drop(listener);
        graceful.shutdown().await;
    }

            return;
        }

                    let client = Arc::clone(&self.poster);
                    let url = url.to_owned();

                        let _resp =

                                .map_err(|_| {
                                    ckb_logger::warn!(
                                        "block assembler notifying {} timed out",
                                        url
                                    );
