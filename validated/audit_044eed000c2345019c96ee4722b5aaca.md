### Title
DAO Withdrawal `header_dep_index` Parsed as Full `u64` in Rust but as Single Byte in C VM, Causing Consensus Split — (`util/dao/src/lib.rs`)

### Summary

The Rust `DaoCalculator::transaction_maximum_withdraw` reads the `header_dep_index` from the witness `input_type` field as a full `u64`, while the on-chain C VM (`dao.c`) reads only the **lowest byte** of that value. For any DAO withdrawal where the encoded index is `>= 256`, the Rust node and the C VM resolve to **different entries** in `header_deps`, creating a consensus split: the C VM accepts the transaction while the Rust node rejects it, and the Rust node computes an incorrect DAO field for block assembly.

### Finding Description

The Nervos DAO withdrawal protocol requires the transaction witness `input_type` field (inside `WitnessArgs`) to carry a `u64` index pointing to the deposit block's hash inside `header_deps`. The Rust `DaoCalculator` in `util/dao/src/lib.rs` reads this index as a full little-endian `u64`:

```rust
// util/dao/src/lib.rs line 91
Ok(LittleEndian::read_u64(&header_deps_index_data.unwrap()))
```

It then uses this value directly to index into `header_deps`:

```rust
// util/dao/src/lib.rs lines 94-98
rtx.transaction
    .header_deps()
    .get(header_dep_index as usize)
    .and_then(|hash| header_deps.get(&hash))
    .ok_or(DaoError::InvalidOutPoint)
```

The on-chain C VM (`dao.c`), however, reads only the **lowest byte** of the same 8-byte field. This is explicitly documented in the test suite:

```
// Position 1: correct deposit block (what C VM resolves via lowest byte).
// Position 257: withdraw block (wrong — Rust resolves this with full u64).
// input_type = 257, lowest byte = 1
```

When a transaction encodes `header_dep_index = 257` (little-endian `u64`):
- **C VM** reads lowest byte → `1` → resolves `header_deps[1]` = deposit block ✓
- **Rust** reads full `u64` → `257` → resolves `header_deps[257]` = withdraw block ✗

The Rust code then compares `deposit_header.number()` against `deposited_block_number` stored in cell data. Because it resolved the wrong header (the withdraw block at number 200 instead of the deposit block at number 100), the check fails and the transaction is rejected with `DaoError::InvalidOutPoint`. [1](#0-0) [2](#0-1) 

### Impact Explanation

**Denial of service on legitimate DAO withdrawals**: Any user whose deposit block hash lands at position `>= 256` in `header_deps` cannot submit a valid withdrawal through the Rust node's tx-pool, even though the on-chain C VM would accept it.

**Incorrect DAO field in assembled blocks**: `DaoCalculator::dao_field` calls `withdrawed_interests`, which calls `transaction_maximum_withdraw`. If a miner bypasses the tx-pool and directly includes such a transaction in a block, the Rust node computes the DAO field using the wrong deposit header (the withdraw block instead of the deposit block). The resulting DAO field is incorrect, causing the assembled block to be rejected by other nodes that correctly compute the DAO field. This is a **consensus split**.

**Wrong block validation**: Nodes that receive a block containing such a DAO withdrawal will compute different DAO field values depending on whether they use the Rust `DaoCalculator` or the C VM logic, leading to divergent chain views. [3](#0-2) 

### Likelihood Explanation

A transaction sender can craft a DAO withdrawal with `>= 256` entries in `header_deps` and encode the deposit block index as a value `>= 256` whose lowest byte points to the correct deposit block. No special privilege is required — this is a standard RPC-submitted transaction. While having 256+ `header_deps` is unusual in normal usage, it is not prohibited by the protocol, and the discrepancy is reachable by any unprivileged transaction sender. The test suite explicitly constructs this scenario with 258 `header_deps`. [4](#0-3) 

### Recommendation

The Rust `DaoCalculator` must parse `header_dep_index` consistently with the C VM. If the C VM reads only the lowest byte, the Rust code should do the same:

```rust
// Instead of:
Ok(LittleEndian::read_u64(&header_deps_index_data.unwrap()))

// Use only the lowest byte, consistent with dao.c:
Ok(header_deps_index_data.unwrap()[0] as u64)
```

Alternatively, if the intended protocol behavior is to use the full `u64`, the C VM (`dao.c`) must be updated to read all 8 bytes. Either way, both the Rust `DaoCalculator` and the C VM must agree on the same interpretation to prevent the consensus split.

### Proof of Concept

The test `check_dao_withdraw_header_dep_index_exceeds_u8` in `util/dao/src/tests.rs` directly demonstrates the discrepancy:

1. Build a transaction with 258 `header_deps`:
   - `header_deps[1]` = deposit block (number 100)
   - `header_deps[257]` = withdraw block (number 200)
2. Set cell data to `100u64` (deposit block number)
3. Set witness `input_type` = `257u64` (little-endian 8 bytes: `[0x01, 0x01, 0x00, ...]`)
4. C VM reads lowest byte = `0x01` = 1 → `header_deps[1]` = deposit block (number 100) → matches cell data → **accepts**
5. Rust reads full `u64` = 257 → `header_deps[257]` = withdraw block (number 200) → does not match cell data (100) → **rejects with `InvalidOutPoint`**

The test asserts `result.is_err()`, confirming the Rust node incorrectly rejects a transaction the C VM would accept. [5](#0-4) [6](#0-5)

    fn transaction_maximum_withdraw(
        &self,
        rtx: &ResolvedTransaction,
    ) -> Result<Capacity, DaoError> {
        let header_deps: HashSet<Byte32> = rtx.transaction.header_deps_iter().collect();
        rtx.resolved_inputs.iter().enumerate().try_fold(
            Capacity::zero(),
            |capacities, (i, cell_meta)| {
                let capacity: Result<Capacity, DaoError> = {
                    let output = &cell_meta.cell_output;
                    let is_dao_type_script = |type_script: Script| {
                        Into::<u8>::into(type_script.hash_type())
                            == Into::<u8>::into(ScriptHashType::Type)
                            && type_script.code_hash() == self.consensus.dao_type_hash()
                    };
                    let is_dao_output = output
                        .type_()
                        .to_opt()
                        .map(is_dao_type_script)
                        .unwrap_or(false);
                    if is_dao_output {
                        // A withdrawing DAO cell has 8 bytes of cell data storing the
                        // block number of the original deposit.
                        let deposited_block_number =
                            match self.data_loader.load_cell_data(cell_meta) {
                                Some(data) if data.len() == 8 => LittleEndian::read_u64(&data),
                                _ => 0,
                            };
                        if deposited_block_number > 0 {
                            let withdrawing_header_hash = cell_meta
                                .transaction_info
                                .as_ref()
                                .map(|info| &info.block_hash)
                                .filter(|hash| header_deps.contains(hash))
                                .ok_or(DaoError::InvalidOutPoint)?;
                            let deposit_header_hash = rtx
                                .transaction
                                .witnesses()
                                .get(i)
                                .ok_or(DaoError::InvalidOutPoint)
                                .and_then(|witness_data| {
                                    // dao contract stores header deps index as u64 in the input_type field of WitnessArgs
                                    let witness =
                                        WitnessArgs::from_slice(&Into::<Bytes>::into(witness_data))
                                            .map_err(|_| DaoError::InvalidDaoFormat)?;
                                    let header_deps_index_data: Option<Bytes> =
                                        witness.input_type().to_opt().map(|witness| witness.into());
                                    if header_deps_index_data.is_none()
                                        || header_deps_index_data.clone().map(|data| data.len())
                                            != Some(8)
                                    {
                                        return Err(DaoError::InvalidDaoFormat);
                                    }
                                    Ok(LittleEndian::read_u64(&header_deps_index_data.unwrap()))
                                })
                                .and_then(|header_dep_index| {
                                    rtx.transaction
                                        .header_deps()
                                        .get(header_dep_index as usize)
                                        .and_then(|hash| header_deps.get(&hash))
                                        .ok_or(DaoError::InvalidOutPoint)
                                })?;

                            let deposit_header = self
                                .data_loader
                                .get_header(deposit_header_hash)
                                .ok_or(DaoError::InvalidHeader)?;
                            if deposit_header.number() != deposited_block_number {
                                return Err(DaoError::InvalidOutPoint);
                            }
                            self.calculate_maximum_withdraw(
                                output,
                                Capacity::bytes(cell_meta.data_bytes as usize)?,
                                deposit_header_hash,
                                withdrawing_header_hash,
                            )
                        } else {
                            Ok(output.capacity().into())
                        }
                    } else {
                        Ok(output.capacity().into())
                    }
                };
                capacity.and_then(|c| c.safe_add(capacities).map_err(Into::into))
            },
        )
    }

#[test]
fn check_dao_withdraw_header_dep_index_exceeds_u8() {
    let deposit_number = 100u64;
    let withdraw_number = 200u64;

    let (_tmp_dir, store, deposit_block, withdraw_block) =
        setup_store_with_headers(deposit_number, withdraw_number);

    let consensus = Consensus::default();
    let dao_type_script = Script::new_builder()
        .code_hash(consensus.dao_type_hash())
        .hash_type(ScriptHashType::Type)
        .build();

    // Pad header_deps to 258 entries so index 257 is valid.
    // Position 1: correct deposit block (what C VM resolves via lowest byte).
    // Position 257: withdraw block (wrong — Rust resolves this with full u64).
    let dummy = h256!("0x1").into();
    let mut header_deps = vec![dummy; 258];
    header_deps[1] = deposit_block.hash();
    header_deps[257] = withdraw_block.hash();

    let cell_data = Bytes::from(deposit_number.to_le_bytes().to_vec());
    let input_cell = CellOutput::new_builder()
        .capacity(capacity_bytes!(1000000))
        .type_(Some(dao_type_script).pack())
        .build();
    let tx_info = TransactionInfo::new(
        withdraw_block.number(),
        withdraw_block.epoch(),
        withdraw_block.hash(),
        0,
    );
    let cell_meta = CellMetaBuilder::from_cell_output(input_cell, cell_data)
        .transaction_info(tx_info)
        .build();

    // input_type = 257, lowest byte = 1
    let witness = WitnessArgs::new_builder()
        .input_type(Some(Bytes::from(257u64.to_le_bytes().to_vec())))
        .build();
    let witness_bytes: Bytes = witness.as_bytes();

    let tx = TransactionBuilder::default()
        .set_header_deps(header_deps)
        .witness(witness_bytes)
        .build();

    let rtx = ResolvedTransaction {
        transaction: tx,
        resolved_cell_deps: vec![],
        resolved_inputs: vec![cell_meta],
        resolved_dep_groups: vec![],
    };

    let data_loader = store.borrow_as_data_loader();
    let calculator = DaoCalculator::new(&consensus, &data_loader);
    let result = calculator.transaction_fee(&rtx);

    // Rust resolves index 257 → withdraw block (number 200), but cell data
    // says deposited at block 100. Block number check catches the mismatch.
    assert!(result.is_err(), "expected Err, got {result:?}");
}
