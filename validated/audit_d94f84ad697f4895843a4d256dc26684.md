### Title
Ghost `edges.inputs` Entry via Duplicate-Input Transaction Enables Targeted UTXO DoS — (`tx-pool/src/component/edges.rs`, `tx-pool/src/component/pool_map.rs`)

### Summary
A transaction with a duplicate input (same `OutPoint` appearing twice in its input list) bypasses all pre-verification checks and reaches `record_entry_edges`. The first occurrence of the outpoint is successfully inserted into `edges.inputs`; the second triggers `Err(Reject::RBFRejected)`. Because there is no rollback, the first outpoint remains as a ghost entry in `edges.inputs` permanently. Any subsequent legitimate transaction spending that outpoint is incorrectly rejected as a double-spend, effectively locking the UTXO in the tx-pool until node restart.

### Finding Description

The tx-pool admission pipeline is:

```
process_tx
  → non_contextual_verify          (NonContextualTransactionVerifier — no duplicate-input check)
  → pre_check / resolve_tx         (ResolvedTransaction::check — seen_inputs extended only at end, no intra-tx dup detection)
  → submit_entry → add_entry
      → check_and_record_ancestors (modifies links — also not rolled back)
      → record_entry_edges         (iterates inputs, inserts one-by-one with `?`)
          insert_input(A, id) → Ok   ← ghost entry written
          insert_input(A, id) → Err(RBFRejected) ← early return, no cleanup
```

`NonContextualTransactionVerifier` checks for duplicate `cell_deps` and `header_deps` via `DuplicateDepsVerifier`, but has no equivalent for transaction inputs. [7](#0-6) 

`ResolvedTransaction::check` passes `seen_inputs` by reference but only extends it after the full input loop completes, so duplicate inputs within the same transaction are invisible to it. [8](#0-7) 

`record_entry_edges` uses `?` to propagate the first error, leaving all previously-inserted inputs as ghost entries. [9](#0-8) 

### Impact Explanation

After the attack:
- `edges.inputs` contains `{A → short_id_of_rejected_tx}` permanently.
- The rejected tx is **not** in `entries` (never reached `insert_entry`).
- Any future tx spending `A` calls `insert_input(A, new_id)`, hits `Entry::Occupied`, and is rejected with `Err(Reject::RBFRejected(...))`.
- The outpoint `A` is effectively unspendable through this node's tx-pool until the node restarts or the pool is fully cleared.
- An attacker can target any live UTXO they know about (e.g., from chain scanning) and permanently block it from being relayed or mined through that node.
- The error message incorrectly attributes the rejection to RBF, leaking internal state and potentially causing divergent behavior across nodes.

Additionally, `check_and_record_ancestors` (which runs before `record_entry_edges`) also modifies `self.links` and is not rolled back, leaving orphaned link entries. [10](#0-9) 

### Likelihood Explanation

The attack requires no privileges. Any RPC caller or P2P peer can submit a transaction. Constructing a tx with duplicate inputs is trivial — the serialization format does not prevent it, and `NonContextualTransactionVerifier` explicitly does not check for it. The attacker only needs to know a live outpoint (trivially obtained from chain data).

### Recommendation

1. Add a `DuplicateInputsVerifier` to `NonContextualTransactionVerifier` that checks for duplicate outpoints in `tx.inputs()`, analogous to `DuplicateDepsVerifier`.
2. As a defense-in-depth fix, add rollback logic to `record_entry_edges`: if any `insert_input` fails, remove all previously-inserted inputs for this entry before returning the error.
3. Change the error returned by `insert_input` when `old_tx == new_tx` (same short ID) to a more specific variant (e.g., `Reject::DuplicateInput`) rather than `RBFRejected`, which is semantically incorrect for this case.

    pub fn verify(&self) -> Result<(), Error> {
        let transaction = self.transaction;
        let mut seen_cells = HashSet::with_capacity(self.transaction.cell_deps().len());
        let mut seen_headers = HashSet::with_capacity(self.transaction.header_deps().len());

        if let Some(dep) = transaction
            .cell_deps_iter()
            .find_map(|dep| seen_cells.replace(dep))
        {
            return Err(TransactionError::DuplicateCellDeps {
                out_point: dep.out_point(),
            }
            .into());
        }
        if let Some(hash) = transaction
            .header_deps_iter()
            .find_map(|hash| seen_headers.replace(hash))
        {
            return Err(TransactionError::DuplicateHeaderDeps { hash }.into());
        }
        Ok(())
    }

                    checked_cells.insert(out_point.clone());
                    Ok(())

                Some(false) => Err(OutPointError::Dead(out_point.clone())),
                None => Err(OutPointError::Unknown(out_point.clone())),
            }
        };

        // // check input

            check_cell(&cell_meta.out_point)?;
        }

        let mut resolved_system_deps: HashSet<&OutPoint> = HashSet::new();
        if let Some(system_cell) = SYSTEM_CELL.get() {

                    .build();

                let dep_group = system_cell.get(&cell_dep);

                    .build();

                if system_cell.get(&cell_dep).is_none()
                    && !resolved_system_deps.contains(&cell_meta.out_point)
                {
                    check_cell(&cell_meta.out_point)?;
                }
            }
        } else {
            for cell_meta in self
                .resolved_cell_deps
                .iter()
                .chain(self.resolved_dep_groups.iter())
            {
                check_cell(&cell_meta.out_point)?;
            }

        for block_hash in self.transaction.header_deps_iter() {
            header_checker.check_valid(&block_hash)?
        }

    fn record_entry_edges(&mut self, entry: &TxEntry) -> Result<(), Reject> {
        let tx_short_id: ProposalShortId = entry.proposal_short_id();
        let header_deps = entry.transaction().header_deps();
        let related_dep_out_points: Vec<_> = entry.related_dep_out_points().cloned().collect();
        let inputs = entry.transaction().input_pts_iter();

        // if input reference a in-pool output, connect it
        // otherwise, record input for conflict check
        for i in inputs {
            self.edges.insert_input(i.to_owned(), tx_short_id.clone())?;
        }

        // record dep-txid
        for d in related_dep_out_points {
            self.edges.insert_deps(d.to_owned(), tx_short_id.clone());
        }
        // record header_deps
        if !header_deps.is_empty() {
            self.edges
                .header_deps
                .insert(tx_short_id, header_deps.into_iter().collect());
        }
        Ok(())
    }

    pub(crate) fn insert_input(
        &mut self,
        out_point: OutPoint,
        txid: ProposalShortId,
    ) -> Result<(), Reject> {
        // inputs is occupied means double speanding happened here
        match self.inputs.entry(out_point.clone()) {
            Entry::Occupied(occupied) => {
                let msg = format!(
                    "txpool unexpected double-spending out_point: {:?} old_tx: {:?} new_tx: {:?}",
                    out_point,
                    occupied.get(),
                    txid
                );
                Err(Reject::RBFRejected(msg))
            }
            Entry::Vacant(vacant) => {
                vacant.insert(txid);
                Ok(())
            }
        }
    }
