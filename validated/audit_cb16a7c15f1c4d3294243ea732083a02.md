### Title
`updatePriceFeedsIfNecessary` Early-Exit Allows Partial Update, Leaving Requested Feeds Stale on Successful Return — (`target_chains/ethereum/contracts/contracts/pyth/Pyth.sol`)

---

### Summary

`updatePriceFeedsIfNecessary` exits as soon as it finds the **first** stale price ID and calls `updatePriceFeeds`. It never verifies that **all** requested price IDs are fresh after the update. An unprivileged relayer can supply `updateData` that contains only a subset of the requested feeds, causing the function to return successfully while leaving other requested feeds stale.

---

### Finding Description

The production override of `updatePriceFeedsIfNecessary` in `Pyth.sol` reads:

```solidity
for (uint i = 0; i < priceIds.length; ) {
    if (latestPriceInfoPublishTime(priceIds[i]) < publishTimes[i]) {
        updatePriceFeeds(updateData);   // called with attacker-supplied blob
        return;                         // exits immediately — no post-check
    }
    unchecked { i++; }
}
revert PythErrors.NoFreshUpdate();
``` [1](#0-0) 

The loop finds the first `priceIds[i]` whose stored `publishTime < publishTimes[i]`, calls `updatePriceFeeds(updateData)`, and **returns**. There is no second pass to confirm every requested feed now satisfies `latestPriceInfoPublishTime(priceIds[i]) >= publishTimes[i]`.

`updatePriceFeeds` itself only stores whatever price IDs happen to be present in the supplied Merkle blob:

```solidity

It does not require the blob to contain every price ID listed in `priceIds`. A blob containing only feed A is perfectly valid and will succeed.

The same early-exit pattern exists in the base `AbstractPyth.updatePriceFeedsIfNecessary`: [3](#0-2) 

**Concrete attack path:**

| Step | State |
|---|---|
| Precondition | `latestPriceInfoPublishTime(A) < T_A` and `latestPriceInfoPublishTime(B) < T_B` |
| Attacker calls | `updatePriceFeedsIfNecessary(updateData_A_only, [A,B], [T_A,T_B])` |
| Loop iteration i=0 | `publishTime(A) < T_A` → true → `updatePriceFeeds(updateData_A_only)` called |
| After `updatePriceFeeds` | A is updated; B is absent from blob, B remains stale |
| Function | `return` — no error, no revert |
| Victim protocol | Reads both A and B, believing both are fresh |

### Impact Explanation

The `IPyth` NatSpec documents the function as a wrapper that "calls `updatePriceFeeds` method to update the prices" when any feed is stale, implying all requested feeds will be fresh on success. [4](#0-3) 

Protocols that rely on this guarantee — accepting `updateData` from an external caller and then reading prices via `getPriceUnsafe` or similar — can be fed a stale price for feed B. In a lending/liquidation context, a stale collateral or debt price enables:

- Forced liquidations at incorrect ratios
- Borrowing against inflated collateral
- Theft of funds proportional to the price deviation between the stale and true value

The impact is direct fund loss in any downstream protocol that treats a successful `updatePriceFeedsIfNecessary` return as a freshness guarantee for all listed feeds.

---

### Likelihood Explanation

- The attacker is fully unprivileged; no key or governance access is required.
- The standard Pyth integration pattern has an external caller (keeper/relayer) supply `updateData` to a victim protocol's function, which then calls `updatePriceFeedsIfNecessary`. This is the exact entry point.
- Constructing a valid Merkle blob for only one feed is straightforward using the Hermes API.
- Protocols that use `getPriceUnsafe` (or equivalent) after the call, trusting the function as a freshness gate, are directly vulnerable.

---

### Recommendation

After calling `updatePriceFeeds`, add a post-update verification loop before returning:

```solidity
function updatePriceFeedsIfNecessary(
    bytes[] calldata updateData,
    bytes32[] calldata priceIds,
    uint64[] calldata publishTimes
) external payable override {
    if (priceIds.length != publishTimes.length)
        revert PythErrors.InvalidArgument();

    bool needsUpdate = false;
    for (uint i = 0; i < priceIds.length; ) {
        if (latestPriceInfoPublishTime(priceIds[i]) < publishTimes[i]) {
            needsUpdate = true;
            break;
        }
        unchecked { i++; }
    }

    if (!needsUpdate) revert PythErrors.NoFreshUpdate();

    updatePriceFeeds(updateData);

    // Post-update: verify ALL requested feeds are now fresh
    for (uint i = 0; i < priceIds.length; ) {
        if (latestPriceInfoPublishTime(priceIds[i]) < publishTimes[i])
            revert PythErrors.PriceFeedNotFoundWithinRange();
        unchecked { i++; }
    }
}
```

This ensures the function's documented invariant — that all requested feeds are fresh on successful return — is actually enforced.
