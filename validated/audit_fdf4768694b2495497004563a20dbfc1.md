### Title
Missing Zero-Authority Validation in `Initialize` Permanently Bricks `InitializePublisher` — (`accounts/config.rs`, `processor/initialize.rs`)

---

### Summary

The `Initialize` instruction accepts any 32-byte value as `authority` and stores it verbatim in the `Config` PDA with no validation. If `authority = [0u8; 32]` (the all-zeros / `Pubkey::default()`) is stored, `validate_authority` can never be satisfied by any real signer, and because the config PDA can only be initialized once, the program is permanently unable to register publishers.

---

### Finding Description

`config::create` stores the caller-supplied authority bytes without any non-zero check: [1](#0-0) 

The `AlreadyInitialized` guard prevents a second `Initialize` call from overwriting the bad value: [2](#0-1) 

`validate_authority` requires both `authority.is_signer` **and** `authority.key.to_bytes() == config.authority`: [3](#0-2) 

If `config.authority` is `[0u8; 32]`, the matching account must be the all-zeros pubkey, which has no known private key — so `is_signer` can never be true for it. There is no `UpdateAuthority` instruction anywhere in the program, and no upgrade path is provided. [4](#0-3) 

The `Initialize` instruction itself has no caller access control — any funded account can be the payer and supply arbitrary `authority` bytes: [5](#0-4) 

---

### Impact Explanation

Every future `InitializePublisher` call routes through `validate_authority`: [6](#0-5) 

With a zero authority stored, every such call returns `MissingRequiredSignature` unconditionally. No publisher can ever be registered, making the entire price-store program permanently inoperable. This matches the scoped impact: *smart contract unable to operate, griefing without profit motive*.

---

### Likelihood Explanation

Two realistic paths exist:

1. **Front-run attack**: `Initialize` is permissionless. An attacker monitoring the network after program deployment can submit `Initialize(authority=[0;32])` before the legitimate operator, locking the zero authority in permanently.
2. **Operator error**: A deployment script that zero-initializes a byte buffer before filling in the authority could accidentally pass all-zeros if the fill step is skipped or fails silently.

Both paths require no privileged access and are locally testable.

---

### Recommendation

Add a non-zero check in `config::create` before storing the authority:

```rust
// In accounts/config.rs, config::create
if authority == [0u8; 32] {
    return Err(ReadAccountError::InvalidAuthority);
}
data.authority = authority;
```

Optionally, also verify in `initialize` that `args.authority` matches the payer's key (i.e., the initializer must prove they control the authority at initialization time), which would prevent both front-running and accidental zero-authority initialization.

---

### Proof of Concept

Call sequence (unit-testable without a live cluster):

```
1. Deploy program.
2. Call Initialize with authority = [0u8; 32] as any funded payer.
3. Config PDA is created; config.authority = [0u8; 32].
4. Call Initialize again → AlreadyInitialized (cannot overwrite).
5. Call InitializePublisher with any real signer as authority →
   validate_authority: authority.key.to_bytes() != [0u8;32] → MissingRequiredSignature.
6. No publisher can ever be registered.
```

Relevant code path: [1](#0-0) [7](#0-6)

```rust
pub fn create(data: &mut [u8], authority: [u8; 32]) -> Result<&mut Config, ReadAccountError> {
    if data.len() < size_of::<Config>() {
        return Err(ReadAccountError::DataTooShort);
    }
    let data: &mut Config = from_bytes_mut(&mut data[..size_of::<Config>()]);
    if data.format != 0 {
        return Err(ReadAccountError::AlreadyInitialized);
    }
    data.format = FORMAT;
    data.authority = authority;
    Ok(data)
```

```rust
pub fn validate_authority<'a, 'b>(
    account: Option<&'b AccountInfo<'a>>,
    config: &AccountInfo<'a>,
) -> Result<&'b AccountInfo<'a>, ProgramError> {
    let authority = account.ok_or(ProgramError::NotEnoughAccountKeys)?;
    ensure!(ProgramError::MissingRequiredSignature, authority.is_signer);
    ensure!(ProgramError::InvalidArgument, authority.is_writable);
    let config_data = config.data.borrow();
    let config = accounts::config::read(*config_data)?;
    ensure!(
        ProgramError::MissingRequiredSignature,
        authority.key.to_bytes() == config.authority
    );
    Ok(authority)
```
