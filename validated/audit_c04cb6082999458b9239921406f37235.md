### Title
Unprivileged RPC caller can trigger panic via `get_fee_rate_statistics` on nodes upgraded from pre-v0.106 — (`rpc/src/util/fee_rate.rs`)

### Summary
`FeeRateCollector::statistics()` calls `.expect()` on `BlockExt::txs_sizes` without guarding against `None`. Nodes upgraded from pre-v0.106 retain old-format `BlockExt` records (5-field `BlockExt`, not 7-field `BlockExtV1`) in their database. Reading those records via `get_block_ext` produces `txs_sizes: None`. Any unprivileged caller of the `get_fee_rate_statistics` RPC during the first ~101 blocks after the upgrade triggers the panic.

### Finding Description
`get_block_ext` in `store/src/store.rs` uses `from_compatible_slice_should_be_ok` and branches on `count_extra_fields()`. For old records with 5 fields (`count_extra_fields() == 0`), it converts via `BlockExtReader::into()`, which hard-codes `cycles: None` and `txs_sizes: None` in `util/types/src/conversion/storage.rs` (lines 154–166). No migration exists to back-fill these fields.

`FeeRateCollector::collect()` uses `filter_map` to skip missing blocks, but old-format blocks are not missing — they return `Some(BlockExt { txs_sizes: None, ... })`. These pass through `filter_map` and reach line 93 of `rpc/src/util/fee_rate.rs`:

```rust
let txs_sizes = txs_sizes.expect("expect txs_size's length >= 1");
```

This panics unconditionally for any old-format block within the query window.

### Impact Explanation
The panic aborts the RPC handler task. In a Tokio-based server this causes the `get_fee_rate_statistics` call to fail with an unhandled panic. Depending on executor configuration, it may crash the node process entirely. The impact is a local RPC API crash, matching the stated scope.

### Likelihood Explanation
Any operator who upgrades a long-running mainnet/testnet node from a version before v0.106 is affected for the first ~101 blocks after the upgrade. The RPC endpoint is unauthenticated by default. An attacker who knows the node was recently upgraded can reliably trigger the panic with a single RPC call.

### Recommendation
Replace the unconditional `.expect()` with a guard that skips blocks whose `txs_sizes` is `None`:

```rust
let Some(txs_sizes) = txs_sizes else {
    return fee_rates; // skip legacy block, no txs_sizes data
};
```

This makes `statistics()` gracefully handle legacy database records, consistent with the documented invariant that the RPC must tolerate pre-v0.106 storage formats.

### Proof of Concept

2. Construct a `Snapshot` over that DB.
3. Call `FeeRateCollector::new(&snapshot).statistics(Some(101))`.
4. Observe panic: `"expect txs_size's length >= 1"` at `rpc/src/util/fee_rate.rs:93`.
