### Title
Arithmetic Underflow in `ema_price_no_older_than` Due to Missing Guard on `current_time - price.publish_time` - (File: `target_chains/fuel/contracts/pyth-contract/src/main.sw`)

---

### Summary

`ema_price_no_older_than` in the Fuel Pyth contract performs `current_time - price.publish_time` without first verifying that `current_time >= price.publish_time`. The sibling function `price_no_older_than` was patched with an explicit guard for this exact case, but `ema_price_no_older_than` was not. In Sway, unsigned integer underflow causes a hard revert, making the function permanently unusable for any price feed whose stored `publish_time` exceeds the current block timestamp.

---

### Finding Description

In `target_chains/fuel/contracts/pyth-contract/src/main.sw`, two functions perform a time-difference check:

**Vulnerable — `ema_price_no_older_than` (lines 311–320):**
```sway
fn ema_price_no_older_than(time_period: u64, price_feed_id: PriceFeedId) -> Price {
    let price = ema_price_unsafe(price_feed_id);
    let current_time = timestamp();
    require(
        current_time - price.publish_time <= time_period,  // ← no guard
        PythError::OutdatedPrice,
    );
    price
}
``` [1](#0-0) 

**Fixed — `price_no_older_than` (lines 330–343):**
```sway
fn price_no_older_than(time_period: u64, price_feed_id: PriceFeedId) -> Price {
    let price = price_unsafe(price_feed_id);
    let current_time = timestamp();
    // Mimicking saturating subtraction to avoid underflow
    let time_difference = if current_time > price.publish_time {
        current_time - price.publish_time
    } else {
        0
    };
    require(time_difference <= time_period, PythError::OutdatedPrice);
    price
}
``` [2](#0-1) 

The comment in `price_no_older_than` explicitly acknowledges the underflow risk and applies a saturating guard. `ema_price_no_older_than` has no such guard. In Sway, subtracting a larger `u64` from a smaller one causes an unconditional revert (panic), not a silent wrap.

---

### Impact Explanation

If any EMA price feed stored in the contract has a `publish_time` greater than the current Fuel block `timestamp()` — which can occur due to clock drift between Pythnet and the Fuel chain, or because a VAA carrying a future-timestamped EMA price was accepted — every call to `ema_price_no_older_than` for that feed will revert with an arithmetic underflow. This permanently DoS-es the EMA price freshness check for the affected feed until the Fuel block timestamp catches up. Any downstream protocol relying on `ema_price_no_older_than` (e.g., for TWAP/EMA-based liquidations or collateral pricing) will be unable to read EMA prices during this window.

---

### Likelihood Explanation

Pyth price updates are permissionlessly submitted by any transaction sender. The `publish_time` field is embedded in the VAA payload signed by Wormhole guardians and reflects Pythnet's clock. Minor clock skew between Pythnet and the Fuel chain is a realistic, non-adversarial condition. Additionally, an attacker who submits a valid VAA (signed by guardians) with a `publish_time` slightly ahead of the Fuel chain's `timestamp()` can reliably trigger the revert. The entry path requires no privileged access — only the ability to call `update_price_feeds` with a legitimate VAA.

---

### Recommendation

Apply the same saturating-subtraction guard used in `price_no_older_than` to `ema_price_no_older_than`:

```sway
fn ema_price_no_older_than(time_period: u64, price_feed_id: PriceFeedId) -> Price {
    let price = ema_price_unsafe(price_feed_id);
    let current_time = timestamp();
    let time_difference = if current_time > price.publish_time {
        current_time - price.publish_time
    } else {
        0
    };
    require(time_difference <= time_period, PythError::OutdatedPrice);
    price
}
```

---

### Proof of Concept

1. Submit a valid Wormhole VAA carrying an EMA price update where `publish_time = current_fuel_timestamp + 1` (one second in the future). This is accepted by `update_price_feeds` since there is no upper-bound check on `publish_time`.
2. Call any public function that internally invokes `ema_price_no_older_than` (e.g., `ema_price` or `parse_price_feed_updates` with EMA).
3. The Sway VM evaluates `current_time - price.publish_time` where `price.publish_time > current_time`, triggering an unsigned integer underflow revert.
4. The call reverts with a panic instead of the expected `PythError::OutdatedPrice`, and continues to revert for every subsequent call until `current_time` advances past `publish_time`. [3](#0-2)
