### Title
`PythAggregatorV3.latestAnswer()` and `latestRoundData()` Use `getPriceUnsafe()`, Silently Returning Arbitrarily Stale Prices - (File: `target_chains/ethereum/sdk/solidity/PythAggregatorV3.sol`)

---

### Summary

`PythAggregatorV3.sol` is Pyth's production Chainlink AggregatorV3 drop-in adapter. Every price-returning function in the contract — `latestAnswer()`, `latestRoundData()`, `getRoundData()`, and `decimals()` — calls `pyth.getPriceUnsafe()` internally. `getPriceUnsafe()` performs no staleness check and never reverts on stale data; it only reverts if the feed has never been initialized (`publishTime == 0`). As a result, any protocol that integrates `PythAggregatorV3` as a Chainlink oracle receives silently stale prices with no on-chain protection.

---

### Finding Description

In `PythAggregatorV3.sol`, all four price-reading functions delegate to `getPriceUnsafe`:

```solidity
// latestAnswer — line 53-56
function latestAnswer() public view virtual returns (int256) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    return int256(price.price);
}

// latestRoundData — line 110
PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
``` [1](#0-0) [2](#0-1) 

The underlying `getPriceUnsafe()` implementation in `Pyth.sol` only reverts when `publishTime == 0` (feed never initialized). It does **not** revert for stale prices:

```solidity
function getPriceUnsafe(bytes32 id) public view override returns (PythStructs.Price memory price) {
    PythInternalStructs.PriceInfo storage info = _state.latestPriceInfo[id];
    price.publishTime = info.publishTime;
    ...
    if (price.publishTime == 0) revert PythErrors.PriceFeedNotFound();
    // No staleness check — returns whatever age the stored price is
}
``` [3](#0-2) 

The safe alternative, `getPriceNoOlderThan()`, exists in `AbstractPyth.sol` and explicitly reverts with `StalePrice` when `block.timestamp - publishTime > age`: [4](#0-3) 

`PythAggregatorV3` never calls it. Critically, even `latestRoundData()` — the function Chainlink integrators are told to use for staleness-aware consumption — returns `updatedAt = price.publishTime` sourced from `getPriceUnsafe()`, so the returned timestamp is the stale feed timestamp, not a freshness guarantee: [5](#0-4) 

---

### Impact Explanation

Any protocol that deploys `PythAggregatorV3` as a Chainlink oracle adapter and calls `latestAnswer()` or `latestRoundData()` without independently verifying `updatedAt` against `block.timestamp` will silently consume arbitrarily stale prices. Concrete consequences include:

- **Incorrect collateral/liquidation valuations** in lending protocols (under- or over-collateralized positions accepted)
- **Mispriced swaps or derivatives** in DEX/perp protocols
- **Free or discounted transactions** in paymaster-style contracts (direct analog to the Boba bug)

The `latestRoundData()` path is particularly dangerous because Chainlink best-practice guides instruct integrators to use `latestRoundData()` and check `updatedAt` — but the `updatedAt` value returned here is the stale feed's own `publishTime`, not a live freshness signal, giving a false sense of safety.

---

### Likelihood Explanation

Pyth operates as a pull oracle; on-chain prices are only as fresh as the last `updatePriceFeeds()` call. During any period of network congestion, Hermes downtime, or simply when no one has pushed an update, the stored price ages freely. An attacker monitoring on-chain `publishTime` values can identify when a feed is stale and then interact with any protocol using `PythAggregatorV3` to exploit the mispricing. No privileged access is required — any transaction sender can trigger the vulnerable path.

---

### Recommendation

Replace `getPriceUnsafe()` with `getPriceNoOlderThan()` in `latestAnswer()` and `latestRoundData()`, and add a configurable `maxAge` parameter to the contract:

```solidity
uint256 public maxAge; // set in constructor, e.g. 60 seconds

function latestAnswer() public view virtual returns (int256) {
    PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
    return int256(price.price);
}

function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
    ...
}
```

This ensures both functions revert with `StalePrice` when the feed has not been updated within the acceptable window, matching the safety guarantee Chainlink integrators expect.

---

### Proof of Concept

1. Deploy `PythAggregatorV3` pointing to a live Pyth contract and a valid `priceId`.
2. Do **not** call `updateFeeds()` for longer than the feed's normal update cadence (e.g., 60+ seconds).
3. Call `latestAnswer()` — returns the last stored price with no revert, regardless of age.
4. Call `latestRoundData()` — returns `updatedAt = price.publishTime` (the stale timestamp). A naive integrator checking `block.timestamp - updatedAt < threshold` will see the staleness, but the function itself does not enforce it, and many integrators omit this check when using a "trusted" adapter.
5. A protocol using this adapter for collateral pricing will accept the stale price as current, enabling an attacker to borrow against over-valued or under-valued collateral.

        bytes32 id,
