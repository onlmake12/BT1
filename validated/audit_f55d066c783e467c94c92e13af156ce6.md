### Title
Governance Message Suppression via Out-of-Order VAA Delivery — (`lazer/contracts/sui/sources/governance.move`)

---

### Summary

The `process_incoming` function in the Sui Lazer governance module uses a `seen_sequence < sequence` guard (strictly less than) rather than enforcing strict sequential delivery (`seen_sequence + 1 == sequence`). Because VAA relay is permissionless and Wormhole VAAs are publicly fetchable once guardian-signed, any unprivileged actor can permanently suppress a lower-sequence governance message (e.g., a signer rotation at seq=5) by front-running the official relayer with a higher-sequence VAA (e.g., seq=6). After seq=6 is accepted, seq=5 is permanently rejected with `EOldSequenceNumber` and can never be applied.

---

### Finding Description

The guard in `process_incoming` is:

```move
assert!(self.seen_sequence < sequence, EOldSequenceNumber);
self.seen_sequence = sequence;
``` [1](#0-0) 

This accepts any VAA whose sequence number is strictly greater than `seen_sequence`, with no requirement that it be the *next* sequence. Once a VAA with sequence N is accepted, `seen_sequence` is set to N, and all VAAs with sequence ≤ N are permanently rejected.

The relay entry points `actions::upgrade` and `actions::update_trusted_signer` are both `public fun` — callable by any account on Sui: [2](#0-1) [3](#0-2) 

Both call `state::unwrap_ptgm`, which calls `governance::process_incoming`: [4](#0-3) 

Wormhole VAAs are publicly available on the guardian network once signed. The official relay infrastructure (`executeGovernanceProposals`) reads `seen_sequence` from chain state and iterates forward from there: [5](#0-4) 

If a malicious actor front-runs the official relayer by submitting seq=6 before seq=5, the official relayer will subsequently read `seen_sequence=6` and begin iterating from 6, permanently skipping seq=5.

---

### Impact Explanation

A governance-approved signer rotation (seq=5) is permanently and silently skipped. The contract's trusted signer set deviates from the governance-approved state. If the skipped VAA was a signer rotation to revoke a compromised key, the compromised key remains trusted indefinitely. If it was an addition of a new signer, price updates from that signer are rejected. The state divergence is irreversible without issuing a new governance VAA.

---

### Likelihood Explanation

- The relay is permissionless: any Sui account can call `actions::update_trusted_signer` or `actions::upgrade` with any valid VAA.
- Wormhole VAAs are public: once guardian-signed, any actor can fetch seq=6 from the Wormhole guardian API and submit it before the official relayer processes seq=5.
- The attack requires only two pending governance VAAs to exist simultaneously — a realistic condition during any governance activity window (e.g., a signer rotation followed by a fee update).
- No privileged access, leaked keys, or oracle manipulation is required.

---

### Recommendation

Change the sequence check in `process_incoming` from a "greater than" check to a strict "exactly next" check:

```move
// Before (vulnerable):
assert!(self.seen_sequence < sequence, EOldSequenceNumber);

// After (fixed):
assert!(self.seen_sequence + 1 == sequence, EOldSequenceNumber);
``` [1](#0-0) 

This enforces that governance messages are always applied in the exact order they were issued, making it impossible to skip any message.

---

### Proof of Concept

1. Governance issues VAA seq=5: `UpdateTrustedSigner` (rotate to new key).
2. Governance issues VAA seq=6: `UpdateTrustedSigner` (unrelated expiry update).
3. Both VAAs are guardian-signed and publicly fetchable.
4. Attacker calls `actions::update_trusted_signer(state, vaa_seq6)` — succeeds, `seen_sequence` → 6.
5. Official relayer calls `actions::update_trusted_signer(state, vaa_seq5)` — aborts with `EOldSequenceNumber` (`5 < 6` is false).
6. `seen_sequence` remains 6; the official relayer's next iteration starts at 6, permanently skipping seq=5.
7. The signer rotation from seq=5 is never applied; the old (possibly compromised) key remains trusted.
