### Title
Fee Charged Unconditionally for Stale (No-Op) Price Updates with No Recovery Path — (`target_chains/sui/contracts/sources/pyth.move`)

---

### Summary

In `update_single_price_feed`, the caller's SUI fee is deposited into `PriceInfoObject` **before** the freshness check is performed. If the submitted price update is stale (`publish_time ≤ cached_timestamp`), the price state is unchanged but the fee is permanently locked inside the `PriceInfoObject`. No withdrawal function exists anywhere in the Sui Pyth contracts, making the loss irrecoverable.

---

### Finding Description

In `target_chains/sui/contracts/sources/pyth.move`, `update_single_price_feed` unconditionally calls `price_info::deposit_fee_coins` at line 277 before delegating to `update_cache`, which only writes the new price if `is_fresh_update` returns true: [1](#0-0) 

`update_cache` then conditionally applies the update: [2](#0-1) 

`is_fresh_update` returns false whenever `update_timestamp <= cached_timestamp`: [3](#0-2) 

The fee is stored as a `Coin<SUI>` dynamic object field inside `PriceInfoObject`: [4](#0-3) 

A search across all production Sui contract sources confirms there is **no** `withdraw_fee_coins` or equivalent function — `get_balance` is read-only and `deposit_fee_coins` is the only mutating fee function: [5](#0-4) 

---

### Impact Explanation

Any caller of `update_single_price_feed` who submits a price update whose `publish_time` is not strictly greater than the cached timestamp pays the `base_update_fee` in SUI and receives nothing in return. The paid SUI is permanently locked inside the `PriceInfoObject` with no user-accessible withdrawal path. This violates the invariant that fees must only be collected for state-changing updates.

---

### Likelihood Explanation

This is reachable by any unprivileged actor through the normal public update path. Realistic triggers include:

1. **Race condition between relayers**: Two relayers submit updates for the same feed; the second one's update is stale after the first lands, but the fee is still charged.
2. **Deliberate replay**: Any actor can re-submit a previously valid (now stale) Wormhole-verified VAA and lose their fee.
3. **Front-running**: A competing relayer submits a fresher update, making the original caller's update stale.

No privileged access, leaked keys, or external oracle misbehavior is required.

---

### Recommendation

Move `deposit_fee_coins` to **after** the freshness check, or refund the fee when `is_fresh_update` returns false. Concretely, restructure `update_single_price_feed` so that the fee is only consumed when `update_cache` actually writes a new price. Alternatively, add a `withdraw_fee_coins` function gated by a governance capability so that at minimum the protocol can recover and redistribute locked fees.

---

### Proof of Concept

```move
// 1. First call: fresh update, timestamp T. Fee charged, price updated.
vec = update_single_price_feed(&pyth_state, vec, &mut pio, coin::split(&mut coins, FEE, ctx), &clock);
assert!(price_info::get_balance(&pio) == FEE, 0); // fee deposited, price updated

// 2. Second call: same VAA (timestamp T == cached T). is_fresh_update returns false.
vec = update_single_price_feed(&pyth_state, vec, &mut pio, coin::split(&mut coins, FEE, ctx), &clock);
assert!(price_info::get_balance(&pio) == 2 * FEE, 0); // fee charged AGAIN despite no state change

// 3. No withdrawal function exists — balance is permanently locked.
// grep across target_chains/sui/contracts/sources/*.move for "withdraw" returns zero matches.
```

The second call deposits another `FEE` into `PriceInfoObject` even though `is_fresh_update` returns false and the price is unchanged, confirming the invariant is broken and the funds are unrecoverable.

    fun is_fresh_update(update: &PriceInfo, price_info_object: &PriceInfoObject): bool {
        // Get the timestamp of the update's current price
        let price_feed = price_info::get_price_feed(update);
        let update_timestamp = price::get_timestamp(&price_feed::get_price(price_feed));

        // Get the timestamp of the cached data for the price identifier
        let cached_price_info = price_info::get_price_info_from_price_info_object(price_info_object);
        let cached_price_feed =  price_info::get_price_feed(&cached_price_info);
        let cached_timestamp = price::get_timestamp(&price_feed::get_price(cached_price_feed));

        update_timestamp > cached_timestamp
    }

    public fun get_balance(price_info_object: &PriceInfoObject): u64 {
        if (!dynamic_object_field::exists_with_type<vector<u8>, Coin<SUI>>(&price_info_object.id, FEE_STORAGE_KEY)) {
            return 0
        };
        let fee = dynamic_object_field::borrow<vector<u8>, Coin<SUI>>(&price_info_object.id, FEE_STORAGE_KEY);
        coin::value(fee)
    }

    public fun deposit_fee_coins(price_info_object: &mut PriceInfoObject, fee_coins: Coin<SUI>) {
        if (!dynamic_object_field::exists_with_type<vector<u8>, Coin<SUI>>(&price_info_object.id, FEE_STORAGE_KEY)) {
            dynamic_object_field::add(&mut price_info_object.id, FEE_STORAGE_KEY, fee_coins);
        }
        else {
            let current_fee = dynamic_object_field::borrow_mut<vector<u8>, Coin<SUI>>(
                &mut price_info_object.id,
                FEE_STORAGE_KEY
            );
            coin::join(current_fee, fee_coins);
        };
    }
