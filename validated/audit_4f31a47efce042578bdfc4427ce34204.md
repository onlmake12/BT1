### Title
`update_price_feeds_if_necessary` Missing `#[payable]` Prevents Fee Payment — (`File: target_chains/stylus/contracts/pyth-receiver/src/lib.rs`)

---

### Summary

The Stylus (Arbitrum Rust) Pyth receiver contract exposes `update_price_feeds_if_necessary` as a public function without the `#[payable]` attribute. Because the function internally calls `update_price_feeds`, which requires `msg.value >= total_fee`, the function can never succeed when the on-chain fee is non-zero: any attempt to attach ETH is rejected by the EVM before the function body executes, and any call without ETH fails the fee check inside `update_price_feeds`.

---

### Finding Description

In `target_chains/stylus/contracts/pyth-receiver/src/lib.rs`, `update_price_feeds` is correctly marked `#[payable]` and enforces a fee: [1](#0-0) 

`update_price_feeds_if_necessary`, however, is declared without `#[payable]`: [2](#0-1) 

At line 250 it calls `self.update_price_feeds(update_data.clone())?`, which reads `self.vm().msg_value()` and returns `InsufficientFee` if the value is below the computed fee. Because `update_price_feeds_if_necessary` is not `#[payable]`, the EVM rejects any transaction that includes ETH before the function body is entered. A call without ETH passes the EVM gate but then fails the fee check inside `update_price_feeds`. Either path results in a revert.

A grep across all Stylus Rust files confirms only a single `#[payable]` annotation exists in the entire contract — on `update_price_feeds` alone: [3](#0-2) 

By contrast, every other Pyth target chain implementation correctly marks the equivalent function as payable. The Fuel interface explicitly annotates it: [4](#0-3) 

The Solidity `IPyth` interface also declares it `payable`: [5](#0-4) 

The Stylus implementation is the only one that omits the attribute.

---

### Impact Explanation

`update_price_feeds_if_necessary` is permanently non-functional on any Stylus deployment where `single_update_fee_in_wei > 0` or `transaction_fee_in_wei > 0`. On all production chains the fee is non-zero (the default is 1 wei per update). Any protocol or user that calls this function — the standard gas-saving variant of `update_price_feeds` — will receive a revert on every invocation. Price feeds cannot be updated through this entry point, breaking any downstream consumer that relies on it.

---

### Likelihood Explanation

The function is part of the public Pyth interface and is the recommended gas-efficient update path. Any unprivileged caller (price updater, keeper, or integrating contract) who follows the standard Pyth integration pattern and calls `update_price_feeds_if_necessary` with the required fee will trigger the revert. The fee is non-zero on every production deployment, so the failure is deterministic and immediate.

---

### Recommendation

Add `#[payable]` to `update_price_feeds_if_necessary`:

```rust
#[payable]
pub fn update_price_feeds_if_necessary(
    &mut self,
    update_data: Vec<Vec<u8>>,
    price_ids: Vec<[u8; 32]>,
    publish_times: Vec<u64>,
) -> Result<(), PythReceiverError> {
```

This mirrors the pattern already used for `update_price_feeds` and matches the Fuel and Solidity implementations.

---

### Proof of Concept

1. Deploy the Stylus `PythReceiver` with `single_update_fee_in_wei = 1` (the production default).
2. Call `update_price_feeds_if_necessary` with valid `update_data`, `price_ids`, and `publish_times` that require an update, attaching `msg.value = 1 wei`.
3. **Result**: The EVM rejects the call before the function body executes because the function is not `#[payable]`. Transaction reverts.
4. Retry the same call with `msg.value = 0`.
5. **Result**: The function body executes, reaches `self.update_price_feeds(...)`, which reads `msg.value == 0`, computes `total_fee == 1`, and returns `InsufficientFee`. Transaction reverts.
6. For comparison, call `update_price_feeds` directly with `msg.value = 1 wei` — this succeeds, confirming the fee logic itself is correct and only the missing `#[payable]` on `update_price_feeds_if_necessary` is the root cause. [2](#0-1)

    pub fn update_price_feeds_if_necessary(
        &mut self,
        update_data: Vec<Vec<u8>>,
        price_ids: Vec<[u8; 32]>,
        publish_times: Vec<u64>,
    ) -> Result<(), PythReceiverError> {
        if (price_ids.len() != publish_times.len())
            || (price_ids.is_empty() && publish_times.is_empty())
        {
            return Err(PythReceiverError::InvalidUpdateData);
        }

        for i in 0..price_ids.len() {
            if self.latest_price_info_publish_time(price_ids[i]) < publish_times[i] {
                self.update_price_feeds(update_data.clone())?;
                return Ok(());
            }
        }

        return Err(PythReceiverError::NoFreshUpdate);
    }
