### Title
Publisher Self-Stake Rewards Permanently Unclaimable When No Stake Account Registered — (`governance/pyth_staking_sdk/src/pyth-staking-client.ts`)

---

### Summary

The `advance_delegation_record` instruction in the OIS integrity pool accepts `publisherStakeAccountPositions` and `publisherStakeAccountCustody` as **optional** accounts. When a publisher has self-delegated tokens tracked in `PoolData.selfDelState` but has not registered a stake account in `PoolData.publisherStakeAccounts`, the SDK passes `null` for these accounts. The on-chain program cannot distribute publisher self-stake rewards in this state, creating a permanent accounting discrepancy: the pool data shows self-delegation exists and rewards should accrue, but the reward distribution path is silently blocked.

---

### Finding Description

In `getAdvanceDelegationRecordInstructions`, the `stakeAccount` field returned by `extractPublisherData` is used to populate `publisherStakeAccountPositions` and `publisherStakeAccountCustody`:

```typescript
publisherStakeAccountCustody: stakeAccount
  ? getStakeAccountCustodyAddress(stakeAccount)
  : null,
publisherStakeAccountPositions: stakeAccount,
``` [1](#0-0) 

`stakeAccount` is derived from `PoolData.publisherStakeAccounts[index]`, and is set to `null` when that entry equals `PublicKey.default` (i.e., the publisher has not called `setPublisherStakeAccount`):

```typescript
stakeAccount:
  poolData.publisherStakeAccounts[index] === undefined ||
  poolData.publisherStakeAccounts[index].equals(PublicKey.default)
    ? null
    : poolData.publisherStakeAccounts[index],
``` [2](#0-1) 

The IDL confirms both accounts are `optional`:

```json
{ "name": "publisher_stake_account_positions", "optional": true },
{ "name": "publisher_stake_account_custody", "optional": true, "writable": true }
``` [3](#0-2) 

Meanwhile, `selfDelState` in `PoolData` independently tracks publisher self-delegation and is populated regardless of whether a stake account is registered:

```typescript
selfDelegation: poolData.selfDelState[index]?.totalDelegation ?? 0n,
``` [4](#0-3) 

A publisher can therefore have a non-zero `selfDelState.totalDelegation` (self-staked tokens contributing to pool rewards) while `publisherStakeAccounts[index]` remains `PublicKey.default`. In this state, every call to `advance_delegation_record` silently skips publisher self-stake reward distribution because the required accounts are null. The publisher's self-stake rewards are permanently unclaimable.

The analog to the Sake bug is direct:
- **Sake**: `if (MasterChef != address(0))` — unstake/liquidate only runs if the user was already staked; documentation claimed auto-staking.
- **Pyth**: `stakeAccount ? getStakeAccountCustodyAddress(stakeAccount) : null` — publisher self-stake rewards are only distributed if a stake account is registered; the protocol tracks self-delegation unconditionally in `selfDelState`, implying rewards should accrue.

The publisher FAQ states:

> "By staking tokens to your pool as a publisher, you can become eligible for on-chain rewards for your published data." [5](#0-4) 

This implies staking to the pool is sufficient. The additional requirement to call `setPublisherStakeAccount` before rewards can be distributed is not documented, creating the same documentation/code inconsistency as the Sake report.

---

### Impact Explanation

When reward rate `y > 0` (currently `y = 0` per OP-PIP-103, but subject to governance change), any publisher who has self-delegated tokens but has not registered their stake account via `setPublisherStakeAccount` will permanently lose all publisher self-stake rewards. The `selfDelState` accounting in `PoolData` will show self-delegation contributing to pool capacity and reward calculations, but the actual token transfer to the publisher's custody will never occur. These rewards are not redirected elsewhere — they are simply not distributed, remaining locked in `poolRewardCustody`. [6](#0-5) 

---

### Likelihood Explanation

The `setPublisherStakeAccount` instruction is a separate, optional step that publishers must explicitly call. The protocol does not enforce that a publisher registers a stake account before self-delegating. Any publisher who self-delegates without completing this registration step is silently affected. The condition is reachable by any publisher operating the staking UI or SDK without following the full registration flow. [7](#0-6) 

---

### Recommendation

1. **Enforce invariant on-chain**: Require that `publisherStakeAccountPositions` is non-null when `selfDelState.totalDelegation > 0` for the publisher, or block self-delegation until a stake account is registered.
2. **Alternatively, accumulate unclaimed publisher rewards**: When `publisherStakeAccountPositions` is null, accumulate publisher self-stake rewards in a separate claimable buffer keyed to the publisher pubkey, redeemable once a stake account is registered.
3. **Update documentation**: Clearly state that publishers must call `setPublisherStakeAccount` before self-stake rewards can be distributed, consistent with the actual code behavior.

---

### Proof of Concept

1. Publisher self-delegates to their own pool via `delegate` instruction — `selfDelState[index].totalDelegation` becomes non-zero in `PoolData`.
2. Publisher does **not** call `setPublisherStakeAccount` — `publisherStakeAccounts[index]` remains `PublicKey.default`.
3. `extractPublisherData` returns `stakeAccount: null` for this publisher.
4. Any call to `getAdvanceDelegationRecordInstructions` (including the user-facing `claim` flow) passes `publisherStakeAccountCustody: null` and `publisherStakeAccountPositions: null`.
5. The on-chain `advance_delegation_record` instruction, receiving null optional accounts, skips publisher self-stake reward distribution.
6. Publisher self-stake rewards are permanently unclaimable; `selfDelState` continues to show non-zero self-delegation, but no tokens are ever transferred to the publisher's custody. [8](#0-7) [9](#0-8)

  async getAdvanceDelegationRecordInstructions(
    stakeAccountPositions: PublicKey,
    payer?: PublicKey,
  ) {
    const poolData = await this.getPoolDataAccount();
    const stakeAccountPositionsData = await this.getStakeAccountPositions(
      stakeAccountPositions,
    );
    const allPublishers = extractPublisherData(poolData);
    const publishers = allPublishers
      .map((publisher) => {
        const positionsWithPublisher =
          stakeAccountPositionsData.data.positions.filter(
            ({ targetWithParameters }) =>
              targetWithParameters.integrityPool?.publisher.equals(
                publisher.pubkey,
              ),
          );

        let lowestEpoch;
        for (const position of positionsWithPublisher) {
          lowestEpoch = bigintMin(position.activationEpoch, lowestEpoch);
        }

        return {
          ...publisher,
          lowestEpoch,
        };
      })
      .filter(({ lowestEpoch }) => lowestEpoch !== undefined);

    const delegationRecords = await Promise.all(
      publishers.map(({ pubkey }) =>
        this.getDelegationRecord(stakeAccountPositions, pubkey),
      ),
    );

    let lowestEpoch: bigint | undefined;
    for (const [index, publisher] of publishers.entries()) {
      const maximum = bigintMax(
        publisher.lowestEpoch,
        delegationRecords[index]?.lastEpoch,
      );
      lowestEpoch = bigintMin(lowestEpoch, maximum);
    }

    const currentEpoch = await getCurrentEpoch(this.connection);

    // Filter out delegationRecord that are up to date
    const filteredPublishers = publishers.filter((_, index) => {
      return !(delegationRecords[index]?.lastEpoch === currentEpoch);
    });

    // anchor does not calculate the correct pda for other programs
    // therefore we need to manually calculate the pdas
    const advanceDelegationRecordInstructions = await Promise.all(
      filteredPublishers.map(({ pubkey, stakeAccount }) =>
        this.integrityPoolProgram.methods
          .advanceDelegationRecord()
          .accountsPartial({
            payer: payer ?? this.wallet.publicKey,
            publisher: pubkey,
            publisherStakeAccountCustody: stakeAccount
              ? getStakeAccountCustodyAddress(stakeAccount)
              : null, // eslint-disable-line unicorn/no-null
            publisherStakeAccountPositions: stakeAccount,
            stakeAccountCustody: getStakeAccountCustodyAddress(
              stakeAccountPositions,
            ),
            stakeAccountPositions,
          })
          .instruction(),
      ),
    );
```
