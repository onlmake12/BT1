### Title
Use of `.transfer()` for Native Token Refund Causes DoS for Smart Contract Callers - (`lazer/contracts/evm/src/PythLazer.sol`)

### Summary
`PythLazer.verifyUpdate()` refunds excess ETH to `msg.sender` using `.transfer()`, which enforces a 2300 gas stipend. Any smart contract caller whose `receive()` or `fallback()` function consumes more than 2300 gas will have the entire `verifyUpdate()` call revert, making the function permanently unusable for that caller.

### Finding Description
In `PythLazer.verifyUpdate()`, when a caller sends more ETH than the `verification_fee`, the excess is refunded via:

```solidity
payable(msg.sender).transfer(msg.value - verification_fee);
``` [1](#0-0) 

The Solidity `.transfer()` primitive forwards exactly 2300 gas to the recipient. This is insufficient for any smart contract whose `receive()` or `fallback()` function performs non-trivial logic (e.g., emitting events, updating state, or interacting with other contracts). When the refund fails, the entire transaction reverts, including the price verification itself.

### Impact Explanation
Any smart contract integrator of `PythLazer` that:
1. Calls `verifyUpdate()` with `msg.value > verification_fee` (e.g., to avoid computing the exact fee off-chain), **and**
2. Has a `receive()` or `fallback()` function consuming more than 2300 gas

...will have every call to `verifyUpdate()` revert. This is a complete denial of service for that caller's price verification flow. The impact is amplified by the fact that `verification_fee` is a mutable storage variable (`verification_fee = 1 wei` initially but changeable), meaning callers may routinely overpay to be safe. [2](#0-1) [3](#0-2) 

### Likelihood Explanation
Smart contract integrators of Pyth Lazer are the primary intended consumers of `verifyUpdate()`. It is common for DeFi protocols (the target audience) to have non-trivial `receive()` functions. Overpaying fees is a standard defensive pattern in on-chain integrations. The combination makes this a realistic and reachable failure mode for any unprivileged Lazer user/integrator.

### Recommendation
Replace `.transfer()` with a low-level `.call()` and check the return value:

```solidity
- payable(msg.sender).transfer(msg.value - verification_fee);
+ (bool success, ) = payable(msg.sender).call{value: msg.value - verification_fee}("");
+ require(success, "Refund failed");
``` [4](#0-3) 

### Proof of Concept
1. Deploy a smart contract `Attacker` with a `receive()` function that emits an event (consuming >2300 gas).
2. From `Attacker`, call `PythLazer.verifyUpdate{value: 2 wei}(validUpdate)` where `verification_fee == 1 wei`.
3. The contract attempts `payable(Attacker).transfer(1 wei)`.
4. The 2300 gas stipend is exceeded by `Attacker`'s `receive()`.
5. The `.transfer()` reverts, rolling back the entire `verifyUpdate()` call.
6. `Attacker` can never successfully call `verifyUpdate()` while overpaying, making Lazer price verification inaccessible to it. [5](#0-4)

    function verifyUpdate(
        bytes calldata update
    ) external payable returns (bytes calldata payload, address signer) {
        // Require fee and refund excess
        require(msg.value >= verification_fee, "Insufficient fee provided");
        if (msg.value > verification_fee) {
            payable(msg.sender).transfer(msg.value - verification_fee);
        }

        if (update.length < 71) {
            revert("input too short");
        }
        uint32 EVM_FORMAT_MAGIC = 706910618;

        uint32 evm_magic = uint32(bytes4(update[0:4]));
        if (evm_magic != EVM_FORMAT_MAGIC) {
            revert("invalid evm magic");
        }
        uint16 payload_len = uint16(bytes2(update[69:71]));
        if (update.length < 71 + payload_len) {
            revert("input too short");
        }
        payload = update[71:71 + payload_len];
        bytes32 hash = keccak256(payload);
        (signer, , ) = ECDSA.tryRecover(
            hash,
            uint8(update[68]) + 27,
            bytes32(update[4:36]),
            bytes32(update[36:68])
        );
        if (signer == address(0)) {
            revert("invalid signature");
        }
        if (!isValidSigner(signer)) {
            revert("invalid signer");
        }
    }
