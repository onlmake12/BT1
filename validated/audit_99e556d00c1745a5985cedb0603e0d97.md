### Title
Attacker-Controlled Peer ID Injected into Victim's Observed Addresses via Identify Protocol — (`network/src/protocols/identify/mod.rs`)

---

### Summary

The `add_observed_addr` callback in the Identify protocol only appends the local peer ID to an `observed_addr` if no `/p2p/` component is present. It performs **no validation** when a `/p2p/` component already exists. An unprivileged remote peer can send a crafted `observed_addr` containing a foreign peer ID, which the victim node stores and subsequently broadcasts to all peers as its own reachable address.

---

### Finding Description

The guard in `IdentifyCallback::add_observed_addr` is:

```rust
fn add_observed_addr(&mut self, mut addr: Multiaddr, session_id: SessionId) -> MisbehaveResult {
    if extract_peer_id(&addr).is_none() {
        addr.push(Protocol::P2P(Cow::Borrowed(
            self.network_state.local_peer_id().as_bytes(),
        )))
    }
    self.network_state.add_observed_addr(session_id, addr);
    MisbehaveResult::Continue
}
``` [1](#0-0) 

The intent is: "if the address has no peer ID, append ours." But the branch is gated only on **presence**, not **correctness**. If the attacker sends `/ip4/<victim_ip>/tcp/8114/p2p/<attacker_peer_id>`, `extract_peer_id` returns `Some(attacker_peer_id)`, the push is skipped, and the addr with the wrong peer ID is stored verbatim.

`NetworkState::add_observed_addr` performs no further validation:

```rust
pub(crate) fn add_observed_addr(&self, session_id: SessionId, addr: Multiaddr) {
    let mut pending_observed_addrs = self.observed_addrs.write();
    pending_observed_addrs.insert(session_id, addr);
}
``` [2](#0-1) 

These poisoned addresses are then included in `local_listen_addrs`, which is broadcast to every newly connected peer:

```rust
fn local_listen_addrs(&mut self) -> Vec<Multiaddr> {
    let mut listens = self.listen_addrs();
    if listens.len() < MAX_RETURN_LISTEN_ADDRS {
        let observe_addrs = self
            .network_state
            .observed_addrs(MAX_RETURN_LISTEN_ADDRS - listens.len());
        listens.extend(observe_addrs);
        listens
    } else {
        listens
    }
}
``` [3](#0-2) 

On every new `connected` event, the victim sends this poisoned list to the new peer:

```rust
let data = IdentifyMessage::new(listen_addrs, session.address.clone(), identify).encode();
let _ = context.quick_send_message(data).await ...;
``` [4](#0-3) 

---

### Impact Explanation

Any node that receives the victim's `listen_addrs` will store `/ip4/<victim_ip>/tcp/8114/p2p/<attacker_peer_id>` as a reachable address for the victim. When those nodes attempt to connect using that address, the Noise/TLS handshake will fail because the actual peer at that IP has a different peer ID. This effectively makes the victim unreachable to any node that received the poisoned address, causing **network routing disruption** and **identity spoofing** of the victim node's advertised identity.

---

### Likelihood Explanation

Any unprivileged peer that can establish a single valid Identify session with the victim can trigger this. No special privileges, no PoW, no key material required. The attacker only needs to be accepted as a peer (standard inbound connection) and send one crafted `IdentifyMessage`.

---

### Recommendation

In `add_observed_addr`, when `extract_peer_id` returns `Some(peer_id)`, verify that `peer_id == self.network_state.local_peer_id()`. If it does not match, strip the foreign `/p2p/` component and append the correct local peer ID (or discard the address entirely). The corrected logic:

```rust
fn add_observed_addr(&mut self, mut addr: Multiaddr, session_id: SessionId) -> MisbehaveResult {
    match extract_peer_id(&addr) {
        None => {
            addr.push(Protocol::P2P(Cow::Borrowed(
                self.network_state.local_peer_id().as_bytes(),
            )));
        }
        Some(peer_id) if peer_id != *self.network_state.local_peer_id() => {
            // Foreign peer ID injected — strip and replace, or discard
            return MisbehaveResult::Continue; // or disconnect
        }
        _ => {}
    }
    self.network_state.add_observed_addr(session_id, addr);
    MisbehaveResult::Continue
}
```
