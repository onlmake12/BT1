### Title
Inbound Peer Can Inject Port-0 Address into Peer Store and Propagate It Network-Wide via Discovery GetNodes — (`network/src/protocols/discovery/mod.rs`, `network/src/protocols/discovery/state.rs`)

### Summary

An unprivileged inbound peer can send a crafted `GetNodes{listen_port: Some(0), version: 1}` message. The discovery handler calls `update_port(0)` on the session's `RemoteAddress::Init`, which replaces the TCP port with `0` and immediately transitions the state to `RemoteAddress::Listen`. The subsequent `is_valid_addr` check only validates IP reachability, not port validity. The port-0 address is then stored in the peer store and later propagated to up to 3 other peers via the periodic announce mechanism.

### Finding Description

**Step 1 — Inbound session starts as `RemoteAddress::Init`:**

In `state.rs`, when a session is inbound, the initial state is `RemoteAddress::Init`: [1](#0-0) 

**Step 2 — `update_port(0)` replaces TCP port and transitions to `Listen`:**

`update_port` only fires when the state is `Init`. It replaces the TCP component with the attacker-supplied port (0) and immediately transitions to `RemoteAddress::Listen`: [2](#0-1) 

**Step 3 — `add_new_addr` is called with the port-0 address:**

Back in `received()`, after `update_port(port)` returns, the state is already `RemoteAddress::Listen`. The `if let RemoteAddress::Listen` guard matches and `add_new_addr` is called unconditionally: [3](#0-2) 

**Step 4 — `is_valid_addr` only checks IP, not port:**

The sole guard before writing to the peer store checks `is_reachable(socket_addr.ip())`. Port 0 with a public IP passes this check: [4](#0-3) 

**Step 5 — `peer_store.add_addr` has no port validation:**

`add_addr` only checks the ban list; there is no port-validity check: [5](#0-4) 

**Step 6 — Port-0 address propagates to up to 3 peers via `check_timer` → `announce_list`:**

`check_timer` returns the `Listen` address (now port 0). `is_valid_addr` passes again. The address is pushed into `announce_list` and queued in `announce_multiaddrs` for up to 3 random peers: [6](#0-5) 

Note: `change_to_listen()` at line 138 is a no-op here because `update_port` already transitioned the state to `Listen`. The vulnerability is fully triggered by the `listen_port: Some(0)` branch alone. [7](#0-6) 

Contrast with `SeedRecord::check()`, which explicitly rejects port 0 — no equivalent guard exists in the discovery path: [8](#0-7) 

### Impact Explanation

- The victim node's peer store is polluted with `/ip4/X.X.X.X/tcp/0/p2p/...` entries.
- These entries are returned by `fetch_addrs_to_feeler`, causing the node to waste connection attempts on port 0 (which always fail).
- The port-0 address is re-announced to up to 3 other peers every `ANNOUNCE_INTERVAL` (24 hours), spreading the pollution network-wide.
- An attacker controlling many inbound connections can fill the peer store (`ADDR_COUNT_LIMIT`) with unreachable addresses, degrading the victim's ability to discover legitimate peers.

### Likelihood Explanation

Any node that accepts inbound connections is vulnerable. No authentication, no PoW, no special role is required. The attacker only needs to establish a TCP connection and send a single crafted discovery message.

### Recommendation

Add a port-validity check in `update_port` and/or in `is_valid_addr`:

```rust
// In update_port or is_valid_addr:
if port == 0 {
    return; // or return false
}
```

Alternatively, add a port check in `add_new_addrs` before calling `peer_store.add_addr`, mirroring the existing check in `SeedRecord::check`.

### Proof of Concept

1. Connect to a CKB node as an inbound peer.
2. Send a discovery `GetNodes` message with `listen_port = Some(0)` and `version = 1`.
3. Wait for the next `notify` tick (default 60 seconds).
4. Inspect the peer store: the entry `/ip4/<your_ip>/tcp/0/p2p/<your_peer_id>` will be present.
5. Observe outbound `Nodes{announce: true}` messages sent to other peers containing the port-0 address.

    pub(crate) fn update_port(&mut self, port: u16) {
        if let RemoteAddress::Init(addr) = self {
            let addr = addr
                .into_iter()
                .map(|proto| {
                    match proto {
                        // TODO: other transport, UDP for example
                        Protocol::Tcp(_) => Protocol::Tcp(port),
                        value => value,
                    }
                })
                .collect();
            *self = RemoteAddress::Listen(addr);
        }
    }
