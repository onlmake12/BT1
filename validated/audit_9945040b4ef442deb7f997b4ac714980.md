### Title
Echo `getFee()` Uses Static Per-Gas Rate Decoupled from Actual Gas Prices, Enabling Permanent User Fund Lockup - (File: target_chains/ethereum/contracts/contracts/echo/Echo.sol)

### Summary

The `Echo` contract's `getFee()` function computes the required fee using a static `feePerGasInWei` rate configured by the provider at registration time, not the actual `tx.gasprice` at execution time. If gas prices spike between when a user submits `requestPriceUpdatesWithCallback()` and when the provider calls `executeCallback()`, the locked fee becomes insufficient to cover the provider's real gas cost. Because there is no user-facing refund path in the contract, the user's funds are permanently locked.

---

### Finding Description

`Echo.getFee()` computes the total fee as:

```
feeAmount = pythFeeInWei + baseFeeInWei + (priceIds.length * feePerFeedInWei) + (callbackGasLimit * feePerGasInWei)
``` [1](#0-0) 

`feePerGasInWei` is a static value stored in `ProviderInfo` and set once at `registerProvider()` or updated via `setProviderFee()`. [2](#0-1) 

At request time, the user pays `msg.value >= getFee(...)` and the contract stores `req.fee = msg.value - pythFeeInWei`. [3](#0-2) 

The code itself contains an explicit acknowledgment that the fee is **not** tied to `tx.gasprice`:

```solidity
// FIXME: this comment is wrong. (we're not using tx.gasprice)
// NOTE: The 60-second future limit on publishTime prevents a DoS vector where
//      attackers could submit many low-fee requests for far-future updates when gas prices
//      are low, forcing executors to fulfill them later when gas prices might be much higher.
//      Since tx.gasprice is used to calculate fees, allowing far-future requests would make
//      the fee estimation unreliable.
``` [4](#0-3) 

When `executeCallback()` runs, the provider receives `req.fee + msg.value - pythFee`. If the actual gas cost at execution time exceeds `req.fee`, the provider has no economic incentive to execute. [5](#0-4) 

There is no user-facing withdrawal or refund function. The only withdrawal paths are `withdrawFees()` (admin-only) and `withdrawAsFeeManager()` (fee-manager-only). [6](#0-5) [7](#0-6) 

---

### Impact Explanation

If the network gas price at execution time exceeds the provider's static `feePerGasInWei`, the provider will not call `executeCallback()` because it is unprofitable. The user's fee (`req.fee`) remains locked in the contract indefinitely with no mechanism to reclaim it. This constitutes a permanent, unrecoverable loss of user funds proportional to the fee paid.

---

### Likelihood Explanation

EVM gas prices are highly volatile. Spikes of 10–100× above baseline are common during periods of high network activity (NFT mints, liquidation cascades, etc.). A provider who sets `feePerGasInWei` at a "normal" gas price will rationally refuse to execute callbacks during spikes. The 60-second `publishTime` window means requests can be submitted at low gas prices and become uneconomical to execute within that same window. The FIXME comment in the production code confirms the developers are aware the fee is not gas-price-aware.

---

### Recommendation

1. Replace the static `feePerGasInWei` with a dynamic fee that incorporates `tx.gasprice` at request time (e.g., `callbackGasLimit * tx.gasprice * multiplier`), so the locked fee reflects realistic execution cost.
2. Add a user-facing refund function that allows the original requester to reclaim `req.fee` after a timeout if the callback has not been executed.
3. Alternatively, implement the penalty/refund mechanism already noted in the TODO comments in `executeCallback()`.

---

### Proof of Concept

1. Provider registers with `feePerGasInWei = 10 gwei`.
2. User calls `requestPriceUpdatesWithCallback()` when `tx.gasprice = 10 gwei`, paying `callbackGasLimit * 10 gwei` as the gas component of the fee.
3. Network gas price spikes to 200 gwei.
4. Provider's cost to call `executeCallback()` is `callbackGasLimit * 200 gwei`, far exceeding `req.fee`.
5. Provider does not execute. User's funds remain locked in the contract.
6. User has no function to call to recover their ETH.

    function getFee(
        address provider,
        uint32 callbackGasLimit,
        bytes32[] calldata priceIds
    ) public view override returns (uint96 feeAmount) {
        uint96 baseFee = _state.pythFeeInWei; // Fixed fee to Pyth
        // Note: The provider needs to set its fees to include the fee charged by the Pyth contract.
        // Ideally, we would be able to automatically compute the pyth fees from the priceIds, but the
        // fee computation on IPyth assumes it has the full updated data.
        uint96 providerBaseFee = _state.providers[provider].baseFeeInWei;
        uint96 providerFeedFee = SafeCast.toUint96(
            priceIds.length * _state.providers[provider].feePerFeedInWei
        );
        uint96 providerFeeInWei = _state.providers[provider].feePerGasInWei; // Provider's per-gas rate
        uint256 gasFee = callbackGasLimit * providerFeeInWei; // Total provider fee based on gas
        feeAmount =
            baseFee +
            providerBaseFee +
            providerFeedFee +
            SafeCast.toUint96(gasFee); // Total fee user needs to pay
    }
```
