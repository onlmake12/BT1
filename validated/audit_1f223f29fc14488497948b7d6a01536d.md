### Title
Unguarded `Initialize` Allows Attacker to Set Unsignable PDA as Authority, Permanently Bricking Publisher Registration — (`target_chains/solana/programs/pyth-price-store/src/processor/initialize.rs`)

---

### Summary

The `Initialize` instruction has no access control and accepts any 32-byte value as `authority` without validating that it corresponds to a signable (non-PDA) keypair. Because the config account is a singleton PDA that can only be initialized once, an attacker who calls `Initialize` first — setting `authority` to a known PDA address — permanently prevents any future `InitializePublisher` call from succeeding.

---

### Finding Description

**Entrypoint — `Initialize` has no caller restriction.**

`processor/initialize.rs` only validates that the `payer` is a signer and that the `config` PDA is writable and uninitialized. There is no check that the caller is the program deployer or any privileged account: [1](#0-0) 

**`authority` is stored as raw bytes with no keypair validation.**

`InitializeArgs.authority` is `[u8; 32]` and is written directly into the config account without any check that the bytes represent a non-PDA public key: [2](#0-1) [3](#0-2) [4](#0-3) 

**Config is a singleton — once initialized, it cannot be re-initialized.**

`accounts::config::create` checks `data.format != 0` and returns `AlreadyInitialized` if the account already has data, making the authority field immutable after the first call: [5](#0-4) 

**`validate_authority` requires `is_signer`, which is impossible for a PDA.**

`InitializePublisher` calls `validate_authority`, which checks both that the account key matches `config.authority` AND that `authority.is_signer` is true. A PDA has no private key, so no external transaction can ever set `is_signer = true` for it: [6](#0-5) 

---

### Impact Explanation

Once `Initialize` is called with a PDA as `authority`, every subsequent `InitializePublisher` call will fail with `MissingRequiredSignature` at line 71 of `validate.rs`. No publisher can ever be registered. The program's core functionality — registering publishers so they can submit prices — is permanently and irrecoverably disabled for that program deployment (assuming the program is not upgradeable or the upgrade authority is not available).

---

### Likelihood Explanation

The attack requires the attacker to call `Initialize` before the legitimate deployer. On Solana this is a front-running race, but:
- There is no atomicity guarantee between program deployment and `Initialize`; any gap is exploitable.
- The cost is only the rent for the config account (a few thousand lamports).
- The attacker needs no privileged access — any funded account suffices.
- The attack is one transaction and is irreversible.

---

### Recommendation

1. **Restrict who can call `Initialize`**: Require the payer/caller to be the program's upgrade authority (BPF upgradeable loader authority), or embed the expected authority pubkey as a program constant.
2. **Validate that `authority` is not a PDA**: Use `Pubkey::is_on_curve()` to reject any authority that lies on a PDA (off-curve) point before storing it.

---

### Proof of Concept

```rust
// 1. Derive the config PDA — this is the address we'll use as the "authority"
let (config_pda, config_bump) =
    Pubkey::find_program_address(&[CONFIG_SEED.as_bytes()], &program_id);

// 2. Attacker calls Initialize with authority = config_pda (a PDA, no private key)
let mut data = vec![Instruction::Initialize as u8, config_bump];
data.extend_from_slice(&config_pda.to_bytes()); // authority = PDA
// ... submit transaction signed by attacker's keypair (payer) ...

// 3. Legitimate operator attempts InitializePublisher — always fails
// validate_authority checks authority.is_signer, but config_pda can never sign
// => ProgramError::MissingRequiredSignature every time
```

The config account is now permanently locked with an unsignable authority. No publisher can ever be registered.

```rust
pub fn validate_authority<'a, 'b>(
    account: Option<&'b AccountInfo<'a>>,
    config: &AccountInfo<'a>,
) -> Result<&'b AccountInfo<'a>, ProgramError> {
    let authority = account.ok_or(ProgramError::NotEnoughAccountKeys)?;
    ensure!(ProgramError::MissingRequiredSignature, authority.is_signer);
    ensure!(ProgramError::InvalidArgument, authority.is_writable);
    let config_data = config.data.borrow();
    let config = accounts::config::read(*config_data)?;
    ensure!(
        ProgramError::MissingRequiredSignature,
        authority.key.to_bytes() == config.authority
    );
    Ok(authority)
```
