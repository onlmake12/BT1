### Title
Fee Collected Unconditionally Before Freshness Check, Permanently Locking SUI With No Withdrawal Path — (`target_chains/sui/contracts/sources/pyth.move` + `price_info.move`)

### Summary

`update_single_price_feed` deposits the fee into `PriceInfoObject` **before** checking whether the price update is fresh. When the update is stale, the fee is consumed and permanently locked — no withdrawal function exists anywhere in the contract. Within a single Sui PTB, the same `HotPotatoVector<PriceInfo>` can be passed through `update_single_price_feed` multiple times against the same `PriceInfoObject`, multiplying the locked amount.

### Impact Explanation

Any SUI deposited via `deposit_fee_coins` is permanently locked inside the `PriceInfoObject`'s dynamic field under `FEE_STORAGE_KEY`. Because `update_single_price_feed` returns the `HotPotatoVector` after each call, a caller can chain N calls within one PTB against the same `PriceInfoObject` using the same stale vector, depositing `N × base_update_fee` SUI while the cached price never changes. There is no on-chain path to recover those coins — no governance withdrawal, no admin drain, no `transfer` of the stored `Coin<SUI>`.

### Likelihood Explanation

- No privileged role required. Any address that can construct a valid `HotPotatoVector<PriceInfo>` (via `create_authenticated_price_infos_using_accumulator` or `create_price_infos_hot_potato`) can trigger this.
- Race conditions are common: a relayer submitting an update that was fresh at construction time may be stale by the time the PTB executes. The fee is taken regardless.
- Deliberate exploitation requires only a valid (but stale) VAA and enough SUI to pay fees — both trivially obtainable.

### Recommendation

Move `deposit_fee_coins` to **after** the freshness check, so fees are only collected when a price update is actually written:

```move
if (is_fresh_update(update, price_info_object)){
    price_info::deposit_fee_coins(price_info_object, fee);   // moved here
    price_info::update_price_info_object(price_info_object, update);
} else {
    // return fee to caller or abort
    transfer::public_transfer(fee, tx_context::sender(ctx));
}
```

Additionally, implement a privileged fee-withdrawal function so accumulated fees can be recovered by the protocol.

### Proof of Concept

Within a single Sui PTB:

1. Obtain a valid `HotPotatoVector<PriceInfo>` whose `publish_time` equals the current cached `publish_time` of `price_info_object_X` (stale by `is_fresh_update`'s strict `>` check).
2. Call `update_single_price_feed(state, vec, &mut price_info_object_X, coin_50, clock)` → fee deposited, price unchanged, `vec` returned.
3. Call `update_single_price_feed(state, vec, &mut price_info_object_X, coin_50, clock)` again → second fee deposited, price still unchanged, `vec` returned.
4. Call `hot_potato_vector::destroy(vec)`.
5. Assert `price_info::get_balance(&price_info_object_X) == 100` while `get_price_unsafe(&price_info_object_X)` is identical to its pre-call value.

The balance grows by `base_update_fee` per call with no corresponding price state change, and no mechanism exists to recover the locked SUI. [6](#0-5) [4](#0-3)

    public fun update_single_price_feed(
        pyth_state: &PythState,
        price_updates: HotPotatoVector<PriceInfo>,
        price_info_object: &mut PriceInfoObject,
        fee: Coin<SUI>,
        clock: &Clock
    ): HotPotatoVector<PriceInfo> {
        let latest_only = state::assert_latest_only(pyth_state);

        // On Sui, users get to choose which price feeds to update. They specify a single price feed to
        // update at a time. We therefore charge the base fee for each such individual update.
        // This is a departure from Eth, where users don't get to necessarily choose.
        assert!(state::get_base_update_fee(pyth_state) <= coin::value(&fee), E_INSUFFICIENT_FEE);

        // store fee coins within price info object
        price_info::deposit_fee_coins(price_info_object, fee);

        // Find price update corresponding to PriceInfoObject within the array of price_updates
        // and use it to update PriceInfoObject.
        let i = 0;
        let found = false;
        while (i < hot_potato_vector::length<PriceInfo>(&price_updates)){
            let cur_price_info = hot_potato_vector::borrow<PriceInfo>(&price_updates, i);
            if (has_same_price_identifier(cur_price_info, price_info_object)){
                found = true;
                update_cache(latest_only, cur_price_info, price_info_object, clock);
                break
            };
            i = i + 1;
        };
        if (found==false){
            abort E_PRICE_UPDATE_NOT_FOUND_FOR_PRICE_INFO_OBJECT
        };
        price_updates
    }

    fun is_fresh_update(update: &PriceInfo, price_info_object: &PriceInfoObject): bool {
        // Get the timestamp of the update's current price
        let price_feed = price_info::get_price_feed(update);
        let update_timestamp = price::get_timestamp(&price_feed::get_price(price_feed));

        // Get the timestamp of the cached data for the price identifier
        let cached_price_info = price_info::get_price_info_from_price_info_object(price_info_object);
        let cached_price_feed =  price_info::get_price_feed(&cached_price_info);
        let cached_timestamp = price::get_timestamp(&price_feed::get_price(cached_price_feed));

        update_timestamp > cached_timestamp
    }

    public fun deposit_fee_coins(price_info_object: &mut PriceInfoObject, fee_coins: Coin<SUI>) {
        if (!dynamic_object_field::exists_with_type<vector<u8>, Coin<SUI>>(&price_info_object.id, FEE_STORAGE_KEY)) {
            dynamic_object_field::add(&mut price_info_object.id, FEE_STORAGE_KEY, fee_coins);
        }
        else {
            let current_fee = dynamic_object_field::borrow_mut<vector<u8>, Coin<SUI>>(
                &mut price_info_object.id,
                FEE_STORAGE_KEY
            );
            coin::join(current_fee, fee_coins);
        };
    }
