### Title
Fee Revenue Permanently Locked in PriceInfoObjects with No On-Chain Withdrawal Path — (`target_chains/sui/contracts/sources/price_info.move`, `pyth.move`)

---

### Summary

Every call to `update_single_price_feed` deposits the caller's fee `Coin<SUI>` into the target `PriceInfoObject` via `deposit_fee_coins`. No function in the entire Sui contract suite withdraws or transfers those coins out. The `fee_recipient_address` stored in `State` is never used to route funds. The `set_fee_recipient.move` module itself contains an explicit comment confirming the fee-recipient mechanism "is not used anymore." All fee revenue paid since deployment is permanently locked inside shared `PriceInfoObject` objects.

---

### Finding Description

**Deposit path (confirmed):**

`update_single_price_feed` in `pyth.move` line 277 unconditionally calls:

```move
price_info::deposit_fee_coins(price_info_object, fee);
``` [1](#0-0) 

`deposit_fee_coins` in `price_info.move` lines 105–116 either adds a new `Coin<SUI>` dynamic object field or merges into the existing one via `coin::join`. The balance grows monotonically with every update call. [2](#0-1) 

**Withdrawal path (absent):**

A full search of `target_chains/sui/contracts/sources/**/*.move` for `withdraw`, `collect_fee`, and `transfer_fee` returns zero hits inside `price_info.move` or `pyth.move`. The only matches are in `state.move` (the `fee_recipient_address` field and its getter/setter) and `set_fee_recipient.move` (governance action to update that address).

`state.move` stores `fee_recipient_address` and exposes `get_fee_recipient()`: [3](#0-2) [4](#0-3) 

But `get_fee_recipient()` is never called anywhere in the production code to actually move coins. The address is stored and settable, but never used as a transfer destination.

**Governance module confirms the mechanism was removed:**

`set_fee_recipient.move` line 1 reads:

> "The previous version of the contract sent the fees to a recipient address but this state is not used anymore. This module is kept for backward compatibility." [5](#0-4) 

The governance action `SET_FEE_RECIPIENT` (action code 5) only updates the `fee_recipient_address` field in `State`; it does not touch any `PriceInfoObject` balance. [6](#0-5) 

---

### Impact Explanation

All SUI paid as update fees by any caller of `update_single_price_feed` accumulates inside the corresponding `PriceInfoObject` with no on-chain path to recover it. The protocol earns fee revenue that is permanently inaccessible. This constitutes permanent freezing of user-paid funds and protocol insolvency with respect to fee revenue.

---

### Likelihood Explanation

This is not a conditional or probabilistic risk. It is a structural invariant of the current code: every fee payment goes in, nothing comes out. It triggers on every legitimate price update call made by any user, with no special preconditions.

---

### Recommendation

1. Add a `withdraw_fees` function to `price_info.move` (or `pyth.move`) that is callable only by the `fee_recipient_address` stored in `State`, which extracts the `Coin<SUI>` dynamic field from a `PriceInfoObject` and transfers it to the recipient.
2. Alternatively, route fees directly to `fee_recipient_address` at the time of deposit in `update_single_price_feed`, bypassing storage in `PriceInfoObject` entirely.
3. Remove or update the misleading comment in `set_fee_recipient.move` once the mechanism is restored.

---

### Proof of Concept

```
1. Call update_single_price_feed N times with valid VAA/accumulator data,
   each time passing base_update_fee SUI coins.
2. After each call, price_info::get_balance(price_info_object) increases by base_update_fee.
3. After N calls, get_balance == N * base_update_fee.
4. Attempt to call any withdraw/collect function — none exists.
5. The fee_recipient_address in State is set but never used to transfer funds.
6. All N * base_update_fee SUI is permanently locked in the PriceInfoObject.
``` [7](#0-6) [8](#0-7)

        pyth_state: &PythState,
        price_updates: HotPotatoVector<PriceInfo>,
        price_info_object: &mut PriceInfoObject,
        fee: Coin<SUI>,
        clock: &Clock
    ): HotPotatoVector<PriceInfo> {
        let latest_only = state::assert_latest_only(pyth_state);

        // On Sui, users get to choose which price feeds to update. They specify a single price feed to
        // update at a time. We therefore charge the base fee for each such individual update.
        // This is a departure from Eth, where users don't get to necessarily choose.
        assert!(state::get_base_update_fee(pyth_state) <= coin::value(&fee), E_INSUFFICIENT_FEE);

        // store fee coins within price info object
        price_info::deposit_fee_coins(price_info_object, fee);

        // Find price update corresponding to PriceInfoObject within the array of price_updates
        // and use it to update PriceInfoObject.
        let i = 0;
        let found = false;
        while (i < hot_potato_vector::length<PriceInfo>(&price_updates)){
            let cur_price_info = hot_potato_vector::borrow<PriceInfo>(&price_updates, i);
            if (has_same_price_identifier(cur_price_info, price_info_object)){
                found = true;
                update_cache(latest_only, cur_price_info, price_info_object, clock);
                break
            };
            i = i + 1;
        };
        if (found==false){
            abort E_PRICE_UPDATE_NOT_FOUND_FOR_PRICE_INFO_OBJECT
        };
        price_updates
    }

    public fun get_balance(price_info_object: &PriceInfoObject): u64 {
        if (!dynamic_object_field::exists_with_type<vector<u8>, Coin<SUI>>(&price_info_object.id, FEE_STORAGE_KEY)) {
            return 0
        };
        let fee = dynamic_object_field::borrow<vector<u8>, Coin<SUI>>(&price_info_object.id, FEE_STORAGE_KEY);
        coin::value(fee)
    }

    public fun deposit_fee_coins(price_info_object: &mut PriceInfoObject, fee_coins: Coin<SUI>) {
        if (!dynamic_object_field::exists_with_type<vector<u8>, Coin<SUI>>(&price_info_object.id, FEE_STORAGE_KEY)) {
            dynamic_object_field::add(&mut price_info_object.id, FEE_STORAGE_KEY, fee_coins);
        }
        else {
            let current_fee = dynamic_object_field::borrow_mut<vector<u8>, Coin<SUI>>(
                &mut price_info_object.id,
                FEE_STORAGE_KEY
            );
            coin::join(current_fee, fee_coins);
        };
    }
