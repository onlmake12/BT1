### Title
Duplicate Indexes in `GetBlockTransactions` Without Deduplication Causes Bandwidth Amplification — (`sync/src/relayer/get_block_transactions_process.rs`)

---

### Summary

`GetBlockTransactionsProcess::execute()` iterates over the attacker-supplied `indexes` field without checking for duplicates. An unprivileged peer can send a `GetBlockTransactions` message with `MAX_RELAY_TXS_NUM_PER_BATCH` identical transaction indexes, causing the victim node to serialize and transmit `MAX_RELAY_TXS_NUM_PER_BATCH` copies of the same transaction per request — a repeatable, bounded bandwidth amplification attack. The analogous handler `GetBlockProposalProcess` already performs this deduplication check; `GetBlockTransactionsProcess` does not.

---

### Finding Description

**Vulnerability class:** User-supplied list iterated without deduplication, causing repeated resource consumption per duplicate entry — the same class as the twTAP `_claimRewardsOn` bug.

In `sync/src/relayer/get_block_transactions_process.rs`, `GetBlockTransactionsProcess::execute()` validates the `indexes` field only by checking its length: [1](#0-0) 

```rust
if get_block_transactions.indexes().len() > MAX_RELAY_TXS_NUM_PER_BATCH {
    return StatusCode::ProtocolMessageIsMalformed.with_context(...);
}
if get_block_transactions.uncle_indexes().len() > shared.consensus().max_uncles_num() {
    return StatusCode::ProtocolMessageIsMalformed.with_context(...);
}
```

It then iterates over the indexes without any deduplication, fetching and collecting the same block transaction once per occurrence: [2](#0-1) 

```rust
let transactions = self
    .message
    .indexes()
    .iter()
    .filter_map(|i| {
        block.transactions().get(Into::<u32>::into(i) as usize).cloned()
    })
    .collect::<Vec<_>>();

let uncles = self
    .message
    .uncle_indexes()
    .iter()
    .filter_map(|i| block.uncles().get(Into::<u32>::into(i) as usize))
    .collect::<Vec<_>>();
```

All collected entries — including duplicates — are serialized and sent back in a `BlockTransactions` response: [3](#0-2) 

**Contrast with `GetBlockProposalProcess`**, which explicitly deduplicates and rejects duplicate proposals: [4](#0-3) 

```rust
let proposals: HashSet<packed::ProposalShortId> =
    self.message.proposals().to_entity().into_iter().collect();
if proposals.len() != message_len {
    return StatusCode::RequestDuplicate.with_context("Request duplicate proposal");
}
```

The `GetBlockTransactions` message schema (`indexes: Uint32Vec`, `uncle_indexes: Uint32Vec`) places no structural constraint on uniqueness: [5](#0-4) 

---

### Impact Explanation

An attacker sends `GetBlockTransactions { block_hash: <any stored block>, indexes: [k; MAX_RELAY_TXS_NUM_PER_BATCH] }` where `k` is the index of the largest transaction in that block. The victim node:

1. Passes the only validation check (length ≤ `MAX_RELAY_TXS_NUM_PER_BATCH`).
2. Fetches and clones the same `TransactionView` `MAX_RELAY_TXS_NUM_PER_BATCH` times.
3. Serializes and transmits all copies in a single `BlockTransactions` response.

The attacker sends a small fixed-size request and receives a response amplified by up to `MAX_RELAY_TXS_NUM_PER_BATCH × max_tx_size`. Since the code never bans or rate-limits a peer for this pattern (the message is structurally valid), the attacker can repeat indefinitely, exhausting the victim's outbound bandwidth and serialization CPU. The same amplification applies to `uncle_indexes` (bounded by `max_uncles_num`, typically 2).

---

### Likelihood Explanation

- Any peer that establishes a P2P relay connection can send `GetBlockTransactions` — no keys, stake, or hashpower required.
- Block hashes are public; the attacker trivially identifies a stored block with a large transaction.
- The victim node does not disconnect or penalize the peer for sending duplicate indexes, so the attack can be sustained indefinitely from a single connection.

---

### Recommendation

Add a duplicate-index check immediately after the length check, mirroring the pattern already used in `GetBlockProposalProcess`:

```rust
// In GetBlockTransactionsProcess::execute(), after the length checks:
let indexes: Vec<u32> = get_block_transactions.indexes().iter()
    .map(Into::into).collect();
let unique_count = indexes.iter().cloned().collect::<HashSet<u32>>().len();
if unique_count != indexes.len() {
    return StatusCode::ProtocolMessageIsMalformed
        .with_context("Duplicate transaction indexes");
}

let uncle_indexes: Vec<u32> = get_block_transactions.uncle_indexes().iter()
    .map(Into::into).collect();
let unique_uncle_count = uncle_indexes.iter().cloned().collect::<HashSet<u32>>().len();
if unique_uncle_count != uncle_indexes.len() {
    return StatusCode::ProtocolMessageIsMalformed
        .with_context("Duplicate uncle indexes");
}
```

### Proof of Concept

1. Connect to a CKB mainnet/testnet node as a relay peer (`SupportProtocols::RelayV3`).
2. Obtain any block hash `H` for a block the node has stored; identify transaction index `k` pointing to a large transaction (e.g., one with many outputs).
3. Craft and send:
   ```
   GetBlockTransactions {
       block_hash: H,
       indexes: [k, k, k, ..., k],   // MAX_RELAY_TXS_NUM_PER_BATCH entries
       uncle_indexes: [],
   }
   ```
4. Observe the node responds with `BlockTransactions` containing `MAX_RELAY_TXS_NUM_PER_BATCH` copies of the same transaction.
5. Repeat in a tight loop; the node never bans the peer for this pattern because the message passes all existing validation checks. [6](#0-5)

    pub async fn execute(self) -> Status {
        let shared = self.relayer.shared();
        {
            let get_block_transactions = self.message;
            if get_block_transactions.indexes().len() > MAX_RELAY_TXS_NUM_PER_BATCH {
                return StatusCode::ProtocolMessageIsMalformed.with_context(format!(
                    "Indexes count({}) > MAX_RELAY_TXS_NUM_PER_BATCH({})",
                    get_block_transactions.indexes().len(),
                    MAX_RELAY_TXS_NUM_PER_BATCH,
                ));
            }
            if get_block_transactions.uncle_indexes().len() > shared.consensus().max_uncles_num() {
                return StatusCode::ProtocolMessageIsMalformed.with_context(format!(
                    "UncleIndexes count({}) > consensus max_uncles_num({})",
                    get_block_transactions.uncle_indexes().len(),
                    shared.consensus().max_uncles_num(),
                ));
            }
        }

        let block_hash = self.message.block_hash().to_entity();
        debug_target!(
            crate::LOG_TARGET_RELAY,
            "get_block_transactions {}",
            block_hash
        );

        if let Some(block) = shared.store().get_block(&block_hash) {
            let transactions = self
                .message
                .indexes()
                .iter()
                .filter_map(|i| {
                    block
                        .transactions()
                        .get(Into::<u32>::into(i) as usize)
                        .cloned()
                })
                .collect::<Vec<_>>();

            let uncles = self
                .message
                .uncle_indexes()
                .iter()
                .filter_map(|i| block.uncles().get(Into::<u32>::into(i) as usize))
                .collect::<Vec<_>>();

            let content = packed::BlockTransactions::new_builder()
                .block_hash(block_hash)
                .transactions(
                    transactions
                        .into_iter()
                        .map(|tx| tx.data())
                        .collect::<Vec<_>>(),
                )
                .uncles(
                    uncles
                        .into_iter()
                        .map(|uncle| uncle.data())
                        .collect::<Vec<_>>(),
                )
                .build();
            let message = packed::RelayMessage::new_builder().set(content).build();

            return async_send_message_to(&self.nc, self.peer, &message).await;
        }

        Status::ok()
    }
