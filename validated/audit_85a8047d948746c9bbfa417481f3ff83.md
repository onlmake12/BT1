### Title
Stale `update_fee` Used in Async Wormhole Callback Causes Incorrect Refunds for Pending Price Updates - (File: target_chains/near/receiver/src/lib.rs)

### Summary
The NEAR Pyth receiver contract's `update_price_feeds` function accepts a user deposit and dispatches an async cross-contract call to Wormhole for VAA verification. The resulting callbacks (`verify_wormhole_batch_callback` and `verify_wormhole_merkle_callback`) compute the user's refund using `self.update_fee` **at callback time**, not at the time of the original call. Because `update_fee` is a mutable global parameter changeable via governance (`GovernanceAction::SetFee`), any fee change that lands between the initial call and the callback causes the user to receive a different refund than they were entitled to — either a loss or an unexpected gain.

### Finding Description

In `update_price_feeds`, the caller's full attached deposit is forwarded to the callback:

```rust
Self::ext(env::current_account_id())
    .with_attached_deposit(env::attached_deposit())
    .verify_wormhole_batch_callback(env::predecessor_account_id(), data),
```

Inside `verify_wormhole_batch_callback`, the refund is computed using the **current** `self.update_fee`:

```rust
Self::refund_storage_usage(
    account_id,
    storage,
    env::storage_usage(),
    env::attached_deposit(),
    Some(self.update_fee),   // ← current global fee, not the fee at call time
)
```

The same pattern appears in `verify_wormhole_merkle_callback`. The `update_fee` is never snapshotted per-request. It is a global field on the `Pyth` struct that governance can change at any time via `GovernanceAction::SetFee`, which calls `set_update_fee`:

```rust
pub fn set_update_fee(&mut self, fee: u64, expo: u64) -> Result<(), Error> {
    self.update_fee = NearToken::from_yoctonear(...);
    Ok(())
}
```

Because NEAR cross-contract calls are asynchronous and span multiple blocks, a governance transaction that changes `update_fee` can be included between the user's `update_price_feeds` call and the Wormhole verification callback.

### Impact Explanation

- **Fee increase between call and callback**: `refund_storage_usage` deducts the new, higher `update_fee` from the user's deposit. The user receives less NEAR back than they paid for. In the extreme case where the new fee exceeds the deposit, `refund_storage_usage` returns `Err(Error::InsufficientDeposit)`, the callback fails, and the user's deposit may be locked (DoS).
- **Fee decrease between call and callback**: The user receives more NEAR back than they paid for. The surplus is drawn from the contract's own balance, draining protocol revenue.

### Likelihood Explanation

Low. Requires a governance `SetFee` transaction to be included in the narrow window between a user's `update_price_feeds` call and the Wormhole verification callback. NEAR's async model means this window spans at least one additional block, making it non-zero but uncommon in practice. The scenario is most plausible during a planned fee migration where governance and user activity overlap.

### Recommendation

Snapshot `self.update_fee` at the time of the initial call and pass it explicitly through to the callback, rather than reading the global field at callback time. For example, add a `fee_at_call_time` parameter to `verify_wormhole_batch_callback` and `verify_wormhole_merkle_callback`, populated from `self.update_fee` before the cross-contract call is dispatched.

### Proof of Concept

1. `update_fee` is currently `F1`. User calls `update_price_feeds` with deposit `D ≥ F1 + storage_cost`.
2. Governance submits a `SetFee` VAA that raises `update_fee` to `F2 > F1`. This transaction is included before the Wormhole callback fires.
3. `verify_wormhole_batch_callback` executes with `self.update_fee = F2`.
4. `refund_storage_usage` computes `cost = storage_cost + F2`. If `D < cost`, it returns `Err(InsufficientDeposit)` — callback fails, deposit is not returned to the user.
5. If `D ≥ cost`, the user receives `D − storage_cost − F2` instead of `D − storage_cost − F1`, losing `F2 − F1` NEAR.

    #[payable]
    #[handle_result]
    pub fn update_price_feeds(&mut self, data: String) -> Result<(), Error> {
        // Attempt to deserialize the Payload based on header.
        let bytes = &*hex::decode(data.clone()).map_err(|_| Error::InvalidHex)?;
        let cursor = &mut Cursor::new(bytes);
        let mut header = [0u8; 4];
        cursor.clone().read_exact(&mut header).unwrap();

        // Handle Accumulator style Price Updates.
        if &header == PYTHNET_ACCUMULATOR_UPDATE_MAGIC {
            let update_data =
                AccumulatorUpdateData::try_from_slice(&cursor.clone().into_inner()).unwrap();

            match update_data.proof {
                Proof::WormholeMerkle { vaa, .. } => {
                    self.verify_encoded_vaa_source(vaa.as_ref())?;
                    let vaa_hex = hex::encode(vaa.as_ref());
                    ext_wormhole::ext(self.wormhole.clone())
                        .with_static_gas(Gas::from_gas(30_000_000_000_000))
                        .verify_vaa(vaa_hex.clone())
                        .then(
                            Self::ext(env::current_account_id())
                                .with_static_gas(Gas::from_gas(30_000_000_000_000))
                                .with_attached_deposit(env::attached_deposit())
                                .verify_wormhole_merkle_callback(
                                    env::predecessor_account_id(),
                                    data,
                                ),
                        )
                        .then(
                            Self::ext(env::current_account_id())
                                .with_static_gas(Gas::from_gas(30_000_000_000_000))
                                .refund_vaa(env::predecessor_account_id(), env::attached_deposit()),
                        );
                }
            };
        } else {
            // We Verify the VAA is coming from a trusted source chain before attempting to verify
            // VAA signatures. Avoids a cross-contract call early.
            self.verify_encoded_vaa_source(bytes)?;

            // Verify VAA and refund the caller in case of failure.
            ext_wormhole::ext(self.wormhole.clone())
                .with_static_gas(Gas::from_gas(30_000_000_000_000))
                .verify_vaa(data.clone())
                .then(
                    Self::ext(env::current_account_id())
                        .with_static_gas(Gas::from_gas(30_000_000_000_000))
                        .with_attached_deposit(env::attached_deposit())
                        .verify_wormhole_batch_callback(env::predecessor_account_id(), data),
                )
                .then(
                    Self::ext(env::current_account_id())
                        .with_static_gas(Gas::from_gas(30_000_000_000_000))
                        .refund_vaa(env::predecessor_account_id(), env::attached_deposit()),
                );
        }

        Ok(())
    }

    fn refund_storage_usage(
        recipient: AccountId,
        before: StorageUsage,
        after: StorageUsage,
        deposit: NearToken,
        additional_fee: Option<NearToken>,
    ) -> Result<(), Error> {
        let fee = additional_fee.unwrap_or_default();

        if let Some(diff) = after.checked_sub(before) {
            // Handle storage increases if checked_sub succeeds.
            let cost = (env::storage_byte_cost().checked_mul(diff.into()).unwrap())
                .checked_add(fee)
                .unwrap();

            // Use match
            match deposit.checked_sub(cost) {
                Some(refund) => {
                    if !refund.is_zero() {
                        Promise::new(recipient).transfer(refund);
                    }
                }
                None => {
                    return Err(Error::InsufficientDeposit);
                }
            }
        } else {
            // If checked_sub fails we have a storage decrease, we want to refund them the cost of
            // the amount reduced, as well the original deposit they sent.
            let storage_refund = env::storage_byte_cost()
                .checked_mul(before.checked_sub(after).unwrap().into())
                .unwrap();
            let refund = storage_refund
                .checked_add(deposit)
                .unwrap()
                .checked_sub(fee)
                .unwrap();
            Promise::new(recipient).transfer(refund);
        }

        Ok(())
    }
