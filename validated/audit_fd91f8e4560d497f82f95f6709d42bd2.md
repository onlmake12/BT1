### Title
Unauthenticated `from` Field in Hole-Punching Protocol Enables NAT Traversal Hijacking via Peer ID Spoofing — (`File: network/src/protocols/hole_punching/component/connection_request.rs`)

### Summary

The CKB hole-punching protocol accepts the `from` field in `ConnectionRequest` and `ConnectionSync` messages as a self-declared peer identity without verifying it against the actual authenticated session peer ID. Any connected peer can spoof the `from` field to poison the `pending_delivered` map on a target node, causing it to attempt NAT traversal to attacker-controlled addresses. This is a direct analog to the external report's "first offer accepted without sender authentication" pattern: the relay channel is the gossip broadcast, the unauthenticated identity is the `from` field, and the blind acceptance is the `pending_delivered.insert(from_peer_id, ...)` call.

### Finding Description

The hole-punching protocol is implemented in `network/src/protocols/hole_punching/`. The three-message handshake is:

1. **`ConnectionRequest`** — initiator broadcasts `{ from: self_id, to: target_id, listen_addrs: [...] }` via gossip.
2. **`ConnectionRequestDelivered`** — target responds with its own listen addresses, keyed by `from`.
3. **`ConnectionSync`** — initiator confirms, triggering NAT traversal on the target using the stored addresses.

**Root cause — `ConnectionRequest` processing:**

In `ConnectionRequestProcess::execute()`, when `self_peer_id == content.to`, the handler calls `respond_delivered(content.from, ...)`:

```rust
// connection_request.rs line 145-147
if self_peer_id == &content.to {
    self.respond_delivered(content.from, &content.to, content.listen_addrs)
        .await
```

Inside `respond_delivered`, the attacker-controlled `from_peer_id` is stored as the key in `pending_delivered` alongside the attacker-supplied `remote_listens`:

```rust
// connection_request.rs lines 234-237
let now = unix_time_as_millis();
self.protocol
    .pending_delivered
    .insert(from_peer_id, (remote_listens, now));
```

At no point is `content.from` verified to equal the peer ID of the actual authenticated session (`self.peer`). The session's real peer ID is available via `context.session.address` (which carries the Secio-authenticated peer ID), but it is never compared against `content.from`.

**Root cause — `ConnectionSync` processing:**

In `ConnectionSyncProcess::execute()`, when `self_peer_id == content.to`, the handler looks up `pending_delivered` using the attacker-controlled `content.from`:

```rust
// connection_sync.rs lines 111-115
let listens_info = self
    .protocol
    .pending_delivered
    .get(&content.from)
    .map(|info| info.0.clone());
```

It then spawns a NAT traversal task to the addresses stored under that key — which the attacker poisoned in step 1.

**Complete attack path:**

1. Attacker (any peer connected to victim node C) sends a `ConnectionRequest` with `from = B_peer_id` (a legitimate peer), `to = C_peer_id`, `listen_addrs = [attacker_ip:port]`.
2. C's `ConnectionRequestProcess::execute()` sees `self_peer_id == content.to`, calls `respond_delivered`. No check that `content.from` matches the session's authenticated peer ID.
3. C stores `pending_delivered[B_peer_id] = ([attacker_ip:port], now)` and sends `ConnectionRequestDelivered` back to the attacker's session.
4. Attacker sends `ConnectionSync { from: B_peer_id, to: C_peer_id, route: [] }` to C.
5. C's `ConnectionSyncProcess::execute()` sees `self_peer_id == content.to`, retrieves `pending_delivered[B_peer_id]` → `[attacker_ip:port]`, and spawns a TCP connection attempt to the attacker's address.

The `forward_rate_limiter` is keyed by `(content.from, content.to, msg_item_id)` — not by the actual session — so the attacker can rotate `msg_item_id` or wait 1 second to re-poison. The `HOLE_PUNCHING_INTERVAL` of 2 minutes means a single poisoning blocks legitimate hole-punching between B and C for that window. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

### Impact Explanation

- **NAT traversal hijacking**: The attacker causes the victim node to initiate TCP connections to attacker-controlled IP addresses, believing it is connecting to a legitimate peer. While the Secio layer will reject the connection if the attacker cannot present the spoofed peer's private key, the TCP SYN is still sent to the attacker's address, enabling port scanning, IP probing, and connection-slot exhaustion.
- **Persistent hole-punching denial**: By poisoning `pending_delivered[B_peer_id]` with unreachable or attacker-controlled addresses, the attacker prevents C from ever successfully hole-punching to B for the duration of `HOLE_PUNCHING_INTERVAL` (2 minutes) or `TIMEOUT` (5 minutes). This can be renewed continuously at 1 request/second.
- **Network topology manipulation**: An attacker can selectively block hole-punching between any pair of peers it knows about, degrading NAT traversal success rates across the network and forcing nodes behind NAT to remain isolated.
- **Outbound connection slot exhaustion**: By sending many spoofed `ConnectionRequest` + `ConnectionSync` pairs with different `from` values, the attacker can cause the victim to spawn many concurrent TCP connection tasks to arbitrary addresses. [5](#0-4) 

### Likelihood Explanation

The precondition is a single authenticated P2P connection to any node in the network — the standard requirement for any peer. No special privileges, no key material, no majority hashpower. The `ConnectionRequest` message is gossip-broadcast to `sqrt(total_peers)` nodes per notify cycle, meaning an attacker connected to any relay node can reach the target. The `forward_rate_limiter` provides only 1 request/second per `(from, to, item_id)` tuple, which is trivially bypassed by varying `from` (any peer ID) or waiting 1 second. The attack is fully scriptable with a standard CKB P2P client. [6](#0-5) [7](#0-6) 

### Recommendation

In `ConnectionRequestProcess::execute()`, before calling `respond_delivered`, verify that `content.from` matches the authenticated peer ID of the sending session:

```rust
// Retrieve the authenticated peer ID from the session
let session_peer_id = extract_peer_id(&context.session.address)
    .expect("connected session must have peer id");
if session_peer_id != content.from {
    return StatusCode::InvalidFromPeerId
        .with_context("from field does not match session peer id");
}
```

Apply the same check in `ConnectionSyncProcess::execute()` — verify `content.from` matches the session's authenticated peer ID before looking up `pending_delivered`. This mirrors the fix for the external report: bind the identity claim to the authenticated transport-layer identity rather than accepting a self-declared field. [8](#0-7) [9](#0-8) 

### Proof of Concept

```
1. Attacker (peer A) establishes a normal authenticated P2P connection to victim node C.

2. Attacker crafts a ConnectionRequest:
   {
     from:        B_peer_id,       // legitimate peer B's ID — attacker-controlled field
     to:          C_peer_id,       // victim node C
     max_hops:    6,
     route:       [],
     listen_addrs: [/ip4/1.2.3.4/tcp/9999/p2p/<B_peer_id>]
                                   // attacker's address, tagged with B's peer ID
   }

3. Attacker sends this directly to C over the authenticated session (session_id = A's session).
   C's ConnectionRequestProcess::execute() sees content.to == self_peer_id.
   No check: content.from (B_peer_id) vs session peer ID (A_peer_id).

4. C calls respond_delivered(B_peer_id, C_peer_id, [/ip4/1.2.3.4/tcp/9999/...]).
   C stores: pending_delivered[B_peer_id] = ([/ip4/1.2.3.4/tcp/9999/...], now).
   C sends ConnectionRequestDelivered back to A's session.

5. Attacker sends ConnectionSync { from: B_peer_id, to: C_peer_id, route: [] } to C.
   C's ConnectionSyncProcess::execute() sees content.to == self_peer_id.
   C looks up pending_delivered[B_peer_id] → [/ip4/1.2.3.4/tcp/9999/...].
   C spawns: try_nat_traversal(bind_addr, /ip4/1.2.3.4/tcp/9999/...).

6. C initiates a TCP SYN to 1.2.3.4:9999 (attacker-controlled).
   Legitimate hole-punching from B to C is blocked for HOLE_PUNCHING_INTERVAL (2 min).
   Attacker repeats every 2 minutes to maintain the block indefinitely.
```

    pub(crate) async fn execute(mut self) -> Status {
        let content = match RequestContent::try_from(&self.message) {
            Ok(content) => content,
            Err(status) => return status,
        };
        if content.listen_addrs.len() > ADDRS_COUNT_LIMIT || content.listen_addrs.is_empty() {
            return StatusCode::InvalidListenAddrLen
                .with_context("the listen address count is too large or empty");
        }

        if content.max_hops > MAX_HOPS {
            return StatusCode::InvalidMaxTTL.into();
        }
        if content.route.len() > MAX_HOPS as usize {
            return StatusCode::InvalidRoute.with_context("the route length is too long");
        }

        let self_peer_id = self.protocol.network_state.local_peer_id();
        if content.route.contains(self_peer_id) {
            return StatusCode::Ignore.with_context("the message is passed, ignore it");
        }

        if self
            .protocol
            .forward_rate_limiter
            .check_key(&(content.from.clone(), content.to.clone(), self.msg_item_id))
            .is_err()
        {
            debug!(
                "from: {}, to {}, item_name: {}, rate limit is reached",
                content.from, content.to, "ConnectionRequest",
            );
            return StatusCode::TooManyRequests.with_context("ConnectionRequest");
        }

        if self_peer_id == &content.to {
            self.respond_delivered(content.from, &content.to, content.listen_addrs)
                .await
        } else if content.max_hops == 0u8 {
            StatusCode::ReachedMaxHops.into()
        } else {
            self.forward_message(self_peer_id, &content.to).await
        }
    }

    async fn notify(&mut self, context: &mut ProtocolContext, _token: u64) {
        let status = self.network_state.connection_status();

        let now = unix_time_as_millis();
        self.pending_delivered
            .retain(|_, (_, t)| (now - *t) < TIMEOUT);
        self.inflight_requests.retain(|_, t| (now - *t) < TIMEOUT);

        if status.non_whitelist_outbound < status.max_outbound && status.total > 0 {
            let target = &self.network_state.required_flags;
            let addrs = self.network_state.with_peer_store_mut(|p| {
                p.fetch_nat_addrs(
                    (status.max_outbound - status.non_whitelist_outbound) as usize,
                    *target,
                )
            });

            let from_peer_id = self.network_state.local_peer_id();
            let listen_addrs = {
                let public_addr = self.network_state.public_addrs(ADDRS_COUNT_LIMIT);
                if public_addr.len() < ADDRS_COUNT_LIMIT {
                    let observed_addrs = self
                        .network_state
                        .observed_addrs(ADDRS_COUNT_LIMIT - public_addr.len());
                    let iter = public_addr
                        .iter()
                        .chain(observed_addrs.iter())
                        .map(Multiaddr::to_vec)
                        .map(|v| packed::Address::new_builder().bytes(v).build());
                    packed::AddressVec::new_builder().extend(iter).build()
                } else {
                    let iter = public_addr
                        .iter()
                        .map(Multiaddr::to_vec)
                        .map(|v| packed::Address::new_builder().bytes(v).build());
                    packed::AddressVec::new_builder().extend(iter).build()
                }
            };

            let mut inflight = Vec::new();
            for i in addrs {
                if let Some(to_peer_id) = extract_peer_id(&i.addr) {
                    let conn_req = {
                        let content = component::init_request(
                            from_peer_id,
                            &to_peer_id,
                            listen_addrs.clone(),
                        );
                        packed::HolePunchingMessage::new_builder()
                            .set(content)
                            .build()
                    };
                    let proto_id = SupportProtocols::HolePunching.protocol_id();

                    // Broadcast to a number of nodes equal to the square root of the total connection count using gossip.
                    let mut total = status.total.isqrt();
                    let _ignore = context
                        .filter_broadcast(
                            TargetSession::Filter(Box::new(move |_| {
                                total = total.saturating_sub(1);
                                total != 0
                            })),
                            proto_id,
                            conn_req.as_bytes(),
                        )
                        .await;
                    inflight.push(to_peer_id);
                }
            }

            let now = unix_time_as_millis();
            for peer_id in inflight {
                self.inflight_requests.insert(peer_id, now);
            }
        }
    }
