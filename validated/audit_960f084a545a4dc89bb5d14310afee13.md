### Title
Missing per-peer rate limit on `GetBlockFilters` handler allows unbounded RocksDB read and outbound bandwidth amplification — (`sync/src/filter/get_block_filters_process.rs`)

### Summary
The `BlockFilter` protocol handler processes `GetBlockFilters` messages with no per-peer rate limiting. Every valid request causes up to 1,000 RocksDB reads and a ~1.8 MB response. A single unprivileged peer can flood the node in a tight loop, saturating disk I/O and outbound bandwidth at negligible attacker cost.

### Finding Description
`Relayer` and `HolePunching` both maintain a `governor::RateLimiter<(PeerIndex, u32)>` keyed by `(peer, message_id)` and reject excess requests before any work is done. `BlockFilter` has no such field and no such check. Its `try_process` dispatches directly to `GetBlockFiltersProcess::execute`, which iterates up to `BATCH_SIZE = 1000` blocks, performs two DB lookups per block (`get_block_hash` + `get_block_filter`), and sends back up to ~1.8 MB. The 1.8 MB cap was introduced solely to avoid tentacle frame-size disconnects (CHANGELOG #4972); it does not throttle request frequency. [8](#0-7) [9](#0-8) 

### Impact Explanation
- **Outbound bandwidth**: one peer can sustain ~1.8 MB/response × N req/s, limited only by the victim's CPU and disk throughput.
- **RocksDB read saturation**: up to 2,000 DB reads per request starve other subsystems (chain sync, tx-pool) that share the same RocksDB instance.
- No ban or disconnect is triggered; the attacker can reconnect immediately if disconnected.

### Likelihood Explanation
Block-filter is an opt-in feature (`block_filter_enable = true` in `ckb.toml`), but nodes that enable it (e.g., to serve light clients) are fully exposed. The attack requires only a TCP connection and a 9-byte message in a loop — no keys, no PoW, no special privileges.

### Recommendation
Add a `governor::RateLimiter<(PeerIndex, u32)>` to `BlockFilter` (mirroring `Relayer::rate_limiter`) and check it at the top of `try_process` before dispatching any variant, returning `StatusCode::TooManyRequests` on excess. A quota of 30 req/s per `(peer, message_id)` matches the existing Relayer policy. [10](#0-9) 

### Proof of Concept
1. Run a CKB node with `block_filter_enable = true`; mine or sync past block 1000 so filter data exists.
2. Connect a custom peer on the Filter protocol.
3. In a tight loop, send `BlockFilterMessage { GetBlockFilters { start_number: 0 } }`.
4. Observe on the victim: outbound bandwidth saturates at ~1.8 MB × loop_rate; `rocksdb.block.cache.miss` and read IOPS spike; no ban is applied; the loop runs indefinitely. [6](#0-5) [11](#0-10)

    pub async fn execute(self) -> Status {
        let active_chain = self.filter.shared.active_chain();
        let start_number: BlockNumber = self.message.to_entity().start_number().into();
        let latest: BlockNumber = active_chain.get_latest_built_filter_block_number();

        if latest >= start_number {
            let mut block_hashes = Vec::new();
            let mut filters = Vec::new();
            let mut current_content_size = 0;
            current_content_size += 8; // Size of start_number
            current_content_size += 4 * 2; // Size of the header field `full-size` of `block_hash` and `block_filter`
            let mut block_number = start_number;
            for _ in 0..BATCH_SIZE {
                if let Some(block_hash) = active_chain.get_block_hash(block_number) {
                    if let Some(block_filter) = active_chain.get_block_filter(&block_hash) {
                        if current_content_size
                            + block_hash.as_slice().len()
                            + 4
                            + block_filter.as_slice().len()
                            + 4
                            >= (1.8 * 1024.0 * 1024.0) as usize
                        {
                            // Break if the encoded size of `block_hash` + `block_filter` + `start_number` + molecule header increase reaches 1.8MB, to avoid frame size too large
                            break;
                        }
                        current_content_size +=
                            block_hash.as_slice().len() + block_filter.as_slice().len() + 4;
                        block_hashes.push(block_hash);
                        filters.push(block_filter);
                    } else {
                        break;
                    }
                } else {
                    break;
                }
                let Some(next_block_number) = block_number.checked_add(1) else {
                    break;
                };
                block_number = next_block_number;
            }
            let content = packed::BlockFilters::new_builder()
                .start_number(start_number)
                .block_hashes(block_hashes)
                .filters(filters)
                .build();
            let message = packed::BlockFilterMessage::new_builder()
                .set(content)
                .build();
            async_send_message_to(&self.nc, self.peer, &message).await
        } else {
            Status::ignored()
        }
    }
