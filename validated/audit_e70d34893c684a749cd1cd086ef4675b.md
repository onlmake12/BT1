### Title
Silent `u128`Ôćĺ`u64` Truncation in `calculate_maximum_withdraw` Returns Incorrect Maximum DAO Withdrawal Amount ÔÇö (File: `util/dao/src/lib.rs`)

---

### Summary

`DaoCalculator::calculate_maximum_withdraw` in `util/dao/src/lib.rs` uses a bare `as u64` cast to narrow a `u128` intermediate result, silently truncating the value if it exceeds `u64::MAX`. Every other analogous `u128`Ôćĺ`u64` narrowing in the same file uses `u64::try_from(...).map_err(|_| DaoError::Overflow)?`. The inconsistency means the function can return a silently incorrect (drastically reduced) maximum withdrawal capacity to any caller ÔÇö including the `calculate_dao_maximum_withdraw` RPC endpoint and the internal fee-calculation path ÔÇö without raising any error.

---

### Finding Description

In `calculate_maximum_withdraw`:

```rust
// util/dao/src/lib.rs  lines 152-156
let withdraw_counted_capacity = u128::from(counted_capacity.as_u64())
    * u128::from(withdrawing_ar)
    / u128::from(deposit_ar);
let withdraw_capacity =
    Capacity::shannons(withdraw_counted_capacity as u64)   // ÔćÉ silent truncation
        .safe_add(occupied_capacity)?;
```

`withdraw_counted_capacity as u64` is a Rust truncating cast. If the 128-bit product exceeds `u64::MAX`, the high bits are silently discarded and the function returns a drastically smaller capacity than the true maximum ÔÇö with no error, no panic, and no indication to the caller.

Every other `u128`Ôćĺ`u64` conversion in the same file uses a checked path:

| Location | Pattern |
|---|---|
| `secondary_block_reward`, line 204 | `u64::try_from(reward128).map_err(|_| DaoError::Overflow)?` |
| `dao_field_with_current_epoch`, line 244ÔÇô245 | `u64::try_from(miner_issuance128).map_err(|_| DaoError::Overflow)?` |
| `dao_field_with_current_epoch`, line 258 | `u64::try_from(ar_increase128).map_err(|_| DaoError::Overflow)?` |

`calculate_maximum_withdraw` is the sole exception. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

---

### Impact Explanation

`calculate_maximum_withdraw` is called from three distinct paths:

1. **`calculate_dao_maximum_withdraw` RPC** (`rpc/src/module/experiment.rs` lines 235ÔÇô298): Any RPC caller (wallet, script author, user) querying the maximum DAO withdrawal receives a silently incorrect (too-small) value. Wallets that rely on this RPC to set the output capacity of a withdrawal transaction would construct an under-valued transaction that the DAO script rejects.

2. **`transaction_fee` Ôćĺ `transaction_maximum_withdraw`** (`util/dao/src/lib.rs` lines 30ÔÇô36): Fee verification for DAO withdrawal transactions uses this path. A silently truncated `maximum_withdraw` makes the computed fee appear negative (underflow in `safe_sub`), causing the node to reject a valid DAO withdrawal transaction.

3. **`withdrawed_interests` Ôćĺ `dao_field_with_current_epoch`** (`util/dao/src/lib.rs` lines 312ÔÇô333, 208ÔÇô264): The DAO field written into each block header is computed using this path. An incorrect `withdrawed_interests` value corrupts the on-chain DAO accounting (`S_i`), which would cause other nodes to reject the block as having an invalid DAO field ÔÇö a consensus-level failure. ### Likelihood Explanation

The overflow condition requires:

```
counted_capacity ├Ś withdrawing_ar / deposit_ar  >  u64::MAX
```

`counted_capacity` is bounded by the total CKB supply (~3.36 ├Ś 10┬╣ÔüŞ shannons). `ar` starts at `10_000_000_000_000_000` (10┬╣ÔüÂ) and grows slowly via secondary issuance. For the result to exceed `u64::MAX` (~1.84 ├Ś 10┬╣Ôü╣), the `ar` ratio must exceed ~5.5├Ś, which would require centuries of accumulation under normal issuance parameters. Likelihood is therefore **very low** under current mainnet conditions, but the defect is structurally present and inconsistent with every other conversion in the same file. [8](#0-7) 

---

### Recommendation

Replace the silent cast with the same checked conversion used everywhere else in the file:

```rust
// Before (silent truncation):
Capacity::shannons(withdraw_counted_capacity as u64).safe_add(occupied_capacity)?

// After (consistent with the rest of the file):
let withdraw_counted_capacity_u64 =
    u64::try_from(withdraw_counted_capacity).map_err(|_| DaoError::Overflow)?;
Capacity::shannons(withdraw_counted_capacity_u64).safe_add(occupied_capacity)?
``` [9](#0-8)

### Proof of Concept

The existing test `check_withdraw_calculation_overflows` (line 296) uses a capacity near `u64::MAX` and asserts `result.is_err()`. However, the error it catches comes from the subsequent `safe_add(occupied_capacity)` call overflowing, **not** from the silent truncation of `withdraw_counted_capacity`. A scenario where `withdraw_counted_capacity` itself exceeds `u64::MAX` but `withdraw_counted_capacity as u64 + occupied_capacity` does not would silently return a wrong value and pass the test.

Concretely: set `deposit_ar = 1`, `withdrawing_ar = 2`, `counted_capacity = u64::MAX / 2 + 1`. Then:

```
withdraw_counted_capacity (u128) = (u64::MAX/2 + 1) * 2 = u64::MAX + 1
withdraw_counted_capacity as u64 = 0   ÔćÉ silently truncated
withdraw_capacity = 0 + occupied_capacity  ÔćÉ returned without error
```

The function returns `occupied_capacity` instead of the true maximum, with no error raised. [10](#0-9) [11](#0-10)

    /// Calculate maximum withdraw capacity of a deposited dao output
    pub fn calculate_maximum_withdraw(
        &self,
        output: &CellOutput,
        output_data_capacity: Capacity,
        deposit_header_hash: &Byte32,
        withdrawing_header_hash: &Byte32,
    ) -> Result<Capacity, DaoError> {
        let deposit_header = self
            .data_loader
            .get_header(deposit_header_hash)
            .ok_or(DaoError::InvalidHeader)?;
        let withdrawing_header = self
            .data_loader
            .get_header(withdrawing_header_hash)
            .ok_or(DaoError::InvalidHeader)?;
        if deposit_header.number() >= withdrawing_header.number() {
            return Err(DaoError::InvalidOutPoint);
        }

        let (deposit_ar, _, _, _) = extract_dao_data(deposit_header.dao());
        let (withdrawing_ar, _, _, _) = extract_dao_data(withdrawing_header.dao());

        let occupied_capacity = output.occupied_capacity(output_data_capacity)?;
        let output_capacity: Capacity = output.capacity().into();
        let counted_capacity = output_capacity.safe_sub(occupied_capacity)?;
        let withdraw_counted_capacity = u128::from(counted_capacity.as_u64())
            * u128::from(withdrawing_ar)
            / u128::from(deposit_ar);
        let withdraw_capacity =
            Capacity::shannons(withdraw_counted_capacity as u64).safe_add(occupied_capacity)?;

        Ok(withdraw_capacity)
    }

    fn calculate_dao_maximum_withdraw(
        &self,
        out_point: OutPoint,
        kind: DaoWithdrawingCalculationKind,
    ) -> Result<Capacity> {
        let snapshot: &Snapshot = &self.shared.snapshot();
        let consensus = snapshot.consensus();
        let out_point: packed::OutPoint = out_point.into();
        let data_loader = snapshot.borrow_as_data_loader();
        let calculator = DaoCalculator::new(consensus, &data_loader);
        match kind {
            DaoWithdrawingCalculationKind::WithdrawingHeaderHash(withdrawing_header_hash) => {
                let (tx, deposit_header_hash) = snapshot
                    .get_transaction(&out_point.tx_hash())
                    .ok_or_else(|| RPCError::invalid_params("invalid out_point"))?;
                let output = tx
                    .outputs()
                    .get(out_point.index().into())
                    .ok_or_else(|| RPCError::invalid_params("invalid out_point"))?;
                let output_data = tx
                    .outputs_data()
                    .get(out_point.index().into())
                    .ok_or_else(|| RPCError::invalid_params("invalid out_point"))?;

                match calculator.calculate_maximum_withdraw(
                    &output,
                    core::Capacity::bytes(output_data.len()).expect("should not overflow"),
                    &deposit_header_hash,
                    &withdrawing_header_hash.into(),
                ) {
                    Ok(capacity) => Ok(capacity.into()),
                    Err(err) => Err(RPCError::custom_with_error(RPCError::DaoError, err)),
                }
            }
