### Title
Unauthenticated `from` Field in Hole-Punching `ConnectionRequest` Allows Any Peer to Poison Relay State and Trigger Outbound Connections to Attacker-Controlled Addresses - (File: `network/src/protocols/hole_punching/component/connection_request.rs`)

---

### Summary

The hole-punching protocol's `ConnectionRequest` handler accepts a `from` field that is never verified against the actual sending peer's identity. Any connected peer can forge a `ConnectionRequest` claiming to be any other peer ID, poisoning the relay node's `pending_delivered` map with attacker-controlled listen addresses. A subsequent `ConnectionSync` message (also forgeable) causes the relay node to initiate outbound TCP connections to arbitrary attacker-controlled IP:port pairs, and disrupts legitimate hole-punching for the impersonated peer.

---

### Finding Description

**Root cause — missing sender identity check in `ConnectionRequestProcess::execute()`**

In `network/src/protocols/hole_punching/component/connection_request.rs`, the handler receives a `ConnectionRequest` message containing a `from` field (a `PeerId`) and a `listen_addrs` field (multiaddresses attributed to `from`). The handler has access to `self.peer` — the actual sender's `PeerIndex` (session ID) — but **never verifies that `content.from` matches the peer ID of `self.peer`**. [1](#0-0) 

The only address validation performed is that any embedded peer ID inside `listen_addrs` matches `content.from`: [2](#0-1) 

This check is trivially satisfied by an attacker who sets `content.from = victim_peer_id` and constructs addresses of the form `attacker_ip/p2p/victim_peer_id`.

When the relay node is the `to` target (`self_peer_id == &content.to`), it calls `respond_delivered`, which stores the attacker-supplied addresses under the victim's peer ID: [3](#0-2) 

**Exploit path via `ConnectionSync`**

The `pending_delivered` map is consumed in `connection_sync.rs` when a `ConnectionSync{from=V, to=R}` arrives. The relay looks up `pending_delivered[V]` and spawns outbound TCP connection tasks to those addresses: [4](#0-3) 

The `ConnectionSync` handler also does not verify that `content.from` matches the actual sender, so the attacker can send both messages themselves.

**Complete no-privilege attack sequence:**
1. Attacker (peer E) connects to relay node R.
2. E sends `ConnectionRequest{from=V, to=R, listen_addrs=[attacker_ip/p2p/V_peer_id]}`.
3. R stores `pending_delivered[V] = ([attacker_ip/p2p/V_peer_id], now)` and sends `ConnectionRequestDelivered` back to E.
4. E sends `ConnectionSync{from=V, to=R}`.
5. R looks up `pending_delivered[V]`, finds attacker's address, and spawns an outbound TCP connection to `attacker_ip`. [5](#0-4) 

---

### Impact Explanation

1. **SSRF / Outbound connection to attacker-controlled addresses**: The relay node initiates TCP connections to arbitrary IP:port pairs chosen by the attacker. This can be used for internal network port scanning, firewall bypass, or connection-slot exhaustion.
2. **Hole-punching disruption (DoS)**: `pending_delivered[V]` is poisoned with wrong addresses. Any subsequent legitimate `ConnectionSync{from=V}` causes the relay to attempt connections to the attacker's addresses instead of V's real addresses, causing V's NAT traversal to fail silently.
3. **Rate-limiter quota theft**: The `forward_rate_limiter` is keyed on `(content.from, content.to, msg_item_id)`, so the attacker exhausts rate-limit quota attributed to the victim peer ID `V`, not to the attacker's own session. [6](#0-5) 

---

### Likelihood Explanation

Any peer connected to a CKB node running the hole-punching protocol can send a `ConnectionRequest` with an arbitrary `from` field. No privileged access, leaked keys, or majority hash power is required. The per-session rate limiter (`rate_limiter` keyed on `(session_id, msg_item_id)`) provides some throttling, but does not prevent the attack — the attacker can use different `to` values to bypass the `forward_rate_limiter` and still poison `pending_delivered` for a target victim peer ID. [7](#0-6) 

---

### Recommendation

Add a check in `ConnectionRequestProcess::execute()` that verifies `content.from` equals the actual sender's peer ID. The peer registry can be queried using `self.peer` (the session ID) to retrieve the authenticated peer ID:

```rust
// After parsing content, before any state mutation:
let actual_sender_peer_id = self.protocol.network_state
    .with_peer_registry(|reg| reg.get_peer(self.peer).map(|p| p.peer_id.clone()));

if actual_sender_peer_id.as_ref() != Some(&content.from) {
    return StatusCode::InvalidFromPeerId
        .with_context("from field does not match actual sender");
}
```

The same check should be applied in `ConnectionRequestDeliveredProcess` and `ConnectionSyncProcess` to prevent analogous spoofing of the `from`/`to` fields in those message types.

### Proof of Concept

**Precondition**: Attacker E is a connected peer to relay node R. Victim V is any known peer ID (e.g., obtained from the discovery protocol).

**Step 1** — Poison `pending_delivered[V]`:
```
E → R: ConnectionRequest {
    from: V_peer_id,
    to:   R_peer_id,
    listen_addrs: ["/ip4/1.2.3.4/tcp/9999/p2p/<V_peer_id>"],
    max_hops: 1,
    route: []
}
```
R executes `respond_delivered(V_peer_id, R_peer_id, ["/ip4/1.2.3.4/tcp/9999/p2p/<V_peer_id>"])`, stores `pending_delivered[V] = ([1.2.3.4:9999], now)`, and sends `ConnectionRequestDelivered` back to E.

**Step 2** — Trigger outbound connection:
```
E → R: ConnectionSync {
    from: V_peer_id,
    to:   R_peer_id,
    route: [],
    sync_route: []
}
```
R looks up `pending_delivered[V]`, finds `1.2.3.4:9999`, and spawns `try_nat_traversal(bind_addr, "/ip4/1.2.3.4/tcp/9999/p2p/<V_peer_id>")`.

**Expected outcome**: R initiates an outbound TCP connection to `1.2.3.4:9999` (attacker-controlled). Any legitimate hole-punching attempt by V through R now fails because `pending_delivered[V]` contains the wrong addresses. [8](#0-7) [9](#0-8)

    pub(crate) async fn execute(mut self) -> Status {
        let content = match RequestContent::try_from(&self.message) {
            Ok(content) => content,
            Err(status) => return status,
        };
        if content.listen_addrs.len() > ADDRS_COUNT_LIMIT || content.listen_addrs.is_empty() {
            return StatusCode::InvalidListenAddrLen
                .with_context("the listen address count is too large or empty");
        }

        if content.max_hops > MAX_HOPS {
            return StatusCode::InvalidMaxTTL.into();
        }
        if content.route.len() > MAX_HOPS as usize {
            return StatusCode::InvalidRoute.with_context("the route length is too long");
        }

        let self_peer_id = self.protocol.network_state.local_peer_id();
        if content.route.contains(self_peer_id) {
            return StatusCode::Ignore.with_context("the message is passed, ignore it");
        }

        if self
            .protocol
            .forward_rate_limiter
            .check_key(&(content.from.clone(), content.to.clone(), self.msg_item_id))
            .is_err()
        {
            debug!(
                "from: {}, to {}, item_name: {}, rate limit is reached",
                content.from, content.to, "ConnectionRequest",
            );
            return StatusCode::TooManyRequests.with_context("ConnectionRequest");
        }

        if self_peer_id == &content.to {
            self.respond_delivered(content.from, &content.to, content.listen_addrs)
                .await
        } else if content.max_hops == 0u8 {
            StatusCode::ReachedMaxHops.into()
        } else {
            self.forward_message(self_peer_id, &content.to).await
        }
    }

    pub(crate) async fn execute(self) -> Status {
        let content = match DeliverdContent::try_from(&self.message) {
            Ok(content) => content,
            Err(status) => return status,
        };
        if content.listen_addrs.len() > ADDRS_COUNT_LIMIT || content.listen_addrs.is_empty() {
            return StatusCode::InvalidListenAddrLen
                .with_context("the listen address count is too large or empty");
        }

        if content.route.len() > MAX_HOPS as usize || content.sync_route.len() > MAX_HOPS as usize {
            return StatusCode::InvalidRoute.with_context("the route length is too long");
        }

        if self
            .protocol
            .forward_rate_limiter
            .check_key(&(content.from.clone(), content.to.clone(), self.msg_item_id))
            .is_err()
        {
            debug!(
                "from: {}, to {}, item_name: {}, rate limit is reached",
                content.from, content.to, "ConnectionRequestDelivered",
            );
            return StatusCode::TooManyRequests.with_context("ConnectionRequestDelivered");
        }

        match content.route.last() {
            Some(next_peer_id) => self.forward_delivered(next_peer_id).await,
            None => {
                let self_peer_id = self.protocol.network_state.local_peer_id();
                if self_peer_id != &content.from {
                    // forward the message to the `from` peer
                    self.forward_delivered(&content.from).await
                } else {
                    // the current peer is the target peer, respond the sync back
                    if let Some(metrics) = ckb_metrics::handle() {
                        metrics.ckb_hole_punching_active_count.inc();
                    }

                    let request_start = self.protocol.inflight_requests.remove(&content.to);

                    match request_start {
                        Some(start) => {
                            let res = self.respond_sync(content.from).await;
                            if !res.is_ok() {
                                return res;
                            }
                            let now = unix_time_as_millis();
                            let ttl = now - start;

                            self.try_nat_traversal(ttl, content.listen_addrs);

                            Status::ok()
                        }
                        None => StatusCode::Ignore.with_context("the request is not in flight"),
                    }
                }
            }
        }
    }
