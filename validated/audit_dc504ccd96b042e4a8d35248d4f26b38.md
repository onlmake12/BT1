### Title
Blast-Deployed `Pyth.sol` Configures Claimable Gas With No Claim/Withdrawal Function, Permanently Locking Accumulated Gas Rewards — (`File: target_chains/ethereum/contracts/chain_patches/blast-gas-claim-patch.diff`)

---

### Summary

The Blast chain-specific patch applied to `Pyth.sol` adds a `configureClaimableGas()` function that switches the contract's Blast gas mode to "claimable," causing all gas fees spent by users interacting with the Pyth contract on Blast to accumulate in the Blast precompile. However, neither the `IBlast` interface defined in the patch nor the patched `Pyth.sol` includes any `claimGas()`, `claimAllGas()`, or equivalent withdrawal function. The accumulated gas is permanently locked with no recovery path.

---

### Finding Description

The patch at `target_chains/ethereum/contracts/chain_patches/blast-gas-claim-patch.diff` modifies `Pyth.sol` for Blast deployment as follows:

```diff
+interface IBlast {
+    function configureClaimableGas() external;
+}
...
+    function configureClaimableGas() external {
+        IBlast(0x4300000000000000000000000000000000000002)
+            .configureClaimableGas();
+    }
```

On Blast, the precompile at `0x4300000000000000000000000000000000000002` supports two gas modes:
- **Void (default):** gas fees are burned.
- **Claimable:** gas fees accumulate and can be claimed by the contract via `claimAllGas()`, `claimGasAtMinClaimRate()`, etc.

Once `configureClaimableGas()` is called, every transaction that touches the Pyth contract on Blast causes gas to accumulate in the precompile, attributed to the Pyth contract address. The full Blast `IBlast` interface exposes claim functions (`claimAllGas`, `claimMaxGas`, `claimGasAtMinClaimRate`), but the patch only exposes `configureClaimableGas()` — none of the claim functions are included.

A grep across all `.sol` files in the repository confirms there is no `claimGas`, `claimAllGas`, or `claimMaxGas` function anywhere in the codebase.

Additionally, `configureClaimableGas()` is declared `external` with no access control modifier, meaning any unprivileged caller can invoke it. [1](#0-0) 

---

### Impact Explanation

All gas fees accumulated by the Pyth contract on Blast are permanently locked in the Blast precompile with no mechanism to retrieve them. As Pyth is a high-throughput price oracle receiving a large volume of `updatePriceFeeds` calls, the volume of locked gas rewards is non-trivial and grows continuously. The Pyth protocol suffers a direct, permanent financial loss of all claimable gas rewards on Blast.

---

### Likelihood Explanation

The vulnerability is triggered passively: every transaction sent to the Pyth contract on Blast after `configureClaimableGas()` is called causes gas to accumulate. No attacker action is required — the loss is automatic and continuous. The patch exists specifically to enable gas claiming on Blast, confirming the intent was to collect these rewards, making the missing claim function a concrete operational loss rather than a theoretical one.

---

### Recommendation

Extend the `IBlast` interface in the patch to include the claim functions:

```solidity
interface IBlast {
    function configureClaimableGas() external;
    function claimAllGas(address contractAddress, address recipientOfGas) external returns (uint256);
    function claimMaxGas(address contractAddress, address recipientOfGas) external returns (uint256);
}
```

Add an access-controlled `claimGas()` function to the patched `Pyth.sol` (restricted to `owner` or `admin`):

```solidity
function claimGas(address recipient) external onlyOwner {
    IBlast(0x4300000000000000000000000000000000000002)
        .claimAllGas(address(this), recipient);
}
```

Also add an access control check to `configureClaimableGas()` itself to prevent unprivileged callers from toggling the gas mode.

---

### Proof of Concept

1. The Blast-specific patch is applied to `Pyth.sol` before deployment on Blast.
2. Anyone calls `Pyth.configureClaimableGas()` (no access control) — this sets the contract's gas mode to claimable on the Blast precompile.
3. Users call `updatePriceFeeds{value: fee}(...)` on the Blast-deployed Pyth contract. Each call causes gas to accumulate in the Blast precompile attributed to the Pyth contract.
4. There is no function in the patched `Pyth.sol` to call `IBlast.claimAllGas()` or any equivalent.
5. All accumulated gas rewards are permanently locked — confirmed by the absence of any `claimGas`/`claimAllGas` function across all `.sol` files in the repository. [2](#0-1)
