### Title
Cache-Bypass DoS via Unbounded `get_overview(EVERYTHING)` Forcing All Five Expensive Backend Operations Per Call — (`rpc/src/module/terminal.rs`)

---

### Summary

`TerminalRpcImpl::get_overview` accepts a caller-controlled `refresh` bitmask. When `refresh = 31` (`EVERYTHING = 0x1F`), the implementation unconditionally calls `clear_all()` to evict all five caches and then forces every sub-operation to bypass its own TTL check, executing `System::new_all()+refresh_all()`, `get_block_template`, `estimate_num_keys_cf`, snapshot acquisition, and peer enumeration on every single call. No rate limiting, cooldown, or minimum interval exists on this endpoint. Any caller who can reach the RPC port can call this in a tight loop, completely defeating the cache TTL invariant the module was designed to enforce.

---

### Finding Description

The double-bypass is structural, not incidental:

**Step 1 — `clear_all()` unconditionally evicts all caches:**

```rust
// rpc/src/module/terminal.rs:446-448
if refresh.contains(RefreshKind::EVERYTHING) {
    self.cache.clear_all();
}
``` [1](#0-0) 

**Step 2 — Each sub-function also skips its TTL check because `EVERYTHING` contains every individual flag:**

```rust
// get_sys_info: line 502
if !refresh.contains(RefreshKind::SYSTEM_INFO)   // false when EVERYTHING is set
    && let Some(cached) = self.cache.get_sys_info()
``` [2](#0-1) 

The same pattern applies to all five sub-functions (`get_mining_info` line 470, `get_tx_pool_info` line 574, `get_cells_info` line 626, `get_network_info` line 657). Since `EVERYTHING = 0b00011111` contains all five individual flags, every sub-function's cache guard evaluates to false and the expensive backend path is always taken. [3](#0-2) 

**The five forced operations per call:**

| Sub-function | Expensive operation |
|---|---|
| `get_sys_info` | `System::new_all()` + `sys.refresh_all()` + `SysDisks::new_with_refreshed_list()` + `SysNetworks::new_with_refreshed_list()` |
| `get_mining_info` | `self.shared.snapshot()` + epoch_ext read |
| `get_tx_pool_info` | `tx_pool.get_tx_pool_info()` + **`self.shared.get_block_template()`** + `get_total_recent_reject_num()` |
| `get_cells_info` | `self.shared.cloned_snapshot()` + `store.estimate_num_keys_cf(COLUMN_CELL)` |
| `get_network_info` | `network_controller.connected_peers()` full enumeration | [4](#0-3) [5](#0-4) [6](#0-5) 

**No rate limiting exists on the Terminal RPC endpoint.** The RPC server only enforces a batch request size limit (`rpc_batch_limit`), not a per-method or per-caller rate limit: [7](#0-6) 

The Terminal module is enabled by default in the production configuration: [8](#0-7) 

---

### Impact Explanation

The most critical contention point is `get_block_template` inside `get_tx_pool_info`. This call acquires the tx-pool lock and builds a full block template — the same operation the miner process calls via RPC to get work. Flooding `get_overview(31)` at high rate directly starves the miner's `get_block_template` calls, causing the miner to stall waiting for the lock, which delays block production and degrades consensus participation.

`System::new_all()` + `refresh_all()` (sysinfo) scans `/proc` on Linux on every call, consuming significant CPU and I/O. `estimate_num_keys_cf` issues a RocksDB property read. Together, 100 req/s of `get_overview(31)` forces 100 full sysinfo scans, 100 block template builds, 100 RocksDB estimations, and 100 snapshot clones per second — all on the node's main thread pool.

---

### Likelihood Explanation

The RPC defaults to `127.0.0.1:8114`, but:
1. The Terminal module is enabled by default in the production `ckb.toml`
2. Many operators expose the RPC publicly for dApps, explorers, and monitoring tools — the very use case the Terminal module targets
3. The exploit requires a single HTTP POST with `{"method":"get_overview","params":[31]}` in a loop — no authentication, no special privileges, no prior state [9](#0-8) 

---

### Recommendation

1. **Add a per-call minimum interval for forced refresh.** Track the last forced-refresh timestamp in `TerminalCache` and reject or silently downgrade `EVERYTHING` requests that arrive within a configurable cooldown (e.g., the shortest TTL, 2 seconds).
2. **Decouple `EVERYTHING` from individual flag checks.** `clear_all()` followed by individual flag checks that also force refresh is redundant and doubly expensive. Either `clear_all()` should be removed (rely on TTL expiry) or the individual flag checks should be skipped when `clear_all()` was just called.
3. **Add per-method rate limiting to the Terminal RPC.** Apply a `governor`-style rate limiter (as already used in the Relayer and HolePunching protocols) keyed by caller IP or connection.
4. **Remove `get_block_template` from the Terminal hot path.** The `committing` field in `TerminalPoolInfo` should be derived from a cheaper source than a full block template build.

---

### Proof of Concept

```bash
# Flood get_overview(EVERYTHING) at 100 req/s
while true; do
  curl -s -X POST http://127.0.0.1:8114 \
    -H 'Content-Type: application/json' \
    -d '{"jsonrpc":"2.0","method":"get_overview","params":[31],"id":1}' &
  sleep 0.01
done
```

Expected observable effects within seconds:
- CPU spikes from repeated `System::new_all()` + `/proc` scans
- Miner RPC `get_block_template` latency increases (lock contention)
- Block template generation stalls, reducing effective mining rate
- RocksDB I/O pressure from repeated `estimate_num_keys_cf` calls [10](#0-9)

    fn get_overview(&self, refresh: Option<u32>) -> Result<Overview> {
        let refresh = refresh
            .and_then(RefreshKind::from_bits)
            .unwrap_or(RefreshKind::NOTHING);

        // If refresh everything, clear cache first
        if refresh.contains(RefreshKind::EVERYTHING) {
            self.cache.clear_all();
        }

        let sys = self.get_sys_info(refresh)?;
        let mining = self.get_mining_info(refresh)?;
        let pool = self.get_tx_pool_info(refresh)?;
        let cells = self.get_cells_info(refresh)?;
        let network = self.get_network_info(refresh)?;

        Ok(Overview {
            sys,
            cells,
            mining,
            pool,
            network,
            version: self.network_controller.version().to_owned(),
        })
    }

    fn get_sys_info(&self, refresh: RefreshKind) -> Result<SysInfo> {
        // Check cache first unless force refresh
        if !refresh.contains(RefreshKind::SYSTEM_INFO)
            && let Some(cached) = self.cache.get_sys_info()
        {
            return Ok(cached);
        }

        // Fetch fresh system data
        let mut sys = System::new_all();
        sys.refresh_all();

        let total_memory = sys.total_memory();
        let used_memory = sys.used_memory();
        let global_cpu_usage = sys.global_cpu_usage();
        let sys_disks = SysDisks::new_with_refreshed_list();
        let disks = sys_disks
            .iter()
            .map(|disk| Disk {
                total_space: disk.total_space(),
                available_space: disk.available_space(),
                is_removable: disk.is_removable(),
            })
            .collect();
        let sys_networks = SysNetworks::new_with_refreshed_list();
        let networks = sys_networks
            .iter()
            .map(|(name, data)| Network {
                interface_name: name.clone(),
                received: data.received(),
                total_received: data.total_received(),
                transmitted: data.transmitted(),
                total_transmitted: data.total_transmitted(),
            })
            .collect();

        let global = Global {
            total_memory,
            used_memory,
            global_cpu_usage,
            disks,
            networks,
        };

        let process = sys
            .process(
                sysinfo::get_current_pid()
                    .map_err(|e| RPCError::custom(RPCError::CKBInternalError, e))?,
            )
            .ok_or_else(|| {
                RPCError::custom(RPCError::CKBInternalError, "failed to get current process")
            })?;

        let sys_disk_usage = process.disk_usage();
        let sys_info = SysInfo {
            global,
            cpu_usage: process.cpu_usage(),
            memory: process.memory(),
            disk_usage: DiskUsage {
                total_written_bytes: sys_disk_usage.total_written_bytes,
                written_bytes: sys_disk_usage.written_bytes,
                total_read_bytes: sys_disk_usage.total_read_bytes,
                read_bytes: sys_disk_usage.read_bytes,
            },
            virtual_memory: process.virtual_memory(),
        };

        // Cache the result
        self.cache.set_sys_info(sys_info.clone());
        Ok(sys_info)
    }

    fn get_tx_pool_info(&self, refresh: RefreshKind) -> Result<TerminalPoolInfo> {
        // Check cache first unless force refresh
        if !refresh.contains(RefreshKind::TX_POOL_INFO)
            && let Some(cached) = self.cache.get_tx_pool_info()
        {
            return Ok(cached);
        }

        // Fetch fresh transaction pool data
        let tx_pool = self.shared.tx_pool_controller();
        let get_tx_pool_info = tx_pool.get_tx_pool_info();
        if let Err(e) = get_tx_pool_info {
            error!("Send get_tx_pool_info request error {}", e);
            return Err(RPCError::ckb_internal_error(e));
        };

        let info = get_tx_pool_info.unwrap();

        let block_template = self
            .shared
            .get_block_template(None, None, None)
            .map_err(|err| {
                error!("Send get_block_template request error {}", err);
                RPCError::ckb_internal_error(err)
            })?
            .map_err(|err| {
                error!("Get_block_template result error {}", err);
                RPCError::from_any_error(err)
            })?;

        let total_recent_reject_num = tx_pool.get_total_recent_reject_num().map_err(|err| {
            error!("Get_total_recent_reject_num result error {}", err);
            RPCError::from_any_error(err)
        })?;

        let tx_pool_info = TerminalPoolInfo {
            pending: (info.pending_size as u64).into(),
            proposed: (info.proposed_size as u64).into(),
            orphan: (info.orphan_size as u64).into(),
            committing: (block_template.transactions.len() as u64).into(),
            total_recent_reject_num: total_recent_reject_num.unwrap_or(0).into(),
            total_tx_size: (info.total_tx_size as u64).into(),
            total_tx_cycles: info.total_tx_cycles.into(),
            last_txs_updated_at: info.last_txs_updated_at.into(),
            max_tx_pool_size: info.max_tx_pool_size.into(),
        };

        // Cache the result
        self.cache.set_tx_pool_info(tx_pool_info.clone());
        Ok(tx_pool_info)
    }
