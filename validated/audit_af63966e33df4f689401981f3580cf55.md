### Title
Unbounded Storage-Writing Loop in `_storePriceUpdates` Allows Permanent DoS of Subscriptions with Maximum Price Feeds — (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`Scheduler.sol` allows subscriptions with up to `MAX_PRICE_IDS_PER_SUBSCRIPTION = 255` price feeds. The `_storePriceUpdates` function writes a full `PriceFeed` struct to storage for every feed on every `updatePriceFeeds` call. With 255 feeds, the cumulative SSTORE cost approaches or exceeds Ethereum's block gas limit (~30M gas), permanently trapping the subscription. A secondary O(n²) nested loop in `_validateSubscriptionParams` and `_clearRemovedPriceUpdates` compounds the issue on subscription creation and update.

---

### Finding Description

**Root cause 1 — `_storePriceUpdates` (O(n) storage writes, called on every update):** [1](#0-0) 

On every `updatePriceFeeds` call, `_storePriceUpdates` iterates over all `priceFeeds` and writes each one to `_state.priceUpdates[subscriptionId][priceFeeds[i].id]`. A `PriceFeed` struct contains `id` (1 slot), `price` (2 packed slots), and `emaPrice` (2 packed slots) — approximately 5 storage slots per feed. With 255 feeds:

- Cold SSTORE: 255 × 5 × 20,000 gas = **25.5M gas** for storage writes alone
- Plus Pyth parsing, slot verification, deviation checks, and keeper fee logic → total easily exceeds the ~30M Ethereum block gas limit

The `updatePriceFeeds` function is the core keeper-callable entry point: [2](#0-1) 

**Root cause 2 — O(n²) nested loop in `_validateSubscriptionParams`:** [3](#0-2) 

Two nested loops check uniqueness of `priceIds` (up to 255) and `readerWhitelist` (up to 255). Each produces ~32,385 iterations. These are called on every `createSubscription` and `updateSubscription`.

**Root cause 3 — O(n²) nested loop with storage deletes in `_clearRemovedPriceUpdates`:** [4](#0-3) 

Called during `updateSubscription`, this function has nested O(n²) loops over `currentPriceIds` and `newPriceIds` (each up to 255), with `delete _state.priceUpdates[subscriptionId][oldPriceId]` inside the outer loop — storage-clearing operations that cost 5,000 gas each.

**The limits that enable this:** [5](#0-4) 

Both `MAX_PRICE_IDS_PER_SUBSCRIPTION` and `MAX_READER_WHITELIST_SIZE` are set to `255` (the maximum value of `uint8`).

---

### Impact Explanation

A subscription created with 255 price feeds becomes permanently un-updatable: `updatePriceFeeds` will always revert with out-of-gas because the storage write loop exceeds the block gas limit. The subscription owner's deposited ETH balance is locked in the contract with no mechanism to recover it (withdrawals require the subscription to be active, and deactivation does not refund). The keeper network cannot service the subscription regardless of gas price. This is a permanent, irreversible DoS of the subscription and its locked funds.

---

### Likelihood Explanation

Any unprivileged user can call `createSubscription` with `priceIds.length = 255`. The minimum balance check (`getMinimumBalance`) only requires `255 × minimumBalancePerFeed` wei, which is a small economic barrier. There is no protocol-level restriction preventing a legitimate user from creating a large subscription that inadvertently becomes un-updatable, nor preventing a griefing attacker from doing so deliberately. The `MAX_PRICE_IDS_PER_SUBSCRIPTION = 255` constant is explicitly designed to be the upper bound, so users are expected to use values near it.

---

### Recommendation

1. **Reduce `MAX_PRICE_IDS_PER_SUBSCRIPTION`** to a value that keeps `updatePriceFeeds` well within block gas limits (e.g., 50–100 feeds). Benchmark with Foundry's `--gas-report` flag at the maximum allowed value.
2. **Replace the O(n²) uniqueness checks** in `_validateSubscriptionParams` with a temporary mapping or sorted-array approach to reduce to O(n).
3. **Replace the O(n²) loop** in `_clearRemovedPriceUpdates` with a mapping-based lookup.
4. **Add a gas consumption test** that calls `updatePriceFeeds` with `MAX_PRICE_IDS_PER_SUBSCRIPTION` feeds and asserts the gas used is below a safe threshold (e.g., 15M gas).

---

    function _clearRemovedPriceUpdates(
        uint256 subscriptionId,
        bytes32[] storage currentPriceIds,
        bytes32[] memory newPriceIds
    ) internal returns (bool newPriceIdsAdded) {
        // Iterate through old price IDs
        for (uint i = 0; i < currentPriceIds.length; i++) {
            bytes32 oldPriceId = currentPriceIds[i];
            bool found = false;

            // Check if the old price ID exists in the new list
            for (uint j = 0; j < newPriceIds.length; j++) {
                if (newPriceIds[j] == oldPriceId) {
                    found = true;
                    break; // Found it, no need to check further
                }
            }

            // If not found in the new list, delete its stored update data
            if (!found) {
                delete _state.priceUpdates[subscriptionId][oldPriceId];
            }
        }

        // Check if any new price IDs were added
        for (uint i = 0; i < newPriceIds.length; i++) {
            bytes32 newPriceId = newPriceIds[i];
            bool found = false;

            // Check if the new price ID exists in the current list
            for (uint j = 0; j < currentPriceIds.length; j++) {
                if (currentPriceIds[j] == newPriceId) {
                    found = true;
                    break;
                }
            }

            // If a new price ID was added, mark as changed
            if (!found) {
                newPriceIdsAdded = true;
                break;
            }
        }

        return newPriceIdsAdded;

    function updatePriceFeeds(
        uint256 subscriptionId,
        bytes[] calldata updateData
    ) external override {
        uint256 startGas = gasleft();

        SchedulerStructs.SubscriptionStatus storage status = _state
            .subscriptionStatuses[subscriptionId];
        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        if (!params.isActive) {
            revert SchedulerErrors.InactiveSubscription();
        }

        // Get the Pyth contract and parse price updates
        IPyth pyth = IPyth(_state.pyth);
        uint256 pythFee = pyth.getUpdateFee(updateData);

        // If we don't have enough balance, revert
        if (status.balanceInWei < pythFee) {
            revert SchedulerErrors.InsufficientBalance();
        }

        // Parse the price feed updates with an acceptable timestamp range of [0, now+10s].
        // Note: We don't want to reject update data if it contains a price
        // from a market that closed a few days ago, since it will contain a timestamp
        // from the last trading period. Thus, we use a minimum timestamp of zero while parsing,
        // and we enforce the past max validity ourselves in _validateShouldUpdatePrices using
        // the highest timestamp in the update data.
        status.balanceInWei -= pythFee;
        status.totalSpent += pythFee;
        uint64 curTime = SafeCast.toUint64(block.timestamp);
        (
            PythStructs.PriceFeed[] memory priceFeeds,
            uint64[] memory slots
        ) = pyth.parsePriceFeedUpdatesWithConfig{value: pythFee}(
                updateData,
                params.priceIds,
                0, // We enforce the past max validity ourselves in _validateShouldUpdatePrices
                curTime + FUTURE_TIMESTAMP_MAX_VALIDITY_PERIOD,
                false,
                true,
                false
            );

        // Verify all price feeds have the same Pythnet slot.
        // All feeds in a subscription must be updated at the same time.
        uint64 slot = slots[0];
        for (uint8 i = 1; i < slots.length; i++) {
            if (slots[i] != slot) {
                revert SchedulerErrors.PriceSlotMismatch();
            }
        }

        // Verify that update conditions are met, and that the timestamp
        // is more recent than latest stored update's. Reverts if not.
        uint256 latestPublishTime = _validateShouldUpdatePrices(
            subscriptionId,
            params,
            status,
            priceFeeds
        );

        // Update status and store the updates
        status.priceLastUpdatedAt = latestPublishTime;
        status.totalUpdates += priceFeeds.length;

        _storePriceUpdates(subscriptionId, priceFeeds);

        _processFeesAndPayKeeper(status, startGas, params.priceIds.length);

        emit PricesUpdated(subscriptionId, latestPublishTime);
    }
