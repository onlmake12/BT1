### Title
Entire Passed `Coin<AptosCoin>` Deposited Instead of Only Required `update_fee` in `update_price_feeds` - (File: `target_chains/aptos/contracts/sources/pyth.move`)

---

### Summary

In `target_chains/aptos/contracts/sources/pyth.move`, the public function `update_price_feeds` accepts a `Coin<AptosCoin>` from the caller, computes the required fee (`update_fee`), asserts the coin is sufficient, but then deposits the **entire** coin into the Pyth contract — not just `update_fee`. Any excess value passed by the caller is permanently lost.

---

### Finding Description

The `update_price_feeds` function at line 170–180 of `target_chains/aptos/contracts/sources/pyth.move` follows this logic:

1. Iterates over each VAA and accumulates `total_updates` by calling `update_price_feed_from_single_vaa`, which returns the count of price updates **contained in** the accumulator message (line 290: `total_updates = vector::length(&result)`), regardless of how many are actually applied to the cache.
2. Computes `update_fee = state::get_base_update_fee() * total_updates` (line 177).
3. Asserts `update_fee <= coin::value(&fee)` — i.e., the passed coin is at least sufficient (line 178).
4. Calls `coin::deposit(@pyth, fee)` — depositing the **entire** coin, not just `update_fee` (line 179).

```move
public fun update_price_feeds(vaas: vector<vector<u8>>, fee: Coin<AptosCoin>) {
    let total_updates = 0;
    while (!vector::is_empty(&vaas)) {
        total_updates = total_updates + update_price_feed_from_single_vaa(vector::pop_back(&mut vaas));
    };
    let update_fee = state::get_base_update_fee() * total_updates;
    assert!(update_fee <= coin::value(&fee), error::insufficient_fee());
    coin::deposit(@pyth, fee);   // <-- entire coin taken, not just update_fee
}
```

The correct pattern would be to use `coin::split` to extract exactly `update_fee` from `fee`, deposit only that split coin, and return the remainder to the caller. The sibling entry function `update_price_feeds_with_funder` correctly uses `coin::withdraw<AptosCoin>(account, update_fee)` to take only the exact required amount (line 155), confirming the intended design.

There is also a secondary dimension: `update_price_feed_from_single_vaa` counts all price updates **present in** the accumulator message (line 290), while `update_cache` only applies those that pass `is_fresh_update` (line 311). So even `update_fee` itself may be higher than the fee for updates actually applied — analogous to the SeaportProxy charging based on the full order price rather than the partial fill. ---

### Impact Explanation

Any Move module that calls `update_price_feeds` and passes a `Coin<AptosCoin>` with value strictly greater than `update_fee` permanently loses the excess to the Pyth contract. In Move, `Coin<T>` is a linear resource — it cannot be silently dropped; the function must consume it entirely, which it does by depositing the whole amount. The caller has no way to recover the overpaid amount after the call. This is a direct, irreversible financial loss to the caller.

---

### Likelihood Explanation

The `update_price_feeds` function is a `public fun` callable by any Move module (not just entry functions). Callers that:
- Use a conservative fee estimate (e.g., rounding up),
- Reuse a pre-minted coin with a round denomination,
- Call `get_update_fee` on a slightly different VAA set than what they ultimately pass,

will silently overpay. The `update_price_feeds_with_funder` entry function is safe because it withdraws exactly `update_fee`, but any integrator using the `public fun` variant is exposed. This is a realistic scenario for any protocol built on Aptos that integrates Pyth price feeds.

---

### Recommendation

Replace the unconditional `coin::deposit` with a split-and-return pattern:

```move
public fun update_price_feeds(vaas: vector<vector<u8>>, fee: Coin<AptosCoin>) {
    let total_updates = 0;
    while (!vector::is_empty(&vaas)) {
        total_updates = total_updates + update_price_feed_from_single_vaa(vector::pop_back(&mut vaas));
    };
    let update_fee = state::get_base_update_fee() * total_updates;
    assert!(update_fee <= coin::value(&fee), error::insufficient_fee());
    // Extract only the required fee and return the remainder
    let required = coin::extract(&mut fee, update_fee);
    coin::deposit(@pyth, required);
    // Return excess to caller — caller must handle the remainder coin
    // (or use a signer-based approach to transfer back)
    coin::destroy_zero(fee); // only valid if remainder is zero; otherwise return it
}
```

The cleanest fix is to accept a `u64` amount and use `coin::withdraw` internally (as `update_price_feeds_with_funder` does), or to split the coin and return the excess to the caller's address.

---

### Proof of Concept

1. Governance sets `base_update_fee = 100` octas.
2. An accumulator VAA contains 3 price feed updates.
3. Caller computes `get_update_fee(&vaas) = 300` octas but passes a coin worth `500` octas (e.g., a pre-minted round-number coin).
4. `update_price_feeds` is called:
   - `total_updates = 3`, `update_fee = 300`
   - `assert!(300 <= 500)` passes
   - `coin::deposit(@pyth, fee)` deposits **500** octas
5. Caller loses **200** octas with no recourse.

The excess is silently absorbed by the Pyth contract, with no event or return value indicating the overcharge. [5](#0-4)

    fun update_price_feed_from_single_vaa(vaa: vector<u8>): u64 {
        let cur = cursor::init(vaa);
        let header: u64 = deserialize::deserialize_u32(&mut cur);
        let total_updates;
        let updates = if (header == PYTHNET_ACCUMULATOR_UPDATE_MAGIC) {
            let result = parse_and_verify_accumulator_message(&mut cur);
            total_updates = vector::length(&result);
            result
        }
        else {
            // Deserialize the VAA
            let vaa = vaa::parse_and_verify(vaa);
            // Check that the VAA is from a valid data source (emitter)
            verify_data_source(&vaa);
            // Deserialize the batch price attestation
            total_updates = 1;
            batch_price_attestation::destroy(batch_price_attestation::deserialize(vaa::destroy(vaa)))
        };
        update_cache(updates);
        cursor::rest(cur);
        total_updates
    }

    /// Update the cache with given price updates, if they are newer than the ones currently cached.
    public(friend) fun update_cache(updates: vector<PriceInfo>) {
        while (!vector::is_empty(&updates)) {
            let update = vector::pop_back(&mut updates);
            if (is_fresh_update(&update)) {
                let price_feed = *price_info::get_price_feed(&update);
                let price_identifier = price_feed::get_price_identifier(&price_feed);
                state::set_latest_price_info(
                    *price_identifier,
                    update,
                );
                event::emit_price_feed_update(price_feed, timestamp::now_microseconds());
            }
        };
        vector::destroy_empty(updates);
    }
