### Title
Expired Alert Replay via Missing Deduplication in `clear_expired_alerts` — (`util/network-alert/src/notifier.rs`, `util/network-alert/src/alert_relayer.rs`)

---

### Summary

The CKB network alert system uses `has_received(alert_id)` as its replay-prevention gate before accepting and re-broadcasting a signed alert. However, `clear_expired_alerts()` removes expired alerts from `received_alerts` without recording their IDs in the permanent `cancel_filter`. After expiry cleanup runs, `has_received()` returns `false` for those IDs, and any P2P peer can replay the original signed alert bytes — which still carry valid Nervos Foundation signatures — causing every receiving node to re-broadcast the alert network-wide and re-notify operators.

---

### Finding Description

**Root cause — `clear_expired_alerts` breaks the deduplication invariant**

`Notifier::has_received` is the sole replay gate:

```rust
// util/network-alert/src/notifier.rs  line 147-149
pub fn has_received(&self, id: u32) -> bool {
    self.received_alerts.contains_key(&id) || self.cancel_filter.contains(&id)
}
```

An alert ID is considered "seen" if it is in `received_alerts` (a plain `HashMap`) **or** in `cancel_filter` (an `LruCache<u32, ()>` of size 128). [1](#0-0) 

`clear_expired_alerts` removes expired entries from `received_alerts` but **never inserts them into `cancel_filter`**:

```rust
// util/network-alert/src/notifier.rs  line 135-144
pub fn clear_expired_alerts(&mut self, now: u64) {
    self.received_alerts.retain(|_id, alert| {
        let notice_until: u64 = alert.raw().notice_until().into();
        notice_until > now
    });
    self.noticed_alerts.retain(|a| { ... });
}
``` [2](#0-1) 

After this runs, `has_received(expired_id)` returns `false`. The `cancel_filter` only receives IDs via the explicit `cancel()` path:

```rust
// util/network-alert/src/notifier.rs  line 125-132
pub fn cancel(&mut self, cancel_id: u32) {
    self.cancel_filter.put(cancel_id, ());
    ...
}
``` [3](#0-2) 

**Trigger path in `AlertRelayer::received`**

The P2P handler checks `has_received` before verifying signatures:

```rust
// util/network-alert/src/alert_relayer.rs  line 147-162
if self.notifier.lock().has_received(alert_id) {
    return;
}
// verify
if let Err(err) = self.verifier.verify_signatures(&alert) { ... }
``` [4](#0-3) 

There is **no check** that `notice_until >= now` in the `received()` handler. [5](#0-4) 

`clear_expired_alerts` is called only inside `connected()`, i.e., whenever a new peer connects:

```rust
// util/network-alert/src/alert_relayer.rs  line 81-95
async fn connected(...) {
    self.clear_expired_alerts();
    ...
}
``` [6](#0-5) 

**End-to-end exploit flow**

1. Nervos Foundation broadcasts a legitimate alert (ID=X, `notice_until=T`, valid signatures). Every node receives and stores it in `received_alerts`.
2. Time advances past `T`. The alert is expired.
3. A new peer connects to a target node, triggering `clear_expired_alerts()`. Alert X is removed from `received_alerts`. `has_received(X)` now returns `false`.
4. The attacker (any P2P peer — no keys required) sends the original alert bytes for ID=X to the target node.
5. `received()`: `has_received(X)` → `false` → passes the gate.
6. `verify_signatures()` passes — the Foundation's signatures are still cryptographically valid.
7. Alert X is re-inserted into `received_alerts`, re-broadcast to all connected peers, and `notify_network_alert` fires again. [7](#0-6) 
8. Steps 3–7 can be repeated indefinitely after each expiry cycle.

The `noticed_alerts.contains(alert)` check at line 112 does not prevent re-notification because `noticed_alerts` was also cleared by `clear_expired_alerts`. [8](#0-7) 

---

### Impact Explanation

Any unprivileged P2P peer can replay any previously broadcast, now-expired alert to the entire CKB network. Each replayed alert:

- Re-triggers `notify_network_alert` on every receiving node, causing node operators to see a stale or misleading critical warning.
- Re-propagates across the full P2P network (the node re-broadcasts to all connected peers that haven't seen the ID in their `known_lists`).
- Can be repeated indefinitely — each expiry cycle resets the deduplication state.

The alert system is specifically designed for critical security notifications ("critical bugs"). Replaying a past alert that warned of a now-patched vulnerability, or a cancelled emergency, can cause operators to take unnecessary emergency actions (halting nodes, rolling back, etc.), constituting unauthorized state manipulation of the network's operator layer.

---

### Likelihood Explanation

- **Attacker preconditions**: None beyond being a P2P peer on the CKB network (open, permissionless).
- **Alert bytes are public**: Once broadcast, any observer captures the raw signed alert bytes.
- **Trigger is automatic**: Any peer connection event causes `clear_expired_alerts`, resetting the deduplication state.
- **No cryptographic work required**: The attacker reuses existing valid signatures verbatim.

Likelihood is **Medium** — requires waiting for an alert to expire and a peer connection event, but both are routine and predictable.

---

### Recommendation

In `clear_expired_alerts`, insert each expired alert's ID into `cancel_filter` before removing it from `received_alerts`:

```rust
pub fn clear_expired_alerts(&mut self, now: u64) {
    let expired_ids: Vec<u32> = self.received_alerts
        .iter()
        .filter_map(|(id, alert)| {
            let notice_until: u64 = alert.raw().notice_until().into();
            if notice_until <= now { Some(*id) } else { None }
        })
        .collect();
    for id in expired_ids {
        self.cancel_filter.put(id, ());  // <-- preserve deduplication
        self.received_alerts.remove(&id);
    }
    self.noticed_alerts.retain(|a| {
        let notice_until: u64 = a.raw().notice_until().into();
        notice_until > now
    });
}
```

Additionally, add an expiry check in `AlertRelayer::received()` to reject alerts whose `notice_until` is already in the past, as a defense-in-depth measure.

### Proof of Concept

```
1. Observe a legitimate alert on the P2P network:
   Alert { id: 42, notice_until: T, signatures: [sig1, sig2], ... }
   Save the raw bytes.

2. Wait until current_time > T (alert expires).

3. Connect a new peer to any CKB node (triggers clear_expired_alerts,
   removing id=42 from received_alerts; has_received(42) → false).

4. Send the saved raw alert bytes to the node via the Alert P2P protocol.

5. Observe:
   - Node accepts the alert (has_received check passes, signatures valid).
   - Node re-broadcasts to all connected peers.
   - notify_network_alert fires; operators see the stale alert again.

6. Repeat from step 3 indefinitely.
```

        &mut self,
        nc: Arc<dyn CKBProtocolContext + Sync>,
        peer_index: PeerIndex,
        data: Bytes,
    ) {
        let alert: packed::Alert = match packed::AlertReader::from_slice(&data) {
            Ok(alert) => {
                if alert.raw().message().is_utf8()
                    && alert
                        .raw()
                        .min_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                    && alert
                        .raw()
                        .max_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                {
                    alert.to_entity()
                } else {
                    info!(
                        "A malformed message fromP peer {} : not utf-8 string",
                        peer_index
                    );
                    nc.ban_peer(
                        peer_index,
                        BAD_MESSAGE_BAN_TIME,
                        String::from("send us a malformed message: not utf-8 string"),
                    );
                    return;
                }
            }
            Err(err) => {
                info!("A malformed message from peer {}: {:?}", peer_index, err);
                nc.ban_peer(
                    peer_index,
                    BAD_MESSAGE_BAN_TIME,
                    String::from("send us a malformed message"),
                );
                return;
            }
        };
        let alert_id = alert.as_reader().raw().id().into();
        trace!("ReceiveD alert {} from peer {}", alert_id, peer_index);
        // ignore alert
        if self.notifier.lock().has_received(alert_id) {
            return;
        }
        // verify
        if let Err(err) = self.verifier.verify_signatures(&alert) {
            debug!(
                "An alert from peer {} with invalid signatures, error {:?}",
                peer_index, err
            );
            nc.ban_peer(
                peer_index,
                BAD_MESSAGE_BAN_TIME,
                String::from("send us an alert with invalid signatures"),
            );
            return;
        }
        // mark sender as known
        self.mark_as_known(peer_index, alert_id);
        // broadcast message
        let selected_peers: Vec<PeerIndex> = nc
            .connected_peers()
            .into_iter()
            .filter(|peer| self.mark_as_known(*peer, alert_id))
            .collect();
        if let Err(err) = nc.quick_filter_broadcast(
            TargetSession::Multi(Box::new(selected_peers.into_iter())),
            data,
        ) {
            debug!("alert broadcast error: {:?}", err);
        }
        // add to received alerts
        self.notifier.lock().add(&alert);
    }
