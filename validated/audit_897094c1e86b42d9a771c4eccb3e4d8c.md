### Title
Out-of-Bounds Panic in `TxConfirmStat::remove_unconfirmed_tx` When `tx_age` Exceeds `MAX_CONFIRM_BLOCKS` — (`util/fee-estimator/src/estimator/confirmation_fraction.rs`)

---

### Summary

`TxConfirmStat::remove_unconfirmed_tx` guards the `block_unconfirmed_txs` array access with a bounds check (`tx_age >= self.block_unconfirmed_txs.len()`), but the immediately following `confirm_blocks_to_failed_txs[tx_age - 1]` access is **outside that guard**. When a tracked transaction has been in the pool for more than `MAX_CONFIRM_BLOCKS` (1000) blocks and is then rejected, `tx_age - 1 >= 1000` causes a Rust index-out-of-bounds **panic**, crashing the node.

---

### Finding Description

`TxConfirmStat` is initialized with three parallel vectors, each of length `MAX_CONFIRM_BLOCKS = 1000`:

- `block_unconfirmed_txs` (length 1000)
- `confirm_blocks_to_confirmed_txs` (length 1000)
- `confirm_blocks_to_failed_txs` (length 1000) [1](#0-0) 

In `remove_unconfirmed_tx`, the code correctly guards the `block_unconfirmed_txs` access:

```rust
if tx_age >= self.block_unconfirmed_txs.len() {
    self.bucket_stats[bucket_index].old_unconfirmed_txs -= 1;
} else {
    let block_index = (entry_height % self.block_unconfirmed_txs.len() as u64) as usize;
    self.block_unconfirmed_txs[block_index][bucket_index] -= 1;
}
if count_failure {
    self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64; // ← NO BOUNDS CHECK
}
``` [2](#0-1) 

The `count_failure` branch is **unconditional** — it executes regardless of whether `tx_age >= MAX_CONFIRM_BLOCKS`. When `tx_age > 1000`, `tx_age - 1 >= 1000`, and `self.confirm_blocks_to_failed_txs[tx_age - 1]` panics with an index-out-of-bounds error.

`count_failure = true` is set by `drop_tx`, which is called by `reject_tx`: [3](#0-2) [4](#0-3) 

`reject_tx` is the only path that sets `count_failure = true`; `process_block_tx` uses `drop_tx_inner(tx_hash, false)` and is safe. [5](#0-4) 

---

### Impact Explanation

When the `ConfirmationFraction` fee estimator is active and a tracked transaction is rejected after residing in the tx-pool for more than 1000 blocks, the node **panics and crashes**. This is a remote denial-of-service: the crash is deterministic and reproducible on any node running this estimator configuration.

---

### Likelihood Explanation

The attacker is an unprivileged tx-pool submitter. The steps are:

1. Submit a valid low-fee-rate transaction via the `send_transaction` RPC. The fee estimator tracks it at the current tip height H.
2. Wait for 1001+ blocks to be mined (the tx remains in the pool because its fee rate is too low to be naturally confirmed, or the attacker simply waits).
3. The transaction is rejected — either naturally (pool eviction due to capacity limits) or via the `remove_transaction` RPC call.
4. `reject_tx` → `drop_tx` → `remove_unconfirmed_tx(entry_height=H, tip_height=H+1001, ..., count_failure=true)` → `tx_age = 1001` → `confirm_blocks_to_failed_txs[1000]` → **panic**.

No privileged access is required. The attacker only needs to submit one transaction and wait.

---

### Recommendation

Add an explicit bounds check before accessing `confirm_blocks_to_failed_txs`:

```rust
if count_failure {
    if tx_age - 1 < self.confirm_blocks_to_failed_txs.len() {
        self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64;
    }
}
```

This mirrors the existing guard on `block_unconfirmed_txs` and is consistent with how `add_confirmed_tx` already silently ignores out-of-range `blocks_to_confirm` values via the range `(blocks_to_confirm - 1)..self.max_confirms()`. [6](#0-5) 

---

### Proof of Concept

```
1. Node is running with ConfirmationFraction fee estimator enabled.
2. Attacker calls send_transaction RPC with a valid tx at tip height H=100.
   → track_tx records entry_height=100, bucket_index=X in tracked_txs.
3. Chain advances to height H=1102 (1002 blocks later).
   → move_track_window runs 1002 times; tx moves to old_unconfirmed_txs
     but remains in tracked_txs.
4. Attacker calls remove_transaction RPC (or pool evicts the tx).
   → reject_tx(tx_hash) → drop_tx(tx_hash) → drop_tx_inner(tx_hash, true)
   → remove_unconfirmed_tx(entry_height=100, tip_height=1102, bucket_index=X, count_failure=true)
   → tx_age = 1102 - 100 = 1002
   → tx_age (1002) >= block_unconfirmed_txs.len() (1000): old_unconfirmed_txs branch taken ✓
   → count_failure=true:
       self.confirm_blocks_to_failed_txs[1001][X] += 1.0
       ^^^ index 1001 >= len 1000 → PANIC: index out of bounds
5. Node process terminates.
```
