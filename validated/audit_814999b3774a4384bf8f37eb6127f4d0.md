### Title
No Minimum TWAP Window Enforcement in `validateTwapPriceInfo()` Allows Near-Spot-Price Manipulation - (File: `target_chains/ethereum/contracts/contracts/pyth/Pyth.sol`)

---

### Summary

The `validateTwapPriceInfo()` function in `Pyth.sol` and `validate_twap_messages()` in the Solana receiver enforce only that `startSlot < endSlot` (a difference of at least 1 slot, ~400 ms on Pythnet). There is no minimum window size check. An unprivileged caller can supply two Wormhole-verified TWAP messages that are only 1 slot apart, producing a result that is functionally equivalent to a spot price and is susceptible to the same flash-loan / single-block price manipulation that TWAP is designed to prevent.

---

### Finding Description

`parseTwapPriceFeedUpdates()` in `Pyth.sol` delegates all temporal validation to `validateTwapPriceInfo()`:

```solidity
// Pyth.sol L586-610
function validateTwapPriceInfo(
    PythStructs.TwapPriceInfo memory twapPriceInfoStart,
    PythStructs.TwapPriceInfo memory twapPriceInfoEnd
) private pure {
    if (twapPriceInfoStart.prevPublishTime >= twapPriceInfoStart.publishTime)
        revert PythErrors.InvalidTwapUpdateData();
    if (twapPriceInfoEnd.prevPublishTime >= twapPriceInfoEnd.publishTime)
        revert PythErrors.InvalidTwapUpdateData();

    if (twapPriceInfoStart.expo != twapPriceInfoEnd.expo)
        revert PythErrors.InvalidTwapUpdateDataSet();
    if (twapPriceInfoStart.publishSlot > twapPriceInfoEnd.publishSlot)   // ← only > not >=
        revert PythErrors.InvalidTwapUpdateDataSet();
    if (twapPriceInfoStart.publishTime > twapPriceInfoEnd.publishTime)
        revert PythErrors.InvalidTwapUpdateDataSet();
}
``` [1](#0-0) 

The slot check uses strict `>`, so `startSlot + 1 == endSlot` passes. `calculateTwap()` then divides by `slotDiff = 1`:

```solidity
uint64 slotDiff = twapPriceInfoEnd.publishSlot - twapPriceInfoStart.publishSlot; // = 1
int128 twapPrice = priceDiff / int128(uint128(slotDiff));                         // = single-slot price
``` [2](#0-1) 

The identical gap exists on Solana. `validate_twap_messages()` only requires `end_msg.publish_slot > start_msg.publish_slot` with no minimum distance:

```rust
require!(
    end_msg.publish_slot > start_msg.publish_slot,
    ReceiverError::InvalidTwapSlots
);
``` [3](#0-2) 

The Solana SDK's `get_twap_no_older_than()` does enforce an exact `window_seconds` match, but this is a consumer-side helper — the core `post_twap_update` instruction itself imposes no minimum:

```rust
pub fn get_twap_no_older_than(
    &self, clock: &Clock, maximum_age: u64, window_seconds: u64, feed_id: &FeedId,
) -> std::result::Result<TwapPrice, GetPriceError> {
    let actual_window = twap_price.end_time.saturating_sub(twap_price.start_time);
    check!(actual_window == i64::try_from(window_seconds).unwrap(), GetPriceError::InvalidWindowSize);
    ...
}
``` [4](#0-3) 

The EVM `parseTwapPriceFeedUpdates()` has no equivalent enforcement at all — it returns the `TwapPriceFeed` with `startTime`/`endTime` populated but never validates the window width. [5](#0-4) 

---

### Impact Explanation

A 1-slot TWAP window on Pythnet (~400 ms) is functionally a spot price. Any downstream EVM protocol that calls `parseTwapPriceFeedUpdates()` and trusts the returned price without independently checking `endTime - startTime` receives a value that can be manipulated within a single Pythnet slot. This defeats the core manipulation-resistance property of TWAP and can lead to incorrect liquidations, mispriced collateral, or drained lending pools in integrating protocols.

---

### Likelihood Explanation

Hermes continuously publishes signed TWAP accumulator messages. An unprivileged caller can freely select any two Wormhole-verified messages from the Hermes API — including two that are only 1 slot apart — and submit them to `parseTwapPriceFeedUpdates()`. No privileged access is required. An attacker with sufficient capital can manipulate the Pythnet aggregate price for a single slot (e.g., via a coordinated large trade on a venue that a Pyth publisher tracks) and then immediately use the resulting 1-slot "TWAP" in a downstream protocol. The cost of the attack scales with the depth of the manipulated market, not with any protocol-level window floor.

---

### Recommendation

Add a minimum slot-difference (and/or time-difference) check inside `validateTwapPriceInfo()` on EVM and `validate_twap_messages()` on Solana. A reasonable floor is on the order of several hundred slots (≈ 1–2 minutes on Pythnet). For example, in `validateTwapPriceInfo()`:

```solidity
uint64 constant MIN_TWAP_SLOTS = 300; // ~2 minutes on Pythnet

if (twapPriceInfoEnd.publishSlot - twapPriceInfoStart.publishSlot < MIN_TWAP_SLOTS) {
    revert PythErrors.InvalidTwapWindowTooShort();
}
```

The Solana `validate_twap_messages()` should receive an analogous `require!(slot_diff >= MIN_TWAP_SLOTS, ...)` guard.

---

### Proof of Concept

1. Attacker queries Hermes for two consecutive `TwapMessage` accumulator updates for a target feed, selecting `start` at slot `N` and `end` at slot `N+1`.
2. Attacker calls `parseTwapPriceFeedUpdates(updateData, priceIds)` on the EVM Pyth contract with these two updates.
3. `validateTwapPriceInfo` passes: `prevPublishTime < publishTime` for both, `expo` matches, `startSlot (N) > endSlot (N+1)` is false → no revert.
4. `calculateTwap` computes `slotDiff = 1`, returning `priceDiff / 1` — the raw single-slot cumulative price delta, i.e., the spot price at slot `N`.
5. The returned `TwapPriceFeed.twap.price` equals the spot price at slot `N`, which the attacker manipulated moments before by executing a large trade on a Pyth-tracked venue.
6. Attacker uses this manipulated "TWAP" price in a downstream lending/derivatives protocol to extract value. [6](#0-5) [7](#0-6)

        bytes[] calldata updateData,
        bytes32[] calldata priceIds
    )
        external
        payable
        override
        returns (PythStructs.TwapPriceFeed[] memory twapPriceFeeds)
    {
        // TWAP requires exactly 2 updates: one for the start point and one for the end point
        if (updateData.length != 2) {
            revert PythErrors.InvalidUpdateData();
        }

        uint requiredFee = getTwapUpdateFee(updateData);
        if (msg.value < requiredFee) revert PythErrors.InsufficientFee();

        // Process start update data
        PythStructs.TwapPriceInfo[] memory startTwapPriceInfos;
        bytes32[] memory startPriceIds;
        {
            uint offsetStart;
            (
                offsetStart,
                startTwapPriceInfos,
                startPriceIds
            ) = extractTwapPriceInfos(updateData[0]);
        }

        // Process end update data
        PythStructs.TwapPriceInfo[] memory endTwapPriceInfos;
        bytes32[] memory endPriceIds;
        {
            uint offsetEnd;
            (offsetEnd, endTwapPriceInfos, endPriceIds) = extractTwapPriceInfos(
                updateData[1]
            );
        }

        // Verify that we have the same number of price feeds in start and end updates
        if (startPriceIds.length != endPriceIds.length) {
            revert PythErrors.InvalidTwapUpdateDataSet();
        }

        // Hermes always returns price feeds in the same order for start and end updates
        // This allows us to assume startPriceIds[i] == endPriceIds[i] for efficiency
        for (uint i = 0; i < startPriceIds.length; i++) {
            if (startPriceIds[i] != endPriceIds[i]) {
                revert PythErrors.InvalidTwapUpdateDataSet();
            }
        }

        // Initialize the output array
        twapPriceFeeds = new PythStructs.TwapPriceFeed[](priceIds.length);

        // For each requested price ID, find matching start and end data points
        for (uint i = 0; i < priceIds.length; i++) {
            bytes32 requestedPriceId = priceIds[i];
            int startIdx = -1;

            // Find the index of this price ID in the startPriceIds array
            // (which is the same as the endPriceIds array based on our validation above)
            for (uint j = 0; j < startPriceIds.length; j++) {
                if (startPriceIds[j] == requestedPriceId) {
                    startIdx = int(j);
                    break;
                }
            }

            // If we found the price ID
            if (startIdx >= 0) {
                uint idx = uint(startIdx);
                // Validate the pair of price infos
                validateTwapPriceInfo(
                    startTwapPriceInfos[idx],
                    endTwapPriceInfos[idx]
                );

                // Calculate TWAP from these data points
                twapPriceFeeds[i] = calculateTwap(
                    requestedPriceId,
                    startTwapPriceInfos[idx],
                    endTwapPriceInfos[idx]
                );
            }
        }

        // Ensure all requested price IDs were found
        for (uint k = 0; k < priceIds.length; k++) {
            if (twapPriceFeeds[k].id == 0) {
                revert PythErrors.PriceFeedNotFoundWithinRange();
            }
        }
    }

            revert PythErrors.InvalidTwapUpdateDataSet();
        }
        if (twapPriceInfoStart.publishTime > twapPriceInfoEnd.publishTime) {
            revert PythErrors.InvalidTwapUpdateDataSet();

```text
    function calculateTwap(
        bytes32 priceId,
        PythStructs.TwapPriceInfo memory twapPriceInfoStart,
        PythStructs.TwapPriceInfo memory twapPriceInfoEnd
    ) private pure returns (PythStructs.TwapPriceFeed memory twapPriceFeed) {
        twapPriceFeed.id = priceId;
        twapPriceFeed.startTime = twapPriceInfoStart.publishTime;
        twapPriceFeed.endTime = twapPriceInfoEnd.publishTime;

        // Calculate differences between start and end points for slots and cumulative values
        uint64 slotDiff = twapPriceInfoEnd.publishSlot -
            twapPriceInfoStart.publishSlot;
        int128 priceDiff = twapPriceInfoEnd.cumulativePrice -
            twapPriceInfoStart.cumulativePrice;
        uint128 confDiff = twapPriceInfoEnd.cumulativeConf -
            twapPriceInfoStart.cumulativeConf;

        // Calculate time-weighted average price (TWAP) and confidence by dividing
        // the difference in cumulative values by the number of slots between data points
        int128 twapPrice = priceDiff / int128(uint128(slotDiff));
        uint128 twapConf = confDiff / uint128(slotDiff);

        // The conversion from int128 to int64 is safe because:
        // 1. Individual prices fit within int64 by protocol design
        // 2. TWAP is essentially an average price over time (cumulativePrice₂-cumulativePrice₁)/slotDiff
        // 3. This average must be within the range of individual prices that went into the calculation
        // We use int128 only as an intermediate type to safely handle cumulative sums
        twapPriceFeed.twap.price = int64(twapPrice);
        twapPriceFeed.twap.conf = uint64(twapConf);
        twapPriceFeed.twap.expo = twapPriceInfoStart.expo;
        twapPriceFeed.twap.publishTime = twapPriceInfoEnd.publishTime;

        // Calculate downSlotsRatio as a value between 0 and 1,000,000
        // 0 means no slots were missed, 1,000,000 means all slots were missed
        uint64 totalDownSlots = twapPriceInfoEnd.numDownSlots -
            twapPriceInfoStart.numDownSlots;
        uint64 downSlotsRatio = (totalDownSlots * 1_000_000) / slotDiff;

        // Safely downcast to uint32 (sufficient for value range 0-1,000,000)
        twapPriceFeed.downSlotsRatio = uint32(downSlotsRatio);

        return twapPriceFeed;
    }

    pub fn get_twap_no_older_than(
        &self,
        clock: &Clock,
        maximum_age: u64,
        window_seconds: u64,
        feed_id: &FeedId,
    ) -> std::result::Result<TwapPrice, GetPriceError> {
        // Ensure the update isn't outdated
        let twap_price = self.get_twap_unchecked(feed_id)?;
        check!(
            twap_price
                .end_time
                .saturating_add(maximum_age.try_into().unwrap())
                >= clock.unix_timestamp,
            GetPriceError::PriceTooOld
        );

        // Ensure the twap window size is as expected
        let actual_window = twap_price.end_time.saturating_sub(twap_price.start_time);
        check!(
            actual_window == i64::try_from(window_seconds).unwrap(),
            GetPriceError::InvalidWindowSize
        );

        Ok(twap_price)
    }
