### Title
Proxy Credentials Exposed in Plaintext via Error Log Messages — (File: `network/src/proxy.rs`)

---

### Summary

The `check_proxy_url` function in `network/src/proxy.rs` includes the raw `proxy_url` string verbatim in error messages during config validation. Because `proxy_url` is documented to support the format `socks5://username:password@host:port`, any validation failure emits the full URL — including embedded credentials — into CKB's log output. Additionally, `ProxyConfig` derives `Debug` with no credential redaction, meaning any `{:?}` log of the config struct also leaks credentials.

---

### Finding Description

`ProxyConfig` in `util/app-config/src/configs/network.rs` holds a `proxy_url` field explicitly documented to accept embedded credentials:

```rust
// like: socks5://username:password@127.0.0.1:1080
pub proxy_url: Option<String>,
``` [1](#0-0) 

The struct derives `Debug` with no custom implementation to redact the credential-bearing field.

`check_proxy_url` in `network/src/proxy.rs` validates this URL at startup. On any validation failure it formats the raw URL directly into the returned error string:

```rust
if parsed_url.host_str().is_none() {
    return Err(format!("missing host in proxy url: {}", proxy_url));
}
...
if parsed_url.port().is_none() {
    return Err(format!("missing port in proxy url: {}", proxy_url));
}
``` [2](#0-1) 

These `Err(String)` values propagate up through the network initialization path and are emitted to CKB's structured log files. A user who configures `proxy_url = "socks5://alice:secret@proxy.example.com"` with a missing port will have `secret` written verbatim into the node's log.

---

### Impact Explanation

Any local user or process with read access to CKB's log directory (e.g., `data/logs/`) can recover the proxy username and password in plaintext. The proxy credential can then be used to authenticate to the SOCKS5 proxy independently, bypassing the intended access control on the proxy server. This is a direct analog to the external report: a secret intended to be opaque is written to a persistent, OS-readable artifact (log file instead of shell history).

---

### Likelihood Explanation

The exposure is triggered whenever `check_proxy_url` returns an error — i.e., whenever a user configures a `proxy_url` that is syntactically incomplete (missing port, missing host, or wrong scheme). This is a realistic misconfiguration scenario during initial setup. The `Debug` derive on `ProxyConfig` provides a second, always-present exposure path if any code path logs the config struct with `{:?}`.

---

### Recommendation

**Short term:** Sanitize the URL before embedding it in error messages. Strip the `username:password@` component before formatting:

```rust
// Replace credentials with *** before logging
let sanitized = redact_url_credentials(proxy_url);
return Err(format!("missing port in proxy url: {}", sanitized));
```

**Long term:** Implement a custom `Debug` (and `Display`) for `ProxyConfig` that replaces the credential portion of `proxy_url` with `***`. This prevents any future `{:?}` log site from leaking credentials.

---

### Proof of Concept

1. In `ckb.toml`, set:
   ```toml
   [network.proxy]
   proxy_url = "socks5://alice:hunter2@127.0.0.1"
   ```
   (port intentionally omitted)

2. Start the CKB node: `ckb run`

3. `check_proxy_url` is called with the raw string and returns:
   ```
   Err("missing port in proxy url: socks5://alice:hunter2@127.0.0.1")
   ``` [3](#0-2) 

4. This error string propagates through the network initialization path and is written to `data/logs/run.log`.

5. Any OS user with read access to the log directory recovers `hunter2` in plaintext — directly analogous to an attacker reading shell history to find `--password="mypassword"` in the external report.

pub(crate) fn check_proxy_url(proxy_url: &str) -> Result<(), String> {
    let parsed_url = Url::parse(proxy_url).map_err(|e| e.to_string())?;
    if parsed_url.host_str().is_none() {
        return Err(format!("missing host in proxy url: {}", proxy_url));
    }
    let scheme = parsed_url.scheme();
    if scheme.ne("socks5") {
        return Err(format!("CKB doesn't support proxy scheme: {}", scheme));
    }
    if parsed_url.port().is_none() {
        return Err(format!("missing port in proxy url: {}", proxy_url));
    }
    Ok(())
}
```
