### Title
Provider Can DOS User Requests via Fee Frontrunning - (`target_chains/ethereum/contracts/contracts/entropy/Entropy.sol`)

### Summary
A registered Entropy provider can repeatedly cause user `requestV2`/`requestWithCallback` transactions to revert by frontrunning them with a `setProviderFee()` call that raises the fee above the user's `msg.value`. The provider can then restore the original fee and repeat the attack indefinitely, permanently blocking any user from successfully requesting randomness from that provider.

### Finding Description
In `Entropy.sol`, the `requestHelper` function computes the required fee at execution time and reverts if `msg.value` is insufficient:

```solidity
uint128 requiredFee = getFeeV2(provider, callbackGasLimit);
if (msg.value < requiredFee) revert EntropyErrors.InsufficientFee();
``` [1](#0-0) 

`getFeeV2(provider, gasLimit)` returns `getProviderFee(provider, gasLimit) + _state.pythFeeInWei`, where `getProviderFee` reads `provider.feeInWei` directly from storage: [2](#0-1) 

The `feeInWei` field is mutable at any time by the provider via `setProviderFee()`, which has no time-lock, rate limit, or minimum notice period:

```solidity
function setProviderFee(uint128 newFeeInWei) external override {
    ...
    provider.feeInWei = newFeeInWei;
    ...
}
``` [3](#0-2) 

A user who queries `getFeeV2(provider)` off-chain and submits a request with exactly that amount is vulnerable: the provider can observe the pending transaction in the mempool, call `setProviderFee(currentFee + 1)` with higher gas to land first, and the user's transaction will revert with `InsufficientFee`. The provider then restores the original fee and repeats.

The same attack is available to the provider's designated fee manager via `setProviderFeeAsFeeManager()`: [4](#0-3) 

### Impact Explanation
Any registered Entropy provider can permanently prevent any user from successfully calling `request`, `requestWithCallback`, or `requestV2` against that provider. Consumer contracts that have hardcoded a specific provider (e.g., the default provider) and pass `msg.value = getFeeV2()` are especially vulnerable because they cannot easily switch providers. The attack costs only the gas for repeated `setProviderFee` calls and is indefinitely repeatable.

### Likelihood Explanation
Any permissionlessly registered provider can execute this attack. Registration requires no trust or vetting. The attack is cheap (one `setProviderFee` call per blocked request), requires no special knowledge beyond watching the mempool, and is directly executable on any EVM chain where Entropy is deployed. Consumer contracts that follow the documented pattern of computing `getFeeV2()` on-chain immediately before calling `requestV2{value: fee}()` are still vulnerable if the provider frontruns within the same block.

### Recommendation
1. **Time-lock fee increases**: Require that fee increases only take effect after a delay (e.g., 256 blocks), while fee decreases can take effect immediately. This eliminates the frontrunning window.
2. **Accept a user-specified maximum fee**: Allow callers to pass a `maxFee` parameter; the contract accepts the request as long as `msg.value >= requiredFee && requiredFee <= maxFee`, so a fee increase beyond the user's tolerance causes a clean revert rather than a silent DOS.
3. **Alternatively**: Cap the rate at which a provider can increase their fee per unit time.

### Proof of Concept
1. Provider is registered with `feeInWei = 100 wei`; `pythFeeInWei = 50 wei`; `getFeeV2(provider) = 150 wei`.
2. User (or consumer contract) queries `getFeeV2(provider)` → `150 wei`, then submits `requestV2{value: 150}(provider, userRandomNumber, 0)`.
3. Provider observes the pending transaction in the mempool and submits `setProviderFee(101)` with higher gas priority.
4. Provider's `setProviderFee` lands first; `getFeeV2(provider)` is now `151 wei`.
5. User's `requestV2` executes: `requiredFee = 151`, `msg.value = 150` → `150 < 151` → `revert InsufficientFee`.
6. Provider calls `setProviderFee(100)` to restore the original fee.
7. Steps 2–6 repeat indefinitely; the user can never successfully request randomness from this provider.

    }

    function getProviderFee(
        address providerAddr,
        uint32 gasLimit
    ) internal view returns (uint128 feeAmount) {
        EntropyStructsV2.ProviderInfo memory provider = _state.providers[
            providerAddr
        ];

        // Providers charge a minimum of their configured feeInWei for every request.
        // Requests using more than the defaultGasLimit get a proportionally scaled fee.
        // This approach may be somewhat simplistic, but it allows us to continue using the
        // existing feeInWei parameter for the callback failure flow instead of defining new
        // configuration values.
        uint32 roundedGasLimit = uint32(roundTo10kGas(gasLimit)) * TEN_THOUSAND;
        if (
            provider.defaultGasLimit > 0 &&
            roundedGasLimit > provider.defaultGasLimit
        ) {
            // This calculation rounds down the fee, which means that users can get some gas in the callback for free.
            // However, the value of the free gas is < 1 wei, which is insignificant.
            uint128 additionalFee = ((roundedGasLimit -
                provider.defaultGasLimit) * provider.feeInWei) /
                provider.defaultGasLimit;
            return provider.feeInWei + additionalFee;
        } else {
            return provider.feeInWei;
        }
    }
```

    function setProviderFee(uint128 newFeeInWei) external override {
        EntropyStructsV2.ProviderInfo storage provider = _state.providers[
            msg.sender
        ];

        if (provider.sequenceNumber == 0) {
            revert EntropyErrors.NoSuchProvider();
        }
        uint128 oldFeeInWei = provider.feeInWei;
        provider.feeInWei = newFeeInWei;
        emit ProviderFeeUpdated(msg.sender, oldFeeInWei, newFeeInWei);
        emit EntropyEventsV2.ProviderFeeUpdated(
            msg.sender,
            oldFeeInWei,
            newFeeInWei,
            bytes("")
        );
    }

    function setProviderFeeAsFeeManager(
        address provider,
        uint128 newFeeInWei
    ) external override {
        EntropyStructsV2.ProviderInfo storage providerInfo = _state.providers[
            provider
        ];

        if (providerInfo.sequenceNumber == 0) {
            revert EntropyErrors.NoSuchProvider();
        }

        if (providerInfo.feeManager != msg.sender) {
            revert EntropyErrors.Unauthorized();
        }

        uint128 oldFeeInWei = providerInfo.feeInWei;
        providerInfo.feeInWei = newFeeInWei;

        emit ProviderFeeUpdated(provider, oldFeeInWei, newFeeInWei);
        emit EntropyEventsV2.ProviderFeeUpdated(
            provider,
            oldFeeInWei,
            newFeeInWei,
            bytes("")
        );
    }
