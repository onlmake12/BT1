### Title
Governance Sequence Gap Allows Permanent Skip of Any Governance Action — (`lazer/contracts/sui/sources/governance.move`)

### Summary

`governance::process_incoming` uses a strictly-less-than comparison (`seen_sequence < sequence`) instead of requiring the exact next sequence number. Because `update_trusted_signer` (and `upgrade`) are `public fun` callable by any unprivileged actor, a relayer can submit a higher-sequence VAA first, advancing `seen_sequence` past any lower-sequence VAA and making it permanently unprocessable.

### Finding Description

In `governance::process_incoming`, the only replay/ordering guard is:

```move
assert!(self.seen_sequence < sequence, EOldSequenceNumber);
self.seen_sequence = sequence;
``` [1](#0-0) 

This accepts **any** sequence number strictly greater than the last seen value — it does not require `sequence == seen_sequence + 1`. Sequence numbers between the old and new value are silently skipped and can never be submitted afterward (they will fail the same `<` check in reverse).

The entry point is fully public:

```move
public fun update_trusted_signer(state: &mut State, vaa: VAA) {
``` [2](#0-1) 

No capability, admin key, or privileged role is required to call it. Any actor who holds a valid Wormhole-verified VAA can invoke it.

### Impact Explanation

The concrete impact is a **permanently skipped signer-removal governance action**:

1. Governance emits VAA at sequence N (payload: remove compromised/expired signer, `expires_at = 0`).
2. Governance later emits VAA at sequence N+2 (any unrelated action).
3. Attacker submits VAA N+2 first → `seen_sequence` advances to N+2.
4. VAA N is now permanently rejected with `EOldSequenceNumber`.
5. The compromised signer remains in `state.trusted_signers` indefinitely.

Because `update_trusted_signer` with `expires_at == 0` is the only on-chain removal path: [3](#0-2) 

…a skipped removal leaves the signer permanently trusted. Any price update signed by that key will continue to be accepted as authentic by all consumers of the Lazer contract.

### Likelihood Explanation

- Both VAAs are publicly available from the Wormhole guardian network immediately after signing.
- The attacker needs no privileged access — only the ability to submit a Sui transaction.
- The window to execute the attack is the time between VAA N+2 becoming available and the legitimate relayer processing VAA N. Given that the off-chain relayer (`executeGovernanceProposals`) iterates sequentially and may have latency, this window is realistic. [4](#0-3) 

### Recommendation

Replace the weak ordering check with a strict consecutive-sequence requirement:

```move
// Before (vulnerable):
assert!(self.seen_sequence < sequence, EOldSequenceNumber);

// After (fixed):
assert!(self.seen_sequence + 1 == sequence, EOldSequenceNumber);
``` [1](#0-0) 

This ensures every governance sequence number is processed exactly once and in order, matching the invariant that every approved governance action must be executed.

### Proof of Concept

```move
#[test]
public fun test_sequence_skip_skips_removal() {
    let mut ctx = tx_context::dummy();
    let mut state = state::new_for_test(&mut ctx, test_governance());
    let current_cap = state.current_cap();

    // Pre-condition: signer is trusted
    state.update_trusted_signer(&current_cap, TEST_TRUSTED_SIGNER, 1799422709);
    assert_eq!(state.trusted_signers(&current_cap).length(), 1);

    // Attacker submits seq=4 (unrelated add, higher sequence) BEFORE seq=3 (removal)
    // TEST_UPDATE_TRUSTED_SIGNER_ADD has sequence=2, TEST_UPDATE_TRUSTED_SIGNER_REMOVE has sequence=4
    // Swap order: submit the higher-sequence VAA first
    let (_, vaa_high) = vaa::parse_test_only(TEST_UPDATE_TRUSTED_SIGNER_ADD); // seq=2
    let (_, vaa_low)  = vaa::parse_test_only(TEST_UPDATE_TRUSTED_SIGNER_REMOVE); // seq=4

    // Submit high-seq first — advances seen_sequence past the removal VAA
    update_trusted_signer(&mut state, vaa_high);

    // Now submit the removal VAA — must abort with EOldSequenceNumber
    // (assert it is rejected)
    update_trusted_signer(&mut state, vaa_low); // expected: aborts

    // Signer is still present — removal was permanently skipped
    assert_eq!(state.trusted_signers(&current_cap).length(), 1);

    destroy(state);
}
```
