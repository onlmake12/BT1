### Title
Missing Minimum Fee Validation in `registerProvider()` and `setProviderFee()` Allows Zero-Fee Provider Registration, Permanently Locking User Funds — (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

`Echo.sol`'s `registerProvider()` and `setProviderFee()` functions accept all-zero fee values (`baseFeeInWei=0`, `feePerFeedInWei=0`, `feePerGasInWei=0`) with no minimum validation. When a provider registers with all-zero fees, `getFee()` returns only the Pyth protocol fee (`pythFeeInWei`), so `req.fee` is stored as 0. In `executeCallback`, the executor earns `(req.fee + msg.value) - pythFee = 0`, meaning no rational actor will ever execute the callback. User requests to a zero-fee provider are permanently stuck and user funds are locked in the contract.

---

### Finding Description

`registerProvider()` stores the three fee components directly with no lower-bound check:

```solidity
function registerProvider(
    uint96 baseFeeInWei,
    uint96 feePerFeedInWei,
    uint96 feePerGasInWei
) external override {
    ProviderInfo storage provider = _state.providers[msg.sender];
    require(!provider.isRegistered, "Provider already registered");
    provider.baseFeeInWei = baseFeeInWei;       // can be 0
    provider.feePerFeedInWei = feePerFeedInWei; // can be 0
    provider.feePerGasInWei = feePerGasInWei;   // can be 0
    provider.isRegistered = true;
    ...
}
``` [1](#0-0) 

`setProviderFee()` has the same absence of minimum validation: [2](#0-1) 

`getFee()` computes the total fee as the sum of all components. With all provider fees at 0, it returns only `_state.pythFeeInWei`: [3](#0-2) 

In `requestPriceUpdatesWithCallback`, `req.fee` is stored as `msg.value - pythFeeInWei`. When provider fees are 0, `msg.value == pythFeeInWei`, so `req.fee = 0`: [4](#0-3) 

In `executeCallback`, the executor is credited `(req.fee + msg.value) - pythFee`. With `req.fee = 0` and `msg.value` covering only the Pyth update fee, the executor earns 0 while spending gas and the Pyth fee out of pocket: [5](#0-4) 

The code itself acknowledges there is no penalty mechanism for providers who never execute callbacks: [6](#0-5) 

---

### Impact Explanation

A malicious actor registers as a provider with all-zero fees. Users are attracted by the apparent low cost (paying only the Pyth protocol fee). After accumulating requests:

1. **Permanently stuck requests**: No rational actor will call `executeCallback` because the executor earns 0 while spending gas and the Pyth update fee. Requests are never fulfilled.
2. **Locked user funds**: The `pythFeeInWei` paid per request is locked in the contract with no path to refund.
3. **Exclusivity period amplification**: During the exclusivity window (default 15 seconds), only the assigned zero-fee provider can execute the callback, blocking any other actor from stepping in even voluntarily.
4. **No penalty for non-execution**: The contract has no slashing or penalty mechanism for providers who register but never fulfill requests.

---

### Likelihood Explanation

- `registerProvider()` is permissionless — any address can register as a provider with zero fees.
- The attack requires no privileged access, no leaked keys, and no governance majority.
- Users have no on-chain signal distinguishing a malicious zero-fee provider from a legitimate one.
- The attack is economically rational for the attacker: zero cost to register, zero cost to attract requests, and user funds are locked with no recourse.

---

### Recommendation

Add a minimum fee floor in both `registerProvider()` and `setProviderFee()`. At minimum, require that the sum of all provider fee components exceeds a protocol-defined constant (e.g., `MIN_PROVIDER_FEE`):

```solidity
uint96 constant MIN_PROVIDER_FEE = /* governance-set value */;

function registerProvider(
    uint96 baseFeeInWei,
    uint96 feePerFeedInWei,
    uint96 feePerGasInWei
) external override {
+   require(
+       baseFeeInWei + feePerFeedInWei + feePerGasInWei >= MIN_PROVIDER_FEE,
+       "Provider fee below minimum"
+   );
    ...
}
```

Apply the same check in `setProviderFee()`. Additionally, implement a penalty/slashing mechanism for providers who fail to execute callbacks within a defined window, as the existing TODO comment already acknowledges.

### Proof of Concept

```solidity
function testZeroFeeProviderLocksUserFunds() public {
    address maliciousProvider = address(0xdead);

    // Step 1: Register provider with all-zero fees (no revert)
    vm.prank(maliciousProvider);
    echo.registerProvider(0, 0, 0);

    // Step 2: User makes a request — pays only pythFeeInWei
    bytes32[] memory priceIds = new bytes32[](1);
    priceIds[0] = bytes32(uint256(1));
    uint96 fee = echo.getFee(maliciousProvider, 100_000, priceIds);
    // fee == pythFeeInWei only; req.fee stored as 0

    vm.deal(address(consumer), fee);
    vm.prank(address(consumer));
    uint64 seq = echo.requestPriceUpdatesWithCallback{value: fee}(
        maliciousProvider,
        SafeCast.toUint64(block.timestamp),
        priceIds,
        100_000
    );

    // Step 3: Verify req.fee == 0 — no incentive for anyone to execute
    EchoState.Request memory req = echo.getRequest(seq);
    assertEq(req.fee, 0); // executor earns 0, request permanently stuck
}

    function getFee(
        address provider,
        uint32 callbackGasLimit,
        bytes32[] calldata priceIds
    ) public view override returns (uint96 feeAmount) {
        uint96 baseFee = _state.pythFeeInWei; // Fixed fee to Pyth
        // Note: The provider needs to set its fees to include the fee charged by the Pyth contract.
        // Ideally, we would be able to automatically compute the pyth fees from the priceIds, but the
        // fee computation on IPyth assumes it has the full updated data.
        uint96 providerBaseFee = _state.providers[provider].baseFeeInWei;
        uint96 providerFeedFee = SafeCast.toUint96(
            priceIds.length * _state.providers[provider].feePerFeedInWei
        );
        uint96 providerFeeInWei = _state.providers[provider].feePerGasInWei; // Provider's per-gas rate
        uint256 gasFee = callbackGasLimit * providerFeeInWei; // Total provider fee based on gas
        feeAmount =
            baseFee +
            providerBaseFee +
            providerFeedFee +
            SafeCast.toUint96(gasFee); // Total fee user needs to pay
    }

    ) external override {
        require(
            _state.providers[provider].isRegistered,
            "Provider not registered"
        );
        require(
            msg.sender == provider ||
                msg.sender == _state.providers[provider].feeManager,
            "Only provider or fee manager can invoke this method"
        );

        uint96 oldBaseFee = _state.providers[provider].baseFeeInWei;

        _state.providers[provider].feePerGasInWei = newFeePerGasInWei;

            oldBaseFee,
            oldFeePerFeed,
