### Title
Whitelisted Readers Are Permanently DoS'd From `getPricesNoOlderThan` / `getEmaPricesNoOlderThan` When `whitelistEnabled` Is True Due to `msg.sender` Context Shift in `this.` External Call — (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` each apply the `onlyWhitelistedReader` modifier to the outer call, then delegate to `this.getPricesUnsafe` / `this.getEmaPricesUnsafe` via an **external** `this.` call. That external call resets `msg.sender` to `address(this)` (the Scheduler contract). The `onlyWhitelistedReader` modifier inside `getPricesUnsafe` / `getEmaPricesUnsafe` then evaluates the Scheduler contract as the caller — which is neither the subscription manager nor a member of `readerWhitelist` — and reverts with `Unauthorized`. The result is that **every** caller, including legitimately whitelisted readers, is permanently blocked from using these two functions whenever `whitelistEnabled` is `true`.

---

### Finding Description

`getPricesNoOlderThan` (line 554) and `getEmaPricesNoOlderThan` (line 597) both delegate to the price-fetch helpers via an external `this.` call:

```solidity
// Scheduler.sol line 554
prices = this.getPricesUnsafe(subscriptionId, priceIds);

// Scheduler.sol line 597
prices = this.getEmaPricesUnsafe(subscriptionId, priceIds);
```

`getPricesUnsafe` and `getEmaPricesUnsafe` are decorated with `onlyWhitelistedReader`:

```solidity
// Scheduler.sol lines 514-522
function getPricesUnsafe(
    uint256 subscriptionId,
    bytes32[] calldata priceIds
)
    external
    view
    override
    onlyWhitelistedReader(subscriptionId)   // <-- re-evaluated with msg.sender = address(this)
    returns (PythStructs.Price[] memory prices)
```

The modifier logic is:

```solidity
// Scheduler.sol lines 750-779
modifier onlyWhitelistedReader(uint256 subscriptionId) {
    if (_state.subscriptionManager[subscriptionId] == msg.sender) { _; return; }
    if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) { _; return; }
    // whitelist membership check against msg.sender ...
    if (!isWhitelisted) { revert SchedulerErrors.Unauthorized(); }
    _;
}
```

When `whitelistEnabled` is `true`:

1. The outer `getPricesNoOlderThan` call passes `onlyWhitelistedReader` because the original `msg.sender` is whitelisted.
2. `this.getPricesUnsafe(...)` is dispatched as an **external** call; `msg.sender` inside `getPricesUnsafe` becomes `address(this)` (the Scheduler contract).
3. `onlyWhitelistedReader` re-runs: the Scheduler contract is not the subscription manager and is not in `readerWhitelist`.
4. The call reverts with `Unauthorized` — for **every** caller, including the subscription manager and all whitelisted readers.

The outer `onlyWhitelistedReader` check on `getPricesNoOlderThan` / `getEmaPricesNoOlderThan` is therefore incomplete: it validates the original caller but the inner `this.` call introduces a second, unintended access-control evaluation against a context (`address(this)`) that can never satisfy the whitelist condition.

---

### Impact Explanation

Any subscription that sets `whitelistEnabled = true` loses access to `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` entirely. These are the only functions that enforce a freshness guarantee (`age_seconds`). Protocols that rely on staleness-bounded price reads — the primary safety feature distinguishing these functions from `getPricesUnsafe` — are silently forced to either use the unchecked `getPricesUnsafe` or receive a hard revert on every call. This constitutes a permanent, deterministic denial-of-service for a core read path whenever the whitelist feature is active.

---

### Likelihood Explanation

The bug is triggered by the ordinary, documented use of `whitelistEnabled = true`. Any subscription manager who enables the whitelist to restrict readers will immediately and reproducibly hit this condition. No adversarial input is required; the failure is deterministic and affects 100% of whitelisted subscriptions that call these two functions.

---

### Recommendation

Replace the external `this.` delegation with a direct call to the internal helper `_getPricesInternal`, which already exists and performs the same data retrieval without re-entering the access-control modifier:

```solidity
// getPricesNoOlderThan — replace:
prices = this.getPricesUnsafe(subscriptionId, priceIds);
// with:
PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
prices = new PythStructs.Price[](priceFeeds.length);
for (uint i = 0; i < priceFeeds.length; i++) { prices[i] = priceFeeds[i].price; }

// getEmaPricesNoOlderThan — replace:
prices = this.getEmaPricesUnsafe(subscriptionId, priceIds);
// with:
PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
prices = new PythStructs.Price[](priceFeeds.length);
for (uint i = 0; i < priceFeeds.length; i++) { prices[i] = priceFeeds[i].emaPrice; }
```

This eliminates the `msg.sender` context shift and ensures the whitelist check is evaluated exactly once, against the original external caller.

---

### Proof of Concept

1. Deploy `SchedulerUpgradeable`. Create a subscription with `whitelistEnabled = true` and `readerWhitelist = [alice]`.
2. Fund the subscription and push a price update via `updatePriceFeeds`.
3. Call `getPricesNoOlderThan(subscriptionId, priceIds, 3600)` as `alice` (a whitelisted reader).
4. Observe revert: `SchedulerErrors.Unauthorized` — emitted from inside `getPricesUnsafe` where `msg.sender == address(scheduler)`.
5. Confirm `getPricesUnsafe` called directly by `alice` succeeds, proving the whitelist entry is valid and the failure is solely due to the `this.` call context shift.

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
