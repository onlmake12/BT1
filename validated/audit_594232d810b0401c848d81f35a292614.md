Audit Report

## Title
Out-of-Bounds Index Panic in `TxConfirmStat::remove_unconfirmed_tx` Crashes Tx-Pool Service Thread — (`util/fee-estimator/src/estimator/confirmation_fraction.rs`)

## Summary
`TxConfirmStat::remove_unconfirmed_tx` guards the `block_unconfirmed_txs` access against `tx_age >= 1000` but applies no equivalent guard to the immediately following `confirm_blocks_to_failed_txs[tx_age - 1]` access. When a tracked transaction ages past 1000 blocks and is then rejected from the tx-pool, the index `tx_age - 1 >= 1000` exceeds the vector's length of 1000, causing a Rust index-out-of-bounds panic that crashes the tx-pool service thread.

## Finding Description
`confirm_blocks_to_failed_txs` and `block_unconfirmed_txs` are both allocated with length `MAX_CONFIRM_BLOCKS = 1000`: [1](#0-0) 

In `remove_unconfirmed_tx`, the guard at line 208 redirects the `block_unconfirmed_txs` access to `old_unconfirmed_txs` when `tx_age >= 1000`, but the `count_failure` branch at lines 214–215 has no corresponding bounds check: [2](#0-1) 

When `tx_age = 1001`, `tx_age - 1 = 1000` is out of bounds for a 1000-element vector (valid indices 0–999), causing a panic.

**Confirmed call chain:**
1. `accept_tx` → `track_tx` records `entry_height = current_tip` (only when `height == self.best_height`).
2. 1001+ blocks pass; `best_height` advances via `commit_block`.
3. Tx is evicted → `register_reject` callback fires → `fee_estimator.reject_tx(&tx_hash)`: [3](#0-2) 

4. `reject_tx` (line 475) → `drop_tx(count_failure=true)` (line 429) → `drop_tx_inner` → `remove_unconfirmed_tx(..., true)`: [4](#0-3) 

5. `tx_age = best_height - entry_height >= 1001` → `confirm_blocks_to_failed_txs[1000]` → **panic**.

The guard at line 208 is insufficient because it only protects `block_unconfirmed_txs`; `confirm_blocks_to_failed_txs` shares the same length but has no guard.

## Impact Explanation
A Rust index-out-of-bounds panic unwinds and terminates the tx-pool service thread. The tx-pool is a critical node component; its crash renders the node unable to accept or relay transactions. This matches the allowed CKB bounty impact: **High (10001–15000 points) — Vulnerabilities which could easily crash a CKB node**. The `ConfirmationFraction` estimator must be explicitly configured (it is not the default `Dummy` estimator), but any node operator who enables it is exposed. [5](#0-4) 

## Likelihood Explanation
The attack requires only a valid transaction submission and patience. The default tx expiry of 12 hours makes the 1001-block threshold (≈2.2 hours at 8 s/block) easily reachable in normal operation — a low-fee tx that is never proposed will naturally age past 1000 blocks without deliberate attack. The attack is repeatable: after the node restarts, a new tx can be submitted to trigger the crash again. No special privileges, PoW, or key material are required.

## Recommendation
Add a bounds check before the `confirm_blocks_to_failed_txs` access in `remove_unconfirmed_tx`, mirroring the existing guard for `block_unconfirmed_txs`:

```rust
if count_failure && tx_age - 1 < self.confirm_blocks_to_failed_txs.len() {
    self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64;
}
```

This silently drops failure samples for txs older than the tracking window, which is the correct semantic (they are already counted as `old_unconfirmed_txs`).

## Proof of Concept

```rust
#[test]
fn test_remove_unconfirmed_tx_oob_panic() {
    let mut stat = TxConfirmStat::default(); // MAX_CONFIRM_BLOCKS = 1000
    let entry_height: u64 = 0;
    let bucket_index = stat.add_unconfirmed_tx(entry_height, FeeRate::from_u64(1000)).unwrap();
    // tx_age = 1001 - 0 = 1001; tx_age - 1 = 1000 >= confirm_blocks_to_failed_txs.len() (1000)
    stat.remove_unconfirmed_tx(entry_height, 1001, bucket_index, true); // panics: index out of bounds
}

    fn remove_unconfirmed_tx(
        &mut self,
        entry_height: u64,
        tip_height: u64,
        bucket_index: usize,
        count_failure: bool,
    ) {
        let tx_age = tip_height.saturating_sub(entry_height) as usize;
        if tx_age < 1 {
            return;
        }
        if tx_age >= self.block_unconfirmed_txs.len() {
            self.bucket_stats[bucket_index].old_unconfirmed_txs -= 1;
        } else {
            let block_index = (entry_height % self.block_unconfirmed_txs.len() as u64) as usize;
            self.block_unconfirmed_txs[block_index][bucket_index] -= 1;
        }
        if count_failure {
            self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64;
        }
    }
