### Title
Excess `msg.value` Not Refunded to Caller in `Echo.requestPriceUpdatesWithCallback` — (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

In `Echo.requestPriceUpdatesWithCallback`, any `msg.value` exceeding the required fee is silently absorbed into the provider's accrued fee balance rather than refunded to the caller. This is a direct, permanent loss of funds for the requester.

---

### Finding Description

`requestPriceUpdatesWithCallback` computes the required fee via `getFee(provider, callbackGasLimit, priceIds)` and enforces a minimum:

```solidity
uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
if (msg.value < requiredFee) revert InsufficientFee();
``` [1](#0-0) 

It then stores the provider's portion as:

```solidity
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);
``` [2](#0-1) 

If `msg.value > requiredFee`, the excess is silently folded into `req.fee`. When `executeCallback` is later called, the provider is credited:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
``` [3](#0-2) 

The entire excess above `requiredFee` flows into the provider's withdrawable balance. There is no refund path to the original requester.

The `IEcho` interface NatSpec says `msg.value must be equal to getFee(callbackGasLimit)`, but does not warn that any overpayment is permanently forfeited to the provider: [4](#0-3) 

This contrasts with `Entropy.sol`, which at least explicitly documents the same behavior: *"Note that excess value is not refunded to the caller."* [5](#0-4) 

Echo has no such disclosure.

---

### Impact Explanation

Any ETH sent above `requiredFee` is permanently credited to the provider's `accruedFeesInWei` and is withdrawable by the provider. The requester has no mechanism to recover the overpayment. This is a direct, unrecoverable loss of user funds.

---

### Likelihood Explanation

Moderate. Callers can query `getFee()` to compute the exact amount, but in practice:
- Integrating contracts often add a small buffer to `msg.value` to guard against fee increases between estimation and execution.
- Wallets and SDKs may round up or add slippage.
- The `getFee()` result includes `callbackGasLimit * feePerGasInWei`, which is an estimate; callers may overshoot intentionally.

Any such overpayment is silently lost to the provider with no on-chain warning.

---

### Recommendation

After enforcing the minimum fee, refund any excess to `msg.sender`:

```solidity
uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
if (msg.value < requiredFee) revert InsufficientFee();

// Refund excess
if (msg.value > requiredFee) {
    (bool ok, ) = payable(msg.sender).call{value: msg.value - requiredFee}("");
    require(ok, "Refund failed");
}

// Store only the required provider portion
req.fee = SafeCast.toUint96(requiredFee - _state.pythFeeInWei);
```

---

### Proof of Concept

1. Provider registers with `baseFeeInWei = 0.001 ether`, `feePerGasInWei = 1 gwei`, `feePerFeedInWei = 0`.
2. `getFee(provider, 500_000, priceIds)` returns `X wei`.
3. Caller sends `msg.value = X + 1 ether` (e.g., a contract that adds a 1 ETH buffer).
4. `req.fee = X + 1 ether - pythFeeInWei` — the 1 ETH excess is stored in `req.fee`.
5. Provider calls `executeCallback`; provider's `accruedFeesInWei` increases by `req.fee - pythFee`, which includes the full 1 ETH excess.
6. Provider calls `withdrawAsFeeManager` and withdraws the 1 ETH.
7. The original requester has permanently lost 1 ETH with no recourse. [6](#0-5)

            _state.providers[provider].isRegistered,

        // FIXME: this comment is wrong. (we're not using tx.gasprice)

        //      are low, forcing executors to fulfill them later when gas prices might be much higher.
        //      Since tx.gasprice is used to calculate fees, allowing far-future requests would make
        //      the fee estimation unreliable.
        require(publishTime <= block.timestamp + 60, "Too far in future");
        if (priceIds.length > MAX_PRICE_IDS) {
            revert TooManyPriceIds(priceIds.length, MAX_PRICE_IDS);
        }
        requestSequenceNumber = _state.currentSequenceNumber++;

        uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
        if (msg.value < requiredFee) revert InsufficientFee();

        Request storage req = allocRequest(requestSequenceNumber);
        req.sequenceNumber = requestSequenceNumber;
        req.publishTime = publishTime;
        req.callbackGasLimit = callbackGasLimit;
        req.requester = msg.sender;
        req.provider = provider;
        req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);

        // Create array with the right size

            // Extract first 8 bytes of the price ID

                prefix := priceId
            }
            req.priceIdPrefixes[i] = prefix;

    }
