### Title
Cancelled Alert Re-Activation via `cancel_filter` LRU Eviction — (`File: util/network-alert/src/notifier.rs`)

### Summary

The `Notifier` in CKB's network alert system tracks cancelled alert IDs in a bounded LRU cache (`cancel_filter`, size 128). Once this cache overflows, evicted cancelled-alert IDs are forgotten. Any P2P peer that retained the original signed alert bytes can replay them, bypassing the deduplication guard and causing a previously cancelled alert to be re-accepted, re-stored, and re-broadcast across the entire network.

### Finding Description

**Root cause — bounded `cancel_filter` LRU:**

The `Notifier` struct uses a fixed-size LRU cache to remember cancelled alert IDs: [1](#0-0) 

```
const CANCEL_FILTER_SIZE: usize = 128;

pub struct Notifier {
    cancel_filter: LruCache<u32, ()>,   // ← bounded at 128 entries
    received_alerts: HashMap<u32, Alert>,
    ...
}
```

When alert ID `X` is cancelled, `cancel()` removes it from `received_alerts` and inserts it into `cancel_filter`: [2](#0-1) 

The deduplication guard in `has_received` checks both stores: [3](#0-2) 

Once 128 additional alert IDs are cancelled, the LRU evicts the oldest entry. At that point:
- `X` is **not** in `received_alerts` (removed by `cancel()`)
- `X` is **not** in `cancel_filter` (evicted by LRU)
- `has_received(X)` returns **false**

**Exploit path in `AlertRelayer::received`:**

The P2P handler checks `has_received` *before* signature verification: [4](#0-3) 

After eviction, a peer replaying the original signed bytes for alert `X` passes the `has_received` guard (returns false), passes signature verification (the original Nervos Foundation signatures are still cryptographically valid), and the alert is re-added to `received_alerts` and `noticed_alerts`, then re-broadcast to all connected peers.

**`clear_expired_alerts` does not populate `cancel_filter`:** [5](#0-4) 

Expired alerts are silently dropped from `received_alerts` without being recorded in `cancel_filter`, creating a second path: any expired alert whose ID was never explicitly cancelled can be replayed via P2P after expiry-clearing, since `has_received` will return false.

**No `notice_until` check in the P2P handler:**

The RPC `send_alert` rejects past-expiry alerts, but the P2P `received` handler has no such check: [6](#0-5) [7](#0-6) 

The P2P path only validates UTF-8 and signatures — not expiry.

**When a new peer connects, stored alerts are re-broadcast without re-verification:** [8](#0-7) 

Once a re-activated alert is in `received_alerts`, it propagates to every newly connecting peer automatically.

### Impact Explanation

An unprivileged P2P peer who stored the raw bytes of a previously cancelled or expired alert can replay them after the `cancel_filter` LRU overflows (128 cancellations over the network's lifetime) or after `clear_expired_alerts` removes the alert. The re-activated alert is re-broadcast to all connected peers and re-displayed to node operators via `get_blockchain_info().alerts`. This allows stale, cancelled, or false-alarm alerts to re-appear network-wide, potentially causing node operators to take unnecessary emergency action (e.g., halting operations, emergency upgrades) based on revoked information. The `noticed_alerts` list, which is the operator-facing output, will show the re-activated alert as active.

### Likelihood Explanation

The `cancel_filter` overflow requires 128 cancellations over the node's lifetime — a realistic threshold for a long-running mainnet node given that the alert system has already been used (e.g., alert `20230001` is present in the test suite). Any peer that connected to the network during the original alert broadcast will have retained the signed bytes. No privileged keys are needed to replay; the attacker only needs the original alert bytes, which were broadcast to all nodes. The expired-alert replay path requires no overflow at all — only waiting for `clear_expired_alerts` to run (triggered on every new peer connection).

### Recommendation

1. Replace the bounded `LruCache` for `cancel_filter` with an unbounded `HashSet<u32>`, or use a persistent store, so cancellations are never forgotten.
2. Add a `notice_until` expiry check in the P2P `received` handler (mirroring the RPC check) to reject replayed expired alerts.
3. When `clear_expired_alerts` removes an alert from `received_alerts`, insert its ID into `cancel_filter` (or equivalent permanent store) to prevent replay.

### Proof of Concept

```
1. Nervos Foundation sends Alert(id=1, ...) — broadcast to all nodes.
   All nodes: received_alerts={1}, cancel_filter={}

2. Nervos Foundation sends Alert(id=2, cancel=1) — cancels alert 1.
   All nodes: received_alerts={2}, cancel_filter={1}

3. Over the network's lifetime, 127 more alerts are cancelled
   (ids 3..129 each with cancel fields).
   cancel_filter now holds 128 entries; entry for id=1 is evicted by LRU.
   All nodes: cancel_filter={2,3,...,129} (id=1 evicted)

4. Attacker (any P2P peer) replays the original raw bytes of Alert(id=1).
   - has_received(1) → false  (not in received_alerts, not in cancel_filter)
   - verify_signatures(alert_1) → Ok  (original Nervos Foundation sigs still valid)
   - alert_1 added back to received_alerts and noticed_alerts
   - alert_1 broadcast to all connected peers

5. All nodes now display the cancelled alert as active in get_blockchain_info().alerts.
``` [9](#0-8) [10](#0-9) [11](#0-10) [12](#0-11)

    pub fn add(&mut self, alert: &Alert) {
        let alert_id = alert.raw().id().into();
        let alert_cancel = alert.raw().cancel().into();
        if self.has_received(alert_id) {
            return;
        }
        // checkout cancel_id
        if alert_cancel > 0 {
            self.cancel(alert_cancel);
        }
        // add to received alerts
        self.received_alerts.insert(alert_id, alert.clone());

        // check conditions, figure out do we need to notice this alert
        if !self.is_version_effective(alert) {
            debug!("Received a version ineffective alert {:?}", alert);
            return;
        }

        if self.noticed_alerts.contains(alert) {
            return;
        }
        self.notify_controller.notify_network_alert(alert.clone());
        self.noticed_alerts.push(alert.clone());
        // sort by priority
        self.noticed_alerts.sort_by_key(|a| {
            let priority: u32 = a.raw().priority().into();
            u32::MAX - priority
        });
    }

    /// Cancel alert id
    pub fn cancel(&mut self, cancel_id: u32) {
        self.cancel_filter.put(cancel_id, ());
        self.received_alerts.remove(&cancel_id);
        self.noticed_alerts.retain(|a| {
            let id: u32 = a.raw().id().into();
            id != cancel_id
        });
    }

    #[allow(clippy::needless_collect)]
    async fn received(
        &mut self,
        nc: Arc<dyn CKBProtocolContext + Sync>,
        peer_index: PeerIndex,
        data: Bytes,
    ) {
        let alert: packed::Alert = match packed::AlertReader::from_slice(&data) {
            Ok(alert) => {
                if alert.raw().message().is_utf8()
                    && alert
                        .raw()
                        .min_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                    && alert
                        .raw()
                        .max_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                {
                    alert.to_entity()
                } else {
                    info!(
                        "A malformed message fromP peer {} : not utf-8 string",
                        peer_index
                    );
                    nc.ban_peer(
                        peer_index,
                        BAD_MESSAGE_BAN_TIME,
                        String::from("send us a malformed message: not utf-8 string"),
                    );
                    return;
                }
            }
            Err(err) => {
                info!("A malformed message from peer {}: {:?}", peer_index, err);
                nc.ban_peer(
                    peer_index,
                    BAD_MESSAGE_BAN_TIME,
                    String::from("send us a malformed message"),
                );
                return;
            }
        };
        let alert_id = alert.as_reader().raw().id().into();
        trace!("ReceiveD alert {} from peer {}", alert_id, peer_index);
        // ignore alert
        if self.notifier.lock().has_received(alert_id) {
            return;
        }
        // verify
        if let Err(err) = self.verifier.verify_signatures(&alert) {
            debug!(
                "An alert from peer {} with invalid signatures, error {:?}",
                peer_index, err
            );
            nc.ban_peer(
                peer_index,
                BAD_MESSAGE_BAN_TIME,
                String::from("send us an alert with invalid signatures"),
            );
            return;
        }
        // mark sender as known
        self.mark_as_known(peer_index, alert_id);
        // broadcast message
        let selected_peers: Vec<PeerIndex> = nc
            .connected_peers()
            .into_iter()
            .filter(|peer| self.mark_as_known(*peer, alert_id))
            .collect();
        if let Err(err) = nc.quick_filter_broadcast(
            TargetSession::Multi(Box::new(selected_peers.into_iter())),
            data,
        ) {
            debug!("alert broadcast error: {:?}", err);
        }
        // add to received alerts
        self.notifier.lock().add(&alert);
    }
