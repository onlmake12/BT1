### Title
Governance VAA Sequence Gap Allows Permanent Skip of Revocation Actions — (`lazer/contracts/cardano/lib/pyth/governance.ak`)

### Summary
The governance sequence check uses a strictly-greater-than comparison (`governance.seen_sequence < seen_sequence`) rather than requiring consecutive sequences (`seen_sequence + 1 == new_sequence`). Combined with the fact that `GovernanceAction` is permissionless (no owner check), any unprivileged relayer can permanently suppress a revocation VAA by submitting a higher-sequence VAA first, leaving a compromised trusted signer active.

### Finding Description

**Root cause — non-consecutive sequence check:**

In `parse_and_verify_action`, the only sequence guard is:

```
expect governance.seen_sequence < seen_sequence
``` [1](#0-0) 

This accepts any sequence strictly greater than the last seen value. A gap of any size is permitted. After applying seq=7, `seen_sequence` becomes 7, and seq=6 can never be applied because `6 < 7` is false.

**Permissionless entry point:**

In `pyth_state.ak`, the `GovernanceAction` branch has no `is_owner` guard:

```aiken
when action is {
  GovernanceAction(action) ->
    execute_governance_action(state, action, guardians)
  PurgeExpiredWithdrawScripts -> {
    expect is_owner   // only this branch is owner-gated
    ...
  }
}
``` [2](#0-1) 

Any relayer can submit any valid (guardian-signed) VAA in any order.

**Revocation mechanism:**

`UpdateTrustedSigner` sets a signer's validity range. Revocation is achieved by setting `expires_at` to a past timestamp, producing `Some(interval.before(past_time))`. If seq=6 carries this revocation and is permanently skipped, the signer remains in `trusted_signers` with its original (active) validity range. [3](#0-2) 

**Price update acceptance:**

`pyth_price.withdraw` iterates over all submitted updates and calls `message.parse_and_verify` against `pyth.trusted_signers`. A signer whose revocation was skipped remains in this list and its signatures continue to be accepted. [4](#0-3) 

### Impact Explanation

A compromised or decommissioned signer that should have been revoked via seq=6 continues to sign price updates that are accepted by `pyth_price.withdraw`. This enables direct oracle price manipulation on Cardano — the attacker (or the holder of the compromised key) can publish arbitrary prices that downstream protocols will consume as valid Pyth data.

### Likelihood Explanation

Wormhole VAAs are publicly observable. A malicious relayer who sees both seq=6 (revocation) and seq=7 (any other action) available can trivially submit seq=7 first. No key compromise, no privileged access, and no forged signatures are required — only selective ordering of publicly available VAAs. The attack is cheap, deterministic, and leaves no on-chain trace distinguishing it from normal relay behavior.

### Recommendation

Replace the gap-permitting check with a consecutive-sequence requirement:

```aiken
// Before (vulnerable):
expect governance.seen_sequence < seen_sequence

// After (fixed):
expect governance.seen_sequence + 1 == seen_sequence
``` [5](#0-4) 

This ensures every governance action is applied in strict order with no gaps, making it impossible to permanently suppress any VAA.

### Proof of Concept

State-transition test (pseudocode):

```
initial state: seen_sequence = 5, trusted_signers = [signer_X (active)]

step 1: submit GovernanceAction(VAA seq=7, action=UpdateTrustedSigner(signer_Y, permanent))
  → check: 5 < 7 ✓ → seen_sequence becomes 7, signer_Y added

step 2: attempt GovernanceAction(VAA seq=6, action=UpdateTrustedSigner(signer_X, expired))
  → check: 7 < 6 ✗ → FAILS, permanently rejected

result: signer_X remains in trusted_signers with active validity
        signer_X can sign price updates accepted by pyth_price.withdraw
``` [6](#0-5)

pub fn execute_governance_action(
  state: State<Pyth>,
  action: PreparedVAA,
  guardians: Guardians,
) {
  let pyth = state.data
  let (action, governance) =
    governance.parse_and_verify_action(action, pyth.governance, guardians)
  let pyth = Pyth { ..pyth, governance }
  when action is {
    UpdateTrustedSigner(key, valid) -> {
      let pyth =
        Pyth {
          ..pyth,
          trusted_signers: message.update_trusted_signer(
            pyth.trusted_signers,
            key,
            valid,
          ),
        }
      state.new(state.home, pyth, state.reference_script)
    }
    UpgradeSpendScript(hash) -> state.new(hash, pyth, state.reference_script)
    UpgradeWithdrawScript { new_script, previous_validity } -> {
      let pyth =
        Pyth {
          ..pyth,
          withdraw_script: new_script,
          deprecated_withdraw_scripts: [
            Pair(previous_validity, pyth.withdraw_script),
            ..pyth.deprecated_withdraw_scripts
          ],
        }
      state.new(state.home, pyth, Some(new_script))
    }
  }
}
```
