### Title
Inactive Owner Permanently Disables Lazer Price Verification — (`File: lazer/contracts/evm/src/PythLazer.sol`)

### Summary
`PythLazer.sol` gates all trusted-signer management behind `onlyOwner`. Every trusted signer carries a hard `expiresAt` timestamp. If the owner (governance executor) becomes inactive before renewing or replacing signers, every signer will eventually expire and `verifyUpdate()` will permanently revert for all callers, making the entire EVM Lazer price-feed service permanently unusable.

### Finding Description
`updateTrustedSigner()` is the sole mechanism to add, renew, or remove trusted signers. It is protected by `onlyOwner` with no time-based fallback. [1](#0-0) 

`isValidSigner()` returns `false` the moment `block.timestamp` reaches or exceeds the stored `expiresAt` value. [2](#0-1) 

`verifyUpdate()` hard-reverts when `isValidSigner` returns false. [3](#0-2) 

The contract stores up to 100 signer slots, all of which carry finite expiry timestamps. [4](#0-3) 

There is no permissionless path, no time-delay override, and no emergency escape hatch that would allow any party other than the owner to extend or replace a signer. Once every slot has expired and the owner cannot act, `verifyUpdate()` is permanently broken.

The owner is set to a governance executor contract at deployment. [5](#0-4) 

The governance executor itself is controlled by Wormhole governance. If governance becomes dysfunctional (guardian-set compromise, quorum failure, or key loss), no renewal transaction can be submitted.

### Impact Explanation
Every EVM consumer of Pyth Lazer calls `verifyUpdate()` to authenticate a price payload before using it. Once all trusted signers expire and the owner is inactive, every such call reverts with `"invalid signer"`. DeFi protocols (lending, perpetuals, options) that depend on Lazer feeds would be permanently unable to update prices, leading to frozen liquidations, stale collateral valuations, and potential loss of user funds. The impact is equivalent to the Fantom `tokensTradeable = false` scenario: a critical protocol function is permanently disabled with no recovery path.

### Likelihood Explanation
The owner is a governance executor contract, so the failure mode requires Wormhole governance to become unable to act (e.g., guardian-set key loss, quorum failure, or a prolonged governance deadlock). This is a lower-probability event than a single EOA losing a key, but it is not zero. Trusted signers are rotated on a schedule with finite expiry windows; a governance outage lasting longer than the remaining signer validity period triggers the condition. The absence of any time-based fallback means there is no grace period once the last signer expires.

### Recommendation
Add a time-based fallback analogous to the Fantom recommendation. For example, allow any caller to call a `renewSigner` function after a signer has been valid for a minimum period, or add a `require` that permits renewal by anyone once the last valid signer is within N days of expiry:

```solidity
function emergencyRenewSigner(address signer, uint256 newExpiresAt) external {
    require(
        msg.sender == owner() ||
        block.timestamp > trustedSignerToExpiresAtMapping[signer] - EMERGENCY_WINDOW,
        "not authorized"
    );
    // update mapping
}
```

Alternatively, remove hard expiry from the contract and rely solely on governance-issued revocations, eliminating the time-bomb property entirely.

### Proof of Concept

1. Governance executor is set as owner of `PythLazer` at `0xACeA761c27A909d4D3895128EBe6370FDE2dF481`.
2. Owner calls `updateTrustedSigner(signerAddr, block.timestamp + 90 days)`.
3. Governance becomes dysfunctional (guardian-set compromise or quorum failure) before the 90-day window closes.
4. After 90 days, `isValidSigner(signerAddr)` returns `false`.
5. Any Lazer consumer calls `verifyUpdate(payload)`.
6. Execution reaches line 103: `if (!isValidSigner(signer)) revert("invalid signer");` — permanent revert.
7. No on-chain path exists to recover; all Lazer price feeds on that EVM chain are permanently frozen. [6](#0-5)
