### Title
Permanently Locked User ETH in Echo Contract Due to Missing Request Cancellation Mechanism - (File: target_chains/ethereum/contracts/contracts/echo/Echo.sol)

### Summary
The `Echo.requestPriceUpdatesWithCallback` function accepts ETH from users and stores the provider-destined portion in `req.fee`, but there is no cancellation or refund function. If a provider never calls `executeCallback`, the user's ETH stored in `req.fee` is permanently locked in the contract with no recovery path.

### Finding Description
In `Echo.requestPriceUpdatesWithCallback` (lines 52–102), when a user creates a price update request, `msg.value` is split as follows:

```solidity
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);
_state.accruedFeesInWei += _state.pythFeeInWei;
``` [1](#0-0) 

The `_state.pythFeeInWei` portion is immediately credited to the Pyth protocol's `accruedFeesInWei`. The remainder — `msg.value - pythFeeInWei` — is stored in `req.fee` and is only ever credited to a provider when `executeCallback` is called:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
``` [2](#0-1) 

There is no `cancelRequest`, `refundRequest`, or timeout-based recovery function anywhere in the contract. The `clearRequest` function is internal and only reachable via `executeCallback`. The `withdrawFees` function only withdraws `accruedFeesInWei` (the Pyth protocol portion), not `req.fee` balances:

```solidity
function withdrawFees(uint128 amount) external override {
    require(msg.sender == _state.admin, "Only admin can withdraw fees");
    require(_state.accruedFeesInWei >= amount, "Insufficient balance");
    _state.accruedFeesInWei -= amount;
    (bool sent, ) = msg.sender.call{value: amount}("");
``` [3](#0-2) 

The developers themselves acknowledge the stuck-funds risk in a TODO comment:

> `// TODO: if this effect occurs here, we need to guarantee that executeCallback can never revert. If executeCallback can revert, then funds can be permanently locked in the contract.` [4](#0-3) 

The comment addresses only the revert-during-callback case, not the case where the provider never calls `executeCallback` at all.

The `EchoState.Request` struct stores `req.fee` as a `uint96` field: [5](#0-4) 

Once a request is created and the provider-destined ETH is stored in `req.fee`, the only code path that ever moves those funds is `executeCallback`. No admin, no user, and no governance action can recover `req.fee` without a contract upgrade.

### Impact Explanation
Any user who calls `requestPriceUpdatesWithCallback` and whose request is never fulfilled by a provider permanently loses the ETH stored in `req.fee`. The Pyth protocol fee (`pythFeeInWei`) is immediately credited and can be withdrawn by the admin, but the provider fee portion is irrecoverable. On a busy network with many pending requests, a provider going offline or selectively ignoring requests could lock significant user ETH.

### Likelihood Explanation
- A registered provider can go offline after users have submitted requests, leaving all pending `req.fee` balances locked.
- The exclusivity period (`exclusivityPeriodSeconds`) can expire without the assigned provider fulfilling, and no other provider is obligated to step in.
- A provider could selectively ignore low-value or unprofitable requests, permanently locking those users' ETH.
- The `publishTime` constraint (`publishTime <= block.timestamp + 60`) means requests can become stale and unfulfillable if not executed promptly, yet there is no refund path. [6](#0-5) 

### Recommendation
Add a time-bounded cancellation function that allows users to reclaim `req.fee` after a configurable timeout (e.g., after the exclusivity period plus a grace period):

```solidity
function cancelRequest(uint64 sequenceNumber) external {
    Request storage req = findActiveRequest(sequenceNumber);
    require(msg.sender == req.requester, "Not requester");
    require(block.timestamp > req.publishTime + CANCEL_TIMEOUT, "Too early");
    uint96 refundAmount = req.fee;
    clearRequest(sequenceNumber);
    (bool sent, ) = msg.sender.call{value: refundAmount}("");
    require(sent, "Refund failed");
}
```

### Proof of Concept
1. User calls `requestPriceUpdatesWithCallback{value: totalFee}(provider, publishTime, priceIds, gasLimit)`.
2. Contract stores `req.fee = totalFee - pythFeeInWei` and credits `pythFeeInWei` to `accruedFeesInWei`.
3. The assigned provider never calls `executeCallback` (goes offline, ignores the request, or the `publishTime` window passes).
4. The exclusivity period expires; no other provider is obligated to fulfill.
5. User's ETH (`req.fee`) remains in the contract indefinitely.
6. User calls every public/external function — `withdrawFees`, `withdrawAsFeeManager`, `getRequest` — and finds no path to recover `req.fee`.
7. ETH is permanently locked unless the contract is upgraded. [7](#0-6)

    function requestPriceUpdatesWithCallback(
        address provider,
        uint64 publishTime,
        bytes32[] calldata priceIds,
        uint32 callbackGasLimit
    ) external payable override returns (uint64 requestSequenceNumber) {
        require(
            _state.providers[provider].isRegistered,
            "Provider not registered"
        );

        // FIXME: this comment is wrong. (we're not using tx.gasprice)
        // NOTE: The 60-second future limit on publishTime prevents a DoS vector where
        //      attackers could submit many low-fee requests for far-future updates when gas prices
        //      are low, forcing executors to fulfill them later when gas prices might be much higher.
        //      Since tx.gasprice is used to calculate fees, allowing far-future requests would make
        //      the fee estimation unreliable.
        require(publishTime <= block.timestamp + 60, "Too far in future");
        if (priceIds.length > MAX_PRICE_IDS) {
            revert TooManyPriceIds(priceIds.length, MAX_PRICE_IDS);
        }
        requestSequenceNumber = _state.currentSequenceNumber++;

        uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
        if (msg.value < requiredFee) revert InsufficientFee();

        Request storage req = allocRequest(requestSequenceNumber);
        req.sequenceNumber = requestSequenceNumber;
        req.publishTime = publishTime;
        req.callbackGasLimit = callbackGasLimit;
        req.requester = msg.sender;
        req.provider = provider;
        req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);

        // Create array with the right size
        req.priceIdPrefixes = new bytes8[](priceIds.length);

        // Copy only the first 8 bytes of each price ID to storage
        for (uint8 i = 0; i < priceIds.length; i++) {
            // Extract first 8 bytes of the price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }
            req.priceIdPrefixes[i] = prefix;
        }
        _state.accruedFeesInWei += _state.pythFeeInWei;

        emit PriceUpdateRequested(req, priceIds);
    }
