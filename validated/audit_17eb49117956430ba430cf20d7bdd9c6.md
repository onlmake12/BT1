Audit Report

## Title
Out-of-Bounds Index Panic in `TxConfirmStat::remove_unconfirmed_tx` Crashes Tx-Pool Service Thread — (`util/fee-estimator/src/estimator/confirmation_fraction.rs`)

## Summary
When a tracked transaction remains unconfirmed for more than `MAX_CONFIRM_BLOCKS` (1000) blocks and is then evicted from the tx-pool with `count_failure = true`, the `confirm_blocks_to_failed_txs[tx_age - 1]` index access in `remove_unconfirmed_tx` exceeds the vector's length of 1000, causing a Rust index-out-of-bounds panic. This panic propagates through the tx-pool reject callback and crashes the tx-pool service thread, rendering the node unable to accept or relay transactions.

## Finding Description
`TxConfirmStat` is initialized with `confirm_blocks_to_failed_txs` and `block_unconfirmed_txs` both as vectors of length `MAX_CONFIRM_BLOCKS = 1000`.

In `remove_unconfirmed_tx` (lines 197–217), the guard at line 208 correctly redirects the `block_unconfirmed_txs` access to `old_unconfirmed_txs` when `tx_age >= 1000`, but the subsequent `count_failure` branch at lines 214–215 has **no corresponding bounds check**:

```rust
if tx_age >= self.block_unconfirmed_txs.len() {   // guards block_unconfirmed_txs only
    self.bucket_stats[bucket_index].old_unconfirmed_txs -= 1;
} else {
    self.block_unconfirmed_txs[block_index][bucket_index] -= 1;
}
if count_failure {
    self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64; // NO BOUNDS CHECK
}
```

When `tx_age = 1001`, `tx_age - 1 = 1000` is out of bounds for a 1000-element vector, causing a panic.

**Call chain:**
1. Attacker submits a valid minimum-fee tx → `register_pending` callback → `fee_estimator.accept_tx(tx_hash, entry_info)` records `entry_height = current_tip`.
2. 1001+ blocks pass; `best_height` advances with each `commit_block` call.
3. Tx is evicted (expiry, pool size limit, or reorg) → `register_reject` callback → `fee_estimator.reject_tx(&tx_hash)`.
4. `reject_tx` → `drop_tx(count_failure=true)` → `drop_tx_inner` → `remove_unconfirmed_tx(entry_height, best_height, bucket_index, true)`.
5. `tx_age = best_height - entry_height >= 1001` → `confirm_blocks_to_failed_txs[1000]` → **panic**.

The existing guard at line 208 is insufficient because it only protects `block_unconfirmed_txs`; `confirm_blocks_to_failed_txs` shares the same length but has no guard.

## Impact Explanation
A Rust index-out-of-bounds panic unwinds and terminates the tx-pool service thread. The tx-pool is a critical node component; its crash renders the node unable to accept or relay transactions. This matches the allowed CKB bounty impact: **High — Vulnerabilities which could easily crash a CKB node** (10001–15000 points). The `ConfirmationFraction` estimator must be explicitly configured (it is not the default `Dummy` estimator), but any node operator who enables it is exposed.

## Likelihood Explanation
The attack requires only a valid transaction submission (no special privileges, no PoW, no key material) and patience. The default tx expiry of 12 hours makes the 1001-block threshold (≈2.2 hours at 8 s/block) easily reachable in normal operation, even without deliberate attack — a low-fee tx that is never proposed will naturally age past 1000 blocks. The attack is repeatable: after the node restarts, a new tx can be submitted to trigger the crash again.

## Recommendation
Add a bounds check before the `confirm_blocks_to_failed_txs` access in `remove_unconfirmed_tx`, mirroring the existing guard for `block_unconfirmed_txs`:

```rust
if count_failure && tx_age - 1 < self.confirm_blocks_to_failed_txs.len() {
    self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64;
}
```

This silently drops failure samples for txs older than the tracking window, which is the correct semantic (they are already counted as `old_unconfirmed_txs`).

    fn remove_unconfirmed_tx(
        &mut self,
        entry_height: u64,
        tip_height: u64,
        bucket_index: usize,
        count_failure: bool,
    ) {
        let tx_age = tip_height.saturating_sub(entry_height) as usize;
        if tx_age < 1 {
            return;
        }
        if tx_age >= self.block_unconfirmed_txs.len() {
            self.bucket_stats[bucket_index].old_unconfirmed_txs -= 1;
        } else {
            let block_index = (entry_height % self.block_unconfirmed_txs.len() as u64) as usize;
            self.block_unconfirmed_txs[block_index][bucket_index] -= 1;
        }
        if count_failure {
            self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64;
        }
    }
