Audit Report

## Title
Peer Store Permanently Blocked via Crafted Group-of-4 Address Distribution — (`network/src/peer_store/peer_store_impl.rs`)

## Summary
`check_purge` contains two compounding logic errors: `take(len / 2)` (floor division) and `addrs.len() > 4` (strict threshold). When an attacker fills the peer store with exactly 4 connectable addresses per `/16` network group, every group fails the `> 4` check, `candidate_peers` is always empty, and `check_purge` permanently returns `EvictionFailed`. Because `add_addr` propagates this error, the node can no longer accept any new peer addresses, leading to eventual network isolation.

## Finding Description
In `check_purge` [1](#0-0)  the function returns early if the store is below `ADDR_COUNT_LIMIT = 16384`. [2](#0-1)  When the store is full and all addresses are connectable, the group-based eviction path executes:

```rust
let len = peers_by_network_group.len();   // e.g. 4096
peers.sort_unstable_by_key(|k| Reverse(k.len()));
peers
    .into_iter()
    .take(len / 2)                        // take(2048) — floor division
    .flat_map(move |addrs| {
        if addrs.len() > 4 {              // 4 > 4 == false for every group
            Some(...)
        } else {
            None                          // always None
        }
    })
    .flatten()
    .collect()                            // always empty Vec
``` [3](#0-2) 

With 4096 groups × 4 peers = 16384 = `ADDR_COUNT_LIMIT`, `len = 4096`, `take(2048)` considers 2048 groups, but every group has `len == 4`, so `4 > 4` is `false` for all of them. `candidate_peers` is empty, and the function hits:

```rust
if candidate_peers.is_empty() {
    return Err(PeerStoreError::EvictionFailed.into());
}
``` [4](#0-3) 

`add_addr` propagates this error via `self.check_purge()?` at line 75. [5](#0-4) 

Freshly injected addresses are created with `AddrInfo::new(addr, 0, score, flags.bits())`, setting `last_connected_at_ms = 0` and `attempts_count = 0`. [6](#0-5)  These pass `is_connectable` because the only failing conditions require `attempts_count >= ADDR_MAX_RETRIES (3)` or `attempts_count >= ADDR_MAX_FAILURES (10)`, neither of which holds at 0. [7](#0-6)  They therefore survive the first eviction pass and feed directly into the dead zone.

## Impact Explanation
Once the dead zone is triggered, `add_addr` returns `Err` for every subsequent call. The node's peer store is permanently locked: it cannot learn about new peers via the discovery protocol. If any existing connections drop, the node has no mechanism to find replacements and becomes isolated from the CKB network. This maps to **High (10001–15000 points): Vulnerabilities which could easily crash a CKB node**, as a fully isolated node is effectively non-functional as a network participant.

## Likelihood Explanation
An attacker needs only a single P2P connection to the victim node to advertise addresses via the discovery protocol. IPv4 has 65536 possible `/16` blocks; filling 4096 of them with exactly 4 addresses each (16384 total) is straightforward. All injected addresses default to `last_connected_at_ms = 0` and `attempts_count = 0` via `AddrInfo::new`. [8](#0-7)  No special privileges, no victim mistakes, and no external dependencies are required. The attack is repeatable and deterministic.

## Recommendation
1. Replace `addrs.len() > 4` with `addrs.len() >= 4` so that groups of exactly 4 are eligible for eviction. [9](#0-8) 
2. Replace `take(len / 2)` with `take((len + 1) / 2)` (ceiling division) to ensure at least one group is always considered when `len` is odd. [10](#0-9) 
3. Add a fallback: if `candidate_peers` is still empty after the group pass, unconditionally evict one address from the largest group, preventing `EvictionFailed` when the store is full of connectable peers. [4](#0-3) 

## Proof of Concept
```rust
let mut peer_store = PeerStore::default();
// Fill with 4096 groups, each with exactly 4 peers from distinct /16 blocks:
for g in 0u16..4096 {
    let a = (g >> 8) as u8 + 1;
    let b = (g & 0xff) as u8 + 1;
    for h in 1u8..=4 {
        let addr = format!("/ip4/{}.{}.0.{}/tcp/8115", a, b, h).parse().unwrap();
        peer_store.add_addr(addr, Flags::COMPATIBILITY).unwrap();
    }
}
// Store is now at ADDR_COUNT_LIMIT with 4096 groups of 4 connectable peers each.
// Any further add_addr call returns Err(EvictionFailed):
let new_addr = "/ip4/5.5.0.1/tcp/8115".parse().unwrap();
assert!(peer_store.add_addr(new_addr, Flags::COMPATIBILITY).is_err());
```

        // do not remove addr tried in last minute
        if self.tried_in_last_minute(now_ms) {
            return true;
        }
        // we give up if never connect to this addr
        if self.last_connected_at_ms == 0 && self.attempts_count >= ADDR_MAX_RETRIES {
            return false;
        }
        // consider addr is not connectable if failed too many times

        true
