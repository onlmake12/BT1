### Title
Excess ETH Sent to `requestPriceUpdatesWithCallback` Is Not Refunded and Is Credited to the Provider - (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary
`Echo.requestPriceUpdatesWithCallback` accepts `msg.value` and checks only that it meets the minimum required fee, but any excess above `requiredFee` is silently credited to the provider rather than refunded to the caller. A user who overpays (e.g., due to a fee change between estimation and execution, or a fixed-value integration) permanently loses the excess ETH to the provider.

### Finding Description
In `Echo.sol`, `requestPriceUpdatesWithCallback` computes `requiredFee = getFee(provider, callbackGasLimit, priceIds)` and reverts if `msg.value < requiredFee`. However, the fee stored for the provider is computed as:

```solidity
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);
``` [1](#0-0) 

`_state.pythFeeInWei` is only the Pyth protocol base fee, not the full `requiredFee`. If `msg.value = requiredFee + X`, then `req.fee = (providerFees + X)` — the excess `X` is folded into the provider's credited amount. When `executeCallback` is later called, the provider receives `req.fee + msg.value_executeCallback - pythFee`, which includes the user's excess `X`. [2](#0-1) 

There is no refund path anywhere in the function.

A secondary instance exists in the core `Pyth.sol` contract: `updatePriceFeeds`, `parsePriceFeedUpdatesWithConfig`, and `parseTwapPriceFeedUpdates` all check `msg.value >= requiredFee` but never refund the excess, which accumulates in the contract and is only recoverable by governance via `withdrawFee`. [3](#0-2) [4](#0-3) [5](#0-4) 

By contrast, `PythLazer.sol` correctly implements the refund pattern:

```solidity
if (msg.value > verification_fee) {
    payable(msg.sender).transfer(msg.value - verification_fee);
}
``` [6](#0-5) 

### Impact Explanation
In `Echo.sol`: any ETH above `requiredFee` sent by the caller is permanently transferred to the provider's accrued balance. The caller has no mechanism to recover it. This is a direct, unintended transfer of user funds to a third party.

In `Pyth.sol`: excess ETH is locked in the contract and is only recoverable by governance, not by the original sender.

### Likelihood Explanation
Low. Users are expected to call `getFee()` / `getUpdateFee()` before each call and send the exact amount. However, realistic scenarios that cause overpayment include:
- Fee changes between the estimation call and the transaction landing (e.g., gas price fluctuation affecting `feePerGasInWei`-based fees in Echo)
- Integrators or contracts that send a fixed ETH buffer to avoid reverts
- Front-end rounding or slippage buffers

### Recommendation
1. **Echo.sol – `requestPriceUpdatesWithCallback`**: After computing `requiredFee`, refund the excess to `msg.sender`:
```solidity
uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
if (msg.value < requiredFee) revert InsufficientFee();
uint256 excess = msg.value - requiredFee;
if (excess > 0) {
    (bool ok, ) = msg.sender.call{value: excess}("");
    require(ok, "refund failed");
}
req.fee = SafeCast.toUint96(requiredFee - _state.pythFeeInWei);
```

2. **Pyth.sol – `updatePriceFeeds`, `parsePriceFeedUpdatesWithConfig`, `parseTwapPriceFeedUpdates`**: Similarly refund `msg.value - requiredFee` after the fee check.

### Proof of Concept
1. Provider registers with `baseFeeInWei = 1000`, `feePerFeedInWei = 500`, `feePerGasInWei = 1`.
2. User calls `getFee(provider, 100000, [feedId])` → returns, say, `requiredFee = 2500`.
3. User calls `requestPriceUpdatesWithCallback{value: 3000}(provider, ...)` (500 wei excess).
4. `req.fee = 3000 - pythFeeInWei`. If `pythFeeInWei = 100`, `req.fee = 2900` instead of the correct `2400`.
5. When `executeCallback` is called, the provider's `accruedFeesInWei` is incremented by `req.fee - pythFee`, which includes the user's 500 wei excess.
6. The user has no way to recover the 500 wei. [7](#0-6)

    function requestPriceUpdatesWithCallback(
        address provider,
        uint64 publishTime,
        bytes32[] calldata priceIds,
        uint32 callbackGasLimit
    ) external payable override returns (uint64 requestSequenceNumber) {
        require(
            _state.providers[provider].isRegistered,
            "Provider not registered"
        );

        // FIXME: this comment is wrong. (we're not using tx.gasprice)
        // NOTE: The 60-second future limit on publishTime prevents a DoS vector where
        //      attackers could submit many low-fee requests for far-future updates when gas prices
        //      are low, forcing executors to fulfill them later when gas prices might be much higher.
        //      Since tx.gasprice is used to calculate fees, allowing far-future requests would make
        //      the fee estimation unreliable.
        require(publishTime <= block.timestamp + 60, "Too far in future");
        if (priceIds.length > MAX_PRICE_IDS) {
            revert TooManyPriceIds(priceIds.length, MAX_PRICE_IDS);
        }
        requestSequenceNumber = _state.currentSequenceNumber++;

        uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
        if (msg.value < requiredFee) revert InsufficientFee();

        Request storage req = allocRequest(requestSequenceNumber);
        req.sequenceNumber = requestSequenceNumber;
        req.publishTime = publishTime;
        req.callbackGasLimit = callbackGasLimit;
        req.requester = msg.sender;
        req.provider = provider;
        req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);

        // Create array with the right size
        req.priceIdPrefixes = new bytes8[](priceIds.length);

        // Copy only the first 8 bytes of each price ID to storage
        for (uint8 i = 0; i < priceIds.length; i++) {
            // Extract first 8 bytes of the price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }
            req.priceIdPrefixes[i] = prefix;
        }
        _state.accruedFeesInWei += _state.pythFeeInWei;

        emit PriceUpdateRequested(req, priceIds);
    }
