### Title
Fee Deposited Before Freshness Check With No Refund Path Enables Front-Running Fee Drain — (`target_chains/sui/contracts/sources/pyth.move`)

---

### Summary

In `update_single_price_feed`, the SUI fee is permanently deposited into the `PriceInfoObject` **before** the freshness check is evaluated. If a front-running attacker pre-updates the same `PriceInfoObject` to a newer timestamp, the victim's subsequent call deposits the fee but receives no price update, with no refund path.

---

### Finding Description

The execution order in `update_single_price_feed` is:

1. **Line 277** — `price_info::deposit_fee_coins(price_info_object, fee)` — fee is irrevocably merged into the object's dynamic field.
2. **Line 287** — `update_cache(...)` is called.
3. **Inside `update_cache`, line 316** — `if (is_fresh_update(update, price_info_object))` — freshness is checked only here.
4. If `is_fresh_update` returns `false`, the `if`-body is skipped silently — **no abort, no refund**. [1](#0-0) [2](#0-1) 

`is_fresh_update` requires strictly `update_timestamp > cached_timestamp`: [3](#0-2) 

`deposit_fee_coins` only adds or joins coins — it has no return or refund branch: [4](#0-3) 

---

### Impact Explanation

The victim's SUI fee is permanently locked inside the `PriceInfoObject`'s `FEE_STORAGE_KEY` dynamic field. There is no withdrawal function, no refund branch, and no abort that would roll back the deposit. The victim's transaction succeeds (no error), but the price feed is unchanged and the fee is gone. This is a direct, permanent loss of user funds matching the stated scope.

---

### Likelihood Explanation

The attack requires no privileges. Any user can:

1. Fetch a fresh accumulator message (timestamp T+1) from the public Pyth HTTP API.
2. Call `create_authenticated_price_infos_using_accumulator` — a fully public function — to produce a valid `HotPotatoVector<PriceInfo>` at T+1.
3. Call `update_single_price_feed` on the shared `PriceInfoObject` before the victim's transaction is processed. [5](#0-4) 

On Sui, transaction ordering within a checkpoint is determined by validators. A well-resourced attacker can reliably order their transaction ahead of the victim's by submitting with higher gas or by monitoring the mempool. The cost to the attacker is only one update fee; the victim loses their fee with zero benefit.

---

### Recommendation

Move the fee deposit to **after** the freshness check, or abort (refunding via returning the `Coin<SUI>` to the caller) when `is_fresh_update` returns `false`. The cleanest fix is to restructure `update_single_price_feed` so that `deposit_fee_coins` is only called when the update will actually be applied:

```move
// pseudocode fix
if (is_fresh_update(cur_price_info, price_info_object)) {
    price_info::deposit_fee_coins(price_info_object, fee);
    update_cache(...);
} else {
    // return fee to caller or abort with refund
    transfer::public_transfer(fee, tx_context::sender(ctx));
}
```

Alternatively, check freshness first and abort with a descriptive error code if the update is stale, so the entire transaction (including the fee transfer) is rolled back by the Sui runtime.

    fun is_fresh_update(update: &PriceInfo, price_info_object: &PriceInfoObject): bool {
        // Get the timestamp of the update's current price
        let price_feed = price_info::get_price_feed(update);
        let update_timestamp = price::get_timestamp(&price_feed::get_price(price_feed));

        // Get the timestamp of the cached data for the price identifier
        let cached_price_info = price_info::get_price_info_from_price_info_object(price_info_object);
        let cached_price_feed =  price_info::get_price_feed(&cached_price_info);
        let cached_timestamp = price::get_timestamp(&price_feed::get_price(cached_price_feed));

        update_timestamp > cached_timestamp
    }

    public fun deposit_fee_coins(price_info_object: &mut PriceInfoObject, fee_coins: Coin<SUI>) {
        if (!dynamic_object_field::exists_with_type<vector<u8>, Coin<SUI>>(&price_info_object.id, FEE_STORAGE_KEY)) {
            dynamic_object_field::add(&mut price_info_object.id, FEE_STORAGE_KEY, fee_coins);
        }
        else {
            let current_fee = dynamic_object_field::borrow_mut<vector<u8>, Coin<SUI>>(
                &mut price_info_object.id,
                FEE_STORAGE_KEY
            );
            coin::join(current_fee, fee_coins);
        };
    }
