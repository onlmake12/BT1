Audit Report

## Title
Peer Store Permanent DoS via Off-by-One in `check_purge` Eviction Guard — (`network/src/peer_store/peer_store_impl.rs`)

## Summary

`check_purge` uses a strict `> 4` guard when selecting eviction candidates in Stage 2. An attacker who fills the peer store with exactly 4 addresses per network group across enough distinct groups causes every subsequent `add_addr` call to return `PeerStoreError::EvictionFailed`, permanently blocking honest peer address admission and enabling node isolation.

## Finding Description

`add_addr` calls `check_purge` before inserting any address. [1](#0-0) 

`check_purge` has two eviction stages:

**Stage 1** (lines 341–355): Collects addresses where `is_connectable` returns `false`. Freshly injected addresses have `last_connected_at_ms = 0` and `attempts_count = 0`. Per `is_connectable`, a never-connected peer is only non-connectable when `attempts_count >= ADDR_MAX_RETRIES (3)`. All attacker-injected addresses therefore pass as connectable — Stage 1 finds nothing. [2](#0-1) 

**Stage 2** (lines 357–402): Groups addresses by network segment, sorts by descending group size, takes the top `len/2` groups, and applies the critical filter: [3](#0-2) 

```rust
if addrs.len() > 4 {   // groups of exactly 4 are NEVER evicted
```

If every group has exactly 4 entries, `4 > 4` is `false`, `flat_map` produces no candidates, `candidate_peers` is empty, and the function returns `Err(PeerStoreError::EvictionFailed)`. [4](#0-3) 

**Attack construction:**
- `ADDR_COUNT_LIMIT = 16384` [5](#0-4) 
- 4096 distinct `/16` IPv4 groups × 4 addresses each = 16384 total — exactly fills the store
- All freshly added → all connectable → Stage 1 skipped
- All groups have `len == 4` → `> 4` never fires → `EvictionFailed` returned on every subsequent `add_addr`

## Impact Explanation

Every `add_addr` call after store saturation returns `Err(PeerStoreError::EvictionFailed)`. The node can no longer learn any new peer addresses from discovery, DNS seeding, or identify handshakes. This permanently disables peer discovery, enabling sustained network isolation and eclipse attacks. This matches **High (10001–15000 points): Vulnerabilities which could easily crash a CKB node** — the node's peer discovery subsystem is permanently disabled, and recovery requires a full restart followed by the attacker's 16384 addresses aging out (requiring `attempts_count >= 3` per address via failed connection attempts to all attacker addresses).

## Likelihood Explanation

The attacker requires 16384 routable IP addresses across 4096 distinct `/16` subnets (4 per subnet). This is achievable via a mid-sized botnet or a cloud provider with diverse IP allocations. The discovery protocol accepts `Nodes` messages from any connected peer, so a single malicious peer can deliver all 16384 addresses across multiple `Nodes` responses. No per-session address-count rate limiting is present in the visible code path. The attack is repeatable after a node restart.

## Recommendation

Change the eviction threshold from strict `> 4` to `>= 4` at line 378 of `network/src/peer_store/peer_store_impl.rs`:

```rust
// Before (vulnerable):
if addrs.len() > 4 {

// After (fixed):
if addrs.len() >= 4 {
```

Additionally:
1. Cap the number of addresses accepted per discovery session to limit the rate of store saturation.
2. Prefer eviction of addresses with `last_connected_at_ms == 0` (never successfully connected) before evicting addresses with connection history.

## Proof of Concept

```rust
#[test]
fn test_eviction_failed_with_exactly_4_per_group() {
    let mut peer_store = PeerStore::default();
    // Fill 4096 /16 groups × 4 addresses each = 16384 total
    for group in 0u16..4096 {
        let hi = (group >> 8) as u8;
        let lo = (group & 0xff) as u8;
        for host in 1u8..=4 {
            let addr: Multiaddr = format!("/ip4/{}.{}.1.{}/tcp/8115", hi, lo, host)
                .parse().unwrap();
            let _ = peer_store.add_addr(addr, Flags::COMPATIBILITY);
        }
    }
    // Any subsequent add_addr must fail with EvictionFailed
    let new_addr: Multiaddr = "/ip4/200.200.1.5/tcp/8115".parse().unwrap();
    assert!(matches!(
        peer_store.add_addr(new_addr, Flags::COMPATIBILITY),
        Err(e) if e.to_string().contains("EvictionFailed")
    ));
}
```
