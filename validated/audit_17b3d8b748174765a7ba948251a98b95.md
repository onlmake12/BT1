Audit Report

## Title
Out-of-bounds panic in `ConfirmationFraction::remove_unconfirmed_tx` when tx tracked at height 0 ages beyond `MAX_CONFIRM_BLOCKS` — (`util/fee-estimator/src/estimator/confirmation_fraction.rs`)

## Summary
`ConfirmationFraction::accept_tx` lacks the `current_tip == 0` early-return guard present in `WeightUnitsFlow::accept_tx`. After IBD exit, both `current_tip` and `best_height` are reset to 0, so a tx submitted before the first `commit_block` is tracked at height 0. When that tx is later rejected after 1001+ blocks, `remove_unconfirmed_tx` accesses `confirm_blocks_to_failed_txs[tx_age - 1]` with `tx_age = 1001`, which is index 1000 on a length-1000 vector, causing a Rust index-out-of-bounds panic and crashing the node.

## Finding Description
`TxConfirmStat::new` allocates `confirm_blocks_to_failed_txs` with exactly `MAX_CONFIRM_BLOCKS = 1000` entries (indices 0–999). [1](#0-0) [2](#0-1) 

In `remove_unconfirmed_tx` (lines 204–216), when `tx_age >= self.block_unconfirmed_txs.len()` (≥ 1000), the code correctly routes to the `old_unconfirmed_txs` counter, but then unconditionally executes `self.confirm_blocks_to_failed_txs[tx_age - 1][bucket_index] += 1f64` without bounding `tx_age` to `MAX_CONFIRM_BLOCKS`. For `tx_age = 1001`, this accesses index 1000 on a length-1000 vector — a Rust panic. [3](#0-2) 

The precondition — a tx tracked at height 0 — is reachable because `track_tx` only checks `height != self.best_height`. [4](#0-3) 

After `update_ibd_state(false)` calls `clear()`, both `current_tip` and `best_height` are 0, so `0 != 0` is `false` and the tx is inserted. [5](#0-4) 

`accept_tx` passes `self.current_tip` (which is 0) directly to `track_tx` with no guard. [6](#0-5) 

`WeightUnitsFlow::accept_tx` avoids this with an explicit `if self.current_tip == 0 { return; }` guard that `ConfirmationFraction::accept_tx` is missing. [7](#0-6) 

Exploit chain:
1. Operator configures `algorithm = "ConfirmationFraction"`
2. Node exits IBD → `update_ibd_state(false)` → `clear()` → `current_tip = 0`, `best_height = 0`, `is_ready = true`
3. Attacker submits a tx before the first `commit_block` → `accept_tx` → `track_tx(hash, fee_rate, 0)` → `0 != 0` is false → tx inserted with `height = 0`
4. Node processes 1001 blocks → `best_height = 1001`; at block 1000, `move_track_window(1000)` moves `block_unconfirmed_txs[0]` into `old_unconfirmed_txs`
5. Tx is evicted → `reject_tx` → `drop_tx_inner(hash, true)` → `remove_unconfirmed_tx(0, 1001, bucket_index, true)` → `tx_age = 1001` → `confirm_blocks_to_failed_txs[1000]` → **index out of bounds → node crash** [8](#0-7) 

## Impact Explanation
The Rust runtime panics on an out-of-bounds index access, terminating the node process. This is a denial-of-service crash of a CKB node, matching the allowed impact: **High (10001–15000 points) — Vulnerabilities which could easily crash a CKB node**.

## Likelihood Explanation
Requires three conditions: (1) non-default `ConfirmationFraction` configuration, limiting the affected population; (2) a tx submitted in the window between IBD exit and the first `commit_block`, which can be seconds to minutes and is observable via P2P sync state; (3) that tx surviving in the pool for 1001+ blocks before rejection, achievable with a low-fee-rate tx that lingers for hours. An attacker who monitors IBD exit can deliberately submit a low-fee tx at the right moment and wait. The attack is repeatable after each node restart that re-enters and exits IBD.

## Recommendation
Add the same guard present in `WeightUnitsFlow::accept_tx` to `ConfirmationFraction::accept_tx`:
```rust
pub fn accept_tx(&mut self, tx_hash: Byte32, info: TxEntryInfo) {
    if self.current_tip == 0 {
        return;
    }
    let weight = get_transaction_weight(info.size as usize, info.cycles);
    let fee_rate = FeeRate::calculate(info.fee, weight);
    self.track_tx(tx_hash, fee_rate, self.current_tip)
}
```
Additionally, bound `tx_age` in `remove_unconfirmed_tx` before indexing `confirm_blocks_to_failed_txs`:
```rust
if count_failure {
    let capped_age = tx_age.min(self.confirm_blocks_to_failed_txs.len());
    self.confirm_blocks_to_failed_txs[capped_age - 1][bucket_index] += 1f64;
}
```
