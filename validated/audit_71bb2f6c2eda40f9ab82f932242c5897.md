### Title
Missing Per-Peer Rate Limit in `BlockFilter` Protocol Allows Unbounded DB Read Amplification — (`sync/src/filter/get_block_filter_hashes_process.rs`)

### Summary

The `BlockFilter` protocol handler has no per-peer rate limiter, unlike the `Relayer` and `HolePunching` protocol handlers which both enforce one. Any unprivileged remote peer can send `GetBlockFilterHashes(start_number=0)` in a tight loop, forcing the server to execute up to 4,000 sequential RocksDB reads per message with no throttle, no ban trigger, and no disconnect.

### Finding Description

`GetBlockFilterHashesProcess::execute` iterates up to `BATCH_SIZE = 2000` times. Each iteration issues two DB reads: `get_block_hash(block_number)` followed by `get_block_filter_hash(&block_hash)`. [1](#0-0) [2](#0-1) 

When `start_number = 0`, the special-case branch returns `packed::Byte32::zero()` as the parent hash and proceeds directly into the loop — no validation rejects it. [3](#0-2) 

The `BlockFilter` struct holds only `shared: Arc<SyncShared>` — there is no `rate_limiter` field, and the `received` handler calls `self.process(...)` unconditionally with no rate check. [4](#0-3) [5](#0-4) 

By contrast, `Relayer` carries a `RateLimiter<(PeerIndex, u32)>` and gates every non-PoW message through `check_key(&(peer, message.item_id()))` before dispatch: [6](#0-5) [7](#0-6) 

`HolePunching` applies the same pattern at the top of `received`: [8](#0-7) 

`BlockFilter` has no equivalent guard anywhere in `sync/src/filter/`.

### Impact Explanation

On a fully-synced node with ≥ 2,000 filter blocks built, a single peer sending `GetBlockFilterHashes(start_number=0)` at line-rate forces 4,000 RocksDB point-reads per message. With multiple concurrent peers (each within the normal peer-count limit) the aggregate read pressure multiplies linearly. This degrades I/O throughput for all other node operations (block validation, sync, tx-pool) without crashing the process — matching the Low (501–2,000) performance-degradation scope.

### Likelihood Explanation

The attack requires only a valid P2P connection and a single well-formed message type. No key, no PoW, no special role. The message is tiny (8 bytes for `start_number`), so bandwidth cost to the attacker is negligible. The exploit is trivially scriptable and locally testable.

### Recommendation

Add a `RateLimiter<(PeerIndex, u32)>` field to `BlockFilter` (mirroring `Relayer`) and check it at the top of `BlockFilter::try_process` before dispatching to any sub-processor. A quota of 30 requests/second per peer per message type (consistent with `Relayer`) would bound the worst-case DB read rate to 120,000 reads/second per peer — still high, so a tighter quota (e.g., 2–5/second) is preferable for filter-serving endpoints.

    pub async fn execute(self) -> Status {
        let active_chain = self.filter.shared.active_chain();
        let start_number: BlockNumber = self.message.to_entity().start_number().into();
        let latest: BlockNumber = active_chain.get_latest_built_filter_block_number();

        let mut block_filter_hashes = Vec::new();

        if latest >= start_number {
            let parent_block_filter_hash = if start_number > 0 {
                match active_chain
                    .get_block_hash(start_number - 1)
                    .and_then(|block_hash| active_chain.get_block_filter_hash(&block_hash))
                {
                    Some(parent_block_filter_hash) => parent_block_filter_hash,
                    None => return Status::ignored(),
                }
            } else {
                packed::Byte32::zero()
            };

            let mut block_number = start_number;
            for _ in 0..BATCH_SIZE {
                if let Some(block_filter_hash) = active_chain
                    .get_block_hash(block_number)
                    .and_then(|block_hash| active_chain.get_block_filter_hash(&block_hash))
                {
                    block_filter_hashes.push(block_filter_hash);
                } else {
                    break;
                }
                let Some(next_block_number) = block_number.checked_add(1) else {
                    break;
                };
                block_number = next_block_number;
            }
            let content = packed::BlockFilterHashes::new_builder()
                .start_number(start_number)
                .parent_block_filter_hash(parent_block_filter_hash)
                .block_filter_hashes(block_filter_hashes)
                .build();

            let message = packed::BlockFilterMessage::new_builder()
                .set(content)
                .build();
            async_send_message_to(&self.nc, self.peer, &message).await
        } else {
            Status::ignored()
        }
    }
