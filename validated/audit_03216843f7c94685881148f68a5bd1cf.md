Audit Report

## Title
Unconditional `.expect()` on `txs_sizes` panics for pre-v0.106 legacy `BlockExt` records — (`rpc/src/util/fee_rate.rs`)

## Summary
`FeeRateCollector::statistics()` calls `.expect()` on `BlockExt::txs_sizes` at line 93 without guarding against `None`. Nodes upgraded from pre-v0.106 retain old-format 5-field `BlockExt` records in their database, which are deserialized with `txs_sizes: None`. Any caller of `get_fee_rate_statistics` during the first ~101 blocks after the upgrade triggers the panic, crashing the RPC handler task.

## Finding Description
`get_block_ext` in `store/src/store.rs` (lines 247–263) branches on `count_extra_fields()`: old 5-field records (`count_extra_fields() == 0`) are converted via `packed::BlockExtReader::into()`, which is implemented in `util/types/src/conversion/storage.rs` (lines 154–166) and hard-codes `txs_sizes: None`. No migration back-fills this field.

`FeeRateCollector::collect()` in `rpc/src/util/fee_rate.rs` (lines 45–47) uses `filter_map` to skip only blocks where `get_block_ext_by_number` returns `None` (block not found). Old-format blocks return `Some(BlockExt { txs_sizes: None, ... })` — they pass through `filter_map` and reach the unconditional `.expect("expect txs_size's length >= 1")` at line 93. The panic fires for every old-format block within the query window. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

## Impact Explanation
The panic aborts the Tokio RPC handler task, causing `get_fee_rate_statistics` to fail with an unhandled panic. This is a local RPC API crash, matching the allowed CKB bounty impact: **Note (0–500 points) — Any local RPC API crash**.

## Likelihood Explanation
Any operator upgrading a long-running mainnet or testnet node from a version before v0.106 is affected for the first ~101 blocks after the upgrade. The `get_fee_rate_statistics` endpoint is unauthenticated by default. An attacker who knows the node was recently upgraded can reliably trigger the panic with a single RPC call during that window.

## Recommendation
Replace the unconditional `.expect()` at `rpc/src/util/fee_rate.rs:93` with a guard that skips blocks whose `txs_sizes` is `None`:

```rust
let Some(txs_sizes) = txs_sizes else {
    return fee_rates; // skip legacy pre-v0.106 block, no txs_sizes data
};
```

This makes `statistics()` gracefully handle legacy database records without panicking.

## Proof of Concept
1. Open a `ChainDB` containing old 5-field `BlockExt` records (from a pre-v0.106 node) within the last 101 blocks of the tip.
2. Construct a `Snapshot` over that DB.
3. Call `FeeRateCollector::new(&snapshot).statistics(Some(101))`.
4. Observe panic: `"expect txs_size's length >= 1"` at `rpc/src/util/fee_rate.rs:93`.
