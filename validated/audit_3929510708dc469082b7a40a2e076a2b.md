### Title
No Subscription Manager Transfer Function Causes Permanent Fund Lock — (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

### Summary
The Pyth Pulse `Scheduler` contract sets the subscription manager address once at creation and provides no mechanism to transfer or update it. If the manager address becomes inaccessible, all manager-gated operations — including fund withdrawal, subscription deactivation, and parameter updates — are permanently blocked with no admin rescue path.

### Finding Description
In `Scheduler.sol`, `createSubscription` records the caller as the permanent manager:

```solidity
_state.subscriptionManager[subscriptionId] = msg.sender;  // line 69
```

Every privileged operation is gated by `onlyManager`:

```solidity
modifier onlyManager(uint256 subscriptionId) {
    if (_state.subscriptionManager[subscriptionId] != msg.sender) {
        revert SchedulerErrors.Unauthorized();
    }
    _;
}
```

`updateSubscription`, `withdrawFunds` both carry this modifier. There is no `transferManager`, no admin override in `SchedulerGovernance`, and no entry in `SchedulerState` for a pending manager transfer. The `subscriptionManager` mapping is write-once. ### Impact Explanation
If the manager address becomes inaccessible (smart contract manager that is later upgraded/self-destructed, multisig that loses quorum, EOA key loss), the following become permanently impossible:

- **`withdrawFunds`** — ETH balance is locked forever in the subscription.
- **`updateSubscription`** — subscription parameters cannot be changed, including deactivation.
- **Subscription deactivation** — the subscription remains active and continues draining its balance to keepers until it hits zero, after which it becomes permanently inert but still unrecoverable.

For non-permanent subscriptions, the entire ETH balance (which can be substantial, as there is no upper cap for non-permanent subscriptions) is irrecoverably locked. The `SchedulerGovernance` contract has no admin function to reassign a manager. [6](#0-5) [7](#0-6) 

### Likelihood Explanation
Subscriptions are created by `msg.sender`, which is commonly a smart contract (e.g., a DeFi protocol's router or proxy). Smart contracts are routinely upgraded, replaced, or deprecated. A protocol that creates a subscription via a contract that is later upgraded to a new address loses all management rights permanently. This is a standard operational pattern and requires no attacker — it is a self-inflicted but protocol-enabled permanent lockout. The entry path is fully unprivileged: any address can call `createSubscription`. [8](#0-7) 

### Recommendation
Add a two-step manager transfer function analogous to the existing `proposeAdmin`/`acceptAdmin` pattern already used in `SchedulerGovernance`:

```solidity
// In SchedulerState.State:
mapping(uint256 => address) pendingManager;

// New functions in Scheduler:
function proposeManagerTransfer(uint256 subscriptionId, address newManager)
    external onlyManager(subscriptionId) {
    _state.pendingManager[subscriptionId] = newManager;
}

function acceptManagerTransfer(uint256 subscriptionId) external {
    require(msg.sender == _state.pendingManager[subscriptionId], "Not pending manager");
    _state.subscriptionManager[subscriptionId] = msg.sender;
    delete _state.pendingManager[subscriptionId];
}
```

Optionally, allow the contract admin to force-reassign a manager as a last-resort rescue path. [9](#0-8) 

### Proof of Concept
1. A DeFi protocol deploys `ProtocolV1` contract and calls `scheduler.createSubscription{value: 10 ether}(params)`. The `subscriptionManager[id]` is set to `address(ProtocolV1)`.
2. The protocol upgrades to `ProtocolV2` at a new address. `ProtocolV1` is deprecated and no longer callable.
3. `ProtocolV2` attempts `scheduler.withdrawFunds(id, 10 ether)` — reverts with `Unauthorized` because `msg.sender == address(ProtocolV2) != subscriptionManager[id]`.
4. No admin function exists in `SchedulerGovernance` to reassign the manager.
5. The 10 ETH is permanently locked. The subscription continues to drain to keepers until balance reaches zero, then becomes permanently inert. [2](#0-1) [10](#0-9)

    function withdrawFunds(
        uint256 subscriptionId,
        uint256 amount
    ) external override onlyManager(subscriptionId) {
        SchedulerStructs.SubscriptionStatus storage status = _state
            .subscriptionStatuses[subscriptionId];
        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        // Prevent withdrawals from permanent subscriptions
        if (params.isPermanent) {
            revert SchedulerErrors.CannotUpdatePermanentSubscription();
        }

        if (status.balanceInWei < amount) {
            revert SchedulerErrors.InsufficientBalance();
        }

        // If subscription is active, ensure minimum balance is maintained
        if (params.isActive) {
            uint256 minimumBalance = this.getMinimumBalance(
                uint8(params.priceIds.length)
            );
            if (status.balanceInWei - amount < minimumBalance) {
                revert SchedulerErrors.InsufficientBalance();
            }
        }

        status.balanceInWei -= amount;

        (bool sent, ) = msg.sender.call{value: amount}("");
        require(sent, "Failed to send funds");
    }

// SPDX-License-Identifier: Apache 2
pragma solidity ^0.8.0;

import "./SchedulerState.sol";
import "@pythnetwork/pulse-sdk-solidity/SchedulerErrors.sol";

/**
 * @dev `SchedulerGovernance` defines governance capabilities for the Pulse contract.
 */
abstract contract SchedulerGovernance is SchedulerState {
    event NewAdminProposed(address oldAdmin, address newAdmin);
    event NewAdminAccepted(address oldAdmin, address newAdmin);
    event SingleUpdateKeeperFeeSet(uint oldFee, uint newFee);
    event MinimumBalancePerFeedSet(uint oldBalance, uint newBalance);

    /**
     * @dev Returns the address of the proposed admin.
     */
    function proposedAdmin() public view virtual returns (address) {
        return _state.proposedAdmin;
    }

    /**
     * @dev Returns the address of the current admin.
     */
    function getAdmin() external view returns (address) {
        return _state.admin;
    }

    /**
     * @dev Proposes a new admin for the contract. Replaces the proposed admin if there is one.
     * Can only be called by either admin or owner.
     */
    function proposeAdmin(address newAdmin) public virtual {
        require(newAdmin != address(0), "newAdmin is zero address");

        _authorizeAdminAction();

        _state.proposedAdmin = newAdmin;
        emit NewAdminProposed(_state.admin, newAdmin);
    }

    /**
     * @dev The proposed admin accepts the admin transfer.
     */
    function acceptAdmin() external {
        if (msg.sender != _state.proposedAdmin)
            revert SchedulerErrors.Unauthorized();

        address oldAdmin = _state.admin;
        _state.admin = msg.sender;

        _state.proposedAdmin = address(0);
        emit NewAdminAccepted(oldAdmin, msg.sender);
    }

    /**
     * @dev Authorization check for admin actions
     * Must be implemented by the inheriting contract.
     */
    function _authorizeAdminAction() internal virtual;

    /**
     * @dev Set the keeper fee for single updates in Wei.
     * Calls {_authorizeAdminAction}.
     * Emits a {SingleUpdateKeeperFeeSet} event.
     */
    function setSingleUpdateKeeperFeeInWei(uint128 newFee) external {
        _authorizeAdminAction();

        uint oldFee = _state.singleUpdateKeeperFeeInWei;
        _state.singleUpdateKeeperFeeInWei = newFee;

        emit SingleUpdateKeeperFeeSet(oldFee, newFee);
    }

    /**
     * @dev Set the minimum balance required per feed in a subscription.
     * Calls {_authorizeAdminAction}.
     * Emits a {MinimumBalancePerFeedSet} event.
     */
    function setMinimumBalancePerFeed(uint128 newMinimumBalance) external {
        _authorizeAdminAction();

        uint oldBalance = _state.minimumBalancePerFeed;
        _state.minimumBalancePerFeed = newMinimumBalance;

        emit MinimumBalancePerFeedSet(oldBalance, newMinimumBalance);
    }
}
