Audit Report

## Title
Single-Sample Epoch Check in `need_clean` Allows Expired Orphan Subtree Eviction Bypass — (`chain/src/utils/orphan_block_pool.rs`)

## Summary

`InnerPool::need_clean` samples exactly one arbitrarily-ordered `HashMap` entry to decide whether to evict an entire leader's subtree. If that entry belongs to a fresh-epoch block, `clean_expired_blocks` skips the entire subtree on every cleanup cycle, permanently retaining all expired-epoch orphans under the same leader. Because `insert()` has no hard capacity limit, this allows unbounded memory and disk growth, crashing the node.

## Finding Description

`need_clean` at lines 113–122 of `chain/src/utils/orphan_block_pool.rs` reads:

```rust
fn need_clean(&self, parent_hash: &packed::Byte32, tip_epoch: EpochNumber) -> bool {
    self.blocks
        .get(parent_hash)
        .and_then(|map| {
            map.iter().next().map(|(_, lonely_block)| {   // only ONE entry checked
                lonely_block.epoch_number() + EXPIRED_EPOCH < tip_epoch
            })
        })
        .unwrap_or_default()
}
``` [1](#0-0) 

The inner map is `HashMap<packed::Byte32, LonelyBlockHash>` (line 18). `map.iter().next()` returns whichever entry occupies the first internal bucket, which is non-deterministic across runs but fixed for a given map state. `clean_expired_blocks` (lines 99–110) calls `need_clean` for every leader and removes the subtree only when it returns `true`. [2](#0-1) 

If the sampled entry is a fresh-epoch block, the entire subtree — including all expired-epoch siblings — is never evicted. `insert()` (lines 36–54) has no size guard; `with_capacity` sets only the initial allocation, not a maximum. [3](#0-2) 

The exploit path is reachable via P2P. In `orphan_broker.rs`, `process_lonely_block` inserts a block into the orphan pool at line 122 when the parent hash is unknown (not stored, not invalid, not pending verify), after only a parent-status check: [4](#0-3) 

An attacker:
1. Connects via P2P and sends blocks whose parent hash does not exist in the chain.
2. Inserts a large batch of expired-epoch orphans (epoch + `EXPIRED_EPOCH` < `tip_epoch`) under a chosen leader hash.
3. Inserts K fresh-epoch siblings under the same leader hash. With K fresh siblings among N expired, the probability that a fresh entry is sampled by `map.iter().next()` is ≈ K/(K+N), approaching certainty as K grows.
4. Once a fresh entry is sampled, `need_clean` returns `false` on every subsequent `clean_expired_orphans` call, permanently bypassing eviction for that subtree.

The existing test `test_remove_expired_blocks` (lines 233–262 of `chain/src/tests/orphan_block_pool.rs`) only covers the all-expired case and does not test the mixed-epoch sibling scenario. [5](#0-4) 

## Impact Explanation

**High — Vulnerability which could easily crash a CKB node.**

Unbounded growth of the `blocks` and `parents` HashMaps and the on-disk unverified block store leads to memory and disk exhaustion, crashing the targeted node.

## Likelihood Explanation

The attacker controls the number of fresh-epoch siblings inserted. With K fresh siblings, the probability of bypassing cleanup approaches K/(K+N). The attacker can retry with a new leader hash if the first attempt fails, and each attempt requires only a P2P connection and the ability to craft blocks with a chosen parent hash. The condition is practically achievable with modest effort.

## Recommendation

Replace the single-sample check with a check over all direct children of the leader, using the minimum epoch:

```rust
fn need_clean(&self, parent_hash: &packed::Byte32, tip_epoch: EpochNumber) -> bool {
    self.blocks
        .get(parent_hash)
        .map(|map| {
            map.values().all(|b| b.epoch_number() + EXPIRED_EPOCH < tip_epoch)
        })
        .unwrap_or_default()
}
```

This ensures the subtree is only evicted when every direct child is expired. Additionally, enforce a hard capacity limit in `insert()` to bound pool size independently of cleanup correctness.

## Proof of Concept

Add the following test to `chain/src/tests/orphan_block_pool.rs`:

```rust
#[test]
fn test_remove_expired_blocks_mixed_epoch() {
    let tip_epoch = 20_u64;
    let expired_epoch = EpochNumberWithFraction::new(1, 0, 10);
    let fresh_epoch   = EpochNumberWithFraction::new(15, 0, 10);

    let pool = OrphanBlockPool::with_capacity(1100);
    let consensus = ConsensusBuilder::default().build();
    let leader_hash = consensus.genesis_block().hash();

    for i in 0..1000_u64 {
        let block = BlockBuilder::default()
            .parent_hash(leader_hash.clone())
            .epoch(expired_epoch)
            .nonce(i)
            .build();
        pool.insert(LonelyBlock { block: Arc::new(block), switch: None, verify_callback: None }.into());
    }

    let fresh_block = BlockBuilder::default()
        .parent_hash(leader_hash.clone())
        .epoch(fresh_epoch)
        .nonce(9999_u64)
        .build();
    pool.insert(LonelyBlock { block: Arc::new(fresh_block), switch: None, verify_callback: None }.into());

    let evicted = pool.clean_expired_blocks(tip_epoch);
    // With the current single-sample implementation this will non-deterministically fail:
    assert_eq!(evicted.len(), 1000);
}
```

Running this test repeatedly demonstrates non-deterministic failures under the current implementation. With the `all()`-based fix, the test passes deterministically.

fn test_remove_expired_blocks() {
    let consensus = ConsensusBuilder::default().build();
    let block_number = 20;
    let mut parent = consensus.genesis_block().header();
    let pool = OrphanBlockPool::with_capacity(block_number);

    let deprecated = EpochNumberWithFraction::new(10, 0, 10);

    for _ in 1..block_number {
        let new_block = BlockBuilder::default()
            .parent_hash(parent.hash())
            .timestamp(unix_time_as_millis())
            .number(parent.number() + 1)
            .epoch(deprecated)
            .nonce(parent.nonce() + 1)
            .build();

        parent = new_block.header();
        let lonely_block = LonelyBlock {
            block: Arc::new(new_block),
            switch: None,
            verify_callback: None,
        };
        pool.insert(lonely_block.into());
    }
    assert_eq!(pool.leaders_len(), 1);

    let v = pool.clean_expired_blocks(20_u64);
    assert_eq!(v.len(), 19);
    assert_eq!(pool.leaders_len(), 0);
