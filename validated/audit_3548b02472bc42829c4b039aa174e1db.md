### Title
Hardcoded `CorsLayer::permissive()` Allows Any Cross-Origin Webpage to Invoke State-Modifying RPC Methods Against a Local CKB Node - (File: rpc/src/server.rs)

### Summary
The CKB RPC HTTP server unconditionally applies `CorsLayer::permissive()` with no operator-configurable restriction. Any webpage a node operator visits can issue cross-origin JSON-RPC calls — including `clear_tx_pool`, `remove_transaction`, and `send_transaction` — against the locally running node without any authentication or origin check.

### Finding Description
In `rpc/src/server.rs`, the `start_server` function builds the axum router and unconditionally layers `CorsLayer::permissive()` onto every HTTP and WebSocket endpoint:

```rust
let app = Router::new()
    .route("/", method_router.clone())
    .route("/{*path}", method_router)
    .route("/ping", get(ping_handler))
    .layer(Extension(Arc::clone(rpc)))
    .layer(CorsLayer::permissive())   // ← hardcoded, no config knob
    ...
```

`tower_http::cors::CorsLayer::permissive()` responds to every preflight with:
```
Access-Control-Allow-Origin: <mirrored request origin>
Access-Control-Allow-Methods: *
Access-Control-Allow-Headers: *
```

This satisfies the browser's CORS check for non-simple requests (e.g., `POST` with `Content-Type: application/json`), so the browser will forward the actual RPC payload and expose the response body to the calling script.

The `RpcConfig` struct in `util/app-config/src/configs/rpc.rs` contains no CORS-related field; there is no way for an operator to tighten this policy without patching the binary.

### Impact Explanation
A malicious webpage visited by a CKB node operator can silently:

| RPC method | Effect |
|---|---|
| `clear_tx_pool` | Wipes the entire pending transaction pool |
| `remove_transaction` | Evicts a specific transaction from the pool |
| `send_transaction` | Injects attacker-crafted transactions into the pool and relays them to peers |
| `get_raw_tx_pool` | Reads all pending transaction hashes/details (information disclosure) |
| `get_block_template` | Reads the current mining template (useful for targeted pool disruption) |

`clear_tx_pool` and `remove_transaction` are the highest-severity primitives: they let an attacker silently evict legitimate user transactions from the pool, causing them to miss block inclusion and potentially expire, with no on-chain trace of the attack.

### Likelihood Explanation
The default `listen_address` is `127.0.0.1:8114`. Any webpage the operator opens in a browser on the same machine can reach this address. The attack requires only that the operator visit a malicious (or compromised) webpage while the node is running — a realistic scenario for developers, miners, and dapp operators who run local nodes. No special privileges, leaked keys, or network access beyond the local machine are required.

### Recommendation
- **Short term**: Replace `CorsLayer::permissive()` with a restrictive default (e.g., `CorsLayer::new()` with no allowed origins) and add an `allowed_origins: Vec<String>` field to `RpcConfig` so operators can explicitly whitelist trusted origins (e.g., a local dapp dev server).
- **Long term**: Add an `Origin` allowlist check as middleware before dispatching any RPC call, and document the CORS posture in the RPC security warning already present in `rpc/README.md`.

### Proof of Concept
From any webpage loaded in the operator's browser:

```javascript
// Clears the entire tx pool of the locally running CKB node
fetch("http://127.0.0.1:8114/", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    id: 1,
    jsonrpc: "2.0",
    method: "clear_tx_pool",
    params: []
  })
}).then(r => r.json()).then(console.log);
```

Because `CorsLayer::permissive()` mirrors the requesting origin in `Access-Control-Allow-Origin`, the browser's preflight succeeds, the POST is sent, and the response body is readable by the script. The pool is wiped with no authentication required.

    fn send_transaction(
        &self,
        tx: Transaction,
        outputs_validator: Option<OutputsValidator>,
    ) -> Result<H256> {
        let tx: packed::Transaction = tx.into();
        let tx: core::TransactionView = tx.into_view();

        self.check_output_validator(outputs_validator, &tx)?;

        let tx_pool = self.shared.tx_pool_controller();
        let submit_tx = tx_pool.submit_local_tx(tx.clone());

        if let Err(e) = submit_tx {
            error!("Send submit_tx request error {}", e);
            return Err(RPCError::ckb_internal_error(e));
        }

        let tx_hash = tx.hash();
        match submit_tx.unwrap() {
            Ok(_) => Ok(tx_hash.into()),
            Err(reject) => Err(RPCError::from_submit_transaction_reject(&reject)),
        }
    }
