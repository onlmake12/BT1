### Title
Gameable `connected_time` Protection in `try_evict_inbound_peer` Allows Attacker to Permanently Occupy Half of Inbound Slots — (`network/src/peer_registry.rs`)

---

### Summary

The connection-time protection tier in `try_evict_inbound_peer` sorts candidates so that the **oldest** connections are dropped from the eviction pool (i.e., protected). An attacker who establishes inbound connections before honest peers can permanently occupy those protected slots, ensuring honest peers are always the eviction candidates when the registry is full.

---

### Finding Description

**`sort_then_drop` semantics** (lines 55–63):

```rust
fn sort_then_drop<T, F>(list: &mut Vec<T>, n: usize, compare: F) {
    list.sort_by(compare);
    if list.len() > n {
        list.truncate(list.len() - n);  // keeps first (len-n), DROPS last n
    }
}
```

The last `n` elements after sorting are removed from the candidate list — they are "protected" from eviction.

**Connection-time protection tier** (lines 184–188):

```rust
let protect_peers = candidate_peers.len() >> 1;
sort_then_drop(&mut candidate_peers, protect_peers, |peer1, peer2| {
    peer2.connected_time.cmp(&peer1.connected_time)  // descending: newest first
});
```

The comparator `peer2.connected_time.cmp(&peer1.connected_time)` sorts **newest-first**. After sorting:

```
[newest_connection, ..., oldest_connection]
```

`truncate(len - protect_peers)` keeps the first half (newest) and drops the last half (oldest). The **oldest half is protected** (removed from candidates). The **newest half remains as eviction candidates**.

**Attack path:**

1. Attacker connects `M/2` inbound peers (from diverse `/16` subnets to survive network-group selection) at time `T-100s`, before honest peers arrive.
2. Honest peers connect at `T-1s`, filling the remaining `M/2` slots.
3. A new honest peer arrives, triggering `try_evict_inbound_peer`.
4. After ping and message-recency protections (which the attacker can also satisfy by maintaining low RTT and sending periodic pings), the connection-time sort runs.
5. Attacker peers have older `connected_time` → they land at the **end** of the descending sort → they are **dropped from candidates** (protected).
6. Honest peers have newer `connected_time` → they land at the **front** → they **remain as eviction candidates**.
7. One honest peer is evicted. The new honest peer takes the slot. On the next connection attempt, the same thing repeats.

The attacker's `M/2` slots are never evicted as long as they maintain their connections. [1](#0-0) [2](#0-1) 

---

### Impact Explanation

The attacker permanently occupies `M/2` inbound connection slots (where `M = max_inbound`). Honest peers are perpetually evicted whenever the registry is full. This:

- Reduces honest peer diversity in the inbound connection set by half.
- Enables a sustained partial eclipse: the attacker controls half of all inbound connections indefinitely.
- Combined with influence over outbound connections (via separate attack vectors), this can escalate to a full eclipse, enabling transaction censorship, double-spend facilitation, or mining isolation. [3](#0-2) 

---

### Likelihood Explanation

- **IP diversity requirement**: The attacker needs `M/2` peers from distinct network groups (`/16` subnets). This is achievable with a botnet or cloud provider diversity. The network-group eviction step (lines 191–203) only selects from the unprotected honest peers after the connection-time sort, so it does not help.
- **Timing requirement**: The attacker must connect before honest peers. This is realistic for newly started nodes, nodes recovering from downtime, or nodes whose peer store has been cleared.
- **Maintenance**: The attacker must keep connections alive and optionally maintain low ping / recent messages to also survive the earlier protection tiers (lines 151–183). This is trivially achievable. [4](#0-3) [5](#0-4) 

---

### Recommendation

1. **Randomize the connection-time protection selection**: Instead of protecting the oldest half deterministically, randomly sample from all candidates weighted by connection age. This prevents an attacker from guaranteeing protection by connecting early.
2. **Add network-group diversity enforcement to the protection tiers**: Ensure that no single network group can occupy more than one protection slot in the connection-time tier.
3. **Cap per-IP/per-subnet inbound slots**: Limit how many inbound connections from the same `/16` subnet can be accepted, reducing the attacker's ability to fill protection slots with diverse-looking but attacker-controlled peers.
4. **Consider a time-bounded protection**: Only protect connections that have been alive for longer than some threshold AND have demonstrated useful behavior (e.g., relayed valid blocks/transactions).

    fn try_evict_inbound_peer(&self, _peer_store: &PeerStore) -> Option<SessionId> {
        let mut candidate_peers = {
            self.peers
                .values()
                .filter(|peer| peer.is_inbound() && !peer.is_whitelist)
                .collect::<Vec<_>>()
        };
        // Protect peers based on characteristics that an attacker hard to simulate or manipulate
        // Protect peers which has the lowest ping
        sort_then_drop(
            &mut candidate_peers,
            EVICTION_PROTECT_PEERS,
            |peer1, peer2| {
                let peer1_ping = peer1
                    .ping_rtt
                    .map(|p| p.as_secs())
                    .unwrap_or_else(|| u64::MAX);
                let peer2_ping = peer2
                    .ping_rtt
                    .map(|p| p.as_secs())
                    .unwrap_or_else(|| u64::MAX);
                peer2_ping.cmp(&peer1_ping)
            },
        );

        // Protect peers which most recently sent messages
        sort_then_drop(
            &mut candidate_peers,
            EVICTION_PROTECT_PEERS,
            |peer1, peer2| {
                let now = Instant::now();
                let peer1_last_message = peer1
                    .last_ping_protocol_message_received_at
                    .map(|t| now.saturating_duration_since(t).as_secs())
                    .unwrap_or_else(|| u64::MAX);
                let peer2_last_message = peer2
                    .last_ping_protocol_message_received_at
                    .map(|t| now.saturating_duration_since(t).as_secs())
                    .unwrap_or_else(|| u64::MAX);
                peer2_last_message.cmp(&peer1_last_message)
            },
        );
        // Protect half peers which have the longest connection time
        let protect_peers = candidate_peers.len() >> 1;
        sort_then_drop(&mut candidate_peers, protect_peers, |peer1, peer2| {
            peer2.connected_time.cmp(&peer1.connected_time)
        });

        // Group peers by network group
        let evict_group = candidate_peers
            .into_iter()
            .fold(
                HashMap::new(),
                |mut groups: HashMap<Group, Vec<&Peer>>, peer| {
                    groups.entry(peer.network_group()).or_default().push(peer);
                    groups
                },
            )
            .values()
            .max_by_key(|group| group.len())
            .cloned()
            .unwrap_or_default();

        // randomly evict a peer
        let mut rng = thread_rng();
        evict_group.choose(&mut rng).map(|peer| {
            debug!("Disconnect inbound peer {:?}", peer.connected_addr);
            peer.session_id
        })
    }
