### Title
Zero `callbackGasLimit` Bypasses Gas Fee Component in Echo, Enabling Underpaid Request Spam - (`target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary

`Echo.requestPriceUpdatesWithCallback` accepts `callbackGasLimit = 0` without any minimum enforcement. Because the fee formula multiplies `callbackGasLimit * feePerGasInWei`, a zero gas limit produces a zero gas fee component. The request is stored and must be fulfilled by the provider, but the provider receives no compensation for the callback gas cost. The callback itself always fails (called with `gas: 0`), yet the request is cleared and the provider is credited — mirroring the Optimism gasless deposit pattern exactly.

---

### Finding Description

`getFee` computes the total fee as:

```
feeAmount = pythFeeInWei + baseFeeInWei + (priceIds.length * feePerFeedInWei) + (callbackGasLimit * feePerGasInWei)
``` [1](#0-0) 

When `callbackGasLimit = 0`, the entire gas fee term is zero. `requestPriceUpdatesWithCallback` enforces only `msg.value >= requiredFee`, which is satisfied at the minimum base fee: [2](#0-1) 

There is no lower-bound check on `callbackGasLimit`. The request is stored with `req.callbackGasLimit = 0`: [3](#0-2) 

When `executeCallback` is later called by the provider, the callback is dispatched with `gas: req.callbackGasLimit`, i.e., `gas: 0`: [4](#0-3) 

The sub-call immediately runs out of gas and falls into the `catch` block, emitting `PriceUpdateCallbackFailed`. Critically, **before** the callback attempt, the provider is already credited and the request is already cleared: [5](#0-4) 

The provider must pay the full gas cost of `executeCallback` (Pyth parsing, storage writes, the failed sub-call overhead) but receives only `req.fee`, which contains zero gas compensation.

---

### Impact Explanation

An attacker can flood the Echo contract with requests at minimum cost (only `pythFeeInWei + baseFeeInWei + N * feePerFeedInWei`), bypassing the `feePerGasInWei` resource metering entirely. Each such request:

1. Occupies one of the 32 fixed request slots (or the overflow mapping), potentially blocking legitimate requests.
2. Forces the provider to execute `executeCallback` — paying for Pyth fee parsing, storage operations, and the failed callback — without receiving gas compensation.
3. Always results in a failed callback, so the consumer never receives price data, making the attack purely griefing/DoS.

The `NUM_REQUESTS = 32` fixed-size ring buffer means 32 simultaneous zero-gas requests can saturate the slot space at near-minimum cost. [6](#0-5) 

---

### Likelihood Explanation

Any unprivileged user can call `requestPriceUpdatesWithCallback` with `callbackGasLimit = 0` and a registered provider address. No special role or key is required. The cost to the attacker is only the base fees per request. The attack is directly reachable from the public interface.

---

### Recommendation

Enforce a minimum `callbackGasLimit` in `requestPriceUpdatesWithCallback`. At minimum, reject zero:

```solidity
require(callbackGasLimit > 0, "callbackGasLimit must be > 0");
```

A stronger fix mirrors the Entropy contract's approach: enforce a provider-configured `defaultGasLimit` as a floor, so that users always pay at least the intrinsic gas cost of a callback invocation. [7](#0-6) 

---

### Proof of Concept

```solidity
// Attacker calls with callbackGasLimit = 0
uint96 minFee = echo.getFee(provider, 0, priceIds);
// minFee = pythFeeInWei + baseFeeInWei + N*feePerFeedInWei  (no gas component)

echo.requestPriceUpdatesWithCallback{value: minFee}(
    provider,
    block.timestamp,
    priceIds,
    0  // callbackGasLimit = 0
);

// Provider must call executeCallback to clear the request.
// The callback is dispatched with gas: 0, immediately fails.
// Provider pays full executeCallback gas cost, receives no gas compensation.
// PriceUpdateCallbackFailed is emitted; consumer gets nothing.
```

Repeat 32 times to saturate the fixed request buffer at minimum cost. [8](#0-7)

    function requestPriceUpdatesWithCallback(
        address provider,
        uint64 publishTime,
        bytes32[] calldata priceIds,
        uint32 callbackGasLimit
    ) external payable override returns (uint64 requestSequenceNumber) {
        require(
            _state.providers[provider].isRegistered,
            "Provider not registered"
        );

        // FIXME: this comment is wrong. (we're not using tx.gasprice)
        // NOTE: The 60-second future limit on publishTime prevents a DoS vector where
        //      attackers could submit many low-fee requests for far-future updates when gas prices
        //      are low, forcing executors to fulfill them later when gas prices might be much higher.
        //      Since tx.gasprice is used to calculate fees, allowing far-future requests would make
        //      the fee estimation unreliable.
        require(publishTime <= block.timestamp + 60, "Too far in future");
        if (priceIds.length > MAX_PRICE_IDS) {
            revert TooManyPriceIds(priceIds.length, MAX_PRICE_IDS);
        }
        requestSequenceNumber = _state.currentSequenceNumber++;

        uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
        if (msg.value < requiredFee) revert InsufficientFee();

        Request storage req = allocRequest(requestSequenceNumber);
        req.sequenceNumber = requestSequenceNumber;
        req.publishTime = publishTime;
        req.callbackGasLimit = callbackGasLimit;
        req.requester = msg.sender;
        req.provider = provider;
        req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);

        // Create array with the right size
        req.priceIdPrefixes = new bytes8[](priceIds.length);

        // Copy only the first 8 bytes of each price ID to storage
        for (uint8 i = 0; i < priceIds.length; i++) {
            // Extract first 8 bytes of the price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }
            req.priceIdPrefixes[i] = prefix;
        }
        _state.accruedFeesInWei += _state.pythFeeInWei;

        emit PriceUpdateRequested(req, priceIds);
    }
