### Title
Unbounded `activeSubscriptionIds` Growth via Subscription Spam Causes Keeper DoS — (`target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

### Summary
Any unprivileged user can call `createSubscription` with only the minimum balance and immediately deactivate the subscription to recover their ETH, paying only gas. By repeating this at scale, an attacker inflates `_state.activeSubscriptionIds` with stale or underfunded entries. The Argus keeper must iterate the entire list on every polling cycle to discover which subscriptions need updating. With enough spam subscriptions, the keeper's polling loop is overwhelmed, causing legitimate subscriptions to miss their heartbeat or deviation update windows.

### Finding Description

`createSubscription` in `Scheduler.sol` has no cap on the total number of subscriptions that can exist simultaneously. [1](#0-0) 

Every new active subscription is appended to `_state.activeSubscriptionIds`, an unbounded dynamic array: [2](#0-1) 

The array is declared with no size limit in `SchedulerState.sol`: [3](#0-2) 

The Argus keeper's `SubscriptionService` calls `get_active_subscriptions()` on every polling tick to discover all subscriptions that need updating: [4](#0-3) 

`getActiveSubscriptions` is paginated on-chain, but the keeper calls it without an explicit page limit, meaning it must make as many RPC calls as needed to drain the full list: [5](#0-4) 

An attacker can create subscriptions with minimum balance, then deactivate them to recover the ETH (paying only gas), leaving the entries removed from the array. However, by continuously cycling — create → let keeper poll → deactivate → repeat — or by simply holding thousands of active subscriptions simultaneously, the attacker keeps the array large. Subscriptions whose balance has been drained below the keeper fee remain `isActive = true` in the array but cause every keeper `updatePriceFeeds` attempt to revert: [6](#0-5) 

There is no automatic deactivation when a subscription's balance falls below the keeper fee, so underfunded subscriptions persist in `activeSubscriptionIds` indefinitely until the manager explicitly deactivates them.

The minimum balance is `minimumBalancePerFeed × numPriceFeeds`. With 1 price feed, the cost per spam subscription is just `minimumBalancePerFeed` (recoverable) plus gas: [7](#0-6) 

### Impact Explanation

The Argus keeper is the sole mechanism that triggers `updatePriceFeeds` for subscriptions. If the keeper's polling loop is overwhelmed by thousands of spam entries, legitimate subscriptions miss their heartbeat or deviation update windows. Protocols relying on the Scheduler for on-chain price freshness receive stale prices, which can cause incorrect liquidations, mispriced derivatives, or other financial losses for downstream users.

### Likelihood Explanation

The attack is cheap: the attacker deposits the minimum balance, creates subscriptions, and can recover the ETH by deactivating them. The net cost is only gas. On low-fee chains (e.g., Arbitrum, Base) where the Scheduler is deployed, creating thousands of subscriptions costs a few dollars. No privileged access is required — `createSubscription` is fully permissionless.

### Recommendation

1. **Impose a global cap** on the total number of active subscriptions (e.g., `MAX_ACTIVE_SUBSCRIPTIONS`) enforced in `createSubscription` and `updateSubscription`.
2. **Automatically deactivate** subscriptions whose balance falls below the keeper fee threshold, removing them from `activeSubscriptionIds`.
3. **Add a per-address subscription limit** to raise the cost of a spam campaign.
4. **Increase the minimum balance** to a level that makes mass subscription creation economically infeasible.

### Proof of Concept

```solidity
// Attacker script (pseudocode)
for (uint i = 0; i < 10_000; i++) {
    // Create subscription with 1 price feed, minimum balance
    uint256 minBal = scheduler.getMinimumBalance(1);
    uint256 subId = scheduler.createSubscription{value: minBal}(spamParams);
    // Optionally deactivate immediately to recover ETH, leaving keeper to
    // discover the subscription was active during the polling window
    // OR hold all subscriptions active to maximize keeper polling load
}
// Keeper now must iterate 10,000 entries on every poll cycle.
// Legitimate subscriptions with 60-second heartbeats miss their windows
// because the keeper's per-cycle work exceeds the heartbeat interval.
```

The benchmark in `PulseSchedulerGasBenchmark.t.sol` already demonstrates that `getActiveSubscriptions(0, 1000)` is measurably expensive at 1,000 subscriptions: [8](#0-7) 

Scaling to 10,000–100,000 subscriptions makes the keeper's polling loop untenable within any reasonable heartbeat window.

    function createSubscription(
        SchedulerStructs.SubscriptionParams memory subscriptionParams
    ) external payable override returns (uint256 subscriptionId) {
        _validateSubscriptionParams(subscriptionParams);

        // Calculate minimum balance required for this subscription
        uint256 minimumBalance = this.getMinimumBalance(
            uint8(subscriptionParams.priceIds.length)
        );

        // Ensure enough funds were provided
        if (msg.value < minimumBalance) {
            revert SchedulerErrors.InsufficientBalance();
        }

        // Check deposit limit for permanent subscriptions
        if (subscriptionParams.isPermanent && msg.value > MAX_DEPOSIT_LIMIT) {
            revert SchedulerErrors.MaxDepositLimitExceeded();
        }

        // Set subscription to active
        subscriptionParams.isActive = true;

        subscriptionId = _state.subscriptionNumber++;

        // Store the subscription parameters
        _state.subscriptionParams[subscriptionId] = subscriptionParams;

        // Initialize subscription status
        SchedulerStructs.SubscriptionStatus storage status = _state
            .subscriptionStatuses[subscriptionId];
        status.priceLastUpdatedAt = 0;
        status.balanceInWei = msg.value;
        status.totalUpdates = 0;
        status.totalSpent = 0;

        // Map subscription ID to manager
        _state.subscriptionManager[subscriptionId] = msg.sender;

        _addToActiveSubscriptions(subscriptionId);

        emit SubscriptionCreated(subscriptionId, msg.sender);
        return subscriptionId;
    }

    function getActiveSubscriptions(
        uint256 startIndex,
        uint256 maxResults
    )
        external
        view
        override
        returns (
            uint256[] memory subscriptionIds,
            SchedulerStructs.SubscriptionParams[] memory subscriptionParams,
            uint256 totalCount
        )
    {
        totalCount = _state.activeSubscriptionIds.length;

        // If startIndex is beyond the total count, return empty arrays
        if (startIndex >= totalCount) {
            return (
                new uint256[](0),
                new SchedulerStructs.SubscriptionParams[](0),
                totalCount
            );
        }

        // Calculate how many results to return (bounded by maxResults and remaining items)
        uint256 resultCount = totalCount - startIndex;
        if (resultCount > maxResults) {
            resultCount = maxResults;
        }

        // Create arrays for subscription IDs and parameters
        subscriptionIds = new uint256[](resultCount);
        subscriptionParams = new SchedulerStructs.SubscriptionParams[](
            resultCount
        );

        // Populate the arrays with the requested page of active subscriptions
        for (uint256 i = 0; i < resultCount; i++) {
            uint256 subscriptionId = _state.activeSubscriptionIds[
                startIndex + i
            ];
            subscriptionIds[i] = subscriptionId;
            subscriptionParams[i] = _state.subscriptionParams[subscriptionId];
        }

        return (subscriptionIds, subscriptionParams, totalCount);
    }

    function _addToActiveSubscriptions(uint256 subscriptionId) internal {
        // Only add if not already in the list
        if (_state.activeSubscriptionIndex[subscriptionId] == 0) {
            _state.activeSubscriptionIds.push(subscriptionId);

            // Store the index as 1-based, 0 means not in the list
            _state.activeSubscriptionIndex[subscriptionId] = _state
                .activeSubscriptionIds
                .length;
        }
    }

    async fn refresh_subscriptions(&self) -> Result<()> {
        match self.contract.get_active_subscriptions().await {
            Ok(subscriptions) => {
                tracing::info!(
                    service_name = self.name,
                    subscription_count = subscriptions.len(),
                    "Retrieved active subscriptions"
                );

                self.subscription_state.update_subscriptions(subscriptions);

                let feed_ids = self.subscription_state.get_feed_ids();
                self.pyth_price_state.update_feed_ids(feed_ids.clone());
                self.chain_price_state.update_feed_ids(feed_ids);

                Ok(())
            }
            Err(e) => {
                tracing::error!(
                    service_name = self.name,
                    error = %e,
                    "Failed to load active subscriptions"
                );
                Err(e)
            }
        }
    }

    function _runGetActiveSubscriptionsBenchmark(
        uint256 numSubscriptions
    ) internal {
        // Setup: As manager, create subscriptions and then deactivate every other one.
        vm.startPrank(manager);

        // Array to store subscription IDs
        uint256[] memory subscriptionIds = new uint256[](numSubscriptions);

        // First create all subscriptions as active (with default 2 price feeds)
        for (uint256 i = 0; i < numSubscriptions; i++) {
            subscriptionIds[i] = addTestSubscription(
                scheduler,
                address(manager)
            );
        }

        // Deactivate every other subscription
        for (uint256 i = 0; i < numSubscriptions; i++) {
            if (i % 2 == 1) {
                (
                    SchedulerStructs.SubscriptionParams memory params,

                ) = scheduler.getSubscription(subscriptionIds[i]);
                params.isActive = false;
                scheduler.updateSubscription(subscriptionIds[i], params);
            }
        }
        vm.stopPrank();

        // Actual benchmark: Measure gas for fetching active subscriptions
        uint256 startGas = gasleft();
        scheduler.getActiveSubscriptions(0, numSubscriptions);
        uint256 gasUsed = startGas - gasleft();
