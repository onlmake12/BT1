Audit Report

## Title
Unconditional `.expect()` on `txs_sizes` in `FeeRateCollector::statistics` panics on legacy `BlockExt` records — (`rpc/src/util/fee_rate.rs`)

## Summary

`FeeRateCollector::statistics` unconditionally unwraps `block_ext.txs_sizes` via `.expect()` at `rpc/src/util/fee_rate.rs:93`. The compatibility reader for pre-v0.106 `BlockExt` records always deserialises `txs_sizes` as `None`. Any node whose canonical chain's last ≤101 blocks include at least one legacy-format record will panic when an unprivileged caller invokes `get_fee_rate_statistics` or its deprecated alias `get_fee_rate_statics`.

## Finding Description

**Panic site** — `rpc/src/util/fee_rate.rs:93`:

```rust
let txs_sizes = txs_sizes.expect("expect txs_size's length >= 1");
```

`txs_sizes` is destructured from `block_ext` as `Option<Vec<u32>>`. There is no prior guard checking whether it is `Some`. [1](#0-0) 

**Why `txs_sizes` can be `None`** — `store/src/store.rs:252-253` dispatches on `count_extra_fields()`. When a record was written in the legacy 5-field `BlockExt` format (before v0.106), `count_extra_fields()` returns `0` and the reader is converted via `reader.into()`: [2](#0-1) 

That conversion is implemented in `util/types/src/conversion/storage.rs:154-165`, which hard-codes both `cycles` and `txs_sizes` to `None`: [3](#0-2) 

The `Unpack` impl at lines 139–151 does the same: [4](#0-3) 

**Why `filter_map` does not protect the caller** — `collect` uses `filter_map` only to skip blocks whose hash or ext is absent from the DB (i.e., `get_block_ext_by_number` returns `None`). A legacy block returns `Some(BlockExt { txs_sizes: None, … })`, which passes through the iterator and reaches the `.expect()` call: [5](#0-4) 

**New blocks are always written as `BlockExtV1`** — `store/src/transaction.rs:246` always packs as `BlockExtV1`, so only blocks committed before v0.106 carry the legacy format: [6](#0-5) 

## Impact Explanation

A Rust `panic!` in a synchronous RPC handler crashes or hangs the handler for that request. Repeated unauthenticated calls to `get_fee_rate_statistics` or `get_fee_rate_statics` keep triggering the panic for as long as the legacy block remains within the rolling window. This constitutes a **local RPC API crash**, matching the in-scope bounty impact: **Note (0–500 pts)**.

## Likelihood Explanation

On a long-running mainnet node all recent blocks are `BlockExtV1`, so the window never reaches legacy records. The condition is realistic for: testnets or devnets with short chains upgraded from a pre-v0.106 binary; any node whose chain height at upgrade time was ≤101 blocks; integration-test environments replaying a short chain from genesis with an old DB snapshot. The RPC requires no authentication and accepts any `target` value from 1 to 101, making it trivially reachable by any unprivileged caller once the precondition is met.

## Recommendation

Replace the unconditional `.expect()` with a graceful skip, consistent with the existing guard on `cycles` at line 97:

```rust
// Before
let txs_sizes = txs_sizes.expect("expect txs_size's length >= 1");

// After
let Some(txs_sizes) = txs_sizes else { return fee_rates; };
```

This makes the function safe for any chain state, including nodes upgraded from legacy databases.

## Proof of Concept

1. Open a `ChainDB` and insert a block whose `BlockExt` is serialised in the legacy 5-field `packed::BlockExt` format — `txs_sizes` field absent. The existing test at `store/src/tests/db.rs:54-68` already constructs a `BlockExt { txs_sizes: None }` and round-trips it through the store. [7](#0-6) 
2. Wrap the store in a `Snapshot` and call `FeeRateCollector::new(snapshot.as_ref()).statistics(Some(1))`.
3. Observe the process panics at `rpc/src/util/fee_rate.rs:93` with message `"expect txs_size's length >= 1"`.
