### Title
Unprivileged Initialization Front-Run Allows Attacker to Seize `config.authority` — (`target_chains/solana/programs/pyth-price-store/src/processor/initialize.rs`)

### Summary
The `Initialize` instruction has no privileged-caller check. Any signer with SOL can call it before the legitimate deployer and embed an attacker-controlled pubkey as `config.authority`. Because the config PDA can only be initialized once, the legitimate deployer is permanently locked out of authority, and the attacker controls all subsequent `InitializePublisher` calls.

### Finding Description

`validate_payer` only checks that the first account is a signer and writable — it performs no identity check against the program deployer or any expected key: [1](#0-0) 

The `initialize` function passes `args.authority` — a fully caller-supplied 32-byte field — directly into `accounts::config::create`: [2](#0-1) 

`config::create` only guards against double-initialization (format != 0), not against who calls it first: [3](#0-2) 

Once the config PDA is created with `format = FORMAT`, any subsequent `Initialize` call returns `AlreadyInitialized` and the authority is permanently set to whatever the first caller supplied.

### Impact Explanation

`validate_authority` in `initialize_publisher` enforces that the signer's key matches `config.authority`: [4](#0-3) 

If the attacker wins the race, they hold the only key that can call `InitializePublisher`. They can:
- Register arbitrary publisher-to-buffer mappings (injecting attacker-controlled publishers into the oracle set), or
- Deny registration to all legitimate publishers (griefing the oracle entirely).

Either outcome corrupts the publisher set and thus oracle price aggregation.

### Likelihood Explanation

On Solana, program deployment and the first `Initialize` transaction are separate on-chain events. Any observer of the mempool/RPC stream can detect a newly deployed program ID and submit a front-running `Initialize` transaction with a higher priority fee before the deployer's transaction lands. This is a well-known Solana attack pattern and requires no privileged access — only a funded keypair.

### Recommendation

Add a deployer/upgrade-authority check inside `initialize`. The canonical Solana pattern is to verify that the payer matches the program's upgrade authority stored in the BPF Loader upgradeable program data account. Alternatively, embed the expected authority pubkey as a program constant at compile time and assert `args.authority == EXPECTED_AUTHORITY` before writing the config.

pub fn validate_authority<'a, 'b>(
    account: Option<&'b AccountInfo<'a>>,
    config: &AccountInfo<'a>,
) -> Result<&'b AccountInfo<'a>, ProgramError> {
    let authority = account.ok_or(ProgramError::NotEnoughAccountKeys)?;
    ensure!(ProgramError::MissingRequiredSignature, authority.is_signer);
    ensure!(ProgramError::InvalidArgument, authority.is_writable);
    let config_data = config.data.borrow();
    let config = accounts::config::read(*config_data)?;
    ensure!(
        ProgramError::MissingRequiredSignature,
        authority.key.to_bytes() == config.authority
    );
    Ok(authority)

pub fn initialize(
    program_id: &Pubkey,
    accounts: &[AccountInfo],
    args: &InitializeArgs,
) -> ProgramResult {
    let mut accounts = accounts.iter();
    let payer = validate_payer(accounts.next())?;
    let config = validate_config(accounts.next(), args.config_bump, program_id, true)?;
    let system = validate_system(accounts.next())?;

    let lamports = (Rent::get()?).minimum_balance(accounts::config::SIZE);

    invoke_signed(
        &system_instruction::create_account(
            payer.key,
            config.key,
            lamports,
            accounts::config::SIZE
                .try_into()
                .expect("unexpected overflow"),
            program_id,
        ),
        &[payer.clone(), config.clone(), system.clone()],
        &[&[CONFIG_SEED.as_bytes(), &[args.config_bump]]],
    )?;

    accounts::config::create(*config.data.borrow_mut(), args.authority)?;

    Ok(())

pub fn create(data: &mut [u8], authority: [u8; 32]) -> Result<&mut Config, ReadAccountError> {
    if data.len() < size_of::<Config>() {
        return Err(ReadAccountError::DataTooShort);
    }
    let data: &mut Config = from_bytes_mut(&mut data[..size_of::<Config>()]);
    if data.format != 0 {
        return Err(ReadAccountError::AlreadyInitialized);
    }
    data.format = FORMAT;
    data.authority = authority;
    Ok(data)
