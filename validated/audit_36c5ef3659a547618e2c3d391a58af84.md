Audit Report

## Title
`GetBlocksProcess` Missing IBD Guard Violates Documented Invariant and Allows Peers to Force Block Serving During Initial Block Download — (`File: sync/src/synchronizer/get_blocks_process.rs`)

## Summary
`sync/src/synchronizer/mod.rs` L4 explicitly documents that during IBD the node will respond with `packed::InIBD` to both `GetHeaders` and `GetBlocks` requests. `GetHeadersProcess::execute()` enforces this, but `GetBlocksProcess::execute()` contains no IBD check, allowing any connected peer to force the IBD node to look up and serve full block bodies, wasting resources that should be dedicated to syncing.

## Finding Description
The module-level documentation at `sync/src/synchronizer/mod.rs` L4 states the invariant:

> "When CKB node is in IBD mode, it will respond `packed::InIBD` to `GetHeaders` and `GetBlocks` requests"

`GetHeadersProcess::execute()` enforces this at `sync/src/synchronizer/get_headers_process.rs` L53–66: it calls `active_chain.is_initial_block_download()`, sends `InIBD`, and returns `Status::ignored()`. [1](#0-0) 

`GetBlocksProcess::execute()` at `sync/src/synchronizer/get_blocks_process.rs` L33–97 has no such check. It immediately reads `block_hashes`, checks `BlockStatus::BLOCK_VALID`, and spawns async tasks to send `SendBlock` responses. [2](#0-1) 

The dispatch in `sync/src/synchronizer/mod.rs` L407–411 routes `GetBlocks` directly to `GetBlocksProcess` with no IBD interception, and `received()` has no top-level IBD gate either. [3](#0-2) 

## Impact Explanation
**Low (501–2000 points) — "Any other important performance improvements for CKB."** During IBD, the node is supposed to dedicate all resources to downloading and verifying the chain. Any connected peer can send repeated `GetBlocks` messages (up to `MAX_HEADERS_LEN` hashes per message, processed up to `INIT_BLOCKS_IN_TRANSIT_PER_PEER` at a time), causing the IBD node to perform block lookups and serialize full block bodies. This degrades IBD throughput and violates the documented protocol invariant. The impact is bounded — the node only serves blocks with `BlockStatus::BLOCK_VALID`, which are limited during early IBD — so this does not rise to node crash or network congestion. [4](#0-3) 

## Likelihood Explanation
Any peer that establishes a sync protocol connection can send a `GetBlocks` message with no authentication or special state required. The `Synchronizer::received()` handler at `sync/src/synchronizer/mod.rs` L890–963 processes all incoming sync messages without an IBD gate. This path is reachable by any unprivileged network peer for the entire duration of IBD. [5](#0-4) 

## Recommendation
Add an IBD guard at the beginning of `GetBlocksProcess::execute()`, mirroring the pattern in `GetHeadersProcess::execute()`:

```rust
pub fn execute(self) -> Status {
    let active_chain = self.synchronizer.shared.active_chain();

    if active_chain.is_initial_block_download() {
        // send InIBD response and return early
        return Status::ignored();
    }

    let block_hashes = self.message.block_hashes();
    // ... rest of existing logic
}
```

This aligns with the documented invariant in `mod.rs` L4 and the `GetHeadersProcess` pattern. [6](#0-5) 

## Proof of Concept
1. Start a CKB node from genesis (IBD mode active — `is_initial_block_download()` returns `true`).
2. Connect a peer that has already synced some blocks with `BlockStatus::BLOCK_VALID` on the IBD node.
3. From the peer, send a `SyncMessage::GetBlocks` containing hashes of genesis-adjacent blocks present in the IBD node's local store.
4. Observe that the IBD node responds with `SendBlock` messages containing full block data, rather than `InIBD`.
5. Contrast with sending `GetHeaders` to the same IBD node — it correctly responds with `InIBD` and returns early.
6. Repeat step 3 in a loop to measure the bandwidth and CPU overhead imposed on the IBD node.

The discrepancy is directly observable at the protocol level via the `GetBlocksProcess` code path at `sync/src/synchronizer/get_blocks_process.rs` L33–97. [7](#0-6)

    pub fn execute(self) -> Status {
        let block_hashes = self.message.block_hashes();
        // use MAX_HEADERS_LEN as limit, we may increase the value of INIT_BLOCKS_IN_TRANSIT_PER_PEER in the future
        if block_hashes.len() > MAX_HEADERS_LEN {
            return StatusCode::ProtocolMessageIsMalformed.with_context(format!(
                "BlockHashes count({}) > MAX_HEADERS_LEN({})",
                block_hashes.len(),
                MAX_HEADERS_LEN,
            ));
        }
        let active_chain = self.synchronizer.shared.active_chain();

        let iter = block_hashes.iter().take(INIT_BLOCKS_IN_TRANSIT_PER_PEER);

        let mut dedup = HashSet::new();
        for block_hash in iter {
            debug!("get_blocks {} from peer {:?}", block_hash, self.peer);
            let block_hash = block_hash.to_entity();

            if block_hash == self.synchronizer.shared().consensus().genesis_hash() {
                return StatusCode::RequestGenesis.with_context("Request genesis block");
            }

            if !dedup.insert(block_hash.clone()) {
                return StatusCode::RequestDuplicate.with_context("Request duplicate block");
            }

            if !active_chain.contains_block_status(&block_hash, BlockStatus::BLOCK_VALID) {
                debug!(
                    "Ignoring get_block {} request from peer={} as it is not verified.",
                    block_hash, self.peer
                );
                continue;
            }

            if let Some(block) = active_chain.get_block(&block_hash) {
                debug!(
                    "respond_block {} {} to peer {:?}",
                    block.number(),
                    block.hash(),
                    self.peer,
                );
                let content = packed::SendBlock::new_builder().block(block.data()).build();
                let message = packed::SyncMessage::new_builder().set(content).build();

                let nc = Arc::clone(self.nc);
                self.synchronizer
                    .shared()
                    .shared()
                    .async_handle()
                    .spawn(async move { async_send_message_to(&nc, self.peer, &message).await });
            } else {
                // TODO response not found
                // TODO add timeout check in synchronizer

                // We expect that `block_hashes` is sorted descending by height.
                // So if we cannot find the current one from local, we cannot find
                // the next either.
                debug!("Stopping getblocks, since {} is not found", block_hash);
                break;
            }
        }

        Status::ok()
    }
