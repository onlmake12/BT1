### Title
Missing `total_difficulty` Validation for Cross-Epoch `HeaderDigest` Allows Malicious Light Client Server to Spoof Chain Work - (File: `util/types/src/utilities/merkle_mountain_range.rs`)

---

### Summary

`HeaderDigest::verify()` only validates the `total_difficulty` field when a digest spans a single epoch. For cross-epoch digests — including the `parent_chain_root` embedded in every `VerifiableHeader` — `total_difficulty` is never checked against the actual compact targets. Because `VerifiableHeader::is_valid()` only verifies the hash of `parent_chain_root` (via `calc_mmr_hash()`, which is derived from the `children_hash` field, not from `total_difficulty`), a malicious light client server can supply a `VerifiableHeader` whose `parent_chain_root.total_difficulty` is arbitrarily inflated. The light client will accept this value as the chain's total work, causing it to follow a weaker fork as if it were the canonical chain.

---

### Finding Description

`HeaderDigest::verify()` enforces a `total_difficulty` consistency check only when `start_epoch_number == end_epoch_number`:

```rust
if start_epoch_number == end_epoch_number {
    // ...
    let total_difficulty_calculated = block_difficulty * blocks_count;
    if total_difficulty != total_difficulty_calculated {
        return Err(...);
    }
}
// No check when start_epoch_number != end_epoch_number
``` [1](#0-0) 

For a single-block (leaf) digest, `start_epoch == end_epoch` always holds, so leaf `total_difficulty` is verified. However, the `parent_chain_root` field of `VerifiableHeader` is a merged `HeaderDigest` spanning all blocks from genesis — it is always cross-epoch on a live chain — and `verify()` is **never called on it**.

`VerifiableHeader::is_valid()` only checks that the block extension begins with `parent_chain_root.calc_mmr_hash()`:

```rust
let parent_chain_root_hash = self.parent_chain_root().calc_mmr_hash();
actual_extension_data.starts_with(parent_chain_root_hash.as_slice())
``` [2](#0-1) 

`calc_mmr_hash()` is derived from the `children_hash` field (the Merkle hash of child nodes), **not** from `total_difficulty`. The `MergeHeaderDigest::merge()` function confirms this: `children_hash` is computed from the children's `calc_mmr_hash()` values, while `total_difficulty` is simply summed and stored as a separate field with no hash commitment:

```rust
let children_hash = {
    hasher.update(&lhs.calc_mmr_hash().raw_data());
    hasher.update(&rhs.calc_mmr_hash().raw_data());
    ...
};
let total_difficulty = { l + r };
``` [3](#0-2) 

Because `total_difficulty` is not part of the hash chain, it is not committed to in the block extension. A malicious server can construct a `VerifiableHeader` with a valid `children_hash` (matching the real block extension) but an arbitrarily inflated `total_difficulty` in `parent_chain_root`. The `is_valid()` check passes, and the light client reads the spoofed value via:

```rust
pub fn total_difficulty(&self) -> U256 {
    let parent_total_difficulty: U256 = self.parent_chain_root.total_difficulty().into();
    let block_difficulty = compact_to_difficulty(self.header.compact_target());
    parent_total_difficulty + block_difficulty
}
``` [4](#0-3) 

---

### Impact Explanation

The light client uses `total_difficulty` from `VerifiableHeader` to select the canonical chain (highest total work). A malicious server can present a minority fork with an inflated `total_difficulty`, causing the light client to treat that fork as canonical. This leads the light client to:

- Report incorrect chain state (wrong tip, wrong balances)
- Accept transactions confirmed only on the minority fork as finalized
- Reject valid transactions confirmed on the real canonical chain

This is a direct analog to the original report: a non-canonical value (`total_difficulty` not derived from actual compact targets) is accepted without a range/consistency check, producing incorrect downstream behavior.

---

### Likelihood Explanation

Any peer that the light client connects to can act as a light client server. No privileged access is required. The attacker only needs to run a CKB full node, serve the light client protocol, and craft a `VerifiableHeader` with a tampered `parent_chain_root.total_difficulty`. The block extension hash check (`is_valid()`) passes because `calc_mmr_hash()` does not cover `total_difficulty`. This is reachable by any unprivileged sync peer.

---

### Recommendation

**Short term:** In `VerifiableHeader::is_valid()`, call `self.parent_chain_root().verify()` and add cross-epoch `total_difficulty` validation to `HeaderDigest::verify()` (e.g., by summing per-epoch block difficulties using the `start_compact_target` / `end_compact_target` bounds).

**Long term:** Include `total_difficulty` in the `calc_mmr_hash()` computation so that it is cryptographically committed to in the block extension. This ensures that any tampered `total_difficulty` produces a hash mismatch that `is_valid()` will catch.

---

### Proof of Concept

1. Attacker runs a CKB full node on a minority fork (lower total work than mainnet).
2. Attacker serves the light client protocol and responds to `GetLastState` with a `SendLastState` message containing a `VerifiableHeader` for the minority fork tip.
3. Attacker sets `parent_chain_root.total_difficulty` to a value larger than the real mainnet total difficulty, while keeping `parent_chain_root.children_hash` correct (matching the real block extension of the minority fork tip).
4. Light client calls `is_valid()`: the extension hash check passes (only `children_hash` is verified), so the `VerifiableHeader` is accepted.
5. Light client calls `total_difficulty()` and receives the inflated value, concluding the minority fork has more work.
6. Light client follows the minority fork, accepting transactions confirmed only there as finalized. [5](#0-4) [6](#0-5) [7](#0-6)

impl HeaderDigest for packed::HeaderDigest {
    /// Verify the MMR header digest
    fn verify(&self) -> Result<(), String> {
        // 1. Check block numbers.
        let start_number: BlockNumber = self.start_number().into();
        let end_number: BlockNumber = self.end_number().into();
        if start_number > end_number {
            let errmsg = format!(
                "failed since the start block number is bigger than the end ([{start_number},{end_number}])"
            );
            return Err(errmsg);
        }

        // 2. Check epochs.
        let start_epoch: EpochNumberWithFraction = self.start_epoch().into();
        let end_epoch: EpochNumberWithFraction = self.end_epoch().into();
        let start_epoch_number = start_epoch.number();
        let end_epoch_number = end_epoch.number();
        if start_epoch != end_epoch
            && ((start_epoch_number > end_epoch_number)
                || (start_epoch_number == end_epoch_number
                    && start_epoch.index() > end_epoch.index()))
        {
            let errmsg = format!(
                "failed since the start epoch is bigger than the end ([{start_epoch:#},{end_epoch:#}])"
            );
            return Err(errmsg);
        }

        // 3. Check difficulties when in the same epoch.
        let start_compact_target: u32 = self.start_compact_target().into();
        let end_compact_target: u32 = self.end_compact_target().into();
        let total_difficulty: U256 = self.total_difficulty().into();
        if start_epoch_number == end_epoch_number {
            if start_compact_target != end_compact_target {
                // In the same epoch, all compact targets should be same.
                let errmsg = format!(
                    "failed since the compact targets should be same during epochs ([{start_epoch:#},{end_epoch:#}])"
                );
                return Err(errmsg);
            } else {
                // Sum all blocks difficulties to check total difficulty.
                let blocks_count = end_number - start_number + 1;
                let block_difficulty = compact_to_difficulty(start_compact_target);
                let total_difficulty_calculated = block_difficulty * blocks_count;
                if total_difficulty != total_difficulty_calculated {
                    let errmsg = format!(
                        "failed since total difficulty is {total_difficulty} but the calculated is {total_difficulty_calculated} \
                        during epochs ([{start_epoch:#},{end_epoch:#}])"
                    );
                    return Err(errmsg);
                }
            }
        }

        Ok(())
    }
}

    /// Checks if the current verifiable header is valid.
    pub fn is_valid(&self, mmr_activated_epoch_number: EpochNumber) -> bool {
        let mmr_activated_epoch = EpochNumberWithFraction::new(mmr_activated_epoch_number, 0, 1);
        let has_chain_root = self.header().epoch() > mmr_activated_epoch;
        if has_chain_root {
            if self.header().is_genesis() {
                if !self.parent_chain_root().is_default() {
                    return false;
                }
            } else {
                let is_extension_beginning_with_chain_root_hash = self
                    .extension()
                    .map(|extension| {
                        let actual_extension_data = extension.raw_data();
                        let parent_chain_root_hash = self.parent_chain_root().calc_mmr_hash();
                        actual_extension_data.starts_with(parent_chain_root_hash.as_slice())
                    })
                    .unwrap_or(false);
                if !is_extension_beginning_with_chain_root_hash {
                    return false;
                }
            }
        }

        let expected_extension_hash = self
            .extension()
            .map(|extension| extension.calc_raw_data_hash());
        let extra_hash_view = ExtraHashView::new(self.uncles_hash(), expected_extension_hash);
        let expected_extra_hash = extra_hash_view.extra_hash();
        let actual_extra_hash = self.header().extra_hash();
        expected_extra_hash == actual_extra_hash
    }
