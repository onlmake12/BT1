### Title
Accumulated `verification_fee` Permanently Locked in `PythLazer` Proxy — No Withdrawal Mechanism and Missing `receive()` Function - (File: `lazer/contracts/evm/src/PythLazer.sol`)

---

### Summary

`PythLazer` is deployed behind a UUPS ERC1967 proxy. Its `verifyUpdate()` function is `payable` and retains `verification_fee` (initially 1 wei) per call inside the proxy's ETH balance. The implementation contract defines no `withdraw()` function and no `receive()` function. All accumulated fees are permanently locked and irrecoverable.

---

### Finding Description

`PythLazer` is the Pyth Lazer on-chain verification contract, deployed via an ERC1967 UUPS proxy pattern. [1](#0-0) 

The `verifyUpdate()` function is `payable` and enforces a `verification_fee`. When `msg.value` exactly equals `verification_fee`, the entire amount is retained in the proxy's ETH balance. When `msg.value` exceeds `verification_fee`, only the excess is refunded to `msg.sender`; the fee portion remains locked. [2](#0-1) 

The deployment script confirms the contract is deployed behind a standard `ERC1967Proxy`: [3](#0-2) 

The full implementation contract contains no `withdraw()` function, no `receive()` function, and no `fallback()` function: [4](#0-3) 

Because the ERC1967 proxy forwards all calls via `delegatecall` to the implementation, any direct ETH transfer to the proxy (e.g., a refund from an external protocol) will also revert, since the implementation provides no `receive()` handler. All ETH that enters the proxy via `verifyUpdate()` is permanently locked.

---

### Impact Explanation

**Impact: Medium**

Every successful call to `verifyUpdate()` locks `verification_fee` (1 wei) in the proxy contract. There is no owner-callable `withdraw()` function, no admin fee-recovery path, and no `receive()` function. The Pyth protocol permanently loses all accumulated `verification_fee` revenue from `PythLazer`. Over high call volumes (Lazer is a high-frequency price verification service), the locked amount grows without bound and is irrecoverable without a contract upgrade.

---

### Likelihood Explanation

**Likelihood: Medium**

Every call to `verifyUpdate()` by any Lazer updater/relayer triggers the fee accumulation. The contract is already deployed in production at `0xACeA761c27A909d4D3895128EBe6370FDE2dF481`. The fee locking occurs automatically on every interaction — no special attacker action is required. The only mitigation path is a UUPS upgrade to add a withdrawal function.

---

### Recommendation

1. Add an owner-restricted `withdraw()` function to the `PythLazer` implementation to allow recovery of accumulated fees:

```solidity
function withdrawFees(address payable recipient, uint256 amount) external onlyOwner {
    require(address(this).balance >= amount, "Insufficient balance");
    recipient.transfer(amount);
}
```

2. Add a `receive()` function to allow the proxy to accept direct ETH transfers (e.g., from external protocol refunds):

```solidity
receive() external payable {}
```

---

### Proof of Concept

1. Deploy `PythLazer` behind an ERC1967 proxy (as done in production).
2. Call `verifyUpdate{value: 1 wei}(validUpdate)` — the 1 wei `verification_fee` is retained in the proxy's balance.
3. Repeat across N calls; the proxy accumulates N wei.
4. Attempt to call any withdrawal function — none exists; the ETH is permanently locked.
5. Attempt to send ETH directly to the proxy address — the proxy's fallback delegates to the implementation, which has no `receive()`, causing a revert.

The test file confirms the fee-retention behavior: [5](#0-4) 

After `verifyUpdate{value: fee}(update)`, `alice.balance` decreases by exactly `fee` — confirming the fee is retained in the contract — and there is no corresponding withdrawal path in the implementation.

import "@openzeppelin/contracts/utils/cryptography/ECDSA.sol";

contract PythLazer is OwnableUpgradeable, UUPSUpgradeable {
    TrustedSignerInfo[100] internal trustedSigners;
    uint256 public verification_fee;
    mapping(address => uint256) trustedSignerToExpiresAtMapping;

    constructor() {
        _disableInitializers();
    }

    struct TrustedSignerInfo {
        address pubkey;
        uint256 expiresAt;
    }

    function initialize(address _topAuthority) public initializer {
        __Ownable_init(_topAuthority);
        __UUPSUpgradeable_init();

        verification_fee = 1 wei;
    }

    function _authorizeUpgrade(address) internal override onlyOwner {}

    function updateTrustedSigner(
        address trustedSigner,
        uint256 expiresAt
    ) external onlyOwner {
        if (expiresAt == 0) {
            for (uint8 i = 0; i < trustedSigners.length; i++) {
                if (trustedSigners[i].pubkey == trustedSigner) {
                    trustedSigners[i].pubkey = address(0);
                    trustedSigners[i].expiresAt = 0;
                    delete trustedSignerToExpiresAtMapping[trustedSigner];
                    return;
                }
            }
            revert("no such pubkey");
        } else {
            for (uint8 i = 0; i < trustedSigners.length; i++) {
                if (trustedSigners[i].pubkey == trustedSigner) {
                    trustedSigners[i].expiresAt = expiresAt;
                    trustedSignerToExpiresAtMapping[trustedSigner] = expiresAt;
                    return;
                }
            }
            // Signer not found - adding a new signer.
            for (uint8 i = 0; i < trustedSigners.length; i++) {
                if (trustedSigners[i].pubkey == address(0)) {
                    trustedSigners[i].pubkey = trustedSigner;
                    trustedSigners[i].expiresAt = expiresAt;
                    trustedSignerToExpiresAtMapping[trustedSigner] = expiresAt;
                    return;
                }
            }
            revert("no space for new signer");
        }
    }

    function isValidSigner(address signer) public view returns (bool) {
        return block.timestamp < trustedSignerToExpiresAtMapping[signer];
    }

    function verifyUpdate(
        bytes calldata update
    ) external payable returns (bytes calldata payload, address signer) {
        // Require fee and refund excess
        require(msg.value >= verification_fee, "Insufficient fee provided");
        if (msg.value > verification_fee) {
            payable(msg.sender).transfer(msg.value - verification_fee);
        }

        if (update.length < 71) {
            revert("input too short");
        }
        uint32 EVM_FORMAT_MAGIC = 706910618;

        uint32 evm_magic = uint32(bytes4(update[0:4]));
        if (evm_magic != EVM_FORMAT_MAGIC) {
            revert("invalid evm magic");
        }
        uint16 payload_len = uint16(bytes2(update[69:71]));
        if (update.length < 71 + payload_len) {
            revert("input too short");
        }
        payload = update[71:71 + payload_len];
        bytes32 hash = keccak256(payload);
        (signer, , ) = ECDSA.tryRecover(
            hash,
            uint8(update[68]) + 27,
            bytes32(update[4:36]),
            bytes32(update[36:68])
        );
        if (signer == address(0)) {
            revert("invalid signature");
        }
        if (!isValidSigner(signer)) {
            revert("invalid signer");
        }
    }

    function version() public pure returns (string memory) {
        return "0.1.1";
    }
}
