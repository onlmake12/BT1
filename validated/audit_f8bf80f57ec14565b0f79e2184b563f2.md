### Title
Unrestricted `public` Access to `get_contract_upgrade_authorized_hash()` Enables Griefing DoS on Contract Upgrade — (File: `target_chains/aptos/contracts/sources/state.move`)

---

### Summary

`state::get_contract_upgrade_authorized_hash()` is declared `public` instead of `public(friend)`. The function uses `move_from` to **destructively consume** the `ContractUpgradeAuthorized` resource. Because it is `public`, any external module can call it to silently drain the authorized upgrade hash before the legitimate `do_contract_upgrade` entry function executes, permanently blocking that upgrade round.

---

### Finding Description

In `target_chains/aptos/contracts/sources/state.move`, the function is:

```rust
public fun get_contract_upgrade_authorized_hash(): Hash acquires ContractUpgradeAuthorized {
    assert!(exists<ContractUpgradeAuthorized>(@pyth), error::unauthorized_upgrade());
    let ContractUpgradeAuthorized { hash } = move_from<ContractUpgradeAuthorized>(@pyth);
    hash
}
``` [1](#0-0) 

`move_from` is a **destructive read** — it removes the resource from global storage entirely. The `Hash` struct carries the `drop` ability, so any caller can receive and silently discard it:

```rust
struct Hash has store, drop {
    hash: vector<u8>,
}
``` [2](#0-1) 

The `state` module already declares `pyth::contract_upgrade` as a `friend`, signalling the intent to restrict this accessor:

```
friend pyth::contract_upgrade;
``` [3](#0-2) 

The only legitimate consumer is `contract_upgrade::do_contract_upgrade`, which calls `state::get_contract_upgrade_authorized_hash()` to verify the hash before publishing new code:

```rust
assert!(matches_hash(code, metadata_serialized, state::get_contract_upgrade_authorized_hash()), ...);
let pyth = state::pyth_signer();
code::publish_package_txn(&pyth, metadata_serialized, code);
``` [4](#0-3) 

Because `get_contract_upgrade_authorized_hash()` is `public`, any third-party module can call it, consume the `ContractUpgradeAuthorized` resource, and leave the contract in a state where `do_contract_upgrade` will always abort with `error::unauthorized_upgrade()`.

---

### Impact Explanation

An attacker who monitors the chain for the governance `execute_governance_instruction` transaction (which sets the authorized hash) can immediately call `state::get_contract_upgrade_authorized_hash()` from their own module. This:

1. Removes `ContractUpgradeAuthorized` from `@pyth` storage.
2. Causes every subsequent `do_contract_upgrade` call to abort at the `exists<ContractUpgradeAuthorized>(@pyth)` assertion.
3. Forces governance to re-issue a new upgrade-authorization VAA and repeat the entire governance process.

The attacker can repeat this indefinitely, permanently blocking any contract upgrade. If the upgrade is a critical security patch, this DoS has high protocol impact.

---

### Likelihood Explanation

Aptos transactions are publicly visible in the mempool and on-chain. An attacker can watch for the governance `execute_governance_instruction` call that sets the authorized hash, then immediately submit a transaction calling their griefing module. No privileged access, leaked key, or social engineering is required — only a deployed Move module and a small gas fee.

---

### Recommendation

Change the visibility from `public` to `public(friend)`:

```rust
// Before
public fun get_contract_upgrade_authorized_hash(): Hash acquires ContractUpgradeAuthorized {

// After
public(friend) fun get_contract_upgrade_authorized_hash(): Hash acquires ContractUpgradeAuthorized {
``` [1](#0-0) 

This restricts the function to the already-declared `friend pyth::contract_upgrade` module, matching the access pattern used for all other sensitive state mutators such as `pyth_signer()`, `set_contract_upgrade_authorized_hash()`, and `set_latest_price_info()`. [5](#0-4) 

---

### Proof of Concept

```rust
// Attacker-controlled module deployed on Aptos
module attacker::griefing {
    use pyth::state;

    // Called immediately after governance sets the authorized hash
    public entry fun consume_upgrade_hash() {
        // Destructively removes ContractUpgradeAuthorized from @pyth storage.
        // The returned Hash has `drop` ability and is silently discarded.
        let _hash = state::get_contract_upgrade_authorized_hash();
        // _hash is dropped here; ContractUpgradeAuthorized resource is now gone.
    }
}
```

**Attack sequence:**

1. Pyth governance submits a VAA → `execute_governance_instruction` sets `ContractUpgradeAuthorized { hash: H }` at `@pyth`.
2. Attacker calls `attacker::griefing::consume_upgrade_hash()` — `move_from<ContractUpgradeAuthorized>(@pyth)` removes the resource; `Hash` is dropped.
3. Pyth team calls `do_contract_upgrade(metadata, code)` → `assert!(exists<ContractUpgradeAuthorized>(@pyth), ...)` **aborts** with `error::unauthorized_upgrade()`.
4. Governance must re-issue the authorization VAA. Attacker repeats step 2 indefinitely. [4](#0-3) [1](#0-0)
