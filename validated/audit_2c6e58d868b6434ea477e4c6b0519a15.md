### Title
`ScriptHashTypeVerifier` Omits Type Script Hash-Type Validation, Allowing Not-Permitted Hash Types into Tx Pool — (File: `verification/src/transaction_verifier.rs`)

---

### Summary

`ScriptHashTypeVerifier::verify()` checks the `hash_type` field only on the **lock script** of each transaction output. It never inspects the **type script** hash type. Because `ENABLED_SCRIPT_HASH_TYPE` only permits `{0, 1, 2, 4}` (Data, Type, Data1, Data2), but the `ScriptHashType` enum exposes variants `Data3`–`Data127` (values `6`–`254`) that pass structural validation, an RPC caller can submit a transaction whose type script carries a not-permitted hash type, have it admitted to the tx pool, and never have it caught by any non-contextual block-level check.

---

### Finding Description

`ENABLED_SCRIPT_HASH_TYPE` is the consensus-level allowlist: [1](#0-0) 

The `ScriptHashType` enum is generated with `seq!(N in 3..=127 { ... })`, producing variants `Data3 = 6` through `Data127 = 254`: [2](#0-1) 

The structural check `verify_value()` accepts **all even values and 1**, so `Data3 = 6` passes it: [3](#0-2) 

`check_data()` uses `verify_value()` for both lock and type scripts, so `Data3` in a type script passes structural validation: [4](#0-3) 

`ScriptHashTypeVerifier::verify()` then applies the consensus-level `ENABLED_SCRIPT_HASH_TYPE` check — but **only on the lock script**. The type script is never examined: [5](#0-4) 

The block-level `BlockVerifier` does not include `ScriptHashTypeVerifier` or any equivalent check for type scripts: [6](#0-5) 

The only place that would eventually reject the transaction is `select_version()` during script execution — which runs only at block-verification time, long after the transaction has been admitted to the tx pool: [7](#0-6) 

---

### Impact Explanation

An unprivileged RPC caller submitting via `send_transaction` can craft a transaction with:
- A valid lock script (permitted hash type, e.g., `Data = 0`)
- A type script with `hash_type = Data3 = 6` (not in `ENABLED_SCRIPT_HASH_TYPE`)
- Valid inputs and fees

The transaction passes `NonContextualTransactionVerifier` (including `ScriptHashTypeVerifier`) and is admitted to the tx pool. It occupies tx pool capacity indefinitely because it can never be included in a valid block — script execution will always reject it with `InvalidScriptHashType`. By repeating this with many transactions, an attacker can exhaust tx pool capacity, evicting legitimate transactions and degrading node availability.

---

### Likelihood Explanation

The attack requires only a standard RPC call to `send_transaction` with a crafted transaction. No special privileges, keys, or hashpower are needed. The `ScriptHashType` enum variants `Data3`–`Data127` are publicly documented in the JSON-RPC types, and the serde `rename_all = "snake_case"` attribute means `"data3"` is a valid JSON string accepted by the RPC deserializer: [8](#0-7) 

---

### Recommendation

Extend `ScriptHashTypeVerifier::verify()` to also validate the type script's hash type for each output. Specifically, after checking `output.lock().hash_type()`, add an equivalent check for `output.type_().to_opt()`:

```rust
// After the existing lock-script check:
if let Some(type_script) = output.type_().to_opt() {
    if let Ok(hash_type) = TryInto::<ScriptHashType>::try_into(type_script.hash_type()) {
        let val: u8 = hash_type.into();
        if !ENABLED_SCRIPT_HASH_TYPE.contains(&val) {
            return Err(TransactionError::ScriptHashTypeNotPermitted { hash_type: val }.into());
        }
    } else {
        return Err(TransactionError::InvalidScriptHashType {
            hash_type: type_script.hash_type(),
        }.into());
    }
}
```

---

### Proof of Concept

Submit via the `send_transaction` RPC:

```json
{
  "version": "0x0",
  "cell_deps": [...],
  "inputs": [{ "previous_output": { "tx_hash": "<valid>", "index": "0x0" }, "since": "0x0" }],
  "outputs": [{
    "capacity": "0x...",
    "lock": { "code_hash": "0x...", "hash_type": "data", "args": "0x" },
    "type": { "code_hash": "0x...", "hash_type": "data3", "args": "0x" }
  }],
  "outputs_data": ["0x"],
  "witnesses": ["0x"]
}
```

`"data3"` deserializes to `ScriptHashType::Data3 = 6`. The lock script uses `"data"` (value `0`), which is in `ENABLED_SCRIPT_HASH_TYPE`. `ScriptHashTypeVerifier` only checks the lock script and returns `Ok(())`. The transaction is admitted to the tx pool. It can never be mined into a valid block because `select_version()` will return `Err(ScriptError::InvalidScriptHashType(...))` for `Data3` during script execution.

seq!(N in 3..=127 {
    /// Specifies how the script `code_hash` is used to match the script code and how to run the code.
    /// The hash type is split into the high 7 bits and the low 1 bit,
    /// when the low 1 bit is 1, it indicates the type,
    /// when the low 1 bit is 0, it indicates the data,
    /// and then it relies on the high 7 bits to indicate
    /// that the data actually corresponds to the version.
     #[derive(Default, Clone, Copy, PartialEq, Eq, Debug, Hash, FromRepr)]
     #[repr(u8)]
    pub enum ScriptHashType {
        /// Type "type" matches script code via cell type script hash.
        Type = 1,
        /// Type "data" matches script code via cell data hash, and run the script code in v0 CKB VM.
        #[default]
        Data = 0,
        /// Type "data1" matches script code via cell data hash, and run the script code in v1 CKB VM.
        Data1 = 2,
        /// Type "data2" matches script code via cell data hash, and run the script code in v2 CKB VM.
        Data2 = 4,
        #(
            #[doc = concat!("Type \"data", stringify!(N), "\" matches script code via cell data hash, and runs the script code in v", stringify!(N), " CKB VM.")]
            Data~N = N << 1,
        )*
    }
});

    pub fn verify(&self) -> Result<(), Error> {
        for output in self.transaction.outputs() {
            if let Ok(hash_type) = TryInto::<ScriptHashType>::try_into(output.lock().hash_type()) {
                let val: u8 = hash_type.into();
                if !ENABLED_SCRIPT_HASH_TYPE.contains(&val) {
                    return Err(
                        TransactionError::ScriptHashTypeNotPermitted { hash_type: val }.into(),
                    );
                }
            } else {
                return Err((TransactionError::InvalidScriptHashType {
                    hash_type: output.lock().hash_type(),
                })
                .into());
            }
        }

        Ok(())
    }
