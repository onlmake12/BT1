### Title
`work_id=0` Tautology Bypass in `update_block_template` Allows Unauthenticated Template Injection via Notify Endpoint — (`miner/src/client.rs`)

---

### Summary

The `update_block_template` function in the CKB miner contains a boolean logic error: the `fetch_update` closure condition `id != work_id || id == 0` becomes a tautology (`id != 0 || id == 0`) whenever the incoming template carries `work_id = 0`. Because the notify HTTP endpoint accepts any POST with no authentication, any process that can reach the listen address can inject a crafted `BlockTemplate` with `work_id = 0` and have it unconditionally accepted and dispatched to workers, regardless of the miner's current work state.

---

### Finding Description

**The logic bug — `miner/src/client.rs` lines 295–301:**

```rust
let updated = |id| {
    if id != work_id || id == 0 {   // ← id is current_work_id; work_id is incoming
        Some(work_id)
    } else {
        None
    }
};
``` [1](#0-0) 

`id` is the current value of the `AtomicU64 current_work_id`; `work_id` is the incoming template's field. When the incoming `work_id = 0`:

| Sub-expression | Evaluates to |
|---|---|
| `id != work_id` | `id != 0` |
| `id == 0` | `id == 0` |
| Combined (`\|\|`) | `id != 0 \|\| id == 0` → **always `true`** |

`fetch_update` therefore always returns `Ok`, and `Works::New(work)` is always sent to the mining workers.

**The unauthenticated HTTP surface — `miner/src/client.rs` lines 358–369:**

```rust
async fn handle(
    client: Client,
    req: Request<hyper::body::Incoming>,
) -> Result<Response<Empty<Bytes>>, Error> {
    let body = BodyExt::collect(req).await?.aggregate();
    if let Ok(template) = serde_json::from_reader(body.reader()) {
        client.update_block_template(template);   // no auth, no IP filter
    }
    Ok(Response::new(Empty::new()))
}
``` [2](#0-1) 

There is no authentication token, no IP allowlist, and no HMAC/signature check. Any HTTP client that can reach the `SocketAddr` bound by `listen_block_template_notify` can POST a `BlockTemplate`. [3](#0-2) 

**Initialization confirms `work_id = 0` is a valid wire value:**

The `BlockAssembler` initialises its counter at `AtomicU64::new(0)` and uses `fetch_add(1, SeqCst)` (which returns the *old* value) for the very first template, so the first legitimate template legitimately carries `work_id = 0`. [4](#0-3) 

This means the miner's deduplication logic was designed to treat `work_id = 0` as a special "always accept" sentinel for startup, but the check `id == 0` tests the *current* atomic value, not the *incoming* value — creating the tautology.

---

### Impact Explanation

An attacker who can reach the notify endpoint (no credential required) can:

1. **Continuously inject a crafted template with `work_id = 0`** — the miner unconditionally replaces its active work with the injected template on every POST.
2. **Redirect coinbase rewards** — the CKB node does not restrict which lock script appears in the coinbase output; a valid template with the attacker's lock script, correct parent hash, and correct difficulty will be accepted by `submit_block` if PoW is solved.
3. **Waste all hashpower on stale work** — injecting a template referencing an old parent hash causes the miner to solve puzzles the network will never accept, permanently suppressing valid block production.

The `work_id` deduplication invariant — that only a strictly different (newer) work ID should replace active work — is completely broken for the `work_id = 0` case.

---

### Likelihood Explanation

- **Notify mode is a documented, production-supported configuration.** The code explicitly logs instructions for enabling it. [5](#0-4) 
- **No authentication exists on the endpoint.** There is no token, no IP filter, no TLS client certificate.
- **The listen address is operator-chosen** (`config.listen: Option<SocketAddr>`). If bound to `0.0.0.0` or any non-loopback interface, the attack is fully remote. Even on `127.0.0.1`, any unprivileged local process (e.g., a compromised dependency, a co-tenant container) can reach it.
- **The bypass requires only a single HTTP POST** with a JSON body containing `"work_id": "0x0"`. No brute force, no PoW, no key material.

---

### Recommendation

1. **Fix the tautology.** The `|| id == 0` branch tests the *current* atomic, not the incoming value. The correct guard for "always accept on startup" is to check the incoming value or remove the special case entirely:

   ```rust
   // Before (buggy):
   if id != work_id || id == 0 { … }

   // After (correct — accept only when work actually changes):
   if id != work_id { … }
   ```

2. **Add authentication to the notify endpoint.** Require a shared secret (e.g., a bearer token in the `Authorization` header) that is configured by the operator and verified before calling `update_block_template`.

3. **Bind the listener to loopback by default** and document that binding to non-loopback addresses requires explicit operator acknowledgement of the exposure.

    async fn listen_block_template_notify(&self, addr: SocketAddr) {
        let listener = TcpListener::bind(addr).await.unwrap();
        let server = auto::Builder::new(TokioExecutor::new());
        let graceful = GracefulShutdown::new();

        loop {
            let client = self.clone();
            let handle = service_fn(move |req| handle(client.clone(), req));
            tokio::select! {
                conn = listener.accept() => {
                    let (stream, _) = match conn {
                        Ok(conn) => conn,
                        Err(e) => {

                            tokio::time::sleep(std::time::Duration::from_secs(1)).await;
                            continue;
                        }
                    };

                    let conn = server.serve_connection_with_upgrades(stream, handle);

                    let conn = graceful.watch(conn.into_owned());
                    tokio::spawn(async move {
                        if let Err(err) = conn.await {
                            info!("connection error: {}", err);
                        }
                    });
                },
                _ = stop_rx.cancelled() => {
                    info!("Miner client received exit signal. Exit now");
                    break;
                }
            }
        }
