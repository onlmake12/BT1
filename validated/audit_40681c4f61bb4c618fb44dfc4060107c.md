### Title
Missing Contract Existence Check in `revealWithCallback` New Callback Flow Causes Silent Callback Failure — (`File: target_chains/ethereum/contracts/contracts/entropy/Entropy.sol`)

---

### Summary

`Entropy.sol`'s `revealWithCallback` function has two code paths. The **old flow** (`gasLimit10k == 0`) explicitly guards the callback with an `extcodesize` check before invoking the requester. The **new flow** (`gasLimit10k != 0`) uses `excessivelySafeCall` with no such guard. Per EVM design, a low-level `CALL` to an address with no deployed code returns `success = true` with empty return data. If the requester contract has been self-destructed between request time and fulfillment time, the new flow silently treats the callback as successful, permanently clears the request, and emits a non-failure event — all without any code having executed in the requester.

---

### Finding Description

In `revealWithCallback`, the branching logic is:

```
if (req.gasLimit10k != 0 && req.callbackStatus == CALLBACK_NOT_STARTED)
    → NEW FLOW: excessivelySafeCall (no extcodesize check)
else
    → OLD FLOW: extcodesize check, then conditional call
```

**Old flow** (protected): [1](#0-0) 

```solidity
// Check if the requester is a contract account.
uint len;
assembly {
    len := extcodesize(callAddress)
}
if (len != 0) {
    IEntropyConsumer(callAddress)._entropyCallback(...);
}
```

**New flow** (unprotected): [2](#0-1) 

```solidity
(success, ret) = req.requester.excessivelySafeCall(
    uint256(req.gasLimit10k) * TEN_THOUSAND,
    256,
    abi.encodeWithSelector(IEntropyConsumer._entropyCallback.selector, ...)
);
...
if (success) {
    ...
    clearRequest(provider, sequenceNumber);   // ← request permanently cleared
}
```

`ExcessivelySafeCall` is a thin wrapper around the EVM `CALL` opcode. It does **not** check `extcodesize`. A call to an address with no code returns `(success=true, ret="")`. The `if (success)` branch then fires, the `RevealedWithCallback` event is emitted with `failed = false`, and `clearRequest` permanently removes the in-flight request — all without any callback code having run.

The requester address is set to `msg.sender` at request time: [3](#0-2) 

Any contract that self-destructs after requesting but before fulfillment triggers this path.

---

### Impact Explanation

1. The user's contract pays fees via `requestV2` with a non-zero `gasLimit`, enabling the new flow.
2. The user's contract self-destructs (e.g., via an upgrade pattern, a bug, or intentional teardown).
3. The provider calls `revealWithCallback`.
4. `excessivelySafeCall` to the now-codeless address returns `(true, "")`.
5. The contract emits `RevealedWithCallback` (indicating success) and calls `clearRequest`.
6. The request is **permanently gone** — no retry, no recovery path.
7. The user paid fees for a random number callback that was never delivered.
8. The provider is paid for work that produced no result for the user.

The `EntropyEventsV2.Revealed` event is emitted with `callbackFailed = false`, so off-chain monitors and indexers also record a false success. [4](#0-3) 

---

### Likelihood Explanation

- Any contract using the new `requestV2(provider, gasLimit)` API that later self-destructs (upgrade proxy teardown, `selfdestruct` in a factory pattern, or a bug) is affected.
- The provider has no incentive to check whether the requester still has code — they are paid regardless.
- The inconsistency with the old flow (which does check `extcodesize`) shows this is an unintentional omission, not a deliberate design choice.
- The entry path is fully unprivileged: any user calling `requestV2` with a non-zero gas limit is in scope.

---

### Recommendation

Add the same `extcodesize` guard used in the old flow before the `excessivelySafeCall` in the new flow:

```solidity
uint len;
assembly { len := extcodesize(req.requester) }
if (len == 0) {
    // Treat as a failed callback — move to CALLBACK_FAILED state
    // or revert so the request remains retryable.
    req.callbackStatus = EntropyStatusConstants.CALLBACK_FAILED;
    emit CallbackFailed(...);
    return;
}
(success, ret) = req.requester.excessivelySafeCall(...);
```

Alternatively, document that requests from contracts that may self-destruct are unsupported and add a check at request time (e.g., require `extcodesize(msg.sender) > 0`).
