### Title
Missing Staleness Check on Price Payload Timestamp in `verifyUpdate` Allows Stale Price Submission — (`File: lazer/contracts/evm/src/PythLazer.sol`)

### Summary
The `verifyUpdate` function in `PythLazer.sol` validates the ECDSA signature and checks that the signer key has not expired, but performs **no validation on the timestamp of the price data within the payload**. An unprivileged Lazer relayer/updater can replay a legitimately signed but arbitrarily old price update. The contract accepts it as valid and returns the stale payload to the caller. Any protocol consuming Lazer prices via `verifyUpdate` without implementing its own staleness check is exposed to stale-price exploitation.

---

### Finding Description

`verifyUpdate` in `PythLazer.sol` performs three checks:

1. EVM format magic number
2. ECDSA signature recovery
3. `isValidSigner(signer)` — checks only that `block.timestamp < trustedSignerToExpiresAtMapping[signer]` [1](#0-0) 

`isValidSigner` is defined as: [2](#0-1) 

The signer expiry check is on the **key's certificate lifetime**, not on the **age of the price data** embedded in the payload. The payload bytes are returned raw to the caller without any timestamp extraction or freshness enforcement.

The Sui and Aptos Lazer implementations share the same design: they check `clock.timestamp_ms() < expires_at_ms` for the signer, but never parse the payload's internal timestamp to enforce freshness. [3](#0-2) [4](#0-3) 

This is the direct analog to the original report's mitigation-review finding: the Chainlink integration accepted prices without a stale-price check (`latestRoundData()` called with no age validation). Here, `verifyUpdate` accepts any signed payload regardless of how old the embedded price timestamp is.

---

### Impact Explanation

A Lazer relayer or any unprivileged caller who has cached a legitimately signed Lazer update from a prior time can:

1. Obtain a signed Lazer payload from when prices were at a favorable level (e.g., before a large price move).
2. Call `verifyUpdate{value: verification_fee}(oldSignedUpdate)`.
3. The contract returns the stale payload as cryptographically valid.
4. Any downstream protocol that uses the returned payload for financial decisions (liquidations, collateral valuation, trade execution) without independently checking the payload's internal timestamp will act on the stale price.
5. The attacker profits from the discrepancy between the stale price and the true current price — equivalent to the sandwich/slippage attack in the original report.

The `verification_fee` is `1 wei`, making the cost of this attack negligible. [5](#0-4) 

---

### Likelihood Explanation

- **Entry path is fully unprivileged**: `verifyUpdate` is `external payable` with no access control. Any address can call it.
- **Signed old payloads are readily available**: A Lazer relayer receives the continuous stream of signed updates and can trivially cache any prior update.
- **Cost is 1 wei**: There is no economic deterrent.
- **No on-chain defense**: The contract does not parse the payload timestamp at all.

---

### Recommendation

Parse the price timestamp from the payload inside `verifyUpdate` and enforce a maximum age (e.g., configurable `maxPriceAge` in seconds). Reject the call if `block.timestamp - priceTimestamp > maxPriceAge`. This mirrors the stale-price fix described in the original report's team response (lines 90–94 of `ChainlinkInterface.sol`).

Alternatively, emit the parsed timestamp as part of the return value so that callers are forced to handle it explicitly, and document a required staleness check in the integration interface.

---

    function verifyUpdate(
        bytes calldata update
    ) external payable returns (bytes calldata payload, address signer) {
        // Require fee and refund excess
        require(msg.value >= verification_fee, "Insufficient fee provided");
        if (msg.value > verification_fee) {
            payable(msg.sender).transfer(msg.value - verification_fee);
        }

        if (update.length < 71) {
            revert("input too short");
        }
        uint32 EVM_FORMAT_MAGIC = 706910618;

        uint32 evm_magic = uint32(bytes4(update[0:4]));
        if (evm_magic != EVM_FORMAT_MAGIC) {
            revert("invalid evm magic");
        }
        uint16 payload_len = uint16(bytes2(update[69:71]));
        if (update.length < 71 + payload_len) {
            revert("input too short");
        }
        payload = update[71:71 + payload_len];
        bytes32 hash = keccak256(payload);
        (signer, , ) = ECDSA.tryRecover(
            hash,
            uint8(update[68]) + 27,
            bytes32(update[4:36]),
            bytes32(update[36:68])
        );
        if (signer == address(0)) {
            revert("invalid signature");
        }
        if (!isValidSigner(signer)) {
            revert("invalid signer");
        }
    }

public(package) fun verify_le_ecdsa_message(
    state: &State,
    clock: &Clock,
    signature: &vector<u8>,
    payload: &vector<u8>,
) {
    let current_cap = state.current_cap();

    // 0 stands for keccak256 hash
    let pubkey = secp256k1_ecrecover(signature, payload, 0);

    // Check if the recovered pubkey is in the trusted signers list
    let trusted_signers = state.trusted_signers(&current_cap);
    let mut maybe_idx = trusted_signers.find_index!(|signer|
        signer.public_key() == &pubkey
    );

    assert!(maybe_idx.is_some(), ESignerNotTrusted);
    let idx = maybe_idx.extract();
    let expires_at_ms = trusted_signers[idx].expires_at_ms();
    assert!(clock.timestamp_ms() < expires_at_ms, ESignerExpired);
