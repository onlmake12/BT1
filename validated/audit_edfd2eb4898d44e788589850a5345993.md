Audit Report

## Title
VerifyQueue Lacks Per-Peer Fairness Scheduling, Enabling Single-Peer DoS of Transaction Verification Pipeline — (File: `tx-pool/src/component/verify_queue.rs`)

## Summary
`VerifyQueue` enforces only a single global 256 MB byte cap with no per-peer occupancy accounting. A single P2P peer can exhaust this cap via the relay protocol's pull-based transaction delivery flow, monopolizing all verification workers and causing every subsequent `submit_remote_tx` and `submit_local_tx` call from any peer or local RPC user to return `Reject::Full` until the attacker's backlog drains.

## Finding Description

**Global-only cap, no per-peer limit.**
`DEFAULT_MAX_VERIFY_QUEUE_TX_SIZE = 256_000_000` is the sole admission guard. [1](#0-0) 

`add_tx()` checks only the global `total_tx_size`; there is no per-peer byte counter anywhere in `VerifyQueue`. [2](#0-1) [3](#0-2) 

**Strict FIFO dequeue.**
`peek()` iterates `iter_by_added_time()` — pure wall-clock insertion order — with no per-peer interleaving. [4](#0-3) 

**Relay injection path.**
The rate limiter allows 30 messages/s keyed by `(peer, message_type)`, and each `RelayTransactions` message may carry up to `MAX_RELAY_TXS_BYTES_PER_BATCH = 1 MiB`. [5](#0-4) [6](#0-5) 

`TransactionsProcess::execute()` filters by `unknown_tx_hashes` (only processes transactions the node explicitly requested), but this pull-based check imposes no cap on the *volume* of transactions that can be requested from a single peer. All announced unknown hashes are requested. [7](#0-6) 

Each accepted transaction is submitted directly to the verify queue with no per-peer occupancy check:
`submit_remote_tx` → `resumeble_process_tx_and_notify_full_reject` → `resumeble_process_tx` → `non_contextual_verify` → `enqueue_verify_queue` → `add_tx`. [8](#0-7) [9](#0-8) 

**Existing mitigations are insufficient.**
`remove_txs_by_peer()` can purge a peer's entries, but only after disconnection — not during an active flood. [10](#0-9) 

The `OnlySmallCycleTx` worker role segregates by cycle size, not by peer, providing no fairness across peers. [11](#0-10) 

## Impact Explanation
Once the 256 MB queue is saturated by one peer, every `submit_remote_tx` and `submit_local_tx` call returns `Reject::Full` (surfaced as RPC error `PoolIsFull -1106`). Legitimate transactions from all other peers and local RPC users are rejected outright. Transactions that entered the queue before saturation are processed only after all earlier attacker transactions complete CKB-VM verification (up to `max_tx_verify_cycles = 70,000,000` cycles each), monopolizing the fixed `max_tx_verify_workers` thread pool. Applied to multiple nodes simultaneously, this constitutes **CKB network congestion achievable at low cost**, matching the **High (10001–15000 points)** impact class: *"Vulnerabilities or bad designs which could cause CKB network congestion with few costs."*

## Likelihood Explanation
No privilege is required beyond completing the standard P2P handshake. Transactions need only pass `non_contextual_verify()` (structural checks, no script execution) to be enqueued — invalid scripts still occupy queue space and worker time until they fail inside the queue. At 30 `RelayTransactions` messages/s × 1 MiB/message the injection rate is ≈30 MiB/s, filling the 256 MB queue in approximately 8–9 seconds. The attack is repeatable: after the attacker disconnects (triggering `remove_txs_by_peer`), reconnecting and repeating the flood immediately re-saturates the queue.

## Recommendation
**Short term:** Add a per-peer byte counter inside `VerifyQueue`. In `add_tx()`, reject a new entry if the submitting peer's current occupancy exceeds `DEFAULT_MAX_VERIFY_QUEUE_TX_SIZE / N` (where N is a configurable peer fraction, e.g., the number of connected relay peers or a fixed cap like 8).

**Long term:** Replace the FIFO dequeue in `peek()` with a fair-queuing discipline (e.g., weighted round-robin over per-peer buckets) so that no single peer can delay transactions from all other peers regardless of submission rate.

## Proof of Concept
1. Connect a custom peer to a CKB node via the RelayV3 protocol.
2. Construct N minimal transactions each passing `non_contextual_verify()` (e.g., using a known-valid structure with an always-success lock script hash the node cannot immediately resolve).
3. Send `RelayTransactionHashes` messages (up to 32767 hashes each, 30/s) to announce all N hashes.
4. When the node replies with `GetRelayTransactions`, respond with `RelayTransactions` messages (up to 1 MiB each, 30/s).
5. `TransactionsProcess::execute()` calls `submit_remote_tx()` for each transaction → `add_tx()` with no per-peer check.
6. After ≈9 seconds, `total_tx_size` reaches 256 MB; `is_full()` returns `true`.
7. Any subsequent `send_transaction` RPC call from a legitimate user returns `PoolIsFull (-1106)`.
8. Transactions already in the queue from legitimate users are processed only after all attacker transactions ahead of them in FIFO order complete CKB-VM verification.

    pub fn add_tx(
        &mut self,
        tx: TransactionView,
        is_proposal_tx: bool,
        remote: Option<(Cycle, PeerIndex)>,
    ) -> Result<bool, Reject> {
        if self.contains_key(&tx.proposal_short_id()) {
            if is_proposal_tx {
                self.remove_tx(&tx.proposal_short_id());
            } else {
                return Ok(false);
            }
        }
        let tx_size = tx.data().serialized_size_in_block();
        let is_large_cycle = remote
            .map(|(cycles, _)| cycles > self.large_cycle_threshold)
            .unwrap_or(false);
        if self.is_full(tx_size) {
            return Err(Reject::Full(format!(
                "verify_queue total_tx_size exceeded, failed to add tx: {:#x}",
                tx.hash()
            )));
        }
        let total_tx_size = self.total_tx_size.checked_add(tx_size).ok_or_else(|| {
            Reject::Full(format!(
                "verify_queue total_tx_size overflowed, failed to add tx: {:#x}",
                tx.hash()
            ))
        })?;
        self.inner.insert(VerifyEntry {
            id: tx.proposal_short_id(),
            added_time: unix_time_as_millis(),
            inner: Entry { tx, remote },
            is_large_cycle,
            is_proposal_tx,
        });
        self.total_tx_size = total_tx_size;
        self.ready_rx.notify_one();
        Ok(true)
    }

    pub fn execute(self) -> Status {
        let shared_state = self.relayer.shared().state();
        let txs: Vec<(TransactionView, Cycle)> = {
            // ignore the tx if it's already known or it has never been requested before
            let mut tx_filter = shared_state.tx_filter();
            tx_filter.remove_expired();
            let unknown_tx_hashes = shared_state.unknown_tx_hashes();

            self.message
                .transactions()
                .iter()
                .map(|tx| (tx.transaction().to_entity().into_view(), tx.cycles().into()))
                .filter(|(tx, _)| {
                    !tx_filter.contains(&tx.hash())
                        && unknown_tx_hashes
                            .get_priority(&tx.hash())
                            .map(|priority| priority.requesting_peer() == Some(self.peer))
                            .unwrap_or_default()
                })
                .collect()
        };

        if txs.is_empty() {
            return Status::ok();
        }

        let max_block_cycles = self.relayer.shared().consensus().max_block_cycles();
        if txs
            .iter()
            .any(|(_, declared_cycles)| declared_cycles > &max_block_cycles)
        {
            self.nc.ban_peer(
                self.peer,
                DEFAULT_BAN_TIME,
                String::from("relay declared cycles greater than max_block_cycles"),
            );
            return Status::ok();
        }

        shared_state.mark_as_known_txs(txs.iter().map(|(tx, _)| tx.hash()));

        let tx_pool = self.relayer.shared.shared().tx_pool_controller().clone();
        let peer = self.peer;
        self.relayer
            .shared
            .shared()
            .async_handle()
            .spawn(async move {
                for (tx, declared_cycles) in txs {
                    if let Err(e) = tx_pool
                        .submit_remote_tx(tx.clone(), declared_cycles, peer)
                        .await
                    {
                        error!("submit_tx error {}", e);
                    }
                }
            });

        Status::ok()
    }
