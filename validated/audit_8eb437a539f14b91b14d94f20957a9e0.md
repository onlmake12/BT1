### Title
Unbounded Duplicate tx_hash DB Lookup Amplification in GetTransactionsProof with No Rate Limiting — (`util/light-client-protocol-server/src/components/get_transactions_proof.rs`)

---

### Summary

`GetTransactionsProofProcess::execute` performs one `get_transaction_info` and one `get_transaction_with_info` DB call **per raw list entry**, with no deduplication. The only guard rejects messages with `len > 1000`, so a message with exactly 1000 identical hashes passes and triggers 2000 DB reads for a single unique transaction. The light client protocol handler has **no rate limiter** at all, unlike the `Relayer` and `HolePunching` handlers.

---

### Finding Description

**Guard check** — the only validation is:

```rust
if self.message.tx_hashes().len() > constant::GET_TRANSACTIONS_PROOF_LIMIT {
    return StatusCode::MalformedProtocolMessage.with_context("too many transactions");
}
```

`GET_TRANSACTIONS_PROOF_LIMIT = 1000`, so `len == 1000` passes. [1](#0-0) [2](#0-1) 

**First amplification — partition step**: iterates every raw entry and calls `get_transaction_info` for each, including duplicates:

```rust
.partition(|tx_hash| {
    snapshot
        .get_transaction_info(tx_hash)   // called 1000× for the same hash
        ...
});
``` [3](#0-2) 

**Second amplification — found loop**: iterates `found` (still contains all 1000 duplicates) and calls `get_transaction_with_info` for each:

```rust
for tx_hash in found {
    let (tx, tx_info) = snapshot
        .get_transaction_with_info(&tx_hash)  // called 1000× for the same hash
        .expect("tx exists");
    txs_in_blocks
        .entry(tx_info.block_hash)
        .or_insert_with(Vec::new)
        .push((tx, tx_info.index));
}
``` [4](#0-3) 

The `txs_in_blocks` `HashMap` is keyed by `block_hash`, so all 1000 duplicates collapse into one block entry. The CBMT proof is therefore built **once** (not 1000 times as the question claims), but the Vec passed to it contains 1000 duplicate `(tx, index)` pairs, causing unnecessary work inside the proof builder. [5](#0-4) 

**No rate limiting**: `LightClientProtocol` has no `rate_limiter` field and `try_process` applies no per-peer throttle, unlike `Relayer` (which uses `governor::RateLimiter<(PeerIndex, u32)>`) and `HolePunching`. [6](#0-5) [7](#0-6) 

---

### Impact Explanation

Each message with 1000 identical hashes causes **2000 DB reads** (1000 `get_transaction_info` + 1000 `get_transaction_with_info`) instead of 2. With no rate limiter, a single peer can flood the server with such messages, multiplying DB I/O by 1000× per message. This can exhaust I/O bandwidth and degrade or halt service on nodes running the light client server. [8](#0-7) 

---

### Likelihood Explanation

The attack requires only a valid P2P connection to a node with the `LightClient` protocol enabled. No key, privilege, or PoW is needed. The attacker needs one known on-chain tx hash. The message is structurally valid and passes all existing checks. [9](#0-8) 

---

### Recommendation

1. **Deduplicate tx_hashes** before any DB access:
   ```rust
   let tx_hashes: Vec<_> = self.message.tx_hashes().to_entity()
       .into_iter()
       .collect::<std::collections::HashSet<_>>()
       .into_iter()
       .collect();
   ```
2. **Add a per-peer rate limiter** to `LightClientProtocol`, mirroring the `governor::RateLimiter<(PeerIndex, u32)>` pattern used in `Relayer`. [3](#0-2) 

---

### Proof of Concept

```
1. Connect to a node with LightClient protocol enabled.
2. Pick any known on-chain tx_hash T.
3. Send GetTransactionsProof { tx_hashes: [T] * 1000, last_hash: <tip> }.
4. Instrument snapshot.get_transaction_info / get_transaction_with_info.
5. Observe 1000 calls each (2000 total) instead of 1 each (2 total).
6. Repeat in a tight loop from one peer — no rate limiter fires.
```

    async fn received(
        &mut self,
        nc: Arc<dyn CKBProtocolContext + Sync>,
        peer: PeerIndex,

        let msg = match packed::LightClientMessageReader::from_slice(&data) {
            Ok(msg) => msg.to_enum(),
            _ => {
                warn!(
                    "LightClient.received a malformed message from Peer({})",
                    peer
                );
                nc.ban_peer(
                    peer,
                    constant::BAD_MESSAGE_BAN_TIME,
                    String::from("send us a malformed message"),
                );
                return;
            }
        };

        let item_name = msg.item_name();
        let status = self.try_process(&nc, peer, msg).await;
        if let Some(ban_time) = status.should_ban() {
            error!(
                "process {} from {}; ban {:?} since result is {}",
                item_name, peer, ban_time, status
            );
            nc.ban_peer(peer, ban_time, status.to_string());
        } else if status.should_warn() {
            warn!("process {} from {}; result is {}", item_name, peer, status);
        } else if !status.is_ok() {
            debug!("process {} from {}; result is {}", item_name, peer, status);
        }
    }

    async fn try_process(
        &mut self,
        nc: &Arc<dyn CKBProtocolContext + Sync>,
        peer_index: PeerIndex,
        message: packed::LightClientMessageUnionReader<'_>,
    ) -> Status {
        match message {
            packed::LightClientMessageUnionReader::GetLastState(reader) => {
                components::GetLastStateProcess::new(reader, self, peer_index, nc)
                    .execute()
                    .await
            }
            packed::LightClientMessageUnionReader::GetLastStateProof(reader) => {
                components::GetLastStateProofProcess::new(reader, self, peer_index, nc)
                    .execute()
                    .await
            }
            packed::LightClientMessageUnionReader::GetBlocksProof(reader) => {
                components::GetBlocksProofProcess::new(reader, self, peer_index, nc)
                    .execute()
                    .await
            }
            packed::LightClientMessageUnionReader::GetTransactionsProof(reader) => {
                components::GetTransactionsProofProcess::new(reader, self, peer_index, nc)
                    .execute()
                    .await
            }
            _ => StatusCode::UnexpectedProtocolMessage.into(),
        }
