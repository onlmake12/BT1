### Title
Missing Payload Timestamp Validation in `verifyUpdate()` Allows Stale Price Replay — (File: `lazer/contracts/evm/src/PythLazer.sol`)

### Summary
`PythLazer.sol::verifyUpdate()` validates the signer's key expiry but never checks the `timestamp` embedded in the signed payload against `block.timestamp`. Any unprivileged relayer can replay an arbitrarily old (but validly-signed) Lazer price update, and the contract will accept it as authentic. Consumer contracts that rely on `verifyUpdate()` as their trust anchor receive stale price data with no on-chain protection.

### Finding Description
`verifyUpdate()` performs three checks: fee sufficiency, EVM format magic, ECDSA signature recovery, and signer key expiry via `isValidSigner()`. It does not inspect the `timestamp` field that is embedded in the payload at bytes 4–12 (after the 4-byte `PAYLOAD_FORMAT_MAGIC`). [1](#0-0) 

The payload structure is documented by `PythLazerLib.sol::parsePayloadHeader()`, which reads the `timestamp` at position 4 of the payload: [2](#0-1) 

`verifyUpdate()` returns the raw `payload` to the caller without ever comparing `timestamp` to `block.timestamp`. The signer key expiry (`isValidSigner`) only bounds how long the *signing key* is trusted, not how fresh the *price data* is: [3](#0-2) 

The Sui and Aptos Lazer contracts do check signer expiry against the current clock, but neither checks the payload timestamp either: [4](#0-3) [5](#0-4) 

### Impact Explanation
A relayer can submit a Lazer price update signed hours or days ago. `verifyUpdate()` succeeds (the signer key is still valid). The consumer contract receives a `payload` containing a stale price. If the consumer does not implement its own maximum-age check — which is easy to omit and not enforced by the SDK — it will act on the stale price. In DeFi contexts (liquidations, collateral valuation, derivatives settlement) this directly enables financial loss.

The developer documentation example only shows a monotonicity guard (`_timestamp > timestamp`), not a maximum-age guard, making the omission easy to miss: [6](#0-5) 

### Likelihood Explanation
The attack requires no privilege: any address that can call `verifyUpdate{value: fee}(oldUpdate)` can execute it. Old signed updates are available from the Lazer WebSocket stream history. The signer key is governance-controlled and valid for extended periods. Consumer contracts that skip the staleness check are a realistic and common integration mistake.

### Recommendation
Add a `maxAge` parameter (or a hardcoded constant) to `verifyUpdate()`. After recovering the signer, parse the payload timestamp and revert if it is older than `block.timestamp - maxAge`:

```solidity
uint64 payloadTimestamp = uint64(bytes8(payload[4:12]));
require(
    block.timestamp <= payloadTimestamp / 1e6 + MAX_PRICE_AGE_SECONDS,
    "price update too old"
);
```

Alternatively, expose a `verifyUpdateWithMaxAge(bytes calldata update, uint64 maxAgeSeconds)` variant so consumers can enforce freshness at the protocol level rather than relying on each integrator to implement it correctly.

### Proof of Concept
1. At time `T`, a valid Lazer update `U` is broadcast. A relayer captures it.
2. At time `T + 3600` (1 hour later), the relayer calls `pythLazer.verifyUpdate{value: fee}(U)`.
3. `isValidSigner(signer)` returns `true` (signer key expires in years).
4. `verifyUpdate` returns `(payload, signer)` with no revert.
5. The consumer contract reads `timestamp = T` from the payload and uses a price that is 1 hour stale.
6. If the asset moved significantly in that hour, the consumer's logic (e.g., a liquidation check) operates on incorrect data. [1](#0-0)

    function verifyUpdate(
        bytes calldata update
    ) external payable returns (bytes calldata payload, address signer) {
        // Require fee and refund excess
        require(msg.value >= verification_fee, "Insufficient fee provided");
        if (msg.value > verification_fee) {
            payable(msg.sender).transfer(msg.value - verification_fee);
        }

        if (update.length < 71) {
            revert("input too short");
        }
        uint32 EVM_FORMAT_MAGIC = 706910618;

        uint32 evm_magic = uint32(bytes4(update[0:4]));
        if (evm_magic != EVM_FORMAT_MAGIC) {
            revert("invalid evm magic");
        }
        uint16 payload_len = uint16(bytes2(update[69:71]));
        if (update.length < 71 + payload_len) {
            revert("input too short");
        }
        payload = update[71:71 + payload_len];
        bytes32 hash = keccak256(payload);
        (signer, , ) = ECDSA.tryRecover(
            hash,
            uint8(update[68]) + 27,
            bytes32(update[4:36]),
            bytes32(update[36:68])
        );
        if (signer == address(0)) {
            revert("invalid signature");
        }
        if (!isValidSigner(signer)) {
            revert("invalid signer");
        }
    }

public(package) fun verify_le_ecdsa_message(
    state: &State,
    clock: &Clock,
    signature: &vector<u8>,
    payload: &vector<u8>,
) {
    let current_cap = state.current_cap();

    // 0 stands for keccak256 hash
    let pubkey = secp256k1_ecrecover(signature, payload, 0);

    // Check if the recovered pubkey is in the trusted signers list
    let trusted_signers = state.trusted_signers(&current_cap);
    let mut maybe_idx = trusted_signers.find_index!(|signer|
        signer.public_key() == &pubkey
    );

    assert!(maybe_idx.is_some(), ESignerNotTrusted);
    let idx = maybe_idx.extract();
    let expires_at_ms = trusted_signers[idx].expires_at_ms();
    assert!(clock.timestamp_ms() < expires_at_ms, ESignerExpired);
}
