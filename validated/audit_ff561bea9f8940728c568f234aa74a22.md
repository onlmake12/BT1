### Title
Unconditional `expect()` on `Option<Vec<u64>>` in `FeeRateCollector::statistics` Panics on Legacy `BlockExt` Records — (`rpc/src/util/fee_rate.rs`)

### Summary

`FeeRateCollector::statistics` calls `.expect()` unconditionally on `BlockExt.txs_sizes`, which is typed `Option<Vec<u64>>`. Any node whose database contains `BlockExt` records with `txs_sizes = None` — a normal state for blocks stored before `BlockExtV1` was introduced — will panic in the RPC handler when `get_fee_rate_statistics` is called by any unprivileged caller.

---

### Finding Description

The panic site is at line 93 of `rpc/src/util/fee_rate.rs`:

```rust
let txs_sizes = txs_sizes.expect("expect txs_size's length >= 1");
``` [1](#0-0) 

`txs_sizes` is destructured from a `BlockExt` value whose field is declared as:

```rust
pub txs_sizes: Option<Vec<u64>>,
``` [2](#0-1) 

The molecule schema has two on-disk formats. The legacy `BlockExt` table has no `txs_sizes` field at all:

```
table BlockExt {
    total_difficulty:   Uint256,
    ...
    verified:           BoolOpt,
}
``` [3](#0-2) 

The newer `BlockExtV1` adds `txs_sizes` as an optional field:

```
table BlockExtV1 {
    ...
    txs_sizes:          Uint64VecOpt,
}
``` [4](#0-3) 

When the store deserializes a legacy `BlockExt` record, the conversion code explicitly sets `txs_sizes: None`:

```rust
fn unpack(&self) -> core::BlockExt {
    core::BlockExt {
        ...
        cycles: None,
        txs_sizes: None,
    }
}
``` [5](#0-4) 

The `collect` helper uses `filter_map` to skip blocks where `get_block_ext_by_number` returns `None`, but it does **not** skip blocks where the returned `BlockExt` has `txs_sizes = None`:

```rust
let block_ext_iter =
    (start..=tip_number).filter_map(|number| self.get_block_ext_by_number(number));
block_ext_iter.fold(Vec::new(), f)
``` [6](#0-5) 

The RPC handler passes the result directly with no panic guard:

```rust
fn get_fee_rate_statistics(&self, target: Option<Uint64>) -> Result<Option<FeeRateStatistics>> {
    Ok(FeeRateCollector::new(self.shared.snapshot().as_ref())
        .statistics(target.map(Into::into)))
}
``` [7](#0-6) 

No migration in the codebase rewrites all legacy `BlockExt` records to `BlockExtV1` with `txs_sizes` populated. The existing migrations (`BlockExt2019ToZero`, `ChangeMoleculeTableToStruct`, etc.) do not address this field: [8](#0-7) 

---

### Impact Explanation

A panic in the RPC handler thread terminates that thread. Depending on whether the jsonrpc-http-server framework wraps the handler in `catch_unwind`, the result is either a 500 error for that request or termination of the RPC serving thread. Either way, the `get_fee_rate_statistics` (and the deprecated `get_fee_rate_statics`) endpoint becomes permanently unavailable or returns errors for every call as long as the legacy records remain in the window iterated by `collect`. This is a denial-of-service against the RPC API surface.

---

### Likelihood Explanation

Any node that has been running since before `BlockExtV1` was introduced will have legacy `BlockExt` records in `COLUMN_BLOCK_EXT`. The `collect` function iterates up to 101 recent blocks; if any of those blocks has a legacy record, the panic fires. The RPC endpoint requires no authentication. The attacker needs only to send a single HTTP POST with `method: "get_fee_rate_statistics"`.

---

### Recommendation

Replace the unconditional `.expect()` with a graceful skip:

```rust
let Some(txs_sizes) = txs_sizes else {
    return fee_rates; // skip blocks without txs_sizes data
};
```

This matches the existing pattern used for `cycles` (which is also `Option` and is handled with `if let Some(cycles) = cycles`). [9](#0-8) 

---

### Proof of Concept

Using the existing `DummyFeeRateProvider` test harness, add a block with `txs_sizes: None`:

```rust
provider.append(1, BlockExt {
    received_at: 0,
    total_difficulty: 0u64.into(),
    total_uncles_count: 0,
    verified: Some(true),
    txs_fees: vec![Capacity::shannons(1000)],
    cycles: Some(vec![100]),
    txs_sizes: None,   // ← legacy record
});
// This call panics unconditionally:
let _ = FeeRateCollector::new(&provider).statistics(None);
``` [10](#0-9) 

The test infrastructure already exists; only the `txs_sizes: None` case is missing from the test suite, confirming the gap.

#[test]
fn test_fee_rate_statics() {
    let mut provider = DummyFeeRateProvider::new(30);
    for i in 0..=21 {
        let ext = BlockExt {
            received_at: 0,
            total_difficulty: 0u64.into(),
            total_uncles_count: 0,
            verified: None,

            // txs_fees length is equal to block_ext.cycles length
            // and txs_fees does not include cellbase
            txs_fees: vec![Capacity::shannons(i * i * 100)],
            // cycles does not include cellbase
            cycles: Some(vec![i * 100]),
            // txs_sizes length is equal to block_ext.txs_fees length + 1
            // first element in txs_sizes is belong to cellbase
            txs_sizes: Some(vec![i * 5678, i * 100]),
        };
        provider.append(i, ext);
    }

    let statistics = FeeRateCollector::new(&provider).statistics(None);
    assert_eq!(
        statistics,
        Some(FeeRateStatistics {
            mean: 11_000.into(),
            median: 11_000.into(),
        })
    );
