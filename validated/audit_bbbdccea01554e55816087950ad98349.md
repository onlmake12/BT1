### Title
Provider Accrued Fees Permanently Locked When No Fee Manager Is Set - (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary
In the `Echo` contract, provider fees accumulate in `_state.providers[provider].accruedFeesInWei` via `executeCallback`, but the **only** withdrawal path is `withdrawAsFeeManager`, which requires `msg.sender == _state.providers[provider].feeManager`. Because `feeManager` defaults to `address(0)` and is never set during `registerProvider`, any provider that does not explicitly call `setFeeManager` has their accrued fees permanently locked in the contract.

### Finding Description
In `Echo.sol`, when `executeCallback` is called, the provider's fees are credited:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
``` [1](#0-0) 

The only function that allows a provider to withdraw these fees is `withdrawAsFeeManager`:

```solidity
function withdrawAsFeeManager(address provider, uint128 amount) external override {
    require(
        msg.sender == _state.providers[provider].feeManager,
        "Only fee manager"
    );
``` [2](#0-1) 

There is no direct `withdraw(amount)` function callable by the provider itself (unlike Entropy, which has `withdraw(uint128 amount)` callable by `msg.sender` directly). [3](#0-2) 

In `EchoState.sol`, `ProviderInfo.feeManager` is an `address` field that is never initialized during `registerProvider`:

```solidity
struct ProviderInfo {
    ...
    address feeManager;
    bool isRegistered;
    ...
}
``` [4](#0-3) 

`registerProvider` sets only `baseFeeInWei`, `feePerFeedInWei`, `feePerGasInWei`, and `isRegistered` — `feeManager` remains `address(0)`: [5](#0-4) 

With `feeManager == address(0)`, the check `msg.sender == _state.providers[provider].feeManager` can never pass (no EOA or contract can call from `address(0)`), so `withdrawAsFeeManager` is permanently blocked. The admin's `withdrawFees` only drains `_state.accruedFeesInWei` (the protocol fee), not provider fees: [6](#0-5) 

### Impact Explanation
Any provider who registers and accumulates fees without first calling `setFeeManager` has their entire `accruedFeesInWei` balance permanently locked in the contract. There is no admin escape hatch, no fallback withdrawal path, and no upgrade mechanism described in the contract itself. The ETH is irrecoverable.

### Likelihood Explanation
`setFeeManager` is an optional, undocumented prerequisite for fee withdrawal. A provider can register, receive callbacks, and accumulate fees without any on-chain signal that withdrawal is impossible. The `defaultProvider` set at initialization time also starts with `feeManager == address(0)` and faces the same issue until it explicitly calls `setFeeManager`. This is certain to affect providers who are unaware of the requirement.

### Recommendation
Add a direct `withdraw(uint128 amount)` function callable by the provider itself (`msg.sender`), mirroring the pattern in `Entropy.sol`:

```solidity
function withdraw(uint128 amount) external {
    ProviderInfo storage provider = _state.providers[msg.sender];
    require(provider.isRegistered, "Provider not registered");
    require(provider.accruedFeesInWei >= amount, "Insufficient balance");
    provider.accruedFeesInWei -= amount;
    (bool sent, ) = msg.sender.call{value: amount}("");
    require(sent, "withdrawal failed");
    emit FeesWithdrawn(msg.sender, amount);
}
```

Alternatively, initialize `feeManager` to the provider's own address in `registerProvider` so that `withdrawAsFeeManager` is immediately usable.

### Proof of Concept
1. Provider calls `registerProvider(baseFee, feedFee, gasFee)` — `feeManager` is `address(0)`.
2. Users call `requestPriceUpdatesWithCallback` — fees accumulate in `req.fee`.
3. Executor calls `executeCallback` — `_state.providers[provider].accruedFeesInWei` grows.
4. Provider attempts to call `withdrawAsFeeManager(providerAddress, amount)` — reverts with `"Only fee manager"` because `msg.sender != address(0)`.
5. Provider has no other withdrawal path. Fees are permanently locked.
