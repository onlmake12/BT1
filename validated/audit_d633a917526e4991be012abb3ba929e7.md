### Title
Hole-Punching `ConnectionRequest` Accepts Private/Loopback `listen_addrs`, Enabling SSRF-Like Internal TCP Probing and Resource Exhaustion — (`network/src/protocols/hole_punching/component/connection_request.rs`)

### Summary
An unprivileged remote peer can send a `ConnectionRequest` with `to = local_peer_id` and `listen_addrs` containing TCP multiaddrs pointing to loopback or RFC-1918 addresses. The victim node stores these addresses in `pending_delivered` without any private-IP check. When the attacker subsequently sends a `ConnectionSync`, the victim spawns async tasks that attempt outbound TCP connections to the attacker-chosen internal endpoints for up to 30 seconds each.

### Finding Description

**Entrypoint:** Any connected peer sends a `ConnectionRequest` P2P message.

**Step 1 — `ConnectionRequestProcess::execute`** [1](#0-0) 

When `self_peer_id == &content.to`, `respond_delivered` is called with the attacker-supplied `listen_addrs`.

**Step 2 — Missing private-IP filter in `respond_delivered`** [2](#0-1) 

The filter only rejects non-TCP transports and addresses without any IP component. `127.0.0.1`, `10.0.0.1`, `192.168.1.1`, `::1`, etc. all pass. The filtered addresses are then stored unconditionally: [3](#0-2) 

**Step 3 — `ConnectionSyncProcess::execute` retrieves and uses the poisoned addresses** [4](#0-3) 

Each stored address is passed directly to `try_nat_traversal`.

**Step 4 — `try_nat_traversal` performs blind TCP connection attempts** [5](#0-4) 

No IP-range check. The function retries every ~200 ms for a full 30-second window. Up to `ADDRS_COUNT_LIMIT = 24` addresses can be submitted per request. [6](#0-5) 

### Impact Explanation

- **SSRF-like internal probing:** The victim's TCP stack attempts connections to attacker-chosen internal addresses. A successful `connect()` vs. `ECONNREFUSED` leaks port-open/closed state of internal services (databases, admin APIs, local RPC, etc.) reachable from the victim host.
- **Resource exhaustion:** Each `try_nat_traversal` call is spawned as an independent async task running for up to 30 seconds. With 24 addresses per request and multiple attacker-controlled `from` peer IDs (each subject only to a 2-minute cooldown per ID), an attacker can maintain a sustained pool of long-lived connection-attempt tasks, consuming file descriptors, async task slots, and CPU.

### Likelihood Explanation

The attacker only needs to be a connected peer — no special privilege, no PoW, no key material. The victim's peer ID is public. The two-message sequence (`ConnectionRequest` → `ConnectionSync`) is straightforward to craft. The `HOLE_PUNCHING_INTERVAL` (2 minutes) rate-limits per `from` peer ID, but an attacker can rotate peer IDs freely. [7](#0-6) 

### Recommendation

In `respond_delivered`, extend the address filter to reject private and loopback IP ranges before inserting into `pending_delivered`. Concretely, after matching `TransportType::Tcp`, extract the `Ip4`/`Ip6` component and reject addresses where `ip.is_loopback() || ip.is_private() || ip.is_link_local() || ip.is_unspecified()`. The same guard should be applied in `try_nat_traversal` as a defense-in-depth check.

### Proof of Concept

1. Connect to a victim CKB node as peer `A` (attacker-controlled).
2. Send a `ConnectionRequest` with `from=A`, `to=<victim_peer_id>`, `listen_addrs=[/ip4/127.0.0.1/tcp/8080/p2p/<A>]`.
3. Receive the `ConnectionRequestDelivered` response (confirms the victim accepted and stored the address).
4. Send a `ConnectionSync` with `from=A`, `to=<victim_peer_id>`.
5. Observe (via `strace -e connect` or equivalent) that the victim node issues `connect(127.0.0.1:8080)` repeatedly for ~30 seconds.
6. Repeat with `/ip4/10.0.0.1/tcp/3306/p2p/<A>` to probe internal MySQL, etc.

pub(crate) async fn try_nat_traversal(
    bind_addr: Option<SocketAddr>,
    addr: Multiaddr,
) -> Result<(TcpStream, Multiaddr), std::io::Error> {
    let net_addr = match multiaddr_to_socketaddr(&addr) {
        Some(addr) => addr,
        None => {
            debug!("Failed to convert multiaddr to socketaddr");
            return Err(std::io::ErrorKind::InvalidInput.into());
        }
    };

    // Use a fixed interval but add a small amount of randomness
    let base_retry_interval = Duration::from_millis(200);

    // total time
    let timeout_duration = Duration::from_secs(30);
    let start_time = Instant::now();
    let mut retry_count = 0u32;
    while start_time.elapsed() < timeout_duration {
        retry_count += 1;

        // Add a small amount of random jitter (±25ms) to avoid conflicts
        // caused by continuous precise synchronization
        let jitter = Duration::from_millis(rand::random::<u64>() % 50);
        let actual_interval = if rand::random::<bool>() {
            base_retry_interval + jitter
        } else {
            base_retry_interval.saturating_sub(jitter)
        };

        let socket = create_socket(bind_addr, net_addr)?;

        match runtime::timeout(
            std::time::Duration::from_millis(200),
            socket.connect(net_addr),
        )
        .await
        {
            Ok(Ok(stream)) => {
                // try get the stored error in the underlying socket
                // if the socket is not connected, it will return an error
                if let Err(err) = check_connection(&stream) {
                    debug!("Failed to connect to NAT(base check): {}", err);
                }
                return Ok((stream, addr));
            }
            Err(err) => {
                debug!("Failed to connect to NAT(timeout): {}", err);
            }
            Ok(Err(err)) => {
                if err.kind() == std::io::ErrorKind::AddrNotAvailable {
                    return Err(err);
                }
                debug!(
                    "Failed to connect to NAT(other error): {}, {}",
                    err.kind(),
                    err
                );
            }
        }
        runtime::delay_for(actual_interval).await;
    }

    debug!("Failed to connect to NAT after {} retries", retry_count);
    Err(std::io::ErrorKind::TimedOut.into())
}
