### Title
Fee Charged for Equal-Timestamp (Non-Fresh) Updates With No Refund or Collection Path — (`target_chains/sui/contracts/sources/pyth.move`)

---

### Summary

`update_single_price_feed` unconditionally deposits the caller's fee into the `PriceInfoObject` **before** the freshness check. When `publish_time == cached_timestamp`, `is_fresh_update` returns `false` (strict `>` comparison), so no cache update and no event occur — but the fee is already locked inside the `PriceInfoObject` with no withdrawal path.

---

### Finding Description

**Step 1 — Fee deposited unconditionally.**

In `update_single_price_feed`, the fee sufficiency check and deposit happen before `update_cache` is ever called: [1](#0-0) 

```move
assert!(state::get_base_update_fee(pyth_state) <= coin::value(&fee), E_INSUFFICIENT_FEE);
// store fee coins within price info object
price_info::deposit_fee_coins(price_info_object, fee);
```

**Step 2 — Freshness check uses strict `>`.**

`is_fresh_update` returns `false` when `update_timestamp == cached_timestamp`: [2](#0-1) 

```move
fun is_fresh_update(update: &PriceInfo, price_info_object: &PriceInfoObject): bool {
    ...
    update_timestamp > cached_timestamp   // strict: equal timestamps → false
}
```

**Step 3 — No event, no cache update when not fresh.**

Inside `update_cache`, both the event emission and the cache write are gated on `is_fresh_update`: [3](#0-2) 

```move
if (is_fresh_update(update, price_info_object)){
    pyth_event::emit_price_feed_update(...);
    price_info::update_price_info_object(price_info_object, update);
}
```

**Step 4 — Fee is permanently locked; no withdrawal path exists.**

`deposit_fee_coins` stores the fee as a dynamic object field on the `PriceInfoObject`. There is no `withdraw_fee_coins` or equivalent function anywhere in `price_info.move`: [4](#0-3) 

The `set_fee_recipient` governance module explicitly comments that the old fee-forwarding mechanism **"is not used anymore"** — confirming fees accumulate in `PriceInfoObject` with no collection path: [5](#0-4) 

---

### Impact Explanation

Any caller who submits a price update whose `publish_time` equals the currently cached timestamp pays `base_update_fee` SUI, receives no cache update, receives no `PriceFeedUpdateEvent`, and has no mechanism to recover the fee. The coins are permanently frozen inside the `PriceInfoObject` dynamic field. This matches the stated scope of **permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

Moderate-low. Pythnet publishes prices at ~400 ms intervals but `publish_time` is a Unix second-granularity timestamp. Two consecutive Wormhole VAAs for the same feed within the same wall-clock second will carry identical `publish_time` values. A user who fetches a VAA and submits it slightly late — after another submitter already applied the same second's update — will hit exactly this condition. This is a realistic race condition in high-throughput environments, not a purely theoretical one.

---

### Recommendation

Move the fee deposit **after** the freshness check, or abort (refund via `transfer::public_transfer`) when `is_fresh_update` returns `false`. The simplest fix:

```move
// inside update_cache, before depositing:
if (!is_fresh_update(update, price_info_object)) {
    transfer::public_transfer(fee, tx_context::sender(ctx));
    return
};
price_info::deposit_fee_coins(price_info_object, fee);
```

Alternatively, change the freshness predicate to `>=` so equal-timestamp updates are accepted (idempotent write), consistent with how Ethereum's `updatePriceFeedsIfNecessary` uses strict `<` on the cached side.

---

### Proof of Concept

```
1. Deploy Sui Pyth with base_update_fee = 50.
2. Submit update A with publish_time = T  → cache updated, PriceFeedUpdateEvent emitted, 50 SUI deposited.
3. Submit update B with publish_time = T  (same VAA or re-signed same second):
   - assert coin::value(fee_paid) == 50          // fee taken
   - assert get_balance(price_info_object) == 100 // fee accumulated (50 + 50)
   - assert PriceFeedUpdateEvent count unchanged  // no event for B
   - assert cached publish_time still == T        // no cache change for B
4. No function exists to recover the second 50 SUI.
```

    fun is_fresh_update(update: &PriceInfo, price_info_object: &PriceInfoObject): bool {
        // Get the timestamp of the update's current price
        let price_feed = price_info::get_price_feed(update);
        let update_timestamp = price::get_timestamp(&price_feed::get_price(price_feed));

        // Get the timestamp of the cached data for the price identifier
        let cached_price_info = price_info::get_price_info_from_price_info_object(price_info_object);
        let cached_price_feed =  price_info::get_price_feed(&cached_price_info);
        let cached_timestamp = price::get_timestamp(&price_feed::get_price(cached_price_feed));

        update_timestamp > cached_timestamp
    }

    public fun deposit_fee_coins(price_info_object: &mut PriceInfoObject, fee_coins: Coin<SUI>) {
        if (!dynamic_object_field::exists_with_type<vector<u8>, Coin<SUI>>(&price_info_object.id, FEE_STORAGE_KEY)) {
            dynamic_object_field::add(&mut price_info_object.id, FEE_STORAGE_KEY, fee_coins);

        else {
            let current_fee = dynamic_object_field::borrow_mut<vector<u8>, Coin<SUI>>(
                &mut price_info_object.id,
                FEE_STORAGE_KEY
            );
            coin::join(current_fee, fee_coins);
        };
    }
