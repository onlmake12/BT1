### Title
EOA Caller of `requestV2`/`requestWithCallback` Permanently Loses Fee With No Callback Delivery — (`File: target_chains/ethereum/contracts/contracts/entropy/Entropy.sol`)

### Summary

`Entropy.sol` does not validate that `msg.sender` is a contract before storing it as the callback recipient (`req.requester`) in `requestV2` / `requestWithCallback`. When `revealWithCallback` is later executed, an EOA requester either has its callback silently skipped (old path) or vacuously succeeds via `excessivelySafeCall` (new path). In both cases the request is cleared, the fee is permanently consumed, and the EOA has no mechanism to retrieve the random number.

### Finding Description

`requestHelper` unconditionally stores `msg.sender` as `req.requester` and sets `callbackStatus = CALLBACK_NOT_STARTED` for all callback-style requests, with no check that the caller is a contract: [1](#0-0) 

When `revealWithCallback` is later called, two execution paths exist:

**Old path (`gasLimit10k == 0`, provider has no `defaultGasLimit`):** The code explicitly checks `extcodesize(callAddress)` and skips the callback for EOAs, but `clearRequest` has already been called before this check, permanently destroying the request: [2](#0-1) 

**New path (`gasLimit10k != 0`, provider has `defaultGasLimit > 0`):** `excessivelySafeCall` is used. A low-level call to an EOA always returns `success = true` with empty return data in the EVM. The `if (success)` branch fires, emits `RevealedWithCallback`, and calls `clearRequest`: [3](#0-2) 

In both paths the request is cleared. The EOA cannot fall back to `reveal` because that function requires `callbackStatus == CALLBACK_NOT_NECESSARY`, which reverts with `InvalidRevealCall` for any `requestV2` request: [4](#0-3) 

There is no refund path. The fee is split between the provider and Pyth protocol and is irrecoverable.

### Impact Explanation

An EOA that calls `requestV2` or `requestWithCallback` pays the full fee, receives a sequence number, but permanently loses both the fee and the random number. The `reveal` path is blocked by `InvalidRevealCall`. The `revealWithCallback` path clears the request without delivering anything useful to the EOA. Funds are permanently locked in provider/protocol fee balances with no recourse.

### Likelihood Explanation

Moderate. The `IEntropyV2` interface documentation states callers "should be a contract," but this is advisory only — no on-chain enforcement exists. Developers integrating Entropy who test directly from a wallet, or who call `requestV2()` (the zero-argument convenience overload) from an EOA during prototyping, will silently lose their fee. The zero-argument `requestV2()` overload is particularly inviting for direct EOA calls: [5](#0-4) 

### Recommendation

Add an EOA guard at the top of `requestHelper` (or in each `requestV2`/`requestWithCallback` entry point) that reverts when `isRequestWithCallback == true` and `msg.sender` has no code:

```solidity
if (isRequestWithCallback) {
    uint256 callerCodeSize;
    assembly { callerCodeSize := extcodesize(caller()) }
    if (callerCodeSize == 0) revert EntropyErrors.CallerMustBeContract();
}
```

This mirrors the existing `extcodesize` guard already present in the old `revealWithCallback` path (lines 670–681) and prevents fee loss at the point of request rather than silently at reveal time.

### Proof of Concept

1. EOA `alice` calls `random.requestV2{value: fee}(provider, userRandomNumber, gasLimit)` — succeeds, fee paid, sequence number `N` returned.
2. Provider calls `random.revealWithCallback(provider, N, userRandomNumber, providerProof)`.
   - **New path**: `excessivelySafeCall(alice, ...)` returns `(true, "")` because `alice` is an EOA. `clearRequest` fires. `RevealedWithCallback` emitted. Request gone.
   - **Old path**: `extcodesize(alice) == 0`, callback skipped, `clearRequest` already called. Request gone.
3. `alice` attempts `random.reveal(provider, N, ...)` → reverts `InvalidRevealCall` (request cleared or wrong `callbackStatus`).
4. Fee is permanently credited to provider/Pyth balances. `alice` has no recourse. [6](#0-5) [7](#0-6)
