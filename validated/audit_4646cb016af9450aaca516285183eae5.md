### Title
`PythAggregatorV3.latestRoundData()` Returns Arbitrarily Stale Prices With No Staleness Guard — (`target_chains/ethereum/sdk/solidity/PythAggregatorV3.sol`)

---

### Summary

`PythAggregatorV3` is Pyth's official Chainlink `AggregatorV3Interface`-compatible adapter, recommended for protocols migrating from Chainlink. Every price-reading function in the contract calls `getPriceUnsafe()`, which by design returns the last stored price with **no recency check whatsoever**. There is no configurable staleness threshold, no revert on stale data, and no way for a downstream caller to distinguish a fresh price from one that is hours old. This is the direct structural analog of the GMX/Arbitrum sequencer issue: a price oracle surface that silently serves stale data.

---

### Finding Description

`PythAggregatorV3.sol` implements the Chainlink `AggregatorV3Interface` so that existing Chainlink-integrated protocols can use Pyth feeds as a drop-in replacement. The migration guide in the repository explicitly recommends deploying this contract and pointing existing apps at it.

Every price-reading function delegates to `getPriceUnsafe()`:

```solidity
// latestRoundData — the primary function called by Chainlink-compatible protocols
function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId); // ← no staleness check
    ...
}

// latestAnswer
function latestAnswer() public view virtual returns (int256) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId); // ← no staleness check
    ...
}

// getRoundData
function getRoundData(uint80 _roundId) external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId); // ← no staleness check
    ...
}
```

`IPyth.getPriceUnsafe()` is explicitly documented:

> *"This function returns the most recent price update in this contract without any recency checks. This function is unsafe as the returned price update may be arbitrarily far in the past."*

The safe alternative, `getPriceNoOlderThan(id, age)`, exists in `AbstractPyth.sol` and reverts with `StalePrice` if `block.timestamp - price.publishTime > age`. It is never used in `PythAggregatorV3`.

The `updatedAt` field returned by `latestRoundData()` is set to `price.publishTime`, so a downstream protocol that checks `updatedAt` against a heartbeat threshold would see the correct timestamp — but the contract itself does not enforce any bound on it, and many Chainlink-compatible protocols do not perform this secondary check (as the GMX report itself demonstrates).

---

### Impact Explanation

Any protocol that integrates `PythAggregatorV3` as its price oracle — lending protocols, perpetuals, liquidation engines — will silently consume arbitrarily stale prices. Concrete scenarios:

1. **L2 sequencer downtime (direct analog to the GMX report):** On Arbitrum or other L2s, if the sequencer goes offline, Pyth price update transactions stop being processed. The on-chain Pyth contract retains the last stored price. `PythAggregatorV3.latestRoundData()` returns that stale price with no error. A user who is also a keeper can submit orders/liquidations at prices that no longer reflect reality.

2. **No one calls `updateFeeds()`:** Because Pyth is a pull oracle, prices only update when someone calls `updateFeeds()`. If the keeper network stalls or the protocol does not enforce fresh updates before reads, `latestRoundData()` returns the last stored price from an arbitrarily long time ago.

3. **Depeg / high-volatility events:** Precisely when staleness matters most (rapid price moves, depegs), the adapter provides no protection, enabling profitable exploitation of the price gap.

Impact: direct financial loss to protocols and their users through mispriced liquidations, undercollateralized borrows, or arbitrage against stale executable prices.

---

### Likelihood Explanation

- `PythAggregatorV3` is a production contract in Pyth's official Solidity SDK, actively recommended in the migration guide.
- Protocols migrating from Chainlink are the explicit target audience; they may not audit the adapter internals and assume staleness protection is inherited.
- No privileged access is required. Any unprivileged transaction sender can trigger the vulnerable path by interacting with a downstream protocol that uses this adapter.
- L2 sequencer downtime events (Arbitrum, Optimism, Base) are historically documented and non-negligible.

---

### Recommendation

Replace all `getPriceUnsafe()` calls in `PythAggregatorV3` with `getPriceNoOlderThan(priceId, maxAge)`, where `maxAge` is a constructor-configurable parameter (e.g., defaulting to 60 seconds):

```solidity
contract PythAggregatorV3 {
    bytes32 public priceId;
    IPyth public pyth;
    uint public maxAge; // staleness threshold in seconds

    constructor(address _pyth, bytes32 _priceId, uint _maxAge) {
        priceId = _priceId;
        pyth = IPyth(_pyth);
        maxAge = _maxAge;
    }

    function latestRoundData() external view returns (...) {
        PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
        // ...
    }
}
```

This causes `latestRoundData()` to revert with `StalePrice` when the price is too old, which is the correct behavior for a Chainlink-compatible adapter — matching how Chainlink's own sequencer uptime feeds work.

### Proof of Concept

1. Deploy `PythAggregatorV3` pointing at the Pyth contract on Arbitrum.
2. Stop submitting price updates (simulate sequencer downtime or keeper failure) for N minutes.
3. Call `latestRoundData()` — it returns the price from N minutes ago with no revert and no error signal.
4. A downstream lending protocol calls `latestRoundData()` to price collateral; the stale price is used for a liquidation or borrow at a price that no longer exists.
