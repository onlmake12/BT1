### Title
Silent Partial Unstake — No Validation That Requested Amount Was Fully Processed (`governance/pyth_staking_sdk/src/pyth-staking-client.ts`)

### Summary
`unstakeFromGovernance` and `unstakeFromPublisher` in the Pyth Staking SDK silently submit a partial (or zero-instruction) transaction when eligible staked positions total less than the user-requested `amount`. No check is performed after the position-closing loop to verify that `remainingAmount` has been reduced to zero. The transaction succeeds, the SDK returns without error, and the user believes the full requested amount was unstaked — but only a fraction (or nothing) was actually moved to cooldown.

### Finding Description
Both `unstakeFromGovernance` (lines 292–344) and `unstakeFromPublisher` (lines 346–401) in `governance/pyth_staking_sdk/src/pyth-staking-client.ts` follow the same pattern:

1. Filter positions to those matching the requested `positionState` and target.
2. Iterate through eligible positions, accumulating `closePosition` / `undelegate` instructions and decrementing `remainingAmount`.
3. After the loop, call `sendTransaction(instructions, ...)` — **with no assertion that `remainingAmount === 0n`**.

```typescript
// unstakeFromGovernance — lines 302–343
let remainingAmount = amount;
const instructionPromises: Promise<TransactionInstruction>[] = [];

for (const { position, index } of eligiblePositions) {
  if (position.amount < remainingAmount) {
    instructionPromises.push( /* close full position */ );
    remainingAmount -= position.amount;
  } else {
    instructionPromises.push( /* close partial position */ );
    break;
  }
}

// ← No check: if (remainingAmount > 0n) throw ...
const instructions = await Promise.all(instructionPromises);
return sendTransaction(instructions, this.connection, this.wallet);
```

Two concrete failure modes exist:

**Case A — No eligible positions**: `eligiblePositions` is empty (e.g., user requests `LOCKED` state but all positions are `LOCKING`). The loop body never executes, `instructionPromises` stays empty, and `sendTransaction([], ...)` submits a valid but no-op Solana transaction that succeeds.

**Case B — Insufficient eligible positions**: User requests to unstake 100 tokens; only 60 tokens are in the requested state. The loop closes 60 tokens worth of positions, exits with `remainingAmount = 40`, and submits a transaction that only partially fulfills the request.

In both cases the SDK resolves successfully, giving the caller no indication that the full amount was not processed.

### Impact Explanation
- A staking user who calls `unstakeGovernance` or `unstakeIntegrityStaking` (the public API wrappers in `apps/staking/src/api.ts` lines 350–396) expecting to move `amount` tokens into cooldown will silently retain staked positions they intended to close.
- Tokens that remain staked continue to accrue OIS slashing exposure, which the user believed they had exited.
- Governance voting power is not reduced as the user intended, which can affect governance participation accounting.
- A user racing to exit before a known slashing event or governance deadline may believe they are safe when they are not.

### Likelihood Explanation
This is triggered by any staking user whose positions are split across states (e.g., some `LOCKED`, some `LOCKING`) or whose total staked amount in the requested state is less than the requested unstake amount — a common real-world scenario when a user has staked in multiple epochs or has partially cooled-down positions. No special privileges or external conditions are required; any staking user can reach this path.

### Recommendation
After the position-closing loop, assert that `remainingAmount` has been fully consumed before submitting the transaction:

```typescript
if (remainingAmount > 0n) {
  throw new Error(
    `Insufficient eligible positions: requested ${amount}, ` +
    `but only ${amount - remainingAmount} tokens are in the ` +
    `${positionState} state for the given target.`
  );
}
```

Apply the same fix to both `unstakeFromGovernance` and `unstakeFromPublisher`.

### Proof of Concept
1. User has a stake account with 60 PYTH tokens in `LOCKED` governance positions and 40 PYTH tokens in `LOCKING` governance positions.
2. User calls `unstakeGovernance(stakeAccount, 100n)` (via `apps/staking/src/api.ts` line 350), which internally calls `unstakeFromGovernance(stakeAccount, PositionState.LOCKED, 100n)`.
3. `eligiblePositions` contains only the 60-token `LOCKED` position.
4. The loop closes 60 tokens; `remainingAmount` becomes `40n`.
5. No error is thrown. `sendTransaction` is called with only the 60-token close instruction.
6. The Solana transaction succeeds. The SDK resolves with no error.
7. The user's UI shows "Unstake successful" but 40 tokens remain staked in `LOCKED` state, still exposed to slashing. [1](#0-0) [2](#0-1) [3](#0-2)

  public async unstakeFromGovernance(
    stakeAccountPositions: PublicKey,
    positionState: PositionState.LOCKED | PositionState.LOCKING,
    amount: bigint,
  ) {
    const stakeAccountPositionsData = await this.getStakeAccountPositions(
      stakeAccountPositions,
    );
    const currentEpoch = await getCurrentEpoch(this.connection);

    let remainingAmount = amount;
    const instructionPromises: Promise<TransactionInstruction>[] = [];

    const eligiblePositions = stakeAccountPositionsData.data.positions
      .map((p, i) => ({ index: i, position: p }))
      .reverse()
      .filter(
        ({ position }) =>
          position.targetWithParameters.voting !== undefined &&
          positionState === getPositionState(position, currentEpoch),
      );

    for (const { position, index } of eligiblePositions) {
      if (position.amount < remainingAmount) {
        instructionPromises.push(
          this.stakingProgram.methods
            .closePosition(index, convertBigIntToBN(position.amount), {
              voting: {},
            })
            .accounts({
              stakeAccountPositions,
            })
            .instruction(),
        );
        remainingAmount -= position.amount;
      } else {
        instructionPromises.push(
          this.stakingProgram.methods
            .closePosition(index, convertBigIntToBN(remainingAmount), {
              voting: {},
            })
            .accounts({
              stakeAccountPositions,
            })
            .instruction(),
        );
        break;
      }
    }

    const instructions = await Promise.all(instructionPromises);
    return sendTransaction(instructions, this.connection, this.wallet);
  }
