### Title
Fee Collected Before Freshness Check Causes Permanent SUI Lock in PriceInfoObject — (`target_chains/sui/contracts/sources/pyth.move`)

---

### Summary

In `update_single_price_feed`, the SUI fee is unconditionally deposited into the `PriceInfoObject` **before** the freshness check. When the submitted update carries a `publish_time` equal to (or less than) the cached timestamp, `is_fresh_update` returns `false`, no state change occurs, no `PriceFeedUpdateEvent` is emitted, but the fee is already locked inside the object. Because no withdrawal path exists anywhere in the contract (the fee-recipient mechanism is explicitly deprecated), those SUI coins are permanently frozen.

---

### Finding Description

**Step 1 — Fee deposited unconditionally** [1](#0-0) 

`deposit_fee_coins` is called at line 277, before `update_cache` is ever reached.

**Step 2 — Freshness check uses strict `>`** [2](#0-1) 

`is_fresh_update` returns `update_timestamp > cached_timestamp`. When `publish_time == cached_timestamp`, this is `false`. The event emission and state write inside `update_cache` are both skipped. [3](#0-2) 

**Step 3 — No withdrawal path**

`price_info.move` exposes `deposit_fee_coins` (public) but contains no corresponding withdrawal or transfer function. [4](#0-3) 

**Step 4 — Fee-recipient mechanism is deprecated**

The governance module that previously forwarded fees to a recipient address is explicitly marked as no longer in use: [5](#0-4) 

Fees accumulate in the `PriceInfoObject`'s dynamic object field (`FEE_STORAGE_KEY`) with no on-chain path to recover them.

---

### Impact Explanation

Every call to `update_single_price_feed` with a non-fresh price (same or older `publish_time`) silently consumes `base_update_fee` SUI from the caller and locks it permanently inside the `PriceInfoObject`. The protocol cannot collect it (fee-recipient path removed), and the caller cannot reclaim it (no withdrawal function). The balance accumulates monotonically and is provably unspendable.

---

### Likelihood Explanation

This is trivially reachable by any unprivileged actor:
- Submitting the same accumulator VAA twice in the same or consecutive transactions (identical `publish_time`).
- Any integrator whose retry logic re-submits a VAA after a partial failure.
- A griefing actor who front-runs a legitimate update with the same VAA, causing the legitimate caller's fee to be wasted on the second call.

No privileged role, leaked key, or governance action is required.

---

### Recommendation

Move the fee deposit **after** the freshness check, or refund the fee when `is_fresh_update` returns `false`:

```move
// In update_cache, after is_fresh_update check:
if (is_fresh_update(update, price_info_object)) {
    price_info::deposit_fee_coins(price_info_object, fee);  // deposit only on accepted update
    pyth_event::emit_price_feed_update(...);
    price_info::update_price_info_object(...);
} else {
    transfer::public_transfer(fee, tx_context::sender(ctx)); // refund
}
```

Alternatively, add a governance-callable `collect_fees` function that transfers the accumulated balance to the fee recipient, and enforce that `deposit_fee_coins` is `public(friend)` only.

    fun is_fresh_update(update: &PriceInfo, price_info_object: &PriceInfoObject): bool {
        // Get the timestamp of the update's current price
        let price_feed = price_info::get_price_feed(update);
        let update_timestamp = price::get_timestamp(&price_feed::get_price(price_feed));

        // Get the timestamp of the cached data for the price identifier
        let cached_price_info = price_info::get_price_info_from_price_info_object(price_info_object);
        let cached_price_feed =  price_info::get_price_feed(&cached_price_info);
        let cached_timestamp = price::get_timestamp(&price_feed::get_price(cached_price_feed));

        update_timestamp > cached_timestamp
    }

    public fun deposit_fee_coins(price_info_object: &mut PriceInfoObject, fee_coins: Coin<SUI>) {
        if (!dynamic_object_field::exists_with_type<vector<u8>, Coin<SUI>>(&price_info_object.id, FEE_STORAGE_KEY)) {
            dynamic_object_field::add(&mut price_info_object.id, FEE_STORAGE_KEY, fee_coins);
        }
        else {
            let current_fee = dynamic_object_field::borrow_mut<vector<u8>, Coin<SUI>>(
                &mut price_info_object.id,
                FEE_STORAGE_KEY
            );
            coin::join(current_fee, fee_coins);
        };
    }
