### Title
Missing Caller Authentication on Critical State-Mutating RPC Functions Allows Any Local Process to Isolate Node and Wipe Transaction Pool - (File: `rpc/src/server.rs`, `rpc/src/module/net.rs`, `rpc/src/module/pool.rs`)

---

### Summary

The CKB RPC server has no authentication middleware of any kind. Critical state-mutating functions — `set_network_active`, `clear_tx_pool`, `remove_transaction`, `set_ban`, and `clear_banned_addresses` — perform zero caller identity verification. Any process that can reach the RPC port (any local process by default, or any remote host if the operator exposes the port) can invoke these functions unconditionally, enabling node network isolation, targeted transaction eviction, and peer-ban manipulation.

---

### Finding Description

**Root cause — no authentication layer in the RPC server:**

`rpc/src/server.rs` builds the Axum router with `CorsLayer::permissive()` and no authentication middleware. The `handle_jsonrpc` dispatcher forwards every incoming request directly to the method handler with no credential check:

```rust
let app = Router::new()
    .route("/", method_router.clone())
    .route("/{*path}", method_router)
    .route("/ping", get(ping_handler))
    .layer(Extension(Arc::clone(rpc)))
    .layer(CorsLayer::permissive())   // ← permissive CORS, no auth layer
    ...
``` [1](#0-0) 

The TCP server path (`start_tcp_server`) is equally unauthenticated — it accepts any connection and pipes it straight into the JSON-RPC dispatcher: [2](#0-1) 

**Critical functions with no authorization check:**

`set_network_active` in `rpc/src/module/net.rs` calls `network_controller.set_active(state)` with no guard:

```rust
fn set_network_active(&self, state: bool) -> Result<()> {
    self.network_controller.set_active(state);
    Ok(())
}
``` [3](#0-2) 

`clear_tx_pool` in `rpc/src/module/pool.rs` wipes the entire mempool with no guard:

```rust

``` [4](#0-3)

None of these implementations check the caller's identity, IP, token, or any credential. The trait definitions confirm no auth parameter is threaded through: [7](#0-6) [8](#0-7) [9](#0-8) 

---

### Impact Explanation

**`set_network_active(false)`** — disables all P2P networking on the node. The node stops receiving new blocks and transactions from peers. A service (exchange, wallet, payment processor) relying on this node for transaction confirmation will see a frozen chain tip. An attacker can exploit this window to execute a double-spend: broadcast a transaction to the isolated node, receive confirmation from the service, then broadcast a conflicting transaction to the real network. Severity: **critical** — enables double-spend against services using the node.

**`clear_tx_pool()`** — atomically removes every pending transaction from the mempool. All fee-paying transactions submitted by users are silently dropped and must be resubmitted. This is a targeted denial-of-service against the node's transaction relay function and against any application that submitted transactions and is waiting for confirmation.

**`remove_transaction(tx_hash)`** — allows targeted eviction of a specific transaction. An attacker who knows a victim's pending transaction hash can evict it from the pool, delaying confirmation and potentially enabling fee-sniping or front-running.

**`clear_banned_addresses()` / `set_ban()`** — allows an attacker to unban previously-banned malicious peers (re-enabling eclipse or Sybil connections) or to ban legitimate peers (degrading connectivity).

---

### Likelihood Explanation

The RPC binds to `127.0.0.1:8114` by default, making it reachable by **any process running on the same host** — including web browsers (via CORS, which is set to `permissive`), other user-space applications, or malicious code executing in the same environment. The prompt explicitly lists "RPC caller" and "supported local CLI/RPC user" as valid attacker profiles.

Additionally, operators frequently expose the RPC port to internal networks or the public internet (the config explicitly supports this via `listen_address`). The documentation warns against this but provides no enforcement mechanism. When exposed, the attack surface is fully remote with no credential barrier. [10](#0-9) 

---

### Recommendation

1. Add an optional API token middleware to the Axum router in `rpc/src/server.rs`. All state-mutating methods (`set_network_active`, `clear_tx_pool`, `remove_transaction`, `set_ban`, `clear_banned_addresses`, `add_node`, `remove_node`) should require the token.
2. Separate read-only RPC methods from write/admin methods, and apply authentication only to the admin group, reducing friction for read-only callers.
3. For the TCP RPC endpoint (`start_tcp_server`), apply the same token check at connection time before dispatching any method.

---

### Proof of Concept

No credentials, no special access — any shell on the host (or any remote host if the port is exposed):

```bash
# 1. Disable all P2P networking — node is now isolated from the chain
curl -s -X POST http://127.0.0.1:8114 \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"set_network_active","params":[false],"id":1}'
# → {"jsonrpc":"2.0","result":null,"id":1}

# 2. Wipe the entire mempool — all pending transactions are dropped
curl -s -X POST http://127.0.0.1:8114 \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"clear_tx_pool","params":[],"id":2}'
# → {"jsonrpc":"2.0","result":null,"id":2}

# 3. Evict a specific victim transaction by hash
curl -s -X POST http://127.0.0.1:8114 \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"remove_transaction","params":["0xa0ef4eb5f4ceeb08a4c8524d84c5da95dce2f608e0ca2ec8091191b0f330c6e3"],"id":3}'
# → {"jsonrpc":"2.0","result":true,"id":3}

# 4. Clear all banned peers — re-enables previously-banned malicious nodes
curl -s -X POST http://127.0.0.1:8114 \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"clear_banned_addresses","params":[],"id":4}'
# → {"jsonrpc":"2.0","result":null,"id":4}
```

All four calls succeed with no authentication required. The `set_network_active(false)` call is immediately effective: the node stops all P2P message processing, confirmed by `sync_state` returning a frozen tip and `get_peers` returning an empty list.

    async fn start_tcp_server(
        rpc: Arc<MetaIoHandler<Option<Session>>>,
        tcp_listen_address: String,
        handler: Handle,
    ) -> Result<SocketAddr, AnyError> {
        // TCP server with line delimited json codec.
        let listener = TcpListener::bind(tcp_listen_address).await?;
        let tcp_address = listener.local_addr()?;
        handler.spawn(async move {
            let codec = LinesCodec::new_with_max_length(2 * 1024 * 1024);
            let stream_config = StreamServerConfig::default()
                .with_channel_size(4)
                .with_pipeline_size(4)
                .with_shutdown(async move {
                    new_tokio_exit_rx().cancelled().await;
                });

            let exit_signal: CancellationToken = new_tokio_exit_rx();
            tokio::select! {
                _ = async {
                        while let Ok((stream, _)) = listener.accept().await {
                            let rpc = Arc::clone(&rpc);
                            let stream_config = stream_config.clone();
                            let codec = codec.clone();
                            tokio::spawn(async move {
                                let (r, w) = stream.into_split();
                                let r = FramedRead::new(r, codec.clone()).map_ok(StreamMsg::Str);
                                let w = FramedWrite::new(w, codec).with(|msg| async move {
                                    Ok::<_, LinesCodecError>(match msg {
                                        StreamMsg::Str(msg) => msg,
                                        _ => "".into(),
                                    })
                                });
                                tokio::pin!(w);
                                if let Err(err) = serve_stream_sink(&rpc, w, r, stream_config).await {
                                    info!("TCP RPCServer error: {:?}", err);
                                }
                            });
                        }
                    } => {},
                _ = exit_signal.cancelled() => {
                    info!("TCP RPCServer stopped");
                }
            }
        });
        Ok(tcp_address)
    }

    fn set_ban(
        &self,
        address: String,
        command: String,
        ban_time: Option<Timestamp>,
        absolute: Option<bool>,
        reason: Option<String>,
    ) -> Result<()> {
        let ip_network = address.parse().map_err(|_| {
            RPCError::invalid_params(format!(
                "Expected `params[0]` to be a valid IP address, got {address}"
            ))
        })?;

        match command.as_ref() {
            "insert" => {
                let ban_until = if absolute.unwrap_or(false) {
                    ban_time.unwrap_or_default().into()
                } else {
                    unix_time_as_millis()
                        + ban_time
                            .unwrap_or_else(|| DEFAULT_BAN_DURATION.into())
                            .value()
                };
                self.network_controller
                    .ban(ip_network, ban_until, reason.unwrap_or_default());
                Ok(())
            }
            "delete" => {
                self.network_controller.unban(&ip_network);
                Ok(())
            }
            _ => Err(RPCError::invalid_params(format!(
                "Expected `params[1]` to be in the list [insert, delete], got {address}"
            ))),
        }
    }
