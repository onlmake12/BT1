### Title
`update_price_feeds_if_necessary` Not Payable But Internally Requires Fee Payment - (`File: target_chains/stylus/contracts/pyth-receiver/src/lib.rs`)

---

### Summary

The `update_price_feeds_if_necessary` function in the Stylus (Arbitrum) Pyth receiver is not marked `#[payable]`, yet it internally delegates to `update_price_feeds`, which enforces a strict `msg_value() >= total_fee` check. Because the outer function cannot receive ETH, `msg_value()` is always zero when the inner fee check runs, causing every call to revert with `InsufficientFee` whenever the on-chain update fee is non-zero.

---

### Finding Description

In `target_chains/stylus/contracts/pyth-receiver/src/lib.rs`, `update_price_feeds` is correctly marked `#[payable]` and enforces a fee:

```rust
#[payable]
pub fn update_price_feeds(
    &mut self,
    update_data: Vec<Vec<u8>>,
) -> Result<(), PythReceiverError> {
    let total_fee = self.get_update_fee(update_data.clone())?;
    let value = self.vm().msg_value();
    if value < total_fee {
        return Err(PythReceiverError::InsufficientFee);
    }
    ...
}
``` [1](#0-0) 

However, `update_price_feeds_if_necessary` is **not** marked `#[payable]`:

```rust
pub fn update_price_feeds_if_necessary(   // ← no #[payable]
    &mut self,
    update_data: Vec<Vec<u8>>,
    price_ids: Vec<[u8; 32]>,
    publish_times: Vec<u64>,
) -> Result<(), PythReceiverError> {
    ...
    self.update_price_feeds(update_data.clone())?;  // ← always sees msg_value() == 0
    ...
}
``` [2](#0-1) 

In Stylus, omitting `#[payable]` causes the framework to revert any call that includes ETH. Therefore:

- If a caller sends ETH → the Stylus framework reverts before the function body executes.
- If a caller sends no ETH → `self.vm().msg_value()` returns `U256::ZERO`, and `update_price_feeds` returns `Err(PythReceiverError::InsufficientFee)` whenever `total_fee > 0`.

The result is that `update_price_feeds_if_necessary` is permanently broken on the Stylus receiver for any non-zero fee configuration.

Compare with the correct Solidity implementation in `Pyth.sol`, where `updatePriceFeedsIfNecessary` is `payable` and the internal call to `updatePriceFeeds` inherits the same `msg.value`:

```solidity
function updatePriceFeedsIfNecessary(
    bytes[] calldata updateData,
    bytes32[] calldata priceIds,
    uint64[] calldata publishTimes
) external payable override {          // ← payable, msg.value flows through
    ...
    updatePriceFeeds(updateData);
    ...
}
``` [3](#0-2) 

The Solidity version works because `payable` allows ETH to be received and `msg.value` is preserved across internal calls. The Stylus version lacks the `#[payable]` attribute, breaking the same pattern.

---

### Impact Explanation

Any on-chain consumer or protocol that calls `update_price_feeds_if_necessary` on the Stylus receiver will always receive `InsufficientFee` when the update fee is non-zero. Price feeds cannot be conditionally updated through this entry point, breaking the gas-optimization pattern the function is designed to provide. Downstream contracts relying on fresh prices via this path will fail silently or revert, potentially causing stale-price reads or denial of service in dependent DeFi protocols.

---

### Likelihood Explanation

The Pyth update fee is non-zero in production deployments. Any unprivileged caller invoking `update_price_feeds_if_necessary` on the Stylus receiver will trigger the failure. No special conditions, privileged access, or attacker-controlled state are required — the function is unconditionally broken for all callers.

---

### Recommendation

Add `#[payable]` to `update_price_feeds_if_necessary` in the Stylus receiver, mirroring the Solidity implementation:

```rust
#[payable]                              // ← add this
pub fn update_price_feeds_if_necessary(
    &mut self,
    update_data: Vec<Vec<u8>>,
    price_ids: Vec<[u8; 32]>,
    publish_times: Vec<u64>,
) -> Result<(), PythReceiverError> {
```

This allows callers to forward ETH with the call, which `self.vm().msg_value()` will then correctly observe inside `update_price_feeds`.

### Proof of Concept

1. Deploy the Stylus Pyth receiver with a non-zero `singleUpdateFeeInWei`.
2. Call `update_price_feeds_if_necessary` with valid `update_data`, `price_ids`, and `publish_times` for a stale feed, sending the required fee as ETH.
3. Observe: the Stylus framework reverts because the function is not `#[payable]`.
4. Call again with `msg.value = 0`.
5. Observe: `update_price_feeds` returns `Err(PythReceiverError::InsufficientFee)` because `msg_value() = 0 < total_fee`.
6. Confirm: `update_price_feeds` called directly with the same ETH succeeds, proving the fee mechanism itself is correct and only the missing `#[payable]` on the wrapper is the root cause. [2](#0-1) [4](#0-3)

    pub fn update_price_feeds_if_necessary(
        &mut self,
        update_data: Vec<Vec<u8>>,
        price_ids: Vec<[u8; 32]>,
        publish_times: Vec<u64>,
    ) -> Result<(), PythReceiverError> {
        if (price_ids.len() != publish_times.len())
            || (price_ids.is_empty() && publish_times.is_empty())
        {
            return Err(PythReceiverError::InvalidUpdateData);
        }

        for i in 0..price_ids.len() {
            if self.latest_price_info_publish_time(price_ids[i]) < publish_times[i] {
                self.update_price_feeds(update_data.clone())?;
                return Ok(());
            }
        }

        return Err(PythReceiverError::NoFreshUpdate);
    }

    function updatePriceFeedsIfNecessary(
        bytes[] calldata updateData,
        bytes32[] calldata priceIds,
        uint64[] calldata publishTimes
    ) external payable override {
        if (priceIds.length != publishTimes.length)
            revert PythErrors.InvalidArgument();

        for (uint i = 0; i < priceIds.length; ) {
            // If the price does not exist, then the publish time is zero and
            // this condition will work fine.
            if (latestPriceInfoPublishTime(priceIds[i]) < publishTimes[i]) {
                updatePriceFeeds(updateData);
                return;
            }

            unchecked {
                i++;
            }
        }

        revert PythErrors.NoFreshUpdate();
    }
