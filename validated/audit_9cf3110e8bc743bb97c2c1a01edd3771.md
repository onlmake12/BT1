### Title
Peer Store Purge Bypass via Crafted /16-Subnet Distribution — (`network/src/peer_store/peer_store_impl.rs`)

### Summary

The `check_purge` function in `PeerStore` has a logical dead zone: its group-based eviction path only removes addresses from groups with **strictly more than 4 members** (`addrs.len() > 4`). An attacker who fills the peer store with exactly 4 addresses per `/16` IPv4 subnet exhausts both eviction paths, causing `check_purge` to return `EvictionFailed`. All subsequent `add_addr` calls from honest peers silently fail, blocking honest peer discovery while the store remains saturated with attacker-controlled addresses.

---

### Finding Description

**`check_purge` — two-stage eviction logic:** [1](#0-0) 

**Stage 1** collects addresses where `!is_connectable(now_ms)`. A freshly-injected address via `add_addr` is created with `last_connected_at_ms = 0`, `attempts_count = 0`, `last_tried_at_ms = 0`: [2](#0-1) 

With `attempts_count = 0 < ADDR_MAX_RETRIES (3)` and `attempts_count = 0 < ADDR_MAX_FAILURES (10)`, every freshly-injected address returns `is_connectable = true`. Stage 1 finds zero candidates.

**Stage 2** groups by `Group::IP4([bits[0], bits[1]])` (the `/16` prefix): [3](#0-2) 

It then sorts groups by size and iterates the top half, but only evicts from groups where `addrs.len() > 4`: [4](#0-3) 

If every group has exactly 4 members, `4 > 4` is `false` for all groups → `candidate_peers` is empty → `EvictionFailed` is returned: [5](#0-4) 

**`add_addr` propagates the error:** [6](#0-5) 

**Discovery path silently drops the error:** [7](#0-6) 

**Attacker entry point — P2P discovery `Nodes` messages:** [8](#0-7) 

A non-announce `Nodes` message can carry up to `MAX_ADDR_TO_SEND = 1000` items per session. With ~17 attacker-controlled sessions (or fewer combined with repeated announce messages of up to 10 items each), the attacker can inject 16,384 addresses spread across 4,096 distinct `/16` subnets (4 per subnet), filling `ADDR_COUNT_LIMIT`: [9](#0-8) 

---

### Impact Explanation

Once the peer store is saturated in this configuration:
- Every `add_addr` call for a new honest peer returns `Err(EvictionFailed)` and is silently dropped.
- The victim node cannot learn about new honest peers through the discovery protocol.
- The attacker can sustain the attack by continuously re-injecting fresh fake addresses to replace any that get evicted after failed connection attempts (3 failures per address × 16,384 addresses = a large but finite replacement budget the attacker can keep replenishing).
- Existing connections are unaffected, but the node's ability to expand its peer set or recover from peer churn is permanently impaired for the duration of the attack.

---

### Likelihood Explanation

- Requires only P2P connectivity — no privileged access, no PoW, no key material.
- The attacker does not need to control the advertised IPs; they only need to advertise fake addresses with the right `/16` distribution.
- `ADDR_COUNT_LIMIT = 16384` with 4 per `/16` requires 4,096 distinct `/16` prefixes, which is trivially achievable with synthetic addresses (e.g., `1.0.x.y`, `1.1.x.y`, … `16.0.x.y`, etc.).
- The off-by-one threshold (`> 4` instead of `>= 4` or `> 0`) is the root cause and is not compensated by any other guard.

---

### Recommendation

Change the eviction threshold from strict greater-than to greater-than-or-equal, or remove the threshold entirely so that any group is eligible for eviction when both stage-1 and stage-2 produce no candidates:

```rust
// Before (vulnerable):
if addrs.len() > 4 {

// After (fixed):
if addrs.len() >= 1 {   // or simply remove the guard
```

Additionally, consider adding a per-session cap on the total number of addresses accepted from a single peer to limit the rate at which any one connection can fill the store.

---

### Proof of Concept

```rust
// Fill peer store with ADDR_COUNT_LIMIT addresses, 4 per /16 subnet
let mut peer_store = PeerStore::default();
let mut count = 0usize;
'outer: for a in 0u8..=255 {
    for b in 0u8..=255 {
        for c in 0u8..4 {
            if count >= ADDR_COUNT_LIMIT { break 'outer; }
            let addr: Multiaddr = format!(
                "/ip4/{}.{}.1.{}/tcp/8114/p2p/{}",
                a, b, c, PeerId::random().to_base58()
            ).parse().unwrap();
            peer_store.add_addr(addr, Flags::COMPATIBILITY).unwrap();
            count += 1;
        }
    }
}
assert_eq!(peer_store.addr_manager().count(), ADDR_COUNT_LIMIT);

// Now any honest peer address fails
let honest: Multiaddr = format!(
    "/ip4/8.8.8.8/tcp/8114/p2p/{}", PeerId::random().to_base58()
).parse().unwrap();
let result = peer_store.add_addr(honest, Flags::COMPATIBILITY);
assert!(result.is_err()); // EvictionFailed — honest peer blocked
```

    fn check_purge(&mut self) -> Result<()> {
        if self.addr_manager.count() < ADDR_COUNT_LIMIT {
            return Ok(());
        }

        // Evicting invalid data in the peer store is a relatively rare operation
        // There are certain cleanup strategies here:
        // 1. First evict the nodes that have reached the eviction condition
        // 2. If the first step is unsuccessful, enter the network segment grouping mode
        //  2.1. Group current data according to network segment
        //  2.2. Sort according to the amount of data in the same network segment
        //  2.3. In the network segment with more than 4 peer, randomly evict 2 peer

        let now_ms = ckb_systemtime::unix_time_as_millis();
        let candidate_peers: Vec<_> = self
            .addr_manager
            .addrs_iter()
            .filter_map(|addr| {
                if !addr.is_connectable(now_ms) {
                    Some(addr.addr.clone())
                } else {
                    None
                }
            })
            .collect();

        for key in candidate_peers.iter() {
            self.addr_manager.remove(key);
        }

        if candidate_peers.is_empty() {
            let candidate_peers: Vec<_> = {
                let mut peers_by_network_group: HashMap<Group, Vec<_>> = HashMap::default();
                for addr in self.addr_manager.addrs_iter() {
                    peers_by_network_group
                        .entry((&addr.addr).into())
                        .or_default()
                        .push(addr);
                }
                let len = peers_by_network_group.len();
                let mut peers = peers_by_network_group
                    .drain()
                    .map(|(_, v)| v)
                    .collect::<Vec<Vec<_>>>();

                peers.sort_unstable_by_key(|k| std::cmp::Reverse(k.len()));

                peers
                    .into_iter()
                    .take(len / 2)
                    .flat_map(move |addrs| {
                        if addrs.len() > 4 {
                            Some(
                                addrs
                                    .iter()
                                    .choose_multiple(&mut rand::thread_rng(), 2)
                                    .into_iter()
                                    .map(|addr| addr.addr.clone())
                                    .collect::<Vec<Multiaddr>>(),
                            )
                        } else {
                            None
                        }
                    })
                    .flatten()
                    .collect()
            };

            for key in candidate_peers.iter() {
                self.addr_manager.remove(key);
            }

            if candidate_peers.is_empty() {
                return Err(PeerStoreError::EvictionFailed.into());
            }
        }
        Ok(())

    pub fn is_connectable(&self, now_ms: u64) -> bool {
        // do not remove addr tried in last minute
        if self.tried_in_last_minute(now_ms) {
            return true;
        }
        // we give up if never connect to this addr
        if self.last_connected_at_ms == 0 && self.attempts_count >= ADDR_MAX_RETRIES {
            return false;
        }
        // consider addr is not connectable if failed too many times
        if now_ms.saturating_sub(self.last_connected_at_ms) > ADDR_TIMEOUT_MS
            && (self.attempts_count >= ADDR_MAX_FAILURES)
        {
            return false;
        }
        true
    }
