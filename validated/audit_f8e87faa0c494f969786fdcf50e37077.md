### Title
`getPricesNoOlderThan` Staleness Check Uses Subscription-Level Max Timestamp, Allowing Stale Individual Feed Prices to Pass Freshness Validation — (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

The `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` functions in `Scheduler.sol` validate freshness using a single subscription-level timestamp (`status.priceLastUpdatedAt`), which is set to the **maximum** `publishTime` across all feeds in the batch. When a subscription contains feeds for closed markets (e.g., equities alongside crypto), those feeds can have `publishTime` values from their last trading session — potentially hours or days old — while the staleness check passes because a different, recently-updated feed (e.g., BTC/USD) drives the max timestamp. A protocol relying on `getPricesNoOlderThan` to guarantee freshness will silently receive stale prices for individual feeds.

---

### Finding Description

**Step 1 — How `priceLastUpdatedAt` is set.**

In `_validateShouldUpdatePrices`, the representative timestamp for the entire subscription update is computed as the **maximum** `publishTime` across all feeds:

```solidity
uint256 updateTimestamp = 0;
for (uint8 i = 0; i < priceFeeds.length; i++) {
    if (priceFeeds[i].price.publishTime > updateTimestamp) {
        updateTimestamp = priceFeeds[i].price.publishTime;
    }
}
```

The code comment explicitly acknowledges the design intent: *"Use the most recent timestamp, as some asset markets may be closed. Closed markets will have a publishTime from their last trading period."*

This `updateTimestamp` (the max) is then stored as `status.priceLastUpdatedAt`:

```solidity
status.priceLastUpdatedAt = latestPublishTime;
``` [1](#0-0) [2](#0-1) 

**Step 2 — How `getPricesNoOlderThan` uses that timestamp.**

```solidity
function getPricesNoOlderThan(...) {
    SchedulerStructs.SubscriptionStatus memory status = _state
        .subscriptionStatuses[subscriptionId];

    if (distance(block.timestamp, status.priceLastUpdatedAt) > age_seconds)
        revert PythErrors.StalePrice();

    prices = this.getPricesUnsafe(subscriptionId, priceIds);
}
```

The check compares `block.timestamp` against the **subscription-level max timestamp**, not against each individual feed's `publishTime`. After passing this single check, `getPricesUnsafe` returns all stored feed prices without any per-feed freshness validation. [3](#0-2) 

The same flaw exists in `getEmaPricesNoOlderThan`: [4](#0-3) 

**Step 3 — The gap.**

Consider a subscription with two feeds:
- **BTC/USD** (crypto, always active): `publishTime = T` (just updated)
- **AAPL/USD** (equity, market closed): `publishTime = T − 8 hours` (last trading session)

After a keeper push, `priceLastUpdatedAt = T` (the max). A caller invoking `getPricesNoOlderThan(subscriptionId, [BTC, AAPL], 60)` at time `T + 30s`:
- Check: `distance(T+30, T) = 30 ≤ 60` → **passes**
- Returns: BTC price (30s old, genuinely fresh) **and** AAPL price (8 hours old, stale)

The caller has no indication that AAPL's price is stale. The `publishTime` field is present in the returned `PythStructs.Price` struct, but the function's name and documented contract (`getPricesNoOlderThan`) implies all returned prices satisfy the age bound. [5](#0-4) [6](#0-5) 

---

### Impact Explanation

Any protocol that integrates with the Scheduler via `getPricesNoOlderThan` and uses the returned prices for financial operations (collateral valuation, liquidation thresholds, swap rates) can be exploited. An attacker who knows the true current market price of a closed-market asset (e.g., from a CEX or off-chain source) can trade against the protocol at the stale on-chain price, profiting from the discrepancy — exactly the arbitrage pattern described in the xRenzo report. The impact is amplified because the protocol has no reason to distrust the prices: it called the "safe" freshness-checked function.

**Severity: Medium.** Financial loss to protocols integrating Scheduler with mixed asset-class subscriptions.

---

### Likelihood Explanation

The Scheduler is explicitly designed to support mixed-asset subscriptions (the code comment about closed markets confirms this). Any subscription containing both crypto and equity/commodity feeds is affected. The `getPricesNoOlderThan` function is the recommended safe API per the SDK documentation and README. Protocols following the documented best practice are silently exposed. An unprivileged attacker needs only to observe the on-chain stale price and trade against a vulnerable protocol — no special access is required.

**Likelihood: Medium-High.** [7](#0-6) [8](#0-7) 

---

### Recommendation

Replace the subscription-level staleness check in `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` with a per-feed check that validates each individual feed's `publishTime` against `age_seconds`:

```solidity
function getPricesNoOlderThan(...) {
    PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
    prices = new PythStructs.Price[](priceFeeds.length);
    for (uint i = 0; i < priceFeeds.length; i++) {
        if (distance(block.timestamp, priceFeeds[i].price.publishTime) > age_seconds)
            revert PythErrors.StalePrice();
        prices[i] = priceFeeds[i].price;
    }
}
```

Alternatively, if the current behavior (max-timestamp check) is intentional for closed-market support, the function's NatSpec and SDK documentation must be updated to clearly state that the age bound applies to the most recently updated feed in the subscription, not to every individual feed returned.

---

### Proof of Concept

1. Deploy `Scheduler` and create a subscription with two price IDs: `BTC_ID` (crypto) and `AAPL_ID` (equity).
2. Keeper calls `updatePriceFeeds` with update data where `BTC publishTime = T` and `AAPL publishTime = T − 28800` (8 hours ago, last trading session). Both share the same Pythnet slot, so `PriceSlotMismatch` is not triggered.
3. `_validateShouldUpdatePrices` sets `updateTimestamp = T` (max). `status.priceLastUpdatedAt = T`.
4. At time `T + 30`, a reader calls `getPricesNoOlderThan(subscriptionId, [BTC_ID, AAPL_ID], 60)`.
5. Check: `distance(T+30, T) = 30 ≤ 60` → no revert.
6. `getPricesUnsafe` returns both prices. AAPL price has `publishTime = T − 28800`.
7. A protocol consuming this result prices AAPL collateral at the 8-hour-old value. An attacker who knows AAPL's true current price exploits the discrepancy. [9](#0-8) [10](#0-9)

    function updatePriceFeeds(
        uint256 subscriptionId,
        bytes[] calldata updateData
    ) external override {
        uint256 startGas = gasleft();

        SchedulerStructs.SubscriptionStatus storage status = _state
            .subscriptionStatuses[subscriptionId];
        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        if (!params.isActive) {
            revert SchedulerErrors.InactiveSubscription();
        }

        // Get the Pyth contract and parse price updates
        IPyth pyth = IPyth(_state.pyth);
        uint256 pythFee = pyth.getUpdateFee(updateData);

        // If we don't have enough balance, revert
        if (status.balanceInWei < pythFee) {
            revert SchedulerErrors.InsufficientBalance();
        }

        // Parse the price feed updates with an acceptable timestamp range of [0, now+10s].
        // Note: We don't want to reject update data if it contains a price
        // from a market that closed a few days ago, since it will contain a timestamp
        // from the last trading period. Thus, we use a minimum timestamp of zero while parsing,
        // and we enforce the past max validity ourselves in _validateShouldUpdatePrices using
        // the highest timestamp in the update data.
        status.balanceInWei -= pythFee;
        status.totalSpent += pythFee;
        uint64 curTime = SafeCast.toUint64(block.timestamp);
        (
            PythStructs.PriceFeed[] memory priceFeeds,
            uint64[] memory slots
        ) = pyth.parsePriceFeedUpdatesWithConfig{value: pythFee}(
                updateData,
                params.priceIds,
                0, // We enforce the past max validity ourselves in _validateShouldUpdatePrices
                curTime + FUTURE_TIMESTAMP_MAX_VALIDITY_PERIOD,
                false,
                true,
                false
            );

        // Verify all price feeds have the same Pythnet slot.
        // All feeds in a subscription must be updated at the same time.
        uint64 slot = slots[0];
        for (uint8 i = 1; i < slots.length; i++) {
            if (slots[i] != slot) {
                revert SchedulerErrors.PriceSlotMismatch();
            }
        }

        // Verify that update conditions are met, and that the timestamp
        // is more recent than latest stored update's. Reverts if not.
        uint256 latestPublishTime = _validateShouldUpdatePrices(
            subscriptionId,
            params,
            status,
            priceFeeds
        );

        // Update status and store the updates
        status.priceLastUpdatedAt = latestPublishTime;
        status.totalUpdates += priceFeeds.length;

        _storePriceUpdates(subscriptionId, priceFeeds);

        _processFeesAndPayKeeper(status, startGas, params.priceIds.length);

        emit PricesUpdated(subscriptionId, latestPublishTime);
    }

    function _validateShouldUpdatePrices(
        uint256 subscriptionId,
        SchedulerStructs.SubscriptionParams storage params,
        SchedulerStructs.SubscriptionStatus storage status,
        PythStructs.PriceFeed[] memory priceFeeds
    ) internal view returns (uint256) {
        // Use the most recent timestamp, as some asset markets may be closed.
        // Closed markets will have a publishTime from their last trading period.
        // Since we verify all updates share the same Pythnet slot, we still ensure
        // that all price feeds are synchronized from the same update cycle.
        uint256 updateTimestamp = 0;
        for (uint8 i = 0; i < priceFeeds.length; i++) {
            if (priceFeeds[i].price.publishTime > updateTimestamp) {
                updateTimestamp = priceFeeds[i].price.publishTime;
            }
        }

        // Calculate the minimum acceptable timestamp (clamped at 0)
        // The maximum acceptable timestamp is enforced by the parsePriceFeedUpdatesWithSlots call
        uint256 minAllowedTimestamp = (block.timestamp >
            PAST_TIMESTAMP_MAX_VALIDITY_PERIOD)
            ? (block.timestamp - PAST_TIMESTAMP_MAX_VALIDITY_PERIOD)
            : 0;

        // Validate that the update timestamp is not too old
        if (updateTimestamp < minAllowedTimestamp) {
            revert SchedulerErrors.TimestampTooOld(
                updateTimestamp,
                block.timestamp
            );
        }

        // Reject updates if they're older than the latest stored ones
        if (
            status.priceLastUpdatedAt > 0 &&
            updateTimestamp <= status.priceLastUpdatedAt
        ) {
            revert SchedulerErrors.TimestampOlderThanLastUpdate(
                updateTimestamp,
                status.priceLastUpdatedAt
            );
        }

        // If updateOnHeartbeat is enabled and the heartbeat interval has passed, trigger update
        if (params.updateCriteria.updateOnHeartbeat) {
            uint256 lastUpdateTime = status.priceLastUpdatedAt;

            if (
                lastUpdateTime == 0 ||
                updateTimestamp >=
                lastUpdateTime + params.updateCriteria.heartbeatSeconds
            ) {
                return updateTimestamp;
            }
        }
