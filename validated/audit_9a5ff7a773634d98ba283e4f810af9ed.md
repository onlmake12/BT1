### Title
Subscription Manager Lock-In: No Transfer or Delegation Mechanism Permanently Locks Funds — (`target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

The `Scheduler` contract enforces subscription management exclusively through a strict `msg.sender == subscriptionManager` check with no mechanism to transfer the manager role or approve a delegate. If the manager address becomes inaccessible (e.g., a deprecated smart contract wallet, a migrated multisig, or a bricked proxy), all subscription funds are permanently locked with no recovery path.

---

### Finding Description

The `onlyManager` modifier in `Scheduler.sol` performs a hard equality check:

```solidity
modifier onlyManager(uint256 subscriptionId) {
    if (_state.subscriptionManager[subscriptionId] != msg.sender) {
        revert SchedulerErrors.Unauthorized();
    }
    _;
}
``` [1](#0-0) 

The manager is set once at subscription creation to `msg.sender` and never updated:

```solidity
_state.subscriptionManager[subscriptionId] = msg.sender;
``` [2](#0-1) 

This modifier gates both critical management functions:

- `updateSubscription()` — modify subscription parameters or deactivate
- `withdrawFunds()` — recover deposited ETH [3](#0-2) [4](#0-3) 

A search across the entire codebase confirms there is no `transferManager`, `setManager`, `delegateManager`, or any equivalent function anywhere in the Solidity contracts. The `SchedulerState` mapping `subscriptionManager` is write-once. [5](#0-4) 

Additionally, `withdrawFunds` sends ETH directly to `msg.sender` (the manager), not to a configurable recipient:

```solidity
(bool sent, ) = msg.sender.call{value: amount}("");
``` [6](#0-5) 

This means even if a new address could somehow prove ownership, the withdrawal destination is hardcoded to the caller.

---

### Impact Explanation

Any ETH deposited into a subscription via `createSubscription` or `addFunds` is permanently locked if the manager address becomes inaccessible. There is no admin override, no governance escape hatch, and no manager transfer path. The `SchedulerGovernance` admin/owner roles have no ability to recover subscription funds on behalf of a user. [7](#0-6) 

Concrete loss scenarios:
1. A protocol deploys a non-upgradeable contract that creates a subscription. The contract is later deprecated with no `withdrawFunds` call path — ETH is permanently locked.
2. A Gnosis Safe multisig creates a subscription, then migrates to a new Safe address (new deployment) due to key compromise. The old Safe address is the manager; the new Safe cannot withdraw.
3. A protocol's contract is exploited and self-destructs. The subscription balance is unrecoverable.

---

### Likelihood Explanation

Smart contract addresses (multisigs, DAOs, protocol contracts) are common subscription creators in DeFi infrastructure. Contract migration, deprecation, and key-compromise-driven Safe redeployment are routine operational events. The absence of any manager transfer mechanism makes this a near-certain fund-loss vector over the lifetime of the protocol, not a theoretical edge case.

---

### Recommendation

1. Add a `transferManager(uint256 subscriptionId, address newManager)` function gated by `onlyManager`, allowing the current manager to hand off control.
2. Optionally add an `approveDelegate(uint256 subscriptionId, address delegate)` pattern to allow operators to act on behalf of the manager without full transfer.
3. Consider adding a governance-level emergency recovery function (e.g., callable by the contract owner) to handle irrecoverable manager addresses, with appropriate time-locks.

---

### Proof of Concept

1. Alice deploys a non-upgradeable `ProtocolV1` contract.
2. `ProtocolV1.setup()` calls `scheduler.createSubscription{value: 1 ether}(params)` — manager is set to `address(ProtocolV1)`.
3. Alice deploys `ProtocolV2` (new address) and deprecates `ProtocolV1` (no `withdrawFunds` selector in `ProtocolV1`).
4. Alice calls `scheduler.withdrawFunds(subscriptionId, 1 ether)` from `ProtocolV2` — reverts with `Unauthorized` because `_state.subscriptionManager[subscriptionId] == address(ProtocolV1) != msg.sender`.
5. Alice calls `scheduler.updateSubscription(subscriptionId, ...)` — same revert.
6. The 1 ETH is permanently locked. No admin, owner, or governance path exists to recover it. [1](#0-0) [8](#0-7)

    function withdrawFunds(
        uint256 subscriptionId,
        uint256 amount
    ) external override onlyManager(subscriptionId) {
        SchedulerStructs.SubscriptionStatus storage status = _state
            .subscriptionStatuses[subscriptionId];
        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        // Prevent withdrawals from permanent subscriptions
        if (params.isPermanent) {
            revert SchedulerErrors.CannotUpdatePermanentSubscription();
        }

        if (status.balanceInWei < amount) {
            revert SchedulerErrors.InsufficientBalance();
        }

        // If subscription is active, ensure minimum balance is maintained
        if (params.isActive) {
            uint256 minimumBalance = this.getMinimumBalance(
                uint8(params.priceIds.length)
            );
            if (status.balanceInWei - amount < minimumBalance) {
                revert SchedulerErrors.InsufficientBalance();
            }
        }

        status.balanceInWei -= amount;

        (bool sent, ) = msg.sender.call{value: amount}("");
        require(sent, "Failed to send funds");
    }

/**

abstract contract SchedulerGovernance is SchedulerState {
    event NewAdminProposed(address oldAdmin, address newAdmin);
    event NewAdminAccepted(address oldAdmin, address newAdmin);
    event SingleUpdateKeeperFeeSet(uint oldFee, uint newFee);
    event MinimumBalancePerFeedSet(uint oldBalance, uint newBalance);

    /**
     * @dev Returns the address of the proposed admin.
     */
    function proposedAdmin() public view virtual returns (address) {
        return _state.proposedAdmin;
    }

    /**
     * @dev Returns the address of the current admin.
     */
    function getAdmin() external view returns (address) {
        return _state.admin;
    }

    /**
     * @dev Proposes a new admin for the contract. Replaces the proposed admin if there is one.
     * Can only be called by either admin or owner.
     */
    function proposeAdmin(address newAdmin) public virtual {
        require(newAdmin != address(0), "newAdmin is zero address");

        _authorizeAdminAction();

        _state.proposedAdmin = newAdmin;
        emit NewAdminProposed(_state.admin, newAdmin);
    }

    /**
     * @dev The proposed admin accepts the admin transfer.
     */
    function acceptAdmin() external {
        if (msg.sender != _state.proposedAdmin)
            revert SchedulerErrors.Unauthorized();

        address oldAdmin = _state.admin;
        _state.admin = msg.sender;

        _state.proposedAdmin = address(0);
        emit NewAdminAccepted(oldAdmin, msg.sender);
    }

    /**
     * @dev Authorization check for admin actions
     * Must be implemented by the inheriting contract.
     */
