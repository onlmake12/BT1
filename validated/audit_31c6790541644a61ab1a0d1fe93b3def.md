### Title
Missing Domain Separator in `verifyUpdate` Enables Cross-Chain Signature Replay - (`lazer/contracts/evm/src/PythLazer.sol`)

### Summary

`PythLazer.verifyUpdate` computes the signed hash as `keccak256(payload)` with no EIP-712 domain separator, no chain ID, and no contract address. A valid Lazer price-update blob signed for one EVM deployment is cryptographically identical on every other EVM chain that trusts the same signer key. Any unprivileged caller can take a signed update observed on Chain A and submit it verbatim to `verifyUpdate` on Chain B; the call succeeds, the fee is collected, and the verified payload is returned to the consumer contract.

### Finding Description

In `PythLazer.sol`, `verifyUpdate` extracts the raw payload and signs only its keccak256 hash:

```solidity
payload = update[71:71 + payload_len];
bytes32 hash = keccak256(payload);          // ← no chain ID, no address
(signer, , ) = ECDSA.tryRecover(
    hash,
    uint8(update[68]) + 27,
    bytes32(update[4:36]),
    bytes32(update[36:68])
);
``` [1](#0-0) 

The signed message is purely `keccak256(payload)`. No EIP-712 `domainSeparator` (which would commit to `chainId` and `verifyingContract`) is mixed in. The contract also maintains no set of consumed update hashes, so the same bytes can be submitted an unlimited number of times. [2](#0-1) 

The Sui and Aptos Lazer contracts sign the raw `payload` / `message` bytes in the same way — no chain-specific domain is committed to in the signed data. [3](#0-2) [4](#0-3) 

The Cardano integration documentation explicitly acknowledges the absence of replay protection:

> "It does not enforce freshness of the update, nor does it disallow verifying the same update multiple times." [5](#0-4) 

### Impact Explanation

**Cross-chain replay**: Pyth Lazer operationally uses the same trusted-signer key set across all EVM deployments (Ethereum, Arbitrum, Base, etc.). A signed update blob observed on any one chain passes `isValidSigner` on every other chain. A consumer contract that calls `verifyUpdate` and does not independently enforce `block.chainid` or a freshness window accepts a cross-chain replayed payload as genuine.

**Same-chain stale-price replay**: Because no consumed-update registry exists, an attacker can replay an arbitrarily old update (e.g., a price from minutes or hours ago) on the same chain. Consumer contracts that do not check the `timestamp` field inside the payload — a check the SDK documentation recommends but does not enforce — will accept the stale price as current.

The test suite itself demonstrates that the same `update` bytes can be submitted twice in the same test without any revert:

```solidity
vm.prank(alice);
pythLazer.verifyUpdate{value: fee}(update);   // first call – succeeds
vm.prank(alice);
pythLazer.verifyUpdate{value: 0.5 ether}(update); // second call – also succeeds
``` [6](#0-5) 

### Likelihood Explanation

- Any unprivileged caller can submit `verifyUpdate` — no role or key is required.
- Signed Lazer update blobs are publicly observable on-chain (calldata) and via the Lazer WebSocket stream.
- Pyth Lazer is deployed on multiple EVM chains with the same signer set; the cross-chain path requires only copying calldata from one chain's explorer to another.
- Consumer contracts that auto-pay the verification fee and do not validate `timestamp` are directly exploitable for stale-price injection.

### Recommendation

1. **Add an EIP-712 domain separator** to the signed hash, committing to `chainId` and the `PythLazer` contract address:
   ```solidity
   bytes32 domainSeparator = keccak256(abi.encode(
       keccak256("EIP712Domain(string name,uint256 chainId,address verifyingContract)"),
       keccak256("PythLazer"),
       block.chainid,
       address(this)
   ));
   bytes32 hash = keccak256(abi.encodePacked("\x19\x01", domainSeparator, keccak256(payload)));
   ```
2. **Optionally track consumed update hashes** (a `mapping(bytes32 => bool) usedUpdates`) to prevent same-chain replay, or enforce a maximum payload age (e.g., `timestamp` within the last N seconds) inside `verifyUpdate` itself rather than delegating that responsibility entirely to consumers.

### Proof of Concept

```solidity
// Attacker observes a valid update blob on Chain A (e.g., Arbitrum).
// The blob is publicly available in calldata on any block explorer.
bytes memory updateFromChainA = hex"2a22999a..."; // copied from Arbitrum tx

// Attacker submits the same blob on Chain B (e.g., Ethereum mainnet).
// verifyUpdate has no chainId check and no consumed-update registry.
(bytes memory payload, address signer) =
    pythLazerOnEthereum.verifyUpdate{value: fee}(updateFromChainA);
// ✓ succeeds — signer is valid, payload is returned
// Consumer contract on Ethereum now processes a price update
// that was signed for Arbitrum, with no indication of the mismatch.

// Same-chain replay: submit the same blob again on Chain A.
(payload, signer) =
    pythLazerOnArbitrum.verifyUpdate{value: fee}(updateFromChainA);
// ✓ also succeeds — no consumed-update check exists.
``` [7](#0-6)

    function verifyUpdate(
        bytes calldata update
    ) external payable returns (bytes calldata payload, address signer) {
        // Require fee and refund excess
        require(msg.value >= verification_fee, "Insufficient fee provided");
        if (msg.value > verification_fee) {
            payable(msg.sender).transfer(msg.value - verification_fee);
        }

        if (update.length < 71) {
            revert("input too short");
        }
        uint32 EVM_FORMAT_MAGIC = 706910618;

        uint32 evm_magic = uint32(bytes4(update[0:4]));
        if (evm_magic != EVM_FORMAT_MAGIC) {
            revert("invalid evm magic");
        }
        uint16 payload_len = uint16(bytes2(update[69:71]));
        if (update.length < 71 + payload_len) {
            revert("input too short");
        }
        payload = update[71:71 + payload_len];
        bytes32 hash = keccak256(payload);
        (signer, , ) = ECDSA.tryRecover(
            hash,
            uint8(update[68]) + 27,
            bytes32(update[4:36]),
            bytes32(update[36:68])
        );
        if (signer == address(0)) {
            revert("invalid signature");
        }
        if (!isValidSigner(signer)) {
            revert("invalid signer");
        }
    }

public(package) fun verify_le_ecdsa_message(
    state: &State,
    clock: &Clock,
    signature: &vector<u8>,
    payload: &vector<u8>,
) {
    let current_cap = state.current_cap();

    // 0 stands for keccak256 hash
    let pubkey = secp256k1_ecrecover(signature, payload, 0);

    // Check if the recovered pubkey is in the trusted signers list
    let trusted_signers = state.trusted_signers(&current_cap);
    let mut maybe_idx = trusted_signers.find_index!(|signer|
        signer.public_key() == &pubkey
    );

    assert!(maybe_idx.is_some(), ESignerNotTrusted);
    let idx = maybe_idx.extract();
    let expires_at_ms = trusted_signers[idx].expires_at_ms();
    assert!(clock.timestamp_ms() < expires_at_ms, ESignerExpired);
