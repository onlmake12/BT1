### Title
Subscription-Level Staleness Check in `getPricesNoOlderThan` Bypasses Per-Feed Freshness for Market-Hours Assets — (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

The `Scheduler` contract's `getPricesNoOlderThan` function validates freshness using a single subscription-level timestamp (`priceLastUpdatedAt`) that equals the **maximum** publish time across all feeds in the last update. For subscriptions containing both 24/7 assets (e.g., BTC) and market-hours assets (e.g., AAPL), the staleness check passes based on the crypto feed's recent timestamp while returning a days-old equity price. Downstream protocols that rely on `getPricesNoOlderThan` for freshness guarantees silently receive stale prices.

---

### Finding Description

In `Scheduler.sol`, `updatePriceFeeds` sets `status.priceLastUpdatedAt` to the **maximum** publish time across all feeds in the submitted update batch:

```solidity
// _validateShouldUpdatePrices
uint256 updateTimestamp = 0;
for (uint8 i = 0; i < priceFeeds.length; i++) {
    if (priceFeeds[i].price.publishTime > updateTimestamp) {
        updateTimestamp = priceFeeds[i].price.publishTime;
    }
}
// ...
status.priceLastUpdatedAt = latestPublishTime;
``` [1](#0-0) [2](#0-1) 

The `getPricesNoOlderThan` function then checks this single subscription-level timestamp against the caller's `age_seconds` threshold:

```solidity
function getPricesNoOlderThan(...) external view ... {
    SchedulerStructs.SubscriptionStatus memory status = _state.subscriptionStatuses[subscriptionId];
    if (distance(block.timestamp, status.priceLastUpdatedAt) > age_seconds)
        revert PythErrors.StalePrice();
    prices = this.getPricesUnsafe(subscriptionId, priceIds);
}
``` [3](#0-2) 

The Scheduler explicitly supports market-hours assets. The `updatePriceFeeds` code comment confirms this design:

> "We don't want to reject update data if it contains a price from a market that closed a few days ago, since it will contain a timestamp from the last trading period." [4](#0-3) 

When a subscription contains BTC (24/7) and AAPL (market-hours), a keeper submits an update where BTC has `publishTime = now` and AAPL has `publishTime = 2 days ago` (last market close). `priceLastUpdatedAt` is set to `now` (BTC's timestamp). A reader calling `getPricesNoOlderThan(subscriptionId, [AAPL_ID], 60)` passes the check (distance = ~0 seconds < 60) but receives a 2-day-old AAPL price via `getPricesUnsafe`. [5](#0-4) 

---

### Impact Explanation

Protocols that integrate with the Scheduler and call `getPricesNoOlderThan` with a tight age threshold (e.g., 60 seconds) for equity or commodity feeds will:

1. Believe the staleness check guarantees freshness.
2. Silently receive prices that are hours or days old for closed-market assets.
3. Execute incorrect liquidations, wrong collateral valuations, or mispriced derivatives based on stale data.

This is directly analogous to the DSC report: the staleness check mechanism fails to protect against stale oracle data, and there is no fallback — `getPricesUnsafe` is called unconditionally after the check passes.

---

### Likelihood Explanation

The Scheduler README and `updatePriceFeeds` code explicitly acknowledge and accommodate market-hours assets. Any subscription mixing 24/7 crypto feeds with equity/commodity feeds (a common DeFi use case) is affected. Any whitelisted reader of such a subscription can trigger the issue by calling `getPricesNoOlderThan` during off-market hours.

---

### Recommendation

Replace the subscription-level staleness check with a per-feed staleness check. In `getPricesNoOlderThan`, after retrieving the feeds via `_getPricesInternal`, validate each individual feed's `publishTime` against `age_seconds`:

```solidity
for (uint i = 0; i < priceFeeds.length; i++) {
    if (distance(block.timestamp, priceFeeds[i].price.publishTime) > age_seconds)
        revert PythErrors.StalePrice();
}
```

This ensures the freshness guarantee applies to every returned price, not just the most recently updated feed in the subscription.

---

### Proof of Concept

1. Deploy `SchedulerUpgradeable` and create a subscription with two price IDs: `BTC_USD` and `AAPL_USD`.
2. Call `updatePriceFeeds` with `BTC_USD.publishTime = block.timestamp` and `AAPL_USD.publishTime = block.timestamp - 2 days`. Both feeds share the same Pythnet slot (required by the slot-match check). `priceLastUpdatedAt` is set to `block.timestamp`.
3. As a whitelisted reader, call `getPricesNoOlderThan(subscriptionId, [AAPL_USD], 60)`.
4. Observe: the call succeeds (no `StalePrice` revert) and returns the 2-day-old AAPL price.
5. The `publishTime` field of the returned price confirms the staleness: `block.timestamp - 2 days`. [6](#0-5) [3](#0-2)

    function updatePriceFeeds(
        uint256 subscriptionId,
        bytes[] calldata updateData
    ) external override {
        uint256 startGas = gasleft();

        SchedulerStructs.SubscriptionStatus storage status = _state
            .subscriptionStatuses[subscriptionId];
        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        if (!params.isActive) {
            revert SchedulerErrors.InactiveSubscription();
        }

        // Get the Pyth contract and parse price updates
        IPyth pyth = IPyth(_state.pyth);
        uint256 pythFee = pyth.getUpdateFee(updateData);

        // If we don't have enough balance, revert
        if (status.balanceInWei < pythFee) {
            revert SchedulerErrors.InsufficientBalance();
        }

        // Parse the price feed updates with an acceptable timestamp range of [0, now+10s].
        // Note: We don't want to reject update data if it contains a price
        // from a market that closed a few days ago, since it will contain a timestamp
        // from the last trading period. Thus, we use a minimum timestamp of zero while parsing,
        // and we enforce the past max validity ourselves in _validateShouldUpdatePrices using
        // the highest timestamp in the update data.
        status.balanceInWei -= pythFee;
        status.totalSpent += pythFee;
        uint64 curTime = SafeCast.toUint64(block.timestamp);
        (
            PythStructs.PriceFeed[] memory priceFeeds,
            uint64[] memory slots
        ) = pyth.parsePriceFeedUpdatesWithConfig{value: pythFee}(
                updateData,
                params.priceIds,
                0, // We enforce the past max validity ourselves in _validateShouldUpdatePrices
                curTime + FUTURE_TIMESTAMP_MAX_VALIDITY_PERIOD,
                false,
                true,
                false
            );

        // Verify all price feeds have the same Pythnet slot.
        // All feeds in a subscription must be updated at the same time.
        uint64 slot = slots[0];
        for (uint8 i = 1; i < slots.length; i++) {
            if (slots[i] != slot) {
                revert SchedulerErrors.PriceSlotMismatch();
            }
        }

        // Verify that update conditions are met, and that the timestamp
        // is more recent than latest stored update's. Reverts if not.
        uint256 latestPublishTime = _validateShouldUpdatePrices(
            subscriptionId,
            params,
            status,
            priceFeeds
        );

        // Update status and store the updates
        status.priceLastUpdatedAt = latestPublishTime;
        status.totalUpdates += priceFeeds.length;

        _storePriceUpdates(subscriptionId, priceFeeds);

        _processFeesAndPayKeeper(status, startGas, params.priceIds.length);

        emit PricesUpdated(subscriptionId, latestPublishTime);
    }
