### Title
Cross-Chain Signature Replay in `PythLazer.verifyUpdate` — (`File: lazer/contracts/evm/src/PythLazer.sol`)

### Summary

`PythLazer.verifyUpdate` computes the signed hash as `keccak256(payload)` with no chain ID or contract address bound into the digest. Because the same trusted signer is registered on every EVM deployment of `PythLazer`, a signed update captured on one chain is cryptographically valid on every other chain.

### Finding Description

`verifyUpdate` in `PythLazer.sol` recovers the signer from the raw payload hash:

```solidity
payload = update[71:71 + payload_len];
bytes32 hash = keccak256(payload);          // ← no chain ID, no address(this)
(signer, , ) = ECDSA.tryRecover(
    hash,
    uint8(update[68]) + 27,
    bytes32(update[4:36]),
    bytes32(update[36:68])
);
``` [1](#0-0) 

The payload itself contains only a magic number, timestamp, channel, and feed data — no chain ID and no target contract address:

```
FORMAT_MAGIC (4) | timestamp (8) | channel (1) | feedsLen (1) | feeds...
``` [2](#0-1) 

`PythLazer` is deployed to a **deterministic address on every EVM chain** via CreateX (the deploy script explicitly sets `uint8(0)` — "No crosschain replay protection" — in the salt): [3](#0-2) 

The same trusted signer address is registered on all chains via `updateTrustedSigner`: [4](#0-3) 

`isValidSigner` only checks expiry, not chain context: [5](#0-4) 

### Impact Explanation

Any signed Lazer price update observed on chain A (e.g., Ethereum) can be submitted verbatim to `verifyUpdate` on chain B (e.g., Arbitrum, BNB Chain). The call succeeds and returns the payload and signer to the consumer contract. Consumer contracts that use the returned `payload` to update on-chain prices will accept stale or chain-mismatched price data. This enables an attacker to feed a DeFi protocol on chain B with price data that was signed for chain A, potentially at a different timestamp or price level, enabling price manipulation, incorrect liquidations, or mispriced trades.

### Likelihood Explanation

- `PythLazer` is live on many EVM mainnets and testnets with the same trusted signer set. [6](#0-5) 
- Every signed update is publicly observable on-chain (emitted in the transaction calldata).
- No special privilege is required: any EOA can call `verifyUpdate` with any `bytes` input.
- The attacker only needs to copy calldata from one chain and replay it on another.

### Recommendation

Bind the signed digest to the chain and contract address. Replace:

```solidity
bytes32 hash = keccak256(payload);
```

with:

```solidity
bytes32 hash = keccak256(abi.encodePacked(block.chainid, address(this), payload));
```

This mirrors the standard EIP-712 domain-separator pattern and ensures a signature produced for one `PythLazer` deployment cannot be accepted by any other.

### Proof of Concept

1. On Ethereum mainnet, call `PythLazer.verifyUpdate{value: fee}(update)` with a freshly signed update. Record the `update` bytes from the transaction calldata.
2. On Arbitrum (or any other chain where the same trusted signer is registered), submit the identical `update` bytes to `PythLazer.verifyUpdate{value: fee}(update)`.
3. The call succeeds: `keccak256(payload)` is identical on both chains, `ECDSA.tryRecover` returns the same signer address, and `isValidSigner` returns `true` because the signer is registered on both chains.
4. The consumer contract on Arbitrum receives the Ethereum-signed price data and updates its state accordingly — without the Lazer signer ever having signed a message for Arbitrum. [7](#0-6)

    function verifyUpdate(
        bytes calldata update
    ) external payable returns (bytes calldata payload, address signer) {
        // Require fee and refund excess
        require(msg.value >= verification_fee, "Insufficient fee provided");
        if (msg.value > verification_fee) {
            payable(msg.sender).transfer(msg.value - verification_fee);
        }

        if (update.length < 71) {
            revert("input too short");
        }
        uint32 EVM_FORMAT_MAGIC = 706910618;

        uint32 evm_magic = uint32(bytes4(update[0:4]));
        if (evm_magic != EVM_FORMAT_MAGIC) {
            revert("invalid evm magic");
        }
        uint16 payload_len = uint16(bytes2(update[69:71]));
        if (update.length < 71 + payload_len) {
            revert("input too short");
        }
        payload = update[71:71 + payload_len];
        bytes32 hash = keccak256(payload);
        (signer, , ) = ECDSA.tryRecover(
            hash,
            uint8(update[68]) + 27,
            bytes32(update[4:36]),
            bytes32(update[36:68])
        );
        if (signer == address(0)) {
            revert("invalid signature");
        }
        if (!isValidSigner(signer)) {
            revert("invalid signer");
        }
    }
