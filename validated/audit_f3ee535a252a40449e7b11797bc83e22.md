### Title
Expired Network Alert Signatures Replayed via P2P After `notice_until` Expiry — (`util/network-alert/src/alert_relayer.rs`)

---

### Summary

The P2P alert receive handler in `alert_relayer.rs` does not validate whether a received alert has already passed its `notice_until` expiry timestamp. The RPC entry point (`rpc/src/module/alert.rs`) enforces this check, but the P2P path does not. Because `clear_expired_alerts()` removes expired alerts from the deduplication map (`received_alerts`) without adding them to the permanent `cancel_filter`, any unprivileged P2P peer can replay a previously-broadcast, validly-signed (by Nervos Foundation keys) alert after it expires. The replayed alert passes signature verification, is accepted into the node's notifier, displayed to the operator, and re-broadcast to all connected peers.

---

### Finding Description

**Signed message scope:**

`RawAlert` is the signed payload. Its hash commits to all fields: `notice_until`, `id`, `cancel`, `priority`, `message`, `min_version`, `max_version`. [1](#0-0) 

`calc_alert_hash()` is simply `blake2b_256(raw_alert.as_slice())` — the full serialized `RawAlert`. [2](#0-1) 

The signatures are therefore cryptographically bound to a specific `notice_until` value. However, nothing in the P2P receive path enforces that this timestamp has not yet passed.

**RPC path — expiry enforced:**

`send_alert` in the RPC module explicitly rejects alerts whose `notice_until` is in the past: [3](#0-2) 

**P2P path — expiry NOT enforced:**

The `received()` handler in `AlertRelayer` only checks:
1. UTF-8 validity of string fields
2. Whether the alert `id` is already in `received_alerts` or `cancel_filter` (`has_received`)
3. Cryptographic signature validity

There is no check that `notice_until >= now`: [4](#0-3) 

**Deduplication gap after expiry:**

`clear_expired_alerts()` removes expired entries from `received_alerts` and `noticed_alerts`, but does **not** insert them into `cancel_filter`: [5](#0-4) 

After this purge, `has_received(id)` returns `false` for the expired alert's `id`: [6](#0-5) 

**Expiry purge is triggered on every new peer connection:**

`clear_expired_alerts()` is called inside `connected()`, meaning the attacker's own connection triggers the purge that opens the replay window: [7](#0-6) 

---

### Impact Explanation

An unprivileged P2P peer can:

1. Capture any validly-signed alert while it is live on the network (all alerts are publicly broadcast).
2. After `notice_until` passes, connect to a target node. The `connected()` callback calls `clear_expired_alerts()`, removing the alert from `received_alerts`.
3. Immediately send the expired alert bytes over the P2P alert protocol.
4. The node's `received()` handler: `has_received(id)` = false → signatures valid → `notifier.add()` called → `notify_network_alert` fires → alert displayed to operator.
5. The alert is also re-broadcast to all of the node's connected peers via `quick_filter_broadcast`, propagating the replay network-wide. [8](#0-7) 

The alert system is the mechanism by which the Nervos Foundation warns operators about critical bugs and required upgrades. Replaying a stale alert (e.g., "Upgrade to v0.105.*") after the affected version is no longer in use causes operators to see misleading, outdated critical warnings. Because the replay propagates to all peers, a single attacker can inject the stale alert across the entire reachable P2P network.

---

### Likelihood Explanation

- **No privileged access required.** Any node that was connected to the network when the original alert was live will have stored the raw alert bytes. Any observer can capture them from the P2P stream.
- **Attacker entry point is a standard P2P connection** — reachable by any unprivileged peer.
- **The attack window is permanent** once an alert expires: `cancel_filter` is never populated by expiry, so `has_received` stays false indefinitely for expired IDs (unless the ID is explicitly cancelled via a new alert).
- **Trigger is automatic:** the attacker's own connection triggers `clear_expired_alerts()`, making the timing trivial to control.

---

### Recommendation

Add an expiry check in the P2P `received()` handler, mirroring the check already present in the RPC `send_alert()` handler:

```rust
// In alert_relayer.rs::received(), after parsing the alert:
let notice_until: u64 = alert.raw().notice_until().into();
let now_ms = ckb_systemtime::unix_time_as_millis();
if notice_until < now_ms {
    // silently drop or optionally ban the peer
    return;
}
```

Additionally, when `clear_expired_alerts()` removes an alert from `received_alerts`, it should insert the `id` into `cancel_filter` so that `has_received(id)` remains true and prevents future replay of the same ID:

```rust
pub fn clear_expired_alerts(&mut self, now: u64) {
    let expired_ids: Vec<u32> = self.received_alerts
        .iter()
        .filter(|(_, alert)| {
            let notice_until: u64 = alert.raw().notice_until().into();
            notice_until <= now
        })
        .map(|(id, _)| *id)
        .collect();
    for id in expired_ids {
        self.received_alerts.remove(&id);
        self.cancel_filter.put(id, ()); // prevent replay
    }
    self.noticed_alerts.retain(|a| {
        let notice_until: u64 = a.raw().notice_until().into();
        notice_until > now
    });
}
```

### Proof of Concept

```
1. Run a CKB node (node A) and connect it to the network.
2. Observe a live alert broadcast (id=X, notice_until=T, with valid Nervos Foundation signatures).
   Save the raw alert bytes.
3. Wait until time T passes (alert expires).
4. Connect a custom P2P peer to node A using the CKB alert protocol.
   - This triggers connected() → clear_expired_alerts() → alert id=X removed from received_alerts.
5. Immediately send the saved raw alert bytes over the alert protocol sub-stream.
6. Node A's received() handler:
   - has_received(X) → false (just cleared)
   - verify_signatures() → OK (signatures are still cryptographically valid)
   - notifier.add() → alert inserted into received_alerts and noticed_alerts
   - notify_network_alert() fires → operator UI displays the stale alert
   - quick_filter_broadcast() → alert re-sent to all of node A's peers
7. The stale alert propagates across the network from node A.
```

Relevant code path confirming no expiry check exists between parse and `notifier.add()`: [9](#0-8)

    async fn received(
        &mut self,
        nc: Arc<dyn CKBProtocolContext + Sync>,
        peer_index: PeerIndex,
        data: Bytes,
    ) {
        let alert: packed::Alert = match packed::AlertReader::from_slice(&data) {
            Ok(alert) => {
                if alert.raw().message().is_utf8()
                    && alert
                        .raw()
                        .min_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                    && alert
                        .raw()
                        .max_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                {
                    alert.to_entity()
                } else {
                    info!(
                        "A malformed message fromP peer {} : not utf-8 string",
                        peer_index
                    );
                    nc.ban_peer(
                        peer_index,
                        BAD_MESSAGE_BAN_TIME,
                        String::from("send us a malformed message: not utf-8 string"),
                    );
                    return;
                }
            }
            Err(err) => {
                info!("A malformed message from peer {}: {:?}", peer_index, err);
                nc.ban_peer(
                    peer_index,
                    BAD_MESSAGE_BAN_TIME,
                    String::from("send us a malformed message"),
                );
                return;
            }
        };
        let alert_id = alert.as_reader().raw().id().into();
        trace!("ReceiveD alert {} from peer {}", alert_id, peer_index);
        // ignore alert
        if self.notifier.lock().has_received(alert_id) {
            return;
        }
        // verify
        if let Err(err) = self.verifier.verify_signatures(&alert) {
            debug!(
                "An alert from peer {} with invalid signatures, error {:?}",
                peer_index, err
            );
            nc.ban_peer(
                peer_index,
                BAD_MESSAGE_BAN_TIME,
                String::from("send us an alert with invalid signatures"),
            );
            return;
        }
        // mark sender as known
        self.mark_as_known(peer_index, alert_id);
        // broadcast message
        let selected_peers: Vec<PeerIndex> = nc
            .connected_peers()
            .into_iter()
            .filter(|peer| self.mark_as_known(*peer, alert_id))
            .collect();
        if let Err(err) = nc.quick_filter_broadcast(
            TargetSession::Multi(Box::new(selected_peers.into_iter())),
            data,
        ) {
            debug!("alert broadcast error: {:?}", err);
        }
        // add to received alerts
        self.notifier.lock().add(&alert);
    }
