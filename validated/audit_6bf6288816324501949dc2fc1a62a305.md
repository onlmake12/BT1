### Title
Missing Admin Setter for `pythFeeInWei` Breaks Post-Deployment Fee Management - (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

The `Echo` contract initializes `_state.pythFeeInWei` at deployment time but provides no admin function to update it afterward. Unlike the `Entropy` contract, which has `EntropyGovernance.setPythFee()`, the `Echo` contract has no equivalent setter. The protocol fee component of every request is permanently locked at the value set during `_initialize()`, and the admin has no on-chain mechanism to adjust it.

---

### Finding Description

In `Echo._initialize()`, `_state.pythFeeInWei` is set once:

```solidity
_state.pythFeeInWei = pythFeeInWei;
``` [1](#0-0) 

This value is then used in two critical places. First, it is deducted from `msg.value` to compute the provider's share of the fee stored in the request:

```solidity
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);
``` [2](#0-1) 

Second, it is credited to the protocol's accrued balance on every request:

```solidity
_state.accruedFeesInWei += _state.pythFeeInWei;
``` [3](#0-2) 

The `IEcho` interface exposes `getPythFeeInWei()` as a getter but defines no setter: [4](#0-3) 

The `Echo` contract itself provides admin setters for `exclusivityPeriodSeconds` and `defaultProvider`, but nothing for `pythFeeInWei`: [5](#0-4) 

By contrast, the `Entropy` contract's governance layer explicitly provides `setPythFee()`:

```solidity
function setPythFee(uint128 newPythFee) external {
    _authoriseAdminAction();
    _state.pythFeeInWei = newPythFee;
    emit PythFeeSet(oldPythFee, newPythFee);
}
``` [6](#0-5) 

No such function exists anywhere in the Echo contract hierarchy (`Echo.sol`, `EchoState.sol`, `EchoUpgradeable.sol`).



---

### Impact Explanation

The Echo protocol's Pyth fee component (`pythFeeInWei`) is permanently frozen at the value set during contract initialization. The admin cannot:

- Reduce the fee to zero to waive protocol fees during a promotional period.
- Increase the fee to reflect changed network economics.
- Correct a misconfigured deployment-time value without a full contract upgrade.

Every call to `getFee()` will return a fee that includes the immutable `pythFeeInWei`, and every call to `requestPriceUpdatesWithCallback()` will accrue that fixed amount to the protocol balance. The fee management feature is broken in the same way as Bond Protocol's `createFeeDiscount`: the state variable exists and is used, but the protocol team has no on-chain path to configure it post-deployment. [7](#0-6) 

---

### Likelihood Explanation

Certain. The setter function is simply absent. Any attempt by the admin to adjust `pythFeeInWei` after deployment will fail because no such function exists in the contract or its interface. This will manifest the first time the protocol team needs to adjust fees. [8](#0-7) 

---

### Recommendation

Add an admin-gated setter to `Echo.sol` (or a dedicated `EchoGovernance` contract, mirroring the Entropy pattern):

```solidity
function setPythFee(uint96 newPythFeeInWei) external {
    require(msg.sender == _state.admin, "Only admin");
    uint96 oldFee = _state.pythFeeInWei;
    _state.pythFeeInWei = newPythFeeInWei;
    emit PythFeeUpdated(oldFee, newPythFeeInWei);
}
```

Add the corresponding declaration to `IEcho.sol` and emit an appropriate event from `EchoEvents.sol`. [9](#0-8) 

---

### Proof of Concept

1. Deploy `EchoUpgradeable` with `pythFeeInWei = 1000 wei`.
2. Call `echo.getPythFeeInWei()` → returns `1000`.
3. As admin, attempt to call any fee-update function on the Echo contract → no such function exists; the ABI has no `setPythFee` entry.
4. Call `echo.getPythFeeInWei()` again → still `1000`; the fee is permanently locked.
5. Compare with `EntropyUpgradable`: calling `random.setPythFee(500)` as admin succeeds and updates the fee. [10](#0-9)

    function getFee(
        address provider,
        uint32 callbackGasLimit,
        bytes32[] calldata priceIds
    ) public view override returns (uint96 feeAmount) {
        uint96 baseFee = _state.pythFeeInWei; // Fixed fee to Pyth
        // Note: The provider needs to set its fees to include the fee charged by the Pyth contract.
        // Ideally, we would be able to automatically compute the pyth fees from the priceIds, but the
        // fee computation on IPyth assumes it has the full updated data.
        uint96 providerBaseFee = _state.providers[provider].baseFeeInWei;
        uint96 providerFeedFee = SafeCast.toUint96(
            priceIds.length * _state.providers[provider].feePerFeedInWei
        );
        uint96 providerFeeInWei = _state.providers[provider].feePerGasInWei; // Provider's per-gas rate
        uint256 gasFee = callbackGasLimit * providerFeeInWei; // Total provider fee based on gas
        feeAmount =
            baseFee +
            providerBaseFee +
            providerFeedFee +
            SafeCast.toUint96(gasFee); // Total fee user needs to pay
    }
