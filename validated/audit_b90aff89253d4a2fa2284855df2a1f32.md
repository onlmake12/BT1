Audit Report

## Title
Unbounded Unauthenticated DoS via `GetBlocksProof` with Mismatched `last_hash`/`block_hashes` Triggering Unpenalized `InternalError` — (`util/light-client-protocol-server/src/components/get_blocks_proof.rs`, `util/light-client-protocol-server/src/lib.rs`)

## Summary
`GetBlocksProofProcess::execute` accepts up to 1000 block hashes that are individually on the main chain but whose block numbers exceed `last_block.number()`. It performs 3 synchronous DB reads per hash (3000 total), then `reply_proof` constructs an MMR anchored at `last_block.number() - 1` that is too small to cover the requested positions, causing `gen_proof` to return `Err` and the handler to return `StatusCode::InternalError` (500). Because `should_ban()` only fires for codes in `400..500`, the peer is never penalized and can repeat the attack indefinitely with no rate limiting.

## Finding Description
`GET_BLOCKS_PROOF_LIMIT = 1000` and the guard is strict (`> 1000`), so exactly 1000 entries pass. [1](#0-0) [2](#0-1) 

The code validates that `last_hash` and each entry in `block_hashes` are individually on the main chain via `snapshot.is_main_chain`, but it never checks that each found block's number is ≤ `last_block.number()`. All 1000 hashes for blocks 2–1001 pass `is_main_chain` and land in `found`. [3](#0-2) 

The loop then performs `get_block_header`, `get_block_uncles`, and `get_block_extension` for each — 3000 DB reads total — with no number-range guard. [4](#0-3) 

In `reply_proof`, the MMR is constructed as `snapshot.chain_root_mmr(last_block.number() - 1)`. With `last_block` = block 1, this calls `chain_root_mmr(0)`. [5](#0-4) 

`chain_root_mmr(n)` calls `leaf_index_to_mmr_size(n)` to set the MMR size, so `chain_root_mmr(0)` produces an MMR of size 1 containing only the genesis leaf. [6](#0-5) 

Calling `mmr.gen_proof` with positions for leaves 2–1001 on an MMR of size 1 returns `Err`, and the function returns `StatusCode::InternalError` (500). [7](#0-6) 

Back in `received`, `status.should_ban()` checks `(400..500).contains(&code)`. Since `InternalError = 500` is not in `[400, 500)`, no ban is issued. `should_warn()` fires for `500..600`, so only a warning is logged. There is no per-peer rate limiter anywhere in the `received` handler. [8](#0-7) [9](#0-8) 

## Impact Explanation
Each malicious request forces ~3000 synchronous database reads and an MMR proof attempt before failing with no cost to the attacker. With no ban and no rate limit, a single persistent connection can saturate the node's I/O and CPU, degrading or crashing a CKB node running the light-client protocol server. This matches the allowed impact: **High (10001–15000 points) — Vulnerabilities which could easily crash a CKB node**.

## Likelihood Explanation
The attack requires only a valid P2P connection to a node running the light-client protocol server. No keys, no PoW, no privileged access. The attacker needs only two publicly available main-chain block hashes at different heights. The attack is trivially scriptable and repeatable without any cooldown.

## Recommendation
1. **Add a number-range check in `execute`**: after resolving each found block's header, reject with `StatusCode::MalformedProtocolMessage` (4xx) any block whose `header.number() > last_block.number()`. This converts the attacker-controlled error path from 5xx to 4xx, triggering the existing ban logic.
2. **Add per-peer rate limiting** to `LightClientProtocol::received`, analogous to rate limiters present in other protocol handlers, to bound the request rate per peer regardless of response code.

## Proof of Concept
```
// Precondition: chain has ≥ 1001 blocks on main chain.
// block_1_hash  = hash of block at height 1
// block_N_hash  = hash of block at height N, for N in 2..=1001

let msg = GetBlocksProof {
    last_hash:    block_1_hash,
    block_hashes: [block_2_hash, block_3_hash, ..., block_1001_hash],  // exactly 1000 entries
};

// Server path:
// 1. block_hashes.len() == 1000 == GET_BLOCKS_PROOF_LIMIT  → passes (check is strictly >)
// 2. is_main_chain(block_1_hash)                           → true
// 3. all block_hashes pass is_main_chain                   → all 1000 in `found`
// 4. 3000 DB reads (get_block_header + get_block_uncles + get_block_extension × 1000)
// 5. reply_proof: mmr = chain_root_mmr(0), size=1
// 6. gen_proof([pos(2)..pos(1001)])                        → Err → InternalError(500)
// 7. should_ban(): 500 ∉ 400..500                         → None  ← no ban
// Repeat indefinitely with no penalty.
```

    async fn received(
        &mut self,
        nc: Arc<dyn CKBProtocolContext + Sync>,
        peer: PeerIndex,
        data: Bytes,
    ) {
        trace!("LightClient.received peer={}", peer);

        let msg = match packed::LightClientMessageReader::from_slice(&data) {
            Ok(msg) => msg.to_enum(),
            _ => {
                warn!(
                    "LightClient.received a malformed message from Peer({})",
                    peer
                );
                nc.ban_peer(
                    peer,
                    constant::BAD_MESSAGE_BAN_TIME,
                    String::from("send us a malformed message"),
                );
                return;
            }
        };

        let item_name = msg.item_name();
        let status = self.try_process(&nc, peer, msg).await;
        if let Some(ban_time) = status.should_ban() {
            error!(
                "process {} from {}; ban {:?} since result is {}",
                item_name, peer, ban_time, status
            );
            nc.ban_peer(peer, ban_time, status.to_string());
        } else if status.should_warn() {
            warn!("process {} from {}; result is {}", item_name, peer, status);
        } else if !status.is_ok() {
            debug!("process {} from {}; result is {}", item_name, peer, status);
        }
    }
