### Title
NEAR Receiver Ignores `PriceAttestation.status`, Storing Arbitrary Non-Trading Prices as Valid — (`target_chains/near/receiver/src/state.rs`)

---

### Summary

The NEAR receiver's `From<&PriceAttestation> for PriceFeed` conversion unconditionally uses the attestation's current `price` and `publish_time` fields, completely ignoring the `status` field. When a price attestation carries a non-trading status (`Unknown`, `Halted`, or `Auction`), the correct behavior — implemented in every other Pyth chain (CosmWasm, Sui, Aptos, Fuel) — is to fall back to `prev_price` / `prev_publish_time` (the last known valid trading price). The NEAR receiver instead stores the current, potentially arbitrary price value with a fresh `publish_time`, making it indistinguishable from a valid trading price to any consumer.

---

### Finding Description

In `target_chains/near/receiver/src/state.rs`, the `From<&PriceAttestation> for PriceFeed` implementation is:

```rust
impl From<&PriceAttestation> for PriceFeed {
    fn from(price_attestation: &PriceAttestation) -> Self {
        Self {
            id: PriceIdentifier(price_attestation.price_id.to_bytes()),
            price: Price {
                price: price_attestation.price.into(),       // ← always current price
                conf: price_attestation.conf.into(),
                expo: price_attestation.expo,
                publish_time: price_attestation.publish_time, // ← always current timestamp
            },
            ema_price: Price {
                price: price_attestation.ema_price.into(),
                conf: price_attestation.ema_conf.into(),
                expo: price_attestation.expo,
                publish_time: price_attestation.publish_time, // ← always current timestamp
            },
        }
    }
}
``` [1](#0-0) 

The `status` field of `PriceAttestation` is never read. Pyth's own documentation explicitly states: *"Only prices with a value of `status=trading` should be used. If the status is not `trading` but is Unknown, Halted or Auction the Pyth price can be an arbitrary value."* [2](#0-1)

```rust

---

### Impact Explanation

When a price attestation with `status = Unknown | Halted | Auction` is submitted to the NEAR receiver (a routine occurrence for equity markets outside trading hours, or any halted asset):

1. The NEAR receiver stores the **arbitrary current price** with a **fresh `publish_time`**.
2. Any consumer calling `get_price_no_older_than` receives this arbitrary price — it passes the age check because `publish_time` is recent.
3. There is no separate validity flag; the only staleness check is `publish_time` age. [7](#0-6) 

Consumers (e.g., DeFi protocols on NEAR) that rely on `get_price` or `get_price_no_older_than` will receive an arbitrary, potentially manipulable price value without any indication that the underlying market is not trading. This can lead to incorrect liquidations, mispriced collateral, or other financial harm.

---

### Likelihood Explanation

- The legacy batch attestation path in the NEAR receiver is still active and reachable by any unprivileged transaction sender.
- Non-trading price statuses are a normal, recurring event (equity market hours, circuit breakers, auctions).
- A VAA containing a non-trading attestation is publicly available from Hermes and can be submitted by anyone.
- No privileged access, key compromise, or Sybil attack is required. [8](#0-7) 

---

### Recommendation

Apply the same status-aware fallback logic used by CosmWasm, Sui, Aptos, and Fuel. In `target_chains/near/receiver/src/state.rs`, the `From<&PriceAttestation> for PriceFeed` implementation should check `price_attestation.status` and use `prev_price` / `prev_publish_time` when the status is not `Trading`:

```rust
impl From<&PriceAttestation> for PriceFeed {
    fn from(price_attestation: &PriceAttestation) -> Self {
        let (price_val, conf_val, publish_time) =
            if price_attestation.status == PriceStatus::Trading {
                (price_attestation.price, price_attestation.conf, price_attestation.publish_time)
            } else {
                (price_attestation.prev_price, price_attestation.prev_conf, price_attestation.prev_publish_time)
            };
        Self {
            id: PriceIdentifier(price_attestation.price_id.to_bytes()),
            price: Price { price: price_val.into(), conf: conf_val.into(),
                           expo: price_attestation.expo, publish_time },
            ema_price: Price { price: price_attestation.ema_price.into(),
                               conf: price_attestation.ema_conf.into(),
                               expo: price_attestation.expo, publish_time },
        }
    }
}
```

### Proof of Concept

1. A market (e.g., a US equity) enters a non-trading state on Pythnet (`status = Halted`). The attestation carries an arbitrary `price` value and a fresh `publish_time`, with the last valid price in `prev_price` / `prev_publish_time`.
2. An unprivileged actor fetches the corresponding Wormhole VAA from Hermes and calls `update_price_feeds` on the NEAR receiver.
3. The NEAR receiver calls `PriceFeed::from(&price_attestation)`, which stores the arbitrary `price` with the fresh `publish_time`.
4. A NEAR DeFi protocol calls `get_price_no_older_than(price_id, 60)`. The age check passes (publish_time is recent). The protocol receives the arbitrary price and acts on it — e.g., allowing a position to be opened or liquidated at a false price.
5. The CosmWasm, Sui, Aptos, and Fuel receivers would have stored `prev_price` / `prev_publish_time` instead, correctly reflecting the last known valid trading price. [1](#0-0) [7](#0-6)

    pub fn update_price_feeds(&mut self, data: String) -> Result<(), Error> {
        // Attempt to deserialize the Payload based on header.
        let bytes = &*hex::decode(data.clone()).map_err(|_| Error::InvalidHex)?;
        let cursor = &mut Cursor::new(bytes);
        let mut header = [0u8; 4];
        cursor.clone().read_exact(&mut header).unwrap();

        // Handle Accumulator style Price Updates.
        if &header == PYTHNET_ACCUMULATOR_UPDATE_MAGIC {
            let update_data =
                AccumulatorUpdateData::try_from_slice(&cursor.clone().into_inner()).unwrap();

            match update_data.proof {
                Proof::WormholeMerkle { vaa, .. } => {
                    self.verify_encoded_vaa_source(vaa.as_ref())?;
                    let vaa_hex = hex::encode(vaa.as_ref());
                    ext_wormhole::ext(self.wormhole.clone())
                        .with_static_gas(Gas::from_gas(30_000_000_000_000))
                        .verify_vaa(vaa_hex.clone())
                        .then(
                            Self::ext(env::current_account_id())
                                .with_static_gas(Gas::from_gas(30_000_000_000_000))
                                .with_attached_deposit(env::attached_deposit())
                                .verify_wormhole_merkle_callback(
                                    env::predecessor_account_id(),
                                    data,
                                ),
                        )
                        .then(
                            Self::ext(env::current_account_id())
                                .with_static_gas(Gas::from_gas(30_000_000_000_000))
                                .refund_vaa(env::predecessor_account_id(), env::attached_deposit()),
                        );
                }
            };
        } else {
            // We Verify the VAA is coming from a trusted source chain before attempting to verify
            // VAA signatures. Avoids a cross-contract call early.
            self.verify_encoded_vaa_source(bytes)?;

            // Verify VAA and refund the caller in case of failure.
            ext_wormhole::ext(self.wormhole.clone())
                .with_static_gas(Gas::from_gas(30_000_000_000_000))
                .verify_vaa(data.clone())
                .then(
                    Self::ext(env::current_account_id())
                        .with_static_gas(Gas::from_gas(30_000_000_000_000))
                        .with_attached_deposit(env::attached_deposit())
                        .verify_wormhole_batch_callback(env::predecessor_account_id(), data),
                )
                .then(

                        .with_static_gas(Gas::from_gas(30_000_000_000_000))
                        .refund_vaa(env::predecessor_account_id(), env::attached_deposit()),
                );

    pub fn get_price_no_older_than(
        &self,
        price_id: PriceIdentifier,
        age: Seconds,
    ) -> Option<Price> {
        self.prices.get(&price_id).and_then(|feed| {
            let block_timestamp = env::block_timestamp() / 1_000_000_000;
            let price_timestamp = feed.price.publish_time;

            // - If Price older than STALENESS_THRESHOLD, set status to Unknown.
            // - If Price newer than now by more than STALENESS_THRESHOLD, set status to Unknown.
            // - Any other price around the current time is considered valid.
            if u64::abs_diff(block_timestamp, price_timestamp.try_into().unwrap()) > age {
                return None;
            }

            Some(feed.price)
        })
