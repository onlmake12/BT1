### Title
Insufficient `callbackGasLimit` Validation Causes Silent Callback Failure and Loss of Fees - (`File: target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

`Echo.requestPriceUpdatesWithCallback` accepts a user-controlled `callbackGasLimit` of any value including `0`, with no minimum enforcement. When `executeCallback` is later called, it forwards exactly `req.callbackGasLimit` gas to the consumer's `_echoCallback`. If the value is too small, the callback silently fails — but the request has already been cleared and fees already credited to the provider, leaving the user with no recourse.

---

### Finding Description

In `Echo.sol`, `requestPriceUpdatesWithCallback` stores the user-supplied `callbackGasLimit` directly without any minimum check:

```solidity
function requestPriceUpdatesWithCallback(
    address provider,
    uint64 publishTime,
    bytes32[] calldata priceIds,
    uint32 callbackGasLimit          // ← no minimum validation
) external payable override returns (uint64 requestSequenceNumber) {
    ...
    uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
    if (msg.value < requiredFee) revert InsufficientFee();
    ...
    req.callbackGasLimit = callbackGasLimit;   // stored as-is
    ...
}
``` [1](#0-0) 

In `executeCallback`, the request is **cleared and fees credited to the provider before the callback is attempted**:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += ...;
clearRequest(sequenceNumber);   // ← request gone before callback
...
try
    IEchoConsumer(req.requester)._echoCallback{
        gas: req.callbackGasLimit   // ← forwards user-supplied gas
    }(sequenceNumber, priceFeeds)
{ ... } catch {
    emit PriceUpdateCallbackFailed(..., "low-level error (possibly out of gas)");
}
``` [2](#0-1) 

With `callbackGasLimit = 0` (or any trivially small value), the `try` call forwards 0 gas, immediately fails, the `catch` block emits `PriceUpdateCallbackFailed`, and execution continues. The user's fee is consumed, the request is permanently cleared, and `echoCallback` never runs.

Compare this to the Entropy contract, which explicitly enforces a minimum gas floor equal to the provider's `defaultGasLimit`:

```solidity
req.gasLimit10k = roundTo10kGas(
    callbackGasLimit < providerInfo.defaultGasLimit
        ? providerInfo.defaultGasLimit
        : callbackGasLimit
);
``` [3](#0-2) 

Echo has no equivalent protection.

The `getFee` function scales the gas component as `callbackGasLimit * providerFeeInWei`, so with `callbackGasLimit = 0` the user pays only the base fees (`pythFeeInWei + providerBaseFee + providerFeedFee`), which are still non-zero. [4](#0-3) 

---

### Impact Explanation

- The user pays `pythFeeInWei + providerBaseFee + providerFeedFee` (all non-zero in a live deployment) and receives nothing in return.
- The request is permanently cleared with no retry mechanism — there is no recovery path for the user.
- The consumer contract's `echoCallback` never executes, breaking any application logic that depends on receiving price updates (e.g., liquidations, settlements, oracle-gated state transitions).
- Provider fees are credited regardless of callback success, so the provider has no incentive to reject or flag the bad request.

---

### Likelihood Explanation

- Any unprivileged user can call `requestPriceUpdatesWithCallback` directly with `callbackGasLimit = 0` or `1`.
- A misconfigured or malicious consumer contract could also pass an insufficient gas limit, causing its own callback to silently fail every time.
- The fee paid is low (no gas component), making this cheap to trigger repeatedly, potentially as a griefing vector against a consumer contract that relies on Echo for critical updates.

---

### Recommendation

Enforce a minimum `callbackGasLimit` in `requestPriceUpdatesWithCallback`, analogous to the Entropy contract's `defaultGasLimit` floor. For example:

```solidity
uint32 minGasLimit = _state.providers[provider].minCallbackGasLimit;
require(callbackGasLimit >= minGasLimit, "callbackGasLimit below minimum");
```

Alternatively, revert in `executeCallback` if `gasleft()` is insufficient to forward `req.callbackGasLimit` to the callback, preventing the request from being cleared when the gas is provably insufficient.

---

### Proof of Concept

```solidity
function testRequestWithZeroCallbackGas() public {
    bytes32[] memory priceIds = createPriceIds();
    uint64 publishTime = SafeCast.toUint64(block.timestamp);

    // callbackGasLimit = 0: user pays only base fees
    uint96 fee = echo.getFee(defaultProvider, 0, priceIds);
    vm.deal(address(consumer), fee);

    vm.prank(address(consumer));
    uint64 seq = echo.requestPriceUpdatesWithCallback{value: fee}(
        defaultProvider,
        publishTime,
        priceIds,
        0   // callbackGasLimit = 0
    );

    // Provider executes callback — request is cleared, fees credited
    // but _echoCallback receives 0 gas and silently fails
    PythStructs.PriceFeed[] memory feeds = createMockPriceFeeds(publishTime);
    mockParsePriceFeedUpdates(pyth, feeds);
    bytes[] memory updateData = createMockUpdateData(feeds);

    // Expect PriceUpdateCallbackFailed event, not PriceUpdateExecuted
    vm.expectEmit(true, false, false, false, address(echo));
    emit PriceUpdateCallbackFailed(seq, defaultProvider, priceIds, address(consumer), "low-level error (possibly out of gas)");

    echo.executeCallback(defaultProvider, seq, updateData, priceIds);

    // Request is gone — no retry possible
    vm.expectRevert();
    echo.getRequest(seq); // NoSuchRequest
}
```

    function requestPriceUpdatesWithCallback(
        address provider,
        uint64 publishTime,
        bytes32[] calldata priceIds,
        uint32 callbackGasLimit
    ) external payable override returns (uint64 requestSequenceNumber) {
        require(
            _state.providers[provider].isRegistered,
            "Provider not registered"
        );

        // FIXME: this comment is wrong. (we're not using tx.gasprice)
        // NOTE: The 60-second future limit on publishTime prevents a DoS vector where
        //      attackers could submit many low-fee requests for far-future updates when gas prices
        //      are low, forcing executors to fulfill them later when gas prices might be much higher.
        //      Since tx.gasprice is used to calculate fees, allowing far-future requests would make
        //      the fee estimation unreliable.
        require(publishTime <= block.timestamp + 60, "Too far in future");
        if (priceIds.length > MAX_PRICE_IDS) {
            revert TooManyPriceIds(priceIds.length, MAX_PRICE_IDS);
        }
        requestSequenceNumber = _state.currentSequenceNumber++;

        uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
        if (msg.value < requiredFee) revert InsufficientFee();

        Request storage req = allocRequest(requestSequenceNumber);
        req.sequenceNumber = requestSequenceNumber;
        req.publishTime = publishTime;
        req.callbackGasLimit = callbackGasLimit;

    function getFee(
        address provider,
        uint32 callbackGasLimit,
        bytes32[] calldata priceIds
    ) public view override returns (uint96 feeAmount) {
        uint96 baseFee = _state.pythFeeInWei; // Fixed fee to Pyth
        // Note: The provider needs to set its fees to include the fee charged by the Pyth contract.
        // Ideally, we would be able to automatically compute the pyth fees from the priceIds, but the
        // fee computation on IPyth assumes it has the full updated data.
        uint96 providerBaseFee = _state.providers[provider].baseFeeInWei;
        uint96 providerFeedFee = SafeCast.toUint96(
            priceIds.length * _state.providers[provider].feePerFeedInWei
        );
        uint96 providerFeeInWei = _state.providers[provider].feePerGasInWei; // Provider's per-gas rate
        uint256 gasFee = callbackGasLimit * providerFeeInWei; // Total provider fee based on gas
        feeAmount =
            baseFee +
            providerBaseFee +
            providerFeedFee +
            SafeCast.toUint96(gasFee); // Total fee user needs to pay
    }
