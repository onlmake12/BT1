### Title
Entropy Fee Not Scaled With Callback Gas Limit for Legacy Providers (`defaultGasLimit == 0`) — (File: `target_chains/ethereum/contracts/contracts/entropy/Entropy.sol`)

---

### Summary

In `Entropy.sol`, when a provider has `defaultGasLimit == 0` (the legacy "opted-out" configuration), the fee charged to a requesting user is always the flat `feeInWei` regardless of the `gasLimit` parameter supplied to `requestV2`. The callback is then executed with **no gas cap** in the legacy path. A malicious user can deploy a gas-intensive callback contract, pay only the minimum base fee, and force the provider's keeper to spend far more on gas than it receives in fees — draining the provider's operational balance.

---

### Finding Description

`getProviderFee` computes the fee a user must pay:

```solidity
uint32 roundedGasLimit = uint32(roundTo10kGas(gasLimit)) * TEN_THOUSAND;
if (
    provider.defaultGasLimit > 0 &&
    roundedGasLimit > provider.defaultGasLimit
) {
    uint128 additionalFee = ((roundedGasLimit - provider.defaultGasLimit) * provider.feeInWei)
        / provider.defaultGasLimit;
    return provider.feeInWei + additionalFee;
} else {
    return provider.feeInWei;   // ← always taken when defaultGasLimit == 0
}
``` [1](#0-0) 

The guard `provider.defaultGasLimit > 0` short-circuits the entire scaling branch. Any `gasLimit` value the caller passes is silently ignored for fee purposes.

In `requestHelper`, the stored gas limit is also zeroed out for these providers:

```solidity
if (providerInfo.defaultGasLimit == 0) {
    req.gasLimit10k = 0;   // ← disables gas-limited callback path
}
``` [2](#0-1) 

When `revealWithCallback` is later called, `req.gasLimit10k == 0` routes execution into the legacy `else` branch, which invokes the callback with **no gas limit**:

```solidity
} else {
    ...
    if (len != 0) {
        IEntropyConsumer(callAddress)._entropyCallback(
            sequenceNumber,
            provider,
            randomNumber
        );
    }
``` [3](#0-2) 

The callback therefore consumes as much gas as the attacker's contract demands, while the provider only received `feeInWei` — a fixed amount calibrated to a normal callback, not an adversarially expensive one.

The test suite explicitly confirms this behaviour: with `defaultGasLimit == 0`, every gas limit value (0, 10 000, 20 000, 100 000) produces the same provider fee of `1`:

```
assertGasLimitAndFee(0,      0, 1);
assertGasLimitAndFee(10000,  0, 1);
assertGasLimitAndFee(100000, 0, 1);
``` [4](#0-3) 

---

### Impact Explanation

A malicious user can repeatedly request entropy from a legacy provider, paying only `feeInWei` per request, while each callback burns an arbitrarily large amount of gas. The provider's keeper must pay the full gas cost of every `revealWithCallback` transaction. Over many requests the provider's ETH balance is drained faster than fees replenish it, making the provider economically unviable and degrading Entropy service availability for all users of that provider.

---

### Likelihood Explanation

Any provider that has not called `setDefaultGasLimit` retains `defaultGasLimit == 0` and is vulnerable. The attack requires no privileged access: any EOA can deploy a gas-burning callback contract and call `requestV2` on the public Entropy contract. The cost to the attacker is only `feeInWei` per request (typically a small amount of native token).

---

### Recommendation

- **Short term:** In `getProviderFee`, remove the `provider.defaultGasLimit > 0` guard so that gas-limit scaling applies even when `defaultGasLimit == 0`. Alternatively, cap the callback gas in the legacy `else` branch to a safe maximum (e.g., the provider's `feeInWei`-implied gas budget).
- **Long term:** Deprecate the legacy `gasLimit10k == 0` path entirely and require all providers to set a non-zero `defaultGasLimit` before accepting new requests.

---

### Proof of Concept

1. Deploy a provider with `defaultGasLimit == 0` (or target an existing legacy provider).
2. Deploy `MaliciousConsumer` whose `entropyCallback` loops until it consumes ~5 000 000 gas.
3. Call `entropy.requestV2{value: feeInWei}(legacyProvider, randomSeed, 0)` — fee is `feeInWei` regardless of callback cost.
4. Provider keeper calls `revealWithCallback`; the legacy path executes `_entropyCallback` with no gas cap.
5. Keeper transaction costs ~5 000 000 × `gasPrice` but only received `feeInWei`.
6. Repeat until provider's accrued fees and wallet balance are exhausted.
