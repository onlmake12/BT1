### Title
Unvalidated `providerToCredit` Address in `executeCallback` Allows Fee Theft — (`target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary

The `executeCallback` function in `Echo.sol` accepts an attacker-controlled `providerToCredit` address without verifying it is a registered provider. After the exclusivity period expires, any caller can pass an arbitrary address as `providerToCredit`, causing request fees to be credited to that address's `accruedFeesInWei` balance. A permissionlessly registered attacker can then drain those fees via `withdrawAsFeeManager`.

### Finding Description

`Echo.executeCallback` enforces the `providerToCredit` address only during the exclusivity window:

```solidity
if (block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds) {
    require(
        providerToCredit == req.provider,
        "Only assigned provider during exclusivity period"
    );
}
```

After that window, no validation occurs. The fee credit line runs unconditionally:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
``` [1](#0-0) [2](#0-1) 

Provider registration is fully permissionless — anyone can call `registerProvider` with zero fees:

```solidity
function registerProvider(
    uint96 baseFeeInWei,
    uint96 feePerFeedInWei,
    uint96 feePerGasInWei
) external override {
    ProviderInfo storage provider = _state.providers[msg.sender];
    require(!provider.isRegistered, "Provider already registered");
    ...
    provider.isRegistered = true;
``` [3](#0-2) 

`setFeeManager` only requires the caller to be a registered provider (no other gate): [4](#0-3) 

`withdrawAsFeeManager` has no registration check — it only verifies `accruedFeesInWei >= amount`: [5](#0-4) 

### Impact Explanation

An attacker can steal the provider fee from every request whose exclusivity period has elapsed. The stolen amount per request is `req.fee + msg.value - pythFee`, which equals the full provider portion of the user-paid fee. Across many requests this constitutes material theft of provider revenue held in the Echo contract. The legitimate provider receives nothing for fulfilling those requests.

### Likelihood Explanation

The exclusivity period is a configurable but finite window (default appears to be seconds-to-minutes scale based on the test suite). Any request that goes unfulfilled past that window is immediately exploitable. An attacker can monitor the chain for expired requests and front-run or simply call `executeCallback` themselves with valid Pyth update data (which is publicly available). Registration is free and permissionless, so there is no barrier to entry.

### Recommendation

Validate that `providerToCredit` is a registered provider before crediting fees:

```solidity
require(
    _state.providers[providerToCredit].isRegistered,
    "providerToCredit not a registered provider"
);
```

Add this check immediately after the exclusivity-period block in `executeCallback`, before the fee credit line.

### Proof of Concept

1. Attacker calls `registerProvider(0, 0, 0)` — registers with zero fees, costs only gas.
2. Attacker calls `setFeeManager(attackerAddress)` — sets self as own fee manager.
3. A legitimate user calls `requestPriceUpdatesWithCallback(legitimateProvider, ...)` and pays the full fee. `req.fee` is stored.
4. Attacker waits for `block.timestamp >= req.publishTime + exclusivityPeriodSeconds`.
5. Attacker obtains valid Pyth `updateData` for `priceIds` (publicly available from Pyth's Hermes API).
6. Attacker calls `executeCallback(attackerAddress, sequenceNumber, updateData, priceIds)`.
   - Exclusivity check passes (period elapsed).
   - `_state.providers[attackerAddress].accruedFeesInWei += req.fee + msg.value - pythFee`.
7. Attacker calls `withdrawAsFeeManager(attackerAddress, stolenAmount)`.
   - `msg.sender == _state.providers[attackerAddress].feeManager` ✓ (set in step 2).
   - `accruedFeesInWei >= stolenAmount` ✓.
   - ETH transferred to attacker.

The legitimate provider receives zero fees for that request.

    function withdrawAsFeeManager(
        address provider,
        uint128 amount
    ) external override {
        require(
            msg.sender == _state.providers[provider].feeManager,
            "Only fee manager"
        );
        require(
            _state.providers[provider].accruedFeesInWei >= amount,
            "Insufficient balance"
        );

        _state.providers[provider].accruedFeesInWei -= amount;

        (bool sent, ) = msg.sender.call{value: amount}("");
        require(sent, "Failed to send fees");

        emit FeesWithdrawn(msg.sender, amount);
    }
