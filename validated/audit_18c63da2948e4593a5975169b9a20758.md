### Title
Fee Deposited Unconditionally Before Freshness Check Enables Fee Loss Without Price Update — (`target_chains/sui/contracts/sources/pyth.move`)

### Summary

In `update_single_price_feed`, the fee is deposited into the `PriceInfoObject` at line 277 **before** the freshness check in `update_cache` at line 316. If `is_fresh_update` returns `false` (because a concurrent transaction already advanced the cached timestamp), the caller's fee is consumed with no oracle state change and no refund path.

### Finding Description

`update_single_price_feed` executes in this order:

1. **Line 274** — assert fee ≥ base update fee
2. **Line 277** — `price_info::deposit_fee_coins(price_info_object, fee)` — **unconditional, irrevocable**
3. **Lines 283–291** — find matching `PriceInfo` in the hot potato, call `update_cache`
4. **`update_cache` line 316** — `if (is_fresh_update(update, price_info_object))` — **conditional write** [1](#0-0) [2](#0-1) 

`is_fresh_update` requires strictly `update_timestamp > cached_timestamp`: [3](#0-2) 

`deposit_fee_coins` merges the coin into the `PriceInfoObject`'s dynamic field with no return value and no refund mechanism: [4](#0-3) 

There is no code path in `update_single_price_feed` that returns or burns the fee if the price write is skipped.

### Impact Explanation

On Sui, `PriceInfoObject` is a **shared object**. Two independent PTBs (each creating their own `HotPotatoVector` from a valid VAA and calling `update_single_price_feed`) can be submitted concurrently. Consensus serializes them. If PTB-B (with timestamp T+1) is ordered before PTB-A (with timestamp T):

- PTB-B executes: fee deposited, price written to T+1. ✓
- PTB-A executes: fee deposited at line 277 ✓, `is_fresh_update` returns `false` (T < T+1), price **not** written. Fee is permanently lost to the caller.

The caller of PTB-A loses their fee with zero oracle benefit. The deposited coins accumulate inside the `PriceInfoObject` and are not recoverable by the caller.

### Likelihood Explanation

- `PriceInfoObject` is a shared object; concurrent updates from independent relayers are the normal operating mode.
- Pyth price feeds update at sub-second to low-second cadence; timestamp collisions and ordering inversions are routine.
- No privileged role is required — any unprivileged updater with a valid VAA is affected.
- A targeted griefing attack is also feasible: an adversary monitors pending Sui transactions, obtains a fresher price attestation from the Pyth price service (publicly available), and submits their own PTB to be ordered first, guaranteeing the victim's fee is consumed without effect.

### Recommendation

Move the fee deposit **after** the freshness check, or abort (refund) if `is_fresh_update` returns `false`. Concretely, inside `update_cache`, return a boolean indicating whether the write occurred, and in `update_single_price_feed` only call `deposit_fee_coins` when that boolean is `true`. Alternatively, refund the fee coin to the caller when no write occurs.

### Proof of Concept

State-transition test (local, no network required):

1. Initialize Pyth state and create a `PriceInfoObject` with cached timestamp `T`.
2. **PTB-A**: call `create_price_infos_hot_potato` with a VAA containing timestamp `T+1`; call `update_single_price_feed` — price written to `T+1`, fee deposited. Record balance = `F`.
3. **PTB-B** (simulating the victim, submitted concurrently but ordered second): call `create_price_infos_hot_potato` with a VAA containing timestamp `T+1` (same or any `T' ≤ T+1`); call `update_single_price_feed`.
4. Assert: `price_info_object` balance = `2F` (fee deposited twice).
5. Assert: cached price timestamp is still `T+1` (unchanged — no second write occurred).

This directly demonstrates fee consumption with no oracle state change, matching the invariant violation described. [5](#0-4)

    fun is_fresh_update(update: &PriceInfo, price_info_object: &PriceInfoObject): bool {
        // Get the timestamp of the update's current price
        let price_feed = price_info::get_price_feed(update);
        let update_timestamp = price::get_timestamp(&price_feed::get_price(price_feed));

        // Get the timestamp of the cached data for the price identifier
        let cached_price_info = price_info::get_price_info_from_price_info_object(price_info_object);
        let cached_price_feed =  price_info::get_price_feed(&cached_price_info);
        let cached_timestamp = price::get_timestamp(&price_feed::get_price(cached_price_feed));

        update_timestamp > cached_timestamp
    }

    public fun deposit_fee_coins(price_info_object: &mut PriceInfoObject, fee_coins: Coin<SUI>) {
        if (!dynamic_object_field::exists_with_type<vector<u8>, Coin<SUI>>(&price_info_object.id, FEE_STORAGE_KEY)) {
            dynamic_object_field::add(&mut price_info_object.id, FEE_STORAGE_KEY, fee_coins);
        }
        else {
            let current_fee = dynamic_object_field::borrow_mut<vector<u8>, Coin<SUI>>(
                &mut price_info_object.id,
                FEE_STORAGE_KEY
            );
            coin::join(current_fee, fee_coins);
        };
    }
