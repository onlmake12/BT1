### Title
Orphan Pool Double-Spend Amplification: N-1 Wasted Full Verifications on Parent Arrival — (`tx-pool/src/component/orphan.rs`, `tx-pool/src/process.rs`)

### Summary

An unprivileged relay peer can fill the orphan pool with up to `DEFAULT_MAX_ORPHAN_TRANSACTIONS` (100) distinct transactions all spending the same unknown parent output. When the parent arrives, `process_orphan_tx` attempts `_process_tx` on every one of them sequentially. The first succeeds; the remaining 99 each proceed through full script execution before being rejected as conflicts. This is a concrete, bounded (100×) CPU amplification attack requiring only 101 P2P relay messages.

### Finding Description

**Step 1 — Orphan pool explicitly allows double-spends of the same unknown input.**

`add_orphan_tx` deduplicates only by `proposal_short_id` (tx hash). Multiple distinct transactions spending the same unknown out-point are all admitted and indexed under the same `by_out_point` key. [1](#0-0) 

This is confirmed by the production test: [2](#0-1) 

**Step 2 — The pool cap is 100 entries total, with no per-peer or per-out-point sub-limit.** [3](#0-2) 

There is no per-peer orphan quota anywhere in the codebase. A single attacker peer can occupy all 100 slots.

**Step 3 — `find_by_previous` returns ALL orphans referencing any output of the parent.** [4](#0-3) 

**Step 4 — `process_orphan_tx` calls `_process_tx` on every returned orphan without pre-filtering conflicts.** [5](#0-4) 

For each orphan with `cycle ≤ max_tx_verify_cycles`, `_process_tx` is called directly (line 626). For higher-cycle orphans, they are enqueued in the verify queue (line 606), which also consumes resources.

**Step 5 — The N-1 conflicting orphans each undergo full script execution before rejection.**

Inside `pre_check`, when `resolve_tx(allow_dead=false)` returns `Dead` (because the first orphan already consumed the parent output), the code falls into the RBF path: it calls `resolve_tx(allow_dead=true)`, finds a conflict in the pool, and returns `Ok` — allowing the call to proceed to `verify_rtx` (full CKB-VM script execution). [6](#0-5) 

Only after script execution does `submit_entry` detect the conflict and return `Err(Reject::Resolve(OutPointError::Dead(...)))` (RBF disabled) or `Err(Reject::RBFRejected(...))` (RBF enabled). [7](#0-6) 

### Impact Explanation

- **Attacker cost**: craft 100 syntactically valid but distinct transactions (different witnesses/outputs) spending the same unknown out-point, relay them as orphans, then relay the parent. Total: 101 P2P messages, negligible CPU.
- **Victim cost**: 100 full `_process_tx` invocations including CKB-VM script execution. 99 of these are entirely wasted.
- **Amplification factor**: bounded at 100× by `DEFAULT_MAX_ORPHAN_TRANSACTIONS`.
- **Repeatability**: the attacker can repeat this continuously with different parent out-points, sustaining elevated CPU load on the victim node.
- **Network effect**: if many peers are targeted simultaneously, or if the attacker connects from multiple IPs, the aggregate CPU drain can cause block-template generation delays and relay latency, contributing to network congestion.

### Likelihood Explanation

The attack path is fully reachable via standard P2P relay (`RelayTransaction` message). No privilege, key, or PoW is required. The orphan pool's explicit design choice to allow double-spends of unknown inputs (confirmed by the test) is the root cause. The 100-entry cap limits but does not eliminate the impact.

### Recommendation

1. **Per-out-point orphan limit**: enforce a maximum number of orphan transactions per unknown out-point (e.g., 3–5). Reject additional double-spending orphans for the same out-point.
2. **Conflict pre-filter in `process_orphan_tx`**: before calling `_process_tx` on each orphan, check whether a sibling orphan for the same out-point has already succeeded in this processing round, and skip the rest.
3. **Per-peer orphan quota**: limit the number of orphan slots a single peer can occupy.

### Proof of Concept

```
1. Attacker connects as relay peer.
2. For i in 1..=100:
     craft orphan_i = tx spending (parent_hash, 0) with unique output/witness
     relay orphan_i with declared_cycle = max_tx_verify_cycles
   → all 100 are admitted to orphan pool (by_out_point[(parent_hash,0)] = {id_1..id_100})
3. Relay parent tx.
4. Node calls process_orphan_tx(parent):
     find_by_previous(parent) → returns all 100 orphan IDs
     for each orphan_i:
       _process_tx(orphan_i) → pre_check → resolve_tx(dead) → RBF path → verify_rtx (FULL SCRIPT) → submit_entry → REJECTED
5. Result: 100 _process_tx calls, 99 full script executions wasted.
   Assert: _process_tx invocation count == 100, failure count == 99 with Dead/RBFRejected errors.
```

    pub fn add_orphan_tx(
        &mut self,
        tx: TransactionView,
        peer: PeerIndex,
        declared_cycle: Cycle,
    ) -> Vec<Byte32> {
        if self.entries.contains_key(&tx.proposal_short_id()) {
            return vec![];
        }

        debug!("add_orphan_tx {}", tx.hash());
        self.entries.insert(
            tx.proposal_short_id(),
            Entry::new(tx.clone(), peer, declared_cycle),
        );

        for out_point in tx.input_pts_iter() {
            self.by_out_point
                .entry(out_point)
                .or_default()
                .insert(tx.proposal_short_id());
        }

        // DoS prevention: do not allow OrphanPool to grow unbounded
        self.limit_size()

fn test_orphan_allows_double_spends_of_unknown_input() {

    let tx1 = build_tx(vec![(&parent_hash, 0)], 1);

    let txs = orphan.find_by_previous(&parent);
    assert_eq!(txs.len(), 2);
    assert!(txs.contains(&&tx1.proposal_short_id()));

    pub(crate) async fn process_orphan_tx(&self, tx: &TransactionView) {
        let mut orphan_queue: VecDeque<TransactionView> = VecDeque::new();
        orphan_queue.push_back(tx.clone());

        while let Some(previous) = orphan_queue.pop_front() {
            let orphans = self.find_orphan_by_previous(&previous).await;
            for orphan in orphans.into_iter() {
                if orphan.cycle > self.tx_pool_config.max_tx_verify_cycles {
                    debug!(
                        "process_orphan {} added to verify queue; find previous from {}",
                        orphan.tx.hash(),
                        tx.hash(),
                    );
                    let orphan_id = orphan.tx.proposal_short_id();
                    match self
                        .enqueue_verify_queue(
                            orphan.tx.clone(),
                            false,
                            Some((orphan.cycle, orphan.peer)),
                        )
                        .await
                    {
                        Ok(_) => {
                            self.remove_orphan_tx(&orphan_id).await;
                        }
                        Err(reject) => {
                            warn!(
                                "process_orphan {} failed to enqueue verify queue: {}; keep orphan from {}",
                                orphan.tx.hash(),
                                reject,
                                tx.hash(),
                            );
                        }
                    }
                } else if let Some((ret, _snapshot)) = self
                    ._process_tx(orphan.tx.clone(), Some(orphan.cycle), None)
                    .await
                {
                    match ret {
                        Ok(_) => {
                            self.send_result_to_relayer(TxVerificationResult::Ok {
                                original_peer: Some(orphan.peer),
                                tx_hash: orphan.tx.hash(),
                            });
                            debug!(
                                "process_orphan {} success, find previous from {}",
                                orphan.tx.hash(),
                                tx.hash()
                            );
                            self.remove_orphan_tx(&orphan.tx.proposal_short_id()).await;
                            orphan_queue.push_back(orphan.tx);
                        }
                        Err(reject) => {
                            debug!(
                                "process_orphan {} reject {}, find previous from {}",
                                orphan.tx.hash(),
                                reject,
                                tx.hash(),
                            );

                            if !is_missing_input(&reject) {
                                self.remove_orphan_tx(&orphan.tx.proposal_short_id()).await;
                                if reject.is_malformed_tx() {
                                    self.ban_malformed(orphan.peer, format!("reject {reject}"))
                                        .await;
                                }
                                if reject.is_allowed_relay() {
                                    self.send_result_to_relayer(TxVerificationResult::Reject {
                                        tx_hash: orphan.tx.hash(),
                                    });
                                }
                                if reject.should_recorded() {
                                    self.put_recent_reject(&orphan.tx.hash(), &reject).await;
                                }
                            }
                        }
                    }
                }
            }
        }
