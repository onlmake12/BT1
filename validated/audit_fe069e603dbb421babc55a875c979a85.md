### Title
Inbound Peer Can Inject Arbitrary Listen Port into Peer Store via `GetNodes.listen_port`, Propagating Poisoned Addresses Network-Wide — (`network/src/protocols/discovery/mod.rs`, `state.rs`)

### Summary
An unprivileged inbound peer can send a `GetNodes` message with an attacker-chosen `listen_port` value. The discovery protocol unconditionally calls `update_port` with that value, replacing the TCP port in the stored `RemoteAddress::Init` multiaddr, then immediately persists the resulting address into the peer store via `add_new_addr`. No port validation exists at any point. The poisoned address (real peer IP, attacker-chosen port) is then returned by `fetch_random_addrs` and relayed to other nodes in `Nodes` responses, propagating the bad address network-wide.

### Finding Description
In `SessionState::new`, an inbound session stores the peer's ephemeral TCP source address as `RemoteAddress::Init(context.session.address)`. When the peer subsequently sends `GetNodes { listen_port: Some(p) }`, the handler at `mod.rs:124-134` calls `state.remote_addr.update_port(p)`. `update_port` (state.rs:153-167) rewrites the `Protocol::Tcp(_)` component of the multiaddr to `Protocol::Tcp(p)` and promotes the state to `RemoteAddress::Listen`. The handler then immediately calls `self.addr_mgr.add_new_addr(session_id, (addr.clone(), flags))`. The `add_new_addrs` implementation filters only on `is_valid_addr`, which checks IP reachability but ignores the port entirely. The address is written to the peer store unconditionally.

### Impact Explanation
The peer store is poisoned with `(real_peer_ip, attacker_port)` entries. These are returned by `fetch_random_addrs` and broadcast to other peers in `Nodes` messages. Receiving nodes will attempt connections to the wrong port, wasting resources and degrading peer discovery quality. A single attacker with many inbound connections (or repeated reconnections) can systematically fill the peer store with non-functional addresses for their own IP, degrading the network's ability to discover and connect to legitimate peers. Impact is scoped to peer store poisoning and discovery degradation — Medium (2001–10000).

### Likelihood Explanation
Requires only an inbound TCP connection — no authentication, no stake, no PoW. The `received_get_nodes` guard prevents duplicate exploitation per session, but reconnection trivially bypasses it. Fully automatable.

### Recommendation
Validate `listen_port` before calling `update_port`:
- Reject port 0.
- Optionally, cross-check the claimed port against the peer's observed source port or a reasonable ephemeral range.
- Consider requiring the peer to prove it actually listens on the claimed port (e.g., via the identify protocol) before writing to the peer store, rather than trusting the self-reported `listen_port` field in `GetNodes`.

### Proof of Concept
1. Establish an inbound TCP connection to a CKB node.
2. Complete the p2p handshake and open the discovery protocol substream.
3. Send `DiscoveryMessage::GetNodes { listen_port: Some(0xDEAD), count: 1, version: 0, required_flags: 0 }`.
4. Observe that `update_port(0xDEAD)` is called, transitioning `remote_addr` to `RemoteAddress::Listen(<peer_ip>/tcp/57005)`.
5. Assert that `peer_store.add_addr` is called with `<peer_ip>/tcp/57005`.
6. Call `peer_store.fetch_random_addrs` and assert the poisoned address is returned.
7. Observe the address propagated to a third peer in a subsequent `Nodes` message.

    pub(crate) fn update_port(&mut self, port: u16) {

    fn get_random(&mut self, n: usize, flags: Flags) -> Vec<(Multiaddr, Flags)> {
        let fetch_random_addrs = self
            .network_state
            .with_peer_store_mut(|peer_store| peer_store.fetch_random_addrs(n, flags));
        let addrs = fetch_random_addrs
            .into_iter()
            .filter_map(|paddr| {
                if !self.is_valid_addr(&paddr.addr) {
                    return None;
                }
                let f = Flags::from_bits_truncate(paddr.flags);
                Some((paddr.addr, f))
            })
            .collect();
        trace!("Discovered random addrs: {:?}", addrs);
        addrs
    }
