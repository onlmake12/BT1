### Title
Unauthenticated `ConnectionSync` Triggers Outbound TCP Connections to Attacker-Controlled Addresses via `pending_delivered` — (`network/src/protocols/hole_punching/component/connection_sync.rs`)

### Summary
An unprivileged remote peer can cause a victim CKB node to initiate a 30-second outbound TCP connection retry loop to arbitrary attacker-controlled IP addresses by (1) pre-populating `pending_delivered` via a crafted `ConnectionRequest`, then (2) sending a crafted `ConnectionSync` referencing that entry.

### Finding Description

The hole-punching protocol's passive side (`ConnectionSyncProcess::execute`) uses `content.from` — a fully attacker-controlled field — as the key to look up stored listen addresses in `pending_delivered`, then unconditionally spawns TCP connection tasks to those addresses. There is no check that the sender of the `ConnectionSync` is the same peer as the `from` field, nor any check that the `from` peer was the legitimate initiator of the original handshake.

**Two-step exploit:**

**Step 1 — Pre-populate `pending_delivered` with attacker addresses:**

The attacker (already connected as a P2P peer) sends a `ConnectionRequest` message with:
- `from = attacker_generated_peer_id` (arbitrary, not verified against session)
- `to = victim_local_peer_id`
- `listen_addrs = [attacker_controlled_ip:port]`

`ConnectionRequestProcess::execute()` sees `self_peer_id == content.to` and calls `respond_delivered()`, which filters addresses to TCP/IPv4/IPv6 only (no private-range block) and inserts:

```
pending_delivered[attacker_generated_peer_id] = ([attacker_controlled_ip:port], now)
``` [6](#0-5) 

**Step 2 — Trigger outbound TCP connections:**

The attacker sends a `ConnectionSync` with:
- `from = attacker_generated_peer_id`
- `to = victim_local_peer_id`
- `route = []`

`ConnectionSyncProcess::execute()` matches `content.to == self_peer_id`, looks up `content.from` in `pending_delivered`, finds the attacker's addresses, and spawns a `runtime::spawn` task calling `select_ok(tasks)` where each task is a `try_nat_traversal` to the attacker's IP: [7](#0-6) 

`try_nat_traversal` loops for 30 seconds, issuing a TCP SYN every ~200ms to the target address: [8](#0-7) 

### Impact Explanation

The victim node makes repeated outbound TCP SYN connections to any IPv4/IPv6 address the attacker specifies. This enables:
- **SSRF against internal network targets**: the victim probes internal services (e.g., `169.254.169.254`, internal databases, admin panels) that are unreachable from the public internet but reachable from the victim host
- **Port scanning**: the attacker can determine whether a port is open on any host reachable from the victim, by observing whether `raw_session` is subsequently established
- **Connection hijacking**: if the attacker controls the target address, a successful `raw_session` is established with `RawSessionInfo::inbound(listen_addr)`, injecting a session into the victim's P2P stack under the attacker's control [9](#0-8) 

### Likelihood Explanation

The attack requires only a standard P2P connection to the victim — no special privileges, no PoW, no leaked keys. The two crafted messages (`ConnectionRequest` then `ConnectionSync`) are straightforward to construct. The `forward_rate_limiter` (1/sec per `(from, to, item_id)` tuple) is trivially bypassed by rotating `from` peer IDs. The `HOLE_PUNCHING_INTERVAL` (2-minute dedup) is similarly bypassed by using fresh peer IDs per attempt. [10](#0-9) 

### Recommendation

1. **Verify sender identity**: In `ConnectionSyncProcess::execute()`, check that the session's actual peer ID matches `content.from` before consulting `pending_delivered`. The session peer ID is available via `context.session` in the caller.
2. **Bind `pending_delivered` entries to the originating session**: Store the `PeerIndex` alongside the entry and reject `ConnectionSync` messages from sessions that don't match.
3. **Restrict listen address ranges**: In `respond_delivered()`, reject RFC-1918, link-local, and loopback addresses in `listen_addrs` to limit SSRF reach.

    pub(crate) async fn execute(self) -> Status {
        let content = match SyncContent::try_from(&self.message) {
            Ok(content) => content,
            Err(status) => return status,
        };

        if content.route.len() > MAX_HOPS as usize {
            return StatusCode::InvalidRoute.with_context("the route length is too long");
        }
        if self
            .protocol
            .forward_rate_limiter
            .check_key(&(content.from.clone(), content.to.clone(), self.msg_item_id))
            .is_err()
        {
            debug!(
                "from: {}, to {}, item_name: {}, rate limit is reached",
                content.from, content.to, "ConnectionSync",
            );
            return StatusCode::TooManyRequests.with_context("ConnectionSync");
        }

        match content.route.last() {
            Some(next_peer_id) => self.forward_sync(next_peer_id).await,
            None => {
                let self_peer_id = self.protocol.network_state.local_peer_id();
                if self_peer_id != &content.to {
                    // forward the message to the `to` peer
                    self.forward_sync(&content.to).await
                } else {
                    // Current node should be the `to` target.
                    if let Some(metrics) = ckb_metrics::handle() {
                        metrics.ckb_hole_punching_passive_count.inc();
                    }

                    let listens_info = self
                        .protocol
                        .pending_delivered
                        .get(&content.from)
                        .map(|info| info.0.clone());

                    match listens_info {
                        Some(listens) => {
                            let tasks = listens
                                .into_iter()
                                .map(|listen_addr| {
                                    Box::pin(try_nat_traversal(self.bind_addr, listen_addr))
                                })
                                .collect::<Vec<_>>();

                            if tasks.is_empty() {
                                return StatusCode::Ignore.with_context("no valid listen address");
                            }

                            debug!(
                                "current peer is the target peer {}, start NAT traversal",
                                content.to
                            );

                            match self
                                .protocol
                                .network_state
                                .config
                                .listen_addresses
                                .first()
                                .cloned()
                            {
                                Some(listen_addr) => {
                                    let control: ServiceAsyncControl = self.p2p_control.clone();
                                    runtime::spawn(async move {
                                        if let Ok(((stream, addr), _)) = select_ok(tasks).await {
                                            debug!("NAT traversal success, addr: {:?}", addr);
                                            if let Some(metrics) = ckb_metrics::handle() {
                                                metrics
                                                    .ckb_hole_punching_passive_success_count
                                                    .inc();
                                            }

                                            let _ignore = control
                                                .raw_session(
                                                    stream,
                                                    addr,
                                                    RawSessionInfo::inbound(listen_addr),
                                                )
                                                .await;
                                        }
                                    });
                                    Status::ok()
                                }
                                None => {
                                    StatusCode::Ignore.with_context("no listen address configured")
                                }
                            }
                        }
                        None => StatusCode::Ignore
                            .with_context("the from peer id is not in the pending list"),
                    }
                }
            }
        }

pub(crate) async fn try_nat_traversal(
    bind_addr: Option<SocketAddr>,
    addr: Multiaddr,
) -> Result<(TcpStream, Multiaddr), std::io::Error> {
    let net_addr = match multiaddr_to_socketaddr(&addr) {
        Some(addr) => addr,
        None => {
            debug!("Failed to convert multiaddr to socketaddr");
            return Err(std::io::ErrorKind::InvalidInput.into());
        }
    };

    // Use a fixed interval but add a small amount of randomness
    let base_retry_interval = Duration::from_millis(200);

    // total time
    let timeout_duration = Duration::from_secs(30);
    let start_time = Instant::now();
    let mut retry_count = 0u32;
    while start_time.elapsed() < timeout_duration {
        retry_count += 1;

        // Add a small amount of random jitter (±25ms) to avoid conflicts
        // caused by continuous precise synchronization
        let jitter = Duration::from_millis(rand::random::<u64>() % 50);
        let actual_interval = if rand::random::<bool>() {
            base_retry_interval + jitter
        } else {
            base_retry_interval.saturating_sub(jitter)
        };

        let socket = create_socket(bind_addr, net_addr)?;

        match runtime::timeout(
            std::time::Duration::from_millis(200),
            socket.connect(net_addr),
        )
        .await
        {
            Ok(Ok(stream)) => {
                // try get the stored error in the underlying socket
                // if the socket is not connected, it will return an error
                if let Err(err) = check_connection(&stream) {
                    debug!("Failed to connect to NAT(base check): {}", err);
                }
                return Ok((stream, addr));
            }
            Err(err) => {
                debug!("Failed to connect to NAT(timeout): {}", err);
            }
            Ok(Err(err)) => {
                if err.kind() == std::io::ErrorKind::AddrNotAvailable {
                    return Err(err);
                }
                debug!(
                    "Failed to connect to NAT(other error): {}, {}",
                    err.kind(),
                    err
                );
            }
        }
        runtime::delay_for(actual_interval).await;
    }

    debug!("Failed to connect to NAT after {} retries", retry_count);
    Err(std::io::ErrorKind::TimedOut.into())
}
