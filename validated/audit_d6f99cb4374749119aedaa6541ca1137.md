Audit Report

## Title
Unbounded HTTP request body buffering in miner notify server leads to OOM crash — (File: miner/src/client.rs)

## Summary
The `handle` function in `miner/src/client.rs` collects the full incoming HTTP request body into memory with no size limit via `BodyExt::collect(req).await?.aggregate()` at line 362. Any local user who can reach the notify listen address can exhaust process memory by streaming an arbitrarily large body, crashing the `ckb-miner` process.

## Finding Description
At line 362 of `miner/src/client.rs`, `handle()` calls `BodyExt::collect(req).await?.aggregate()` directly on the raw `hyper::body::Incoming` body. The import at line 14 brings in `BodyExt`, `Empty`, and `Full` from `http_body_util`, but notably omits `Limited` — no size cap is applied before or during collection. The `hyper::body::Incoming` type carries no application-level size restriction. The notify listener is spawned unconditionally when `config.listen` is `Some(addr)` (lines 206–224), a documented production feature with explicit operator instructions. Each accepted connection is dispatched into its own `tokio::spawn`ed task (lines 257–261), so multiple concurrent oversized requests compound memory pressure. No connection limit, byte-count guard, or frame-count limit exists anywhere in the path.

## Impact Explanation
An attacker streams an arbitrarily large chunked HTTP body to the notify port. The Tokio task servicing that connection allocates heap memory proportional to the body size before any application logic runs. Sufficient data triggers the Linux OOM killer or a Rust allocator panic, terminating the `ckb-miner` process. This matches the allowed CKB bounty impact: **"Any local command line crash" (Note, 0–500 points)**.

## Likelihood Explanation
The notify listen address defaults to loopback (`127.0.0.1`). Any unprivileged local user account on the same host can open a TCP connection to a loopback port without elevated privileges, no authentication, no token, and no special capability. If the operator binds to `0.0.0.0`, any network peer qualifies. The attack requires a single HTTP request, no prior knowledge beyond the port number, and is trivially repeatable.

## Recommendation
Wrap the incoming body with `http_body_util::Limited` before collecting:

```rust
use http_body_util::Limited;

const MAX_BODY: u64 = 4 * 1024 * 1024; // 4 MiB

async fn handle(
    client: Client,
    req: Request<hyper::body::Incoming>,
) -> Result<Response<Empty<Bytes>>, Error> {
    let limited = Limited::new(req, MAX_BODY);
    let body = BodyExt::collect(limited).await?.aggregate();
    // ...
}
```

`Limited` returns `Err` (mapped to `LengthLimitError`) once the threshold is exceeded, aborting the connection without buffering the full payload.

import socket

HOST, PORT = "127.0.0.1", <configured_notify_port>
