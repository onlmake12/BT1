### Title
PeerStore Permanent DoS via Crafted Address Distribution Bypassing `check_purge` Eviction — (`network/src/peer_store/peer_store_impl.rs`)

### Summary

A logic flaw in `PeerStore::check_purge` allows a remote attacker to permanently fill the peer store by advertising exactly 4 addresses per network group across enough groups to reach `ADDR_COUNT_LIMIT`. Once full, every subsequent `add_addr` call returns `EvictionFailed`, blocking all new peer discovery.

### Finding Description

**Constants:**

- `ADDR_COUNT_LIMIT = 16384`
- `ADDR_MAX_RETRIES = 3`
- `ADDR_MAX_FAILURES = 10` [1](#0-0) 

**Step 1 — Addresses with `last_connected_at_ms=0`, `attempts_count=0` are connectable.**

`is_connectable` evaluates three conditions:

```
tried_in_last_minute(now_ms)          → 0 >= now_ms-60000 → false
last_connected_at_ms==0 && count>=3   → true && 0>=3      → false
now_ms-0 > ADDR_TIMEOUT_MS && 0>=10  → true && false      → false
```

Returns `true`. The first pass of `check_purge` finds nothing to remove. [2](#0-1) 

**Step 2 — `check_purge` second pass is defeated by the `> 4` threshold.**

When the first pass removes nothing, the second pass groups all addresses by network segment, sorts by group size (descending), takes the top `len/2` groups, and only evicts from groups where `addrs.len() > 4`:

```rust
peers.into_iter()
    .take(len / 2)
    .flat_map(move |addrs| {
        if addrs.len() > 4 {   // ← strictly greater than 4
            Some(...)
        } else {
            None               // ← groups of exactly 4 are skipped
        }
    })
```

With 4096 groups × 4 addresses each: every group has exactly 4 entries, `4 > 4` is `false` for all, `candidate_peers` is empty, and the function returns `Err(PeerStoreError::EvictionFailed)`. [3](#0-2) 

**Step 3 — `add_addr` always stores discovered addresses with `last_connected_at_ms=0`.** [4](#0-3) 

**Step 4 — Discovery protocol entry point has no per-peer total address limit.**

`DiscoveryAddressManager::add_new_addrs` iterates every address in a received `Nodes` message and calls `peer_store.add_addr` directly, with no per-session or per-peer cumulative cap: [5](#0-4) 

Per-message limits are `MAX_ADDR_TO_SEND = 1000` nodes × `MAX_ADDRS = 3` addresses = 3000 addresses per message. Filling 16384 slots requires only ~6 such messages, achievable from a single connection. [6](#0-5) 

The `is_valid_addr` filter only requires globally routable IPs — the attacker does not need to control those IPs, only advertise them. [7](#0-6) 

### Impact Explanation

Once the store is saturated with the crafted distribution, every call to `add_addr` (from discovery or identify) returns `EvictionFailed`. The victim node can no longer learn about any new peers. Existing connections are unaffected, but after natural churn the node cannot replenish its peer set, leading to progressive network isolation. Deployed at scale against many nodes simultaneously, this collapses peer discovery network-wide.

### Likelihood Explanation

The attack requires only a single P2P connection and ~6 `Nodes` messages. No PoW, no key material, no privileged role. The attacker advertises globally routable IPs from 4096 distinct /16 subnets — fabricated addresses that pass `is_reachable` but need not be live. This is straightforward to automate.

### Recommendation

Fix the eviction logic in `check_purge` with two changes:

1. **Change `> 4` to `>= 1`** (or any positive threshold) so that any non-empty group is eligible for eviction when the store is full.
2. **Change `take(len / 2)` to `take(len)`** so all groups are considered, not just the largest half.
3. Alternatively, add a **per-session cumulative address admission cap** in `add_new_addrs` to bound how many addresses a single peer can inject per time window.

### Proof of Concept

```rust
// Fill store with 4096 groups × 4 addresses (4096 × 4 = 16384 = ADDR_COUNT_LIMIT)
let mut peer_store = PeerStore::default();
let mut count = 0usize;
'outer: for subnet in 0u16..=255 {
    for host in 1u8..=255 {
        for port in 1u16..=4 {
            if count >= ADDR_COUNT_LIMIT { break 'outer; }
            let addr: Multiaddr = format!(
                "/ip4/1.{}.{}.1/tcp/{}/p2p/{}",
                subnet, host, port, PeerId::random().to_base58()
            ).parse().unwrap();
            peer_store.add_addr(addr, Flags::COMPATIBILITY).unwrap();
            count += 1;
        }
    }
}
assert_eq!(peer_store.addr_manager().count(), ADDR_COUNT_LIMIT);

// Any new address now fails
let new_addr: Multiaddr = format!(
    "/ip4/200.0.0.1/tcp/9999/p2p/{}", PeerId::random().to_base58()
).parse().unwrap();
let result = peer_store.add_addr(new_addr, Flags::COMPATIBILITY);
assert!(result.is_err()); // EvictionFailed — store permanently blocked
```
