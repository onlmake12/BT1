### Title
Integer Overflow in `GetLastStateProofProcess` Bounds Check Enables Unbounded Server-Side Work via Light Client P2P Protocol — (`File: util/light-client-protocol-server/src/components/get_last_state_proof.rs`)

---

### Summary

An integer overflow in the `GetLastStateProofProcess::execute()` bounds check allows a remote, unprivileged peer to bypass the `GET_LAST_STATE_PROOF_LIMIT = 1000` guard. The attacker crafts a `GetLastStateProof` P2P message with a specific `last_n_blocks` value that causes the multiplication `(last_n_blocks as usize) * 2` to wrap to zero in Rust release mode, making the guard trivially pass. The server then performs O(n) database lookups and O(n log n) MMR root computations proportional to the full chain height, causing severe CPU and memory exhaustion per request.

---

### Finding Description

In `GetLastStateProofProcess::execute()`, the guard intended to cap the total work is:

```rust
if self.message.difficulties().len() + (last_n_blocks as usize) * 2
    > constant::GET_LAST_STATE_PROOF_LIMIT   // 1000
{
    return StatusCode::MalformedProtocolMessage.with_context("too many samples");
}
```

`last_n_blocks` is a `u64` read directly from the attacker-controlled message field `GetLastStateProof.last_n_blocks`. In Rust release builds, integer arithmetic wraps on overflow (two's complement). If the attacker sends `last_n_blocks = u64::MAX / 2 + 1 = 9223372036854775808`, then:

- `last_n_blocks as usize` = `9223372036854775808` (on 64-bit, `usize` = `u64`)
- `9223372036854775808 * 2` wraps to `0`
- `difficulties.len() + 0 = 0`, which is **not** `> 1000`

The guard passes. The server then proceeds to build `reorg_last_n_numbers`:

```rust
let reorg_last_n_numbers = if start_block_number == 0
    || snapshot.get_ancestor(&last_block_hash, start_block_number)
        .map(|header| header.hash() == start_block_hash)
        .unwrap_or(false)
{
    Vec::new()
} else {
    let min_block_number = start_block_number - min(start_block_number, last_n_blocks);
    (min_block_number..start_block_number).collect()
};
```

The attacker sets `start_block_hash` to a random hash that does not match the actual block at `start_block_number`, forcing the `else` branch. With `last_n_blocks` huge, `min(start_block_number, last_n_blocks) = start_block_number`, so `min_block_number = 0`, and `(0..start_block_number).collect()` produces a Vec with up to `last_block_number` entries (the full chain height).

These entries are then passed to `complete_headers`, which for each block number performs:
- `snapshot.get_ancestor(last_hash, number)` — DB traversal
- `snapshot.get_block(ancestor_header.hash())` — DB lookup
- `snapshot.chain_root_mmr(number - 1).get_root()` — MMR root computation (O(log n) per call)

Total work per request: **O(n log n)** where n is the chain height.

---

### Impact Explanation

Any peer on the CKB P2P network can send a single crafted `GetLastStateProof` message to force the light-client protocol server to perform O(n log n) database and MMR operations proportional to the full chain height. Repeated requests from one or more peers can saturate CPU and memory, causing the node to become unresponsive or crash. This is a liveness/availability impact matching the bounty scope ("liveness issues that cause validators/nodes to crash or be unavailable").

---

### Likelihood Explanation

The `LightClientProtocol` handler accepts `GetLastStateProof` messages from any connected peer without authentication or rate limiting beyond the broken guard. The only prerequisite is knowing a valid main-chain block hash (`last_hash`), which is public information. The overflow value (`u64::MAX / 2 + 1`) is trivially constructable. The attack is fully remote and requires no stake, key, or privileged access.

---

### Recommendation

Replace the unchecked multiplication with a saturating or checked variant, or validate `last_n_blocks` independently before the combined check:

```rust
// Option 1: validate last_n_blocks independently
if last_n_blocks as usize > constant::GET_LAST_STATE_PROOF_LIMIT / 2 {
    return StatusCode::MalformedProtocolMessage.with_context("too many samples");
}
if self.message.difficulties().len() + (last_n_blocks as usize) * 2
    > constant::GET_LAST_STATE_PROOF_LIMIT
{
    return StatusCode::MalformedProtocolMessage.with_context("too many samples");
}

// Option 2: use saturating arithmetic
if self.message.difficulties().len()
    .saturating_add((last_n_blocks as usize).saturating_mul(2))
    > constant::GET_LAST_STATE_PROOF_LIMIT
{
    return StatusCode::MalformedProtocolMessage.with_context("too many samples");
}
```

Additionally, consider enabling `overflow-checks = true` in the release profile in `Cargo.toml` to catch such issues at runtime.

    fn complete_headers(
        &self,
        positions: &mut Vec<u64>,
        last_hash: &packed::Byte32,
        numbers: &[BlockNumber],
    ) -> Result<Vec<packed::VerifiableHeader>, String> {
        let mut headers = Vec::new();

        for number in numbers {
            if let Some(ancestor_header) = self.snapshot.get_ancestor(last_hash, *number) {
                let position = leaf_index_to_pos(*number);
                positions.push(position);

                let ancestor_block = self
                    .snapshot
                    .get_block(&ancestor_header.hash())
                    .ok_or_else(|| {
                        format!(
                            "failed to find block for header#{} (hash: {:#x})",
                            number,
                            ancestor_header.hash()
                        )
                    })?;
                let uncles_hash = ancestor_block.calc_uncles_hash();
                let extension = ancestor_block.extension();

                let parent_chain_root = if *number == 0 {
                    Default::default()
                } else {
                    let mmr = self.snapshot.chain_root_mmr(*number - 1);
                    match mmr.get_root() {
                        Ok(root) => root,
                        Err(err) => {
                            let errmsg = format!(
                                "failed to generate a root for block#{number} since {err:?}"
                            );
                            return Err(errmsg);
                        }
                    }
                };

                let header = packed::VerifiableHeader::new_builder()
                    .header(ancestor_header.data())
                    .uncles_hash(uncles_hash)
                    .extension(Pack::pack(&extension))
                    .parent_chain_root(parent_chain_root)
                    .build();

                headers.push(header);
            } else {
                let errmsg = format!("failed to find ancestor header ({number})");
                return Err(errmsg);
            }
        }

        Ok(headers)
    }

                item_name, peer, ban_time, status
            );

        } else if status.should_warn() {
            warn!("process {} from {}; result is {}", item_name, peer, status);
        } else if !status.is_ok() {
            debug!("process {} from {}; result is {}", item_name, peer, status);
        }
    }

    /// Gets ancestor block header by a base block hash and number
    fn get_ancestor(&self, base: &packed::Byte32, number: BlockNumber) -> Option<HeaderView> {
        let header = self.get_block_header(base)?;
        if number > header.number() {
            None
        } else if number == header.number() {
            Some(header)
        } else if self.is_main_chain(base) {
            self.get_block_hash(number)
                .and_then(|hash| self.get_block_header(&hash))
        } else {
            self.get_ancestor(&header.parent_hash(), number)
        }
    }
