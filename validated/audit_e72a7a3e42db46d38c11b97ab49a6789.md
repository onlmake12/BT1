### Title
Unvalidated `providerToCredit` in `Echo.executeCallback` Allows Fee Theft from Legitimate Providers — (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

In `Echo.executeCallback`, the `providerToCredit` parameter is only validated against `req.provider` during the exclusivity window. Once that window expires, any caller may supply an arbitrary `providerToCredit` address and redirect the entire stored request fee to an attacker-controlled provider account, stealing funds that were meant for the legitimate provider.

---

### Finding Description

`Echo.requestPriceUpdatesWithCallback` stores the user's payment (minus the fixed Pyth protocol fee) in `req.fee`:

```solidity
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);
```

Later, `executeCallback` credits a provider with those funds:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
```

The only guard on `providerToCredit` is:

```solidity
if (block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds) {
    require(
        providerToCredit == req.provider,
        "Only assigned provider during exclusivity period"
    );
}
```

After the exclusivity period (default 15 seconds), **there is no check that `providerToCredit` equals `req.provider`**. Any address — including an attacker-controlled one — may be supplied. Because `registerProvider` is permissionless and accepts zero-value fees, an attacker can pre-register a provider with `baseFeeInWei = feePerFeedInWei = feePerGasInWei = 0`, set themselves as fee manager via `setFeeManager`, and then call `executeCallback` on any expired request, routing the victim provider's earned fee to their own account. They then drain it via `withdrawAsFeeManager`.

---

### Impact Explanation

Legitimate providers lose 100% of the fees they earned for a request whenever the exclusivity period lapses before they fulfill it. The attacker receives those fees with no legitimate service rendered. This is a direct, quantifiable financial loss to every provider in the Echo ecosystem whose callbacks are not fulfilled within the exclusivity window.

---

### Likelihood Explanation

The exclusivity period is 15 seconds by default. Network congestion, provider downtime, or deliberate front-running can easily cause a provider to miss this window. The attack requires no privileged access: `registerProvider`, `setFeeManager`, `executeCallback`, and `withdrawAsFeeManager` are all permissionlessly callable by any EOA. The attacker can monitor the mempool for pending requests and act immediately after the exclusivity period expires.

---

### Recommendation

After the exclusivity period, restrict `providerToCredit` to a whitelist of registered providers, or — more simply — always credit `req.provider` for the stored `req.fee` and only allow the surplus (`msg.value - pythFee`) to go to the caller who actually executed the callback. Alternatively, require `providerToCredit` to be the original `req.provider` unconditionally, and separately compensate the executor from a penalty pool.

---

### Proof of Concept

```
1. Attacker calls registerProvider(0, 0, 0)          // zero-fee provider
2. Attacker calls setFeeManager(attackerAddress)      // can now withdraw
3. Victim user calls requestPriceUpdatesWithCallback(legitimateProvider, ...)
   paying, e.g., 1 000 000 wei; req.fee = 1 000 000 - pythFeeInWei
4. Legitimate provider fails to call executeCallback within 15 seconds
5. Attacker calls executeCallback(attackerAddress, sequenceNumber, updateData, priceIds)
   - providerToCredit == attackerAddress passes (exclusivity expired)
   - _state.providers[attackerAddress].accruedFeesInWei += req.fee - pythFee
6. Attacker calls withdrawAsFeeManager(attackerAddress, stolenAmount)
   - Receives the fee that was owed to legitimateProvider
```

    function withdrawAsFeeManager(
        address provider,
        uint128 amount
    ) external override {
        require(
            msg.sender == _state.providers[provider].feeManager,
            "Only fee manager"
        );
        require(
            _state.providers[provider].accruedFeesInWei >= amount,
            "Insufficient balance"
        );

        _state.providers[provider].accruedFeesInWei -= amount;

        (bool sent, ) = msg.sender.call{value: amount}("");
        require(sent, "Failed to send fees");

        emit FeesWithdrawn(msg.sender, amount);
    }
