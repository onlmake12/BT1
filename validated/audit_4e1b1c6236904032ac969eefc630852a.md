### Title
Unauthenticated GetHeaders Bandwidth Amplification via Crafted Locator — (`sync/src/synchronizer/get_headers_process.rs`)

---

### Summary

An unprivileged remote peer can send a `GetHeaders` message with exactly `MAX_LOCATOR_SIZE` (101) locator entries — 100 all-zero hashes followed by the genesis hash — to force a non-IBD node to respond with up to 2,000 headers per request, with no ban, no rate limit, and no cost to the attacker beyond maintaining a TCP connection.

---

### Finding Description

**Step 1 — Locator size check passes**

The guard in `execute` uses a strict `>` comparison:

```rust
if locator_size > MAX_LOCATOR_SIZE {   // 101 > 101 == false → passes
```

`MAX_LOCATOR_SIZE = 101`, so a locator with exactly 101 entries is accepted. [1](#0-0) [2](#0-1) 

**Step 2 — `locate_latest_common_block` returns `Some(0)` without banning**

`locate_latest_common_block` first requires the last locator entry to equal the genesis hash:

```rust
let locator_hash = locator.last().expect("empty checked");
if locator_hash != &self.sync_shared.consensus().genesis_hash() {
    return None;   // would trigger GetHeadersMissCommonAncestors → ban
}
```

The attacker satisfies this by placing the real genesis hash at index 100. The function then iterates all 101 entries calling `snapshot.get_block_number(hash)` — the 100 zero hashes all return `None`, and the genesis hash returns `Some(0)`. The early-exit condition fires immediately:

```rust
if index == 0 || latest_common == Some(0) {
    return latest_common;   // returns Some(0)
}
``` [3](#0-2) 

**Step 3 — `get_locator_response(0, Byte32::zero())` returns up to 2,000 headers**

With `block_number = 0` and `hash_stop = Byte32::zero()`:

```rust
std::iter::successors(Some(start_number), ...)   // starts at block 1
    .take_while(|n| *n <= tip_number)
    .take(MAX_HEADERS_LEN)                        // cap = 2,000
    .filter_map(|n| self.snapshot.get_block_hash(n))
    .take_while(|h| h != hash_stop)              // zero never matches real hashes
    .filter_map(|h| self.sync_shared.store().get_block_header(&h))
    .collect()
```

On a synced mainnet node (tip >> 2000), this collects the full 2,000-header batch. [4](#0-3) [5](#0-4) 

**Step 4 — No ban, no rate limit**

`execute` returns `Status::ok()` on the success path. `post_sync_process` only bans on 4xx status codes:

```rust
pub fn should_ban(&self) -> Option<Duration> {
    if !(400..500).contains(&(self.code as u16)) {
        return None;   // OK(100) → no ban
    }
    ...
}
``` [6](#0-5) [7](#0-6) 

There is no per-peer rate limit, request counter, or cooldown applied to incoming `GetHeaders` messages anywhere in the synchronizer path. [8](#0-7) 

---

### Impact Explanation

| Metric | Value |
|---|---|
| Request size | ~3.3 KB (101 × 32-byte hashes + framing) |
| Response size | ~416 KB (2,000 headers × ~208 bytes each) |
| Byte amplification | ~126× |
| DB reads forced on victim | 101 (locator lookups) + up to 4,000 (hash + header per block) |
| Ban triggered | No |
| Rate limit | None |

An attacker with a single persistent connection can saturate the victim node's upload bandwidth and I/O by sending this message in a tight loop. The "crash the entire network" framing in the question is overstated — the impact is **per-node bandwidth and I/O exhaustion**, not a network-wide outage. However, a node under this attack will degrade its ability to serve legitimate peers and process new blocks.

---

### Likelihood Explanation

The attack requires only a standard P2P connection (no privileges, no PoW, no keys). The crafted locator is trivially constructable. The existing integration test `InvalidLocatorSize` confirms the node correctly bans for `locator_size > 101`, but the at-limit case (`== 101`) with a valid genesis tail is never tested or rate-limited. [9](#0-8) 

---

### Recommendation

1. **Rate-limit incoming `GetHeaders` per peer** — e.g., allow at most N requests per second per peer before returning `StatusCode::TooManyRequests` (which is already defined but unused here).
2. **Charge cost proportional to locator size** — if `locator_size == MAX_LOCATOR_SIZE` and the common ancestor resolves to genesis (block 0), treat this as a suspicious pattern and apply a cooldown or reduced response window.
3. **Change the size check to `>=`** — `locator_size >= MAX_LOCATOR_SIZE` would reject the maximally-crafted case, though this alone is insufficient since smaller locators can still anchor to genesis.

---

### Proof of Concept

```rust
// Build locator: 100 zero hashes + genesis hash
let genesis_hash = node.consensus().genesis_hash();
let mut hashes: Vec<Byte32> = (0..100).map(|_| Byte32::zero()).collect();
hashes.push(genesis_hash);
assert_eq!(hashes.len(), MAX_LOCATOR_SIZE); // exactly 101

let msg = SyncMessage::new_builder()
    .set(GetHeaders::new_builder()
        .block_locator_hashes(hashes)
        .hash_stop(Byte32::zero())
        .build())
    .build()
    .as_bytes();

// Send repeatedly; measure response size vs request size.
// Assert: response contains 2000 headers, no ban is issued,
// and the loop can run indefinitely.
loop {
    net.send(node, SupportProtocols::Sync, msg.clone());
    // victim sends back ~416 KB per iteration
}
```

        let active_chain = self.synchronizer.shared.active_chain();

        let block_locator_hashes = self
            .message
            .block_locator_hashes()
            .iter()
            .map(|x| x.to_entity())
            .collect::<Vec<Byte32>>();
        let hash_stop = self.message.hash_stop().to_entity();
        let locator_size = block_locator_hashes.len();
        if locator_size > MAX_LOCATOR_SIZE {
            return StatusCode::ProtocolMessageIsMalformed.with_context(format!(
                "Locator count({locator_size}) > MAX_LOCATOR_SIZE({MAX_LOCATOR_SIZE})"
            ));
        }

        if active_chain.is_initial_block_download() {
            info!(
                "Ignoring getheaders from peer={} because the node is in initial block download stage.",
                self.peer
            );
            self.send_in_ibd();
            let shared = self.synchronizer.shared();
            if let Some(flag) = shared.state().peers().get_flag(self.peer)
                && (flag.is_outbound || flag.is_whitelist || flag.is_protect)
            {
                shared.insert_peer_unknown_header_list(self.peer, block_locator_hashes);
            };
            return Status::ignored();
        }

        if let Some(block_number) =
            active_chain.locate_latest_common_block(&hash_stop, &block_locator_hashes[..])
        {
            debug!(
                "headers latest_common={} tip={} begin",
                block_number,
                active_chain.tip_header().number(),
            );

            self.synchronizer.peers().getheaders_received(self.peer);
            let headers: Vec<core::HeaderView> =
                active_chain.get_locator_response(block_number, &hash_stop);
            // response headers

            debug!("headers len={}", headers.len());

            let content = packed::SendHeaders::new_builder()
                .headers(headers.into_iter().map(|x| x.data()).collect::<Vec<_>>())
                .build();
            let message = packed::SyncMessage::new_builder().set(content).build();
            let nc = Arc::clone(self.nc);
            self.synchronizer
                .shared()
                .shared()
                .async_handle()
                .spawn(async move { async_send_message_to(&nc, self.peer, &message).await });
        } else {
            return StatusCode::GetHeadersMissCommonAncestors
                .with_context(format!("{block_locator_hashes:#x?}"));
        }
        Status::ok()
    }

    pub fn get_locator_response(
        &self,
        block_number: BlockNumber,
        hash_stop: &Byte32,
    ) -> Vec<core::HeaderView> {
        let tip_number = self.tip_header().number();
        let Some(start_number) = block_number.checked_add(1) else {
            return Vec::new();
        };
        std::iter::successors(Some(start_number), |number| number.checked_add(1))
            .take_while(|number| *number <= tip_number)
            .take(MAX_HEADERS_LEN)
            .filter_map(|block_number| self.snapshot.get_block_hash(block_number))
            .take_while(|block_hash| block_hash != hash_stop)
            .filter_map(|block_hash| self.sync_shared.store().get_block_header(&block_hash))
            .collect()
    }
