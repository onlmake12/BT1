Audit Report

## Title
Compact Block Path Skips `version_check()`, Enabling Consensus Split via Version=1 Block — (`sync/src/relayer/compact_block_process.rs`)

## Summary
`HeaderAcceptor::version_check()` is the sole enforcement point that rejects block headers with `version != 0` and marks them `BLOCK_INVALID`. This check is called in the sync-headers path but is entirely absent from the compact block (`RelayV3`) path. A miner who produces a single valid-PoW block with `header.version = 1` can cause nodes receiving it via compact block to fully accept it, while nodes receiving the same header via `SendHeaders` permanently mark it invalid — a deterministic, persistent consensus split.

## Finding Description
`contextual_check()` in `compact_block_process.rs` constructs a `HeaderVerifier` and calls `header_verifier.verify()`, which checks only PoW, block number, epoch, and timestamp — confirmed at [1](#0-0)  — with no version check anywhere in that function. After `header_verifier.verify()` passes, `contextual_check()` returns `Status::ok()` with no further validation. [2](#0-1) 

The caller then proceeds to call `shared.insert_valid_header()` and `relayer.accept_block()`. [3](#0-2) 

In contrast, `HeaderAcceptor::accept()` in the sync-headers path explicitly calls `version_check()` after `non_contextual_check()`, writing `BLOCK_INVALID` for any header with `version != 0`. [4](#0-3) 

The `version_check()` method itself: [5](#0-4) 

The chain service's block acceptance path runs `BlockVerifier::verify()`, which checks only proposals limit, bytes, cellbase, duplicates, and merkle root — no version check. [6](#0-5) 

`ContextualBlockVerifier::verify()` likewise has no version check, running only epoch, uncles, two-phase commit, DAO header, reward, extension, and transaction verifiers. [7](#0-6) 

Because the compact block path checks for `BLOCK_INVALID` status at entry, the outcome is order-of-receipt-dependent: whichever message arrives first determines whether the block is accepted or permanently rejected. [8](#0-7) 

## Impact Explanation
This is a **Critical** vulnerability matching the allowed impact: "Vulnerabilities which could easily cause consensus deviation." Two honest nodes receiving the same valid-PoW block with `version=1` via different P2P paths will permanently disagree on chain state. Nodes that accepted the block via compact block will build descendants on it; nodes that received the header via `SendHeaders` first will reject all those descendants as having an invalid parent. The split is persistent and not self-healing.

## Likelihood Explanation
Any permissionless miner can produce a single valid-PoW block with `header.version = 1`. No majority hashpower is required. The attacker controls which message type is sent to which peer, making the differential deterministic and locally reproducible. The exploit requires no victim mistakes and no external dependencies beyond standard P2P connectivity.

## Recommendation
Add a version check inside `contextual_check()` in `sync/src/relayer/compact_block_process.rs`, immediately after `header_verifier.verify()` succeeds:

```rust
if compact_block_header.version() != 0 {
    shared.shared().insert_block_status(block_hash.clone(), BlockStatus::BLOCK_INVALID);
    return StatusCode::CompactBlockHasInvalidHeader
        .with_context(format!("{block_hash} invalid version"));
}
```

Alternatively, move the version check into `HeaderVerifier::verify()` so it is enforced uniformly across all ingestion paths.

## Proof of Concept
1. Run two nodes A and B connected to each other.
2. Mine a block with `header.version = 1` and valid PoW on top of the current tip.
3. Send the block to node A via a `RelayV3` compact block message.
4. Send the same block header to node B via a `Sync` `SendHeaders` message.
5. Assert: node A has the block stored (`BLOCK_STORED`); node B has the hash in `BLOCK_INVALID` state.
6. Mine a subsequent block on top of the version=1 block and broadcast to both nodes. Node A accepts it; node B rejects it as having an invalid parent — confirming a persistent consensus split.

    pub fn verify(
        &'a self,
        resolved: &'a [Arc<ResolvedTransaction>],
        block: &'a BlockView,
    ) -> Result<(Cycle, Vec<Completed>), Error> {
        let parent_hash = block.data().header().raw().parent_hash();
        let header = block.header();
        let parent = self
            .context
            .store
            .get_block_header(&parent_hash)
            .ok_or_else(|| UnknownParentError {
                parent_hash: parent_hash.clone(),
            })?;

        let epoch_ext = if block.is_genesis() {
            self.context.consensus.genesis_epoch_ext().to_owned()
        } else {
            self.context
                .consensus
                .next_epoch_ext(&parent, &self.context.store.borrow_as_data_loader())
                .ok_or_else(|| UnknownParentError {
                    parent_hash: parent.hash(),
                })?
                .epoch()
        };

        if !self.switch.disable_epoch() {
            EpochVerifier::new(&epoch_ext, block).verify()?;
        }

        if !self.switch.disable_uncles() {
            let uncle_verifier_context = UncleVerifierContext::new(&self.context, &epoch_ext);
            UnclesVerifier::new(uncle_verifier_context, block).verify()?;
        }

        if !self.switch.disable_two_phase_commit() {
            TwoPhaseCommitVerifier::new(&self.context, block).verify()?;
        }

        if !self.switch.disable_daoheader() {
            DaoHeaderVerifier::new(&self.context, resolved, &parent, &block.header()).verify()?;
        }

        if !self.switch.disable_reward() {
            RewardVerifier::new(&self.context, resolved, &parent).verify()?;
        }

        if !self.switch.disable_extension() {
            BlockExtensionVerifier::new(&self.context, self.chain_root_mmr, &parent)
                .verify(block)?;
        }

        let ret = BlockTxsVerifier::new(
            self.context.clone(),
            header,
            self.handle,
            &self.txs_verify_cache,
            &parent,
        )
        .verify(resolved, self.switch.disable_script())?;
        Ok(ret)
    }
