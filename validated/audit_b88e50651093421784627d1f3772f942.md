Audit Report

## Title
Truncated 32-bit Genesis Hash in `identify_name()` Allows Cross-Chain Peers to Bypass Network Identity Filter — (File: `spec/src/consensus.rs`)

## Summary
`Consensus::identify_name()` constructs the P2P identity string using only the first 8 hex characters (4 bytes / 32 bits) of the 32-byte genesis hash. `Identify::verify()` performs a full string equality check against this truncated identifier, meaning an attacker who crafts a genesis block whose first 4 bytes match the target network's prefix will pass the identity gate and be admitted as a peer. This enables peer-slot exhaustion and sustained resource consumption against any reachable CKB node with no privileged access required.

## Finding Description
`Consensus::identify_name()` at `spec/src/consensus.rs` lines 964–968 produces the identity string:

```rust
pub fn identify_name(&self) -> String {
    let genesis_hash = format!("{:x}", Into::<H256>::into(&self.genesis_hash));
    format!("/{}/{}", self.id, &genesis_hash[..8])  // 8 of 64 hex chars = 32 bits
}
```

The `self.id` field is set directly from `ChainSpec.name` via `spec/src/lib.rs` line 573 (`.id(self.name.clone())`). The mainnet spec sets `name = "ckb"` (`resource/specs/mainnet.toml` line 1), and this field is freely configurable in any custom chain spec.

`Identify::verify()` at `network/src/protocols/identify/mod.rs` lines 541–551 performs a full string equality check against this truncated identifier:

```rust
fn verify(&self, data: &[u8]) -> Option<(Flags, String)> {
    let name = reader.name().as_utf8().ok()?.to_owned();
    if self.name != name {
        return None;  // triggers 5-minute ban and disconnect
    }
    ...
}
```

A `None` return triggers a 5-minute ban (`BAN_ON_NOT_SAME_NET = Duration::from_secs(5 * 60)`, line 25). A matching return admits the peer fully into the sync/relay pipeline.

**Exploit path:**
1. Create a custom chain spec with `name = "ckb"`.
2. Iterate over a mutable genesis field (e.g., `genesis.nonce`, a `u128`) and recompute the genesis hash until `&genesis_hash_hex[..8]` matches the mainnet prefix. Expected cost: ~2^32 Blake2b computations.
3. Start a CKB node on this custom chain and connect to a legitimate mainnet peer.
4. `Identify::verify()` passes because `"/ckb/<4-byte-prefix>" == "/ckb/<4-byte-prefix>"`.
5. The attacker node is admitted and can occupy peer slots and send syntactically valid but semantically invalid sync/relay messages (e.g., `GetHeaders`, `SendBlock`) that enter `HeadersProcess`/`BlockProcess` and consume CPU and memory before being rejected at the ancestry/validation layer.

No additional network-layer chain-identity gate exists beyond the identify protocol.

## Impact Explanation
**High — Vulnerabilities or bad designs which could cause CKB network congestion with few costs.**

An attacker running multiple nodes with colliding genesis hash prefixes can exhaust inbound peer slots on targeted nodes, preventing legitimate peers from connecting, and force targeted nodes to process sync/relay messages that consume CPU and memory before being rejected at the block-validation layer. At scale, this constitutes network-wide congestion with negligible cost to the attacker.

## Likelihood Explanation
- **No privileged access required** — any unprivileged network peer can attempt this.
- **Computational cost is low** — a 32-bit preimage search requires ~2^32 Blake2b hash computations on average. At ~100M hashes/second on a single CPU core, this completes in under a minute.
- **Chain ID is freely settable** — the attacker sets `name = "ckb"` in their custom `chainspec.toml`; no key material or insider knowledge is needed.
- **Repeatable** — the 5-minute ban applies per session; the attacker can reconnect with new sessions or rotate IPs, and can pre-compute a pool of colliding genesis specs.

## Recommendation
Replace the 8-character genesis hash slice with the full 64-character hex string in `identify_name()` at `spec/src/consensus.rs`:

```rust
pub fn identify_name(&self) -> String {
    let genesis_hash = format!("{:x}", Into::<H256>::into(&self.genesis_hash));
    format!("/{}/{}", self.id, &genesis_hash)  // full 256-bit hash
}
```

This is a non-breaking change for honest nodes (all derive the same full hash from the same genesis block) and raises the collision cost from ~2^32 to ~2^256.

## Proof of Concept
1. Clone the CKB repository and create a custom chain spec with `name = "ckb"`.
2. Write a script that iterates `genesis.nonce` and calls `ChainSpec::build_genesis()` + `consensus.genesis_hash()` until `&genesis_hash_hex[..8]` matches the mainnet prefix (obtainable from any public mainnet node).
3. Start a CKB node on this custom chain.
4. Connect to a mainnet peer. `Identify::verify()` passes because the identity strings are equal.
5. Confirm the peer is admitted (no ban triggered). Send `GetHeaders` messages; observe they enter `HeadersProcess` and consume CPU/memory before being rejected at the ancestry check, confirming resource consumption past the identity gate.
