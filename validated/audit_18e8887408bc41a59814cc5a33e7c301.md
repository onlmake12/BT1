### Title
Unauthenticated `from` Field in HolePunching `ConnectionRequest` Enables Peer Impersonation and `pending_delivered` Cache Poisoning — (`network/src/protocols/hole_punching/component/connection_request.rs`)

---

### Summary

The CKB HolePunching protocol processes a `ConnectionRequest` message that contains a self-declared `from: PeerId` field. Although the underlying P2P transport (SecIO) authenticates each session's peer identity cryptographically, the application-level handler **never validates that `content.from` matches the actual authenticated peer ID of the sending session**. Any connected peer can forge an arbitrary `from` value, impersonating any other node in the network. This enables peer impersonation, rate-limiter bypass, and `pending_delivered` cache poisoning — all reachable by any unprivileged peer that can open a connection to the node's P2P port.

---

### Finding Description

The `ConnectionRequest` molecule message carries three identity-bearing fields:

```
table ConnectionRequest {
    from: Bytes,   // claimed sender PeerId
    to:   Bytes,   // target PeerId
    ...
    listen_addrs: AddressVec,  // addresses the "from" peer claims to listen on
}
``` [1](#0-0) 

In `ConnectionRequestProcess::execute()`, the `from` field is parsed directly from the message payload into a `PeerId`:

```rust
let from = PeerId::from_bytes(value.from().raw_data().to_vec())...;
``` [2](#0-1) 

The handler then uses `content.from` for:
1. **Rate-limiting** — keyed on `(content.from, content.to, msg_item_id)` [3](#0-2) 
2. **Routing decisions** — `respond_delivered(content.from, ...)` when `self_peer_id == content.to` [4](#0-3) 
3. **`pending_delivered` insertion** — `self.protocol.pending_delivered.insert(from_peer_id, (remote_listens, now))` [5](#0-4) 

**At no point is `content.from` compared against the actual session peer ID** (which is cryptographically established by SecIO). The `HolePunching` struct holds `network_state: Arc<NetworkState>` which can resolve the real peer ID from the session, but this lookup is never performed. [6](#0-5) 

The `listen_addrs` in the message are only validated to ensure the embedded peer ID (if any) matches `content.from` — not the actual session peer:

```rust
if let Some(peer_id) = extract_peer_id(&addr) {
    if peer_id != from {   // "from" is the attacker-controlled claim, not the real session peer
        return Err(...);
    }
}
``` [7](#0-6) 

This means an attacker can craft `listen_addrs` containing their own IP, embed the victim's `PeerId` as `from`, and the check passes.

---

### Impact Explanation

**Attack 1 — Peer Impersonation / `pending_delivered` Poisoning:**

```
Attacker (connected to Target node T):
  Sends: ConnectionRequest {
    from: victim_V_peer_id,       // impersonation
    to:   T_peer_id,
    listen_addrs: [attacker_ip:port]  // malicious address
  }

Target T:
  - Stores pending_delivered[victim_V] = ([attacker_ip:port], now)
  - Sends ConnectionRequestDelivered back to attacker
  - On subsequent ConnectionSync, T dials attacker_ip:port
    believing it is connecting to victim V
```

This causes the target node to establish a connection to attacker-controlled infrastructure under the identity of a legitimate peer, enabling traffic interception or further protocol-level attacks.

**Attack 2 — Rate-Limiter Bypass:**

The rate limiter is keyed by `(content.from, content.to, msg_item_id)`. By rotating fake `from` values, an attacker bypasses the per-peer rate limit entirely and floods the target with `ConnectionRequest` processing work.

**Attack 3 — `pending_delivered` Cache Exhaustion / Slot Blocking:**

The `pending_delivered` map is checked with a `HOLE_PUNCHING_INTERVAL` guard. An attacker can pre-fill entries for all known peer IDs, blocking legitimate hole-punching requests from those peers for the duration of the interval. [8](#0-7) 

---

### Likelihood Explanation

**HIGH.** The HolePunching protocol is open to any peer that connects to the node's P2P port, which must be publicly reachable for the node to participate in the network. No special role, key material, or stake is required. The attacker only needs a single TCP connection. The forged `from` field requires no cryptographic material — it is a plain byte sequence parsed without signature verification. [9](#0-8) 

---

### Recommendation

After parsing `content.from`, resolve the **actual** peer ID of the sending session from `network_state` and reject the message if they do not match:

```rust
// Resolve the real peer ID for this session
let actual_peer_id = self.protocol.network_state
    .with_peer_registry(|reg| {
        reg.get_peer(self.peer)
            .and_then(|p| p.connected_addr.peer_id().cloned())
    });

match actual_peer_id {
    Some(real_id) if real_id == content.from => { /* proceed */ }
    _ => return StatusCode::InvalidFromPeerId
             .with_context("from field does not match session peer id"),
}
```

This mirrors the correct pattern already used in `SeedRecord::decode_with_pubkey`, which recovers a public key from a signature and then explicitly checks it against the expected key before trusting any payload field. [10](#0-9)

    pub(crate) async fn execute(mut self) -> Status {
        let content = match RequestContent::try_from(&self.message) {
            Ok(content) => content,
            Err(status) => return status,
        };
        if content.listen_addrs.len() > ADDRS_COUNT_LIMIT || content.listen_addrs.is_empty() {
            return StatusCode::InvalidListenAddrLen
                .with_context("the listen address count is too large or empty");
        }

        if content.max_hops > MAX_HOPS {
            return StatusCode::InvalidMaxTTL.into();
        }
        if content.route.len() > MAX_HOPS as usize {
            return StatusCode::InvalidRoute.with_context("the route length is too long");
        }

        let self_peer_id = self.protocol.network_state.local_peer_id();
        if content.route.contains(self_peer_id) {
            return StatusCode::Ignore.with_context("the message is passed, ignore it");
        }

        if self
            .protocol
            .forward_rate_limiter
            .check_key(&(content.from.clone(), content.to.clone(), self.msg_item_id))
            .is_err()
        {
            debug!(
                "from: {}, to {}, item_name: {}, rate limit is reached",
                content.from, content.to, "ConnectionRequest",
            );
            return StatusCode::TooManyRequests.with_context("ConnectionRequest");
        }

        if self_peer_id == &content.to {
            self.respond_delivered(content.from, &content.to, content.listen_addrs)
                .await
        } else if content.max_hops == 0u8 {
            StatusCode::ReachedMaxHops.into()
        } else {
            self.forward_message(self_peer_id, &content.to).await
        }
    }

    async fn respond_delivered(
        &mut self,
        from_peer_id: PeerId,
        to_peer_id: &PeerId,
        remote_listens: Vec<Multiaddr>,

        if let Some((_, t)) = self.protocol.pending_delivered.get(&from_peer_id) {
            let now = unix_time_as_millis();
            if now - t < HOLE_PUNCHING_INTERVAL {
                return StatusCode::Ignore
                    .with_context("a same message is already replied in a moment ago");
            }
        }
        let listen_addrs = {
            let public_addr = self.protocol.network_state.public_addrs(ADDRS_COUNT_LIMIT);
            if public_addr.len() < ADDRS_COUNT_LIMIT {
                let observed_addrs = self
                    .protocol
                    .network_state
                    .observed_addrs(ADDRS_COUNT_LIMIT - public_addr.len());
                let iter = public_addr
                    .iter()
                    .chain(observed_addrs.iter())
                    .map(Multiaddr::to_vec)
                    .map(|v| packed::Address::new_builder().bytes(v).build());
                packed::AddressVec::new_builder().extend(iter).build()
            } else {
                let iter = public_addr
                    .iter()
                    .map(Multiaddr::to_vec)
                    .map(|v| packed::Address::new_builder().bytes(v).build());
                packed::AddressVec::new_builder().extend(iter).build()

        };

            Ok(msg) => msg.to_enum(),

                warn!(
                    "HolePunching.received a malformed message from {}",
                    session_id

                self.network_state.ban_session(

                    session_id,
                    BAD_MESSAGE_BAN_TIME,
                    String::from("send us a malformed message"),

                return;
            }
        };

        let item_name = msg.item_name();

        if self
            .rate_limiter
            .check_key(&(session_id, msg.item_id()))
            .is_err()

                "process {} from {}; result is {}",
                item_name,
                session_id,

            return;
