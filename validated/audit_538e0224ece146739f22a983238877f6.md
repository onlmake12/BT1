### Title
No Replay Protection on Signed Lazer Price Updates Allows Stale Price Injection - (File: `lazer/contracts/evm/src/PythLazer.sol`)

---

### Summary

`PythLazer.sol`'s `verifyUpdate()` function verifies an ECDSA-signed Lazer price update but performs no replay protection. The same signed update bytes can be submitted an unlimited number of times and will be accepted as valid on each call. Any unprivileged actor who captures a past signed update from on-chain calldata can replay it to any integrator contract, causing that contract to receive a cryptographically valid but arbitrarily stale price.

---

### Finding Description

`PythLazer.sol` `verifyUpdate()` (lines 70–106) performs three checks:

1. Fee sufficiency (`msg.value >= verification_fee`)
2. Magic-byte and length validation
3. ECDSA signature recovery and trusted-signer lookup [1](#0-0) 

There is no fourth step: the contract never records which `(signature, payload)` pairs have already been processed. No hash set, no nonce, no sequence number, no monotonic counter. The test suite itself demonstrates this: the identical `update` bytes are submitted twice in `test_verify()` and both calls succeed without error. [2](#0-1) 

The same structural absence exists in the Aptos Lazer contract's `verify_message()`: [3](#0-2) 

And in the Sui Lazer contract's `parse_and_verify_le_ecdsa_update_v2()`: [4](#0-3) 

None of these functions record consumed update hashes, unlike Wormhole's `ConsumedVAAs` pattern used elsewhere in the same repository: [5](#0-4) 

---

### Impact Explanation

An integrator contract calls `pythLazer.verifyUpdate(update)` and trusts the returned `payload` as a freshly-verified price. Because `verifyUpdate()` accepts any previously-valid signed blob, an attacker can:

1. Observe a signed Lazer update from on-chain calldata (fully public).
2. Wait for the real price to move significantly (e.g., a large market move).
3. Replay the old signed update to the integrator's `verifyPrice` entry point.
4. The integrator receives a cryptographically valid payload whose embedded `timestampUs` is arbitrarily old.
5. If the integrator does not independently enforce timestamp freshness (which the Lazer contract does not require), it will act on the stale price — enabling liquidation manipulation, arbitrage against AMMs, or collateral mispricing.

The payload does contain a `timestampUs` field, but `verifyUpdate()` never checks it against `block.timestamp`. The contract's trust guarantee is therefore weaker than it appears: "this signature is from a trusted signer" does not imply "this price is current."

---

### Likelihood Explanation

- **Entry path is fully unprivileged.** `verifyUpdate()` is `external payable` with no access control. Any EOA or contract can call it.
- **Signed update bytes are public.** They appear as calldata in every historical transaction that called `verifyUpdate()` or any integrator that forwarded them.
- **Cost is only the verification fee.** The attacker pays `verification_fee` per replay call, which is negligible compared to DeFi manipulation profit.
- **No timing constraint.** The only expiry is the trusted signer's `expiresAt` timestamp (set far in the future, e.g., `3000000000000000` ms in tests). Old updates signed by a still-valid signer remain replayable indefinitely. [6](#0-5) 

---

### Recommendation

1. **Hash-based deduplication (preferred):** Maintain a `mapping(bytes32 => bool) public usedUpdates` keyed on `keccak256(update)`. Revert if the hash has been seen before. This mirrors the `ConsumedVAAs` pattern already used in the Wormhole/Pyth ecosystem.

2. **Timestamp freshness enforcement:** Inside `verifyUpdate()`, parse the `timestampUs` field from the payload and require it to be within an acceptable staleness window (e.g., `block.timestamp - MAX_AGE_SECONDS`). This is a lighter-weight alternative but does not prevent replay within the freshness window.

3. **Monotonic sequence number:** Embed a per-signer sequence number in the signed payload and require it to be strictly increasing, analogous to the recommendation in the referenced ChainlinkAtlasWrapper report.

interface IPythLazer {
    function verifyUpdate(bytes calldata update)
        external payable
        returns (bytes calldata payload, address signer);
    function verification_fee() external view returns (uint256);
}

contract ReplayAttack {
    IPythLazer public pythLazer;

    constructor(address _pythLazer) {
        pythLazer = IPythLazer(_pythLazer);
    }

    /// @notice Replay a previously-captured signed Lazer update.
    /// `oldUpdate` is any bytes calldata observed from a past on-chain call.
    /// The function succeeds and returns the stale payload as "verified".
    function replayOldUpdate(bytes calldata oldUpdate)
        external payable
        returns (bytes memory stalePricePayload)
    {
        uint256 fee = pythLazer.verification_fee();
        require(msg.value >= fee, "need fee");

        // First submission — accepted
        (bytes calldata p1, ) = pythLazer.verifyUpdate{value: fee}(oldUpdate);

        // Second submission of the IDENTICAL bytes — also accepted, no revert
        (bytes calldata p2, ) = pythLazer.verifyUpdate{value: fee}(oldUpdate);

        // p1 == p2: same stale payload returned both times
        // An integrator calling verifyUpdate with oldUpdate will receive
        // a cryptographically valid but arbitrarily old price.
        stalePricePayload = p2;
    }
}

    function verifyUpdate(
        bytes calldata update
    ) external payable returns (bytes calldata payload, address signer) {
        // Require fee and refund excess
        require(msg.value >= verification_fee, "Insufficient fee provided");
        if (msg.value > verification_fee) {
            payable(msg.sender).transfer(msg.value - verification_fee);
        }

        if (update.length < 71) {
            revert("input too short");
        }
        uint32 EVM_FORMAT_MAGIC = 706910618;

        uint32 evm_magic = uint32(bytes4(update[0:4]));
        if (evm_magic != EVM_FORMAT_MAGIC) {
            revert("invalid evm magic");
        }
        uint16 payload_len = uint16(bytes2(update[69:71]));
        if (update.length < 71 + payload_len) {
            revert("input too short");
        }
        payload = update[71:71 + payload_len];
        bytes32 hash = keccak256(payload);
        (signer, , ) = ECDSA.tryRecover(
            hash,
            uint8(update[68]) + 27,
            bytes32(update[4:36]),
            bytes32(update[36:68])
        );
        if (signer == address(0)) {
            revert("invalid signature");
        }
        if (!isValidSigner(signer)) {
            revert("invalid signer");
        }
    }

public fun parse_and_verify_le_ecdsa_update_v2(s: &State, clock: &Clock, update: vector<u8>): Update {
    let mut cursor = bcs::new(update);

    // Parse and validate message magic
    let magic = cursor.peel_u32();
    assert!(magic == UPDATE_MESSAGE_MAGIC, EInvalidUpdateMagic);

    // Parse signature
    let mut signature = vector::empty<u8>();
    let mut sig_i = 0;
    while (sig_i < SECP256K1_SIG_LEN) {
        signature.push_back(cursor.peel_u8());
        sig_i = sig_i + 1;
    };

    // Parse expected payload length and get remaining bytes as payload
    let payload_len = cursor.peel_u16();
    let payload = cursor.into_remainder_bytes();

    // Validate expected payload length
    assert!(payload_len as u64 == payload.length(), EInvalidPayloadLength);

    // Parse payload
    let mut payload_cursor = bcs::new(payload);
    let payload_magic = payload_cursor.peel_u32();
    assert!(payload_magic == PAYLOAD_MAGIC, EInvalidPayloadMagic);

    // Verify the signature against trusted signers
    verify_le_ecdsa_message(s, clock, &signature, &payload);

    update_v2::parse_from_cursor(payload_cursor)
}

module wormhole::consumed_vaas {
    use sui::tx_context::{TxContext};

    use wormhole::bytes32::{Bytes32};
    use wormhole::set::{Self, Set};

    /// Container storing VAA hashes (digests). This will be checked against in
    /// `parse_verify_and_consume` so a particular VAA cannot be replayed. It
    /// is up to the integrator to have this container live in his contract
    /// in order to take advantage of this no-replay protection. Or an
    /// integrator can implement his own method to prevent replay.
    struct ConsumedVAAs has store {
        hashes: Set<Bytes32>
    }

    public fun new(ctx: &mut TxContext): ConsumedVAAs {
        ConsumedVAAs { hashes: set::new(ctx) }
    }

    public fun consume(self: &mut ConsumedVAAs, digest: Bytes32) {
        set::add(&mut self.hashes, digest);
    }
