### Title
Silent `u128 as u64` Truncation in `calculate_maximum_withdraw` Bypasses Overflow Guard — (File: `util/dao/src/lib.rs`)

---

### Summary

CKB's custom capacity safe-math library (`Capacity` in `util/occupied-capacity/core/src/units.rs`) has **no unit tests** for any of its arithmetic operations (`safe_add`, `safe_sub`, `safe_mul`, `safe_mul_ratio`, `bytes`). This is the direct analog to the external report's vulnerability class. As a concrete consequence, a silent truncating `as u64` cast in `calculate_maximum_withdraw` bypasses the overflow guard that every other analogous u128→u64 conversion in the same file applies via `u64::try_from(…).map_err(|_| DaoError::Overflow)?`. An attacker-submitted DAO withdrawal transaction referencing a large-capacity cell can trigger a silently wrong (too-small) computed maximum withdrawal, causing the node to incorrectly reject a valid DAO withdrawal.

---

### Finding Description

**The untested custom safe-math library**

`util/occupied-capacity/core/src/units.rs` defines CKB's custom capacity arithmetic type `Capacity` with safe operations: [1](#0-0) 

There is no `#[cfg(test)]` block, no `mod tests`, and no test file anywhere under `util/occupied-capacity/core/src/` — confirmed by directory listing:



This library is used pervasively in all capacity accounting, DAO field calculation, reward calculation, and transaction verification paths.

**The concrete bug: silent truncation in `calculate_maximum_withdraw`**

In `util/dao/src/lib.rs`, `calculate_maximum_withdraw` uses u128 intermediate arithmetic to avoid overflow, then converts back to u64: [2](#0-1) 

The cast `withdraw_counted_capacity as u64` **silently truncates** if `withdraw_counted_capacity > u64::MAX`. This is inconsistent with every other u128→u64 conversion in the same file, which uses the checked form: [3](#0-2) [4](#0-3) 

Both of those use `u64::try_from(…).map_err(|_| DaoError::Overflow)?`. The `calculate_maximum_withdraw` path does not.

**Why the existing overflow test does not cover this path**

The test `check_withdraw_calculation_overflows` expects `result.is_err()`, but the error is caught by the subsequent `safe_add` call (the truncated small value plus `occupied_capacity` still overflows u64), not by the `as u64` cast itself: [5](#0-4) 

If `withdraw_counted_capacity` wraps to a small value (e.g., wraps to `k` where `k + occupied_capacity ≤ u64::MAX`), `safe_add` succeeds and the function returns a silently wrong, too-small capacity with no error.

---

### Impact Explanation

When `withdraw_counted_capacity` silently wraps to a small value and `safe_add` does not catch it, `calculate_maximum_withdraw` returns a capacity far below the correct maximum. The node then compares this wrong value against the actual withdrawal amount in the transaction. A valid DAO withdrawal claiming the correct (larger) amount is rejected as exceeding the computed maximum. This is a correctness/consensus violation: a user with a sufficiently large DAO deposit and accumulated interest is permanently unable to withdraw via any node running this code.

The function is called both during consensus-critical transaction verification and via the public RPC `calculate_dao_maximum_withdraw`: [6](#0-5) [7](#0-6) 

---

### Likelihood Explanation

Triggering the truncation requires `counted_capacity * withdrawing_ar / deposit_ar > u64::MAX`. Since `counted_capacity ≤ u64::MAX` and `withdrawing_ar / deposit_ar > 1` (interest always grows), a deposit near the maximum u64 capacity combined with a meaningful accumulate-rate increase is sufficient. The accumulate rate (`ar`) starts at `10^16` and grows monotonically; a large enough deposit held long enough can satisfy the condition. An unprivileged transaction sender controls the deposit amount and the header deps (which determine `withdrawing_ar`), making this attacker-reachable via the P2P tx-pool submission path.

---

### Recommendation

1. Replace the silent cast with a checked conversion, consistent with the rest of the file:

```rust
// Before (line 156, util/dao/src/lib.rs):
Capacity::shannons(withdraw_counted_capacity as u64).safe_add(occupied_capacity)?;

// After:
let withdraw_counted_u64 =
    u64::try_from(withdraw_counted_capacity).map_err(|_| DaoError::Overflow)?;
Capacity::shannons(withdraw_counted_u64).safe_add(occupied_capacity)?;
```

2. Add dedicated unit tests for all `Capacity` arithmetic operations in `util/occupied-capacity/core/src/units.rs`, covering boundary values (`u64::MAX`, zero, overflow, underflow) for `safe_add`, `safe_sub`, `safe_mul`, `safe_mul_ratio`, and `bytes`. The `RationalU256` library has property-based tests as a model: [8](#0-7) 

---

### Proof of Concept

```
counted_capacity  = u64::MAX - occupied_capacity   (e.g., ~1.844 × 10^19 shannons)
withdrawing_ar    = deposit_ar + 1                 (any positive interest)

withdraw_counted_capacity (u128) = counted_capacity * withdrawing_ar / deposit_ar
                                 > u64::MAX

withdraw_counted_capacity as u64  →  wraps to small value k
Capacity::shannons(k).safe_add(occupied_capacity)  →  Ok(small_wrong_value)

Node computes maximum_withdraw = small_wrong_value
Actual withdrawal amount       = correct_large_value  (> small_wrong_value)
Node rejects valid transaction  →  user funds locked
```

The attacker-controlled entry path is: submit a DAO withdrawal transaction via P2P (`send_transaction` RPC or relay) referencing a large-capacity DAO cell with header deps pointing to blocks with a high accumulate rate. The node's `transaction_maximum_withdraw` → `calculate_maximum_withdraw` path is triggered during tx-pool admission and block verification. [9](#0-8)

impl Capacity {
    /// Capacity of zero Shannons.
    pub const fn zero() -> Self {
        Capacity(0)
    }

    /// Capacity of one Shannon.
    pub const fn one() -> Self {
        Capacity(1)
    }

    /// Views the capacity as Shannons.
    pub const fn shannons(val: u64) -> Self {
        Capacity(val)
    }

    /// Views the capacity as CKBytes.
    pub fn bytes(val: usize) -> Result<Self> {
        (val as u64)
            .checked_mul(BYTE_SHANNONS)
            .map(Capacity::shannons)
            .ok_or(Error::Overflow)
    }

    /// Views the capacity as Shannons.
    pub fn as_u64(self) -> u64 {
        self.0
    }

    /// Adds self and rhs and checks overflow error.
    pub fn safe_add<C: IntoCapacity>(self, rhs: C) -> Result<Self> {
        self.0
            .checked_add(rhs.into_capacity().0)
            .map(Capacity::shannons)
            .ok_or(Error::Overflow)
    }

    /// Subtracts self and rhs and checks overflow error.
    pub fn safe_sub<C: IntoCapacity>(self, rhs: C) -> Result<Self> {
        self.0
            .checked_sub(rhs.into_capacity().0)
            .map(Capacity::shannons)
            .ok_or(Error::Overflow)
    }

    /// Multiplies self and rhs and checks overflow error.
    pub fn safe_mul<C: IntoCapacity>(self, rhs: C) -> Result<Self> {
        self.0
            .checked_mul(rhs.into_capacity().0)
            .map(Capacity::shannons)
            .ok_or(Error::Overflow)
    }

    /// Multiplies self with a ratio and checks overflow error.
    pub fn safe_mul_ratio(self, ratio: Ratio) -> Result<Self> {
        self.0
            .checked_mul(ratio.numer())
            .and_then(|ret| ret.checked_div(ratio.denom()))
            .map(Capacity::shannons)
            .ok_or(Error::Overflow)
    }

                            && type_script.code_hash() == self.consensus.dao_type_hash()

                        .type_()
                        .to_opt()
                        .map(is_dao_type_script)
                        .unwrap_or(false);
                    if is_dao_output {
                        // A withdrawing DAO cell has 8 bytes of cell data storing the
                        // block number of the original deposit.
                        let deposited_block_number =
                            match self.data_loader.load_cell_data(cell_meta) {
                                Some(data) if data.len() == 8 => LittleEndian::read_u64(&data),
                                _ => 0,
                            };
                        if deposited_block_number > 0 {
                            let withdrawing_header_hash = cell_meta
                                .transaction_info
                                .as_ref()
                                .map(|info| &info.block_hash)
                                .filter(|hash| header_deps.contains(hash))
                                .ok_or(DaoError::InvalidOutPoint)?;
                            let deposit_header_hash = rtx
                                .transaction
                                .witnesses()
                                .get(i)
                                .ok_or(DaoError::InvalidOutPoint)
                                .and_then(|witness_data| {
                                    // dao contract stores header deps index as u64 in the input_type field of WitnessArgs
                                    let witness =
                                        WitnessArgs::from_slice(&Into::<Bytes>::into(witness_data))
                                            .map_err(|_| DaoError::InvalidDaoFormat)?;
                                    let header_deps_index_data: Option<Bytes> =
                                        witness.input_type().to_opt().map(|witness| witness.into());
                                    if header_deps_index_data.is_none()
                                        || header_deps_index_data.clone().map(|data| data.len())
                                            != Some(8)
                                    {
                                        return Err(DaoError::InvalidDaoFormat);
                                    }
                                    Ok(LittleEndian::read_u64(&header_deps_index_data.unwrap()))
                                })
                                .and_then(|header_dep_index| {
                                    rtx.transaction
                                        .header_deps()
                                        .get(header_dep_index as usize)
                                        .and_then(|hash| header_deps.get(&hash))
                                        .ok_or(DaoError::InvalidOutPoint)
                                })?;

                            let deposit_header = self
                                .data_loader
                                .get_header(deposit_header_hash)
                                .ok_or(DaoError::InvalidHeader)?;
                            if deposit_header.number() != deposited_block_number {
                                return Err(DaoError::InvalidOutPoint);
                            }
                            self.calculate_maximum_withdraw(
                                output,
                                Capacity::bytes(cell_meta.data_bytes as usize)?,
                                deposit_header_hash,
                                withdrawing_header_hash,
                            )
                        } else {
                            Ok(output.capacity().into())
                        }
                    } else {
                        Ok(output.capacity().into())
                    }
                };
                capacity.and_then(|c| c.safe_add(capacities).map_err(Into::into))
            },
        )
    }

    /// Calculate maximum withdraw capacity of a deposited dao output
    pub fn calculate_maximum_withdraw(
        &self,
        output: &CellOutput,
        output_data_capacity: Capacity,
        deposit_header_hash: &Byte32,
        withdrawing_header_hash: &Byte32,
    ) -> Result<Capacity, DaoError> {
        let deposit_header = self
            .data_loader
            .get_header(deposit_header_hash)
            .ok_or(DaoError::InvalidHeader)?;
        let withdrawing_header = self
            .data_loader
            .get_header(withdrawing_header_hash)
            .ok_or(DaoError::InvalidHeader)?;
        if deposit_header.number() >= withdrawing_header.number() {
            return Err(DaoError::InvalidOutPoint);
        }

        let (deposit_ar, _, _, _) = extract_dao_data(deposit_header.dao());
        let (withdrawing_ar, _, _, _) = extract_dao_data(withdrawing_header.dao());

        let occupied_capacity = output.occupied_capacity(output_data_capacity)?;
        let output_capacity: Capacity = output.capacity().into();
        let counted_capacity = output_capacity.safe_sub(occupied_capacity)?;
        let withdraw_counted_capacity = u128::from(counted_capacity.as_u64())
            * u128::from(withdrawing_ar)
            / u128::from(deposit_ar);
        let withdraw_capacity =
            Capacity::shannons(withdraw_counted_capacity as u64).safe_add(occupied_capacity)?;

        Ok(withdraw_capacity)
    }

        .number(200)
        .epoch(epoch)
        .dao(pack_dao_data(
            10_000_000_001_123_456,
            Default::default(),
            Default::default(),
            Default::default(),
        ))
        .build();

    let tmp_dir = TempDir::new().unwrap();
    let db = RocksDB::open_in(&tmp_dir, COLUMNS);
    let store = ChainDB::new(db, Default::default());
    let txn = store.begin_transaction();
    txn.insert_block(&deposit_block).unwrap();
    txn.attach_block(&deposit_block).unwrap();
    txn.insert_block(&withdrawing_block).unwrap();
    txn.attach_block(&withdrawing_block).unwrap();
    txn.commit().unwrap();

    let consensus = Consensus::default();
    let data_loader = store.borrow_as_data_loader();
    let calculator = DaoCalculator::new(&consensus, &data_loader);
    let result = calculator.calculate_maximum_withdraw(
        &output,
        Capacity::bytes(0).expect("should not overflow"),
        &deposit_block.hash(),
        &withdrawing_block.hash(),
    );
    assert!(result.is_err());
