### Title
Negative Price Denominator Causes Price Deviation Update Condition to Never Trigger — (`File: apps/price_pusher/src/price-config.ts`)

---

### Summary

In `shouldUpdate()` within `apps/price_pusher/src/price-config.ts`, the `priceDeviationPct` calculation applies `Math.abs()` only to the numerator (the price difference) but divides by the raw, potentially negative `targetLatestPrice.price`. When the on-chain price is negative — which is valid for Pyth feeds representing funding rates or other derivative products — the resulting `priceDeviationPct` is negative. Because the threshold `priceConfig.priceDeviation` is always a positive number, the comparison `priceDeviationPct >= priceConfig.priceDeviation` is permanently false, silently disabling the price-deviation update trigger for the entire lifetime of that feed's negative price.

---

### Finding Description

`PriceInfo.price` is typed as `string` in `apps/price_pusher/src/interface.ts` and originates from the on-chain Pyth price, which is an `int64` (signed). Pyth explicitly supports negative prices for derivative products (e.g., funding rates).

The faulty calculation in `shouldUpdate()`:

```typescript
const priceDeviationPct =
    (Math.abs(
      Number(sourceLatestPrice.price) - Number(targetLatestPrice.price),
    ) /
      Number(targetLatestPrice.price)) *   // ← denominator not abs'd
    100;
```

When `targetLatestPrice.price` is, say, `-500`:

- Numerator: `Math.abs(source - (-500))` → positive
- Denominator: `Number("-500")` → `-500`
- Result: `priceDeviationPct` is **negative**

The subsequent threshold check:

```typescript
priceDeviationPct >= priceConfig.priceDeviation   // negative >= positive → always false
```

…and the early-update variant:

```typescript
priceDeviationPct >= priceConfig.earlyUpdatePriceDeviation  // same, always false
```

Both branches are permanently dead for any feed whose on-chain price is negative. The `confidenceRatioPct` calculation does not share this bug because it wraps the entire expression in `Math.abs()`.

---

### Impact Explanation

The price-deviation update condition is silently disabled for all feeds with a negative on-chain price. The price pusher will only push updates when the heartbeat (`timeDifference`) or confidence-ratio threshold is met. For feeds where the price moves significantly but the heartbeat has not elapsed (e.g., a funding rate swinging from −0.01% to −5% within a short window), the on-chain price will remain stale until the heartbeat fires. Consumers relying on the deviation-triggered freshness guarantee receive incorrect data, which can lead to mispriced positions or incorrect liquidation decisions in protocols that integrate Pyth.

---

### Likelihood Explanation

Pyth explicitly documents that prices can be negative for derivative products. Any operator running the price pusher against such a feed is affected deterministically — no special attacker action is required. The bug is triggered purely by the sign of the on-chain price, which is a normal, expected value for a subset of Pyth feeds.

---

### Recommendation

Apply `Math.abs()` to the denominator so the percentage is always computed relative to the magnitude of the reference price:

```typescript
const priceDeviationPct =
    (Math.abs(
      Number(sourceLatestPrice.price) - Number(targetLatestPrice.price),
    ) /
      Math.abs(Number(targetLatestPrice.price))) *   // ← fix
    100;
```

Additionally, guard against a zero denominator (which would produce `Infinity` or `NaN`) before performing the division.

---

### Proof of Concept

**Setup**: Configure the price pusher for a feed whose on-chain price is currently `"-500"` (a valid negative `int64` Pyth price). Set `priceDeviation: 1` (1%) in the config. Publish a new Hermes price of `"-450"` — a 10% deviation.

**Trace through `shouldUpdate()`**:

```
sourceLatestPrice.price = "-450"
targetLatestPrice.price = "-500"

numerator  = Math.abs(Number("-450") - Number("-500"))
           = Math.abs(-450 - (-500))
           = Math.abs(50)
           = 50

denominator = Number("-500") = -500          // ← no Math.abs

priceDeviationPct = (50 / -500) * 100 = -10  // negative!

Check: -10 >= 1  →  false                    // deviation condition never fires
```

**Result**: `shouldUpdate()` returns `UpdateCondition.NO` (or `EARLY` only if the heartbeat early-update path happens to match), even though the price has moved 10% — ten times the configured threshold.
