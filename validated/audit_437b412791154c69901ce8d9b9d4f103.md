### Title
Missing Upper-Bound Validation on `heartbeatSeconds` and `deviationThresholdBps` in `createSubscription` Allows Permanent Locking of Deposited ETH - (File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol)

---

### Summary

`_validateSubscriptionParams` in `Scheduler.sol` only enforces a lower-bound of `> 0` on `heartbeatSeconds` and `deviationThresholdBps`, but imposes no upper-bound. A user can create a **permanent** subscription with `heartbeatSeconds = type(uint32).max` (~136 years) or `deviationThresholdBps = type(uint32).max` (~43 billion bps), causing the subscription to never trigger an update. Because permanent subscriptions prohibit withdrawal and deactivation, the deposited ETH (up to `MAX_DEPOSIT_LIMIT = 100 ETH`) is permanently locked in the contract.

---

### Finding Description

`_validateSubscriptionParams` validates `UpdateCriteria` as follows:

```solidity
if (
    params.updateCriteria.updateOnHeartbeat &&
    params.updateCriteria.heartbeatSeconds == 0
) {
    revert SchedulerErrors.InvalidUpdateCriteria();
}
if (
    params.updateCriteria.updateOnDeviation &&
    params.updateCriteria.deviationThresholdBps == 0
) {
    revert SchedulerErrors.InvalidUpdateCriteria();
}
``` [1](#0-0) 

There is no maximum value check. `heartbeatSeconds` is `uint32`, allowing values up to `4,294,967,295` seconds (~136 years). `deviationThresholdBps` is also `uint32`, allowing values up to `4,294,967,295` bps (~43 billion percent). Either extreme value makes the update condition practically unreachable.

The heartbeat trigger logic is:
```solidity
if (
    lastUpdateTime == 0 ||
    updateTimestamp >= lastUpdateTime + params.updateCriteria.heartbeatSeconds
) {
    return updateTimestamp;
}
``` [2](#0-1) 

With `heartbeatSeconds = type(uint32).max`, this condition will never be satisfied after the first update.

For permanent subscriptions, `withdrawFunds` and `updateSubscription` both revert unconditionally:
```solidity
if (params.isPermanent) {
    revert SchedulerErrors.CannotUpdatePermanentSubscription();
}
``` [3](#0-2) 

The deposit cap for permanent subscriptions is `MAX_DEPOSIT_LIMIT = 100 ether`:
```solidity
if (subscriptionParams.isPermanent && msg.value > MAX_DEPOSIT_LIMIT) {
    revert SchedulerErrors.MaxDepositLimitExceeded();
}
``` [4](#0-3) 

`MAX_DEPOSIT_LIMIT` is defined as:
```solidity
uint256 public constant MAX_DEPOSIT_LIMIT = 100 ether;
``` [5](#0-4) 

---

### Impact Explanation

**Impact: High.**

A user who creates a permanent subscription with `heartbeatSeconds = type(uint32).max` (and `updateOnDeviation = false`) or `deviationThresholdBps = type(uint32).max` (and `updateOnHeartbeat = false`) will have their deposited ETH — up to 100 ETH — permanently locked in the contract with no recovery path. The subscription can never be deactivated, updated, or have funds withdrawn once `isPermanent = true` is set. The balance drains only when `updatePriceFeeds` is called successfully, but with an unreachable update condition, no keeper will ever call it, and the funds are frozen forever.

---

### Likelihood Explanation

**Likelihood: Low.**

This requires user error or misconfiguration — the same class as the reference report. A user must set an extreme value for `heartbeatSeconds` or `deviationThresholdBps` and also mark the subscription as permanent. This is unlikely in normal operation but is a realistic misconfiguration scenario (e.g., intending to set a very long heartbeat like `type(uint32).max` as a "disable heartbeat" sentinel, combined with a deviation-only mode that also has an extreme threshold).

---

### Recommendation

Add maximum bounds in `_validateSubscriptionParams`:

```solidity
uint32 public constant MAX_HEARTBEAT_SECONDS = 7 days; // e.g., 604800
uint32 public constant MAX_DEVIATION_THRESHOLD_BPS = 10_000; // 100%

if (params.updateCriteria.updateOnHeartbeat &&
    params.updateCriteria.heartbeatSeconds > MAX_HEARTBEAT_SECONDS) {
    revert SchedulerErrors.InvalidUpdateCriteria();
}
if (params.updateCriteria.updateOnDeviation &&
    params.updateCriteria.deviationThresholdBps > MAX_DEVIATION_THRESHOLD_BPS) {
    revert SchedulerErrors.InvalidUpdateCriteria();
}
```

This mirrors the pattern already used for `MAX_PRICE_IDS_PER_SUBSCRIPTION` and `MAX_READER_WHITELIST_SIZE`. [6](#0-5) 

---

### Proof of Concept

```solidity
// Attacker/user creates a permanent subscription with unreachable heartbeat
SchedulerStructs.UpdateCriteria memory criteria = SchedulerStructs.UpdateCriteria({
    updateOnHeartbeat: true,
    heartbeatSeconds: type(uint32).max, // ~136 years — passes the != 0 check
    updateOnDeviation: false,
    deviationThresholdBps: 0
});

SchedulerStructs.SubscriptionParams memory params = SchedulerStructs.SubscriptionParams({
    priceIds: priceIds,
    readerWhitelist: new address[](0),
    whitelistEnabled: false,
    isActive: true,
    isPermanent: true,   // <-- permanent: no withdraw, no update, no deactivate
    updateCriteria: criteria
});

// Deposit up to 100 ETH — passes _validateSubscriptionParams (heartbeatSeconds != 0)
scheduler.createSubscription{value: 100 ether}(params);

// Now: 100 ETH is permanently locked.
// withdrawFunds() → CannotUpdatePermanentSubscription
// updateSubscription() → CannotUpdatePermanentSubscription
// updatePriceFeeds() → UpdateConditionsNotMet (heartbeat never fires)
```

The `_validateSubscriptionParams` call at line 35 passes because `heartbeatSeconds != 0`, but no upper-bound check exists. [7](#0-6)

        SchedulerStructs.SubscriptionParams memory subscriptionParams
    ) external payable override returns (uint256 subscriptionId) {
        _validateSubscriptionParams(subscriptionParams);

        // Calculate minimum balance required for this subscription
        uint256 minimumBalance = this.getMinimumBalance(
            uint8(subscriptionParams.priceIds.length)
        );

        // Ensure enough funds were provided
        if (msg.value < minimumBalance) {
            revert SchedulerErrors.InsufficientBalance();
        }

        // Check deposit limit for permanent subscriptions
        if (subscriptionParams.isPermanent && msg.value > MAX_DEPOSIT_LIMIT) {
            revert SchedulerErrors.MaxDepositLimitExceeded();
        }

        // Set subscription to active
        subscriptionParams.isActive = true;

        subscriptionId = _state.subscriptionNumber++;

        // Store the subscription parameters
        _state.subscriptionParams[subscriptionId] = subscriptionParams;

        // Initialize subscription status

        status.priceLastUpdatedAt = 0;
        status.balanceInWei = msg.value;
        status.totalUpdates = 0;
        status.totalSpent = 0;

        _state.subscriptionManager[subscriptionId] = msg.sender;

        emit SubscriptionCreated(subscriptionId, msg.sender);

    /// @notice Validates subscription parameters.
    /// @param params The subscription parameters to validate.
    function _validateSubscriptionParams(
        SchedulerStructs.SubscriptionParams memory params
    ) internal pure {
        // No zero‐feed subscriptions
        if (params.priceIds.length == 0) {
            revert SchedulerErrors.EmptyPriceIds();
        }

        // Price ID limits and uniqueness
        if (
            params.priceIds.length >
            SchedulerConstants.MAX_PRICE_IDS_PER_SUBSCRIPTION
        ) {
            revert SchedulerErrors.TooManyPriceIds(
                params.priceIds.length,
                SchedulerConstants.MAX_PRICE_IDS_PER_SUBSCRIPTION
            );
        }
        for (uint i = 0; i < params.priceIds.length; i++) {
            for (uint j = i + 1; j < params.priceIds.length; j++) {
                if (params.priceIds[i] == params.priceIds[j]) {
                    revert SchedulerErrors.DuplicatePriceId(params.priceIds[i]);
                }
            }
        }

        // Whitelist size limit and uniqueness
        if (params.readerWhitelist.length > MAX_READER_WHITELIST_SIZE) {
            revert SchedulerErrors.TooManyWhitelistedReaders(
                params.readerWhitelist.length,
                MAX_READER_WHITELIST_SIZE
            );
        }
        for (uint i = 0; i < params.readerWhitelist.length; i++) {
            for (uint j = i + 1; j < params.readerWhitelist.length; j++) {
                if (params.readerWhitelist[i] == params.readerWhitelist[j]) {
                    revert SchedulerErrors.DuplicateWhitelistAddress(
                        params.readerWhitelist[i]
                    );
                }
            }
        }

        // Validate update criteria
        if (
            !params.updateCriteria.updateOnHeartbeat &&
            !params.updateCriteria.updateOnDeviation
        ) {
            revert SchedulerErrors.InvalidUpdateCriteria();
        }
        if (
            params.updateCriteria.updateOnHeartbeat &&
            params.updateCriteria.heartbeatSeconds == 0
        ) {
            revert SchedulerErrors.InvalidUpdateCriteria();
        }
        if (
            params.updateCriteria.updateOnDeviation &&
            params.updateCriteria.deviationThresholdBps == 0
        ) {
            revert SchedulerErrors.InvalidUpdateCriteria();
        }
    }
