### Title
Unbounded While Loop in `executeCallback` Can DoS Request Fulfillment — (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

The `Echo.executeCallback` function contains an unbounded `while` loop that advances `_state.firstUnfulfilledSeq` past all inactive (fulfilled) requests. An unprivileged caller can create many requests and fulfill them out of order, causing a single subsequent `executeCallback` call to perform O(N) cold storage reads and writes. If N is large enough (~12,000+), the call hits the EVM block gas limit and reverts, permanently preventing the targeted request from ever being fulfilled and locking the requester's funds.

---

### Finding Description

After clearing a fulfilled request, `executeCallback` runs the following loop:

```solidity
// Echo.sol lines 169–174
while (
    _state.firstUnfulfilledSeq < _state.currentSequenceNumber &&
    !isActive(findRequest(_state.firstUnfulfilledSeq))
) {
    _state.firstUnfulfilledSeq++;
}
```

`findRequest` performs a `keccak256` hash and one or two `SLOAD`s per iteration. `firstUnfulfilledSeq++` is an `SSTORE` per iteration. With cold storage, each iteration costs roughly 2,100–2,500 gas. At the 30 M gas block limit, ~12,000 iterations suffice to exhaust the block.

`firstUnfulfilledSeq` is a persistent state variable that only moves forward. It does **not** advance past an active request. Therefore, if request #1 is still active while requests #2 … #N are all fulfilled, `firstUnfulfilledSeq` stays at 1 across every one of those N−1 `executeCallback` calls. The moment request #1 is finally cleared, the loop must scan all N inactive slots in a single transaction. [1](#0-0) 

There is no cap on the number of requests that can be created:

```solidity
// Echo.sol line 73
requestSequenceNumber = _state.currentSequenceNumber++;
``` [2](#0-1)

### Impact Explanation

When the while loop exceeds the block gas limit, the entire `executeCallback` transaction reverts. Because `firstUnfulfilledSeq` is part of the reverted state, every subsequent attempt to fulfill request #1 restarts the same O(N) scan and reverts again. The request is permanently un-fulfillable, and the requester's ETH (fee) is locked in the contract with no recovery path.

---

### Likelihood Explanation

The attack is reachable by any unprivileged caller:

1. Attacker calls `requestPriceUpdatesWithCallback` N times (paying the per-request fee each time). `publishTime` for request #1 is set to `block.timestamp + 60` (the maximum allowed); requests #2 … #N use a past timestamp.
2. Attacker calls `executeCallback` for requests #2 … #N using publicly available Hermes price data. Each call costs normal gas; the while loop stops immediately at request #1 (still active), so `firstUnfulfilledSeq` stays at 1.
3. After 60 seconds, the provider attempts to fulfill request #1. `clearRequest` marks it inactive, then the while loop scans all N slots — hitting the block gas limit if N ≥ ~12,000.
4. The call reverts; request #1 is never fulfilled.

Cost to attacker: N × fee + (N−1) × gas per `executeCallback`. At a modest fee of 0.001 ETH, 12,000 requests costs ~12 ETH — feasible for a motivated attacker targeting a high-value consumer contract.

---

### Recommendation

Decouple the `firstUnfulfilledSeq` advancement from `executeCallback`. Options:

1. **Remove the while loop entirely** from `executeCallback` and expose a separate, gas-bounded `advanceFirstUnfulfilledSeq(uint256 maxSteps)` function that keepers can call off-chain.
2. **Cap the loop** to a fixed maximum number of iterations per call (e.g., 50), accepting that `firstUnfulfilledSeq` may lag but never blocks fulfillment.
3. **Use a doubly-linked list** of active requests (as noted in the existing TODO comment on line 166) so that `firstUnfulfilledSeq` can be updated in O(1). [4](#0-3)

N = 15_000

// Step 1 – attacker creates N requests
for i in 1..N:
    publishTime = block.timestamp + (60 if i == 1 else -1)
    echo.requestPriceUpdatesWithCallback{value: fee}(provider, publishTime, priceIds, gasLimit)
// Sequence numbers 1..N assigned; firstUnfulfilledSeq = 1

// Step 2 – attacker fulfills requests 2..N (loop stops at seq=1 each time)
for i in 2..N:
    echo.executeCallback(provider, i, updateData, priceIds)
// firstUnfulfilledSeq still = 1

// Step 3 – 60 seconds pass; provider fulfills request 1
echo.executeCallback(provider, 1, updateData, priceIds)
// clearRequest(1) marks seq 1 inactive
// while loop: scans seq 1 (inactive) → seq 2 (inactive) → … → seq 15_000 (inactive)
// ~15_000 × 2_500 gas ≈ 37.5 M gas > 30 M block limit → REVERT
// firstUnfulfilledSeq unchanged; request 1 permanently un-fulfillable

    function executeCallback(
        address providerToCredit,
        uint64 sequenceNumber,
        bytes[] calldata updateData,
        bytes32[] calldata priceIds
    ) external payable override {
        Request storage req = findActiveRequest(sequenceNumber);

        // Check provider exclusivity using configurable period
        if (
            block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds
        ) {
            require(
                providerToCredit == req.provider,
                "Only assigned provider during exclusivity period"
            );
        }

        // Verify priceIds match
        require(
            priceIds.length == req.priceIdPrefixes.length,
            "Price IDs length mismatch"
        );
        for (uint8 i = 0; i < req.priceIdPrefixes.length; i++) {
            // Extract first 8 bytes of the provided price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }

            // Compare with stored prefix
            if (prefix != req.priceIdPrefixes[i]) {
                // Now we can directly use the bytes8 prefix in the error
                revert InvalidPriceIds(priceIds[i], req.priceIdPrefixes[i]);
            }
        }

        // TODO: should this use parsePriceFeedUpdatesUnique? also, do we need to add 1 to maxPublishTime?
        IPyth pyth = IPyth(_state.pyth);
        uint256 pythFee = pyth.getUpdateFee(updateData);
        PythStructs.PriceFeed[] memory priceFeeds = pyth.parsePriceFeedUpdates{
            value: pythFee
        }(

            priceIds,
            SafeCast.toUint64(req.publishTime),
            SafeCast.toUint64(req.publishTime)
        );
