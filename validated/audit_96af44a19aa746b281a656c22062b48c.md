### Title
`stake_caps_parameters::set_parameters` Does Not Validate `m` and `z` Inputs, Enabling Any Caller to Corrupt Stake Cap Parameters — (`pythnet/stake_caps_parameters/programs/stake_caps_parameters/src/lib.rs`)

---

### Summary

The `set_parameters` instruction in the `stake_caps_parameters` Solana program accepts the `Parameters` struct (containing `current_authority`, `m`, and `z`) with no bounds or sanity checks on `m` or `z`. Additionally, when the PDA has never been written (i.e., `current_authority == Pubkey::default()`), the authority check is bypassed entirely, meaning **any signer** can call the instruction and write arbitrary values — including `m = 0` or `z = 0` — before the legitimate operator does.

---

### Finding Description

```rust
// pythnet/stake_caps_parameters/programs/stake_caps_parameters/src/lib.rs
pub fn set_parameters(ctx: Context<SetParameters>, parameters: Parameters) -> Result<()> {
    let stored_parameters = &mut ctx.accounts.parameters;
    require!(
        ctx.accounts.signer.key() == stored_parameters.current_authority
            || stored_parameters.current_authority == Pubkey::default(),
        ErrorCode::WrongAuthority
    );
    **stored_parameters = parameters;   // ← no validation of m or z
    Ok(())
}
```

The `Parameters` account is declared with `init_if_needed`:

```rust
#[account(
    init_if_needed,
    seeds = ["parameters".as_bytes()],
    bump,
    payer = signer,
    space = Parameters::LEN
)]
pub parameters: Account<'info, Parameters>,
```

On the very first call the account is created with all-zero fields, so `current_authority == Pubkey::default()`. The `require!` guard passes for **any** signer in this state. The caller can therefore supply:

- `m = 0` and/or `z = 0` — values that are semantically invalid for a stake-cap multiplier/divisor
- `current_authority = <attacker pubkey>` — permanently locking out the legitimate operator

After this call the account is owned by the attacker's key, and the legitimate deployer can never overwrite it (the `WrongAuthority` error will fire for every subsequent call from the real operator).

Even after legitimate initialization, the function still accepts any `m` and `z` from the stored authority with no range checks, mirroring the Bond `setDefaults` pattern exactly.

---

### Impact Explanation

`m` and `z` are consumed by the integrity pool program to compute per-publisher stake caps. Setting either to `0` will cause the cap formula to produce `0` (or trigger an arithmetic panic / division-by-zero) for every publisher, effectively freezing all new delegations and breaking the Oracle Integrity Staking reward accounting. The attacker also seizes permanent control of the parameter account, preventing recovery without a program upgrade.

---

### Likelihood Explanation

The window exists from the moment the `stake_caps_parameters` program is deployed until the operator's initialization transaction is confirmed. On Solana, any observer of the RPC can submit a competing transaction in the same slot or the next slot. The attack requires no special privilege, no capital, and no prior relationship with the protocol — only the ability to send a Solana transaction.

---

### Recommendation

1. **Add input validation** inside `set_parameters` before writing:
   ```rust
   require!(parameters.m > 0, ErrorCode::InvalidM);
   require!(parameters.z > 0, ErrorCode::InvalidZ);
   // optionally: require!(parameters.m <= SOME_MAX, ErrorCode::InvalidM);
   ```
2. **Remove the `Pubkey::default()` bypass** or replace `init_if_needed` with a dedicated one-time `initialize` instruction that hard-codes the authority at deploy time, preventing any race on first initialization.

---

### Proof of Concept

1. Deploy (or observe deployment of) the `stake_caps_parameters` program before the operator calls `set_parameters`.
2. Immediately call `set_parameters` with:
   ```
   parameters = Parameters {
       current_authority: <attacker_pubkey>,
       m: 0,
       z: 0,
   }
   ```
3. The `require!` guard passes because `stored_parameters.current_authority == Pubkey::default()`.
4. The PDA now stores `m = 0`, `z = 0`, `current_authority = attacker`.
5. Any subsequent call from the legitimate operator fails with `WrongAuthority`.
6. The integrity pool reads `m = 0` / `z = 0` from the PDA and the stake-cap calculation produces `0` (or panics), halting all publisher delegation accounting. [1](#0-0) [2](#0-1) [3](#0-2)
