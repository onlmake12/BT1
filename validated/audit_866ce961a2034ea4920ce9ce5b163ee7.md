### Title
`MaturityVerifier` Does Not Check `resolved_dep_groups` for Cellbase Immaturity — (`File: verification/src/transaction_verifier.rs`)

---

### Summary

`MaturityVerifier::verify()` checks `resolved_inputs` and `resolved_cell_deps` for cellbase immaturity, but silently skips `resolved_dep_groups`. A transaction sender can reference an immature cellbase output as a `DepGroup`-type cell dependency and bypass the cellbase maturity consensus rule.

---

### Finding Description

`ResolvedTransaction` holds three distinct cell collections:

- `resolved_inputs` — consumed input cells
- `resolved_cell_deps` — code/data dependency cells (including sub-cells expanded from dep groups)
- `resolved_dep_groups` — the dep-group container cells themselves [1](#0-0) 

When `resolve_transaction_dep()` processes a `DepType::DepGroup` entry, the container cell (the one whose data encodes the list of sub-out-points) is pushed into `resolved_dep_groups`, while the sub-cells it points to are pushed into `resolved_cell_deps`. [2](#0-1) 

`MaturityVerifier::verify()` iterates over `resolved_inputs` and `resolved_cell_deps` to enforce the cellbase maturity rule, but never iterates over `resolved_dep_groups`: [3](#0-2) 

The doc comment on `CellbaseImmaturity` even acknowledges only `Inputs` and `CellDeps` as valid sources, omitting dep groups entirely: [4](#0-3) 

This is structurally identical to the reported Brahma/Spearbit finding: one specific category is validated while a parallel category of equal power is silently skipped.

---

### Impact Explanation

The cellbase maturity rule exists to prevent spending or referencing outputs from very recent coinbase transactions before they are considered settled. A dep-group container cell is a live cell on-chain; if it was created as a cellbase output in a recent block, it is subject to the same maturity requirement as any other cellbase-derived cell. Because `resolved_dep_groups` is never checked, a transaction that uses such a cell as a `DepGroup` dep will pass `MaturityVerifier` and be accepted into the tx-pool and committed into a block, violating the consensus rule.

This is a **consensus validation bypass**: nodes that apply the fix would reject the transaction; nodes running the current code would accept it. This creates a chain-split vector.

---

### Likelihood Explanation

Any unprivileged transaction sender can craft this scenario:
1. Mine a block (or observe a freshly mined block) containing a cellbase output.
2. Construct a cell whose data is a valid `OutPointVec` pointing to other live cells, funded from that immature cellbase output — making the dep-group container cell itself a cellbase-derived cell.
3. Submit a transaction referencing that container cell as a `DepType::DepGroup` cell dep before the maturity window expires.

The tx-pool admission path (`verify_rtx` → `ContextualTransactionVerifier::verify` → `TimeRelativeTransactionVerifier::verify` → `MaturityVerifier::verify`) will pass without error. [5](#0-4) 

---

### Recommendation

Extend `MaturityVerifier::verify()` to also iterate over `self.transaction.resolved_dep_groups` using the same `cellbase_immature` closure, returning `TransactionError::CellbaseImmaturity` with a new `TransactionErrorSource::DepGroups` variant (or reusing `CellDeps`) if an immature dep-group container cell is found.

---

    pub fn verify(&self) -> Result<(), Error> {
        let cellbase_immature = |meta: &CellMeta| -> bool {
            meta.transaction_info
                .as_ref()
                .map(|info| {
                    info.block_number > 0 && info.is_cellbase() && {
                        let threshold =
                            self.cellbase_maturity.to_rational() + info.block_epoch.to_rational();
                        let current = self.epoch.to_rational();
                        current < threshold
                    }
                })
                .unwrap_or(false)
        };

        if let Some(index) = self
            .transaction
            .resolved_inputs
            .iter()
            .position(cellbase_immature)
        {
            return Err(TransactionError::CellbaseImmaturity {
                inner: TransactionErrorSource::Inputs,
                index,
            }
            .into());
        }

        if let Some(index) = self
            .transaction
            .resolved_cell_deps
            .iter()
            .position(cellbase_immature)
        {
            return Err(TransactionError::CellbaseImmaturity {
                inner: TransactionErrorSource::CellDeps,
                index,
            }
            .into());
        }

        Ok(())
    }
