### Title
`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` Always Revert for Whitelist-Enabled Subscriptions Due to `this.` External Call Changing `msg.sender` - (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` in `Scheduler.sol` internally call `this.getPricesUnsafe` and `this.getEmaPricesUnsafe` respectively. Because `this.f()` in Solidity is an **external call**, `msg.sender` inside the inner function becomes the Scheduler contract's own address. The `onlyWhitelistedReader` modifier then checks whether the Scheduler contract itself is the subscription manager or is in the whitelist — it is neither — causing an unconditional `Unauthorized` revert for every subscription with `whitelistEnabled = true`.

---

### Finding Description

`getPricesNoOlderThan` applies `onlyWhitelistedReader` to the original caller, passes that check, then delegates to `this.getPricesUnsafe`:

```solidity
// Scheduler.sol line 535-555
function getPricesNoOlderThan(
    uint256 subscriptionId,
    bytes32[] calldata priceIds,
    uint256 age_seconds
)
    external view override
    onlyWhitelistedReader(subscriptionId)   // ← checks original caller ✓
    returns (PythStructs.Price[] memory prices)
{
    ...
    prices = this.getPricesUnsafe(subscriptionId, priceIds); // ← external call; msg.sender = address(this)
}
```

Inside `getPricesUnsafe`, the modifier fires again:

```solidity
// Scheduler.sol line 514-533
function getPricesUnsafe(...)
    external view override
    onlyWhitelistedReader(subscriptionId)   // ← msg.sender is now the Scheduler contract
    ...
```

The `onlyWhitelistedReader` modifier:

```solidity
// Scheduler.sol line 750-779
modifier onlyWhitelistedReader(uint256 subscriptionId) {
    if (_state.subscriptionManager[subscriptionId] == msg.sender) { _; return; }
    if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) { _; return; }
    // whitelist check — Scheduler contract is never in the whitelist
    if (!isWhitelisted) revert SchedulerErrors.Unauthorized();
    _;
}
```

When `whitelistEnabled = true`, the Scheduler contract address is not the manager and is not in the whitelist, so the inner call always reverts. The same defect exists in `getEmaPricesNoOlderThan` → `this.getEmaPricesUnsafe`.

---

### Impact Explanation

Any subscription that enables the reader whitelist (`whitelistEnabled = true`) renders `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` permanently unusable. These are the **staleness-checked** price getters — the safe API that the interface explicitly recommends over the `Unsafe` variants. Whitelisted consumers are silently forced to fall back to `getPricesUnsafe` / `getEmaPricesUnsafe`, which carry no recency guarantee, degrading the security model of every whitelist-protected subscription.

---

### Likelihood Explanation

The whitelist feature is a first-class, documented capability of the Scheduler. Any integrator who creates a subscription with `whitelistEnabled = true` and then calls the recommended staleness-checked getter will hit this revert on every invocation. The entry path requires no privilege: any whitelisted reader calling `getPricesNoOlderThan` triggers it.

---

### Recommendation

Replace the `this.` external calls with direct calls to the internal helper `_getPricesInternal`, which already performs the same storage lookup without re-applying the access-control modifier:

```solidity
// getPricesNoOlderThan — replace:
prices = this.getPricesUnsafe(subscriptionId, priceIds);
// with:
PythStructs.PriceFeed[] memory feeds = _getPricesInternal(subscriptionId, priceIds);
prices = new PythStructs.Price[](feeds.length);
for (uint i = 0; i < feeds.length; i++) prices[i] = feeds[i].price;

// getEmaPricesNoOlderThan — replace:
prices = this.getEmaPricesUnsafe(subscriptionId, priceIds);
// with:
PythStructs.PriceFeed[] memory feeds = _getPricesInternal(subscriptionId, priceIds);
prices = new PythStructs.Price[](feeds.length);
for (uint i = 0; i < feeds.length; i++) prices[i] = feeds[i].emaPrice;
```

---

### Proof of Concept

1. Deploy `SchedulerUpgradeable` and create a subscription with `whitelistEnabled = true` and `readerWhitelist = [alice]`.
2. Fund the subscription and push a price update.
3. As `alice` (a whitelisted reader), call `getPricesNoOlderThan(subscriptionId, priceIds, 60)`.
4. The call reverts with `SchedulerErrors.Unauthorized` because the inner `this.getPricesUnsafe` sees `msg.sender = address(scheduler)`, which is not in the whitelist.
5. Calling `getPricesUnsafe` directly as `alice` succeeds, confirming the outer whitelist check passes but the inner one does not.

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
