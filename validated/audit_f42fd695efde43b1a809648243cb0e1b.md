### Title
Cross-Network Alert Signature Replay: `RawAlert` Hash Lacks Network Domain Separator - (File: `util/network-alert/src/verifier.rs`)

---

### Summary

The CKB network alert system signs and verifies alerts using a raw `blake2b_256` hash of the `RawAlert` molecule struct. Because the signed hash contains no network/chain identifier (no domain separator), a valid alert signature produced for one CKB network (e.g., mainnet `ckb`) can be replayed verbatim on any other CKB network (e.g., testnet `ckb_testnet`) that shares the same authorized public keys. Any unprivileged P2P peer or RPC caller can perform this replay.

---

### Finding Description

`Verifier::verify_signatures` in `util/network-alert/src/verifier.rs` computes the message to verify as:

```rust
let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
``` [1](#0-0) 

`calc_alert_hash` delegates to `calc_hash`, which is simply `blake2b_256(self.as_slice())` over the raw molecule bytes of `RawAlert`:

```rust
impl<'r> packed::RawAlertReader<'r> {
    pub fn calc_alert_hash(&self) -> packed::Byte32 {
        self.calc_hash()   // = blake2b_256(self.as_slice())
    }
}
``` [2](#0-1) 

The `RawAlert` molecule schema contains only application-level fields — `notice_until`, `id`, `cancel`, `priority`, `message`, `min_version`, `max_version` — with **no network name, genesis hash, or any chain-specific domain separator**: [3](#0-2) 

The authorized public keys are the same hard-coded set across all CKB networks by default: [4](#0-3) 

Because the hash is purely a function of the alert content and not of the network, a signature `(r, s, v)` over a mainnet alert is cryptographically identical to a valid signature over the same alert on testnet or any devnet using the same keys.

---

### Impact Explanation

An attacker who observes a legitimately signed alert on mainnet (publicly broadcast over P2P) can replay it to testnet nodes — or vice versa — without possessing any private key. Testnet nodes will accept the alert as fully valid (2-of-4 threshold satisfied), store it, display it to operators, and re-broadcast it to all connected peers via `AlertRelayer::received`: [5](#0-4) 

This causes nodes on the wrong network to display misleading critical alerts (e.g., "upgrade immediately" or "critical bug in version X.Y") that were never intended for them. Conversely, a testnet-only alert could be injected into mainnet nodes, potentially triggering unnecessary operator responses or panic.

---

### Likelihood Explanation

- All mainnet alerts are publicly broadcast over P2P and are trivially observable by any node.
- The `send_alert` RPC endpoint is accessible to any local or remote RPC caller with no authentication beyond the alert's own signatures.
- The same 4 hard-coded public keys are the default for all CKB network configurations.
- No cryptographic capability is required — the attacker only needs to copy a valid alert from one network and submit it to another. [6](#0-5) 

---

### Recommendation

Include a network domain separator in the signed data. The natural choice is the genesis block hash (already used as a network identifier in `Consensus::identify_name`): [7](#0-6) 

Concretely, `calc_alert_hash` should hash `blake2b_256(genesis_hash || raw_alert_bytes)` rather than `blake2b_256(raw_alert_bytes)` alone, or the `RawAlert` molecule struct should include a `network_id` / `genesis_hash` field that is part of the signed payload. The `Verifier` would need to receive the genesis hash at construction time and incorporate it into the message derivation.

---

### Proof of Concept

1. Run a CKB mainnet node and a CKB testnet node, both using the default `alert_signature` config (same 4 public keys).
2. Observe a legitimately signed alert broadcast on mainnet (e.g., alert id `20230001` with the two known signatures from `generate_alert_signature.rs`).
3. Submit the identical `packed::Alert` bytes to the testnet node via `send_alert` RPC:
   ```json
   {
     "id": "0x1337beef",
     "cancel": "0x0",
     "priority": "0x14",
     "message": "CKB v0.105.* have bugs. Please upgrade.",
     "notice_until": "0x...",
     "signatures": ["0x8dca28...c97c00", "0x4554b3...4e0a01"]
   }
   ```
4. The testnet node's `Verifier::verify_signatures` recomputes `blake2b_256(raw_alert_bytes)` — identical to the mainnet hash — recovers the same two public keys, passes the 2-of-4 threshold check, stores the alert, and re-broadcasts it to all testnet peers. [8](#0-7)

    pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
        trace!("Verifying alert {:?}", alert);
        let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
        let signatures: Vec<Signature> = alert
            .signatures()
            .into_iter()
            .filter_map(
                |sig_data| match Signature::from_slice(sig_data.as_reader().raw_data()) {
                    Ok(sig) => {
                        if sig.is_valid() {
                            Some(sig)
                        } else {
                            debug!("invalid signature: {:?}", sig);
                            None
                        }
                    }
                    Err(err) => {
                        debug!("signature error: {}", err);
                        None
                    }
                },
            )
            .collect();
        verify_m_of_n(
            &message,
            self.config.signatures_threshold,
            &signatures,
            &self.pubkeys,
        )
        .map_err(|err| err.kind())?;
        Ok(())
    }

#[async_trait]
impl AlertRpc for AlertRpcImpl {
    fn send_alert(&self, alert: Alert) -> Result<()> {
        let alert: packed::Alert = alert.into();
        let now_ms = ckb_systemtime::unix_time_as_millis();
        let notice_until: u64 = alert.raw().notice_until().into();
        if notice_until < now_ms {
            return Err(RPCError::invalid_params(format!(
                "Expected `params[0].notice_until` in the future (> {now_ms}), got {notice_until}",
            )));
        }

        let result = self.verifier.verify_signatures(&alert);

        match result {
            Ok(()) => {
                // set self node notifier
                self.notifier.lock().add(&alert);

                self.network_controller.broadcast_with_handle(
                    SupportProtocols::Alert.protocol_id(),
                    alert.as_bytes(),
                    &self.handle,
                );
                Ok(())
            }
            Err(e) => Err(RPCError::custom_with_error(
                RPCError::AlertFailedToVerifySignatures,
                e,
            )),
        }
    }
