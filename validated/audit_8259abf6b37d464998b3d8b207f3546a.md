### Title
`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` Always Revert for Whitelist-Enabled Subscriptions Due to `this.` External Call Bypassing `msg.sender` — (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

The `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` functions in `Scheduler.sol` delegate to `this.getPricesUnsafe(...)` and `this.getEmaPricesUnsafe(...)` via external `this.` calls. Because Solidity external calls reset `msg.sender` to the calling contract's own address, the `onlyWhitelistedReader` modifier on the inner functions sees the Scheduler contract itself as the caller — not the original user. For any subscription with `whitelistEnabled = true`, this causes an unconditional `Unauthorized` revert, making the staleness-checked price getters permanently broken for those subscriptions.

---

### Finding Description

`getPricesNoOlderThan` applies `onlyWhitelistedReader`, performs a staleness check, then delegates:

```solidity
prices = this.getPricesUnsafe(subscriptionId, priceIds);
``` [1](#0-0) 

`getPricesUnsafe` also carries `onlyWhitelistedReader`:

```solidity
function getPricesUnsafe(
    uint256 subscriptionId,
    bytes32[] calldata priceIds
)
    external
    view
    override
    onlyWhitelistedReader(subscriptionId)
``` [2](#0-1) 

The modifier logic is:

```solidity
modifier onlyWhitelistedReader(uint256 subscriptionId) {
    if (_state.subscriptionManager[subscriptionId] == msg.sender) { _; return; }
    if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) { _; return; }
    // linear scan of whitelist ...
    if (!isWhitelisted) revert SchedulerErrors.Unauthorized();
    _;
}
``` [3](#0-2) 

When `getPricesNoOlderThan` calls `this.getPricesUnsafe(...)`, Solidity issues an external call. Inside `getPricesUnsafe`, `msg.sender` is now the **Scheduler contract address**, not the original caller. The Scheduler contract is never the subscription manager and is never added to any reader whitelist. Therefore, for any subscription where `whitelistEnabled = true`, the inner `onlyWhitelistedReader` check unconditionally reverts with `Unauthorized`.

The identical defect exists in `getEmaPricesNoOlderThan`, which calls `this.getEmaPricesUnsafe(...)`: [4](#0-3) 

The root cause is that both `NoOlderThan` functions should call the internal helper `_getPricesInternal` directly instead of routing through the external-facing `this.` call. [5](#0-4) 

---

### Impact Explanation

Any subscription created with `whitelistEnabled = true` loses access to the staleness-checked price getters (`getPricesNoOlderThan`, `getEmaPricesNoOlderThan`). Callers are silently forced to fall back to `getPricesUnsafe` / `getEmaPricesUnsafe`, which carry no recency guarantee. DeFi integrators relying on the `NoOlderThan` API for freshness enforcement will either receive unexpected reverts or, if they catch errors and fall back, consume arbitrarily stale prices — a direct price-feed integrity risk.

---

### Likelihood Explanation

The whitelist feature is a first-class, documented subscription option. Any subscription manager who enables it (e.g., to restrict price-feed access to their own protocol contracts) will trigger this defect on every call to the staleness-checked getters. No special attacker action is required; normal usage of the intended API is sufficient to reproduce the revert.

---

### Recommendation

Replace the `this.` external delegation with a direct call to the internal helper, then extract the relevant price field:

```solidity
function getPricesNoOlderThan(
    uint256 subscriptionId,
    bytes32[] calldata priceIds,
    uint256 age_seconds
)
    external view override onlyWhitelistedReader(subscriptionId)
    returns (PythStructs.Price[] memory prices)
{
    SchedulerStructs.SubscriptionStatus memory status =
        _state.subscriptionStatuses[subscriptionId];
    if (distance(block.timestamp, status.priceLastUpdatedAt) > age_seconds)
        revert PythErrors.StalePrice();

    // Use internal helper directly — avoids resetting msg.sender
    PythStructs.PriceFeed[] memory feeds = _getPricesInternal(subscriptionId, priceIds);
    prices = new PythStructs.Price[](feeds.length);
    for (uint i = 0; i < feeds.length; i++) prices[i] = feeds[i].price;
}
```

Apply the same fix to `getEmaPricesNoOlderThan`.

---

### Proof of Concept

1. Deploy `SchedulerUpgradeable` and create a subscription with `whitelistEnabled = true`, adding address `0xBEEF` to `readerWhitelist`.
2. Fund the subscription and call `updatePriceFeeds` to store a fresh price.
3. As `0xBEEF` (a whitelisted reader), call `getPricesNoOlderThan(subscriptionId, priceIds, 3600)`.
4. **Expected**: returns current prices (staleness check passes, caller is whitelisted).
5. **Actual**: reverts with `SchedulerErrors.Unauthorized` because `this.getPricesUnsafe(...)` sees `msg.sender == address(scheduler)`, which is neither the manager nor in the whitelist.
6. Calling `getPricesUnsafe(subscriptionId, priceIds)` directly as `0xBEEF` succeeds — confirming the whitelist check itself is not the problem, only the `this.` indirection.

    function _getPricesInternal(
        uint256 subscriptionId,
        bytes32[] calldata priceIds
    ) internal view returns (PythStructs.PriceFeed[] memory priceFeeds) {
        if (!_state.subscriptionParams[subscriptionId].isActive) {
            revert SchedulerErrors.InactiveSubscription();
        }

        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        // If no price IDs provided, return all price feeds for the subscription
        if (priceIds.length == 0) {
            PythStructs.PriceFeed[]
                memory allFeeds = new PythStructs.PriceFeed[](
                    params.priceIds.length
                );
            for (uint8 i = 0; i < params.priceIds.length; i++) {
                PythStructs.PriceFeed memory priceFeed = _state.priceUpdates[
                    subscriptionId
                ][params.priceIds[i]];
                // Check if the price feed exists (price ID is valid and has been updated)
                if (priceFeed.id == bytes32(0)) {
                    revert SchedulerErrors.InvalidPriceId(
                        params.priceIds[i],
                        bytes32(0)
                    );
                }
                allFeeds[i] = priceFeed;
            }
            return allFeeds;
        }

        // Return only the requested price feeds
        PythStructs.PriceFeed[]
            memory requestedFeeds = new PythStructs.PriceFeed[](
                priceIds.length
            );
        for (uint8 i = 0; i < priceIds.length; i++) {
            PythStructs.PriceFeed memory priceFeed = _state.priceUpdates[
                subscriptionId
            ][priceIds[i]];

            // Check if the price feed exists (price ID is valid and has been updated)
            if (priceFeed.id == bytes32(0)) {
                revert SchedulerErrors.InvalidPriceId(priceIds[i], bytes32(0));
            }
            requestedFeeds[i] = priceFeed;
        }
        return requestedFeeds;
    }

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
