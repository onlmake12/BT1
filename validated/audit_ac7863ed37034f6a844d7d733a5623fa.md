### Title
Echo Protocol Fee Double-Counted in `executeCallback` — Provider Credited with Already-Accrued Pyth Fee - (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary
In `Echo.sol`, the Echo protocol fee (`_state.pythFeeInWei`) is accrued to `_state.accruedFeesInWei` at request time, but `req.fee` (which equals the full `msg.value` paid by the user, including `_state.pythFeeInWei`) is then used in `executeCallback` to credit the provider. This causes the Echo protocol fee to be double-counted: once to Pyth's accrued balance and once to the provider's accrued balance, making the contract insolvent by `pythFeeInWei` per fulfilled request.

### Finding Description

In `requestPriceUpdatesWithCallback`, the Echo protocol fee is immediately accrued:

```solidity
_state.accruedFeesInWei += _state.pythFeeInWei;
```

The full `msg.value` (= `pythFeeInWei + providerFees`) is stored in `req.fee`. [1](#0-0) 

Later, in `executeCallback`, the provider is credited:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
``` [2](#0-1) 

Here `pythFee` is `pyth.getUpdateFee(updateData)` — the fee paid to the **underlying Pyth oracle contract** — which is a completely different value from `_state.pythFeeInWei` (the Echo protocol fee). [3](#0-2) 

The `getFee` function confirms the total user fee includes `_state.pythFeeInWei`:

```solidity
uint96 baseFee = _state.pythFeeInWei; // Fixed fee to Pyth
...
feeAmount = baseFee + providerBaseFee + providerFeedFee + SafeCast.toUint96(gasFee);
``` [4](#0-3) 

**Accounting breakdown per request:**

| Step | `_state.accruedFeesInWei` | Provider `accruedFeesInWei` | ETH in contract |
|---|---|---|---|
| After request | `+pythFeeInWei` | 0 | `+pythFeeInWei + providerFees` |
| After callback | `pythFeeInWei` | `+pythFeeInWei + providerFees + msg.value_cb - pyth_oracle_fee` | `+msg.value_cb - pyth_oracle_fee` |

Total credited = `2 × pythFeeInWei + providerFees + msg.value_cb - pyth_oracle_fee`
Total ETH held = `pythFeeInWei + providerFees + msg.value_cb - pyth_oracle_fee`

**Shortfall = `pythFeeInWei` per request.**

Compare with the correct pattern in `Entropy.sol`, where the fee split is done atomically and correctly:

```solidity
uint128 providerFee = getProviderFee(provider, callbackGasLimit);
providerInfo.accruedFeesInWei += providerFee;
_state.accruedPythFeesInWei += (SafeCast.toUint128(msg.value) - providerFee);
``` [5](#0-4) 

### Impact Explanation
The contract becomes insolvent by `pythFeeInWei` per fulfilled request. Over time, the sum of `_state.accruedFeesInWei` (admin-withdrawable) plus all providers' `accruedFeesInWei` exceeds the contract's actual ETH balance. Either the admin's `withdrawFees` call will revert due to insufficient balance, or providers' `withdrawAsFeeManager` calls will fail. The degree of insolvency scales linearly with the number of fulfilled requests and the configured `pythFeeInWei`. [6](#0-5) 

### Likelihood Explanation
This is triggered by normal protocol usage. Any user calling `requestPriceUpdatesWithCallback` followed by any provider calling `executeCallback` exercises the bug. No privileged access is required. The entry path is fully permissionless. [7](#0-6) 

### Recommendation
In `executeCallback`, subtract `_state.pythFeeInWei` from `req.fee` before crediting the provider, so the Echo protocol fee is not double-counted:

```solidity
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee - _state.pythFeeInWei + msg.value) - pythFee);
```

Alternatively, store only the provider portion of the fee in `req.fee` at request time (i.e., `req.fee = msg.value - _state.pythFeeInWei`), mirroring the clean split used in `Entropy.sol`.

### Proof of Concept

1. Admin sets `pythFeeInWei = 1000 wei`.
2. User calls `requestPriceUpdatesWithCallback{value: 1000 + providerFees}(...)`.
   - `_state.accruedFeesInWei` becomes `1000`.
   - `req.fee = 1000 + providerFees`.
3. Provider calls `executeCallback{value: pyth_oracle_fee}(...)`.
   - `pythFee = pyth.getUpdateFee(updateData)` (e.g., 100 wei).
   - Provider credited: `(1000 + providerFees + pyth_oracle_fee) - pyth_oracle_fee = 1000 + providerFees`.
4. Contract ETH balance = `providerFees + pyth_oracle_fee - pyth_oracle_fee = providerFees`.
5. Total credited = `1000 (admin) + 1000 + providerFees (provider)`.
6. Admin calls `withdrawFees(1000)` → succeeds (drains `providerFees` ETH).
7. Provider calls `withdrawAsFeeManager(provider, 1000 + providerFees)` → **reverts** (only `providerFees - 1000` ETH remains if `providerFees > 1000`, or admin withdrawal already failed). [8](#0-7)
