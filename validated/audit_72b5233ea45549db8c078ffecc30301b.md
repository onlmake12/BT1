### Title
Index Confusion After Filter in `extractPublisherData` Misattributes Pool State to Wrong Publishers - (`File: governance/pyth_staking_sdk/src/utils/pool.ts`)

### Summary

`extractPublisherData` in the Pyth Staking SDK filters `poolData.publishers` to remove empty (`PublicKey.default`) slots, then uses the **post-filter** array index to access all parallel arrays (`delState`, `selfDelState`, `delegationFees`, `publisherStakeAccounts`, `events[].eventData`). These parallel arrays are indexed by the **original** position in the on-chain `PoolData` account. Whenever any publisher slot before a live publisher is empty (i.e., a publisher was removed), every subsequent publisher's staking state, delegation fees, APY history, and stake account are read from the wrong slot.

### Finding Description

The `PoolData` on-chain account stores publishers and their associated data in parallel fixed-size arrays of 1024 entries, all indexed by the same position:

- `publishers[i]` — publisher pubkey (or `PublicKey.default` if slot is empty)
- `delState[i]` — delegator state for publisher at slot `i`
- `selfDelState[i]` — self-delegation state for publisher at slot `i`
- `delegationFees[i]` — delegation fee for publisher at slot `i`
- `publisherStakeAccounts[i]` — stake account for publisher at slot `i`
- `events[e].eventData[i]` — per-epoch reward data for publisher at slot `i`

`extractPublisherData` does:

```typescript
return poolData.publishers
  .filter((publisher) => !publisher.equals(PublicKey.default))  // removes gaps
  .map((publisher, index) => ({                                  // index = filtered position
    delegationFee: poolData.delegationFees[index] ?? 0n,        // WRONG: uses filtered index
    selfDelegation: poolData.selfDelState[index]?.totalDelegation ?? 0n,
    stakeAccount: poolData.publisherStakeAccounts[index],
    totalDelegation: (poolData.delState[index]?.totalDelegation ?? 0n) + ...
    apyHistory: poolData.events.map(event => ({
      apy: event.eventData[index]?.otherRewardRatio ...          // WRONG: uses filtered index
    }))
  }));
```

If the `publishers` array has any empty slot before a live publisher (e.g., slot 0 is `PublicKey.default`, slot 1 is `PublisherA`), then after filtering, `PublisherA` gets `index = 0` in the filtered array. The code then reads `delState[0]`, `delegationFees[0]`, `publisherStakeAccounts[0]`, etc. — all of which belong to the empty slot, not to `PublisherA`. `PublisherA`'s actual data is at index 1 in the original arrays.

This is the exact same class of bug as the Telcoin `topUp` issue: a loop counter (`index` from `.map`) is used to index into a parallel array, but the loop counter reflects a filtered/compressed position rather than the original position.

### Impact Explanation

1. **Wrong delegation amounts, fees, and APY history displayed** to all users of the staking dashboard and SDK. Users see incorrect pool utilization, incorrect estimated APY, and incorrect delegation fees for publishers whose original slot index differs from their filtered index.

2. **Wrong `publisherStakeAccountPositions` passed to `advanceDelegationRecord` instructions** in `getAdvanceDelegationRecordInstructions`. The `stakeAccount` field comes from `extractPublisherData` and is used directly as an account in the CPI call. Passing the wrong stake account causes the instruction to fail or reference an unintended account, breaking reward claiming for affected users.

3. **Incorrect reward expiry calculation** in `getClaimableRewards`, which relies on `getAdvanceDelegationRecordInstructions` and the `lowestEpoch` derived from the (potentially wrong) publisher data.

### Likelihood Explanation

The bug is triggered whenever any publisher is removed from the pool, leaving a gap in the `publishers` array. Publisher removal is a governance action that has already occurred or will occur in the normal lifecycle of the protocol. Once any gap exists, every publisher whose original slot index is greater than the gap's index is affected. With 100+ active publishers, the probability of at least one gap existing is high over time.

### Recommendation

Track the original index before filtering:

```typescript
return poolData.publishers
  .map((publisher, originalIndex) => ({ publisher, originalIndex }))
  .filter(({ publisher }) => !publisher.equals(PublicKey.default))
  .map(({ publisher, originalIndex }) => ({
    delegationFee: poolData.delegationFees[originalIndex] ?? 0n,
    selfDelegation: poolData.selfDelState[originalIndex]?.totalDelegation ?? 0n,
    selfDelegationDelta: poolData.selfDelState[originalIndex]?.deltaDelegation ?? 0n,
    stakeAccount: poolData.publisherStakeAccounts[originalIndex],
    totalDelegation:
      (poolData.delState[originalIndex]?.totalDelegation ?? 0n) +
      (poolData.selfDelState[originalIndex]?.totalDelegation ?? 0n),
    totalDelegationDelta:
      (poolData.delState[originalIndex]?.deltaDelegation ?? 0n) +
      (poolData.selfDelState[originalIndex]?.deltaDelegation ?? 0n),
    apyHistory: poolData.events
      .filter((event) => event.epoch > 0n)
      .map((event) => ({
        apy: convertEpochYieldToApy(
          (event.y * (event.eventData[originalIndex]?.otherRewardRatio ?? 0n)) /
            FRACTION_PRECISION_N,
        ) * computeDelegatorRewardPercentage(poolData.delegationFees[originalIndex] ?? 0n),
        epoch: event.epoch,
        selfApy: convertEpochYieldToApy(
          (event.y * (event.eventData[originalIndex]?.selfRewardRatio ?? 0n)) /
            FRACTION_PRECISION_N,
        ),
      }))
      .sort((a, b) => Number(a.epoch) - Number(b.epoch)),
    pubkey: publisher,
  }));
```

### Proof of Concept

Given a `PoolData` where:
- `publishers[0]` = `PublicKey.default` (removed publisher)
- `publishers[1]` = `PublisherA`
- `publishers[2]` = `PublisherB`
- `delegationFees[1]` = 500 (PublisherA's fee)
- `delegationFees[2]` = 1000 (PublisherB's fee)

After `.filter(...)`, the array is `[PublisherA, PublisherB]` with filtered indices `[0, 1]`.

The current code reads:
- `PublisherA` → `delegationFees[0]` = 0 (empty slot's fee) ❌ should be `delegationFees[1]` = 500
- `PublisherB` → `delegationFees[1]` = 500 (PublisherA's fee) ❌ should be `delegationFees[2]` = 1000

The same misattribution applies to `delState`, `selfDelState`, `publisherStakeAccounts`, and `events[].eventData` for all publishers after the gap. [1](#0-0) [2](#0-1) [3](#0-2)
