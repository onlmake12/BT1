### Title
Unbounded `tx.gasprice`-Scaled Keeper Fee Drains Subscription Balance — (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

### Summary
`Scheduler._processFeesAndPayKeeper` computes `totalKeeperFee` as `(gasUsed + GAS_OVERHEAD) * tx.gasprice` with no upper bound relative to the subscription's deposited balance. Any unprivileged keeper can call `updatePriceFeeds` with an arbitrarily inflated `tx.gasprice`, extracting the entire subscription balance in a single transaction while netting a guaranteed profit of `GAS_OVERHEAD * tx.gasprice`.

### Finding Description

In `_processFeesAndPayKeeper`:

```solidity
uint256 gasCost = (startGas - gasleft() + GAS_OVERHEAD) * tx.gasprice;
uint256 keeperSpecificFee = uint256(_state.singleUpdateKeeperFeeInWei) * numPriceIds;
uint256 totalKeeperFee = gasCost + keeperSpecificFee;

if (status.balanceInWei < totalKeeperFee) {
    revert SchedulerErrors.InsufficientBalance();
}

status.balanceInWei -= totalKeeperFee;
(bool sent, ) = msg.sender.call{value: totalKeeperFee}("");
```

`tx.gasprice` is fully attacker-controlled. There is no cap on `totalKeeperFee` as a fraction of `status.balanceInWei`. The only guard is the `InsufficientBalance` revert, which passes when the subscription has enough balance — exactly the condition the attacker exploits.

`GAS_OVERHEAD` is a fixed constant of `30000` gas units. [1](#0-0) 

The fee calculation with no upper bound: [2](#0-1) 

The call site in `updatePriceFeeds` that triggers this: [3](#0-2) 

### Impact Explanation

A keeper sets `tx.gasprice` = `subscription_balance / (actual_gas_used + GAS_OVERHEAD)`. The contract deducts `totalKeeperFee ≈ subscription_balance` from the subscriber and pays it to the keeper. The keeper's own network cost is `actual_gas_used * tx.gasprice`, so their net profit is `GAS_OVERHEAD * tx.gasprice > 0`. The subscriber loses their entire deposited balance in one update call, receiving a valid price update in return but at a cost equal to 100% of their balance. Permanent subscriptions can hold up to `MAX_DEPOSIT_LIMIT = 100 ether`, making the maximum extractable value substantial. [4](#0-3) 

### Likelihood Explanation

The entry point `updatePriceFeeds(uint256 subscriptionId, bytes[] calldata updateData)` is permissionless — any address can call it with valid update data. The attacker only needs to:
1. Obtain valid Pyth price update data (publicly available from Hermes).
2. Submit the transaction with a high `maxPriorityFeePerGas` (EIP-1559) or `gasPrice` (legacy), sized to consume the target subscription's balance.

No privileged access, leaked keys, or governance majority is required. The attack is profitable as long as `GAS_OVERHEAD > 0`. [5](#0-4) 

### Recommendation

Cap `totalKeeperFee` to a reasonable fraction of `status.balanceInWei`, or introduce a protocol-level `maxGasPriceInWei` parameter that bounds the `tx.gasprice` used in the fee calculation:

```solidity
uint256 effectiveGasPrice = maxGasPriceInWei > 0
    ? Math.min(tx.gasprice, maxGasPriceInWei)
    : tx.gasprice;
uint256 gasCost = (startGas - gasleft() + GAS_OVERHEAD) * effectiveGasPrice;
```

Alternatively, cap `totalKeeperFee` at a fixed percentage (e.g., 10%) of `status.balanceInWei`.

### Proof of Concept

1. Subscriber creates a subscription with `balanceInWei = B` (e.g., 10 ETH).
2. Attacker fetches valid Pyth update data from Hermes for the subscribed price IDs.
3. Attacker estimates `actual_gas_used ≈ G` for the `updatePriceFeeds` call.
4. Attacker sets `tx.gasprice = B / (G + 30000)`.
5. Attacker calls `updatePriceFeeds(subscriptionId, updateData)`.
6. Inside `_processFeesAndPayKeeper`:
   - `gasCost = (G + 30000) * (B / (G + 30000)) = B`
   - `totalKeeperFee ≈ B` (plus `keeperSpecificFee`, so attacker sets price slightly lower)
   - `status.balanceInWei < totalKeeperFee` → false → no revert
   - `status.balanceInWei -= B` → subscriber balance zeroed
   - Attacker receives `B` wei
7. Attacker's network cost: `G * tx.gasprice ≈ G * B / (G + 30000) < B`
8. Net profit: `30000 * B / (G + 30000) > 0` [6](#0-5)

    function updatePriceFeeds(
        uint256 subscriptionId,
        bytes[] calldata updateData
    ) external override {
        uint256 startGas = gasleft();

        SchedulerStructs.SubscriptionStatus storage status = _state
            .subscriptionStatuses[subscriptionId];
        SchedulerStructs.SubscriptionParams storage params = _state
            .subscriptionParams[subscriptionId];

        if (!params.isActive) {
            revert SchedulerErrors.InactiveSubscription();
        }

        // Get the Pyth contract and parse price updates
        IPyth pyth = IPyth(_state.pyth);
        uint256 pythFee = pyth.getUpdateFee(updateData);

        // If we don't have enough balance, revert
        if (status.balanceInWei < pythFee) {
            revert SchedulerErrors.InsufficientBalance();
        }

        // Parse the price feed updates with an acceptable timestamp range of [0, now+10s].
        // Note: We don't want to reject update data if it contains a price
        // from a market that closed a few days ago, since it will contain a timestamp
        // from the last trading period. Thus, we use a minimum timestamp of zero while parsing,
        // and we enforce the past max validity ourselves in _validateShouldUpdatePrices using
        // the highest timestamp in the update data.
        status.balanceInWei -= pythFee;
        status.totalSpent += pythFee;
        uint64 curTime = SafeCast.toUint64(block.timestamp);
        (
            PythStructs.PriceFeed[] memory priceFeeds,
            uint64[] memory slots
        ) = pyth.parsePriceFeedUpdatesWithConfig{value: pythFee}(
                updateData,
                params.priceIds,
                0, // We enforce the past max validity ourselves in _validateShouldUpdatePrices
                curTime + FUTURE_TIMESTAMP_MAX_VALIDITY_PERIOD,
                false,
                true,
                false
            );

        // Verify all price feeds have the same Pythnet slot.
        // All feeds in a subscription must be updated at the same time.
        uint64 slot = slots[0];
        for (uint8 i = 1; i < slots.length; i++) {
            if (slots[i] != slot) {
                revert SchedulerErrors.PriceSlotMismatch();
            }
        }

        // Verify that update conditions are met, and that the timestamp
        // is more recent than latest stored update's. Reverts if not.
        uint256 latestPublishTime = _validateShouldUpdatePrices(
            subscriptionId,
            params,
            status,
            priceFeeds
        );

        // Update status and store the updates
        status.priceLastUpdatedAt = latestPublishTime;
        status.totalUpdates += priceFeeds.length;

        _storePriceUpdates(subscriptionId, priceFeeds);

        _processFeesAndPayKeeper(status, startGas, params.priceIds.length);

    function _processFeesAndPayKeeper(
        SchedulerStructs.SubscriptionStatus storage status,
        uint256 startGas,
        uint256 numPriceIds
    ) internal {
        // Calculate fee components
        uint256 gasCost = (startGas - gasleft() + GAS_OVERHEAD) * tx.gasprice;
        uint256 keeperSpecificFee = uint256(_state.singleUpdateKeeperFeeInWei) *
            numPriceIds;
        uint256 totalKeeperFee = gasCost + keeperSpecificFee;

        // Check balance
        if (status.balanceInWei < totalKeeperFee) {
            revert SchedulerErrors.InsufficientBalance();
        }

        status.balanceInWei -= totalKeeperFee;
        status.totalSpent += totalKeeperFee;

        // Pay keeper and update status
        (bool sent, ) = msg.sender.call{value: totalKeeperFee}("");
        if (!sent) {
            revert SchedulerErrors.KeeperPaymentFailed();
        }
    }
