### Title
Lazer Signed Price Update Can Be Replayed Unlimited Times Without Replay Protection — (`lazer/contracts/evm/src/PythLazer.sol`, `lazer/contracts/aptos/sources/pyth_lazer.move`)

---

### Summary

The Pyth Lazer verification contracts (`PythLazer.sol` on EVM, `pyth_lazer.move` on Aptos, `pyth_lazer.move` on Sui) accept any valid signed price-update message and verify it, but store **no record of previously used messages**. The same signed update bytes can be submitted an unlimited number of times across separate transactions. Any consumer contract that relies on `verifyUpdate` / `verify_message` / `parse_and_verify_le_ecdsa_update_v2` without independently enforcing timestamp freshness can be fed an arbitrarily stale price that was legitimately signed by a trusted Lazer signer.

---

### Finding Description

```solidity

    bytes calldata update
) external payable returns (bytes calldata payload, address signer) {
    require(msg.value >= verification_fee, "Insufficient fee provided");
    ...
    bytes32 hash = keccak256(payload);
    (signer, , ) = ECDSA.tryRecover(hash, ...);
    if (!isValidSigner(signer)) revert("invalid signer");
    // ← no mapping/set of consumed hashes, no nonce, no timestamp check

```

The contract holds no state tracking which `(hash, signer)` pairs have already been consumed. Any caller can re-submit the identical `update` calldata in a new transaction and receive the same `(payload, signer)` return values.

**Sui — `lazer/contracts/sui/sources/pyth_lazer.move::parse_and_verify_le_ecdsa_update_v2` (lines 87–118)**

The function parses the update, calls `verify_le_ecdsa_message`, and returns an `Update` object. `State` (via `state::trusted_signers`) stores only signer public keys and expiry times — no consumed-update set. The same raw bytes can be passed in a second transaction and will produce a fresh `Update` object.

---

### Impact Explanation

A Lazer price update payload contains a `timestampUs` field. The Lazer contract itself **never checks** this timestamp against `block.timestamp`. Any consumer contract that:

1. Calls `verifyUpdate` / `verify_message` to obtain a "verified" payload, and
2. Does not independently reject payloads whose `timestampUs` is older than an acceptable staleness window,

can be fed a legitimately-signed but arbitrarily old price. An attacker who observed a favorable price update on-chain (e.g., BTC/USD = $50 000 when the current price is $60 000) can replay that update to:

- Manipulate a lending protocol's collateral valuation, enabling under-collateralised borrows or unfair liquidations.
- Execute trades at a stale price on a DEX that uses Lazer as its oracle.
- Drain a protocol's reserves by repeatedly triggering actions gated on the stale price.

The fee per call is `verification_fee = 1 wei` (EVM) / `single_update_fee = 1` nominal APT (Aptos) — economically negligible relative to the profit from a price-manipulation trade.

---

### Likelihood Explanation

- The attacker needs no privileged keys; any EOA can call `verifyUpdate` / `verify_message_with_funder`.
- Signed Lazer updates are broadcast publicly (WebSocket / on-chain calldata), so historical updates are freely available.
- Consumer contracts that omit a staleness check are a realistic and documented failure mode (analogous to Chainlink's well-known "no staleness check" class of bugs).
- The `verification_fee` is set to 1 wei / 1 nominal unit, making repeated replay essentially free.

---

### Recommendation

1. **On-chain timestamp enforcement**: Inside `verifyUpdate` / `verify_message`, parse the `timestampUs` field from the payload and revert if `block.timestamp - (timestampUs / 1e6) > MAX_STALENESS_SECONDS`.
2. **Consumed-hash registry**: Maintain a `mapping(bytes32 => bool) usedHashes` (EVM) or `Table<Bytes32, bool>` (Move) keyed on `keccak256(payload)`. Revert if the hash has already been consumed.
3. **Nonce in payload**: Require the Lazer signer to embed a monotonically increasing nonce or a chain-specific domain separator so that the same price observation cannot produce a reusable signature across chains or time windows.

    function verifyUpdate(
        bytes calldata update
    ) external payable returns (bytes calldata payload, address signer) {
        // Require fee and refund excess
        require(msg.value >= verification_fee, "Insufficient fee provided");
        if (msg.value > verification_fee) {
            payable(msg.sender).transfer(msg.value - verification_fee);
        }

        if (update.length < 71) {
            revert("input too short");
        }
        uint32 EVM_FORMAT_MAGIC = 706910618;

        uint32 evm_magic = uint32(bytes4(update[0:4]));
        if (evm_magic != EVM_FORMAT_MAGIC) {
            revert("invalid evm magic");
        }
        uint16 payload_len = uint16(bytes2(update[69:71]));
        if (update.length < 71 + payload_len) {
            revert("input too short");
        }
        payload = update[71:71 + payload_len];
        bytes32 hash = keccak256(payload);
        (signer, , ) = ECDSA.tryRecover(
            hash,
            uint8(update[68]) + 27,
            bytes32(update[4:36]),
            bytes32(update[36:68])
        );
        if (signer == address(0)) {
            revert("invalid signature");
        }
        if (!isValidSigner(signer)) {
            revert("invalid signer");
        }

    public fun verify_message(
        message: vector<u8>,
        signature: vector<u8>,
        trusted_signer: vector<u8>,
        fee: coin::Coin<AptosCoin>
    ) acquires Storage {
        let storage = borrow_global<Storage>(@pyth_lazer);

        // Verify fee amount
        assert!(coin::value(&fee) >= storage.single_update_fee, EINSUFFICIENT_FEE);

        // Transfer fee to treasury
        coin::deposit(storage.treasury, fee);

        // Verify signer is trusted and not expired
        let i = 0;
        let valid = false;
        while (i < storage.trusted_signers.length()) {
            let signer_info = vector::borrow(&storage.trusted_signers, (i as u64));
            if (&signer_info.pubkey == &trusted_signer
                && signer_info.expires_at > timestamp::now_seconds()) {
                valid = true;
                break
            };
            i = i + 1;
        };
        assert!(valid, EINVALID_SIGNER);

        // Verify signature
        let sig = ed25519::new_signature_from_bytes(signature);
        let pk = ed25519::new_unvalidated_public_key_from_bytes(trusted_signer);
        assert!(
            ed25519::signature_verify_strict(&sig, &pk, message),
            EINVALID_SIGNATURE
        );
    }

public fun parse_and_verify_le_ecdsa_update_v2(s: &State, clock: &Clock, update: vector<u8>): Update {
    let mut cursor = bcs::new(update);

    // Parse and validate message magic
    let magic = cursor.peel_u32();
    assert!(magic == UPDATE_MESSAGE_MAGIC, EInvalidUpdateMagic);

    // Parse signature
    let mut signature = vector::empty<u8>();
    let mut sig_i = 0;
    while (sig_i < SECP256K1_SIG_LEN) {
        signature.push_back(cursor.peel_u8());
        sig_i = sig_i + 1;
    };

    // Parse expected payload length and get remaining bytes as payload
    let payload_len = cursor.peel_u16();
    let payload = cursor.into_remainder_bytes();

    // Validate expected payload length
    assert!(payload_len as u64 == payload.length(), EInvalidPayloadLength);

    // Parse payload
    let mut payload_cursor = bcs::new(payload);
    let payload_magic = payload_cursor.peel_u32();
    assert!(payload_magic == PAYLOAD_MAGIC, EInvalidPayloadMagic);

    // Verify the signature against trusted signers
    verify_le_ecdsa_message(s, clock, &signature, &payload);

    update_v2::parse_from_cursor(payload_cursor)
}
