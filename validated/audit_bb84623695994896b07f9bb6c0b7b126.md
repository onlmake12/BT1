### Title
`createSubscription` Hardcodes `msg.sender` as Manager with No Transfer Mechanism — Funds Permanently Locked When Called via Factory/Intermediary Contract - (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`Scheduler.createSubscription` unconditionally assigns `msg.sender` as the subscription manager. There is no `transferManager` function anywhere in the contract or interface. When any smart contract (factory, router, DeFi protocol) calls `createSubscription` on behalf of a user, the intermediary contract becomes the irrecoverable manager, and the user permanently loses the ability to update or withdraw from the subscription.

---

### Finding Description

In `Scheduler.createSubscription`, the manager is set as:

```solidity
_state.subscriptionManager[subscriptionId] = msg.sender;
``` [1](#0-0) 

All privileged subscription operations gate on `onlyManager`:

```solidity
modifier onlyManager(uint256 subscriptionId) {
    if (_state.subscriptionManager[subscriptionId] != msg.sender) {
        revert SchedulerErrors.Unauthorized();
    }
    _;
}
``` [2](#0-1) 

Both `updateSubscription` and `withdrawFunds` are gated by this modifier: [3](#0-2) [4](#0-3) 

`withdrawFunds` also sends ETH to `msg.sender`, not to a user-specified recipient:

```solidity
(bool sent, ) = msg.sender.call{value: amount}("");
``` [5](#0-4) 

A search across the entire Solidity codebase confirms there is **no** `transferManager`, `setManager`, or `changeManager` function anywhere: [6](#0-5) 

The `IScheduler` interface exposes no manager-transfer primitive, and `SchedulerGovernance` only manages the protocol-level admin, not per-subscription managers: [7](#0-6) 

The manager address is stored in a private mapping with no public getter: [8](#0-7) 

---

### Impact Explanation

When a DeFi protocol, factory contract, or any smart contract intermediary calls `createSubscription` on behalf of a user:

1. The intermediary contract becomes the permanent manager.
2. The actual user cannot call `updateSubscription`, `withdrawFunds`, or deactivate the subscription.
3. The ETH deposited as subscription balance is locked — `withdrawFunds` sends to `msg.sender` (the manager), not to the original depositor.
4. There is no recovery path: no `transferManager` exists anywhere in the codebase.

This results in **permanent user fund lockup** proportional to the subscription balance deposited.

---

### Likelihood Explanation

The Scheduler README explicitly describes the Manager role as "Typically an EOA or a protocol controlled by its governance," directly anticipating smart-contract callers. DeFi protocols routinely wrap subscription creation in factory or router contracts to batch operations, abstract gas, or manage subscriptions on behalf of their users. Any such integration pattern triggers this issue without any malicious intent. The entry path is a standard `createSubscription` call — no special privilege is required.

---

### Recommendation

Add a `manager` parameter to `createSubscription` so callers can designate the intended manager explicitly (analogous to how `EntropyUpgradable.initialize` accepts an explicit `owner` address rather than using `msg.sender`):

```solidity
function createSubscription(
    SchedulerStructs.SubscriptionParams memory subscriptionParams,
    address manager  // <-- add this
) external payable override returns (uint256 subscriptionId) {
    ...
    _state.subscriptionManager[subscriptionId] = manager != address(0) ? manager : msg.sender;
    ...
}
```

Additionally, add a two-step `transferManager` / `acceptManager` function (mirroring the `proposeAdmin` / `acceptAdmin` pattern already present in `SchedulerGovernance`) so existing subscriptions can recover from misconfigured managers.

---

### Proof of Concept

```solidity
// Factory contract that creates subscriptions on behalf of users
contract SubscriptionFactory {
    IScheduler public scheduler;

    constructor(address _scheduler) {
        scheduler = IScheduler(_scheduler);
    }

    // User calls this, sending ETH, expecting to become the manager
    function createForUser(
        SchedulerStructs.SubscriptionParams calldata params
    ) external payable returns (uint256 subId) {
        // msg.sender here is the user, but inside createSubscription,
        // msg.sender is address(this) — the factory
        subId = scheduler.createSubscription{value: msg.value}(params);
        // Factory is now the manager, not the user
    }
    // Factory has no withdrawFunds or updateSubscription forwarding
    // User's ETH is permanently locked
}
```

1. User calls `SubscriptionFactory.createForUser{value: 1 ether}(params)`.
2. Factory calls `scheduler.createSubscription{value: 1 ether}(params)`.
3. Inside `createSubscription`, `msg.sender == address(factory)` → `subscriptionManager[id] = factory`.
4. User calls `scheduler.withdrawFunds(id, 1 ether)` → reverts with `Unauthorized` because `msg.sender (user) != manager (factory)`.
5. Factory has no `withdrawFunds` forwarding → 1 ETH permanently locked. [9](#0-8)

    function createSubscription(
        SchedulerStructs.SubscriptionParams memory subscriptionParams
    ) external payable override returns (uint256 subscriptionId) {
        _validateSubscriptionParams(subscriptionParams);

        // Calculate minimum balance required for this subscription
        uint256 minimumBalance = this.getMinimumBalance(
            uint8(subscriptionParams.priceIds.length)
        );

        // Ensure enough funds were provided
        if (msg.value < minimumBalance) {
            revert SchedulerErrors.InsufficientBalance();
        }

        // Check deposit limit for permanent subscriptions
        if (subscriptionParams.isPermanent && msg.value > MAX_DEPOSIT_LIMIT) {
            revert SchedulerErrors.MaxDepositLimitExceeded();
        }

        // Set subscription to active
        subscriptionParams.isActive = true;

        subscriptionId = _state.subscriptionNumber++;

        // Store the subscription parameters
        _state.subscriptionParams[subscriptionId] = subscriptionParams;

        // Initialize subscription status
        SchedulerStructs.SubscriptionStatus storage status = _state
            .subscriptionStatuses[subscriptionId];
        status.priceLastUpdatedAt = 0;
        status.balanceInWei = msg.value;
        status.totalUpdates = 0;
        status.totalSpent = 0;

        // Map subscription ID to manager
        _state.subscriptionManager[subscriptionId] = msg.sender;

        _addToActiveSubscriptions(subscriptionId);

        emit SubscriptionCreated(subscriptionId, msg.sender);
        return subscriptionId;
    }

abstract contract SchedulerGovernance is SchedulerState {
    event NewAdminProposed(address oldAdmin, address newAdmin);
    event NewAdminAccepted(address oldAdmin, address newAdmin);
    event SingleUpdateKeeperFeeSet(uint oldFee, uint newFee);
    event MinimumBalancePerFeedSet(uint oldBalance, uint newBalance);

    /**
     * @dev Returns the address of the proposed admin.
     */
    function proposedAdmin() public view virtual returns (address) {
        return _state.proposedAdmin;
    }

    /**
     * @dev Returns the address of the current admin.
     */
    function getAdmin() external view returns (address) {
        return _state.admin;
    }

    /**
     * @dev Proposes a new admin for the contract. Replaces the proposed admin if there is one.
     * Can only be called by either admin or owner.
     */
    function proposeAdmin(address newAdmin) public virtual {
        require(newAdmin != address(0), "newAdmin is zero address");

        _authorizeAdminAction();

        _state.proposedAdmin = newAdmin;
        emit NewAdminProposed(_state.admin, newAdmin);
    }

    /**
     * @dev The proposed admin accepts the admin transfer.
     */
    function acceptAdmin() external {
        if (msg.sender != _state.proposedAdmin)
            revert SchedulerErrors.Unauthorized();

        address oldAdmin = _state.admin;
        _state.admin = msg.sender;

        _state.proposedAdmin = address(0);
        emit NewAdminAccepted(oldAdmin, msg.sender);
    }
