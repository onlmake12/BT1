### Title
Missing Per-Peer Rate Limiting on `GetBlocksProof` Allows Single-Peer Amplification DoS — (`util/light-client-protocol-server/src/lib.rs`, `util/light-client-protocol-server/src/components/get_blocks_proof.rs`)

---

### Summary

`LightClientProtocol::try_process()` dispatches `GetBlocksProof` messages with no rate limiting. A single unauthenticated peer can send a stream of max-size (1000-hash) `GetBlocksProof` messages, each triggering ~3000 RocksDB reads plus an MMR proof generation over 1000 positions, with no ban and no throttle.

---

### Finding Description

`GET_BLOCKS_PROOF_LIMIT` is set to 1000. [1](#0-0) 

`GetBlocksProofProcess::execute()` enforces only an upper-bound size check (≤ 1000), then for every hash in the `found` partition performs three sequential RocksDB reads (`get_block_header`, `get_block_uncles`, `get_block_extension`), followed by `reply_proof()` which calls `mmr.gen_proof(1000 positions)`. [2](#0-1) [3](#0-2) 

A successful execution returns `Status::ok()` (code 200). The ban logic in `should_ban()` only fires for 4xx codes, so a valid max-size request is never penalized. [4](#0-3) 

`LightClientProtocol::try_process()` contains **no rate limiter** of any kind — no `governor::RateLimiter`, no per-peer counter, no token bucket. [5](#0-4) 

This is in direct contrast to `Relayer::try_process()`, which gates every non-PoW message through a `governor::RateLimiter` keyed by `(PeerIndex, message_item_id)` before dispatching. [6](#0-5) 

---

### Impact Explanation

Each max-size `GetBlocksProof` message from a single peer forces the server to execute:
- 1000× `snapshot.is_main_chain()` (line 74)
- 1000× `snapshot.get_block_header()` (line 83)
- 1000× `snapshot.get_block_uncles()` (line 89)
- 1000× `snapshot.get_block_extension()` (line 91)
- 1× `mmr.gen_proof(1000 positions)` (line 210)

A single TCP connection can sustain this indefinitely. The server issues no ban, no warning, and no disconnect for well-formed requests. This allows one unauthenticated peer to monopolize the light-client server's RocksDB I/O and CPU, degrading or denying service to all legitimate light clients.

---

### Likelihood Explanation

The attack requires only a TCP connection to a node with the light client protocol enabled. No PoW, no key, no stake. The attacker needs only a list of 1000 valid main-chain block hashes (trivially obtained from any block explorer or by syncing headers) and a valid `last_hash`. The message is structurally valid, so no ban is ever issued. The attack is repeatable at will from a single peer.

---

### Recommendation

Add a `governor::RateLimiter<(PeerIndex, u32), ...>` to `LightClientProtocol` (mirroring the pattern in `Relayer::new()`), and check it at the top of `try_process()` before dispatching any handler. A per-peer, per-message-type quota of 1–2 requests/second is sufficient to eliminate the amplification while preserving legitimate light-client usage. [7](#0-6) 

---

### Proof of Concept

1. Start a CKB node with the light client protocol server enabled and a chain of height ≥ 1000.
2. Collect 1000 distinct main-chain block hashes (e.g., blocks 1–1000) and the tip hash as `last_hash`.
3. Open a single P2P connection and send a tight loop of `GetBlocksProof` messages, each containing all 1000 hashes.
4. Observe: server CPU and RocksDB I/O saturate; no ban is issued; the peer remains connected; legitimate light-client peers receive degraded or no responses.

    async fn try_process(
        &mut self,
        nc: &Arc<dyn CKBProtocolContext + Sync>,
        peer_index: PeerIndex,
        message: packed::LightClientMessageUnionReader<'_>,
    ) -> Status {
        match message {
            packed::LightClientMessageUnionReader::GetLastState(reader) => {
                components::GetLastStateProcess::new(reader, self, peer_index, nc)
                    .execute()
                    .await
            }
            packed::LightClientMessageUnionReader::GetLastStateProof(reader) => {
                components::GetLastStateProofProcess::new(reader, self, peer_index, nc)
                    .execute()
                    .await
            }

                components::GetBlocksProofProcess::new(reader, self, peer_index, nc)
                    .execute()
                    .await
            }
            packed::LightClientMessageUnionReader::GetTransactionsProof(reader) => {
                components::GetTransactionsProofProcess::new(reader, self, peer_index, nc)
                    .execute()
                    .await
            }
            _ => StatusCode::UnexpectedProtocolMessage.into(),
        }
    }
