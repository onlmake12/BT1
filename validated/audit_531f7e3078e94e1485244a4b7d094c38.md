### Title
Admin Transfer Cancellation Can Be Front-Run — (`File: target_chains/ethereum/contracts/contracts/entropy/EntropyGovernance.sol`, `target_chains/ethereum/contracts/contracts/pulse/SchedulerGovernance.sol`)

---

### Summary

Both `EntropyGovernance` and `SchedulerGovernance` implement a two-step admin transfer via `proposeAdmin` / `acceptAdmin`. There is no explicit cancel function, and `proposeAdmin` forbids `address(0)` as input. The only way to revoke a pending proposal is to overwrite it with a new `proposeAdmin` call. A previously-proposed admin who is no longer trusted can monitor the mempool and front-run that overwrite transaction with `acceptAdmin()` at a higher gas price, permanently seizing the admin role before the cancellation lands.

---

### Finding Description

`EntropyGovernance.proposeAdmin` stores the candidate in `_state.proposedAdmin` and explicitly blocks cancellation via zero-address:

```solidity
// EntropyGovernance.sol line 34
require(newAdmin != address(0), "newAdmin is zero address");
_state.proposedAdmin = newAdmin;
```

`acceptAdmin` completes the transfer for whoever is currently stored as `_state.proposedAdmin`:

```solidity
// EntropyGovernance.sol lines 45-53
function acceptAdmin() external {
    if (msg.sender != _state.proposedAdmin)
        revert EntropyErrors.Unauthorized();
    address oldAdmin = _state.admin;
    _state.admin = msg.sender;
    _state.proposedAdmin = address(0);
    emit NewAdminAccepted(oldAdmin, msg.sender);
}
```

The identical pattern exists in `SchedulerGovernance.sol` lines 34–54.

**Attack sequence:**

1. Admin/owner calls `proposeAdmin(attackerAddress)` (e.g., a key that was later compromised, or a mistaken address).
2. Admin/owner decides to revoke the proposal and broadcasts `proposeAdmin(trustedAddress)` to overwrite it.
3. The attacker observes the pending overwrite in the public mempool and immediately submits `acceptAdmin()` with a higher gas price.
4. The attacker's `acceptAdmin()` is mined first; they become the permanent admin. The owner's overwrite transaction then either reverts (if the attacker already cleared `proposedAdmin`) or sets a new proposal that is irrelevant — the damage is done.

---

### Impact Explanation

Once the attacker holds the `admin` role in `EntropyGovernance`, they can:

- Call `withdrawFee(attackerAddress, accruedPythFeesInWei)` to drain all accumulated protocol fees to an arbitrary address (lines 103–116).
- Call `setPythFee` to set an arbitrarily high fee, breaking Entropy for all users.
- Call `setDefaultProvider` to redirect new entropy requests to a malicious provider.

In `SchedulerGovernance`, the attacker can manipulate `singleUpdateKeeperFeeInWei` and `minimumBalancePerFeed`, disrupting the Pulse scheduler economics.

The `withdrawFee` path represents a direct, irreversible loss of accumulated protocol funds.

---

### Likelihood Explanation

- EVM chains have a public mempool; front-running is a well-known and actively practiced technique.
- The scenario requires the admin/owner to have previously proposed the attacker's address — plausible via key compromise, social engineering, or operational error.
- The attacker only needs to watch for a `proposeAdmin` transaction targeting their address and be ready to call `acceptAdmin` if a subsequent overwrite is broadcast.
- No special infrastructure is needed; standard MEV bots or a simple mempool watcher suffice.

---

### Recommendation

Add an explicit cancel function that sets `_state.proposedAdmin = address(0)` without the zero-address guard, callable only by the current admin or owner:

```solidity
function cancelAdminProposal() external {
    _authoriseAdminAction();
    _state.proposedAdmin = address(0);
}
```

Alternatively, remove the `require(newAdmin != address(0), ...)` guard from `proposeAdmin` so that calling `proposeAdmin(address(0))` acts as a cancellation. Apply the same fix to `SchedulerGovernance`.

---

### Proof of Concept

```
State: _state.proposedAdmin = ATTACKER, _state.admin = CURRENT_ADMIN

Block N (mempool):
  Tx A (owner): proposeAdmin(TRUSTED_ADDRESS)  ← broadcast to cancel attacker

  Attacker sees Tx A in mempool, submits:
  Tx B (attacker): acceptAdmin()  ← higher gas price

Block N+1 (mined order):
  Tx B mines first → _state.admin = ATTACKER, _state.proposedAdmin = address(0)
  Tx A mines second → _state.proposedAdmin = TRUSTED_ADDRESS  (admin already stolen)

Result: ATTACKER is now admin and can call withdrawFee(ATTACKER, totalAccruedFees)
```
