### Title
Lack of Rate Limiting and Authentication on Governance Proposer Server Endpoint — (File: `governance/xc_admin/packages/proposer_server/src/index.ts`)

---

### Summary

The `proposer_server` is an Express.js HTTP server that acts as a relay between the `xc_admin` frontend and the Pyth governance Squads multisig. It exposes a `POST /api/propose` endpoint with no rate limiting, no authentication, and fully open CORS. The server holds a privileged keypair loaded from disk and uses it to submit governance proposals on-chain. Any attacker who can reach the server can trigger unlimited on-chain proposal submissions, draining the server's SOL balance and flooding the multisig with spam proposals.

---

### Finding Description

The entire server implementation is contained in a single file:

```
governance/xc_admin/packages/proposer_server/src/index.ts
```

The server is initialized with:

```typescript
app.use(cors());
app.use(express.json({ limit: "50mb" }));
app.use(express.urlencoded({ extended: true, limit: "50mb" }));
```

No authentication middleware, no IP allowlist, and no rate-limiting middleware is applied anywhere. The single route handler is:

```typescript
app.post("/api/propose", async (req: Request, res: Response) => {
  ...
  const wallet = new NodeWallet(KEYPAIR);
  ...
  await vault.proposeInstructions(instructions, cluster, {
    computeUnitPriceMicroLamports: COMPUTE_UNIT_PRICE_MICROLAMPORTS!,
  });
  ...
});
```

The `KEYPAIR` is the server's own private key, loaded at startup from disk. Every call to this endpoint causes the server to sign and broadcast one or more Solana transactions to the Squads multisig program, paying transaction fees from the server's wallet.

The `cors()` call with no options defaults to `Access-Control-Allow-Origin: *`, meaning any browser origin can POST to this endpoint. The server description in `package.json` confirms its role: `"A server that proposes the instructions that it receives to the multisig"`. [1](#0-0) [2](#0-1) 

---

### Impact Explanation

An attacker who can reach the server (which is plausible given the open CORS and the fact that the frontend calls it from a browser) can:

1. **Drain the server's SOL balance** — each `proposeInstructions` call submits one or more Solana transactions, each paying transaction fees from the server's keypair. Flooding the endpoint exhausts the wallet.
2. **Spam the Squads multisig with garbage proposals** — the multisig's `transactionIndex` is incremented for every proposal created. Thousands of spam proposals pollute the proposal queue, making legitimate governance operations harder to track and execute.
3. **Cause DoS of the governance workflow** — once the server's SOL balance is depleted, legitimate governance proposals from the `xc_admin` frontend can no longer be submitted until the wallet is manually refunded.

The multisig still requires approval from a threshold of signers, so no unauthorized governance action can be *executed*. However, the availability and integrity of the governance submission pipeline is directly impacted. [3](#0-2) 

---

### Likelihood Explanation

The `xc_admin` frontend calls this server via `axios.post(proposerServerUrl + "/api/propose", ...)`, where `proposerServerUrl` is a configured URL. The open `cors()` with no origin restriction and the absence of any network-level access control in the code itself indicates the server is designed to be reachable from a browser. If the server is deployed on a publicly routable address (which is the operational requirement for the frontend to work), any attacker can discover and abuse the endpoint. No credentials, tokens, or prior knowledge beyond the server URL are required. [4](#0-3) 

---

### Recommendation

1. **Add IP allowlisting** — restrict the `/api/propose` endpoint to known frontend server IPs or an internal network only.
2. **Add a shared secret / bearer token** — require a pre-shared API key in the `Authorization` header; the frontend already knows the server URL and can include the token.
3. **Add rate limiting** — apply a middleware such as `express-rate-limit` to cap requests per IP per time window, analogous to the fix applied in the referenced SSP Relay remediation.
4. **Remove the wildcard CORS** — replace `cors()` with `cors({ origin: "<known-frontend-origin>" })`.

### Proof of Concept

```bash
# Flood the endpoint with arbitrary proposal payloads
for i in $(seq 1 500); do
  curl -s -X POST http://<proposer-server-host>:4000/api/propose \
    -H "Content-Type: application/json" \
    -d '{
      "cluster": "mainnet-beta",
      "instructions": [{
        "programId": "11111111111111111111111111111111",
        "keys": [],
        "data": [0,0,0,0]
      }]
    }' &
done
wait
```

Each successful request causes the server to sign and submit a Solana transaction from its own keypair, incrementing the multisig proposal counter and paying fees. After enough requests the server wallet is drained and legitimate governance proposals fail with insufficient funds errors. [5](#0-4)

```typescript
import NodeWallet from "@coral-xyz/anchor/dist/cjs/nodewallet";
import type { PythCluster } from "@pythnetwork/client";
import { getPythClusterApiUrl } from "@pythnetwork/client";
import {
  envOrErr,
  getMultisigCluster,
  MultisigVault,
  PRICE_FEED_MULTISIG,
} from "@pythnetwork/xc-admin-common";
import type { Cluster } from "@solana/web3.js";
import {
  Connection,
  Keypair,
  PublicKey,
  TransactionInstruction,
} from "@solana/web3.js";
import SquadsMesh from "@sqds/mesh";
import cors from "cors";
import express, { type Request, type Response } from "express";
import * as fs from "fs";

const PORT: number = Number(process.env.PORT ?? "4000");
const KEYPAIR: Keypair = Keypair.fromSecretKey(
  Uint8Array.from(JSON.parse(fs.readFileSync(envOrErr("WALLET"), "ascii"))),
);
const MAINNET_RPC: string =
  process.env.MAINNET_RPC ?? getPythClusterApiUrl("mainnet-beta");
const DEVNET_RPC: string =
  process.env.DEVNET_RPC ?? getPythClusterApiUrl("devnet");
const TESTNET_RPC: string =
  process.env.TESTNET_RPC ?? getPythClusterApiUrl("testnet");
const LOCALNET_RPC: string =
  process.env.LOCALNET_RPC ?? getPythClusterApiUrl("localnet");

const RPC_URLS: Record<Cluster | "localnet", string> = {
  devnet: DEVNET_RPC,
  localnet: LOCALNET_RPC,
  "mainnet-beta": MAINNET_RPC,
  testnet: TESTNET_RPC,
};

const COMPUTE_UNIT_PRICE_MICROLAMPORTS: number | undefined = process.env
  .COMPUTE_UNIT_PRICE_MICROLAMPORTS
  ? Number(process.env.COMPUTE_UNIT_PRICE_MICROLAMPORTS)
  : undefined;

const app = express();

app.use(cors());
app.use(express.json({ limit: "50mb" }));
app.use(express.urlencoded({ extended: true, limit: "50mb" }));

app.post("/api/propose", async (req: Request, res: Response) => {
  try {
    const instructions: TransactionInstruction[] = req.body.instructions.map(
      (ix: any) =>
        new TransactionInstruction({
          data: Buffer.from(ix.data),
          keys: ix.keys.map((key: any) => {
            return {
              isSigner: key.isSigner,
              isWritable: key.isWritable,
              pubkey: new PublicKey(key.pubkey),
            };
          }),
          programId: new PublicKey(ix.programId),
        }),
    );

    const cluster: PythCluster = req.body.cluster;

    const wallet = new NodeWallet(KEYPAIR);
    const proposeSquads: SquadsMesh = new SquadsMesh({
      connection: new Connection(RPC_URLS[getMultisigCluster(cluster)]),
      wallet,
    });

    const vault = new MultisigVault(
      wallet,
      getMultisigCluster(cluster),
      proposeSquads,
      PRICE_FEED_MULTISIG[getMultisigCluster(cluster)],
    );

    // preserve the existing API by returning only the first pubkey
    const proposalPubkey = (
      await vault.proposeInstructions(instructions, cluster, {
        computeUnitPriceMicroLamports: COMPUTE_UNIT_PRICE_MICROLAMPORTS!,
      })
    )[0];
    res.status(200).json({ proposalPubkey: proposalPubkey });
  } catch (error) {
    if (error instanceof Error) {
      res.status(500).json(error.message);
    } else {
      res.status(500).json("An unknown error occurred");
    }
  }
});

app.listen(PORT);
```
