### Title
Peer Store Permanently Locked by Single-Group Address Flood: `len/2` Integer Truncation in `check_purge` Network-Group Fallback — (`network/src/peer_store/peer_store_impl.rs`)

---

### Summary

`PeerStore::check_purge` contains an integer-division bug: when all stored addresses belong to a single network group, `len / 2 = 1 / 2 = 0`, so `.take(0)` selects zero groups, zero addresses are evicted, and the function returns `Err(PeerStoreError::EvictionFailed)`. An unprivileged attacker who sends ~6 P2P discovery `Nodes` messages containing 16 384 addresses from a single `/16` subnet permanently prevents the victim node from adding any new peer addresses, disabling peer discovery for up to 7 days.

---

### Finding Description

**Root cause — integer truncation in the network-group fallback:** [1](#0-0) 

```
let len = peers_by_network_group.len();   // = 1 when all addrs share one /16
...
peers.into_iter()
    .take(len / 2)          // take(0) — integer division truncates 1/2 → 0
    .flat_map(|addrs| {
        if addrs.len() > 4 { Some(...choose 2...) } else { None }
    })
    .flatten()
    .collect()               // always empty
```

When `candidate_peers` is empty the function returns: [2](#0-1) 

**Network group definition** — IPv4 group key is the first two octets (`/16`): [3](#0-2) 

So every address in `1.2.0.0/16` maps to `Group::IP4([1, 2])` — one group.

**Step 1 of `check_purge` does not help** — freshly injected addresses have `last_connected_at_ms = 0` and `attempts_count = 0`. With `ADDR_MAX_RETRIES = 3`, `is_connectable` returns `true` for all of them: [4](#0-3) 

So step 1 evicts nothing, step 2 evicts nothing, and `check_purge` returns `Err`.

**`add_addr` propagates the error:** [5](#0-4) 

**Callers silently swallow the error** — discovery protocol: [6](#0-5) 

Identify protocol: [7](#0-6) 

Both callers log and discard the error, so the node silently stops learning new peers.

**Attack capacity** — `MAX_ADDR_TO_SEND = 1000` items × `MAX_ADDRS = 3` addresses = 3 000 addrs per `Nodes` message. `ADDR_COUNT_LIMIT = 16384`: [8](#0-7) 

⌈16 384 / 3 000⌉ = **6 messages** from a single connection suffice to fill the store.

---

### Impact Explanation

Once the store is saturated with single-group addresses:
- Every subsequent `add_addr` call returns `Err(EvictionFailed)`.
- The node cannot record any newly discovered peer addresses via the discovery or identify protocols.
- Peer discovery is effectively disabled until the injected addresses age out (`ADDR_TIMEOUT_MS = 7 days`) or exhaust their retry budget (`ADDR_MAX_RETRIES = 3` failed dial attempts).
- This is a prerequisite for an eclipse attack: the victim's outbound connection pool can only draw from the attacker-controlled addresses already in the store.

---

### Likelihood Explanation

- Requires only a single inbound P2P connection and ~6 `Nodes` messages — no PoW, no stake, no privileged role.
- No per-session rate limit exists on `add_new_addrs`.
- `is_valid_addr` only rejects loopback/private addresses; it does not limit per-subnet concentration.
- The condition is permanent until natural address expiry, so a one-time burst is sufficient.

---

### Recommendation

1. **Fix the off-by-one**: replace `take(len / 2)` with `take(len.saturating_add(1) / 2)` (ceiling division) so a single group is still selected when `len == 1`.
2. **Cap per-subnet concentration at insertion time**: reject or replace an incoming address when its network group already holds more than `N` entries (e.g., `N = ADDR_COUNT_LIMIT / 32`).
3. **Rate-limit per-session address additions** in `add_new_addrs` to bound how quickly any single peer can fill the store.

---

### Proof of Concept

    let addr: Multiaddr = format!("/ip4/{}/tcp/8115", ip).parse().unwrap();

    peer_store.mut_addr_manager()

// Any add_addr call must go through check_purge:

assert!(result.is_err()); // Err(EvictionFailed) — peer discovery permanently blocked

    pub fn is_connectable(&self, now_ms: u64) -> bool {
        // do not remove addr tried in last minute
        if self.tried_in_last_minute(now_ms) {
            return true;
        }
        // we give up if never connect to this addr
        if self.last_connected_at_ms == 0 && self.attempts_count >= ADDR_MAX_RETRIES {
            return false;
        }
        // consider addr is not connectable if failed too many times

            && (self.attempts_count >= ADDR_MAX_FAILURES)
        {
            return false;
        }
        true
    }
