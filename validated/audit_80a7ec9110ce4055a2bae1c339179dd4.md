### Title
TOCTOU in Async Tx-Pool Verification Pipeline Allows Pool Pollution via RBF Race - (File: tx-pool/src/process.rs)

### Summary

The CKB tx-pool's multi-stage async verification pipeline has a time-of-check/time-of-use gap analogous to the external report's callback-based reentrancy. Between `pre_check` (read lock) and `submit_entry` (write lock), script verification runs with no lock held. When the chain tip has not changed, `check_rtx` — which re-validates cell liveness against the **current** pool state — is unconditionally skipped. An unprivileged submitter can exploit this window via RBF to inject a transaction whose input no longer exists in the pool or snapshot, permanently polluting the pool.

---

### Finding Description

`_process_tx` in `tx-pool/src/process.rs` follows a three-phase pipeline:

**Phase 1 — `pre_check` (read lock)** [1](#0-0) 

The transaction is resolved against the current pool overlay and snapshot. The snapshot's `tip_hash` is captured as `pre_resolve_tip`. The read lock is then released.

**Phase 2 — `verify_rtx` (no lock)** [2](#0-1) 

Script execution runs without any pool lock. For complex scripts this window can span the full `max_tx_verify_cycles` budget (70 M cycles by default). During this window the pool is fully writable by other tasks.

**Phase 3 — `submit_entry` (write lock)** [3](#0-2) 

The critical conditional:

```rust
if pre_resolve_tip != tip_hash {
    status = check_rtx(tx_pool, &snapshot, &entry.rtx)?;
    ...
    time_relative_verify(snapshot, Arc::clone(&entry.rtx), tx_env)?;
}
```

`check_rtx` — which calls `check_rtx_from_pool` and re-validates every resolved input against the **live pool state** — is executed **only when the tip has changed**. [4](#0-3) 

When the tip is unchanged, the only guard that runs is `find_conflict_outpoint` (RBF disabled) or `check_rbf` (RBF enabled):

```rust
let conflicted_outpoint =
    tx_pool.pool_map.find_conflict_outpoint(entry.transaction());
``` [5](#0-4) 

`find_conflict_outpoint` only checks whether any input of the new transaction is already registered in `edges.inputs` (i.e., claimed by another pool tx). It does **not** verify that the resolved inputs still exist in the pool or snapshot. [6](#0-5) 

`check_rbf` returns early with `Ok(HashSet::new())` when there are no conflicting pool transactions, which is the case after the parent has been RBF-evicted: [7](#0-6) 

`check_rtx_from_pool` — the function that would catch the stale input — uses `OverlayCellChecker` to verify liveness against both the pool and the snapshot: [8](#0-7) 

`PoolCell::cell` returns `CellStatus::Unknown` for any outpoint not present as a pool output, which `check_rtx_from_pool` would surface as `OutPointError::Unknown`: [9](#0-8) 

Because this check is bypassed when the tip is unchanged, a transaction whose pool-only input was evicted by RBF is silently admitted.

---

### Impact Explanation

A transaction admitted with an unresolvable input (`CellStatus::Unknown` for both pool and snapshot) can never be included in a block. Every call to `resolve_tx_from_pool` during block assembly will fail for it. The transaction occupies pool space indefinitely until the pool's size-eviction mechanism removes it. An attacker who repeats this pattern can fill the pool with permanently unmined entries, crowding out legitimate transactions and degrading miner revenue and user experience. This is a targeted DoS against the tx-pool, reachable by any unprivileged transaction submitter.

---

### Likelihood Explanation

The attack requires:
1. Submitting `tx_Y` (creates output `X`) — accepted into the pool.
2. Submitting `tx_A` (spends `X` from `tx_Y`'s pool output) — enters the verify queue.
3. Submitting `tx_Z` (RBF replacement of `tx_Y`, higher fee) while `tx_A` is in Phase 2.

The verification window is bounded by `max_tx_verify_cycles` (default 70 M cycles). An attacker can craft `tx_A`'s lock script to consume close to this limit, maximising the window. The cost is the RBF fee premium for `tx_Z`. Multiple workers run concurrently: [10](#0-9) 

This makes the race reliably exploitable with modest effort. The orphan pool also allows two transactions spending the same unknown input simultaneously, providing an additional staging area: [11](#0-10) 

---

### Recommendation

`check_rtx_from_pool` should be called unconditionally inside `submit_entry`, not only when the tip changes. The tip-change guard is correct for `time_relative_verify` (which depends on block height), but pool-state liveness must be re-validated on every submission because the pool can change independently of the tip (via RBF). A minimal fix:

```rust
// Always re-check pool cell liveness, regardless of tip change
status = check_rtx(tx_pool, &snapshot, &entry.rtx)?;

// Only redo time-relative verification when the tip changed
if pre_resolve_tip != tip_hash {
    let tip_header = snapshot.tip_header();
    let tx_env = status.with_env(tip_header);
    time_relative_verify(snapshot, Arc::clone(&entry.rtx), tx_env)?;
}
```

---

### Proof of Concept

```
1. Attacker submits tx_Y  →  pool accepts it; output X is live in pool_map.
2. Attacker submits tx_A  →  pre_check resolves X from pool (tx_Y's output),
                              captures tip_hash = H; tx_A enters verify queue.
3. [Phase 2 begins for tx_A — no lock held]
4. Attacker submits tx_Z  →  RBF evicts tx_Y; X is removed from pool_map.
5. [Phase 2 completes — scripts pass against old snapshot]
6. submit_entry(tx_A, pre_resolve_tip=H):
     - tip_hash == H  →  check_rtx SKIPPED
     - find_conflict_outpoint(tx_A): X not in edges.inputs  →  no conflict
     - tx_A inserted into pool with input X (non-existent in pool or snapshot)
7. Block assembly: resolve_tx_from_pool(tx_A) → OutPointError::Unknown(X)
   tx_A is never mined; pool slot permanently wasted.
```

The entry path is fully attacker-controlled via the standard `send_transaction` RPC or P2P relay, requiring no privileged access.

        entry: TxEntry,

                // check_rbf must be invoked in `write` lock to avoid concurrent issues.
                let conflicts = if tx_pool.enable_rbf() {
                    tx_pool.check_rbf(&snapshot, &entry)?
                } else {
                    // RBF is disabled but we found conflicts, return error here
                    // after_process will put this tx into conflicts_pool
                    let conflicted_outpoint =
                        tx_pool.pool_map.find_conflict_outpoint(entry.transaction());
                    if let Some(outpoint) = conflicted_outpoint {
                        return Err(Reject::Resolve(OutPointError::Dead(outpoint)));
                    }
                    HashSet::new()

                // if snapshot changed by context switch we need redo time_relative verify
                let tip_hash = snapshot.tip_hash();

                        pre_resolve_tip,
                        tip_hash

                    let tip_header = snapshot.tip_header();
                    let tx_env = status.with_env(tip_header);
                    time_relative_verify(snapshot, Arc::clone(&entry.rtx), tx_env)?;
                }

    pub(crate) async fn pre_check(
        &self,
        tx: &TransactionView,
    ) -> (Result<PreCheckedTx, Reject>, Arc<Snapshot>) {
        // Acquire read lock for cheap check
        let tx_size = tx.data().serialized_size_in_block();

        let (ret, snapshot) = self
            .with_tx_pool_read_lock(|tx_pool, snapshot| {
                let tip_hash = snapshot.tip_hash();

                // Same txid means exactly the same transaction, including inputs, outputs, witnesses, etc.
                // It's also not possible for RBF, reject it directly
                check_txid_collision(tx_pool, tx)?;

                // Try normal path first, if double-spending check success we don't need RBF check
                // this make sure RBF won't introduce extra performance cost for hot path
                let res = resolve_tx(tx_pool, &snapshot, tx.clone(), false);
                match res {
                    Ok((rtx, status)) => {
                        let fee = check_tx_fee(tx_pool, &snapshot, &rtx, tx_size)?;
                        Ok((tip_hash, rtx, status, fee, tx_size))
                    }
                    Err(Reject::Resolve(OutPointError::Dead(out))) => {
                        let (rtx, status) = resolve_tx(tx_pool, &snapshot, tx.clone(), true)?;
                        let fee = check_tx_fee(tx_pool, &snapshot, &rtx, tx_size)?;
                        let conflicts = tx_pool.pool_map.find_conflict_outpoint(tx);
                        if conflicts.is_none() {
                            // this mean one input's outpoint is dead, but there is no direct conflicted tx in tx_pool
                            // we should reject it directly and don't need to put it into conflicts pool
                            error!(
                                "{} is resolved as Dead, but there is no conflicted tx",
                                rtx.transaction.proposal_short_id()
                            );
                            return Err(Reject::Resolve(OutPointError::Dead(out)));
                        }
                        // we also return Ok here, so that the entry will be continue to be verified before submit
                        // we only want to put it into conflicts pool after the verification stage passed
                        // then we will double-check conflicts txs in `submit_entry`

                        Ok((tip_hash, rtx, status, fee, tx_size))
                    }
                    Err(err) => Err(err),
                }
            })
            .await;
        (ret, snapshot)
    }

    async fn run(mut self) {
        let queue_ready = self.tasks.read().await.subscribe();
        self.refresh_status();
        loop {
            tokio::select! {
                _ = self.exit_signal.cancelled() => {
                    break;
                }
                _ = self.command_rx.changed() => {
                    self.status = self.command_rx.borrow_and_update().to_owned();
                    self.process_inner().await;
                }
                _ = queue_ready.notified() => {
                    self.process_inner().await;
                }
            };
        }
    }

    fn refresh_status(&mut self) {
        self.status = self.command_rx.borrow().to_owned();
    }

    async fn process_inner(&mut self) {
        loop {
            if self.exit_signal.is_cancelled() {
                info!("Verify worker::process_inner exit_signal is cancelled");
                return;
            }
            self.refresh_status();
            if self.status != ChunkCommand::Resume {
                return;
            }
            // cheap query to check queue is not empty
            if self.tasks.read().await.is_empty() {
                return;
            }

            self.refresh_status();
            if self.status != ChunkCommand::Resume {
                return;
            }

            // pick a entry to run verify
            let entry = {
                let mut tasks = self.tasks.write().await;
                match tasks.pop_front(self.role == WorkerRole::OnlySmallCycleTx) {
                    Some(entry) => entry,
                    None => {
                        if !tasks.is_empty() {
                            tasks.re_notify();
                            debug!(
                                "Worker (role: {:?}) didn't got tx after pop_front, but tasks is not empty, notify other Workers now",
                                self.role
                            );
                        }
                        return;
                    }
                }
            };

            if let Some((res, snapshot)) = self
                .service
                ._process_tx(
                    entry.tx.clone(),
                    entry.remote.map(|e| e.0),
                    Some(&mut self.command_rx),
                )
                .await
            {
                self.service
                    .after_process(entry.tx, entry.remote, &snapshot, &res)
                    .await;
            } else {
                info!("_process_tx for tx: {} returned none", entry.tx.hash());
            }
        }
    }
