### Title
Instantaneous Mempool State in `WeightUnitsFlow` Fee Estimator Enables Attacker-Controlled Fee Rate Inflation — (File: `util/fee-estimator/src/estimator/weight_units_flow.rs`)

---

### Summary

The `WeightUnitsFlow` fee estimator and its fallback (`pool_map::estimate_fee_rate`) both derive their output directly from the instantaneous, attacker-influenced mempool state. Any unprivileged tx-pool submitter can flood the pool with minimum-fee-rate transactions to inflate `current_weight_buckets`, causing the `estimate_fee_rate` RPC to return an artificially elevated fee rate recommendation. Because the historical `flow_speed_buckets` are also populated from accepted transactions, a sustained attacker can corrupt both the spot and the historical signal simultaneously.

---

### Finding Description

The `estimate_fee_rate` RPC (exposed via `rpc/src/module/experiment.rs`) delegates to `TxPoolService::estimate_fee_rate` in `tx-pool/src/process.rs`, which reads the live pool snapshot and passes it to the `WeightUnitsFlow` algorithm.

**Primary path — `WeightUnitsFlow::do_estimate`**

`estimate_fee_rate` in `weight_units_flow.rs` builds `sorted_current_txs` from the raw, instantaneous pool contents: [1](#0-0) 

`do_estimate` then constructs `current_weight_buckets` directly from those transactions: [2](#0-1) 

The decision for each bucket is: [3](#0-2) 

`removed_weight` is a constant (`MAX_BLOCK_BYTES * 85/100 * target_blocks`). `current_weight` is the attacker-controlled variable. By submitting enough transactions at the minimum fee rate, an attacker can make every low-fee bucket fail the `current_weight + added_weight <= removed_weight` check, forcing the estimator to return a higher fee rate.

**Historical signal is also attacker-influenced**

`accept_tx` records every admitted transaction into `self.txs` keyed by block number: [4](#0-3) 

`flow_speed_buckets` is derived from this same store: [5](#0-4) 

A sustained attacker who submits many transactions over `historical_blocks` (up to 256 blocks for `NoPriority`) inflates `added_weight` as well, compounding the upward bias.

**Fallback path — `pool_map::estimate_fee_rate`**

When the primary estimator returns `LackData` or `NotReady`, the fallback is invoked: [6](#0-5) 

The fallback is a purely instantaneous scan of the pool sorted by fee rate: [7](#0-6) 

It returns the fee rate of the transaction sitting at the `target_blocks`-th simulated block boundary. Flooding the pool with high-fee-rate transactions directly raises this boundary value.

---

### Impact Explanation

Any wallet, exchange, or dApp that calls `estimate_fee_rate` to determine what fee to attach to a transaction will receive an inflated recommendation. Users will systematically overpay transaction fees. In a miner-controlled or cartel scenario, the attacker can simultaneously collect the inflated fees as a miner. The `NoPriority` mode (`DEFAULT_TARGET = 128` blocks) is the default, giving the attacker the widest manipulation window.

---

### Likelihood Explanation

The attack requires only the ability to submit valid transactions to the mempool — the standard, unprivileged `send_transaction` RPC path. The minimum fee rate is 1,000 shannons/KW: [8](#0-7) 

To exceed `removed_weight` for `NoPriority` (`128` blocks): `MAX_BLOCK_BYTES * 85/100 * 128 ≈ 65 MB` of transaction weight. At 1,000 shannons/KW this costs roughly 65,000 shannons (≈ 0.00065 CKB) in fees — economically trivial. The pool accepts up to 180 MB: [9](#0-8) 

The attacker needs sufficient UTXOs to construct valid inputs, which is the only practical barrier.

---

### Recommendation

1. **Time-weight the current pool snapshot**: Instead of using the raw instantaneous pool weight, apply an exponential moving average or a TWAP-style smoothing over recent blocks before computing `current_weight_buckets`.
2. **Cap the influence of any single block's transactions** on the historical `flow_speed_buckets` to limit burst manipulation.
3. **Rate-limit or bound the `estimate_fee_rate` RPC** to prevent it from being used as a probe to verify manipulation success.
4. **Consider open-interest-style caps**: bound the maximum fee rate the estimator can return (e.g., a multiple of the recent median) to limit the magnitude of any manipulation.

---

### Proof of Concept

1. Attacker holds many UTXOs (obtainable by splitting a single large UTXO over several blocks).
2. Attacker calls `send_transaction` repeatedly, submitting transactions with fee rate = `min_fee_rate` (1,000 shannons/KW) until the pool contains > 65 MB of weight in the lowest fee bucket.
3. Victim calls `estimate_fee_rate` with default parameters (`NoPriority`, `enable_fallback=true`).
4. In `do_estimate`, `current_weight_buckets[1]` (the lowest non-zero bucket) now satisfies `current_weight + added_weight > removed_weight`, so bucket 1 fails.
5. The estimator walks up to higher buckets until it finds one that passes, returning a fee rate significantly above the true market rate.
6. Victim's wallet attaches the inflated fee to its transaction, overpaying.

The fallback path (`pool_map::estimate_fee_rate`) is triggered on a fresh node (before `historical_blocks` of data accumulate). The attacker submits high-fee-rate transactions to push the simulated block boundary to a high-fee entry, causing the fallback to return an inflated rate immediately.     pub fn estimate_fee_rate(

        target_blocks: BlockNumber,
        all_entry_info: TxPoolEntryInfo,

        if !self.is_ready {
            return Err(Error::NotReady);

        let sorted_current_txs = {
            let mut current_txs: Vec<_> = all_entry_info
                .pending
                .into_values()
                .chain(all_entry_info.proposed.into_values())
                .map(TxStatus::new_from_entry_info)
                .collect();
            current_txs.sort_unstable_by(|a, b| b.cmp(a));
            current_txs
        };

        self.do_estimate(target_blocks, &sorted_current_txs)
    }

        {
            Ok(fee_rate) => Ok(fee_rate),

                if enable_fallback {

                        .read()
                        .await
                        .estimate_fee_rate(target_blocks)
                        .map_err(Into::into)
                } else {
                    Err(err.into())
                }
            }
        }
    }
