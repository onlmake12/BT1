### Title
Zero-Fee Provider Registration Enables Denial of Service and Fund Drainage of Off-Chain Executor — (`File: target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary

`Echo.sol`'s `registerProvider` and `setProviderFee` functions accept all-zero fee values with no minimum enforcement. When a provider's fees are zero, `requestPriceUpdatesWithCallback` requires only the Pyth protocol fee (`pythFeeInWei`) as `msg.value`. An unprivileged attacker can flood the contract with near-free requests, forcing the provider's off-chain executor to execute every callback at its own gas cost, draining the provider wallet and causing denial of service.

---

### Finding Description

`registerProvider` stores caller-supplied fee values directly with no lower-bound check:

```solidity
function registerProvider(
    uint96 baseFeeInWei,
    uint96 feePerFeedInWei,
    uint96 feePerGasInWei
) external override {
    ProviderInfo storage provider = _state.providers[msg.sender];
    require(!provider.isRegistered, "Provider already registered");
    provider.baseFeeInWei = baseFeeInWei;
    provider.feePerFeedInWei = feePerFeedInWei;
    provider.feePerGasInWei = feePerGasInWei;
    provider.isRegistered = true;
    ...
}
``` [1](#0-0) 

`setProviderFee` has the same absence of minimum validation: [2](#0-1) 

`getFee` computes the total required fee as the sum of the Pyth protocol fee plus the three provider-controlled components. When all provider components are zero, the total fee collapses to `pythFeeInWei` alone:

```solidity
feeAmount =
    baseFee +           // pythFeeInWei (fixed, can be very small)
    providerBaseFee +   // 0
    providerFeedFee +   // 0
    SafeCast.toUint96(gasFee); // 0
``` [3](#0-2) 

`requestPriceUpdatesWithCallback` enforces only that `msg.value >= requiredFee`. With zero provider fees, this threshold is trivially met:

```solidity
uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
if (msg.value < requiredFee) revert InsufficientFee();
``` [4](#0-3) 

The fee stored in the request for the provider is:

```solidity
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);
``` [5](#0-4) 

When `msg.value == pythFeeInWei` and provider fees are zero, `req.fee = 0`. The provider's off-chain executor must still call `executeCallback`, paying full on-chain gas costs, but accrues zero compensation.

The `_initialize` function also permits `pythFeeInWei = 0`, making the total required fee literally zero: [6](#0-5) 

---

### Impact Explanation

1. **Denial of Service**: The provider's off-chain executor polls `getFirstActiveRequests` and processes every pending request. An attacker paying only `pythFeeInWei` (potentially 1 wei or zero) per transaction can create thousands of requests, flooding the executor queue and delaying or blocking legitimate requests.

2. **Fund Drainage**: Each `executeCallback` call costs real gas. With `req.fee = 0`, the provider's executor wallet is drained paying gas for callbacks with zero reimbursement. The `accruedFeesInWei` for the provider remains zero while the executor's ETH balance is consumed. [7](#0-6) 

---

### Likelihood Explanation

- **Realistic misconfiguration**: A provider deploying for the first time may call `registerProvider(0, 0, 0)` as a placeholder before setting real fees, or may reduce fees to zero via `setProviderFee` during a fee adjustment window.
- **No privileged access required**: The attacker only needs to call `requestPriceUpdatesWithCallback` — a public payable function — with `msg.value = pythFeeInWei`.
- **Low attacker cost**: On chains where `pythFeeInWei` is small (e.g., 1 wei), the attacker's cost per request is negligible. The `EchoState` struct shows `pythFeeInWei` is a `uint96` with no enforced minimum. [8](#0-7) 

---

### Recommendation

1. **Enforce a minimum provider fee** in `registerProvider` and `setProviderFee`: require that at least one of `baseFeeInWei`, `feePerFeedInWei`, or `feePerGasInWei` is above a protocol-defined minimum, or require `baseFeeInWei > 0`.

2. **Enforce a minimum total fee** in `requestPriceUpdatesWithCallback`: add a check such as `require(requiredFee >= MIN_REQUEST_FEE, "Fee too low")` where `MIN_REQUEST_FEE` covers the expected gas cost of `executeCallback`.

3. **Enforce a non-zero `pythFeeInWei`** during `_initialize` to guarantee a baseline cost per request regardless of provider configuration.

---

### Proof of Concept

```solidity
// Attacker registers as a provider with zero fees
vm.prank(attacker);
echo.registerProvider(0, 0, 0);

// Admin sets attacker as default provider (or attacker targets any zero-fee provider)
// pythFeeInWei is 0 or 1 wei

bytes32[] memory priceIds = new bytes32[](1);
priceIds[0] = bytes32(uint256(1));

// Flood with 10,000 free (or near-free) requests
for (uint i = 0; i < 10_000; i++) {
    echo.requestPriceUpdatesWithCallback{value: 0}(
        attacker,                        // zero-fee provider
        block.timestamp,
        priceIds,
        100_000                          // callbackGasLimit
    );
}
// Off-chain executor must now process 10,000 callbacks at its own gas cost
// Provider accrues 0 wei in fees
``` [9](#0-8)

    function requestPriceUpdatesWithCallback(
        address provider,
        uint64 publishTime,
        bytes32[] calldata priceIds,
        uint32 callbackGasLimit
    ) external payable override returns (uint64 requestSequenceNumber) {
        require(
            _state.providers[provider].isRegistered,
            "Provider not registered"
        );

        // FIXME: this comment is wrong. (we're not using tx.gasprice)
        // NOTE: The 60-second future limit on publishTime prevents a DoS vector where
        //      attackers could submit many low-fee requests for far-future updates when gas prices
        //      are low, forcing executors to fulfill them later when gas prices might be much higher.
        //      Since tx.gasprice is used to calculate fees, allowing far-future requests would make
        //      the fee estimation unreliable.
        require(publishTime <= block.timestamp + 60, "Too far in future");
        if (priceIds.length > MAX_PRICE_IDS) {
            revert TooManyPriceIds(priceIds.length, MAX_PRICE_IDS);
        }
        requestSequenceNumber = _state.currentSequenceNumber++;

        uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
        if (msg.value < requiredFee) revert InsufficientFee();

        Request storage req = allocRequest(requestSequenceNumber);
        req.sequenceNumber = requestSequenceNumber;
        req.publishTime = publishTime;
        req.callbackGasLimit = callbackGasLimit;
        req.requester = msg.sender;
        req.provider = provider;
        req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);

        // Create array with the right size
        req.priceIdPrefixes = new bytes8[](priceIds.length);

        // Copy only the first 8 bytes of each price ID to storage
        for (uint8 i = 0; i < priceIds.length; i++) {
            // Extract first 8 bytes of the price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }
            req.priceIdPrefixes[i] = prefix;
        }
        _state.accruedFeesInWei += _state.pythFeeInWei;

        emit PriceUpdateRequested(req, priceIds);
    }

    function getFee(
        address provider,
        uint32 callbackGasLimit,
        bytes32[] calldata priceIds
    ) public view override returns (uint96 feeAmount) {
        uint96 baseFee = _state.pythFeeInWei; // Fixed fee to Pyth
        // Note: The provider needs to set its fees to include the fee charged by the Pyth contract.
        // Ideally, we would be able to automatically compute the pyth fees from the priceIds, but the
        // fee computation on IPyth assumes it has the full updated data.
        uint96 providerBaseFee = _state.providers[provider].baseFeeInWei;
        uint96 providerFeedFee = SafeCast.toUint96(
            priceIds.length * _state.providers[provider].feePerFeedInWei
        );
        uint96 providerFeeInWei = _state.providers[provider].feePerGasInWei; // Provider's per-gas rate
        uint256 gasFee = callbackGasLimit * providerFeeInWei; // Total provider fee based on gas
        feeAmount =
            baseFee +
            providerBaseFee +
            providerFeedFee +
            SafeCast.toUint96(gasFee); // Total fee user needs to pay
    }
