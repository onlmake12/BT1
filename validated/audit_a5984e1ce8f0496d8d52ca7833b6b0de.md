### Title
`PythPriceOracleGetter.getAssetPrice()` Always Reverts on Stale/Invalid Price Instead of Returning 0, Breaking Aave's Fallback Oracle Mechanism - (File: `target_chains/ethereum/contracts/contracts/aave/PythPriceOracleGetter.sol`)

### Summary

`PythPriceOracleGetter.getAssetPrice()` unconditionally reverts on stale price, non-positive price, or normalization overflow. In Aave V2's dual-oracle design, the primary oracle is expected to return `0` on failure so that `AaveOracle` can fall through to its `_fallbackOracle`. Because `PythPriceOracleGetter` reverts instead of returning `0`, the fallback oracle is never triggered, causing a full DoS of all Aave operations that depend on the price oracle.

### Finding Description

`PythPriceOracleGetter` implements Aave's `IPriceOracleGetter` interface and is intended to serve as the price oracle for Aave V2 deployments. The function `getAssetPrice()` contains four hard-revert paths that prevent it from ever returning `0`:

1. **Stale price** — `pyth().getPriceNoOlderThan(priceId, validTimePeriodSeconds())` reverts with `PythErrors.StalePrice` when the cached price is older than `validTimePeriodSeconds`.
2. **Non-positive raw price** — `if (price.price <= 0) { revert InvalidNonPositivePrice(); }` (line 69–71).
3. **Normalizer overflow** — `if (!isNormalizerExpoNeg && normalizer > type(uint192).max) { revert NormalizationOverflow(); }` (line 80–82).
4. **Normalized-to-zero** — `if (normalizedPrice <= 0) { revert InvalidNonPositivePrice(); }` (line 88–90).

In Aave V2's `AaveOracle`, the fallback oracle is triggered when the primary oracle source returns `0`:

```solidity
// Aave V2 AaveOracle (external reference)
int256 price = IChainlinkAggregator(source).latestAnswer();
if (price > 0) {
    return uint256(price);
} else {
    return _fallbackOracle.getAssetPrice(asset);  // triggered on 0
}
```

When `PythPriceOracleGetter` is deployed as the primary oracle (or as the `_fallbackOracle`), any of the four revert conditions above propagates the revert up through `AaveOracle.getAssetPrice()`, preventing the fallback from ever being reached. The fallback oracle is designed to keep the protocol operational when one source fails; the revert behavior defeats this entirely.

### Impact Explanation

All Aave V2 operations that require a price (borrow, repay, liquidate, health-factor checks) call `oracle.getAssetPrice(asset)`. If `PythPriceOracleGetter` reverts, every one of these operations reverts. This is a complete DoS of the lending protocol for any asset whose Pyth feed is stale or returns a non-positive price. Liquidations are blocked, which can lead to bad debt accumulation. The fallback oracle — the safety net designed for exactly this scenario — is never invoked.

### Likelihood Explanation

Pyth price feeds can become stale during network congestion, keeper downtime, or deliberate griefing of the update pipeline. The `validTimePeriodSeconds` window is a fixed constructor parameter; once it elapses without an on-chain update, every call to `getAssetPrice()` reverts. This is a realistic, externally observable condition that requires no privileged access to trigger.

### Recommendation

Wrap the Pyth call in a `try/catch` and return `0` on any failure, consistent with how Chainlink's `latestAnswer()` signals bad data in Aave's design:

```solidity
function getAssetPrice(address asset) external view override returns (uint256) {
    bytes32 priceId = _registryState.assetsPriceIds[asset];
    if (asset == BASE_CURRENCY) return BASE_CURRENCY_UNIT;
    if (priceId == 0) return 0; // signal bad data instead of reverting

    try pyth().getPriceNoOlderThan(priceId, PythAssetRegistry.validTimePeriodSeconds())
        returns (PythStructs.Price memory price)
    {
        if (price.price <= 0) return 0;
        // ... normalization logic, return 0 on overflow or zero result
    } catch {
        return 0; // allows AaveOracle to fall through to _fallbackOracle
    }
}
```

### Proof of Concept

1. Deploy `PythPriceOracleGetter` with `validTimePeriodSeconds = 60`.
2. Push a valid Pyth price update at `t = 0`.
3. Advance block time by 61 seconds without pushing a new update.
4. Call `pythOracleGetter.getAssetPrice(asset)`.
5. Observe revert with `PythErrors.StalePrice` — confirmed by the existing test `testGetAssetRevertsIfPriceNotRecentEnough` in `Pyth.Aave.t.sol` lines 165–176.
6. In a live Aave V2 deployment, `AaveOracle.getAssetPrice(asset)` would also revert, blocking all borrow/liquidate operations and never reaching `_fallbackOracle.getAssetPrice(asset)`. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)
