### Title
Single-Step Ownership Transfer in `PythLazer` Locks `updateTrustedSigner` When Executor Becomes Owner - (File: `lazer/contracts/evm/src/PythLazer.sol`)

### Summary

`PythLazer` inherits from `OwnableUpgradeable` (single-step ownership) instead of `Ownable2StepUpgradeable`. The intended production deployment flow transfers ownership to the `ExecutorUpgradable` contract via `PythLazerChangeOwnership.s.sol`. Because `transferOwnership()` is single-step and immediate, if the wrong executor address is supplied (wrong chain, undeployed contract, or typo in `NEW_OWNER`), ownership is irrevocably transferred to an address that has no mechanism to call `updateTrustedSigner()`. All trusted signers will eventually expire, permanently breaking `verifyUpdate()` for every Lazer consumer.

### Finding Description

`PythLazer.sol` inherits `OwnableUpgradeable`:

```solidity
contract PythLazer is OwnableUpgradeable, UUPSUpgradeable {
``` [1](#0-0) 

Both critical owner-gated functions are protected by `onlyOwner`:

```solidity
function updateTrustedSigner(address trustedSigner, uint256 expiresAt) external onlyOwner { ... }
function _authorizeUpgrade(address) internal override onlyOwner {}
``` [2](#0-1) 

The deployment script `PythLazerDeploy.s.sol` initializes the proxy with a hardcoded deployer EOA as `topAuthority`:

```solidity
address topAuthority = deployer;
...
abi.encodeWithSignature("initialize(address)", topAuthority)
``` [3](#0-2) 

Ownership is then transferred to the executor contract via `PythLazerChangeOwnership.s.sol`, which calls the single-step `transferOwnership`:

```solidity
lazer.transferOwnership(NEW_OWNER);
``` [4](#0-3) 

`NEW_OWNER` is loaded from an environment variable with no on-chain validation:

```solidity
address public NEW_OWNER = vm.envAddress("NEW_OWNER");
``` [5](#0-4) 

With `OwnableUpgradeable`, `transferOwnership()` immediately and irrevocably sets the new owner. There is no `pendingOwner` / `acceptOwnership()` two-step safety net. If `NEW_OWNER` resolves to any address that cannot call `updateTrustedSigner()` — an undeployed contract, wrong-chain executor, or zero address — the function is permanently locked.

By contrast, the `ExecutorUpgradable` contract (the intended new owner) correctly uses `Ownable2StepUpgradeable` for its own ownership:

```solidity
contract ExecutorUpgradable is
    Initializable,
    Ownable2StepUpgradeable,
    UUPSUpgradeable,
    Executor
``` [6](#0-5) 

The inconsistency — `Ownable2StepUpgradeable` for the executor but `OwnableUpgradeable` for `PythLazer` — means the most sensitive ownership transfer (to a contract) has no safety net.

The codebase itself acknowledges the operational fragility: the `updateTrustedSigner` helper in `evm.ts` carries the note:

> `@note The privateKey should be the owner of the Lazer contract. It's not possible to run this function if the executor contract is the owner.` [7](#0-6) 

This confirms that once the executor is owner, the only recovery path is via governance VAA. If ownership is transferred to the wrong address, that path is also gone.

### Impact Explanation

Every `TrustedSignerInfo` entry carries an `expiresAt` timestamp:

```solidity
mapping(address => uint256) trustedSignerToExpiresAtMapping;
...
function isValidSigner(address signer) public view returns (bool) {
    return block.timestamp < trustedSignerToExpiresAtMapping[signer];
}
``` [8](#0-7) 

Once all signers expire and `updateTrustedSigner()` is locked, every call to `verifyUpdate()` reverts with `"invalid signer"`. The entire EVM Lazer price-feed verification service is permanently broken for all consumers. Additionally, `_authorizeUpgrade()` is also `onlyOwner`, so the contract cannot be upgraded to recover.

### Likelihood Explanation

The `PythLazerChangeOwnership.s.sol` script is the documented, intended production ownership-transfer procedure. `NEW_OWNER` is supplied via environment variable with no on-chain guard. Operational mistakes (wrong chain, undeployed executor, copy-paste error) are realistic in a multi-chain deployment. The absence of a two-step confirmation means there is zero recovery window once the transaction is broadcast.

### Recommendation

Replace `OwnableUpgradeable` with `Ownable2StepUpgradeable` in `PythLazer.sol`:

```solidity
import "@openzeppelin/contracts-upgradeable/access/Ownable2StepUpgradeable.sol";

contract PythLazer is Ownable2StepUpgradeable, UUPSUpgradeable {
```

Update `PythLazerChangeOwnership.s.sol` to call `transferOwnership` (which now only sets `pendingOwner`) and add a separate step where the executor calls `acceptOwnership()` via a governance VAA. This matches the pattern already used by `ExecutorUpgradable`, `EntropyUpgradable`, `SchedulerUpgradeable`, and `EchoUpgradeable`.

### Proof of Concept

1. Deploy `PythLazer` proxy via `PythLazerDeploy.s.sol`; owner = deployer EOA.
2. Run `PythLazerChangeOwnership.s.sol` with `NEW_OWNER` set to an address that has no `updateTrustedSigner` capability (e.g., wrong-chain executor, undeployed address, or `address(0)`).
3. `lazer.transferOwnership(NEW_OWNER)` executes; `owner()` immediately returns `NEW_OWNER`.
4. Any call to `updateTrustedSigner()` reverts: `"Ownable: caller is not the owner"`.
5. All existing `TrustedSignerInfo` entries reach their `expiresAt` timestamp.
6. `isValidSigner()` returns `false` for every signer.
7. Every `verifyUpdate()` call reverts: `"invalid signer"` — the Lazer EVM service is permanently non-functional.

    function _authorizeUpgrade(address) internal override onlyOwner {}

    function updateTrustedSigner(
        address trustedSigner,
        uint256 expiresAt
    ) external onlyOwner {
        if (expiresAt == 0) {
            for (uint8 i = 0; i < trustedSigners.length; i++) {
                if (trustedSigners[i].pubkey == trustedSigner) {
                    trustedSigners[i].pubkey = address(0);
                    trustedSigners[i].expiresAt = 0;
                    delete trustedSignerToExpiresAtMapping[trustedSigner];
                    return;
                }
            }
            revert("no such pubkey");
        } else {
            for (uint8 i = 0; i < trustedSigners.length; i++) {
                if (trustedSigners[i].pubkey == trustedSigner) {
                    trustedSigners[i].expiresAt = expiresAt;
                    trustedSignerToExpiresAtMapping[trustedSigner] = expiresAt;
                    return;
                }
            }
            // Signer not found - adding a new signer.
            for (uint8 i = 0; i < trustedSigners.length; i++) {
                if (trustedSigners[i].pubkey == address(0)) {
                    trustedSigners[i].pubkey = trustedSigner;
                    trustedSigners[i].expiresAt = expiresAt;
                    trustedSignerToExpiresAtMapping[trustedSigner] = expiresAt;
                    return;
                }
            }
            revert("no space for new signer");
        }
    }

    function isValidSigner(address signer) public view returns (bool) {
