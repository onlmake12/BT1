### Title
`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` Always Revert for Whitelisted Subscriptions Due to Dual `onlyWhitelistedReader` Check via `this.` External Call ŌĆö (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

`getPricesNoOlderThan` and `getEmaPricesNoOlderThan` in the Pulse `Scheduler` contract are permanently broken for any subscription with `whitelistEnabled = true`. Both functions carry the `onlyWhitelistedReader` modifier themselves, then delegate to `this.getPricesUnsafe` / `this.getEmaPricesUnsafe` via an external `this.` call. Because `this.` creates an external call, `msg.sender` inside the delegated function becomes the Scheduler contract's own address ŌĆö which is neither the subscription manager nor a whitelisted reader ŌĆö causing the second `onlyWhitelistedReader` check to always revert.

---

### Finding Description

`getPricesNoOlderThan` is declared `external view` and guarded by `onlyWhitelistedReader`:

```solidity
// Scheduler.sol line 535-555
function getPricesNoOlderThan(
    uint256 subscriptionId,
    bytes32[] calldata priceIds,
    uint256 age_seconds
)
    external
    view
    override
    onlyWhitelistedReader(subscriptionId)   // ŌåÉ Check 1: original msg.sender
    returns (PythStructs.Price[] memory prices)
{
    ...
    prices = this.getPricesUnsafe(subscriptionId, priceIds); // ŌåÉ external call
}
```

`getPricesUnsafe` is also `external view` and guarded by the same modifier:

```solidity
// Scheduler.sol line 514-533
function getPricesUnsafe(
    uint256 subscriptionId,
    bytes32[] calldata priceIds
)
    external
    view
    override
    onlyWhitelistedReader(subscriptionId)   // ŌåÉ Check 2: msg.sender = Scheduler address
    returns (PythStructs.Price[] memory prices)
{
    ...
}
```

The `onlyWhitelistedReader` modifier:

```solidity
// Scheduler.sol line 750-779
modifier onlyWhitelistedReader(uint256 subscriptionId) {
    if (_state.subscriptionManager[subscriptionId] == msg.sender) { _; return; }
    if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) { _; return; }
    // linear scan of whitelist
    if (!isWhitelisted) revert SchedulerErrors.Unauthorized();
    _;
}
```

When `getPricesNoOlderThan` calls `this.getPricesUnsafe(...)`, Solidity issues an external call. Inside `getPricesUnsafe`, `msg.sender` is the Scheduler contract's own address. The Scheduler contract is not the subscription manager and is not in any user-defined `readerWhitelist`. Therefore, for every subscription where `whitelistEnabled = true`, the second modifier invocation unconditionally reverts with `Unauthorized`.

The identical pattern exists in `getEmaPricesNoOlderThan` ŌåÆ `this.getEmaPricesUnsafe`.

---

### Impact Explanation

- `getPricesNoOlderThan` and `getEmaPricesNoOlderThan` are **completely non-functional** for all subscriptions with `whitelistEnabled = true`.
- The Pulse README explicitly recommends these functions as the safe reading path: *"Readers are recommended to use the SDK's functions `get(Ema)PricesNoOlderThan`"*.
- Whitelisted readers who funded a subscription cannot use the age-validated read path at all, even though they are legitimately authorized.
- The subscription manager is equally blocked when calling these functions.
- The default test subscription parameters use `whitelistEnabled: true`, confirming this is the expected common configuration.
- Impact: permanent DoS of the recommended safe price-reading interface for all whitelisted subscriptions.

---

### Likelihood Explanation

- Any whitelisted reader or subscription manager calling `getPricesNoOlderThan` or `getEmaPricesNoOlderThan` on a subscription with `whitelistEnabled = true` triggers the revert.
- No special attacker setup is required; the bug is triggered by normal, legitimate usage.
- The default subscription template in the test suite uses `whitelistEnabled: true`, making this the most common deployment configuration.
- Likelihood: **High** ŌĆö affects every whitelisted subscription on every call to these functions.

---

### Recommendation

Replace the `this.getPricesUnsafe(...)` and `this.getEmaPricesUnsafe(...)` external calls with direct calls to the internal helper `_getPricesInternal(...)`, which already contains the shared logic without the access-control modifier:

```solidity
// Before (broken):
prices = this.getPricesUnsafe(subscriptionId, priceIds);

// After (fixed):
PythStructs.PriceFeed[] memory priceFeeds = _getPricesInternal(subscriptionId, priceIds);
prices = new PythStructs.Price[](priceFeeds.length);
for (uint i = 0; i < priceFeeds.length; i++) {
    prices[i] = priceFeeds[i].price;
}
```

The `onlyWhitelistedReader` check on the outer function (`getPricesNoOlderThan`) is sufficient; the inner helper does not need to re-check it.

---

### Proof of Concept

1. Deploy `SchedulerUpgradeable` with a non-zero `singleUpdateKeeperFeeInWei`.
2. Call `createSubscription` with `whitelistEnabled = true` and `readerWhitelist = [readerAddress]`.
3. Fund the subscription and call `updatePriceFeeds` to populate price data.
4. As `readerAddress` (a legitimately whitelisted reader), call `getPricesNoOlderThan(subscriptionId, priceIds, 300)`.
5. Observe revert with `SchedulerErrors.Unauthorized`.

- `getPricesNoOlderThan` passes `onlyWhitelistedReader` (msg.sender = readerAddress Ō£ō).

    modifier onlyWhitelistedReader(uint256 subscriptionId) {
        // Manager is always allowed
        if (_state.subscriptionManager[subscriptionId] == msg.sender) {
            _;
            return;
        }

        // If whitelist is not used, allow any reader
        if (!_state.subscriptionParams[subscriptionId].whitelistEnabled) {
            _;
            return;
        }

        // Check if caller is in whitelist
        address[] storage whitelist = _state
            .subscriptionParams[subscriptionId]
            .readerWhitelist;
        bool isWhitelisted = false;
        for (uint i = 0; i < whitelist.length; i++) {
            if (whitelist[i] == msg.sender) {
                isWhitelisted = true;
                break;
            }
        }

        if (!isWhitelisted) {
            revert SchedulerErrors.Unauthorized();
        }
        _;
    }
