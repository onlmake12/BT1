### Title
Observed-Address Peer-ID Spoofing in Identify Protocol Poisons Node's Self-Advertised Address List — (`network/src/protocols/identify/mod.rs`)

### Summary
`IdentifyCallback::add_observed_addr` accepts a remote-supplied `observed_addr` that already contains an arbitrary `/p2p/<peer_id>` component without verifying that the embedded peer ID matches the local node's own peer ID. The attacker-controlled address is stored verbatim in `NetworkState::observed_addrs` and subsequently included in the `listen_addrs` field sent to every peer that connects afterward, causing the victim node to advertise a foreign address as its own.

### Finding Description
During the identify handshake, a remote peer sends an `IdentifyMessage` containing an `observed_addr` field — the address at which the remote peer claims to have observed the local node. `IdentifyProtocol::process_observed` forwards this value without any validation to `IdentifyCallback::add_observed_addr`:

```rust
// mod.rs line 497-507
fn add_observed_addr(&mut self, mut addr: Multiaddr, session_id: SessionId) -> MisbehaveResult {
    if extract_peer_id(&addr).is_none() {          // ← only checks presence, not ownership
        addr.push(Protocol::P2P(Cow::Borrowed(
            self.network_state.local_peer_id().as_bytes(),
        )))
    }
    self.network_state.add_observed_addr(session_id, addr);  // stored verbatim
    MisbehaveResult::Continue
}
```

If the attacker crafts `observed_addr = /ip4/<attacker_ip>/tcp/<attacker_port>/p2p/<attacker_peer_id>`, `extract_peer_id` returns `Some(attacker_peer_id)`, the `push` branch is skipped, and the address is stored as-is. `NetworkState::add_observed_addr` inserts it into `observed_addrs: RwLock<HashMap<PeerIndex, Multiaddr>>` with no further checks.

When any subsequent peer connects, `local_listen_addrs` is called, which appends `observed_addrs` to the list of addresses sent in the identify message. The victim node now advertises the attacker's address as one of its own listen addresses.

### Impact Explanation
- Every peer that connects after the poisoning receives the attacker's address in the victim's `listen_addrs`.
- Those peers add the address to their peer stores (via their own `add_remote_listen_addrs` → `peer_store.add_addr`).
- When they later attempt to connect to what they believe is the victim node, they dial the attacker's endpoint instead.
- The attacker can use this to redirect connections, impersonate the victim, or amplify an eclipse attack by making themselves appear reachable under the victim's advertised identity.
- The `observed_addrs` map is keyed by `session_id`, so a single attacker occupies one slot; multiple colluding attackers can fill all slots (bounded by `MAX_RETURN_LISTEN_ADDRS = 10`).

### Likelihood Explanation
Any inbound peer that passes the identify handshake (valid network name + non-zero flags) can trigger this with a single crafted message. No special privileges, no PoW, no key material required. The `is_reachable` filter applied in `connected` only requires the injected IP to be a globally routable address — trivially satisfied by using the attacker's own public IP.

### Recommendation
In `IdentifyCallback::add_observed_addr`, after extracting the peer ID from the supplied address, verify it matches the local node's peer ID. If it does not match, strip the foreign `/p2p/` component and append the correct local peer ID (or reject the address entirely):

```rust
fn add_observed_addr(&mut self, mut addr: Multiaddr, session_id: SessionId) -> MisbehaveResult {
    match extract_peer_id(&addr) {
        Some(peer_id) if peer_id == *self.network_state.local_peer_id() => {
            // peer ID already correct, use as-is
        }
        Some(_) => {
            // foreign peer ID — strip it and append the correct one
            // (or return MisbehaveResult::Disconnect)
            addr = addr.iter()
                .filter(|p| !matches!(p, Protocol::P2P(_)))
                .collect();
            addr.push(Protocol::P2P(Cow::Borrowed(
                self.network_state.local_peer_id().as_bytes(),
            )));
        }
        None => {
            addr.push(Protocol::P2P(Cow::Borrowed(
                self.network_state.local_peer_id().as_bytes(),
            )));
        }
    }
    self.network_state.add_observed_addr(session_id, addr);
    MisbehaveResult::Continue
}
```

This mirrors the validation already present in the hole-punching protocol's `connection_request.rs` and the startup check in `NetworkState::from_config`.

### Proof of Concept
1. Start a victim CKB node V with peer ID `QmV`.
2. Connect attacker node A (peer ID `QmA`, listening at `1.2.3.4:8115`) to V as an inbound peer.
3. A completes the identify handshake with a valid network name and flags, but sets `observed_addr = /ip4/1.2.3.4/tcp/8115/p2p/QmA` (its own address, not V's).
4. V's `add_observed_addr` stores `/ip4/1.2.3.4/tcp/8115/p2p/QmA` in `observed_addrs`.
5. Connect a third node C to V.
6. Inspect the `listen_addrs` field of the identify message C receives from V.
7. Assert that `/ip4/1.2.3.4/tcp/8115/p2p/QmA` is present — C now believes `1.2.3.4:8115` is one of V's listen addresses and will add it to its peer store, directing future connection attempts to A instead of V.
