### Title
Ban List Not Enforced at Outbound Dial Initiation — Banned Peers Receive TCP Connection Attempts - (File: `network/src/network.rs`)

---

### Summary

CKB's P2P ban list is checked only when *accepting* a session (`PeerRegistry::accept_peer`), but is never consulted before *initiating* an outbound TCP connection (`NetworkState::can_dial`). This is the direct analog of the USDM blocklist checking only the token sender: the restriction is applied on one side of the interaction (inbound acceptance) but not the other (outbound dial). As a result, banned peers can still receive TCP connection attempts from the node — both via the `add_node` RPC and via the automatic `OutboundPeerService` dialing loop — wasting resources and leaking node liveness to peers that were explicitly banned for misbehavior.

---

### Finding Description

**Root cause — missing ban check in `can_dial()`**

`NetworkState::can_dial()` is the gate that decides whether a dial attempt is sent. It checks five conditions (not self, not own public address, not already in registry, not already dialing, peer-id present), but it never consults the ban list: [1](#0-0) 

Every outbound dial path flows through `can_dial()`:

- `dial_inner()` → `can_dial()` → `p2p_control.dial()` (TCP connection sent)
- `dial_identify()` → `dial_inner()`
- `dial_feeler()` → `dial_inner()`
- `NetworkState::add_node()` → `dial_identify()`
- `NetworkController::add_node()` → `NetworkState::add_node()`
- RPC `add_node` → `NetworkController::add_node()` [2](#0-1) [3](#0-2) [4](#0-3) 

**Where the ban IS checked — only after TCP is established**

The ban check lives exclusively in `PeerRegistry::accept_peer()`, which is called only after the TCP handshake completes and a session is opened: [5](#0-4) 

**Second path: automatic dialing does not filter banned subnets**

`OutboundPeerService::try_dial_peers()` calls `fetch_addrs_to_attempt()`, and `dial_feeler()` calls `fetch_addrs_to_feeler()`. Neither of these filters out addresses that belong to a banned subnet: [6](#0-5) [7](#0-6) 

`ban_addr()` removes the specific address from `addr_manager`, but `ban_network()` (used by the `set_ban` RPC) does **not**: [8](#0-7) 

So after a subnet ban via `set_ban`, every address in that subnet that was already in `addr_manager` remains there and will be returned by `fetch_addrs_to_attempt` / `fetch_addrs_to_feeler`, causing the node to repeatedly dial banned peers.

---

### Impact Explanation

1. **Liveness leakage to banned peers**: A peer banned for sending malformed messages or invalid blocks still receives TCP SYN packets from the node. The banned peer can observe that the node is alive and actively trying to reconnect, undermining the isolation intent of the ban.

2. **Repeated resource consumption**: When a subnet is banned via `set_ban` RPC, the `OutboundPeerService` loop continuously selects addresses from that subnet, initiates TCP connections, completes the handshake, and then immediately tears them down when `accept_peer` rejects the session. This cycle repeats on every dial interval, wasting CPU, file descriptors, and bandwidth proportional to the number of banned-but-still-indexed addresses.

3. **Incomplete ban semantics**: The ban list is documented and exposed as a mechanism to fully isolate a peer. The asymmetry between the dial path and the accept path means the ban is only half-enforced, which is the direct structural analog of the USDM blocklist only checking the sender.

---

### Likelihood Explanation

- **Automatic path**: Any node that uses `set_ban` (RPC or automatic banning triggered by malformed-message detection) and has the banned IP already in `addr_manager` will exhibit this behavior continuously without any further attacker action.
- **RPC path**: Any local RPC caller (operator or script) that calls `add_node` with a banned peer's address will trigger an outbound TCP connection to that peer, bypassing the ban at the dial stage.
- Both paths are reachable without privileged keys; the automatic path requires no operator action at all after the initial ban.

---

### Recommendation

Add a ban list check inside `can_dial()` before returning `true`, mirroring the check already present in `accept_peer()`:

```rust
// In NetworkState::can_dial(), before returning true:
if self.with_peer_store(|peer_store| peer_store.is_addr_banned(addr)) {
    trace!("Do not dial banned addr: {}", addr);
    return false;
}
```

Additionally, `ban_network()` should remove all matching addresses from `addr_manager` (as `ban_addr()` already does for individual addresses), so that subnet bans are fully reflected in the dial candidate pool.

---

### Proof of Concept

1. Node A and Node B are connected. Node B sends a malformed sync message; Node A bans Node B's IP via automatic ban or `set_ban` RPC (`192.168.1.5/32`).
2. Node B's address `/ip4/192.168.1.5/tcp/8115/p2p/<peer_id>` remains in Node A's `addr_manager` (if banned via `ban_network`, which does not call `addr_manager.remove()`).
3. On the next `OutboundPeerService` tick, `fetch_addrs_to_attempt()` returns Node B's address (no ban filter in the query).
4. `dial_identify()` → `dial_inner()` → `can_dial()` returns `true` (no ban check) → `p2p_control.dial()` sends a TCP SYN to Node B.
5. TCP handshake completes; session-open event fires; `accept_peer()` checks the ban list → `PeerError::Banned` → session closed.
6. Node B has received and logged a TCP connection from Node A, confirming Node A's IP and liveness. Step 3–6 repeats on every dial interval for the duration of the ban. [1](#0-0) [9](#0-8) [10](#0-9) [11](#0-10)

    pub(crate) fn can_dial(&self, addr: &Multiaddr) -> bool {
        let peer_id = extract_peer_id(addr);
        if peer_id.is_none() {
            error!("Do not dial addr without peer id, addr: {}", addr);
            return false;
        }
        let peer_id = peer_id.as_ref().unwrap();

        if self.local_peer_id() == peer_id {
            trace!("Do not dial self: {:?}, {}", peer_id, addr);
            return false;
        }
        if self.public_addrs.read().contains(addr) {
            trace!(
                "Do not dial listened address(self): {:?}, {}",
                peer_id, addr
            );
            return false;
        }

        let peer_in_registry = self.with_peer_registry(|reg| {
            reg.get_key_by_peer_id(peer_id).is_some() || reg.is_feeler(addr)
        });
        if peer_in_registry {
            trace!("Do not dial peer in registry: {:?}, {}", peer_id, addr);
            return false;
        }

        if let Some(dial_started) = self.dialing_addrs.read().get(peer_id) {
            trace!(
                "Do not send repeated dial commands to network service: {:?}, {}",
                peer_id, addr
            );
            if Instant::now().saturating_duration_since(*dial_started) > DIAL_HANG_TIMEOUT {
                #[cfg(feature = "with_sentry")]
                with_scope(
                    |scope| scope.set_fingerprint(Some(&["ckb-network", "dialing-timeout"])),
                    || {
                        capture_message(
                            &format!(
                                "Dialing {:?}, {:?} for more than {} seconds, \
                                 something is wrong in network service",
                                peer_id,
                                addr,
                                DIAL_HANG_TIMEOUT.as_secs(),
                            ),
                            Level::Warning,
                        )
                    },
                );
            }
            return false;
        }

        true
    }

        &mut self,
        count: usize,
        required_flags: Flags,
        filter: F,
    ) -> Vec<AddrInfo>
    where
        F: Fn(&AddrInfo) -> bool,
    {
        // Get info:
        // 1. Not already connected
        // 2. Connected within 3 days

        let now_ms = ckb_systemtime::unix_time_as_millis();
        let peers = &self.connected_peers;
        let addr_expired_ms = now_ms.saturating_sub(ADDR_TRY_TIMEOUT_MS);

        let filter = |peer_addr: &AddrInfo| {
            filter(peer_addr)
                && extract_peer_id(&peer_addr.addr)
                    .map(|peer_id| !peers.contains_key(&peer_id))
                    .unwrap_or_default()
                && peer_addr

                && required_flags_filter(required_flags, Flags::from_bits_truncate(peer_addr.flags))
        };

        // get addrs that can attempt.
        self.addr_manager.fetch_random(count, filter)
    }
