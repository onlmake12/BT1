### Title
Stale `total_tx_size` Accounting: `remove_expired` Only Triggered by New Block Arrival, Not Callable Independently - (File: `tx-pool/src/process.rs`)

### Summary
The tx-pool's `remove_expired` function, which removes time-expired transactions and decrements `total_tx_size`/`total_tx_cycles`, is only invoked inside `_update_tx_pool_for_reorg`, which is itself only triggered when a new best block is processed. There is no periodic timer or public RPC/API path that allows an operator or RPC caller to trigger expiry cleanup independently. If no new blocks arrive while the pool is at capacity, expired transactions remain, keeping `total_tx_size` inflated, and new transactions submitted via RPC are rejected with `Reject::Full` even though the pool should have space.

### Finding Description

`TxPool` tracks the total serialized size of all pending/proposed transactions in `pool_map.total_tx_size`. Transactions have a configurable time-to-live (`expiry_hours`, default 12 hours). The cleanup function is: [1](#0-0) 

This function is called exclusively from `_update_tx_pool_for_reorg`: [2](#0-1) 

`_update_tx_pool_for_reorg` is itself only invoked from the chain verifier when a new best block is committed: [3](#0-2) 

The pool-full eviction check compares against the stale (inflated) `total_tx_size`: [4](#0-3) 

There is no standalone RPC method, periodic background timer, or public API in the tx-pool service that calls `remove_expired` independently of block arrival. The `clear_tx_pool` RPC exists but requires privileged local access and destroys all transactions, not just expired ones. [5](#0-4) 

### Impact Explanation

When a node stops receiving new blocks (network partition, eclipse attack, or extended low-hashrate period) while the tx-pool is at or near `max_tx_pool_size` (default 180 MB), transactions that have passed their 12-hour TTL are not evicted. `total_tx_size` remains inflated. Any subsequent `send_transaction` RPC call triggers `limit_size`, which finds `total_tx_size > max_tx_pool_size` and returns `Reject::Full` to the caller — even though the pool would have ample space if expiry cleanup ran. The caller has no mechanism to force cleanup; they must wait for the next block to be processed by the node. [6](#0-5) 

### Likelihood Explanation

On mainnet, blocks arrive approximately every 8–10 seconds, so the window where this matters is narrow under normal conditions. However, during a network partition or eclipse attack lasting more than 12 hours, the condition is fully reachable by any RPC caller submitting transactions. The attacker-controlled entry path is: submit transactions to fill the pool → isolate the node from the network → after 12 hours, the pool is full of expired-but-not-cleaned-up transactions → any new `send_transaction` RPC call is rejected with `PoolIsFull`. The root cause is entirely within CKB's own tx-pool code.

### Recommendation

Add a periodic background timer in `TxPoolService` (analogous to how the block assembler has its own update loop) that calls `remove_expired` at a fixed interval (e.g., every few minutes), independent of block arrival. This mirrors the fix described in the report: making the cleanup callable without requiring the triggering action (buying a policy / receiving a block).

### Proof of Concept

1. Fill the tx-pool to `max_tx_pool_size` (180 MB) with valid transactions paying just above `min_fee_rate`.
2. Isolate the node from the network so no new blocks arrive.
3. Wait 12 hours (`expiry_hours` default). All transactions in the pool are now past their TTL.
4. Submit a new transaction via `send_transaction` RPC.
5. Observe `PoolIsFull` rejection, because `remove_expired` has never been called (no block arrived), so `pool_map.total_tx_size` still reflects the expired transactions.
6. Confirm: reconnect the node to the network, wait for one block to be processed — `remove_expired` fires, `total_tx_size` drops to 0, and the same transaction is now accepted. [1](#0-0) [7](#0-6)

    // Expire all transaction (and their dependencies) in the pool.
    pub(crate) fn remove_expired(&mut self, callbacks: &Callbacks) {
        let now_ms = ckb_systemtime::unix_time_as_millis();

        let removed: Vec<_> = self
            .pool_map
            .iter()
            .filter(|&entry| self.expiry + entry.inner.timestamp < now_ms)
            .map(|entry| entry.inner.clone())
            .collect();

        for entry in removed {
            let tx_hash = entry.transaction().hash();
            debug!("remove_expired {} timestamp({})", tx_hash, entry.timestamp);
            self.pool_map.remove_entry(&entry.proposal_short_id());
            let reject = Reject::Expiry(entry.timestamp);
            callbacks.call_reject(self, &entry, reject);
        }
    }

    // Return a `Reject` for current inserting entry if it's removed
    pub(crate) fn limit_size(
        &mut self,
        callbacks: &Callbacks,
        current_entry_id: Option<&ProposalShortId>,
    ) -> Option<Reject> {
        let mut ret = None;
        while self.pool_map.total_tx_size > self.config.max_tx_pool_size {
            let next_evict_entry = || {
                self.pool_map
                    .next_evict_entry(Status::Pending)
                    .or_else(|| self.pool_map.next_evict_entry(Status::Gap))
                    .or_else(|| self.pool_map.next_evict_entry(Status::Proposed))
            };

            if let Some(id) = next_evict_entry() {
                let removed = self.pool_map.remove_entry_and_descendants(&id);
                for entry in removed {
                    let tx_hash = entry.transaction().hash();
                    debug!(
                        "Removed by size limit {} timestamp({})",
                        tx_hash, entry.timestamp
                    );
                    let reject = Reject::Full(format!(
                        "the fee_rate for this transaction is: {}",
                        entry.fee_rate()
                    ));
                    if let Some(short_id) = current_entry_id
                        && entry.proposal_short_id() == *short_id
                    {
                        ret = Some(reject.clone());
                    }
                    callbacks.call_reject(self, &entry, reject);
                }
            }
        }
        self.pool_map.entries.shrink_to_fit();
        ret

fn _update_tx_pool_for_reorg(
    tx_pool: &mut TxPool,
    attached: &LinkedHashSet<TransactionView>,
    detached_headers: &HashSet<Byte32>,
    detached_proposal_id: HashSet<ProposalShortId>,
    snapshot: Arc<Snapshot>,
    callbacks: &Callbacks,
    mine_mode: bool,
) {
    tx_pool.snapshot = Arc::clone(&snapshot);

    // NOTE: `remove_by_detached_proposal` will try to re-put the given expired/detached proposals into
    // pending-pool if they can be found within txpool. As for a transaction
    // which is both expired and committed at the one time(commit at its end of commit-window),
    // we should treat it as a committed and not re-put into pending-pool. So we should ensure
    // that involves `remove_committed_txs` before `remove_expired`.
    tx_pool.remove_committed_txs(attached.iter(), callbacks, detached_headers);
    tx_pool.remove_by_detached_proposal(detached_proposal_id.iter());

    // mine mode:
    // pending ---> gap ----> proposed
    // try move gap to proposed
    if mine_mode {
        let mut proposals = Vec::new();
        let mut gaps = Vec::new();

        for entry in tx_pool.pool_map.entries.get_by_status(&Status::Gap) {
            let short_id = entry.inner.proposal_short_id();
            if snapshot.proposals().contains_proposed(&short_id) {
                proposals.push((short_id, entry.inner.clone()));
            }
        }

        for entry in tx_pool.pool_map.entries.get_by_status(&Status::Pending) {
            let short_id = entry.inner.proposal_short_id();
            let elem = (short_id.clone(), entry.inner.clone());
            if snapshot.proposals().contains_proposed(&short_id) {
                proposals.push(elem);
            } else if snapshot.proposals().contains_gap(&short_id) {
                gaps.push(elem);
            }
        }

        for (id, entry) in proposals {
            debug!("begin to proposed: {:x}", id);
            if let Err(e) = tx_pool.proposed_rtx(&id) {
                debug!(
                    "Failed to add proposed tx {}, reason: {}",
                    entry.transaction().hash(),
                    e
                );
                callbacks.call_reject(tx_pool, &entry, e);
            } else {
                callbacks.call_proposed(&entry)
            }
        }

        for (id, entry) in gaps {
            debug!("begin to gap: {:x}", id);
            if let Err(e) = tx_pool.gap_rtx(&id) {
                debug!(
                    "Failed to add tx to gap {}, reason: {}",
                    entry.transaction().hash(),
                    e
                );
                callbacks.call_reject(tx_pool, &entry, e.clone());
            }
        }
    }

    // Remove expired transaction from pending
    tx_pool.remove_expired(callbacks);

    // Remove transactions from the pool until its size <= size_limit.
    let _ = tx_pool.limit_size(callbacks, None);
