### Title
Zero-Fee TWAP and Parse Operations Enable Spam/DoS — (`target_chains/stylus/contracts/pyth-receiver/src/lib.rs`)

### Summary

The Stylus (Arbitrum) Pyth receiver contract contains two zero-cost callable paths analogous to the zero-weight extrinsic vulnerability: `get_twap_update_fee` is hardcoded to return `0`, and `parse_price_feed_updates` / `parse_price_feed_updates_with_config` are public functions that perform expensive on-chain work (Wormhole VAA verification, Merkle proof checking, optional storage writes) with no fee check and no `#[payable]` attribute.

### Finding Description

In `target_chains/stylus/contracts/pyth-receiver/src/lib.rs`:

**Issue 1 — `get_twap_update_fee` hardcodes zero:**

```rust
pub fn get_twap_update_fee(&self, _update_data: Vec<Vec<u8>>) -> U256 {
    U256::from(0u8)
}
``` [1](#0-0) 

This is the fee-quoting function for TWAP updates. It unconditionally returns 0 regardless of the number of price feeds or update complexity. Any caller querying this function to determine the required fee will be told the fee is zero.

**Issue 2 — `parse_price_feed_updates` and `parse_price_feed_updates_with_config` have no fee enforcement:**

```rust
pub fn parse_price_feed_updates(
    &mut self,
    update_data: Vec<u8>,
    price_ids: Vec<[u8; 32]>,
    min_publish_time: u64,
    max_publish_time: u64,
) -> Result<Vec<PriceFeedReturn>, PythReceiverError> {
    let price_feeds = self.parse_price_feed_updates_with_config(
        vec![update_data], price_ids, min_publish_time, max_publish_time,
        false, false, false,
    );
    price_feeds
}
``` [2](#0-1) 

Neither `parse_price_feed_updates` nor `parse_price_feed_updates_with_config` is marked `#[payable]`, and neither calls `self.vm().msg_value()` or checks against any fee. Compare this to `update_price_feeds`, which is correctly gated:

```rust
#[payable]
pub fn update_price_feeds(...) -> Result<(), PythReceiverError> {
    let total_fee = self.get_update_fee(update_data.clone())?;
    let value = self.vm().msg_value();
    if value < total_fee {
        return Err(PythReceiverError::InsufficientFee);
    }
    ...
}
``` [3](#0-2) 

`parse_price_feed_updates_with_config` with `store_updates_if_fresh = true` calls `update_price_feeds_internal`, which writes price data to contract storage — a state-mutating operation performed for free. [4](#0-3) 

The EVM reference implementation (`Pyth.sol`) correctly enforces fees on all parse paths:

```solidity
uint requiredFee = getTwapUpdateFee(updateData);
if (msg.value < requiredFee) revert PythErrors.InsufficientFee();
``` [5](#0-4) 

The Stylus implementation is missing this enforcement entirely.

### Impact Explanation

An unprivileged attacker can call `parse_price_feed_updates_with_config` (with `store_updates_if_fresh = true`) in a tight loop, paying only Arbitrum gas costs (no protocol fee). Each call triggers:
- An external call to the Wormhole contract for VAA verification
- Merkle proof verification over the update set
- Storage writes for each price feed when updates are fresh

This allows an attacker to exhaust node resources and degrade service for legitimate users at a cost far below what the protocol intends to charge. The `get_twap_update_fee` returning 0 means any future TWAP-consuming integrator will be told no fee is required, compounding the issue when TWAP parsing is fully implemented.

### Likelihood Explanation

The entry path is fully permissionless — any Ethereum address can call `parse_price_feed_updates` or `parse_price_feed_updates_with_config` on the deployed Stylus contract with zero ETH value. No privileged role, key, or governance action is required. The attacker only needs valid (or even stale) Wormhole-signed accumulator update data, which is freely available from the Hermes API.

### Recommendation

1. Mark `parse_price_feed_updates` and `parse_price_feed_updates_with_config` as `#[payable]` and add a fee check using `self.get_update_fee(...)` and `self.vm().msg_value()`, mirroring the pattern in `update_price_feeds`.
2. Implement `get_twap_update_fee` to compute the actual fee based on the number of updates in the provided data, rather than returning a hardcoded `0`.
3. Audit all other public `&mut self` functions in the Stylus receiver for missing `#[payable]` + fee-check guards.

### Proof of Concept

```rust

    pub fn parse_price_feed_updates_with_config(
        &mut self,
        update_data: Vec<Vec<u8>>,
        price_ids: Vec<[u8; 32]>,
        min_allowed_publish_time: u64,
        max_allowed_publish_time: u64,
        check_uniqueness: bool,
        check_update_data_is_minimal: bool,
        store_updates_if_fresh: bool,
    ) -> Result<Vec<PriceFeedReturn>, PythReceiverError> {
        let mut all_parsed_price_feeds = Vec::new();
        for data in &update_data {
            if store_updates_if_fresh {
                all_parsed_price_feeds.extend(self.update_price_feeds_internal(
                    data.clone(),
                    min_allowed_publish_time,
                    max_allowed_publish_time,
                    check_uniqueness,
                )?);
            } else {
                all_parsed_price_feeds.extend(self.parse_price_feed_updates_internal(
                    data.clone(),
                    min_allowed_publish_time,
                    max_allowed_publish_time,
                    check_uniqueness,
                )?);
            }
        }

        if check_update_data_is_minimal && all_parsed_price_feeds.len() != price_ids.len() {
            return Err(PythReceiverError::InvalidUpdateData);
        }

        let mut result: Vec<PriceFeedReturn> = Vec::with_capacity(price_ids.len());

        for price_id in price_ids {
            if let Some(price_info) = all_parsed_price_feeds
                .iter()
                .find(|feed| feed.0 == price_id)
            {
                result.push(price_info.clone());
            } else {
                return Err(PythReceiverError::PriceFeedNotFoundWithinRange);
            }
        }

        Ok(result)
    }
