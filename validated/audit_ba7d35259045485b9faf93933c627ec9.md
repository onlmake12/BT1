### Title
Unguarded `.expect()` on `txs_sizes: None` in `FeeRateCollector::statistics` Panics RPC Handler — (`rpc/src/util/fee_rate.rs`)

### Summary

An unprivileged caller invoking `get_fee_rate_statistics` (or `get_fee_rate_statics`) with any `target` value large enough to reach pre-`BlockExtV1`-migration blocks will trigger an unconditional `.expect()` on a `None`-valued `txs_sizes` field, panicking the RPC handler thread.

---

### Finding Description

**The unguarded `.expect()` call:**

In `rpc/src/util/fee_rate.rs` line 93, the closure passed to `collect()` unconditionally unwraps `txs_sizes`:

```rust
let txs_sizes = txs_sizes.expect("expect txs_size's length >= 1");
```

There is no `None` check before this call. [1](#0-0) 

**`txs_sizes` is legitimately `None` for old blocks:**

The old `packed::BlockExt` deserialization (pre-`BlockExtV1`) always produces `txs_sizes: None`:

```rust
impl<'r> Unpack<core::BlockExt> for packed::BlockExtReader<'r> {
    fn unpack(&self) -> core::BlockExt {
        core::BlockExt {
            ...
            cycles: None,
            txs_sizes: None,   // always None for old format
        }
    }
}
``` [2](#0-1) 

Even after migration to `BlockExtV1`, blocks that were originally stored without `txs_sizes` retain `txs_sizes: None` — the migration test itself stores a `BlockExt { txs_sizes: None }` as `BlockExtV1`, confirming no backfill occurs: [3](#0-2) 

**`collect()` does not filter `txs_sizes=None` blocks:**

The `collect()` method only skips blocks where `get_block_ext_by_number` returns `None` (i.e., missing from DB entirely). Blocks that exist but carry `txs_sizes: None` pass through unfiltered into the closure:

```rust
let block_ext_iter =
    (start..=tip_number).filter_map(|number| self.get_block_ext_by_number(number));
block_ext_iter.fold(Vec::new(), f)
``` [4](#0-3) 

**The RPC entrypoint is fully unprivileged:**

`get_fee_rate_statistics` and `get_fee_rate_statics` are public RPC methods with no authentication. Both delegate directly to `FeeRateCollector::statistics` with the caller-supplied `target`: [5](#0-4) 

`MAX_TARGET` is 101, so a caller passing `target=101` maximizes the look-back window: [6](#0-5) 

---

### Impact Explanation

A panic in the RPC handler thread crashes that worker. Depending on the async runtime's panic strategy (tokio workers are not `catch_unwind`-wrapped by default in CKB's RPC layer), this can kill the tokio worker thread processing the request, degrading or crashing the RPC service. The full node process itself may survive, but the RPC endpoint becomes unreliable or unresponsive. Scoped impact: **local RPC API crash (Note, 0–500)**.

---

### Likelihood Explanation

Any node that has been running since before the `BlockExtV1` migration (i.e., upgraded from an older CKB version) will have historical blocks with `txs_sizes=None` in its DB. The RPC endpoint is open to any local or network-accessible caller. The trigger is a single valid JSON-RPC call with `target=101`. No special privileges, keys, or hashpower are required.

---

### Recommendation

Replace the unconditional `.expect()` with a graceful `None` skip:

```rust
// Before (panics):
let txs_sizes = txs_sizes.expect("expect txs_size's length >= 1");

// After (safe):
let Some(txs_sizes) = txs_sizes else {
    return fee_rates; // skip blocks without txs_sizes data
};
```

This matches the existing guard on line 94 (`if txs_sizes.len() > 1 && !txs_fees.is_empty()`) which already handles the empty-data case gracefully. [7](#0-6) 

---

### Proof of Concept

Using the existing `DummyFeeRateProvider` test harness in `rpc/src/tests/fee_rate.rs`, add a block with `txs_sizes: None`:

```rust
#[test]
#[should_panic(expected = "expect txs_size's length >= 1")]
fn test_fee_rate_panics_on_none_txs_sizes() {
    let mut provider = DummyFeeRateProvider::new(101);
    provider.append(1, BlockExt {
        received_at: 0,
        total_difficulty: 0u64.into(),
        total_uncles_count: 0,
        verified: None,
        txs_fees: vec![Capacity::shannons(1000)],
        cycles: Some(vec![100]),
        txs_sizes: None,   // pre-migration block
    });
    // This panics at rpc/src/util/fee_rate.rs:93
    let _ = FeeRateCollector::new(&provider).statistics(Some(101));
}
```

The `#[should_panic]` annotation confirms the panic is unconditionally triggered by a `None` `txs_sizes` value, reachable via the public RPC endpoint on any node with pre-migration DB state. [8](#0-7)
