### Title
Any Registered Provider Can Steal Fees From Another Provider's Pending Request After Exclusivity Period - (File: `target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary
In `Echo.sol`, the `executeCallback` function enforces that only the originally assigned provider (`req.provider`) can be credited during the exclusivity window. However, once that window expires, **no check is performed on `providerToCredit`**. Any registered provider can call `executeCallback` with their own address as `providerToCredit` and receive the full fee that was meant for the originally assigned provider.

### Finding Description
The `executeCallback` function in `Echo.sol` accepts a caller-controlled `providerToCredit` parameter and credits that address with the request's accrued fee:

```solidity
function executeCallback(
    address providerToCredit,
    uint64 sequenceNumber,
    bytes[] calldata updateData,
    bytes32[] calldata priceIds
) external payable override {
    Request storage req = findActiveRequest(sequenceNumber);

    // Check provider exclusivity using configurable period
    if (
        block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds
    ) {
        require(
            providerToCredit == req.provider,
            "Only assigned provider during exclusivity period"
        );
    }
    ...
    _state.providers[providerToCredit].accruedFeesInWei += SafeCast
        .toUint128((req.fee + msg.value) - pythFee);
``` [1](#0-0) [2](#0-1) 

The guard at lines 114–121 only enforces `providerToCredit == req.provider` **during** the exclusivity period. After the period elapses, the `if` block is skipped entirely and `providerToCredit` is never validated against `req.provider`. Any address supplied by the caller is credited unconditionally.

Provider registration is fully permissionless:

```solidity
function registerProvider(...) external override {
    ProviderInfo storage provider = _state.providers[msg.sender];
    require(!provider.isRegistered, "Provider already registered");
    ...
    provider.isRegistered = true;
``` [3](#0-2) 

A registered provider can set themselves as their own fee manager:

```solidity
function setFeeManager(address manager) external override {
    require(
        _state.providers[msg.sender].isRegistered,
        "Provider not registered"
    );
    ...
    _state.providers[msg.sender].feeManager = manager;
``` [4](#0-3) 

And then withdraw the stolen accrued fees via `withdrawAsFeeManager`:

```solidity
function withdrawAsFeeManager(address provider, uint128 amount) external override {
    require(
        msg.sender == _state.providers[provider].feeManager,
        "Only fee manager"
    );
    ...
    _state.providers[provider].accruedFeesInWei -= amount;
    (bool sent, ) = msg.sender.call{value: amount}("");
``` [5](#0-4) 

### Impact Explanation
A competing provider can steal the full fee (`req.fee + msg.value - pythFee`) from any pending request once the exclusivity period expires. The originally assigned provider loses their earned fee entirely. Because `registerProvider` is permissionless, any EOA can become a "registered provider," execute another provider's pending callback, and redirect all fees to themselves. This is a direct, repeatable financial loss for legitimate providers.

### Likelihood Explanation
The exclusivity period is a finite, admin-configured window (`_state.exclusivityPeriodSeconds`). Every pending request that is not fulfilled within that window becomes vulnerable. A malicious actor who monitors the mempool or on-chain state can trivially identify unfulfilled requests past their exclusivity window and front-run or race the legitimate provider. Registration as a provider costs only gas. The attack is fully permissionless and requires no privileged access.

### Recommendation
After the exclusivity period, `providerToCredit` should still be validated. The simplest fix is to always require `providerToCredit == req.provider`, regardless of timing, and separately implement a penalty/redistribution mechanism for late fulfillment. Alternatively, require that `providerToCredit` is a registered provider **and** is either the originally assigned provider or has been explicitly authorized by them:

```solidity
// Always enforce that fees go to the originally assigned provider
require(
    providerToCredit == req.provider,
    "providerToCredit must be the assigned provider"
);
```

If the intent is to allow any provider to fulfill late requests and earn a fee, the contract should credit `msg.sender` (the actual executor) rather than an arbitrary caller-supplied address, and should verify `msg.sender` is a registered provider.

### Proof of Concept

1. **Setup**: Alice registers as a provider (`registerProvider`) and sets herself as her own fee manager (`setFeeManager(alice)`). Bob (the legitimate provider) is the assigned provider for a pending request with a fee of `F` wei.

2. **Wait**: The exclusivity period (`req.publishTime + exclusivityPeriodSeconds`) elapses without Bob fulfilling the request.

3. **Attack**: Alice calls:
   ```solidity
   echo.executeCallback(
       alice,          // providerToCredit = Alice
       sequenceNumber, // Bob's pending request
       updateData,
       priceIds
   );
   ```
   The `if (block.timestamp < req.publishTime + exclusivityPeriodSeconds)` check is **false**, so the `require(providerToCredit == req.provider)` guard is **skipped**. `_state.providers[alice].accruedFeesInWei += F` is executed.

4. **Withdraw**: Alice calls `withdrawAsFeeManager(alice, F)` and receives Bob's fee.

Bob receives nothing despite being the assigned provider. Alice, a competing provider, has stolen Bob's fee with no privileged access beyond a permissionless `registerProvider` call. [6](#0-5)

        address provider,
        uint128 amount
    ) external override {
        require(
            msg.sender == _state.providers[provider].feeManager,
            "Only fee manager"
        );
        require(
            _state.providers[provider].accruedFeesInWei >= amount,
            "Insufficient balance"
        );

        _state.providers[provider].accruedFeesInWei -= amount;

        (bool sent, ) = msg.sender.call{value: amount}("");
        require(sent, "Failed to send fees");

        emit FeesWithdrawn(msg.sender, amount);
