### Title
Provider Can Front-Run User Requests by Instantly Raising Fees — (`File: target_chains/ethereum/contracts/contracts/entropy/Entropy.sol`)

### Summary
A registered Entropy provider can observe a user's pending `requestV2()` / `requestWithCallback()` transaction in the mempool and front-run it with a `setProviderFee()` call at a higher gas price. Because the fee is enforced at execution time against the live on-chain value, the user's transaction reverts with `InsufficientFee`, wasting the user's gas and denying them service.

### Finding Description
`setProviderFee()` takes effect immediately with no timelock, delay, or minimum-notice period:

```solidity
function setProviderFee(uint128 newFeeInWei) external override {
    // ...
    provider.feeInWei = newFeeInWei;   // instant, no delay
    // ...
}
``` [1](#0-0) 

Inside `requestHelper`, the fee is validated against the current on-chain value at execution time:

```solidity
uint128 requiredFee = getFeeV2(provider, callbackGasLimit);
if (msg.value < requiredFee) revert EntropyErrors.InsufficientFee();
``` [2](#0-1) 

A user who calls `getFeeV2()` off-chain and then submits a request with exactly that amount is vulnerable between the moment they read the fee and the moment their transaction is mined. A provider who sees the pending request in the mempool can submit `setProviderFee(higherFee)` with a higher gas price, landing first and causing the user's transaction to revert.

The same pattern exists in `setProviderFeeAsFeeManager()`, which the keeper's `adjust_fee_if_necessary` loop calls automatically: [3](#0-2) [4](#0-3) 

### Impact Explanation
- Any user's `requestV2()` / `requestWithCallback()` / `request()` transaction can be made to revert by the provider they chose, at will.
- The user loses gas on every reverted attempt.
- For time-sensitive consumers (lotteries, NFT mints, gaming), the provider can selectively block specific users or block requests at critical moments.
- Because `revealWithCallback` is permissionless (anyone can call it), the provider can also choose *not* to reveal, compounding the DoS.

### Likelihood Explanation
- Provider registration is permissionless via `register()`. Any attacker can become a provider and attract users (e.g., by advertising a low fee), then grief them.
- A legitimate provider running the Fortuna keeper already has automated fee-adjustment logic (`adjust_fee_if_necessary`) that fires `setProviderFeeAsFeeManager` whenever gas prices move. This creates an unintentional race condition even without malicious intent: a user who reads the fee during a gas-price spike may find their request reverted by the keeper's routine fee update.
- Mempool visibility is standard on all EVM chains where Entropy is deployed.

### Recommendation
Introduce a **fee-change delay / timelock**: when a provider calls `setProviderFee()`, record the new fee and a future activation timestamp (e.g., `block.timestamp + MIN_FEE_NOTICE`). The `requestHelper` fee check should use the currently *active* fee, not the pending one. This mirrors the standard ERC-4626 / AMM pattern of announcing parameter changes in advance.

Alternatively, allow users to pass a `maxFeeInWei` parameter to `requestV2()` so the transaction self-protects with a slippage guard and reverts with a clear error rather than silently losing gas.

### Proof of Concept
1. Provider registers with `feeInWei = 1000 wei`.
2. User calls `getFeeV2(provider, 0)` → returns `1000 + pythFee`.
3. User submits `requestV2{value: 1000 + pythFee}()` to the mempool.
4. Provider observes the pending transaction and submits `setProviderFee(2000)` with `gasPrice + 1`.
5. `setProviderFee` is mined first; `provider.feeInWei` is now `2000`.
6. User's `requestV2` executes: `getFeeV2` returns `2000 + pythFee > msg.value` → `revert InsufficientFee`.
7. User loses gas; provider can repeat this indefinitely to permanently deny service. [5](#0-4) [6](#0-5)

    function requestHelper(
        address provider,
        bytes32 userCommitment,
        bool useBlockhash,
        bool isRequestWithCallback,
        uint32 callbackGasLimit
    ) internal returns (EntropyStructsV2.Request storage req) {
        EntropyStructsV2.ProviderInfo storage providerInfo = _state.providers[
            provider
        ];
        if (_state.providers[provider].sequenceNumber == 0)
            revert EntropyErrors.NoSuchProvider();

        // Assign a sequence number to the request
        uint64 assignedSequenceNumber = providerInfo.sequenceNumber;
        if (assignedSequenceNumber >= providerInfo.endSequenceNumber)
            revert EntropyErrors.OutOfRandomness();
        providerInfo.sequenceNumber += 1;

        // Check that fees were paid and increment the pyth / provider balances.
        uint128 requiredFee = getFeeV2(provider, callbackGasLimit);
        if (msg.value < requiredFee) revert EntropyErrors.InsufficientFee();

    function setProviderFee(uint128 newFeeInWei) external override {
        EntropyStructsV2.ProviderInfo storage provider = _state.providers[
            msg.sender
        ];

        if (provider.sequenceNumber == 0) {
            revert EntropyErrors.NoSuchProvider();
        }
        uint128 oldFeeInWei = provider.feeInWei;
        provider.feeInWei = newFeeInWei;
        emit ProviderFeeUpdated(msg.sender, oldFeeInWei, newFeeInWei);
        emit EntropyEventsV2.ProviderFeeUpdated(
            msg.sender,
            oldFeeInWei,
            newFeeInWei,
            bytes("")
        );
    }

    function setProviderFeeAsFeeManager(
        address provider,
        uint128 newFeeInWei
    ) external override {
        EntropyStructsV2.ProviderInfo storage providerInfo = _state.providers[
            provider
        ];

        if (providerInfo.sequenceNumber == 0) {
            revert EntropyErrors.NoSuchProvider();
        }

        if (providerInfo.feeManager != msg.sender) {
            revert EntropyErrors.Unauthorized();
        }

        uint128 oldFeeInWei = providerInfo.feeInWei;
        providerInfo.feeInWei = newFeeInWei;

        emit ProviderFeeUpdated(provider, oldFeeInWei, newFeeInWei);
        emit EntropyEventsV2.ProviderFeeUpdated(
            provider,
            oldFeeInWei,
            newFeeInWei,
            bytes("")
        );
    }
