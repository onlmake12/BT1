### Title
Missing Contract Existence Check Before Low-Level Call in `revealWithCallback` (`gasLimit10k != 0` Branch) — (File: `target_chains/ethereum/contracts/contracts/entropy/Entropy.sol`)

### Summary
`Entropy.sol`'s `revealWithCallback` function has two execution branches. The legacy branch (`gasLimit10k == 0`) correctly guards its callback with an `extcodesize` check before invoking `_entropyCallback`. The newer branch (`gasLimit10k != 0`) uses `excessivelySafeCall` — a raw low-level CALL — with no `extcodesize` guard. When `req.requester` is an EOA or a selfdestructed contract, the EVM CALL succeeds silently (`success = true`, empty return data), causing the request to be permanently cleared and success events to be emitted, while the callback is never actually delivered.

### Finding Description

In `revealWithCallback`, the `gasLimit10k != 0` branch performs:

```solidity
(success, ret) = req.requester.excessivelySafeCall(
    uint256(req.gasLimit10k) * TEN_THOUSAND,
    256,
    abi.encodeWithSelector(
        IEntropyConsumer._entropyCallback.selector,
        sequenceNumber,
        provider,
        randomNumber
    )
);
``` [1](#0-0) 

`excessivelySafeCall` is a raw low-level CALL. On the EVM, a CALL to an address with zero bytecode (EOA or selfdestructed contract) always returns `(1, "")` — success with empty return data. There is no `extcodesize` guard here.

When `success == true`, the code immediately clears the request and emits success events:

```solidity
if (success) {
    emit RevealedWithCallback(...);
    emit EntropyEventsV2.Revealed(...);
    clearRequest(provider, sequenceNumber);
}
``` [2](#0-1) 

By contrast, the `gasLimit10k == 0` (legacy) branch explicitly checks `extcodesize` before invoking the callback:

```solidity
uint len;
assembly {
    len := extcodesize(callAddress)
}
if (len != 0) {
    IEntropyConsumer(callAddress)._entropyCallback(...);
}
``` [3](#0-2) 

This is a direct inconsistency: the developers were aware of the EOA/no-code case and guarded it in one branch but omitted the guard in the other.

### Impact Explanation

When `req.requester` is an EOA or a contract whose code has been removed (via `selfdestruct`):

1. `excessivelySafeCall` returns `(true, "")`.
2. The `if (success)` branch executes.
3. `clearRequest(provider, sequenceNumber)` permanently removes the request from state — it cannot be retried or recovered.
4. `RevealedWithCallback` and `EntropyEventsV2.Revealed` are emitted, signaling to off-chain infrastructure that the callback was successfully delivered.
5. The requester's fee is consumed and the random number is never actually delivered to any contract logic.

The request enters a state that is indistinguishable from a successful fulfillment, yet no callback code ever ran. Off-chain relayers and monitoring systems that rely on the `Revealed` event will incorrectly conclude the request was fulfilled.

### Likelihood Explanation

- `requestWithCallback` is callable by any unprivileged address, including EOAs. An EOA can set `gasLimit10k != 0` and pay the required fee.
- `revealWithCallback` is a public function callable by anyone — no privileged role is required to trigger the vulnerable path.
- The scenario also applies to any consumer contract that selfdestructs after registering a request (e.g., a factory-deployed contract with a `selfdestruct` path).
- No special attacker capability is needed beyond submitting a standard transaction.

### Recommendation

Add an `extcodesize` check in the `gasLimit10k != 0` branch before the `excessivelySafeCall`, mirroring the guard already present in the `gasLimit10k == 0` branch:

```solidity
uint len;
assembly { len := extcodesize(req.requester) }
if (len == 0) revert EntropyErrors.InvalidCallbackAddress();
(success, ret) = req.requester.excessivelySafeCall(...);
```

Alternatively, revert or emit a distinct `CallbackSkipped` event (consistent with the `gasLimit10k == 0` branch behavior) rather than treating a no-code call as a successful delivery.

### Proof of Concept

1. EOA `attacker` calls `requestWithCallback` on the Entropy contract with `gasLimit10k = 1` (non-zero), paying the required fee. `req.requester = attacker` (EOA).
2. A relayer calls `revealWithCallback(provider, sequenceNumber, userRandom, providerRandom)`.
3. Execution enters the `gasLimit10k != 0` branch at line 574.
4. `req.requester.excessivelySafeCall(...)` issues a CALL to the EOA `attacker`. The EVM returns `(1, "")`.
5. `success = true`. The `if (success)` block at line 601 executes.
6. `clearRequest(provider, sequenceNumber)` is called at line 620 — the request is gone.
7. `RevealedWithCallback` and `EntropyEventsV2.Revealed` events are emitted with no error indication.
8. `attacker` (EOA) never executed any callback logic. The random number is lost. [4](#0-3)
