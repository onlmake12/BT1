### Title
`PythAggregatorV3` Exposes Stale Prices via `getPriceUnsafe()` with No Staleness Guard and Fabricated `answeredInRound` — (`File: target_chains/ethereum/sdk/solidity/PythAggregatorV3.sol`)

---

### Summary

`PythAggregatorV3` is Pyth's official Chainlink `AggregatorV3Interface`-compatible adapter, distributed as part of the production EVM SDK. Every price-reading function in the contract — `latestAnswer()`, `latestRoundData()`, `getRoundData()`, `decimals()`, and `latestTimestamp()` — calls `getPriceUnsafe()`, which explicitly returns prices from arbitrarily far in the past with no staleness enforcement. Compounding this, `latestRoundData()` fabricates `answeredInRound = roundId = uint80(price.publishTime)`, causing the canonical Chainlink staleness guard (`require(answeredInRound >= roundId)`) to always pass silently, even when the price is hours or days old.

---

### Finding Description

`PythAggregatorV3.sol` is the official Pyth SDK adapter for protocols migrating from Chainlink. It implements the full `AggregatorV3Interface` surface.

**Root cause 1 — universal use of `getPriceUnsafe()`:**

Every read function delegates to `getPriceUnsafe()`:

```solidity
// line 53-56
function latestAnswer() public view virtual returns (int256) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    return int256(price.price);
}

// line 99-119
function latestRoundData() external view returns (...) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    roundId = uint80(price.publishTime);
    return (roundId, int256(price.price), price.publishTime, price.publishTime, roundId);
}
```

`getPriceUnsafe()` is documented in `IPyth.sol` as: *"This function returns the most recent price update in this contract without any recency checks. This function is unsafe as the returned price update may be arbitrarily far in the past."* [1](#0-0) [2](#0-1) [3](#0-2) 

**Root cause 2 — `latestRoundData()` fabricates `answeredInRound = roundId`:**

The function sets both `roundId` and `answeredInRound` to `uint80(price.publishTime)`:

```solidity
roundId = uint80(price.publishTime);
return (roundId, int256(price.price), price.publishTime, price.publishTime, roundId);
//       ^^^^^^                                                               ^^^^^^
//  answeredInRound == roundId always
```

The canonical Chainlink staleness check used by virtually every Chainlink-integrated protocol is:

```solidity
require(answeredInRound >= roundID, "Chainlink Price Stale");
require(updateTime != 0, "Incomplete round");
require(price > 0, "Chainlink Malfunction");
```

All three checks pass unconditionally against `PythAggregatorV3.latestRoundData()` output:
- `answeredInRound >= roundId` → always true (they are identical)
- `updateTime != 0` → always true (`publishTime` is a Unix timestamp)
- `price > 0` → passes for any normal price

A protocol that performs these checks and believes it is protected against stale data is not protected at all. [4](#0-3) 

**Root cause 3 — `decimals()` unsafe narrowing cast:**

```solidity
function decimals() public view virtual returns (uint8) {
    PythStructs.Price memory price = pyth.getPriceUnsafe(priceId);
    return uint8(-1 * int8(price.expo));  // int32 → int8: silent truncation
}
```

`price.expo` is `int32`. Casting to `int8` silently truncates any exponent outside `[-128, 127]`. If the truncated value is positive, `-1 * int8(expo)` is negative, and `uint8(negative)` reverts in Solidity ≥0.8, making `decimals()` a DoS vector for any exponent that truncates to a positive `int8`. [5](#0-4) [6](#0-5)

### Impact Explanation

Any protocol that deploys `PythAggregatorV3` as a drop-in Chainlink replacement and applies standard Chainlink staleness guards will silently consume arbitrarily stale prices. Concrete consequences include:

- **Lending/borrowing protocols**: stale collateral prices allow under-collateralized borrows or block valid liquidations.
- **Perpetuals/derivatives**: stale mark prices cause incorrect funding rates and mis-priced positions.
- **AMMs with oracle-based pricing**: stale prices enable arbitrage extraction at the expense of LPs.

The `answeredInRound == roundId` fabrication is the critical amplifier: it defeats the one staleness check that Chainlink-migrating protocols are most likely to apply, giving a false sense of safety.

---

### Likelihood Explanation

`PythAggregatorV3` is the official Pyth-distributed migration path for Chainlink users, documented in the developer hub. Protocols migrating from Chainlink will copy their existing staleness-check boilerplate (`answeredInRound >= roundId`, `updateTime != 0`, `price > 0`) and believe they are protected. The Pyth price feed is a pull oracle — if no one submits an update, the on-chain price ages indefinitely. An attacker can deliberately withhold updates (or simply wait during low-activity periods) to exploit a stale price window. No privileged access is required.

---

### Recommendation

1. Replace `getPriceUnsafe()` with `getPriceNoOlderThan(priceId, maxAge)` in all read functions, where `maxAge` is a constructor-configurable parameter.
2. In `latestRoundData()`, do not set `answeredInRound = roundId`. Instead, encode staleness information so that downstream checks can detect it, or revert if the price is stale.
3. Fix the `decimals()` cast: use `int32` arithmetic throughout and add bounds checks before narrowing to `int8`/`uint8`.

```solidity
// Recommended pattern
uint256 public maxAge;

function latestRoundData() external view returns (
    uint80 roundId, int256 answer, uint256 startedAt,
    uint256 updatedAt, uint80 answeredInRound
) {
    PythStructs.Price memory price = pyth.getPriceNoOlderThan(priceId, maxAge);
    roundId = uint80(price.publishTime);
    return (roundId, int256(price.price), price.publishTime, price.publishTime, roundId);
}
```

### Proof of Concept

1. Deploy `PythAggregatorV3` pointing to a live Pyth contract and a valid `priceId`.
2. Submit a price update at time `T`.
3. Wait `N` hours without submitting any further updates (Pyth is a pull oracle; no one is forced to update).
4. Call `latestRoundData()`. Observe:
   - `answer` returns the price from time `T` (hours stale).
   - `answeredInRound == roundId` → the check `require(answeredInRound >= roundId)` passes.
   - `updatedAt == price.publishTime` (hours in the past) → `require(updateTime != 0)` passes.
   - `answer > 0` → `require(price > 0)` passes.
5. A lending protocol using this adapter with standard Chainlink guards accepts the stale price as fresh, enabling under-collateralized borrows or blocked liquidations. [7](#0-6) [8](#0-7)

        bytes32 id,
