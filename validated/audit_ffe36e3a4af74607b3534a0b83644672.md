Audit Report

## Title
Unconditional `.expect()` on `BlockExt.txs_sizes` in `FeeRateCollector::statistics` panics on legacy/IBD blocks — (`rpc/src/util/fee_rate.rs`)

## Summary

`FeeRateCollector::statistics` unconditionally unwraps `BlockExt.txs_sizes` at line 93 via `.expect()`. `txs_sizes` is legitimately `None` for any block stored under the old `BlockExt` molecule schema (pre-hardfork) or written via the IBD fast-sync path. Any unauthenticated caller invoking `get_fee_rate_statistics` with a target window that includes such a block triggers a panic, crashing the RPC task.

## Finding Description

In `rpc/src/util/fee_rate.rs` line 93, the closure unconditionally unwraps `txs_sizes`:

```rust
let txs_sizes = txs_sizes.expect("expect txs_size's length >= 1");
``` [1](#0-0) 

`BlockExt.txs_sizes` is `Option<Vec<u64>>` and is `None` in two concrete, production-reachable situations:

**1. Old `BlockExt` schema (pre-hardfork blocks):** The old molecule schema has no `txs_sizes` field. Its deserialization always produces `txs_sizes: None`: [2](#0-1) 

**2. IBD fast-sync path:** When the chain verifier runs with all checks disabled, it calls `insert_ok_ext` with explicit `None` for both `cycles` and `txs_sizes`: [3](#0-2) 

**3. `collect` does not filter `None` txs_sizes:** The `filter_map` in `collect` only skips blocks missing from the DB entirely. A block that exists but has `txs_sizes: None` passes through to the closure unchanged: [4](#0-3) 

**4. The RPC endpoint is public:** `get_fee_rate_statistics` is exposed to any caller with no authentication: [5](#0-4) 

## Impact Explanation

A Rust `panic!` in the RPC handler task crashes that task, causing the RPC call to return an error to the caller. This constitutes a **local RPC API crash** — matching the allowed CKB bounty impact: **Note (0–500 points): Any local RPC API crash.**

## Likelihood Explanation

- Any node that synced its chain history via IBD will have blocks with `txs_sizes: None` in the database.
- Any node within 101 blocks of the hardfork activation height has old-format blocks in the default 21-block (or up to 101-block) target window.
- The trigger requires only a single unauthenticated JSON-RPC call to `get_fee_rate_statistics` with no special privileges.

## Recommendation

Replace the `.expect()` with a graceful skip, consistent with how `cycles` is already handled:

```rust
let Some(txs_sizes) = txs_sizes else {
    return fee_rates; // skip blocks without size data
};
``` [6](#0-5) 

## Proof of Concept

The existing `DummyFeeRateProvider` test infrastructure in `rpc/src/tests/fee_rate.rs` can reproduce the panic directly:

```rust
let mut provider = DummyFeeRateProvider::new(101);
provider.append(1, BlockExt {
    received_at: 0,
    total_difficulty: 0u64.into(),
    total_uncles_count: 0,
    verified: Some(true),
    txs_fees: vec![Capacity::shannons(1000)],
    cycles: Some(vec![100]),
    txs_sizes: None,   // simulates pre-hardfork or IBD block
});
// panics: "expect txs_size's length >= 1"
let _ = FeeRateCollector::new(&provider).statistics(Some(3));
``` [7](#0-6)

use crate::util::{FeeRateCollector, FeeRateProvider};
use ckb_jsonrpc_types::FeeRateStatistics;
use ckb_types::core::{BlockExt, BlockNumber, Capacity};
use std::collections::HashMap;

struct DummyFeeRateProvider {
    tip_number: BlockNumber,
    block_exts: HashMap<BlockNumber, BlockExt>,
    max_target: u64,
}

impl DummyFeeRateProvider {
    pub fn new(max_target: u64) -> DummyFeeRateProvider {
        DummyFeeRateProvider {
            tip_number: 0,
            block_exts: HashMap::new(),
            max_target,
        }
    }

    pub fn append(&mut self, number: BlockNumber, ext: BlockExt) {
        if number > self.tip_number {
            self.tip_number = number;
        }
        self.block_exts.insert(number, ext);
    }

    pub fn set_max_target(&mut self, max_target: u64) {
        self.max_target = max_target
    }
}

impl FeeRateProvider for DummyFeeRateProvider {
    fn get_tip_number(&self) -> BlockNumber {
        self.tip_number
    }

    fn get_block_ext_by_number(&self, number: BlockNumber) -> Option<BlockExt> {
        self.block_exts.get(&number).cloned()
    }

    fn max_target(&self) -> u64 {
        self.max_target
    }
}
