### Title
Entropy Provider Can Set `feeInWei` to `type(uint128).max` With No Upper Bound, Causing Permanent DoS via Overflow in `getFeeV2` - (File: `target_chains/ethereum/contracts/contracts/entropy/Entropy.sol`)

---

### Summary

The `setProviderFee` and `setProviderFeeAsFeeManager` functions in `Entropy.sol` accept any `uint128` value for `newFeeInWei` with no upper-bound validation. A registered Entropy provider (or their fee manager) can set `feeInWei` to `type(uint128).max`. When `_state.pythFeeInWei > 0`, the addition in `getFeeV2` overflows and reverts under Solidity 0.8+ checked arithmetic, making it impossible for any user to query the required fee or submit a randomness request to that provider. If the affected provider is the default provider, the entire Entropy service is rendered unusable.

---

### Finding Description

`setProviderFee` is callable by any registered Entropy provider with no restriction on the value:

```solidity
function setProviderFee(uint128 newFeeInWei) external override {
    EntropyStructsV2.ProviderInfo storage provider = _state.providers[msg.sender];
    if (provider.sequenceNumber == 0) revert EntropyErrors.NoSuchProvider();
    uint128 oldFeeInWei = provider.feeInWei;
    provider.feeInWei = newFeeInWei;   // ← no upper bound check
    ...
}
``` [1](#0-0) 

The same pattern exists in `setProviderFeeAsFeeManager`: [2](#0-1) 

The fee is consumed in `getFeeV2`:

```solidity
function getFeeV2(address provider, uint32 gasLimit) public view override returns (uint128 feeAmount) {
    return getProviderFee(provider, gasLimit) + _state.pythFeeInWei;
}
``` [3](#0-2) 

When `provider.feeInWei = type(uint128).max` and `_state.pythFeeInWei > 0`, the addition `type(uint128).max + pythFeeInWei` overflows and reverts under Solidity 0.8+ checked arithmetic. Even when `pythFeeInWei == 0`, `getFeeV2` returns `type(uint128).max` (≈ 3.4 × 10³⁸ wei), which no user can practically supply as `msg.value`.

The fee check in `requestHelper` enforces:

```solidity
uint128 requiredFee = getFeeV2(provider, callbackGasLimit);
if (msg.value < requiredFee) revert EntropyErrors.InsufficientFee();
``` [4](#0-3) 

This means all `requestV2`, `requestWithCallback`, and `request` calls revert, and even `getFeeV2` itself reverts (overflow case), making the provider's service completely inaccessible.

The interface exposes `setProviderFee` as a first-class provider operation with no protocol-level cap: [5](#0-4) 

---

### Impact Explanation

- **Scenario A (default provider)**: If the default provider (Pyth's own Fortuna keeper) sets `feeInWei = type(uint128).max`, `getFeeV2()` (the zero-argument convenience overload used by most integrators) reverts on every call. No user can request randomness. The entire Entropy service is bricked until the provider lowers the fee — but if the provider key is lost or the provider is unresponsive, recovery is impossible without a contract upgrade.
- **Scenario B (non-default provider)**: Any registered provider can permanently DoS their own service, affecting all users who explicitly selected that provider.
- **Overflow amplification**: The `getProviderFee` internal function also performs arithmetic on `feeInWei` when `defaultGasLimit > 0`, causing additional overflow paths. [6](#0-5) 

---

### Likelihood Explanation

The Immunefi scope explicitly lists **Entropy user/provider** as a valid attacker role. Any address that has called `register(...)` is a provider and can immediately call `setProviderFee(type(uint128).max)` in a single transaction. No special privilege, leaked key, or governance majority is required. The attack is a single on-chain call with no preconditions beyond provider registration, which is permissionless.

---

### Recommendation

Add a protocol-enforced maximum fee constant and validate against it in both `setProviderFee` and `setProviderFeeAsFeeManager`:

```solidity
uint128 public constant MAX_PROVIDER_FEE_IN_WEI = 1e18; // 1 ETH, adjust as appropriate

function setProviderFee(uint128 newFeeInWei) external override {
    if (newFeeInWei > MAX_PROVIDER_FEE_IN_WEI) revert EntropyErrors.FeeTooHigh();
    ...
}
```

This mirrors the recommended fix in the original report: declare a `MAXIMUM_DELAY` constant and require `minimumDelay ≤ MAXIMUM_DELAY`.

### Proof of Concept

```solidity
// 1. Provider registers (permissionless)
entropy.register(1000, commitment, metadata, chainLength, uri);

// 2. Provider sets fee to type(uint128).max — no revert, no cap check
entropy.setProviderFee(type(uint128).max);

// 3. Any user attempting to query the fee now gets a revert (overflow)
//    assuming pythFeeInWei > 0:
entropy.getFeeV2(providerAddress, 0); // ← reverts: arithmetic overflow

// 4. Any user attempting to request randomness also reverts:
entropy.requestV2{value: 1 ether}(providerAddress, 0); // ← reverts: InsufficientFee or overflow
```

    function getProviderFee(
        address providerAddr,
        uint32 gasLimit
    ) internal view returns (uint128 feeAmount) {
        EntropyStructsV2.ProviderInfo memory provider = _state.providers[
            providerAddr
        ];

        // Providers charge a minimum of their configured feeInWei for every request.
        // Requests using more than the defaultGasLimit get a proportionally scaled fee.
        // This approach may be somewhat simplistic, but it allows us to continue using the
        // existing feeInWei parameter for the callback failure flow instead of defining new
        // configuration values.
        uint32 roundedGasLimit = uint32(roundTo10kGas(gasLimit)) * TEN_THOUSAND;
        if (
            provider.defaultGasLimit > 0 &&
            roundedGasLimit > provider.defaultGasLimit
        ) {
            // This calculation rounds down the fee, which means that users can get some gas in the callback for free.
            // However, the value of the free gas is < 1 wei, which is insignificant.
            uint128 additionalFee = ((roundedGasLimit -
                provider.defaultGasLimit) * provider.feeInWei) /
                provider.defaultGasLimit;
            return provider.feeInWei + additionalFee;
        } else {
            return provider.feeInWei;
        }
    }
```

    function setProviderFeeAsFeeManager(
        address provider,
        uint128 newFeeInWei
    ) external override {
        EntropyStructsV2.ProviderInfo storage providerInfo = _state.providers[
            provider
        ];

        if (providerInfo.sequenceNumber == 0) {
            revert EntropyErrors.NoSuchProvider();
        }

        if (providerInfo.feeManager != msg.sender) {
            revert EntropyErrors.Unauthorized();
        }

        uint128 oldFeeInWei = providerInfo.feeInWei;
        providerInfo.feeInWei = newFeeInWei;

        emit ProviderFeeUpdated(provider, oldFeeInWei, newFeeInWei);
        emit EntropyEventsV2.ProviderFeeUpdated(
            provider,
            oldFeeInWei,
            newFeeInWei,
            bytes("")
        );
    }
