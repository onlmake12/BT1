### Title
Echo.sol `executeCallback` Uses Dynamic Pyth Oracle Fee Against Request-Time Stored Fee, Enabling Permanent Fund Lock - (File: target_chains/ethereum/contracts/contracts/echo/Echo.sol)

### Summary
`Echo.sol` stores the provider's fee portion at request time as `req.fee = msg.value - _state.pythFeeInWei`, where `_state.pythFeeInWei` is the Echo protocol fee. At execution time, `executeCallback` deducts `pyth.getUpdateFee(updateData)` — the actual Pyth oracle fee, a separate and dynamic value — from `req.fee`. If the Pyth oracle fee increases between request and execution, the subtraction `(req.fee + msg.value) - pythFee` underflows and reverts, permanently locking the user's deposited funds with no cancellation path.

### Finding Description
In `requestPriceUpdatesWithCallback`, the stored provider fee is:

```solidity
req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);
```

`_state.pythFeeInWei` is the Echo protocol fee (accrued to the Echo admin). The provider's portion `req.fee` is expected to cover the Pyth oracle's `getUpdateFee` at execution time.

In `executeCallback`, the provider is credited:

```solidity
uint256 pythFee = pyth.getUpdateFee(updateData);
_state.providers[providerToCredit].accruedFeesInWei += SafeCast
    .toUint128((req.fee + msg.value) - pythFee);
```

`pyth.getUpdateFee(updateData)` is dynamic — it is derived from `singleUpdateFeeInWei` on the Pyth oracle contract, which is updatable via Pyth governance. These are two distinct fees:

| Fee | Where set | Who receives it |
|---|---|---|
| `_state.pythFeeInWei` | Echo contract | Echo admin |
| `pyth.getUpdateFee(updateData)` | Pyth oracle contract | Pyth oracle |

If `pyth.getUpdateFee(updateData)` increases after a request is submitted (e.g., via a Pyth governance `SetFee` action), then `pythFee > req.fee + msg.value_at_execution`, causing an arithmetic underflow revert in Solidity 0.8+. The Echo contract has no `cancelRequest` function, so the user's deposited funds are permanently locked. [1](#0-0) [2](#0-1) 

### Impact Explanation
User funds deposited via `requestPriceUpdatesWithCallback` are permanently locked in the Echo contract if the Pyth oracle fee increases after the request is submitted. There is no `cancelRequest` or refund mechanism. The provider cannot be compelled to send additional ETH to cover the increased oracle fee, and even if they do, they bear an unexpected cost not reflected in the original fee agreement. [3](#0-2) 

### Likelihood Explanation
The Pyth oracle's `singleUpdateFeeInWei` is governed on-chain and has been changed historically. Any legitimate governance fee increase — even a modest one — applied while requests are pending would trigger this condition for all in-flight requests whose `req.fee` was computed under the old fee schedule. The Echo contract's comment explicitly acknowledges the provider must cover the Pyth fee, but provides no mechanism to handle fee changes between request and execution. [4](#0-3) 

### Recommendation
At request time, snapshot the current Pyth oracle fee and store it alongside `req.fee`:

```solidity
uint256 snapshotPythOracleFee = IPyth(_state.pyth).getUpdateFee(/* estimated */);
req.pythOracleFee = snapshotPythOracleFee;
```

Then at execution time, use the snapshotted value rather than re-querying the live fee. Alternatively, add a `cancelRequest` function that allows users to reclaim their funds if the callback has not been executed within a timeout window, so that fee changes do not permanently lock funds.

### Proof of Concept
1. Pyth oracle fee is 100 wei. Provider sets `feePerGasInWei` such that `req.fee = 500 wei` (covers oracle fee).
2. User calls `requestPriceUpdatesWithCallback{value: pythFeeInWei + 500}(...)`. `req.fee = 500` is stored.
3. Pyth governance increases `singleUpdateFeeInWei` so that `pyth.getUpdateFee(updateData) = 600 wei`.
4. Provider calls `executeCallback{value: 0}(...)`.
5. Line 162 computes `(500 + 0) - 600` → underflow → revert.
6. No `cancelRequest` exists. User's `pythFeeInWei + 500` wei is permanently locked. [5](#0-4) [6](#0-5)
