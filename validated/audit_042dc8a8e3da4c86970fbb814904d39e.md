Audit Report

## Title
Unbounded Wait Loop in `read_from_json` Causes Permanent CLI Hang on Crafted Import File — (`util/instrument/src/import.rs`)

## Summary

`Import::read_from_json` unconditionally sets `largest_block_number` to the maximum block number seen on the channel, then enters an infinite polling loop waiting for `snapshot.get_block_hash(largest_block_number)` to return `Some`. A crafted JSONL file whose second block carries a bogus `parent_hash` causes that block to be silently parked in the orphan pool with its callback never invoked, so `COLUMN_INDEX` is never written for that height and the `ckb import` process never exits.

## Finding Description

**Unconditional `largest_block_number` update.** [1](#0-0) 
Every non-genesis block received from the channel unconditionally raises `largest_block_number`, regardless of whether the chain will ever accept that block.

**Partial first-block guard is bypassable.** [2](#0-1) 
The parent-existence check applies only to the first block in the file. If the first block is genesis (`is_genesis()` returns `true`), the guard is skipped entirely. Any subsequent block with a bogus `parent_hash` is never checked.

**Fire-and-forget dispatch.** [3](#0-2) 
`asynchronous_process_lonely_block` returns immediately with no success/failure signal back to the caller.

**Block inserted into DB, then handed to orphan broker.** [4](#0-3) 
`insert_block` performs a raw DB write with no parent-existence check. After that, the block is passed to `orphan_broker.process_lonely_block`.

**Orphan pool insertion with no callback.** [5](#0-4) 
When the parent hash is unknown (not stored, not pending-verify, not invalid), the block is silently inserted into the orphan pool. The `verify_callback` is never invoked.

**Unbounded wait loop.** [6](#0-5) 
The loop polls `get_block_hash(largest_block_number)` with a 1-second sleep and no timeout or failure exit path.

**Orphan TTL cleanup does not unblock the loop.** [7](#0-6) 
`clean_expired_orphans` removes the orphan from the pool but never calls its callback and never writes a block hash to `COLUMN_INDEX`, so the wait condition remains permanently unsatisfied after eviction.

## Impact Explanation

The `ckb import` CLI process hangs permanently, consuming a thread sleeping in a 1-second loop. The only recovery is `SIGKILL`. This matches the in-scope bounty impact: **Note (0–500 points) — Any local command line crash/hang**. Impact is bounded to the local import tool; it does not affect a running node, the network, or consensus.

## Likelihood Explanation

Triggering the hang requires only the ability to supply a crafted JSONL file to `ckb import`. With `--skip-all-verify`, all non-contextual checks are bypassed. Without it, a structurally well-formed block with a bogus `parent_hash` still passes `BlockVerifier` and `NonContextualBlockTxsVerifier` (neither checks parent existence), proceeds through `insert_block`, and enters the orphan pool. No cryptographic material, PoW solution, or network access is required. The file can be as small as two lines.

## Recommendation

Replace the unconditional `largest_block_number` update with tracking only blocks that are successfully committed to the chain, **or** add a bounded timeout to the wait loop, **or** plumb a shared atomic/channel that the `verify_callback` writes to on permanent failure (orphan eviction, `BLOCK_INVALID`) so the wait loop can break early. A minimal safe fix is to add a deadline:

```rust
let deadline = std::time::Instant::now() + std::time::Duration::from_secs(MAX_WAIT_SECS);
while self.shared.snapshot().get_block_hash(largest_block_number).is_none() {
    if std::time::Instant::now() > deadline {
        return Err("import timed out waiting for block commitment".into());
    }
    std::thread::sleep(std::time::Duration::from_secs(1));
}
```

## Proof of Concept

```jsonl
{"header":{"version":"0x0","compact_target":"0x207fffff","timestamp":"0x174...","number":"0x0","parent_hash":"0x0000...0000",...},...}
{"header":{"version":"0x0","compact_target":"0x207fffff","timestamp":"0x174...","number":"0xffffffffffffffff","parent_hash":"0xdeadbeef..."},...}
```

```bash
ckb import --skip-all-verify crafted.jsonl
# Process enters the while loop at import.rs:224 and never exits.
# `kill -9 <pid>` is the only way to terminate it.
```

The first block is genesis so the parent-existence guard at `import.rs:105–133` is skipped. The second block passes non-contextual verification, is written to the DB by `insert_block`, and is silently parked in the orphan pool. `largest_block_number` is set to `0xffffffffffffffff`; `get_block_hash` returns `None` on every iteration indefinitely.
