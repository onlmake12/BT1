### Title
`getPricesNoOlderThan()` and `getEmaPricesNoOlderThan()` validate staleness against a batch-level timestamp, not per-feed `publishTime`, allowing stale closed-market prices to bypass the freshness check — (`File: target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

The `Scheduler` contract's `getPricesNoOlderThan()` and `getEmaPricesNoOlderThan()` functions check staleness against `status.priceLastUpdatedAt`, which is the **maximum** `publishTime` across all feeds in the last batch update. For subscriptions that include closed-market feeds (e.g., equities), individual feed prices can be days old while the batch-level timestamp is recent. The staleness guard passes, but the returned prices are arbitrarily stale.

---

### Finding Description

When `updatePriceFeeds()` is called, `_validateShouldUpdatePrices()` computes `updateTimestamp` as the **maximum** `publishTime` across all feeds in the batch:

```solidity
// Scheduler.sol lines 366–371
uint256 updateTimestamp = 0;
for (uint8 i = 0; i < priceFeeds.length; i++) {
    if (priceFeeds[i].price.publishTime > updateTimestamp) {
        updateTimestamp = priceFeeds[i].price.publishTime;
    }
}
```

This maximum is stored as `status.priceLastUpdatedAt`:

```solidity
// Scheduler.sol line 340
status.priceLastUpdatedAt = latestPublishTime;
```

The code comment explicitly acknowledges that closed-market feeds will have old timestamps:

> *"Use the most recent timestamp, as some asset markets may be closed. Closed markets will have a publishTime from their last trading period."*

When a reader calls `getPricesNoOlderThan()`, the staleness check is:

```solidity
// Scheduler.sol lines 551–552
if (distance(block.timestamp, status.priceLastUpdatedAt) > age_seconds)
    revert PythErrors.StalePrice();

prices = this.getPricesUnsafe(subscriptionId, priceIds);
```

`getPricesUnsafe()` returns each feed's individually stored price, which for a closed-market feed may have a `publishTime` days in the past. The staleness check passes because `priceLastUpdatedAt` reflects the most recent open-market feed in the batch, not the feed being queried.

The same flaw exists in `getEmaPricesNoOlderThan()` at lines 594–597.

---

### Impact Explanation

A reader calling `getPricesNoOlderThan(subscriptionId, [equityFeedId], 60)` expects a price no older than 60 seconds. The check passes because `priceLastUpdatedAt` is recent (set by a crypto feed in the same subscription), but the returned equity price may be from the last trading session — potentially days old. DeFi protocols relying on this "safe" API for liquidations, collateral valuation, or minting calculations will silently consume stale prices, enabling value extraction analogous to the reported issue.

**Impact: Medium** — Incorrect price data returned from a function explicitly designed to guarantee freshness; downstream protocol losses are realistic.

---

### Likelihood Explanation

**Likelihood: Medium** — The Scheduler README and code explicitly support mixed-market subscriptions (crypto + equities). Any subscription containing at least one closed-market feed alongside an open-market feed triggers this condition. The Scheduler is a production push-oracle service with permissionless readers; any whitelisted reader contract that trusts `getPricesNoOlderThan()` is affected without any special attacker capability.

---

### Recommendation

Replace the batch-level staleness check with a per-feed check against each individual price's `publishTime`. In `getPricesNoOlderThan()` and `getEmaPricesNoOlderThan()`, after fetching the feeds via `_getPricesInternal()`, iterate over each returned price and verify `distance(block.timestamp, prices[i].publishTime) <= age_seconds` before returning. This mirrors how `AbstractPyth.getPriceNoOlderThan()` validates staleness per feed.

---

### Proof of Concept

1. Create a subscription with two feeds: `cryptoFeedId` (always open) and `equityFeedId` (closed market, last traded 2 days ago).
2. A keeper calls `updatePriceFeeds()`. `_validateShouldUpdatePrices()` sets `updateTimestamp = cryptoFeed.publishTime ≈ now`. `status.priceLastUpdatedAt = now`.
3. The equity feed is stored with `publishTime = 2 days ago`.
4. A reader calls `getPricesNoOlderThan(subscriptionId, [equityFeedId], 60)`.
5. The check `distance(block.timestamp, status.priceLastUpdatedAt) > 60` evaluates to `false` (passes), because `priceLastUpdatedAt ≈ now`.
6. `getPricesUnsafe()` returns the equity price with `publishTime = 2 days ago` — a 2-day-old price is returned from a function that promised a price no older than 60 seconds.
