### Title
Expired Alert Replay via P2P — Missing `notice_until` Validation in P2P Handler Allows Signature Reuse — (`util/network-alert/src/alert_relayer.rs`)

### Summary

The `AlertRelayer::received()` P2P handler accepts alerts without checking `notice_until`, while `Notifier::clear_expired_alerts()` removes expired alerts from the deduplication set (`received_alerts`) without adding them to `cancel_filter`. After expiry and cleanup, `has_received(alert_id)` returns `false`, allowing any peer who stored the original alert bytes to re-inject the same alert with the same valid signatures. The node re-accepts, re-notifies, and re-broadcasts the expired alert to all connected peers.

### Finding Description

**Root cause — two cooperating gaps:**

**Gap 1:** `AlertRelayer::received()` checks `has_received` and verifies signatures, but never validates `notice_until`:

```
// ignore alert
if self.notifier.lock().has_received(alert_id) { return; }
// verify
if let Err(err) = self.verifier.verify_signatures(&alert) { ... ban_peer ... return; }
// broadcast + add
self.notifier.lock().add(&alert);
```

The RPC path (`send_alert`) does check `notice_until < now_ms` and rejects expired alerts. The P2P path has no equivalent guard. [1](#0-0) [2](#0-1) 

**Gap 2:** `Notifier::clear_expired_alerts()` removes expired alerts from `received_alerts` and `noticed_alerts`, but does **not** add their IDs to `cancel_filter`. After cleanup, `has_received(id)` returns `false` for the evicted alert:

```rust
pub fn has_received(&self, id: u32) -> bool {
    self.received_alerts.contains_key(&id) || self.cancel_filter.contains(&id)
}
``` [3](#0-2) 

**Gap 3:** `Notifier::add()` itself performs no `notice_until` check before inserting into `received_alerts` and calling `notify_network_alert`: [4](#0-3) 

**Exploit flow:**

1. A legitimate alert (ID=`X`, `notice_until=T`, valid 2-of-4 signatures) is broadcast. Every peer that receives it stores the raw bytes.
2. Time advances past `T`; the alert expires.
3. Attacker (any peer that received the original alert) connects to a target node. `connected()` calls `clear_expired_alerts()`, which removes alert `X` from `received_alerts`. Now `has_received(X)` → `false`. [5](#0-4) 
4. Attacker immediately sends the stored (expired) alert bytes via P2P.
5. `received()`: `has_received(X)` is `false` → proceeds to signature verification.
6. `verify_signatures()` passes — the signatures are cryptographically valid; there is no nonce or sequence number in the signed payload. [6](#0-5) 
7. Alert is re-added to `received_alerts`, `notify_network_alert` fires, and the alert is re-broadcast to all connected peers.
8. The cycle repeats: any peer that receives the re-broadcast can replay it again after the next `clear_expired_alerts()` call.

The `cancel_filter` LRU cap of 128 entries introduces a second, independent replay window: if 128 distinct alert IDs are cancelled after alert `X` was cancelled, `X` is evicted from `cancel_filter`, and the same replay becomes possible for a cancelled (not just expired) alert — though filling the LRU requires 128 valid cancel-alerts, which requires the Nervos Foundation keys. [7](#0-6) 

### Impact Explanation

An unprivileged peer can force all reachable nodes to re-display and re-broadcast any previously-seen, now-expired alert. This causes:

- **Unauthorized state change**: Expired alerts are re-activated in `noticed_alerts` and re-delivered to all RPC subscribers and node operators.
- **Network-wide propagation**: The re-injected alert is re-broadcast to all connected peers, amplifying the effect across the entire P2P network.
- **Repeated exploitation**: The attack can be repeated every time `clear_expired_alerts()` runs (i.e., on every new peer connection), making suppression difficult.

The alert system is used for critical security notices. Re-displaying stale or superseded alerts can cause operator confusion, erode trust in the alert system, or mask a real current alert.

### Likelihood Explanation

The attacker preconditions are minimal:

- Must be a connectable P2P peer (no privileged keys required).
- Must have stored the raw bytes of any previously-broadcast alert (trivially available to any node that was online during the original broadcast, or from any peer that still holds them).

The trigger is automatic: connecting to a node causes `clear_expired_alerts()` to run, immediately opening the replay window. No timing precision is required.

### Recommendation

1. **Add `notice_until` check in `AlertRelayer::received()`** — mirror the RPC guard:
   ```rust
   let notice_until: u64 = alert.as_reader().raw().notice_until().into();
   if notice_until < ckb_systemtime::unix_time_as_millis() {
       // silently drop or ban peer
       return;
   }
   ```

2. **Add expired IDs to `cancel_filter` in `clear_expired_alerts()`** — prevent re-injection after cleanup:
   ```rust
   self.received_alerts.retain(|id, alert| {
       if notice_until <= now { self.cancel_filter.put(*id, ()); false }
       else { true }
   });
   ```

3. **Add `notice_until` check in `Notifier::add()`** as a defense-in-depth layer.

### Proof of Concept

```
1. Node A receives alert {id:1, notice_until: now+10s, signatures:[valid_sig1, valid_sig2]}
   → stored in received_alerts[1]

2. Wait 10+ seconds for alert to expire.

3. Attacker peer connects to Node A.
   → connected() calls clear_expired_alerts()
   → received_alerts[1] is removed
   → has_received(1) now returns false

4. Attacker immediately sends the original alert bytes via P2P (AlertRelayer protocol).
   → received(): has_received(1) == false → proceeds
   → verify_signatures() passes (same valid signatures, no nonce)
   → notifier.add() called: alert re-inserted into received_alerts, notify_network_alert fired
   → alert re-broadcast to all connected peers

5. All peers connected to Node A receive the expired alert and repeat steps 3-4.
```

    pub fn add(&mut self, alert: &Alert) {
        let alert_id = alert.raw().id().into();
        let alert_cancel = alert.raw().cancel().into();
        if self.has_received(alert_id) {
            return;
        }
        // checkout cancel_id
        if alert_cancel > 0 {
            self.cancel(alert_cancel);
        }
        // add to received alerts
        self.received_alerts.insert(alert_id, alert.clone());

        // check conditions, figure out do we need to notice this alert
        if !self.is_version_effective(alert) {
            debug!("Received a version ineffective alert {:?}", alert);
            return;
        }

        if self.noticed_alerts.contains(alert) {
            return;
        }
        self.notify_controller.notify_network_alert(alert.clone());
        self.noticed_alerts.push(alert.clone());
        // sort by priority
        self.noticed_alerts.sort_by_key(|a| {
            let priority: u32 = a.raw().priority().into();
            u32::MAX - priority
        });
    }

    pub fn clear_expired_alerts(&mut self, now: u64) {
        self.received_alerts.retain(|_id, alert| {
            let notice_until: u64 = alert.raw().notice_until().into();
            notice_until > now
        });
        self.noticed_alerts.retain(|a| {
            let notice_until: u64 = a.raw().notice_until().into();
            notice_until > now
        });
    }

    /// Whether id received

    pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
        trace!("Verifying alert {:?}", alert);
        let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
        let signatures: Vec<Signature> = alert
            .signatures()
            .into_iter()
            .filter_map(
                |sig_data| match Signature::from_slice(sig_data.as_reader().raw_data()) {
                    Ok(sig) => {
                        if sig.is_valid() {
                            Some(sig)
                        } else {
                            debug!("invalid signature: {:?}", sig);
                            None
                        }
                    }
                    Err(err) => {
                        debug!("signature error: {}", err);
                        None
                    }
                },
            )
            .collect();
        verify_m_of_n(
            &message,
            self.config.signatures_threshold,
            &signatures,
            &self.pubkeys,
        )
        .map_err(|err| err.kind())?;
        Ok(())
    }
