### Title
No Cancellation or Refund Mechanism for Unfulfilled Echo Requests Leads to Permanently Locked User Funds — (`target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

---

### Summary

`Echo.sol`'s `requestPriceUpdatesWithCallback` locks the user's ETH fee (`req.fee`) inside the contract at request time. There is no `cancelRequest`, timeout, or refund path. If the assigned provider never calls `executeCallback`, the user's funds are permanently locked with no on-chain recovery mechanism. The Pyth protocol fee portion (`_state.accruedFeesInWei`) is also credited immediately at request time, inflating the withdrawable balance even for requests that are never fulfilled.

---

### Finding Description

When a user calls `requestPriceUpdatesWithCallback`, the contract:

1. Immediately credits `_state.accruedFeesInWei += _state.pythFeeInWei` (Pyth protocol fee, line 99).
2. Stores the remaining provider fee in `req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei)` (line 84). [1](#0-0) 

The provider fee (`req.fee`) is only released when `executeCallback` is called, which credits it to the provider and clears the request: [2](#0-1) 

There is **no cancellation function, no timeout/expiry, and no refund path** anywhere in `Echo.sol`. The entire contract interface (`IEcho`) exposes no such mechanism: [3](#0-2) 

The developers themselves acknowledge the locking risk in a TODO comment inside `executeCallback`: [4](#0-3) 

The `EchoState.State` struct confirms there is no expiry or cancellation field: [5](#0-4) 

---

### Impact Explanation

**1. Permanently locked user funds.** The `req.fee` (provider portion) is held in the contract and can only exit via `executeCallback`. If the provider is offline, unresponsive, or deliberately withholds fulfillment, the user's ETH is locked forever with no on-chain recourse.

**2. Inflated `accruedFeesInWei` accounting.** The Pyth protocol fee is credited to `_state.accruedFeesInWei` at request time, before any service is rendered. The admin can call `withdrawFees` to extract these fees even for requests that are never fulfilled, meaning Pyth collects fees for services not delivered: [6](#0-5) 

This is the direct analog of the Midas report: a pending request leaves funds locked in a contract, `pendingTokenOutAmount()` (here `accruedFeesInWei`) remains inflated, and there is no mechanism to clear the stuck state.

---

### Likelihood Explanation

Any unprivileged user who submits a request is exposed. Realistic trigger scenarios:

- The assigned provider's keeper service goes offline (network outage, key rotation, deregistration).
- The provider is in the exclusivity window and refuses to fulfill (no penalty mechanism exists — the TODO at line 157 explicitly notes this gap).
- A malicious provider registers, collects fees from many users, then goes silent.
- The `publishTime` window passes and no valid price data is available, making fulfillment impossible.

The entry path requires no special privilege: `requestPriceUpdatesWithCallback` is callable by any EOA or contract. [7](#0-6) 

---

### Recommendation

1. Add a `cancelRequest(uint64 sequenceNumber)` function that allows the original requester to reclaim `req.fee` after a configurable timeout (e.g., after the exclusivity period plus a grace window).
2. Decrement `_state.accruedFeesInWei` on cancellation to keep accounting consistent, or defer the Pyth fee credit to fulfillment time.
3. Consider a penalty mechanism against providers who fail to fulfill within the exclusivity window, as noted in the existing TODO.

---

### Proof of Concept

```
1. Alice calls requestPriceUpdatesWithCallback{value: 1 ether}(provider, publishTime, priceIds, gasLimit)
   → req.fee = 1 ether - pythFeeInWei is stored in the request struct
   → _state.accruedFeesInWei += pythFeeInWei  (immediately credited)

2. Provider never calls executeCallback (offline / malicious).

3. Alice has no cancelRequest() to call.
   → req.fee is permanently locked in the Echo contract.
   → _state.accruedFeesInWei is inflated; admin can withdraw Pyth fees for the unrendered service.
   → getFirstActiveRequests() shows the stuck request indefinitely, and firstUnfulfilledSeq never advances past it.
``` [8](#0-7)

    function requestPriceUpdatesWithCallback(
        address provider,
        uint64 publishTime,
        bytes32[] calldata priceIds,
        uint32 callbackGasLimit
    ) external payable override returns (uint64 requestSequenceNumber) {
        require(
            _state.providers[provider].isRegistered,
            "Provider not registered"
        );

        // FIXME: this comment is wrong. (we're not using tx.gasprice)
        // NOTE: The 60-second future limit on publishTime prevents a DoS vector where
        //      attackers could submit many low-fee requests for far-future updates when gas prices
        //      are low, forcing executors to fulfill them later when gas prices might be much higher.
        //      Since tx.gasprice is used to calculate fees, allowing far-future requests would make
        //      the fee estimation unreliable.
        require(publishTime <= block.timestamp + 60, "Too far in future");
        if (priceIds.length > MAX_PRICE_IDS) {
            revert TooManyPriceIds(priceIds.length, MAX_PRICE_IDS);
        }
        requestSequenceNumber = _state.currentSequenceNumber++;

        uint96 requiredFee = getFee(provider, callbackGasLimit, priceIds);
        if (msg.value < requiredFee) revert InsufficientFee();

        Request storage req = allocRequest(requestSequenceNumber);
        req.sequenceNumber = requestSequenceNumber;
        req.publishTime = publishTime;
        req.callbackGasLimit = callbackGasLimit;
        req.requester = msg.sender;
        req.provider = provider;
        req.fee = SafeCast.toUint96(msg.value - _state.pythFeeInWei);

        // Create array with the right size
        req.priceIdPrefixes = new bytes8[](priceIds.length);

        // Copy only the first 8 bytes of each price ID to storage
        for (uint8 i = 0; i < priceIds.length; i++) {
            // Extract first 8 bytes of the price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }
            req.priceIdPrefixes[i] = prefix;
        }
        _state.accruedFeesInWei += _state.pythFeeInWei;

        emit PriceUpdateRequested(req, priceIds);
    }
