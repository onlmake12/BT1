### Title
Unprivileged `Initialize` Allows Attacker to Seize Authority and Permanently Block Publisher Registration — (`target_chains/solana/programs/pyth-price-store/src/processor/initialize.rs`)

---

### Summary

The `Initialize` instruction has no deployer-gating check. Any signer can call it before the legitimate deployer, supply an arbitrary `authority` pubkey in the instruction data, and permanently own the config PDA. Because the config PDA can only be created once, the legitimate deployer's subsequent `Initialize` call fails. The attacker then controls all `InitializePublisher` calls, permanently blocking legitimate publishers.

---

### Finding Description

`validate_payer` only checks that the caller is a signer and writable — there is no check that the caller is the program's upgrade authority or any other privileged account: [1](#0-0) 

`initialize()` accepts the `authority` field directly from attacker-controlled instruction data, with no constraint on who supplies it: [2](#0-1) 

The config account is a deterministic PDA seeded only by `CONFIG_SEED`, so any party can compute its address and race to create it: [3](#0-2) 

`accounts::config::create` enforces one-time initialization via a format-magic check, meaning once the attacker creates the account, the legitimate deployer's call is permanently rejected: [4](#0-3) 

`InitializePublisher` enforces that the signer matches `config.authority`, so only the attacker-controlled authority can ever register publishers: [5](#0-4) 

Each `publisher_config` PDA is also one-time-only (created via `system_instruction::create_account`), so a publisher that was never registered by the attacker can never be registered later: [6](#0-5) 

---

### Impact Explanation

- The attacker permanently owns the `authority` stored in the config PDA.
- No legitimate publisher can ever be registered (their `publisher_config` PDA will never be created).
- `SubmitPrices` requires a valid `publisher_config` PDA, so all legitimate publisher price submissions are permanently frozen.
- The program cannot be re-initialized; the only remediation is a program upgrade and redeployment.

---

### Likelihood Explanation

On Solana, program deployment and the first `Initialize` call are separate transactions. An attacker watching on-chain activity (e.g., via a validator or RPC subscription) can detect the program deployment and immediately submit an `Initialize` transaction with their own authority. There is no mempool ordering guarantee that favors the deployer. The attack requires no privileged access, no leaked keys, and no governance majority — only the ability to submit a Solana transaction.

---

### Recommendation

Gate `Initialize` on the program's BPF upgrade authority. Inside `initialize()`, load the program's `ProgramData` account (the upgradeable BPF loader stores the upgrade authority there) and assert that `payer.key` matches that upgrade authority. Alternatively, hardcode the expected authority pubkey as a program constant and verify it against `args.authority` before writing to the config account.

---

### Proof of Concept

```
1. Deploy the pyth-price-store program → program_id is now known on-chain.
2. Attacker computes config PDA:
     (config_pda, bump) = find_program_address(["CONFIG"], program_id)
3. Attacker submits Initialize with:
     key[0] = attacker_keypair  (signer, writable)
     key[1] = config_pda        (writable)
     key[2] = system_program
     data    = [0x00, bump, attacker_pubkey_bytes[32]]
4. Config PDA is created; config.authority = attacker_pubkey.
5. Legitimate deployer submits Initialize → system_instruction::create_account
   fails with AccountAlreadyInitialized.
6. Attacker calls InitializePublisher only for publishers they control.
7. All legitimate publishers are permanently locked out of SubmitPrices.
```

pub fn validate_authority<'a, 'b>(
    account: Option<&'b AccountInfo<'a>>,
    config: &AccountInfo<'a>,
) -> Result<&'b AccountInfo<'a>, ProgramError> {
    let authority = account.ok_or(ProgramError::NotEnoughAccountKeys)?;
    ensure!(ProgramError::MissingRequiredSignature, authority.is_signer);
    ensure!(ProgramError::InvalidArgument, authority.is_writable);
    let config_data = config.data.borrow();
    let config = accounts::config::read(*config_data)?;
    ensure!(
        ProgramError::MissingRequiredSignature,
        authority.key.to_bytes() == config.authority
    );
    Ok(authority)

pub fn create(data: &mut [u8], authority: [u8; 32]) -> Result<&mut Config, ReadAccountError> {
    if data.len() < size_of::<Config>() {
        return Err(ReadAccountError::DataTooShort);
    }
    let data: &mut Config = from_bytes_mut(&mut data[..size_of::<Config>()]);
    if data.format != 0 {
        return Err(ReadAccountError::AlreadyInitialized);
    }
    data.format = FORMAT;
    data.authority = authority;
    Ok(data)
