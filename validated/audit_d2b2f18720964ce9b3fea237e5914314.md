### Title
`update_price_feeds_if_necessary()` Missing `#[payable]` Causes Permanent DoS — (`target_chains/stylus/contracts/pyth-receiver/src/lib.rs`)

---

### Summary

The Stylus (Rust/WASM) implementation of `update_price_feeds_if_necessary` is not marked `#[payable]`, yet it internally calls `update_price_feeds` which is `#[payable]` and enforces `msg_value() >= total_fee`. Because the outer function cannot receive ETH, `msg_value()` is always `0` when the inner fee check executes, causing every call to revert with `InsufficientFee` whenever the protocol fee is non-zero.

---

### Finding Description

In `target_chains/stylus/contracts/pyth-receiver/src/lib.rs`, `update_price_feeds` is correctly marked `#[payable]` and enforces a fee: [1](#0-0) 

```rust
#[payable]
pub fn update_price_feeds(
    &mut self,
    update_data: Vec<Vec<u8>>,
) -> Result<(), PythReceiverError> {
    let total_fee = self.get_update_fee(update_data.clone())?;
    let value = self.vm().msg_value();
    if value < total_fee {
        return Err(PythReceiverError::InsufficientFee);
    }
    ...
}
```

However, `update_price_feeds_if_necessary` is **not** marked `#[payable]` and directly calls `self.update_price_feeds(...)`: [2](#0-1) 

```rust
pub fn update_price_feeds_if_necessary(   // ← no #[payable]
    &mut self,
    update_data: Vec<Vec<u8>>,
    price_ids: Vec<[u8; 32]>,
    publish_times: Vec<u64>,
) -> Result<(), PythReceiverError> {
    ...
    for i in 0..price_ids.len() {
        if self.latest_price_info_publish_time(price_ids[i]) < publish_times[i] {
            self.update_price_feeds(update_data.clone())?;  // ← fee check fires with value=0
            return Ok(());
        }
    }
    return Err(PythReceiverError::NoFreshUpdate);
}
```

In Stylus, the `#[payable]` attribute is the mechanism that allows a function to accept ETH. Without it, the Stylus framework rejects any call carrying non-zero value before execution. Consequently:

- A caller **cannot** send ETH to `update_price_feeds_if_necessary` (non-payable → rejected).
- `self.vm().msg_value()` inside the subsequently called `update_price_feeds` therefore always returns `0`.
- The check `if value < total_fee` fails whenever `singleUpdateFeeInWei > 0`, reverting with `InsufficientFee`.

The Solidity counterpart (`Pyth.sol`) does not have this bug because `updatePriceFeedsIfNecessary` is declared `external payable` and the internal call to `updatePriceFeeds` shares the same `msg.value` context: [3](#0-2) 

The Stylus implementation diverges from this correct pattern.

---

### Impact Explanation

`update_price_feeds_if_necessary` is permanently broken (DoS) on any Stylus deployment where `singleUpdateFeeInWei > 0`. Any protocol or user relying on this gas-saving wrapper to conditionally update prices will have every transaction revert. The Solidity equivalent is a widely-used entry point (documented in the SDK and API reference), so the Stylus version being silently non-functional is a high-severity correctness and availability issue. [4](#0-3) 

---

### Likelihood Explanation

Any unprivileged user or protocol integrating the Stylus Pyth receiver and calling `update_price_feeds_if_necessary` will trigger the revert. No special role or precondition is required beyond the protocol fee being non-zero, which is the standard production configuration.

---

### Recommendation

Add `#[payable]` to `update_price_feeds_if_necessary` so that callers can forward the required fee, mirroring the Solidity implementation:

```rust
// Before
pub fn update_price_feeds_if_necessary(

// After
#[payable]
pub fn update_price_feeds_if_necessary(
``` [5](#0-4) 

---

### Proof of Concept

1. Deploy the Stylus Pyth receiver with `singleUpdateFeeInWei = 1` (any non-zero fee).
2. Call `update_price_feeds_if_necessary(updateData, priceIds, publishTimes)` with `msg.value = 0` (the only option, since the function is non-payable).
3. The function reaches `self.update_price_feeds(update_data.clone())`.
4. Inside `update_price_feeds`, `self.vm().msg_value()` returns `0`.
5. `0 < total_fee` → returns `Err(PythReceiverError::InsufficientFee)`.
6. The outer call propagates the error and reverts.

Calling with non-zero ETH is impossible because the Stylus framework rejects ETH on non-payable functions before execution begins.
