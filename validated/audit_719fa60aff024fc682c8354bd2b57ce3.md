### Title
Use-After-Delete State Inconsistency in `Echo.executeCallback` Causes Consumer Callback Failure for Overflow Requests — (`File: target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary

`Echo.executeCallback` obtains a `storage` pointer to the active request, then calls `clearRequest` before invoking the consumer's `_echoCallback`. For requests stored in the overflow mapping (`_state.requestsOverflow`), `clearRequest` issues a full `delete`, zeroing every field of the struct the pointer still references. The subsequent callback is therefore dispatched to `address(0)` with `0` gas, silently failing. The provider is paid, the request is permanently gone, and the consumer never receives their price-feed callback.

### Finding Description

`executeCallback` follows this sequence:

1. `Request storage req = findActiveRequest(sequenceNumber)` — obtains a live storage pointer, which may point to `_state.requestsOverflow[key]` when the primary slot is occupied by a different request.
2. Validates price IDs and calls `pyth.parsePriceFeedUpdates`.
3. Credits provider fees: `_state.providers[providerToCredit].accruedFeesInWei += ...`
4. **`clearRequest(sequenceNumber)`** — for overflow entries this executes `delete _state.requestsOverflow[key]`, zeroing the entire struct in storage.
5. Calls `IEchoConsumer(req.requester)._echoCallback{gas: req.callbackGasLimit}(...)` — but `req.requester` is now `address(0)` and `req.callbackGasLimit` is now `0`. [1](#0-0) 

`clearRequest` for overflow entries: [2](#0-1) 

`findRequest` returning the overflow pointer: [3](#0-2) 

A request lands in the overflow mapping whenever `allocRequest` finds the primary slot already occupied by a different active request: [4](#0-3) 

With 256 primary slots and sequence numbers incrementing from 1, a birthday-paradox collision is expected after roughly 16–20 requests (~50 % probability), making this routinely reachable in production.

### Impact Explanation

- The consumer paid `msg.value ≥ getFee(provider, callbackGasLimit, priceIds)` when calling `requestPriceUpdatesWithCallback`.
- After `clearRequest` zeroes the overflow entry, the `try` call dispatches to `address(0)` with `0` gas, which fails immediately.
- The `catch` block emits `PriceUpdateCallbackFailed`, but the request has already been cleared — the consumer cannot retry.
- The provider's `accruedFeesInWei` was already incremented before the callback attempt; the provider keeps the fee.
- **Net result**: consumer funds are permanently lost; the service (price-feed callback) is never delivered. [5](#0-4) 

### Likelihood Explanation

Any unprivileged user can trigger this by submitting enough requests to cause a slot collision. With 256 slots and uniformly distributed `keccak256(sequenceNumber)` values, the expected number of requests before the first collision is ~20. On a busy deployment this happens continuously. No special privilege is required — only the ability to call `requestPriceUpdatesWithCallback`.

### Recommendation

Save the fields needed after `clearRequest` into local (memory) variables **before** calling `clearRequest`, then use those locals for the callback:

```solidity
address callbackTarget = req.requester;
uint32  callbackGas    = req.callbackGasLimit;

_state.providers[providerToCredit].accruedFeesInWei += SafeCast.toUint128((req.fee + msg.value) - pythFee);
clearRequest(sequenceNumber);
// ... firstUnfulfilledSeq update ...

try IEchoConsumer(callbackTarget)._echoCallback{gas: callbackGas}(sequenceNumber, priceFeeds) {
    ...
} catch { ... }
```

Additionally, add `ReentrancyGuardUpgradeable` (already available via the OpenZeppelin import chain used by `EchoUpgradeable`) and apply `nonReentrant` to `executeCallback`, `withdrawFees`, and `withdrawAsFeeManager` to prevent cross-function reentrancy during the consumer callback. [6](#0-5) 

### Proof of Concept

1. Deploy `EchoUpgradeable` with a registered provider.
2. Consumer submits **two** requests whose sequence numbers share the same `shortKey` (i.e., `uint8(keccak256(abi.encodePacked(seqA))[0] & MASK) == uint8(keccak256(abi.encodePacked(seqB))[0] & MASK)`). The first request is moved to `_state.requestsOverflow` when the second is allocated.
3. Provider calls `executeCallback` for the **first** (overflow) request with valid `updateData`.
4. `clearRequest` deletes the overflow entry; `req.requester` and `req.callbackGasLimit` are now `0`.
5. The `try` call to `address(0)` with `0` gas fails; `PriceUpdateCallbackFailed` is emitted.
6. Provider's `accruedFeesInWei` was already incremented; consumer's funds are gone with no recourse. [7](#0-6)

    // TODO: does this need to be payable? Any cost paid to Pyth could be taken out of the provider's accrued fees.
    function executeCallback(
        address providerToCredit,
        uint64 sequenceNumber,
        bytes[] calldata updateData,
        bytes32[] calldata priceIds
    ) external payable override {
        Request storage req = findActiveRequest(sequenceNumber);

        // Check provider exclusivity using configurable period
        if (
            block.timestamp < req.publishTime + _state.exclusivityPeriodSeconds
        ) {
            require(
                providerToCredit == req.provider,
                "Only assigned provider during exclusivity period"
            );
        }

        // Verify priceIds match
        require(
            priceIds.length == req.priceIdPrefixes.length,
            "Price IDs length mismatch"
        );
        for (uint8 i = 0; i < req.priceIdPrefixes.length; i++) {
            // Extract first 8 bytes of the provided price ID
            bytes32 priceId = priceIds[i];
            bytes8 prefix;
            assembly {
                prefix := priceId
            }

            // Compare with stored prefix
            if (prefix != req.priceIdPrefixes[i]) {
                // Now we can directly use the bytes8 prefix in the error
                revert InvalidPriceIds(priceIds[i], req.priceIdPrefixes[i]);
            }
        }

        // TODO: should this use parsePriceFeedUpdatesUnique? also, do we need to add 1 to maxPublishTime?
        IPyth pyth = IPyth(_state.pyth);
        uint256 pythFee = pyth.getUpdateFee(updateData);
        PythStructs.PriceFeed[] memory priceFeeds = pyth.parsePriceFeedUpdates{
            value: pythFee
        }(
            updateData,
            priceIds,
            SafeCast.toUint64(req.publishTime),
            SafeCast.toUint64(req.publishTime)
        );

        // TODO: if this effect occurs here, we need to guarantee that executeCallback can never revert.
        // If executeCallback can revert, then funds can be permanently locked in the contract.
        // TODO: there also needs to be some penalty mechanism in case the expected provider doesn't execute the callback.
        // This should take funds from the expected provider and give to providerToCredit. The penalty should probably scale
        // with time in order to ensure that the callback eventually gets executed.
        // (There may be exploits with ^ though if the consumer contract is malicious ?)
        _state.providers[providerToCredit].accruedFeesInWei += SafeCast
            .toUint128((req.fee + msg.value) - pythFee);

        clearRequest(sequenceNumber);

        // TODO: I'm pretty sure this is going to use a lot of gas because it's doing a storage lookup for each sequence number.
        // a better solution would be a doubly-linked list of active requests.
        // After successful callback, update firstUnfulfilledSeq if needed
        while (
            _state.firstUnfulfilledSeq < _state.currentSequenceNumber &&
            !isActive(findRequest(_state.firstUnfulfilledSeq))
        ) {
            _state.firstUnfulfilledSeq++;
        }

        try
            IEchoConsumer(req.requester)._echoCallback{
                gas: req.callbackGasLimit
            }(sequenceNumber, priceFeeds)
        {
            // Callback succeeded
            emitPriceUpdate(sequenceNumber, priceIds, priceFeeds);
        } catch Error(string memory reason) {
            // Explicit revert/require
            emit PriceUpdateCallbackFailed(
                sequenceNumber,
                providerToCredit,
                priceIds,
                req.requester,
                reason
            );
        } catch {
            // Out of gas or other low-level errors
            emit PriceUpdateCallbackFailed(
                sequenceNumber,
                providerToCredit,
                priceIds,
                req.requester,
                "low-level error (possibly out of gas)"
            );
        }
    }

    ) internal view returns (Request storage req) {

        req = _state.requests[shortKey];
        if (req.sequenceNumber == sequenceNumber) {
            return req;
        } else {
            req = _state.requestsOverflow[key];

    function clearRequest(uint64 sequenceNumber) internal {
        (bytes32 key, uint8 shortKey) = requestKey(sequenceNumber);

        Request storage req = _state.requests[shortKey];
        if (req.sequenceNumber == sequenceNumber) {
            req.sequenceNumber = 0;
        } else {
            delete _state.requestsOverflow[key];
        }
    }
