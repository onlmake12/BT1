### Title
Permanent Subscription ETH Permanently Locked With No Admin Recovery Path - (File: `target_chains/ethereum/contracts/contracts/pulse/Scheduler.sol`)

---

### Summary

ETH deposited into `isPermanent` subscriptions in the Pyth Pulse `Scheduler` contract is permanently and irrecoverably locked. The `withdrawFunds()` function unconditionally reverts for permanent subscriptions, and `SchedulerGovernance.sol` contains no emergency withdrawal or rescue function. If the contract is deprecated, upgraded, or if subscribed price feeds become unavailable, all ETH held in permanent subscriptions is lost forever.

---

### Finding Description

The `Scheduler` contract allows any user to create a subscription with `isPermanent = true`, depositing up to `MAX_DEPOSIT_LIMIT` (100 ETH) per subscription.

`withdrawFunds()` explicitly and unconditionally blocks all withdrawals from permanent subscriptions:

```solidity
// Prevent withdrawals from permanent subscriptions
if (params.isPermanent) {
    revert SchedulerErrors.CannotUpdatePermanentSubscription();
}
``` [1](#0-0) 

Similarly, `updateSubscription()` blocks all parameter changes (including deactivation) for permanent subscriptions:

```solidity
// Updates to permanent subscriptions are not allowed
if (currentParams.isPermanent) {
    revert SchedulerErrors.CannotUpdatePermanentSubscription();
}
``` [2](#0-1) 

The `SchedulerGovernance.sol` contract — the only admin-privileged surface — provides only fee and minimum-balance setters. It contains **no emergency withdrawal, no rescue function, and no admin override** for permanent subscription balances: [3](#0-2) 

The `isPermanent` flag is a one-way door: once set, it cannot be unset:

```solidity
// Make it permanent
params.isPermanent = true;
scheduler.updateSubscription(subscriptionId, params);
// Verify we can't make it non-permanent again → CannotUpdatePermanentSubscription
``` [4](#0-3) 

The `MAX_DEPOSIT_LIMIT` is 100 ETH per permanent subscription: [5](#0-4) 

---

### Impact Explanation

Any ETH deposited into a permanent subscription is irrecoverable by anyone — including the subscription manager and the admin — through any currently deployed contract function. Concrete loss scenarios:

1. **Contract deprecation/upgrade**: The `SchedulerUpgradeable` pattern implies future upgrades are expected. If the contract is deprecated before a permanent subscription's balance is fully consumed by keepers, the remaining ETH is permanently lost.
2. **Price feed delisting**: If a Pyth price feed subscribed to by a permanent subscription is delisted or becomes unavailable, keepers stop calling `updatePriceFeeds`, the balance is never consumed, and the ETH is permanently locked.
3. **Keeper abandonment**: If no keeper services the subscription (e.g., keeper infrastructure failure), the ETH balance accumulates and is unrecoverable.

**Impact: 3/5** — Direct ETH loss for subscription managers; up to 100 ETH per subscription permanently locked.

---

### Likelihood Explanation

- Any unprivileged user can create a permanent subscription with a single transaction.
- Contract upgrades and price feed lifecycle changes are routine protocol operations.
- The `isPermanent` flag is explicitly designed to be irreversible, making this a structural property of the contract rather than an edge case.

**Likelihood: 3/5** — The trigger conditions (contract upgrade, feed delisting, keeper failure) are realistic and expected over the protocol's lifetime.

---

### Recommendation

Add an admin-only emergency withdrawal function in `SchedulerGovernance.sol` that can recover ETH from permanent subscriptions in exceptional circumstances (e.g., contract deprecation):

```solidity
function emergencyWithdrawPermanentSubscription(
    uint256 subscriptionId,
    address payable recipient
) external {
    _authorizeAdminAction();
    SchedulerStructs.SubscriptionParams storage params =
        _state.subscriptionParams[subscriptionId];
    require(params.isPermanent, "Not a permanent subscription");
    uint256 amount = _state.subscriptionStatuses[subscriptionId].balanceInWei;
    _state.subscriptionStatuses[subscriptionId].balanceInWei = 0;
    (bool sent, ) = recipient.call{value: amount}("");
    require(sent, "Transfer failed");
}
```

Alternatively, implement a contract-level deprecation flag that, when set by the admin, allows permanent subscription managers to reclaim their remaining balances.

### Proof of Concept

1. User calls `createSubscription` with `isPermanent = true` and deposits 50 ETH. [6](#0-5) 

2. The subscription is now permanent and active. The user's 50 ETH is held in `status.balanceInWei`. [7](#0-6) 

3. User attempts `withdrawFunds(subscriptionId, 1 ether)` → reverts with `CannotUpdatePermanentSubscription`. [1](#0-0) 

4. User attempts `updateSubscription(subscriptionId, params_with_isActive_false)` → reverts with `CannotUpdatePermanentSubscription`. [2](#0-1) 

5. Admin inspects `SchedulerGovernance.sol` — no function exists to recover the ETH. [8](#0-7) 

6. Contract is deprecated. The 50 ETH remains permanently locked in the contract with no recovery path.

    function createSubscription(
        SchedulerStructs.SubscriptionParams memory subscriptionParams
    ) external payable override returns (uint256 subscriptionId) {
        _validateSubscriptionParams(subscriptionParams);

        // Calculate minimum balance required for this subscription
        uint256 minimumBalance = this.getMinimumBalance(
            uint8(subscriptionParams.priceIds.length)
        );

        // Ensure enough funds were provided
        if (msg.value < minimumBalance) {
            revert SchedulerErrors.InsufficientBalance();
        }

        // Check deposit limit for permanent subscriptions
        if (subscriptionParams.isPermanent && msg.value > MAX_DEPOSIT_LIMIT) {
            revert SchedulerErrors.MaxDepositLimitExceeded();
        }

        // Set subscription to active
        subscriptionParams.isActive = true;

        subscriptionId = _state.subscriptionNumber++;

        // Store the subscription parameters
        _state.subscriptionParams[subscriptionId] = subscriptionParams;

        // Initialize subscription status
        SchedulerStructs.SubscriptionStatus storage status = _state
            .subscriptionStatuses[subscriptionId];
        status.priceLastUpdatedAt = 0;
        status.balanceInWei = msg.value;
        status.totalUpdates = 0;
        status.totalSpent = 0;

        // Map subscription ID to manager
        _state.subscriptionManager[subscriptionId] = msg.sender;

        _addToActiveSubscriptions(subscriptionId);

        emit SubscriptionCreated(subscriptionId, msg.sender);
        return subscriptionId;

import "./SchedulerState.sol";
import "@pythnetwork/pulse-sdk-solidity/SchedulerErrors.sol";

/**
 * @dev `SchedulerGovernance` defines governance capabilities for the Pulse contract.
 */
abstract contract SchedulerGovernance is SchedulerState {
    event NewAdminProposed(address oldAdmin, address newAdmin);
    event NewAdminAccepted(address oldAdmin, address newAdmin);
    event SingleUpdateKeeperFeeSet(uint oldFee, uint newFee);
    event MinimumBalancePerFeedSet(uint oldBalance, uint newBalance);

    /**
     * @dev Returns the address of the proposed admin.
     */
    function proposedAdmin() public view virtual returns (address) {
        return _state.proposedAdmin;
    }

    /**
     * @dev Returns the address of the current admin.
     */
    function getAdmin() external view returns (address) {
        return _state.admin;
    }

    /**
     * @dev Proposes a new admin for the contract. Replaces the proposed admin if there is one.
     * Can only be called by either admin or owner.
     */
    function proposeAdmin(address newAdmin) public virtual {
        require(newAdmin != address(0), "newAdmin is zero address");

        _authorizeAdminAction();

        _state.proposedAdmin = newAdmin;
        emit NewAdminProposed(_state.admin, newAdmin);
    }

    /**
     * @dev The proposed admin accepts the admin transfer.
     */
    function acceptAdmin() external {
        if (msg.sender != _state.proposedAdmin)
            revert SchedulerErrors.Unauthorized();

        address oldAdmin = _state.admin;
        _state.admin = msg.sender;

        _state.proposedAdmin = address(0);
        emit NewAdminAccepted(oldAdmin, msg.sender);
    }

    /**
     * @dev Authorization check for admin actions
     * Must be implemented by the inheriting contract.
     */
    function _authorizeAdminAction() internal virtual;

    /**
     * @dev Set the keeper fee for single updates in Wei.
     * Calls {_authorizeAdminAction}.
     * Emits a {SingleUpdateKeeperFeeSet} event.
     */
    function setSingleUpdateKeeperFeeInWei(uint128 newFee) external {
        _authorizeAdminAction();

        uint oldFee = _state.singleUpdateKeeperFeeInWei;
        _state.singleUpdateKeeperFeeInWei = newFee;

        emit SingleUpdateKeeperFeeSet(oldFee, newFee);
    }

    /**
     * @dev Set the minimum balance required per feed in a subscription.
     * Calls {_authorizeAdminAction}.
     * Emits a {MinimumBalancePerFeedSet} event.
     */
    function setMinimumBalancePerFeed(uint128 newMinimumBalance) external {
        _authorizeAdminAction();

        uint oldBalance = _state.minimumBalancePerFeed;
        _state.minimumBalancePerFeed = newMinimumBalance;

        emit MinimumBalancePerFeedSet(oldBalance, newMinimumBalance);
    }
}
