### Title
Unbounded Per-Peer Request Rate in Light Client `GetBlocksProof` Handler Enables Storage I/O Exhaustion — (`util/light-client-protocol-server/src/components/get_blocks_proof.rs`)

### Summary

The `GetBlocksProofProcess::execute` handler accepts up to 1000 block hashes per request and performs up to 3 RocksDB reads per hash plus an MMR proof generation, with **no per-peer rate limiting of any kind**. A single unprivileged light-client peer can sustain back-to-back maximum-cost requests, monopolizing RocksDB I/O and degrading storage throughput for all other peers on the same node.

---

### Finding Description

`GET_BLOCKS_PROOF_LIMIT` is set to 1000. [1](#0-0) 

The guard in `execute` rejects only requests **exceeding** 1000, so exactly 1000 hashes is accepted: [2](#0-1) 

For every hash that is on the main chain, the handler issues three synchronous RocksDB reads — `get_block_header`, `get_block_uncles`, and `get_block_extension` — in a tight loop: [3](#0-2) 

After the loop, `reply_proof` builds an MMR proof over all 1000 positions: [4](#0-3) 

The `LightClientProtocol` struct carries **no rate-limiter field**, and `try_process` dispatches directly to the handler with no rate check: [5](#0-4) 

This is in direct contrast to the `Relayer` protocol, which maintains a `RateLimiter<(PeerIndex, u32)>` and checks it before processing every message type: [6](#0-5) [7](#0-6) 

A grep over the entire `util/light-client-protocol-server/` tree returns zero matches for `rate_limit`, `RateLimiter`, or `TooManyRequests`. The omission is complete.


---

### Impact Explanation

Per request at maximum load: **3 000 RocksDB point reads** (1 000 × header + uncles + extension) plus one MMR proof traversal over a chain of potentially millions of blocks. A single peer sending these requests in a tight loop saturates the shared RocksDB read path, increasing latency for block sync, transaction relay, and all other storage-dependent operations on the same node.

---

### Likelihood Explanation

The attacker needs only a valid P2P connection to a node that has the light-client protocol enabled. No key, no hashpower, no Sybil attack is required. The 1 000-hash payload is trivially constructed from any 1 000 known main-chain block hashes (e.g., genesis + 999 sequential blocks). The attack is repeatable indefinitely because no ban or backpressure is triggered by well-formed requests.

---

### Recommendation

Add a per-peer, per-message-type rate limiter to `LightClientProtocol` mirroring the pattern already used in `Relayer`:

1. Add a `rate_limiter: RateLimiter<(PeerIndex, u32)>` field to `LightClientProtocol`.
2. In `try_process`, check the limiter before dispatching any message, returning `StatusCode::TooManyRequests` on excess.
3. Consider also reducing `GET_BLOCKS_PROOF_LIMIT` (currently 1 000) or splitting large requests across multiple round-trips.

---

### Proof of Concept

```
# 1. Connect to a light-client-enabled CKB full node.
# 2. Collect 1000 consecutive main-chain block hashes (h_1 … h_1000).
# 3. In a loop:
#      send GetBlocksProof { last_hash: tip_hash, block_hashes: [h_1..h_1000] }
#      do not wait for the reply before sending the next request
# 4. Observe: RocksDB read IOPS spike to maximum; concurrent sync peers
#    experience increased header/block fetch latency.
```

The loop is valid because each request is well-formed (≤ 1 000 unique main-chain hashes, no duplicates) and triggers no ban. [8](#0-7)

    pub(crate) async fn execute(self) -> Status {
        if self.message.block_hashes().is_empty() {
            return StatusCode::MalformedProtocolMessage.with_context("no block");
        }

        if self.message.block_hashes().len() > constant::GET_BLOCKS_PROOF_LIMIT {
            return StatusCode::MalformedProtocolMessage.with_context("too many blocks");
        }

        let snapshot = self.protocol.shared.snapshot();

        let last_block_hash = self.message.last_hash().to_entity();
        if !snapshot.is_main_chain(&last_block_hash) {
            return self
                .protocol
                .reply_tip_state::<packed::SendBlocksProof>(self.peer, self.nc)
                .await;
        }
        let last_block = snapshot
            .get_block(&last_block_hash)
            .expect("block should be in store");

        let block_hashes: Vec<_> = self
            .message
            .block_hashes()
            .to_entity()
            .into_iter()
            .collect();

        let mut uniq = HashSet::new();
        if !block_hashes
            .iter()
            .chain([last_block_hash].iter())
            .all(|hash| uniq.insert(hash))
        {
            return StatusCode::MalformedProtocolMessage
                .with_context("duplicate block hash exists");
        }

        let (found, missing): (Vec<_>, Vec<_>) = block_hashes
            .into_iter()
            .partition(|block_hash| snapshot.is_main_chain(block_hash));

        let mut positions = Vec::with_capacity(found.len());
        let mut block_headers = Vec::with_capacity(found.len());
        let mut uncles_hash = Vec::with_capacity(found.len());
        let mut extensions = Vec::with_capacity(found.len());

        for block_hash in found {
            let header = snapshot
                .get_block_header(&block_hash)
                .expect("header should be in store");
            positions.push(leaf_index_to_pos(header.number()));
            block_headers.push(header.data());

            let uncles = snapshot
                .get_block_uncles(&block_hash)
                .expect("block uncles must be stored");
            let extension = snapshot.get_block_extension(&block_hash);

            uncles_hash.push(uncles.data().calc_uncles_hash());
            extensions.push(packed::BytesOpt::new_builder().set(extension).build());
        }

        let proved_items = (
            block_headers.into(),
            uncles_hash.into(),
            packed::BytesOptVec::new_builder().set(extensions).build(),
        );
        let missing_items = missing.into();

        self.protocol
            .reply_proof::<packed::SendBlocksProofV1>(
                self.peer,
                self.nc,
                &last_block,
                positions,
                proved_items,
                missing_items,
            )
            .await
    }
