### Title
Expired Network Alert Replay by Unprivileged Peer — (`util/network-alert/src/notifier.rs`)

---

### Summary

`clear_expired_alerts` removes expired alerts from `received_alerts` without recording their IDs in `cancel_filter`. After expiry, `has_received(alert_id)` returns `false`, so any connected peer can replay a previously valid (but now expired) signed alert, causing the node to re-accept and re-broadcast it as if it were new.

---

### Finding Description

The `Notifier` struct tracks seen alerts in two structures:

- `received_alerts: HashMap<u32, Alert>` — active/live alerts keyed by ID
- `cancel_filter: LruCache<u32, ()>` — IDs of explicitly cancelled alerts

The deduplication gate in `add()` is:

```rust
// notifier.rs:96-98
if self.has_received(alert_id) {
    return;
}
```

where `has_received` is:

```rust
// notifier.rs:147-149
pub fn has_received(&self, id: u32) -> bool {
    self.received_alerts.contains_key(&id) || self.cancel_filter.contains(&id)
}
```

When `clear_expired_alerts` runs (triggered on every new peer connection in `alert_relayer.rs:87`), it purges expired entries from `received_alerts` and `noticed_alerts`:

```rust
// notifier.rs:135-143
pub fn clear_expired_alerts(&mut self, now: u64) {
    self.received_alerts.retain(|_id, alert| {
        let notice_until: u64 = alert.raw().notice_until().into();
        notice_until > now
    });
    self.noticed_alerts.retain(|a| { ... });
}
```

Critically, it does **not** insert the expired IDs into `cancel_filter`. After this call, `has_received(expired_id)` returns `false`. The alert's ID is now completely untracked.

A peer who stored the original alert bytes can then re-send them. In `alert_relayer.rs:received()`:

1. `has_received(alert_id)` → `false` (ID was cleared) — gate passes
2. `verifier.verify_signatures(&alert)` → `Ok(())` (signatures are still cryptographically valid)
3. `notifier.lock().add(&alert)` — alert is re-inserted into `received_alerts` and `notify_controller.notify_network_alert` fires again [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

A secondary, lower-likelihood path exists via `cancel_filter`'s bounded `LruCache` of size 128: if more than 128 distinct alert IDs are cancelled, the oldest entries are evicted, and a peer can replay those cancelled alerts as well. [5](#0-4) [6](#0-5) 

---

### Impact Explanation

An unprivileged peer can force any CKB node to:

1. **Re-broadcast** an expired alert across the P2P network, amplifying the replay to all connected peers.
2. **Re-fire** `notify_network_alert`, causing the node's subscribers (RPC clients, wallets, monitoring tools) to receive a stale alert as if it were a new one.
3. **Re-execute the `cancel` side-effect**: if the replayed alert carries `cancel > 0`, `notifier.cancel(cancel_id)` is called again, removing the target alert from `received_alerts` and `noticed_alerts`. If the Nervos Foundation re-issued an alert with the same numeric ID after the cancellation alert expired, the replay silently suppresses the active security warning for all users of that node. [7](#0-6) [8](#0-7) 

---

### Likelihood Explanation

- Any peer connected to the CKB P2P network can send raw alert bytes; no privilege is required.
- The trigger condition (alert expiry + `clear_expired_alerts`) fires automatically on every new peer connection.
- Storing a valid alert for later replay requires only that the peer was online when the original alert was broadcast — a trivial precondition.
- The `cancel_filter` LRU eviction path requires 128+ cancellations and is therefore low-probability in practice, but the expiry-based path has no such threshold. [9](#0-8) [10](#0-9) 

---

### Recommendation

**Short term**: In `clear_expired_alerts`, insert each removed alert's ID into `cancel_filter` before dropping it from `received_alerts`:

```rust
pub fn clear_expired_alerts(&mut self, now: u64) {
    self.received_alerts.retain(|id, alert| {
        let notice_until: u64 = alert.raw().notice_until().into();
        if notice_until <= now {
            self.cancel_filter.put(*id, ()); // mark as permanently seen
            false
        } else {
            true
        }
    });
    ...
}
```

**Long term**: Replace the bounded `LruCache` for `cancel_filter` with an unbounded set (or a set bounded by a much larger, operationally justified limit), or persist seen alert IDs across restarts so that replay is impossible even after a node reboot. [11](#0-10) 

---

### Proof of Concept

```
1. Node A receives Alert(id=5, notice_until=T, message="critical bug").
   → received_alerts = {5: Alert5}; cancel_filter = {}

2. Time advances past T. Peer B connects.
   → alert_relayer::connected() calls clear_expired_alerts(now > T)
   → received_alerts = {}; cancel_filter = {} (ID 5 is untracked)

3. Peer B (unprivileged) sends the original Alert5 bytes.
   → alert_relayer::received():
       has_received(5) == false  ✓ (gate passes)
       verify_signatures(Alert5) == Ok(())  ✓ (signatures still valid)
       notifier.add(Alert5)
         → received_alerts = {5: Alert5}
         → notify_network_alert fires  ← stale alert re-delivered to all subscribers
         → Alert5 is re-broadcast to all connected peers
``` [12](#0-11) [13](#0-12)

    pub fn add(&mut self, alert: &Alert) {
        let alert_id = alert.raw().id().into();
        let alert_cancel = alert.raw().cancel().into();
        if self.has_received(alert_id) {
            return;
        }
        // checkout cancel_id
        if alert_cancel > 0 {
            self.cancel(alert_cancel);
        }
        // add to received alerts
        self.received_alerts.insert(alert_id, alert.clone());

        // check conditions, figure out do we need to notice this alert

            debug!("Received a version ineffective alert {:?}", alert);
            return;

            return;

        self.noticed_alerts.push(alert.clone());
        // sort by priority
        self.noticed_alerts.sort_by_key(|a| {
            let priority: u32 = a.raw().priority().into();
            u32::MAX - priority
        });

    pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
        trace!("Verifying alert {:?}", alert);
        let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
        let signatures: Vec<Signature> = alert
            .signatures()
            .into_iter()
            .filter_map(
                |sig_data| match Signature::from_slice(sig_data.as_reader().raw_data()) {
                    Ok(sig) => {
                        if sig.is_valid() {
                            Some(sig)
                        } else {
                            debug!("invalid signature: {:?}", sig);
                            None
                        }
                    }
                    Err(err) => {
                        debug!("signature error: {}", err);
                        None
                    }
                },
            )
            .collect();
        verify_m_of_n(
            &message,
            self.config.signatures_threshold,
            &signatures,
            &self.pubkeys,
        )
        .map_err(|err| err.kind())?;
        Ok(())
    }
