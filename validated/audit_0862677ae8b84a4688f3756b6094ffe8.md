### Title
Missing `setPythFee` Function in Echo Contract Permanently Locks Protocol Fee Rate - (`target_chains/ethereum/contracts/contracts/echo/Echo.sol`)

### Summary

The `Echo` contract stores a `pythFeeInWei` protocol fee in its state and accumulates it on every request, but provides no admin function to update this value post-deployment. Unlike the analogous `Entropy` contract — which has `EntropyGovernance.setPythFee()` — the `Echo` contract permanently fixes the protocol fee at the value set during initialization, preventing the admin from ever adjusting it.

### Finding Description

`EchoState.sol` defines `pythFeeInWei` as a field in the `State` struct: [1](#0-0) 

`Echo._initialize()` sets this value once at deployment: [2](#0-1) 

Every call to `requestPriceUpdatesWithCallback()` accumulates `_state.pythFeeInWei` into `_state.accruedFeesInWei`: [3](#0-2) 

The contract exposes `getPythFeeInWei()` and `withdrawFees()`, but neither `Echo.sol`, `IEcho.sol`, nor `EchoUpgradeable.sol` contains any function to update `_state.pythFeeInWei` after initialization: [4](#0-3) 

By contrast, the `Entropy` contract's governance layer provides `setPythFee()` for exactly this purpose: [5](#0-4) 

### Impact Explanation

Once `EchoUpgradeable` is deployed and initialized, `_state.pythFeeInWei` is immutable for the lifetime of the proxy. The admin cannot:
- Increase the fee to reflect higher operational costs or governance decisions.
- Decrease the fee to attract users or respond to competitive pressure.
- Set the fee to zero to temporarily waive protocol charges.

The protocol will continue to accumulate fees at the stale initialization rate indefinitely, with no on-chain mechanism to correct it short of a full contract upgrade. This directly mirrors the Superposition M-12 finding where `fee_protocol` had no setter.

### Likelihood Explanation

This is a certainty, not a probability: the function simply does not exist. Any deployment of `EchoUpgradeable` is immediately and permanently affected. The impact materializes whenever the admin needs to adjust the protocol fee — a routine governance operation that is explicitly supported in the sibling `Entropy` contract.

### Recommendation

Add an admin-gated setter to `Echo.sol` and expose it in `IEcho.sol`, mirroring the pattern in `EntropyGovernance.sol`:

```solidity
// In Echo.sol
function setPythFee(uint96 newPythFeeInWei) external {
    require(msg.sender == _state.admin, "Only admin");
    uint96 oldFee = _state.pythFeeInWei;
    _state.pythFeeInWei = newPythFeeInWei;
    emit PythFeeUpdated(oldFee, newPythFeeInWei);
}
```

Declare the corresponding event in `EchoEvents.sol` and add the function signature to `IEcho.sol`.

### Proof of Concept

1. Deploy `EchoUpgradeable` with `pythFeeInWei = 1000`.
2. Call `getPythFeeInWei()` → returns `1000`.
3. Attempt to call any function to update `pythFeeInWei` → no such function exists in the ABI.
4. Call `requestPriceUpdatesWithCallback(...)` with sufficient value → `_state.accruedFeesInWei` increments by `1000` permanently.
5. Call `getAccruedPythFees()` → confirms fees accumulate at the fixed rate with no way to change it. [6](#0-5) [7](#0-6) [8](#0-7)
