### Title
Missing `notice_until` Expiry Enforcement in P2P Alert Relay Path Allows Replay of Expired Alerts - (File: `util/network-alert/src/alert_relayer.rs`)

### Summary
The CKB network alert system enforces the `notice_until` expiry deadline only in the RPC `send_alert` handler, but not in the P2P `received()` relay handler. Any unprivileged P2P peer can replay a previously valid, now-expired alert (bearing legitimate Nervos foundation signatures) to any node. The node will accept it, add it to its `noticed_alerts` list (displaying it to users), and re-broadcast it to all connected peers — indefinitely re-injecting stale alerts across the network.

---

### Finding Description

The `RawAlert` molecule schema includes a `notice_until: Uint64` field (a millisecond-precision expiry timestamp), and `calc_alert_hash()` hashes the entire `RawAlert` including this field. The signature therefore commits to the expiry value. However, the expiry is only validated in one of the two alert ingestion paths.

**RPC path — expiry IS enforced:**

```rust
// rpc/src/module/alert.rs:104-110
fn send_alert(&self, alert: Alert) -> Result<()> {
    let alert: packed::Alert = alert.into();
    let now_ms = ckb_systemtime::unix_time_as_millis();
    let notice_until: u64 = alert.raw().notice_until().into();
    if notice_until < now_ms {
        return Err(RPCError::invalid_params(...));
    }
    ...
}
```

**P2P relay path — expiry is NOT enforced:**

```rust
// util/network-alert/src/alert_relayer.rs:98-179
async fn received(&mut self, nc: ..., peer_index: PeerIndex, data: Bytes) {
    let alert: packed::Alert = match packed::AlertReader::from_slice(&data) { ... };
    let alert_id = alert.as_reader().raw().id().into();
    if self.notifier.lock().has_received(alert_id) { return; }
    // verify — only checks cryptographic signature, NOT notice_until
    if let Err(err) = self.verifier.verify_signatures(&alert) { ... return; }
    // broadcasts and stores without any expiry check
    nc.quick_filter_broadcast(...);
    self.notifier.lock().add(&alert);
}
```

`Notifier::add()` also performs no expiry check:

```rust
// util/network-alert/src/notifier.rs:93-122
pub fn add(&mut self, alert: &Alert) {
    let alert_id = alert.raw().id().into();
    if self.has_received(alert_id) { return; }
    self.received_alerts.insert(alert_id, alert.clone());
    if !self.is_version_effective(alert) { return; }
    self.notify_controller.notify_network_alert(alert.clone());
    self.noticed_alerts.push(alert.clone());
    // no notice_until check anywhere
}
```

The only expiry cleanup is `clear_expired_alerts()`, called only on peer `connected()` events — not on `received()`:

```rust
// util/network-alert/src/alert_relayer.rs:87-94
async fn connected(&mut self, nc: ..., peer_index: PeerIndex, _version: &str) {
    self.clear_expired_alerts();
    for alert in self.notifier.lock().received_alerts() { ... }
}
```

This means: after `clear_expired_alerts()` runs on connection, the attacker immediately sends an expired alert in the same session, and it is accepted with no further expiry check.

---

### Impact Explanation

- **Stale alert display to users**: Expired alerts are added to `noticed_alerts` and surfaced via `get_blockchain_info().alerts` (RPC) and `notify_network_alert` (notification subsystem). Users see alerts that the Nervos foundation intended to expire.
- **Network-wide re-propagation**: The relay path broadcasts the expired alert to all connected peers, which in turn relay it further. A single attacker can re-flood the entire P2P network with an old alert.
- **Social engineering surface**: An attacker can selectively replay an old "upgrade your node" or "critical bug" alert to targeted nodes to manipulate operator behavior.
- **Persistent re-injection**: Because `clear_expired_alerts()` is only called on `connected()`, an attacker can reconnect repeatedly to clear the dedup state and re-inject the same expired alert.

---

### Likelihood Explanation

- Any unprivileged P2P peer can connect to a CKB node and send an alert message on the Alert protocol.
- The attacker only needs a previously broadcast, now-expired alert with valid Nervos foundation signatures — these are public on-chain/network history (e.g., alert `20230001` from 2023 is hardcoded in the test suite).
- No key compromise, no majority hashpower, no privileged access required.
- The `KNOWN_LIST_SIZE` LRU cache for deduplication is only 64 entries and is per-peer, so it does not prevent cross-peer replay.

---

### Recommendation

Add a `notice_until` expiry check in the P2P `received()` handler, mirroring the RPC path:

```rust
// util/network-alert/src/alert_relayer.rs — inside received()
let now_ms = ckb_systemtime::unix_time_as_millis();
let notice_until: u64 = alert.as_reader().raw().notice_until().into();
if notice_until < now_ms {
    debug!("Received an expired alert {} from peer {}", alert_id, peer_index);
    return; // silently drop; do not ban (peer may have cached it legitimately)
}
```

Also add the same check inside `Notifier::add()` as a defense-in-depth guard:

```rust
// util/network-alert/src/notifier.rs — inside add()
let now_ms = ckb_systemtime::unix_time_as_millis();
let notice_until: u64 = alert.raw().notice_until().into();
if notice_until < now_ms {
    return;
}
```

### Proof of Concept

1. Obtain the real expired alert `20230001` (its raw bytes and signatures are hardcoded in `util/network-alert/src/tests/generate_alert_signature.rs`; `notice_until = 1681574400000` = April 2023).
2. Connect to any CKB mainnet/testnet node as a P2P peer on the Alert protocol.
3. Wait for the `connected()` callback to fire on the target node (which calls `clear_expired_alerts()`, removing any cached copy of alert `20230001`).
4. Immediately send the serialized `packed::Alert` bytes for alert `20230001` over the P2P connection.
5. Observe: the target node's `received()` handler verifies the signatures (valid), skips the expiry check, calls `notifier.add()`, and broadcasts the expired alert to all its peers. The alert appears in `get_blockchain_info().alerts` on the target node.

```rust
            Err(err) => {

                nc.ban_peer(

                    BAD_MESSAGE_BAN_TIME,
                    String::from("send us a malformed message"),
                );
                return;
            }
        };
        let alert_id = alert.as_reader().raw().id().into();
        trace!("ReceiveD alert {} from peer {}", alert_id, peer_index);

            debug!(

                peer_index, err

            nc.ban_peer(

                BAD_MESSAGE_BAN_TIME,
                String::from("send us an alert with invalid signatures"),
            );
            return;

        self.mark_as_known(peer_index, alert_id);
        // broadcast message

            .connected_peers()
            .into_iter()
            .filter(|peer| self.mark_as_known(*peer, alert_id))
            .collect();

            TargetSession::Multi(Box::new(selected_peers.into_iter())),
            data,
        ) {
            debug!("alert broadcast error: {:?}", err);
        }
        // add to received alerts
        self.notifier.lock().add(&alert);
    }
```

```rust
    /// Add an alert
    pub fn add(&mut self, alert: &Alert) {
        let alert_id = alert.raw().id().into();
        let alert_cancel = alert.raw().cancel().into();
        if self.has_received(alert_id) {
            return;
        }
        // checkout cancel_id
        if alert_cancel > 0 {
            self.cancel(alert_cancel);
        }
        // add to received alerts
        self.received_alerts.insert(alert_id, alert.clone());

        // check conditions, figure out do we need to notice this alert
        if !self.is_version_effective(alert) {
            debug!("Received a version ineffective alert {:?}", alert);
            return;
        }

        if self.noticed_alerts.contains(alert) {
            return;
        }
        self.notify_controller.notify_network_alert(alert.clone());
        self.noticed_alerts.push(alert.clone());
        // sort by priority
        self.noticed_alerts.sort_by_key(|a| {
            let priority: u32 = a.raw().priority().into();
            u32::MAX - priority
        });
    }
```

```rust
    fn send_alert(&self, alert: Alert) -> Result<()> {
        let alert: packed::Alert = alert.into();
        let now_ms = ckb_systemtime::unix_time_as_millis();
        let notice_until: u64 = alert.raw().notice_until().into();
        if notice_until < now_ms {
            return Err(RPCError::invalid_params(format!(
                "Expected `params[0].notice_until` in the future (> {now_ms}), got {notice_until}",
            )));
        }

        let result = self.verifier.verify_signatures(&alert);

        match result {
            Ok(()) => {
                // set self node notifier
                self.notifier.lock().add(&alert);

                self.network_controller.broadcast_with_handle(
                    SupportProtocols::Alert.protocol_id(),
                    alert.as_bytes(),
                    &self.handle,
                );
                Ok(())
            }
            Err(e) => Err(RPCError::custom_with_error(
                RPCError::AlertFailedToVerifySignatures,
                e,
            )),
        }
    }
```
