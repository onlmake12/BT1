### Title
Network Alert Signature Replay Across Different CKB Networks (No Chain-ID in Signed Message) - (File: `util/network-alert/src/verifier.rs`)

### Summary
The CKB network alert system signs and verifies alerts using a hash of `RawAlert` bytes that contains no chain/network identifier. Because the same hard-coded public keys are used across all networks running the same CKB binary (mainnet, testnet, devnet), a valid alert signature produced for one network is cryptographically valid on every other network. An unprivileged P2P peer or RPC caller who captures a legitimately-signed alert from any network can replay it verbatim on mainnet, causing every receiving node to accept, store, and re-broadcast the alert.

### Finding Description

The `RawAlert` molecule schema contains only application-level fields:

```
notice_until, id, cancel, priority, message, min_version, max_version
``` [1](#0-0) 

`calc_alert_hash()` is a plain `blake2b_256` over the raw serialized bytes of this struct — no chain ID, genesis hash, or network tag is mixed in:

```rust
pub fn calc_alert_hash(&self) -> packed::Byte32 {
    self.calc_hash()   // = blake2b_256(self.as_slice())
}
``` [2](#0-1) 

`verify_signatures` derives the message to verify from exactly this hash:

```rust
let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
``` [3](#0-2) 

It then checks the recovered public keys against the hard-coded key set:

```rust
verify_m_of_n(&message, self.config.signatures_threshold, &signatures, &self.pubkeys)
``` [4](#0-3) 

The hard-coded key set is identical for every node running the same CKB binary:

```toml
signatures_threshold = 2
public_keys = [
  "0x03933a9b116c5017561742c37ae69acb0dca3329a52c479c85df5bb387c8ac8715",
  ...
]
``` [5](#0-4) 

Because the signed digest is identical for the same `RawAlert` content regardless of which network it was originally intended for, a signature that is valid on testnet is equally valid on mainnet.

### Impact Explanation

An attacker who observes any legitimately-signed alert on testnet (alerts are broadcast publicly over P2P) can:

1. Capture the raw `Alert` bytes (including the two valid signatures).
2. Submit them to a mainnet node via the `send_alert` RPC endpoint or directly over the P2P alert protocol.
3. The `verify_signatures` check passes because the recovered public keys match the hard-coded mainnet key set.
4. The alert is stored locally and re-broadcast to all connected peers. [6](#0-5) 

Every mainnet node then displays the replayed alert message to operators and propagates it further. A stale or misleading alert (e.g., "upgrade immediately", a false version range, or a cancelled-but-replayed message) reaches the entire mainnet P2P network without any new authorization from the key holders.

### Likelihood Explanation

- Alerts are broadcast publicly over the P2P network; capturing one requires only running a CKB node.
- The `send_alert` RPC is reachable by any local or remote caller with RPC access.
- The P2P `AlertRelayer` accepts alert messages from any connected peer.
- The only partial mitigation is the `notice_until` expiry check in the RPC handler, but this only applies to the RPC path and only if the alert has already expired; a non-expired testnet alert is fully replayable on mainnet. [7](#0-6) 

### Recommendation

Include a network/chain identifier in the signed data. The simplest fix is to mix the genesis block hash (which is unique per network) into the alert hash computation:

```rust
pub fn calc_alert_hash(&self, genesis_hash: &Byte32) -> packed::Byte32 {
    let mut blake2b = new_blake2b();
    blake2b.update(genesis_hash.as_slice());
    blake2b.update(self.as_slice());
    let mut result = [0u8; 32];
    blake2b.finalize(&mut result);
    result.into()
}
```

This ensures a signature produced for testnet is cryptographically invalid on mainnet, directly mirroring the recommendation in the reference report to include the wallet address in the signed data.

### Proof of Concept

1. Run a CKB testnet node and a CKB mainnet node (same binary version, same hard-coded alert keys).
2. On testnet, have the key holders sign and broadcast a valid alert (e.g., `id=99`, `message="test"`, `notice_until=<future>`).
3. Capture the raw `Alert` bytes from the testnet P2P stream.
4. Call `send_alert` on the mainnet RPC with the identical alert bytes.
5. Observe that `verify_signatures` returns `Ok(())` and the alert is stored and re-broadcast to all mainnet peers — without any new signature from the key holders. [8](#0-7) [9](#0-8)

    pub fn verify_signatures(&self, alert: &packed::Alert) -> Result<(), AnyError> {
        trace!("Verifying alert {:?}", alert);
        let message = Message::from_slice(alert.calc_alert_hash().as_slice())?;
        let signatures: Vec<Signature> = alert
            .signatures()
            .into_iter()
            .filter_map(
                |sig_data| match Signature::from_slice(sig_data.as_reader().raw_data()) {
                    Ok(sig) => {
                        if sig.is_valid() {
                            Some(sig)
                        } else {
                            debug!("invalid signature: {:?}", sig);
                            None
                        }
                    }
                    Err(err) => {
                        debug!("signature error: {}", err);
                        None
                    }
                },
            )
            .collect();
        verify_m_of_n(
            &message,
            self.config.signatures_threshold,
            &signatures,
            &self.pubkeys,
        )
        .map_err(|err| err.kind())?;
        Ok(())
    }

    async fn received(
        &mut self,
        nc: Arc<dyn CKBProtocolContext + Sync>,
        peer_index: PeerIndex,
        data: Bytes,
    ) {
        let alert: packed::Alert = match packed::AlertReader::from_slice(&data) {
            Ok(alert) => {
                if alert.raw().message().is_utf8()
                    && alert
                        .raw()
                        .min_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                    && alert
                        .raw()
                        .max_version()
                        .to_opt()
                        .map(|x| x.is_utf8())
                        .unwrap_or(true)
                {
                    alert.to_entity()
                } else {
                    info!(
                        "A malformed message fromP peer {} : not utf-8 string",
                        peer_index
                    );
                    nc.ban_peer(
                        peer_index,
                        BAD_MESSAGE_BAN_TIME,
                        String::from("send us a malformed message: not utf-8 string"),
                    );
                    return;
                }
            }
            Err(err) => {
                info!("A malformed message from peer {}: {:?}", peer_index, err);
                nc.ban_peer(
                    peer_index,
                    BAD_MESSAGE_BAN_TIME,
                    String::from("send us a malformed message"),
                );
                return;
            }
        };
        let alert_id = alert.as_reader().raw().id().into();
        trace!("ReceiveD alert {} from peer {}", alert_id, peer_index);
        // ignore alert
        if self.notifier.lock().has_received(alert_id) {
            return;
        }
        // verify
        if let Err(err) = self.verifier.verify_signatures(&alert) {
            debug!(
                "An alert from peer {} with invalid signatures, error {:?}",
                peer_index, err
            );
            nc.ban_peer(
                peer_index,
                BAD_MESSAGE_BAN_TIME,
                String::from("send us an alert with invalid signatures"),
            );
            return;
        }
        // mark sender as known
        self.mark_as_known(peer_index, alert_id);
        // broadcast message
        let selected_peers: Vec<PeerIndex> = nc
            .connected_peers()
            .into_iter()
            .filter(|peer| self.mark_as_known(*peer, alert_id))
            .collect();
        if let Err(err) = nc.quick_filter_broadcast(
            TargetSession::Multi(Box::new(selected_peers.into_iter())),
            data,
        ) {
            debug!("alert broadcast error: {:?}", err);
        }
        // add to received alerts
        self.notifier.lock().add(&alert);
    }
